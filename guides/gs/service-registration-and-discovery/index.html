服务注册和发现

本指南将引导您完成建立和使用Netflix Eureka服务注册表的过程。

你会建立什么

您将设置Netflix Eureka服务注册表 ，然后构建一个客户端，该客户端会同时在注册表中注册自己并使用它来解析自己的主机。服务注册表非常有用，因为它可以实现客户端负载平衡，并使服务提供商与使用者分离，而无需DNS。

你需要什么

  • 约15分钟

  • 最喜欢的文本编辑器或IDE

  • JDK 1.8或更高版本

  • Gradle 4+或Maven 3.2+

  • 您还可以将代码直接导入到IDE中：

    • Spring Tool Suite（STS）

    • IntelliJ IDEA

如何完成本指南

像大多数Spring 入门指南一样，您可以从头开始并完成每个步骤，也可以绕过您已经熟悉的基本设置步骤。无论哪种方式，您最终都可以使用工作代码。

要从头开始 ，请继续使用Gradle构建 。

要跳过基础知识 ，请执行以下操作：

  • 下载并解压缩本指南的源存储库，或使用Git对其进行git clone https://github.com/spring-guides/gs-service-registration-and-discovery.git ： git clone https://github.com/spring-guides/gs-service-registration-and-discovery.git

  • cd进入gs-service-registration-and-discovery/initial

  • 跳起来站起来，建立一个Eureka服务注册表 。

完成后 ，您可以根据gs-service-registration-and-discovery/complete中的代码检查结果。

Spring Boot gradle插件提供了许多方便的功能：

  • 它收集类路径上的所有jar，并构建一个可运行的单个“über-jar”，这使执行和传输服务更加方便。

  • 它搜索public static void main()方法以将其标记为可运行类。

  • 它提供了一个内建的依赖解析器，设置了版本号，以匹配Spring Boot dependencies 。您可以覆盖所需的任何版本，但是它将默认为Boot选择的一组版本。

站起一个尤里卡服务注册中心

您首先需要一个Eureka服务注册表。您可以使用Spring Cloud的@EnableEurekaServer来建立其他应用程序可以@EnableEurekaServer的注册表。这是一个常规的Spring Boot应用程序，添加了一个注释以启用服务注册表。

eureka-service/src/main/java/hello/EurekaServiceApplication.java

package hello;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.cloud.netflix.eureka.server.EnableEurekaServer;

@EnableEurekaServer
@SpringBootApplication
public class EurekaServiceApplication {

    public static void main(String[] args) {
        SpringApplication.run(EurekaServiceApplication.class, args);
    }
}

注册表启动时，它将使用堆栈跟踪信息来抱怨没有副本节点可供注册表连接。在生产环境中，您将需要多个注册表实例。但是，出于简单的目的，足以禁用相关日志记录。

默认情况下，注册表也将尝试将其自身注册，所以你需要禁用，以及。

在本地使用此注册表时，最好将此注册表放在单独的端口上。

将一些属性添加到您的eureka-service/src/main/resources/application.properties以处理所有这些要求。

eureka-service/src/main/resources/application.properties

server.port=8761

eureka.client.register-with-eureka=false
eureka.client.fetch-registry=false

logging.level.com.netflix.eureka=OFF
logging.level.com.netflix.discovery=OFF

与注册表交谈

现在我们已经建立了一个服务注册表，让我们建立一个客户端，该客户端既向注册表注册自身，又使用Spring Cloud DiscoveryClient抽象为其自身的主机和端口查询注册表。@EnableDiscoveryClient激活Netflix Eureka DiscoveryClient实现。其他服务注册中心还有其他实现，例如Hashicorp的Consul或Apache Zookeeper 。

eureka-client/src/main/java/hello/EurekaClientApplication.java

package hello;

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.cloud.client.ServiceInstance;
import org.springframework.cloud.client.discovery.DiscoveryClient;
import org.springframework.cloud.client.discovery.EnableDiscoveryClient;
import org.springframework.web.bind.annotation.PathVariable;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

import java.util.List;

@EnableDiscoveryClient
@SpringBootApplication
public class EurekaClientApplication {

    public static void main(String[] args) {
        SpringApplication.run(EurekaClientApplication.class, args);
    }
}

@RestController
class ServiceInstanceRestController {

    @Autowired
    private DiscoveryClient discoveryClient;

    @RequestMapping("/service-instances/{applicationName}")
    public List<ServiceInstance> serviceInstancesByApplicationName(
            @PathVariable String applicationName) {
        return this.discoveryClient.getInstances(applicationName);
    }
}

无论选择哪种实现，您都将很快看到eureka-client以您在spring.application.name属性中指定的名称注册。该属性在Spring Cloud中经常使用，通常是在服务配置的最早阶段。此属性用于服务引导中，因此按照惯例，该属性位于eureka-client/src/main/resources/bootstrap.properties ，该位置在 src/main/resources/application.properties 。

eureka-client/src/main/resources/bootstrap.properties

spring.application.name=a-bootiful-client

eureka-client定义了一个Spring MVC REST端点ServiceInstanceRestController ，该端点返回在http://localhost:8080/service-instances/a-bootiful-client注册表中注册的所有ServiceInstance实例的枚举。请参阅《 构建RESTful Web服务》指南，以了解有关使用Spring MVC和Spring Boot构建REST服务的更多信息。

测试应用

首先启动eureka-service ，然后在加载后启动eureka-client ，以测试端对端结果。eureka-client大约需要一分钟时间才能在注册表中进行注册，并从注册表中刷新其自己的注册实例列表。所有这些阈值都是可配置的。参观eureka-client在浏览器中http://localhost:8080/service-instances/a-bootiful-client 。在那里，你应该看到ServiceInstance的eureka-client反映的响应。
