重要
 该项目已弃用。它已迁移到： Cloud Pipelines 。您可以在此处了解更多信息。

Spring，Spring Boot和Spring Cloud是允许开发人员加快创建新业务功能的时间的工具。但是众所周知，该功能只有在投入生产时才有价值。这就是公司花大量时间和资源来建立自己的部署管道的原因。

该项目试图解决以下问题：

  • 创建通用部署管道

  • 传播良好的测试和部署实践

  • 加快将功能部署到生产所需的时间

运行，配置和部署应用程序的常用方法降低了新开发人员在变更项目时融入其中所需的支持成本和时间。

Spring Cloud Pipelines具有以下功能

  • 自以为是的微服务平台

  • Jenkins使用声明性管道（Jenkinsfile）的自以为是的微服务管道

  • Cloud Foundry PaaS支持

  • Kubernetes PaaS支持

运行Cloud Foundry的演示

您可以通过运行默认演示设置轻松开始，以使用Docker / PCF Dev或Minikube在本地运行整个系统。

我们提供了一个将构建和部署的演示管道-Github Webhook应用程序。

总而言之，以下项目参与了整个microservice setup 。

  • Github Analytics-具有REST端点并使用消息传递的应用程序。我们的业务应用程序。

  • Github Webhook-发出Github Analytics使用的消息的项目。我们的业务应用程序。

  • 尤里卡 -简单的尤里卡服务器。这是一个基础结构应用程序。

  • Github Analytics Stub Runner引导程序 -用于与Github Analytics进行测试的Stub Runner Boot服务器。使用Eureka和消息传递。这是一个基础结构应用程序。

有关详细信息，请查看文档 。

大堂

  • 分叉Github Webhook和https://github.com/spring-cloud-samples/github-analytics [Github Analytics]存储库

  • 启动PCF开发人员

  • 克隆spring-cloud-pipelines项目

  • 进入concourse文件夹

  • 运行./setup-docker-compose.sh设置Concourse的密钥

  • 运行./start.sh 192.168.99.100以启动Concourse和Artifactory（例如，将192.168.99.100作为docker映像的外部IP。如果您没有Docker Machine，则可以运行./what_my_ip.sh脚本并在此处传递该值）

  • 运行./deploy_infra.sh将基础设施应用程序（ eureka和stub runner运行器）上传到Artifactory

  • 运行./login.sh 192.168.99.100登录到您的Concourse（其中192.168.99.100是您的Docker Machine ip-如果您使用的是localhost ，则不必提供该值）

  • 根据提供的credentials-sample-cf.yml设置您的credentials.yml credentials-sample-cf.yml （指向分叉的[Github Webhook]（ https://github.com/spring-cloud-samples/github-webhook） ）

  • 运行./set-pipeline.sh为github-webhook设置管道

  • 取消暂停并在Concourse中运行管道！

  • 管道通过后，您可以为github-analytics建立新的管道

有关Kubernetes逐步指南的信息，请查阅文档 。

詹金斯

  • 分叉Github Webhook和Github Analytics存储库

  • 启动PCF开发人员

  • 克隆spring-cloud-pipelines项目

  • 转到jenkins文件夹

  • 运行./start.sh yourGithubUser yourGithubPass yourGithubOrg来启动Jenkins和Artifactory（ yourGithubUser和yourGithubPass是管道所需的Github凭据，以标记位于yourGithubOrg的分叉yourGithubOrg ；如果您更喜欢使用密钥，也可以传递yourGithubOrg -key path_to_private_key Git存储库的基于身份的身份验证）

  • 运行./deploy_infra.sh将基础设施应用程序（ eureka和stub runner运行器）上传到Artifactory

  • 转到Jenkins并单击jenkins-pipeline-seed以生成管道作业

  • 转到github-webhook管道视图，然后单击play按钮以运行管道

对于Kubernetes逐步指南，只需查看文档即可

根据您的需求调整管道

该存储库可被视为管道的模板。我们提供了一些自以为是的实现，您可以根据自己的需要进行更改。使用它构建生产项目的最佳方法是将Spring Cloud Pipelines存储库下载为ZIP，然后在此处初始化Git项目并根据需要进行修改。

使用master分支中的代码的示例。您可以使用标签，例如1.0.0.M8 。然后您的URL将如下所示https://github.com/spring-cloud/spring-cloud-pipelines/archive/1.0.0.M8.zip 。

$ curl -LOk https://github.com/spring-cloud/spring-cloud-pipelines/archive/master.zip
$ unzip master.zip
$ cd spring-cloud-pipelines-master
$ git init
$ ./gradlew customize
$ # modify the pipelines to suit your needs
$ git add .
$ git commit -m "Initial commit"
$ git remote add origin ${YOUR_REPOSITORY_URL}
$ git push origin master

您还可以克隆存储库，以防与上游存储库中的更改保持一致。为了避免发生很多合并冲突，建议使用自定义文件夹钩子覆盖函数。
