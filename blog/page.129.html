Spring Integration 4.3 GA可用

发布
阿尔特姆·比兰（Artem Bilan）
六月14，2016

尊敬的Spring社区，

我很高兴地宣布，Spring集成框架4.3 GA是现在可从repo.spring.io ，以及Maven的中央！维护4.2.8版本也已发布。连同内部4.2.7版本一起，它包含一些重要的错误修复 。

除了4.3 RC1公告中提到的功能主题外，GA版本还包括以下几个主要更改：

  • 升级到Spring Framework 4.3 GA，Spring AMQP 1.6 GA，Spring Security 4.1 GA;

  • FtpRemoteFileTemplate现在提供ExistsMode选项来控制ExistsMode exists()操作的工作方式。

  • FileSplitter可以将FileMarker对象表示为JSON字符串（ markersJson选项），以避免在分布式环境中对使用者应用程序产生额外的spring-integration-file依赖性；

  • 引入了新的JdbcLockRegistry功能。特别感谢Dave Syer ！

  • 集成Graph已通过通用网关节点进行了改进，并且LinkNode现在具有type （ input ， output ， error ， discard ， route ）属性，以提供更好的可视化体验；

  • 其他一些小的改进，并且不仅是简单的（ WebSocketStompSessionManager.connect ）错误修复。

引入Spring Cloud Cloud Foundry Service Broker

发布
斯科特·弗雷德里克
2016年6月7日

我很高兴地宣布Spring Cloud系列的最新成员。Spring Cloud Cloud Foundry Service Broker是一个框架，用于为Cloud Foundry平台构建服务代理 。

Cloud Foundry服务经纪人

服务代理提供了一种通过托管服务扩展Cloud Foundry的方法，这些服务可由部署到平台的应用程序使用。托管服务通常向应用程序公开某种资源，例如数据库或其他持久性存储，消息系统或其他支持基础结构。服务代理发布服务和服务计划的目录，管理服务实例的供应和取消供应，并为应用程序消耗资源提供连接详细信息和凭据。服务代理已注册到Cloud Foundry，并使用定义良好的Service Broker REST API与平台进行通信。

Spring Cloud Cloud Foundry Service Broker将完整的服务代理REST API实施为Spring MVC端点。这使服务代理作者可以专注于管理支持资源所需的逻辑，而不必担心实现代理API语义。
