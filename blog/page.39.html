Spring Cloud Data Flow 1.7 GA发布

发布
马克·波拉克
十月25，2018

Spring Cloud Data Flow团队很高兴宣布发布1.7.0 。遵循本地服务器 ， Cloud Foundry和Kubernetes的入门指南 。在接下来的几周中寻找适用于SCDF的更新的Cloud Foundry Tile 。

这是亮点

  • 改进的用户界面

  • 流应用DSL

  • 审计追踪

  • 并发任务启动限制

  • 流和任务验证

  • 强制升级流

  • Kubernetes上的任务调度

改进的用户界面

用户界面具有全新的外观。导航已从选项卡移至左侧导航系统。这样可以增加使用Flo Designer创建流的屏幕空间，并且可以通过最小化左侧导航来获得更多屏幕空间。有一个快速搜索功能，可以搜索所有不同的数据流类别。添加了其他颜色和整体主题更改，以使UI看起来更生动。更深入的核心是，路由管理得到了改进，并且我们使用BrowserStack / SauceLabs增加了端到端测试的覆盖范围。如果白名单为空，则属性白名单功能已得到改进，默认情况下不会显示所有应用程序属性。观看视频了解用户界面。

Spring Cloud功能中的功能Bean注册

工程
戴夫·瑟
十月22，2018

Spring Cloud Function在2.0中有几个新功能（仍处于里程碑阶段），可能最引人注目的是“完全运行”的功能。通过在Spring Boot 2.1和Spring Framework 5.1中进行更改，可以实现这一点，这意味着在Spring应用程序中以不同的方式考虑bean定义，但也大大提高了启动性能。

AWS节省成本

从图片开始总是好事，特别是当它讲述一个故事时。下图显示了Spring Cloud Function 2.0在1.0之上的改进，并比较了AWS中冷启动的成本：

x轴是以MB为单位的内存，y轴是以GBsec为单位的冷启动成本。最显着的效果是对于低内存容器，其成本几乎是2.0的4倍。“自定义”功能甚至更快（比Spring Cloud Function 1.0快10倍）-它是使用带有功能Bean的Spring Cloud Function的自定义AWS运行时 。改进的根源在于大大缩短了启动时间，而启动时间又来自于在应用程序中使用bean定义的功能形式。如果您需要介绍的话，Josh做了一段关于功能bean注册的视频 （在YouTube上）。现在，让我们仔细看看它在Spring Cloud Function中的工作方式。

Spring Cloud Data Flow 1.7 RC1发布

发布
马克·波拉克
十月19，2018

Spring Cloud Data Flow团队很高兴宣布发布1.7 RC1 。遵循本地服务器 ， Cloud Foundry和Kubernetes的入门指南 。

RC1版本以1.7 M1中引入的核心功能为基础，并进行了一些改进。

这是亮点

  • 改进的用户界面 -用户界面具有全新的外观。如果白名单为空，则属性白名单功能已得到改进，默认情况下不会显示所有应用程序属性。

  • 流应用程序DSL-已集成到UI中，因此您可以设计由任意Boot应用程序组成的数据管道。

  • 支持Kubernetes上的任务计划 。

  • 审计追踪 。 —已添加支持的任务和计划程序。

  • 并发任务启动限制 。

  • 流和任务验证 。 —添加了对注册为“ app”类型的应用程序的验证的支持。

  • 强制升级Streams 。

  • 添加了用于自定义Docker组成部分以包括Skipper的部分。请参阅“ 定制Spring Cloud数据流”中的项目5。
