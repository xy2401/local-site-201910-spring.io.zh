SpringOne2GX 2015重播：使用Spring Boot，Nashorn和React的同构模板

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2016年3月15日

记录在SpringOne2GX 2015中。
演讲者：塞巴斯蒂安·德勒兹（Sebastian Deleuze）
Web / JavaScript跟踪
幻灯片： http ： //www.slideshare.net/SpringCentral/isomorphic-templating-with-spring-boot-nashorn-and-react
本讲座的第一部分介绍即将推出的Spring Framework 4.2脚本模板支持，
展示了如何使用Nashorn，Jython或JRuby通过基于Javascript，Python或Ruyby的模板引擎渲染模板。我们将在脚本编写方面看到您可以轻松扩展现成的功能。
根据基准，可用功能和代码示例，我们将了解它是否可以替代通常用于Spring MVC的模板技术（如JSP，Velocity，FreeMarker或Thymeleaf），并已投入生产。
演讲的第二部分将说明如何构建一个同构应用程序，该应用程序基于Spring Boot，Nashorn和React（Facebook发布的最新游戏规则Javascript库）在客户端和服务器上执行相同的模板渲染。借助于Spring MVC + Nashorn集成，最初在服务器端执行Javascript模板的呈现，然后在客户端通过服务器发送事件推送机制进行丰富。
React通过他的虚拟DOM机制带来的这种同构模板+颠覆性的创新使构建新的应用程序成为可能：具有响应能力，具有出色的UX且可访问。

SpringOne2GX 2015重播：基于RabbitMQ，Spring和Zipkin的CallTracing（tm）简介

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2016年3月15日

记录在SpringOne2GX 2015中。
演讲者：Monish Unni
数据/集成跟踪
幻灯片： http ： //www.slideshare.net/SpringCentral/introducing-calltracing-tm-based-on-rabbitmq-spring-and-zipkin
您是否生活在StackTraces还不够的世界中？没有简单的方法可以预测一组服务的调用方式或使用方式？在您投入生产之前，一切都会在DIT / SIT / UAT / PELT中起作用，并且由于服务的分发而开始发生奇怪的事情吗？解决方案：使用RabbitMQ（AMQP协议）和Spring 代理/拦截器来启用带外工具来跟踪请求并获得有关某些请求在分布式系统中的执行情况的深入知识。在2014年，作为基础架构范围内变更的一部分，我引入了calltracing（tm），以将跨E * Trade异构系统的单个用户的请求关联起来。然后，各种大数据分析工具都会使用此“跟踪”，以生成汇总报告。Zipkin（tm）是收集器，摘要和聚合数据的可视化前端。换句话说，它是一个分布式跟踪系统，可以显示各个节点上的服务的时序数据。Zipkin通过收集器和查询服务管理数据的收集和查找。在本节中，我将专门讨论如何使用RabbitMQ（AMQP协议）和Spring Proxies将E * Trade的各种服务组合在一起，以形成支持层，以向Zipkin提供数据。

增压操作和分析：使用Spring XD支持分析和CEP

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2016年3月15日

记录在SpringOne2GX 2015中。
演讲者：CapitalOne的约瑟夫·保尔切尔
大数据轨道
幻灯片： http ： //www.slideshare.net/SpringCentral/supercharging-operations-analysis-using-spring-xd-to-support-analytics-and-cep
运营，分析和业务团队要求提供给大数据分析平台和工具的数据量不断增加。希望更好地了解用户体验，服务质量，系统的实时性能，以及为改善服务，销售产品和使用户满意而存在的未发现模式和机会，从而推动了这一需求。Capital One Technology使用Spring Extreme Data（XD）和自定义Java库来解决面向消费者的应用程序对数据的不断增长的需求，这些数据实时地将数据从应用程序流传输到Hadoop HDFS，Mongo，Kafka，Splunk等。在本次演讲中，我们将讨论从使用数据库的面向批处理的过程到实时数据流解决方案的过程以及所获得的重大收益。我们将介绍采用流式解决方案的含义，为什么选择Spring XD以及我们正在实现的用于将数据放入HDFS中的目标体系结构。我们将介绍如何自动进行环境供应，我们的客户端库（Java，Spring）的设计，我们的XD环境以及如何利用所有这些数据。我们将描述在此过程中克服的挑战，包括连接到我们的HDFS集群，使用多个Mongo存储，使用Kerberos以及确保端到端敏感数据的保护和加密。最后，我们将讨论评估Spring XD的许多用例，并看到潜在的好处，包括“滚动窗口”系统分析，数字消息传递和事件驱动的体系结构。

Reactor Core 3.0成为基于Java 8的统一Reactive Foundation

发布
斯蒂芬·马尔蒂尼（Stephane Maldini）
2016年3月11日

[更新]
在引入Reactor 2.5之后，我们将版本更改为3.0，以更好地反映已进行的重大重新设计工作。

有意义的合成API

以前的反应器流模块已合并到Reactor Core 3.0中。助焊剂和单声道分别代表0 ..N和0..1序列类型现在涵盖以下类别的可靠操作范围：

  • 冷热或多播： publish ， publishNext ， cache ， multicast …
  • 汇总/归约（转换）： buffer ， reduce ， scan ， window ， sample …
  • 筛选： filter ， exists ， single …
  • 调理： timeout ， take ， takeUntil ， skip ， skipUntil …
  • 结合： withLatestFrom ，combineLatest
  • 反压onBackpressureDrop ， onBackpressureLatest …

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2016年3月9日

使用数据微服务在云中进行流处理
演讲者：Pivotal的Marius Bogoevici

可伸缩数据处理的未来是事件驱动的微服务！它们提供了强大的范例，可以解决通常与分布式应用程序相关的问题，例如可用性，数据一致性或通信复杂性，并允许创建复杂且可扩展的数据处理管道。

基于Spring Boot提供的易于开发和部署以及Spring Cloud的云原生功能，Spring Cloud Stream项目提供了一个简单而强大的框架来创建事件驱动的微服务。它们使开发基于Spring Integration功能的数据处理Spring Boot应用程序变得容易。在更高的抽象层次上，Spring Cloud Data Flow是一个集成的编排层，可为部署和管理由独立微服务组成的复杂数据管道提供高生产力的体验。流是使用DSL抽象定义的，可以通过外壳和Web UI进行管理。此外，可插入的运行时SPI允许Spring Cloud Data Flow跨各种分布式运行时平台（例如Apache YARN，Cloud Foundry，Kubernetes或Apache Mesos）协调这些应用程序。
