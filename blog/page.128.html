  • 添加了对将命令行args配置为单独的一组属性的支持，以便在部署应用程序时将其传递给应用程序。这是运行Spring Batch作业的重要要求
  • 多项Ambari插件改进，以简化Spring Cloud Data Flow的YARN服务器的配置

Kubernetes 1.0.0的Spring Cloud数据流。RC1发布

发布
托马斯·里斯伯格
2016年6月21日

我很高兴宣布1.0.0。Kubernetes的Spring Cloud Data Flow的RC1发行候选版，这是一个团队的工作，其中包含了许多新功能。

该候选版本基于最新的1.0.0。Spring Cloud Data Flow的RC1版本，还增加了Cloud Foundry特定的改进。一些重点包括：

  • 实现TaskLauncher SPI以支持启动短命应用程序（也称为任务）。它们将作为Kubernetes作业创建，并且每次任务启动都会在Kubernetes作业中创建一个新的容器。
  • 添加了对将命令行args配置为单独的一组属性的支持，以便在部署应用程序时将其传递给应用程序。这是使用Spring Cloud Task编程模型运行作为任务编排的Spring Batch作业的重要要求
  • 增加了对RDBMS“数据源”的支持，以补充持久性存储库要求
  • 新增/info查询健康检查并补充探测要求的API
  • 改进健康端点，以标准化对Kubernetes中精心策划的所有应用程序的就绪状态的探测
  • 为Spring Cloud Data Flow的Kubernetes服务器添加docker映像，以便可以将其部署为Kubernetes复制控制器和服务
