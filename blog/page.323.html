Spring Batch 2.0新功能概要

工程
戴夫·瑟
2008年10月21日

在本文中，我们概述了Spring Batch 2.0的主要主题，并重点介绍了1.x的更改。新版本的开发工作正在进行中，上周发布了M2版本，引起了我们的极大兴趣，因此现在似乎是个提供一些建议的好时机。

Spring Batch 2.0主题

新版本的四个主要主题是


  • Java 5和Spring 3.0

  • 非顺序执行

  • 可扩展性

  • 配置：注释和XML名称空间


在Spring Batch 2.0.0中，项目的物理布局没有任何变化。M2（相同的旧下载，Java包的基本布局相同）。我们尚未删除任何功能，但已利用这次机会修改了几个API，对于从1.x更新项目的人们也做了一些小的更改。Spring Batch尚不成熟，我们添加了一些非常重要的功能，因此我们认为对主要版本进行更改是一个很好的机会，可以进行一些清理。我们不希望任何人在升级方面遇到任何困难，如果您是现有用户，那么本文将帮助您评估更改的程度。

了解OSGi使用指令

工程
格林·诺明顿
2008年10月20日

如果您为SpringSource dm Server或任何其他OSGi平台构建应用程序，则可能不久就会遇到using指令。除非您对指令的用途有清楚的了解，否则您将不知道何时编写该指令，并且您将只能猜测由于使用冲突而导致束无法解决的时间。这篇文章应该给你的使用指令，当使用它，以及如何调试用途冲突的全面理解。

捆绑分辨率

OSGi的设计宗旨是，一旦“解决”了捆绑包，通常就不应该遇到由于类型不匹配而导致的类强制转换异常和类似问题。这是非常重要的，因为OSGi为每个捆绑软件使用了一个类加载器，因此有足够的机会使用户暴露于各种类型的不匹配。
