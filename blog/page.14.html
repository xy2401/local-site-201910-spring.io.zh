Java CFEnv 1.1.0。RC1发布

发布
马克·波拉克
2019年6月12日

介绍

我很高兴代表社区宣布Java CFEnv 1.1 RC1的发布。

RC1版本添加了以下功能：

  • 检查类路径以正确确定MySQL或MariaDB驱动程序类名称的设置。

  • 使用引导支持时，如果Spring Cloud Connector库位于类路径上，则会引发异常。这仅适用于以下服务：DataSource，RabbitMQ，Cassandra，MongoDB和Redis。异常消息指示设置环境变量JBP_CONFIG_SPRING_AUTO_RECONFIGURATION '{enabled: false}'

  • 通过将日志记录实用程序类复制到项目中来支持Boot1.5.x。

项目README具有更多信息。

GA版本计划于下周发布。请尝试一下，并在我们的github问题页面上提供反馈。

宣布nohttp

工程
罗伯·温奇
2019年6月10日

我很高兴地宣布nohttp项目，该项目使用户可以查找，替换和阻止使用http:// 。

背景

今天， 乔纳森·莱特舒 （ Jonathan Leitschuh）发布了一个名为“ 想要接管Java生态系统？您只需要一个MITM！ 。该博客演示了数百个Java库正在通过HTTP下载依赖项。这使项目容易受到潜在的MITM（中间人）攻击 。

不幸的是，有多个使用HTTP下载依赖项的Spring项目。幸运的是，我们没有发现成功进行MITM攻击的迹象。我们还解决了该问题，以确保将来无法进行MITM攻击。

Java CFEnv 1.1.0。M1发布

发布
马克·波拉克
五月31，2019

介绍

我很高兴代表社区宣布Java CFEnv 1.1 M1的发布。

此版本带来了几个团队的贡献

  • EMC卷服务

  • 关键单点登录服务

  • 重要的Redis服务

支持批量服务是一项新功能。单一登录功能已得到改进，可以为Spring Security 5的OAuth支持设置Spring Security自动配置属性。Redis支持已得到改进，以支持TLS的自动配置。

项目README具有更多信息。

下个候选版本将在下周出炉，紧接着是GA发布。请尝试一下，并在我们的github问题页面上提供反馈。
