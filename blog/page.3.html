可用Spring Integration 5.2 GA

发布
阿尔特姆·比兰（Artem Bilan）
十月2，2019

亲爱的春季社区，

我很高兴代表Spring Integration小组宣布Spring Integration 5.2.0.RELEASE可用。

可以从我们的发布存储库和Maven Central中下载：

compile "org.springframework.integration:spring-integration-core:5.2.0.RELEASE"

这代人有什么新东西？

  • RSocket支持-一个RSocketInboundGateway和RSocketOutboundGateway在RSocket协议中为响应流支持而实现。看到spring-integration-rsocket模块。

  • FluxAggregatorMessageHandler根据Project Reactor生成窗口或缓冲区结果Flux分组和窗口功能。

  • RateLimiterRequestHandlerAdvice和CacheRequestHandlerAdvice可用于AOP方面MessageHandler.handleMessage() 。

  • Kotlin脚本的JSR223实现已添加到spring-integration-scripting模块。

  • 现在，FTP和SFTP模块为某些Apache Mina FTP / SFTP服务器事件提供了事件侦听器。

  • 现在提供了简单的Apache Avro变压器。

  • 在整个项目中还有许多其他改进。

看到What’s New?有关详细信息和可能的重大更改，请参见参考手册和迁移指南 。

即将推出GA的Spring Batch 4.2！

发布
Mahmoud Ben Hassine
十月2，2019

我代表Spring Batch团队高兴地宣布Spring Batch 4.2全面上市，其中包括许多新功能和性能改进！

新的功能

Spring Batch 4.2包含以下新功能：

  • 千分尺批处理指标
  • 支持Apache Kafka
  • 支持Apache Avro

千分尺批处理指标

Spring Batch始终通过作业存储库为用户提供各种指标。但是，使用数据库作为度量标准存储已使使用这些度量标准并对实时做出反应的能力静音。此版本引入了一项新功能，使您可以通过Micrometer使用真实的指标存储来监视批处理作业。默认情况下，Spring Batch收集指标（例如作业持续时间，步骤持续时间，项目读写吞吐量等），并将其注册到Micrometer的全局指标注册表中。 spring.batch字首。这些度量可以发送到Micrometer支持的任何监视系统。

Spring Data Moore正式上线

发布
马克·帕鲁奇
2019年10月1日

我很荣幸地代表社区和所有贡献者，从Maven Central和我们的发布存储库中普遍提供Spring Data Moore。

Spring Data Moore需要JDK 8或更高版本，并且特别支持JDK 11作为当前的长期支持分支以及JDK 13作为最新的OpenJDK版本。这是此发行列车处理的960张车票的摘录：

  • 强制性和响应式EntityCallback API，用于不可变对象支持，并作为将来的响应式审核的基础
  • 对Kotlin协程的一流支持
  • 性能提升
  • 声明式聚合和反应式Querydsl支持Spring Data MongoDB存储库
  • Spring Data MongoDB中的反应式事务管理器
  • 对Spring Data Elasticsearch的反应式支持
  • Spring Data Redis对Redis Streams的支持
