网络研讨会重播：使用数据微服务在云中进行流处理

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2016年3月30日

主讲人：Pivotal的Marius Bogoevici

可伸缩数据处理的未来是事件驱动的微服务！它们提供了强大的范例，可以解决通常与分布式应用程序相关的问题，例如可用性，数据一致性或通信复杂性，并允许创建复杂且可扩展的数据处理管道。

基于Spring Boot提供的易于开发和部署以及Spring Cloud的云原生功能，Spring Cloud Stream项目提供了一个简单而强大的框架来创建事件驱动的微服务。它们使开发基于Spring Integration功能的数据处理Spring Boot应用程序变得容易。在更高的抽象层次上，Spring Cloud Data Flow是一个集成的编排层，可为部署和管理由独立微服务组成的复杂数据管道提供高生产力的体验。流是使用DSL抽象定义的，可以通过外壳和Web UI进行管理。此外，可插入的运行时SPI允许Spring Cloud Data Flow跨各种分布式运行时平台（例如Apache YARN，Cloud Foundry，Kubernetes或Apache Mesos）协调这些应用程序。

使用Apache Kafka开发实时数据管道http://kafka.apache.org/为开发人员提供了有关为什么以及如何使用Apache Kafka的介绍。Apache Kafka是一个发布-订阅消息传递系统，被重新认为是分布式提交日志。Kafka旨在允许单个群集充当中央数据主干。单个Kafka代理可以处理来自数千个客户端的每秒数百兆的读写操作。它可以弹性透明地扩展，而无需停机。数据流被分区并分布在一组机器上，以允许数据流大于任何一台机器的能力，并允许一组协调的使用者。消息将保留在磁盘上并在群集内复制，以防止数据丢失。每个代理可以处理TB级的消息。对于Spring用户，Spring Integration Kafka和Spring XD提供与Apache Kafka的集成。

在本系列的第五部分中，我们将仔细研究STS 3.7.3中添加到Spring Boot Dashboard的对多种启动配置的新支持。

每个项目有多个启动配置

启动仪表板的第一个版本使您可以快速启动和停止本地Spring Boot应用程序。因此，启动仪表板将重点放在该项目的一个特定启动配置上，而完全忽略了其他启动配置。但是，每个项目具有多个启动配置可能非常有用，例如，以略有不同的配置多次启动同一应用程序。

我们很高兴宣布Spring AMQP 1.6的第二个里程碑已经发布。

自第一个里程碑以来的新增功能包括：

  • 缓存连接工厂现在公开缓存统计信息

  • @RabbitListener现在，方法将类型信息传递给消息转换器以获取入站消息。例如，这意味着Jackson2JsonMessageConverter当收到标头中没有类型信息的消息时（例如从不是Spring AMQP应用程序的发件人处），不再需要自定义类映射器。相反， @RabbitListener转换中使用方法参数类型。除此之外Method和bean也可以通过消息属性提供给自定义转换器。

强调

布里克斯顿发行火车的一些亮点是：
