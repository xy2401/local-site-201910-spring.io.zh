用于Apache Hadoop 2.2 GA的Spring发布

发布
托马斯·里斯伯格
2015年6月11日

我们很高兴地宣布Apache Hadoop 2.2 GA的春季发布。

Spring中针对Apache Hadoop 2.2版本最重要的更改/增强是：

  • 现在需要删除对与JDK 6，Java 7或更高版本一起运行的支持
  • HDFS编写器的改进，以支持可同步写入和新的超时选项
  • 对HBase支持的改进-将PUT和DELETE添加到HbaseTemplate
  • 添加对Pivotal HD 3.0的支持
  • 更新Cloudera CDH 5以使用版本5.3.3
  • 将Hortonworks HDP 2.2版本更新到2.2.4.0
  • 将Kite SDK更新到1.0版
  • 将Spring项目版本更新为最新版本

Spring Framework中的CORS支持

工程
塞巴斯蒂安·德勒兹（SébastienDeleuze）
2015年6月8日

出于安全原因，浏览器禁止AJAX调用当前来源之外的资源。例如，当您在一个标签中检查银行帐户时，可以在另一个标签中打开evil.com网站。evil.com的脚本不应该向您的银行API发出AJAX请求（从您的帐户中提取资金！）使用您的凭据。

跨域资源共享 （CORS）是由大多数浏览器实施的W3C规范 ，使您可以灵活地指定对哪种跨域请求进行授权，而不必使用一些安全性和功能不强的hack（例如IFrame或JSONP）。

Apache Ambari遇到Spring XD

工程
珍妮·瓦尔基亚拉赫蒂
2015年6月2日

我们很高兴宣布Spring XD的Apache Ambari插件的第一个版本。

Ambari是一个群集配置工具，用于安装和管理各种Apache项目，例如HDFS，YARN，Zookeeper或Kafka。Ambari具有堆栈定义的概念，Pivotal和Hortonworks均使用该定义。堆栈定义通常由维护者支持的组件组成。

这种Ambari集成增加了对使用Pivotal HD 3.0和Hortonworks HDP 2.2设置Spring XD的支持。

Spring XD不是PHD或HDP的主要Ambari发行版的一部分。但是，通过修改现有堆栈定义的简单RPM安装，可以轻松扩展现有的Ambari安装。

Spring XD 1.2 RC1发布

发布
马克·波拉克
2015年6月2日

我非常荣幸地代表Spring XD团队宣布Spring XD 1.2 Release Candidate现在可以下载了 。您也可以使用rpm安装

1.2 RC1版本包括错误修复以及几个新功能和增强功能。

  • Apache Ambari插件可自动将Spring XD部署到Hadoop集群。在插件发布博客中了解更多信息。
  • 用户界面中的新“ Google Analytics（分析）”标签可让您轻松查看量规，计数器，字段值计数器和汇总计数器。
  • Kafka消息总线的性能改进使其与Kafka附带的性能基准测试应用程序相当。即将发布的博客将更详细地讨论XD性能
  • 改进了对Rabbit Message Bus的HA支持，XD容器连接到承载正在使用的队列的Rabbit Broker，
  • 支持Sqoop元存储和其他Sqoop集成改进
  • 支持Kafka来源中的多个主题。
  • 将rolloverTime选项添加到HDFS接收器
  • 通过jdbchdfs作业支持增量数据导入
  • 能够利用Spark流处理器
  • 在邮件源中支持任意JavaMail属性
  • 几个新的源/接收器模块
    • 视频捕获源 （社区贡献）
    • FTP源和接收器（社区贡献）
    • 负载生成器源和吞吐量接收器
    • xslt-transformer （社区贡献）
    • 语言分类处理器
  • 现在，gpfdist接收器包含在XD发行版中
  • 基于反应堆的处理器的性能改进
  • 支持将Oracle作为批处理存储库
  • 从log4j更改为Logback日志记录框架
