Spring Cloud Pipelines 1.0.0。M6发布

发布
马尔辛·格热兹扎克（Marcin Grzejszczak）
2017年8月4日

我很荣幸代表Spring Cloud团队宣布Spring Cloud Pipelines的新里程碑发布1.0.0.M6 。

什么是新的？

管道描述符

每个应用程序可以包含一个具有以下结构的文件pipeline.yml ：

lowercaseEnvironmentName1:
    services:
        - type: service1Type
          name: service1Name
          coordinates: value
        - type: service2Type
          name: service2Name
          key: value
lowercaseEnvironmentName2:
    services:
        - type: service3Type
          name: service3Name
          coordinates: value
        - type: service4Type
          name: service4Name
          key: value

对于给定的环境，我们声明要部署的基础结构服务的列表。服务有

  • type （例如： eureka ， mysql ， rabbitmq ， stubrunner ） -这个值被随后施加到deployService bash函数

  • name要部署的服务的名称

  • coordinates -允许您获取服务二进制文件的坐标。示例：它可以是Maven坐标groupid:artifactid:version ，泊坞窗图像organization/nameOfImage等。

  • 任意键值对-您可以根据需要自定义服务

stubrunner类型也可以将useClasspath标志打开为true或false。

例：

test:
  services:
    - type: rabbitmq
      name: rabbitmq-github-webhook
    - type: mysql
      name: mysql-github-webhook
    - type: eureka
      name: eureka-github-webhook
      coordinates: com.example.eureka:github-eureka:0.0.1.M1
    - type: stubrunner
      name: stubrunner-github-webhook
      coordinates: com.example.eureka:github-analytics-stub-runner-boot-classpath-stubs:0.0.1.M1
      useClasspath: true
stage:
  services:
    - type: rabbitmq
      name: rabbitmq-github
    - type: mysql
      name: mysql-github
    - type: eureka
      name: github-eureka
      coordinates: com.example.eureka:github-eureka:0.0.1.M1

当部署到test或部署stage时，春季云管道将：

  • 对于test环境，请删除现有服务，然后从列表中重新部署这些服务

  • 对于stage环境，如果服务不可用，则会对其进行部署。否则什么都不会发生

发行说明

您可以单击此处以查看发行说明 ，以获取有关此里程碑所做的更多信息。

Spring Security OAuth 2.2发布

发布
乔·格兰贾
2017年7月28日

我很高兴代表社区宣布Spring Security OAuth 2.2.0的发布。发布。

2.2.0。RELEASE包括以下新功能：

  • JwtClaimsSetVerifier提供验证JWT Claims Set中包含的索赔的功能。

  • IssuerClaimVerifier ，用于验证JWT Claims Set中包含的Issuer（iss）声明。

  • DelegatingJwtClaimsSetVerifier仅将声明验证委派给JwtClaimsSetVerifier的内部列表。

  • ProviderDiscoveryClient ，它能够发现OpenID Connect Discovery 1.0规范定义的提供程序配置信息。

  • JwkTokenStore现在支持多个JWK设置URL。

  • 向JwkTokenStore提供自定义AccessTokenConverter的能力。

Spring Batch 4.0.0。M3现在可用

发布
迈克尔·米内拉
2017年7月26日

我们很高兴地宣布Spring Batch 4.0.0。现在可以通过Github和Pivotal下载存储库使用M3。这个版本代表了Spring Batch 4.0版本的第三个里程碑。非常感谢对此版本做出的所有贡献。

什么是新的？

该里程碑是先前4.0.0里程碑中列出的工作的延续。具体更新包括：

  • 依赖关系更新
  • 持续的Java配置改进
  • 文档的初始更新

依赖关系更新

随着Spring Boot 2依赖关系图的发展，我们一直在更新依赖关系以使其与它们保持一致。这包括更新Spring Data Kay和Spring Integraiton 5依赖关系，以及删除对Log4J 1.x的支持，而改用较新的2.x系列。
