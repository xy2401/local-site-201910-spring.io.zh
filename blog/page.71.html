Spring Cloud Data Flow 1.3.0。M3发布

发布
马克·波拉克
2017年11月29日

我们很高兴宣布1.3.0。M3版本的Spring Cloud Data Flow及其相关项目生态系统。

本地服务器： 入门指南

发布要点

流更新和回滚

精心策划为一系列微服务应用程序的流数据管道一直是Spring Cloud Data Flow设计的核心价值。在1.3.0中。M3提供了独立更新源，处理器和接收器的能力，而无需取消部署和重新部署整个流。

流更新功能是通过将部署过程委派给名为Skipper的新Spring Cloud项目来实现的。在此博客中介绍的Spring Cloud Skipper是一台独立的服务器，可将Spring Boot应用程序部署到多个云平台。它还跟踪应用程序版本，应用程序属性以及已部署的一个或多个应用程序的部署属性，以便可以根据更新请求来计算对这些属性中的任何一个的更改。

当请求更新Stream中的应用程序时，仅更新需要更改的一个或多个应用程序。执行简单的红色/黑色更新，并且设计依赖于与新应用程序相关联的Spring Boot的运行状况端点。通过跟踪部署，Skipper还可以回滚到流的先前版本。

以下新的数据流命令显示了将日志接收器从版本1.1.0更新到版本1.2.0，然后回滚到1.1.0的基本生命周期。假定日志接收器的版本1.1.0和1.2.0均已部署为Maven工件。请注意，在对日志接收器进行的所有更改中，HTTP源应用程序始终保持部署状态。

dataflow:> app register --name http --type source --uri maven://org.springframework.cloud.stream.app:http-source-rabbit:1.2.0.RELEASE

dataflow:> app register --name log --type sink --uri maven://org.springframework.cloud.stream.app:log-sink-rabbit:1.1.0.RELEASE

dataflow:> stream create --name httptest --definition "http --server.port=9000 | log"

dataflow:> stream skipper deploy --name httptest

dataflow:> stream skipper update --name httptest --properties version.log=1.2.0.RELEASE

dataflow:> stream skipper rollback --name httptest

该文档的入门和带队长流部分详细介绍了整个过程。

Spring REST Docs 2.0.0。发布

发布

2017年11月28日

谨代表每个贡献者，宣布Spring REST Docs 2.0.0。RELEASE已发布，可从Maven Central，JCenter和我们的版本库中获得

什么是新的？

Spring REST Docs 2.0是该项目的最新一代，并且需要Spring Framework 5.0和Java 8。

2.0版本的主要亮点是支持使用Spring WebFlux的WebTestClient记录API。 WebTestClient可以用于使用WebFlux Fn和WebFlux的基于注释的编程模型实现的API的轻量级测试和文档记录。它也可以用来记录可通过HTTP访问的任何API，这以前只能在使用REST Docs的REST保证支持时才能实现。
