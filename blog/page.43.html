九月24，2018

在我以前的博客文章中 ，我描述了如何设置和使用Spring Data JDBC。我还描述了使Spring Data JDBC比JPA更易于理解的前提。一旦考虑了引用，这将变得很有趣。作为第一个示例，请考虑以下域模型：

class PurchaseOrder {

  private @Id Long id;
  private String shippingAddress;
  private Set<OrderItem> items = new HashSet<>();

  void addItem(int quantity, String product) {
    items.add(createOrderItem(quantity, product));
  }

  private OrderItem createOrderItem(int quantity, String product) {

    OrderItem item = new OrderItem();
    item.product = product;
    item.quantity = quantity;
    return item;
  }
}

Spring Web Services 3.0.4 / 2.4.3发布了！

工程
格雷格·特恩奎斯特
九月19，2018

春季问候社区，

Spring Web Services团队同时发布两个版本。3.0.4。发行版是与2.4.3一起开发的主要分支。发布维护。

3.0.4。RELEASE带有一些下面列出的功能。它还具有对Java 11的新支持， Java 11是Java的下一个长期受支持的版本。

Java 11及更高版本

从Java 9开始，一些关键的Java EE软件包的可见性降低了。这些包括核心XML和基于SOAP的软件包。在Java 11中，这些软件包已被完全删除。由于Java 11是自Java 8以来的第一个长期发行版本，因此它是Spring Web Services与之对应的版本，以提供相应的长期支持。

这对您意味着什么？要在Java 9+上使用Spring Web Services，您不再可以依赖JDK提供的关键XML和基于SOAP的库。不用担心，解决方案就在这里！

如果您访问项目的构建文件，您将发现一个新的Java 11配置文件 。该配置文件包含必须添加到自己的构建文件中的其他依赖关系，才能使用Spring Web Services（或与此相关的任何基于XML / SOAP的库）。

Spring Web Services不会提供这些依赖项，因此您必须将它们添加到自己的构建文件中。但是，一旦您这样做，就不必再担心了（除非这些API的较新版本发布了）。

还在使用Spring Web Services 2.x维护分支吗？我们已经覆盖了您。在不破坏向后兼容性的情况下，2.4.3。RELEASE 还支持Java 11 。它仅使用稍旧的SOAP API版本（从1.3.8代替1.4.0）。一切都没有强迫您执行，因此您可以继续使用其他所有版本的相同版本。

当然，Java 11是否破坏Spring Web Services之外的应用程序的任何部分都取决于您。

介绍Spring Data JDBC

工程
延斯·肖德（Jens Schauder）
九月17，2018

随着即将发布的Lovelace GA发布，我们将发布一个新的Spring Data Module： Spring Data JDBC 。

Spring Data JDBC背后的思想是提供对关系数据库的访问，而不必考虑JPA的复杂性。 JPA提供了诸如延迟加载，缓存和脏跟踪等功能。如果您需要这些功能，虽然它们很棒，但实际上它们会使JPA及其行为的思考比必须的更加困难。

当您不期望延迟加载时，延迟加载可能会触发昂贵的语句，否则可能会因异常而失败。当您实际上想比较一个实体的两个版本时，缓存可能会妨碍您的工作，而肮脏使您很难找到所有持久性操作都经过的单个点。
