这篇文章重点介绍Spring 3.1 M2中Spring MVC的新增功能。这里是主题：

  • MVC名称空间的基于代码的等效项。
  • 可自定义的@MVC处理。

Spring MVC的基于代码的配置

正如Chris在上周五的博客文章中所指出的那样，XML名称空间大大减少了配置，但同时也降低了透明度，有时还降低了灵活性。对于MVC名称空间来说，这是正确的，它支持许多自定义，但不支持所有可用的自定义。这意味着您可以使用它，也可以保留它。我们相信基于代码的配置可以解决此问题，并提供从简单到高级的路径。

Spring Framework 3.1 M2发布

工程
于尔根·霍勒（Juergen Hoeller）
2011年6月9日

Spring Framework 3.1 M2已于本周发布 ，标志着3.1里程碑阶段的结束。我们现在进入候选发布阶段，准备在7月完成功能完整的RC1，并在9月发布GA版本。

3.1 M2完成了从2月的3.1 M1开始的几个主要主题的工作：

  • 我们已经稳定了环境抽象和环境配置文件机制 。如果您还没有尝试过，那么现在是检查它的好时机！

  • 我们基于Java的应用程序配置方法已从M1中的@Feature方法更改为M2中常规@Configuration类的@ Enable *注释 。

  • 缓存抽象已经过修订，可提供最小的缓存交互SPI。我们的声明性缓存解决方案（@Cacheable等）始终位于其之上。

春天的一周：2011年6月7日

工程
乔什·朗（Josh Long）
2011年6月8日

欢迎回到另一个激动人心的综述！这周真是一片模糊。老实说上周在欧洲的S2G论坛热闹非凡之后 ，发生了如此多的新事物。让一个男人喘不过气来，但是很兴奋。继续阅读！



  1. 网上可以观看Mark Fisher和Ramnivas Laddad的网络研讨会“ 从零到云端的60分钟之春” 。
    这次网络研讨会是对CloudFoundry上某些Spring，Spring Roo和Grails支持的快速浏览。看看这个！
    但是，在您开始观看之前，请快速在CloudFoundry.com上注册以访问公共的免费Beta版云服务。
    如果您想签出代码并了解更多信息，请签出CloudFoundry.org 。


  2. Spring BlazeDS负责人兼一般“ Spring web dude”的Jeremy Grelle宣布了Spring Flex项目的第一个候选版本 。Spring Flex项目将Flex BlazeDS中间件与Spring集成在一起，提供了一种非常简单的方式来公开Spring Bean，其方式可以被Flex或Adobe AIR Web和桌面客户端使用。Spring Flex项目还提供了与Spring Security的集成，并为完全基于BlazeDS或通过JMS或Spring Integration的基于服务器端推送的消息传递提供了紧密的支持。是否曾经想通知登录到应用程序的用户服务器端发生了某些事情（Twitter消息，新的AMQP消息，新的XMPP消息等等）？Spring Flex使它变得容易。

  3. SpringSource Tool Suite团队负责人Martin Lippert接受了有关SpringSource Tool Suite 2.6最新和最强大功能的采访。他谈到了许多亮点，包括STS 2.6的经过改进的Spring Webflow可视化效果，Java配置支持，云，基于代理的重新加载以及下一步。看看这个！

  4. Thomas Risberg宣布了对MongoDB 1.0.0版的Spring Data Document支持。M3 。Spring Data Document 1.0.0中的更改和新功能。M3包括大大改进的映射和转换支持。MappingMongoConverter现在是MongoTemplate和SimpleMongoConverter已被弃用，将被移除。默认集合名称的概念也已删除，MongoTemplate的所有操作均基于用于该操作目标的实体类的集合名称。实体类使用的集合名称默认为以小写字母开头的类名称，但可以使用@Document注解。请参阅更改日志以获取更多详细信息。

  5. 处女座3.0.0的里程碑5可供下载。这是一个重要的里程碑，它增加了重要的功能增强功能，将多个依赖项升级到了最新级别，包括Spring 3.0.5，Tomcat 7.0.12和Servlet 3.0，并修复了许多错误。发行说明中提供了完整的详细信息。EclipseRT的Virgo Web Server是完全基于模块的Java应用程序服务器，旨在以高度的灵活性和可靠性运行企业Java应用程序和Spring支持的应用程序。它提供了一个简单而全面的平台来开发，部署和服务企业Java应用程序。



  6. Spring Integration的提交人Marius Bogoevici已撰写
    关于使用JPA的选项的精彩文章EntityManager在JBoss AS中使用Spring 。该帖子的主要目的是应用程序服务器自动创建一个EntityManager ，默认情况下，因此可能不需要在Spring中重新创建-您只需插入现有引用即可。该方法专门用于解决功能完善的应用程序服务器的冒昧行为。如果您想在Tomcat中运行，那么Spring是配置JPA的最简单方法EntityManager 。Marius还解释了如何通过禁用应用程序服务器行为来让Spring完全运行该节目。这具有多个优点，这些优点通常是性能更高，以及与应用程序本身（而不是服务器）保持配置。

  7. Matt Raible已发布了有关Web应用程序安全性的博客文章和屏幕录像的后续内容。以前，他演示了如何以伪相同的方式使用Spring Security ， Apache Shiro和Java EE安全性来保护Web应用程序，并着重强调了这些差异。
    这篇后续文章讨论了所有这三种技术，并提供了在集成到Spring MVC应用程序中时启用程序化登录的比较。Spring Security支持已经存在很长时间了，并且可以在许多容器中工作（不只是Java EE 6兼容容器），不必大惊小怪。真好！


  8. 您是否涉足其他基于JVM的语言？您看过Scala吗？在最近在加利福尼亚州帕洛阿尔托举行的Scala Days会议上，Cloud Foundry团队宣布在CloudFoundry.com上提供了新的Scala支持 ！

  9. 如果您上周参加了在伦敦举行的S2G论坛，那么您将获得一份免费的《 开源杂志》 -印刷版（可免费下载） .PDF ）杂志。该出版物在涵盖某些Spring Framework 技术方面做得很出色。可从发行商的网站上免费下载。在此处查看第一个和第二个问题。

倒计时Grails 2.0：单元测试

工程
彼得·莱德布鲁克
2011年6月7日

Grails 1.4的第一个里程碑 （现在为2.0） 现在已经发布，我们正处在迈向旅程的最后阶段 1.4 2.0最终版。当我们接近这一点时，我将撰写一系列博客文章，其中涵盖了 1.4 2.0版本带来。我将从新的测试支持开始。

从一开始，Grails就为开发人员提供了三个级别的测试支持：单元，集成和功能。单元测试曾经并且仍然具有独立于Grails进行运行的好处，但是它们通常需要大量的模拟形式的额外工作。Grails 1.1引入的单元测试框架帮助进行了这种模拟，但是它仍然不能涵盖所有用例，因此开发人员需要比期望的早一些方法来进行集成测试，该集成测试在自举的Grails实例中运行。
