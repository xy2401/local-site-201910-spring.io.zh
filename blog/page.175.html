展望Spring Roo 2.0

工程
彼得·汉弗莱（Pieter Humphrey）
2015年3月30日

原作者：DISID Corporation

尊敬的Spring社区，

DISID Spring Roo团队正在忙于2.0发行版。使用Spring Roo 2.0，我们想恢复定义Roo项目的本质：

  • 作为快速的应用程序开发工具 ，Roo既不是框架也不是Maven插件。
  • 智能代码生成可提高生产率 。
    • 开发人员经常被迫花费太多的时间去做那些增加了价值的事情，Spring Roo的代码生成可以帮助他们保持专注。
  • 基于广泛的Spring技术创建应用程序：Spring Boot，Spring Data，Spring MVC，Spring Security等。
  • 根据最佳架构实践生成应用程序，例如：服务层，存储库层
  • 新的脚手架模型可基于jQuery和Bootstrap生成功能齐全的响应式UI 。
  • 提高可扩展性并增强Spring Roo项目的协作 ：
    • 我们将创建“ Roo市场 ”：Roobot的替代产品，易于维护且所有人均可使用，可以找到并跟踪第三方插件和Roo Addon Suites。
    • “ Roo Addon Suite ”是将一组加载项打包和分发在一起的好方法，例如，如果您想分发Roo自定义发行版。Roo Addon Suite基于OSGi R5子系统，可提供真正方便的部署模型，而不会损害Roo的模块化。

Spring Data Fowler有什么新功能？

工程
托马斯·达里蒙特
2015年3月26日

Spring Data发布火车Fowler的GA发布标志着6个月开发的终点。现在是时候让您对该版本的内容有一个印象，并简要介绍各个功能。Fowler发行培训的主要主题是性能改进和Java 8支持的增强，这些主要体现在Spring Data JPA和MongoDB模块中，但许多其他模块也实现了显着改进。

升级到Spring Data Fowler发布系列最简单的方法是使用Spring Boot并配置spring-data-releasetrain.version财产Fowler-RELEASE 。如果您尚未使用Spring Boot，请将Spring Data BOM添加到您的 Maven POM部分。

发布

2015年3月26日

尊敬的Spring社区，

我们很高兴地宣布，Spring Integration Kafka Support 1.1 GA现在可用。
将发布存储库与Maven或Gradle一起使用
compile "org.springframework.integration:spring-integration-kafka:1.1.1.RELEASE"
或下载发行档案 ，以进行试用。

在1.0 GA之后如此快速发布下一版本的主要原因是要解决即将发布的Spring XD的许多关键修复程序 1.1.1和1.2发布。此外，此版本还提高了消息生成流方案的性能，这需要升级到Spring Integration。 4.1 ，这也是Spring XD的最小值。
