Spring Cloud Task 1.1.0。RC1现在可用

发布
格伦·伦弗罗（Glenn Renfro）
2016年11月7日

我们很高兴地宣布Spring Cloud Task 1.1.0。RC1现在可以通过Github和Pivotal下载存储库获得。非常感谢所有对此版本做出贡献的人。

Spring Cloud Task 1.1.0提供以下功能：

该版本继续以1.1.0的主要主题为基础，该主题改进了整体任务功能以及将Spring XD中现有的功能迁移到Spring Cloud Task。1.1.0的功能集包括：

  • 更新的错误消息处理 -1.1.0将新列添加到TASK_EXECUTION表， ERROR_MESSAGE 。在此新列中，将抛出引发导致记录任务失败的任何异常的记录（此记录已从EXIT_MESSAGE以前版本中的字段）。此更改允许EXIT_MESSAGE以便更好地用于任务编排的领域。
  • 用于自定义启动分区工作程序的改进选项 -使用DeployerPartitionHandler ，1.1.0现在提供了围绕每个分区自定义环境变量和命令行参数的抽象方法。
  • 对外部执行ID的支持 -在PaaS或其他云基础架构上执行任务时，通常需要在任务的执行ID与基础系统提供的ID之间进行映射。此功能提供了将两者关联的功能，例如日志管理，审计等。
  • 支持外部任务执行记录 -此功能与对外部执行ID的支持一起使用，因为它允许启动系统在任务执行开始之前对其进行记录。
  • DB2支持 -感谢社区添加了对任务库的DB2支持。
  • 分区作业样本 -在本地，Cloud Foundry，Mesos，YARN和Kubernetes中签出更新的分区批处理作业样本以及有关如何将它们编排为短期任务的更新说明。

2016年11月2日

我很高兴代表团队宣布发布针对Kubernetes 1.1的Spring Cloud Data Flow的第二个里程碑。

用于Kubernetes的Spring Cloud Data Flow提供了对在Kubernetes上协调长时间运行的（流）和短期的（任务/批处理）数据微服务的支持。

注意：开始使用此新版本的一个好方法是遵循项目页面上的版本列表 ，其中包括下载坐标和参考指南的链接。

什么是Spring Vault和Spring Cloud Vault？

Spring Vault是HashiCorp Vault的客户端，它提供熟悉的Spring抽象。它带有@VaultPropertySource从Vault公开加密属性Environment和VaultTemplate访问保险柜中存储和加密的机密信息。

@Configuration
@VaultPropertySource("secret/my-application")
public class AppConfig extends AbstractVaultConfiguration {

    /**
     * Specify an endpoint for connecting to Vault.
     */
    @Override
    public VaultEndpoint vaultEndpoint() {
        return VaultEndpoint.create("localhost", 8200);
    }

    /**
     * Configure a client authentication.
     */
    @Override
    public ClientAuthentication clientAuthentication() {
        return new TokenAuthentication("…");
    }
}
