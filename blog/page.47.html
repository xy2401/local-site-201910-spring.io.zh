Spring Boot 2.1 M2现已上市

发布
斯特凡·尼科尔（StéphaneNicoll）
2018年8月21日

我代表团队和所有贡献者，高兴地宣布，Spring Boot 2.1的第二个里程碑已经发布，可以从我们的里程碑存储库中获得 。此版本关闭了75个问题并提出了请求 ！

第二个里程碑与Spring Framework 5.1保持一致。RC2 ，是支持Java 11的第一个版本。

第二个里程碑的其他亮点包括：

  • 自动配置任务执行（ @EnableAsync ）和任务计划（ @EnableScheduling ）
  • JPA的可配置引导程序模式
  • OAuth2资源服务器支持
  • 改进了Web应用程序的日志输出
  • 改进了数据大小属性的绑定。

Spring Session Bean-M1和Apple-SR4发布

发布
罗伯·温奇
八月17，2018

该帖子由VedranPavić撰写

我很高兴代表社区宣布Spring Session BOM的发布Bean-M1和Apple-SR4 。Spring Boot用户将很高兴得知最近发布了这些发行版2.1.0.M1和2.0.4.RELEASE分别发布了Spring Boot。

春季会议Bean-M1

的Bean-M1是基于Spring Session的第一个里程碑版本2.1.0.M1 。

下表概述了所有包含的模块及其各自的版本：

模组 版

春季会议核心

2.1.0.M1

春季会议数据GemFire

2.0.3.RELEASE

春季会议数据Geode

2.0.3.RELEASE

春季会议数据MongoDB

2.0.2.RELEASE

春季会议数据Redis

2.1.0.M1

春季会议Hazelcast

2.1.0.M1

Spring Session JDBC

2.1.0.M1

春季会议2.1.0.M1

的2.1.0.M1是第一个里程碑版本2.1.x生命周期。此版本的重点是对Same-Site Cookie的支持，该机制是帮助开发人员防止Cross-Site Request Forgery的另一种机制，并且对HttpSessionBindingListener 。该版本还包括通常的依赖项升级，包括使用Spring Framework。 5.1.0.RC1作为基准。您可以在changelog中找到该发行版的完整详细信息。

使用物料清单

使用Maven：

<dependencyManagement>
	<dependencies>
		<dependency>
			<groupId>org.springframework.session</groupId>
			<artifactId>spring-session-bom</artifactId>
			<version>Bean-M1</version>
			<type>pom</type>
			<scope>import</scope>
		</dependency>
	</dependencies>
</dependencyManagement>
<dependencies>
	<dependency>
		<groupId>org.springframework.session</groupId>
		<artifactId>spring-session-data-redis</artifactId>
	</dependency>
	...
</dependencies>

使用Gradle：

plugins {
	id 'io.spring.dependency-management' version '1.0.6.RELEASE'
}

dependencyManagement {
	imports {
		mavenBom 'org.springframework.session:spring-session-bom:Bean-M1'
	}
}

dependencies {
	compile 'org.springframework.session:spring-session-data-redis'
	...
}

Spring Framework 5.1 RC2现在可用

发布
于尔根·霍勒（Juergen Hoeller）
八月17，2018

尊敬的Spring社区，

我很高兴地宣布，我们的里程碑存储库中现已提供第二个Spring Framework 5.1发行候选版。

Spring Framework 5.1 RC2附带了相关性更新，例如最新的Reactor Californium M2，RxJava 2.2和JUnit 5.3 RC1。它解决了有关JDK 11和GraalVM的几个已报告的问题（跟踪其发布候选对象），并在我们的5.1主题中提供了进一步的改进，特别是在日志输出和启动性能方面。

这是下周即将发布的Spring Boot 2.1 M2的基础。我们的目标仍然是9月初推出Spring Framework 5.1 RC3，采用Reactor Californium RC1并结束我们的JDK 11工作，并于9月下旬升级到5.1 GA。
