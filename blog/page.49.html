Spring Boot 1.x EOL 2019年8月1日

新闻与活动
菲尔·韦伯
七月30，2018

所有美好的事物都必须结束，对于Spring Boot的1.x系列来说，这意味着我们需要从今天起十二个月（即2019年8月1日）停止维护。

现有的Spring Boot 1.x用户应进行相应的计划，以确保在该日期之前已升级到最新的2.x版本。另请注意，Spring Boot 2.0及更高版本需要Java 8+，因此，如果您运行的是Java 6或7，则还应计划升级JDK。

Spring Security 5.1.0。M2发布

发布
罗伯·温奇
七月30，2018

我很高兴代表社区宣布Spring Security 5.1.0的发布。M2。此版本附带100张以上的门票 。

一如既往，我们期待收到您的反馈 ！您可以在下面找到要点：

OAuth2

OAuth2资源服务器

添加了对OAuth2资源服务器的基本支持。请参阅oauth2resourceserver

授权码流程

用户现在可以使用OAuth 2.0授权码授予来获得访问令牌。请参阅authcodegrant示例。

WebClient和OAuth2支持

现在内置了对OAuth2和WebClient支持的支持。支持允许：

  • 将访问令牌添加到请求中

  • 访问令牌过期时自动刷新

  • 解析要使用的访问令牌

例如，在Servlet环境中，您可以这样配置Bean：

@Bean
WebClient webClient(OAuth2AuthorizedClientRepository repository) {
    ServletOAuth2AuthorizedClientExchangeFilterFunction filter =
        new ServletOAuth2AuthorizedClientExchangeFilterFunction(repository);
    return WebClient.builder()
        .filter(new OAuth2AuthorizedClientExchangeFilterFunction())
        .apply(filter.oauth2Configuration())
        .build();
 }

现在，您可以通过多种不同方式添加OAuth令牌。如果需要，可以使用Spring MVC支持来解析OAuth2AuthorizedClient 。如果授权服务器返回了刷新令牌，并且访问令牌即将到期，Spring Security将透明地更新访问令牌并提交更新后的访问令牌。

@GetMapping("/users")
Mono users(@RegisteredOAuth2AuthorizedClient("client-id")
        OAuth2AuthorizedClient authorizedClient) {
    return this.webClient.get()
        .uri("https://api.example.com/user")
        .attributes(oauth2AuthorizedClient(authorizedClient))
        .retrieve()
        .bodyToMono(String.class);
}

您也可以通过WebClient解析访问令牌。前例：

Mono users() {
    return this.webClient.get()
        .uri("https://api.example.com/user")
        .attributes(clientRegistrationId("client-id"))
        .retrieve()
        .bodyToMono(String.class);
}

如果您使用OAuth2登录或OIDC进行了身份验证，则可以在没有用户交互的情况下应用默认访问令牌。

Mono users() {
    // if Authenticated with OIDC
    // OAuth2 Log In use the access token associated to log in
    return this.webClient.get()
        .uri("https://api.example.com/user")
        .retrieve()
        .bodyToMono(String.class);
}

Spring Framework 5.1进入RC1

发布
于尔根·霍勒（Juergen Hoeller）
七月26，2018

尊敬的Spring社区，

我很高兴地宣布，现在可以从我们的里程碑资料库中获得功能完整的Spring Framework 5.1发行候选版！在我们的GitHub Wiki上找到新功能和改进的全面列表以及相应的升级说明 。

Spring Framework 5.1需要JDK 8或更高版本，并且特别支持JDK 11作为下一个长期支持版本。对于任何针对JDK 11的应用程序，我们强烈建议升级到5.1，以在类路径和模块路径上提供无警告的体验。除此之外，对GraalVM兼容性的初步改进使其加入了此版本，从而自动适应了核心Spring设施中本机映像的运行时约束。
