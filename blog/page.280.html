即时结果-使Java有趣

Spring Roo是一种轻量级的开发人员工具，可快速轻松地交付即时结果。最重要的是，您可以100％用Java编写代码，并可以重用所有现有的Java知识，技能和经验。您会喜欢它的，而且也有很多乐趣！了解TomTom如何使用Spring Roo 将开发时间减少一半 。

下载Josh Long和Steve Mayzak的书，名为Roo入门 。您也可以按需印刷书籍或从O'Reilly网站上以其他格式购买。

Ken Rimple和Srini Penchikala还编写了权威的Roo参考书，名为Spring Roo in Action，可以从Manning Publications购买。

令人兴奋的功能-GWT。 Solr。云。柔性...

带有JPA的MVC应用程序对Roo来说是一件轻而易举的事，但是它也使您轻松进入下一代技术。我们已经支持一些很酷的功能，例如用于高级下一代用户界面的Google Web Toolkit（GWT），用于搜索服务器集成的Solr和诸如Google App Engine的云部署。另外，我们还具有增量数据库自省，Flex，Spring Integration等功能，并且在开发中更加积极。

SpringSource和Google携手合作，将Spring Roo的开发速度与Google Web Toolkit（GWT）的强大功能相结合，以构建丰富的浏览器应用程序。试用Spring Roo 1.1现在提供的令人兴奋的GWT和Google App Engine支持。

零风险-生产力永不妥协

使用Roo，您仍然可以使用Java编写所有程序。它的创新方法完全在编译时运行，并且与您的IDE完全兼容。另外，由于没有运行时部分，因此采用也被简化并且风险低。从第一天开始，您所有的Java开发人员都将感到舒适并富有成效。Roo确实代表了生产力，而毫不妥协。

Spring Roo是面向Java开发人员的下一代快速应用程序开发工具。使用Roo，您可以在几分钟内轻松构建完整的Java应用程序。它与其他生产力工具的不同之处在于：

生产力不折不扣

Roo的创新方法没有任何妥协：

  • 用Java编写程序！
  • 全面的IDE支持（具有代码辅助，重构和调试等功能）
  • 没有运行时部分，这意味着具有批准要求的企业更容易采用
  • 出色的运行时性能，因为没有运行时内存使用，部署占用空间，控制流调用时间等
  • 由于Roo完全在开发时运行（就像您的IDE一样），因此降低了影响项目的错误风险。
  • 无需锁定Roo，只需几分钟即可将其从项目中删除
  • 借助基于OSGi的附加扩展性，可以轻松扩展新功能
  • 我们可以继续，但是您明白了：Roo不会让您妥协（曾经）:-)

怎么运行的

简而言之，Roo是轻量级的控制台外壳，可在开发项目时加载。尽管Roo Shell可以通过一个快速命令来完成耗时的操作，但大多数情况下，您将忽略Roo并继续在IDE或文本编辑器中开发项目。

 

当您以常规方式编辑代码时，Roo会密切注意您的项目文件，并根据您的操作自动对其进行修改。根据您正在运行的Roo插件，Roo可以帮助您处理不同类型的文件。例如，Roo的JPA插件意味着您可以编写一个非常简单的Hello.java文件，如下所示：

@RooJpaActiveRecord public class Hello {
	private String world;
}

您可能想知道只有一个私有字段的Java类有什么好处。好吧，不多。但是Roo通过使用编译器的“ mixins”功能在编译时（而不是运行时）将额外的东西添加到.class文件中来解决此问题。这意味着Hello.class实际上包含大量有用的成员，您不必自己编写，维护，调试和测试麻烦的成员：

public class Hello {
	private String world {..}
	public String getWorld() {..}
	public void setWorld(String world) {..}
	public Long getId() {..}
	public void setId(Long id) {..}
	public Integer getVersion() {..}
	public void setVersion(Integer version) {..}
	public String toString() {..}
	public void persist() {..}
	public void remove() {..}
	public void flush() {..}
	public static Long countHellos() {..}
	public static Hello findHello(Long id) {..}
	public static List findAllHellos() {..}
        public static List findHelloEntries(int start, int finish) {..} // there are even more methods
}

Roo方法的优点是，您的Hello.java完全没有杂音和精力，但是Hello.class非常有用，并且与您的IDE（代码辅助，调试等）和运行时环境（没有Roo依赖项或其他）完全兼容妥协）。另外，因为它只是一个普通的Java类，所以它在运行时也表现出色，并且在运行时不会消耗额外的内存。Roo完全了解往返，并会在您开发实体时维护mixins。例如，如果添加另一个字段，Roo将立即生成getter / setter并更新toString（）方法。如果您添加@Id字段，Roo将停止产生其标识符字段，而您的标识符字段将具有优先权。如果您自己编写toString（）方法，则将使用您的方法，而Roo将消失。实际上，如果您注释掉toString（）方法，Roo将立即返回。这一切都非常容易（也很有趣！）。

上面没有立即显而易见的更微妙的因素是@ Roo *注释是“源保留”。这意味着它们不会出现在.class文件中，反映出Roo的“零锁定”和“无运行时”策略。另外，通过将Roo生成的mixin源代码的内容直接复制到您的.java文件中，您可以轻松摆脱Roo。实际上，Eclipse中甚至有一个重构命令，它将在几秒钟内为您完成此操作！哦，如果您不想编写Hello.java源文件，则可以只使用两个支持Roo TAB的命令来完成它：“ entity --class Hello”，“ field String world”。就像我们说的那样，这一切都非常容易（也很有趣！）。

Roo的方法为您提供了最好的免费锁定，高性能生产力工具。它只是工作。试试看 ，自己看看。或访问我们的参考指南，以了解更多有关为何选择Roo作为生产力工具的信息。


尝试Spring Roo

开始使用Roo非常容易。只需按照此页面上的步骤操作，您将在几分钟内拥有一个正在运行的环境。

下载

Roo要求您下载并安装：

  •  Java 6 JDK
  •  阿帕奇Maven

要获得Roo，您可以下载其独立的命令行Shell版本，也可以使用SpringSource Tool Suite内置的版本。如果您愿意，也可以下载两者并一起使用：

  •  Spring Roo
  •  SpringSource工具套件

安装

如果您下载了独立的Spring Roo，只需将其解压缩到您选择的位置即可。然后：

  • Windows用户：将％ROO_HOME％\ bin添加到您的路径
  • * nix用户：创建到$ ROO_HOME / bin / roo.sh的符号链接（例如sudo ln -s〜/ spring-roo-1.xx / bin / roo.sh / usr / bin / roo）

 

看视频

如果您想观看安装Roo的过程有多么容易， 该视频将向您展示如何使用。或者，完成以下步骤，然后自己尝试。

第一个项目-Roo Standalone

让我们建立一个项目。在以下步骤中，我们将省略控制台输出。我们也强烈建议您在每个步骤之后键入“提示”，以帮助您学习如何使用Shell。请注意，您需要Roo 1.2.0或更高版本才能执行以下命令：

mkdir hello
cd hello
roo
roo> hint
roo> project --topLevelPackage com.foo
roo> jpa setup --provider HIBERNATE --database HYPERSONIC_IN_MEMORY         
roo> entity jpa --class ~.Timer --testAutomatically
roo> field string --fieldName message --notNull
roo> hint web mvc
roo> web mvc setup
roo> web mvc all --package ~.web
roo> selenium test --controller ~.web.TimerController
roo> web gwt setup
roo> web gwt all --proxyPackage ~.client.proxy --requestPackage ~.client.request          
roo> perform tests
roo> quit

哇！现在，您已经拥有一个Web应用程序，其中包含JUnit测试，Selenium测试，MVC前端，Google Web Toolkit前端。只需使用“ mvn gwt：run”来玩GWT客户端，或者使用“ mvn tomcat：run”来启动Tomcat MVC前端并使用http://localhost:8080/foo / timers进行查看。

接下来，您可能想要键入“ mvn eclipse：eclipse”，然后使用Eclipse的“常规”>“将现有项目导入工作区”功能。保持Roo Shell运行，并随时在Eclipse中进行编辑。您会看到Roo在运行时会维护文件。如果您在不运行Roo的情况下进行了任何更改，请放心-只需加载Roo，它就会赶上您所做的更改。

第一个项目–通过STS

如果安装了SpringSource Tool Suite，则只需使用File> New> Roo Project。然后输入“ Foo”的“项目名称”和“ com.foo”的“包”。单击“下一步”和“完成”后，您将在STS的底部看到Roo shell加载。然后输入与在“第一个项目（Roo独立）”部分中看到的命令相同的命令，您可以通过WTP轻松运行该应用程序。

Roo社区由使用Roo的数千人组成，从初学者Java开发人员到JavaOne摇滚明星-以及介于两者之间的每个人！Roo社区具有包容性，友好性，并热忱欢迎您的参与。我们不仅会帮助您学习Roo，而且我们也非常乐意为您提供使用技巧，路线图信息和体系结构建议。本页详细介绍了参与Roo社区的方式。我们期待您的回音！

如果您发现了错误或有新功能，请将其记录在我们的问题跟踪器中。我们的问题跟踪器还可用于查找其他人提交的功能，对需求进行评论并为您认为最有用的功能投票。这是我们优先处理增强请求的常用方法。

博客

Roo核心工程师在SpringSource Team Blog上定期发布有关Roo的博客 。您还可以通过遵循Roo类别直接访问与Roo相关的文章。

写代码

如果您想深入了解Roo代码，则第一步是克隆我们的Git存储库并遵循根目录中的readme.txt。我们使用Maven来构建Roo，因此构建自己的发行版非常容易（因为您可能已经安装了Maven才能使用Roo！）。您需要的Git命令是：

git clone git://github.com/SpringSource/spring-roo.git

为Roo编写代码的一种简单方法是开发附加组件。由于Roo使用基于OSGi的加载项体系结构，因此您可以轻松构建其他人可以找到并安装到其Roo Shell中的加载项。

如果您想为Roo本身做出贡献（而不是编写外部附加组件），也非常欢迎您这样做。浏览问题跟踪器或在社区论坛上发布消息，概述您感兴趣的工作，我们将很乐意为您提供帮助。

奥利弗·德罗博姆（Oliver Drotbohm）
2011年9月2日

亲爱的春季社区，

我很高兴宣布Spring Data MongoDB 1.0.0 M4的可用性。以下是添加的最重要的功能：

  • 支持MongoTemplate中的map-reduce操作
  • 提供了使用外部查询由存储库执行的方法
  • 支持MongoTemplate和存储库上的geoNear查询
  • MongoTemplate上的新DocumentCallbackHandler接口
  • 现在可以在SimpleDbFactory上配置数据库范围的WriteConcern
  • 可在MongoFactoryBean上配置的WriteConcern
  • 添加了QuerydslRepositorySupport基类，以简化基于Querydsl的存储库的实现
  • 可配置的TypeMapper接口，用于控制如何在Mongo文档中写入和检索类型信息

春天的一周：2011年8月30日

工程
乔什·朗（Josh Long）
2011年8月31日

欢迎使用另一版
“春季的这个星期”有很多事情要做，所以我们将继续。快速说明：如果您在阳光明媚的拉斯维加斯参加VMworld 2011，请转到Cloud Application Platform展位，打个招呼。



  1. CloudFoundry的一周！一周的时间是Micro Cloud Foundry的发布和可用性，这是可免费下载的“ PaaS-on-a-stick”。
    Micro Cloud Foundry是流行的开放源代码平台即服务的完整本地版本，使开发人员可以在Mac或PC上运行功能全面的云。使用Micro Cloud Foundry，开发人员可以在本地构建端到端的云应用程序，而无须配置中间件的麻烦，同时又可以保留部署位置的选择和扩展其应用程序的能力，而无需更改代码行。

    要了解有关Micro Cloud Foundry的更多信息，请查看以下三篇博客文章，向Spring开发人员和Grails开发人员介绍Micro Cloud Foundry，并在SpringSource Tool Suite中介绍对Micro CloudFoundry的支持 。
  2. Thomas Risberg blogged today about using PostgreSQL on Cloud Foundry. The recently announced PostgreSQL support makes CloudFoundry the natural place to deploy your enterprise applications: between MySQL and PostgreSQL there's very likely few speed or feature that you can't match on CloudFoundry. Try it out today!
  3. Mark Thomas had an article on Tomcat Expert last week about Tomcat 8 and Java 7, answering the question: will they work together?
  4. Roy Clarkson, lead of the Spring Android project, recently blogged about the various dependency injection options for Android and how to write cleaner code. Check it out for a look at the modern Android developers toolkit.
  5. Spring provides many ways to support configuration. Spring 2.5 enhanced the options available with stereotypes, which let you specify a marker annotation as a way to define an injection site. This very nice blog post first introduces some of the earlier ways to make maximum use of the type system with @Autowired, then introduces the stereotype model, cleanly and consistently. I wholly recommend it for a great look at an easy way to achieve cleaner code.
  6. In addition to the MicroCloud, last week saw the announcement of the availability of PHP and Python support through community CloudFoundry providers. This is an ideal situation for developers on those platforms and if you've always wanted first-class support for Python and PHP on a cloud, this release is for you! And, of course, if you're a Python developer, you should no doubt be looking at Spring Python, the Pythonic port of the Spring framework.
  7. Last week, Alan Stewart, lead of the Spring Roo project, announced the availability of the O'REILLY book, Getting Started with Spring Roo, that my co-author Steve Mayzak and I put together. O'REILLY worked closely with us to help make the book concise, and useful. O'REILLY have graciously agreed to make the book available for free download here, as a .PDF. Readers who want an alternative e-reader-specific format or want to get a print copy of the book on-demand may of course pursue those options at the O'Reilly page for the book. The book features an introduction to Spring Roo as well as an introduction to two of the many powerful, new addons being developed for Spring Roo: the Spring Data Neo4j addon, and the Vaadin web framework addon.
  8. Virgo 3.0 and Gemini Web 2.0 have been released. The theme of Virgo 3.0 is better integration with EclipseRT technologies. To that end, we have created a Jetty variant of the Virgo web server and have switched from Felix to Equinox implementations of some OSGi services. Gemini Web and the Tomcat variant of the Virgo web server have been upgraded to Tomcat 7 and Servlet 3.0.
  9. Spring AMQP 1.0 GA has been released! I don't know about you guys, but I've been waiting for this one for a looong time. RabbitMQ is one of the most powerful messaging options available to enterprise Java developers today in traditional Spring applications as well as in the cloud. Spring AMQP provides the idiomatic Spring support for RabbitMQ, including support for a PlatformTransactionManager abstraction, and a message listener container abstraction similar to the support available for JMS. The GA release of the Spring AMQP integration clears the way for the inclusion of the Spring Integration AMQP support in the imminent Spring Integration 2.1. Powerful stuff! Developers looking to get started with messaging in Spring might take a look at this GreenBeans post ("Getting Started with Enterprise Messaging with Spring"), which may be a bit out of date but provides a clear, concise introduction to the relevant concepts and options for both AMQP and JMS integration with Spring.
  10. Spring GemFire 1.1.0.M2 has been released! This new release includes dedicated support for continuous query (Message Driven POJOs for GemFire), extensive client cache support, and namespace support for region expiration. Very cool release, and users are highly encouraged to check it out. The continuous query support underlies the new Spring Integration Gemfire adapter that's expected to be in Spring Integration 2.1, as well: very powerful stuff!
  11. Thomas Risberg今天写了关于在Cloud Foundry上使用PostgreSQL的博客。最近宣布的PostgreSQL支持使CloudFoundry成为部署企业应用程序的自然之地：在MySQL和PostgreSQL之间，极少数速度或功能是CloudFoundry所无法比拟的。今天尝试一下！
  12. 马克·托马斯（Mark Thomas）上周在Tomcat Expert上发表了一篇有关Tomcat 8和Java 7的文章 ，回答了这个问题： 它们可以一起工作吗？
  13. Spring Android项目负责人Roy Clarkson 最近在博客上介绍了Android的各种依赖项注入选项以及如何编写更简洁的代码。请查看一下现代Android开发人员工具包。
  14. Spring提供了许多支持配置的方法。Spring 2.5增强了原型的可用选项，可让您指定标记注释，以定义注射部位。这篇非常不错的博客文章首先介绍了一些早期的方法，可以通过@Autowired最大限度地利用类型系统，然后干净，一致地介绍@Autowired 型模型。我完全推荐使用它，以更好地了解实现简洁代码的简便方法。
  15. 除了MicroCloud，上周还宣布了通过社区CloudFoundry提供程序提供PHP和Python支持的消息。对于那些平台上的开发人员来说，这是理想的情况，如果您一直想在云上对Python和PHP提供一流的支持，那么此版本适合您！而且，当然，如果您是Python开发人员，那么毫无疑问，您应该看看Spring Python （Spring框架的Pythonic端口）。
  16. 上周，Spring Roo项目的负责人艾伦·斯图尔特（Alan Stewart） 宣布推出了我的合著者史蒂夫·梅扎克和我合着的O'REILLY书，《 Spring Roo入门》 。O'REILLY与我们紧密合作，以使本书简洁而实用。O'REILLY亲切地同意将这本书作为本书免费下载 .PDF希望使用特定于电子阅读器的替代格式或希望按需获得该书的印刷版的读者当然可以在O'Reilly页面上寻求这些选项。该书介绍了Spring Roo，并介绍了为Spring Roo开发的许多功能强大的新插件中的两个：Spring Data Neo4j插件和Vaadin Web框架插件。
  17. 处女座3.0和双子座Web 2.0已发布。Virgo 3.0的主题是与EclipseRT技术更好地集成。为此，我们创建了Virgo Web服务器的Jetty变体，并已从Felix转换为某些OSGi服务的Equinox实现。Gemini Web和Virgo Web服务器的Tomcat变体已升级到Tomcat 7和Servlet 3.0。
  18. Spring AMQP 1.0 GA已发布！ 我不了解你们，但我已经等了很久了 。RabbitMQ是当今的企业Java开发人员在传统的Spring应用程序以及云中可用的最强大的消息传递选项之一。 Spring AMQP为RabbitMQ提供了惯用的Spring支持，包括对PlatformTransactionManager抽象的支持以及类似于JMS可用的消息侦听器容器抽象。 GA AM版本的Spring AMQP集成为即将到来的Spring Integration 2.1中包含Spring Integration AMQP支持扫清了道路。强大的东西！希望在Spring中开始使用消息传递的开发人员可以看一下GreenBeans的这篇文章（“使用Spring进行企业消息传递入门”） ，这可能有点过时了，但是提供了有关相关概念和选项的清晰简洁的介绍。用于将AMQP和JMS与Spring集成。
  19. 春季GemFire 1.1.0。M2已发布！这个新版本包括对连续查询的专门支持（针对GemFire的消息驱动POJO），广泛的客户端缓存支持以及对区域到期的名称空间支持。非常酷的发行版，强烈建议用户检查出来。连续查询支持是新的Spring Integration Gemfire适配器的基础，该适配器也有望在Spring Integration 2.1中使用：功能非常强大！
