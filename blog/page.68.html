具有Spring Security的下一代OAuth 2.0支持

工程
乔·格兰贾
一月30，2018

当前状态

Spring项目组合中的OAuth 2.0支持的当前状态分布在Spring Security OAuth ， Spring Cloud Security ， Spring Boot 1.5.x和Spring Security 5中引入的新支持之间。作为OAuth的用户，您可能会问：“我使用哪个项目？为什么Spring Security 5引入了新的支持？”

简而言之，有必要将OAuth 2.0支持统一到一个项目中，以便为用户提供明确的选择并避免任何潜在的混乱。此外，OAuth 2.0支持需要更进一步，并为OAuth 2.0和OpenID Connect 1.0提供更广泛的支持。另外，根据社区反馈，需要重新编写文档，以便易于使用并提高开发人员的生产力。基于所有这些因素，我们决定重新开始并在Spring Security 5中构建下一代OAuth 2.0支持。

Spring Cloud Data Flow 1.3.0。RC1发布

发布
马克·波拉克
一月24，2018

我们很高兴宣布1.3.0。RC1发布了Spring Cloud Data Flow及其相关的项目生态系统。

遵循本地服务器 ， Cloud Foundry和Kubernetes的入门指南。

发布要点

船长功能切换

为了简化选择使用Skipper部署流的总体体验，功能切换为您提供了在船长模式和先前的“经典”模式之间切换的功能。命令行管理程序和服务器中均使用功能切换。默认值是使用“经典”非船长模式。要启用跳过模式，请传入以下Spring Boot选项。

java -jar spring-cloud-dataflow-server-local-1.3.0.RC1.jar  \
  --spring.cloud.dataflow.features.skipper-enabled=true

java -jar spring-cloud-dataflow-shell-1.3.0.RC1.jar --dataflow.mode=skipper

这也使我们可以删除stream skipper 1.3 M3中引入的命令，并将斗篷行为折叠为stream命令。

Spring Cloud Skipper 1.0 RC1发布

发布
马克·波拉克
一月19，2018

我很高兴代表团队宣布Spring Cloud Skipper 1.0 RC1的发布。

Skipper是一款轻巧的工具，可让您发现Spring Boot应用程序并在多个Cloud Platform上管理其生命周期。您可以独立使用Skipper或将其与Continuous Integration管道集成在一起，以帮助实施Continuous Deployment的实践。

1.0 RC1版本修复了多个错误，并引入了一些新功能。

  • OAuth安全支持。
  • 发布使用Spring StateMachine项目管理的安装，升级和回滚工作流。
  • REST API的改进。
  • 使用Flyway管理的数据库架构。
  • 软件包删除，检查活动版本。
  • 使用可选的软件包删除来释放删除。
  • Shell命令遵循一致的格式，例如平台列表，发布状态。
  • 在包模板中添加对ResourceMetadata URI的支持。
  • 支持交互式和非交互式外壳模式。
  • 从java.util改进的转换。属性，到外壳中的YAML。
