Spring for Apache Hadoop 1.0.1。RC1发布-添加了Hadoop 2.0和Pivotal HD 1.0支持

发布
托马斯·里斯伯格
2013年7月11日

我们很高兴宣布推出新的候选版本1.0.1。Spring for Apache Hadoop项目的RC1。此发行候选版本引入了对Apache Hadoop 1.1.2，Hortonworks HDP 1.2，Apache Hadoop 2.0.3-alpha的支持以及对Pivotal HD 1.0的支持。除了已经支持的Apache Hadoop 1.0.4和Cloudera CDH 3和CDH 4版本之外，对这些版本的支持。这使我们支持至少7种不同的Hadoop版本。

为了使使用这些不同版本的项目构建更容易，我们将介绍Maven工件和POM的特定形式以及它们各自的传递依赖项。为此1.0.1。RC1版本中，我们引入了三个附加的工件版本– 1.0.1。RC1-cdh4（Cloudera CDH 4），1.0.1。RC1-pdh1（Pivotal HD 1.0）和1.0.1。RC1-hadoop11（Apache Hadoop 1.1.2和Hortonworks HDP 1.2）。通过使用这些版本作为依赖项，您将获取它们各自的可传递依赖项，而不是使用纯1.0.1时将获得的来自Apache Hadoop 1.0.4发行版的依赖项。RC1版本。

网络研讨会：抵抗不是徒劳的：如何谈论春天并影响人们

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2013年7月8日

当然，Spring Framework 4.0中出现的新功能非常令人兴奋，但是对于那些仍在向初级开发人员解释依赖注入的人来说，又该如何呢？尽管Spock，Geb和spring-test-mvc彻底改变了我们测试应用程序的能力，但高级开发人员又如何向其经理证明单元测试的价值呢？强大的技术领导能力可以克服组织惯性，而组织惯性通常会阻止您的团队采用Spring技术。通过从Liberty University的软件开发部门从ColdFusion中的“剪切和粘贴编码”迁移到Spring Framework上的企业级应用程序的过程中汲取的教训，提高您的领导技能。学习计划有效的技术采用策略，以避免“新技术过载”，并在技术改进的速度与继续生产的必要性之间取得平衡。与经理，初级开发人员和生产系统管理员的关系将非常重要。对非技术经理有更好的了解，并探索提供正确决策所需的条件的策略。检查为初级开发人员建立指导计划的方法，这些方法包括但不限于SpringSource University的培训和认证，以便您可以花费更少的时间进行教学，而花费更多的时间进行编码。
