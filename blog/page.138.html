Core Spring赛道

我们每天编写Java代码，然后将其编译为字节码并在JVM上运行。这篇演讲解释了JVM内部字节码发生了什么。首先概述了即时（JIT）编译器。本概述讨论了JIT优化，例如方法内联和循环展开。然后，演讲会检查JIT日志，以解释在Spring Boot应用程序上执行的优化。在讲座结束之前，您应该能够检查JIT编译器对您自己的代码的影响。

SpringOne2GX 2015重放：使用事件源和CQRS构建微服务

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2016年3月22日

记录在SpringOne2GX 2015中。
演讲者：Michael Ploed
云原生跟踪
幻灯片： http : //www.slideshare.net/SpringCentral/building-microservices-with-event-sourcing-and-cqrs-53211567
CQRS和事件源是两种架构模式，它们随着云架构和No-SQL数据库的兴起而变得非常流行。与“经典企业方法”相比，两者可以很好地协同工作，并且在数据的处理和存储上采取不同的态度。CQRS将您的应用程序分为读取部分和写入部分，而通过事件源，数据的状态由事件序列表示。本讲座介绍了这两种模式，并展示了如何利用Spring生态系统来实现利用CQRS和事件源的应用程序。演讲将有幻灯片和现场演示。

STS中的Spring Boot仪表板-第4部分：使用Cloud Foundry清单文件

工程
马丁·利珀特
2016年3月22日

欢迎回到Spring社区，

Spring Tool Suite的最新版本3.7.3在Spring Boot Dashboard周围引入了许多新功能。因此，我们继续从去年开始的博客文章系列，向您介绍了在IDE中使用基于Spring Boot的微服务项目的新方法（您可以在底部找到指向前面各部分的链接）。

Cloud Foundry清单文件

在本系列的这一新部分中，我们将仔细研究Cloud Foundry清单文件。它们是Cloud Foundry概念，用作为Cloud Foundry上的应用程序定义配置的快捷方式。不必在执行“ cf push”时将所有参数和配置传递给命令行，而是可以将所有配置数据放入YAML文件中，然后将其传递给push命令。可以在此处找到有关Cloud Foundry Manifest文件的更多详细信息。

Spring Data Hopper的第一个候选发布

发布
奥利弗·德罗博姆
2016年3月18日

我很高兴代表Spring Data团队宣布Hopper发布系列的第一个也是最后一个发布候选。该版本附带一些重要的更改和更新：

  • 将Spring Framework基准升级到4.2（尤其是4.2.5）。
  • 在Spring Data JPA和MongoDB中支持示例查询。
  • 支持Spring Data Redis的对象映射和存储库。
  • 改进了对组合注释的支持。
  • 在Spring Data REST中支持查找类型和嵌套关联
  • 升级到Elasticsearch 2.2，从而将Spring Data Elasticsearch的版本号更改为2.0
