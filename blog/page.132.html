Spring AMQP 1.6.0发行候选版（和1.5.6）可用

发布
加里·罗素
2016年5月6日

我们很高兴地宣布Spring AMQP 1.6发行候选版（1.6.0。RC1）现在在春季里程碑回购中可用。

1.5.6维护版本还提供了一些错误修复 。

以下是1.6.0版本内容的摘要，有关更多详细信息，请参考参考文档中的新增功能以及此版本已关闭的JIRA问题 。

  • 一个新罐子spring-rabbit-test包含测试工具以帮助进行测试@RabbitListener方法;参见测试章节 。

  • 多@RabbitListener方法的注释（使用Java 8时）和@RabbitListeners注释（对于Java 8之前的版本），每个注释都允许同一方法成为多个侦听器容器的侦听器方法。

  • 完全支持延迟消息交换 RabbitMQ插件。

  • 一个AsyncRabbitTemplate返回ListenableFuture用于请求/回复消息。

  • 发布选项ApplicationEvents当侦听器容器空闲时。

  • 缓存连接工厂现在公开缓存统计信息

  • @RabbitListener现在，方法将类型信息传递给消息转换器以获取入站消息。例如，这意味着Jackson2JsonMessageConverter当收到标头中没有类型信息的消息时（例如从不是Spring AMQP应用程序的发件人处），不再需要自定义类映射器。相反， @RabbitListener转换中使用方法参数类型。除此之外Method和bean也可以通过消息属性提供给自定义转换器。

Spring Data Geode 1.0.0。APACHE-GEODE-INCUBATING-M2发布

工程
约翰·布鲁姆
2016年4月29日

我很高兴地宣布发布Spring Data Geode 1.0.0。APACHE-GEODE-INCUBATING-M2，为最近宣布的 Apache Geode 1.0.0-incubating提供支持。M2发布。

Spring Data Geode是Spring Data GemFire的特定版本，支持Apache Geode，而不是Pivotal GemFire 。

Spring Data GemFire和Spring Data Geode都基于GitHub中相同的源存储库 ，但是Apache Geode支持位于apache-geode分支中。因此，您不会找到专门用于Apache Geode的单独的Spring Data项目 ，但这没有什么意义，因为您对Pivotal GemFire与Spring Data GemFire结合使用的现有知识将立即有用并可以转移到Apache Geode和Spring Data Geode中 。
