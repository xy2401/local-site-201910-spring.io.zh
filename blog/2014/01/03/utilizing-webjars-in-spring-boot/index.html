在Spring Boot中使用WebJars

工程
罗伊·克拉克森
一月03，2014

欢迎来到2014！2013年对于春季来说是令人兴奋的一年，我们期待着另一个伟大的一年。我们在最近的几篇文章中都专注于客户端开发，其中包括我们发布了一些新的客户端入门指南 。在上一篇文章中，我还回顾了使用Spring Boot 提供静态Web内容有多么容易。

在本文中，当我们探索另一种内置功能时，我将继续关于使用Spring Boot进行客户端开发的讨论。我的上一篇文章包括以下摘录自的源代码 WebMvcAutoConfiguration它说明了如何将静态资源自动添加到Spring MVC ResourceHandlerRegistry使用Spring Boot时。

@Override
public void addResourceHandlers(ResourceHandlerRegistry registry) {
    if (!registry.hasMappingForPattern("/webjars/**")) {
        registry.addResourceHandler("/webjars/**").addResourceLocations(
                "classpath:/META-INF/resources/webjars/");
    }
    if (!registry.hasMappingForPattern("/**")) {
        registry.addResourceHandler("/**").addResourceLocations(
                RESOURCE_LOCATIONS);
    }
}

我在上一篇文章中没有讨论的代码的一个方面是“ webjars”资源处理程序。您可能会想，“很好，但是什么是webjar？”。

作为Java开发人员，您可能熟悉JAR（Java存档）文件格式，该文件格式用于将许多类文件及其关联的元数据打包到一个文件中。WebJars只是采用JAR的概念并将其应用于客户端库或资源。例如，jQuery库可以打包为JAR，并可供您的Spring MVC应用程序使用。使用WebJars有很多好处，包括对Java构建工具（例如Gradle和Maven）的支持。通过在构建工具中使用依赖管理，可以自动为您处理传递依赖。

也许您想向网页添加jQuery动画。以“ 通过Spring MVC提供Web内容”指南为起点，修改build.gradle文件以包含jQuery依赖项。可通过Maven Central获得许多WebJars，其GroupID为org.webjars 。完整列表可从webjars.org获得 。

buildscript {
    repositories {
        maven { url "http://repo.spring.io/libs-milestone" }
        mavenLocal()
    }
    dependencies {
        classpath("org.springframework.boot:spring-boot-gradle-plugin:0.5.0.M6")
    }
}

apply plugin: 'java'
apply plugin: 'eclipse'
apply plugin: 'idea'
apply plugin: 'spring-boot'

jar {
    baseName = 'gs-serving-web-content'
    version =  '0.1.0'
}

repositories {
    mavenCentral()
    maven { url "http://repo.spring.io/libs-milestone" }
}

dependencies {
    compile("org.springframework.boot:spring-boot-starter-web:0.5.0.M6")
    compile("org.thymeleaf:thymeleaf-spring3:2.0.17")
    compile("org.webjars:jquery:2.0.3-1")
    testCompile("junit:junit:4.11")
}

task wrapper(type: Wrapper) {
    gradleVersion = '1.8'
}

添加依赖项后，可以在HTML中引用jQuery WebJar并利用它来使段落标记中的文本动画化。注意jQuery依赖项的位置！

<!DOCTYPE HTML>
<html xmlns:th="http://www.thymeleaf.org">
<head> 
    <title>Getting Started: Serving Web Content</title> 
    <meta http-equiv="Content-Type" content="text/html; charset=UTF-8" />
    <script src="webjars/jquery/2.0.3/jquery.min.js"></script>
    <script type="text/javascript">
        $(document).ready(function() {
            $('p').animate({
                fontSize: '48px'
            }, "slow");
        });
    </script>
</head>
<body>
    <p th:text="'Hello, ' + ${name} + '!'" />
</body>
</html>

JavaScript软件包管理不是一个新概念。实际上， npm和bower是两个更流行的工具，目前提供用于管理JavaScript依赖项的解决方案。Spring的《 了解JavaScript程序包管理器》指南提供了更多有关这些信息。大多数JavaScript开发人员可能熟悉npm和bower并在其项目中使用它们。相比之下，WebJars利用Maven的依赖管理模型将JavaScript库包含在项目中，从而使Java开发人员更易于访问。

Spring Boot提供了一种简单的方法来构建需要很少或不需要配置的Spring应用程序。这篇文章说明了在Spring MVC应用程序中使用WebJars是多么简单，以及WebJars如何提供一种管理JavaScript包和依赖关系的便捷方法。有关更多信息，请参见Spring Boot项目页面和入门指南目录。
