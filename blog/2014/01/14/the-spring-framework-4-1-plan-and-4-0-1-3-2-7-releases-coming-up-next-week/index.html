Spring Framework 4.1计划-下周将发布4.0.1和3.2.7版本

发布
于尔根·霍勒（Juergen Hoeller）
2014年1月14日

在上周我的Spring Framework 4网络研讨会之后，我想分享那里介绍的Spring Framework 4.1计划 。我们针对4.1制定了一些关键主题，以在2014年8月及时交付GA，并在6月发布4.1 RC：

全面的Web资源处理-资源流水线化，缓存控制优化

重新考虑了缓存支持-与JCache 1.0最终，用户要求的增强功能保持一致

JMS支持大修-与我们的消息传递模块，注释驱动的端点保持一致

性能改进-应用程序启动，SpEL表达式评估

这些主题很大程度上是由我们总体的Spring Framework 4愿景驱动的，并且非常适合此4.1早期迭代。详细信息中计划了许多特定功能，并且我确定我们将看到仍在考虑的新建议。

请注意，Spring Framework 4.1发行是受日期驱动的 ，将于8月交付。无法使用4.1的功能将被视为4.2，计划于2015年第二季度交付。

另外，快速提醒一下，我们下周将发布Spring Framework 4.0.1和3.2.7双重包装 ：这是您借此机会及时提交有关这些发行版的最新问题；我们保证会在可行的情况下考虑它们。如果您在JIRA中有较旧的问题报告，请随时通过评论引起注意，以澄清它们是否仍然与您相关。

请注意，3.2.x行已经处于维护模式，因此，在微调其对Quartz和EhCache的支持之后，3.2.7将主要修复错误和回归。也就是说，对于4.0.1，我们仍然非常灵活-特别是可以务实地重新访问4.0功能细节。

在https://jira.springsource.org/browse/SPR#selectedTab=com.atlassian.jira.plugin.system.project%3Aroadmap-panel中查看我们的JIRA版本路线图，以解决所有相关问题，并告诉我们您的情况认为…

于尔根

PS：如果您错过了上周的网络研讨会，则计划在4.0.1版本的当天（1月23日，星期四）重复一次。请继续关注官方公告！
