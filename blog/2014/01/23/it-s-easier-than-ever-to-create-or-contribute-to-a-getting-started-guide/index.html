创建或贡献《入门指南》比以往任何时候都更加容易。

工程
格雷格·特恩奎斯特
2014年1月23日

问候春天社区！

我们一直收到有关Spring 入门指南大受欢迎的反馈。

到目前为止，我们已经将这些指南迁移到了Asciidoctor 。我们还增加了指南，以包括面向客户的指南。还有更多！

使这些指南可以长期运行的链中的最后一个链接是每个指南侧栏中的导航链接。我们现在使用元数据将各种指南标记和链接在一起，同时还将它们连接到相关项目。

例如，如果您使用AngularJS访问Consuming RESTful Web Service ，则会看到一个侧边栏，其中包含：

  • 该指南顶层部分的快捷方式
  • 两个标签： rest和JavaScript ，链接使您可以搜索包含相同标签的其他指南
  • 鉴于此指南着重于将Spring MVC与AngularJS结合在一起，因此它是Spring Framework项目的链接
  • 链接到了解REST和了解JSON

侧栏现在需要零维护。

注意：如果有我喜欢的东西，那就是零维护！

如果您对我们的指南之一有所贡献，则不再需要编辑有关侧栏的单独文件。相反，每个指南的顶部都标记有标签和项目元数据。这就是在侧栏上建立导航链接所需要的全部。

这使得发布新的相关指南变得更加容易，这些指南通过一些相互链接链接在一起。再举一个例子，签出所有Spring Data guides 。

当您访问指南列表时，您可以根据标题和指南说明以及标签进行即时搜索。正如您从上方的所有链接中看到的那样，可以将搜索条件添加为书签，捕获和保存。

仅供参考：标签数据可能会发生变化，因此请不要以为它是永远的。

通过对指南的创建和管理方式进行的所有这些更新，创建新指南或编辑现有指南的过程比以往任何时候都更加简单。

如果您有新指南的想法，可以在自己的github存储库中进行研究。只需确保检查我们的模板项目以及编写准则，以了解编写，编辑和贡献的过程。

我们渴望看到您的贡献，并且还将广为传播，以便各个Spring领导也可以为其项目编写更多指南。
