Spring Boot 1.2中的“ Bootiful” Java EE支持

工程
乔什·朗（Josh Long）
2014年11月23日

在本博客中，我想研究并演示Spring Boot 1.2中的许多新功能， 这些新功能使来自Java EE或以Java EE为基础的人的生活变得更加轻松。

值得一提的是，Spring以前当然可以提供很多支持，但是现在有了Spring Boot 1.2，就这么简单！

首先，这是一个带有注释的示例程序。

package demo;

import org.glassfish.jersey.jackson.JacksonFeature;
import org.glassfish.jersey.server.ResourceConfig;
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.jms.annotation.JmsListener;
import org.springframework.jms.core.JmsTemplate;

import javax.annotation.PostConstruct;
import javax.inject.Inject;
import javax.inject.Named;
import javax.jms.JMSException;
import javax.persistence.*;
import javax.transaction.Transactional;
import javax.ws.rs.*;
import javax.ws.rs.core.MediaType;
import java.io.Serializable;
import java.util.Collection;
import java.util.logging.Logger;

@SpringBootApplication
public class Application {

    @Named
    public static class JerseyConfig extends ResourceConfig {

        public JerseyConfig() {
            this.register(GreetingEndpoint.class);
            this.register(JacksonFeature.class);
        }
    }

    @Named
    @Transactional
    public static class GreetingService {

        @Inject
        private JmsTemplate jmsTemplate;

        @PersistenceContext
        private EntityManager entityManager;

        public void createGreeting(String name, boolean fail) {
            Greeting greeting = new Greeting(name);
            this.entityManager.persist(greeting);
            this.jmsTemplate.convertAndSend("greetings", greeting);
            if (fail) {
                throw new RuntimeException("simulated error");
            }
        }

        public void createGreeting(String name) {
            this.createGreeting(name, false);
        }

        public Collection<Greeting> findAll() {
            return this.entityManager
                    .createQuery("select g from " + Greeting.class.getName() + " g", Greeting.class)
                    .getResultList();
        }

        public Greeting find(Long id) {
            return this.entityManager.find(Greeting.class, id);
        }
    }

    @Named
    @Path("/hello")
    @Produces({MediaType.APPLICATION_JSON})
    public static class GreetingEndpoint {

        @Inject
        private GreetingService greetingService;

        @POST
        public void post(@QueryParam("name") String name) {
            this.greetingService.createGreeting(name);
        }

        @GET
        @Path("/{id}")
        public Greeting get(@PathParam("id") Long id) {
            return this.greetingService.find(id);
        }
    }

    @Entity
    public static class Greeting implements Serializable {

        @Id
        @GeneratedValue
        private Long id;

        @Override
        public String toString() {
            return "Greeting{" +
                    "id=" + id +
                    ", message='" + message + '\'' +
                    '}';
        }

        private String message;

        public String getMessage() {
            return message;
        }

        public Greeting(String name) {
            this.message = "Hi, " + name + "!";
        }

        Greeting() {
        }
    }

    @Named
    public static class GreetingServiceClient {

        @Inject
        private GreetingService greetingService;

        @PostConstruct
        public void afterPropertiesSet() throws Exception {
            greetingService.createGreeting("Phil");
            greetingService.createGreeting("Dave");
            try {
                greetingService.createGreeting("Josh", true);
            } catch (RuntimeException re) {
                Logger.getLogger(Application.class.getName()).info("caught exception...");
            }
            greetingService.findAll().forEach(System.out::println);
        }
    }

    @Named
    public static class GreetingMessageProcessor {

        @JmsListener(destination = "greetings")
        public void processGreeting(Greeting greeting) throws JMSException {
            System.out.println("received message: " + greeting);
        }
    }

    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}

完整的代码清单，包括非常稀疏的application.properties和Maven build可以在线获得。

JAX-RS与泽西岛

该示例演示了Boot的新JAX-RS自动配置 （在这种情况下，使用Jersey 2.x ）。 GreetingEndpoint 。请注意使所有功能正常运行非常方便！唯一要烦的是您需要指定一个ResourceConfig子类，让Jersey知道要注册的组件。

与JTA的全球交易

它通过新的自动配置的JTA支持演示了全局事务。JTA是用于X / Open XA协议的Java API，它允许多个兼容的事务资源（例如消息队列和数据库）参与单个事务。为此，我们使用了Atomikos独立的JTA提供程序。我们也可以像使用Bitronix一样容易地使用它。如果带上合适的启动器，两者都将自动配置。在此示例中， GreetingService ，JMS和JPA工作是作为全局事务的一部分完成的。我们通过创建3个事务并在第三个事务上模拟回滚来演示这一点。您应该看到打印到控制台上的是从JDBC返回的两条记录javax.sql.DataSource嵌入式JMS接收到的数据源和两条记录javax.jms.Destination目的地。

Undertow嵌入式Web服务器

这个例子也使用了Wildfly（从红帽）应用服务器的真棒 暗潮嵌入式HTTP服务器 ，而不是（默认）的Apache Tomcat。使用Undertow和使用Jetty或Tomcat一样容易-只需排除org.springframework.boot:spring-boot-starter-tomcat并添加org.springframework.boot:spring-boot-starter-undertow ！此贡献源自第三方PR-感谢Ivan Sopov！这很棒。

什物

为了保持一致性，该示例还使用了JSR 330。JSR 330描述了一组注释，您可以在专用应用程序服务器（如WebLogic）中以及以可移植方式在依赖注入容器（如Google Guice或Spring）中使用这些注释。我还使用JSR 250注释（定义为Java EE 5的一部分）来演示生命周期挂钩。

这个例子依赖于Spring Boot 自动配置和嵌入式 H2内存javax.sql.DataSource和-一个Spring Boot 自动配置并嵌入的内存HornetQ javax.jms.ConnectionFactory 。如果要连接到传统的非嵌入式实例，则可以直接在其中指定属性application.yml要么application.properties喜欢spring.hornetq.host或简单地定义@Bean适当类型的。

本示例还使用了新的@SpringBootApplication结合了注释@Configuration ， @EnableAutoConfiguration和@ComponentScan 。真好！

部署方式

尽管此示例使用了许多相当熟悉的Java EE API，但这仍然只是典型的Spring Boot，因此默认情况下，您可以使用java -jar ee.jar或轻松地将其部署到以流程为中心的平台即服务产品（如Heroku或Cloud Foundry） 。如果要将其部署到独立的应用程序服务器（例如Apache Tomcat或Websphere或两者之间的任何设备）中，可以很容易地将构建版本转换为.war并将其相应地部署到任何Servlet 3容器。

如果将应用程序部署到更经典的应用程序服务器，Spring Boot可以利用AS的功能。例如，使用受JNDI约束的JMS非常简单ConnectionFactory ， JDBC DataSource或JTA UserTransaction 。

Spring Boot 1.2：选择和力量

我个人会问很多这样的API。您真的需要分布式的多资源交易吗？在当今的分布式世界中，将全球事务管理器视为一种架构气味 。当Spring提供更丰富的，基于Spring MVC的集成堆栈，并带有MVC，REST，HATEOAS，OAuth和websockets支持时，您真的需要使用JAX-RS吗？JPA是用于与基于SQL对话的不错的API javax.sql.DataSource ，但是Spring Data存储库（当然包括对JPA的支持， 还包括对Cassandra，MongoDB，Redis，CouchBase的支持，以及越来越多的替代技术列表），这些都将很多样板简化为常见情况的简单接口定义。那么，您真的需要所有这些吗？您很可能会这样做，而且-一如既往-选择是您的。这就是为什么此版本如此酷！更多动力，更多选择。
