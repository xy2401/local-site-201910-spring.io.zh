Spring Cloud 1.0.0。RC1现在可用

发布
戴夫·瑟
2014年12月19日

Spring团队另一个愉快的假期信息： Spring Cloud 1.0.0。RC1现在可在http://repo.spring.io Maven存储库中获得。有很多新功能，包括

  • 通过注释支持Hystrix指标聚合@EnableTurbine和@EnableTurbineAmqp （对于基于AMQP的收集器）

  • 对Ribbon配置进行了重新调整，使其对Spring用户更加友好。现在，您可以配置每个功能区客户端ApplicationContext使用@RibbonClient并覆盖各个位，例如LoadBalancer ， 或者ServerListFilter ， 通过提供@Bean定义。

  • DiscoveryHealthIndicator现在是用户可以通过声明添加信息的组合@Beans类型的DiscoveryHealthIndicator 。

  • 现在，发现已从Eureka提取到新的spring-cloud-commons库中，并通过诸如@EnableDiscoveryClient （而不是旧的@EnableEurekaClient ）。相同的模式也适用于断路器
    和@EnableCircuitBreaker取代@EnableHystrix 。

  • Zuul代理的多项改进，包括Eureka目录更改时的自动更新，对表单编码的POST的支持，路由的外部配置以及每个客户端的身份验证方案。

  • 在Spring Cloud Security中以声明方式配置哪些路由需要OAuth2身份验证。

  • 支持Config Server的“本机”配置文件中的标签（如git分支）（在搜索位置的子目录中查找）。

  • 如果查找配置数据所需的URI无效，则“配置服务器和客户端”中的“快速失败”选项。

  • Spring Cloud Bus中对JSON消息的开箱即用支持。

  • 一个基于新的Feign配置的不错的框架@FeignClient注释（有点像Spring Data存储库）。

某些总线和入门模块已被重新组织和重命名，并且为用户应用提供了一个新的（可选的）父pom。RC1中的启动器的完整列表是

  • spring-cloud-starter：提供配置客户端和基本的Spring Boot依赖项

  • spring-cloud-starter-bus-amqp：用于通过AMQP的总线客户端

  • spring-cloud-starter-cloudfoundry：Cloud Foundry的特定功能

  • spring-cloud-starter-eureka：带有Netflix Eureka的服务发现客户端

  • spring-cloud-starter-eureka-server：具有Netflix Eureka的服务发现服务器

  • spring-cloud-starter-hystrix：Netflix Hystrix的断路客户端

  • spring-cloud-starter-hystrix-dashboard：带有Netflix Hystrix的断路仪表板

  • spring-cloud-starter-security：单一登录和OAuth2资源服务器快捷方式

  • spring-cloud-starter-turbine：与Netflix Turbine 1.0一起基于HTTP的Hystrix指标聚合

  • spring-cloud-starter-turbine-amqp：与Netflix Turbine 2.0一起基于AMQP的Hystrix指标聚合

  • spring-cloud-starter-zuul：具有自动注册发现服务的基本Zuul代理

spring-cloud-samples存储库中有大量样本 。请尝试一下可能会用到的东西，对于那些已经做了的人，谢谢，并继续尽可能地努力，然后再回馈反馈。节日快乐！
