Spring Boot 1.2.0发布

发布
菲尔·韦伯
2014年12月11日

我很高兴地宣布，Spring Boot 1.2.0已经发布，可以从repo.spring.io和Maven Central获得 。此版本在1.1之上添加了大量新功能和改进，建议所有用户升级。有关升级说明和“新功能和值得注意的功能”，请参阅发行说明 。

以下是此版本的一些重点：

Servlet 3.1

当与嵌入式Servlet容器一起运行时，Spring Boot现在使用Servlet 3.1。Tomcat 8，Jetty 9和Undertow 1.1都是受支持的选项。此外，WebSocket支持已得到改进，现已自动为所有受支持的服务器配置。如果您需要坚持使用Servlet 3.0，则仍支持Tomcat 7和Jetty 8。

@SpringBootApplication

如果您发现自己在打字@Configuration + @EnableAutoConfiguration + @ComponentScan每当您启动新的Spring Boot应用程序时，您可能都会喜欢新的@SpringBootApplication注解。您可以将其用作这三个注释的快速替代方法，并节省一些键入时间。如果您按照参考文档中的建议来构建代码，则此方法特别有效。

JTA支持和Java EE

Spring Boot 1.2现在使用Atomikos或Bitronix嵌入式事务管理器支持跨多个XA资源的分布式JTA事务。部署到合适的Java EE应用程序服务器时，还支持JTA事务。

总体而言，Spring Boot 1.2刷新了Java EE支持。Josh Long的这篇博客文章提供了出色的入门。

杰克逊定制

Spring Boot 1.2大大改善了Jackson的配置。您现在可以从您的配置大多数杰克逊选项application.properties文件。请参阅SébastienDeleuze的这篇优秀博客文章 ，以获取有关Spring对Jackson的一般支持的更多背景。

JMS支持

Spring 4.1 对其JMS支持进行了一些不错的增强，现在您可以从Spring Boot中无缝使用它们。的@EnableJms您也可以随时自动配置注释spring-jms.jar在您的课程路径上。

执行器更新

Spring Boot 1.2对执行器模块进行了许多更新。现在，我们提供了其他指标（包括数据源指标，Tomcat会话指标和改进的系统指标）。与Dropwizard“指标”库的集成也更好。

的/health端点已得到改进，以提供数据源和磁盘空间监视。现在，它也更加安全，并且不会在未经身份验证的连接上公开太多信息。

CLI更新

的spring CLI工具已通过几个新命令得到了改进。您现在可以输入spring init使用start.spring.io服务。例如：

$ spring init -d=web myapp.zip

将下载一个包含基本Spring Boot Web应用程序的zip。

CLI现在还支持扩展。您可以spring install 从Maven存储库添加扩展。

性能提升

尽管增加了更多的自动配置，Spring Boot 1.2实际上应该比1.1快一点。我们试图尽可能地优化代码库，并做了许多小技巧来使启动时间尽可能快。在大多数机器上，Tomcat示例应用程序的启动时间不到3秒。

其他许多变化

我们对Spring Boot 1.2进行了许多其他小的改进。有更好的JNDI支持，增强的横幅支持，对“ Spring Cloud Connectors”的支持，改进的Spring MVC默认值和新的电子邮件“ starter POM”。此外，对第三方库的支持已扩展到涵盖Log4J2 ， GSON和Jersey 。有关更改的完整列表，请参考发行说明和更新的参考指南 。


再次感谢继续支持并为Spring Boot做出贡献的每个人。此版本中的许多增强功能都是由社区贡献或驱动的。请继续做好工作，继续提出问题并提出要求 ！
