Spring XD 1.1 M2和1.0.3发布

工程
马克·波拉克
2014年12月23日

我非常荣幸地代表Spring XD团队宣布Spring XD 1.1的第二个里程碑版本和1.0.3维护版本。

下载链接：
* 1.1.0。M2发布： zip ，1.0.3。发布： zip

1.1 M2版本包含错误修复和增强功能以及几个新功能：

  • 基于Kafka的消息总线 。如果Kafka是您的中间件，那么您会感到宾至如归。
  • 对RabbitMQ的批处理和压缩支持大大提高了有效消息速率的吞吐量。
  • 使用Project Reactor的 Stream API开发处理器模块。 Stream API基于Reactive Streams规范 ，该规范提供了具有无阻塞背压的异步流处理的标准。这使您可以使用功能性编程模型作为处理事件流的基础。结合XD 对消息总线上数据分区的支持，您可以创建功能非常强大的分布式流处理解决方案。
  • 轻松开发，测试和打包自定义模块 。父pom引导程序构建自定义模块所需的所有依赖关系，并将模块和依赖关系捆绑到单个uberjar文件中。还支持使用嵌入式单节点服务器对模块与JUnit进行集成测试 。
  • 通过HTTP将自定义模块上传到管理服务器。
  • REST端点的基于角色的授权使您可以控制“谁可以做什么”。
  • 基于文件的身份验证可以替代LDAP。
  • Sqoop批处理作业集成

有关使用Reactor的Stream API的示例，请参见反应堆平均移动项目 ；有关开发自定义模块的完整示例，请参见si-dsl- module。

对于Spring XD 1.1 RC1的下一发行版，我们将包括对以下功能的支持：

  • 使用Spark Streaming的处理器模块
  • 使用RxJava的处理器模块
  • Kafka Message Bus和Kafka源代码的改进基于经过改进的Spring Kafka库
  • 细粒度的安全ACL支持谁可以创建或销毁流和作业
