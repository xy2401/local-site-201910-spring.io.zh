适用于Android 2.0.0的Spring。M2发布

发布
罗伊·克拉克森
2014年12月9日

我很高兴宣布Spring for Android 2.0.0。M2现在可以在Spring里程碑存储库中使用。重点包括：

  • 通过HttpComponentsClientHttpRequestFactory支持HttpClient 4.3的Android端口
  • Android附带的HttpClient 4.0支持现已弃用，但仍可通过HttpComponentsAndroidClientHttpRequestFactory获得。
  • RestTemplate中的HTTP PATCH支持
  • Spring Core的类型转换包现已在Spring for Android Core中提供。
  • Spring 3.2的许多改进和修补程序现已合并到Android的Spring中，以使Spring 3.2具有基线兼容性，同时还包含了Spring 4.1的某些RestTemplate功能以支持新的HttpClient。
  • 其他错误修复和改进

HttpClient 4.3

如果类路径上有适用于Android的HttpClient 4.3，它将被配置为默认HTTP客户端。在您的文件中包含以下依赖项build.gradle利用新的HttpClient。

dependencies {
    compile 'org.apache.httpcomponents:httpclient-android:4.3.5'
}

最低版本

支持的最低版本是Android 2.2（API级别8），但是现在它占Android开发人员仪表板上 Android设备的0.5％。我们建议将Android 2.3作为新应用程序的最低目标，因为诸如Jackson Jackson 2.4和OkHttp之类的第三方库需要Android 2.3及更高版本。

结论

有关更改的完整列表，请参见JIRA中的发行说明 。 参考文档也已更新，以涵盖新功能。

再次感谢为发布做出贡献的每个人！请尝试这个新的里程碑，并报告JIRA或GitHub上的任何问题。
