Spring Integration 3.0.2和4.0 Milestone 4发布

发布
阿尔特姆·比兰（Artem Bilan）
2014年3月31日

我们很高兴地宣布针对Spring Integration 4.0的最终里程碑版本以及3.0.x流的下一个维护版本。
3.0.2。RELEASE包含针对3.0版本的少量重要修补程序。
鼓励Spring Integration 3.0用户尽快升级到此版本。
请参阅3.0.2发行说明和项目页面以获取更多信息。

Spring集成4.0是下一代的框架，这是目前基于新的Spring框架4.0消息系统 。
有关将应用程序从Spring Integration 3.0迁移到4.0的信息，请参阅《 迁移指南》 。

Spring Integration 4.0发行版的另一个主要目标是在框架中添加改进的Java和注释配置功能。让我们来看看其中的一些……

__ @ EnableIntegration__

Spring Integration提供了许多环境和内置bean，以支持运行时企业集成模式和消息传递基础结构。
使用XML配置时， NamespaceHandler根据需要自动声明它们。在纯Java的纯配置中，没有名称空间处理程序，并且需要另一种机制来设置集成环境。@EnableIntegration添加了@EnableIntegrationannotation。
它类似于@EnableWebMvc从spring-webmvc或@Enable*Repositories从春数据注解，应该并排放置@Configurationannotation至少一个类。

请注意，只需要在ApplicationContext具有一个@EnableIntegrationannotation。
有了适当的注释，您可以开始从Spring @Configuration类配置集成流程：

@Configuration
@EnableIntegration
public static class MyConfiguration {

    @Bean
    public MessageChannel fileWritingChannel() {
         return new DirectChannel();
    }

    @Bean
    public FileWritingMessageHandler fileWritingMessageHandler() {
        return new FileWritingMessageHandler(this.outputDir);
    }

   @Bean
    public ConsumerEndpointFactoryBean fileWritingEndpoint() {
        ConsumerEndpointFactoryBean endpoint = new ConsumerEndpointFactoryBean();
        endpoint.setHandler(this.fileWritingMessageHandler());
        endpoint.setInputChannel(this.fileWritingChannel());
        return endpoint;
    }

}

当然，与组分扫描，现有Spring集成配置说明（ @MessageEndpoint ， @ServiceActivator ， @Router ， @Filter等）可用于限定的流动。有关示例，请参见本文后面的Spring Boot应用程序。

__ @ MessagingGateway__

消息Messaging Gateway是另一个有用且重要的消息传递组件。对于XML，我们使用组件，以提供接口的实现作为消息流的网关。
使用Spring Integration 4.0可以通过使用新的@MessagingGatewayannotation来避免XML配置。
此annotation提供的属性与元素，放置在网关的服务接口上：

@MessagingGateway(defaultRequestChannel = "gatewayChannel",
     defaultHeaders = @GatewayHeader(name = "foo", value = "FOO"))
public interface MyGateway {

	@Gateway(headers = @GatewayHeader(name = "calledMethod",
                                      expression = "#gatewayMethod.name"))
	String echo(String payload);

}

重要提示 ：由于这部分是不自动Spring容器和默认可见@ComponentScan不起作用与接口，
引入了另一个新注释@IntegrationComponentScan 。该注释类似于Spring Data中的@Enable*Repositories
并提供用于配置basePackages属性以扫描集成组件的选项，并且应与@Configuration一起放置。

Spring Boot @EnableAutoConfiguration

利用SpringFactoriesLoader机制，还可以通过Spring Boot @EnableAutoConfigurationannotation来使用Spring Integration基础结构。
只需将Spring Integration 4.0添加到类路径并使用Spring Boot自动配置功能。

这是一个非常简单的Spring Boot应用程序：

@EnableAutoConfiguration  // enables integration infrastructure
@IntegrationComponentScan // looks for gateways
@ComponentScan			  // looks for Spring Beans
public class Integration {

	public static void main(String[] args) throws Exception {
		ConfigurableApplicationContext ctx = SpringApplication.run(Integration.class, args);
		String reply = ctx.getBean(GW.class).sendAndReceive("foo");
		System.out.println(reply);
		ctx.close();
	}

	@MessagingGateway(defaultRequestChannel="in")
	public interface GW {

		String sendAndReceive(String payload);
	}

	@MessageEndpoint
	public static class MyService {

		@ServiceActivator(inputChannel="in")
		public String foo(String payload) {
			return payload.toUpperCase();
		}
	}

}

其他变化

此外，还引入了其他用于Java和注释配置的有用和便捷的组件：
@EnableMessageHistory ， @EnablePublisher ， @EnableIntegrationMBeanExport ， @GlobalChannelInterceptor ， @IntegrationConverter等。
