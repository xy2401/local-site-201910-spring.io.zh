使用WebDriver进行Spring MVC测试

工程
罗伯·温奇
2014年3月26日

在第二篇文章中，我介绍了如何将Spring MVC Test与HtmlUnit一起使用。在本文中，我们将利用WebDriver中的其他抽象使事情变得更加容易。

为什么要使用WebDriver？

我们已经可以使用HtmlUnit和MockMvc，那么为什么要使用WebDriver？WebDriver提供了非常优雅的API，并允许我们轻松地组织代码。为了更好地理解，让我们探索一个例子。

-
注意尽管是Selenium的一部分，但WebDriver不需要Selenium Server即可运行测试。

假设我们需要确保正确创建一条消息。测试涉及找到html输入，填写它们并进行各种声明。

有很多测试，因为我们也想测试错误条件。例如，我们要确保如果仅填写表单的一部分，则会出现错误。如果我们填写整个表格，则会显示新创建的消息。

如果将其中一个字段命名为“ summary”，那么在我们的测试中，我们可能会在每个地方重复以下内容：

HtmlTextInput summaryInput = createMsgFormPage.getHtmlElementById("summary");
summaryInput.setValueAttribute("Spring Rocks");

因此，如果我们将id更改为“ smmry”会发生什么。这意味着我们将必须更新所有测试！相反，我们希望我们写出一些更优雅的代码，用自己的方法填写表格：

public HtmlPage createMessage(HtmlPage currentPage, String summary, String text) {
  ...
  setSummary(currentPage, summary);
  ...
}

public void setSummary(HtmlPage currentPage, String summary) {
  HtmlTextInput summaryInput = currentPage.getHtmlElementById("summary");
  summaryInput.setValueAttribute(summary);
}

这样可以确保如果我们更改UI，则不必更新所有测试。

我们可能会更进一步，并将此逻辑放置在代表HtmlPage我们目前在。

public class CreateMessagePage {
  private final HtmlPage currentPage;

  ...

  public T createMessage(Class<T> resultPage, String summary, String text) {
    ...
    setSummary(currentPage, summary);
    ...
    HtmlPage result = submit.click();
    ...
    return (T) error ? new CreateMessagePage(result) : new ViewMessagePage(result);
  }

  public void setSummary(String summary) {
    HtmlTextInput summaryInput = currentPage.getHtmlElementById("summary");
    summaryInput.setValueAttribute(summary);
  }
}

以前，此模式称为Page Object Pattern 。虽然我们当然可以使用HtmlUnit做到这一点，但WebDriver提供了一些工具，我们将在以下各节中探讨这些工具，使该模式更加容易。

更新依赖项

在使用项目之前，必须确保更新依赖项。有关Maven和Gradle的说明，请参见站点文档。

使用WebDriver

现在我们有了正确的依赖关系，我们可以在单元测试中使用WebDriver了。我们的示例假定您已经具有JUnit作为依赖项。如果尚未添加，请相应地更新您的类路径。在MockMvcHtmlUnitDriverCreateMessageTests中可以找到使用WebDriver和Spring MVC Test的完整代码示例。

创建MockMvc

为了使用WebDriver和Spring MVC Test，我们必须首先创建一个MockMvc实例。关于如何创建MockMvc实例，但我们将回顾如何创建一个MockMvc在本节中非常快地实例化。

第一步是创建一个新的JUnit类，其注释如下所示：

@RunWith(SpringJUnit4ClassRunner.class)
@ContextConfiguration(classes = {WebMvcConfig.class, MockDataConfig.class})
@WebAppConfiguration
public class MockMvcHtmlUnitDriverCreateMessageTests {

  @Autowired
  private WebApplicationContext context;

  ...
}
  • @RunWith(SpringJUnit4ClassRunner.class)允许Spring对我们的对象执行依赖项注入MockMvcHtmlUnitDriverCreateMessageTests 。这就是为什么我们@Autowired注释将受到尊重。
  • @ContextConfiguration告诉Spring加载什么配置。您会注意到，我们正在加载数据层的模拟实例，以提高测试性能。如果需要，我们可以选择对真实数据库运行测试。但是，这具有我们前面提到的缺点。
  • @WebAppConfiguration表示SpringJUnit4ClassRunner它应该创建一个WebApplicationContext而不是ApplicationContext 。

接下来，我们需要创建我们的MockMvc来自的实例context 。下面提供了如何执行此操作的示例：

@Before
public void setup() {
  MockMvc mockMvc = MockMvcBuilders.webAppContextSetup(context).build();
  ...  
}

当然，这只是创建广告素材的一种方法MockMvc实例。我们可能已经决定添加Servlet过滤器 ，使用独立设置等。重要的是，我们需要一个Instance MockMvc 。有关创建广告的其他信息MockMvc实例请参阅Spring MVC测试文档 。

初始化WebDriver

现在，我们已经创建了MockMvc实例，我们需要创建一个MockMvcHtmlUnitDriver确保我们使用MockMvc我们在上一步中创建的实例。

private WebDriver driver;

@Before
public void setup() {
	MockMvc mockMvc = MockMvcBuilders.webAppContextSetup(context).build();
	driver = new MockMvcHtmlUnitDriver(mockMvc, true);
}

使用WebDriver

现在，我们可以像往常一样使用WebDriver，而无需部署应用程序。例如，我们可以请求视图创建以下消息：

CreateMessagePage messagePage = CreateMessagePage.to(driver);

然后，我们可以填写表格并将其提交以创建一条消息。

ViewMessagePage viewMessagePage = 
    messagePage.createMessage(ViewMessagePage.class, expectedSummary, expectedText);

通过利用Page Object Pattern，这可以改善我们的HtmlUnit测试的设计。正如我们在“ 为什么要使用WebDriver？ ，我们可以将Page Object Pattern与HtmlUnit一起使用，但是现在它要容易得多。让我们来看看我们的CreateMessagePage 。

public class CreateMessagePage extends AbstractPage {
    private WebElement summary;

    private WebElement text;

    @FindBy(css = "input[type=submit]")
    private WebElement submit;

    public CreateMessagePage(WebDriver driver) {
        super(driver);
    }

    public <T> T createMessage(Class<T> resultPage, String summary, String details) {
        this.summary.sendKeys(summary);
        this.text.sendKeys(details);
        this.submit.click();
        return PageFactory.initElements(driver, resultPage);
    }

    public static CreateMessagePage to(WebDriver driver) {
        driver.get("http://localhost:9990/mail/messages/form");
        return PageFactory.initElements(driver, CreateMessagePage.class);
    }
}

您会注意到的第一件事是CreateMessagePage扩展AbstractPage 。我们不会详细介绍AbstractPage ，但总而言之，它包含了我们所有页面的所有常用功能。例如，如果您的应用程序具有导航栏，全局错误消息等，则可以将此逻辑放置在共享位置。

接下来您会发现，对于HTML的每个部分，我们都有一个成员变量， WebElement ，我们对此感兴趣。 WebDriver的PageFactory允许我们从HtmlUnit版本的CreateMessagePage通过自动解决每个WebElement 。

的PageFactory#initElements方法将自动解决每个WebElement通过使用字段名称并尝试通过ID或HTML页面上元素的名称进行查找。我们还可以使用@FindBy批注覆盖默认值。我们的示例演示了如何使用@FindBy注释，以使用input [type = submit]的css选择器查找提交按钮。

最后，我们可以验证是否成功创建了新消息

assertThat(viewMessagePage.getMessage()).isEqualTo(expectedMessage);
assertThat(viewMessagePage.getSuccess()).isEqualTo("Successfully created a new message");

我们可以看到我们的ViewMessagePage可以返回一个Message除个人以外的对象Message属性。这使我们可以轻松地与我们的富域对象进行交互，而不仅仅是String 。然后，我们可以在声明中利用富域对象。为此，我们创建了一个自定义的fest断言 ，该断言使我们可以验证实际的所有属性Message等于预期Message 。您可以在断言和MessageAssert中查看自定义断言的详细信息

最后，别忘了关闭WebDriver完成后的实例。

@After
public void destroy() {
	if(driver != null) {
		driver.close();
	}
}

有关使用WebDriver的其他信息，请参阅WebDriver文档 。
