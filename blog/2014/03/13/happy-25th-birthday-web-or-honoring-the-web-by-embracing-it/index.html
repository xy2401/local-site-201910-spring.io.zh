祝25岁生日快乐，网站！或：通过拥抱来尊重网络

工程
乔什·朗（Josh Long）
2014年3月13日

昨天，2014年3月，12日，是在 25周年 的 爵士 蒂姆· 伯纳的 李已经发明了网络 。正如他解释的那样：“我只需要接受超文本概念，并将其连接到传输控制协议和域名系统概念，以及_ta-da！_ —万维网。”小学 ，真的。（哈！）

图中 （取自Wikimedia）是原始的NeXT工作站（其操作系统NeXT Step ，支撑了当今的OS X和iOS设计），蒂姆·伯纳斯·李爵士将最初的HTTP服务和客户端放在了该工作站上。

顺便说一句：我一直想知道在那个实验室里呆在那里会是什么样子。为了防止意外绊倒和摔倒并拔下他的机器，并（因此）拔掉整个网络的电源-哈！试想一下，如果一小时内您可以立即关闭网络，您的工作效率将提高多少！ :)

在HTTP 的第一个修订版中，HTTP具有很多设计权 。这是一个相当容易实现的协议。它是无状态的，这使得对服务进行分区变得容易。到1996年为1.0，它支持查询，写入数据和一些基本的客户端/服务器协商。它使用标头来注释请求和响应，从而为客户端和服务提供一种内置机制，以丰富所传递的有效负载。HTTP旨在供具有不同编码的不同客户端使用。它旨在从诸如缓存和代理之类的网络元素中受益。HTTP 1.1（正式地）与内容无关，并且具有内容类型的概念（包括Accept和Content-Type标头）。

到Roy Fielding （也从事HTTP 1.1的工作）在为加利福尼亚大学尔湾分校撰写他著名的博士学位论文，介绍REST架构约束时，该平台已经在建造中了！REST形式化了HTTP中已有的一些好主意。它将HTTP动词暗示的状态突变映射到数据的生命周期。将REST视为HTTP：面向服务的部分 。

HTTP使我们走了很长一段路。REST也已经改变了游戏规则。REST推动了移动革命，推动了RIA革命，而且-我认为-它为大众提供了迄今为止独角兽般的SOA实现。

Spring的HTTP和REST支持概述

Spring 平台吐气 HTTP和REST。实际上，为Spring Framework 编写的第一行代码是为了支持Web编程 。Spring Framework 为构建基于HTTP和REST的应用程序提供了非常丰富的堆栈。

在本文中，我希望简要介绍一下Spring丰富的REST支持。

智慧的客户

Spring Framework 包括RestTemplate这将常见的HTTP操作简化为与熟悉的样式相同的单行JdbcTemplate和新的NIO-2供电AsyncRestTemplate 。在文档中，“它简化了与HTTP服务器的通信，并实施了RESTful原则。它处理HTTP连接，留下应用程序代码以提供URL（带有可能的模板变量）并提取结果。”对。

的RestTemplate支持拦截器模型，该模型可用于插入安全性，审核，会话处理等内容。特别是，可以透明地处理基于状态的基于HTTP的交换，例如OAuth 。

OAuth是一种协议，允许以简单，标准的方式从Web，移动和桌面应用程序进行安全授权。它确保了众多的Web服务的安全，包括Facebook，Twitter，GitHub，LinkedIn，TripIt和Google的各种API。如果您想使用Web上的Web服务，则很难忽略OAuth，而且-借助Spring Social-您不需要！

Spring Social是一个保护伞项目。它在基础上提供了构建OAuth服务客户端的机制（使用RestTemplate ），然后在各种类型的API的类型安全的Java绑定之上进行分层，这些API包括Spring Social Twitter ， Spring Social Facebook和Spring Social LinkedIn 。对于许多替代API，还有许多其他（ 已知和未知）第三方绑定。

Spring 社会进一步提供紧密集成Spring MVC应用，处理OAuth认证为你跳舞 。Spring Social使使用安全的OAuth API变得非常简单，并支持其他常见用例，例如Sign In With Facebook 。

Spring for Android项目提供了RestTemplate适用于Android客户。您还可以使用Android上的Spring Social客户，进一步扩大您的覆盖范围。

您可以找到许多入门指南 [详细说明如何使用REST服务]（详细说明如何使用REST服务）（以及其他内容！），而不仅仅是RestTemplate ，但来自Android，流行的JavaScript客户端等 。还提供了一些方便的入门指南， 以开始使用Spring Social Twitter和Facebook 。

广泛，可扩展，安全的Web

Spring MVC是Spring Framework 的一部分。Spring MVC为构建基于HTTP和REST的应用程序提供了非常丰富的基础。正如我前面提到的，REST是一种体系结构约束， 而不是一成不变的标准。相反，有准则。这是一种力量和美德。REST可能是一个移动的目标。Leaonard Richardson博士设计了所谓的Richardson成熟度模型，该模型本身提供了一个矩阵， 您可以据此判断给定REST API是否符合RESTful原则 。

Spring MVC的核心REST支持不难达到第2级。将其带入更高的层次3，需要了解一些有关超媒体的知识 ，或者通过链接和HATEOAS将HTTP资源连接在一起的能力。在HATEOAS上的Wikipedia页面上：“ REST客户端通过简单的固定URL进入REST应用程序。在服务器返回的资源表示中发现了客户端将来可能执行的所有操作。用于这些表示的媒体类型以及它们可能包含的链接关系是标准化的。客户端可以通过从表示形式内的链接中进行选择，或通过其媒体类型提供的其他方式来操纵表示形式，从而在应用程序状态之间进行转换。通过这种方式，RESTful交互是由超媒体而不是带外信息驱动的。”

那么，安全性如何？

开放的Web应用程序或REST API就像一个开放的公开数据库。 可能不是您想要的。保护Web应用程序对许多人来说意味着很多事情：客户端和服务器之间的连接是否已加密？应用程序知道谁在发出请求吗？我们是否在保护Web应用程序？REST服务？消息通道？幸运的是，答案不只一个： Spring Security 。Spring Security是您的一站式安全商店，其中包括后端身份提供商（LDAP，Active Directory），身份验证和授权，加密以及对常用功能（如应用程序登录和注销）的集成的集成。

例如，您可以使用Spring Security OAuth 来保护REST端点 ，当然，您可以从Spring Social中使用它！

Spring Boot为所有这些技术提供了理想的切入点。使用Spring Boot部署的应用程序附带了管理和指标。前面提到的大多数入门指南都是基于Spring Boot的。对于REST开发，有很多建议可以推荐使用Spring Boot，但是一定要确定一个指标是Tweetable！ 。而且，当然，另一个是Spring Boot使构建Spring应用程序像饼一样容易，并且它包含了各种功能，这些功能旨在简化以一致的方式生产的任务。

以下代码是David Syer博士的史诗性示例 的轻微衍生 ，该示例演示了如何使用Spring Boot和Spring Security OAuth保护REST端点 。我要做的就是添加一个Spring Data REST支持的JPA存储库和一些种子数据（请参阅存储库）。Spring Data REST托管数据库（默认为嵌入式H2 javax.sql.DataSource ，尽管我们可以轻松地使用其他任何东西DataSource或者，实际上，完全是许多其他后端NoSQL存储中的任何一个！

package demo;

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.EnableAutoConfiguration;
import org.springframework.context.annotation.ComponentScan;
import org.springframework.context.annotation.Configuration;
import org.springframework.context.annotation.Import;
import org.springframework.data.jpa.repository.JpaRepository;
import org.springframework.data.rest.core.annotation.RepositoryRestResource;
import org.springframework.data.rest.webmvc.config.RepositoryRestMvcConfiguration;
import org.springframework.security.authentication.AuthenticationManager;
import org.springframework.security.config.annotation.web.builders.HttpSecurity;
import org.springframework.security.oauth2.config.annotation.configurers.ClientDetailsServiceConfigurer;
import org.springframework.security.oauth2.config.annotation.web.configuration.AuthorizationServerConfigurerAdapter;
import org.springframework.security.oauth2.config.annotation.web.configuration.EnableAuthorizationServer;
import org.springframework.security.oauth2.config.annotation.web.configuration.EnableResourceServer;
import org.springframework.security.oauth2.config.annotation.web.configuration.ResourceServerConfigurerAdapter;
import org.springframework.security.oauth2.config.annotation.web.configurers.OAuth2AuthorizationServerConfigurer;
import org.springframework.security.oauth2.config.annotation.web.configurers.OAuth2ResourceServerConfigurer;

import javax.persistence.Entity;
import javax.persistence.GeneratedValue;
import javax.persistence.Id;

// standard @Configuration class 
@Configuration
@ComponentScan
@EnableAutoConfiguration
@Import(RepositoryRestMvcConfiguration.class) // import Spring Data REST
public class Application {

    public static final String CRM_RESOURCE_ID = "crm";
    
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
	
	    @Configuration
    @EnableResourceServer
    protected static class ResourceServerConfiguration
    	 extends ResourceServerConfigurerAdapter {

		// tell SS OAuth which URI paths to secure (optional!)
        @Override
        public void configure(HttpSecurity http) throws Exception {
            // @formatter:off
            http
                    .requestMatchers().antMatchers("/*", "/admin/beans").and()
                    .authorizeRequests()
                    .anyRequest().access("#oauth2.hasScope('read')");
            // @formatter:on
        }
		
        @Override
        public void configure(OAuth2ResourceServerConfigurer resources) throws Exception {
            resources.resourceId(CRM_RESOURCE_ID);
        }

    }

    @Configuration
    @EnableAuthorizationServer
    protected static class OAuth2Configuration
    		 extends AuthorizationServerConfigurerAdapter {
		
        @Autowired
        private AuthenticationManager authenticationManager;

		// plugin a Spring Security AuthenticationManager. One is created for us by Boot,
		// though we could plugin to *any* Identity Provider (LDAP, ActiveDirectory, etc)
        @Override
        public void configure(OAuth2AuthorizationServerConfigurer oauthServer) throws Exception {
            oauthServer.authenticationManager(authenticationManager);
        }

        @Override
        public void configure(ClientDetailsServiceConfigurer clients) throws Exception {
            // @formatter:off
            
			// we could describe multiple clients with varying settings
            clients.inMemory()    
            		 .withClient("ios-crm")  // describe how our iOS client should connect
                    .authorizedGrantTypes("client_credentials", "password")
                    .authorities("ROLE_CLIENT")
                    .scopes("read")
                    .resourceIds(CRM_RESOURCE_ID)
                    .secret("secret");
            // @formatter:on
        }

    }
}


// a Spring Data JPA repository that's 
// exposed as a REST endpoint using Spring Data REST
@RepositoryRestResource
interface CustomerRepository extends JpaRepository<Customer, Long> {
}

// JPA entity.
@Entity
class Customer {

    @Id
    @GeneratedValue
    private Long id;
    private String firstName, lastName;


    Customer() {
    } // for JPA

    public Customer(String firstName, String lastName) {
        this.firstName = firstName;
        this.lastName = lastName;
    }

    public String getLastName() {
        return lastName;
    }

    public Long getId() {
        return id;
    }

    public String getFirstName() {
        return firstName;
    }
}
