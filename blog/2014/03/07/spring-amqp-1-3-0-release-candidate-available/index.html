Spring AMQP 1.3.0发行候选版可用

发布
加里·罗素
2014年3月7日

我们很高兴宣布Spring AMQP（用于Java）1.3.0的可用性。RC1版本候选。有望成为通用航空之前的最终候选人。

该版本包括一些重要的新功能，包括：

  • 侦听器容器

  • 可以在不先停止容器的情况下更改侦听器容器的并发性，并且将相应地调整侦听器

  • 侦听器容器可以根据工作负载动态调整并发使用者
  • 侦听器容器现在支持使用者优先级（RabbitMQ 3.2.x或更高版本）
  • 侦听器容器现在支持独占使用者的配置

  • 兔子模板

  • RabbitTemplate现在具有几个便捷的receiveAndReply方法

  • 该RabbitTemplate现在可以用配置RetryTemplate ，使客户能够不必直接与经纪人连接问题要处理。

  • 连接工厂

  • 连接工厂现在可以缓存连接，而不是所有共享同一连接的用户（并且由于M2，通道可以缓存在缓存的连接中）
  • 现在有一个SimpleRoutingConnectionFactory来确定在运行时使用哪个连接工厂

  • 其他主要特点

  • 现在提供了流畅的Java API来构建Message （和MessageProperties ）

  • 现在提供了流畅的Java API，用于构建侦听器容器重试拦截器
  • 使用重试拦截器时，可以使用新的RepublishMessageRecoverer将失败的消息（带有堆栈跟踪）重新发布到另一个交换

有关新功能的完整列表，请参阅“ 新增功能和发行说明”。请访问项目页面以获取文档等的链接。
