2014年SpringOne2GX-第2天

工程
乔什·朗（Josh Long）
2014年9月15日

第2天的爆炸声开始了！我的理论是，到那时为止，人们已经看完一天的会议，因此，主题演讲必须充满影响力，大爆炸性的公告和讨论，以保持难忘的气氛。它需要提高标准并表现出领导才能。它做到了。

Spring Framework 4.1

Spring框架负责人Juergen Hoeller通过最近发布的 Spring 4.1中的新功能的介绍拉开了序幕。

Spring Framework 4.1提供了带注释的JMS侦听器方法，这些方法全面适用于最终的JSR 107规范，灵活的解析和静态Web资源的转换，其他MVC视图（例如Groovy Server Pages），websocket支持的改进以及许多其他新功能。

春季XD

Spring XD的共同负责人Mark Fisher于去年在SpringOne2GX 2013上首次亮相一年之后，现阶段介绍该项目的现状。马克（Mark）邀请我们考虑自春季初期以来情况已经发展了多少，并考虑了今天机会仍然多么有趣。然后……他讲了个笑话 ：“过去，现在和未来都陷入了障碍。太紧张了。”

过了一会儿，但是房间恢复了。勉强。马克表示愿意继续担任全职开发人员，而不转向喜剧。

Mark向我们展示了Spring XD a对未来的看法。它建立在一流的组件之上，例如Spring Batch ， Spring Integration ， Spring Data ， Redis ， RabbitMQ ， Zookeeper和Netflix的Zookeeper-addition Curator 。它提供了一种简单的，类似于Linux的特定领域语言（DSL），不需要任何复杂的处理，从而使Hadoop对初学者和专家而言都更加易于使用。扩展Spring XD是在Spring / Java中完成的，通常在Spring Integration中进行，以适应客户独特的系统，如Spring XD源和接收器。

它以操作员和开发人员熟悉的软件包提供强大的数据集成和流处理功能。Mark介绍了一个演示，该演示分析了智能电网的资源消耗，然后使用Spring XD中的PMML机器学习支持来预测未来的消耗水平。该示例使用了部署在多个群集节点上的大量节点，但是Spring XD本身中解决方案的组装和描述非常简单！（人群似乎也是如此！）

微服务

* Dave Syer博士 *（ Spring Batch的共同创始人， Spring Security OAuth和Spring Boot的共同负责人）重新加入了舞台，在Spring Boot的第2天更新中进行了跟进，并提供了有关该产品的一些新闻和介绍。新的Spring Cloud伞项目。过去，Spring Cloud包含“平台即服务”连接器，使您可以从PaaS环境中使用服务-数据库，消息队列等。Spring Cloud现在具有扩展的范围来定义和提供软件，以更好地支持现代的云架构，例如Netflix在Spring Boot 及其自身的开源堆栈之上构建的微服务 。

Spring Cloud旨在为大规模的，通常基于云的应用程序中的紧急模式提供解决方案：

实际上，新的Spring Cloud项目集成了流行的Netflix OSS组件，例如Hystrix （提供其著名的可靠性模式的实现 ）和Eureka （提供服务发现和定位）。

Syer博士介绍了一种简单的微服务，该微服务演示了两种服务，一种提供客户存储库，另一种提供对商家（商店）的访问。客户服务返回有关附近商店的信息。此信息来自商店服务。如果将商店服务下线，则客户服务将继续运行。

这项工作基于Spring Boot，着眼于简化微服务的开发。该消息似乎引起了很好的共鸣，我们认为Spring Boot，Spring Cloud和CloudFoundry的结合将为需要以整体形式开始生活但随后需要分散和分散以进行扩展的JVM应用程序提供致命的结合。阿德里安·考克克罗夫特（Adrian Cockcroft） 在InfoQ的一次采访中解释了为什么分配和权力下放如此重要非常重要。
