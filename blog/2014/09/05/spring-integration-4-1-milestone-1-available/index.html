Spring Integration 4.1里程碑1可用

发布
阿尔特姆·比兰（Artem Bilan）
2014年9月5日

尊敬的Spring社区，

我们很高兴地宣布Spring Integration 4.1里程碑
1已发布。请使用带有Maven或Gradle的Milestone库 ，
下载一个
发行档案 ，
或查看项目主页以获取更新文档的链接，
和Maven / Gradle配置详细信息。

该版本包括一些新功能和进一步的改进，以及一些GA发行版的错误修复。
大约在十月底。

以下是主要更改的摘要：

Spring Framework 4.1

Spring Integration充分利用了Spring Framework 4.1中的许多新功能，并需要该版本。

更改包括：

  • 不再将其他Spring Integration API迁移到Core Spring Messaging模块，并且不建议使用Spring Integration等效项
    或删除以避免在目标应用程序中造成混淆；

  • Spring Framework 4.1中包含了许多消息传递性能改进。

  • Spring Framework 4.1引入了[SpEL编译器]（ http：//docs.spring
    .io / spring / docs / current / spring-framework-reference / html / expressions.html＃expressions-spel-compilation）。
    对于Spring Integration（在运行时广泛使用SpEL）非常有用。您可以使用以下命令启用SpEL编译器：
    spring.expression.compiler.mode系统属性，值为IMMEDIATE或MIXED 。

WebSocket适配器

已引入WebSocket入站和出站通道适配器。它们基于Spring WebSocket构建
和消息基础。

WebSockets的主要功能之一是streaming协议，从Java的角度来看，它是基于
与服务器端和客户端相同的API，因此，我们可以使用类似的组件构建集成流程
双方：

服务器端

@Configuration
@EnableIntegration
public class ServerConfig {

	@Bean
	public ServerWebSocketContainer serverWebSocketContainer() {
		return new ServerWebSocketContainer("/ws").withSockJs();
	}

	@Bean
	public MessageProducer webSocketInboundChannelAdapter() {
		WebSocketInboundChannelAdapter webSocketInboundChannelAdapter = 
					new WebSocketInboundChannelAdapter(serverWebSocketContainer());
		webSocketInboundChannelAdapter.setOutputChannel(webSocketInputChannel());
		return webSocketInboundChannelAdapter;
	}

	@Bean
	@Transformer(inputChannel = "webSocketInputChannel", outputChannel = "webSocketOutputChannel")
	public ExpressionEvaluatingTransformer transformer() {
		return new ExpressionEvaluatingTransformer(PARSER.parseExpression("'Hello ' + payload"));
	}

	@Bean
	@ServiceActivator(inputChannel = "webSocketOutputChannel")
	public MessageHandler webSocketOutboundMessageHandler() {
		return new WebSocketOutboundMessageHandler(serverWebSocketContainer());
	}

}

客户端

@Configuration
@EnableIntegration
public class ClientConfig {
	@Bean
	public WebSocketClient webSocketClient() {
		return new SockJsClient(Collections.singletonList(
						new WebSocketTransport(new JettyWebSocketClient())));
	}

	@Bean
	public IntegrationWebSocketContainer clientWebSocketContainer() {
		return new ClientWebSocketContainer(webSocketClient(), "ws://host:port/ws");
	}

	@Bean
	public MessageProducer webSocketInboundChannelAdapter() {
		WebSocketInboundChannelAdapter webSocketInboundChannelAdapter =
				new WebSocketInboundChannelAdapter(clientWebSocketContainer());
		webSocketInboundChannelAdapter.setOutputChannel(webSocketInputChannel());
		return webSocketInboundChannelAdapter;
	}

	@Bean
	@ServiceActivator(inputChannel = "webSocketOutputChannel")
	public MessageHandler webSocketOutboundMessageHandler() {
		return new WebSocketOutboundMessageHandler(clientWebSocketContainer());
	}

}

与Spring WebSockets和STOMP子协议集成但使用集成流程的另一种简单方法
基于@MessageMappingannotation来处理请求和发送响应，
但在@MessagingGateway接口上：

@MessagingGateway
@Controller
public interface WebSocketGateway {

	@MessageMapping("/greeting")
	@SendToUser("/queue/answer")
	@Gateway(requestChannel = "greetingChannel")
	String greeting(String payload);

}

反应堆支持

已添加Promise Gateway has been added to support Project Reactor.
Now Spring Integration message flows can be used as a part of Reactive Streams:

@MessagingGateway(reactorEnvironment = "reactorEnv")
public interface PromiseGateway {

	@Gateway(requestChannel = "promiseChannel")
	Promise multiply(Integer value);

}

	    ...

@ServiceActivator(inputChannel = "promiseChannel")
public Integer multiply(Integer value) {
	return value * 2;
}

		...

Streams.defer(Arrays.asList("1", "2", "3", "4", "5"))
	.env(this.environment)
	.get()
	.map(Integer::parseInt)
	.mapMany(integer -> promiseGateway.multiply(integer))
	.collect()
	.consume(integers -> ...)
	.flush();

In addition, support of Spring Framework ListenableFuture网关方法返回类型。

Boon JSON映射器

Boon JsonObjectMapper实现已添加到
JSON变形金刚。事实证明，Boon比Jackson具有更好的JSON映射性能。在Spring Integration中启用了它
通过在类路径中检测其jar来自动进行。

分离器迭代器

现在可以将Iterator and Iterable作为payload返回，以实现streaming行为，
当使用Iterator.next()每个项目作为答复消息发出时。
