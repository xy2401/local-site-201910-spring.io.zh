以编程方式使用Spring Cloud

工程
拉姆尼瓦斯·拉达德（Ramnivas Laddad）
2014年7月29日

在上一个博客中 ，我向您展示了如何使用Spring Cloud的Java配置选项以声明方式获取服务连接器（如果需要，还提供XML名称空间支持）。在此博客中，我们将仔细研究如何以编程方式使用Spring Cloud。这将在无法使用Java或XML配置的情况下有所帮助。它还将揭秘Spring Cloud的工作原理，并为本系列的下一个博客做准备，在此我们讨论扩展Spring Cloud。

要使用Spring Cloud，我们需要访问的对象Cloud类。但是，您无法创建Cloud直接对象（其构造函数不是公共的）。相反，您将通过以下方式获取它CloudFactory 。

CloudFactory cloudFactory = new CloudFactory();
Cloud cloud = cloudFactory.getCloud();

在此情况下创建的云对象适用于应用程序在其中运行的环境。例如，如果应用程序在Cloud Foundry中运行，则将其配置为了解其如何向应用程序公开服务。请注意，创建一个CloudFactory实例有些昂贵，因此您应避免创建多个实例。如果使用像Spring这样的依赖注入框架，它将避免多个实例。否则，请自己进行管理。

一旦有了云对象，就可以获取应用程序实例信息，使用各种条件的服务信息对象以及使用指定条件的服务连接器。假设您想获得ServiceInfo绑定到应用程序的所有服务的对象，并显示关系服务的JDBC URL，您可以使用以下代码段：

List<ServiceInfo> serviceInfos = cloud.getServiceInfos();
for (ServiceInfo serviceInfo : serviceInfos) {
    if (serviceInfo instanceof RelationalServiceInfo) {
        System.out.println(((RelationalServiceInfo) serviceInfo).getJdbcUrl());
    }
}

这将产生如下输出：
jdbc:postgresql://babar.elephantsql.com:5432/tbsonrjm?user=***&password=***

使用获取的对象getServiceInfos()及其变体包含足够的信息（例如URL和凭据）以创建服务连接器。在某些情况下，获得ServiceInfo对象可能就是您所需要的，因为您始终可以创建合适的连接器（例如DataSource ）。但是在大多数情况下，您应该让Spring Cloud为服务创建合适的服务连接器。例如，如果您想获得一个DataSource对于“库存数据库”服务，您可以使用以下代码段：
DataSource inventoryDataSource = cloud.getServiceConnector("inventory-db", DataSource.class, null);
此方法有一个变体： getSingletonServiceConnector() ，您可以按以下方式使用：
java DataSource inventoryDataSource = cloud.getSingletonServiceConnector(DataSource.class, null);
在这里，它将返回一个DataSource用于绑定到应用程序的唯一关系数据库服务；如果没有这样的服务，或者有不止一项服务，它将抛出异常。我们通过了null作为这两种方法的最后一个参数，以对创建的连接器使用默认配置。但是，您可以传递想要覆盖的配置。例如，在这里我们为要创建的数据源指定池配置以及连接配置。

PoolConfig poolConfig = new PoolConfig(20, 200);
ConnectionConfig connectionConfig = new ConnectionConfig("characterEncoding=UTF-8");
DataSourceConfig serviceConfig = new DataSourceConfig(poolConfig, connectionConfig);
DataSource invetoryDataSource = cloud.getSingletonServiceConnector(DataSource.class, serviceConfig);

最后，有一种获取应用程序信息的方法，该信息包含应用程序ID（取决于云，但通常是应用程序名称），应用程序实例ID和松散定义的应用程序属性。让我们打印所有这些信息：

ApplicationInstanceInfo appInstanceInfo = cloud.getApplicationInstanceInfo();
System.out.println("Application id: " + appInstanceInfo.getAppId());
System.out.println("Application instance id: " + appInstanceInfo.getInstanceId());
for (Map.Entry<String, Object> entry: appInstanceInfo.getProperties().entrySet()) {
    System.out.println("Application property: " + entry.getKey() + "=" + entry.getValue());
}

在Cloud Foundry中运行的应用程序中执行此代码时，您将获得类似于以下内容的输出（此处缩写）。如果同一个应用程序在Heroku中运行，它将产生类似的输出，但是具有不同的键集：
