春季本周（春季XD版）-2014年7月29日

工程
乔什·朗（Josh Long）
2014年7月29日

欢迎来到春季本周的另一期！这周看到了惊人的Spring XD 1.0.0的发布。发布 。发布公告是开始大数据之旅的好地方 。在这里，您会找到指向其他精彩文章，学习内容等的链接。这是评估您希望从数据中获取什么以及是否获取它的绝佳机会。Spring XD是您的大数据瑞士军刀：它可以支持实时流工作负载；以批处理为中心的脱机工作负载；和通用数据集成解决方案。如果您从本周的综述中只摘录了一篇文章，请使其成为Spring XD版本 ！（然后，编写数据集成解决方案以使用Spring XD读取和组织其余的数据！）

当然，关于此事以及关于Spring和其他事情的事情还有很多要说的，所以一定要尽快在9月8日至11 日在德克萨斯州达拉斯的SpringOne2GX 2014上预订您的位置 。这是发现第一手一切并提供直接反馈的最佳机会。将在Spring XD上进行深入探讨，并进行一般的大数据讲座，以介绍开发大数据应用程序的前景和挑战。

顺便说一句，让我们继续本周的综述...

  1. 从… Spring XD 1.0开始。GA ！我知道我知道。我们就是这样做的。但是，您可能需要花一些时间来阅读一些有关Spring XD的精彩文章，首先从InfoQ上对Spring XD的精彩介绍开始。
  • Spring Framework 4.1 RC1上周发布，本周我们对Spring 4.1的功能进行了一些介绍， 包括MVC改进，如JSONP，Google Protocol Buffers支持和Java 8。 java.util.Optional在适当的地方提供支持…
  • …以及有关Spring MVC 4.1的非常明智的静态Web资源处理方法的文章 。也阅读评论！那里也有很多反馈。
  • 我将在9月16日举行一个网络研讨会， 探讨Spring Boot与微服务的关系 ，希望您能与我一起提出问题，评论和更多内容。
  • Activiti项目成员Joram Barrez和我将在网络研讨会上介绍如何使用Spring和Activiti为以流程为中心的应用程序提供一两个功能 -加入我们！
  • Spring Data SR2刚刚发布 。此服务发布，鼓励用户尽早升级。
  • 我很幸运能够在Spring Boot上出席Toronto Pivotal开源中心聚会。现在可以在线观看录音了 ，尽管我提醒您，音频有时会有些断断续续。
  • Spring Cloud主管Ramnivas Laddad撰写了一篇不错的文章，介绍了如何以编程方式使用Spring Cloud （与通过XML甚至Spring的Java配置相反）。Spring Cloud是PaaS绑定服务（例如数据库，消息代理，缓存等）的客户端。它可在Heroku和Cloud Foundry上运行。除了致力于Spring Cloud，Ramnivas还是AspectJ和Spring的AOP支持的最初贡献者之一，并且是Cloud Foundry背后的早期首席架构师之一。Ramnivas住在Spring和云之间的交汇处，而Spring Cloud 正是医生要求的！
  • Spring Batch主管Michael Minella刚刚宣布Spring Batch Admin 1.3.0现已可用。最新版本提高了对最新和最好的第三方库的兼容性，也代表了最后一个切入点，其中包括现在作为Spring Batch 3.0的一部分提供的Spring Batch Integration模块。
  • 您是否看到MichaelPlöd的网络研讨会，向他介绍了为什么向客户推荐Spring Framework ？我认为，该Slidedeck的在线版本对下一个项目中使用Spring的原因（有时是反对）提供了适当的实用见解。
  • ZeroTurnaround团队对XRebel进行了很好的介绍，XRebel是一个交互式概要文件，旨在向运行中的应用程序揭示性能问题。他们剖析了经典的Petclinic示例应用程序对会话的使用，并了解如何对其进行优化 。有趣的阅读！
  • Idriss Mrabti撰写了一篇不错的文章，解释了如何从国际化（“ i18n”）加载JSR 303验证消息MessageSource由Spring管理 -很方便！
  • Shazin Sadakath重新审视了他以前的一篇关于注册和使用Spring管理的文章的介绍。 Filter使用Spring MVC，这次是在Java配置中
  • Cloud Foundry团队产品总监James Watters 在2张幻灯片中对Cloud Foundry的功能进行了简述。I ..尝试过……但是找不到任何错误。如果您理解这两张幻灯片，那么您就有足够的理由来介绍Cloud Foundry。显然，这些并不是实现说明，而只是制定业务案例的一个明智的第一步。为什么为您和我提供Cloud Foundry？因为它是运行基于Spring的工作负载的理想平台。那里。可能甚至不需要整个幻灯片。 :)
