Spring Data JPA @Query定义中的SpEL支持

工程
托马斯·达里蒙特
2014年7月15日

Spring Data JPA允许使用存储库方法手动定义要由存储库方法执行的查询@Query注解。不幸的是，JPQL中的参数绑定非常有限，仅允许您设置一个值并提供某种类型转换。Evans发行列车的最新Spring Data JPA M1发行版通过添加支持减轻了这种痛苦
使用SpEL表达式在语句中使用动态绑定的参数@Query注释，在手动定义查询时提供了更大的灵活性。在此博客文章中，我将向您介绍此功能的功能。

方法参数表达式

SpEL支持提供对查询方法参数的访问。这使您可以直接按原样绑定参数，也可以在绑定之前执行其他操作。

@Query("select u from User u where u.age = ?#{[0]}")
List<User> findUsersByAge(int age);

@Query("select u from User u where u.firstname = :#{#customer.firstname}")
List<User> findUsersByCustomersFirstname(@Param("customer") Customer customer);

公开了用于索引访问的参数（ [0]在第一种方法中）或通过使用声明的名称@Param 。实际的SpEL表达式绑定由以下方式触发?#要么:# 。我们支持这两种类型，以使您与标准JPQL参数绑定（在查询定义中也可能出现）保持一致。
特殊类型的参数，例如Sort和` Pageable以其简单的类名作为变量公开。

高级SpEL表达式

尽管高级参数绑定是非常有用的功能，但SpEL的真正功能源于以下事实：表达式可以引用框架抽象或其他应用程序组件。SpEL的一个非常常见的情况是安全约束的定义。因此，如果我们可以限制查询以仅返回与当前经过身份验证的用户相关的结果，那将很酷：

@Query("select u from User u where u.emailAddress = ?#{principal.emailAddress}")
List<User> findCurrentUserWithCustomQuery();

如您所见，我们指的是Spring Security的principal 。那么，Spring Data SpEL支持如何与Spring Security集成。

SpEL EvaluationContext扩展模型

Spring Data公开了一个扩展点EvaluationContextExtension 。该接口允许实施者自定义EvaluationContext以非常详细的方式，但为了方便起见，我们提供了EvaluationContextExtensionSupport基类，以方便您仅实现您感兴趣的部分：

class SecurityEvaluationContextExtension extends EvaluationContextExtensionSupport {

  @Override
  public String getExtensionId() {
    return "security";
  }

  @Override
  public SecurityExpressionRoot getRootObject() {
    Authentication authentication = SecurityContextHolder.getContext().getAuthentication();
    return new SecurityExpressionRoot(authentication) {};
  }
}

对于我们的Spring Security扩展，我们扩展了EvaluationContextExtensionSupport并覆盖getRootObject()
方法并返回一个新的SecurityExpressionRoot实例，它公开了您从使用中已经知道的所有安全属性和表达式@PreAuthorize 。此步骤还使它们在我们的SpEL表达式中可用@Query注解。

我们需要采取的最后一步是将安全扩展注册为Bean：

@Configuration
@EnableJpaRepositories
class SecurityConfiguration {

    @Bean
    EvaluationContextExtension securityExtension() {
        return new SecurityEvaluationContextExtension();
    }
}

Spring Data JPA将拾取所有类型的bean EvaluationContextExtension并用它们来准备EvaluationContext用于评估在中定义的SpEL表达式@Query 。

现在，已有的扩展将使您能够利用Spring Security SpEL函数的全部功能。想象一下，一个存储库查询方法将返回BusinessObject当前用户是或的所有者的s BusinessObject如果当前用户是admin，则为s。查询方法定义如下所示：

interface SecureBusinessObjectRepository extends Repository<BusinessObject,Long>{

    @Query("select o from BusinessObject o where o.owner.emailAddress like "+
      "?#{hasRole('ROLE_ADMIN') ? '%' : principal.emailAddress}")
    List<BusinessObject> findBusinessObjectsForCurrentUser();
}

您可以在Spring-Data-Examples存储库中找到此处的代码片段的工作示例。
