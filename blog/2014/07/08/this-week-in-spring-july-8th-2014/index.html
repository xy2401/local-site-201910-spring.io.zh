春天的这个星期-2014年7月8日

工程
乔什·朗（Josh Long）
2014年7月8日

欢迎来到春季本周的另一期！我希望我们的美国读者在7月4日过得愉快，也希望其他所有人在星期五过得愉快！

像往常一样，本周有很多讨论，所以让我们开始吧！

  1. Spring Security负责人Rob Winch刚刚宣布了Spring Session的第一个里程碑 。通过Spring Session，可以轻松地从任何环境（例如，Web，消息基础结构等）访问会话，以供应商中立的方式支持集群，插入用于确定会话ID的策略，并在WebSocket处于活动状态时轻松地使HttpSession保持活动状态活性。这真是太酷了！而且，实际上，它可以解决我上周遇到的一个问题，该问题是我想从Web套接字处理程序访问HTTP会话！
  2. Andy Wilkinson刚刚宣布了Spring Boot 1.1.4版本 。新版本包括许多错误修复和一些小的新功能，包括对Tomcat 8的第一个稳定预览版的支持。
  3. Andy 还刚刚宣布了Spring IO的第一个维护版本，该版本更新了整个平台上的版本，包括Spring AMQP 1.3.5，Spring Boot 1.1.4，Spring Batch 3.0.1，适用于Apache Hadoop 2.0.1的Spring和Spring Framework 4.0。 6。干得好，安迪！
  4. Thomas Risberg刚刚宣布了适用于Apache Hadoop 2.0.1的Spring，该版本修订了依赖版本，包括Hadoop 2.4.1，Spring Framework 4.0.6。发布，将Spring Batch升级至3.0.1。发布，将Spring Boot升级到1.1.4。发布。
  5. StéphaneNicoll刚刚宣布Spring Framework 4.0.6现已可用 。此版本包括大约50个错误修正。
  6. Spring Social主管Craig Walls刚刚宣布了Spring Social Facebook 2.0.0。M1现在可用 。此版本支持新的Facebook v2.0.0 Graph API，该API引入了一些重大的，强制性的更改（感谢Facebook！）。
  7. Groovy项目负责人Guillaume LaForge宣布了新Groovy网站的Beta版，并在征求整个社区的反馈。
  8. 从战es中加入：7月15日，与Casey Doolittle和Phil Dutson一起进行有关疯狂项目的网络研讨会，以及他们如何使用Spring在24小时内构建购物车 。
  9. 查看即将举行的Michael Plod网络研讨会，他为什么推荐Spring-了解技术（以及一些业务）案例！迈克尔是一位出色的演讲者，经常参加会议。立即注册参加7月22日的活动。
  10. 7月29日，来自Crafter软件的Russ Danner将向我们介绍如何使用Spring和Groovy征服支持内容的Web和移动应用程序 。
  11. 去年，Spring工具忍者Andy Clement和Martin Lippert在SpringOne2GX 2013上做了精彩的演讲， 介绍了最新，最出色的工具 。
  12. 去年在2013年的SpringOne2GX上，朱利安·杜波依斯（Julien Dubois，您可能会从JHipster那里认识他。）就如何运行和性能调整经典的Spring Petclinic应用程序进行了精彩的演讲。
  13. Dobb博士在一篇不错的文章中介绍了新的Spring IO版本 。
  14. InfoQ的Matt Raible也对Spring IO进行了很好的介绍。看看这个！
  15. Literate Java博客上有一篇有趣的文章，介绍了Spring中的隐式 （以组件扫描为中心）与显式配置（使用XML）。我认为应该真正考虑Java配置，因为它提供了显式XML配置（集中式，易于配置的原因）和隐式但类型安全的Java注释和以组件扫描为中心的最佳配置组态。无论哪种方式，有趣的帖子！
  16. Spring Boot 现在正在评估 Thoughtwork的技术范围内。
  17. Codeleak.pl网站很好地介绍了Spring 4中较小但功能强大的功能之一-CGLIB代理不再需要默认构造函数！
  18. 我喜欢使用Spring和Angular.js的这一系列教程 。你还不如。干得好，克里斯托弗·汉高！
  19. 学徒的注意事项博客有关于如何使用一个很好的系列教程中的Spring MVC，Thymeleaf和引导在一起 。
  20. Autoletics博客很好地介绍了他们对Reactor项目进行概要分析的经验 。有趣的东西！一些观点也很有趣。如本综述的读者所知， Reactor是一个高度并行的事件驱动的IO就绪平台。而且，这是可笑的快。
  21. 我想向Paul Snively讲一点建议 ，他一直在使用Spring（和Scala）的1-2打孔组合以及Alfresco的Spring MVC的第三方集成来构建Alfresco驱动的解决方案 。真好！
  22. 无论您是否喜欢新的Groovy网站（尽管为什么不这样做？），您都会喜欢今天刚发布的新Groovy 2.3.4版本 ，其中包括对静态编译和AST转换的改进。
  23. HornetQ项目负责人Clebert Suconic正在邀请社区对HornetQ对Apache ActiveMQ项目的拟议贡献进行反馈。我们知道社区中的开发人员既了解又使用这两者，因此，如果此提议的合并影响到您，也许现在是一个审慎的时机！我个人很喜欢这个主意。ActiveMQ.next（长代号为“ Apollo”）似乎可以使用HornetQ贡献所提供的拥抱。而且，当然，Spring Boot现在支持ActiveMQ和HornetQ，所以无论哪种方式，您都可以设置！
