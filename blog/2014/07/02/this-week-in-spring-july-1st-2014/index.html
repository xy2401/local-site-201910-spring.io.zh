春天的这个星期-2014年7月1日

工程
乔什·朗（Josh Long）
2014年7月2日

欢迎来到春季本周的另一期！已经是七月了，夏天来了！你相信吗？时间肯定过得很快。像往常一样，本周我们要介绍的内容很多，所以让我们开始吧。

  1. 上周， 我们宣布了Spring IO Platform 。我赢得了这一奖，他们让我写了博客，但这反映了所有Spring项目之间一年多的内部讨论，计划，努力，合作与协调。我竭尽全力在此博客中介绍Spring IO Platform 。您应该阅读它。如果它听起来不那么棒，而且非常非常有用，那么比我所做的解释很糟糕！ ：D不要犹豫，伸出手 。Spring IO Platform是一种处理所有Spring产品组合项目中的依赖项的简单方法。使用Spring项目时，它也是一个更简单的逻辑模型。
  • Dave Syer博士刚刚宣布Spring Boot 1.1.3现已可用 。这主要是一个影响Windows用户的错误修正版本。
  • Spring Data负责人Oliver Gierke宣布了Spring Data Dijkstra SR1 。其中包括跨多个模块的59个错误修正。
  • 要使用Spring Boot吗？很多人也是如此。并且有很多入门方法。我试图在这个小博客中记录一些常用选项-Spring Initializr ， STS以及诸如JHipster之类的其他选项
  • Zoltan的博客上有一篇很好的文章，介绍了Spring的灵活缓存抽象 。该博客演示了如何轻松使用本地缓存（如EHCache）或分布式缓存（如Hazelcast）。酷的东西！另外，一定要检查一下Spring 4.1缓存抽象更新中的出色工作。
  • 正如该领域的读者所知道的，我们（以及RedHat，Typesafe，Netflix等）正在合作进行反应式流项目，该项目旨在为反应式应用程序提供基础层。例如，我们的实现基于Reactor项目，该项目为Spring 4的非常强大的websocket支持奠定了基础。Scalac.io博客有一个很好的示例，说明了如何将Typesafe的协作实现与RabbitMQ一起使用 。 Reactor的实现不应该太过遥远，与此同时，这是使用RabbitMQ的一种方法的很好看。
  • 说到RabbitMQ，Google Cloud Platform博客上有一篇很棒的文章，介绍如何设置RabbitMQ集群以通过Google的GCE（谷歌计算引擎）一次单击就每秒处理超过一百万条消息。太酷了！很高兴看到RabbitMQ被添加到GCE上受支持的中间件中。 （如果我不提Cloud Foundry以及基本上所有其他平台即服务都具有RabbitMQ的无痛设置，那我会很失落。）
  • Lingohub的Helmet M. Juskewycz刚刚发表了一个非常酷的博客，介绍了Lingohub为什么从Ruby on Rails迁移到Spring MVC 。我应该提到的是，即使在Pivotal（一家拥有各行各业的开发人员的公司）中，我们也看到了向Spring（和Spring Boot）的一些迁移 ！
  • Spring Security和Spring Data团队正在为我们准备一些很棒的东西。查看一些工作 ！
  • 我刚刚找到了有关如何将Spring框架添加到Eclipse项目的文章 。但是，提供的步骤并非特定于Spring。我建议您只使用Spring Tool Suite并转到File -> New -> Spring Starter Project 。有关STS如何简化使用Spring的更多信息，请观看此介绍视频 。
