Spring Framework 4.1-Spring MVC改进

工程
罗森·斯托扬切夫（Rossen Stoyanchev）
2014年7月28日

最近Juergen Hoeller
宣布上市
两个4.1发行候选版本中的第一个。
Brian Clozel跟进了
静态Web资源处理增强功能。
先前，Stephane Nicoll发表了有关
缓存和
与JMS相关的改进。
这篇文章的目的是总结Spring MVC的改进。

JDK 1.8 java.util.Optional现在支持@RequestParam ， @RequestHeader和
@MatrixVariable控制器方法参数，而ListenableFuture作为
返回值替代DeferredResult ，其中基础
服务（或对AsyncRestTemplate的调用）已经返回ListenableFuture。

@ResponseBody和ResponseEntity直接支持Jackson的@JsonView
序列化同一POJO的不同细节量的控制器方法，
例如摘要与详细信息页面。基于View的渲染也支持此功能
通过在特殊键下将序列化视图类型添加为模型属性。

杰克逊现在支持JSONP 。
对于响应主体方法，声明一个@ControllerAdvice ，如下所示。
对于基于View的渲染，只需配置JSONP查询参数名称
在MappingJackson2JsonView 。

@ControllerAdvice
private static class JsonpAdvice extends AbstractJsonpResponseBodyAdvice {

    public JsonpAdvice() {
        super("callback");
    }

}

请注意， @ControllerAdvice @ExceptionHandler是3.2中为@ExceptionHandler引入的，
@ModelAttribute和@InitBinder方法在全部或部分子对象之间共享
控制器。ResponseEntityExceptionHandler是全局的一个示例
通过将错误详细信息写入主体来处理异常
响应。在4.1中， @ControllerAdvice也可以实现
ResponseBodyAdvice在这种情况下将在控制器方法之后调用
返回，但在编写响应并因此提交响应之前。这有一个数字
@JsonView的有用应用程序的JSONP已经作为两个示例
建立在它上面。

两种新的HttpMessageConverter类型：
* Gson-占地面积更轻
比杰克逊已在Spring Android中使用。
* Google协议缓冲区
（只是错过了RC1，但已经在4.1 RC2的主服务器中使用了）
–高效和有效的跨服务
企业内的通信数据协议，但也可以公开为
浏览器的JSON和XML。这来自于
亚历克斯·安东诺夫（Alex Antonov） 。

MvcUriComponentsBuilder在4.0中引入，是一种构建与
通过控制器方法调用的控制器方法（类似于模拟测试）。
在4.1中，例如JSP的视图也可以通过引用控制器来建立到控制器的链接。
按名称映射。默认名称分配给每个@RequestMapping。例如
默认情况下，具有FooController方法的handleFoo被分配为“ FC＃handleFoo”，但
命名策略是可自定义的，也可以通过
@RequestMapping上的新name属性。一个新的mvcUrl Spring JSP标记使
这在JSP页面中易于使用。对于任何其他视图技术都可以这样做。

熟悉的ResponseEntity现在具有一个构建器样式的API，该API可以指导控制器
准备服务器端响应的方法，例如ResponseEntity.ok() 。
对于客户端，有一个新的RequestEntity也提供了构建器样式
指导准备客户端HTTP请求的API。

MVC Java配置和XML名称空间：
*视图解析器配置–如果您必须使用
内容协商，您将不胜感激。见
更新的文档 。
*增强的“视图控制器” –除了直接将URL映射到视图名称之外
无需控制器逻辑，视图控制器现在具有内置支持
用于重定向和设置响应状态。应用程序可以使用它来
配置重定向URL，使用视图呈现404响应，发送“无内容”
响应等。一些用例
在这里列出 。
*路径匹配–内置了常用的自定义项
并已反向移植到4.0。见
更新的文档 。
