Spring XD 1.0.0。M6发布

发布
马克·波拉克
2014年4月16日

Spring XD团队高兴地宣布，Spring XD Milestone 6现在可以下载了 。

这是我们最大的发行版！团队一直在努力工作，并且Milestone 6包含了许多新功能，这些新功能在可靠性，性能和用户体验方面都可以满足企业要求。以下是快速排行的前十名（排名不分先后），但是如果您查看发行说明，您将意识到从299列表中挑选出十个是多么困难。

  • 分布式和容错运行时 ：在多个xd-admin服务器之间进行领导者选举，并在出现故障的情况下自动将模块重新部署到其他xd容器。ZooKeeper被引入来管理集群及其部署状态。

  • 支持在YARN上运行XD ：在Hadoop YARN群集上而不是在需要管理的VM或物理服务器上运行管理节点和容器节点。有简单的配置和shell脚本使此过程非常容易。

  • 部署清单 ：部署流时，您可以提供一个部署清单，该清单描述了如何将逻辑流定义（例如，http | hdfs）转换为集群上的物理部署。您可以指定要部署的每个模块的实例数，还可以指定条件表达式（使用SpEL），该条件表达式评估群集中的每个可用容器，以确定与这些模块实例的最佳匹配。随着我们扩展清单以包括对数据分区策略的支持，这将是下一版本活跃开发的领域。

  • 机器学习评分算法的实时评估 ：与JPMML-Evaluator库集成，该库提供对多种模型类型的支持，并且可与从流行数据分析包（例如R）导出的模型互操作。通过提供XD的Analytic和MappedAnalytic抽象的实现，支持与其他库的集成。尝试使用基于经典虹膜数据集的示例应用程序 。

  • 更新的UI ：重新设计和重写具有现代外观的UI。

  • 增强的DSL自动完成功能：现在，在创建流和作业时，选项卡补全在–definition选项的值内起作用。创建流和作业从未如此简单。通过xd-shell尝试一下！

  • 批处理作业生命周期的默认流 ：批处理作业是事件数据的源。在“批处理作业”上添加一个水龙头，以在作业成功/失败，各个步骤动作甚至项目级动作时接收事件。此功能以及通过将消息发送到命名通道来启动作业的支持，使您的应用程序在批处理域和流传输域之间具有复杂的事件驱动流。

  • 改进的服务器和模块配置选项 ：服务器和模块配置已分离，以帮助独立升级它们的生命周期。使模块配置更具模块化，同时允许相关模块之间进行共享配置。

  • 高性能TCP / UDP源 ：基于Reactor项目-商品硬件上的Reactor IP源每秒可消耗约100万毫秒。

  • FTP到HDFS作业 ：即时支持将文件从FTP传输到HDFS的作业。这些作业利用了另一个新功能：跨多个XD容器的步骤分区。
