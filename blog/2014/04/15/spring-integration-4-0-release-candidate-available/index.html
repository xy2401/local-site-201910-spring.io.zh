Spring Integration 4.0版本候选可用

发布
阿尔特姆·比兰（Artem Bilan）
2014年4月15日

我们很高兴地宣布Spring Integration 4.0发行候选版（4.0.0。RC1）现在可用。
请使用带有Maven或Gradle的Milestone库 ，
下载发行档案 ，
或查看项目主页以获取更新文档的链接以及Maven / Gradle配置的详细信息。

该版本包含一些错误修复，一些新功能以及将于4月底发布的GA版本的进一步改进。

这是自上一个里程碑以来的主要变化的摘要

__ @ Poller和@InboundChannelAdapter__

在M4版本中宣布的注释支持的广泛改进的基础上，
新的@Poller注释已添加到每个消息注释（ @ServiceActivator ， @Router等等。）。
的@Poller注解属性可用于配置poller的选项inputChannel消息注释的内容。
这样可以将带注释的端点PollingConsumer s。以前，带注释的端点只能使用SubscribableChannel并受事件驱动。

这也为我们提供了介绍@InboundChannelAdapter方法消息注释。
现在，可以在Spring Boot应用程序中不使用XML对其进行配置，例如：
Java
@EnableAutoConfiguration //启用集成基础架构
@MessageEndpoint //使此类成为集成组件
@PropertySource（“ classpath：integration.properties”）//属性-占位符配置
公共类集成{

public static void main(String[] args) throws Exception {
    ConfigurableApplicationContext ctx = SpringApplication.run(Integration.class, args);
    Thread.sleep(10000);
    ctx.close();
}

@InboundChannelAdapter(value = "countChannel",
     poller = @Poller(fixedDelay = "${poller.interval}", maxMessagesPerPoll = "1"))
public Integer count() {
    return this.counter.incrementAndGet();
}

@ServiceActivator（inputChannel =“ countChannel”）
公共无效foo（整数有效负载）{
System.out.println（有效载荷）;
}

分布式MetadataStore和LockRegistry

的MetadataStore用于存储各种类型的通用元数据（例如，最后一次提要的发布日期
已处理的条目）以帮助组件，例如Feed Adapter ，以维持状态并避免重复。
对于分布式环境和多实例应用程序，以及要在所有应用程序之间维护持久元数据状态
应用程序重启， RedisMetadataStore和GemfireMetadataStore此版本已引入。

例如，一个可以配置FileSystemPersistentAcceptOnceFileListFilter ，
可以配置为使用其中一种分布式MetadataStore s。这允许filter keys成为
在多个应用程序实例之间共享，或者当多个服务器正在使用网络文件共享时。

对于类似的分布式（跨JVM）环境，当只有一个实例可以访问该对象时
（例如AggregatorHandler对于MessageGroup消息到达时），已分发LockRegistry
引入了实现- RedisLockRegistry和GemfireLockRegistry 。

优先通道和消息存储

在此版本中，您可以配置使用外部MessageStore 。

Twitter搜索网关

现有的推特仅允许对每个轮询执行固定查询。为了获得更大的灵活性，
