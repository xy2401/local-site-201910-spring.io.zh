Sagan项目：升级到JDK 8

工程
克里斯·比姆斯
2014年4月18日

我确信大多数读者都知道， Java SE 8于上个月发布 。结合最近发布的STS 3.5.0及其对JDK 8的完全支持，我们认为现在是升级Sagan来利用新语言功能和API的理想时机。

注意 ：还不熟悉Sagan项目 ？是Spring参考应用程序为该博客以及spring.io上的所有其他内容提供了动力。有关更多详细信息，请查看本系列的前 三篇 文章 。

自发布之日起，Cloud Foundry的Java buildpack就为JDK 8提供了支持。为了开始使用它，我们创建了自己的buildpack 分支 ， 将JDK版本从1.7.0升级到1.8.0，并配置了Gradle构建以在将Sagan部署到生产中时使用我们的fork。

注意 ：在不久的将来，1.8.0将成为Cloud Foundry构建包的默认版本，并且不再需要描述的分支。

您可能已经注意到， Sagan使用Travis CI进行持续集成和部署，并且更新Travis配置以使用JDK 8非常简单。

当然，升级最有趣的方面是对代码本身的更改。查看拉取请求＃348中的提交，您将看到更改分为几类，包括重构以使用lambda表达式以及利用JDK 8强大的新Stream API。

对于一个非常简单的示例 ， 很高兴从这样的代码开始：
java List projectVersions = new ArrayList<>(); for (ProjectRelease projectRelease : project.getProjectReleases()) { projectVersions.add(projectRelease.getVersion()); }

对此：
java List projectVersions = project.getProjectReleases().stream() .map(ProjectRelease::getVersion) .collect(Collectors.toList());

或者，在单元测试中 ，从充满仪式的内容中：
java mockMvc.perform(createPostRequest) .andExpect(new ResultMatcher() { @Override public void match(MvcResult result) { String redirectedUrl = result.getResponse().getRedirectedUrl(); assertThat(redirectedUrl, startsWith("/admin/blog")); } });

严格而易读：
java mockMvc.perform(createPostRequest) .andExpect(result -> { String redirectedUrl = result.getResponse().getRedirectedUrl(); assertThat(redirectedUrl, startsWith("/admin/blog")); });

当然，您在上面看到的内容仅能初步了解JDK 8中的流，lambda表达式和方法引用的可能性。有一些新概念需要学习，但是您会很高兴的！为此，我们建议阅读Venkat Subramanian出色的Java函数式编程 。
