Spring Boot 1.0 GA发布

发布
菲尔·韦伯
2014年4月1日

我非常荣幸地代表整个Spring Boot团队宣布Spring Boot 1.0的全面上市！您可以在此处下载具有重要安全修复程序的1.0.1。

您将在projects.spring.io/spring-boot以及我们不断增长的“入门”指南集合（其中大部分使用Spring Boot）中找到所需的一切。

提出“改进无容器Web应用程序体系结构”的最初要求已经18个月了，该要求催生了Spring Boot。从那时起，我们看到54个不同的贡献者 提交了1720份文章 ，我们已经结束了549期 ，并且代码分叉了398次 。谢谢！

为什么没有容器？当今的PaaS环境已经提供了许多管理，横向扩展和可靠性功能，因此我们专注于使Spring Boot成为超轻型容器，非常适合在云中进行应用程序或服务部署。如果您还没有看到Spring Boot的实际应用，请参阅经典的“ Hello World！“您可以使用CLI工具实际运行的Web应用程序。

groovy @RestController class Example { @RequestMapping("/") String helloWorld() { "Hello World!" } }

CLI使用精湛的Groovy语言来编译您的代码。当然，您也可以自由地直接将Spring Boot库与Java或任何其他基于JVM的语言一起使用来编写应用程序。请务必查看详尽的参考文档以获取完整的功能细分。

如果您有兴趣查看使用Spring Boot构建的应用程序的更大示例，请尝试spring.io本身。我们最近将整个站点开源，作为Spring参考应用程序。

对于较小的示例（如服务），请尝试“构建RESTful Web服务”指南。它显示了如何在<100 LoC中编写服务非常简单，甚至可以在一条推文 （140个字符）中放入可执行程序！您可以在Dan Woods的InfoQ优秀文章中阅读有关微服务和引导的更多信息。

感谢Java Buildpack团队，您还将发现Spring Boot应用程序可以无缝部署到Cloud Foundry ；当然，您也可以将Boot应用程序与其他云提供商一起使用，或者直接在自己的数据中心中使用。

最后，如果您是STS用户，请查看最新的v3.5版本 ，其中包括对Spring Boot的集成支持（请参见下面的视频）。如果您更喜欢其他IDE（或者您更像Vim / Emacs这样的人），则可以使用start.spring.io初始化您的项目。


再次感谢所有测试，编写并提交了Spring Boot拉取请求的早期采用者。您的反馈对我们来说是无价的，所以请继续保持 ！
