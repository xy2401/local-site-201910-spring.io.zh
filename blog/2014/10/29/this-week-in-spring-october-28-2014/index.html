春天的这个星期-2014年10月28日

工程
乔什·朗（Josh Long）
2014年10月29日

您好，纽约市！今晚，我将向Cloud Foundry用户组致辞，以介绍如何构建在Cloud Foundry上运行的Spring Boot和基于Spring Cloud的微服务 -希望您加入我们！如果您在波士顿，那么当我星期四在波士顿进行同样的演讲时 ，请出去玩 ！

  • Spring Social领导者以及所有Spring Ninja忍者Craig Walls 宣布了Spring Sync项目 ，然后对其进行了出色的介绍 ，该文章摘自该帖子，因为它在其中得到了很好的说明。基于补丁的交换。因此，基本上，客户端/服务器通信（如REST）中缺少的一块（在基本REST，HATEOAS和超媒体，OAuth和安全性之后，Spring能够以一流的方式全部支持）现在在这里！如果这个星期你读一件事...
  • Spring Batch主管Michael Minella 刚刚宣布了Spring Batch 3.0.2
  • Spring Integration摇滚明星Artem Bilan刚刚宣布了Spring Integration 4.1 RC1-带有websocket适配器，路由滑模支持，JDK Optional支持，Redis Queue网关，基于Reactor的PersistentQueue实施和可以跳过民意测验的民意测验（而且-哇！-还有更多！）现在可用！嗯棒极了。我一直想知道为什么我们没有听到来自Spring Integration团队的消息-他们很忙 ！
  • Spring Integration和AMQP忍者Artem Bilan在Spring Integration发布后并没有为自己的桂冠而骄傲，他刚刚宣布了Spring AMQP 1.4 RC1版本 ，该版本提供了一种基于声明式，基于注释的模型，用于以与新引入的注解相同的样式构建消息侦听器容器，用于JMS的Spring框架中的组件模型等。
  • 然后，在11月4日举行的Syer博士的网络研讨会上记下您的日历，以介绍带有Spring和OAuth2的微服务安全性
  • 继续11月18日举行的微服务网络研讨会系列，为您的Redmonk Made for Each of彼此的Matt Stine和Donnie Berkholz记下日历：微服务+ PaaS
  • 11月20日，与我和JRebel团队的Adam Koblentz一起谈论JRebel 6.0和Spring Boot！
  • 微服务网络研讨会系列的第二次回放已经发布！（顺便说一下 ，第一个在这里 ）。因此，不要错过Matt Stine的史诗网络研讨会： 持续交付的架构：带有Pivotal CF和Spring Cloud的微服务
  • 我们刚刚发布了更新的入门教程（与使用Spring构建REST服务的非常有用但侧重于目标的Spring IO指南相对 。我们已经对其进行了迭代，欢迎您提出任何反馈。
  • 您想在Spring Boot中使用Jade模板吗？看看这个方便的Spring Boot Jade 启动器 ！
  • 查看Spring Data忍者Thomas Darimont的示例Spring Boot和jOOQ的示例
  • Spring Data负责人Oliver Gierke在Stack Overflow上回答了一些我认为值得一读的问题。第一： 为什么将Spring Boot应用程序部署到WebSphere 7应用服务器会导致ClassCastException？ 然后： 我应该使用Java 8默认方法来手动实现Spring Data存储库方法吗？ 两者都是很棒的读物，请随时检查它们！
  • 签出使用Kotlin语言编写的Spring Boot应用程序
  • 我们的好朋友Rafal Borowiec整理了一篇不错的文章，介绍了他的步骤，以获取在OpenShift 上运行的Java 8上的简单Spring Boot应用程序的步骤 。当然，Spring Boot具有尽可能的可移植性，但是他的示例使用的Java 8并不完全符合您希望在OpenShift上使用的方式（到目前为止），因此他遇到了一些麻烦，并且很好地记录了所有内容。他们。看看这个！
  • 我们的朋友Toshiaki Maki 在Spring Cloud配置服务上撰写了史诗（日语）帖子。我的日语有点生锈（_ahem！_），但即使通过Google Translate也能提供有用的信息。
