Pivotal在英国伦敦举行的Spring eXchange 2014上

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2014年10月29日

您要在11月6日至7日前往伦敦的Spring eXchange吗？

在不到一周的时间里，Spring专家将与数百名Spring爱好者一起在Spring中学习和分享技能两天。涵盖的主题将包括大数据和Spring XD，Spring Data，REST，微服务和现代体系结构。在星光熠熠的阵容中，Pivotal专家将有很多人参加。请参阅下面的一些重要演讲者和演讲。

前往节目页面了解更多详细信息和完整的发言人列表。 要预订您的位置并保证有座位，请访问此处的预订页面 。Pivotal赞助了Spring eXchange，因此请通过Twitter @springcentral #springx与我们联系，以在会议期间取得联系！

伦敦Java社区成员的特别优惠 ：想要成为我们2014年春季eXchange的重要演讲嘉宾，并获得免费的演出门票吗？我们将把它们提供给前15个LJC成员，这些成员在@springcentral和@cloudfoundry上通过推特网络服务上已部署的Spring应用程序的URL向我们发送推文。先到先得，先到先得，我们15位发言人有15张门票！我们将通过查看Spring应用程序的URL来验证获胜者，如果您获胜，将在metup.com上寻找伦敦Java社区的成员，因此请确保您具有metup.com的个人资料并成为会员！

星期四-2014年11月6日
10:00 AM

  • David Syer博士
  • 春季新消息
  • 在2014年Spring eXchange的开幕主题演讲中，Dave Syer讲述了Spring生态系统的最新发展，可以追溯到大约12个月左右。自从2013年Spring Boot上街以来，Spring再次变得有趣，所有很酷的孩子都想玩它。在Spring IO和Spring Cloud的背景下，Dave还将涵盖Foundation项目（如Spring 4.1和4.2，Spring Batch 3.0，Spring Integration 4.1和Spring Data Evans）的最新更新，以及诸如Spring XD for Big Data的执行层项目。和Grails用于全栈Web应用程序。您会惊讶并高兴地看到构建应用程序和服务并将它们粘合到分布式系统中是多么容易，特别是如果您没有使用过Spring或最近没有使用过它。

上午10:45

  • 马克·波拉克
  • 大数据，很少的时间：消除创建大数据应用程序的复杂性
  • 创建大数据应用程序是一种相对较新的技术，开发全面的解决方案可能会面临挑战。一些关键挑战来自复杂领域的融合，例如流分析，批处理工作流程编排以及数据导入和导出。此外，建立此类应用程序通常需要将不同的产品缝合在一起。Spring XD是一个旨在解决大数据复杂性的新项目。它基于成熟的Spring技术将这些域联系在一起，并提供了可扩展的，分布式的和容错的运行时环境。您无需成为Java开发人员即可上手。简单的DSL可让您构建复杂的流和作业定义，只需几个简单的步骤即可对机器学习算法进行评分。
  • 斯蒂芬·尼科尔（ Brian Clozel）
  • Spring入门
  • Spring不仅仅是一个单一的框架-它是一个具有多个项目的丰富生态系统，所有项目都有助于构建现代应用程序。但是所有这些对于Spring初学者来说可能有点吓人。StéphaneNicoll和Brian Clozel将带您了解Spring框架背后的核心概念和思想模型。它们将介绍框架的主要方面，还向您介绍其他Spring项目中的关键功能，包括最新的Spring IO Platform。在本次会议之后，您将对所有Spring投资组合项目以及所有成功在2014年Spring成功启动的所有关键方法保持一致。

12:00 PM

  • 斯图尔特·威廉姆斯
  • Spring和RTI的异步设计：每秒1M个事件
  • 应用程序设计师通常必须选择在何处进行灵活性以换取特异性（因此通常是性能）。知道何时何地这样做是一门艺术，需要经验。本讲座将分享数十年的经验，这些经验将有助于您做出这些决定，以及使用Spring项目的最新版本开发Pivotal成功的实时智能（RTI）产品的经验教训：集成，数据，引导，MVC / REST和XD。逐步浏览RTI架构将为解释Spring如何每秒执行数百（百万）个事件/操作以及您现在可以在自己的Spring应用程序中使用的技术提供基础，以最大程度地减少资源利用率并提高性能。
  • 乔什·朗（Josh Long）
  • 使用Spring Boot构建“ Bootiful”应用程序
  • 好吧，所以“可恶”也许永远都行不通，但是我尝试过，这也是值得的，因为您正在阅读本文。Spring Boot是Pivotal的Spring团队提供的一种新的基于约定的配置中心框架，它将Spring的灵活性与常规的常识默认值结合在一起，使应用程序开发不仅轻而易举，而且令人愉快！加入Spring开发倡导者Josh Long的行列，看看Spring Boot是什么，为何引人注目，为什么要在下一个应用程序中考虑使用它（REST，Web，批处理，大数据，集成等等）。以及如何开始。

下午3:15

  • 埃里克·博塔德
  • Spring XD内部导览
  • 将Stream部署到Spring XD集群时会发生什么？流处理和数据分区如何工作？当Spring XD容器发生故障时，群集如何恢复？在本次有关Spring XD的深入探讨中，我们将涵盖所有这些主题，并使用实时演示来介绍更多内容。
  • 斯蒂芬·马尔蒂尼（Stephane Maldini）
  • 使用Reactor的反应式微服务
  • 微服务架构现已成为众多公司的标准。在构建微服务时要解决的问题中，开发人员可能需要考虑异步。Reactor为任何JVM应用程序及以后的应用程序提供了渐进的，无目的的并发处理。它不仅是方便的轻量级工具包，而且还是Reactive-Streams规范的一部分，因此可以与RxJava或Akka之类的朋友进行互操作。快来发现在Reactor之上构建微服务时的一些技巧和窍门，了解它是如何在Reactive-Streams规范之上构建的，以及为什么这在当今的软件开发中会改变游戏规则。

星期五-2014年11月7日

上午11:15

  • 本·黑尔
  • 使用Spring Boot设计REST-ful微服务
  • 随着数据驱动的应用程序无处不在，提供数据的服务正在激增。随着团队对越来越多的此类服务负责，以尽可能轻便的方式设计和实施团队变得至关重要。本课程将介绍如何将微服务设计为RESTful API，并使用Spring Boot以最少的代码实现它们。它将着重于使用REST和HATEOAS进行API设计，实时编码将一直发展到在云中运行的成熟应用程序。
  • 乔什·朗（Josh Long）
  • 使用Spring Cloud构建“ Bootiful”应用程序
  • 好吧，所以“可恶”也许永远都行不通，但是我尝试过，这也是值得的，因为您正在阅读本文。Spring Boot是Pivotal的Spring团队提供的一种新的基于约定的配置中心框架，它将Spring的灵活性与常规的常识默认值结合在一起，使应用程序开发不仅轻而易举，而且令人愉快！加入Spring开发倡导者Josh Long @starbuxman，看看Spring Boot是什么，为什么它会引起轰动，为什么您应该在下一个应用程序（REST，Web，批处理，大数据，集成等等）中考虑使用它呢！以及如何开始。让我们利用虚拟JUG的动态特性：在介绍技术时，我将实时编码并（尝试）回答有关Spring和Spring Boot的所有问题。

下午12:30

  • 奥利弗·吉尔克（Oliver Gierke）
  • Spring Data REST-存储库符合Hypermedia
  • Spring数据存储库抽象极大地简化了Java中数据访问层的实现。最重要的是，总括项目的REST模块提供了一个编程模型，可以轻松地公开超媒体驱动的REST资源以与存储库进行交互。演讲讨论了Spring Data REST实现的模式，互联网标准和规范如何影响设计以及模块如何公开元数据以构建松耦合的客户端。该演示文稿包含许多动手代码示例，以演示所讨论的方面。

  • Inside Spring.io：生产型Spring参考应用程序
  • 您是否想看一下为spring.io网站提供动力的新开源参考应用程序？然后加入Spring Guru Brian Clozel，他将研究协调平台背后的一些功能。本演讲将涵盖以下内容：

    • 在此处了解有关Spring.io的更多信息！

下午2:30

  • 托马斯·达里蒙特
  • Spring Data有什么新功能？
  • 本讲座将对最新的Spring Data发布系列中引入的新功能进行广泛概述。我们将介绍Spring Data Commons的最新添加和改进，Spring Data Commons是商店特定模块之间共享的模块。然后，我们将研究各个商店模块的最新和最大功能，例如JPA，MongoDB，Neo4j，Solr以及社区功能。
  • 安迪·威尔金森
  • 记录RESTful API
  • API文档是使其易于理解和易于使用的重要组成部分。RESTful API没什么不同。在本次演讲中，我们将研究RESTful API文档中应包含的内容，以及同样重要的是，不应包含的内容。在考虑为生成使用Spring构建的RESTful API的文档的多种不同方法之前，我们将查看大量有关API文档好坏的现实示例。

下午3:45

  • 克里斯多夫·斯特罗布尔
  • 使用Spring启动搜索
  • 在Google时代，搜索数据已成为关键任务。我们要寻找，而不是寻找！本节重点讨论将搜索集成到您的应用程序中的解决方案。我们将看看Spring Data对Apache Solr，Elasticsearch和Mongodb的搜索抽象，并使用Spring Boot进行一些自动配置（保证100％XML免费设置）。您选择技术，Spring为您提供了前进的工具。
  • 罗伯·温奇
  • 使用Spring保护RESTful API的状态
  • 了解如何正确保护RESTful端点，如何在将安全性应用于RESTful API时探索一些常见的陷阱，并了解Spring Security中的新功能如何极大地简化RESTful API的保护。 RESTful架构的许多优点使其成为设计基于Web的API的标准方法。例如，REST原则规定我们应该利用标准的HTTP动词，这有助于使我们的API保持简单。被认为是RESTFul的服务器组件应该是无状态的，这有助于确保它们可以轻松扩展。我们可以利用缓存来获得进一步的性能和可伸缩性优势。但是，REST和安全性的最佳实践通常似乎冲突。如何在无状态应用程序中对用户进行身份验证？安全资源如何还支持缓存？安全最佳实践发展得如此迅速，这一事实使RESTful端点的安全变得更加复杂。在这次演讲中，Rob将讨论如何正确保护RESTful端点。在将安全性应用于RESTful API的过程中，我们将探索一些常见的陷阱。最后，我们将看到Spring Security中的新功能如何极大地简化保护RESTful API的安全。
