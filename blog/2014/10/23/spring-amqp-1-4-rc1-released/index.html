Spring AMQP 1.4 RC1发布

发布
阿尔特姆·比兰（Artem Bilan）
2014年10月23日

我们很高兴地宣布Spring AMQP 1.4的发布候选版本已经发布； GA计划于11月初发布。

首先，感谢所有提供里程碑1.4 M1版本反馈并提交报告（错误或新功能）的人。
特别感谢StéphaneNicoll ，他介绍了基于注释的消息
听众（ @RabbitListener ）和新RabbitMessagingTemplate 。

此版本中包含的功能和修复（自里程碑以来的更改）

  • 借助基于注释的消息侦听器，您可以使用@SendTo注释以将方法的结果作为回复消息发送（当没有replyTo在入站消息中）：
@RabbitListener(queues = "capitalize")
@SendTo("capitalize.reply")
public String capitalize(String s) {
	return s.toUpperCase();
}

（基于注释的侦听器需要Spring Framework 4.1.x）。

  • 一个新的RabbitMessagingTemplate已经引入，允许使用spring-messagingMessage抽象替代Spring AMQP Message （春季信息Message带注释的侦听器也支持）。

  • 登陆回AmqpAppender现在可用，其配置类似于现有的Log4j AmqpAppender ：

<appender name="AMQP" class="org.springframework.amqp.rabbit.logback.AmqpAppender">
	<layout>
		<pattern><![CDATA[ %d %p %t [%c] - <%m>%n ]]></pattern>
	</layout>
	<abbreviation>36</abbreviation>
	<applicationId>MyApplication</applicationId>
	<routingKeyPattern>%property{applicationId}.%c.%p</routingKeyPattern>
	<generateId>true</generateId>
	<charset>UTF-8</charset>
	<durable>false</durable>
	<deliveryMode>NON_PERSISTENT</deliveryMode>
</appender>
  • 的RabbitTemplate现在可以用AbstractRoutingConnectionFactory与expression确定目标的属性ConnectionFactory取决于RabbitTemplate操作：
<rabbit:template connection-factory="RoutingConnectionFactory"
     send-connection-factory-selector-expression="messageProperties.userId"
     receive-connection-factory-selector-expression="#root == 'fooQueue' ? 'foo' : null"/>
  • 除此之外mandatory-expression属性已添加到RabbitTemplate设置mandatory每个已发送消息的标志：
<rabbit:template connection-factory="connectionFactory"
      mandatory-expression="messageProperties.contentType == 'application/json'"/>

（先前， mandatory是在模板级别设置的）。

  • 的AbstractRoutingConnectionFactory现在也支持MessageListenerContainer ，其中key对于
    目标ConnectionFactory基于配置queueNames数组

请参阅发行说明
和项目页面以获取更多信息。
