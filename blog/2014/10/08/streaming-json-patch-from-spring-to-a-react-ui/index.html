从Spring将JSON补丁流式传输到React UI

工程
布莱恩·卡瓦里尔
2014年10月8日

我们正在探索帮助开发人员创建易于与Spring后端集成的丰富，现代的前端的方法。如果您今年参加了SpringOne，您已经看到了我们一直在努力的一些事情：

  1. 由Greg Turnquist，Oliver Gierke和Roy Clarkson 提供的Spring Data REST中的超媒体支持
  2. RaveJS： John Hann的JavaScript应用程序的Spring Boot概念
  3. 差异同步和JSON补丁作者Craig Walls和我

简而言之，我们希望简化Spring后端与客户端之间的有效通信，并轻松集成最佳和最受欢迎的客户端技术。

React +流式更新

JSON修补程序是一种用于向结构化数据发送增量更改的格式。我认为尝试通过STOMP将JSON Patch格式的流更新一直流到Web UI会很有趣。

Craig Walls已经使用新的Spring Sync项目通过STOMP上的JSON Patch将分数更新推送到浏览器，从而建立了简单的足球分数概念证明 。我能够获取他的POC，放入RaveJS，添加客户端反应流，并将更新集成到React UI组件中。

您可以在我的rave-most-react分支中的github上找到完整的代码 。该服务器与Craig的原始版本相同。Web客户端是操作所在。在深入了解某些细节之前，请查看main.js以获得对应用程序的总体了解 。

起步

我使用RaveJS和npm来管理客户端依赖性，因此启动很简单：

> cd src/main/resources/public

> npm init

<answer a few questions>

> npm install --save rave most jiff react rave-load-jsx rave-node-process stompjs

然后，我只需要在现有HTML中使用一个脚本标签，就可以开始编码了！

<script src="node_modules/rave/rave.js"></script>

介绍most.js

多亏了Craig，我已经有了通过STOMP流到客户端的更新流（JSON补丁格式）。我决定使用补丁流的最好方法是（惊奇！）实际使用流 。

Most.js是cujoJS的新JavaScript反应流包。它提供了一小组但功能强大的API，用于创建，转换和使用事件流。我用它包装了stompjs API。

事实证明，有两个STOMP订阅：一个带有所有乐谱数据的初始完整副本，另一个带所有后续更改。通过使用jiff.js在到达时应用JSON补丁，我能够将这两个订阅打包为一个代表“最新分数集”的反应性流。

这一段代码创建了一个从STOMP订阅到初始数据的流，并获取了第一个事件（所有乐谱的完整快照），并将其与仅包含JSON补丁更新的第二个流组合在一起，以生成变化的乐谱视图随着时间的推移。

function getScoresStream(initDestination, updateDestination, client) {
	// Create a stream containing one full copy of the data, and
	// flatMap that to a stream containing the time-varying
	// current set of scores, by accumulating each patch
	// and emitting the updated scores data.
	return getInitialDataStream(initDestination, client)
		.flatMap(function(data) {
			return getUpdatesStream(updateDestination, client, data);
		});
}

function getInitialDataStream (initDestination, client) {
	// Await a copy of the data from the STOMP subscription
	// that is sending the full scores data, then unsubscribe.
	return streamFromStompJson(initDestination, client)
		.take(1);
}

function getUpdatesStream (updateDestination, client, data) {
	// Incrementally accumulate patches from the STOMP subscription
	// that is carrying JSON Patches onto the scores data to produce
	// an updated view of the scores.
	return streamFromStompJson(updateDestination, client)
		.startWith([])
		.scan(updateWithJsonPatch, data);
}

当流结束时，most.js还会自动清理基础资源。在完整分数数据的初始副本到达后，很容易安排取消第一个订阅，仅留下一个订阅：更改。

用React制作UI

现在，我有了一个代表最新分数的most.js流，我创建了一个React组件来显示它们 。我安装（见上） rave-load-jsx扩展 ，它是由社区成员创建的RaveJS扩展，它允许在RaveJS中直接加载JSX组件。我要做的就是创建一个Scoreboard.jsx文件并开始编写一个简单的React Scoreboard组件。

React组件具有内部state包含将用于呈现组件的数据的对象。我要做的就是在创建组件时传递最新的分数流，然后让组件观察流并更新其state 。相关的代码只有几行：

// this.props.scores is the scores stream provided when the
// Scoreboard component is created

this.props.scores.observe(function(scores) {
	self.setState({ scores: scores });
});

React负责自动保持DOM与state 。

从位到像素

公认这是一个玩具应用程序。但是，它显示了很多强大的概念，它们在很小的空间内一起工作，只用很少的代码：小型的，服务器生成的增量通过STOMP + WebSocket传递到客户端上的反应流，由React组件观察。变化从Spring后端一直流到UI，从位到像素。

我希望这个小小的应用程序也可以让我们对我们的发展方向有所了解。我们希望提供工具和客户端软件包，以帮助开发人员构建与领先的客户端技术和Spring后端集成的丰富，现代的客户端应用程序。
