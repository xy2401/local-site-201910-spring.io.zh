Spring 的这个星期-2014年9月30日

工程
乔什·朗（Josh Long）
2014年10月1日

欢迎来到春季本周的另一期！ 这个星期，Spring团队将在JavaOne上 ！过去将会有很多值得一看的地方 ！我们还在摊位上闲逛，与尽可能多的用户交流。看看我刚刚给出的链接，列出JavaOne上所有很酷的Pivotal（和非Pivotal）演讲。（啊当然，如果您发表我的演讲-Spring4TW ，我个人将喜欢它！ -明天上午8:30，在西里尔·马格宁II / III广场55上 ！）。


除了在Spring上使用NetFlixOSS进行IoT，大数据和微服务演示外 ，学生们还在编写编码挑战以使用Spring Boot构建REST服务，并在不到 15分钟的时间内将其推向Pivotal Web Services ，以从American Apparel获得preshavedyak帽衫 。了解当今市场上最好的Java PaaS； Java 8 / Tomcat 8支持是在几天前发布的，并且对Spring，Groovy和Grails都提供了强大的支持。


  • 我真的很喜欢Michael Plod的“ Why I Recommend Spring网络研讨会” ，现在可以在线重播

  • 上周，Activiti BPMN2引擎联合创始人Joram Barrez和我就如何构建以流程（阅读：工作流）为中心的Spring应用程序进行了演讲， 该应用程序现在也可以在线获得。

  • 2014年10月21日，星期二，与Matt Stine一起参加微服务架构系列2： 持续交付的架构设计：带有Pivotal CF和Spring Cloud的微服务

  • 2014年10月23日，星期四，2014年使用Spring Boot和Vaadin构建“启动” UI

  • 2014年11月4日，星期二-微服务架构系列＃3-与Dave Syer博士 一起探讨Spring和OAuth2的微服务安全性

  • 缺少微服务架构＃1？在Spring Developer Channel上观看使用Spring Boot构建“ Bootiful”微服务的重播

  • Spring Boot 1.1.7现已发布 ！

  • 在Spring Boot Web应用程序中使用Active Directory LDAP设置Spring Security （昨天我刚在Spring BOF上被问到这个问题！）

  • 最近有一个关于微服务的有趣辩论。“叔叔”鲍勃·马丁（Bob Martin）写了一个帖子，声称.jar s和Ruby gem可以为您带来微服务的相同好处 。乔治·西罗尼（Giorgio Sironi）在他的博客上有一个很大的反驳（我恰好同意。） 。关于InfoQ的整个讨论甚至都有一个很好的总结。

  • 我真的很喜欢Matt Stine的PaaSes 风洞比较 。首先：Heroku。我等不及要看该系列的后续文章！

  • 这篇关于使用Spring Data Redis的帖子是在去年发布的（并包含在TWiS ..中）， 但是它又出现了，我仍然认为这是值得阅读的

  • eHarmony博客上有一篇不错的文章， 介绍如何设置Spring Batch Admin（棘手的部分..）

  • 在“ 必须相信它”部门中，您是否知道有一个Spring for Delphi（Spring4D）项目 ？有受Spring启发的C ++，ColdFusion，JavaScript，.NET，Python以及许多其他工具。酷将其添加到列表中。

  • Intertech博客在其史诗般的 Spring Integration教程中还有另一部分 。这个是服务激活器 ，基本上是Spring Integration的转义线，因此您可以调用常规Java对象。

  • 本杰明·鲍尔（Benjamin Ball） 撰写了有关使用RabbitMQ和Spring的精彩文章

  • Yashwant Chavan在构建简单的Spring Boot（带有Spring MVC）示例并在Eclipse中对其进行编辑方面有一篇不错的文章。

  • 我们在Codecentric的朋友又来了！他们刚刚发布了他们的Spring Boot Admin的第一版 。看起来很适合Spring Cloud项目…

  • 这一定是有史以来最酷的Spring Boot演示之一 ！小心，如果您晕车，不建议全屏观看！ :)

  • 您是否坚持使用Apache CXF，但仍然想要一流的安全性？这篇文章介绍了如何在几分钟内将Spring Security插入CXF之上

  • Biju Kunjummen很好地阐明了Spring与WebApplicationInitializer和ApplicationContextInitializer
