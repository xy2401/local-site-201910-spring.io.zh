Spring XD 1.0.0。M7发布

发布
马克·波拉克
2014年6月3日

Spring XD团队高兴地宣布，现在可以下载 Spring XD Milestone 7。

此版本的重点

  • 传输数据分区 ：默认情况下，消息以循环方式传递到流模块的多个实例。但是，如果模块执行的操作不能吸收流中的随机消息，则可以根据流的内容对流进行分区，以便始终将相似的消息传递到同一模块实例。例如，如果处理模块正在基于每个客户执行有状态操作，则可以基于消息中的customerId字段对流进行分区。这是通过在部署清单中指定分区属性来完成的。一个小例子如下所示。

  • HDFS和HDFS DataSet接收器改进：这些接收器现在支持基于分区功能写入多个路径和文件。查看HDFS分区样本 ，了解使用分区功能的几种方法。

  • 分布式运行时环境

  • 作业的部署清单

  • 更新以支持较新的Hadoop发行版 ：现在共有8个。

  • 管理员界面增强功能

  • Rabbit Message Bus的可配置选项：配置选项，例如消息传递选项，并发设置和高可用性策略。这些选项也可以为特定模块覆盖，例如modue.http.producer.deliveryMode = NON_PERSISTENT

  • 改进了自动化系统测试中的模块覆盖率

数据分区示例

为了演示数据分区功能，请使用Rabbit作为传输启动两个容器。在壳

stream create words --definition "http | splitter --expression=payload.split(' ') | log"  

stream deploy words --properties module.splitter.producer.partitionKeyExpression=payload,module.log.count=2  

http post --data "How much wood would a woodchuck chuck if a woodchuck could chuck wood"

请注意，仅在将RabbitMQ用作传输方式时才支持分区。在下一发行版中将支持Redis作为传输
