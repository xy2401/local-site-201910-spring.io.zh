使用新的when.js 3.2.2为Spring Data REST构建前端

工程
格雷格·特恩奎斯特
2014年6月2日

问候春天社区！

Roy Clarkson和我将在今年的SpringOne 2014大会上发表题为Spring Data REST-Data Meets Hypermedia的演讲。我们将探索如何快速弥合功能强大的Spring Data后端与启用了超媒体的RESTful前端之间的鸿沟。

在演讲的一部分中，我们将深入研究JavaScript前端，该前端使用户可以拍照并将其上传到网站。该网站转身并从后端获取图像。就其本身而言，这要归功于Spring Data REST提供的完全加载的RESTful API。

但是，直接获取多个图像并不是很有效，并且容易冻结Web浏览器。多亏了我们团队中的CujoJS团队（ Brian Cavalier和John Hann ），我得以使用最近发布的when.js模块并编写了更加流畅的体验。

下面的部分显示了rest.js的核心用法以及通过when.js的promises，以及如何使它易于编写可读性和功能性代码。

首先，我们引入一些关键模块：

var rest = require('rest');
var when = require('when');
var defaultRequest = require('rest/interceptor/defaultRequest');
var mime = require('rest/interceptor/mime');
var hateoas = require('rest/interceptor/hateoas');

然后我们配置一个api带一个对象mime拦截器hateoas拦截器，并将其配置为默认Accept标头application/hal+json因此，Spring Data REST会使用HAL。

var api = rest
    .wrap(mime)
    .wrap(hateoas)
    .wrap(defaultRequest, {headers: {'Accept': 'application/hal+json'}});

使用该配置，我们可以执行一些RESTful调用来获取图像数组，而不会破坏用户的体验：

when.all(api({
    method: 'GET',
    path: gallery._links.items.href,
    params: {projection: "noImages"}
}).then(function (response) {
    if (response.entity._embedded) {
        return response.entity._embedded.items.map(function (itemWithoutImage) {
            return api({path: itemWithoutImage._links.self.href})
        })
    } else {
        return [];
    }
})).done(function(itemsWithImages) {
    itemsWithImages.forEach(function(item) {
        items[item._links.self.href] = item.entity;
        nestedTable.append(createItemRowForGallery(item.entity, gallery));
    })
})

那是怎么回事？让我们看一下每个小块，探索正在发生的事情。

api({
    method: 'GET',
    path: gallery._links.items.href,
    params: {projection: "noImages"}
})

这是在调用以检索与此特定画廊相关的物品数组。它返回了一个承诺，为我们提供了一些不错的选择。

注意：它使用?projection=noImages以获取不含图像数据的项目URI列表。（一次检索十张2MB图像的图像！）

在 - 的里面then功能，代码寻找_embedded数据，如果存在的话，它将无像项目的数组1对1转换为GET承诺，获取每件商品的实际图片。如果没有_embedded数据，则返回一个空数组。

提取图像数据的承诺数组用when.all包装， 这是一个方便的函数，它将等待每个承诺完成后再继续。

由于我们打算消耗所有这些输出GET通过在屏幕上显示它们，我们完成了done() 。 itemsWithImages ，由提供when.all ，是大小相等的数组，其中包含每个诺言的结果。

顺便说一句：如果您不知道……使用诺言时的规则＃1是then（）函数调用必须返回一个对象（将作为诺言包装）或诺言本身。如果您打算实际使用结果并进行处理，请使用done（）代替。

如果您对javascript不太熟悉，那么可以考虑使用它。但是我自己是一名JavaScript新手 ，因此我发现该API可以让我轻松表达自己想做的事情。
