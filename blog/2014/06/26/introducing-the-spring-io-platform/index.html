介绍Spring IO平台

工程
乔什·朗（Josh Long）
2014年6月26日

我们很高兴宣布Spring IO 1.0的发布！

首先，Spring IO是对许多用户已经知道并用作以Spring为中心的统一的，统一的，统一的平台的逻辑描述。

小型（Java）软件包中的大功能

Spring IO平台包括基础层模块和执行层域特定的运行时（DSR）。基础层代表了核心的Spring模块和相关的第三方依赖关系，它们已经进行协调以确保流畅的开发体验。Spring IO执行层提供的DSR大大简化了基于JVM的生产就绪型构建工作。Spring IO的第一个版本包括两个DSR：Spring Boot和Grails。Spring XD将在2014年晚些时候添加到Spring IO平台。

Spring支持许多应用程序工作负载。是否想与数据存储区对话？大的还是专业的数据存储 ？建立批处理解决方案 ？整合不同的系统和数据 ？使用AMQP吗？使用Groovy编程语言通过Grails快速构建Web应用程序 ？是否连接到Twitter ， Facebook之类的OAuth安全的REST服务，或者实际上是任何OAuth安全的服务 ？保护应用程序 ？使用REST，websockets等构建Web应用程序 ？（ 使用HATEOAS吗？）连接到您喜欢的平台即服务（如Cloud Foundry）上的基础架构服务？是否需要管理快速，响应式，并发，消息和事件调度 ？公开基于SOAP的Web服务 ？我需要继续吗？如果您知道要看的地方，可能会为您找到解决方案！

使用Spring Boot ，很容易拉出这些模块（此外还有更多！）一起成为由惯例驱动的解决方案。Spring Boot是引导和简化正在进行的应用程序开发的好方法。

一个平台BOM

Spring IO平台也是通过Maven物料清单依赖项实现的API的实际协调。现在，很容易推断出整个投资组合中哪个版本的依赖项可以一起工作。如果您想获得最新和最伟大的API版本，只需更新您的Spring IO依赖项即可。作为平台，Spring IO还指定了流行的第三方库（例如Hibernate）的已知工作版本。

最终，Spring IO仅仅是CLASSPATH您的应用程序。您可以选择使用构建工具的特定版本的依赖项设施。版本管理最终是一种建议和便利，而不是要求 。

Spring IO被证明可以与Java 1.7和1.8一起使用，尽管Spring IO下的特定项目通常也可以与更老的JDK一起使用。它仅需要Java SE，并支持Groovy，Grails和某些Java EE。作为平台，它可以部署在嵌入式运行时，经典应用程序服务器部署和PaaS环境中。

少即是多

这是一个如何使用Spring IO的示例。这是一个简单的Spring Boot REST端点，运行时会响应以下形式的HTTP请求： http://127.0.0.1:8080/hello/SpringIO 。

嗨/ pom.xml

<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>
    
    <groupId>com.example</groupId>
    <artifactId>your-application</artifactId>
    <version>1.0.0-SNAPSHOT</version>
    
    <!-- our one dependency on Spring Boot's web support. No version. -->
    <dependencies>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
        </dependency>
    </dependencies>

    <!-- Transitively bring in the Spring IO Platform Bill-of-Materials `pom.xml` -->    
    <dependencyManagement>
        <dependencies>
            <dependency>
                <groupId>io.spring.platform</groupId>
                <artifactId>platform-bom</artifactId>
                <version>1.0.0.RELEASE</version>
                <type>pom</type>
                <scope>import</scope>
            </dependency>
        </dependencies>
    </dependencyManagement>

</project>

嗨/ src / main / java / io / Application.java


package io;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.EnableAutoConfiguration;
import org.springframework.context.annotation.ComponentScan;
import org.springframework.context.annotation.Configuration;
import org.springframework.web.bind.annotation.PathVariable;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

@Configuration
@EnableAutoConfiguration
@RestController
public class Application {

    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }

    @RequestMapping("/hello/{name}")
    String hi(@PathVariable String name) {
        return "Hello, " + name + "!";
    }
}
