预览Spring安全测试：HtmlUnit

工程
罗伯·温奇
2014年5月23日

2015年3月31日更新
该博客已过时，不再维护。请参考参考文档的“ 测试”部分以获取更新的文档。

在我之前的博客中，我们探讨了如何在Spring MVC Test中使用测试支持。现在，我们将看到相同的支持如何与Spring Test MVC HtmlUnit一起使用 。

最低版本
Spring Security测试支持不适用于spring-test-mvc-htmlunit-1.0.0。M1.jar。相反，您只使用最新的快照。这是由于对Spring Security和MockMvcHtmlUnitDriver可以一起使用进行了一些小的修改。

设置HtmlUnit和Spring Security

的设置HtmlUnit Spring Security的集成与使用Spring MVC Test和HtmlUnit的集成非常相似。下面我们重点介绍不同之处：

@RunWith(SpringJUnit4ClassRunner.class)
@ContextConfiguration(classes = {WebMvcConfig.class, WebSecurityConfig.class, MockDataConfig.class})
@WebAppConfiguration
@TestExecutionListeners(listeners={ServletTestExecutionListener.class,
        DependencyInjectionTestExecutionListener.class,
        DirtiesContextTestExecutionListener.class,
        TransactionalTestExecutionListener.class,
        WithSecurityContextTestExcecutionListener.class})
@WithMockUser
public class MockMvcHtmlUnitCreateMessageTest {
  @Autowired
  private Filter springSecurityFilterChain;

  @Autowired
  private WebApplicationContext context;

  private WebClient webClient;


  @Before
  public void setup() {
      MockMvc mockMvc = MockMvcBuilders
              .webAppContextSetup(context)
              .addFilters(springSecurityFilterChain)
              .build();
      driver = new MockMvcHtmlUnitDriver(mockMvc, true);
  }
}

源代码
您可以在github上找到此博客的完整源代码。

  • 第一个区别是确保我们添加了安全配置， WebSecurityConfig ，到@ContextConfiguration
  • 接下来的区别是确保添加WithSecurityContextTestExcecutionListener正如我们在测试基于方法的安全性时所做的那样。
  • 然后，我们需要使用Spring Security WithSecurityContext注解。在这种情况下，我们使用@WithMockUser ，但是可以利用方法安全性部分中使用的任何注释。
  • 我们需要得到springSecurityFilterChain并将其添加到MockMvcBuilders过滤器。

现在，假设我们已经通过用户名“ user”和角色“ ROLE_USER”进行了身份验证，就可以运行测试。当然，就像我们在方法安全性中所做的一样，我们可以轻松地更改批注以反映要作为其运行的用户。

无需在测试之前进行显式身份验证，我们就可以确保更轻松地隔离测试。通过仅以用户身份运行，我们就可以大大简化测试（尤其是当身份验证像SSO工作流程那样复杂时，尤其如此）。我们还消除了实际认证的需要，从而大大加快了测试速度。此外，我们可以使用上述相同步骤将Spring Security Test支持与HtmlUnit / WebDriver或HtmlUnit / Geb集成。
