Spring Integration Java DSL Milestone 1发布

发布
阿尔特姆·比兰（Artem Bilan）
2014年5月8日

Spring Integration开发团队很高兴宣布Java DSL扩展的第一个里程碑的发布
对于Spring Integration ！

Java DSL的一般目的是为基于以下内容的消息流提供一种流畅而便捷的API。
EIP并避免样板式Spring Integration配置。

的org.springframework.integration:spring-integration-java-dsl:1.0.0.M1工件可以从
Spring IO里程碑存储库 。

这个概念

Spring Integration Java DSL与现有的Scala不同
和Groovy DSls；也不类似于
Apache Camel Java DSL ，尽管它确实实现了方法链原理。

我们的主要目标是重用Spring Framework JavaConfig（ @Configuration类），以及Spring Integration Core中的类
组件。为了实现方法链原则，我们提供了一组构建器及其工厂 。
Java 8 Lambdas还为干净语法提供了一些其他帮助。

基础

中央阶级是IntegrationFlows -的IntegrationFlowBuilder工厂-应该从@Bean定义方法来填充IntegrationFlow bean使用* method-chain * s和Lambdas作为参数。的IntegrationFlowBuilder提供了很多
重载的方法来配置集成端点。我们称它们为EIP方法 。规范样本：

@Bean
public IntegrationFlow helloWorldFlow() {
	return IntegrationFlows.from("helloWorldInput")
			.filter("World"::equals)
			.transform("Hello "::concat)
			.handle(System.out::println)
			.get();
}

使用Spring Integration术语，以上示例的描述为：
* 一种Message发送到“ helloWorldInput”通道；
*的filter仅在其接受时接受payload是字符串“ World”；
*的transformer前缀payload与Hello ;
*的handler将结果打印到STDOUT；
*如果是原始的payload不是“世界”，我们不会在控制台中看到任何内容- Message被丢弃
filter 。

Spring Integration Java DSL可以按原样在现有应用程序中使用，并且可以与exising连接
消息注解配置或Spring Integration XML配置，当然还有Spring Boot应用程序中的配置。

有关更多信息，请参阅上文提到的参考手册。
