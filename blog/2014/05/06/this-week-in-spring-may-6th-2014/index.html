春天的这个星期-2014年5月6日

工程
乔什·朗（Josh Long）
2014年5月6日

欢迎来到春季本周的另一期！本周，我在洛杉矶，与正在考虑在Spring和Spring Boot之上构建下一代体系结构的大型公司进行交谈。哦，我还去参观超爽枢纽洛杉矶的办公室！

  • Spring Data Hadoop忍者Thomas Risberg（顺便说一句，他是JdbcTemplate ，以及他的许多其他成名声！）刚刚宣布了适用于Hadoop 2.0的Spring。RC3 。新版本增加了受支持的Hadoop发行版集，并包括对YARN的支持，您可以在一些“ 入门指南”中实际看到它。
  • Spring Integration 4.0是一个重大升级- 新的4.0版本提供了对注释和Java配置的完全支持，并提供了一些Spring Boot支持！使用4.0，您将能够制作XML的免费集成应用程序。加入项目负责人Gary Russell ，在5月13日的网络研讨会Spring Integration 4.0（新领域）中带您领略所有新热点 。
  • Spring Framework 4.0.4是一个维护版本，其中包含一些较小的依赖项更新，建议使用 。更新版本并mvn clean install / gradle package现在！
  • 新的Spring Data版本，Spring Data Dijkstra RC1（以Edsger W. Dijkstra命名）终于发布了，其中包含了许多很棒的功能！我个人的最爱之一是能够返回Java 8 Optional存储库中的值。我在这里举一个例子 。新版本的发布是巨大的 ，并且包括一些（历史上完全由社区驱动）模块，例如Spring Data Cassandra，Spring Data Solr和Spring Data Couchbase。不要错过！
  • Spring和Groovy / Grails工具套件的新3.5.1版本现已发布 。新版本包含新功能，更新和新基准，因此请查看发行说明。
  • Groovy 2.3在这里！ （现在就去阅读和下载！）
  • 令人惊叹的，不眠的（我敢肯定！只要看看他发布的博客！），Rob Winch就宣布了具有测试支持功能的Spring Security 4.0.0的第一个里程碑 。
  • Spring HATEOAS 0.11现在可用，并且具有称为Traverson的用于客户端服务遍历的API 。
  • Spring Data Neo4j的负责人Michael Hunger和我将在5月20日举行有关Spring Data Neo4j 3.0和Neo4j 2.0的新功能的网络研讨会。
  • XavierPadró毫不犹豫地撰写了一篇有关使用Spring Integration的Java API的非常不错的文章 。
  • Spring Framework ninjaStéphaneNicoll刚刚发表了一篇不错的博客文章，在即将发布的Spring Framework 4.1中介绍了注释驱动的JMS支持 。
  • 想更多地了解Neo4j和Spring Data Neo4J ？在JavaBeat上有一个关于Spring Data Neo4j的不错的小教程。英文有点断断续续 ，代码示例杂乱无章，但我认为您仍然应该能够理解它的要旨。如果您想要更现代的，以Java为中心的替代方法，请查看这些指南，以了解如何使用Spring Data Neo4j以及将Spring Data Neo4j与Spring Data REST一起使用
  • 如果那还不够Neo4j，如果您在湾区，那么您可能想看看Spring Data Neo4J带领Michael Hunger（和我）将于5月8日（星期四 ）举行的聚会！
  • 我和我的好朋友Andy Piper在SpringOne2GX 2013上发表了有关在Cloud Foundry上运行和编写Spring应用程序的演讲。看看这个！
  • Scott Frederick和Cornelia Davis 在SpringOne2GX 2013的精彩演讲中介绍了如何向Cloud Foundry添加自定义服务和运行时 。
  • 您是否签出了新的Project Reactor版本 ？新版本是挤满了很酷的东西（包括API改进，通用对象池和更多 ！）并为Reactor 2.0铺平了道路Promise以及对反应流项目的流支持。真好！
  • ActiveState博客上的Phil Whelan在一篇不错的文章中介绍了buildpack 。Buildpacks来自Heroku，但是您当然可以在Cloud Foundry中使用它们，并且与ActiveState自己的Stackato项目一样，它们也可以与基于Cloud Foundry构建的项目一起使用。
