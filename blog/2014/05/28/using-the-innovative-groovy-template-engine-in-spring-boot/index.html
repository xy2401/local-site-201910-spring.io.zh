在Spring Boot中使用创新的Groovy模板引擎

工程
塞德里克·尚波
2014年5月28日

随着Spring Boot 1.1.0的发布。M2附带了对Groovy 2.3提供的新模板引擎的支持。在这篇文章中，我们将描述使用这种引擎的好处，当然还有您如何在Boot中使用它。

这篇文章中的所有源代码都可以在GitHub上找到，因此随时克隆存储库并尝试一下：

git clone https://github.com/melix/springboot-groovytemplates.git
cd springboot-groovytemplates
./gradlew run

然后在http：// localhost：8080上打开浏览器

该应用程序是完全用Groovy编写的，并且还使用GORM for Boot ，但是当然可以仅将Groovy用于模板部分，并用Java编写其余的应用程序。从现在开始，我们将只专注于该项目的模板方面。

依存关系

在Spring Boot中与Groovy 2.3模板集成非常容易。您只需要向groovy-templates生成文件中的模块。例如，如果您使用Gradle，则只需使用以下命令：

dependencies {
  compile "org.codehaus.groovy:groovy:${groovyVersion}"
  compile "org.codehaus.groovy:groovy-templates:${groovyVersion}"
  compile "org.springframework.boot:spring-boot-starter-web:${springBootVersion}"
  compile "org.grails:gorm-hibernate4-spring-boot:1.0.0.RELEASE"

  runtime "com.h2database:h2:1.3.173"
}

Groovy模板

Groovy标记模板引擎基于构建器语法提供了创新的模板系统。它提供了各种关键功能：

  • 分层（生成器）语法来生成类似XML的内容（尤其是HTML5）
  • 模板包括
  • 将模板编译为字节码以便快速渲染
  • 国际化
  • 共享结构模式的布局机制
  • 可选类型检查

和更多！您可以在文档中找到此模板引擎功能的完整列表。模板基本上是Groovy代码，对模板用例具有特殊的支持。

让我们从一个非常简单的示例开始，在该示例中，我们想显示一个带有简单消息的索引，其中包含当前正在使用的Spring Boot和Groovy的版本号：

yieldUnescaped ''
html {
  head {
    title('Spring Boot - Groovy templates example')
    link(rel: 'stylesheet', href: '/css/bootstrap.min.css')
  }
  body {
    div(class: 'container') {
      div(class: 'navbar') {
        div(class: 'navbar-inner') {
          a(class: 'brand',
              href: 'http://beta.groovy-lang.org/docs/groovy-2.3.2/html/documentation/markup-template-engine.html',
              'Groovy - Template Engine docs')
          a(class: 'brand',
              href: 'hhttp://projects.spring.io/spring-boot/') {
            yield 'Spring Boot docs'
          }
        }
      }
      div("This is an application using Boot $bootVersion and Groovy templates $groovyVersion")
    }
  }
}

在第一行，您可以阅读yieldUnescaped指令。它指示渲染器按原样渲染参数。该指令可用于呈现任何类型的基于文本的内容。在这里，它用于呈现HTML文件的doctype声明，但是您实际上可以使用它来呈现任何内容。模板引擎提供了许多辅助功能，例如yield ，在文档中进行了介绍 。

模板的其余部分由与HTML输出匹配的分层结构组成，因此呈现HTML内容非常自然。例如，代码： link(rel: 'stylesheet', href: '/css/bootstrap.min.css')将呈现为：

同样，这是：

a(class: 'brand',
  href: 'http://beta.groovy-lang.org/docs/groovy-2.3.2/html/documentation/markup-template-engine.html',
  'Groovy - Template Engine docs')

将呈现为：

Groovy - Template Engine docs

请注意模板中的属性如何映射到标记的HTML中的属性。最后一个参数对应于标签的主体。另外，也可以使用yield渲染标签正文的指令：

a(class: 'brand',
  href: 'http://beta.groovy-lang.org/docs/groovy-2.3.2/html/documentation/markup-template-engine.html') {
  yield 'Groovy - Template Engine docs'
}

选择通常取决于您是否要嵌套内容。但是到目前为止，我们的模板生成的所有内容都是静态的。模板的最后一部分更有趣：

div("This is an application using Boot $bootVersion and Groovy templates $groovyVersion")

如您所料，它将呈现为：

This is an application using Boot 1.1.0.M2 and Groovy templates 2.3.2

此处的模板利用了在模型中找到的两个变量：

  • bootVersion
  • groovyVersion

我们的应用程序将这些变量作为模板中的变量公开，因此让我们看看如何做到这一点。

控制器

我们要做的唯一一件事就是创建一个控制器，该控制器将呈现我们的视图，并且与Spring Boot一样，只需几行代码即可：

package sample

import org.springframework.boot.Banner
import org.springframework.stereotype.Controller
import org.springframework.web.bind.annotation.RequestMapping
import org.springframework.web.servlet.ModelAndView

@Controller
class SampleController {
  @RequestMapping("/")
  def home() {
    new ModelAndView(
        "views/home",
        [bootVersion: Banner.package.implementationVersion, 
         groovyVersion: GroovySystem.version])
  }
}

我们的home方法返回一个ModelAndView实例，并且该模型仅包含两个元素，分别是Spring Boot版本和Groovy版本。借助其视图，可以自动找到该视图， views/home 。Spring Boot希望可以在以下位置找到该视图src/main/resources/templates/views 。能更容易吗？

现实生活数据

在现实生活中，模板不可能这么简单。您将拥有实体，数据库，CRUD操作等。因此，下一步是向您展示如何使用新的模板引擎来渲染更复杂的模型。为此，我们使用了GORM，因此我们将首先定义一个名为Person ：

package sample

import grails.persistence.*

@Entity
class Person {
  String firstName
  String lastName
}

我们将要做的是，例如：

  • 列出数据库中的人员
  • 添加/编辑新人

因此，我们将需要两个模板：一个用于列出人员，另一个用于创建人员（或编辑）。该列表示例很有趣，因为它将使我们展示如何在模板中的列表上进行迭代。因此，在此之前，让我们使用list操作创建一个控制器：

@Controller
@RequestMapping("/person")
class PersonController {

  @RequestMapping("list")
  def list() {
    new ModelAndView('views/person/list', [persons: Person.list()])
  }
}

您可以看到，与在简单示例中所做的类似，我们返回了ModelAndView例如，但是这次，模型包含人员列表。因此，让我们看看模板是什么样的：

yieldUnescaped ''
html {
  head {
    title('Spring Boot - Groovy templates example')
    link(rel: 'stylesheet', href: '/css/bootstrap.min.css')
  }
  body {
    div(class: 'container') {
      div(class: 'navbar') {
        div(class: 'navbar-inner') {
          a(class: 'brand',
              href: 'http://beta.groovy-lang.org/docs/groovy-2.3.2/html/documentation/markup-template-engine.html',
              'Groovy - Template Engine docs')
          a(class: 'brand',
              href: 'hhttp://projects.spring.io/spring-boot/') {
            yield 'Spring Boot docs'
          }
        }
      }

      ul {
        persons.each { person ->
          li {
            a(href:"/person/$person.id", "$person.lastName $person.firstName")
          }
        }
      }

      div {
        a(href:'/person/add', 'Add new person')
      }
    }
  }
}

大部分模板实际上与页面的装饰相对应，并从原始模板复制而来。在这一点上，您可能想知道您可以采取什么措施来改善这一点，但我们稍后会再讲到这一点，并将重点放在该模板最有趣的部分，即迭代：

ul {
  persons.each { person ->
    li {
      a(href: "/person/$person.id", "$person.lastName $person.firstName")
    }
  }
}

遍历persons多亏了传统each Groovy开发人员习惯的方法。这是正常的，因为模板实际上是Groovy代码！这样我们就可以遍历人，给我们起个名字（ person ）中的当前人，然后在a标签。

在数据库中有几个人的情况下，生成的HTML将如下所示：

记住，我们实际上有两个共享相同结构的模板。他们俩都使用Twitter Bootstrap，它们共享相同的菜单，最终，唯一改变的是页面标题和主体。如果我们可以从模板中提取并共享该怎么办？

布局介绍

为此进行了布局。因此，让我们将模板的公共部分提取到main.tpl文件中，然后将其保存到src / main / resources / templates / layouts中 ：

yieldUnescaped ''
html {
  head {
    title(pageTitle)
    link(rel: 'stylesheet', href: '/css/bootstrap.min.css')
  }
  body {
    div(class: 'container') {
      div(class: 'navbar') {
        div(class: 'navbar-inner') {
          a(class: 'brand',
              href: 'http://beta.groovy-lang.org/docs/groovy-2.3.2/html/documentation/markup-template-engine.html',
              'Groovy - Template Engine docs')
          a(class: 'brand',
              href: 'hhttp://projects.spring.io/spring-boot/') {
            yield 'Spring Boot docs'
          }
        }
      }
      mainBody()
    }
  }
}

这看起来与标准模板非常相似，但是实际上您可以找到两个特殊之处：

  • title(pageTitle)哪里pageTitle预期是我们要提供的页面标题
  • mainBody() ，这将导致使用该布局的页面呈现为主体。注意括号很重要。

现在，让我们更新主页模板以使用此布局：

layout 'layouts/main.tpl',
    pageTitle: 'Spring Boot - Groovy templates example with layout',
    mainBody: contents {
      div("This is an application using Boot $bootVersion and Groovy templates $groovyVersion")
    }

您可以看到我们称layout方法，并提供几个参数：

  • 要使用的布局文件的名称（ layouts/main.tpl ）
  • pageTitle ，一个简单的字符串
  • mainBody ， 使用contents块

使用contents块将触发呈现的内容mainBody里面的布局时mainBody()找到指令。因此，使用此布局文件，我们绝对可以针对多个模板共享一个通用的结构化模式。作为说明，让我们看看list.tpl模板现在看起来像：

layout 'layouts/main.tpl',
    pageTitle: 'List persons',
    mainBody: contents {
      ul {
        persons.each { person ->
          li {
            a(href:"/person/$person.id", "$person.lastName $person.firstName")
          }
        }
      }

      div {
        a(href:'/person/add', 'Add new person')
      }
    }

当然，布局本身是可组合的，因此您可以在布局内部使用布局…

结论

在本文中，我们向您展示了Spring Boot如何使Groovy在Groovy 2.3中引入的新模板引擎的使用变得非常容易。该模板引擎提供了非常自然而强大的语法，可以生成任何类型的基于文本的内容。在Groovy文档中可以找到模板引擎功能的完整描述，在Spring Boot示例中可以找到使用相同技术的替代应用程序。

最后但并非最不重要的一点是，此模板引擎的本机支持即将出现在Spring 4.1中！因此，希望将来有更多Groovy的爱！
