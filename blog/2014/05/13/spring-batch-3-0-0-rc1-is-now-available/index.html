Spring Batch 3.0.0。RC1现在可用

发布
迈克尔·米内拉
2014年5月13日

今天，我们很高兴地宣布Spring Batch 3.0的候选版本。Spring Batch的这个版本提供了对我们致力于为现有Spring Batch社区提供的JSR-352的支持以及许多新功能。


候选版本1中的功能


此版本的主要功能包括：



  • JSR-352支持

  • 促进将Spring Batch集成到Spring Batch中

  • 彻底检查依赖项

  • 支持SQLite

  • 工作范围

  • 从Maven切换到Gradle以构建Spring Batch


JSR-352支持


JSR-352是批处理JSR，已于去年下半年发布为1.0。当Spring Batch成为该JSR大部分编程模型的灵感时，Spring致力于对其进行支持。在3.0版本中，Spring Batch通过所有TCK测试，符合JSR-352。它提供了经过生产测试最多的JSR实现。

Spring Batch的JSR-352实现的开发旨在为现有Spring Batch用户提供最大的灵活性。我们允许开发人员将现有的ItemReader，ItemProcessor，ItemWriters等与JSR-352提供的配置工具结合使用。这为开发人员提供了经过生产测试的组件的完整库，用于构建可靠的批处理作业。


要了解有关Spring Batch JSR-352实施的更多信息，请在此处访问我们的参考文档： http://docs.spring.io/spring-batch/trunk/reference/html/jsr-352.html


将Spring Batch集成到Spring Batch


何时使用Spring Batch或Spring Integration之间的界线通常是模糊的。在许多用例中，一种比另一种效果更好。但是，在许多用例中，可以将它们一起使用以构建健壮且可伸缩的数据处理系统。Spring Batch Integration提供了一组组件，以将Spring Batch和Spring Integration一起使用。Spring Batch Integration提供的用例包括：



  • 异步项目处理

  • 远程分块

  • 通过消息启动批处理作业

  • 远程分区


这些功能将批处理处理超出了JSR-352所提供的范围，并允许用户开发可扩展到单个JVM之外的批处理应用程序。您可以在参考文档中阅读有关Spring Batch Integration及其组件的更多信息： http://docs.spring.io/spring-batch/trunk/reference/html/springBatchIntegration.html 。


彻底检查依赖项


我们借此机会检查了Spring Batch的所有第三方依赖项，并更新了它们。作为此练习的一部分，我们还使它们与Spring产品组合中的其他项目保持一致，以提供将Spring Batch添加到现有应用程序（或将其他项目添加到Spring Batch项目）的最简单的体验。


支持SQLite


尽管HSQLDB在许多测试场景中很有用，但基于文件的系统（如SQLite）也可能非常有用。在3.0版本中，我们为SQLite添加了作业存储库DDL以解决这些用例。


工作范围


Spring Batch的步骤范围使开发人员可以延迟对象的创建，直到执行特定步骤为止。此功能还提供了在批处理工件中提供后期属性绑定的功能。在此3.0版本中，Spring Batch引入了Job范围。该作用域的工作方式与步骤作用域的工作方式相同（通过代理延迟对象的创建），但是它将延迟创建，直到作业执行而不是步骤执行为止。当在上下文中定义多个作业或步骤级组件中发生繁重的初始化过程时，这可能会很有帮助。您可以在以下参考手册中阅读有关工作范围的更多信息： http://docs.spring.io/spring-batch/trunk/reference/html/configureStep.html#job-scope 。

从Maven移至Gradle


最终，此版本是Spring Batch从Maven迁移到Gradle进行内部构建系统的第一个版本。此更改对使用jar的开发人员的影响为零。它们仍将通过Maven Central提供给Maven用户。


结论


3.0.0。RC1代表了Spring Batch的下一个重要里程碑的完成。它为我们的社区带来了JSR-352提供的标准，并为高级用户提供了详尽的其他功能集合。我们期待您在论坛，社交媒体以及SpringOne2GX上的亲自反馈！
