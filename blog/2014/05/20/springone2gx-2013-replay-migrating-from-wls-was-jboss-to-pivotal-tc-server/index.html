SpringOne2GX 2013重放：从WLS，WAS，JBoss迁移到Pivotal tc服务器

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2014年5月20日

记录在加利福尼亚州圣克拉拉的SpringOne2GX 2013

演讲者：李志勇

SAS®Institute有大量的Java EE应用程序组合。SAS以前提供了在WebLogic，WebSphere和JBoss中部署和运行所有这些应用程序的支持。从2013年7月发布的SAS 9.4开始，SAS更新了其基础架构和中间层平台，以交付并在Pivotal tc Server上运行。在本次演讲中，我们将讨论SAS用于为其客户提高价值的动机，技术选择，体系结构，系统管理，自动安装和配置等。具体来说，我们将详细讨论以下领域：

  • 技术选择：为了使tc Server可行，我们包括消息传递，缓存和事务管理系统。
  • 体系结构：为了在SAS产品中利用tc Server的可伸缩性和可靠性，我们通过使用Pivotal Web Server和mod_proxy支持集群。应用程序迁移：我们向Java开发人员和配置开发人员提供有关如何将其应用程序迁移到tc Server环境的指南。
  • 安全性：我们支持SSL，单点登录和其他企业安全协议，例如集成Windows身份验证，CA Site Minder，IBM Web Seal，SAML等。
  • 系统管理：我们提供了一个单一入口点，以利用Hyperic产品管理所有SAS应用程序堆栈，包括所有Web应用程序。
  • 自动化的安装/配置：我们提供了自动化的过程来安装和配置Hyperic以及所有Pivotal Application Fabric产品（tc Server，vFWS和GemFire）和SAS Web应用程序。
  • 交付和支持：SAS将嵌入式tc服务器作为SAS Web应用服务器提供，可与我们所有产品一起使用，并且在我们所有受支持的主机平台上都具有中间层。这使SAS能够提供完整的受支持的应用程序体系结构，并具有对关键软件的更完整的可见性和控制力。
  • 云部署：这种方法还利用虚拟化和云部署策略为我们的客户提供了优势。
