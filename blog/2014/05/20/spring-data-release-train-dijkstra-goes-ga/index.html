春季数据发布列车Dijkstra上GA

发布
奥利弗·德罗博姆
2014年5月20日

紧随Spring 3.2.9和4.0.5发行版之后，我很高兴地宣布推出名为Dijkstra的Spring Data发行列车的GA发行版。

我们很高兴地欢迎5个新模块加入发行流程 ： Elasticsearch ， Cassandra ， Couchbase ， Gemfire和Redis 。他们中的两个甚至庆祝自己的第一个GA版本。因此，我要表示特别的祝贺，并感谢Spring Data Elasticsearch和David Webb的项目负责人Mohsin Husen以及Spring Data Cassandra的Matthew Adams 。

以下是参与模块的完整列表：

  • Spring Data Commons 1.8 GA- 工件 -JavaDocs- 文档 -Changelog
  • Spring Data JPA 1.6 GA- 工件 -JavaDocs- 文档 -Changelog
  • Spring Data MongoDB 1.5 GA- 工件 -JavaDocs- 文档 -Changelog
  • Spring Data Neo4j 3.1 GA- 工件 -JavaDocs- 文档 -Changelog
  • Spring Data Solr 1.2 GA- 工件 -JavaDocs- 文档 -Changelog
  • Spring Data Couchbase 1.1 GA- 工件 -JavaDocs- 文档 -Changelog
  • Spring Data Cassandra 1.0 GA- 工件 -JavaDocs- 文档 -Changelog
  • 春季数据Elasticsearch 1.0 GA - 文物 - 的JavaDoc - 文档 - 更新日志
  • Spring Data Gemfire 1.4 GA- 工件 -JavaDocs- 文档 -Changelog
  • Spring Data Redis 1.3 GA- 工件 -JavaDocs- 文档 -Changelog
  • Spring Data REST 2.1 GA- 工件 -JavaDocs- 文档 -Changelog

到GA的返程使我们能够修复55张门票 ，而剩下的379张门票已为Dijkstra整体修复并实施。在我们的Wiki中找到发布培训的精选变更日志。这篇博客文章将引导您逐步了解发行培训中最相关的新增功能。

我们将为名为Codd的上一列火车删除一个或两个以上的服务版本，然后集中精力于下一个火车迭代的第一个里程碑，即Evans。
