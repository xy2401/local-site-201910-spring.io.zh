春天的这个星期-2016年12月6日

工程
乔什·朗（Josh Long）
2016年12月6日

欢迎来到春季本周的另一期！本周，我在YOW的第二站，在澳大利亚的布里斯班！会议电路。如果您在附近， 请打个招呼，或者在YOW时找到我！几天后搬到悉尼。

我们有很多内容需要介绍，让我们开始吧！

  • 如果您不熟悉JUnit 5.0和Spring Framework，那么这是必须的-请参阅Swiftmind的Sam Brannen的演讲
  • 好消息！由Axon领导和创始人Allard Buijze和我提供的介绍Axon和Spring Boot的CQRS网络研讨会现在在线
  • Spring Integration的负责人Gary Russell刚刚宣布了Spring Integration 5.0 M1 。此版本具有许多不错的新功能，并简化了一些较旧的功能。我最喜欢的功能之一：它现在将Java DSL打包在框架本身中，而不是单独发行！弄碎钻头，踢轮胎。
  • Janne Valkealahti刚刚宣布了Spring Statemachine 1.2.0。RC1 。此版本具有许多小的改进和错误修复，以及用于跟踪和监视的新API，以及对Redis，MongoDB和JPA中存储库的全面支持。
  • 这次即将举行的网络研讨会将是您必看的工具，您应该了解这些工具，以升级基本Spring Cloud之外的微服务架构 。
  • 观看有关使用Angular.js构建现代Spring应用程序的 SpringOne平台讲座。
  • 查看Phill Webb关于Spring Boot 1.4的新测试功能的演讲
  • 查看Netflix Zuul主持人Mikey Cohen的演讲， 介绍SpringOne Platform 2016中的Zuul
  • 上周，继续阅读Spring Tips系列，我研究了如何在Spring应用程序中利用（以及何时不利用） JTA（以XOpen协议表示 ）的优势。
  • 观看 Dan Wood在介绍Ratpack的SpringOne平台精彩演讲中的重播
  • 西蒙·沃德利（Simon Wardley）的这篇博客文章是个老歌，但又是个好东西： Cloud Foundry如何从Yak Shaving中拯救世界
  • Richard Seroter在使用Concourse CI向Microsoft Azure上的Pivotal Cloud Foundry连续交付基于服务总线的Java（春季启动）的应用程序方面发表了一篇不错的文章。
  • SkipJaq的前SpringSourcer Rob Harrop在2016年SpringOne平台上发表了有关反应式编程和微服务的演讲，来自Spring I / O Barcelona和S1P的非常受欢迎的演讲
  • RedHat的Mark Little博士在这个引人入胜的InfoQ帖子中评论了工程师Piotr Gankiewicz的微服务之旅
  • 最后一个Redis RC已发布，其中包含许多功能 ！

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2016年12月6日

记录在SpringOne Platform 2016中。
演讲者：Swiftmind的Sam Brannen
幻灯片： http ： //www.slideshare.net/SpringCentral/testing-with-spring-43-junit-5-and-beyond

对4.3中的Spring测试支持中的新功能或Spring 5和JUnit 5中的新功能感到好奇吗？然后加入Spring Test组件负责人和JUnit 5核心提交者Sam Brannen，以了解答案！本节的第一部分将向与会者概述Spring 4.3中最新的核心集成测试改进，例如SpringJUnit4ClassRunner的新SpringRunner别名，可以推断出@ContextConfiguration时的省略，对@Sql和@SqlGroup的支持。组成annotation，限制测试套件中上下文缓存的大小以及用于自定义测试的ApplicationContext的新ContextCustomizer API。下一部分将介绍Spring MVC Test的最新改进，例如服务器端测试支持对响应标头的期望，表单数据处理，响应内容中的JSON前缀，自定义HTTP谓词以及验证是否调用了特定的控制器方法以及客户端REST测试支持，用于指示期望请求的次数以及对请求正文中表单数据的期望。然后，重点将转移到Spring Framework 5.0中预期的测试类型。最后但并非最不重要的一点是，Sam将使用Spring Framework对JUnit 5的即将支持对Java 8以及更高版本进行现代测试的示例进行深入研究。
