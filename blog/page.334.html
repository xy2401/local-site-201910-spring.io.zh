Spring Java配置向前发展

工程
罗德·约翰逊
2007年11月5日

一些用户询问我们是否致力于Spring Java Configuration以及它在Spring 2.5中引入的注释配置选项的位置 。答案是肯定的，我们致力于Java Config。并且这两种方法不是互相排斥的。

这两种配置方法完全不同：Spring Framework中的@Autowired注释使用业务对象中的注释来配置组件，而Spring Java Config采用了一种独特的方法来专用化专用配置类中的注释。这些方法都不是唯一正确或错误的方法，它们在不同情况下都具有吸引力。甚至没有理由不能在同一应用程序中同时使用它们。

Spring Framework 2.5 RC1发布

发布
阿德里安·科耶尔（Adrian Colyer）
2007年10月23日

尊敬的Spring社区，

我很高兴地宣布，第一个Spring Framework 2.5发行候选版已经发布！Spring 2.5是从Spring 2.1里程碑开始的努力的顶点，它通过许多新功能增强了Spring 2.0，例如：

  • 完全支持Java 6和Java EE 5（JDBC 4.0，JTA 1.1，JavaMail 1.4，JAX-WS 2.0等）
  • 全功能注释驱动的依赖项注入（包括对“限定符”的支持）
  • 支持类路径中的组件扫描（自动检测带注释的类）
  • AspectJ切入点表达式中的bean名称切入点元素
  • 对AspectJ加载时编织的内置支持（基于LoadTimeWeaver抽象）
  • 更多的XML配置名称空间（“ context”，“ jms”），以提供最大的便利
  • 完全修订的集成测试框架（支持JUnit 4和TestNG）
  • Servlet MVC和Portlet MVC的基于注释的新控制器样式
  • 扩展的SimpleJdbcTemplate功能（支持命名参数等）
  • 官方认证的WebSphere支持（对WebSphere 6 UOWManager工具的支持）
  • Spring框架jar作为兼容OSGi的捆绑包出厂时提供
  • 可以将Spring ApplicationContext部署为JCA RAR文件（用于无头应用程序模块）
  • JCA 1.5消息端点管理（用于Spring管理的JMS和CCI消息侦听器）
