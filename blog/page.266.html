在Spring中使用Cloud Foundry Worker

工程
乔什·朗（Josh Long）
2012年5月9日

您无疑已经阅读了Jennifer Hickey的精彩博客文章，其中介绍了Cloud Foundry工人 ，他们在设置Ruby Resque后台作业中的应用以及今天的文章中介绍了Spring支持 。

Spring开发人员要点

  1. 您需要更新您的vmc与gem update vmc 。
  2. Cloud Foundry工作者让您运行public static void main职位。也就是说，Cloud Foundry工作者基本上是一个过程，其级别低于Web应用程序，它自然地映射到许多所谓的后台工作。
  3. 您需要提供Cloud Foundry将运行的命令。您可以提供java你想使用它的咒语，但是要发布一个shell脚本，让Cloud Foundry代为运行该shell脚本要容易得多。您提供的命令应采用$JAVA_OPTS ，Cloud Foundry已经提供了该版本，以确保一致的内存使用情况和JVM设置。
  4. 有多种方法可以自动创建Cloud Foundry可部署应用程序。如果您使用的是Maven，则org.codehaus.mojo:appassembler-maven-plugin插件将帮助您创建启动脚本并打包您的.jars为了易于部署以及指定入口点类。
  5. 其他一切基本相同。当你做vmc push在Java上.jar项目，Cloud Foundry将询问您该应用程序是否为独立应用程序。确认，然后它将引导您完成设置。

Spring MVC 3.2预览版：实时更新技术

工程
罗森·斯托扬切夫（Rossen Stoyanchev）
2012年5月8日
最后更新时间为2012年11月5日（Spring MVC 3.2 RC1）

在上一篇文章中，我介绍了Spring MVC 3.2中基于Servlet 3的新异步支持，并讨论了长时间运行的请求。异步处理的第二个非常重要的动机是浏览器需要接收实时更新。示例包括在浏览器中聊天，股票报价，状态更新，实时体育比赛结果等。可以肯定的是，并非所有示例都对延迟敏感，但是它们都具有相似的需求。

在标准的HTTP请求-响应语义中，浏览器会发起请求，服务器会发送响应，这意味着服务器只有在收到来自浏览器的请求后才能发送新信息。已经发展了几种方法，包括传统的轮询 ， 长轮询和HTTP流传输 ，最近，我们有了WebSocket协议。

发布
马丁·利珀特
2012年5月3日

亲爱的春季社区，

我很高兴宣布第一个里程碑版本3.0.0。SpringSource工具套件（STS）的M1。

此里程碑的重点包括：


  • 该发行版现在位于Eclipse Juno M6（4.2M6）软件包的顶部。

  • 更新至tc Server 2.6.5

  • 与Spring相关的内容助手和代码模板的一些改进

  • Groovy 2.0支持

  • Grails 2.0.3支持

  • Java7对AspectJ / AJDT的支持

由于6月份的Eclipse Juno版本将基于新的Eclipse 4.2平台（而不是3.x开发流），因此我们决定根据最新的Juno里程碑版本而不是最新的Indigo SR2版本发布此具有里程碑意义的STS版本。 。我们期望在Eclipse平台上有很多修复，直到六月的最终Eclipse版本发布为止，因此对于此发行版，使用M1构建的STS可能会遇到一些故障。尽管如此，我们还是鼓励您尝试使用此M1，并报告使用过程中遇到的问题和不兼容性。
