在Spring中使用Cloud Foundry Worker

工程
乔什·朗（Josh Long）
2012年5月9日

您无疑已经阅读了Jennifer Hickey的精彩博客文章，其中介绍了Cloud Foundry工人 ，他们在设置Ruby Resque后台作业中的应用以及今天的文章中介绍了Spring支持 。

Spring开发人员要点

  1. 您需要使用gem update vmc更新您的vmc版本。
  2. Cloud Foundry工作者可让您运行public static void main作业。也就是说，Cloud Foundry工作者基本上是一个过程，其级别低于Web应用程序，它自然地映射到许多所谓的后台工作。
  3. 您需要提供Cloud Foundry将运行的命令。您可以提供您希望使用的java指令，但要提供一个Shell脚本要简单得多，而让Cloud Foundry代您运行该Shell脚本。您提供的命令应该使用$JAVA_OPTS ，Cloud Foundry已经提供了$JAVA_OPTS以确保一致的内存使用和JVM设置。
  4. 有多种方法可以自动创建Cloud Foundry可部署应用程序。如果您使用的是Maven，则org.codehaus.mojo:appassembler-maven-plugin插件将帮助您创建启动脚本并打包.jars以便于部署，并指定入口点类。
  5. 其他一切基本相同。当您在Java .jar项目上执行vmc push时，Cloud Foundry会询问您该应用程序是否为独立应用程序。确认，然后它将引导您完成设置。

Spring MVC 3.2预览版：实时更新技术

工程
罗森·斯托扬切夫（Rossen Stoyanchev）
2012年5月8日
最后更新时间为2012年11月5日（Spring MVC 3.2 RC1）

我在去年的文章中，我介绍了新的Servlet 3为主，异步Spring MVC中3.2的支持，并谈到了长时间运行的请求。异步处理的第二个非常重要的动机是浏览器需要接收实时更新。示例包括在浏览器中聊天，股票报价，状态更新，实时体育比赛结果等。可以肯定的是，并非所有示例都对延迟敏感，但是它们都具有相似的需求。

在标准的HTTP请求-响应语义中，浏览器启动请求，服务器发送响应，这意味着服务器只有在收到来自浏览器的请求后才能发送新信息。已经发展了几种方法，包括传统的轮询 ， 长轮询和HTTP流传输 ，最近我们有了WebSocket协议。
