OSGi服务平台4.2版规范的早期草案现已发布

工程
阿德里安·科耶尔（Adrian Colyer）
2008年9月1日

OSGi联盟已经发布了服务平台规范4.2版的早期草案 。SpringSource员工是联盟内核心平台专家组（CPEG）和企业专家组（EEG）的活跃成员。我的个人参与主要涉及EEG，尤其是RFC 124“ OSGi的组件模型”。

RFC 124是Spring动态模块背后核心思想的标准化。如果看一下配置模式，您会发现它非常类似于Spring Dynamic Modules（DM）提供的“ osgi”命名空间。RFC 124吸收了我们在过去几年中开发Spring DM所学到的所有知识，并将其与核心和企业专家组其他成员的一些重要见解相结合，以产生既基于现实世界的经验又紧密结合的规范。与OSGi服务平台本身集成。非常感谢Spring DM开发团队：-Costin Leau，Oracle的Hal Hildebrand和BEA（现为Oracle）的Andy Piper在帮助开发和测试Spring DM以及随后帮助我们采用该模型方面的辛勤工作作为OSGi服务平台中标准化的基础。

谁是Apache项目的负责人？

工程
马克·布鲁尔
2008年8月22日

答案是Apache Software Foundation （ASF），这是一个非营利， 基于精英的组织，由个人（而非公司）组成，他们贡献代码，漏洞修复，用户问题的答案，时间，有时甚至是金钱来确保他们提供的软件不仅满足实际需求，而且功能强大，安全且被广泛采用。

ZDNet的达娜·布兰肯霍恩（Dana Blankenhorn） 对戴夫·罗森伯格（Dave Rosenberg） 博客的回应是 ，有关公司出售对他们不拥有或贡献的技术的支持，这激发了我对Apache项目和“支持”的公司直接提出一些意见。他们。SpringSource通过收购Covalent而成为许多Apache项目的重要贡献者。我同意Dave的观点，即那些试图仅通过销售“支持”来通过开源项目获利的供应商。这些技术虽然对软件没有贡献，但仅是寄生虫。这些公司不仅没有为项目做任何贡献，而且由于缺乏参与，导致支持不佳，从而损害了用户，并最终损害了项目的声誉和接受度。

优化和调整Apache Tomcat

工程
马克·托马斯
2008年8月8日

在星期三，我举行了关于优化和调整Apache Tomcat的网络研讨会。可以从SpringSource网站的网络研讨会部分获得网络研讨会的记录和幻灯片的副本。同一页面上包含所有以前的SpringSource网络研讨会以及Covalent网络研讨会档案的链接 。

在问答环节中，我无法解决所有问题，因此，如所承诺的，这是剩余的问题和我的答案。


  • 调整在Apache上运行的XYZ应用程序的最佳实践是什么
    Tomcat？

    对于任何应用程序都没有理想的配置设置，因为最佳设置取决于许多因素。首先，使用应用程序随附的建议。之后，请按照网络研讨会中列出的过程进行操作，如果可以提高性能，请不要害怕与应用程序一起提供的建议。

    • <li><strong>Do you have any recommendations for open source tools for troubleshooting, performance benchmarking or testing web applications?</strong>

    故障诊断的起点是使用Eclipse构建一个简单的测试用例。然后，我使用内置的调试器逐步遍历代码，以真正了解正在发生的事情。如果您需要一个探查器，那么NetBeans会提供一个探查器，尽管我从未使用过它。我使用商业分析器（ YourKit ），因为它们向开源开发人员提供免费许可证。对于性能基准测试，我使用Apache JMeter ，对于测试，我使用JUnit和Apache JMeter的组合。

    <li><strong>What tool do you recommend for monitoring memory, cpu, etc.?</strong>

    对于Tomcat特定的工具，我喜欢LambdaProbe 。对于更通用的工具，您可能需要看一下SpringSource Application Management Suite 。

  • 是否建议在生产环境中使用JMeter，LambdaProbe或Profiler？

    这取决于环境。过去，我曾在生产环境中使用过这三个工具，但同样有时候我故意不使用这些工具。您必须权衡该工具引起问题（性能，功能，安全性等）的风险与使用该工具的潜在利益。该余额必须根据具体情况进行评估。


  • 拥有硬件负载平衡器和Apache httpd负载平衡器是一个好主意吗？

    这也取决于情况。硬件负载平衡器带来了一些好处，但也带来了成本。通常，我希望在具有很高可用性要求的环境中看到硬件负载平衡器。


  • 可以使用iPlanet对Tomcat进行集群和负载均衡吗？

    是的，iPlanet可以使用mod_jk版本。您可以从Apache Tomcat下载页面下载源代码版本和二进制版本。您应该意识到，iPlanet的mod_jk用户相对较少，因此如果您使用较流行的Web服务器之一作为前端，可能会遇到一些错误。
