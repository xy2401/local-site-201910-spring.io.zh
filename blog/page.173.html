Spring如何实现与Java 6、7和8的兼容性

工程
斯特凡·尼科尔（StéphaneNicoll）
2015年4月3日

从Spring Framework 4.0开始，Java 8被作为一等公民而受支持，从那时起，我们就在Spring社区中看到了一些困惑。毕竟，我们如何设法支持Java 8并保持与Java 6和Java 7的兼容性？这篇博客文章提供了一些有关我们如何在框架代码库中进行处理的见解。

Java 8语言功能与Java 8 API

首先，必须在给定的Java一代（如Java 8）中使用新语言功能和新API之间进行区分。如果类使用lambda表达式之类的Java 8语言功能，则必须使用-source 1.8 -target 1.8进行编译，因此整个编译单元仅适用于Java 8+。但是，如果库中的特定类可以选择使用新的Java 8接口，例如java.util.stream.Stream ，图书馆仍然可以在以前的Java运行一代，只要它正在与如编译-source 1.6 -target 1.6 -只要使用特定的Stream实际运行时基础的类把守，只踢在Java 8+上。您可能已经猜到了，我们在Spring Framework代码库中广泛使用了这种安排！
