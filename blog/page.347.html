在Spring中使用JPA而不引用Spring

工程
本·黑尔
2006年8月7日

Spring 2.0增加了对JPA数据访问标准的支持，以及人们期望的所有标准Spring支持类 。马克·费舍尔（Mark Fisher）在如何使用此新支持方面有出色的文章 。但是，我们不断遇到的问题之一就是为什么要使用Spring类（ JpaTemplate ）来访问EntityManager 。这个问题的最佳答案在于JpaTemplate提供的增值。除了提供作为Spring数据访问特征的单线便捷方法之外 ，它还提供自动参与事务以及从PersistenceException到Spring DataAccessException层次结构的转换的功能。

Spring 2.0 RC2发布

发布
于尔根·霍勒（Juergen Hoeller）
2006年7月6日

尊敬的Spring社区，

我们很高兴地宣布Spring 2.0 RC2已发布。下载 | 说明文件 | 变更日志

这是Spring 2.0最终版的第二个候选版本。它引入了许多错误修复和较小的改进，特别是在AOP框架和JPA支持中。请参阅更改日志以获取详细信息。即将发布的2.0 RC3版本将解决许多其他已知问题。有关详细信息，请参见我们的JIRA路线图。

请尝试使用此版本的应用程序，让我们知道您可能遇到的任何问题！需要强调的是， Spring 2.0提供了与Spring 1.x系列的向后兼容性 。Spring 2.0还继续支持JDK 1.3+和J2EE 1.3+ （除了JSP标记和EJB支持之外，甚至还有J2EE 1.2+）。兼容性对我们的用户群至关重要，我们致力于提供兼容性。因此，我们也有兴趣了解任何此类平台上的潜在兼容性问题。

Spring Web Flow 1.0 RC3发布

发布
基思·唐纳德
2006年6月26日

 

亲爱的春季社区，

我们很高兴地宣布， Spring Web Flow （SWF）1.0 RC3（候选版本3）已经发布。下载 | 说明文件 | 变更日志

此稳定版本包含错误修复和较小的改进。目前，我们希望这是即将到来的Spring Web Flow 1.0 final之前的最后一个候选版本。值得注意的包括……

值得改进

改进的流异常层次结构。  总体而言，异常层次结构现在可以更清楚地组织从流定义访问（FlowLocatorException）到运行时流执行（FlowExecutionException）和执行持久性（FlowExecutionRepositoryException）的故障类别。
