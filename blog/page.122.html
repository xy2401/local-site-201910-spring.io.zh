检查您的Spring Security SAML配置-XXE安全问题

工程
罗伯·温奇
2016年8月24日

引起我们注意的是spring-security-saml示例应用程序包含一个XML外部实体（XXE）漏洞 。这意味着恶意用户可以查看Spring Application进程可以访问的任何文件。

该问题是OpenSAML Java ParserPool和Decrypter容易受到XML攻击的直接结果。ParserPool实现的默认行为已在OpenSAML 2.6.1+（Spring Security SAML使用）中修复。但是，如果用户在没有正确设置的情况下构建自己的ParserPool ，则仍然可能出现此漏洞。

用于Cloud Foundry 1.0.0的Spring Cloud Data Flow。RC1现在可用

发布
埃里克·博塔德
2016年8月24日

我们很高兴宣布1.0.0。用于Cloud Foundry的Spring Cloud Data Flow的RC1版本。

在即将到来的GA发布即将完成时，这个里程碑带来了以下改进：

  • 以1.0.0为基础。Spring Cloud Deployer Cloud Foundry的RC1版本本身建立在Project Reactor 3.0.0之上。发布和CF-Java-Client 2.0.0。发布

  • 添加了在Cloud Foundry中编排短期任务（包括Spring Batch Jobs）的功能，可以将其作为实验性功能切换启用

  • 添加对命令行参数的支持，将其作为单独的属性集在启动时传递给Task

  • 添加了对单独的流和任务特定服务绑定的支持。这允许将流和任务特定的服务固定到流和任务应用程序

  • 添加了覆盖按应用程序部署使用的buildpack的功能

  • 对Spring Cloud Deployer Cloud Foundry TCK测试覆盖率的重大改进允许改进应用程序状态计算

  • 添加有关滚动升级以及如何与Pivotal Cloud Foundry中的Spring Cloud Config Server“服务” 集成的文档
