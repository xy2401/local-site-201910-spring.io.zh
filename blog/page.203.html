乔什·朗（Josh Long）
2014年8月13日

欢迎来到春季本周的另一期。正如您可能想像的那样，我们发生了很多很酷的事情，整个团队都在忙于为SpringOne2GX 2014做准备，这将在短短的几周内完成！如果您尚未注册， 现在是时候注册了 ！

  1. Spring Security负责人和全能的不眠忍者Rob Winch刚刚宣布了Spring MVC Test HtmlUnit 1.0.0。M2已发布。如果您想使用Spring MVC对真实页面进行单元测试，那么这是一个很棒的版本
  • Spring Boot 1.1.5刚刚发布 。新版本主要解决了一些问题，建议所有用户升级。
  • 我真的很喜欢这个博客， “Quées Spring Framework？ ，它试图解释Spring的基本价值是什么。很大程度上，（西班牙语）文章解释说，Spring以一致的方式处理对象的生命周期。处理基本生命周期时，很容易在边缘插入各种框架和软件。我要指出的是，这篇文章使用了Spring 3.0和XML配置，虽然确实可以使用，但是这已经有些过时了。
  • RebelLabs的工作人员对Web框架的用法进行了有趣的介绍，以支持它。查看此最新报告，该报告由Spring MVC负责（很多）。
  • 这是一篇不错的文章，介绍如何使用Spring MVC（作为Spring Framework 4.x的一部分）来处理文件上传
  • *先生。 Haki *博客上有一篇很好的文章，介绍了如何使用Gradle分析依赖关系 ，并提供了一个基于Spring Boot的示例。当然，Spring Boot易于上手 ，可以与Maven 或 Gradle一起使用，并且具有许多好处，其中最重要的一点是，它使得在项目中使用Spring库变得非常简单，而不必担心版本不匹配。
  • 坚持使用JAX-RS和JSF？仍要使用Spring Security？这篇帖子（ 来自JSF实用性帖子 ）具有一些代码片段，这些代码片段展示了前进的道路。当然，使用直接的Spring可以更轻松地完成此操作。无需添加所有额外的Java EE重量。
  • 这是一种老套的东西：关于如何在Spring MVC中实现基于JQuery的自动保存示例的文章
  • 这是一个如何编写使用Primefaces并在Wildfly应用程序服务器上运行的Spring应用程序的（特别是受折磨的）示例。当Spring具有开箱即用的良好JSF支持时，我不明白为什么需要所有这些间接访问。此外，没有理由重新创建JdbcTemplate或注入ApplicationContext变成 bean 不过，它可能会帮助某人。

Spring MVC测试HtmlUnit 1.0.0。M2发布

发布
罗伯·温奇
2014年8月6日

我很高兴宣布Spring MVC Test HtmlUnit的第二个里程碑版本。

该项目的目的是在[Spring MVC Test]（Spring MVC测试框架）和HtmlUnit之间提供集成。使用基于HTML的视图时，这简化了端到端测试的执行。

变更日志

您可以在github上查看完整的变更日志 。以下是该版本的重点内容：

  • 该版本包含参考文档并发布API文档
  • 工件名称已从spring-test-mvc-htmlunit更改为spring-test-htmlunit。请参阅更新依赖项，以了解如何将Spring MVC Test HtmlUnit添加为Maven或Gradle依赖项
  • 项目名称已更改为Spring MVC Test HtmlUnit，以便更好地与Spring MVC Test的名称保持一致
  • 现在支持“”的上下文根
  • 添加了对外部资源的支持 。有关其他详细信息，请参见DelegatingWebConnection的javadoc。
  • Bug修复

扩展Spring Cloud

工程
拉姆尼瓦斯·拉达德（Ramnivas Laddad）
2014年8月5日

Spring Cloud最有趣的功能之一就是其可扩展性。您可以扩展它以支持其他云，增强已经支持的云，支持新服务，新服务连接器-所有这些都无需修改Spring Cloud代码本身。在此博客中，我们探索了这种功能。如果您还没有这样做，请阅读本系列的第一个和第二个博客以获取足够的背景知识。

可扩展性的三个轴

Spring Cloud提供了沿三个正交方向的可扩展性。您可以在这些方向之一中进行扩展，并且正交性可确保您继续从其他方向中受益。

Spring XD 1.0 GA发布

发布
马克·波拉克
2014年7月30日

我非常荣幸地代表Spring XD团队宣布Spring XD 1.0的全面上市！您可以下载zip发行版 。您也可以使用自制软件在OSX上安装，以及使用yum安装在RHEL / CentOs上。

Spring XD的目标是成为您开发和部署大数据应用程序的一站式服务。这种应用程序需要广泛的技术来解决不同的用例，同时又需要相互协作。此过程中的步骤包括：

  • 数据采集
  • 实时流和分析
  • 数据清理
  • 批处理（Hadoop上和下）
  • 机器学习和探索性数据分析
  • 可视化和报告
  • 实时和批处理之间的闭环分析

网络研讨会：使用Activiti进行流程驱动的Spring应用程序-9月23日

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2014年7月29日

主讲人：Activiti的Josh Long和Joram Barrez

当今的应用程序是复杂的分布式系统，这些系统共同产生了结果。但是，随着自动和人工驱动的任务集成到业务流程中，通过该系统跟踪工作流程变得越来越痛苦。企业无法对看不见的事物做出反应。企业无法衡量看不见的事物。 “工作流”描述了从开始到完成一件工作所经过的过程顺序。像Activiti之类的工作流系统描述并执行这些过程。Activiti是Apache 2许可的开源工作流引擎，可与Spring和Spring Boot一起使用。在此网络研讨会中，请加入Spring Developer Advocate Josh Long和Activiti-ninja Joram Barrez，以了解如何使用Spring（和Spring Boot）和Activiti提取，描述和重用复杂的业务流程。
