Spring Tools 4.3.1发布

发布
马丁·利珀特
2019年7月12日

亲爱的 Spring 社区，

我很高兴地宣布适用于Eclipse，Visual Studio Code和Theia的Spring Tools 4的4.3.1版本。

此版本的重点包括：

  • （所有语言服务器）性能：进一步改善了语言服务器的启动时间
  • （Spring Boot）修复：属性文件中错误的错误标记（ ＃314 ）
  • （VS代码）已修复：带有lsp的vscode goto定义错误（ ＃309 ）
  • （Eclipse）已修复：在JDK8上运行时，现在可以将STS4重新安装到现有Eclipse安装中
  • （Eclipse）修复：启动Spring Boot语言服务器以处理具有多个项目的工作区时出现死锁
  • （Eclipse）修复：使用早期版本的Eclipse 2019-09（4.13）时无法再在启动仪表板中启动应用程序
  • （Eclipse）有所改进，但尚未完全解决：打开pom.xml文件时冻结-如果您在此仍然遇到问题，请按照问题注释中的说明进行操作（ 对＃314的评论 ）

Spring Cloud Data Flow团队很高兴宣布里程碑版本2.2.0。M1。

除了一些增强功能和错误修复之外，以下是该版本的一些重点：

任务指标

在start.spring.io上探索项目

发布
达米安·维特拉克（Damien Vitrac）
七月4，2019

在start.spring.io上引入新功能： 探索项目 。
有了它，您现在可以浏览生成的项目文件，而无需实际下载，解压缩等。

许多开发人员要求使用此功能 ，因为他们想与另一个项目进行比较，或升级现有的应用程序。我们希望将来进一步探索这些可能性，因此请以我们的方式发送您的反馈 ！

探索项目：

  • 根据项目的不同，默认情况下将显示pom.xml / build.gradle / build.gradle.kts
  • 突出显示代码（1），预览降价
  • 复制并下载文件
