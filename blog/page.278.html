发布
马丁·利珀特
2011年10月25日

亲爱的 Spring 社区，

我很高兴地宣布针对SpringSource Tool Suite（STS）的Cloud Foundry集成的新里程碑发布M5。

此里程碑版本包括对在Cloud Foundry中调试应用程序的支持（适用于本地云和即将推出的Micro Cloud Foundry版本）。

Spring Data Neo4j 2.0.0。M1发布

发布
迈克尔·汉格
2011年10月21日

亲爱的 Spring 社区，

我们很高兴地宣布，第一个里程碑版本 （2.0.0。新的Spring Data Neo4j主版本2.0的M1）现已上市！

在过去的几周中，工程师们一直在忙于以新名称对现有库进行改造，以使其适合其展示
下周在Spring One 2GX上 。

内部重大重构将框架分为几个子模块，每个子模块都解决了不同的问题。

  • spring-data-neo4j：Neo4jTemplate，用于使用持久性实体元信息轻松复制对象图映射和Spring Data Repository
  • spring-data-neo4j-aspects：使用AspectJ进行透明的对象图映射
  • spring-data-neo4j-cross-store：JPA和Neo4j之间基于AspectJ的跨存储持久性
  • spring-data-neo4j-rest：远程访问Neo4j REST服务器的透明访问

升级SpringSource Tool Suite 2.8.0的Maven集成

工程
安德鲁·艾森伯格
2011年10月18日

更新，11月23日：我们发布了STS 2.8.1，该版本修复了人们遇到的许多m2eclipse问题的安装/卸载。阅读发行公告。 现在，可以轻松升级/降级m2eclipse。STS 2.8.1完全支持m2eclipse的两个版本。

今天，我们发布了SpringSource Tool Suite的 2.8.0版本。 随着更新到Eclipse的靛蓝SR1（其为您带来的Java7完全支持IDE），我们还更新了从旧的和没有维护Maven的整合m2eclipse的版本0.12.x到最新版本M2E 1.0 -这是现在出货的部分Eclipse发布火车。STS 2.8.0软件包附带了m2e 1.0，并预先安装了几个兼容的加载项（项目配置器）。我们还实施了一个迁移助手，可以帮助您将现有项目升级到新的m2e版本。该迁移助手会将所有旧版Maven项目的类路径，构建器和项目性质转换为与m2e 1.0兼容。它不会更新旧的启动配置或项目/工作区设置。这些将需要手动迁移。

具有Oracle数据库支持1.0.0的Spring Data JDBC扩展。RC1发布

发布
托马斯·里斯伯格
2011年10月18日

尊敬的Spring社区，我们很高兴地宣布候选发布版（1.0.0。具有Oracle数据库支持的Spring Data JDBC Extensions 1.0项目的RC1）现在可用！

创建新的Spring Data JDBC Extensions项目是为了为特定于供应商的JDBC扩展提供附加支持，以及使用QueryDSL之类的JDBC的新方法。大部分支持由从SpringSource项目“ Oracle数据库高级包”移植的代码组成，该代码可用于支持订阅客户。现在，我们将该代码提供给所有Spring用户，并且任何新的开发都将在Spring Data JDBC Extensions项目中进行。
