其他演讲

Spring MVC 3.1更新 -Rossen Stoyanchev ：
该会议将深入探讨有关Spring MVC 3.1的更新。主要主题包括用于处理带注释的控制器方法的新基础结构，名称空间配置的改进，对话管理和Servlet 3.0支持。
 
Spring集成更新 -Josh Long
本课程提供有关Spring Integration 2.0的新功能（包括新的Spring Integration ROO附加组件）的导览。在此过程中，您将了解Spring Integration对Spring Framework 3.0功能的支持，例如Spring Expression Language，ConversionService和RestTemplate。您还将了解一些新适配器，包括AMQP，XMPP，TCP / UDP，JDBC，JMX等。
 
针对虚拟环境调整Java - Ben Corrie
本实践课程详细讨论如何在虚拟运行时如何从JVM中获得最佳性能，尤其着重于内存管理。该会议将花一些时间研究现有的最佳实践及其背后的技术原因，同时还将提供最新的数据和建议以及SpringSource为增强Java所做的工作的前瞻性观点。在虚拟上。
 
Spring开发人员的Cloud Foundry - Josh Long
Cloud Foundry是VMware提供的新的开源平台即服务，它提供了一个出色的部署平台，可以在自动缩放，自动管理的云环境中运行Spring应用程序。本演示文稿描述了如何利用MongoDB，Redis和RabbitMQ等云服务，并阐明了在Cloud Foundry上执行Java背后的架构细节。
 
Apache Tomcat 7更新 -Mark Thomas
Apache Tomcat是当今企业市场中部署最广泛的应用服务器。开发人员，QA团队和IT经理都在各种部署中使用Tomcat，并取得了令人难以置信的成功。本次会议在受欢迎的Apache项目中进行了分析，以分析Apache Tomcat 7中可用的新功能，包括异步请求处理，内存泄漏检测/预防和安全性增强。
 
Spring AMQP和Rabbit消息传递 -Dave Syer
Spring AMQP项目的目标是简化基于AMQP协议的消息传递应用程序的开发。Spring AMQP提供了可跨RabbitMQ和Apache Qpid实现的可移植Java和.NET API，以及方便的抽象，这些抽象促进了基于POJO的编程模型。如果您熟悉Spring的JMS支持，您将感到宾至如归。在本节中，您将浏览Spring AMQP功能，例如发布，消息转换器和创建多线程使用者。还将涵盖对RabbitMQ服务器管理的支持及其使用Spring的配置。
 
使用Gemfire进行缓存和可扩展的数据访问-Costin Leau
关于如何通过缓存在Spring应用程序中轻松获得收益的实用见解。本课程将分析常见应用程序堆栈中常见的瓶颈以及解决这些问题的方法。将讨论各种缓存模式，不仅关注性能，还关注可伸缩性，包括如何充分利用vFabric Gemfire。
 
云中的 Grails-彼得·莱德布鲁克
本演讲将调查提供Grails支持的各种云运行时平台，并讨论每个平台固有的设计约束。该会话将提供有关创建Grails应用程序的建议，以充分利用这些新的部署目标。
 
调整Grails应用程序 -Peter Ledbrook
Grails使Web应用程序的启动和运行变得异常容易，但是并不能保证该应用程序的性能和伸缩性。如果每个请求发出数百个数据库查询，则您的应用程序将不会成为Speedy Gonzalez。了解如何跟踪和修复应用程序各个部分的瓶颈，尤其是数据库访问和视图呈现。

Spring BlazeDS集成1.5.0 GA发布

发布
杰里米·格莱尔
2011年6月28日

亲爱的春季社区，

我很高兴地宣布， Spring BlazeDS Integration的1.5.0 GA版本已发布，该版本是使用Adobe Flex构建基于Spring的RIA的开源解决方案。

下载 | 参考文档 | JavaDocs | 变更日志

自发布候选版本以来，此版本仅包含一些错误修复。回顾以前的里程碑版本，1.5.0中包含的主要功能包括：

  • 增强的AMF序列化支持，可以正确处理Hibernate惰性属性和集合，可以选择使用直接字段映射而不是基于属性的映射，并支持将备用构造函数指定为默认的无参数构造函数。

  • 对Spring Security支持的重大重构，使得Flex应用程序与各种Spring Security功能（如“记住我”，会话固定保护和并发会话控制）的集成更加容易。

  • 框架提供的服务（例如LoginCommand和MessageInterceptor）的自定义和命名空间支持的配置更加容易。

  • 支持使用RESTful Spring MVC @Controllers读写AMF内容。这可以作为远程服务的直接RPC方法的不错选择，从而减少了需要支持多种客户端类型的应用程序中的重复工作。（有关使用示例，请参阅更新的“测试驱动器”。）

  • 将最低依赖关系升级到Spring 3.0，BlazeDS 4.0，Spring Security 3.0和Spring Integration 2.0

  • 基于Maven的Test Drive示例的完整修订版，包括使用Flex 4的全面更新，以及演示RESTful编程模型的新示例。

春天的一周：2011年6月14日

工程
乔什·朗（Josh Long）
2011年6月14日

欢迎回到“春季的这一周”的另一期，已经过去了一周！上周这是Spring 3.1 M2 和 vFabric 5的发布 ！那里有很多令人兴奋的话题以及社区新闻，所以让我们开始吧！

  1. 今天，VMware 宣布发布了VMware vFabric 5 ，该应用程序平台为云和虚拟化执行环境定义了企业Java的未来。 vFabric 5包含Spring社区已经熟悉的许多技术，包括tc Server ， Hyperic ， GemFire和RabbitMQ ，但是现在增加了一些新技术。
    • Java弹性内存 （EM4J）：tc Server的一项新功能，可在应用程序服务器和基础虚拟机之间提供全新的协调级别。EM4J使用基础的vSphere虚拟化来克服Java静态内存堆的某些限制。
    • Spring Insight Operations ：利用与Spring Insight项目相同的代码级跟踪技术，但将来自多个应用程序服务器的信息汇总到一个控制台中，并具有汇总视图，下钻和历史比较，可用于生产系统。
    • SQLFire ： vFabric SQLFire利用了久经考验的vFabric GemFire基础，以内存速度和水平规模提供数据，但vFabric SQLFire向该服务添加了熟悉的标准SQL和JDBC接口。

    Rod Johnson在其最新博客中讨论了该版本的所有详细信息。请务必查看最新版本并进行尝试。

  2. Spring核心负责人Juergen Hoeller宣布Spring 3.1.0 M2已发布 ！终于，稳步迈向Spring 3.1 GA的下一步了！新版本的功能与上一版本一样，其中包含许多主要的新功能，包括（但绝对不限于！）改进的Java配置支持，无XML和无麻烦的基于Servlet 3.0的Spring MVC应用程序引导，新Builder JPA和Hibernate的API，还有更多！在此处查看发行公告，并从您选择的构建依赖项管理工具或下载页面中获取内容。
  3. 紧随Spring 3.1发布公告之后， Chris Beams加入了经过改进的Spring 3.1 M2中以Java为中心的配置模型，即使与M1相比也是如此！这些功能真的开始融合在一起，使它成为最流畅，排列合理的版本之一！
  4. Spring 3.1 M2代表了核心Spring以及Spring MVC的显着改进！ Rossen Stoyanchev会介绍很多（确实，您需要阅读详细的博客才能了解一个主意-我什至不用在这里列举所有这些！）Spring MVC 3.1 M2中的新功能，包括Servlet 3.0容器上不包含XML的Web应用程序。看看这个！

  6. 使用Spring Roo？想塑造未来吗？现在是提供意见的好时机！Spring Roo小组希望您填写有关Spring Roo方向的快速调查。
