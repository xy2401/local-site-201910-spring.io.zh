Spring Web Flow 1.0.2发布

发布
基思·唐纳德
2007年4月11日

亲爱的春季社区，

我们很高兴地宣布Spring Web Flow 1.0.2已经发布。这是一个错误修复和增强版本，解决了针对Spring Web Flow 1.0系列报告的所有错误，并显着增强了Spring Web Flow对Java Server Faces（JSF）用户的支持。我们建议从以前的版本升级到此版本。

 
下载 | 实用介绍 | 样品 参考手册 变更日志

Spring Web Flow是下一代Java Web应用程序控制器框架。该框架提供了一个功能强大的系统，用于在各种环境中一致地实现导航逻辑和管理应用程序状态。

此版本中的新功能和值得注意的功能

Spring Web Flow 1.0.2是一个稳定可靠的发行版，可以作为现有用户的直接替代品，并且还包含一些值得注意的增强功能。特别值得一提的是其中的三个增强功能：

大大增强了JSF集成

现在，当用作JSF扩展时，Spring Web Flow提供了：

  • 一个NavigationHandler，它将Web Flow导航系统的全部功能带给JSF开发人员。该系统允许实现动态导航规则，并立即解决臭名昭著的后退按钮问题。
  • 一种状态管理系统，向UI组件提供对在Web Flow的任何对话数据范围（包括“对话”，“流”和“ Flash”范围）中管理的bean的完全访问权限。这些范围补充了JSF的默认范围，对于使用诸如Ajax4JSF之类的库应用Ajax技术的交互式Web应用程序特别有用。
  • 对所有主要JSF视图技术的本地支持。使用Spring Web Flow 1.0.2，无论是使用JSP还是Facelets构建的，参与流程的视图都是标准的JSF视图。现在，任何JSF UI组件都可以绑定到在任何会话范围内无缝管理的bean。

请运行并查看sellitem-jsf示例 ，以快速评估这些增强功能。它们极大地提高了在JSF环境中使用Spring Web Flow的便利性，并且是更大的努力的开始，并将继续到1.1版本及以后的版本中。

扩展的实用文档

现在，参考手册中提供了每个Spring Web Flow示例应用程序的完整演练。这些演练将引导您完成每个示例的实现，并在此过程中解释最佳实践和设计注意事项。下载该版本后，请参见“实用”一章 ，以获取如何在IDE中运行示例应用程序的信息。

Spring IDE 2.0集成

即将发布的Spring IDE Eclipse插件2.0版本具有图形Web流程编辑器和XML流程定义编辑器。从Spring Web Flow 1.0.2开始，每个示例应用程序均已启用Spring IDE 2.0，从而使您可以在这些工具进展时轻松评估它们。要评估Spring IDE 2.0，只需将示例项目导入Eclipse，然后从每晚更新站点安装最新版本的Spring IDE 2.0插件。

Spring Web Flow 1.1路线图

Spring Web Flow 1.1的工作已经开始，预计第一个候选版本将在JavaOne上可用。此发行版以1.0.2为基础，将提供主要的新功能，包括对会话持久性上下文的支持，Acegi Security集成，统一的EL集成，集成的Spring 2.0自定义范围，对基于Java的流的增强支持以及对流组成和继承的支持。

特别感谢Jeremy Grelle，Rossen Stoyanchev和Christian Dupuis对此版本的主要贡献。加入Web Flow社区是一个令人兴奋的时刻！
