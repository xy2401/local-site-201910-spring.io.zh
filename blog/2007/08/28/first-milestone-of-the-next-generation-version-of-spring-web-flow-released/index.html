Spring Web Flow下一代版本的第一个里程碑发布

发布
基思·唐纳德
2007年8月28日

亲爱的春季社区，

我们很高兴地宣布，Spring Web Flow的下一代版本的第一个里程碑现已发布。Spring Web Flow 2.0 M1引入了几个主要的新功能，包括对流管理的持久性上下文的支持，对Java Server Faces的改进支持，完整的统一表达语言（EL）支持以及更全面的示例Web应用程序 。

Spring Web Flow 2.0是用于开发Java Web应用程序控制器的下一代框架。该框架提供了统一的运行时，用于在各种环境中执行无状态和有状态的客户端交互。

Web Flow 2.0版本的目标

今天，大多数应用程序开发人员通过将Web Flow插入其“基本” Web框架（如Spring MVC）中，使用Spring Web Flow来实现线性页面流（如向导）。这样的页面流表现出线性的导航规则，是有状态的，并形成Web应用程序功能的动态，独立的模块。贷款申请流程就是一个很好的例子：Web Flow独特的流程定义语言为实现这些类型的流程提供了自然的编程模型。

Spring Web Flow 2.0的总体目标是正式将产品从如今的形式（主要用于实现线性向导的框架）发展为始终设计为：支持各种类型的客户端交互的通用应用程序控制器引擎 。 。这样的交互包括向导，无状态的“ RESTful”交互以及“ web 2.0”应用程序中经常出现的更细粒度的非线性/异步交互。

Spring Web Flow 2.0将为Spring社区提供统一的应用程序控制器框架和运行时，适用于执行所有类型的客户端交互，并能够集成各种视图呈现技术和UI组件模型。这种统一的运行时将使管理工具，安全性，AJAX和受管持久性以及其他独特功能的一致性应用成为可能。

Web Flow 2.0 M1全新且值得关注

第一个2.0发行里程碑介绍了通往2.0最终版的几个主要新功能。这些功能如下所述。

使用JPA和Hibernate进行流管理的持久性上下文

Spring Web Flow 2.0 M1通过Hibernate和JPA引入了对Flow Managed Persistence Context的支持。版本中包含的新的酒店预订样本应用程序演示了此功能。以下是预订示例中的工作方式：

  • 当新的酒店预订流程开始时，将自动为您创建一个持久性上下文。
  • 随着流程的进行，持久化上下文将自动用于所有数据访问操作。您不必担心定位受流量限制的EntityManager实例，或以任何方式对其进行管理。
  • 授权预订后，对托管持久性实体的所有更改都会被提交并自动刷新回数据库。如果您选择取消预订，则不会进行任何更改。

改进了对Java Server Faces（JSF）的支持

Spring Web Flow 2.0 M1引入了Spring Faces模块（spring-faces-2.0-m1.jar），该组件随Web Flow发行版一起提供，其中包含对使用Java Server Faces开发Web应用程序的组织的一流支持。该项目已经排除了先前存在的Web Flow + JSF集成，并且该项目将成为将来所有JSF集成工作的所在地。

Spring Faces模块为Spring社区提供了一个专门的项目，用于探索其他JSF集成机会。2.0 M1中的最初工作引入了与流行的Javascript GUI小部件框架Ext的集成。提供了几个轻量级的JSF UI组件，这些组件封装了丰富的Ext小部件的呈现。我们采用的方法允许Ext小部件装饰标准的JSF组件，添加类似于桌面的外观和其他UI行为，例如客户端验证。2.0 M1中提供了以下Ext组件装饰器：

  • 日期验证器组件，用于对日期文本字段执行丰富的客户端验证，并带有美观的日期选择器控件。
  • 文本验证器，可在自由格式的文本输入字段上执行丰富的客户端验证。
  • 一个数字验证器，它对数字文本字段执行丰富的客户端验证。

Ext控件看起来很棒，并且由于所有UI行为都在客户端执行，因此UI的响应性极佳。作为Spring Faces模块的一部分，我们将在以后的Web Flow发行里程碑中在此领域中提供进一步的支持。

拥有专用的Spring Faces模块还可以简化在JSF环境中设置Spring Web Flow的过程。在2.0 M1之前，开发人员需要手动修改faces-config.xml来设置样板Spring / SWF集成管道。使用2.0 m1，只需在类路径中包含spring-faces-2.0-m1.jar，即可自动为您完成此设置。

统一表达语言（EL）支持

此版本中的新增功能是对Unified EL的完全支持，它提供了Spring Web Flow中默认使用的基于OGNL的表达式解析器的替代方法。现在建议在JSF环境中使用统一的EL，并且在将来的里程碑中，它将成为JSF开发人员的默认方法。

2.0发行说明

Spring Web Flow 2.0需要Spring 2.0或更高版本以及Java 1.4或更高版本。

Spring Web Flow 2.0将尽可能与基于XML的1.0.x流定义语言保持向后兼容性。随着Web Flow演变成一个完整的控制器框架，预计某些SWF SPI将在将来的Web Flow 2.x里程碑中发生变化。

Spring Web Flow 2.0的未来里程碑将引入新的方言，以实际的编程语言定义流程，为当前基于XML的流程定义语法提供更强大的替代方法。基于POJO的Java流定义语法以及基于Groovy的方法都在研究中。此外，还将介绍用于实现REST-ful和CRUD流的高级流方言。因为Web Flow定义API和执行运行时与任何具体的流定义语法是分开的，所以我们自然可以引入新的语法，以不同的方式在运行时对应用程序控制器进行工程设计。

对于JSF用户，Spring Web Flow 2.0和Spring Faces可在JSF 1.1和JSF 1.2上运行。Spring Faces模块中的UI组件库独立于Web Flow运行时，并且还可以与JSF的默认导航处理程序一起使用。

Spring Web Flow 2.0将继续提供对Servlet 2.4或更高版本，Portlet 1.0或更高版本以及JUnit 3.8.1或更高版本环境的全面支持。

Spring Web Flow 2.0将继续在Struts 1.2或更高版本以及Spring MVC 2.0或更高版本的环境中运行，并且在最终版本2.0之前也可以用作独立的Web应用程序控制器框架。
