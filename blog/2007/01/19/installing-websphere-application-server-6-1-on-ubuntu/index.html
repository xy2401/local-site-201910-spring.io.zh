在Ubuntu上安装WebSphere Application Server 6.1

工程
罗伯·哈罗普
2007年1月19日

最近，我一直在与WAS 6.1的客户端进行一些工作。由于WAS上有许多Spring用户，并且我需要测试该应用程序，因此我决定是时候在我的一台工作笔记本电脑上运行WAS的副本了。我说“其中之一”是因为我目前同时在Mac（带OSX）和ThinkPad（带Ubuntu）上工作-最近，我一直在使用ThinkPad，因为我可以运行Oracle XE和WAS而不需要用于诸如Parallels的VM工具。我仍然更喜欢Mac，但是老实说，每天之间并没有太大的区别-我只是想念一些更有用的Mac工具，例如Spotlight ， Quicksilver ， TextMate和NewsFire 。

无论如何，回到主要主题-在Ubuntu上安装WAS 6.1。我使用的是Ubuntu Edgy，而我的第一次安装尝试完全失败了，我只是不知道为什么。幸运的是，一个快速的Google发表了这篇文章。我完全不知道/ bin / sh是链接到破折号而不是bash的-到底是什么拥有了它们。我真的不喜欢运行安装程序的建议解决方案，让安装程序失败，然后更改已安装目录中的所有脚本。相反，我只是用快速的sudo unlink / bin / sh重新链接/ bin / sh ，然后是sudo ln -s / bin / bash / bin / sh 。之后，安装程序如梦似幻地运行，我在大约15分钟的时间内启动并运行了WAS安装。

即使在同时运行Oracle XE的ThinkPad上，WAS的运行速度也相当快。关于WAS的最好的事情之一就是所提供的工具（管理控制台，命令行工具）确实非常强大。管理控制台的性能引人注目-许多其他服务器的控制台速度非常慢。
