Spring Web Flow Bean范围和JSF

工程
本·黑尔
2007年5月8日

我最近在Spring Web Flow中完成了一个有趣的问题。该问题（ SWF-163 ）涉及为Spring Web Flow的内部范围添加Spring 2.0 bean范围支持。实现并不是那么有趣（毕竟Scope接口很容易实现），但是我想确切地提到您将如何在应用程序中使用类似的东西。

Spring 2.0范围界定

在Spring 1.x中，我们想到了singleton和原型bean作用域，但是这种表示法是固定的，并且对singleton =” [true |假]” 。因此，在Spring 2.0中，此表示法已从XSD的配置样式中删除，现在您看到的表示法在scope =” [singleton |原型| …]” 。Spring本身增加了三个bean作用域；与Web应用程序相关的request ， session和globalSession 。

使用Spring Web Flow 1.1的最新快照，我们现在可以看到三个主要Web Flow范围（即flash ， flow和session）的 bean作用域。


<bean id="sale" class="org.springframework.webflow.samples.sellitem.Sale" scope="flash"/>
<bean id="sale" class="org.springframework.webflow.samples.sellitem.Sale" scope="flow"/>
<bean id="sale" class="org.springframework.webflow.samples.sellitem.Sale" scope="conversation"/>

要利用这些bean作用域，您需要利用新的1.1版配置（包含在Web Flow jar中），并向bean定义中添加一个元素。


<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
        xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
        xmlns:flow="http://www.springframework.org/schema/webflow-config"
        xsi:schemaLocation="
            http://www.springframework.org/schema/beans
            http://www.springframework.org/schema/beans/spring-beans-2.0.xsd
            http://www.springframework.org/schema/webflow-config
            http://www.springframework.org/schema/webflow-config/spring-webflow-config-1.1.xsd">

	<flow:enable-scopes/>

	<bean id="sale" class="org.springframework.webflow.samples.sellitem.Sale" scope="conversation"/>

</beans>

的标签仅需要在给定的应用程序上下文中存在一次，并且允许您使用Spring Web Flow贡献的三个范围中的任何一个。

范围豆和JSF

现在，这些新范围的最引人注目的用途是在JSF中。通过在标准Spring bean定义文件中使用范围表示法，而不是在流定义中使用符号表示法，JSF用户的体验更接近于标准JSF行为。要向JSF添加此功能，请注册标准Spring JSF DelegatingVariableResolver 。其他定义（ FlowNavigationHandler和FlowPhaseListener ）是将JSF与Spring结合使用的标准定义。


<?xml version="1.0"?>
<!DOCTYPE faces-config PUBLIC
  "-//Sun Microsystems, Inc.//DTD JavaServer Faces Config 1.0//EN"
  "http://java.sun.com/dtd/web-facesconfig_1_0.dtd">

<faces-config>
    <application>
        <navigation-handler>
            org.springframework.webflow.executor.jsf.FlowNavigationHandler
        </navigation-handler>
        <variable-resolver>
            org.springframework.web.jsf.DelegatingVariableResolver
        </variable-resolver>
    </application>

    <lifecycle>
        <phase-listener>
            org.springframework.webflow.executor.jsf.FlowPhaseListener
        </phase-listener>
    </lifecycle>
</faces-config>

与以前启用Web Flow的配置相比，主要的变化是现在变量解析器是Spring而不是Spring Web Flow的。当JSP页面查找sale变量时，JSF将委托Spring进行bean解析，并且将根据其定义上的scope属性对bean实例进行范围划分。

结论

最终的结果是，现在JSF用户现在可以使用类似于内置样式的语法，但是使用的容器功能更强大。

如果您想使用此新功能，它将在Spring Web Flow 1.1-m1发行版中很快推出，或者您可以通过下载最新的Spring Web Flow 1.1-m1每夜快照来预览。
