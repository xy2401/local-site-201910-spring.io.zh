春季产品组合中的新版本

工程
阿德里安·科耶尔（Adrian Colyer）
2007年5月25日

去年下半年，我们开始谈论春季“发行火车”的概念。发布流程背后的想法是，我们在Spring产品组合中发布了产品的协调发布：一起测试并一起工作。您仍然可以选择所需的零件，但是在需要时可以更轻松地将各种产品一起使用。我们还没有到那儿，但是我们正在路上。

Interface21上对我们的挣扎之一是对我们的支持服务，培训和咨询的需求如此之高，以至于我们一直在努力让所有人都努力去实现它。这使得我们很难获得一致的和可预测的产品开发时间，而这正是我们需要发布发行培训之类的东西的时间。这只是我对Benchmark Capital在Interface21上进行的1000万美元投资感到非常兴奋的众多原因之一（ 新闻稿 ）。这将使我们能够加快产品开发工作，并比过去更快地为您带来更多出色的版本。

所以我们还没有到那儿，但是我们正在路上。同时，您是否注意到目前几乎同时发布了多少Spring Portfolio版本；）。


  • 春季批 宣布

  • Spring Web Services发布了1.0rc1 ，并且1.0 即将发布

  • Spring OSGi发布了m2版本，我们正在朝m3迈进

  • Acegi刚刚发布了1.0.4 ，很快您就会看到“ Spring Security 2.0.0 m1”，因为我们将Acegi完全引入了Spring Portfolio，并将其与Spring 2中的名称空间支持集成在一起以简化配置。

  • Spring Java Configuration发布了1.0平方米 。

  • Spring Web Flow即将发布1.0.4版本，以及1.1 m1。我不会在这里窃取Keith的雷声，但您会喜欢它的:)。在他最近在此站点上的博客文章中查看视频 ，以了解一些很棒的新JSF集成。

  • Spring Framework 2.1 m1已发布，它具有对基于注释的配置的改进支持（可以选择 ：使用注释，xml或满足您需求的任何组合。如果清楚的是一件事，那就是不同的人都有不同的偏好。

  • Spring .NET 1.1 M1迫在眉睫

  • Spring IDE 2.0 m4已发布，而2.0 final即将推出。感谢Spring IDE团队在这里所做的出色工作。我已经看到了即将到来的Mylar集成的屏幕截图，而且这会令人震惊！

  • Spring LDAP 1.2 RC1已发布（这也是一个社区主导的项目，因此Interface21对此不承担任何责任，但我们当然非常感谢）。


进行了如此多的工作，您一定不想错过今年的SpringOne ，在那里您可以听到所有有关这些令人兴奋的新版本的信息。

哦，对于那些好奇的人，我们目前的目标是在7月底之前发布Spring Framework 2.1的最终版本。
