会议季节由SpringOne决定！

工程
罗德·约翰逊
2007年5月28日

自从我有时间写博客已经有一段时间了。我们一直很忙。我们筹集了1000万美元 。正如Adrian指出的那样 ，我们在产品开发方面一直非常活跃。在过去的几个月中，我自己编写了比平常更多的代码。（主要是实验性的东西，可能看不到，但是很有趣，有时我做的事情很有用。）我已经花了很多时间与新闻界和分析师们交流。这些天来，我们引起了媒体的极大兴趣。新闻/分析人员的电话可能很累，但也可能很有价值，因为其中许多人都很聪明，会提出发人深省的问题。

会议季节占用了很多时间。JavaOne对我们来说是一个大展示，我们所有的会议对Spring都充满了兴趣。Interface21首次成为参展商，我们所有人都精疲力尽，想与来到我们展位的每个人交谈。如果您还没有做过，那么您将不感激使用这些徽章扫描仪之一会带来多大的疲劳……下周， Mark Pollack和我在Tech-Ed上谈论Spring.NET 。这是我的第一本Tech-Ed，将形成有趣的对比。

对于Sun和Microsoft的节目来说，就这么多。夏季发布会上赛季为Spring团队的亮点将是我们在欧洲的演出， SpringOne上 ，在比利时安特卫普，从6月20-22今年，SpringOne已延长至3天，这将是与Spring开发人员建立联系的绝好机会，同时您可以享受世界上最好的啤酒。注册在这里 。

所有最近和即将发布的版本都意味着我们将有很多话题。

正如Mark Fisher最近写的那样 ，我们在Spring 2.1M1中引入了一个新的配置选项，该选项允许使用注释来补充甚至替换XML bean定义。

在Spring团队中，我们一直在谈论有关配置的最佳实践，并计划编写和讨论我们的结论。值得一提大图，因为很多人似乎都错过了它：

春季> XML



Spring容器具有其自己的内部元数据，并且可以从不同的配置源进行贡献。例如，长期以来我们一直支持属性文件覆盖。Pitchfork项目混合了Java EE 5.0注释。定义Spring组件的方法有很多。无论您做什么，您都可以使用Spring的现成服务，扩展钩子（例如后处理器），真正的AOP，Spring与无数第三方产品的集成……

因此，通过支持其他配置格式，我们不会弃用人们通常用于配置Spring的XML语法：仅提供与Spring架构一致的更多选择。

在SpringOne上，Costin Leau将讨论配置Spring容器的方法 ，这应该是一个有趣的环节，并将继续解决这一问题。Costin领导的Spring Portfolio项目之一是Spring Java Configuration ，它源于几年前的一些实验， 我在去年11月写过博客 。该项目使用Java配置Spring：当然可以将其与其他形式的配置混合使用。

SpringOne应该是与Spring团队互动并反馈您的建议的好机会。

春季一期过后，我期待在同一地方定居超过几个星期。而且我希望有更多时间来写我过去几个月一直在想的所有博客文章！
