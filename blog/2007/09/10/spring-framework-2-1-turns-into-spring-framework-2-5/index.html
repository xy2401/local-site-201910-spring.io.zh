Spring Framework 2.1变成了Spring Framework 2.5！

工程
于尔根·霍勒（Juergen Hoeller）
2007年9月10日

我的第一篇博客文章-以及什么是重要的宣布:-)

在一系列Spring 2.1里程碑发布之后，我们一直在审查引入的全部功能：


  • 完全支持Java 6和Java EE 5 （JDBC 4.0，JTA 1.1，JavaMail 1.4，JAX-WS 2.0等）

  • 全功能注释驱动的依赖项注入 （包括对“限定符”注释的支持）

  • 支持类路径中的组件扫描 （自动检测带注释的类）

  • AspectJ切入点表达式中的bean名称切入点元素

  • 对AspectJ加载时编织的内置支持（基于Spring的LoadTimeWeaver抽象）

  • 更多的XML配置名称空间 （“ context”，“ jms”），以提供最大的便利

  • 扩展的SimpleJdbcTemplate功能（支持命名参数等）

  • 官方认证的WebSphere支持 （对WebSphere 6 UOWManager工具的支持等）

  • Spring Framework jar作为兼容OSGi的捆绑包出厂时提供

  • 可以将Spring ApplicationContext 部署为JCA RAR文件 （用于无头应用程序模块）

  • JCA 1.5消息端点管理 （用于Spring管理的JMS和CCI消息侦听器）

  • 完全修订的集成测试框架 （支持JUnit 4和TestNG）


以及细节方面的许多改进。在最近发布的2.1 M4中，几乎所有这些功能都可用。

我们得出的结论是，这大大超出了我们最初针对Spring 2.1计划的范围。里程碑使用的版本号2.1并未反映此主要版本中功能的重要性和全面性。

因此，我很高兴地宣布， 作为即将发布的主要版本 ，2.1里程碑将无缝地转变为Spring Framework 2.5！ Spring 2.5计划于10月最终发布，首批候选版本预计于10月初发布。当我们接近最终版本时，请注意即将发布的有关Spring 2.5特定功能的博客文章！

请注意，按照最初的计划，Spring 2.5仍将与JDK 1.4.2+和J2EE 1.3+兼容 。尽管许多新的主要功能固有地需要Java 5或更高版本，但Spring 2.5还包括Java 1.4用户也可以使用的重大增强功能：例如，bean名称切入点元素，jms配置名称空间，经过认证的WebSphere支持和JCA RAR。部署选项。
