Spring Framework 2.1 M4发布

发布
于尔根·霍勒（Juergen Hoeller）
2007年9月10日

亲爱的春季社区，

我很高兴地宣布Spring Framework 2.1 M4已发布！此里程碑版本介绍：

  • 'qualifier'注释支持，用于选择特定的@Autowired匹配项；
  • 支持JUnit4的下一代Spring TestContext框架；
  • SimpleJdbcCall操作对象，用于存储过程调用；
  • 支持脚本对象（Groovy，JRuby，BeanShell）的自动装配；
  • 在Spring Web MVC中支持Tiles2视图。

下载 | 说明文件 | 变更日志

 

这使我想到了更多令人振奋的消息……

我们一直在审查最近2.1里程碑中引入的全部功能：

  • 全面的Java 6和Java EE 5支持
  • 全功能注释驱动的依赖注入
  • 支持类路径中的组件扫描
  • AspectJ切入点表达式中的“ beanName”切入点元素
  • 内置对AspectJ加载时编织的支持
  • 其他XML配置名称空间（上下文，jms）
  • 扩展的SimpleJdbcTemplate功能
  • 官方认证的WebSphere支持
  • Spring ApplicationContext可以部署为RAR文件
  • JCA 1.5消息端点管理（用于JMS和CCI）
  • 完全修订的集成测试框架

我们得出的结论是，这大大超出了我们最初针对Spring 2.1计划的范围。里程碑使用的版本号2.1并未反映此发行版中功能的重要性和全面性。

因此，我很高兴地宣布，下一个版本将称为Spring Framework 2.5 RC1，其中2.1里程碑将无缝引入Spring Framework 2.5作为即将发布的主要版本！在我们逐步接近最终版本时，请查看Interface21团队博客以获取正式声明和有关特定Spring 2.5功能的后续帖子。

Spring 2.1 M4在这里。Spring 2.5万岁！ :-)

请享用，
