Maven工件

工程
本·黑尔
2007年9月18日

到现在为止，不确定地创建了Spring Portfolio Maven工件，尤其是快照，并散布在各个位置。在过去的几周中，我们一直在努力使项目在创建和上传这些工件时更加一致。

Maven仓库

Spring产品组合中对Maven支持的最有用的改进之一是使用一致的存储库位置。根据您对代码的熟悉程度，共有三个不同的存储库。

版本库

对于任何最终版本（Spring 2.5，Spring Web Flow 2.0等），该版本的Maven工件都将上传到Maven Central存储库（ http://repo1.maven.org/maven2 ）。使用该存储库无需您费力，因为Maven会自动在此处查找工件。

此存储库中的工件确实遵循预期的存储库行为，并且不会（也无法）不会被删除。

里程碑资料库

对于任何里程碑版本（Spring 2.5-RC1，Spring Web Flow 2.0-M2等），该版本的Maven工件都将上传到Spring Milestone存储库（ http://s3.amazonaws.com/maven.springframework）。 org / milestone ）。使用此存储库需要您将条目添加到 POM中的元素。它看起来应该像这样：


<repository>
    <id>spring-milestone</id>
    <name>Spring Portfolio Milestone Repository</name>
    <url>http://s3.amazonaws.com/maven.springframework.org/milestone</url>
</repository>

该存储库中的工件不遵循预期的存储库行为，将定期删除。在发布最终版本（Spring 2.6，Spring Web Flow 2.1等）之后，将删除工件先前版本中的所有里程碑版本。例如，当发布Spring 2.6时，将删除Spring 2.5里程碑，而将保留Spring 2.6里程碑。

快照存储库

对于任何快照构建（Spring 2.5-SNAPSHOT，Spring Web Flow 2.0-SNAPSHOT等），该构建的Maven工件都将上传到Spring Snapshot存储库（ http://s3.amazonaws.com/maven.springframework.org / snapshot ）。使用此存储库需要您将条目添加到 POM中的元素。它看起来应该像这样：


<repository>
    <id>spring-snapshot</id>
    <name>Spring Portfolio Snapshot Repository</name>
    <url>http://s3.amazonaws.com/maven.springframework.org/snapshot</url>
</repository>

该存储库中的工件不遵循预期的存储库行为，将定期删除。至少将保留给定工件的最后10个快照构建。如果从发行版中删除工件，其快照构建将立即删除。在发布里程碑或最终版本时，将删除工件的所有快照，并为下一个版本创建新的快照。

资料库浏览

里程碑和快照存储库都托管在Amazon的S3服务上，因此目录结构不易于阅读。要以人类可读的格式查看存储库，请使用S3Browse实用程序。

  • 里程碑资料库： http : //s3browse.com/explore/maven.springframework.org/milestone
  • 快照存储库： http : //s3browse.com/explore/maven.springframework.org/snapshot

仅将这些URL用于人类可读的查看。如果将它们用作POM的URL，则会遇到错误。

神器来源

另一个重要的改进是为所有发行版添加了源工件。您会注意到，在里程碑存储库中，所有工件都已部署了源。当我们继续开发所有最终版本时，也是如此。具体来说，从Spring 2.5发行版开始，除了合并的Spring源代码外，每个模块还将具有一个源工件。

春季快照

最后的改进是尚未完成的。 春季的夜间快照 。我很高兴地说这已经接近完成。我仍在就Maven Ant Tasks进行最后的处理，但这最终将开始显示，并在其发布时再次宣布。同样，您可以期望该功能最终会应用到所有其他基于ANT的Spring组合项目中，以便所有项目都将创建Maven快照以及里程碑。
