春季：企业Java编程中的实际标准

工程
阿德里安·科耶尔（Adrian Colyer）
2007年6月13日

昨天，GigaSpaces宣布了其最新版本的基于空间的体系结构，并且也有了一个新名称： GigaSpaces eXtreme应用程序平台（XAP） 。引用他们的新闻稿：

新版本提供了一个完整的中间件平台，用于管理需要高性能和跨数百台机器水平扩展能力的应用程序的数据，消息传递和业务逻辑。

公告中引起我注意的部分是：
作为新产品发布的一部分，GigaSpaces包含了一个更简单的，非侵入式的编程模型，该模型允许开发人员使用普通旧Java对象（POJO），普通.Net和普通C ++对象编写应用程序。对于Java，GigaSpaces通过支持Spring Framework 来实现这一目标，Spring Framework 已迅速成为企业Java编程中的实际标准 。

看到这种认可真是太好了，我对声明所做的仅有一点改变就是删除了“迅速成为”的一部分：Spring Framework 是 Enterprise Java编程中的事实上的标准。

像这样的公告是一个良性循环的一部分（例如，Geoffrey Moore在他的《大猩猩游戏》一书中进行了描述），Spring Framework 的广泛普及使厂商非常有必要在其产品中提供Spring Framework 集成，这反过来增加Spring的整体价值。当然，这有助于使Spring更加普及，对更多的供应商施加更大的压力以使其更深入地集成。

那么，在产品中“支持Spring”意味着什么？从最简单的角度讲，这意味着要接受Spring的哲学：支持外部化配置和易于测试的简单Java对象。以下是一些使您的产品“春季友好”的建议：


  1. 允许配置由Spring管理 。从最基本的角度讲，这意味着拥有一组配置元数据类，这些元数据类可以在应用程序上下文中作为Spring bean进行连接。尽可能避免创建自己的自定义配置文件和格式。为了进一步简化用户的操作，您可以考虑添加对Spring命名空间的支持，以使其更易于配置。Gigaspace提供了一个“ gigaspaces”命名空间，例如，允许诸如和等元素直接在Spring配置文件中使用。

  2. 在您的API中使用Spring抽象和设计习惯用法 。例如，Spring用户非常熟悉“模板”的概念。GigaSpaces提供了一个“ GigaSpacesTemplate”。

  3. 支持单元和集成测试 。以这样一种方式设计API，即在使用您的产品的Spring应用程序中轻松进行单元测试和集成测试业务逻辑。

  4. 与Spring使用的基础结构服务抽象集成 。例如，GigaSpaces的JMS和JDBC抽象可以直接与Spring一起使用。GigaSpaces还提供了Spring的PlatformTransactionManager的几种实现，以允许Spring Framework 划分基于空间的事务。


当然，并不是所有的集成选项都适用于每种产品，但是这些想法至少应该帮助您入门。
