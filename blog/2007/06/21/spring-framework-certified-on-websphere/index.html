经WebSphere认证的Spring Framework

工程
罗德·约翰逊
2007年6月21日

SpringOne很好地嗡嗡作响。今年是一场为期3天的展览，比去年的2天有所增加，很高兴见到数百名参加春季会议的与会者。有一次，我在一个展览上非常放松，因为在开幕式主题演讲之后，我不再进行任何会议，也不需要处理幻灯片。

目前，Adrian正在准备发布有关Spring工具的重要公告。好吧，实际上他在鸭式打字方面有着独特的个性，我相信您会听到的……

有关更多信息，但首先，我需要分享昨天的一些新闻。我很高兴能够宣布我们已经与IBM合作，以在IBM的旗舰WebSphere Application Server上对Spring进行认证。由于Interface21的大部分客户群都来自大型企业，因此我们看到了很多WebSphere，并希望确保多年来的理想集成。很高兴看到IBM拥有这个目标。他们对集成的承诺很大程度上也是由客户推动的，其中许多客户已经采用了Spring，并在其中取得了丰硕的成果。随着Spring的采用已从逐个项目转变为战略性，这些客户希望知道这种组合是可支持的，并且两家供应商都支持这种组合。

您可以阅读新闻稿 。但是，由于新闻稿是针对管理人员的，往往会在相关公司与公关机构之间进行审查和反弹，以至于毫无意义，因此，对于我来说，用英语而不是用市场术语解释这实际上意味着什么可能更容易。

  • IBM和Interface21已经密切合作了近一年，以在Websphere上测试Spring框架。这远远超出了单纯的营销工作。它已经在运行WebSphere的许多不同平台（包括大型机平台）上进行了广泛的测试。
  • 我们已经解决了有关合并的所有已知问题。无论您使用哪种硬件和OS，都可以在WebSphere平台上放心使用Spring。显然，Spring在WebSphere上已经运行良好，但是这可以解决一些极端的情况，最重要的是，这意味着即使是风险最大的经理也可以理解，在WebSphere上使用Spring是安全的并且可以得到支持。
  • 增强功能中最重要的特定领域涉及事务管理，在WebSphere和Spring中都添加了代码。这项工作由Interface21方面的Juergen Hoeller和IBM的WebSphere事务架构师Ian Robinson领导。

如果您熟悉Java EE和JTA规范，就会知道它们之间存在差距-尤其是如何处理UserTransaction无法做到的重要事情，例如事务挂起。这些是极端情况，但是当您遇到其中一种情况时，可能很难解决。尽管将事务管理委派给Spring比直接使用JTA（出于可测试性，异常处理和语义完整性的原因）要好得多，但如果在底层可以使用所有标准JTA功能，则可能存在问题。

幸运的是，有一个解决方案可以使Spring最大化潜在平台的潜力。Spring提供了一个事务抽象-PlatformTransactionManager-从语义上讲比JTA更完整，但不对其环境进行任何假设。这是所谓的可移植服务抽象的关键示例。通常，您不会直接使用PlatformTransactionManager，因为Spring的声明式事务管理为您提供了将程序化事务应用于任何POJO的独特强大功能，但这是Spring程序化和声明性事务管理的基础。因为我们不对环境做任何假设，并且环境细节不会泄漏到您的代码中并降低可移植性，所以我们可以安全地与平台相关。我们已经为WebLogic做到了这一点，O racle已经为OC4J做到了 ，现在我们已经与IBM合作为WebSphere生成一流的集成。

遵循Spring的核心价值主张，我们会照顾管道问题，因此您不必这样做。如您所料，新的WebSphere事务管理器非常易于使用。如果您将WebSphere与Spring一起使用，则应使用我们与IBM合作开发的新的PlatformTransactionManager抽象。这是一个琐碎的配置设置。您可以使用新的WebSphereUowTransactionManager来代替JtaTransactionManager，如下所示：
