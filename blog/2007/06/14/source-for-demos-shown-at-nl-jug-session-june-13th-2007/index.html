2007年6月13日在NL-JUG会议上演示的演示源

工程
阿列夫·阿伦森（Alef Arendsen）
2007年6月14日

昨天，Joris和我在Dutch Java Users Group进行了一次会议。我们进行了两次会议，总共约有250人参加了会议。很多人要求提供我们在会议期间进行的演示的代码。附件中，您将找到AOP和依赖注入演示的代码。它显示了一个简单的方面，它在每个JDBC操作之前刷新Hibernate会话（虽然不如您在生产代码中所希望的那样健壮，但这只是一个开始），并且还显示了CarPlant系统（在其他会话中已演示，并在之前已附加）到另一个Blog条目），并在Spring 2.1中使用多种方法进行依赖注入（即，使用@Bean和@Autowired）。

这是一个Maven2项目，因此您需要安装Maven2。要准备包括所有库的Eclipse项目，请在carplant目录中的命令行上运行mvn eclipse：eclipse。

来源： carplant.zip

在会议期间，出现了一个应用程序上下文中有关多个块的问题。听众中的那个人不确定两个或更多的AOP配置块是否显示了预期的结果（建议两次或更多次）。我没有这些人的电子邮件地址，所以我想在这里澄清一下。考虑下面的代码。将建议doIt（）。实际建议（为简单起见）与主方法自举ApplicationContext一样，保留在同一类中。


public class Logger {

  public void doIt() {

  }

  public void log() {
    System.out.println("Log!");
  }

  public static void main(String args[]) {
    ApplicationContext context = 
      new ClassPathXmlApplicationContext(
        new String[] {"com/carplant/context1.xml", "com/carplant/context2.xml"});

    Logger logger = (Logger)context.getBean("logger");

    logger.doIt();
  }
}

配置1：一个简单的AOP配置块

在这种情况下，文件context2.xml为空，而context1.xml包含以下代码：


<bean id="logger" class="Logger"/>

<aop:config>
	<aop:pointcut id="doItOperation" expression="execution(* doIt(..))"/>
	<aop:aspect ref="logger">
		<aop:before pointcut-ref="doItOperation" method="log"/>
	</aop:aspect>
</aop:config>

不出所料，调用doIt（）方法后，记录器将仅获得一行输出（仅建议一次）。

配置2：两个不同文件中的两个AOP配置块

现在context2.xml与context1.xml（在上面的示例中）相同，唯一的区别是在context2.xml中我们没有名为logger的bean。运行此方案时，我们将看到两个Log！输出条目。建议两次使用doIt（）方法。

配置3：同一配置文件中的两个AOP配置块

context2.xml再次为空。另一方面，context1.xml现在包含两个块。两者之间的唯一区别是切入点标识符（这是一个XML ID，因此在XML文件中应该是唯一的）。


<bean id="logger" class="Logger"/>

<aop:config>
	<aop:pointcut id="doItOperation" expression="execution(* doIt(..))"/>
	<aop:aspect ref="logger">
		<aop:before pointcut-ref="doItOperation" method="log"/>
	</aop:aspect>
</aop:config>

<aop:config>
	<aop:pointcut id="doItOperation2" expression="execution(* doIt(..))"/>
	<aop:aspect ref="logger">
		<aop:before pointcut-ref="doItOperation2" method="log"/>
	</aop:aspect>
</aop:config>

运行此命令还将显示两次建议使用该bean。

请注意，我在这里使用的是2.1里程碑版本。
