WebLogic 10 Tech Preview发行-在Spring Framework上构建

工程
罗德·约翰逊
2007年2月11日

祝贺WebLogic团队发布了通过Java EE 5 CTS 的WebLogic 10预览版 。很高兴看到BEA在使用J2EE 1.4异常之后重新回到了迅速摆脱困境的传统。

对于Spring社区而言，这是一个有趣的消息，因为WebLogic 10在内部使用Pitchfork 。Pitchfork项目是一个由Interface21领导的开源项目，与BEA共同开发，该项目在Spring的顶部实现了EJB 3.0拦截和JSR-250注入。它可以在WebLogic 10 EJB容器内部以及服务器的其他部分使用，以满足新的规范要求，但并不与WebLogic绑定。Pitchfork可以独立使用（尽管这不是其主要目标），也可以被其他应用程序服务器采用。

顺便说一句，在干草叉项目不应该混同项目干草叉 ，一个德国乐队。Stop Press：我刚刚发现他们有一首歌叫做It's Spring 。但是不能说我是粉丝。

使用Pitchfork意味着从WebLogic 10的运行时角度来看，每个EJB 3.0会话或消息Bean也是Spring Bean。BEA做出此选择的动机是为了缩短上市时间，并尽可能以经过验证的代码为基础来实施新的规范功能。Spring已经做了很多他们需要的事情，这有助于他们制定积极的时间表。它还应有助于确保安全的升级路径。Pitchfork在经过验证的Spring代码之上提供了许多新功能；事实证明，绝大多数现有的WebLogic EJB代码库都是不变的。当然，采用这种方法强调了BEA对Spring质量的信任。

从长远来看，在表面之下使用Spring的事实应该使运行WebLogic的Spring用户受益，允许服务器（已经“有效地”自然地理解了）服务器对Spring组件的深入管理，并启用Spring组件。用于补充Java EE的模型。 EJB负责人Patrick Linskey最近暗示了其中一些功能：


这种体系结构还为我们提供了一些机会，可以在更高的集成水平上向WebLogic Server开发人员展示Spring工件。
[例如]…在WebLogic Server群集中部署Spring Bean； Spring会话复制（在WebLogic HTTP会话复制上pi带）； WebLogic与Spring bean的安全集成；基于WebLogic的Spring Bean的集群远程处理； WebLogic Server管理控制台中对Spring Bean的管理和管理。此外，我们在内部为一些引人注目的新的面向外部的Spring集成功能奠定了基础，但我们尚未在WebLogic Server 10中公开这些功能。

在以前的预览中，可以了解服务器内部的Spring-ness并对其进行实际编程。Spring IDE团队的精力充沛的Christian Dupuis在第一次预览发布时就在博客上发表了这篇文章，展示了每个EJB 3.0会话或消息bean也是一个Spring bean，并且有可能对这些对象应用更强大的Spring注入和真正的AOP。只需提供与EJB同名的Spring bean定义即可使用这些组件，这是一种简单而优雅的方法。我不确定该机制是否在当前预览版中有效，还没有时间检查。我知道BEA的目标是首先交付兼容的服务器，然后研究如何最好地打包和记录增值扩展。

与Pitchfork上的WebLogic EJB团队合作以及在WebLogic中使用它（尤其是Michael Chen和Patrick Linskey）非常有趣。不可避免地，与帕特里克一起工作是因为喝啤酒。

显然，我对Spring角度特别感到兴奋（尤其是当我和Costin Leau参与其中时），但是WebLogic 10中当然还有很多其他重要内容：特别是将Kodo / OpenJPA合并为JPA持久性提供者。该产品的核心是Apache License开放源代码。很高兴看到继续强调QOS功能，而不仅仅是API追逐：例如，自动JMS故障转移。特别是大型企业客户通常比规格级别更关注操作和正常运行时间。

过滤类加载器听起来特别受欢迎（如果过期）。传统上，复杂应用程序中的J2EE类加载是痛苦的事情，即使将它们编码为标准并使用可移植框架，也仍然是移植应用程序的真正障碍。随着服务器（包括WebLogic和WebSphere）在内部越来越多地使用开放源代码，冲突变得越来越普遍，需要进行修复。（就像与Hibernate 3和WebLogic 8.1的ANTRL冲突一样，可以通过使WAR中的库优先于服务器中的库来解决此问题。）WebLogic 10似乎在这里将有一个可靠的解决方案，该解决方案使特定的应用程序能够以更加灵活的方式筛选出服务器所依赖的类。
