Spring 2.0 RC1发布

发布
于尔根·霍勒（Juergen Hoeller）
2006年6月21日

尊敬的Spring社区，

我们很高兴地宣布Spring 2.0 RC1已发布。下载 | 说明文件 | 变更日志

在过去六个月中已发布了五个里程碑，这是Spring 2.0的第一个候选版本。这个稳定的版本正在等待广泛的用户反馈，以期在7月初推出2.0最终版的过程中。

Spring 2.0的主要新主题是：

  1. 简化的配置 -您将发现编写整体的代码要少得多，从而可以配置应用程序。您编写的代码是高级的并且是特定于域的，得益于智能默认设置以及丰富的验证和工具支持。

  2. 强大的AOP统一 -您可以使用AspectJ在对象之前，之后和对象周围应用编织自定义行为的方面，同时仍然受益于Spring AOP运行时的简单性。

  3. JSR 220 Java持久性架构（JPA） -为您提供持久性提供程序实现的选择。Spring是第一个在JEE和JSE环境中都支持JPA且两者之间具有完全可移植性的人。

  4. 异步JMS消息传递-消息驱动的POJO通过选择JMS提供程序为您提供了用于JSE环境的轻量级异步组件。

  5. JSR-168 Portlet -Spring Portlet MVC提供了一个用于开发JSR 168 Portlet的框架，其中包括与Spring Web Flow的集成，以协调用户交互。

Spring Web Flow 1.0

发布
基思·唐纳德
2006年6月10日

 

产品线索
SpringSource的 Keith Donald
Erwin Vervaet， Ervacon
 
下载 | 实用介绍 | 样品 参考手册 变更日志 | 1.0升级指南
Spring Web Flow是下一代Java Web应用程序控制器框架，允许开发人员将用户操作建模为称为流的高级模块，这些模块可以在任何环境中运行。该框架提供了提高的生产力和可测试性，同时为执行导航规则和管理应用程序状态提供了强大的解决方案。

大图景

Spring Web Flow允许开发人员构建可重用的，自包含的控制器模块，称为流。流程定义了一个用户对话框，该对话框响应用户事件以驱动应用程序代码的执行以完成业务目标。考虑以下流程定义的图形视图，该图形视图与用户一起执行搜索过程：
 
上面的流程是使用丰富的领域特定语言（DSL）以声明方式定义的。这种语言的XML（XSD）形式在用户中最受欢迎。定义后，流就可以独立进行测试，可以作为子流嵌入其他流中，并且可以部署为在包括Java EE Servlet和Portlet（JSR 168）在内的多个环境中执行而无需更改。
实际操作中将看到此简单搜索流程，并查看完整的流程定义 。

产品亮点

Spring Web Flow包含许多受用户欢迎的创新功能。这些功能包括：
  • 在一个地方定义一个应用程序任务（例如搜索过程）的所有控制器逻辑，而不是在多个地方散布该逻辑。
  • 组合流可以从简单的零件创建丰富的控制器模块。
  • 使用自然的，面向对象的线性编程模型来执行严格的用户导航规则，而无需对冗长的if / else块进行编码。
  • 在流程执行期间分配的内存在执行结束或到期时自动清除。
  • 使用您选择的基本Web框架部署流以在Servlet环境中执行。Spring Web Flow提供了与领先的Web框架Struts，Spring MVC和Java Server Faces的便捷集成。
  • 更改Web框架，例如从Struts更改为Spring MVC，而无需更改流定义。
  • 一起更改环境，例如从JUnit测试到Portlet，而无需更改流定义。
  • 在开发时动态地发展应用程序的导航规则，而无需重新启动容器。
  • 接收自动浏览器按钮支持（后退，前进，刷新），而无需自定义编码。
  • 将任务数据存储在四个托管范围中的任何一个中：请求，闪存，流和对话；每个都有自己独特的语义。
  • 测试流无需容器即可隔离。在部署之前，请确保您的应用程序控制逻辑有效。
  • 使用Spring IDE 2.0图形化地可视化和编辑您的流程导航逻辑。

入门

开始使用Spring Web Flow的最佳方法是阅读Erwin的实用介绍并遍历示例应用程序。我们建议您检查所有样品，并从一开始就根据需要补充参考手册材料。该发行版附带了十个示例应用程序，每个示例应用程序都展示了一套独特的功能。样本是：
  1. 电话簿 -演示大多数功能（包括子流）的原始样本
  2. Sellitem-演示具有条件转换，流执行重定向，自定义文本字段格式和继续的向导
  3. Flowlauncher-演示启动和恢复流程的所有可能方法
  4. 项目清单 -演示REST风格的URL和内联流
  5. Shippingrate-与Ajax技术一起展示Spring Web Flow
  6. NumberGuess-演示有状态Bean，评估操作以及“单键”流执行重定向。
  7. 生日 -演示Struts集成
  8. Fileupload-演示分段文件上传，设置操作和Flash作用域
  9. Phonebook-Portlet-Portlet环境中的电话簿示例（请注意流定义如何保持不变）
  10. Sellitem-JSF -JSF环境中的Sellitem示例
  11. Booking-Faces-一个更全面的Spring Web Flow + JSF应用程序。
所有示例项目都是可作为“动态Web项目”直接导入Eclipse的Spring IDE项目，也可以使用Eclipse Web Tools部署在Eclipse IDE中。
 
要深入全面地了解整个Spring Web Flow功能集，最好的选择是阅读Working with Spring Web Flow 。

最新发行说明

  • Spring Web Flow 1.0.x被证明是适合生产使用的软件。
  • Spring Web Flow在Java SE 1.3或更高版本以及Java EE 1.3（Servlet 2.3，Portlet 1.0）或更高版本上运行。Spring Web Flow在所有主要的应用服务器平台上运行。
  • Spring 1.x系列需要Spring 1.2.7或更高版本，Spring 2.x系列需要Spring 2.0或>。
  • Struts Classic（1.2.9），Spring MVC和JSF集成是现成的。
  • 1.0版本表示在整个1.x系列中将保留公共API和XSD的向后兼容性。

即将发行说明

  • Spring Web Flow 2.0是该框架的下一个主要发行版，着重于增强的集成性和易用性。
  • Spring Web Flow 2.0最终版将于2008年4月发布。
