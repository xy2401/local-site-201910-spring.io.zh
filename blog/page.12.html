Spring Tools 4.3.0发布

发布
马丁·利珀特
2019年6月21日

亲爱的春季社区，

我很高兴地宣布适用于Eclipse，Visual Studio Code和Theia的Spring Tools 4的4.3.0版本。

此版本的重点包括：

  • （Eclipse Theia与Atom） Eclipse Theia现在正式受支持，Atom支持已弃用
  • （Spring Boot）改进：现在，项目类路径通知现在在启动时成批出现，以进一步优化Eclipse方面的性能和工作负载
  • （Spring Boot）改进：现在，如果依赖类型发生更改，则可以重新创建符号
  • （Spring Boot）已修复：缓慢的代码完成需要1秒钟以上的时间。（ ＃293 ）
  • （Spring Boot）已修复：Spring XML配置文件的内容辅助现在可以在VS Code和Theia中再次使用
  • （Spring Boot）修复：匿名内部类型Bean没有启动提示
  • （CF清单）已修复：CF清单编辑器支持清单实例设置为零的清单文件
  • （主题）修复：增加了对vars任务步骤中的属性
  • （Eclipse）已修复：多次快速重新启动启动仪表板中的应用程序可能会导致失败

CVE-2019-11272：Spring Security 4.2.13发布了

发布
乔什·卡明斯
六月19，2019

我们已经发布了Spring Security 4.2.13以解决CVE-2019-11272：PlaintextPasswordEncoder对null的编码密码进行身份验证 。

鼓励用户立即更新。

使用Spring Boot，您可以像这样在Maven中覆盖Spring Security版本：

<properties>
    <spring-security.version>4.2.13.RELEASE</spring-security.version>
</properties>

或像这样在Gradle中：

ext['spring-security.version'] = '4.2.13.RELEASE'

请注意，Spring Security 5+的用户不受此漏洞影响。

Java CFEnv 1.1 GA发布

发布
马克·波拉克
六月19，2019

我很高兴代表Java CFEnv社区宣布Java CFEnv 1.1 GA的发布。

此版本增加了对以下服务的支持：

  • 批量服务

  • 关键单点登录服务

  • 重要的Redis服务

它还添加了以下功能：

  • 检查类路径以正确确定MySQL或MariaDB驱动程序类名称的设置。

  • 使用Spring Boot支持时，如果Spring Cloud Connector库位于类路径上，则会引发异常。这仅适用于以下服务：DataSource，RabbitMQ，Cassandra，MongoDB和Redis。异常消息指示设置环境变量JBP_CONFIG_SPRING_AUTO_RECONFIGURATION '{enabled: false}'

  • 通过将日志记录实用程序类复制到项目中来支持Boot1.5.x。
