java-cfenv 1.0 RC1发布

发布
马克·波拉克
2019年2月27日

java-cfenv库是用于访问Cloud Foundry Services的新库。对于Spring Boot用户，它会从VCAP_SERVICES环境变量并设置众所周知的Spring Boot自动配置属性。

在此版本中，支持以下Cloud Foundry服务

  • 数据库-DB2，MySQL，Oracle，Postgresql，SqlServer

  • 兔子MQ

  • 卡桑达拉

  • MongoDB

  • 雷迪斯

在1.0 RC1中，引入了另外两个模块，其中包含对Pivotal的Spring Cloud Service和Single Sign on tile的支持。的java-cfenv-boot-pivotal-scs模块提供对Config Server和java-cfenv-boot-pivotal-sso模块提供对单点登录的支持。

Spring Tools 4.1.2发布

发布
马丁·利珀特
2019年2月22日

亲爱的春季社区，

我很高兴地宣布适用于Eclipse，Visual Studio Code和Atom的Spring Tools 4的4.1.2版本。

此版本的重点包括：

  • （Spring Boot）新增功能：Bean连线的实时悬停信息现在支持在本地服务器安装中运行的，打包了战争的启动应用程序
  • （Spring Boot）新功能：实时悬停信息@Value注释（ ＃177 ）
  • （Spring Boot）新功能：XML配置文件中的Bean符号现在包含确切的位置信息
  • （Spring Boot）已修复：在实时悬停中导航至资源，CF上运行的应用程序将再次运行
  • （Spring Boot）已修复：现在在服务器端进行项目中符号的搜索，以避免在开始查询之前在客户端上没有出现与项目相关的符号
  • （Spring Boot）性能：改进以进一步减少检查实时悬停进程时的CPU负载（ ＃140 ）
  • （Spring Boot）性能：语言服务器不再触发Maven项目的完整源代码和javadoc下载
  • （Concourse）新增功能：添加了对文件中的分层符号的支持，现在可以生成漂亮的轮廓视图信息
  • （主题）新增：支持YAML锚，引用，扩展添加（ ＃58 ）
  • （Eclipse）新增功能：快速文本搜索可以切换到非模式模式（ ＃189 ）
  • （Eclipse）新增功能：快速文本搜索允许针对某些文件类型过滤结果（ ＃185 ）
  • （Eclipse）已修复：在Eclipse 4.11的早期构建中发现了启动性能下降
  • 各种其他错误修复和改进
