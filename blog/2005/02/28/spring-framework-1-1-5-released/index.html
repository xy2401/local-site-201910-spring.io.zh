Spring Framework 1.1.5发布

发布
科林·桑帕林（Colin Sampaleanu）
2005年2月28日

我们很高兴地宣布，Spring Framework 1.1.5刚刚发布。





这是1.1.x系列中的最新错误修复和次要增强版本，
具有许多小的改进，例如：


  • 向错误接口和BindException添加了没有默认消息的重载“ reject”和rejectValue”方法


  • 向JndiTemplate添加了“ lookup（name，requiredType）”便捷方法，将JNDI对象与给定类型匹配


  • 向AbstractRemoteSlsbInvokerInterceptor添加了“ homeInterface”属性，用于指定要缩小到的home接口


  • 引入了MailMessage接口，作为SimpleMailMessage和JavaMail MIME消息的通用接口


  • Log4jConfigurer也接受“类路径：” URL或“文件：” URL作为位置，而不仅仅是纯文件路径


  • Log4jConfigurer接受不存在于文件系统中的配置文件，只要没有刷新间隔



  • 向JdbcTemplate添加了“ int [] batchUpdate（String [] sql）”方法，以作为批处理执行一组SQL语句


  • 为C3P0 0.8.5或更高版本添加了C3P0NativeJdbcExtractor（对于较早的C3P0版本，请使用SimpleNativeJdbcExtractor）


  • 向JdbcTemplate添加了“ maxRows” bean属性，从而允许指定要获取的最大行数


  • 向RdbmsOperation添加了“ fetchSize”和“ maxRows” bean属性，并将值传递给内部JdbcTemplate


  • 为iBATIS SQL Maps 2.0.9添加了ClobStringTypeHandler，BlobByteArrayTypeHandler和BlobSerializableTypeHandler


  • 如果没有剩余生存时间（尝试操作之前），ResourceHolderSupport将引发TransactionTimedOutException


  • TransactionSynchronization对象可以通过实现Ordered接口来影响其执行顺序


  • JtaTransactionManager仅能与JTA TransactionManager一起使用（即没有UserTransaction句柄）



  • 将MockHttpServletRequest升级到Servlet API 2.4（添加了getRemotePort，getLocalName，getLocalAddr，getLocalPort）


  • 将MockPageContext升级到JSP API 2.0（添加了getExpressionEvaluator，getVariableResolver，重载包括）


  • 在ServletContextPropertyPlaceholderConfigurer中添加了“ contextOverride”选项，从而使web.xml覆盖本地设置


  • 添加了“ searchContextAttributes”选项
    ServletContextPropertyPlaceholderConfigurer，解析上下文属性


  • 在ModelAndView中添加了“ clear”和“ isEmpty”方法，从而可以清除给定ModelAndView对象的视图


  • 添加了JasperReportsMultiFormatView，从而允许通过模型中的鉴别器动态指定输出格式


  • Spring的JSP标记中的JSP EL表达式将与JSP 2.0上的JSP 2.0 ExpressionEvaluator一起解析（否则为Jakarta JSTL）


  • 将“ spring：transform”标记的“ value”属性从String更改为Object，以允许使用JSP 2.0解析表达式



请参阅更改日志以获取详细信息。
