Spring Web Flow 1st预览版已发布

发布
科林·桑帕林（Colin Sampaleanu）
2005年3月30日

我们很高兴地
宣布用于Spring的Spring Web Flow预览版
1.2发布路径。在此版本中，我们考虑了网络流量
系统适合开发使用。




此版本中包含我们希望通过网络进行的新功能
对页面流有苛刻要求的应用程序
水平。这些包括：

  • 捕捉逻辑的能力
    的页面流
    您的应用程序作为独立的模块可以在多个环境中重复使用
    情况。
  • 嵌套流量“ n”级的能力
    深（如
    子流），其中每个流都是遵循明确定义的“黑匣子”
    输入/输出合同。
  • 表达页面流的能力
    声明性地
    具有可插入流定义格式（当前基于Java和
    提供了基于XML的构建器格式。）
  • 影响路径的能力
    通过一个流
    通过在任何受支持范围内推理上下文数据来实现动态方式。
  • 跟踪和响应的能力
    生命周期
    页面流。
  • 自动状态管理
    有能力
    在多个范围（事件，请求，流）中存储模型数据
  • 集中相关能力
    动作逻辑
    与一类流相关联。
  • 支持申请交易
    使用基于令牌的机制进行同步。
  • 对HTTP没有硬连接依赖
    小服务程序
    请求/响应或任何请求URL，用于其他环境，例如
    Portlet，JSF和Tapestry。
    -过期的流量检测器，用于自动清除废弃的流量。
  • 开箱即用的辅助动作
    表单数据绑定
    和验证。
  • 开箱即用的Struts 1.x
    集成（样本
    即将发布。）
  • 开箱即用的Spring MVC
    积分。
  • 专为可扩展性而设计
    有据可查
    定制的插件点（定制状态类型，定制流程
    建筑商等）
     



和往常一样，凭借Spring Seal的品质您吗？我期待着，
我们提供：

  • 质量文档，包括javadocs和
    即将发布的参考文档
  • 强大的测试覆盖率
  • 仔细注意命名，包装和易用性
  • 异常错误报告
  • 最终用户驱动的功能集
    需要
     



要下载预览版本：
 

  1. 访问springframework项目文件列表
  2. 向下滚动到“ spring-webflow”发行包
    并选择“ spring-webflow-preview1.zip”进行下载。
     

发布档案包含两个jar：

  1. spring-webflow.jar-核心Web流系统
    （org.springframework.web.flow。*）
  2. spring-webflow-support.jar-不支持软件包
    尚未发布为
    Spring核心框架的一部分。
     

由于发行档案未随Spring Framework一起提供，请
如果尚未下载Spring 1.2 RC1，请先下载。
 



一旦下载，请访问的Webflow
空间来查看模块文档和“快速入门”。
 



为了使Spring Web Flow运作起来更好，请检查样本
在samples / webflow目录内的发行档案中。看到
样本/webflow/readme.txt文件，介绍如何构建和部署
示例应用程序。
