2016年3月4日

Spring Framework 4.3。RC1即将来临，并带来了不错的核心容器改进，我们将在本文中探讨…

单构造器场景的隐式构造器注入

考虑以下服务类别：

@Service
public class FooService {

    private final FooRepository repository;

    @Autowired
    public FooService(FooRepository repository) {
        this.repository = repository
    }
}

相当常见的用例，但是如果您忘记了@Autowired除非在bean定义设置中明确指示自动装配模式“ constructor”（例如，在XML中），否则容器将在构造函数上添加注解，容器将引发异常以寻找默认构造函数 ）。

SpringOne2GX 2015重播：Cloud Native Dataflow业务流程

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2016年3月1日

记录在SpringOne2GX 2015中。
主讲人：马克·费舍尔和帕特里克·佩拉尔塔
大数据轨道
幻灯片： http ： //www.slideshare.net/SpringCentral/cloud-native-data-flow-orchestration

Spring Cloud Stream和Spring Cloud Task项目提供了一个简单而强大的框架，用于创建用于流和批处理的云原生数据微服务。这些分布式系统中的每个微服务都包含一个独立的Spring Boot应用程序。

尽管可以手动定义这些微服务应用程序之间的数据管道，但Spring Cloud Data Flow是一个集成的编排层，可为开发和运营提供高效的部署和管理经验。使用Spring XD引入的相同DSL和Shell / UI定义流和任务。此外，可插入的运行时SPI允许Spring Cloud Data Flow在包括Apache YARN，Lattice和Cloud Foundry在内的各种分布式运行时平台上协调这些应用程序。
