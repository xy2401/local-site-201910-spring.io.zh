Spring Security 4.0.0。RC1发布

发布
罗伯·温奇
2014年12月11日

我们很高兴地宣布Spring Security 4.0.0的发布。RC1。此版本解决了40张票 。您可以在下面找到这些更改的重点。

  • 更新的默认值 -随着安全性的发展，Spring Security也随之发展。我们借此机会确保默认设置更加安全。例如，XML命名空间支持现在默认情况下启用CSRF保护。
  • 波兰语WebSocket安全性 -我们从社区获得了非常宝贵的反馈，这使我们可以完善WebSocket安全性。我们还添加了对WebSocket安全性的XML命名空间配置支持。可以在更新博客上找到详细信息预览Spring Security WebSocket支持
  • 最低依赖性版本 -最低依赖性版本已提高。例如，Spring Security现在需要Spring4.1.x。
  • 已删除的弃用 -虽然这可能不是每个人都喜欢的功能，但有必要清理不必要的代码。这使代码库更易于理解和维护。
  • 文档格式 -当我们更新为使用asciidoctor时，PDF和HTML Multi Page输出已删除。我们再次提供HTML Single ， HTML Multi Page ， EPub和PDF输出。

Spring Boot 1.2.0发布

发布
菲尔·韦伯
2014年12月11日

我很高兴地宣布，Spring Boot 1.2.0已经发布，可以从repo.spring.io和Maven Central获得 。此版本在1.1之上添加了大量新功能和改进，建议所有用户升级。有关升级说明和“新功能和值得注意的功能”，请参阅发行说明 。

以下是此版本的一些重点：

Servlet 3.1

当与嵌入式Servlet容器一起运行时，Spring Boot现在使用Servlet 3.1。Tomcat 8，Jetty 9和Undertow 1.1都是受支持的选项。此外，WebSocket支持已得到改进，现已自动为所有受支持的服务器配置。如果您需要坚持使用Servlet 3.0，则仍支持Tomcat 7和Jetty 8。

网络研讨会重播：将Reactor用于异步/非阻塞微服务

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2014年12月11日

演讲者：斯蒂芬·马尔蒂尼

幻灯片： https ： //speakerdeck.com/smaldini/introduction-to-reactor-and-reactive-streams

异步，非阻塞样式通信在微服务中的作用是什么？与Stephane Maldini一起，深入了解为何在最终一致的方法（如微服务体系结构）中，反应式组件如此重要。当服务拥有其数据并完全独立时，具有（响应）抽象层可以执行各种角色。能够并行化微服务拥有的资源是一项重要技术。另一个可能是事件总线，它从各种微服务中动态提取状态数据，并对照缓存的，相当最新的本地副本进行检查。尽管微服务拥有其他服务“所需”的所有数据来响应请求，但该数据不一定是最新的，这是准备接受的重要约束。Stephane还将讨论微服务之外的其他异步用例，以及www.reactive-streams.org协议支持如何实现跨供应商实现的可移植性级别。
