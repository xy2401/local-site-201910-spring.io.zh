提供了用于Spring Integration 1.2版本的Java DSL

发布
阿尔特姆·比兰（Artem Bilan）
十月14，2016

亲爱的春季社区，

我很高兴宣布Java DSL for Spring Integration 1.2 GA现在可用。

神器org.springframework.integration:spring-integration-java-dsl:1.2.0.RELEASE在Release repo和Maven Central中可用。

自上一版本候选版本1以来，我们已收到一些反馈，并添加了以下附加功能：

线程屏障支持

一个新的.barrier()和它的镜子.trigger() EIP方法已添加到IntegrationFlow定义：

private static final String BARRIER = "barrier";

@Bean
public IntegrationFlow barrierFlow() {
    return f -> f
        .barrier(10000, b -> b
                .correlationStrategy(
                         new HeaderAttributeCorrelationStrategy(BARRIER))
                 .outputProcessor(g ->
                         g.getMessages()
                                 .stream()
                                 .skip(1)
                                 .findFirst()
                                 .get()))
         .channel(c -> c.queue("barrierResults"));
}

@Bean
public IntegrationFlow releaseBarrierFlow(
                          MessageTriggerAction barrierTriggerAction) {
    return IntegrationFlows.from((Channels c) -> c.queue("releaseChannel"))
        .trigger(barrierTriggerAction,
            e -> e.poller(p -> p.fixedDelay(100)))
        .get();
}

Spring Cloud Task 1.1.0。M2现在可用

发布
格伦·伦弗罗（Glenn Renfro）
十月12，2016

我们很高兴地宣布Spring Cloud Task 1.1.0。现在可以通过Github和Pivotal下载存储库使用M2。非常感谢所有对此版本做出贡献的人。

Spring Cloud Task 1.1.0的新功能。M2

该版本继续以1.1.0的主要主题为基础，该主题改进了整体任务功能以及将Spring XD中现有的功能迁移到Spring Cloud Task。此版本的主要更新如下：

  • 对生成的任务ID的支持-Spring Cloud Task应用程序的常见用例是在PaaS中启动它，例如Cloud Foundry或其他云基础架构。因此，启动请求与实际应用程序开始之间可能会有延迟。更糟糕的是，负责启动应用程序的系统可能由于Spring Cloud Task无法控制的原因（资源不足，网络问题等）而实际上未启动任务。此功能允许调用方注册已请求运行任务。这样，在启动应用程序的请求和实际启动之间，不会丢失任何任务请求。
  • 对外部执行ID的支持 -在PaaS或其他云基础架构上执行任务时，通常需要在任务的执行ID与基础系统提供的ID之间进行映射。此功能提供了将两者关联的功能，例如日志管理，审计等。
