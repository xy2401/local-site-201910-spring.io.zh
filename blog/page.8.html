Spring Tools 4.3.2发布

发布
马丁·利珀特
2019年8月14日

亲爱的春季社区，

我很高兴地宣布适用于Eclipse，Visual Studio Code和Theia的Spring Tools 4的4.3.2版本。

此版本的重点包括：

  • （Spring Boot）已修复：CTRL单击导航无法正确处理超类的属性（ ＃326 ）
  • （Spring Boot）已修复：配置属性分析不能正确处理用snake_case编写的属性（ ＃327 ）
  • （Eclipse）已修复：在特定情况下，语言服务器进程未正确终止，导致许多语言服务器进程在Eclipse打开时仍保持运行
  • （Eclipse）已修复：由于违反了使用约束，因此无法解析Eclipse Language Server（ ＃325 ）
  • （Eclipse）已修复：UI冻结是由xmlnamespace支持引起的吗？（ ＃318 ）
  • （Eclipse）修复：缺少javax.inject导入会导致启动仪表板中的CF集成失败
  • 许多其他错误修复…

Spring Statemachine在3.0.0版本中开始响应。M1

发布
珍妮·瓦尔基亚拉赫蒂
2019年8月8日

我很高兴地宣布Spring Statemachine 3.0.0的第一个里程碑版本。M1和这些话，我可以说Statemachine正在变得被动。

Statemachine本身不需要为自己的执行而做出反应，但是一旦机器脱离其受控环境执行诸如Actions和Guards之类的用户定义的逻辑，就无法保证这些功能不会被阻塞。

当世界的另一端变得更加被动时，那些使用数据库的持久性功能的用户将受益于被动更改。基本上，每次我们需要走出机器世界的一步时，我们都可能会遇到阻塞IO操作的情况。

的Corn-M3发布基于：

  • Spring Session核心模块2.2.0.M3

  • Spring Session Data Geode2.2.0.M2

  • Spring Session Data MongoDB2.2.0.RC1

  • 解决会话索引的通用策略
