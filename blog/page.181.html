记录在SpringOne2GX 2014中。

演讲者：史蒂夫·彭伯

G＆G专题

幻灯片： http ： //www.slideshare.net/SpringCentral/reactive-grails

对于应用程序开发人员来说，自然的趋势是以过程性，同步，整体式模式构建其代码。经验丰富的开发人员知道，这会导致容易出错，无法扩展，运行缓慢的软件-但它非常流行。多年来，已经出现了几种架构模式，这些模式试图缓解此问题。最受欢迎的服务之一是面向服务的体系结构，该体系结构涉及将单片应用程序分解为包含多个较小服务的分布式系统。在本次演讲中，我认为SOA是一个不错的开始，但还不够好。我将讨论Reactive Pattern的租户，以及从Monolithic转向Reactive体系结构的重要性。我们将讨论各种Groovy友好技术，这些技术使我们能够构建基于分布式，微服务的应用程序，并涵盖每种服务之间的有效通信策略。我们将看到，对于大型，高效，可扩展的系统，Reactive是唯一可行的架构。如果您对Reactive不熟悉，那么这将是一个很好的介绍。

Spring Cloud 1.0.0。RC3现在可用

发布
戴夫·瑟
2015年2月13日

Spring Cloud 1.0.0。RC3现在可从https://repo.spring.io/libs-milestone-local获得 。 （希望如此）这是1.0.0之前的最后一个里程碑版本。自1.0.0开始，已修复了一些错误。RC2以及一些小的新功能：

  • 重构了Feign支持，使其看起来更像Spring Data（因此@EnableFeignClients而不是@FeignClientScan ）。
  • 在Zuul代理中支持multipart / form-data
  • 支持在Zuul的自动路由注册中包括和排除远程服务
  • 在Zuul中支持声明式功能区重试
  • 清理了很多依赖项。如果您使用spring-cloud-starters，则应该获得一致的添加和减去功能的经验。Gradle用户需要使用依赖项管理插件才能获得相同的体验。
  • 添加了一些小规模的示例项目，以对类路径隔离问题进行集成测试。

Spring XD 1.1 GA和1.0.4发布

发布
马克·波拉克
2015年2月12日

1.0 GA发行六个月后，团队很高兴宣布Spring XD 1.1 GA可用。

下载链接：
* 1.1.0。发布： zip ， 自制软件 ， yum 。
* 1.0.4。发布： zip 。

请阅读Sabby Anandan的博客 ，以获取Spring XD的一般概述。特定于1.1，已添加了大量功能。以下是一些可以最大程度发挥您的大数据优势的方法。

  • 使用Reactor ， RxJava ， Spark流和Python进行流处理
  • 编排Spark和Sqoop批处理作业
  • 基于Kafka的消息总线， 源和接收器以及新的通用高级Kafka API
  • RabbitMQ的批处理和压缩
  • 轻松开发， 测试和打包自定义模块，然后将其上传到管理服务器。
  • 在YARN上运行XD时的容器组管理
  • REST端点的基于角色的授权以及基于LDAP / 文件的身份验证
  • 支持Pivotal，Hortonworks和Cloudera Hadoop发行版。
