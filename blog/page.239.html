引入用于开发Apache Hadoop YARN应用程序的Spring YARN框架

发布
珍妮·瓦尔基亚拉赫蒂
2013年9月10日

我们非常高兴地摆脱困境，并在Apache Hadoop 2.0 M1发行版Spring的一部分中释放支持编写基于YARN的应用程序。在这篇博客文章中，我将向您介绍YARN ，您可以使用它做什么，以及Spring如何简化基于YARN的应用程序的开发。

如果过去一两年来一直在关注Hadoop社区，那么您可能已经看到了很多有关YARN和Hadoop的下一版本MapReduce（称为MapReduce v2）的讨论。YARN（另一个资源协商者）是MapReduce项目的一个组件，旨在克服Hadoop原始设计中的某些性能问题。MapReduce v2的基本思想是将JobTracker，资源管理和Job Scheduling / Monitoring的功能拆分为单独的守护程序。这个想法是拥有一个全局资源管理器（RM）和一个按应用程序的应用程序主机（AM）。可以从YARN体系结构中找到有关YARN组件依赖关系的通用图。

    反应器1.0.0。M2 – JVM上的快速反应数据应用程序的基础

    工程
    乔恩·布里斯宾
    2013年8月27日

    我很高兴宣布Reactor即将迈向1.0的第二个里程碑版本！适用于Reactor 1.0.0的Maven工件。M2在常规的里程碑存储库中可用。

    什么是反应堆？

    Reactor是用于在JVM上构建高吞吐量，异步，反应式应用程序的基础框架。它提供了Selector风格的主题匹配功能，用于事件路由，动态消费者分配，超快速任务处理器以及反应性Stream和Promise API，用于异步处理数据和协调异步任务。它通过提供语言扩展来提供全面的Groovy语言支持，从而使在Groovy中编写Reactor应用程序变得非常精明。它还具有易于使用的Spring支持，可自动将带注释的POJO连接到Reactor。

    Spring Framework 3.2和SpringSource EBR

    新闻与活动
    彼得·汉弗莱（Pieter Humphrey）
    2013年8月23日

    从3.2版开始， Spring Framework JAR文件（例如spring-core，spring-context和spring-webmvc）不再包含带有OSGi元数据的MANIFEST.MF文件。同样，构建不会自动升级到SpringSource EBR 。为了确保OSGi用户能够升级到Spring Framework 3.2，SpringSource将在GA版本发布后的一个单独过程中为Spring Framework 3.2 GA创建捆绑包并将其发布到EBR。还将发布至少一个3.2里程碑或候选发布版本，以便社区可以在进入GA之前验证OSGi元数据。请注意，Spring Framework 3.1.x系列中的所有未来发行版将继续包含OSGi元数据，并将照常立即发布到EBR。感兴趣的用户可能希望将手表放在SPR-8903上 ，以通知进一步的更新，例如，当Spring Framework 3.2捆绑包发布到EBR时。
