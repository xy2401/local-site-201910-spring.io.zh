Spring Framework 3.2 RC1发布

工程
于尔根·霍勒（Juergen Hoeller）
2012年11月5日

尊敬的Spring社区，

我很高兴地宣布，第一个Spring Framework 3.2候选版本现已发布 。

继去年的Spring Framework 3.1之后，这一代核心框架是下一步的直接步骤，延续了几个已确立的主题。Spring Framework 3.2的主要功能包括：

  • 一个新的基于Gradle的框架构建 ，比以往更轻松地为GitHub上的Spring Framework 项目做贡献
  • 内联的CGLIB 3.0和ASM 4.0 ，完全支持Java 7字节代码，并使基于CGLIB的功能可用，而无需明确声明CGLIB依赖项
  • 允许将@Autowired和@Value用作元注释 ，例如，结合特定限定符来构建自定义注入注释
  • 支持@Configuration类中的自定义@Bean定义注释 ，例如，与特定限定符，@ Lazy，@ Primary等结合使用
  • Servlet 3.0上的异步MVC处理 ，其中Spring MVC处理程序方法能够返回Callables和DeferredResults（请参阅Rossen的系列博客文章）
  • 包含以前独立的Spring MVC Test项目，可以对Spring MVC应用程序进行一流的测试（敬请关注Rossen的博客文章）
  • 支持将WebApplicationContexts加载到TestContext框架中 （敬请关注Sam的博客文章）
  • 早期支持JCache 0.5（JSR-107）作为Spring缓存抽象的后端

发布了Spring Data Solr的第一个里程碑

发布
奥利弗·德罗博姆
2012年10月31日

我很高兴地宣布Spring Data Solr项目的第一个里程碑可用。这是Christoph Strobl领导的第一个社区驱动的Spring Data模块。它本质上是用于访问Solr存储的Spring Data Repository API的实现，并提供了与Spring Data JPA，MongoDB，Neo4j和Gemfire相同的编程模型。除此之外，Spring Data Solr还提供了较低的级别SolrTemplate以及用于轻松引导嵌入式Solr服务器的抽象。
