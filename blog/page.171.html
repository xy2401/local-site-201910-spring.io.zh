Spring Security Kerberos 1.0.0发布

发布
珍妮·瓦尔基亚拉赫蒂
2015年4月23日

我们很高兴地宣布Spring Security Kerberos 1.0.0的发布。发布。首先，我要感谢所有通过为新功能创建PR或仅提供反馈帮助我们的社区成员。

这是对两个候选版本和一个GA版本所做的更改的简要总结：

  • 由于从第一个里程碑开始就拖延了很长时间，因此我们做了很多整理工作以使整个项目保持最新状态。
  • Spring团队添加了KerberosRestTemplate，而所有其他新功能都来自社区。
  • 社区贡献是：
    • KerberosLdapContextSource允许使用现有的Kerberos凭据针对Windows AD进行身份验证，以获取有关已登录用户的更多信息。
    • 调整Kerberos协商，以实现与WinRM更好的互操作性。
    • 更改SpnegoEntryPoint的使用方式，如果Kerberos协商失败，该方式允许回退到其他身份验证方法。
  • 样本已从头开始重写，现在已引导以提供更好的用户体验。

SpringOne2GX 2014重放：使用Spring和Apache ZooKeeper进行集中的应用程序配置

工程
彼得·汉弗莱（Pieter Humphrey）
2015年4月21日

记录在SpringOne2GX 2014中。

演讲者：Ryan Gardner，Dealer.com

核心春季赛道

幻灯片： http ： //www.slideshare.net/SpringCentral/zookeeper-config2014-2c-41123708

应用程序配置是一个演进。它以应用程序中的硬编码字符串开始，并有望进行到某些外部操作，例如无需部署即可更改的文件或系统属性。但是，当其他企业关注的问题（例如审核要求或对谁可以进行更改的访问控制）进入组合时，会发生什么？您如何在过多的应用程序服务器之间保持值的一致性，以使其无法从终端窗口一次进行管理？应用程序配置演变的下一步是集中式配置，当您的应用程序在生产环境中穿越各种环境时，您可以访问它们。这样的服务将配置的所有权从最后接触代码的开发人员转移到负责所有环境中应用程序配置的精通应用程序所有者。在Dealer.com，我们创建了一个这样的解决方案，该解决方案依靠Apache ZooKeeper来处理配置数据的存储和协调，并依靠Spring来处理每个应用程序中已配置对象的检索，创建和注册。最终结果是一个透明的框架，该框架提供了可以使用Spring配置，配置文件和属性值连接创建的相同配置对象。本讲座将讨论解决方案的原因和方式，并将重点介绍如何利用Apache ZooKeeper和Spring的强大属性来摆脱本地配置文件的应用程序，并为企业中的应用程序配置提供一致的机制。

尊敬的Spring社区，

首先，特别感谢ErenAvşaroğulları ，他发起了该项目，并做出了积极而积极的贡献。拜托，不要错过他今年在SpringOne上与我们的对话 ！

总览

Spring Integration实现了众所周知的企业集成模式，并在基于Spring的应用程序中提供了轻量级消息传递，并通过声明性适配器支持与外部系统的集成。Spring Integration的主要目标是为构建企业集成解决方案提供一个简单的模型，同时保持关注点的分离，这对于生成可维护，可测试的代码至关重要。

春季社交Facebook 2.0.0。发布发布

发布
克雷格·沃尔斯
2015年4月15日

亲爱的春季社区，

我很高兴地宣布发布Spring Social Facebook 2.0.0。发布。此版本完成了对Spring Social Facebook的全面修订，以针对Facebook Graph API的2.3版本为目标.Facebook将在4月30日关闭其Graph API的1.0版本，因此强烈建议您升级到Spring Social Facebook 2.0.0。尽快发布。

正如上周发布候选版本和之前的里程碑版本中提到的那样，Facebook的Graph API引入了一些重大更改，导致Spring Social Facebook 1.1.x和Spring Social Facebook 2.0.0之间发生了许多重大更改。同样，即使某些仍可以运行的操作也无法像以前那样运行。例如，不再可能获取用户所有 Facebook朋友的列表；您只能获取也已通过Facebook授权您的应用程序的朋友列表。
