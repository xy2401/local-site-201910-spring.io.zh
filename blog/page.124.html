网络研讨会重播：Spring Cloud Task介绍

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2016年7月29日

演讲者：Pivotal的Michael Minella
云的主要承诺之一就是灵活性。如今，大多数部署到云的应用程序都是长期运行的流程，它们利用云扩展的灵活性。但是计算充满了短暂的任务，这些任务可以启动，完成工作然后终止。这些任务是出色的云用例，因为可以快速调配和回收资源。

在此网络研讨会中，我们将探索Spring Cloud产品组合中的一个新项目Spring Cloud Task，这是一个用于开发和编排短期微服务的新框架。我们将探索各种用例，构建您的第一个任务，并讨论如何使用各种技术来编排任务。最后，我们将窥视Spring Cloud Task项目的路线图。

Spring 5.0 M1的反应式编程

工程
罗森·斯托扬切夫（Rossen Stoyanchev）
2016年7月28日

正如Juergen在他的Spring Framework 5 M1 发布公告中提到的那样，我们的Spring Reactive计划已被合并到Spring Framework中，以保留所有的贡献和超过一年的全部历史。

它是什么？

简而言之，反应式编程是关于无阻塞，事件驱动的应用程序，该应用程序以少量线程进行扩展，并带有背压作为关键成分，旨在确保生产者不会压倒消费者。Reactive Streams规范 （也在Java 9中采用）使能够跨不同提供商的层和库传达需求。例如，写给客户端的HTTP连接可以传达其可用性，以一直向上游写入数据存储库，从而从数据库中获取数据，从而在HTTP客户端速度慢的情况下，存储库也可能变慢甚至暂停。有关反应式编程的更广泛介绍，请参阅Dave Syer的多部分系列文章“关于反应式编程的说明” 。
