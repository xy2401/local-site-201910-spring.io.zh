春天的这个星期-2015年3月3日

工程
乔什·朗（Josh Long）
2015年3月3日

欢迎来到春季本周的另一期！本周，我在堪萨斯州堪萨斯城，与我的朋友和Spring Security负责人Rob Winch闲逛。本周，我将在本地用户组和Spring用户组中发言，并且我将与Cerner和其他一些大型企业讨论Spring，Spring Boot和Spring Cloud。

和往常一样，我们有很多很棒的东西要谈论，让我们开始吧！

  1. Spring Boot的共同负责人Phil Webb 刚刚宣布了Spring Boot 1.2.2和Spring Boot 1.1.11 。这两个版本都是推荐的升级，并且Spring Boot 1.2.2甚至包括对Mustache模板库的新支持。真好！
  2. 查看John Hann的SpringOne2GX 2014演讲，介绍Rave.js，它将Spring Boot概念引入JavaScript。
  3. 我汇总了一个博客，内容涉及构建应用程序的过程，这些应用程序可以轻松地从应用程序服务器，Web服务器和云中以可扩展的方式和可移植的方式使用Servlet HTTP会话 。
  4. 克里斯·比姆斯（Chris Beams）在SpringOne2GX 2014演讲中， “革命不会被集中化”现在可以在线获得。
  5. 想更多地了解Spring的缓存？查看Michael Plod在SpringOne2GX 2014演讲中的重播内容，该演讲介绍了Spring框架中的高级缓存改进
  6. Brian Clozel和Rossen Stoyanchev的SpringOne2GX 2014演讲在Spring MVC 4.1中引入了资源处理， 真棒 ！
  7. Spring Security负责人Rob Winch 刚刚宣布了Spring Security 4.0 RC2 ，其中包括对Websocket的支持！检查一下，然后尝试一下！
  8. Rob Winch还刚刚宣布了Spring Security 3.2.6，它改进了Java configuratino和Spring Boot的集成 ，因此请查看！
  9. 这本身并不是特定于Spring或任何技术的，但它启发了人们对机器学习如何用于预测对抗癌药物的临床反应的启发
  10. 谈到Rob Winch和教育界的忍者Ken Kreuger，请查看他们在SpringOne2GX 2014上有关集成测试的演讲，该演讲现已在InfoQ上提供 。
  11. Alex Zhitnitsky将Spring Boot和Dropwizard进行了很好的比较
  12. 查看这些基于Cloud Foundry的IBM Bluemix的Spring Cloud PaaS连接器
  13. Rahman Usta组合了一个基于Web的SSH客户端的惊人酷演示（后端带有Spring Boot）
  14. 该视频演示了如何使用Codeenvy和Spring Boot

SpringOne2GX 2014重放：RaveJS-JavaScript应用程序的Spring Boot概念

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2015年3月3日

记录在SpringOne2GX 2014中。

演讲者：约翰·汉恩

Web / JavaScript跟踪

幻灯片： http ： //www.slideshare.net/SpringCentral/rave-js-springone-2gx-2014-41117770

现代JavaScript框架已经变得非常复杂。不幸的是，它们也变得非常复杂。这些框架的演示和示例项目看起来很简单。但是，要构建和部署实际的应用程序，开发人员必须搭建，配置和维护大量复杂的机器。直到最近，Java世界还是没有很大的不同。最后，Spring Boot使创建独立的生产级Spring应用程序变得容易，您可以“运行”它们。我们可以对JavaScript做同样的事情吗？我们可以！介绍RaveJS。 Rave消除了配置，机械和复杂性。停止配置和调整文件监视程序，缩小程序和编译器之类的机制，仅是为了获得可运行的应用程序。而是在30秒内从零变为“ hello world”。在接下来的30秒内，只需安装npm和领先的JavaScript软件包管理器Bower的* Rave Extensions *和* Rave Starter *软件包，即可轻松地向您的应用程序添加功能和框架。最后，安装其他* Rave Extension *软件包以应用您喜欢的构建，部署和测试模式。

SpringOne2GX 2014重播：革命将不会集中

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2015年3月3日

记录在SpringOne2GX 2014中。

演讲者：克里斯·比姆斯

数据/集成跟踪

幻灯片： http : //www.slideshare.net/SpringCentral/the-revolution-will-not-be-centralizedhow-to-build-a-blockchain-using-spring-to-develop-a-bitcoinlike-virtual-currency

大型政府运营的数据拖网。基于广告的“免费”云服务不利于用户的最大利益。巨大的，不可读的最终用户许可协议。似乎无论我们走到哪里，围墙都在封闭个人的隐私和自主权。情况很复杂。出于困惑和沮丧，许多用户已经放弃了。常见的限制是“隐私已死”和“我没有什么可隐瞒的”。但是革命即将到来。互联网和Web通过设计分散。诸如DNS，SMTP和HTTP之类的协议采用了对等网络，但是在最近15年（我们对这些技术的青春期）中，我们无意间将互联网的许多基础结构集中了起来。这并不是出于邪恶目的。这样做是出于方便。拥有GMail帐户比运行自己的邮件服务器简单得多。将所有内容存储在云中比维护自己的备份要容易得多。出乎意料的结果是，我们使监视变得更加容易，并且使基于侵入式广告的商业模型成为规范。越来越多的技术人员正在努力以令人惊讶且深刻的方式对网络进行去中心化。自由软件和创新的点对点网络在这项工作中发挥了重要作用，但是可能被证明是最重要的工具是一个新工具：加密货币。有了比特币，我们现在有了本机数字货币。网上现金；一种与互联网其余部分一样分散和灵活的货币。乍一看，比特币可能看起来就像是另一种支付方式，一种时尚或投机泡沫。通过仔细检查，人们开始发现它可以通过促进同行之间以前不可能的经济激励来启用新的业务模型。一旦掌握了加密货币的基础知识，就会看到它的长期影响和可能性与互联网本身一样广泛和深入。就像互联网一样，比特币也不是万灵药。它充满了自己的问题，面临着自己的生存威胁。在这次演讲中，克里斯·比姆斯（Chris Beams）将分享他对比特币及相关技术进行两年多研究的发现：承诺与风险；比特币如何能够为自由软件的开发创造第一个可持续的商业模式；隐私如何从死里复活呢？以及为什么革命不会集中化。

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2015年3月3日

记录在SpringOne2GX 2014中。

演讲者：Michael Plod

核心春季赛道

幻灯片： http ： //www.slideshare.net/SpringCentral/spring-one2gx-caching-with-spring

缓存与广泛的业务应用相关，并且市场上有各种各样的产品，从易于采用的基于本地堆的缓存到功能强大的分布式数据网格。本演讲介绍了Spring缓存抽象的高级用法，例如集成了默认Spring软件包未集成的缓存提供程序。除此之外，我还将概述JCache规范及其在Spring生态系统中的采用。最后，演讲还将讨论将各种缓存解决方案集成到企业级应用程序中的各种最佳实践，这些最佳解决方案不具有将“最终一致性”作为非功能性要求的奢侈之处。该演讲包含许多实时演示，其中一些演示在Raspberry Pis和Lego Mindstorms机器人（运行Spring）上的分布式缓存集群上进行。

可移植的，支持云的HTTP会话

工程
乔什·朗（Josh Long）
2015年3月1日

适用于所有季节的框架（和体系结构）

春天走了一条有趣的路线。无论它在哪里运行，它都提供了很多价值；而且-因为它是建立在依赖注入层上的，所以它在基础层和在其之上运行的应用程序之间提供了自然的间接。这种间接关系通过解耦来提高代码的可移植性：您的应用程序代码不知道javax.sql.DataSource （或其他）它使用的句柄来自JNDI查找，环境变量或Spring提供的简单的new-d-up bean。Spring的这种脱钩和丰富的功能工具箱支持各种用例（批处理，集成，流处理，Web服务，微服务，操作，Web应用程序，安全性等），这使Spring成为开发人员的合理选择部署到（有时是嵌入式的）Web容器（如Apache Tomcat或Eclipse Jetty），应用程序服务器（如WebSphere和WildFly）以及云运行时（如Google App Engine，Heroku，OpenShift和（如今我个人最喜欢的）Cloud Foundry）。这种便携性也使叉车最容易（合理编写！）从应用程序服务器到轻量级的Web容器，最终到云中的应用程序。
