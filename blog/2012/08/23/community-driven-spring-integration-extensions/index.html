GitHub-based repository is to promote a vibrant community of adapter and extension developers, and to encourage a broader segment of the community to contribute to the project. The Spring Integration Extensions project is hosted at:

  • https://github.com/SpringSource/spring-integration-extensions

This new repository will give us the flexibility to incorporate new contributions much more rapidly, without having to go through the core Spring Integration framework’s stricter vetting process. Since each community contributed extension is a separate project in the Spring Integration Extensions repository, it will have its own release life-cycle independent from the one that governs the core Spring Integration framework. As a result, this will allow for more frequent releases.

社区驱动的Spring集成扩展

工程
冈纳·希尔特（Gunnar Hillert）
2012年8月23日

Spring Integration团队自豪地宣布Spring Integration Extensions存储库的可用性。这个基于GitHub的新存储库的主要目标是促进一个充满活力的适配器和扩展开发人员社区，并鼓励该社区的广大用户为该项目做出贡献。Spring Integration Extensions项目托管在：

  • https://github.com/SpringSource/spring-integration-extensions

这个新的存储库将使我们能够灵活地更快地合并新的贡献，而不必经历核心Spring Integration框架更严格的审核过程。由于每个社区贡献的扩展都是Spring Integration Extensions存储库中的一个单独项目，因此它将拥有自己的发行生命周期，而不受控制核心Spring Integration框架的生命周期的影响。结果，这将允许更频繁的发布。

随着时间的流逝，我们希望一些使用较少的Spring Integration模块可以过渡到Extensions存储库。另一方面，对于某些扩展模块， Spring Integration Extensions项目也可以充当孵化器，从而最终将它们合并到核心Spring Integration框架中。

由于社区的一些贡献，新的存储库已经提供了几个适配器，例如XQuery Adapter （包括对XPath 2.0的支持）和SMB Adapter 。在未来几周内，将围绕Amazon Web Services （AWS）添加几个适配器（ Amazon S3 ， Amazon SNS ， Amazon SQS ）。此外，我们正在提供打印适配器的早期版本，并且还在开发使用Atmosphere提供WebSocket支持的适配器。

我如何才能成为贡献者？

如果您有关于新扩展和/或适配器的想法，请随时提交JIRA票证。如果您已经创建了扩展模块并且想要贡献代码，请派生Spring Integration Extensions存储库，添加模块并发出拉取请求 。但是，在我们合并您的代码之前，您必须填写《 贡献者许可协议》 。提交后，我们和/或其他社区成员将审核您的拉取请求并将其合并。有关更多详细信息，请仔细查看Spring Integration Extensions项目的贡献者准则 。

创建新的Spring集成扩展模块

Spring Integration Adapters是最受欢迎的扩展类型之一。为了简化编写用于Spring Integration的自定义适配器的过程，我们现在提供用于SpringSource Tool Suite （STS）3.0.0的适配器模板 。这个模板将使贡献者非常容易快速地构建新的适配器，同时仍然确保与核心Spring Integration框架（包结构，命名约定，名称空间处理程序/解析器等）中提供的那些适配器保持一致。

作为Spring Integration Templates项目的一部分的Spring Integration Adapter模板创建一个功能全面的Spring Integration Module ，并且该模板将创建以下常用组件：

  • 入站通道适配器
  • 出站通道适配器
  • 出站网关

该模板不仅将创建核心Java类和一些基本的单元测试，而且还将提供XML命名空间支持，其中包括XML Schema和关联的解析器类。即使您创建其他类型的组件，此模板仍应为您提供有关开发这些组件的宝贵见解。有关使用STS模板进行Spring集成的更多信息，请参见以下博客文章：

  • 使用STS创建Spring集成项目 。

创建的适配器项目将使用Gradle作为其构建工具，并且紧密遵循针对Spring Integration定义的构建任务：

生成适配器项目

    $ ./gradlew build
将Adapter项目安装到本地Maven存储库

    $ ./gradlew install
生成JavaDoc

    $ ./gradlew api
使用Sonar生成指标

    $ ./gradlew sonar
建立完整的发行版

    $ ./gradlew dist

有关更多信息，请参见Spring Integration Extensions Wiki上的How to Create New Components 。
