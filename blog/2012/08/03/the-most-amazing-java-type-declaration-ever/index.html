有史以来最神奇的Java类型声明

工程
大卫·图兰斯基（David Turanski）
2012年8月3日

我想认为我对Java和泛型非常熟悉，但是最近我遇到了这段Java代码，它使我停滞不前：



public abstract class AnnotationBasedPersistentProperty

> extends AbstractPersistentProperty

{..}

此类是Spring Data Framework的存储库支持的内部组件，该类在实现数据访问层时不再需要编写样板代码，并且还提供了用于映射域对象和管理对任何类型的持久性存储的数据访问的通用编程模型。Spring Data当前的存储库实现包括关系数据库（ JPA ）， Gemfire ， MongoDB ， Neo4 。

幸运的是，如果在Java应用程序中使用Spring Data，您将永远不会看到此代码。它被隐藏在Spring Data简单而强大的Repository抽象之后，该抽象提供查询，分页，排序和常见的CRUD操作。例如：


public class SomeClient {
 @Autowired
 private PersonRepository repository;
 public void doSomething() {
 List<Person> persons = repository.findByLastname(“Matthews”);
 }
}


如Spring Data项目任务说明中所述：
Spring Data为NoSQL和关系存储提供了熟悉且一致的基于Spring的编程模型，同时保留了特定于存储的功能。

对于存储库，这种熟悉且一致的编程模型的关键功能是使用批注来配置持久对象映射。这个想法首先在Hibernate中得到证明，随后又适应了JPA。保留特定于商店的功能要求核心框架提供一些通用注释，例如@Id和@Transient ，并且还必须支持特定于商店的注释。

为了支持映射域对象的属性，Spring Data Repository实现应子类AnnotationBasedPersistentProperty 。此类提供跨持久性的通用注释处理逻辑，并提供用于给定商店的自定义注释处理的钩子。此类恰巧扩展了另一个称为
声明如下的AbstractPersistentProperty ：


public abstract class AbstractPersistentProperty

> implements PersistentProperty



AbstractPersistentProperty自然实现PersistentProperty接口。因此，在我们拆解该代码之前，让我们看一下PersistentProperty的接口声明：


public interface PersistentProperty

>


哇-我的头在旋转着鸡和鸡蛋，河内的塔楼以及Godel，Escher和Bach的影像！这个递归的宝石说核心PersistentProperty接口需要一个扩展它的类型。换句话说，形式参数P代表PersistentProperty的子类型，并且由商店特定的实现定义。特定于商店的PersistentProperty类型是对象映射策略的核心，并通过多个Spring Data通用接口（包括PersistentProperty本身）公开。例如，PersistentProperty声明以下方法：

PersistentEntity getOwner();

PersistentEntity是持久化域对象的类的抽象表示。它旨在解决类级别映射问题，并包含对-等待它-持久属性的引用。PersistentEntity被强类型化为扩展的PersistentProperty接口，并且还公开了由通配符？表示的域类类型。形式参数。

因此，让我们再来看一下初始类型声明：

public abstract class AnnotationBasedPersistentProperty

> extends AbstractPersistentProperty

{..}


希望现在的意思很清楚：Spring Data commons库提供了AbstractPersistentProperty ，它实现了PersistentProperty来管理所有底层字段和类型信息。AnnotationBasedPersistentProperty扩展了此类，以添加注释支持。所有这些类型都需要一个扩展PersistentProperty的类型参数。一些例子：


public interface MongoPersistentProperty extends PersistentProperty{…}


public interface GemfirePersistentProperty extends PersistentProperty{…}


最后，所有这些参数化类型实际上是一种优雅的方法，允许那些实现Spring Data Repository的人员使用该框架来公开特定于商店的功能，而不必诉诸类型转换，instanceof等。工作中已经有许多实现的事实证明了该框架的强大功能。在撰写本文时，所有Spring Data项目都可以作为里程碑版本获得。
