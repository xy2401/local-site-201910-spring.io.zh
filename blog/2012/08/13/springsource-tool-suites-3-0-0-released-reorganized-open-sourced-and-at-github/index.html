SpringSource Tool Suites 3.0.0发布-重新组织，开源并在GitHub上

工程
马丁·利珀特
2012年8月13日

介绍

我们自豪地宣布，基于Eclipse的开发人员工具的最新主要版本现已发布。这不仅是新功能的主要发布，而且还因为其他重大变化（例如组件化，开放源代码以及我们首次提供了多个发行版，每个发行版都是针对不同类型的开发人员而定制）而发布的。让我们看一下细节：

Spring工具套件和Groovy / Grails工具套件

过去，SpringSource Tool Suite是完整的发行版下载，大多数Spring开发人员都可以使用。与Groovy / Grails相比，开发人员必须在其开发环境中手动安装多个扩展才能上手。这已经改变了。现在，我们发布了两个完整的发行版：



  • Spring Tool Suite： Spring Tool Suite是我们基于Eclipse的工具的完整发行版，其中预先装有处理Spring项目所需的所有必需部件。它包括对Spring Core框架本身，Spring Integration，Spring Batch，Spring Webflow，Spring Data等的支持。它随附了最新版本的tc Server Developer Edition和Spring Roo，以及用于Maven的最新Eclipse集成，并且以最新的Eclipse Juno 4.2版本为基础。这与以前称为SpringSource Tool Suite的非常相似。


  • Groovy / Grails工具套件： Groovy / Grails工具套件是针对Groovy和Grails开发而定制的基于Eclipse的工具的完整发行版。它预先安装了Groovy-Eclipse以及我们的Grails工具，支持直接部署到tc Server，并且随发行版一起提供了随时可用的Grails安装。它还基于最新的Eclipse Juno 4.2版本构建，并为我们的Groovy-Grails用户提供了即用型体验。

开源和GitHub

我们坚决致力于开源，并积极参与我们工具所包含的许多开源项目，例如AspectJ，AJDT和Groovy-Eclipse。Spring IDE过去是SpringSource Tool Suite的主要部分之一，也一直是开源的。现在，我们在GitHub的SpringSource组织下，在GitHub的Eclipse Public License下开放了工具套件的所有部分。
Spring工具以前的商业附加组件，如Spring Roo的集成，或提供更好的内容辅助，更好的代码完成和高级重构支持的附加组件，以及Spring的项目模板，为Spring IDE项目做出了贡献。其他部分则提取到全新的开源项目中，例如tc Server的Eclipse集成。

组成项目

为了允许在工具套件的不同部分之间进行单独安装和更好的模块化，我们将不同部分组成了自己的项目。它们都生活在GitHub上，提供它们自己的每晚更新站点，并且可以单独安装到简单的Eclipse JEE安装中。


  • Spring IDE：这为您提供了用于Spring框架的所有工具，以及与各种其他与Spring相关的其他技术（例如AJDT，Spring Integration，Spring Webflow，Spring Data，Spring Security和Spring Roo）的集成。对Maven和Spring Roo的支持（以前仅是STS的一部分）已集成到该项目中。 （ https://github.com/SpringSource/spring-ide ）

  • Grails IDE：为您提供完整的Grails开发人员工具，该工具以前可以从仪表板安装到SpringSource Tool Suite实例中。它建立在Groovy-Eclipse项目的顶部。 （ https://github.com/SpringSource/grails-ide ）

  • Eclipse Integration for tc Server：此组件提供了创建tc Server的新实例，使用现有实例，直接从工作区部署和更新应用程序，配置tc Server实例以及激活Spring Insight的功能。 （ https://github.com/SpringSource/eclipse-integration-tcserver ）

  • Eclipse Integration for Gradle：这在Eclipse中提供了Gradle支持。它允许用户直接导入gradle配置的项目，并将根据gradle配置自动管理依赖项。它还允许直接从Eclipse执行gradle任务。（ https://github.com/SpringSource/eclipse-integration-gradle ）

  • Eclipse Integration Commons：该项目包含上述组件之间共有的共享基础结构。此外，它包含UAA和SpringSource仪表板。 （ https://github.com/SpringSource/eclipse-integration-commons ）

由于这种重组和开源的关系，这些项目之间的依赖性降低了。因此，如果需要，您可以从项目更新站点单独使用它们，并且只会引入最少的依赖项集。例如，可以将VMware vFabric tc Server的Eclipse集成安装到纯Eclipse JEE中，而无需同时安装Spring IDE，Grails IDE或其他组件。
您可以始终使用Dashboard（每个项目都随附于Dashboard，例如UAA）来轻松地将其他项目添加到现有安装中，就像以前的SpringSource Tool Suite版本所习惯的那样。

贡献过程

关于GitHub的最好的事情之一是请求请求的概念。如果您还不熟悉，请继续阅读，但可以说拉取请求就像补丁和代码审查一样，都集成到一个紧密的流程和简单的UI中。纵览一下Spring Integration，Spring Framework和许多其他Spring项目的拉取请求历史记录，您将看到很多有趣且有用的示例。与将补丁文件附加到JIRA相比，此过程要平滑得多。当您将其与Git的功能相结合时，这意味着为Spring工具项目做出贡献从未如此简单。
但是，并非每个拉取请求都会进入。结果取决于审核过程，但是，尽管有许多讨论，但经过适当的讨论和完善，它们将会发挥作用。有一个来自Spring核心框架的贡献者指南文档，该文档将使您对流程的工作方式有一个印象。对于工具项目，某些部分会有所不同，但是如果您有想要回馈给框架的内容，请阅读一下。这是将您的问题摆在最前沿的好方法。自然而然地，我们会优先考虑那些花时间整理高质量作品的用户。感谢所有已经这样做的人，并在此先感谢将来的贡献者！
