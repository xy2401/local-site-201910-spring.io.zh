Spring Data Neo4j 2.1.0里程碑1发布

发布
迈克尔·汉格
2012年3月2日

尊敬的Spring-NOSQL社区，

自从上一版Spring Data - Neo4j发布以来，我们致力于解决许多重要改进和扩展问题。

感谢Mark Spitzler，Oliver Gierke，Rajaram Ganeshan，Laurent Pireyn的贡献以及所有其他社区成员的反馈和讨论。

我们希望鼓励您尝试一下，尤其是新事物，然后将您的反馈发送给我们。

我们知道尚有待解决的问题，并希望在计划于3月底发布的2.1版本中解决这些问题-
与Neo4j 1.7一致。

如果您有任何疑问或建议，请随时在Spring论坛 ， Neo4j Google Group上提问，或者在JIRA中提出问题。

另请参阅InfoQ上的Spring Data Neo4j指南 。它以PDF和ePUB的形式提供，很快也将以印刷形式提供。

请享用！

2.1版中的更改。M1（2012-03-02）

  • DATAGRAPH-181通过template.getOrCreateNode和@Indexed（unique = true）添加了对唯一实体的支持
  • DATAGRAPH-198添加了对自定义目标类型的支持，例如，存储转换为Long @GraphProperty（propertyType = Long.class）的日期
  • DATAGRAPH-102图形中的固定类型表示形式，支持@TypeAlias，以允许图形中较短的类型标识符
  • DATAGRAPH-204 pom.xml清理（存储库）以及对SFW的依赖性现在为3.0.7。发布-3.2
  • 单个字段的DATAGRAPH-185密码查询在没有结果的情况下返回null
  • DATAGRAPH-182在单个RelationshipEntity字段上允许@RelatedTo +内部重构
  • DATAGRAPH-202在Neo4jTemplate中提供了getRelationshipsBetween（）方法
  • GH-＃34修复了使用Neo4j高可用性的问题
  • DATAGRAPH-176添加了用于cypher和gremlin查询的调试日志输出以及派生的查询方法
  • DATAGRAPH-186只读关系集合的默认值
  • DATAGRAPH-173固定的接口验证方法，为类型表示策略增加了接口支持
  • DATAGRAPH-169在派生的查找程序查询的所有可变部分中加反引号，以容纳非标识符名称。
  • DATAGRAPH-164增加了确定Neo4j-template和crud-repository的Java类型的方法
  • DATAGRAPH-166修复了多个排序参数
  • 文档更新
