使用STS创建Spring集成项目

工程
冈纳·希尔特（Gunnar Hillert）
2012年4月9日

就在几天前， SpringSource Tool Suite™ （STS）2.9.1已发布。除了许多新功能之外 ，它还提供了一些特别适合Spring Integration用户的功能。首先，最新的STS版本增加了对Spring Integration 2.1的支持，并改进了Spring Integration流的可视化功能。STS现在支持所有最近添加的Spring Integration适配器，例如：


  • 雷迪斯

  • 宝石火

  • AMQP

  • 储存程序


另外，所有现有的Spring Integration适配器都已更新，以支持新的可视化元素。Spring Integration用户的另一个出色补充是，SpringSource Tool Suite 2.9.x现在附带对Spring Integration的模板支持。因此，当使用Spring Template Project Wizard创建一个新项目时，您现在可以在以下3个新的Spring Integration目标模板之间进行选择：

  • Spring集成项目（独立）- 简单

  • Spring集成项目（独立）- 文件

  • 春季整合项目（ 战争 ）


Simple模板创建一个基本的Spring Integration项目，该项目仅使用核心Spring Integration组件作为独立的Java应用程序运行。为了说明文件轮询功能，“ 文件”模板使用其他组件来轮询文件目录以及路由这些文件。最后， War模板允许用户轻松创建基本的Spring Integration项目，这些项目旨在作为WAR部署的一部分在servlet容器（例如Tomcat）中运行。为了说明目的， War模板使用Spring Integration Twitter适配器 。

创建一个新的Spring Integration Project


为了使用STS Spring模板开始一个新项目，请进入主菜单中的File ，然后是NEW ，然后是Spring Template Project 。接下来将显示模板选择屏幕 。

 

[标题id =“ attachment_10681” align =“ aligncenter” width =“ 342” caption =“模板选择屏幕”] [/字幕]

该屏幕提供了所有可用Spring模板的列表，包括前面提到的3个用于Spring Integration的新模板。如果您在模板名称前面看到小箭头-则意味着尚未下载实际的模板。一旦下载，模板将被缓存在您的计算机上，并且您无需再次下载模板文件，除非您按下“ 刷新”按钮并且可以远程获得该模板的新版本。

让我们选择模板Spring Integration Project（Standalone）-Simple ，然后按下一步。如果尚未下载模板，您将看到以下弹出窗口：

继续并下载了模板后，将显示“ 项目设置”屏幕 。在这里，您必须提供项目名称和某些属性，以使将新创建的项目轻松部署到Maven存储库。

实际上，除了创建特定于Eclipse的项目文件之外，此Template向导还将生成一个项目pom.xml文件，这也将使该项目在STS之外也易于使用（例如，在持续集成过程中构建该项目以及Jenkins之类的工具）。因此，除了项目名称之外，还必须提供一个groupId ， artifactId和一个version属性。最后，您还需要定义一个顶层（基本）程序包 ，相关的源文件将位于该程序包下。

一旦按下“ 完成 ”，将创建该项目，并且该项目应在“ STS 包资源管理器”窗口的左侧列出。至此，您已经有了一个功能齐全的Spring Integration项目。

运行项目


在声明的基本包下，您将找到一个Main类，该类允许您执行Spring Integration应用程序。在STS中，可以通过右键单击Main类，然后选择“ 运行方式 ”-> Java应用程序来执行它。该应用程序启动，您将看到以下控制台输出：


 


除了使用STS，您还可以使用Maven从命令行非常简单地启动项目。只需转到项目目录并执行：

$ mvn clean package exec:java

该命令将清除目标构建目录，编译和打包应用程序，并使用Exec Maven插件执行应用程序。该应用程序将像在STS中一样启动。按“ q”将退出应用程序。

Spring集成配置


该应用程序的作用是允许您通过命令行输入一些自由格式的文本，然后将其转换为大写形式并打印到控制台。

为了实现这一点，正在使用Spring Integration。但是，如果您查看引导Spring应用程序上下文的Main类的详细信息，您会发现它没有对Spring Integration本身的引用。主要类的主要作用是从Spring Application Context中检索“ StringConversionService ”接口的实例。但是，Spring Integration Gateway使用此接口来捕获输入的String并将其转发到请求通道。

项目的应用程序上下文XML文件位于META-INF / spring / integration下 。如您所见，该应用程序包含一个非常基本的Spring Integration流。我们从使用前面提到的StringConversionService接口的简单网关开始。

有2个已定义的消息通道。第一个通道会将包含输入字符串的消息路由到服务激活器。服务激活器本身将使用Spring Expression Language（SpEL）将输入消息的有效负载转换为大写。并通过回复通道将大写的String返回给调用方并打印到命令行。

此外，请注意，我们在此模板中使用了Service Activator，以说明其他基于Java的业务服务的执行。如果您的目标是纯粹地转换Spring Integration Message Payloads，请也考虑使用Transformer ，这在语义上可能更合适。

这是一个非常简单的用例，但是它为您提供了一个不错的起点来实现自己的Spring Integration组件。我们希望这些模板将大大降低在您自己的项目中学习和体验使用Spring Integration的好处的入门门槛。如果您有任何疑问，我们希望收到您的来信。如有任何疑问，请访问Spring Integration论坛，位于forum.springsource.org 。有关总体上有关Spring Integration的更多信息，请访问我们的项目网站，网址为： http://www.springintegration.org/ 。
