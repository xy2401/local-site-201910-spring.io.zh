Spring MVC 3.2预览版：聊天示例

工程
罗森·斯托扬切夫（Rossen Stoyanchev）
2012年5月16日

最后更新时间为2012年11月5日（Spring MVC 3.2 RC1）

在 以前的 博客 文章中，我介绍了Spring MVC 3.2中基于Servlet 3的异步功能，并使用spring-mvc-showcase和Spring AMQP 库存示例进行了演示。这篇文章介绍了一个聊天示例，其中外部事件不是AMQP消息，而是带有聊天消息的HTTP POST请求。在文章的第二部分，我将切换到分布式聊天，其中事件是Redis通知。

聊天不是Web应用程序的常见要求。但是，这是仅通过实时通知才能满足的要求的一个很好的例子。与电子邮件或状态警报相比，它对时间延迟更敏感，并且在浏览器中与朋友，同事在网络研讨会期间或与购物网站上的现场人员聊天的情况并不少见。您可以想象其他类型的在线协作。

这个样本

spring-mvc-chat示例可在Github上获得。尽管不是本博客的重点，但客户端使用Thymeleaf ， 基因敲除 .js和jQuery。Thymeleaf是JSP的绝佳替代品，它支持预览功能，从而使HTML模板更干净，并允许预览者双击HTML模板并进行查看，这与需要Servlet容器的JSP不同。 knockout.js是一个客户端MVC框架，对于将行为附加到HTML元素非常方便。要快速了解它，请遵循其出色的教程之一 。 jQuery用于DOM脚本和Ajax请求。

聊天控制器

ChatController公开了获取和发布聊天消息的操作。这是获取消息的方法：


@RequestMapping(method=RequestMethod.GET)
@ResponseBody
public DeferredResult<List<String>> getMessages(@RequestParam int messageIndex) {

  final DeferredResult<List<String>> deferredResult = new DeferredResult<List<String>>(null, Collections.emptyList());
  this.chatRequests.put(deferredResult, messageIndex);

  deferredResult.onCompletion(new Runnable() {
    @Override
    public void run() {
      chatRequests.remove(deferredResult);
    }
  });

  List<String> messages = this.chatRepository.getMessages(messageIndex);
  if (!messages.isEmpty()) {
    deferredResult.setResult(messages);
  }

  return deferredResult;
}

一个新的DeferredResult被创建并保存在地图中，注册者将从中删除该地图onCompletion异步请求完成时回调。然后该方法使用内存检查新消息ChatRepository 。如果发现新消息，则DeferredResult立即设置。否则，它将在以后收到新消息时进行设置。

以下是保存聊天消息并更新所有已保存内容的方法DeferredResult实例：


@RequestMapping(method=RequestMethod.POST)
@ResponseBody
public void postMessage(@RequestParam String message) {

  this.chatRepository.addMessage(message);

  // Update all chat requests as part of the POST request
  // See Redis branch for a more sophisticated, non-blocking approach

  for (Entry<DeferredResult<List<String>>, Integer> entry : this.chatRequests.entrySet()) {
    List<String> messages = this.chatRepository.getMessages(entry.getValue());
    entry.getKey().setResult(messages);
  }
}

分布式聊天

上面的聊天示例使用简单的内存持久性，并且仅在部署到单个服务器时才有效。Redis 分支使用Redis支持的ChatRepository 。Redis是一个简单的键值存储，借助于Spring Redis项目，它易于在Java中使用。

的RedisChatRepository使用Spring Redis RedisTemplate查找和保存聊天消息。随意看一下代码。

现在，保存新聊天消息的controller方法为一行：


@RequestMapping(method=RequestMethod.POST)
@ResponseBody
public void postMessage(@RequestParam String message) {
  this.chatRepository.addMessage(message);
}

接收新消息也非常简单。它涉及实施Spring Redis MessageListener界面，可以直接在控制器中完成：


@Controller
@RequestMapping("/mvc/chat")
public class ChatController implements MessageListener {

  // ...

  public void onMessage(Message message, byte[] pattern) {
    for (Entry<DeferredResult<List<String>>, Integer> entry : this.chatRequests.entrySet()) {
      List<String> messages = this.chatRepository.getMessages(entry.getValue());
      entry.getKey().setResult(messages);
    }
  }

}

聊天的Redist版本将在群集中工作。消息可以发布在任何服务器上，所有其他服务器都将收到Redis通知。Spring Redis项目使以消息驱动的POJO样式接收那些通知变得非常简单。

聊天消息也可以从非Java Redis客户端发布。例如，使用Redis命令行外壳连接，键入以下命令，聊天消息将传递到所有订阅的服务器和连接的浏览器：

redis 127.0.0.1:6379> RPUSH chat:archive "hello from the redis cli"
redis 127.0.0.1:6379> PUBLISH chat "a new chat message is available"
