Spring Integration 2.1现在已发布

工程
马克·费舍尔
2012年1月9日

Spring Integration 2.1已发布！您可以在此处下载它或配置Maven POM 。这个版本是一年多的工作成果，自从Spring Integration 2.0发布以来，该团队已经解决了数百个问题。有趣的是，距Spring Integration首次公开发布（2008年1月3日发布0.5版）已有4年零几天了。在本文中，我只是想快速概述2.1的主要新功能。

RabbitMQ / AMQP支持

在“ spring-integration-amqp”模块中，我们具有用于单向消息传递的通道适配器以及用于请求/回复的网关。适配器和网关可用于与RabbitMQ代理进行入站和/或出站集成。我们还有AMQP支持的消息通道实现-点对点和发布/订阅-都可以在流程中间的任何地方使用。有关更多信息，请参见参考手册的这一部分 。

GemFire支持

在“ spring-integration-gemfire”模块中，我们具有入站通道适配器，用于连续查询以及简单的缓存侦听。我们还提供了出站通道适配器，用于写入缓存区域。如果要在集合器，声明检查或缓冲消息通道之后使用GemFire进行持久化，则可以配置GemFire消息存储。有关更多信息，请参见参考手册的这一部分 。

Redis支持

在“ spring-integration-redis”模块中，我们有依赖Redis的发布/订阅功能的入站和出站通道适配器。另外，您可以配置Redis支持的消息存储以在聚合器，声明检查或缓冲消息通道之后保持持久性。有关更多信息，请参见参考手册的这一部分 。

MongoDB支持

在“ spring-integration-mongodb”模块中，我们还有另一个消息存储实现，可以在聚合器，声明检查或缓冲消息通道之后使用。有关更多信息，请参见参考手册的这一部分 。

JSR-223脚本支持

在“ spring-integration-scripting”模块中，我们提供了必要的功能，以支持对任何核心EIP组件（过滤器，分离器，路由器，转换器等）使用多种脚本语言。支持的语言包括：Groovy，Ruby / JRuby，Python / Jython和JavaScript。像2.0版以来存在的Groovy支持一样，这些脚本可以在运行时动态刷新。有关更多信息，请参见参考手册的这一部分 。

存储过程支持

在“ spring-integration-jdbc”模块中，我们在支持简单选择/插入/更新操作的现有适配器旁边添加了存储过程适配器。有关更多信息，请参见参考手册的这一部分 。

Spring资源抽象支持

在“ spring-integration-core”模块中，我们添加了一个新的入站通道适配器，该适配器针对Spring的Resource抽象实例运行。这样可以为许多基础资源（例如文件，类路径资源或可以通过URL定位的任何资源）提供灵活性。有关更多信息，请参见参考手册的这一部分 。

有效负载丰富器

在“ spring-integration-core”模块中，我们添加了一个“ enricher”组件，该组件允许从另一个流中“调用”消息流，以便可以使用该流的结果来丰富消息流的有效负载。信息。它补充了现有的头文件增强器组件。有关更多信息，请参见参考手册的这一部分 。

FTP和SFTP出站网关

在“ spring-integration-ftp”和“ spring-integration-sftp”模块中，除了现有的单向通道适配器之外，我们还添加了出站网关适配器以支持请求/答复操作。有关更多信息，请参阅本节的FTP和本节的SFTP 。

从Spring Integration 2.0迁移

如果您打算从Spring Integration版本2.0升级应用程序，请参阅我们的迁移指南 。我们将根据需要更新该文档，因此，如果您遇到该指南中未提及的任何问题，请向我们发送反馈。

摘要

如您所见，我们一直很忙。除了上述新功能之外，还有许多改进和重构。您可能希望从参考手册的这一部分开始，除了本文中提到的功能之外，您还将找到有关这些其他改进的更多信息。另外，请确保在github上签出Spring Integration Samples 。

一如既往，我们欢迎您的反馈。请加入Spring Integration论坛中的讨论 ，并访问Spring Integration主页以获取其他参考以及指向源代码，问题跟踪器，博客等的链接。
