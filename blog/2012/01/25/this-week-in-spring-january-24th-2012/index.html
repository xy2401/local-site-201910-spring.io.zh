        Welcome to another installment of This Week in Spring .

We are already almost done with January (which, honestly, shocks me. Where does the time go?)

       

         
    
      1.  Did you guys see last week's webinar introducing Spring 3.1 with Spring project lead, and VMWare/SpringSource Principal Engineer, Juergen Hoeller?  
       Last week's webinar was very well-attended, and represents the best turnout yet, by far, for all of the SpringSource webinars. So, thanks to all those who came, and, to the handful of people on earth that couldn't make it, don't worry! You can watch

      Spring Source YouTube页面上的Spring 3.1网络研讨会录像 。

    2. 下一个视频是我的最爱之一。这是两部分演讲的第二部分，Ramnivas Laddad，Scott Andrews和Jennifer Hickey –现在所有也在Spring Cloud上工作的SpringSource重量级人物–在演讲中
      SpringOne 2GX 2011关于将Cloud Foundry与Spring结合使用 。
      那么，第一部分在哪里？我不确定。我希望InfoQ也能收到该视频，但是无论哪种方式，这都非常有用
      视频。并且，它们很好地证明了为SpringOne 2GX创建的内容的价值。
    3. The SpringSource Tool Suite, 2.9.0.M2, has been released. This release of the SpringSource Tool Suite has many new features, including an update to TcServer 2.6.3, an update to Spring Roo 1.2.0 (and updated support for the new Spring Roo service arrangements - repositories, services, etc.), an update to Grails 2.0.0 and Groovy 1.8, and initial support for editing Gradle files.
    4. Google App Engine Product Manager Chris Ramsdale gave a great talk about using Spring on Google App Engine for SpringOne2GX 2011. This a good talk, and also a strong testament to Spring's unique portability story.
    5. A blogger, whose name I was not able to uncover, wrote up a fantastic post on his first steps with Spring Integration 2.1. In the blog's example, he talks a little about channels, the Spring Integration FTP and XML support, and then.. the blog's done! Because, Spring Integration's just that simple. This is definitely worth a read. I love that he came to Spring Integration in basically the same fashion as I - and numerous others - have: we'd used big, fat, horrid ESBs and wanted a leaner, cleaner, open-source (and, tangentially, Spring-friendly) alternative. Nice job, and welcome to the community.


    6. Spring Social和Spring Social Twitter 1.0.2版本主要修复了自1.0.1版本以来发现的错误，并进行了一些小的增强 。

               The  Spring Social LinkedIn release includes API binding for LinkedIn's Group API, which completes the LinkedIn API binding. 
         The Spring Social LinkedIn binding's very mature, at this point, and the steady march towards 1.0 will start with a first RC, soon.
    7. Last April, Spring project lead and SpringSource Principal Engineer Juergen Hoeller gave an interview to JSF Central's Kito D. Mann on Spring 3.1, JSF, and more. I'm not too sure why this podcast took so long to materialize, but it's here, and still fairly timely, almost a year later. Enjoy.
    8. Our pals over at Chariot Solutions have posted a very comprehensive roundup of the new features in Spring 3.1. As usual, they knock it out of the park. There's a reason these guys are great trainers...
    9. Willie Wheeler followed up on the nature of the contributions he'd made to Spring Social GitHub, which we talked about here in this very column, last week.
    10. Tobias Trelle is back with the second installment of his exploration of Spring Data. This week, he's introducing Spring Data JPA. Definitely worth a read (and bookmark - he promises even more great stuff to come). Check it out!
    11. Lots of people are taking the first, intrepid steps into the brave new world of Scala development and want to establish good, reusable practices for working with Spring and Spring MVC with Scala. The Orange Apple blog offers a Scala-based Spring and Hibernate wireframe application to get you started. This, the blogger (ykameshrao, on GitHub) volunteers, is his first open source project, and I think it's a wonderful one. Really nice work! Keep it up.
    12. The Monitis blog has some good tips on optimizing database performance with Tomcat.
    13. 去年4月，Spring项目负责人兼SpringSource首席工程师Juergen Hoeller在Spring 3.1，JSF等上接受了JSF Central的Kito D. Mann的采访。我不太确定为什么播客花了这么长时间才能实现，但是差不多一年后，它就在这里，而且还很及时。请享用。
    14. 我们在Chariot Solutions的朋友发布了Spring 3.1中新功能的非常全面的摘要 。和往常一样，他们把它赶出了公园。这些人是很棒的教练是有原因的...
    15. 威利·惠勒（Willie Wheeler）跟进了他对Spring Social GitHub所做的贡献的性质 ，上周我们在本专栏文章中谈到了这一点。
    16. Tobias Trelle返回了他的Spring Data探索的第二部分。本周， 他将介绍Spring Data JPA 。绝对值得一读（和书签-他承诺还会有更多很棒的东西）。看看这个！
    17. 许多人正在迈入勇敢的Scala开发新世界的第一步而又勇敢无畏的步骤，并希望建立与Spring和Spring MVC与Scala一起使用的良好，可重用的实践。Orange Apple博客提供了一个基于Scala的Spring和Hibernate 线框应用程序，以帮助您入门 。这是博主（在ykameshrao ，在GitHub上）的志愿者，这是他的第一个开源项目，我认为这是一个很棒的项目。真的很好！保持。
    18. Monitis博客提供了一些有关使用Tomcat优化数据库性能的好技巧。


    19. 在TomcatExpert.com上 ，Stacey Schneider宣布了Apache Tomcat 7.0.25发行版 ，其中包括一些增强功能，包括

                an alignment of the  Servlet 3.0 implementation with the changes defined in the first maintenance release (also known as Rev. A.), 
         added support for connectors to automatically select a free port to bind to, and an update  to Commons Pool 1.5.7, Commons Daemon 1.0.8 and Eclipse JDT compiler 3.7.1.
    20. TomcatExpert.com also has a great post that explains how to use the new, highly concurrent connection pool that ships with Tomcat 7 (org.apache.tomcat.jdbc.pool.DataSourceFactory) as a replacement for the Commons DBCP pool. This is one handy, dandy pool! You only need to scroll down the page to see all the new, enterprise grade features supported. I'm personally excited to try it!
    21. TomcatExpert.com上也有一篇很棒的文章，解释了如何使用Tomcat 7（ org.apache.tomcat.jdbc.pool.附带的新的高度并发连接池org.apache.tomcat.jdbc.pool.DataSourceFactory ），以替代Commons DBCP池。这是一个方便的花花公子池！您只需要向下滚动页面即可查看所有受支持的企业级新功能。我个人很高兴尝试一下！
