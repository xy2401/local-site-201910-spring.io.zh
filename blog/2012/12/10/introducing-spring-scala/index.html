介绍Spring Scala

工程
Arjen Poutsma
2012年12月10日

去年10月，在SpringOne2GX上 ，我向世界介绍了Spring Scala项目。从那时起，我还在Devoxx上介绍了这个项目。在此博客文章中，我想提供有关此项目以及如何在Scala项目中使用它的更多详细信息。

为什么选择Spring Scala？

Spring Scala项目的目的只是为了使在Scala中使用Spring Framework 更加容易 。我们相信，有很多Spring用户想要试用Scala，但又不想将他们对Spring的体验抛在后面。这个项目是为那些人准备的。

显然，您可以在今天的Scala中使用（Java）Spring Framework，而无需使用Spring Scala。但是这样做在某些地方会很尴尬。就像任何编程语言一样，Scala具有自己的不同处理方式，并且在Scala中使用像Spring这样的纯Java框架只会感觉“像Java一样”。Spring Scala试图通过使Spring成为Scala语言的一等公民来解决此问题。

Spring Scala概述

Spring Scala正在进行中。在本文的其余部分，我将重点介绍当前实现的功能。但是，我们希望在接下来的几个月中可以添加许多其他功能，希望能通过您提供给我们的反馈来实现。因此，如果您有一个使Spring在Scala中更令人愉快使用的功能的想法，请通过提交JIRA问题或发表评论让我们知道。

用XML连接bean

在Spring XML应用程序上下文中连接Scala Bean的最简单且首选的方法是使用构造函数注入。例如，假设我们有以下Scala类：


class Person(val firstName: String, val lastName: String)

您可以使用c名称空间像这样连接此类：


<bean id="person" class="Person" c:firstName="John" c:lastName="Doe"/>

请注意，通过结合使用构造函数注入和val ，我们还制作了Person类不可变。功能语言（例如Scala）鼓励不可变的数据结构。因此， 在Scala中使用Spring时，首选构造方法注入 。还要注意，构造函数注入是开箱即用的。您不需要在类路径上使用Spring Scala jar即可工作。

当涉及到二传手注入时，事情变得更加复杂。默认情况下，Scala类不遵循JavaBeans属性协定（例如String getFoo()和void setFoo(String) ）。相反，Scala拥有自己的财产合同：“ getters”没有前缀get ，而是使用字段名称作为方法名称（例如foo: String ）。Scala设置器在字段名后缀_= （例如。 foo_=(String): Unit ）。

要变通解决此问题，您可以指示Scala编译器使用以下命令来生成JavaBeans getter和setters： @scala.reflect.BeanProperty注解;或者您可以将Spring Scala添加到您的类路径中以启用对Scala setter的支持。

使用@BeanProperty就像注释一个var用它：


class Person(@BeanProperty var firstName: String, @BeanProperty var lastName: String)

这将导致为该类创建JavaBean样式的setter，因此您可以非常简单地将其连接到XML：


<bean id="constructor" class="Person" p:firstName="John" p:lastName="Doe"/>

作为使用此注释的替代方法，从3.2版开始，Spring使用以下命令支持任意的getter和setter方法： BeanInfoFactory策略界面。Spring Scala项目包含此接口的实现，该接口支持Scala getter和setter。Spring会自动检测到此实现，无需其他配置。

实际上，这意味着将Spring Scala jar放在类路径上将启用对Scala属性的支持，并且您可以使用上述XML配置来连接Person没有添加的类@BeanProperty注释。

有关在Spring XML中连接Scala bean的更多信息，请参阅Spring Scala文档Wiki的相关部分 。还可以查看有关在Spring XML中连接Scala集合的部分。

在Scala中接线 bean

除了用XML定义bean之外，Spring Scala还提供了一种替代方法，它使用Scala类而不是XML文件来配置Spring bean。这种方法类似于Spring Java中的@Configuration ，不同之处在于它基于函数而不是注释。

要创建功能性的Spring配置，您只需混合使用FunctionalConfiguration特质进入您的配置类。Bean是通过调用bean特征的方法，并传递创建Bean的函数。

鉴于Person上面显示的类，我们可以按以下功能配置将其连接起来：


class PersonConfiguration extends FunctionalConfiguration {
    bean() {
        new Person("John", "Doe")
    }
}

当然，您也可以使用特定名称注册Bean，提供别名或设置范围：


class PersonConfiguration extends FunctionalConfiguration {
    bean("john", aliases = Seq("doe"), scope = BeanDefinition.SCOPE_PROTOTYPE) {
        new Person("John", "Doe")
    }
}

然后，您可以使用此配置类创建Spring应用程序上下文：


object PersonConfigurationDriver extends App {
    val applicationContext = new FunctionalConfigApplicationContext(classOf[PersonConfiguration])
    val john = applicationContext.getBean(classOf[Person])
    println(john.firstName)
}

有关功能Bean配置的更多信息，包括有关强类型Bean引用，如何导入XML文件和@Configuration类以及如何在配置文件中包装Bean定义的部分 ，请参阅Spring Scala文档Wiki的相关部分 。

在Scala中使用Spring模板

Spring的模板是有用的实用程序类，可促进各种数据访问或一般的资源处理。Spring Scala包含一些包装器，这些包装器使（Java）模板适应于Scala更加友好。通常，在使用Scala语言时，这些Scala模板包装器对Java版本进行了三处改进：

  • 使用函数代替回调接口
  • 用于Option Java版本可以返回的位置null
  • 使用类清单而不是类参数

通过这三个改进，您可以使用JmsTemplate ，例如，如下所示：


val connectionFactory : ConnectionFactory = ...
val template = new JmsTemplate(connectionFactory)

template.send("queue") {
    session: Session => session.createTextMessage("Hello World")
}

template.receive("queue") match {
    case Some(textMessage: TextMessage) => println(textMessage.getText)
    case _ => println("No text message received")
}

通常，Scala版本的模板包装器与Java打包器位于相同的程序包中，只是存在一个scala包之间。因此，Scala版本JmsTemplate住在org.springframework.scala.jms.core ， 例如。在撰写本文时，存在以下Scala友好模板：

  • SimpleJdbcTemplate
  • JmsTemplate
  • RestTemplate
  • 交易模板

有关在Scala中使用Spring模板的更多信息，请参阅Spring Scala文档Wiki的相关部分 。

可用性

Spring Scala的第一个里程碑可在我们的里程碑资料库中下载， http://repo.springsource.org/libs-milestone 。对于Maven用户：


<repositories>
    <repository>
        <id>milestone.repo.springsource.org</id>
        <name>repo.springsource.org-milestone</name>
        <url>https://repo.springsource.org/libs-milestone</url>
    </repository>
</repositories>
<dependency>
    <groupId>org.springframework.scala</groupId>
    <artifactId>spring-scala</artifactId>
    <version>1.0.0.M2</version>
</dependency>

该项目本身可从GitHub获得 。如果您想贡献，可以通过为功能请求提交JIRA或在GitHub上留下pul请求来进行。

请注意 ，自2013年4月2日起，新的里程碑已发布。阅读论坛公告 。
