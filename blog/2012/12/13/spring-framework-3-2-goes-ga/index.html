Spring Framework 3.2正式发布

工程
于尔根·霍勒（Juergen Hoeller）
2012年12月13日

尊敬的Spring社区，

在Spring Framework 3.1发布大约一年之后，我很高兴地宣布Spring Framework 3.2现在已经可以普遍使用了！

我们建议对所有以前的Spring版本进行升级，特别是从Spring Framework 3.1.x升级，这是其直接后继。

如前所述，Spring Framework 3.2的主要功能包括：

  • 框架内完善的Java SE 7支持 ，以及通过升级到CGLIB 3.0，ASM 4.0（我们现在都在其中内嵌）和AspectJ 1.7
  • 跨框架的并发优化 ，尽可能避免使用同步-尤其是对于作用域/原型Bean
  • 允许将@Autowired和@Value用作元注释 ，例如，结合特定限定符来构建自定义注入注释
  • 支持@Configuration类中的自定义@Bean定义注释 ，例如，与特定限定符，@ Lazy，@ Primary等结合使用
  • Servlet 3.0上的异步MVC处理 ，并且Spring MVC处理程序方法能够返回Callables和DeferredResults
  • 包括以前独立的Spring MVC Test项目，可以对Spring MVC应用程序进行一流的测试
  • 支持在TestContext框架中加载WebApplicationContext ，以及对Web应用程序测试的进一步改进
  • 为0.5的JCache（JSR-107）作为后端早期支持Spring的高速缓存抽象

有关更多详细信息，尤其是Spring MVC中的许多增强功能，请查看我们文档中的Spring Framework 3.2中的新功能和增强功能部分。

此版本包含了许多社区反馈，因此，让我借此机会对所有记者和撰稿人表示感谢！在我们迈入2013年之际，继续努力，致力于下一代框架的开发……

干杯，

于尔根
