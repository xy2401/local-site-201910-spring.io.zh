今年春季-2012年12月25日

新闻与活动
乔什·朗（Josh Long）
2012年12月25日

欢迎回到春季的本周另一个非常特殊的假期和年终分期付款！
如果您是这次综述的追随者，那么您会知道2012年对于Spring来说是非常激动人心的一年！
首先，让我们看一些要点，然后再进行每周综述：


  1. 向前迈进当然，今年是Spring 3.2的发布版本，恰好是从Spring 3.1发行到发布的一年，它具有许多新功能，并有助于Spring保持其作为构建Web应用程序的首要平台的地位。今年也取得了许多重大进步
    和其他Spring项目中的迭代，例如Spring Integration 2.2.0 GA，Spring Data

  2. 云端
    由于与基础平台的自然分离，Spring在所有云平台上都能很好地工作
    依赖关系注入提供了，但是它一直（并且将继续） 在开放源代码的PaaS Cloud Foundry上享有特殊的地位。而且，对于Cloud Foundry来说已经过去了一年！我们已经看到像App Fog这样的生态系统合作伙伴采用Cloud Foundry位并与之一起运行。我们已经看到，Cloud Foundry对Spring应用程序的支持通过独立流程等新功能大大提高了。

  3. RESTful Web如果您问我，今年最激动人心的部分是观看Spring的Web支持的改进。如果您要构建一个Web应用程序（包括在Servlet 3环境中）或公开RESTful API端点，那么Spring MVC是自然的选择。如果您想保护那些RESTful端点，
    Spring Security OAuth是一种易于使用的绑定，它在REST之上支持OAuth。是否需要通过OAuth连接到Twitter，Facebook，LinkedIn和GitHub等社交服务提供商？使用Spring Social。是否想在RESTful端点中支持HATEOAS的原理？查看Spring HATEOAS。您是否想透明，轻松地公开Spring Data存储库以用作RESTful端点？您只需要看Spring Data REST。
    没有比Spring提供的功能更丰富，更全面或更集成的解决方案集来构建功能丰富的RESTful Web应用程序。

  4. Git的 “ING参与今年以来，特别是锯社区互动
    在Spring开源项目激增之后，现在所有项目都完全在GitHub.com/SpringSource上 。Spring和其他项目一直都是开源的，但是Git支持的协作模型使诸如
    Spring Social，Spring Integration和Spring Data可以依靠社区的投入和贡献而繁荣。

  5. 扩展SpringSource内容的覆盖范围我们一直在努力为所有开发人员带来有关SpringSource的丰富内容，并且已经扩展
    今年很多。例如，除了在SpringSource.org上发布内容之外，您还知道吗
    您可以在Twitter的@SpringSource上找到SpringSource，
    Google+上的+ SpringFramework ，
    在YouTube SpringSourceDev频道上以及（对于中国的许多粉丝来说特别有用） 在新浪微博上的SpringFramework上 ？此外，如果您喜欢此综述，请确保在“春季本周”聚合页面中添加书签。


现在，继续本周综述！有很多要讲的，希望您不希望阅读任何东西
这个星期，如果您要休假，还有一些空闲时间！
