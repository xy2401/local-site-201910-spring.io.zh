春天的这个星期-2012年9月11日

工程
乔什·朗（Josh Long）
2012年9月11日




欢迎来到春季本周的另一个装置！我将去挪威奥斯陆参加JavaZone会议，与人们讨论如何在Cloud Foundry上使用Spring Integration和Spring Batch 。再次，这是一个自然的用例：Cloud Foundry使其易于扩展以处理最大的工作负载，而Spring Integration和Spring Batch（大概位于RabbitMQ之上）负责管道工作，并在整个工作范围内进行繁重的工作负载分配集群。

 
Wrapping up from last week's Cloud Foundry   Open Tour - India, touring Bangalore, and Pune, I got great feedback about using Spring MVC for multi-client applications   with REST. We looked at deploying those applications to Cloud Foundry, the open-source PaaS from   VMware, where you can control the range of channels that your Spring MVC/REST   architecture can deliver to. It's been a truly exciting time. Cloud Foundry   gives developers a platform to deploy the Spring applications they want to   build, how they want to build them, without having to worry about infrastructure   and middleware concerns. 

 
Also don't miss the Spring, Groovy and Grails event of the year in Washington, DC: SpringOne2GX.  If you haven't already registered, now's the time!

像往常一样，我们有很多有趣的内容要看，所以让我们开始吧！



  1. Spring Social的领导者Craig Walls宣布一切开始，宣布了Spring Social Facebook的最新版本1.0.2 。新版本与最新的Facebook API更新保持同步。

  2. Shekhar Gulati在介绍Spring Roo方面做得非常出色。在他关于IBM Developer Works的系列文章的最新两期中，Shekhar介绍了构建Spring MVC和GWT应用程序以及开发Spring Data MongoDB应用程序 。

  3. Hubberspot博客上的此博客条目介绍了如何使用Spring的XML应用程序上下文标记，用于通过参数索引指定bean构造函数参数 。

  4. MacPhail博士组建了一个非常不错的博客，介绍了对经典Spring PetClinic应用程序进行现代化改造，对默认servlet的使用进行了改进以及引入了注释驱动的配置。上的信息如何在default servlet的工作原理以及从3.0版本开始的Spring MVC如何让您利用它，这非常棒，并且对任何人来说都是快速，值得阅读的内容。

  5. 企业消息传递中的一种常见用例是尝试在事件流被其他客户端消耗之后从事件流中推断状态。如果有一种方法可以针对特定的值在RabbitMQ上获得追溯消费者的行为，那不是很好吗？借助RabbitMQ的可插拔交换功能，您可以。我刚刚看到了RabbitMQ 的最后值缓存的实现。我承认，我没有时间尝试它，但是它看起来确实很有趣且功能强大！

  6. Boyko Todorov整理了一个分步博客，介绍如何创建Maven Web应用程序以及如何安装构成Spring Batch应用程序的有趣部分 。
    这些步骤有效，并且确实有一些好处，例如安装Quartz作业引擎，
    但如果您使用的是开放源代码的免费Spring Tool Suite（STS） ，则不是最简单的方法。在STS中，您只需转到File > New > Spring Template Project菜单，那里有几个选项，包括一个用于在Web应用程序中配置完整Spring Batch Admin的项目！

  7. 克里希纳的博客上有一篇不错的文章
    Spring Web Service出色的测试驱动开发支持 （ 带有代码！ ）。
    他展示了如何使用MockWebServiceClient将负载驱动的请求驱动到Web服务，并验证结果。为了站起Web服务，他使用Spring Integration入站Web服务网关。这是设置Web服务端点的另一种方法。另外，您可以使用Spring Web Services并直接在Web应用程序中或在Spring MVC的Web处理机制之上公开它们。
    这种方法对我来说更有意义，因为它使您能够使用Spring Integration工具箱中包含的许多谚语工具更好地表达请求处理管道。

  8. 的Tshikatshikaa ，或Technical Notes博客很好地介绍了如何使用JPA 2和Spring 3.1的测试支持以及Java配置对Spring服务以及数据访问对象（DAO）进行单元测试 。

  9. JBoss的Mark Proctor和ValueMomentum软件服务Pvt。 Ldt的Vinod Kiran有一篇不错的文章， 介绍了流行的开源规则引擎的最新版本Drools在5.5的beta版本中如何更好地支持Spring配置 。新的支持采用了版本5.3中提供的现有支持，并将其扩展为支持知识运行时记录器（控制台，文件，线程文件）的声明式配置。帖子中也包含许多代码示例！

  10. Uttesh Kumar在测试RESTful Spring MVC服务的各种方法上有一篇不错的文章。第一个示例使用Spring的内置RestTemplate集成测试Web服务的对象。
    该文章还链接到一些非常好的客户端，包括SoapUI，Firefox REST Client插件和Chrome的POST MAN插件。

  11. Jijo Mathew在如何使用Spring MVC构建基于Ajax的应用程序方面写了一篇很棒的文章。他没有解释，而是用大量的代码示例进行了演示。
