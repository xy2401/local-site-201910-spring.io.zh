新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2012年11月30日

 

使用Spring 3设计REST-ful API



在创建VAS的过程中，我们做了很多工作来设计有用的REST API。


REST通常是一个非常松散的主体集合，可以用多种方法来解释，因此本演讲将描述一个REST-ful API应该是什么样的更具体的想法。


此外，该API的实现是使用Spring 3中的许多新功能完成的，很好地展示了其提供的功能。










关于本·黑尔



Ben Hale是Springsource的高级软件工程师，也是SpringSource dm Server项目的核心开发人员。Ben专门从事中间件开发，使用OSGi和面向切面的编程等技术，并指导Spring和SpringSource产品组合中所有产品的构建和发布过程。


他的兴趣包括中间层体系结构以及有效的构建和发布管理策略。


在加入SpringSource之前，Ben在电信行业的大型企业管理应用程序的体系结构和开发中花费了数年的领导团队。


 



 


 


IOC + Javascript



较厚的Web客户端和服务器端JavaScript产生了必须通过体系结构模式进行管理的复杂性。JavaScript尚未接受从其他平台（如Java + Spring）汲取的教训。现有的JavaScript MVC框架过于僵化，缺乏足够的体系结构。Javascript需要灵活的架构基础架构来构建更大，更好的应用程序。


在本演讲中，Brian和John将介绍几个概念，包括JavaScript模块和控制反转，并演示它们如何减轻构建大型JavaScript应用程序时遇到的许多主要问题。与会者将对新的架构模式有更深入的了解，并亲眼目睹这些模式如何提高可测试性，重构性，可组合性，工作分工和团队可伸缩性的实例。
