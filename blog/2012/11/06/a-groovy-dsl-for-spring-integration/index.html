用于弹簧集成的Groovy DSL

工程
大卫·图兰斯基（David Turanski）
2012年11月6日

Spring Integration使用Spring编程模型实现企业集成模式，以在基于Spring的应用程序中启用消息传递。Spring Integration还使用声明性适配器提供与外部系统的集成，该声明性适配器支持jms，http，amqp，tcp，ftp（s），smtp等。当前，配置消息流主要是通过Spring XML完成的，Spring Integration支持多个命名空间，以使其尽可能简洁。今年早些时候，SpringSource发布了用于Spring Integration的Scala DSL 。现在，我们很高兴地宣布第一个里程碑版本（1.0.0。Groovy DSL的 M1）。

这两种DSL都有一个共同的目标-为Spring Integration提供XML配置的强大而灵活的替代方案。由于Groovy DSL借鉴了Scala DSL引入的概念，因此这两种语言在语义上也相似。此外，两者本质上都是Spring Integration之上的外观。但是，相似之处到此结束。许多差异可以归因于Scala和Groovy之间的语言差异，最明显的是静态和动态类型。Groovy DSL主要针对Groovyist，他们对DSL所基于的构建器模式的分层语法感到满意。这也应该吸引Java开发人员，他们可以利用DSL必须提供的丰富功能，并且会发现语法非常容易实现。

你好，世界

让我们从Groovy示例开始：


def builder = new IntegrationBuilder()

def flow = builder.messageFlow {
   transform {"hello, $it"}
   handle {println it}
}

flow.send('world')

这将创建一个Spring应用程序上下文，构造一个包含一个转换器（转换）和一个服务激活器（句柄）的Spring Integration消息流，并将这些端点与直接通道连接起来，以便它们将按顺序执行。转换器将消息有效负载（在这种情况下为“ world”）附加到字符串“ hello”，然后服务激活器将结果打印到STDOUT。瞧！在这里，我们看到了构建器模式的一个简单实例。对于那些不熟悉Groovy的人，这是有效的Groovy语法。 messageFlow ， transform和handle是DSL定义的所有方法。 {}是用于闭包的Groovy语法。由于在Groovy中括号和分号是可选的，因此等效于：


def flow = builder.messageFlow({
   transform({"hello, $it"});
   handle({println(it)});
});

另外，我应该提到，默认情况下，Groovy闭包需要一个名为'it'的参数。闭包参数可以命名，也可以选择键入。例如：


   transform {String payload -> "hello,  $payload"}

还有一件事。Groovy允许您将变量表达式嵌入双引号字符串中。这并非打算成为Groovy教程，但足以说所有Groovy的语法糖都可以使代码非常甜美，简洁和易读。顺便说一句，与此等效的XML是


<beans xmlns="http://www.springframework.org/schema/beans" xmlns:si="http://www.springframework.org/schema/integration" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd http://www.springframework.org/schema/integration http://www.springframework.org/schema/integration/spring-integration.xsd">
   
   <si:transformer id="t1" input-channel="flow1.inputChannel" output-channel="sa1.inputChannel" expression="'Hello,' + payload"/>
   <si:service-activator id="sa1" input-channel="sa.inputChannel" expression = "T(java.lang.System).out.println(payload)"/>
</beans>

然后，我们将不得不编写大约十行代码来初始化Spring应用程序上下文并发送一条消息。

要在Java中运行DSL示例，有几个选项。一种方法是加载并运行外部DSL脚本：

HelloWorld.groovy


 messageFlow {
    transform {"hello, $it"}
    handle {println it}
 }

Main.java


public class Main {
    public static void main(String[] args) {
          IntegrationBuilder builder = new IntegrationBuilder();
          MessageFlow flow = (MessageFlow) builder.build(new File("HelloWorld.groovy"));
          flow.send("world");
    }
}

除了File实例之外，如上所述，build（）方法还接受InputStream，GroovyCodeSource，Spring Resource甚至是groovy.lang。脚本。因此，如果使用Groovy编译器编译项目，则可以执行类似的操作，其中HelloWorld是groovy.lang的实例。脚本。


public class Main {
    public static void main(String[] args) {
          IntegrationBuilder builder = new IntegrationBuilder();
          MessageFlow flow = (MessageFlow) builder.build(new HelloWorld()));
          flow.send("world");
    }
}
