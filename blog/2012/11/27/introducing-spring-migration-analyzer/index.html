介绍Spring Migration Analyzer

工程
安迪·威尔金森
2012年11月27日

我很高兴地宣布，我们已经发布了Spring Migration Analyzer（SMA）的第一个里程碑， SMA是一个命令行实用程序，可以分析企业Java应用程序并生成描述该应用程序以及如何将其迁移到Spring的报告。

为什么将应用程序迁移到Spring？

我们看到人们选择使用Spring的两个主要原因。首先，Spring提供了包括云和PaaS在内的最大范围的部署选项，使您能够以较低的运营成本将应用程序部署到重量更轻的运行时。其次，正如Adrian最近解释的那样， Spring提供了对企业Java最前沿的一系列技术的访问。

在考虑将现有应用程序迁移到Spring时，通常是部署灵活性会激发这一迁移，因为它可以显着降低应用程序的运营成本。

使用SMA分析应用程序

要开始使用SMA，请下载发行版 。下载后，将其解压缩：

unzip spring-migration-analyzer-1.0.0.M1-dist.zip

设置JAVA_HOME后 ，您可以运行迁移分析脚本来分析应用程序：

./spring-migration-analyzer-1.0.0.M1/bin/migration-analysis.sh ~/dev/apps/my-app.ear

这将在当前工作目录的名为my-app.ear的目录中生成HTML报告。该报告描述了该应用程序，并提供了有关将其迁移到Spring的指导。

除了迁移指南之外，该报告还提供了诸如API使用率，EJB和部署描述符之类的细节，这些细节在迁移应用程序时需要加以考虑。
