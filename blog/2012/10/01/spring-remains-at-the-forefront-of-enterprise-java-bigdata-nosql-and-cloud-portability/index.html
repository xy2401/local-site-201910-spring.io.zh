Spring始终处于企业Java的最前沿：BigData，NoSQL和云可移植性

工程
阿德里安·科耶尔（Adrian Colyer）
2012年10月1日

再次是JavaOne，今年您将找到有关Spring如何与Java SE一起使用以解决大数据 ， NoSQL和企业集成难题的讲座。我们一直致力于通过为新兴趋势和技术提供及时的支持来确保Spring处于企业Java的最前沿。数字表明我们的用户对此表示赞赏。Evans Data的最新报告（“ Evans Data Corporation的Spring资源使用研究”，2011年9月）显示：

  • Spring是Java专业开发人员的首选框架 。目前，超过三分之二的Java开发人员正在使用Spring或计划使用Spring
  • 春季使用量继续增长 。超过一半的现有Spring用户期望增加或显着增加对Spring的使用。
  • 春天正在取得成果 。 70％的Spring用户表示使用Spring至少可提高50％的生产率，而73％的用户表示交付完整项目所需的时间至少减少50％。
  • Spring用户处于企业Java的最前沿 。Spring用户更适合云计算，Spring采纳者更可能已经将应用程序部署到云中。

Spring一直提供部署灵活性。在早期，这意味着可以选择应用程序服务器，但是现在，这已扩展到包括我们自己的Cloud Foundry在内的云平台选择。Spring一直提供强大的数据访问支持。在早期，这意味着可以选择一种用于访问关系数据库的方法，而如今，这已扩展到大数据和NoSQL存储，数据处理以及内存中数据网格功能的范围。Spring一直重视开发人员的生产力，并为编写服务器端Web应用程序提供了一流的支持。今天，它已经扩展到可以满足基于HATEOAS的REST API的现代要求。

这些是当今企业中迫切需要解决的现实问题。云无处不在，现在大数据也已牢牢地放在企业项目雷达上，从而产生了一系列新要求：

  • 大数据：需要从千兆字节，千兆字节或PB的非结构化或半结构化数据中存储和获取分析数据
  • 快速数据：对低延迟与大量数据进行交互的需求不断增长，通常由当今的移动和社交应用推动
  • 灵活的数据：需要使数据访问适应每个应用程序的最合适模型
  • 云交付：在您选择的云上配置的访问数据即服务的需求不断增长。

最新的《 哈佛商业评论》调查表明，有85％的组织已计划或正在进行大数据计划。应用程序开发人员需要提供功能强大的高质量解决方案，以与这个新世界集成在一起，并且可以在内部或外部部署任何位置。NoSQL和NewSQL还用于解决各种用例，包括：

  • 管理非交易数据的大数据流（日志，点击流）
  • 欺诈识别
  • 时间轴（Twitter）
  • 同步离线和在线数据
  • 线上游戏
  • 文件，目录和内容管理
  • 归档-存储仍可在线访问的大量连续数据流
  • 嵌入式系统（设备中的小型数据库）

有关更多详细信息，请参见此High Scalability博客条目 。

451 Research的最新调查数据突显了数据格局不断变化的形状。一种尺寸不再适合所有尺寸。

现在，企业开发人员可以通过Spring和熟悉的Spring编程模型开始利用这些新技术。春季优惠：

  • 通过Spring Data对Apache Hadoop和HBase的支持以及Spring Batch和Spring Integration的大数据处理
  • 通过Spring Data GemFire支持内存中的数据网格
  • 通过Spring Data Redis支持NoSQL键值存储
  • 通过Spring Data MongoDB支持NoSQL文档存储
  • 通过Spring Data Neo4j支持NoSQL图形数据库
  • 通过SQLFire支持NewSQL数据库
  • 当然，传统的关系数据库（例如MySQL， Postgres ，Oracle和DB2） 仍然存在 ，可以通过Spring JPA和Spring JDBC Extensions访问，并可以通过Spring Data REST进行访问。

围绕着NoSQL蓬勃发展的开源社区以及Spring，使下一代数据存储和数据处理的开发今天变得更加高效。我们没有为所有持久性存储提供单一的Java API。考虑到它们之间的根本差异，这是不现实的。但是我们确实基于模板和存储库（如果有）的概念提供了一个常见且熟悉的编程模型。有关更多详细信息，请参见Tobias Trelle最近的InfoQ文章 。

总而言之，JavaOne 2012肯定会是一个充满动感的一周，将有许多精彩的Java技术讲座，主题演讲和会议，其中包括我们自己的大数据 ， NoSQL和集成计划的内容。想要接触这些技术的开发人员现在也不会想错过2012年10月15日至18日在华盛顿特区举行的SpringOne 2GX 。我们有一个紧凑的技术议程，涉及现代企业应用程序开发的所有方面。
