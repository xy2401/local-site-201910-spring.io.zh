Spring MVC：从JSP和Tiles到Thymeleaf

工程
迈克尔·艾维（Michael Isvy）
2012年10月30日

当涉及到视图层时，Spring @MVC提供了多种选择。在本文中，我们将首先讨论您在过去几年中最有可能使用视图层的方式： JSP 。我们将看到使用它们的坏方法和更好的方法（纯JSP ，带有自定义标签的JSP ， Apache Tiles ）。

然后，我们将讨论一个名为Thymeleaf的新项目，您可以将其用作JSP的替代方法。

与往常一样，您可以在github上的相应应用程序中找到讨论的所有代码示例。

从8年前可能以完全相同的方式编写该代码示例以来，它并不完全是“最新技术”。这是否意味着它已经过时了 ？让我们讨论一些可能的限制。
1）布局
我们已经使用为了包括JSP片段（页眉和页脚）。显然有因为它会阻止您进行大量复制/粘贴。但是，如果您有数百个JSP文件，您会发现自己复制/粘贴了这些文件标签添加到所有JSP中。最好将所有布局信息外部化到一个专用文件中。
2）详细
我们的用户页面很小，因为它只显示元素列表。我们已经有50行代码（上面的代码示例已略有减少）。您可以想象如果我们必须显示很多内容会有多大。
3）符合HTML / CSS
此页面不兼容HTML / CSS 。假设Web设计器已将其原型化，则必须完全重写它才能使用侵入式JSP语法。在谈论ThymeLeaf时，我们将回到这一点。

利弊

标准标签有很多好处：

     
  • 我可以使用它们做更多的事情，而不仅仅是外部化布局信息。最后，它们可以轻松地使您的JSP缩小5倍。

    •  
  • Eclipse / STS与自定义标记配合使用很好，因此您可以使用CTRL + space来自动完成。


不利的一面：

     
  • 文档不是最好的。

    •  
  • 尽管自定义标签已经非常整洁，但我不记得过去几年中对它们的任何改进。

    •  
  • 使用自定义标签意味着您正在Web容器内使用JSP引擎。然后，您将根据使用的Web容器（Apache Tomcat，IBM Websphere，Oracle Weblogic…）预期会有一些细微的差异。

    •  
  • 对视图层进行单元测试也比较困难。如果您对此主题感兴趣，可以查看Spring MVC Test框架 。

利弊

光明的一面：

     
  • 百里香叶（ThymeLeaf）是一个健康的开源项目：每个月都会有新功能，良好的文档，反应灵敏的用户论坛……

    •  
  • 如果您希望您的网站设计师能够读取视图文件，则它是理想的模板引擎

    •  
  • 使用的表达式语言（实际上称为标准方言）比JSP表达式语言强大得多。

    •  
  • 与JSP不同，Thymeleaf适用于Rich HTML电子邮件（请参阅http://www.thymeleaf.org/springmail.html）。


不利的一面：

     
  • Thymeleaf还没有与自定义标签（.tagx文件）等效的标签。

    •  
  • 在此阶段，ThymeLeaf与JSP标签库不兼容。


结论

我们已经看到JSP和Thymeleaf并存。如果您的应用程序使用数百个JSP，我们并不是说您应该抛弃所有的JSP，然后再次使用Thymeleaf重新开始。但是，您可以考虑将Thymeleaf用于Web容器之外的HTML页面，例如Rich HTML电子邮件。
如果您要开始一个新项目，我们强烈建议您将Thymeleaf和JSP进行比较，以确定哪个更适合您的需求。
另外，我的同事Rob Winch就Spring MVC的现代模板选项进行了精彩的演讲。除了JSP和Thymeleaf，它还讨论了Mustache模板。
