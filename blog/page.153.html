工程
马丁·利珀特
2015年10月22日

欢迎回到Spring社区，

有关Spring Tool Suite中新的Spring Boot Dashboard的系列文章的最后一部分。在最后一部分中，我们将更深入地研究将Spring Boot Devtools与启动仪表板结合使用。同样，此功能是Spring Boot 1.3的新增功能，因此您必须使用该版本才能使用以下功能。

快速部署代码更改

对于本地应用程序，使用Spring Boot Devtools非常简单明了。一旦将Spring Boot Devtools作为依赖项添加到项目中（上下文菜单的Spring类别中有一个简单的菜单选项）并启动您的应用程序，它将监听配置文件和类文件的本地更改，并自动为您启动应用程序的重启。您甚至不需要自己重新启动应用程序，Spring Boot Devtools会自动为您完成此操作。由于每当您保存文件时STS / Eclipse都会生成和更新类文件，因此在STS中工作时，所有这些都会自动发生。

新闻与活动
彼得·汉弗莱（Pieter Humphrey）

记录在SpringOne2GX 2015
演讲者：约翰·菲尔德，肖恩·麦金尼
幻灯片： http ： //www.slideshare.net/SpringCentral/a-how-to-guide-to-security-in-the-paas-cloud
大多数开发人员仍然孤立地处理应用程序安全性问题，而不了解“全栈”的安全性。结果，安全性有时会不一致，并且可以看作是将应用程序迁移到云的障碍。该会话将检查企业部署中典型Java Web应用程序的安全性。然后，我们将研究将安全Java应用程序“铲车”到Cloud Foundry中时需要更改的内容。最后，我们将了解采用云原生安全协议（例如OAuth2和SAML2）的好处。此旅程将涵盖从现实世界中的客户问题中提取的5种常见应用程序安全体系结构模式。我们将比较独立应用程序和云本机应用程序之间安全集成模式的差异。从传统的企业身份管理集成到微服务的安全性，该技术会议包括实用的实践指南，以使用Spring和Cloud Foundry正确实施身份验证，授权和机密性控制。我们都听说过“全栈”开发人员的想法，即可以在整个栈中上下左右理解应用程序的人。本届会议的目的是描述完整的安全性堆栈，并展示其在独立部署和PAAS部署之间的区别。
