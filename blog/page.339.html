Spring LDAP 1.2 RC1发布

发布
乌尔里克·桑德伯格
2007年5月13日

尊敬的Spring社区，


我们很高兴地宣布Spring LDAP 1.2的第一个候选版本，其中包含许多功能和错误修复。这里只列出最重要的。有关完整列表，请参阅changelog 。
该版本可从此处下载 。

  • 为Spring LDAP实现了客户端事务支持。有关更多信息，请参见参考文档（ LDAP-29 ）。
  • 将异常层次结构更改为JNDI NamingException层次结构（ LDAP-4 ）的未经检查的镜像。
  • 现在，无论包装的NamingException是否（并非总是如此）（ LDAP-14 ），Spring LDAP引发的异常始终可以序列化。
  • 重新编写LdapEncoder.nameDecode（）以解决国家字符问题，并删除解析中使用的正则表达式，从而极大地提高了专有名称解析性能（ LDAP-30 ）。
  • 内部升级到Spring 2.0.4。仍支持Spring 1.2.x（ LDAP-35 ， LDAP-51 ）。

Spring Web Flow Bean范围和JSF

工程
本·黑尔
2007年5月8日

我最近在Spring Web Flow中完成了一个有趣的问题。该问题（ SWF-163 ）涉及为Spring Web Flow的内部范围添加Spring 2.0 bean范围支持。实现并不是那么有趣（毕竟Scope接口很容易实现），但是我想确切地提到您将如何在应用程序中使用类似的东西。

Spring 2.0范围界定

在Spring 1.x中，我们想到了singleton和原型bean作用域，但是这种表示法是固定的，并且对singleton =” [true |假]” 。因此，在Spring 2.0中，此表示法已从XSD的配置样式中删除，现在您看到的表示法在scope =” [singleton |原型| …]” 。Spring本身增加了三个bean作用域；与Web应用程序相关的request ， session和globalSession 。
