SpringSource dm Server 1.0 RC2发布

工程
罗伯·哈罗普
2008年9月11日

我很高兴地宣布SpringSource dm Server （以前称为SpringSource应用程序平台）的RC2的可用性。此版本功能完善，除非出现严重问题，否则两周后将变为1.0 GA。

此版本修复了一些关键的错误，升级到Tomcat 6.0.18，并更新了所有代码，文档和支持材料以反映新名称。

由于产品的重命名， PlatformOsgiBundleXmlWebApplicationContext已重命名为ServerOsgiBundleXmlWebApplicationContext，并已从com.springsource.platform.web.dm包移至com.springsource.server.web.dm包。因此，如果您在共享服务WAR中的web.xml中为Spring MVC的ContextLoaderListener或DispatcherServlet设置contextClass，请确保将标准路径更改为com.springsource.server.web.dm。ServerOsgiBundleXmlWebApplicationContext 。

重命名PlatformOsgiBundleXmlWebApplicationContext还会影响对Web模块或共享服务WAR的BundleContext的编程访问：用于从ServletContext检索BundleContext的常量现在为ServerOsgiBundleXmlWebApplicationContext。BUNDLE_CONTEXT_ATTRIBUTE 。

表示个性特定类型的清单标头的名称已从Platform-ModuleType更改为Module-Type 。因此，为了使现有的Web模块继续运行，您将需要更新Web模块中的META-INF / MANIFEST.MF并将Platform-ModuleType：Web替换为Module-Type：Web 。

请注意，所有示例应用程序中的Web应用程序（例如，宠物诊所，表格标签和Spring Travel）都已更新，以反映这些更改。

以前在com.springsource.platform下发布的MBean现在在com.springsource.server下发布 。

下载，样本和文档可在以下位置找到： http://www.springsource.com/beta/dmserver 。
