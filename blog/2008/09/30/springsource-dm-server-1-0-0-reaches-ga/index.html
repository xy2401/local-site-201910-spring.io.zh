SpringSource dm Server 1.0.0到达GA

工程
彼得·库珀·埃利斯
2008年9月30日

您可能已经注意到，SpringSource今天宣布了SpringSource dm Server™的全面上市。dm服务器是SpringSource应用程序平台的一部分 。由于这是dm Server的首次使用，因此我想对产品做一些简短的评论。

我们相信dm Server总体上将改变企业Java软件的开发和部署方式。特别是，dm服务器是从头开始设计的，它是轻量级的（dm内核的占用空间约为3兆字节），灵活且快速。它还旨在促进模块化开发并简化依赖关系管理。在操作方面，dm Server旨在在运行时配置，部署，启动和停止服务。随着服务越来越多地部署在网格，虚拟化和云环境中，最初仅提供内核并根据服务请求动态加载其他服务的能力将成为一个重要的区别。

祝贺Rob Harrop和工程团队发布了非常重要的版本。团队必须解决一些棘手的问题 ，才能在dm服务器上无缝部署标准WAR文件。我特别赞扬该团队坚持他们的愿景，即创建一个用于部署Spring应用程序的高级平台。该产品提供其他产品堆栈所没有的真正的应用程序和基础结构模块化。它为开发人员提供了处理企业Java开发中固有的复杂性（例如处理共享库和依赖项）的切实好处。此外，服务器通过高级日志记录和跟踪功能在可维护性方面建立了新的标准。

展望未来，我们的愿景是提供一个从头开始设计的全方位服务平台，用于将企业Java应用程序部署到现代数据中心环境中。仍有大量工作要做，但团队取得了良好的开端，他们在这里建立了一些非常有趣的技术。实际上，SpringSource 实现了 BEA和IBM等其他商业供应商都难以实现的愿景。

我们从一些Beta版用户和一些关注该版本的分析师那里收到的反馈是，我们的发展方向正确。我们鼓励您下载 dm服务器并亲自尝试。并且，当然，请给我们反馈。该团队现在正在设计下一个版本，因此时机适合合并用户反馈。

感谢您对SpringSource dm Server的关注！
