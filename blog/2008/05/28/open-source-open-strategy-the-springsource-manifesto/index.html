开源，开放战略：SpringSource宣言

工程
罗德·约翰逊
2008年5月28日

作为开源软件提供商，我们认为我们也应该对我们的策略持开放态度。我们想分享我们到达这里的方式，去向以及为什么这个旅程对Spring有益，对Spring用户有利，对SpringSource也有利。

我们的历史

春天的故事始于2001年，那时我开始研究与2002年与专家一对一J2EE设计和开发一起发布的30,000行框架代码。我的目标是帮助其他人避免自1999年以来完成J2EE项目时遇到的陷阱。

很快就很清楚，其他人喜欢该代码中的想法，例如Dependency Injection和Spring数据访问抽象，并从将它们付诸实践中受益。那些要求我发布代码并希望做出贡献的读者与我联系。

我很快看到开放源代码的一些重要好处。


  • 大多数用户免费获得所需的功能

    •         <li> It develops a strong community, which contributes to making the software better </li>

        <li>Open source is inherently anti-bureaucratic </li>
        <li>Open source can underpin a business model with lower sales and marketing costs than a traditional enterprise software company---meaning better value for customers. </li>
        <li>Open source projects (and, hence, open source companies) can attract developers worldwide, rather than in any one geographical area. This represents a huge pool of talent that isn't available to traditional software vendors.</li>

实际价值，实际成本

这些好处是巨大的，但是它们并没有中止重力定律或经济学定律。

魔术无法开发任何软件。对于Spring，我们很早就召集了一批杰出的开发人员。这些人付出了巨大的个人代价，这不可能永远维持下去。我花了大约18个月的工作时间从事Spring及其背后的想法。这影响了我家庭的财务稳定：我们甚至需要从抵押贷款中提取大量款项。Juergen Hoeller很幸运有一位雇主，他了解Spring在构建其软件产品方面可以带来的潜在节省。但是，他很快需要转而从事兼职工作，并做出相应的薪水牺牲，以保持对Spring的承诺。正是Juergen和我本人的长期，紧张，专注的努力才使Spring迎接了今天。诸如Colin Sampaleanu和Thomas Risberg之类的其他核心开发人员可以通过投入自己的个人时间以及牺牲家人和朋友的方式做出更加断断续续的贡献。这种情况的可持续性不足以支撑企业的关键基础架构。

从长远来看，所有软件开发都需要投资。不仅是编写代码方面的投资：还需要长期维护代码。

我们于2004年成立了SpringSource（当时称为Interface21），以实现这一投资。

第二年，我们得以聘请了第一位高调的员工，他帮助提高了团队的火力及其智力领导力-Adrian Colyer：IBM高级技术人员，AspectJ项目负责人。这是一个重要的里程碑。这是我们第一次能够使某人能够在Spring上工作，而其他人无法这样做。

最初，我们的业务以咨询和培训为基础。显然，这些业务无法维持用户社区期望的投资水平，也无法实现我们热衷于以技术人员的身份实现的愿景。我们的成长受到限制；我们太多最好的人花了大量时间和精力在提供服务上，而几乎没有编写软件的余地；而且我们担心我们会耗尽我们被迫要求的差旅费和加班费。

当Spring 2.0发布的时间比原计划晚了几个月时，我们意识到我们当时的业务模型只是一个起点，而不是终点。同样在2006年，随着Spring产品组合的扩大以及我们对OSGi和Spring模型潜力的探索，我们的愿景得以增强，这证明了基于Spring的服务器平台的变革潜力。

我们决定在2007年筹集资金以实现这一愿景，并弥合我们从服务业务到软件公司的过渡，该公司可以维持高质量软件的创建。

好处是巨大的。我们能够招募更多的明星开发者加入团队，并使他们能够为开源做出更多贡献。我们能够将才华横溢的产品开发人员集中到我们现有的员工中，以增强和扩展Spring产品组合。在IBM减少投资之后，我们能够确保AspectJ的未来。

我们最近几个月的开源发行记录不言而喻：

    <ul>
<li> Spring 2.5 </li>
        <li>Spring.NET 1.1 </li>
        <li>Spring Security 2.0 </li>
        <li>Spring Web Flow 2.0</li>
        <li>Spring Batch 1.0  (co-developed by SpringSource and Accenture) </li>
        <li>Spring Web Services 1.5 </li>
        <li>Spring Integration </li>
        <li>Spring Dynamic Modules 1.0 </li>
        <li>Spring IDE 2.0</li>
        <li>AspectJ 1.6</li>

所有这些版本都极大地推动了这些项目的发展，并为用户带来了实实在在的收益。

我们还为其他开放源代码项目做出了重大贡献，例如Apache Tomcat，Apache HTTPD（支持大部分Internet的Apache Web服务器）和其他Apache项目，以及Eclipse的Mylyn项目。

我们的价值

在过去的5年中，Spring团队已经从项目团队发展成为公司，我们的业务模式已经从服务业务转变为具有出色服务能力的软件公司。在整个转型过程中，我们一直保持着核心价值。特别是：


  • 我们一直专注于技术领先和卓越。

  • 我们的目标不是提供过分的解决方案，而是要提高技术水平。

  • 我们提供务实的技术解决方案。软件的价值仅在于其在现实世界中提供的结果。

  • 我们认为，开源业务的长期成功需要对开源做出重大贡献。

  • 我们为自己的诚信感到自豪。我们对社区，用户和客户诚实。

  • 我们努力为客户提供最大的价值。

  • 我们尊重用户，客户，合作伙伴和竞争对手。

  • 我们珍视我们的社区，并为自己的最大利益而努力。

我们的行动源于这些价值观。例如：


  • 我们不会重造一个好轮子。如果可能的话，我们将使用现有项目，如果我们认为它们对我们的用户很重要，请参与其中，例如AspectJ，Tomcat和Equinox。我们是其中前两个项目的最积极参与者。为此，您将看到我们更多地参与了Apache和Eclipse社区。我们的目标是成为企业Java开源的领先提供商，因此我们很自然地在重要的开源项目背后的社区中发挥积极作用。

  • 我们将创建没有好的解决方案的新项目。Spring Batch是一个很好的例子-将Power的功能，简单性和一致性的Spring值带到Java基础架构中被人们忽视的领域。

这些价值观的力量帮助我们公司度过了快速增长的时期。这也是我们与Covalent整合成功的关键因素， Covalent是我们在2008年1月收购的一家历史悠久的开源公司 。这两个组织具有相似的文化，使员工自然而然地融入在一起。它有助于我们继续吸引杰出的技术和业务人才。

我们（和您 ）的软件

在SpringSource，我们开发三种不同的产品：

     <ul>
  • 无处不在的编程模型和基础架构。这涵盖了Spring Portfolio，以及AspectJ（我们领导）和Tomcat（我们是主要贡献者）。我们希望每个人都能使用这些项目。其中许多是事实上的标准。
    •      <li>

    SpringSource应用平台 。 基于Spring产品组合以及其他Apache和EPL软件的完整应用服务器产品。

         <li>

    企业增值 。 增值到两类开源产品。我们通过每年为客户订购商业许可来提供这些服务。它们提高了构建Spring应用程序的效率（如SpringSource Tool Suite），或在生产环境中运行这些产品的操作经验（如SpringSource Application Management Suite）。他们没有定义编程模型或部署模型，而是通过使用产品的前两类来增强可获得的体验。用户不会被迫购买这些增值产品（与传统软件许可证不同）；他们可以自己确认他们提供物有所值。

    我们的经营策略

    我们用Spring重新定义了企业Java。我们的使命是继续提供技术领导力和解决方案，以将企业Java推向新的高度。我们正在围绕这个建立一个伟大的软件公司。

    当前，围绕企业Java的交付的软件价值和经济活动之间存在明显的偏差。绝大部分的收入都流向了BEA（Oracle）和IBM，但客户使用的大部分运行时都是开源的，而企业Java中重要的创新则主要来自其他地方。

    显然，企业Java市场需要新的解决方案，而且显然，市场需要开源公司的解决方案。我们相信它将是我们。

    我们通过以下方式赚钱：

        <ul>

  • 提供世界一流的支持和服务。这包括可靠的24x7全天候支持，出色的培训和咨询服务以及对可以避免风险的企业客户的赔偿。

  • 添加可带来价值的订购产品，以补充Spring产品组合。

  • 出售我们全栈产品的企业版订阅。

    • 我们的许可策略

    我们最近在GPL v3下发布的SpringSource应用程序平台引起了很多讨论。我想借此机会解释我们的许可策略，以及为什么我们认为这是Spring社区的正确选择。

    首先，让我们完全清楚，并解决一个重要问题：

    我们不会更改，也不会更改任何现有项目的许可证 。Spring产品组合将仍受Apache许可。它涵盖了Spring Framework ，Spring Security，Spring Web Flow和Spring产品组合的其余部分。

    我们一直在使用Apache许可证（和EPL）的任何地方。但是，并非所有软件都一样。不同的许可证适用于不同的产品。今年，SpringSource将几种重要的新产品推向市场，这些新产品适用于不同的许可。

    Apache许可证以外的许可证对我们有两个用途：

        <ul>
<li><em>For additional products available only to our customers</em>. These products satisfy a real need for those customers, and help to sustain the open source software that they and others benefit from. </li>

        <li><em>To ensure that ISVs and OEMs using our new stack products don't get a free ride from software we develop for our community, and that software vendors can't compete with us with our own code</em>. The GPL v3 license used for the SpringSource Application Platform meets this goal, while remaining free to end users or open source usage.

    让我们详细考虑涉及SpringSource应用程序平台的第二点。这是一个全栈产品，可与Oracle / BEA（WebLogic，OC4J），IBM（WebSphere）和Red Hat（JBoss）的产品竞争。所有这些供应商都意识到，他们还需要基于OSGi的成熟运行时。我们在这一领域拥有领先的技术。所有这三个都需要完成我们使用SpringSource Application Platform的dm-Kernel™完成的繁重工作。

    假设我们在ASL下发布了SpringSource Application Platform。我们可以期望这些供应商在短期内与我们竞争，他们可能会向客户收取使用该技术的产品的费用。这不仅不公平，而且会降低我们投资该产品的能力-最终伤害了整个社区。

    因此，我们选择了许可，这意味着最终用户可以从我们的工作中自由受益，但竞争对手无法使用我们自己的代码与我们竞争。

    接下来呢？

    我们旨在基于Spring项目和Spring哲学创建一个完整的Java堆栈。到目前为止，无论走到哪里，我们都使事情变得更容易，更好和更快。我们将去更多的地方。有些人表示担心我们的努力会被淡化，但是证据（例如，最新的Spring Portfolio发布）证明了我们正在加速。在过去的6个月中，我们以前所未有的速度发布了更多的开源。我们正在以与开发基础产品相同的模块化方式来努力。我们的产品策略本质上是反整体的，随着组织的发展，这种策略可以很好地转换。

    多年以来，我们创造了伟大的技术。今天，我们正在创造比以往更多的东西。我们为推动企业Java从EJB 1.x和2.x的苦难转向POJO敏捷开发而感到自豪。我们已经为企业客户提供了数十亿美元的价值，并且在未来，我们将提供更多的价值。

    我们很高兴继续讲述并提供更多，更好的基础架构。企业Java社区需要一家专注于提供最佳解决方案的公司。我们曾经用Spring重新定义了企业Java，而这又是使用SpringSource Application Platform进行企业Java的又一个新赛季。请挑战我们，因为我们挑战现状。
