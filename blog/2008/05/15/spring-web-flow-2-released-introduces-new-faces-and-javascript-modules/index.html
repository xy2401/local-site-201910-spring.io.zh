Spring Web Flow is the project in the Spring Portfolio that focuses on providing the infrastructure for building and running rich web applications. As a Spring project, Web Flow builds on the Spring Web MVC framework to provide:

  • A domain-specific-language for defining reusable controller modules called flows
  • An advanced controller engine for managing conversational state
  • First-class support for using Ajax to construct rich user interfaces
  • First-class support for using JavaServerFaces with Spring

Spring Web Flow 2发布；推出新面孔和JavaScript模块

发布
基思·唐纳德
2008年5月15日

亲爱的春季社区，

我们很高兴地宣布Spring Web Flow 2全面上市。下载 | 文献资料

Spring Web Flow是Spring产品组合中的项目，致力于提供用于构建和运行富Web应用程序的基础结构。作为Spring项目，Web Flow建立在Spring Web MVC框架的基础上，以提供：

  • 用于定义称为流的可重用控制器模块的领域特定语言
  • 用于管理对话状态的高级控制器引擎
  • 对使用Ajax构建丰富的用户界面的一流支持
  • 对在Spring中使用JavaServerFaces的一流支持

Web Flow 2发行版的模块及其与Spring Framework 的关系如下所示：

Web Flow 2中的内容

 

Spring Web MVC

Spring Web MVC框架是Spring Framework发行版的模块，为使用Spring使用成熟的ModelViewController范例开发Web应用程序提供了基础。Web Flow发行版的每个模块都基于此基础。

Spring Web Flow

Web Flow模块是MVC扩展，允许您使用特定于域的语言定义控制器。此语言旨在对需要数个请求才能完成的用户交互进行建模，或者可以从不同的上下文中调用。

春季JavaScript

Spring JavaScript是一个JavaScript抽象框架，可以轻松编写流畅的JavaScript来逐步增强具有行为的Web页面。该框架由公共JavaScript API以及在Dojo Toolkit上构建的实现组成。Spring.js旨在简化Dojo在常见企业场景中的使用，同时保留其在高级用例中的全部功能。

Spring JavaScript可以与任何服务器端框架一起使用。Web Flow 2发行版包括Spring JavaScript和Spring Web MVC之间的便捷集成，用于处理Ajax请求。

春天的面孔

Spring Faces模块包含Spring对JavaServerFaces的支持。这种支持使您可以在熟悉的Spring MVC和Web Flow Controller环境中将JSF用作View技术。通过这种架构方法，您可以将JSF UI组件模型的优点与Web MVC架构的优点相结合。Spring Faces还包括一个基于Spring JavaScript的轻量级组件库，用于以渐进方式声明性地启用Ajax和客户端验证行为。

Web Flow 2发行版的主题

除了引入新的Spring Faces和Spring Javascript模块之外，Web Flow 2的发布还解决了两个主要主题：集成和简单性。

积分

在每个模块中，Web Flow 2发行版添加了许多有趣的集成，这些集成使您可以丰富自己的Web应用程序。这些集成支持：

  • 使用Spring Security以声明的方式保护您的流程
  • 使用Tiles进行JSP页面合成和Ajax部分渲染
  • 使用JSF时，使用Facelets进行页面合成和布局
  • 使用JSF时，请使用Apache Trindad和JBoss RichFaces组件库
  • 以循序渐进的方式使用Dojo小部件系统；如果客户端上没有JavaScript，则会优雅地降低性能

简单

流定义语言已经在Web Flow 2中得到了极大的简化，同时整体上变得更加强大。这些简化包括：

  • 将版本2流定义与其等效版本1进行比较时，代码行总体减少了大约50％（例如： 版本2与版本1 ）
  • 使用表达式语言（EL）调用动作的简洁语法，同时支持Unified EL和OGNL
  • 声明式模型绑定和验证，并支持基于约定的配置
  • 支持使用流定义继承在流和状态级别重用
  • 增强的模块化，允许将流及其相关资源打包到一个独立的包中

发行说明

  • Web Flow 2需要Java 1.4或更高版本，并且可以在所有主要的Java EE平台上运行，包括Tomcat，Jetty，Websphere，WebLogic和JBoss。
  • Web Flow 2需要Spring Framework 2.5.4或更高版本。
  • Web Flow 2已通过SpringSource认证为“平台就绪”，适合在启用OSGi的Web应用程序中的SpringSource dm服务器上运行。


入门



  • 通过下载发行版并逐步浏览参考应用程序来开始使用Web Flow。当您熟悉功能集时，请补充文档 。



  • 要开始使用Maven或Ant + Ivy之类的构建系统，请从Maven中央存储库访问Web Flow工件。



  • 要在SpringSource Application Platform的OSGi环境中开始使用Web Flow，请下载平台并从SpringSource Enterprise Bundle Repository获取Web Flow捆绑包 。
