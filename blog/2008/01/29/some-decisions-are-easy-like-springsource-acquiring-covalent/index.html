有些决定很容易–例如SpringSource获得共价

工程
罗德·约翰逊
2008年1月29日

我的上一个博客显示了Spring如何超越EJB。 BZ Media等人的研究表明，Apache Tomcat是领先的开源应用程序服务器，具有64％的市场渗透率。Spring和Tomcat的统治地位是众所周知的。人们可能还不知道，成千上万的组织正在Tomcat上运行Spring作为其中间件基础架构。这些组织希望一家公司寻求成功所需的产品和服务。

今天，我们宣布了对Covalent Technologies的收购。Covalent不仅带来了Apache的领导地位，而且我们合并后的公司现在在Apache Tomcat和HTTP方面拥有重要的领导地位。两周前，Sun 以10亿美元的价格收购了LAMP中的“ M” 。现在，Covalent出色的Apache专业知识和服务已成为SpringSource的一部分，我们已成为“ A”领域的坚强领导者。我们一直致力于技术领导地位，因此我们对与Covalent一起可以做什么感到非常兴奋。在过去的几年中，Covalent通过支持Apache项目（包括Tomcat和Apache HTTP）在市场上赢得了很高的声誉。它的数百个支持客户包括超过一半的《财富》 500强企业，以及家喻户晓的名字，例如辉瑞，强生，英国电信（BT），NASA，英特尔，苏格兰皇家银行和贝尔斯登。今天我们的公告开始了我们公司历史上的一个新阶段，并且-我相信-快速变化的应用程序平台市场。

开发人员，架构师和IT经理都喜欢Tomcat，并且在生产中越来越多地使用它 。他们也喜欢Spring，它是越来越多的关键任务应用程序的核心。SpringSource和Covalent在大型组织中一直在相互交流，因此联手创建了Spring和相关Apache产品和服务的事实提供者。在最近的Spring Experience展会上 ，与会的数百名开发人员，架构师和高级IT经理都对此进行了回应。百分之九十三（93）的Spring用户也在Tomcat上运行他们的应用程序，其中许多是关键任务。

SpringSource现在是支持Apache Tomcat，HTTP和其他Apache软件项目以及Spring产品组合的领先提供商。我们进一步加强了我们已经非常出色的工程和支持团队，并有能力为客户提供跨越他们使用的关键开源项目的打包支持产品。

在所有情况下，我们的支持都将由关键项目团队的核心提交者提供支持。 例如，考虑Apache HTTPD上的Ohloh排名 ，该排名显示SpringSource的Bill Rowe是多年来最活跃的提交者。Ohloh是用于对开源项目的活动进行排名的工具，在选择开源支持的提供者时这是一种方便的资源。您可以查看您是否有任何支持资金用于开发开源软件（从而帮助确保您所依赖的项目明天可靠），还可以判断是否为获得酬劳的员工解决了您的问题具有代码的专业知识和精通知识，以确保他们能够以最佳的方式快速执行此操作。

我们很高兴加深与Apache Software Foundation（ASF）的关系。SpringSource希望在为ASF项目做出贡献方面发挥重要作用，并继续延续Covalent重视，推广和赞助ASF的传统。我们将在许多项目，活动和计划中与Apache社区互动，并且我们将为客户提供Apache项目的高质量支持。

此次收购具有很好的战略意义，原因有很多，为什么SpringSource的现有员工及其来自Covalent的新同事都对此感到兴奋。当我们合并两个组织时，我们发现越来越多的协同作用，但是这里有一些关键点：


  • 互补产品组合 。随着Spring和Tomcat成为用于企业Java部署的最受欢迎的平台，将我们在Spring方面的领导地位与Covalent在Tomcat和其他Apache技术方面的深厚专业知识相结合，就具有了独特的吸引力。
  • 类似的文化和价值观 。SpringSource是一家由杰出的技术专家创立的公司，并且在其整个历史中雇用了越来越多的人才。Covalent聘用了杰出的工程师和支持人员，包括ASF主席和Apache HTTPD运行时间最长的活动提交者Jim Jagielski，Apache名人Bill Rowe以及领先的Tomcat开发人员Filip Hanik等。我们都为合作感到兴奋，并且我们可以互相学习。我很高兴有这些家伙加入我们的团队，并为他们的热情感到非常高兴。
  • 可靠的开源支持 。我们相信，只有在开源开发方面进行持续投资并在受支持项目中雇用关键提交者的公司才能提供可靠的开源支持。
  • 调整业务目标 。我们很高兴有Covalent首席执行官Mark Brewer和首席运营官Ryan Lindsay加入我们的执行团队。他们对通过多年的经验获得的开源业务有深刻的理解，并且可以帮助我们实现我们的共同愿景。
  • 能够提供最佳支持的能力。 随着支持对我们的业务变得越来越重要，获得Covalent的支持基础结构和专业知识将使我们能够提供Covalent众所周知的运行时，生产支持。反过来，我们的全球影响力将使为Covalent客户提供24x7全天候支持变得更加容易。
  • 对满足客户的共同承诺和成功的证明 。Covalent在其订阅客户中享有95％的续订率。
  • 开源业务部分与临界质量有关 ：围绕中心极点建立关系网络。SpringSource + Covalent现在是最重要的企业Java开源支持的领先提供商。

这对SpringSource客户意味着什么？除了Spring，我们现在还可以为Apache项目提供支持，包括Tomcat。一个电话重要项目的更广泛的技术组合。Spring产品组合将继续保持其在环境之间可移植性的出色记录。我们将继续支持使用所有中间件平台的客户，包括WebLogic和WebSphere。您很快就会看到有关这些平台的SpringSource产品计划的其他公告。

这对共价客户意味着什么？您所依赖的强大团队仍然存在，但现在得到了一个遍布全球的大型组织的支持。我们将继续提供Covalent的所有产品，但您还将受益于Spring产品组合的打包支持。Covalent工程师将继续积极参与众多开源项目，并为ASF的生命做出重要贡献。

在SpringSource，我们致力于成为开源企业Java的定义公司。我们正在建立长期成功，而不仅仅是短期目标。这意味着我们需要能够为客户提供成功所需的基础架构软件方面的最佳专业知识，并且随着开源对企业的重要性越来越高，我们需要与所有开源社区进行互动这个问题，从长远来看。

我们理解尊重社区开发和维持社区并产生优质技术的不同方式来开发开源软件的尊重。对我们的用户来说，最重要的三个是Spring Portfolio方法，ASF和Eclipse Foundation 。

SpringSource创建并维护了Spring产品组合，并且我们将越来越多地与其他两个社区进行互动，作为我们确保在客户需要的地方拥有深厚专业知识的策略的一部分。从企业Java的重要性来看，很明显，除了Spring产品组合之外，Apache项目至关重要，尤其是Tomcat，但还有其他一些项目。如果要创建高度可扩展的Web应用程序，请将Apache HTTP放在Java中间件的前面。在开发中，您最有可能使用Eclipse平台，也许还有Mylyn。 如果您对OSGi和OSGi项目的Spring动态模块 （似乎参加我们活动的每个人都感兴趣）感兴趣，那么您很可能会使用Eclipse Foundation的Equinox OSGi运行时 。从Spring 2.0开始，另一个Eclipse项目AspectJ在实践中一直是Spring Portfolio的核心。

我们正在为所有这些项目做出贡献。现在，我们聘用了参与多个ASF项目的关键ASF成员和提交者，包括在开发Tomcat和Apache HTTP中扮演着特别积极的角色。我们雇用了最活跃的AspectJ开发人员，以确保该项目在Eclipse的未来。通过与Tasktop Technologies的合作开发SpringSource Spring Tool Suite ，我们正在为Eclipse生态系统（尤其是Mylyn）贡献代码。最近被聘为杰出工程师Glyn Normington是Equinox的提交者。

随着Spring产品组合在去年的扩展，Spring已经不仅仅是Spring Framework。尽管Spring是我们业务，客户应用程序以及企业Java的未来的核心 ，但SpringSource作为一家公司，将成为开源企业Java成功的主要合作伙伴，而不仅仅是Spring –一个继续为我们骄傲的合作伙伴跟踪领导力和创新的记录，并为继续改变行业的开源项目提供深厚的专业知识和承诺。
