SpringSource旗帜下的第一个Grails版本

工程
格雷姆·罗彻（Graeme Rocher）
2008年11月14日

我很高兴宣布自SpringSource 收购G2One以来的第一个Grails版本。Grails 1.0.4包括许多改进以及对支持Grails的关键库的升级，可以从Grails下载页面下载 。更具体地说，Grails 1.0.4附带了大约一周前发布的最新Spring 2.5.6版本 。

除了这些改进之外，此版本中还有几个有趣的新功能。首先是添加了一项功能，该功能可以更好地支持GORM中Hibernate用户类型定义的映射。现在，您可以将自定义用户类型映射到多个列，包括自定义基础SQL类型的功能：


static mapping =  {
     amount type: MonetaryUserType, {
         column name: "value"
         column name: "currency", sqlType: "char", length: 3
     }
}

其他新功能包括通过在grails-app / conf / spring / resources.groovy中将PropertyEditorRegistrar实例指定为Spring bean即可定义用于数据绑定的自定义PropertyEditor实例的功能。Grails在底层使用了Spring的数据绑定和验证功能，因此其机制本质上与Spring MVC相同。

现在，我们将重点转移到Grails 1.1版本上，该版本旨在显着改善Grails中内置的单元测试功能，为Maven构建系统提供支持，为GSP中的JSP标签添加支持，以及对现有的关键产品进行增量改进。 GORM和插件系统等功能。

在Grails 1.1发布之后，由于对插件系统的改进，许多用于Grails的关键插件将成为可能，其中包括Java Content Repository（JCR）API，Java Persistence API（JPA）和Portlet API的插件。对于那些愿意活在最前沿的人，其中一些已经在插件存储库中处于alpha状态。

总的来说，我们对来年感到非常兴奋，并希望在短期内为您带来许多重要的版本，以及Groovy，Grails和关键Spring产品组合（如Spring Batch，Spring Integration和Spring）之间的集成带来的激动人心的可能性。 dm Server从长远来看。
