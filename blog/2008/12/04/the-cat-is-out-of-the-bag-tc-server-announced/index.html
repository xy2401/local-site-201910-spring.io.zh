这只猫出了大笔钱– tc服务器宣布

工程
彼得·库珀·埃利斯
2008年12月4日

我们刚刚在本周的SpringOne Americas大会上宣布了一款名为SpringSource tc Server的新产品。Springsource tc Server是基于Apache Tomcat的企业级Web应用程序服务器。

尽管SpringSource并不是第一家围绕Apache Tomcat构建产品的公司（WebSphere Community Edition和JBoss都在其J2EE应用服务器中嵌入了Tomcat版本，而JBoss Web 2.1.1的开发人员版本也将Tomcat嵌入了），但是tc Server在它保留了Tomcat Servlet / JSP编程模型。写入Tomcat的应用程序可100％移植到tc Server，反之亦然。

tc Server保留了开发人员关心的Tomcat的所有最佳方面，并添加了操作和支持团队所需的基本管理和诊断功能，以降低在关键任务生产中运行Tomcat托管的应用程序的风险和总体成本。就像Tomcat一样，tc Server轻巧，易于使用且速度很快。它具有约7兆字节的内存占用空间，并在3秒内冷启动。在Web应用程序的吞吐量和可伸缩性方面，它还可以与任何功能强大的Java EE应用程序服务器竞争。

如果今天使用Tomcat，您已经知道版本6在解决可靠性方面已经走了很长一段路。Tomcat V6中的群集已经得到了很大的改进。 tc Server对软件包进行了完善，并为企业Java Web应用程序部署提供了基本的“您需要什么而不需要什么”解决方案。

tc Server包括一个操作控制台，允许对服务器组进行定义和单步管理，它还与Tomcat紧密集成，以提供无与伦比的可见性和对分布式Tomcat服务器场的控制。

tc Server是我们多年将代码提交给Tomcat并在生产中支持Tomcat 的经验的产物。根据我们的经验，我们看到了两个关键的用例。

首先，我们了解到，尽管开发人员喜欢开源Apache Tomcat，但在生产中维护这些应用程序的成本常常包括隐性成本。管理大型Tomcat服务器场可能很昂贵。大多数组织最终都会滚动自己的脚本来进行配置和部署。由于现有的Tomcat具有有限的诊断功能，因此对测试和生产中的问题进行故障排除也可能既缓慢又昂贵。

其次，我们了解到，如今在仅使用Servlet API的完整Java EE应用服务器上部署了大量应用程序。大多数使用Spring容器而不是EJB容器的Spring应用程序都属于此类。通常，组织将在Tomcat上进行开发并在Java EE应用程序服务器上进行部署。他们之所以在Tomcat上进行开发，是因为Tomcat为他们提供了开发人员的生产力（再次是因为它既简单又快速）。之所以将它们部署在功能强大的Java EE应用程序服务器上，是因为运营团队受益于Java EE应用程序服务器随附的管理和诊断功能。

tc Server旨在解决这两个用例。一方面，升级到tc Server的Tomcat用户将受益于tc Server中的操作管理和高级诊断。另一方面，可以在tc Server上支持部署在商用Java EE应用程序服务器上的Web应用程序，而成本仅为Java EE服务器的一小部分。 tc Server将免费提供给所有开发人员，订阅生产部署的费用约为500美元/ CPU。

对于需要精简，功能强大且低成本的平台来部署Web应用程序的组织，tc Server应该非常有用。Spring / Tomcat现在是Java Web应用程序的领先堆栈 。因为部署在Tomcat或tc Server上的任何应用程序都可以无缝升级到SpringSource dm Server，所以随着公司转向模块化OSGi架构，这也提供了一条平滑的路线图。

我们将于2009年1月发布tc Server。试试看 ，让我们知道您的想法。
