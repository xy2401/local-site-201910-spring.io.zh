Spring Web Flow 2.0 M4发布

发布
基思·唐纳德
2008年3月11日

亲爱的春季社区，

我们很高兴地宣布，Spring Web Flow 2.0 M4现在可用。下载 | 文献资料

此版本巩固了Web Flow 2架构模型，包括SWF 2如何与Spring MVC，JavaServerFaces和Ajax集成。它还引入了许多新功能和改进，包括：

2.0 M4全新且值得关注

  • 引入了简化的XML流定义语法。  有关新语法的示例，请参见Spring Travel参考应用程序。
    • 使用版本2语法可将版本1流定义的大小减少多达50％。例如，Spring Travel 1.0.5由跨越六个工件的约200行流应用程序代码组成。最新的2.0 M4版本由跨越两个工件的93行代码组成，减少了50％，减少了四个维护文件。
    • 主要通过更强大的表达式语言（EL）集成和更简单的动作执行和数据映射标签来实现节省。
  • Spring Security集成 。提供了对确保流，状态和转换的全面支持。
    通过使用新的“ currentUser” EL变量，可以轻松地从流定义或视图模板引用经过身份验证的主体。
  • 对Spring MVC内部流异常处理的灵活支持 ，包括对自动重启已终止或已过期流的默认支持。
  • 支持处理不会更改当前页面的Ajax事件。  一个新的“ render”元素允许您在处理Ajax事件之后有选择地重新呈现页面的片段。
  • 查看变量。 视图变量在其包含的视图状态进入时分配，而在状态退出时超出范围。这些变量提供页面上下文，对于通过同一页面的一系列Ajax请求更新模型特别有用。
  • @Autowired流量变量 。流变量现在可以由Spring使用@Autowired依赖，从而使它们能够保存对Spring管理的@Services的引用。变量反序列化后，将在请求之间自动重新连接对服务的引用。
  • 支持弹出窗口 。使用popup = true标记一个视图状态，当客户端启用Javascript时，它将在模式弹出对话框中呈现。
  • Web Flow的JSF支持从称为“ Spring Javascript”的Javascript抽象层中分解出来 。当前，提供了基于Dojo和Ext的该层的实现。Spring.js提供：
    • Ajax的通用界面，无论使用哪种工具包
    • 一种类似于方面的API，用于用行为（包括客户端验证行为）装饰HTML DOM节点。
  • 一个小的JSF组件库 ，该组件库在下面使用Spring.js来逐步增强使用JSF的Spring Web应用程序。如果Javascript在客户端上不可用，则该库会降级。有关示例，请参见Spring Travel示例。关闭Java脚本，然后将应用程序与打开Java脚本时的应用程序进行比较。
  • 支持在标准Spring MVC环境中呈现JSF视图 。这使得Facelets模板可以由普通的Spring MVC控制器以及Flow呈现。

请参阅发行版中包含的Spring Travel参考应用程序，以获取所有这些功能的实际演示。参考项目可以作为动态Web项目直接导入Eclipse。

1.x兼容性

现有Web Flow用户的特别说明：即将发布的2.0 RC1将在版本2环境中提供对版本1流的支持。这将允许1.0.x流在同一应用程序中沿版本2的流不变地运行。

有关更多信息，请参阅完整的Web Flow 2 ChangeLog和RoadMap 。我们还鼓励您访问我们的支持论坛和JIRA系统，以向开发团队提供有关M4的反馈。

请享用！2.0决赛即将来临。
