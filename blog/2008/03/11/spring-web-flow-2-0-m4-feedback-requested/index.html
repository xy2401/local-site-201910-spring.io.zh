Spring Web Flow 2.0 M4-请求反馈

工程
基思·唐纳德
2008年3月11日

Web Flow团队一直在努力开发Web Flow 2。我们刚刚达到了第四个里程碑 ，最早将在下周进入发布候选状态。2.0最终版本计划于本月底发布。

从现在到2.0决赛，我们希望您的反馈！如果您是当前正在使用Web Flow 1.x的应用程序开发人员，或者正在评估要在项目中使用的Web Flow，请给2.0 M4进行评估，并告诉我们您的想法 。如果您是将Web Flow引擎集成到您的框架中的Web框架提供商，我们建议您评估2.0 M4中经过改进的挂钩，并向我们大喊大叫。

在接下来的几段中，我将为使用Web Flow的应用程序开发人员和将其集成到他们自己的Web框架中的框架开发人员提供2.0 M4重点：

应用程式开发人员

首先，到目前为止，评估最新Web Flow功能集的最佳方法是试用发行版中包含的参考应用程序。2.0 M4中有两个参考应用程序，都具有“ Spring Travel”旅行预订应用程序的风格。一种显示将Web Flow集成到具有JSP视图的Spring MVC中，另一种显示Web Flow集成到具有JSF视图的Spring MVC中。 （请参阅： 如何部署 ）

2.0版本的主要主题之一是易用性。Web Flow 2.0 M4引入了许多简化了流程实现的新功能，我们很高兴收到您的反馈。特别：


  • 我们引入了一种简化的XML流定义语法。将最新的Spring Travel参考应用程序用于测试驱动器以查看它，或者将示例流程之一的旧版本与最新版本进行比较 。你怎么看？大多数简化来自增强使用EL和更简单的标签来执行动作。例如，除了任何流变量之外，您现在可以使用EL在任何Spring Bean上执行任何方法。

  • 我们介绍了与Spring Security 2的集成。Spring Travel参考应用程序显示了如何使用“安全”元素保护流程的安全。您还可以保护状态和过渡元素。

  • 2.0 M4中有几个新的“丰富网络”功能。您可以将视图状态标记为“弹出”，并且其视图将为您呈现在弹出对话框中（默认为Dojo弹出窗口）。您还可以在过渡块内使用新的“ render”标签，以在处理Ajax事件之后请求页面的部分呈现。最后，尝试在Spring Travel应用程序中关闭Javascript，并注意该应用程序如何继续运行。我们一直在努力构建对降级的支持，这对不能依赖客户端上可用Javascript的客户而言非常重要。

  • 我们已大大减少了FlowExecution键的长度。1.x密钥格式的流执行URls看起来像这样： http：// localhost / booking？_flowExecutionKey = _c1E11178E-485F-3913-98BA-86E17A09FA3D_kE977CB0B-47FE-3319-A688-FE4F4FFFCE2 。
    默认的2.0格式为您提供了更简单的URL，例如： http：// localhost / booking？execution = c1v1 ，其中第一个数字是您的sessionId ，第二个数字是您的视图（或延续）ID。URL格式当然仍然可以配置。

  • 还有更多要结帐的地方；有关其他新功能的说明，请参见发行公告 。


框架开发人员

Web Flow的目标之一一直是“使其专注于真正做好一件事情（实现流程），并使其易于扩展”。Web Flow的核心扮演着提供最完整的流引擎的重担。在这个角色中，我们鼓励其他框架提供者在我们的工作基础上继续发展，并让他们知道集成是否应该像应该的那样容易。Grails社区已经做到了这一点，已经将Web Flow合并到Grails中以在Groovy中实现流（Grails基于Spring MVC构建，Web Flow是自然的MVC扩展，因此很合适）。

为了简化与框架开发人员的集成，我们在Web Flow中做了几件事。首先，基本上有三个主要扩展点：


  1. FlowExecutor SPI允许将Web Flow引擎嵌入到其他环境中，例如Web框架。例如，这就是Web Flow如何集成到Spring MVC中。我们将Spring MVC视为基于Java的权威MVC平台，但是，如果您不能将其用作基础，则可以使用该挂钩。

  2. FlowBuilder SPI允许您合并用于定义流的另一种语法。我们的XmlFlowBuilder附带了Web Flow发行版。Graeme Rocher为Grails开发了基于Groovy的流生成器。

  3. ViewFactory SPI允许您将视图框架集成到Web Flow中以在流中呈现视图。这就是Spring Faces如何为Web Flow提供JSF支持，以及剑桥大学的RSF团队如何将Web Flow与他们的框架集成在一起。我可以想到的使用此扩展点进行集成的其他候选产品包括Wicket，Flex以及GWT。

    4. 这些扩展点中的每一个都在版本2中得到了简化，并且ViewFactory扩展点是新的。2.0 M4还引入了新的“ AjaxHandler”扩展点，用于集成Ajax请求/响应处理，以及与Spring Javascript兼容的实现。我们目前还与IceFaces团队合作，以期在2.0最终版之前与他们的Ajax组件库集成。

    Web Flow 2普遍可用的版本指日可待。帮助我们使其易于使用和扩展！
