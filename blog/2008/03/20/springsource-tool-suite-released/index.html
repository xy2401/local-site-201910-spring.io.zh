SpringSource工具套件发布

工程
克里斯蒂安·杜普依斯（Christian Dupuis）
2008年3月20日

今年的EclipseCon是我们作为公司参加的第一次。我们有相当多的谈判，这都深受欢迎，我们公布了测试版程序的SpringSource工具套件 。总的来说，我们所有人都度过了非常愉快的时光，并且我们从Eclipse社区获得了很多有关Spring Portfolio中项目的反馈，尤其是我们在Web空间中所做的事情。我将确保在接下来的几周内将所有反馈转发给项目负责人。由于EclipseCon即将结束，所以我终于有时间坐下来写下我们在会议上介绍的内容。

我们为中亮相SpringSource工具套件（STS）的个人使用版使用BoF在周一和我花了很多时间讨论和邮寄给什么额外的价值它带给人们。在此博客中，我将更详细地概述SpringSource Tool Suite的功能。首先，我听说过很多次，人们认为SpringSource Tool Suite的发布意味着我们将停止改进和塑造Spring IDE 。请让我指出，事实并非如此。实际上，相反的事实是正确的，并且通过查看Spring IDE JIRA和Subversion存储库已经很明显了。实际上，自从我在一月份加入SpringSource以来，我能够增加在并行开发工具套件时花在Spring IDE上的时间。

SpringSource工具套件的目标是给你的最先进的开发工具 Eclipse平台做企业发展与Spring组合-听起来像营销俚语但这是使命陈述的工具团队致力。STS当然是在经过验证的Spring IDE的基础上构建的，但它与Eclipse Mylyn紧密相连，并且已经扩展了两个开源工具，以将企业级扩展引入IDE。为了尽力提供最好的Spring工具，我们已经开始并继续在Spring核心框架和其他Spring项目中添加与工具相关的功能，挂钩和扩展点。

因此，使用该工具套件，您最终可以获得Mylyn的任务集中型用户界面的所有好处，从而可以进行Spring开发。在处理Spring应用程序蓝图时，您可以进行上下文管理并集中精力。此外，我们将“以任务为中心”的UI方法提升到一个新的水平，并引入了一项新技术，即“以任务为中心的教程” 。以任务为中心的教程扩展了Eclipse Cheat Sheet框架，并在教程的每个步骤中添加了以任务为中心的内容。通过以任务为中心的教程，用户（Spring的新手或想要学习某些功能或Spring项目的用户）将能够单击导入一个正在运行的示例应用程序，并开始探索示例。然后，每个教程步骤将提供对教程某个方面的解释，并仅显示与理解和探究解释有关的Java和Spring元素。

从上面的屏幕截图中，您可以看到它在Eclipse中的外观。该教程将PetClinic示例应用程序带到Eclipse工作区中，并集中工作区以仅显示@Autowiredannotation以及如何将其应用于应用程序组件。在右侧，您可以看到展开的步骤，该步骤说明了如何以及为什么应使用@Autowiredannotation。在本教程的最后，用户可以在自动安装和配置的捆绑Tomcat服务器上启动示例应用程序。再次单击即可。

我们认为，这种引入新功能和产品的方法是一种非常强大的方法，并且使人们真正轻松地探索Spring产品组合非常容易。开发人员需要具备的所有知识，以及在入门时都知道的一切都正确地存在于IDE内部。当您下载Personal Use Edition Beta时，您将立即访问大约20个教程，这些教程都是由项目负责人或与客户直接合作的经验丰富的顾问创建的。最著名的Adrian Colyer花了一些时间来创建7个教程，其中介绍了OSGi概念，Equinox和Spring Dynamic Modules 。如果您想立即参加OSGi培训，那么这些教程真的很棒。该工具套件包含一个自动OSGi捆绑包更新/刷新机制，该机制最终允许在应用程序上工作并基于OSGi运行时的功能获得即时重新部署的语义（您是否曾经使用EJB尝试过；我做到了）。

我想谈的下一个功能称为运行时错误分析 ：SpringSource Tool Suite能够提供有关如何通过分析Java堆栈跟踪来解决运行时问题的信息。为此，我们创建了一个在线知识库，该知识库已集成到套件中，并且可以在Eclipse IDE中进行查询。知识库对使用该工具的所有人开放，我们甚至鼓励人们使用内置的丰富编辑功能来贡献自己的分析。看一下下面的截图。

我还没有涉及更多功能。您在此功能阵容中看到主旋律了吗？最后列出的三个功能为开发强大的基于Spring的应用程序提供了宝贵的帮助。在内部，我们开始为该功能集使用一个隐喻：盒子里的顾问。借助SpringSource Tool Suite，我们希望可以访问SpringSource知识，就像您旁边有位顾问一样！
