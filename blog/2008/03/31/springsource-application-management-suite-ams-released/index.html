SpringSource应用管理套件（AMS）发布

工程
珍妮弗·希基
2008年3月31日

自从SpringSource与Hyperic合作将我们的应用程序管理套件（AMS）产品推向市场以来，已经忙了几个月。我很高兴地宣布，SpringSource AMS beta版本现已向所有人开放。请花一点时间评估该软件并将您的想法发布到beta论坛 。我们致力于为Spring支持的应用程序提供最佳的应用程序管理体验，非常感谢您的反馈！

那些在12月的“春季体验”中对SpringSource AMS感兴趣的人收到了一封电子邮件，宣布该beta版本。这是该电子邮件的摘录，其中介绍了SpringSource AMS并概述了其一些其他功能：

SpringSource应用程序管理套件（AMS）是一个全面的企业应用程序管理工具。它旨在管理和监视所有Spring支持的应用程序，Spring运行时以及各种平台和应用程序服务器。SpringSource AMS建立在Hyperic的HQ Enterprise Edition之上，HQ Enterprise Edition是一种经过验证的系统和应用程序管理解决方案。SpringSource AMS的其他功能包括：-部署时自动发现Spring支持的应用程序-自动监视应用程序中运行的各种Spring，第三方和构造型组件。SpringSource AMS在Spring Framework 中提供了各种组件。只需将已检测的jar文件放到您的应用程序中，SpringSource AMS将开始监视您的应用程序并通过JMX公开统计信息。无需其他配置步骤！-自动监视Spring运行时的组件，包括应用程序上下文和bean工厂

在此博客中，我将更详细地说明这些功能，讨论其背后的技术，并向您展示如何使用SpringSource AMS监视您的应用程序。

SpringSource AMS的主要功能

应用程序自动发现
SpringSource AMS在部署时会自动发现您的Web和独立应用程序。它评估通过ApplicationContext创建的每个bean。如果它知道如何管理和监视该bean，则将bean导出为托管资源。此外，有一些部件（如DispatcherServlet的）创建一个ApplicationContext以外即使当，通过建议使用创建的SpringSource AMS发现的AspectJ的方面。可以在AMS产品文档中找到托管的Spring Framework组件的完整列表。

SpringSource AMS对您的应用程序及其包含的ManagedResources进行建模。每个ManagedResource都有一组固定的属性，控制操作和度量标准。这样便形成了一个简单，一致的清单模型，该模型描述了整个应用程序 。该模型将自动导出到JMX ModelMBeans。然后，任何JMX客户端（包括SpringSource AMS代理）都可以访问这些MBean，该代理从Spring驱动的应用程序中检索数据以显示在SpringSource AMS仪表板上。

SpringSource AMS仪表板当前显示部署在Tomcat，WebLogic和WebSphere服务器以及独立应用程序中的Spring托管资源。JBoss服务器的Spring托管资源显示计划在SpringSource AMS的最终1.0版本中进行。如果您使用其他服务器并向其部署单个应用程序，则可能仍可以通过在服务器启动时包含“ spring.managed.application.name”系统属性来在仪表板上查看服务，这将导致发现一个名为“ Spring Application”的服务器。如果您希望将来看到对其他服务器的支持，请在Beta论坛上告诉我们。

监控方式
SpringSource AMS使用在编译时编织的AspectJ方面来进行大多数操作监视，以建议所监视的方法。这给您的方法执行增加了非常小的开销（在大多数情况下，正是执行该方法之前和之后需要花费快速时间戳的时间）。对于吞吐量和平均执行时间之类的指标，跨给定方法执行的指标数据聚合是异步完成的。

SpringSource AMS还能够监视某些构造型组件-用@ Controller，@ Repository，@ Transactional，@ Component和@Service标记的组件。发现这些组件后，SpringSource AMS将它们包装在Spring AOP代理中（或将MethodInterceptor添加到现有代理中）。该代理用于收集组件的每种方法的执行时间（以毫秒为单位）。然后，可以通过导出到JMX的清单模型获得每种方法的平均执行时间。

入门

检测您的Spring-Powered应用程序
SpringSource AMS包含用于您的应用程序的Spring Framework 2.5.2的工具版本（spring-framework-instrumented-management.zip）。Spring Framework的组件通过结合代码钩子和AspectJ编译器的编译时织法进行管理管理。

要检测您的应用程序，只需用spring-framework-instrumented-management.zip中包含的检测版本替换现有的Spring Framework jar。您还需要将springsource-ams-instrumentation-agent.zip中的jar添加到您的类路径中。有关更多详细信息，请参阅AMS产品文档中的“使用AMS来管理您的应用程序”。

如果您在已经公开了JMX MBeanServer和远程JMX连接器的容器（例如WebLogic或WebSphere）中运行，那么您要做的就是！Tomcat用户可以按照此处概述的方式快速配置Tomcat进行监视。SpringSource AMS还可以与独立应用程序一起使用，可以通过添加Tomcat设置中建议的相同系统属性或使用Spring JMX类（例如MBeanServerFactoryBean和ConnectorServerFactoryBean）来启用JMX连接。

如果运行的是独立应用程序，则需要包含“ spring.managed.application.name”系统属性，以便SpringSource AMS知道您的应用程序名称（例如，-Dspring.managed.application.name = PetClinic ）。

使用SpringSource AMS仪表板
使用SpringSource AMS仪表板可完整了解应用程序以及运行它们的平台和容器的运行状况-适用于单个节点或整个集群！只需按照springsource-ams-installer发行版中随附的《安装指南》中的说明进行操作，即可启动并运行AMS服务器和代理。

AMS代理在启动时自动发现您的应用程序服务器或独立的Spring支持的应用程序。您应该在仪表板上的“自动发现” Portlet中看到已发现的服务器（可通过登录http：// server：7080进行访问） ，如下图所示：

选择“添加到库存”按钮以将服务器添加到库存。这是您要监视的每个服务器的一次性步骤。

将服务器添加到清单后，可以通过从导航菜单中选择“资源”->“浏览”来访问它。一旦为Tomcat，WebLogic，WebSphere或Spring Application Server配置了连接属性，SpringSource AMS应该开始发现您的应用程序服务。它们将显示在“服务”下的“资源”视图中，还将显示在服务器的“监视和库存”选项卡中，如下图所示：

服务名称将包含应用程序名称，以方便分组。要将服务分组到应用程序中以便于查看，请从导航菜单中选择“资源”->“浏览”。然后从“工具”菜单中选择“新应用程序”并创建一个新应用程序。

在应用程序视图中，选择“服务层”选项卡，然后单击“添加到列表”。然后，您应该看到服务列表。您可以根据应用程序名称过滤服务，并将它们添加到您的应用程序中，如下所示：

现在，您可以开始查看指标，定义警报，运行报告等等！请查阅测试版下载中随附的AMS文档手册，以获取有关所有AMS功能的全面指南。敬请关注更多博客文章，重点介绍使用AMS来防止应用程序停机，监视性能，执行服务级别协议等的一些方式。

下一步是什么？

SpringSource AMS还有更多的发展空间。将来会寻找更多Spring产品组合的仪器版本。Spring Source AMS的最终1.0发行版应提供Spring Security和Spring Web Flow的工具版本 。还要在应用程序模型中寻找更多的第三方组件，例如Apache ActiveMQ和Quartz。

我们需要您的反馈！您希望SpringSource AMS自动公开哪些管理数据？您是否希望在仪表板中看到特定的视图或报告？您是否希望产品具有特定功能？作为唯一专门针对Spring应用程序的应用程序管理解决方案，我们希望对开发人员和运营人员都使其尽可能强大和易于使用！
