Spring Dependency Injection和Java 5（包括幻灯片和代码）

工程
阿列夫·阿伦森（Alef Arendsen）
2008年3月18日

我正在写这本书，正要去开罗。我们仅在意大利西部飞行，我在意大利海岸线上看得清清楚楚，蓝色的海水和海浪轻轻地向岸移动。现在那边一定很好。我将前往开罗参加由艾哈迈德·哈辛（Ahmed Hashim）组织的埃及用户组会议，毫无疑问， 埃及用户组将做得很好，我敢肯定。这次我将在Spring上的主题是依赖注入，类型安全和Java 5。昨天（即3月14日），我在NL的Loenen的 Profict Wintercamp进行了几乎相同的演讲，我认为70个观众中有60个观众。

最近，关于类型安全，Spring和其他依赖项注入方法的讨论很多。很多时候，我发现人们指的是Spring具有依赖类型不安全甚至更糟的依赖注入特性，而人们指的是Spring依赖于XML。由于我永远无法一直望着飞机窗外（嗯，也许可以，但是那没什么用），所以我想写一篇关于Spring和Java 5依赖注入的当前状态的博客文章。

在Spring 2.5中增加了功能，并在Spring的子项目JavaConfig中添加了类型安全性参数以及将Spring与XML绑定的参数不再是事实。过去，我们一直说过 Spring没有与XML耦合，当然，表达配置细节的唯一可行选择是XML 。但是从Spring 2.5开始，这不再是理论上的争论。有一个实用的选项，用于使用纯Java配置依赖项。

Spring @Autowired支持

Spring 2.5本身提供了基于注释的依赖项注入，其中使用@Autowired注释和@Qualifier注释（或与此相关的任何其他自定义注释）向Spring容器提供注入依赖项的提示。我不会在这里介绍整个机制。相反，我将重点介绍涉及@Autowired方法的博客条目和文章：

  • Rod Johnson 介绍Spring Framework 2.5

  • Juergen Hoeller对Spring 2.5的全面注释支持

  • 在Spring 2.5中自定义注释配置和组件检测作者：Mark Fisher

    • Spring JavaConfig

    除了@Autowired支持之外，Spring JavaConfig还提供了一种全新的依赖项注入方法。很多博客条目已经在谈论JavaConfig，因此在此不再赘述。最近，我对JavaConfig的状态提出了一些疑问。一段时间以来，我们尚未发布该项目的任何里程碑。这当然不是没有道理的。尽管该模型已经可以很好地运行，但是我们仍然没有完善一些细节。我们想发布一些我们100％满意的东西，而现在，这简直不是
    做完了。密切关注JavaConfig项目页面和此博客。一个新的里程碑版本指日可待。

    JavaConfig方法在各种博客条目中也得到了更详细的介绍：

    • Rod Java的Spring Java配置前进
    • Rod Johnson的Spring Java配置选项
    • Costin Leau撰写的有关Java配置的更多信息
    • Guice与Spring JavaConfig： Craig Walls 的DI风格比较
    • 春季： Solomon Duskis的原型范围，查找方法和JavaConfig
    • IoC和模块化 -Solomon Duskis的Spring，OSGi，Guice等
    • 所罗门·杜斯基斯的《 Spring 配置注释》

    正如我在Loenen和开罗的会议上所说的那样，Spring JavaConfig尚未完成。尚需充实一些细节，以创建一种平滑的DI语言，以支持基于XML的DI语言也具有的所有功能。在这里（如果您愿意）可以为我们提供很多帮助。试试Spring JavaConfig并告诉我们您的想法！

    结论

    我在昨天的演讲中试图传达的另一个非常重要的观点是，Spring容器作为依赖项注入平台的思想，并在其之上实现了各种DI风格。第一种味道已经存在了5年（基于XML的方法）。第二个也已经存在了一段时间，现在它为BEA WebLogic版本10中的EJB3 SessionBean功能提供了基础（也可以在公共领域以Pitchfork的名称获得）。我们发布的最后一个风味是@Autowired风味，接下来是JavaConfig。拥有平台对我们而言至关重要。无论您使用的是哪种方式，它都能帮助我们为您带来全面向后兼容的良好体验
    JavaConfig，@ Autowired或我们基于XML的方法（或与此同时全部使用）。
