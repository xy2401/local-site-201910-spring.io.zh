Spring Security 2.0最终版本：不再有死神

工程
罗德·约翰逊
2008年4月17日

Spring Security 2.0已发布 。这是Spring产品组合向前迈出的重要一步。春季（Acegi）安全性已经是Java平台使用最广泛的企业安全框架，在SourceForge上的下载量超过25万，每个版本的下载量超过20,000。通过使它变得更加简单易用，此版本无疑将使采用达到一个新的水平。

我对此版本感到特别满意，原因有很多：


  • 对于Spring社区来说，这是一件很棒的事情。使用（很多）更简单，也更强大。它将最强大的企业Java安全解决方案提供给更多用户，几乎消除了采用的障碍。请参阅本教程 ，以获取有关如何简化保护典型Web应用程序的示例。XML bean定义的泛滥已经成为过去。

  • 这是Spring 2.x工作的延续，通过应用自定义XML名称空间的功能来实现积极的默认设置，同时仍允许自定义。

  • 像Spring 2.5一样，它展现了当前的Spring Portfolio趋势，即对XML需求的根本减少。
  • 这证明了SpringSource业务模型的价值。我们的收入模型使我们能够比以往任何时候都更多地投资于创建开源软件。如果不能同时雇用Acegi / Spring Security的创建者Ben Alex和其他主要提交者Luke Taylor，那么此发行版本要么不会发生，要么发行范围会广泛得多。

  • 对童话王国有好处 。

Acegi / Spring Security的创建者Ben Alex和Luke Taylor做得很好。Ben将在下个月的Java One上讨论Spring Security。如果您要去旧金山，那将是一个很好的机会来了解新功能并有机会与产品背后的人交谈。
