SpringSource工具套件中的运行时错误分析

工程
阿列夫·阿伦森（Alef Arendsen）
2008年4月14日

三周前，SpringSource工具套件发布了。Christian，负责该产品的博客已经对此进行了博客发布 ，我们还为希望快速了解其当前提供的所有功能的您提供了一个网络研讨会。在此条目中，我想特别强调运行时错误报告功能。

编程时，有时由于我引起的某些错误，控制台窗口会显示数十个堆栈跟踪。有时，我很幸运，并且堆栈跟踪看起来很熟悉。如果是这样，那么问题可能很容易解决。但是有时，您需要的信息被埋在所有这些堆栈跟踪中如此之深，以至于弄清楚真正的问题是需要一段时间的。

随着SpringSource Tool Suite的引入，我们开始建立一个在线知识库，其中包括针对您可能遇到的常见问题的解决方案。使用知识库，您不再需要浏览异常堆栈跟踪信息的页面和页面。而是单击一个简单的按钮将对知识库进行查询，如果找到可能的原因，您将自动获得相关信息，以帮助您解决特定问题。下面的快速截屏视频应该使您对功能有所了解。请注意，blip.tv的全屏功能运行良好，因此，如果某些字体在屏幕的某些部分变小，则使用全屏模式可以解决此问题。

我们刚刚开始填充知识库，并且随着工具套件的发布，它可能已经包含很多错误。但是显然，知识库无法为您可能遇到的每个问题提供解决方案。这就是为什么我们会不断自己添加有关潜在错误的信息， 而且还允许您使用建议的解决方案添加特定的错误条件 。编写工具可直接在IDE内部使用，因此您不必费心使用某种专有的XML格式。您（和其他开发人员）提交的所有内容将定期进行审核并作为知识库的更新发布。在下一个屏幕截图中，我将重点介绍此功能。

我对SpringSource Tool Suite的这一特殊功能感到非常兴奋，因为它将帮助我比过去更快地解决错误。如果您有兴趣查看SpringSource Tool Suite，请访问我们网站上的产品页面 ，当前，您可以在注册后下载Beta。在该页面上，我们还有一个较长的网络研讨会，其中Christian Dupuis会更详细地介绍SpringSource Tool Suite。
