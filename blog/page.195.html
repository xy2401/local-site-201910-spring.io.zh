Spring Security OAuth 2.0.4。现在发布

发布
戴夫·瑟
2014年11月11日

Spring Security OAuth 2.0.4。现在可以在通常的存储库中使用RELEASE。这是一个错误修复版本，因此建议升级，但还有一小部分新功能：

  • 现在可以显式查询OAuth2Request （以及OAuth2Authentication ）以找到关联令牌的授予类型。如果要刷新令牌，则在OAuth2Request提供给TokenEnhancer的授予类型是原始授予类型，而不是“ refresh_token”。

  • 客户权限在“ / check_token”端点中公开

  • 密码授予更加灵活并且可以扩展，因为客户端和服务器都可以向请求中添加其他参数。服务器端的自定义AuthenticationManager仍应使用UsernamePasswordAuthenticationToken ，但是其他参数将在AuthenticationDetails可用。例如，可以以这种方式实现针对移动设备的多因素认证。

  • Keystore支持JWT令牌签名和验证。用户提供资源和密码，然后可以按名称将密钥从商店中取出。只要它们是RSA密钥，就可以将它们注入到JwtAccessTokenConverter （使用新的setter）。

SpringOne2GX 2014重放：调整大型Java平台

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2014年11月9日

记录在SpringOne2GX 2014中。

演讲者：Emad Benjamin，Jamie O'Meara

幻灯片： http ： //www.slideshare.net/SpringCentral/tuning-large-scale-java-platforms

该会议将涵盖各种GC调优技术，尤其侧重于调优大规模JVM部署。参加本课程以了解GC调优配方，该配方可以为对延迟敏感的应用程序提供最佳配置。大多数情况下，大多数企业级Java工作负载都可以容纳小于4GB JVM堆的横向扩展JVM实例集，但内存数据库空间中的工作负载需要相当大的JVM。在本节中，我们将深入探讨问题以及优化4GB至128GB范围内大型JVM的优化调整配置。在本次会议中，我们分享了GC调优方法，这是对15年的GC参与的精炼，以及对近年来使用普通的HotSpot和CMS GC策略来调优行业中一些最大的JVM的适应。您应该能够自行完成不错的GC调整练习。该会议确实总结了完成此任务所需的技术和必要的JVM选项。自然，在调整大规模JVM平台时，不可忽略基础硬件调整，因此该会话将绕开传统的GC调整讨论，并深入探讨如何优化平台大小以增加内存消耗。最后，本次会议还将涵盖进行了全面性能研究的Pivotal Application Fabric参考架构。
