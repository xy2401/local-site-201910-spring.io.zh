2018年8月1日

    <artifactId>spring-cloud-stream-binder-kinesis</artifactId>

</dependency>

Spring Cloud Edgware EOL 2019年8月1日

新闻与活动
斯宾塞·吉布
七月30，2018

Spring Cloud Edgware将遵循Spring Boot 1.x EOL声明 。从2019年8月1日起，我们将在12个月后停止维护Edgware版本培训。

截止到那时，我们将继续发布偶尔的Edgware服务发布，然后结束发布过程。

提醒一下，卡姆登（Camden）发行火车已经到达停产期，达斯顿（Dalston）将在2018年12月底到达停产期。

现有的Spring Cloud用户应进行相应的计划，以确保在该日期之前已升级到最新的Finchley +版本。另外，请注意，Spring Cloud Finchley及更高版本需要Spring Boot 2.x，后者需要Java 8+。因此，如果您运行的是Java 6或7，则还应计划升级JDK。

Spring Cloud Data Flow 1.6 GA发布

发布
马克·波拉克
七月30，2018

Spring Cloud Data Flow团队很高兴宣布发布1.6.0 。遵循本地服务器 ， Cloud Foundry和Kubernetes的入门指南 。

1.6 GA的功能亮点

  • PCF上的任务计划

  • 仪表板即兴

  • Kubernetes支持增强

  • 应用托管工具

  • 组成的任务运行器安全

  • DSL和部署属性解析的改进

  • 批处理数据库架构和优化

PCF上的任务计划

我们很高兴在SCDF中为Cloud Foundry实施引入PCF Scheduler的本机集成！

批处理数据的典型工作流程涉及调度批处理应用程序。例如，调度程序系统接受cron表达式，并在表达式与当前时间匹配时启动应用程序。

数据流提供了计划和取消计划任务定义的功能。该计划基于cron表达式。在PCF Java Client的基础上，团队在Spring Cloud Scheduler SPI项目 （服务提供商接口）中创建了一个可移植的调度程序接口，并为PCF实施了Cloud Foundry的Spring Cloud Scheduler 。仪表板提供了计划和取消计划任务的访问权限，如下面的屏幕快照所示。
