dm Server 2.0.0发布

工程
安迪·威尔金森
2010年1月12日

正如Adrian所说，今天是dm Server 2.0的日子，我很高兴地宣布dm Server 2.0现已可用 。感谢您在2.0的开发过程中获得的所有反馈，它有助于将其塑造为我们认为对企业OSGi迈出的一大步。现在，我们在Eclipse.org上期待着旅程的下一步。

2.0版中有很多很棒的东西。以下是简要概述：

  • dm Shell-我们在2.0中添加了一个全新的Shell，它可以在过程中使用，也可以通过SSH远程使用。它为管理dm Server实例及其部署的工件提供了广泛的支持。
  • 管理控制台 -2.0版中对管理控制台进行了全面修订。除其他功能外，它还提供了对管理和部署工件，检查诊断转储以及在实时系统中以及在解决故障时检查束接线的支持。
  • 计划 -dm Server 2.0引入了对计划的支持，这些计划是1.0的PAR的强大替代方案，用于部署模块化应用程序。可以使用一个计划来引用构成您的应用程序的所有工件，可以选择使该应用程序具有原子性，以将子工件的生命周期联系在一起，并且范围可以将应用程序与部署在同一dm Server实例中的其他应用程序隔离。
  • 调配 -dm Server始终具有出色的调配支持，而在2.0中，它甚至变得更好。除了根据1.0中可用的已安装应用程序的需求自动配置捆绑包外，dm Server 2.0还支持本地和远程存储库中的配置计划，PAR和配置。
  • Web支持 -dm Server 2.0建立在1.0的Web支持的基础上，并嵌入了针对OSGi Web容器规范的基于Tomcat的参考实现，从而允许用户将所有依赖项的原始WAR文件部署在WEB-INF / lib和Web应用程序捆绑包中通过OSGi清单元数据导入其依赖项。可使用标准Tomcat server.xml配置Web容器。
  • 用户区域 -dm Server 2.0引入了一个用户区域，该区域将dm内核与用户安装的应用程序隔离开。除其他外，这简化了管理，因为它使用户可以专注于用户应用程序工件及其依赖性，而不必同时处理内核的工件。
  • Spring 3.0-与许多其他版本一样，我们一直在跟踪Spring 3.0的里程碑并发布候选版本，而dm Server 2.0现在将Spring 3.0最终版打包了。如果您希望将dm Server 2.0与其他版本的Spring结合使用，则可以轻松地对其进行配置。
  • 热部署 -除了支持通过管理控制台进行部署外，还可以通过将工件以归档或分解形式复制到dick Server中，将工件部署到dm Server。
  • 作为服务运行 -dm Server现在可以作为Windows服务或Unix后台进程运行。
  • 日志记录 -与1.0中一样，可以通过SLF4J获得dm Server的日志记录支持。在2.0中，后端已被LogBack取代， 可以通过使用大量可用的附加程序来修改dm Server的config / serviceability.xml文件来对其进行广泛配置 。
  • Equinox 3.5 -dm Server 2.0嵌入了Equinox 3.5，这是OSGi 4.2的参考实现。

Grails 1.2发布

工程
格雷姆·罗彻（Graeme Rocher）
2009年12月23日

继续发布，今天我们很高兴宣布Grails 1.2 final正式发布。Grails 1.2代表了迄今为止最稳定，性能最高的Grails版本，是JVM首要动态语言框架的重要新版本。

除了具有Spring 3的所有优点之外，此版本还为Grails用户提供了许多重要的新功能：


  • 依赖关系解析DSL ：Grails用户现在基于Ivy，可以完全控制JAR依赖关系，包括从框架继承的依赖关系和任何已安装的插件。

  • 更好的Spring集成 ：除了支持组件扫描之外，Grails现在还允许您将控制器实现为常规MVC @Controller实例。

  • 命名查询支持 ：现在可以在GORM中定义命名的，可重复使用的条件查询，该查询可以与常规动态查找器结合使用，从而使查询更加干燥

  • 改进的性能和内存消耗 ：Grails的视图层（GSP）的性能已得到显着提高，吞吐量提高了2-3倍。通过实施GSP视图的预编译，我们还改善了Grails的内存消耗以及对其他PermGen的需求。

  • 命名URL映射 ：现在可以命名一个单独的URL映射 ，这使您可以在GSP中创建更明确和更具表现力的链接

  • 重构测试基础架构 ：Grails的测试基础架构现在可以完全插入新的提供商。默认的JUnit提供程序仍然存在，但是可以实施可以在特定阶段（例如“单元”，“集成”和“功能”阶段）运行的新测试提供程序。已经有一个Spock插件可以利用这一新的基础结构，从而进行BDD样式测试。

  • 可插拔Web容器 ：Grails现在允许安装不同的开发时间容器，并且可以使用Tomcat和Jetty的插件。
