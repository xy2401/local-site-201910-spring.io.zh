今年春季-2014年12月30日

工程
乔什·朗（Josh Long）
2014年12月30日

欢迎来到春季本周的另一期！本周，按照我们的传统，我们将回顾一些让2014年成为Spring开发人员的美好一年的重要事情。而且，那是一年！

  • 去年年底看到了Spring 4 GA，而今年看到的是Spring 4.1发布，距离4.0之后仅短短9个月！Spring 4.x拥有一系列令人眼花new乱的新功能！Java 8支持，Java EE 7支持，新的Spring MVC视图，JSR 107支持，强大的资源管道和@Conditional注释，这当然使之成为可能...
  • Spring Boot的第一个1.0 GA发布于今年4月，此后经历了两次重大更新（ 1.1和1.2 ）。Spring Boot席卷了整个社区，引发了我们团队以及更重要的是其他所有人的各种活动！比赛试图淡化或复制它。现有的Spring用户蜂拥而至（包括Netflix和Ticketmaster之类）。但是，如果您遵循此每周综述，那么您就不需要进一步的证据来证明社区的普及！观看人们关于使用Spring Boot来做事的博客真是太有趣了……似乎一切！
  • Spring XD 1.0已于今年早些时候发布，其后续版本Spring XD 1.1即将完成。Spring XD（也是基于Spring Boot，BTW构建的！）通过在Spring Integration ， Spring Data和Spring Batch的基础上提供了一个数据处理工作台。它光滑，快速，优雅，当然人们很快就将其包含在军械库中 。
  • Spring Cloud于今年在德克萨斯州达拉斯举行的SpringOne2GX 2014上宣布。 Spring Cloud支持构建基于云的微服务风格的应用程序，并基于Spring Boot公开的编程模型构建。它集成了流行的库和基础架构，例如许多Netflix OSS堆栈和Consul 。
  • Cloud Foundry也度过了一年的欢乐时光！去年年底，我们开始看到Cloud Foundry v2的第一个细流，而今年我们看到了它的成熟，并且还看到了Cloud Foundry v2.1！下一代Cloud Foundry的令人兴奋的工作正在进行中。这是对快速运行的Go中所有组件的自下而上的重写，提供了可独立于较高的PaaS-y位使用的运行时（“格”）。当然，Cloud Foundry是Spring Boot和基于Spring Cloud的应用程序的理想目标。Pivotal Cloud Foundry是Cloud Foundry的紧缩包装，其发展迅猛。即将推出：磁贴以及对Spring XD和Spring Cloud的支持！而且，当然，我们在结束本节之前必须提到所有最令人振奋的事情： Cloud Foundry Foundation现在已全面运行 ！

Spring XD 1.1 M2和1.0.3发布

工程
马克·波拉克
2014年12月23日

我非常荣幸地代表Spring XD团队宣布Spring XD 1.1的第二个里程碑版本和1.0.3维护版本。

下载链接：
* 1.1.0。M2发布： zip ，1.0.3。发布： zip

1.1 M2版本包含错误修复和增强功能以及几个新功能：

  • 基于Kafka的消息总线 。如果Kafka是您的中间件，那么您会感到宾至如归。
  • 对RabbitMQ的批处理和压缩支持大大提高了有效消息速率的吞吐量。
  • 使用Project Reactor的 Stream API开发处理器模块。 Stream API基于Reactive Streams规范 ，该规范提供了具有无阻塞背压的异步流处理的标准。这使您可以使用功能性编程模型作为处理事件流的基础。结合XD 对消息总线上数据分区的支持，您可以创建功能非常强大的分布式流处理解决方案。
  • 轻松开发，测试和打包自定义模块 。父pom引导程序构建自定义模块所需的所有依赖关系，并将模块和依赖关系捆绑到单个uberjar文件中。还支持使用嵌入式单节点服务器对模块与JUnit进行集成测试 。
  • 通过HTTP将自定义模块上传到管理服务器。
  • REST端点的基于角色的授权使您可以控制“谁可以做什么”。
  • 基于文件的身份验证可以替代LDAP。
  • Sqoop批处理作业集成

Spring Cloud 1.0.0。RC1现在可用

发布
戴夫·瑟
2014年12月19日

Spring团队另一个愉快的假期信息： Spring Cloud 1.0.0。RC1现在可在http://repo.spring.io Maven存储库中获得。有很多新功能，包括

  • 通过注释支持Hystrix指标聚合@EnableTurbine和@EnableTurbineAmqp （对于基于AMQP的收集器）

  • 对Ribbon配置进行了重新调整，使其对Spring用户更加友好。现在，您可以配置每个功能区客户端ApplicationContext使用@RibbonClient并覆盖各个位，例如LoadBalancer ， 或者ServerListFilter ， 通过提供@Bean定义。

  • DiscoveryHealthIndicator现在是用户可以通过声明添加信息的组合@Beans类型的DiscoveryHealthIndicator 。

  • 现在，发现已从Eureka提取到新的spring-cloud-commons库中，并通过诸如@EnableDiscoveryClient （而不是旧的@EnableEurekaClient ）。相同的模式也适用于断路器
    和@EnableCircuitBreaker取代@EnableHystrix 。

  • Zuul代理的多项改进，包括Eureka目录更改时的自动更新，对表单编码的POST的支持，路由的外部配置以及每个客户端的身份验证方案。

  • 在Spring Cloud Security中以声明方式配置哪些路由需要OAuth2身份验证。

  • 支持Config Server的“本机”配置文件中的标签（如git分支）（在搜索位置的子目录中查找）。

  • 如果查找配置数据所需的URI无效，则“配置服务器和客户端”中的“快速失败”选项。

  • Spring Cloud Bus中对JSON消息的开箱即用支持。

  • 一个基于新的Feign配置的不错的框架@FeignClient注释（有点像Spring Data存储库）。

我们很高兴宣布Spring for Apache Hadoop 2.1的第三个里程碑版本。该版本现已在Spring IO存储库中提供

我们一直在努力改进YARN和数据存储编写器。@Configuration还进行了新的更改，并对Boot自动配置功能进行了改进。有关更多详细信息，请参阅完整的变更日志 。
