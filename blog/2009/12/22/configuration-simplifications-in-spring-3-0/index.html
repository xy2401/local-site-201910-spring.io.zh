Spring 3.0中的配置简化

工程
克里斯·比姆斯
2009年12月22日

在昨天由Keith发表的有关“ Spring 3 Simplifications”的系列文章中的第二篇，我想对Spring的新@Configuration注释和相关支持进行非常简短的动手介绍。

正如跟随Spring JavaConfig项目的人们所知道的那样，带有@Configuration注释的类的作用与Spring XML文件几乎相同。它提供了一种以代码为中心的方式声明Spring bean定义，仅使用方法和注释。您可能将其称为“ 普通旧配置” * :)这意味着在简单情况下，不需要XML！

让我们开始吧。为了演示@Configuration功能，我在新的spring-samples SVN存储库中创建了一个非常简单的项目 。您可能需要立即进行同步并进行构建。您将需要Subversion客户端和最新版本的Maven。


svn co https://src.springframework.org/svn/spring-samples/configuration-basic/trunk configuration-basic
cd configuration-basic
mvn clean test
[…]
Tests run: 1, Failures: 0, Errors: 0, Skipped: 0

[INFO] ————————————————————————
[INFO] BUILD SUCCESSFUL
[INFO] ————————————————————————

结帐中还包括Eclipse .classpath和.project元数据，因此您可以使用SpringSource Tool Suite或安装了m2eclipse插件的任何最新版本的Eclipse导入项目。无论哪种情况，都可以使用文件->导入->现有项目进入工作区。

首先，让我们看一下AppConfig.java 。带有@Configuration注释的类与基于XML的Spring应用程序中的app-config.xml担当相同的角色。在审查项目时，这是一个很好的起点，因为它是对象实例在运行时如何连接和管理的“蓝图”。


@Configuration
public class AppConfig {

    public @Bean TransferService transferService() {
        return new TransferServiceImpl(accountRepository());
    }

    public @Bean AccountRepository accountRepository() {
        return new InMemoryAccountRepository();
    }

}

当然，这是一个琐碎的应用程序示例，甚至没有真正的JDBC DataSource。没关系，本文的目的只是传达基本概念。上面的@Bean方法将在适当的时候被Spring容器识别并调用，返回的对象将像其他任何bean一样在Spring容器中进行管理。您可以看到，bean之间的依赖关系可以简单地表示为从一个bean方法到另一个bean方法的调用。TransferServiceImpl需要一个AccountRepository构造函数参数，因此只需调用accountRepository（）方法即可。

精明的Spring用户将研究这种情况，并问“关于bean范围界定的问题”？这是一个好问题。您可能知道，所有Spring Bean都有一个作用域。默认情况下，bean的作用域是'singleton'的作用域，这意味着每个Spring容器将只有一个bean实例。看一下上面的代码，似乎如果我们多次调用accountRepository（），我们实际上将创建多个实例，但是实际上并非如此！在运行时处理@Configuration类时，将动态子类化（使用CGLIB），并增强@Bean方法的子类实现以确保遵循范围定义语义。

如您所见，定义@Bean方法非常简单。现在，让我们引导容器并使用这些对象。

看一下TransferServiceTest JUnit系统测试及其transfer100Dollars（）@Test方法。您会注意到的第一件事是AnnotationConfigApplicationContext的使用。Spring 3中添加了此新的ApplicationContext实现，以支持直接使用@Configuration类实例化Spring容器。使用AppConfig.class作为构造函数参数创建上下文，然后使用getBean（Class ）方法按类型检索TransferService和AccountRepository Bean。该方法的其余部分只是行使TransferService和AccountRepository API的标准JUnit断言，以确保一切正常。


public class TransferServiceTest {

    @Test
    public void transfer100Dollars() {
        // create the spring container using the AppConfig @Configuration class
        ApplicationContext ctx = new AnnotationConfigApplicationContext(AppConfig.class);

        // retrieve the beans we'll use during testing
        AccountRepository accountRepository = ctx.getBean(AccountRepository.class);
        TransferService transferService = ctx.getBean(TransferService.class);

        // create accounts to test against
        accountRepository.add(new Account("A123", 1000.00));
        accountRepository.add(new Account("C456", 0.00));

        // check account balances before transfer
        assertThat(accountRepository.findById("A123").getBalance(), equalTo(1000.00));
        assertThat(accountRepository.findById("C456").getBalance(), equalTo(0.00));

        // perform transfer
        transferService.transfer(100.00, "A123", "C456");

        // check account balances after transfer
        assertThat(accountRepository.findById("A123").getBalance(), equalTo(900.00));
        assertThat(accountRepository.findById("C456").getBalance(), equalTo(100.00));
    }

}

而已！简单的纯Java类型安全配置。我们希望您会发现它是对Spring核心依赖注入支持的便捷而强大的补充。

当然，今天我们才刚刚开始。@Configuration类可以完成更多工作，我们将在以后的文章中探讨这些功能。但是，请不要等我-您现在可以阅读Spring 3参考文档的@Configuration部分，自行检查所有内容 。我鼓励每个人都使用此示例项目作为起点，从中您可以快速路测此新支持的其余部分 。

期待您的反馈。享受@Configuration和Spring 3的所有新功能，祝您假期愉快！
