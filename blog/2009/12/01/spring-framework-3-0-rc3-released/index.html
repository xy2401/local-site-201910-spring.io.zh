Spring Framework 3.0 RC3发布

工程
于尔根·霍勒（Juergen Hoeller）
2009年12月1日

我们决定在发布GA之前再发布一个Spring 3.0候选版本：从下载页面获取它，进行全面的测试，然后让我们知道它如何为您工作。Spring 3.0现在正在等待您的集成测试反馈 ，最终将在12月中旬发布。

此版本的候选者具有多项增强功能：例如，新名称空间中的扩展功能 ，以及对启动/关闭行为 （影响消息侦听器和计划任务）的进一步修订 。随意尝试这些功能！我们也渴望了解现有Spring 2.5应用程序的升级体验，因为我们希望您的许多应用程序有选择地采用3.0功能……同时暂时将大多数代码保持其2.5形状。

与第三方框架和库的兼容性也是一个重要的目标。您现有的大多数库都应该继续工作，甚至不需要升级。但是，在某些情况下，我们在所需版本方面提高了标准：例如， Spring 3.0现在需要Hibernate 3.2或更高版本，并且已经明确支持Hibernate 3.3和Hibernate 3.5 beta。 再举一个例子，Spring 3.0现在需要Tiles 2.1，现在不再支持Tiles 2.0。我们通常建议使用此类第三方库的最新生产版本，但与Hibernate一样，我们将继续支持仍然使用的较旧版本。

有时，由于以前对此有所困惑：Spring 3.0代码库现在完全基于Java SE 5（JDK 1.5）和Java 5语言功能，但是与此同时， Spring 3.0与J2EE 1.4服务器完全兼容。以及Java EE 5服务器，并且已经为Java EE 6提供了早期支持。 特别是，您可以在Tomcat 5.5和WebSphere 6.1之类的操作系统上运行基于Spring 3.0的应用程序，并在已建立的J2EE 1.4生成平台（幸运的是，其在下面的JDK 1.5上运行）上提供完整的Spring 3.0功能集。您甚至可以在该组合中添加一个全新的JPA 2.0提供程序...充分利用您所获得的东西。

最后，基于Spring 3.0，尤其是基于该候选版本，我们获得了一系列完整的项目版本：例如， Grails，ROO，dm Server，Spring Security，Spring Batch和Spring Integration的新主要版本。 关注此空间！
