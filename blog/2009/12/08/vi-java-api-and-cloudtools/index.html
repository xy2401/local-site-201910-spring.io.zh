VI Java API和CloudTools

工程
查尔斯·李
2009年12月8日

虚拟基础架构Java API或vSphere API的创建者史蒂夫·金（Steve Jin）最近将他为VMworld 2009主题演讲所做的工作贡献给了CloudTools存储库。CloudTools是一个开源项目，为Cloud Foundry提供了核心基础架构配置和应用程序部署功能。Steve的代码不仅演示了如何通过Cloud Foundry用户界面将Java应用程序部署到vSphere基础架构，还展示了如何轻松地向CloudTools添加适配器以将Java应用程序部署到不同的云提供商。以下是Steve在其VMware Infrastructure（vSphere）Java API Blog上发表的博客文章的重印，以宣布其贡献。


VI Java API和CloudTools使DIY PaaS成为可能

2009年11月23日


如前所述，VIWorld API在VMWorld 2009 Keynote演示中得到了利用。现在，我获得了法律批准，并为Google托管的CloudTools代码贡献了相关的适配器。


CloudTools / CloudFoudry最初是为EC2设计的。CloudTools是开源的； CloudFoudry不是。借助我们提供的代码，您可以在vSphere中运行CloudTools，以将基于Java（Groovy）的Web应用程序部署到内部云中。它提供了Maven和Grails插件，因此您可以使用一行命令完成所有部署。更好的是，您可以将plugin命令与Spring Tools Suite（STS）集成在一起，并在基于Eclispe的IDE中拥有一个上下文菜单。这就是我所谓的DIY PaaS（自己做平台即服务） ：vSphere + VI Java API +适配器+ CloudTools。


vCloud适配器是使用Terremark vCloudExpress平台设计的，用于SpringOne 2GX主题演示。该适配器不使用VI Java API，而是利用vCloud REST API。除了基本部分之外，Terremark vCloud API还提供了用于管理网络的扩展，例如公共IP，InternetService和节点。


尽管您看到两个不同的适配器，但是用户体验是相同的。两个适配器都实现了CloudTools定义的必需接口。从技术上讲，这不是什么大问题，但是从商业角度来说，这是大问题–您可以无缝部署到私有（vSphere）云和公共（服务提供商，如Terremark）云，无论哪种最适合您的需求。


有关更多详细信息，请查看CloudTools项目主页 。


史蒂夫·金
