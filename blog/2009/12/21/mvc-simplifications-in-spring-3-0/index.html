Spring 3.0中的MVC简化

工程
基思·唐纳德
2009年12月21日

正如Juergen和Arjen所提到的，各地的Java开发人员都可以通过Spring 3.0顺利升级。既然Spring 3已经发布，我想带您了解一些您可能不知道的MVC新功能。我希望您发现这些功能有用，并且可以立即将其用于Web应用程序中。

这也是“春季3简化”系列文章的开始，因此，预计在未来几天和几周内会有更多此类文章。

简化配置

Spring 3引入了一个mvc命名空间，该命名空间大大简化了Spring MVC的设置。除了其他增强功能之外，启动和运行Spring Web应用程序从未如此简单。这可以通过mvc-basic示例进行说明，现在我将向您介绍。

mvc-basic旨在说明Spring MVC的基本功能。该项目在我们的spring-samples SVN存储库中可用，可以用Maven构建，并且可以导入Eclipse。从web.xml开始审阅，并注意那里的配置。值得注意的是，DispatcherServlet配置有单个主Spring配置文件，该文件初始化所有其他应用程序组件。DispatcherServlet被配置为应用程序的默认servlet（映射为“ /”），允许使用纯净的，基于REST的URL。

在主servlet-context.xml内部，您将找到典型的设置。在第一行，已打开组件扫描以从类路径中发现应用程序组件。在下一行，您将找到第一个正在使用的Spring MVC 3新功能：


<!-- Configures the @Controller programming model -->
<mvc:annotation-driven />

此标记注册将请求分派到@Controllers所需的HandlerMapping和HandlerAdapter。另外，它根据您的类路径中存在的内容应用合理的默认值。这些默认值包括：


  • 使用Spring 3 Type ConversionService作为JavaBeans PropertyEditors的更简单，更强大的替代品
  • 支持使用@NumberFormat格式化数字字段
  • 支持使用@DateTimeFormat格式化日期，日历和乔达时间字段（如果乔达时间位于类路径上）
  • 如果类路径上有JSR-303提供程序，则支持使用@Valid验证@Controller输入
  • 如果类路径上包含JAXB，则支持读写XML
  • 如果杰克逊在类路径中，则支持读取和写入JSON

太酷了吧？

继续，下一行演示了另一个新功能：


<!-- Forwards requests to the "/" resource to the "welcome" view -->
<mvc:view-controller path="/" view-name="welcome" />

在后台，mvc：view-controller注册了一个ParameterizableViewController，它选择要渲染的视图。在这种情况下，当请求“ /”时，将显示欢迎视图。实际的视图模板是在/ WEB-INF / views目录中解析的.jsp文件。

继续，下一行显示了另一个新功能：


<!-- Configures Handler Interceptors -->	
<mvc:interceptors>
    <!-- Changes the locale when a 'locale' request parameter is sent; e.g. /?locale=de -->
    <bean class="org.springframework.web.servlet.i18n.LocaleChangeInterceptor" />
</mvc:interceptors>

mvc：interceptors标记允许您注册HandlerInterceptors以应用于所有控制器。以前，要执行此操作，必须向每个 HandlerMapping bean显式注册此类拦截器，这是重复的。还要注意，此标记现在使您可以限制某些拦截器应用于哪些URL路径。

继续，下一行突出显示了版本3.0.4中添加的新功能：


<!-- Handles GET requests for /resources/** by efficiently serving static content in the ${webappRoot}/resources dir -->
<mvc:resources mapping="/resources/**" location="/resources/" />

mvc：resources标记可让您为静态资源（例如css和javascript文件）配置处理程序。在这种情况下，对/ resources / **的请求将映射到/ resources目录内的文件。

数据绑定简化

我将说明的下一组新功能与@Controller绑定和验证有关。当我在几周前写博客时 ，这方面有很多新内容。

在示例中，如果激活@Controller Example链接，则应呈现以下形式：

从那里，如果您更改区域设置，您应该会看到国际化的字段格式。例如，从en到de切换将导致续订日期12/21/10的格式为21.12.10。此行为和表单的验证规则由模型注释驱动：


public class Account {
	
	@NotNull
	@Size(min=1, max=25)
	private String name;
	
	@NotNull
	@NumberFormat(style=Style.CURRENCY)
	private BigDecimal balance = new BigDecimal("1000");
	
	@NotNull
	@NumberFormat(style=Style.PERCENT)
	private BigDecimal equityAllocation = new BigDecimal(".60");

	@DateTimeFormat(style="S-")
	@Future
	private Date renewalDate = new Date(new Date().getTime() + 31536000000L);

}

表单提交由以下AccountController方法处理：


@RequestMapping(method=RequestMethod.POST)
public String create(@Valid Account account, BindingResult result) {
    if (result.hasErrors()) {
        return "account/createForm";
    }
    this.accounts.put(account.assignId(), account);
    return "redirect:/account/" + account.getId();
}

在绑定和验证之后调用此方法，其中@Valid批注触发对帐户输入的验证。如果存在任何验证错误，则将重新渲染createForm，否则将保存帐户，并将用户重定向；例如http：// localhost：8080 / mvc-basic / account / 1 。

为了说明另一个很酷的新功能，请尝试请求一个不存在的帐户。例如/ account / 99。
