使用Maven获取Spring 3工件

工程
基思·唐纳德
2009年12月2日

最近有一位评论员对此表示不满：“使用Maven的只是世界的一半”，并指出如何使用Maven获得Spring 3工件并不明显。在此条目中，我将向您展示如何执行此操作以及选项。此信息还将集成到即将发布的Spring 3最终版本的参考文档中。

发布了春季工件的Maven存储库

通常，Spring将其工件发布到两个不同的地方：


  1. Maven Central ，它是Maven查询的默认存储库，不需要任何特殊配置即可使用
  2. 企业捆绑存储库 （EBR），由SpringSource运行，还托管与Spring集成的所有库

因此，在使用Maven获取Spring时，需要决定的第一件事就是从哪里获得Spring。通常， 如果您关心OSGi ，请使用EBR，因为它为所有Spring依赖项（例如Hibernate和Freemarker）都提供了OSGi兼容工件。如果OSGi对您而言并不重要，那么这两个地方都可以工作，尽管它们之间有一些优缺点。通常，为您的项目选择一个地方或另一个地方；不要混合它们。这一点特别重要，因为EBR工件使用与Maven Central工件不同的命名约定。

从企业捆绑存储库（EBR）获取Spring版本

要从EBR获得Spring项目的最终版本，请在您的.pom中添加以下存储库：



    com.springsource.repository.bundles.release
    EBR Spring Release Repository
    http:// repository.springsource.com/maven/bundles/release


    com.springsource.repository.bundles.external
    EBR External Release Repository
    http:// repository.springsource.com/maven/bundles/external

获取春季里程碑版本

里程碑和候选发布者可能不会直接发布到Maven Central，并且通常与最终版本分开发布。SpringSource托管两个用于获取Spring里程碑的存储库。第一个应与Maven Central结合使用，第二个应与EBR结合使用。

从Maven中央兼容存储库中获取里程碑

要从Maven Central兼容存储库中获取Spring 里程碑 ，请将以下存储库添加到您的.pom中：



    org.springframework.maven.milestone
    Maven Central Compatible Spring Milestone Repository
    http:// maven.springframework.org/milestone

里程碑版本号格式为。 。 。M＃;例如3.0.0。M4。发行候选版本号格式为。 。 。RC＃;例如3.0.0。RC3。

例如，添加以下依赖项将检索版本3.0.0。弹簧上下文工件的RC3：



  org.springframework
  spring-context
  3.0.0.RC3

从企业捆绑存储库（EBR）获取里程碑

要从EBR获得Spring 里程碑 ，请将以下存储库添加到您的.pom中：



    com.springsource.repository.bundles.milestone
    EBR Spring Milestone Repository
    http:// repository.springsource.com/maven/bundles/milestone

确保记住不同的EBR工件命名约定。例如，添加以下依赖项将检索版本3.0.0。org.springframework.context工件的RC3：



  org.springframework
  org.springframework.context
  3.0.0.RC3

获取每晚春季快照

Spring项目的快照每天晚上发布，使用户可以在下一个版本之前验证报告的问题是否已解决。像里程碑一样，有一个单独的Maven Central兼容快照存储库和一个EBR快照存储库。

从Maven Central兼容存储库中获取快照

要从与Maven Central兼容的存储库中获取Spring 夜间快照 ，请将以下存储库添加到您的.pom中：



    org.springframework.maven.snapshot
    Maven Central Compatible Spring Snapshot Repository
    http:// maven.springframework.org/snapshot

快照版本格式为。 。 。建立快照;例如3.0.1。建立快照。

例如，添加以下依赖关系将检索spring-context工件的最新快照：



  org.springframework
  spring-context
  3.0.1.BUILD-SNAPSHOT

请注意。 。 。BUILD-SNAPSHOT格式与的传统Maven 2快照格式略有不同。 。 -快照这是因为xyz-SNAPSHOT不是有效的OSGi版本号。现在，所有Spring项目都遵循OSGi版本编号方案 （Maven 3也是如此）。

从企业捆绑存储库（EBR）获取快照

要从EBR获取Spring 夜间快照 ，请将以下存储库添加到您的.pom中：



    com.springsource.repository.bundles.snapshot
    EBR Spring Snapshot Repository
    http:// repository.springsource.com/maven/bundles/snapshot

作为最后一个示例，添加以下依赖关系将检索org.springframework.context工件的最新快照：



  org.springframework
  org.springframework.context
  3.0.1.BUILD-SNAPSHOT

春季项目生产力工具

最后，我想简要介绍一下Spring为使用Maven的项目提供的工具。SpringSource Tool Suite和Spring Roo都提供了可以使用预配置的.poms生成新Spring项目的向导。Roo在这方面已走得很远–在执行需要下载其他工件的代码生成命令时，它实际上可以为您管理.pom。

Cloud Foundry还具有一项新功能，该功能无需外部依赖就可以进行云部署，从而大大减少了部署时间。为了使此工作有效，Cloud Foundry在发布完成后与EBR同步以完成部署。
