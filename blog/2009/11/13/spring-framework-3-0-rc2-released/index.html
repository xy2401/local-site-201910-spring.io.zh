JSR-330, a.k.a. “Dependency Injection for Java”, basically standardizes an @Inject annotation with a qualifier model. The “javax.inject” annotations can now be used as alternative to Spring’s own @Autowired and @Qualifier annotations. The newly revised version of our formatting package also comes with support for the popular Joda Time library.

  • Revised lifecycle management at runtime: Auto-starting message listener containers and schedulers will now kick off once context configuration has been fully completed. On context close, a specific shutdown order can be defined for ‘phased’ stopping of lifecycle components. A configurable LifecycleProcessor delegate provides corresponding customization hooks.

Spring Framework 3.0 RC2发布

工程
于尔根·霍勒（Juergen Hoeller）
2009年11月13日

我很高兴地宣布，我们今天发布了第二个Spring 3.0候选版本（ 下载页面 ）。此版本在几个方面对RC1进行了重要改进，特别是：

  • Spring 3.0 RC2完全兼容JSR-330，并通过了TCK的最终版本。 JSR-330，又称“Java依赖注入”，基本上是使用限定符模型标准化@Inject注释。现在，可以将“ javax.inject”注释用作Spring自己的@Autowired和@Qualifier注释的替代方法。Spring的元素也会自动激活JSR-330处理（如果存在“ javax.inject” API）。

  • 专用的AnnotationConfigApplicationContext：在不涉及任何XML的情况下，使程序化引导尽可能方便。支持带注释的类的显式注册以及类路径中的组件扫描。这不仅适用于Spring的@Component模型，而且还适用于@Configuration类（也称为“JavaConfig”）和符合JSR-330的类。

  • 一个新的mvc配置名称空间： 激活@Controller类的丰富处理，包括对注释驱动的格式和注释驱动的验证的支持（如果存在JSR-303“ Bean验证”提供程序，例如Hibernate Validator 4.0）。我们格式包的最新修订版还支持流行的Joda Time库。

  • 修订的运行时生命周期管理：上下文配置完全完成后，自动启动消息侦听器容器和调度程序将立即启动。在上下文关闭时，可以为生命周期组件的“阶段性”停止定义特定的关闭顺序。可配置的LifecycleProcessor委托提供相应的自定义挂钩。

最后，RC2在细节方面进行了许多改进：例如，对URI编码进行了修订，以对请求URL的任何部分中的特殊字符进行适当处理；父子关系中改进的范围处理，以继承bean的范围；支持在JBoss 5.x上进行加载时编织；等等

Spring 3.0 GA即将来临...

PS：特别感谢“扩展的” Spring开发团队中的所有人员（包括SpringSource内部和外部），他们使此发行成为可能！你摇滚:-)
