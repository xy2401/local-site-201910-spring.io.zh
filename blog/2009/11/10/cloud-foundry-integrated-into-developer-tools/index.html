将Cloud Foundry集成到开发人员工具中

工程
查尔斯·李
2009年11月10日

感谢其他SpringSource项目团队的努力，我们很高兴将三个开发人员工具与Cloud Foundry集成在一起，即SpringSource Tool Suite，Grails和Roo。这样的结果是，您可以创建和开发Spring或Grails应用程序，并在云中运行它们，而不必离开自己喜欢的IDE或命令行界面。这些工具利用Cloud Foundry客户端Web服务API（即将发布），并利用云中的其他智能和SpringSource工件存储库，以将Java Web应用程序优化，高效地上传到Cloud Foundry。这意味着您仅需几分钟即可从“代码到云”。

对于那些今年参加了新奥尔良SpringOne 2GX的人来说，现在是使用我们提供的免费试用帐户来Cloud Foundry进行体验的绝佳时机。

作为开发人员，如果最终将您的应用程序部署在Cloud Foundry或Amazon EC2上进行生产，则在开发过程中使用Cloud Foundry可以为您提供一种快速，便捷的方式来验证与生产环境完全相同的环境中的代码更改。 。但是，即使您不使用EC2进行生产，在工具箱中使用Cloud Foundry仍然有很多好处。


  • 在沙箱环境中立即访问Web堆栈，而无需经历在开发机器上设置和配置Web服务器，应用程序服务器和数据库的麻烦

  • 轻松配置多实例环境，以便您可以在开发的早期阶段暴露基础架构的问题

  • 功能/负载测试

  • 允许他人访问您正在开发的应用程序


鉴于所有这些好处，与以下开发工具的集成使Cloud Foundry可以无缝，有效地整合到您的开发流程中。

SpringSource工具套件（STS）


从SpringSourceTool Suite版本2.2.0开始，您将把Cloud Foundry服务作为一种新的Server类型集成到IDE中。这支持将Web应用程序项目拖放到Cloud Foundry服务器上，并支持在Cloud Foundry中部署和控制应用程序。使用内置的浏览器，您可以查看应用程序和Cloud Foundry Web用户界面，而无需离开IDE。查看此截屏视频以查看实际效果，并下载STS以开始使用。感谢Christian Dupuis和STS团队做出了出色的整合努力。

ils子


Cloud Foundry Grails插件是在本周发布的，我们已经从用户那里听到了他们的经验。Graeme Rocher创作了该插件，并通过Subversion提供了源代码：
svn co https://src.springsource.org/svn/cloudfoundry_plugins/grails/cloud-foundry

通过最少的设置，使用几个简单的命令即可轻松使用Grails命令行部署到Cloud Foundry。查看此截屏视频以查看实际效果，并转到插件页面以获取有关如何安装和使用插件的详细信息。

o


最后但并非最不重要的是新的Roo插件，它需要1.0.0。RC3或更高版本。在SpringOne 2GX上， 本·亚历克斯 （ Ben Alex ）在Roo举行的会议是出席人数最多的会议之一，并且仅在会议室举行。我们欣喜若狂，Ben看到了Cloud Foundry带来的生产率提高，并使其成为Roo的第一个第三方附加组件。转至此文章以获取有关如何在Roo中安装附加组件的说明，并使用制表符完成功能查找“ cloud Foundry”的全部功能。也可以通过subversion获得源代码：

svn co https://src.springsource.org/svn/cloudfoundry_plugins/roo/com.cloudfoundry.roo.addon

我们将汇总Roo / Cloud Foundry的截屏视频，并将很快在www.cloudfoundry.com上提供 。

准备好Cloud Foundry


除了这些项目之外，我们还引入了Cloud Foundry Ready计划。下图将出现在网站或应用程序的用户界面上，只要工具或服务与Cloud Foundry集成在一起即可增强用户体验。
