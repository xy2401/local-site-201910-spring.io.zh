介绍Spring Batch Admin

工程
戴夫·瑟
2009年11月10日

Spring Batch Admin提供了一个基于Web的用户界面，具有用于Spring Batch应用程序和系统的管理控制台。这是SpringSource的新开源项目。里程碑版本1.0.0。M1即将推出，具有以下所有功能，我们希望在2010年初发布1.0.0最终版本。

主要用例

快速概览Spring Batch Admin的最简单方法是查看一些主要用例的屏幕截图。用户界面是一个Web应用程序（使用Spring MVC构建）。

检查工作


用户可以检查系统已知的作业。作业是可启动的或不可启动的（在屏幕快照中，它们都是可启动的）。区别在于，可启动作业是在应用程序本身中定义和配置的，而不可启动的作业被检测为在另一个进程中执行作业所留下的状态。（Spring Batch使用关系数据库来跟踪作业和步骤的状态，因此可以查询历史执行以显示不可启动的作业。）

启动工作


可启动作业可以使用名称为名称/值对的作业参数从用户界面启动，也可以通过配置到应用程序中的增量器启动。

检查执行


一旦作业正在执行或已经执行，该视图就可以用来查看最新的执行，以及它们的状态的简要摘要（STARTED，COMPLETED，FAILED等）。


每个单独的执行都有一个更详细的视图（如上所示），用户可以从那里单击以查看作业中每个步骤执行的视图（在这种情况下只有一个）。想要执行此操作的常见原因是查看失败的原因。


步骤执行详细信息视图的顶部显示了所有作业执行中该步骤执行的历史记录。这对于获得性能特征的统计感觉很有用。在集成测试环境中运行作业的开发人员可能会使用此处的统计信息来比较作业的不同参数，以查看在项目处理步骤中更改（例如）提交间隔的效果。


步骤执行视图的底部具有该步骤的详细元数据（状态，读取计数，写入计数，提交计数等），以及从任何导致该步骤失败的异常中提取的堆栈跟踪（如上例所示）。

停止执行


用户可以停止正在执行的作业（无论该作业是否可启动）。停止信号通过数据库发送，并且在任何正在运行该作业的进程中被Spring Batch检测到后，该作业即被停止（状态从STOPPING移至STOPPED），并且不再进行任何处理。

从哪里获得

最好的起点是SpringSource社区下载页面 。本文还附有快照下载，或者您可以从subversion获取源代码并自己进行编译。快照构建也每晚都通过S3到达Maven存储库：


	spring-snapshots
	Spring Maven Snapshot Repository
	http://s3.amazonaws.com/maven.springframework.org/snapshot

有两个JAR工件和一个WAR示例（org.springframework.batch：spring-batch-admin-sample：war），它们都在Maven存储库和SVN中。对于本文中的WAR下载链接，JAR文件包含在WEB-INF / lib中。

要部署示例应用程序，只需将WAR安装到您喜欢的servlet容器中，或将源代码导入STS并使用标准Eclipse WTP功能启动示例（例如，将其拖到服务器上或以“运行方式”运行在服务器上）。它开箱即用地使用了内存数据库，因此您将在每次重新启动应用程序时获得一个全新的副本。
