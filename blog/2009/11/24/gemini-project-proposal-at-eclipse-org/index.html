Eclipse.org上的Gemini项目提案

工程
阿德里安·科耶尔（Adrian Colyer）
2009年11月24日

本周， Gemini项目提案在Eclipse.org上揭晓。Gemini是Eclipse RT（运行时）项目的一部分，该项目“旨在在Equinox框架和OSGi的基础上促进，促进和容纳Eclipse中的运行时工作”（ Eclipse RT任务说明 ）。Gemini本身是一个总括项目，它将为企业应用程序开发设计的模块集合在一起。SpringSource和Oracle是提供该项目提交者的前两家公司，其他几家公司也表示有兴趣加入。

为Gemini贡献的最初项目代表了我们在OSGi联盟企业专家小组中共同努力的成果。既然该小组的工作正在加快步伐，我们希望在某个地方将规范组件的集体参考实现汇集在一起，以使您更容易找到所有相关部分。此外，Gemini还为您提供了一种通过参与项目而对企业OSGi的发展产生影响的方法。SpringSource将贡献两个初步项目：


  • Gemini Web容器 -OSGi Web容器（RFC 66）参考实现，当前位于SpringSource git存储库中。Rob Harrop 在此博客的上一篇文章中描述了Web容器RI 。我们在SpringSource dm Server中使用了Web容器，但是经过精心设计，可以用作独立实体。迁移到Eclipse.org上自己的项目将使您更加轻松地将基于标准的Web应用程序部署添加到OSGi服务平台。

  • 双子座蓝图服务 -这很重要！那些一直关注OSGi世界发展的人会知道，自从我们在大约4年前启动Spring Dynamic Modules项目 （当时称为Spring OSGi）以来，它已经发展成为OSGi服务平台上企业应用程序开发的非常流行的基础。 。通过OSGi联盟企业专家组，我们致力于创建基于Spring动态模块编程模型的标准，该标准作为“蓝图服务”作为OSGi R4.2汇编规范的一部分发布。Spring Dynamic Modules v2是Blueprint Service规范的参考实现。我们仍在研究细节，但是Spring Dynamic Modules v2代码库将作为Gemini Blueprint Service项目移至Eclipse.org，在该项目中它将与其他企业项目一起继续开发并跟踪Blueprint的发展未来OSGi联盟更新中的规范。


这两个项目都将在EPL和Apache许可下获得双重许可。

除了SpringSource提供的两个参考实现之外，Oracle将为企业专家组当前正在开发的以下RFC提供参考实现项目：


  • RFC 98-交易

  • RFC 122-数据库访问

  • RFC 139-JMX集成

  • RFC 142-JNDI集成

  • RFC 143-JPA集成

  • RFC 146-JCA连接器集成


这是许多企业OSGi参考实现的一个地方，并且当然与OSGi服务平台本身的参考实现Equinox并置在一起。

Gemini社区论坛现已开放，您将在此处找到Mike Keith发布的常见问题解答 ，其中涵盖了您可能遇到的一些最常见问题。
