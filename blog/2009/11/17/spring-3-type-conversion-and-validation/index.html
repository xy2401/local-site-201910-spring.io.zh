Spring 3类型转换和验证

工程
基思·唐纳德
2009年11月17日

Spring 3最终版本指日可待 ，它将是一个不错的版本。在此博客文章中，我将带您了解Spring 3的一些类型转换和验证增强功能。无论您是开发传统的Web应用程序，桌面应用程序还是“下一代” RIA，数据绑定，类型转换和验证都是重要的领域。正如您将在此条目中看到的那样，Spring 3在这些区域的每个方面都进行了重大升级，同时保留了与先前版本的向后兼容性。

新系统目标

在介绍功能之前，我首先要强调一下我们着手改善Spring 3的数据绑定系统时所达到的目标：


  1. 提供一个取代JavaBean PropertyEditor的无状态，强类型的Type Converter SPI

  2. 提供统一的Type Conversion API，以在需要进行转换的任何地方使用，包括Spring的DataBinder和Expression Language
  3. 允许类型转换由Java注释元数据驱动

  4. 通过注册合理的默认值并应用约定优于配置来简化


随着春季3的最后阶段临近，我相信我们已经实现了每个目标。继续阅读，您将对此做出判断。

特征

充分利用新类型转换系统的第一个环境是Spring MVC，我将借鉴它来演示新功能。这从新的Spring MVC 3配置名称空间开始：


<beans xmlns="http://www.springframework.org/schema/beans"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xmlns:mvc="http://www.springframework.org/schema/mvc"
        xsi:schemaLocation="
            http://www.springframework.org/schema/beans
            http://www.springframework.org/schema/beans/spring-beans-3.0.xsd
            http://www.springframework.org/schema/mvc
            http://www.springframework.org/schema/mvc/spring-mvc-3.0.xsd">

    <mvc:annotation-driven />

</beans>

前面的最小配置使Spring自动安装用于对Number和Date字段进行本地化的默认类型转换器，包括对流行的Joda Time库的完全支持（如果类路径中存在的话）。另外，如果您的类路径中存在JSR-303提供程序，例如Hibernate Validator，Spring会自动启用注释驱动的声明式验证。

现在，当您绑定到一个字段时，将使用明智的默认格式打印并解析该字段以适应用户的语言环境。为了说明，请考虑以下模型：


public class Account {

    private Date activationDate = new Date(1258466400);

    private BigDecimal balance = new BigDecimal("3000.25");
}

…打印在美国和DE语言环境用户的表格上：

用注释覆盖默认值

通常，您需要以不同于默认格式的方式来格式化字段。在上一个示例中，您可能希望使用短日期格式来设置ActivationDate字段，并使用货币格式来设置余额字段。在Spring的早期版本中，您将在Controller中注册一个自定义的PropertyEditor以实现此目的。在春季3中，您只需注释字段：


private class Account {

    @DateTimeFormat(style="S-")
    private Date activationDate = new Date(1258466400);

    @NumberFormat(style=Style.CURRENCY)
    private BigDecimal balance = new BigDecimal("3000.25");
}

使用注释替代，将为美国和DE语言环境打印以下内容：

启用日期：平衡：
启用日期：平衡：

参数注释

注释驱动的替代也可以应用于方法参数。考虑下面的Controller方法，该方法获取特定日期的即将到来的约会，其中day是以ISO日期格式编码的URL路径变量：


    @RequestMapping(value = "/appointments/{day}", method = RequestMethod.GET)
    public String getAppointmentsForDay(@PathVariable @DateTimeFormat(iso=ISO.DATE) Date day) {
        ...
    }

向/ appointments / 2009-11-17发送GET请求会在2009年11月17日获取所有约会。将@DateTimeFormat iso属性设置为ISO.DATE指示Spring将传入的日期字符串解析为ISO日期（yyyy-mm-dd）。

验证方式

在将用户输入绑定到模型之后，通常需要对其进行验证。Spring 3支持使用JSR-303进行声明式验证。如果您的类路径中存在JSR-303提供程序（例如Hibernate Validator），则会自动启用此支持。启用后，您可以简单地通过使用@Valid注释对Controller方法参数进行注释来触发验证：


    @RequestMapping(value = "/appointments", method = RequestMethod.POST)
    public String add(@Valid AppointmentForm form, BindingResult result) {
        ....
    }

    static class AppointmentForm {

        @NotNull @Future
        private Date date;
    }

绑定传入的POST参数后，将验证AppointmentForm；在这种情况下，要验证日期字段值不为null并在将来发生。

约定优于配置

最后，考虑如何将约定优于配置的原理应用于类型转换。在业务应用程序中，您通常会定义自己的自定义字段类型。在Spring的早期版本中，要格式化此类类型，您将创建一个自定义的PropertyEditor实现，并将其注册到Controller中。在大多数情况下，使用Spring 3，您可以遵循以下约定：


  1. 定义一个静态valueOf（String）方法或Constructor（String）以从其String表示形式解析您的值
  2. 实现toString（）以打印您的值以供显示

考虑遵循以下约定的SocialSecurityNumber类型：


    public class SocialSecurityNumber {

        @Size(9)
        @Mask("###-##-####")
        private String value;

        public SocialSecurityNumber(String value) {
            this.value = value;
        }

        public String toString() {
            return this.value;
        }
    }

当显示显示SocialSecurityNumber字段时，将调用toString（）；解析客户值时，将调用构造函数。不需要单独的Formatter或PropertyEditor实现。

摘要

此项涵盖了一些新的Spring 3类型转换和验证功能。要了解更多信息，包括如何实现自己的类型转换器，请查阅Spring 3 Reference Guide 。另外，请注意未来几周修订后的Petclinic 3示例应用程序的发布（此示例应用程序演示了此处突出显示的所有功能，目前可在此处的SVN存储库中进行早期访问）。
