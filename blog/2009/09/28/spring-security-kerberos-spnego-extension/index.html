Spring Security Kerberos / SPNEGO扩展

工程
迈克·威斯纳
2009年9月28日

我们高兴地宣布，Spring Security Kerberos Extension的第一个里程碑现已可供下载 。也可以通过Maven里程碑资料库（ http://maven.springframework.org/milestone）获得该版本。使用Spring Security Kerberos Extension，仅通过打开URL即可针对您的Web应用程序对用户进行身份验证。无需输入用户名/密码，也无需安装其他软件。

在深入探讨Kerberos之前，我想介绍Spring Security Extensions ，这是一个新的Spring Extension项目，专用于为核心Spring Security项目提供扩展模块。当前，我们在那里开发了两个扩展：SAML2集成和Kerberos / SPNEGO集成。每个模块都有自己的发布周期，因此人们可以在准备好这些扩展后立即受益，而不必等待下一个Spring Security版本。如果您有任何想法或什至需要进一步扩展的代码，请告诉我们！

Kerberos / SPNEGO

在本模块的第一个里程碑中，我们为您提供了针对Web应用程序的现成的Kerberos / SPNEGO解决方案。Kerberos是一种标准化的网络身份验证协议，旨在为客户端/服务器应用程序提供强身份验证，例如以浏览器为客户端的Web应用程序。这也是在Windows网络中验证用户身份的推荐方法，它取代了过时且相对不安全的NTLM。除此之外，它还广泛用于* NIX环境，并且每个主要平台都有实现。因此，很可能您已经安装了Kerberos，现在您也可以在自己的Web应用程序中使用它。这意味着您的用户只需输入URL，便会自动使用其域用户名进行身份验证，例如mikewiesner@SPRINGSOURCE.COM 。然后，您可以通过Spring Security或什至与request.getRemoteUser（）一起找到该用户名。这是如何运作的？这里是一个简短的概述：

浏览器将GET请求发送到您的Web应用程序（1），然后返回该请求，要求进行“协商”身份验证（2）。然后，浏览器将要求Kerberos服务器获取所谓的服务票证（3）。然后，浏览器将该服务票证发送给Web应用程序，以证明呼叫者的身份以及一些其他内容（5）。验证票证之后，根据Web应用程序和Kerberos服务器之间的某些共享机密，您将获得用户名。

为此，每个Web应用程序都需要在Kerberos服务器上注册，并获得服务主要权限和分配的共享机密。对于Web应用程序，服务主体必须为“ HTTP / @DOMAIN”。例如，如果您的应用程序在web.springsource.com上运行，则为“ HTTP / web .springsource.com @ SPRINGSOURCE.COM” 。然后，您需要将此主体的凭据导出到keytab文件（共享机密），并将其提供给您的应用程序。每个基于Kerberos的系统都将以这种方式工作，但是在两个系统之间，此服务主体和密钥表的创建是不同的。我将向您展示如何使用Microsoft Windows和MIT Kerberos进行此操作，但它也应与其他实现一起使用。

使用Microsoft Windows 2008 Server创建服务主体

尽管这是指Microsoft Windows 2008 Server，但在2003年甚至2000 Server中，它应该非常相似。在ActiveDirectory中，您只需创建一个普通的域用户，然后为其分配服务主体（SPN），然后使用命令行实用程序创建密钥表。现在一步一步来：

创建一个普通用户，它将成为服务主体。用户名和密码对Kerberos没有意义，但是您当然应该选择一个有用的名称，例如http-web.springsource.com。只需确保取消激活“用户必须在下次登录时更改密码”选项并激活“密码永不过期”选项即可。

之后，您必须使用命令行工具“ ktpass.exe”。它已经包含在Windows 2008 Server中，在早期版本中，您必须自己安装它。只需确保您使用的版本与服务器版本相匹配，并且语言环境也应匹配。该工具将服务主体名称（SPN）分配给您先前创建的用户，并将用户密钥导出到密钥表文件。如果您的服务主体是“ HTTP / web .springsource.com @ SPRINGSOURCE.COM ”，而您的用户是http-web.springsource.com，则您的ktpass命令应如下所示：


ktpass /out http-web.keytab /mapuser [email protected] /princ HTTP/[email protected]  /pass *

ktpass将提示您输入一些密码。您应该为此选择一些安全的随机数。如果目录中现在有一个文件http-web.keytab，那么一切正常。该文件稍后在您的应用程序中需要，因为它包含用于验证服务票证的共享密钥。

使用MIT Kerberos创建服务主体

在* NIX系统上以及在Mac OS X中，MIT Kerberos实现被广泛使用。使用MIT Kerberos，它甚至更简单。只需打开kadmin控制台并执行以下命令：


kadmin:  addprinc -randkey HTTP/web.springsource.com
kadmin:  ktadd -k /http-web.keytab HTTP/web.springsource.com

然后，您应该在根目录下有一个文件http-web.keytab。以后在您的应用程序中需要此文件，因为它包含用于验证服务票证的共享密钥。

配置Spring Security

首先，要求：


  • Spring Security 3.0.0 M2

  • SUN JRE / JDK 1.6.x

  • Kerberos环境

  • 支持SPNEGO的浏览器（Firefox，IE，Safari）

为了在Spring Security中使用Kerberos模块，您只需要声明一个过滤器，一个认证入口点和一个认证提供者。我们提供了一个示例Web应用程序，您可以将其用作起点。您只需要配置服务主体名称并将生成密钥表放在此处即可。示例应用程序包含在上述下载中。

如果打开示例应用程序的security.xml文件（位于/ src / main / webapp / WEB-INF下），则会看到使用新Kerberos模块的基本Spring Security配置。


<sec:http entry-point-ref="spnegoEntryPoint">
	<sec:intercept-url pattern="/secure/**" access="IS_AUTHENTICATED_FULLY" />
	<sec:custom-filter ref="spnegoAuthenticationProcessingFilter" position="BASIC_PROCESSING_FILTER" />
</sec:http>

<bean id="spnegoEntryPoint" class="org.springframework.security.extensions.kerberos.web.SpnegoEntryPoint" />

<bean id="spnegoAuthenticationProcessingFilter" class="org.springframework.security.extensions.kerberos.web.SpnegoAuthenticationProcessingFilter">
	<property name="authenticationManager" ref="authenticationManager" />
</bean>

<sec:authentication-manager alias="authenticationManager">
	<sec:authentication-provider ref="kerberosServiceAuthenticationProvider" />
</sec:authentication-manager>

<bean id="kerberosServiceAuthenticationProvider" class="org.springframework.security.extensions.kerberos.KerberosServiceAuthenticationProvider">
	<property name="ticketValidator">
		<bean class="org.springframework.security.extensions.kerberos.SunJaasKerberosTicketValidator">
			<property name="servicePrincipal" value="HTTP/web.springsource.com" />
			<property name="keyTabLocation" value="classpath:http-web.keytab" />
		</bean>
	</property>
	<property name="userDetailsService" ref="dummyUserDetailsService" />
</bean>

<!-- Just returns the User authenticated by Kerberos and gives him the ROLE_USER -->
<bean id="dummyUserDetailsService" class="org.springframework.security.extensions.kerberos.sample.DummyUserDetailsService"/>

前两个bean（SpnegoEntryPoint和SpnegoAuthenticationProcessingFilter）负责握手，然后KerberosServiceAuthenticationProvider最终验证服务票证。当前，我们仅支持SUN的JRE / JDK中包含的Kerberos / SPNEGO实现。由于仅从Kerberos取回用户名，因此还需要一个UserDetailsService来获取角色以及其他一些用户属性。在此示例中，我们仅使用虚拟实现来简化测试。

如您所见，我们已经填写了服务的主要名称和密钥表位置。根据需要更改这些值，并确保在此位置下可以使用以前生成的密钥表。

现在启动您的服务器，尝试尝试SPNEGO身份验证。您应该在浏览器中看到完整的域用户名。在您的代码中，您可以使用常规Spring Security类甚至标准Java servlet调用request.getRemoteUser（）来检索用户名。如果它不起作用（也许您会看到一个空白页），请检查以下内容：


  • 检查日志文件

  • 确保在URL中使用完整的合格域名（不是IP地址，也不是简称）。

  • 如果您使用的是Internet Explorer：打开“ Windows集成身份验证”，并确保在IE的本地Intranet网站部分中列出了域（在我们的示例中为web.springsource.com）。

  • 如果您使用的是Firefox：请在此处查看 。

  • 如果使用Windows客户端：客户端和服务器必须位于不同的计算机上，否则Windows将使用NTLM而不是Kerberos。

  • 检查所有相关机器上的时间是否同步。

  • 如果您使用的是Microsoft AD，则可以在此处找到更多帮助： http://msdn.microsoft.com/en-us/library/ms995329.aspx

      • 除此之外，设置适当的Kerberos环境可能会很复杂，因此在开始使用Spring Security Kerberos扩展之前，请务必正确设置此环境。我们在咨询过程中遇到的大多数问题是Kerberos环境的问题，而不是应用程序本身的问题。

      如果您不想在自己的Maven项目中使用Spring Security Kerberos Extension，则必须将Spring Milestone存储库添加到pom.xml中。它看起来应该像这样：


      <repositories>
	<repository>
		<id>spring-milestone</id>
		<name>Spring Portfolio Milestone Repository</name>
		<url>http://maven.springframework.org/milestone </url>
	</repository>
</repositories>

      当然是依赖项：


      <dependency>
	<groupId>org.springframework.security.extensions</groupId>
	<artifactId>spring-security-kerberos-core</artifactId>
	<version>1.0.0.M1</version>
</dependency>

      还有许多工作要做，例如，还为Java客户端（不仅为服务器）提供Kerberos，但我们希望您可以尝试此里程碑版本并提供一些反馈。社区论坛是提出问题或开始讨论新功能的最佳场所。另外，如果发现不对劲，可以提出《 吉拉问题》 。
