SpringSource TC Server-合理的下一步

工程
吉姆·贾吉尔斯基
2009年4月28日

轻量级AND企业级Java应用服务器的时机已经成熟，而Apache Tomcat是首选。现在，借助SpringSource tc Server ，我们在SpringSource使其成为现实。

如果您熟悉Covalent，现在又熟悉SpringSource，那么您很可能了解ERS（ 企业就绪服务器 ）。ERS是我们对Apache httpd Web服务器和Apache Tomcat进行的预先构建，预打包和完全质量检查的发行版。发行版中还包括一些非常有用的增强功能，这些功能以Apache的模块形式出现，例如PHP，mod_perl和mod_snmp。该发行版有些不幸的方面是，除了mod_jk之外，Tomcat方面增加的额外位非常少。

tc Server完全更改了此设置。

而且我们都已经看到Tomcat在许多环境中取代了更多的“传统”（和重量级）JEE服务器，但是总会有一小群人即使它们不需要EJB功能（而对于Spring，为什么有人需要它）无论如何），这些服务器的“企业级功能”使它们变得非常有用，而Tomcat中缺少这些功能使得很难从它们迁移到Tomcat。

tc Server也会完全更改此设置。

tc Server是SpringSource的Tomcat新软件发行版，它添加了人们想要和需要的所有缺失的企业功能。该功能集令人印象深刻：


  • 应用管理

    • 列出在服务器实例的分布式集合中运行的应用程序

    • 将应用程序定向，部署和取消部署到分布式服务器实例

    • 启动，停止和重新加载为分布式服务器实例运行的应用程序

    • 控制Web应用程序参数，例如缓存，JSP行为和提供静态内容



  • 服务器配置和管理

    • 服务器实例的远程配置控制：

    • 配置JDBC数据源和连接池

    • 定义虚拟主机，访问日志以及与Web服务器的集成

    • 配置JVM服务器启动参数，例如Java堆大小和垃圾收集特征

    • 定义服务器组（tc服务器或Tomcat实例）



  • 高级服务器诊断

    • 当线程竞争受限资源时，应用程序线程锁定检测会以警告方式危害应用程序完整性，从而发出警告

    • 可配置的自动和按需线程和堆转储，用于故障和异常分析

    • 与URL关联的线程可在分析请求处理问题时更快地进行诊断




上面包含的功能是可以从命令行脚本环境执行各种管理任务，而不是GUI，GUI非常适合自动化任务和各种cronjob。当然，您还可以获得整个软件包的SpringSource Enterprise支持和服务。

tc Server的独特之处在于，它为您提供了您了解，喜爱和信任的Tomcat，但具有所需的企业功能。当然，我们不仅止步于此。tc Server中还有一个新的高并发连接池，它击败了Tomcat的传统DBCP和其他外部解决方案。并且，根据我们对开放源代码社区的长期承诺，我们将其捐赠回ASF 。

我们已经有很多计划要在下一个主要版本中添加其他诊断功能。我们还致力于下一代ERS，它将ERS和Apache httpd的这些企业功能添加到Web层。

从Web前端到中间件业务逻辑，SpringSource都会为您服务。因此，无论您是使用Apache httpd，Tomcat和Spring的老手，还是进行合理的（可能是长期拖延的）迁移，请仔细研究我们能提供的所有帮助。
