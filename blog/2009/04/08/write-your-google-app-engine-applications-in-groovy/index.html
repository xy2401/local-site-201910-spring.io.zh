用Groovy编写您的Google App Engine应用程序

工程
纪尧姆·拉福格（Guillaume Laforge）
2009年4月8日

[caption id =” attachment_1577” align =” alignright” width =“ 250” caption =” Google App Engine Groovy”] [/字幕]

谷歌刚刚宣布 ，其Google App Engine云托管平台现在支持除Python外的其他语言：Java和Groovy！

您现在可以在Groovy中有效地编写Google App Engine应用程序 ！

几周前，SpringSource Groovy团队与Google App Engine Java团队携手合作，共同解决了细节问题，以确保流行且屡获殊荣的JVM Groovy动态语言可以在此平台上很好地运行。令人兴奋的平台。在约束和严格的安全管理器策略领域为Groovy创建了一些补丁之后，Groovy开发团队集成了这些补丁，并在D日当天发布了更新的Groovy 1.6.1版本。使用此新发行版，您可以直接在WEB-INF / lib目录中使用“ groovy-all” JAR，并立即开始在Groovy中编写应用程序，并将其托管在Google的基础架构上。

在本文的其余部分，我将引导您完成一些简单的步骤，以使您构建第一个由Groovy驱动的App Engine Webapp。我将跳过基本的安装步骤，因为它们在App Engine文档中得到了概述和很好的说明，而我将集中精力于构建Groovy应用程序本身的各个方面。如您所见，这相当容易。

入门


首先，显然，首先，您需要在Google App Engine上注册一个Google帐户，以便您可以在平台上创建应用程序，并能够将它们上传到云中。您还需要下载并安装Google App Engine Java SDK。对于所有这些步骤，您应该签出在线文档，该文档提供了所需的所有详细信息。

一旦安装了SDK，就本教程而言，您还应该下载并安装Groovy 1.6 。您只需要在本文的第一部分中真正安装Groovy，就需要编译servlet，否则，对于本文的其余部分，您将不再需要它，因为我们将使用已编译的Groovlet。通过Groovy运行时本身。

安装Java，SDK和Groovy之后，我们可以继续进行操作，并从该支持Groovy的项目模板中开始一个新项目。下载该骨架，将其解压缩到您喜欢的目录中，让我们来看看我们拥有的骨架！就像打开圣诞礼物吗？

我已将模板项目解压缩到名为gaedemo的目录中。在该目录的根目录下，您将看到一个src目录，其中包含需要编译的所有Groovy和Java源代码（servlet，域类，实用程序类等）。deploy目录基本上对应于我们分解的webapp：您将看到一个用于编译类的classes目录，用于各种JAR（Groovy JAR以及Google App Engine自己的API JAR）的lib和用于包含我们将开发的Groovlet的groovy。在本文的第二部分。您当然也已经注意到appengine-web.xml文件，它是App Engine特定的描述符。还存在规范的web.xml文件，用于定义servlet，映射等。

编译课程


考虑到项目结构的概述，让我们看看每个密钥文件包含什么。让我们从build.groovy文件开始。我没有创建Ant构建，而是利用Groovy的AntBuilder ，它是在Ant和Ant任务之上的轻量级DSL /脚本包装器：
def ant = new AntBuilder().sequential {
    webinf = “deploy/WEB-INF”
    taskdef name: “groovyc”, classname: “org.codehaus.groovy.ant.Groovyc”
    groovyc srcdir: “src”, destdir: “${webinf}/classes”, {
        classpath {
            fileset dir: “${webinf}/lib”, {
                include name: "*.jar“
            }
            pathelement path: ”${webinf}/classes“
        }
        javac source: ”1.5“, target: ”1.5“, debug: ”on"
    }
}

我们实例化AntBuilder，为目标WEB-INF目录创建一个属性，我们定义groovyc Ant任务，它是Groovy Joint Compiler，它能够通过将Java类的编译委托给javac来将Groovy和Java相互依赖的类一起编译。编译器-两种语言之间无缝互操作性的又一证明。定义完该任务后，我们可以使用WEB-INF / lib中的JAR组成的类路径以及已编译的类来调用它来编译源代码。

为了调用该构建文件（授予已安装Groovy的权限），您只需使用以下命令来编译您的项目：

groovy build

设置项目描述符


appengine-web.xml文件包含Google App Engine部署应用程序所需的一些元数据。特别是，您可以在其中定义应用程序的名称或其版本号。您必须更新文件以使用自己的应用程序名称。到目前为止，我们的描述符如下所示：
<appengine-web-app xmlns="http://appengine.google.com/ns/1.0">;
    <application>myowngroovy</application>
    <version>1</version>
</appengine-web-app>

创建您的第一个servlet


在深入研究Groovlets之前，我们将从一个好的旧Servlet开始！由于Google App Engine支持Servlet 2.5规范，因此我们可以编写一个简单的Hello World！servlet。我们在src目录中创建一个名为HelloServlet.groovy的文件，该文件将包含以下代码：
import javax.servlet.http.*
class HelloServlet extends HttpServlet {
    void doGet(HttpServletRequest req, HttpServletResponse resp) throws IOException {
        resp.contentType = “text/plain”
        resp.writer.println “Hello Google App Engine Groovy!”
 }
}

尽管您会注意到Groovy提供的更简单的语法，这看起来很像普通的Java servlet：缺少分号，可选的公共关键字，getter / setter的属性符号，省略了分号。

下一步：我们需要在web.xml中引用该servlet，如下所示：

<web-app xmlns=“http://java.sun.com/xml/ns/javaee” version=“2.5”>
    <servlet>
        <servlet-name>HelloServlet</servlet-name>
        <servlet-class>HelloServlet</servlet-class>
    </servlet>
    <servlet-mapping>
        <servlet-name>HelloServlet</servlet-name>
        <url-pattern>/hello</url-pattern>
    </servlet-mapping>
</web-app>

一旦配置了servlet，我们就不要忘记用我们的小型构建文件来编译servlet：
groovy build

将您的应用程序上传到云中！


如果您的Google App Engine SDK已正确配置，则应该能够从项目的根目录运行以下命令：
appcfg.sh update deploy/

该命令将在首次使用时提示您输入凭据，进一步的调用将显示类似于以下行的输出：

如果看到“成功”一词，那肯定是因为一切都进行得很顺利，并且您的应用程序已准备就绪，可以访问！从类似于该URL的URL中获取servlet（取决于您选择的应用程序的名称）： http://myowngroovy.appspot.com/hello将向您展示漂亮的“ Hello Google App Engine Groovy！”信息！

Groovlets进行救援！


好吧，用Groovy语言编写普通的Java servlet很令人兴奋，对吧？感觉就像是您编写并开始使用第一个servlet的第一天？事实并非如此，似乎已经过时了。幸运的是，Groovy推出了Groovlets来进行救援！

简而言之，Groovlet只是存储在WEB-INF / groovy中的Groovy脚本，它们由Groovy servlet调度程序呈现，并编译并呈现这些脚本。

首先，让我们更新web.xml以将GroovyServlet添加到混合中，并添加一个URL映射以将遵循.groovy模式的所有URL重定向到该URL ：

<web-app xmlns=“http://java.sun.com/xml/ns/javaee” version=“2.5”>
    <servlet>
        <servlet-name>GroovyServlet</servlet-name>
        <servlet-class>groovy.servlet.GroovyServlet</servlet-class>
    </servlet>
    <servlet>
        <servlet-name>HelloServlet</servlet-name>
        <servlet-class>HelloServlet</servlet-class>
    </servlet>
    <servlet-mapping>
        <servlet-name>GroovyServlet</servlet-name>
        <url-pattern>.groovy</url-pattern>
    </servlet-mapping>
    <servlet-mapping>
        <servlet-name>HelloServlet</servlet-name>
        <url-pattern>/hello</url-pattern>
    </servlet-mapping>
</web-app>

完成此操作后，我们将在WEB-INF / groovy下创建第一个Groovlet脚本，并将其称为hello.groovy：
html.html {
    head {
        title “Hello”
    }
    body {
        p “Hello Groovy World!”
    }
}

该Groovy脚本使用绑定到脚本绑定的html变量，该脚本是MarkupBuilder的实例。这是一个小型实用程序DSL，用于创建任何类型的XML或HTML标记。与使用println语句以字符串形式输出原始HTML相比，MarkupBuilder提供了一种更简洁的语法。当然，您可以通过在某些循环或有条件的条件中进行混合等方式来使此标记具有多变的动态性。

重新上传应用程序之后，您现在可以通过单击URL来访问此Groovlet。 http://myowngroovy.appspot.com/hello.groovy 。这次无需编译任何东西，因为这是GroovyServlet的工作，用于编译那些Groovlets脚本。

那不是那么容易吗？

下一步是什么？


到目前为止，我们只涉及了一些问题，但是足以开始使用Groovy。正如您将在Google App Engine API中发现的那样，您可以从Groovlets和servlet中利用一些有趣的服务：

  • 可用于通过低级无模式API或JDO存储对象的数据存储区API

  • 图像API进行各种转换并在图像上应用过滤器

  • 邮件API发送电子邮件

  • memcache API，用于缓存昂贵的数据结构或计算结果

  • URL提取程序API来检索远程URL内容

  • 使用Google的用户帐户满足身份验证需求的用户API


当然，所有这些服务都可以从您的Groovlet脚本中使用。您也可以使用第三方库并将其放入WEB-INF / lib中。进一步研究可用的API，以查看它们之上的Groovy薄层是否能够以更Groovier的方式进一步简化其用法，将是很有趣的。

当前，完全支持Groovlets和常规servlet，但是例如， Grails应用程序无法在当前版本的Google App Engine上运行。我们将继续与Google App Engine团队合作，以便您也可以将Grails用于要求更高的应用程序。

如果您想了解有关Groovy和Grails的更多信息，以及如何编写支持Groovy的App Engine应用程序，则还可以考虑注册并参加GR8会议 ，该会议是专门针对Groovy，Grails和Griffon的 会议 ，由这些技术的专家或制造商进行作为演讲者，动手实践会议。并宣布了Groovy在App Engine中的支持，毫无疑问，我们还将在大会上谈到这一点！

我们期待着您阅读有关App Engine上Groovy的反馈，并且很高兴听到您可能提出的所有建议，这些建议可以帮助我们进一步改善在云中开发Groovy应用程序的体验。
