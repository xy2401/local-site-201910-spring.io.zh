Spring Web Flow项目更新

工程
基思·唐纳德
2009年4月20日

Spring Web Flow项目团队刚刚完成了Web Flow 2的第7个维护版本 。这是迄今为止我们最好的发行版，距Web Flow 2.0.0最终发行版将近一年。现在，随着2.0.x产品线的成熟和稳定，我们将开始开发下一个主要版本。在这篇文章中，我想回顾一下过去的一年，并提供一些有关我们前进方向的信息。

Web Flow 2采用

去年发布Web Flow 2时，发布后的前两个月我们看到了50,000次下载。从那时起，我们的论坛流量一直在稳定增长，并且我们看到了几个令人振奋的行业的新应用。你们中的许多人都知道Spring Web Flow是Orbitz在线旅行平台的基础，如今该平台为ebookers.com和nwa.com等网站提供了动力。如果您一直在关注2009 NBA季后赛，您可能还会发现它很有趣，Web Flow也是nba.com的重要组成部分。

过去一年的工作

像所有Spring项目一样，Web Flow依靠反馈来取得成功。去年，与客户和SpringSource支持工程师的现场互动推动了我们在2.0.x上的许多工作。社区在报告错误 ， 贡献补丁 ，突出使用场景以及讨论项目可以继续改进的方式方面也表现出色。

我想快速回顾一下自2.0.0起所做的一些具体改进。发布：

  • 流URL映射的配置简化和约定

    此版本在2.0.5中应用，并受到客户反馈以及Dan Allen的JSFOne演示的推动 ，将典型的webflow-config.xml的大小减少了一半，减少到约20行配置。如您所见 ，这是通过对流定义进行通配符搜索，并结合了基于流ID将流定义绑定到URL的约定而实现的。

  • 支持显式视图状态模型绑定

    响应此安全公告，此改进首先提供给SpringSource客户，然后在2.0.3中发布。您可以通过视图状态来限制允许的模型绑定集。这是通过此处所示的声明式样式实现的。

  • 事后重定向改进

    Web Flow最有用的功能之一就是后重定向模式可以正常工作，这是在受控导航下获得良好后退按钮支持的关键先决条件。2.0.5、2.0.6和2.0.7均对此支持进行了后续改进。最新的改进确保了重定向行为在所有情况下（包括存在绑定或验证错误时）都始终适用。您可以在ViewState.java的doEnter和doResume方法中查看控制此模式的实施的源代码。

  • 支持流操作

    社区弄清楚了如何将文件流传输回参与流程的客户端。文件支持此网络中的流量2.0.0被忽略和2.0.6中加入。

  • 类型转换改进

    从2.0.2到2.0.6应用了对视图状态模型绑定提供支持的系统的许多改进。该系统提供了Spring DataBinder的所有独特功能，例如，与Java PropertyEditor相比，使用更简单的类型Converter API，支持转换通用集合的元素。

除了这些核心改进之外，去年我们还看到了许多有趣的Web Flow集成，例如Grails 1.1 ， ZK RIA框架 ， Terracotta ， IceFaces ，SpringSource的richweb培训课程 ， IntelliJ ， Skyway Software和第一个Web Flow 2。书 。

我们要去的地方

我们对未来有很多计划。我将保留所有技术细节的时间，但想总结一下这项工作的一些关键主题。首先，Web Flow 3将是第一个需要Java 5的版本，因为它将基于Spring Framework 3作为基础。其次，您可以期望看到@Flow模型的引入，该模型补充了Spring MVC的无状态@Controller模型，并允许将有状态的Web流定义为POJO。第三，您可以期望将Spring JavaScript和Spring Faces（这两个源于Web Flow 2的成果）发展为顶级Spring项目。Spring JavaScript将成为Spring的官方Ajax集成项目，而Spring Faces将成为Spring的官方JavaServerFaces集成项目。

我期待下周在SpringOne与大家见面，讨论您在应用该项目方面的经验以及我们的未来方向！
