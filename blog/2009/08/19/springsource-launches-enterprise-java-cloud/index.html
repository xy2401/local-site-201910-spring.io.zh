SpringSource推出企业Java云

工程
罗德·约翰逊
2009年8月19日

今天，我们再次发布重要声明。SpringSource正在启动企业Java云– SpringSource Cloud Foundry 。

该计划是我们集成的Build / Run / Manage方法的逻辑扩展，以统一应用程序生命周期，将我们简化企业Java的愿景扩展到传统数据中心之外。随着云计算变得越来越重要，我们希望确保Java开发人员可以充分利用它。我们相信，我们在Java开发方面的领导地位，加上我们在运行时和管理阶段的实力，使我们能够提供引人注目的解决方案，从而使开发人员社区受益。

这箱子里有什么东西？


SpringSource Cloud Foundry提供了一种易于使用的自助服务方法，使开发人员可以使用简单的Web界面在几分钟内部署其应用程序。它通过自动扩展和自动基础结构修复，提供了常见部署蓝图的自动化和SLA驱动的智能资源分配。

Cloud Foundry建立在SpringSource领导或做出的重大贡献的熟悉Java技术的基础上。核心运行时是我们的tc Server产品 ，该产品基于Apache Tomcat构建并与之完全兼容。我们使用Apache HTTP Web服务器进行负载平衡，并集成Hyperic技术进行监视和管理。我们还提供了一个关系数据库（MySQL）。Cloud Foundry允许用户通过Web界面指定所需的应用程序服务器和数据库配置，从而隐藏了实现所需结果的复杂性，确保所需步骤反映了意图而非细节，并且配置和部署是可重复的。我们期望大多数用户将部署Spring或Grails应用程序，尽管Cloud Foundry不仅限于Spring和Grails应用程序。

因此，编程和部署模型完全基于开源。Cloud Foundry服务还基于开源：由Chris Richardson创立和领导的Cloud Tools项目，该项目提供了一套在云环境中部署，管理和测试企业Java应用程序的工具。

此初始版本在Amazon EC2云基础架构上运行。但是，Cloud Foundry的关键要素之一是它在此类基础架构上提供的抽象。与Spring Framework 一样，我们将致力于增强开发人员的选择。将来，Cloud Foundry将为开发人员提供公共（非本地）云和私有（本地）云的一致体验，使组织能够做出最适当的选择。我们与客户的对话清楚表明，这种统一的方法是当前行业中尚未满足的关键需求。满足此需求的关键部分将是Cloud Foundry，它也在VMware vSphere基础架构上运行，该架构提供了独特的高级本地云基础架构。

未来发展方向


Cloud Foundry对于当今的开发人员而言非常有价值，但是更好的是我们才刚刚开始！

更长远的眼光更远了。我们将增强您已经看到的管理和其他功能。正如我们使用tc Server产品本身所做的那样，我们将增强Spring和Grails应用程序的运行时可见性。我们希望通过SpringSource dm Server提供模块化部署。我和Adrian都在博客中谈到了完全集成的堆栈的可能性，其中应用程序框架和应用程序服务器与云基础架构紧密协作，以提供最佳的体验。借助Cloud Foundry，我们正在向开发人员开放这条道路。

我们正在免费提供的SpringSource Tool Suite（STS）中与Cloud Foundry集成，从而使Spring开发人员更加轻松地将应用程序部署到云中。我们还计划进行Grails集成，从而使将Grails应用程序部署到云中变得像创建它们一样容易。

立即开始！


今天的发布是一个beta服务。作为我们在Spring项目中的传统，我们将在转向GA版本时努力获得开发人员的反馈。

只需访问CloudFoundry.com ，您就可以在几分钟内部署应用程序。我们希望收到你的来信！例如，我们愿意提供除MySQL外的持久性选择，并希望得到您的投入。

欢迎克里斯·理查森


一些读者会认识到Cloud Foundry的名称。将云确定为重要的战略方向后，由于其在该领域带来的创新和领导地位，我们选择收购Cloud Foundry Inc.。从那时起，我们增加了对Cloud Foundry技术的投资。

你们中的许多人都将认识Cloud Foundry的创始人Chris Richardson，他是Spring社区的长期成员和企业Java思想领袖。我很高兴欢迎克里斯和他的团队加入SpringSource。他们的工作与SpringSource在框架，服务器和管理方面的实力相结合，令人振奋！
