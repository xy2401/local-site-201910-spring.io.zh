dm Server 2.0 M4

工程
安迪·威尔金森
2009年8月6日

dm服务器 2.0.0。M4已发布，现在可以下载 。

自2.0.0版以来，我们已经取得了很大进步。M3，添加了许多新功能，用户可以通过这些新功能跟踪夜间版本，这已经给了我们一些很好的反馈。如果您有兴趣查看我们一直在努力的所有内容，请查看M4发行说明 。请在我们的博客， 论坛和JIRA上保留您的反馈意见。

新的和值得注意的

dm Server现在嵌入Medic：我们的OSGi可维护性项目

作为M4工作的一部分，我们已经将dm Server的可维护性支持移到了一个单独的ASL许可项目Medic中。作为此举的一部分，我们还借此机会进行了一些重大改进：


  • 日志支持现在基于LogBack ，这意味着用户现在可以完全控制日志输出的格式，并且可以使用完整的LogBack附加程序范围。
  • 现在，可维护性转储的支持是完全可扩展的。现在，只需将DumpContributor实现发布到服务注册表中，即可在dm Server中运行的用户代码有助于维护性转储。
  • 事件日志支持（输出到控制台的编码消息）也可用于用户代码。可以使用EventLogger生成事件日志条目，可以从服务注册表中获取事件日志。

我们将在未来几周内发布有关Medic的更多博客，介绍如何在dm Server和普通OSGi中使用它。

支持部署属性文件

现在可以通过管理控制台将属性文件部署到dm Server上，也可以将它们放到Pickup目录中。部署属性文件时，将使用从属性文件的名称派生的pid创建ConfigurationAdmin 配置 。例如，部署名为com.foo.bar.properties的属性文件将使用pid com.foo.bar创建一个Configuration。
