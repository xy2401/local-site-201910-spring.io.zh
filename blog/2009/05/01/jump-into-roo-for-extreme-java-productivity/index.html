进入Roo，以实现极高的Java生产率

工程
本·亚历克斯
2009年5月1日

更新： “ Spring Roo简介”博客系列的第二部分现已发布，其中包括详细的分步教程，可帮助您开始使用Roo。第三部分详细介绍了Roo的内部体系结构。

本周，来自SpringOne Europe的新闻使Twitter领域变得非常热闹。SpringSource的新开源生产力工具（代号为“ Roo”）引起了极大关注。

Roo是一个复杂的往返代码生成器，它使您比想像中创建和开发Spring应用程序更快，更轻松。即使您对代码生成有所保留，仍然值得一看Roo。它包含了重大的创新，可以解决所有主要的反对代码生成的问题，同时仍然提供最佳实践Spring应用程序，并且在整个应用程序生命周期中仍然有用。

这是我系列中的第一个博客，我将讨论Roo，从其一般设计目标和功能开始，然后再介绍如何使用它以及Roo工程内部。鉴于此第一个博客为介绍，让我们从重复指导Roo开发的使命声明开始：

Roo的使命是从根本上可持续地提高Java开发人员的生产力，而又不影响工程完整性或灵活性。

这如何转化为工具？一位看到Roo的会议代表将其描述为“一个坐在后台，处理我不想担心的事情的小精灵”。由于两个原因，该描述特别合适。首先，Roo确实“有点”。它的当前下载量不到3 MB，它使您可以继续使用基于Spring的常规Java编程模型，并且不需要采用特殊的Roo运行时或构建技术。其次，它确实静静地坐在后台，为您照顾事物。最重要的是，Roo不会妨碍您。如果您想自己写东西，那就去做吧，Roo会悄悄地自动调整相关文件作为响应（即Roo支持往返）。如果要编辑Java或JSP文件，只需执行此操作。Roo不需要您使用其界面。或者，如果您想停止使用Roo，只需退出程序即可。Roo不会成为您的构建系统或运行时的一部分，因此在您极有可能发生的极少数情况下，停止使用Roo非常容易。

我们相信Roo填补了现有IDE的强大功能，现代Web RAD框架所展示的生产力潜力以及Java开发人员强烈希望拥有一种能够以其希望的工作方式工作并反映其工程原理的工具之间的甜头。他们重视。这样就产生了一种非侵入性的后台工具，该工具异常易学，易于使用，既可以应用于现有项目也可以应用于新项目，并以非凡的速度简化了世界最佳实践应用程序的开发。

生产力和可用性

当您拥有易于使用并基于现有知识，技能和经验的高度可用的工具时，提高生产力和享受开发就变得更加容易。因此，在设计Roo时，可用性是我们的关键设计因素。开发人员通过使用Tab键完成用户界面（例如* nix shell提示）而享有的明显的生产力优势，使我们深受其影响。但是我们又走了一步，专门为Roo写了一个shell。通过提供外壳程序（而不是GUI或命令行实用程序），我们能够实现可编写脚本的，用户友好的，易于学习的，宽容的界面。如有疑问，可以按TAB键，Roo将完成命令。您可以在需要时键入“提示”，并接收特定于项目阶段的指导。您可以忽略命令中的选项，Roo会根据您最近所做的操作自动推断您的意思。您可以输入非法命令，Roo会仔细撤消失败命令可能进行的任何更改。您可以编写脚本并进行回放以再次产生您的项目。您甚至不需要完全键入命令-只需键入足够的内容，以便Roo识别出它是唯一的。您甚至可以在不安装IDE的情况下从操作系统命令提示符下完成所有这些操作。

当然，大多数Java开发人员都使用IDE，并发现它们对于提高生产力非常宝贵。因此，Roo意识到了这一点，并在您在IDE内处理项目时愉快地继续执行。您将不需要任何特定于Roo的IDE插件（尽管我们确实有SpringSource Tool Suite的插件，我将在下面进一步讨论）。自然，在您的IDE中，您会喜欢Roo旨在通过代码辅助和调试支持使Java开发工作变得轻松的事实。调试器将像往常一样运行，逐步浏览构成应用程序一部分的所有代码-包括Roo帮助您的那些文件。Roo生成的每个源文件都会正确显示在代码辅助（控件+空格）列表中，因此您无需记住方法签名。

由于Roo支持您使用Java进行编程，因此您将获得常规的增量验证，这有助于我们所有人识别丢失的方括号或输入错误的字段名称。IDE重构工具将继续为您提供服务，使您有信心即使在庞大的开发团队，不断变化的团队成员以及多年的不断开发和维护的情况下，应用程序代码质量也将保持很高的水平。最后，所有Roo注释当前都采用“ Roo *”的命名约定。该命名约定确保您可以在需要时轻松找到注释-根本不需要真正键入任何Roo特定的注释。注释本身为您提供了对Roo如何帮助您进行开发的非常细粒度的控制，允许您在希望的情况下以详细的方式控制成员名称和自定义行为（默认情况也很好）。

在欧洲SpringSource上，我们还展示了SpringSource Tool Suite（STS）中Roo的使用，STS是我们流行的， 即将免费提供的基于Eclipse的IDE。这使您可以在IDE中享受Roo的所有好处。您可以在屏幕底部的“ Roo View”中输入命令，然后立即在Package Explorer和Editors中看到更改。或者，您可以在“编辑器”中进行更改，然后立即看到“ Roo视图”，以显示Roo对您所做的更改所做的操作。这是一个小精灵，挡不住您的路，照看您不想要的东西。

技术功能

Roo建立在一个附加架构上，该附加架构可构建不同类型的Java应用程序。目前，大约有18个“附加组件”，这些附加组件旨在通过单独的基于JPA的实体层支持轻松开发请求响应MVC Web应用程序。Roo当前包含的一些特定功能：

  • 从空目录开始新应用程序（不过，Roo也可以与现有应用程序一起使用）
  • 配置构建系统并设置特定于该构建系统的项目结构（我们最初支持Maven 2）
  • 安装您选择的JPA提供程序和数据库
  • 查看和配置数据库配置详细信息
  • 设置Log4J配置文件，并允许使用TAB完成命令对其进行修改
  • 创建具有自动依赖项注入和持久性方法的JPA实体
  • 向JPA实体添加字段，包括自动为JPA和新的Bean验证（JSR 303）标准设置正确的注释
  • 管理自动JUnit集成测试，即使您在实体之间存在关系并且正在应用Bean验证约束，该测试也可以立即使用
  • 创建和维护认真遵循REST约定的基于Spring MVC的控制器，包括为这些控制器生成和维护JSP页面
  • 轻松完成Spring MVC控制器的存根，从而省去了查找此类控制器中最常用的注释，约定和方法签名的需要
  • 在实体上动态创建查找器，因此无需编写JPA查询语言即可生成典型的“ findByField1LikeOrField2Between（Field1 like，Field2 from，Field2 to）”样式查询
  • 轻松安装和配置Spring Security（只需键入“ security setup”，然后按Enter）

所有这些功能都建立在Roo的内部元数据模型上，该模型是项目的全面表示。因此，尽管像IDE这样的技术提供了成员结构模型，但Roo通过提供反映通用开发约定的企业特定于应用程序的元数据，将其提升到了更高的抽象水平。例如，尽管IDE可能知道存在一个名为“ Foo”的类，该类具有一种名为“ persist”的方法，该方法定义了某些参数和返回类型，但Roo也具有此信息，但还包含一系列其他元数据，例如该方法从逻辑观点。

作为元数据模型的结果，我们将在以后的版本中为Roo添加的功能将大大扩展到请求-响应Web应用程序开发之外。通过此元数据模型实现的可能性非常广泛，包括例如第四代Web应用程序的往返管理（基于组件的，客户端执行的技术），往返UML集成，备用持久性模型（例如针对云优化的）计算）和往返富客户端。
