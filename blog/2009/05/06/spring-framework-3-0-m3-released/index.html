Spring Framework 3.0 M3发布

工程
于尔根·霍勒（Juergen Hoeller）
2009年5月6日

我们很高兴地宣布，第三个Spring 3.0里程碑现已可用（ 下载页面 ）！此版本具有许多新功能和改进，包括…

参考文档 ：M3是HTML 3.0和PDF格式的参考文档随附的第一个Spring 3.0里程碑。即使该文档仍在进行中，目前它确实涵盖了许多3.0功能区域。我们希望您会发现本文的早期内容对了解有关3.0里程碑功能的更多信息很有用。

带注释的工厂方法： Spring 3.0 M3包含Spring JavaConfig项目的核心功能，即带有定义带管工厂的带注释工厂方法的配置类。

@Bean @Primary @Lazy
public RewardsService rewardsService() {
return new RewardsServiceImpl(…);
}

任何带注释的组件类 （例如，普通@Component类）均支持此类工厂方法，并基于组件的状态构建和公开bean对象。除了包含组件的bean定义之外，它们还将被简单地视为从该组件类派生的其他bean定义。这种bean定义的默认名称是其工厂方法的名称（在上面的示例中：“ rewardsService”）。每当容器需要获取新实例时，将通过工厂方法调用获得这些bean实例。这在语义上接近于具有工厂bean /工厂方法引用的XML bean定义。

除了简单的工厂方法外，我们还支持JavaConfig的操作模式 ：如果在标有@Configurationannotation的类上定义了工厂方法，则将应用特殊的行为……此类显式配置组件上的工厂方法可以在上调用其他工厂方法。相同的组件，这些内部工厂方法调用将通过容器重新路由！这允许构建容器管理的Bean实例的图形，而构建器代码仅由链接的Java工厂方法调用组成。像原始的JavaConfig项目一样，我们正在生成此类配置类的CGLIB子类，以提供那些扩展的语义。

请注意，带注释的工厂方法可以以无缝的方式与XML bean定义混合并匹配 。它们也可以与常规带注释的组件类混合并匹配，重用许多常见的注释，例如@ Scope，@ Lazy，@ Primary和@Qualifier。包含带注释的工厂方法的组件可以用XML定义或通过在类路径中进行组件扫描来检测。基本上，适用于所有常规Spring bean的规则；现在，这是Spring 2.5对带注释的组件的自然支持。

对元注释的扩展支持： Spring的@Scope和@Transactional注释以及特殊的构造型（如@Service和@Controller）现在可以用作元注释，用于自定义注释。例如：

@Service
@Scope(“request”)
@Transactional(rollbackFor=Exception.class)
@Retention(RetentionPolicy.RUNTIME)
public @interface MyService {
}

@MyService
public class RewardsService {
…
}

这是定义具有扩展默认语义的自定义构造型注释的有效方法，如上所示。或者，您可以构建简单的自定义范围注释：例如，标有@Scope（“ request”）的自定义@RequestScoped注释，或自定义事务注释：例如，标有@Transactional（rollbackFor = Exception.class）的@MyTx。换句话说，这允许为您首选的配置变量创建方便的快捷方式注释 ！

TaskScheduler抽象： Spring现在提供了完整的调度外观API，包括用于标准ScheduledExecutorServices和CommonJ TimerManager的适配器。它带有一个触发器抽象和一个CronTrigger实现，用于基于cron表达式的简单调度。在3.0 RC1中，我们将在此之上添加一个调度名称空间，以方便基于XML的配置。

新型的类型转换SPI和转换器API ：受Spring Web Flow绑定子系统功能的启发，Spring现在支持基于无状态Java 5的类型转换器。它已经与Spring 3.0的表达式解析器集成在一起，并且将完全受支持，作为3.0 RC1中所有Spring绑定目的的标准JDK PropertyEditor的替代。同样，Spring Web Flow 3.0也将使用此新的统一转换子系统。

现在，我们将转移到计划于六月发布的Spring 3.0 RC1 。RC1计划的一项主要新功能是支持基于注释的验证：特别是JSR-303 Bean验证提供程序的集成 。我们还在研究ApplicationContexts中启动/停止生命周期的改进，并且正在为Spring管理的作用域代理开发可序列化解决方案。关注此空间！

PS：我们将发布旧版PetClinic示例应用程序的完整修订版，以作为现代Spring 3.0 Web应用程序的展示，它将单独发行。Keith将在未来几天内就此发表博客。
