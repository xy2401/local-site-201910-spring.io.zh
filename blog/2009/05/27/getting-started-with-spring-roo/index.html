进度更新

安装


现在，我们已经向Spring Roo 1.0.0添加了简化的安装。M1，安装非常简单，只需下载ZIP，解压缩然后将“ bin”目录添加到您的路径即可：

  • Windows用户应加载控制面板，然后单击“系统”，“高级”，“环境变量”，然后双击“路径”条目。追加到路径的末尾，“; C：\ spring-roo-1.0.0。M1 \ bin”（或安装Roo的任何位置）。

  • * nix用户应创建指向roo.sh shell脚本的符号链接。诸如“ sudo ln -s〜/ spring-roo-1.0.0”的命令。M1 / bin / roo.sh / usr / bin / roo”通常可以解决问题。


虽然可以独立使用Spring Roo，但我也建议您下载SpringSource Tool Suite （STS）2.3.0或更高版本。STS是我们的免费基于Eclipse的IDE，并提供许多功能来简化Spring应用程序的开发。例如，如果您下载STS 2.3.0或更高版本，您将获得内置的Spring Roo支持，甚至不需要安装独立的Spring Roo shell。

假设您使用的是独立的Spring Roo Shell，将撰写此博客。除非另有说明，否则所有命令在标准Spring Roo和STS 2.3.0或更高版本中均相同。主要区别之一是在使用Roo Shell时，按TAB键以选择完成选项，而在STS中，TAB键由CTRL + SPACE代替。我们使用CTRL + SPACE是因为它是基于Eclipse的IDE中完成操作的更传统的组合键。

您还应该验证是否已安装Maven 2.0.9或更高版本。尽管Roo本身不使用Maven并且可以在未安装Maven的情况下运行，但是Roo创建的项目当前使用Maven。另外，如果安装了较早版本的Roo alpha版本，请确保删除ROO_HOME变量。

申请条件


现在您已经安装了Roo，让我们考虑一下我们的婚礼RSVP应用程序的需求。然后，我们将结合使用Roo和一些手动编码来构建应用程序。对某些手动编码的要求反映了Roo的基本理念，即您仍然需要开发应用程序中与下一个应用程序不同的那些部分。但是，您会看到Roo旨在以完全透明，熟悉且往返的支持方式来进行手动编码和自定义 。

尽管有在线RSVP，我们仍然发送了纸质结婚请柬。每个邀请的背面都有一个小的“邀请码”。这些不容易猜到，但易于阅读和输入（没有UUID ！）。婚礼邀请文字邀请客人访问我们的婚礼RSVP网站以进行RSVP。当他们访问婚礼RSVP网站时，将要求客人输入邀请代码。

访客输入邀请代码后，将检索任何现有的RSVP记录并允许他们进行编辑。如果以前未进行过RSVP审核，则会为他们提供新的RSVP表单以进行填写。该表格将仅询问客人有多少人参加，以及任何特殊要求（例如饮食需求）。还将要求他们输入电子邮件地址，如果提供了电子邮件地址，则应用程序将发送基于电子邮件的RSVP确认。我们还将保留其RSVP的日期和时间的记录，如果他们多次更改其RSVP将会很有用。

创建一个持久项目


现在我们有了一个想要构建的实际应用程序的想法，让我们使用Roo来构建它。第一步是创建一个空目录并加载Roo：

$ mkdir wedding
$ cd wedding
$ roo

如果您按照上述安装说明进行操作，则应该看到下面显示的Roo徽标。如果没有看到这样的消息，请返回并检查安装是否正确。

Roo中有很多可用性功能。如果您键入“提示”，则将向您提供分步说明。或者，如果您键入“ help”，您将看到当前可用的所有命令（这些更改在项目生命周期的不同阶段）。另外，在几乎所有情况下，您都可以按TAB键以获得完成服务。 如果您正在学习Roo，那么“提示”就是您的朋友。它不仅会教您命令，还会教您有关外壳以及键盘功能的工作原理。养成在有疑问时使用“提示”的习惯，尤其是在您最初的几个Roo项目中。

让我们开始我们的婚礼项目。在Roo Shell中键入“创建项目”命令后，您应该收到以下输出：

roo> project --topLevelPackage com.wedding

Created SRC_MAIN_RESOURCES

Created SRC_TEST_RESOURCES

如控制台输出所示，Roo创建了一个Maven 2项目结构。即使您此时退出Roo并从不重新加载它，此刻，您仍具有正确配置的Spring 3 Web应用程序，包括URL重写 ，基于注释的类路径扫描和任何类的依赖项注入，甚至包括使用“ new”关键字或通过像Hibernate这样的ORM 。您甚至可以使用“ mvn tomcat：run”并启动嵌入式Tomcat容器。

如果此时输入“提示”，那么Roo建议您安装JPA提供程序和数据库。让我们现在开始：

roo> persistence setup --provider HIBERNATE --database HYPERSONIC_PERSISTENT

注意，我们在命令中选择了Hibernate和持久性Hypersonic数据库。该选择是通过TAB按钮进行的，因此我们实际上不需要完全键入这些命令和参数。还要注意底部的两个“ Managed”语句。这些表示Roo正在管理的文件或目录。Roo具有内置的文件撤消服务，因此如果出现问题，它将自动回滚有问题的命令。

由于Roo使用JPA，因此我们可以享受不同JPA实现之间的可移植性。如果查看Roo生成的代码，您会发现其中没有一行是特定于持久性提供程序的。您还将看到Roo提供的代码非常高效。 Java的强项之一就是其出色的性能，Roo可以进行所有工作，从避免反射到优化toString （）方法中的字符串操作（以及两者之间的所有操作），以最大程度地提高应用程序的运行时性能。

当我们要求一个持久数据库时，默认情况下它存储在〜/ wedding。*中。有一个“数据库属性列表”命令，它将显示数据库配置：

虽然默认位置可以正常工作，但让我们将其更改为其他位置：

roo> database properties set --key database.url --value jdbc:hsqldb:/home/balex/our-wedding
Managed SRC_MAIN_RESOURCES/META-INF/spring/database.properties

如控制台输出所示，Roo所做的所有工作都是编辑一个库存标准的database.properties文件。您可能已经使用文本编辑器或IDE有效地编辑了项目文件。鲁不介意。它是从一开始就进行设计的，因此您可以与Roo同时使用其他工具，并且一切仍然可以正常进行。

您可能想通过Roo使用“数据库属性集”命令的一个原因是，因为您正在制作一个独立的脚本，以后可以重播。您可以使用“ script filename.roo”命令执行脚本，这些脚本只是标准文本文件格式的Roo命令。为了方便起见，我在Roo 1.0.0发行版中包含了wedding.roo脚本。请注意，也可以使用常规Java注释语法（//，/ *和* /）将注释包括在脚本中。

创建一个实体


现在让我们创建一个JPA实体。该实体将存储在我们的数据库中，并代表我们应用程序的整个域模型。您可以使用选择的IDE，文本编辑器或Roo Shell创建实体：

roo> entity --class ~.domain.Rsvp
Created SRC_MAIN_JAVA/com/wedding/domain
Created SRC_MAIN_JAVA/com/wedding/domain/Rsvp.java
Created SRC_MAIN_JAVA/com/wedding/domain/Rsvp_Roo_Entity.aj
Created SRC_MAIN_JAVA/com/wedding/domain/Rsvp_Roo_ToString.aj
Created SRC_MAIN_JAVA/com/wedding/domain/Rsvp_Roo_Configurable.aj

在这一点上，我想你们中有些人会想，“那些.aj文件是什么？”。简而言之，这些是AspectJ 类型间声明 （ITD），它们非常有效地实现了关注点分离，同时还保持了与相关Roo附加组件未来版本的兼容性。Roo会自动创建，维护和删除.aj文件，从而使最终用户可以安全地忽略它们。实际上，STS 2.1.0+默认情况下会自动隐藏它们，就像基于Eclipse的IDE隐藏.classpath，.project和.settings资源一样。毕竟，这些资源只是该工具的内部输出，很少有您打开它们-更不用说自己维护它们了。在我的下一个博客条目中 ，我将详细讨论这些以及其他Roo内部问题，因此在此之前，我将进一步讨论。

您可能已经注意到我们在“〜.domain”包中创建了Rsvp实体。“〜”字符会自动扩展到项目的顶级程序包，您可能还记得我们最初创建项目时指定的程序包。因此，Roo充分理解了Java包的概念，并允许您以最直观的方式构造项目包。

自然地，一个实体通常会包含一些字段，因此让我们添加它们（省略Roo的输出，因为它只管理与上面列出的相同的文件）：

roo> field string code --notNull --sizeMin 1 --sizeMax 30
roo> field string email --sizeMax 30
roo> field number attending --type java.lang.Integer
roo> field string specialRequests --sizeMax 100
roo> field date confirmed --type java.util.Date

在第一行中，您会注意到我们使用–notNull参数以及–sizeMin和–sizeMax参数。这些引用新的Bean验证标准，否则称为JSR 303 。该特定标准提供了自动的Web和持久层验证，包括为数据库中的表创建正确的DDL 。使用Roo的优点之一是，您无需付出任何额外努力即可获得相关标准的好处，例如JSR 303，JPA， Servlet规范和REST 。自然，如果您不想使用JSR 303参数，则无需使用。

上面的field命令要注意的另一点是，我们没有指定要将字段插入到哪个实体中。Roo自动确定您可能要将字段添加到Rsvp，因为这是您使用的最后一个实体。您也可以指定“ –class〜”。“ SomeEntity”参数，如果您希望明确显示该字段或将字段定位到另一个实体（在这种情况下，该实体将成为后续与实体相关的命令的默认值）。

PublicRsvpController将响应HTTP GET和POST请求。在PublicRsvpController.java源文件中已自动提供了两种用于这些操作的方法。

如果考虑GET用例，我们的目标是为特定的邀请代码检索正确的RSVP。Spring Security的工作方式是要求人们登录才能使用该应用程序，并且我们将每个邀请代码视为唯一的登录名。这样，GET方法需要从Spring Security获取当前登录的用户名，然后从数据库中检索相应的RSVP。通常，此时您将编写一个JPA QL查询来获取具有匹配代码的特定Rsvp实例，但是由于您使用的是Roo，因此您可以省去麻烦，而可以使用动态查找器。

动态查找器为您提供了几乎无限的预装查询范围。这些查询全部在内部使用JPA QL，以提供最大的基于标准的兼容性和可移植性。所有动态查找器（和其他Roo方法）均以格式正确，类型安全的Java方法实现-带来了熟悉性，IDE代码辅助，调试器集成和显着的运行时性能的所有常规优势 。您可以使用以下命令列出可用的动态查找器：

roo> finder list --class ~.domain.Rsvp --filter code,equ
findRsvpsByCodeEquals(String code)
findRsvpsByCodeNotEquals(String code)

注意，“ – filter”参数将输出限制为仅包含“ code”和“ equ”字符串的建议方法签名。您可以通过省略“ -filter”参数或指定“ -depth 2”（或3、4等，如果您希望查询中包含更多属性）来指示Roo您希望看到更多组合。

@RequestMapping("/publicrsvp/**")
@Controller
@SessionAttributes("rsvp")
public class PublicRsvpController {

    @Autowired
    private transient MailSender mailTemplate;

    @RequestMapping
    public String get(ModelMap modelMap) {
    	modelMap.put("rsvp", getRsvp());
    	return "publicrsvp";
    }

    @RequestMapping(method = RequestMethod.POST)
    public String post(@ModelAttribute("rsvp") Rsvp rsvp, ModelMap modelMap) {
  	rsvp.setConfirmed(new Date());
        if (rsvp.getId() == null) {
        	rsvp.persist();
        } else {
        	rsvp.merge();
        }
    	if (rsvp.getEmail().length() > 0) {
        	sendMessage("Ben Alex <[email protected]>", "RSVP to our wedding", rsvp.getEmail(), "Your RSVP has been saved: " + rsvp.toString());
    	}
    	modelMap.put("rsvp", rsvp);
    	return "thanks";
    }

    private Rsvp getRsvp() {
    	Rsvp rsvp = new Rsvp();
    	try {
        	String code = SecurityContextHolder.getContext().getAuthentication().getName();
        	rsvp.setCode(code);
    		// Cast due to http://java.sun.com/javaee/5/docs/api/javax/persistence/Query.html#getSingleResult()
    		rsvp = (Rsvp) Rsvp.findRsvpsByCodeEquals(code).getSingleResult();
    	} catch (DataAccessException ignored) { /* no Rsvp for this code was found, so start a new Rsvp */ }
    	return rsvp;
    }

    private void sendMessage(String mailFrom, String subject, String mailTo, String message) {
        SimpleMailMessage simpleMailMessage = new SimpleMailMessage();
        simpleMailMessage.setFrom(mailFrom);
        simpleMailMessage.setSubject(subject);
        simpleMailMessage.setTo(mailTo);
        simpleMailMessage.setText(message);
        mailTemplate.send(simpleMailMessage);
    }
}

结论


在此博客文章中，我们介绍了安装Roo，使用其Shell并快速构建具有以下功能的实际应用程序所需的步骤：

  • 基于Spring Framework 3的最佳实践应用程序体系结构

  • 基于Maven 2的项目结构

  • 基于JPA的持久性，在这种情况下使用Hibernate

  • 数据库存储到自定义Hypersonic文件位置

  • Bean验证（JSR 303）支持，包括将约束向下传播到数据库DDL

  • 基于Spring Framework集成测试功能的自动化JUnit集成测试

  • 自动化的RESTful应用程序后端

  • 您的Web层的自动Selenium测试

  • 在我们的应用程序中满足实际用例的动态查找器

  • Spring Security集成，包括Web URL安全性和自定义的登录页面

  • 电子邮件传输（在Stefan的博客文章中了解有关通过Roo发送电子邮件的更多信息）

  • Log4J配置支持

  • URL重写可保持URL整洁和RESTful

  • 手动Web控制器

  • 嵌入式Tomcat服务器容器使用情况

  • Eclipse和STS集成
