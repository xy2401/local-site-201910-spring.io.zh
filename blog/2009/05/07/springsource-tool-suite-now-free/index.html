SpringSource Tool Suite现在免费

工程
克里斯蒂安·杜普依斯（Christian Dupuis）
2009年5月7日

4月27日下午2时39分左右，Rod在他的SpringOne Europe开幕主题演讲中宣布：

“ STS是免费的！”

Reto Meier和我们的合作伙伴namics拍了张照片作为当时的证明，并将其发布在flickr.com上 。此外，SpringOne的观众似乎对这一宣布很感兴趣，因此这个词很快就传播到了会议之外。

所以就在这里，因为罗德答应了，我们无法摆脱它；-)

今天，我们很自豪地兑现Rod的承诺，并宣布STS的第一个免费版本已经发布，可以从产品页面下载。

作为用户，您可以在两个版本之间做出选择：我们提供了STS 2.0.2。RELEASE发行版实质上是以前的商业2.0.1。根据我们的新免费EULA加上新的tc服务器工具的条款，RELEASE重新获得了许可。

如果您想了解Adrian，Ben和我在SpringOne上演示的所有新东西，可以下载即将发布的2.1.0版本的M1版本。

STS 2.1.0中的新功能。M1

如果您以前没有使用过STS，请确保查看有关较早版本中新功能的较旧的博客文章：


  • 在Eclipse中使用Bundlor

  • Bundlor入门

  • 宣布SpringSource Tool Suite 2.0

  • STS 2.0中的OSGi开发工具

此M1早期里程碑版本包含几个新功能：

  • 新的项目模板可帮助快速启动基于Spring的新项目。有适用于Spring MVC，Web Flow，Faces，Batch，Roo和OSGi捆绑项目的模板
  • 集成了Roo Shell，Quick Roo命令提示符（CRTL + R或CMD + R）以及一些STS特定的Roo命令，例如“部署”“服务器”并“运行所有测试”

  • 通过自动设置服务器群集和负载均衡器基础架构，将其部署到在Amazon EC2上运行的dm和tc服务器。EC2群集与WTP Server基础结构集成在一起，可以使用“新建服务器”向导进行设置。dm和tc Server的AMI已发布，可从EC2进行部署。我们将在另一篇文章中继续详细介绍。

  • 用于Spring Batch的可视编辑器，作为用于UI辅助开发Spring Batch和Spring Integration配置的概念验证
  • 捆绑的dm和tc服务器可以在STS中使用，也可以独立使用

 

“免费”实际上是什么意思？

我们对在STS上下文中“免费”到底意味着什么有一些疑问。因此，让我尝试解释一下：首先从版本2.0.2开始。发布，STS免费提供，免费用于所有开发目的；没有任何附加条件。它是根据商业许可获得许可的，您可以在此处进行查看 。

其次，我们致力于改善和推动我们的开源，EPL许可项目Spring IDE和dm Server Tools。这两个部分是STS的一部分，并将继续与STS一起发展。

实际上，您会在我们的合并更新站点上看到所有STS组件（包括Spring IDE和dm服务器工具）都已可用（这将在M2时间范围内发生）。这个新的更新站点将成为从SpringSource安装任何基于Eclipse的工具的唯一位置。更新站点位置将确保您仅可以安装完整的STS或某些功能，而无需查找其他更新站点位置并担心依赖关系解决问题。

除了整合各种更新站点外，我们还将在以下位置使用STS JIRA项目： http://issuetracker.springsource.com可以主动和公开地跟踪我们所有工具的功能请求，错误和改进。在接下来的几天中，您会注意到STS开发团队将开始跟踪他们在JIRA中的日常工作。

我认为您会同意我的观点，即在STS框架下进行的整合工作将使Spring开发人员的工作变得更加轻松：只有地方可以查找我们工具的最新版本，文件问题并与开发团队进行互动。

说到与团队的互动，我想鼓励每个Spring开发人员和Tomcat用户在那里下载STS，并尝试通过我们的论坛 JIRA ， Twitter （ #STS ）和电子邮件（sts）加入社区。 AT springsource DOT com）。作为用户，您现在可以积极影响STS的发展方向以及对您有价值的东西。此外，您无需花费时间来确定从何处获得某些工具功能，或者Spring IDE和STS之间的区别是什么。

我希望您像SpringSource的团队一样兴奋：

“ STS现在免费！！去拿吧…… ”

更新：由于流行的社区需求，我们发布了STS 2.1.0。M1到我们的综合更新站点，网址为 http://www.springsource.org/milestone/e3.4 （注意：此URL在浏览器中不起作用，但在Eclipse中起作用）。请在JIRA中提交该网站的所有问题。这是一项正在进行的工作，更新站点的结构将来可能会更改。
