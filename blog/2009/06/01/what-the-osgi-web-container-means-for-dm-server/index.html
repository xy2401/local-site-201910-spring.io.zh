OSGi Web容器对dm Server意味着什么

工程
罗伯·哈罗普
2009年6月1日

在我上一篇文章发表之后，许多人一直在问迁移到OSGi Web容器将对dm Server产生的影响。最常见的问题是：

  • 添加了什么？
  • 什么保持不变？
  • 发生了什么变化？
  • 我如何保持最新状态？

我将分别解决这些问题。如果您还有其他问题，请随时发表评论。

添加了什么？

与Web容器RI集成将使dm Server可以访问OSGi Web容器标准的所有功能。这包括有关如何处理WAR的标准模型，对webbundle URL方案的支持以及对Web容器扩展器的支持。

我正在探索RI的一些不错的增值功能，包括使用ConfigAdmin进行动态配置，这是一个全面的MBean接口，用于内省已部署的Web捆绑包，以及EventAdmin集成以监视生命周期事件。所有这些功能都将添加到dm Server以及RI中。

什么保持不变？

您会很高兴知道，您在dm Server中从Web应用程序中学到的很多东西都保持不变。

使用dm Server部署程序

除了支持Webbundle URL，仍可以使用dm Server部署程序部署WAR文件。支持进入部署程序的所有路径，包括拾取目录，部署程序MBean和管理控制台。

使用dm Server部署程序进行部署时，将从配置的存储库链中可用的捆绑包中自动安装WAR文件依赖项。

WAR部署模式

程序员指南中提到的所有WAR类型都保留了-实际上它们是Web容器标准的一部分。

系统包导入

使用dm Server部署的WAR将自动导入所有已配置的系统程序包，即使此功能未使其成为标准功能。如果使用Webbundle URL进行部署，则可以使用URL参数触发系统软件包导入。我希望规范将包括该领域的一些标准行为

发生了什么变化？

我们的目标是使dm Server中的大多数功能保持相同，但是迁移到Web容器确实需要进行一些更改。同时，我们利用代码重做功能来集成一些我们从用户那里看到的更流行的功能请求。

Web模块被删除

最大的变化是删除了Web模块 。我们倾向于支持基于标准的方法，现在我们已经能够与OSGi联盟合作，为OSGi上的Web应用程序创建基于标准的方法，因此我们将优先于特定于dm Server的方法迁移到该方法。解。

对于那些你们谁今天正在使用的网络模块，我听到了什么功能，你最喜欢的，并会伤心失去真正的兴趣。没有理由不能在Web Container Web捆绑包的顶部重做重要的Web模块功能。

切换到Tomcat配置格式

在1.0.x行中，使用JSON配置文件格式配置dm Server中嵌入的Tomcat实例。我们的许多用户都要求我们切换回使用Tomcat的XML格式。Web容器RI使用标准的Tomcat格式，并且dm Server切换到Web容器时，它也会切换配置文件格式。

我仍在最后确定配置文件存储位置的确切细节。我希望能够使用可以从ConfigAdmin填充的占位符来参数化Tomcat配置文件

我如何保持最新状态？

紧跟进度的最简单方法是跟踪Web容器和dm Server Web的SVN存储库。您可以通过以下URL访问这些存储库：

  • OSGi Web容器-https: //anonsvn.springsource.org/svn/dm-server-osgi-web-container/trunk
  • dm服务器网站-https: //anonsvn.springsource.org/svn/dm-server-web/main-branches/jersey

我会定期在这里博客，您可以使用#osgi和#dmserver跟踪Twitter上的进度 。
