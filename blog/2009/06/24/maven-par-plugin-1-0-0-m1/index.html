Maven PAR插件1.0.0。M1

工程
本·黑尔
2009年6月24日

在引入SpringSource dm Server（当时的应用程序平台）后不久，Thorsten Maus创建了一个用于创建PAR文件的Maven插件 。这是社区做出的巨大贡献，我们甚至在dm Server 1.0.x 入门指南中使用了它。

随着dm Server 2.0里程碑的发展，我们一直在向PAR文件添加新功能。最有趣的功能是，现在一个PAR文件可以包含的不仅仅是JAR文件。随着OSGi RFC 66的引入，dm服务器已弃用了Web模块 ，而支持使用.war扩展名的标准Web容器文件。因此，需要更新PAR插件以支持这些不同类型的依赖关系。Thorsten慷慨地同意将其代码捐赠给SpringSource，我很高兴宣布1.0.0。Apache许可的Maven PAR插件的M1版本。


  • Git- git：//git.springsource.org/maven-par-plugin/maven-par-plugin.git


用法


该插件的使用仅发生了微小变化，现在可以默认所有所需信息。

资料库

要获取插件，您需要将SpringSource Enterprise Bundle存储库引用为

<pluginRepositories>
    <pluginRepository>
          <id>com.springsource.repository.bundles.milestone</id>
          <name>SpringSource Enterprise Bundle Repository - SpringSource Bundle Milestones</name>
          <url>http://repository.springsource.com/maven/bundles/milestone </url>
     </pluginRepository>
</pluginRepositories>

构建插件

接下来，您需要将包装类型设置为par ，并将插件添加到构建插件列表中。

<build>
     <plugins>
          <plugin>
               <groupId>org.apache.maven.plugins</groupId>
               <artifactId>maven-par-plugin</artifactId>
               <version>1.0.0.M1</version>
          </plugin>
     </plugins>
</build>

默认情况下，所有PAR清单标头都是使用POM中的默认值填充的，但是可以用配置覆盖。

标头 POM元素
Application-SymbolicName $ {project.artifactId}
应用名称 $ {project.name}
应用说明 $ {project.description}
应用版本 $ {project.version} 
<build>
     <plugins>
          <plugin>
               <groupId>org.apache.maven.plugins</groupId>
               <artifactId>maven-par-plugin</artifactId>
               <version>1.0.0.M1</version>
               <configuration>
                   <applicationSymbolicName>override.symbolic.name</applicationSymbolicName>
                   <applicationName>Override Name</applicationName>
                   <applicationDescription>Override Description</applicationDescription>
                   <applicationVersion>1000.0.0.override</applicationVersion>
               </configuration>
          </plugin>
     </plugins>
</build>

还有一个配置标记，用于在PAR文件中包含的文件名之间选择$ {groupId}。$ {artifactId}-$ {version}。$ {type}和$ {artifactId}。$ {type} 。该值默认为false 。

帕尔

接下来，添加依赖项列表。该插件打包了POM中列出的所有直接依赖项。它不在乎作用域或可传递依赖项。

<dependencies>
    <dependency>
        <groupId>com.springsource.dmserver</groupId>
        <artifactId>greenpages.app-solution</artifactId>
        <version>${project.version}</version>
    </dependency>
    <dependency>
        <groupId>com.springsource.dmserver</groupId>
        <artifactId>greenpages.jpa-solution</artifactId>
        <version>${project.version}</version>
    </dependency>
    <dependency>
        <groupId>com.springsource.dmserver</groupId>
        <artifactId>greenpages.db-solution</artifactId>
        <version>${project.version}</version>
    </dependency>
    <dependency>
        <groupId>com.springsource.dmserver</groupId>
        <artifactId>greenpages.web-solution</artifactId>
        <version>${project.version}</version>
        <type>war</type>
    </dependency>
</dependencies>

建立

运行mvn package 。

[INFO] [par:par]
[INFO] Assembling Artifacts for PAR 'solution/greenpages/target/greenpages-solution-1.0.0.SNAPSHOT.par'
[INFO]   Added 'greenpages.app-solution.jar'
[INFO]   Added 'greenpages.jpa-solution.jar'
[INFO]   Added 'greenpages.db-solution.jar'
[INFO]   Added 'greenpages.web-solution.war'
