使用Spring BlazeDS集成1.0

工程
杰里米·格莱尔
2009年6月11日

今天，我们宣布了开源Spring项目组合中最新成员Spring BlazeDS Integration的1.0 GA版本公开发布。与此事件相对应，我认为是时候更新我之前的入门文章了。回顾一下：

该项目的目的是使使用Adobe Flex作为前端客户端的构建基于Spring的Rich Internet Application更加容易。它旨在通过为使用开源Adobe BlazeDS项目及其强大的远程处理和消息传递功能以及熟悉的Spring编程模型提供一流的支持来实现此目的。

以Spring BlazeDS集成进行测试

自从第一个M1版本发布以来，我们确实扩展了功能集，使其包括：


  • 完整的Spring Security集成

  • 异步消息支持（具有3种不同的消息目标类型）

  • 完整的XML配置名称空间

  • 基于注释的远程配置选项

  • 众多高级定制挂钩

现在，与项目发行版一起提供的是与Adobe合作构建的示例的完整集合，这些示例演示了各种功能（称为Spring BlazeDS集成测试驱动器）的用法。这些示例是启动和运行项目的好方法，在这里，我将为您快速介绍所有内容如何组合在一起。

如果您想继续使用IDE，请继续下载该发行版，并按照以下说明使用Maven构建示例并将其导入Eclipse。最终结果是，您将有很多新项目导入到Eclipse工作区中。大多数项目都是针对单个Flex示例的（即，它们包含.mxml和.as源并编译为.swf文件）。在“ testdrive”项目中可以找到可部署WTP的实际Web应用程序结构，这是我们首先要关注的地方。

在项目中值得一看的第一件事是/testdrive/src/main/webapp/WEB-INF/web.xml。在这里，您将看到通过ContextLoaderListener配置的根Web应用程序上下文的相当典型的设置，Spring Security过滤器链的基本设置以及Spring DispatcherServlet的此配置：


<servlet>
    <servlet-name>testdrive</servlet-name>
    <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
    <load-on-startup>1</load-on-startup>
</servlet>

<servlet-mapping>
    <servlet-name>testdrive</servlet-name>
    <url-pattern>/messagebroker/*</url-pattern>
</servlet-mapping>

这代替了在典型的独立BlazeDS应用程序中可以找到的MessageBrokerServlet配置。/ messagebroker / *路径的映射对应于BlazeDS AMF通信通道的典型设置，您可以在/testdrive/src/main/webapp/WEB-INF/flex/services-config.xml中找到该设置，例如：


<channel-definition id="my-amf" class="mx.messaging.channels.AMFChannel">
    <endpoint url="http://{server.name}:{server.port}/{context.root}/messagebroker/amf"
        class="flex.messaging.endpoints.AMFEndpoint"/>
</channel-definition>

这是BlazeDS的主要配置文件。在样本中检查此文件时，请注意，“服务”部分中没有引用remoting-config.xml或messages-config.xml。Spring BlazeDS集成的好处之一是，以前可以在特定于BlazeDS的文件中定义的设置现在可以通过提供的Spring XML配置名称空间和Java批注完全定义。这样可以减少很多思想上的上下文切换，此外，如果您使用免费的SpringSource Tool Suite ，则在编辑Spring配置文件时会获得完整的代码完成支持。

现在让我们看一下该示例的Spring应用程序上下文的某些方面。

使用从/ testdrive / src / main / webapp / WEB-INF / spring中找到的* -context.xml文件组装的根Web应用程序上下文以及本地的子应用程序上下文来设置该应用程序。前面提到的DispatcherServlet，并从/testdrive/src/main/webapp/WEB-INF/testdrive-servlet.xml组装（其名称和位置由惯例确定）。我们已将所有特定于Flex的配置分离到此子上下文中。

让我们进一步检查testdrive-servlet.xml。该文件以必要的XML序言开头，以设置标准的Spring“ beans”配置名称空间以及新的“ flex”名称空间。然后，您会看到的第一件事是BlazeDS MessageBroker的基本设置：


<flex:message-broker>
    <flex:message-service
        default-channels="my-streaming-amf,my-longpolling-amf,my-polling-amf" />
    <flex:secured />
</flex:message-broker>

message-broker标签触发了MessageBroker作为Spring托管Bean的引导启动，并自动设置了所有必需的Spring MVC基础结构，包括HandlerMapping和HandlerAdapter 。您可以通过可选的属性和标签（例如，services-config.xml的位置，映射到MessageBroker的特定路径等）进一步自定义内容，但是在通常情况下，这就是所需要的。

消息服务子标记正在为BlazeDS MessageService设置默认的通信通道（按优先顺序）。该标签是完全可选的，但是通常需要类似的设置，因为发布/订阅消息传递的通信通道要求往往与直接远程处理的要求不同。有关某些准则，请参阅BlazeDS文档的本部分 。

安全标签是启用与/testdrive/src/main/webapp/WEB-INF/spring/security-context.xml中定义的现有Spring Security设置集成所需要的全部。有了这个标签，通过使用Flex客户端ChannelSet API驱动的身份验证请求将被路由到Spring Security AuthenticationManager 。成功的身份验证返回的结果ActionScript对象包含一些其他有用的信息，例如用户授予的权限的数组。（本文中没有提供其他一些安全功能和配置选项，因此，如果您需要更多详细信息，请参考相关文档 。）

一旦完成了基本设置，就可以开始创建远程处理和消息传递目标，以将Flex客户端应用程序连接到Spring支持的服务。如果您查看/testdrive/src/main/webapp/WEB-INF/spring/services-context.xml，您将看到几个数据访问对象的定义，其中包括：


<bean id="contactService" class="org.springframework.flex.samples.contact.ContactDAO">
    <constructor-arg ref="dataSource" />
</bean>

然后，可以使用testdrive-servlet.xml中的以下内容公开该基本Spring bean以便将其远程处理到Flex客户端：


<flex:remoting-destination ref="contactService" />

这会将Bean公开为名为“ contactService”的远程目标（默认情况下，目标名称与导出的Bean的ID相同）。要从客户端访问此目标，我们只需要使用Flex RemoteObject类。例如，请参见/insync01/src/main/flex/insync01.mxml中的以下代码段：


<mx:RemoteObject id="ro" destination="contactService"/>

<mx:ApplicationControlBar width="100%">
    <mx:TextInput id="searchStr"/>
    <mx:Button label="Search" click="ro.findByName(searchStr.text)"/>
</mx:ApplicationControlBar>

<mx:DataGrid id="dg" dataProvider="{ro.findByName.lastResult}" width="100%" height="100%"/>

如您所见，连接RemoteObject后 ，您可以轻松地在其上调用方法并将结果绑定到Flex UI控件（例如DataGrid） 。Spring管理的MessageBroker负责处理传入HTTP消息的路由以及AMF和Java之间的序列化。基于POJO的Spring编程模型在服务器端保持不变，这在/testdrive/src/main/java/org/springframework/flex/samples/contact/ContactDAO.java中的“ findByName”方法的实现中很明显：


public List<Contact> findByName(String name) {
    return this.template.query("SELECT * FROM contact WHERE UPPER(CONCAT(first_name, ' ', last_name)) LIKE ? ORDER BY first_name, last_name", 
        this.rowMapper, "%" + name.toUpperCase() + "%");
}

异步pub / sub样式的通信同样简单。Spring BlazeDS集成为三种不同的消息传递适配器提供集成支持：


  1. BlazeDS ActionScriptAdapter用于基本AMF消息，包括使用提供的MessageTemplate从简单的POJO将这些消息推送到订阅的客户端的功能

  2. JmsAdapter用于连接到Spring管理的JMS目标

  3. 用于连接到Spring Integration MessageChannel的IntegrationAdapter


使用XML命名空间以与远程处理目标类似的方式来设置使用这些适配器的消息目标。例如，请参见testdrive-servlet.xml中的以下定义：


<flex:message-destination id="chat" />

这将设置一个名为“聊天”的基本AMF目标。定义此目标后，Flex客户端可以使用Producer和Consumer API通过它进行通信。要查看如何使用此特定目的地，请查看/chat/src/main/flex/chat.mxml中的Flex聊天示例源。

使用标签的jms-message-destination变体来布线由JMS目标支持的消息目标是相似的：


<flex:jms-message-destination id="jms-chat" jms-destination="chatTopic" />

该版本引用了Spring管理的JMS主题（在这种情况下，此主题在/testdrive/src/main/webapp/WEB-INF/spring/infrastructure-context.xml中定义）。在/jmschat/src/main/flex/jmschat.mxml中查看Flex jms-chat示例的源代码，您会注意到它与使用基本AMF“聊天”目的地的版本相同。

同样，连接到Spring Integration MessageChannel只需使用Integration-message-destination标签即可。一定要检查一下测试驱动器中的“ POJO Messaging”示例，以了解将简单的POJO消息处理程序引入混合是多么容易。
