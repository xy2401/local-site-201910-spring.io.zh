SpringSource TC服务器6.0.20。现在可用

工程
吉姆·贾吉尔斯基
2009年7月29日

今年4月，SpringSource全面推出了我们的第一版tc Server版本6.0.19。答：该产品取得了巨大的成功 ，部分原因是基于每天都在使用的，经过验证的，真正的Apache Tomcat基础上，该基础已经有很多人（并且越来越多地一直在使用）。正如我们在先前的博客文章中介绍6.0.19所概述的那样。答：我们已经通过扩展扩展了久经考验的真正核心，这些扩展使其更易于使用，并为Tomcat提供了面向生产的关键扩展。而且，我们已经通过SpringSource管理产品的企业级，生产级管理和监视功能来建立坚实的基础。

跟上Apache Tomcat


我们继续专注于客户需要的定期维护。对我们客户而言，最优先的要求是密切跟踪tc Server所基于的Apache Tomcat版本。Apache Tomcat于5月底发布，经过社区的全面测试以及SpringSource内的扩展生产级别测试，我们现在正在提供tc Server 6.0.20版。一种。

一些新的钟声和口哨声


我们没有到此为止。此版本还引入了对tc Server的一些改进和扩展。有关更完整的描述，请转到在线文档的“ 新增和更改的功能”部分。

  • tc服务器版本控制和迁移-使您可以从一个已安装的tc服务器二进制文件（CATALINA_HOME）创建和管理tc服务器实例（CATALINA_BASE）的实例脚本得到扩展，以允许您安装多个版本的tc Server（例如6.0）。 19A和6.0.20。A）并轻松在任何已安装实例之间交换特定的应用程序实例。请记住，与其他应用程序服务器容器不同，tc Server的磁盘占用空间很小，因此完全不必维护多个已安装实例。

  • server.xml，context.xml和web.xml中的加密密码-添加了在配置文件中存储和处理加密密码的功能。

  • 允许使用Oracle代理身份验证功能的数据源。

  • 在实例创建时创建* nix boot.rc文件模板，以便更轻松地将实例集成到系统启动中。

  • tc Server Application Server仅针对只安装应用程序服务器的客户下载。该下载文件远小于完整堆栈。

    • 支持平台


    我们为6.0.19支持的所有内容。6.0.20也支持A（请参阅支持的平台 ）。在GA。此外，我们的质量检查团队已准备就绪，可以根据6.0.19的直接客户反馈来认证和支持新平台。发布。

    期货


    我们已经在计划下一个维护更新（暂时称为6.0.20。B，但如果此期间有Apache Tomcat版本，则可能会发生变化），下一个将包含错误修复和对中央管理服务器的一些小改进。下一步，我们计划引入有关服务器和应用程序配置以及回滚等重要功能。我们很乐意在这里的评论中或在tc Server论坛组中听到您对您想看到的内容的输入。
