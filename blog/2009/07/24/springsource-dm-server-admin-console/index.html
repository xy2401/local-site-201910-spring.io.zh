SpringSource dm服务器管理控制台

工程
克里斯托弗·弗罗斯特
2009年7月24日

新的功能

每晚的构建中都提供了对SpringSource dm Server管理控制台的一组更新。这也显示了Plans和RFC66 Web容器的使用 。有两个新功能可用，第一个功能使您可以查看dm Server系统可能产生的任何服务转储，第二个功能使您可以探索OSGi中捆绑软件的状态。

转储可能包含许多转储项，例如堆栈跟踪，线程转储，具体取决于初始原因。可在管理控制台中查看转储项目，只需在左侧选择感兴趣的转储，然后在右侧选择转储项目即可显示。

[caption id =” attachment_2519” align =” aligncenter” width =” 785” caption =““转储的'摘要'项目视图。”] [/字幕]

有些转储项目不是纯文本格式。其中的一项是转储时OSGi解析状态的zip。当发生转储时，状态查看器也可以用来浏览OSGi系统的状态。查看状态可让您在分发包及其各种导入/导出的包中浏览。

它也可以用于查看正在运行的实例的实时状态。实时视图提供了OSGi状态转储中不存在的其他信息，包括捆绑软件是否由Spring提供动力以及OSGi中捆绑软件的状态。

[caption id =” attachment_2529” align =” aligncenter” width =” 749” caption =““某些捆绑包的视图，其中包含有关碎片/主机，Spring上下文和状态的信息。”] [/字幕]

即将到来及任何要求...

管理控制台正在积极开发中，并且状态查看器当前正在改进中，以使诊断问题更容易，并显示有关任何捆绑软件和任何Spring支持捆绑软件的bean的服务的更多信息。我们已经讨论了其他将来的功能，例如存储库查看器，因此您可以看到各种已配置存储库中可用的内容以及它们形成的链。此外，还可以使用Config Admin查看器查看服务器配置的内容。

我们真的很想听到您想看到的其他信息，这将使您的OSGi生活更加轻松。请发表评论或在论坛上进行讨论 ，或者仅提出JIRA ，我们将进行看看。如果您想知道夜间版本中何时有可用的新功能，而不必等待下一个里程碑发布和博客，则可以在Twitter上关注我。
