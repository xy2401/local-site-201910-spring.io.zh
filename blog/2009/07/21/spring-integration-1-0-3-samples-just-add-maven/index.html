Spring Integration 1.0.3示例：仅添加Maven

工程
马克·费舍尔
2009年7月21日

Spring Integration 1.0.3现在可用。您可以在Spring Integration 主页上找到下载链接，参考手册等。此版本包含许多更改 ，但是在此博客中，我将特别关注一个更改 。从该版本开始，这些示例是完全独立的且已启用Maven。这意味着您可以下载发行版，进入“ samples”目录，运行“ mvn install”，然后将项目导入到Maven感知的Eclipse实例中，例如SpringSource Tool Suite 。这是分步细分...

安装和运行示例

  1. 确保已安装Maven并且在路径中（需要2.0.9或更高版本）。如果不是，请下载该文件并按照设置说明进行操作： http : //maven.apache.org
  2. 如果您还没有支持Maven的Eclipse版本，则可以下载SpringSource工具套件 （STS），该套件将立即支持这些项目（STS甚至包括对启用OSGi的示例的支持）。另外，您可以手动将Maven插件（例如m2eclipse）添加到现有的Eclipse安装中。
  3. 下载Spring Integration Samples并解压缩。
  4. 在解压缩的“样本”目录中，运行“ mvn install”。您最终应该看到类似于以下内容的输出：
  5. 一旦STS / Eclipse启动并运行，请选择“文件->导入...”，然后在向导中选择“常规->现有项目到工作区”。然后浏览到解压缩的“样本”目录，您将看到默认情况下选择的所有项目： 要么全部接受，要么选择要导入的单个样本，在构建工作区片刻之后，您应该可以运行了。*

* 注意：如果导入“ osgi-inbound”项目，则将在该特定项目上看到一些错误（但在工作区构建完成后，其他项目上将不会看到任何错误）。在配置运行时和捆绑软件存储库之后，将解决这些错误。如果您想浏览OSGi示例，请参考《参考手册》中的专用章节。另外，我们将在几天内发布一个博客，详细介绍这些项目。

每个示例在“ Demo”类中都有一个主要方法（“ osgi-inbound”和“ osgi-outbound”除外，它们打算在OSGi环境中部署-即将在其后的博客文章中介绍）。例如，“ helloworld”项目具有“ HelloWorldDemo”。在每个演示类的javadoc中，您将找到该单独样本中展示的功能的简要概述。

*注意：所有示例均具有某些基本模式，这些模式对于底层Spring Integration核心至关重要：


  • 消息 ：Spring Integration Messages封装了POJO有效负载和标头Map（ 参考 ）。

  • 消息通道 ：Spring Integration包括许多用于点对点和发布-订阅的消息通道选项。有些包含用于缓冲的队列，而另一些则直接分配给订阅者（ 参考 ）。

  • 消息端点 ：高层，包括所有连接到通道以进行输入和/或输出的组件。

  • 消息映射器 ：Spring Integration将入站消息绑定到方法参数，并将方法返回值绑定到消息有效负载和/或头。

  • Message Dispatcher ：在Spring Integration中，没有队列的通道使用Message Dispatchers来调用其订阅者。

  • 管道和过滤器 ：这是描述消息处理组件通过通道以松耦合方式连接的最通用模式。

  • 消息总线 ：Spring Integration本质上将Spring ApplicationContext变成了轻量级的消息总线，在其中托管了所有其他这些组件。
