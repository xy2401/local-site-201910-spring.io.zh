推出tc Server开发人员版-带有Spring Insight

工程
乔恩·特拉维斯
2009年10月21日

我们很荣幸为Spring社区提供一个新工具，该工具可以帮助他们更快地将其应用程序投入生产： tc Server Developer Edition是您熟悉和喜爱的标准100％Tomcat，以及一个新的Spring Insight控制台。

Spring Insight可在逐个请求的基础上深入了解应用程序的实际活动。对于任何请求，您都可以查看它进行的所有JDBC查询，渲染所需的时间或任何主要Spring Bean的计时。

当您单独查看特定请求时，这种细粒度的信息非常有价值，但是在成千上万的群中很难找到特定请求。幸运的是，Spring Insight还具有10k英尺的视图，可以快速将您指向问题区域。

Spring Insight的“应用程序运行状况”屏幕在性能测试环境中效果很好。它将向您显示哪些Spring MVC控制器不正常，并允许您深入研究特定的问题请求。只需单击几下，您就可以从10k脚视图导航到特定的远程Web服务呼叫。

要查看运行中的tc Server Developer Edition，只需单击下面的图像以开始截屏 ：

我们认为该工具对于希望获得应用程序内部运行时运行时可视性的开发人员和质量检查人员而言都是有价值的。Spring Insight为质量检查工程师提供了后视镜。他们可以正常测试他们的应用程序，并且在遇到问题时，只需导航至Spring Insight以查看原因。

现在来一些技术优势。

Spring Insight使用AspectJ来加载Web应用程序的加载时间。这意味着您不需要对应用程序进行任何更改即可使用Spring Insight。零。

Spring Insight会在内存中收集其数据，不需要访问数据库或磁盘-这使尝试变得不容易！在将应用程序部署到Spring Insight时，您需要为其提供更多内存，以容纳跟踪，响应时间等的存储。当达到内部限制时，Spring Insight将放弃跟踪以保持较低的内存占用。它提供了配置选项以调整内存占用量。

Spring Insight不应该在生产中使用。它没有安全性概念，并且能够显示敏感的低层信息。

应用程序运行状况提供了一些有关响应时间的非常有用的信息。所有Web请求都由最终处理该请求的Spring MVC控制器进行分组。这些分组称为端点，代表了进入应用程序的主要入口。预览版仅支持Spring MVC端点，但我们计划在最终版之前添加更多类型。

这给我带来了该体系结构的一个不错的功能：可插入性。分析Hibernate或JDBC或Spring-MVC的能力全部来自一组插件，这些插件被放入tc Server的目录中。在SpringOne 2GX期间，我们有许多要求打开此插件体系结构，并允许人们访问以编写自己的插件。对我们来说，这是当务之急，我们将在以后的版本中继续努力。

该项目进展迅速，我们收到的反馈非常好。请试一试 ， 让我们知道您的想法 。绝对让我们知道它是否不适用于您的应用程序。

性能洞察力，我们来了！
