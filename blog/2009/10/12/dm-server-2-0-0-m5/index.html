dm服务器2.0.0。M5

工程
本·黑尔
2009年10月12日

dm服务器 2.0.0。M5现在可用，可以从此处下载。

我们即将结束2.0.0发布周期，因此此里程碑中的大部分工作都集中在锁定用户交互模型上。看看M5发行说明，了解我们一直在努力的全部细节。请在我们的博客， 论坛和JIRA上保留您的反馈意见。

还请观看博客，以获取有关本“新功能和值得注意的功能”中引用的许多新功能的详细信息。

新的和值得注意的

部署配置文件

现在可以在拾取目录或任何已配置的存储库中部署属性文件。然后，可以在OSGi的ConfigurationAdmin中以指定为文件名的PID（不带.properties扩展名）访问这些文件的内容。

计划属性和属性

计划现在可以具有用于替换的全局定义的属性。

<attributes>
	<attribute name="version" value="[1.0.0, 2.0.0)"/>
</attributes>

<artifact type="bundle" name="org.springframework.context.support" version="${version}"/>

计划中的工件可以具有定义的属性，这些属性在部署时会转换为Bundle清单头。

<artifact type="bundle" name="my.webapp" version="[1.0.0, 2.0.0)">
	<property name="Web-ContextPath" value="/foo"/>
</artifact>

管理控制台

管理控制台的Artifacts页面经过精心整理，可以将用户安装的工件的依赖关系呈现为树。另外，这些树的每个级别都延迟加载，从而提高了加载时间性能。

命令行外壳

dm Server现在随附命令行外壳程序，以替换以前使用的Equinox控制台。可使用-shell startup命令在本地访问此控制台。

$ bin/startup.sh -shell

基础设施搬迁

我们还对dm Server基础结构的各个部分进行了一些更改。首先，存储库区域已简化为仅具有ext和usr文件夹。usr文件夹配置为受监视的存储库，这意味着您无需重新启动服务器即可引用已添加到其中的捆绑软件。

config /目录已展平，凭证存储和密钥库已移动。现在，密钥库位于config / keystore中 。现在，凭据存储位于config / com.springsource.kernel.users.properties中 ，并且以属性文件格式驻留了指定的用户和角色。

##################
# User definitions
##################
user.admin=springsource

##################
# Role definitions
##################
role.admin=admin

可运行的dm内核

最后但并非最不重要的一点是，dm Kernel现在是一个独立的可运行发行版。您可以直接从源代码构建zip包，也可以从此处下载。
