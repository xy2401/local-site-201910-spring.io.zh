Grails 1.1发布

工程
格雷姆·罗彻（Graeme Rocher）
2009年3月10日

紧随Groovy 1.6发行版之后 ，我们很高兴地宣布Grails 1.1 final已经发布，可以从Grails站点上获得 。发行说明中详细列出了许多改进。但是，一些关键的是：

独立GORM：现在可以在Grails外部使用Grails的ORM层（基于Hibernate构建）。在发行版的samples / petclinic-mvc目录中的Spring MVC应用程序内部有一个使用GORM的示例。该示例使用Spring配置启用GORM的SessionFactory，如下所示：

Maven和Ant + Ivy支持： Grails应用程序现在可以使用Java空间中两个最出色的构建工具来构建。将Grails应用程序集成到Java生态系统中变得非常容易，Grails现在涵盖了从构建到部署的整个应用程序生命周期中与Java的集成。

更好的插件： Grails体验的关键部分是插件生态系统 ，而现在有了Grails 1.1，该体验变得更好了。现在，插件是从项目元数据自动安装的，安装过程是过渡性的（这意味着插件依赖关系是自动安装的），并且已添加了对全局插件（跨多个应用程序的插件）的支持。

Spring命名空间支持：除了支持Spring的本机XML用于定义新bean之外，Grails还支持Groovy DSL用于定义bean定义。该DSL已扩展为支持Spring名称空间：


beans = {
    xmlns aop:"http://www.springframework.org/schema/aop"

    fred(Person) {
       name = "Fred"
       age = 45
    }
    birthdayCardSenderAspect(BirthdayCardSender)

     aop {
         config("proxy-target-class":true) {
             aspect( id:"sendBirthdayCard",ref:"birthdayCardSenderAspect" ) {
                   after method:"onBirthday",
                   pointcut: "execution(void ..Person.birthday()) and this(person)"
              }
         }
     }
}

JSP标记库支持：现在可以在GSP内使用任何JSP标记库，从而如今更容易迁移到Grails。您甚至可以在Grails（基于Spring MVC构建）上使用标准的Spring MVC标签库：


<%@ taglib prefix="form" uri="http://www.springframework.org/tags/form" %>

对于Groovy，Grails和Spring来说，这是激动人心的时刻，如果您想了解更多信息，那么您应该参加SpringOne Europe和/或gr8conf ，这两者都提供了Groovy和Grails的大量介绍。
