建筑弹簧3

工程
克里斯·比姆斯
2009年3月3日

更新-2012年2月21日：Spring Framework已移至GitHub ，对于3.2.x开发已从Ant移至Gradle。看一下README的源代码部分中的建筑物 （大大简化了！）。说明。

介绍

正如Juergen 上周宣布的那样 ，Spring 3.0 Milestone 2现在可用。在本文中，我将分六个步骤向您展示如何下载和构建最新的Spring 3源代码，以及如何在Eclipse中启动并运行开发环境。

最后，我们将讨论跟踪Spring 3开发的最佳方法，如何提交错误和改进请求以及在Maven环境中针对本地构建的Spring 3二进制文件进行编译的注意事项。


先决条件

在开始以下步骤之前，请确保您具有以下各项：

  • Subversion客户端，版本1.5或更高版本
  • Java 6或更高版本（Spring 3需要Java 5才能运行，但需要Java 6才能构建）
  • Ant 1.7或更高
  • Eclipse（可以使用3.3或3.4），也可以选择使用最新的SpringSource Tool Suite


步骤1：检查来源

您可能已经知道，Spring源代码现在托管在springsource.org上，与以前的CVS所控制的Spring版本不同，Spring 3现在使用Subversion进行管理。

在检查源代码之前，让我们快速看一下存储库的布局。

cbeams @ kaizen：〜 $ svn ls https://src.springsource.org/svn/spring-framework
分支机构/
标签/
树干/

如您所见，该存储库以传统的Subversion形式进行布局，带有分支，标签和主干目录。如果您需要同步到特定的里程碑或版本，则始终可以在tags /目录中找到它们：

cbeams @ kaizen：〜 $ svn ls https://src.springsource.org/svn/spring-framework/tags
弹簧框架-3.0.0。M1 /
弹簧框架-3.0.0。M2 /

让我们同步到最新的trunk /：

cbeams @ kaizen：〜 $ svn co https://src.springsource.org/svn/spring-framework/trunk spring-framework
…

签出完成后，您将找到以下目录列表：


cbeams @ kaizen：〜 $ cd spring-framework
cbeams @ kaizen：〜/ spring-framework $ ls -1

有经验的Spring黑客会立即注意到目录结构与以前的版本有些不同。现在，Spring的内部结构更加模块化-每个org.springframework。*目录代表一个单独的项目或模块，每个目录都有各自的源目录，测试套件和构建构件。在使用框架时，这是个好消息：例如，对org.springframework.core的更改可以独立于其他模块中的数千个无关单元测试进行单元测试。这带来了更加便利和愉悦的开发经验。这也意味着查找现有的单元测试往往是一个更直观的过程。自己戳一下，看看您的想法！


步骤2：建立

与许多其他Spring项目一起，Spring 3是使用Ant和Ivy的组合构建的，我们简称为“ Spring Build”。除了构建基础结构的详细信息之外，启动构建很简单：


cbeams @ kaizen：〜/ spring-framework $ cd build-spring-framework
cbeams @ kaizen：〜/ spring-framework / build-spring-framework $ ant
[…]
建立成功

根据连接速度，此初始构建将花费20分钟或更长时间。您会注意到，Ivy从SpringSource Enterprise Bundle存储库中自动地自动下载所有依赖项，这可能需要一些时间。

随着构建的进行，继续并完成步骤3-5，以设置Eclipse / STS工作区。


步骤3：设置新的工作区（可选）

因为我们将更改默认的JRE并引入一个新的classpath变量，所以建议您在Eclipse中专门用于Spring 3的地方设置一个新的工作区，但这不是严格要求的。如果您希望跳过此步骤并使用现有的工作空间，请继续执行步骤4。

转到文件>切换工作区>其他…

为您的Spring 3工作区指定一个新位置。您可以选择任何目录；正如您在下面看到的那样，为了方便起见，我选择在我的春季结帐中创建一个“工作区”目录。


步骤4：创建IVY_CACHE类路径变量

在这一步中，我们将在Eclipse中创建一个classpath变量，各种Spring 3项目将使用该变量来确定它们所依赖的jar的位置。正如我们在前面观察Ant构建进度时所注意到的那样，Ivy正在下载许多jar文件-这些jar文件正被下载到Spring签出根目录下的ivy-cache / repository目录中。

打开全局首选项对话框

输入“ classpath”以缩小菜单项的列表，然后选择“ Classpath Variables”

单击“新建”以创建新的类路径变量，然后键入以下值

名称： IVY_CACHE
路径： [your-checkout-root] / ivy-cache / repository


步骤5：将工作区默认JRE设置为Java 6

如上所述，Spring 3需要Java 6来构建。确保处理此依赖关系的最简单方法是简单地将Eclipse工作区的默认JRE设置为Java 6。如果您的Eclipse安装已经存在这种情况，那么您当然可以跳过此步骤。

再次打开全局“首选项”对话框。这次，键入“ installed JRE”以缩小选项范围。

确保选择了1.6 JVM。如果已安装的JRE中没有列出1.6 JVM，请单击“添加...”，然后将Eclipse指向系统上Java 6的位置。


步骤6：将源导入到Eclipse

我们几乎已经准备好导入Spring源。首先让我们确保构建完成。回头看开始构建的窗口。它可能仍在进行中。如果是这样，请等待直到完成。完成后，您应该会看到以下内容：

请记住，您的时间将取决于您的连接速度。如果初始构建需要一段时间，请不要感到惊讶！

构建完成后，将项目导入Eclipse：

转到文件>导入，然后键入“现有”以缩小选项

选择“现有项目到工作区”。出现文件选择对话框。选择Spring 3签出的根目录。Eclipse将遍历目录，并为每个Spring 3项目检测.project文件。完成后，您应该看到以下对话框：

取消选中第一个未标记的复选框，然后单击“完成”。

等待Eclipse导入项目。完成后，您应该会看到如下所示的Package Explorer，其中包含22个项目，没有错误。

而已！现在，您可以轻松获得Spring 3的源代码，并且可以随意浏览和编辑。玩得开心！


在Spring 3开发之后

与Spring开发保持联系的最佳方法是使用SpringSource的Fisheye服务通过RSS跟踪Subversion提交。

去 https://fisheye.springsource.org/browse/spring-framework/trunk ，然后单击右上角的RSS图标。

请注意，您可以订阅Spring源代码树中的任何节点。因此，如果您关心某个特定模块甚至单个软件包，则可以选择任何粒度接收通知。

为了快速参考，遵循所有Spring 3提交的RSS feed URL是 https://fisheye.springsource.org/changelog/~rss/spring-framework/trunk/rss.xml 。将其添加到供稿阅读器中，您就可以开始了。


提供有关Spring 3的反馈

随着里程碑的进展以及我们越来越接近Spring 3.0 GA，您的反馈越来越重要。如果您发现错误或对现有功能有改进要求，请花一点时间访问 http://jira.springsource.org并针对Spring Framework 项目创建一个新期刊。

如果可能，请附加一个简单的JUnit测试用例，该用例会重现所有错误。如果没有，请提供详细的复制步骤。

另外，在报告错误或改进时，请不要犹豫地亲自实施更改并附加补丁文件。我们喜欢从社区中获得贡献！

如果您将Subclipse Subversion插件用于Eclipse，则生成补丁文件非常容易：

从软件包资源管理器的上下文菜单中，转到“ 团队”>“创建补丁”…

在“创建补丁”对话框中，选择“保存在文件系统中”并指定一个文件

单击“下一步”以获取高级选项，然后选择“项目”作为修补程序根。

然后将生成的补丁文件附加到JIRA问题。您可以放心，补丁程序的问题具有自然的优先权;）


在Maven环境中使用本地构建的Spring 3二进制文件

最后，Maven用户将想知道如何最有效地将其本地构建的Spring 3二进制文件安装到其本地Maven存储库中。

从Spring检出目录的根目录，或从任何单个模块中，可以使用'install-maven-central'目标自动将工件从target /复制到本地m2存储库。


cbeams @ kaizen：〜/ spring-framework / org.springframework.core $ ant install-maven-central
生成文件：build.xml
[…被剥夺…]
install-maven-central：
[maven：install] [INFO]将/Users/cbeams/spring-framework/org.springframework.core/target/artifacts/org.springframework.core.jar安装到/Users/cbeams/.m2/repository/org/springframework/ org.springframework.core / 3.0.0。BUILD-20090302225538 / org.springframework.core-3.0.0。BUILD-20090302225538.jar
[maven：install] [INFO]将/Users/cbeams/spring-framework/org.springframework.core/target/artifacts/org.springframework.core-sources.jar安装到/Users/cbeams/.m2/repository/org/ springframework / org.springframework.core / 3.0.0。BUILD-20090302225538 / org.springframework.core-3.0.0。BUILD-20090302225538-sources.jar

建立成功
总时间：4秒

您可以结合使用“测试”和“安装-maven-中心”目标，以实现大致等同于“ mvn install”的功能：


cbeams @ kaizen：〜/ spring-framework / org.springframework.core $ ant test install-maven-central

注意：“ install-maven-central”假定您的本地存储库位于$ HOME / .m2 / repository的默认位置。


摘要

我希望您喜欢遍历Spring 3开发的整个过程，并且我认为您会发现，在上述详细的一次性设置之后，很容易针对Spring 3的源进行开发。我们鼓励您关注发展并通过JIRA与我们保持联系。您的反馈一直是Spring的命脉，我们对此表示感谢。保持它来！
