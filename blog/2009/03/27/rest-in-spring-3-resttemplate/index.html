Spring 3：REST

工程
Arjen Poutsma
2009年3月27日

在较早的帖子中 ，我在博客中介绍了我们添加到Spring @MVC 3.0版中的REST功能。后来，Alef 写了有关使用引入的功能向Pet Clinic应用程序添加Atom视图的文章。在本文中，我要介绍我们在里程碑2中添加的客户端功能。

RestTemplate

RestTemplate是用于客户端HTTP访问的中央Spring类。从概念上讲，它与JdbcTemplate ， JmsTemplate以及Spring Framework和其他项目组合项目中的各种其他模板非常相似。例如，这意味着RestTemplate一旦构建便是线程安全的，并且您可以使用回调来自定义其操作。

RestTemplate方法

模板的主要入口点以六个主要的HTTP方法命名：

HTTPRestTemplate
删除delete（字符串，字符串...）
得到getForObject（String，Class ，String ...）
头headForHeaders（字符串，字符串...）
选项optionsForAllow（字符串，字符串...）
开机自检postForLocation（字符串，对象，字符串...）
放put（字符串，对象，字符串...）

这些方法的名称清楚地表明了它们调用的HTTP方法，而名称的第二部分则表明了返回的内容。例如， getForObject（）将执行GET，将HTTP响应转换为您选择的对象类型，然后返回该对象。 postForLocation将执行POST，将给定对象转换为HTTP请求，并返回响应HTTP Location标头，在该标头中可以找到新创建的对象。如您所见，这些方法尝试实施REST最佳实践。

URI模板

这些方法中的每一个都以URI作为第一个参数。该URI可以是URI模板 ，并且可以使用变量将模板扩展为普通URI。模板变量可以两种形式传递：作为String变量arguments数组或Map 。字符串varargs变体按顺序扩展给定的模板变量，以便


String result = restTemplate.getForObject("http://example.com/hotels/{hotel}/bookings/{booking}", String.class, "42", "21");

将在http://example.com/hotels/42/bookings/21上执行GET。 map变量根据变量名称扩展模板，因此在使用多个变量或多次使用单个变量时更有用。例如：


Map<String, String> vars = new HashMap<String, String>();
vars.put("hotel", "42");
vars.put("booking", "21");
String result = restTemplate.getForObject("http://example.com/hotels/{hotel}/bookings/{booking}", String.class, vars);

还将在http://example.com/hotels/42/rooms/42上执行GET。

HttpMessageConverters

传递给方法getForObject（） ， postForLocation（）和put（）并从中返回的对象，并通过HttpMessageConverters从HTTP响应转换为HTTP请求。默认情况下，已注册用于主要mime类型和Java类型的转换器，但是您也可以编写自己的转换器并将其插入RestTemplate中。在下面的示例中，我将向您展示如何完成此操作。

使用RestTemplate从Flickr检索照片

我不会向您介绍RestTemplate的各种方法，而是向您展示如何使用它从Yahoo!的在线照片共享应用程序Flickr中检索照片。该示例应用程序在Flickr中搜索与给定搜索词匹配的照片。然后使用简单的Swing UI显示这些图片。要自己运行该应用程序，您需要创建一个Flickr帐户并申请API密钥 。

搜索照片

Flickr公开了各种API来操纵其庞大的照片库。flickr.photos.search方法允许您通过在http://www.flickr.com/services/rest?method=flickr.photos.search&api+key=xxx&tags=penguins上发出GET请求来搜索照片。输入您的API密钥和要搜索的内容（在这种情况下为企鹅）。如此一来，您将获得一个XML文档，其中描述了符合查询条件的照片。就像是：


<photos page="2" pages="89" perpage="10" total="881">
	<photo id="2636" owner="[email protected]" 
		secret="a123456" server="2" title="test_04"
		ispublic="1" isfriend="0" isfamily="0" />
	<photo id="2635" owner="[email protected]"
		secret="b123456" server="2" title="test_03"
		ispublic="0" isfriend="1" isfamily="1" />
	<photo id="2633" owner="[email protected]"
		secret="c123456" server="2" title="test_01"
		ispublic="1" isfriend="0" isfamily="0" />
	<photo id="2610" owner="[email protected]"
		secret="d123456" server="2" title="00_tall"
		ispublic="1" isfriend="0" isfamily="0" />
</photos>

使用RestTemplate，检索这样的文档非常简单：


final String photoSearchUrl =
   "http://www.flickr.com/services/rest?method=flickr.photos.search&api+key={api-key}&tags={tag}&per_page=10";
Source photos = restTemplate.getForObject(photoSearchUrl, Source.class, apiKey, searchTerm);

其中apiKey和SEARCHTERM是命令行给出两个字符串。此方法使用SourceHttpMessageConverter将HTTP XML响应转换为javax.xml.transform。Source （请注意，SourceHttpMessageConverter是在我们发布Spring 3.0 M2之后不久引入的，因此您必须获取最近的快照（或即将发布的M3）才能使用它。下面提供了示例项目，以通过Maven检索这些项目。

检索照片

接下来，我们将使用XPath表达式来检索文档的所有照片元素。为此，我们将使用Spring Web Services中的XPathTemplate 。我们将执行// photo表达式，返回出现在文档中任何位置的所有photo元素。NodeMapper是一个回调接口，将为文档中的每个照片元素调用其mapNode（）方法。在这种情况下，我们将检索此元素的server ， id和secret属性，并使用它们填充Map。最后，我们再次使用RestTemplate，以java.awt.image的形式检索照片。BufferedImage 。因此，当完成XPath评估时，生成的imageList将包含XML文档中每张照片的图像。


List<BufferedImage> imageList = xpathTemplate.evaluate("//photo", photos, new NodeMapper() {
    public Object mapNode(Node node, int i) throws DOMException {
        Element photo = (Element) node;

        Map<String, String> variables = new HashMap<String, String>(3);
        variables.put("server", photo.getAttribute("server"));
        variables.put("id", photo.getAttribute("id"));
        variables.put("secret", photo.getAttribute("secret"));

        String photoUrl = "http://static.flickr.com/{server}/{id}_{secret}_m.jpg";
        return restTemplate.getForObject(photoUrl, BufferedImage.class, variables);
    }
});

例如，给定上面给出的XML文档， imageList将包含4张图像。检索到的第一个图像的URL将是http://static.flickr.com/2/2636_ a123456_m.jpg ，第二个是http://static.flickr.com/2/2635_ b123456_m.jpg等。

转换影像

为了使代码正常工作，还需要做另外一件事情：我们将需要编写一个能够从HTTP响应中读取的HttpMessageConverter ，并从中创建一个BufferedImage 。使用Java Image I / O API这样做非常简单，我们只需要实现HttpMessageConverter接口中定义的read（）方法即可 。总体而言，我们的简单转换器如下所示：


public class BufferedImageHttpMessageConverter implements HttpMessageConverter<BufferedImage> {

    public List<MediaType> getSupportedMediaTypes() {
        return Collections.singletonList(new MediaType("image", "jpeg"));
    }

    public boolean supports(Class<? extends BufferedImage> clazz) {
        return BufferedImage.class.equals(clazz);
    }

    public BufferedImage read(Class<BufferedImage> clazz, HttpInputMessage inputMessage) throws IOException {
        return ImageIO.read(inputMessage.getBody());
    }

    public void write(BufferedImage image, HttpOutputMessage message) throws IOException {
        throw new UnsupportedOperationException("Not implemented");
    }

}

请注意，我们没有实现write（），因为我们不是上传图像，而是下载图像。现在，我们只需要将此转换器插入RestTemplate。我们在Spring应用程序上下文中执行此操作：


<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd">

    <bean id="flickrClient" class="com.springsource.samples.resttemplate.FlickrClient">
        <constructor-arg ref="restTemplate"/>
        <constructor-arg ref="xpathTemplate"/>
    </bean>

    <bean id="restTemplate" class="org.springframework.web.client.RestTemplate">
        <property name="messageConverters">
            <list>
                <bean class="org.springframework.http.converter.xml.SourceHttpMessageConverter"/>
                <bean class="com.springsource.samples.resttemplate.BufferedImageHttpMessageConverter"/>
            </list>
        </property>
    </bean>

    <bean id="xpathTemplate" class="org.springframework.xml.xpath.Jaxp13XPathTemplate"/>

</beans>

显示照片

最后阶段是在简单的GUI中显示照片。为此，我们使用Swing：


JFrame frame = new JFrame(searchTerm + " photos");
frame.setLayout(new GridLayout(2, imageList.size() / 2));
for (BufferedImage image : imageList) {
    frame.add(new JLabel(new ImageIcon(image)));
}
frame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
frame.pack();
frame.setVisible(true);
