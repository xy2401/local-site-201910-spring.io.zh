宣布SpringSource Tool Suite 2.0

工程
克里斯蒂安·杜普依斯（Christian Dupuis）
2009年3月17日

今天，我们很高兴地宣布SpringSource Tool Suite（STS）的2.0版本已全面上市。您可以在这里找到新闻稿。

在2.0迭代中，我们根据客户群和社区的反馈意见，重点关注了两个主要的功能区域：为基于Spring的应用程序开发提供最佳的环境，并提供基于OSGi和Windows Server 2003开发企业应用程序的工具。 SpringSource dm服务器。

我想使用此博客介绍STS 2.0的一些新Spring开发工具。新的OSGi开发工具已经在较早的博客中进行了介绍 。

STS中的Spring开发工具

STS 2.0具有许多针对日常开发人员活动的增强功能，可以帮助提高开发人员的生产力。

借助新的Visual Spring Configuration Editor（参见下文），由于新的用户辅助功能（如XML内容的图形表示，内容辅助，超链接和集成），现在可以更快，更轻松地创建和编辑Spring XML配置。文档。可视编辑器为每个Spring XML配置名称空间集成了选项卡，并提供了动态添加和删除名称空间的方法。

要创建或编辑单个XML bean定义或名称空间元素，开发人员可以使用UI元素，这些元素将提供有关所需配置参数和内联验证的指南。还可以使用“新建Bean创建向导”来加快Bean定义的创建和修改。此外，Visual Editor通过单击即可执行常见任务，从而提供帮助。例如，使用Spring DM配置名称空间将Spring Bean作为OSGi服务导出，并且可以轻松扩展以了解自定义的Spring配置名称空间。

对于那些喜欢使用XML的开发人员，SpringSource Tool Suite在Spring IDE的XML编辑功能的基础上提供了增强功能。STS 2.0可以按需验证XML内容，从而无需保存文件即可揭示验证问题。除了按需输入验证之外，STS还为报告的验证问题提供了相关的“快速修复”（请参见下文）。

使用新的快速修复程序，现在可以非常容易地首先以XML创建配置模板，然后让STS自动创建包括所有已配置属性的Java类。新的编辑和用户辅助功能可以帮助大大减少编辑Spring配置文件所需的时间，并使STS的XML编辑功能达到开发人员用来编辑Java文件的类似水平。

另外，STS带有Spring IDE Spring Explorer的扩展，可帮助在大型代码库中导航Spring批注（例如@ Service，@ Transactional或@Repository）的用法（请参见下文）。通过使用Bean批注分组，可以很容易地基于@RequestMapping批注导航复杂的URL到控制器的映射。

除了上面概述的Spring开发功能之外，开发人员还可以从STS 1.0功能中受益，该功能还提供用户帮助和指导。事实证明，以任务为中心的教程，运行时错误分析和体系结构查看工具对Spring开发人员来说特别有价值。

下载及安装

您可以从产品页面下载SpringSource Tool Suite 2.0。STS是完全集成的Eclipse发行版，建立在最新的Eclipse Ganymede SR2版本的基础上，并捆绑了Spring IDE，dm Server Tools和Eclipse AJDT。

除了STS 2.0，我们今天还发布了新版本的Spring IDE和dm Server Tools。为了使这两个安装尽可能简单，我们创建了一个包含两个插件的统一Eclipse更新站点。新的合并更新站点URL为： http://www.springsource.org/update/e3.4

当然，您也可以在知名的更新站点上找到新版本。由于Spring IDE和dm Server Tools都需要安装Eclipse WTP，因此最简单的做法是从“ 面向Java EE开发人员的Eclipse IDE ”软件包开始。该软件包带有所有必需的依赖项。
