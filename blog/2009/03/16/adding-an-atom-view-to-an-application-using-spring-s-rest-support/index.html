使用Spring的REST支持向应用程序添加Atom视图

工程
阿列夫·阿伦森（Alef Arendsen）
2009年3月16日

在Spring 3.0中，将通过REST支持来增强Spring MVC。这篇文章描述了如何使用REST支持在一个简单的示例应用程序之上实现AtomView。请按照以下逐步过程进行操作，以了解在Spring MVC中使用新的REST支持在简单应用程序之上实现AtomView有多么容易。

步骤1：下载应用程序框架

在此博客文章的底部附近，您将找到一个简单的下载，其中包含Web应用程序的框架。在内部，您将找到此应用程序所需的所有Spring 3.0二进制文件，以及Atom功能所需的一些其他功能。Spring二进制文件基于每晚构建，并且在Spring 3.0最终发布后可能会被最终构建替换。

接下来，使用“导入>将现有项目导入到工作区”向导（从“文件”菜单）在Eclipse中加载项目。该应用程序是一个简单的Eclipse动态Web项目，具有用于Spring MVC设置的所有基础结构。因此，如果您对Spring MVC熟悉，那么这没什么大不了的。

步骤2：检查应用程序的设置

在/WEB-INF/web.xml中，您将找到正在定义的Spring MVC DispatcherServlet。它从/WEB-INF/rest-servlet.xml文件加载应用程序上下文。该文件又包含一个组件扫描程序，该组件扫描程序扫描com.springsource.samples.rest包中的@Components（也称为@Controllers）。

接下来，在com.springsource.samples.rest包中，您将找到带有两个控制器方法的ContentController。


@Controller
public class ContentController {
	
	private List contentList = new ArrayList();
	
	@RequestMapping(value="/content.*", method=RequestMethod.GET)
	public ModelAndView getContent() {
		ModelAndView mav = new ModelAndView();
		mav.setViewName("content");
		mav.addObject("sampleContentList", contentList);
		return mav;
	}
	
	@RequestMapping(value="/content.html", method=RequestMethod.POST)
	public String addContent() {
		contentList.add(SampleContent.generateContent("Alef Arendsen", new Date()));
		return "redirect:content.html";
	}
}

第一个处理程序返回SampleContent项目的列表。第二个处理程序通过使用SampleContent.generateContent（）方法添加一个新的SampleContent项。第一个处理程序对GET请求作出反应，第二个处理程序对POST请求作出反应。方法本身已使用@RequestMapping注释进行了注释。

在组件服务器扫描程序的rest-servlet.xml应用程序上下文文件中，您还将获得一个ViewResolver，在本例中是一个InternalResourceViewResolver。它将负责视图名称（在getContent（）处理程序中为“内容”）和JSP页面之间的转换。

将其部署到例如Tomcat之后，您应该可以转到 HTTP：//本地主机：8080 /弹簧休息/休息 。这会将您重定向到/ rest / content，该URL由处理程序获取。

步骤3：实现AtomView

为了实现AtomView，我们将使用Rome项目，该项目可从以下网站获得 https://rome.dev.java.net/ 。在应用程序的原始设置中，视图名称由视图解析器转换为InternalResourceView实例，在这种情况下，该实例将呈现JSP。我们将创建自己的View接口专用实例，而不是呈现Atom提要。

在com.springsource.samples.rest包中创建一个名为SampleContentAtomView的类，然后粘贴以下代码。该代码使用Spring MVC中的Atom支持类以及Rome项目中Atom提要的文档对象模型。


public class SampleContentAtomView extends AbstractAtomFeedView {

	@Override
	protected void buildFeedMetadata(Map model, Feed feed, HttpServletRequest request) {
		feed.setId("tag:springsource.com");
		feed.setTitle("Sample Content");
		@SuppressWarnings("unchecked")
		List contentList = (List)model.get("sampleContentList");
		for (SampleContent content : contentList) {
			Date date = content.getPublicationDate();
			if (feed.getUpdated() == null || date.compareTo(feed.getUpdated()) > 0) {
				feed.setUpdated(date);
			}
		}
	}

	@Override
	protected List buildFeedEntries(Map model,
			HttpServletRequest request, HttpServletResponse response) throws Exception {

		@SuppressWarnings("unchecked")
		List contentList = (List)model.get("sampleContentList");
		List entries = new ArrayList(contentList.size());

		for (SampleContent content : contentList) {
			Entry entry = new Entry();
			String date = String.format("%1$tY-%1$tm-%1$td", content.getPublicationDate());
			// see http://diveintomark.org/archives/2004/05/28/howto-atom-id#other
			 entry.setId(String.format("tag:springsource.com,%s:%d", date, content.getId()));
			entry.setTitle(String.format("On %s, %s wrote", date, content.getAuthor()));
			entry.setUpdated(content.getPublicationDate());

			Content summary = new Content();
			summary.setValue(content.getText());
			entry.setSummary(summary);
			
			entries.add(entry);
		}

		return entries;

	}
}

步骤4：设置内容协商

框架Web应用程序已经提供了HTML视图，现在我们还实现了生成Atom提要的视图。我们需要做的最后一件事是确保对Atom提要的请求实际上将使用SampleContentAtomView呈现，并且在请求HTML视图时将呈现JSP。

在理想情况下，客户端会使用“接受HTTP”标头来请求其偏爱的某种表示形式。Accept HTTP标头（由HTTP规范定义）可以采用一种或多种媒体类型，并且应由浏览器（或任何与此相关的HTTP客户端）发送，以指示其喜欢哪种表示形式。例如，Atom提要的适当媒体类型是'application / atom + xml'，而HTML视图可能只是发送'text / html'或'text / xhtml'作为Accept标头。但是，这有一个小问题。浏览器通常具有一组固定的媒体类型，它们作为Accept HTTP标头发送，并且没有办法（除了使用JavaScript）修改浏览器发送的Accept标头。这就是文件扩展名是向服务器指示所需的表示形式的好选择的原因。

Spring 3.0具有一个ContentNegotiatingViewResolver，它可以与扩展名和Accept标头一起使用。找到合适的媒体类型后，它委托一组其他视图解析器为我们完成此操作。需要将以下内容粘贴到rest-servlet.xml中，以使其正常工作。如您所见，ContentNegotiatingViewResolver将委派给BeanNameViewResolver（如果需要，解析为SampleContentAtomView）或InternalResourceViewResolver。顺便说一下，下面的代码片段应替换在rest-servlet.xml文件中已经配置的InternalResourceViewResolver。

ContentNegotiatingViewResolver首先查看文件扩展名，然后使用Accept标头（通过某种程度可定制的方式）。我们必须将适当的扩展名映射到适当的媒体类型。在此示例中，我们将.html映射到text / html媒体类型，并将.atom映射到application / atom + xml。这将确保呈现适当的视图。

如果出现.atom请求，则ContentNegotiatingViewResolver将查找与application / atom + xml媒体类型匹配的视图。视图解析器将查找媒体类型为text / html的视图呈现内容，这是一个.html请求传入的。

将其放在您的应用程序上下文中后，让服务器重新启动应该可以解决问题。去 http://localhost:8080/spring-rest / rest / content.html查看所有内容项并生成新的内容项。去 http://localhost:8080/spring-rest / rest / content.atom订阅Atom提要。

我希望这个小博客条目向您展示将Atom提要添加到您的应用程序有多么简单。除了Atom，Spring还提供视图支持类，用于呈现PDF和Excel文件，JSON表示形式和XML文档。检查他们，让我们知道您的想法！
