适用于Apache Kafka 2.0里程碑1的Spring

发布
加里·罗素
2017年4月27日

我们非常高兴地宣布Apache Kafka 2.0发行版2.0.0的Spring的第一个里程碑可用。M1 。

2.0系列中的重要新功能包括：

  • 支持时间戳KafkaTemplate 。

  • 寻求话题的开始/结束。

  • KIP-62促进了新的线程模型-现在，消费者客户端不再依赖poll()被频繁调用的线程模型要简单得多。避免需要内部pause() / resume()处理。现在总是在使用者线程上调用侦听器。反过来，这促进了：

  • ConsumerAwareMessageListener （和BatchConsumerAwareMessageListener ），以便侦听器实现可以访问Consumer对象执行诸如pause() ， resume() ， metrics()等等

  • @KafkaListener POJO方法现在可以用注释@SendTo将方法结果发送到其他主题。

Spring Cloud Task 1.2.0。现在可以发布

发布
格伦·伦弗罗（Glenn Renfro）
2017年4月25日

我们很高兴地宣布Spring Cloud Task 1.2.0。现在可以通过Github，Maven Central和Pivotal下载存储库获得RELEASE。非常感谢所有对此版本做出贡献的人。此版本代表1.2系列的第一个GA版本。

Spring Cloud Task 1.2.0中的新增功能。发布

Spring Cloud Task 1.2.0。RELEASE旨在继续进行Spring Cloud Data Flow所需的集成，此版本提供以下功能。

  • 更好DataSource任务和批处理之间的集成-此版本使配置共享DataSource任务和批处理存储库之间的连接更加简单。
  • 允许用户轻松地为任务表配置前缀-与Spring Batch允许用户为批处理存储库表配置前缀的方式类似，Spring Cloud Task现在提供了通过Spring Boot为任务存储库表配置前缀的功能属性。
  • 允许外部进程更新外部执行ID-在此版本之前，必须由任务本身更新外部执行ID（基础平台的执行ID）。在某些用例中，这是不可能的。此版本提供了在任务本身范围之外更新外部执行ID的功能。
  • 添加对父级执行ID的支持-在复杂的用例中，一项任务启动多个其他任务的能力非常普遍。Spring Cloud Task已经通过分区批处理作业启动工作节点作为任务来支持此功能。此版本提供了任务存储库中的功能，以将由此功能产生的父子关系（在批处理用例中以及与原始任务）相关联。
  • 在创建任务时可以设置任务名称TaskExecution在外部，允许启动器在永久性庄园中设置名称。
  • 添加Ordered与任务事件和批处理作业事件的接口，以便用户可以确定何时从其Spring Cloud Task应用程序发出任务或批处理事件。
  • 更名closecontext.enable至closecontext.enabled与启用的属性的Spring Boot样式匹配。
  • 各种错误修复和文档更新。
