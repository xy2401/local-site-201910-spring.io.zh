       Well, it's that time of the year again!

假期即将到来，许多人将庆祝！无论您正在庆祝哪个假期（或者如果您正在庆祝），我都希望您度过一个愉快的一周，并度过最快乐的假期。

  1. 圣诞老人的精灵可以从SpringSource工程师那里学到一两个东西！艾伦·斯图尔特（Alan Stewart）于周六 宣布了Spring Roo 1.2.0 GA 。
    这个版本的Roo吸引了很多人的注意。尽管有许多新功能，但我最喜欢的是多模块Maven项目支持（例如，它使您可以在与Web应用程序不同的项目中构建模型类），
    支持JSF / PrimeFaces脚手架（而不只是GWT，Spring MVC，Vaadin和Flex），
    支持服务/存储库（使用Spring Data JPA和Spring Data MongoDB；这是已经可用的Active记录样式数据访问对象的补充），
    和
    （更多）灵活的GWT支持。
  2. When Spring 3.1 was released last week, I imagine Spring Data Gemfire and Spring Data Redis lead Costin Leau hit a big red button that he had prepared for just this occasion. First, Costin released Spring Data Gemfire 1.1.0 , which has numerous features! Besides supporting Spring 3.1 (including a 3.1-compatible implementation of the Spring Cache API), Spring Data Gemfire 1.1.0 supports PDX (the binary object serialization type supported by Gemfire) attributes on the <cache/>, <client-cache/> namespace, it provides dedicated support for indices, better support for Region creation, CacheServer support, queries with variable parameters, and a MessageListenerContainer like abstraction that works with Gemfire's continuous queries (CQs). The CQ support is the basis of the GemFire inbound adapter support available in Spring Integration 2.1, due any minute now..
  3. Then, Costin released Spring Data Redis 1.0.0, a major milestone in of itself! This release, besides being compatible with Spring 3.1, supports atomic counters, a RedisTemplate, data access exception translation (consistent with the support you expect from all of Spring's data access technologies), a MessageListenerContainer abstraction that builds on Redis' pub-sub support, and much more!
  4. But Costin wasn't the only one sitting on a Spring 3.1 compatible release. Spring Social lead Craig Walls announced Spring Social 1.0.1, which, besides being Spring 3.1- and Spring Security 3.1-compatible, also provides numerous bug fixes and updates.
  6. 上周发布Spring 3.1时，我想像Spring Data Gemfire和Spring Data Redis的负责人Costin Leau碰到了他为此准备的一个红色大按钮。首先，Costin发布了Spring Data Gemfire 1.1.0 ，它具有许多功能！除了支持Spring 3.1（包括与Spring Cache API的3.1兼容实现）之外，Spring Data Gemfire 1.1.0在 ， 支持PDX（Gemfire支持的二进制对象序列化类型）属性。命名空间，它提供了对索引的专用支持，对Region创建的更好支持，对CacheServer支持，具有可变参数的查询以及像Gemfire的连续查询（CQ）一样的MessageListenerContainer类似于抽象）。CQ支持是Spring Integration 2.1中提供的GemFire入站适配器支持的基础，现在可能需要一分钟。
  7. 然后，Costin发布了Spring Data Redis 1.0.0 ，这本身就是一个重要的里程碑！除了与Spring 3.1兼容之外，该版本还支持原子计数器， RedisTemplate ，数据访问异常转换（与您期望从Spring的所有数据访问技术中获得的支持一致），基于Redis的pub-sub支持的MessageListenerContainer抽象，以及更多！
  8. 但是Costin并不是唯一一个与Spring 3.1兼容的发行版。Spring Social负责人Craig Walls 宣布了Spring Social 1.0.1 ，除了与Spring 3.1和Spring Security 3.1兼容之外，它还提供了许多错误修复和更新。

That is, it's one thing to think of REST as being an HTTP-centric RPC mechanism, but to really grok it is something else entirely. On this subject, I can recommend no finer reference than Jim's book and of course Roy Fielding's original work. In Jim's book, however, he establishes an example of an automated coffee shop, RestBucks, which exposes an API. (This example itself is a twist on the example established in the simialarly seminal Patterns of Enterprise Architecture) It's a wonderful book, a fantastic example, and, more to the point, it makes perfect fodder for blogger "Furious Bob" to demonstrate a cleanly architected application using Spring 3.1's enhanced REST support.

    以及所有这些新功能，在Grails 2.0的开发过程中，Grails团队修复了900多个问题，并围绕Grails不断发展插件生态系统 。在Grails 2.0的开发过程中， 源代码以多种重要方式得到了发展。我们通过转移到Gradle作为构建工具来对源代码进行模块化。我们现在使用Artifactory进行仓库管理； Spock已成为我们的事实测试工具；并且我们重写了内部结构以利用Groovy AST转换的优势。通过所有这些更改，Grails的性能将大大改善，并且用户将看到Grails 2.0的好处。除了Grails 2.0的发布之外，我们还有许多其他激动人心的公告，它们将从Cloud开始。

    云中的Grails 2.0

    对于那些关注云空间的人（谁不是！）您会很高兴知道Grails 2.0已经支持Cloud。 Cloud Foundry -VMware开放的PaaS- 开箱即用地支持Grails 2.0 ，我们已经在努力将Grails网站迁移到cloudfoundry.com ！在Peter Ledbrook撰写的这篇出色的博客文章中介绍了Grails和Cloud Foundry的一步部署 ，它展示了如何使用Cloud Foundry插件（用于Grails）开始Cloud Foundry部署。Heroku的家伙们还宣布了对Grails的支持 -支持将Grails应用程序持续部署到Heroku的云平台 。Grails应用程序可以通过简单的“ git push”进行部署，从而触发完整的构建和部署工作流程。要在Heroku上开始使用Grails，请遵循此Grails快速入门指南 。Grails部署到Heroku当前处于公开测试阶段。随着更多支持Grails的云平台的出现，云将在2012年继续成为一个巨大的主题，其中许多平台将基于开源Cloud Foundry。

    IDE中的Grails 2.0

    对于Eclipse用户，请使用SpringSource Tool Suite（STS）版本2.9.0。M1（2011年12月可用）为Grails 2.0提供了最佳支持。STS 2.8.1版为那些不准备依赖里程碑版本的用户提供了许多基本支持。STS版本可在以下位置找到： http : //www.springsource.com/downloads/sts最新的STS新版本和值得注意的版本可在以下位置找到： http : //download.springsource.com/release/STS/doc/STS-new_and_noteworthy .pdf它描述了Groovy和Grails开发的所有最新增强功能。有关STS可能存在的任何问题的支持论坛，请访问这里： http : //forum.springsource.org/forumdisplay.php? 32-SpringSource-Tool-Suite对于Intellij IDEA用户， JetBrains一直在忙于开发Intellij。上周发布的IDEA 11。Intellij IDEA 11包含许多支持Grails 2.0的新功能，因此请确保在开始使用Grails 2.0和Intellij IDEA之前进行升级。

    Grails 2.0和NoSQL

    Grails 2.0是Grails的第一个版本，该版本真正抽象了GORM层，因此可以使用GORM的新实现。随着Grails 2.0的发布，我们还很高兴地宣布针对NoSQL数据存储发布了许多与Grails 2.0兼容的插件：
    • 适用于MongoDB 1.0 RC3的GORM
    • 适用于Redis 1.0 M8的GORM
    • 适用于Riak 1.0 M4的GORM
    MongoDB插件处于最终发布候选阶段，它基于出色的Spring Data MongoDB项目 ，该项目也以RC形式提供。此外，我们还为希望参与该项目的人们编写了一份开发人员指南 ，说明如何构建GORM实施。Grails的用户可以期待在2012年更加精彩的NoSQL宣布与格姆的即将到来的未来版本中对Neo4j的 ， 亚马逊的SimpleDB的作品和卡桑德拉。
