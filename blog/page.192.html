网络研讨会：使用Spring和GemFire构建可扩展的数据应用程序

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2014年12月2日

演讲者：卢克·香农

GemFire是Pivotal的“分布式内存数据”网格。在本课程中，我们将介绍使用Spring Data GemFire简化配置GemFire网格的过程，并探讨随着网格资源的增加或减少以满足客户不断增长的需求，如何动态分布数据并使数据具有高可用性。将详细介绍使用Spring的GemFire访问和测试模式。我们将逐步创建域对象，并使用Spring Data GemFire存储库在GemFire网格上执行CRUD操作，创建一个使用GemFire作为其持久层的Spring MVC / Spring Boot应用程序，并回顾如何使用Spring Boot和Spring Data GemFire。帮助GemFire测试用例以及Spring Data GemFire如何使GemFire函数的创建和执行更加容易。我们还将探讨如何使用Spring XD加载和监视网格中的数据更改。这将是一个动手的，面向代码的会话。

春季数据发布的第一个里程碑火车福勒问世

发布
奥利弗·德罗博姆（Oliver Drotbohm）
2014年12月1日

我很高兴代表Spring Data团队宣布Spring Data Release火车Fowler的第一个里程碑。

该版本发行了170张固定的票 ，其中119 张是福勒发行列车特有的。在此处找到策划的变更日志。该版本的重点包括：

  • 升级到Spring 4.0.8和Querydsl 3.6.0。
  • 通过使用字节码生成，可以提高转换器子系统的性能。
  • 对JSR-310日期/时间类型（MongoDB）的现成支持。
  • SpEL（JPA）用法的各种（性能）改进。
  • 支持$bit ， $mul和$position运算符（MongoDB）。
  • 改进了JPA和REST的代理ID检测。
  • 介绍QueryDslPredicateExceutor.findAll(OrderSpecifier... orders) （Querydsl）。
  • Cassandra模块的CDI存储库扩展。
  • 升级到Gemfire 8（Gemfire）。
  • 基于版本属性的ETag生成（REST）。
  • 升级到Solr 4.10.1，Elasticsearch 1.2，Neo4J 2.1.5，并与即将推出的MongoDB服务器和驱动程序兼容。

Spring Cloud 1.0.0。M3现在可用

发布
戴夫·瑟
2014年11月25日

Spring Cloud 1.0.0。M3现在可在repo.spring.io存储库中使用。以下项目的版本均为1.0.0。M3版本：

  • Spring Cloud Config：集中式键值（或YAML）配置管理。现在支持将配置服务器完全嵌入到另一个应用程序中。

  • Spring Cloud Netflix。还具有对服务器组件可嵌入性的更好支持。现在，还可以在启用了Ribbon的Spring RestTemplate正确记录负载均衡器统计信息。

  • 适用于Amazon Web Services的Spring Cloud。具有新的Spring Boot集成点，可将AWS元数据的配置外部化。

  • Spring Cloud Security：声明式样式的超级简单OAuth2。

  • Spring Cloud Bus：向Spring Cloud组件广播框架级别的事件。这里的大新闻是，我们现在有一个用于Hystrix指标的基于RabbitMQ的聚合器 （基于Turbine 2），因此您不必依赖对所有服务实例的直接HTTP访问。

  • Spring Cloud CLI：Groovy CLI，用于在自包含脚本中编写微服务。

  • Spring Cloud for Cloud Foundry：现在在Spring Cloud Security和Cloud Foundry服务绑定之间架起了桥梁，使在Cloud Foundry中执行单点登录和OAutth2保护的资源变得超级容易。
