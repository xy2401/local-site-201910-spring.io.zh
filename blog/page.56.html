Spring Cloud Data Flow 1.5 RC1发布

发布
马克·波拉克
2018年5月9日

Spring Cloud Data Flow团队很高兴宣布发布1.5.0 RC1 。遵循本地服务器 ， Cloud Foundry和Kubernetes的入门指南 。

以下是重点内容：

总体改进

  • 切换到Hikari连接池并重组代码以使用更少的连接。

  • 底层部署程序库中的几个错误修复。

仪表板

  • 现在可以从Stream Builder中编辑创建/部署的流。可以编辑和重新部署应用程序和部署属性。App版本也可以切换。

  • 新的分页器组件将添加到所有列表页面。每页可能会从20、30、50或100个项目的列表中切换。这进一步简化了批量操作工作流程。

  • 介绍通过Selenium和SauceLabs进行的端到端测试。

2018年5月9日

今天发布了以下CVE：

  • 适用于Spring Framework 5.0.6，4.3.17的CVE- 2018-1257 。
  • 用于Spring Security 5.0.5的 CVE-2018-1258 。
  • Spring Data Ingalls SR12，Kay SR7的 CVE-2018-1259 。
  • CVE-2018年至一二六零年春季安全的OAuth 2.3.3，2.2.2，2.1.2，2.0.15 。

请查看CVE报告中的信息并立即升级。

Spring Boot用户：今天早些时候发布的Spring Boot 2.0.2和1.5.13包含上述漏洞的修复程序。

	<dependencies>
		<dependency>
			<groupId>org.springframework.session</groupId>
