Maven PAR插件1.0.0。M1

工程
本·黑尔
2009年6月24日

在引入SpringSource dm Server（当时的应用程序平台）后不久，Thorsten Maus创建了一个用于创建PAR文件的Maven插件 。这是社区做出的巨大贡献，我们甚至在dm Server 1.0.x 入门指南中使用了它。

随着dm Server 2.0里程碑的发展，我们一直在向PAR文件添加新功能。最有趣的功能是，现在一个PAR文件可以包含的不仅仅是JAR文件。随着OSGi RFC 66的引入，dm服务器已弃用了Web模块 ，而支持使用.war扩展名的标准Web容器文件。因此，需要更新PAR插件以支持这些不同类型的依赖关系。Thorsten慷慨地同意将其代码捐赠给SpringSource，我很高兴宣布1.0.0。Apache许可的Maven PAR插件的M1版本。

OSGi测试存根1.0.0。M1

工程
本·黑尔
2009年6月23日

我很高兴宣布1.0.0。M1版本的SpringSource的OSGi测试存根。这些存根提供了一种无需复杂的OSGi容器即可对复杂的OSGi框架交互进行单元测试的方法。


  • 套餐- 下载

  • Git- git：//git.springsource.org/osgi-test-stubs/osgi-test-stubs.git

问题

随着dm Server团队的不断发展，我们发现对我们来说最大的测试问题之一是BundleActivator 。我们的BundleActivator确实向服务注册表发布了许多服务，并使用ServiceTracker消费了服务。这些任务涉及对BundleContext ， Bundle ， ServiceRegistration和ServiceReference的许多交织调用。最初，这些激活器非常简单，因此无需对它们进行太多的单元测试，并且我们依靠集成测试来捕获所引入的任何错误。随着时间的流逝，激活器变得越来越复杂，单元测试也变得更加迫切。我们开始使用EasyMock进行这些测试，但是发现它们非常复杂，难以维护，最重要的是难以理解。

通过JMX的dm服务器存储库内容

工程
本·黑尔
2009年6月22日

在dm Server 2.0.0中。M1版本中，我们增加了对共享存储库的支持。作为此新功能的后续，我们添加了对通过JMX浏览这些存储库的支持。

现在，使用JMX客户端连接到正在运行的dm Server实例将显示一个存储库组。在此组下，每个配置的存储库（链式存储库除外）都有一个Bean，该Bean返回包含在其中的所有工件的概要。该bean还公开了一种根据其类型，名称和版本返回给定bean的整个详细ArtifactDescriptor的方法。
