Spring Cloud Data Flow 1.2 M1发布

发布
托马斯·里斯伯格
2017年2月15日

我很高兴代表团队宣布发布Spring Cloud Data Flow 1.2的第一个里程碑。

注意：开始使用此新版本的一个好方法是遵循项目页面上的版本列表 ，其中包括下载坐标和参考指南的链接。

在过去的几周中，我们为数据微服务的整体编排添加了新功能并进行了改进。1.2.0中包含以下新功能。M1版本：

核心

  • 为部署属性引入专用前缀。使用部署程序属性非常简单deployer..xxx相对于app..spring.cloud.deployer.xxx
  • 引入新的REST-API控制器和外壳程序支持以清理任务执行
  • 基础工作，以巩固在任务部署和任务执行之间使用控制器
  • 合并REST-API调用跟踪和返回代码以保持一致性
  • 添加基于角色的访问控制，以定义谁有权创建，部署，销毁或查看流/任务。这可以与支持的身份验证方法无缝协作。
  • 针对“流列表”操作的性能优化。新引入的方法不是对与流相关的每个应用程序单独调用，而是MultiStateAppDeployer SPI操作按流调用一个查询，该查询在单个网络调用中查询所有应用程序状态
  • 改进了“流列表”操作的错误报告

Spring Cloud Task 1.2.0。M1现在可用

发布
迈克尔·米内拉
2017年2月9日

我们很高兴地宣布Spring Cloud Task 1.2.0。现在可以通过Github和Pivotal下载存储库使用M1。非常感谢所有对此版本做出贡献的人。

Spring Cloud Task 1.2.0。M1提供以下功能：

这是Spring Cloud Task 1.2.x系列的第一个里程碑。为了继续Spring Cloud Data Flow所需的集成，此版本提供了以下新功能：

  • 任务和批处理之间更好的数据源集成-此版本使配置任务/批处理集成所使用的数据源更加容易。

  • 允许外部进程更新外部执行ID-在此版本之前，必须由任务本身更新外部执行ID（基础平台的执行ID）。在某些用例中，这是不可能的。此版本提供了在任务本身范围之外更新外部执行ID的功能。

  • 允许用户为任务表配置前缀-与Spring Batch允许用户为批处理存储库表配置前缀的方式类似，Spring Cloud Task现在也提供了为任务存储库表配置前缀的功能。

  • 添加对父级执行ID的支持-在复杂的用例中，一项任务启动多个其他任务的能力非常普遍。Spring Cloud Task已经通过分区批处理作业启动工作节点作为任务来支持此功能。此版本提供了任务存储库中的功能，以将由此功能产生的父子关系（在批处理用例中以及与原始任务）相关联。

  • 升级到Spring Cloud Camden SR5-此版本与Spring Cloud Camden的最新SR5版本兼容。
