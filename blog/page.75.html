Spring 的这个星期-2017年10月31日

工程
乔什·朗（Josh Long）
2017年10月31日

嗨，春季粉丝们，欢迎来到春季本周的另一期！本周我将在旧金山参加2017年Kotlin Conf大会 ，这是第一个，而且肯定会令人惊讶 。我将谈论Spring Boot和Kotlin，希望您在身边时也能加入我的行列。

这周我们有很多工作要做，让我们开始吧！

  • Spring Batch主管Michael Minella 刚刚宣布了Spring Batch 4.0.0。M5 ，其中包括ItemProcessor支持Java 8的实现Function实施，更新的文档等等。
  • Spring Framework的负责人Juergen Hoeller 宣布了Spring Framework 5.0.1 ，其中包括一些错误修复，这些错误修复进而在Reactor Netty中进行了修复，并欢迎AspectJ 1.8.12中的性能改进。
  • 上周，在Spring Tips的另一部分中， 我通过Spring Cloud Contract研究了基于HTTP的合同 。
  • Spring Cloud联合创始人Spencer Gibb宣布了Spring Cloud Edgware的首个发布候选版本。对于各种启动器，没有在工件名称中进行一些更改。Spring Cloud Edgware 具有许多新功能，因此请务必研究发行说明。
  • 您是否想知道Apache Geode和Pivotal Gemfire之间的区别是什么？让Spring Data Geode / Gemfire领导John Blum阐明本文中的差异 。TL; DR：不多！
  • Project Reactor负责人Stephane Maldini刚刚发布了Reactor Bismuth SR3 ，该版本包含许多错误修复和改进。Spring Framework 5.0.1中提供了Reactor Bismuth SR3。
  • Spring REST Docs负责人Andy Wilkinson 刚刚宣布了Spring REST Docs 2.0.0。RC1 。新版本需要Spring Framework 5.0和Java 8。2.0版本的主要亮点是支持使用Spring WebFlux的WebTestClient记录API。
  • Spring Web Services负责人Greg Turnquist 刚刚宣布了Spring Web Services 3.0.0。发布 。新版本具有许多新功能，包括更好地处理大型附件，SAML回调，升级到AspectJ的最新版本，支持即将到来的Axiom 1.3等。值得注意的是，感谢3.0版的75％是由社区贡献的！
  • Mark Pollack博士刚刚宣布了Spring Cloud Skipper 1.0 M1 。摘自文章：“ Skipper是一种轻量级工具，可让您发现Spring Boot应用程序并管理它们在多个Cloud Platform上的生命周期。您可以单独使用Skipper，也可以将其与Continuous Integration管道集成在一起，以帮助实施Continuous Deployment的实践。”
  • 我喜欢从Spring Framework 5.0.1开始的新Spring Framework文档 。新的登录页面非常漂亮，只需单击任何部分，您就会看到轮廓如何真正被很好地清理。
  • Spring Integration忍者Artem Bilan 在这篇推荐的新文章中探讨了集成测试Spring Cloud Stream应用程序的艰巨主题。
  • 在谈到消费者驱动的合同测试时，Spring Cloud合同负责人Marcin Grzejszczak 在今年早些时候的演讲中提出了更有趣的观点
  • Oleh Dokuka 查看了Reactive Streams规范的Publisher根据Pivotal的Reactor实施类型输入 。正如Lightbend的Konrad Malawski所说 ：“可能唯一有关#reactivestreams实现和#flow＃jdk9的帖子可以很好地说明如何使用tck / rules ：）”

Spring Cloud Skipper 1.0 M1发布

发布
马克·波拉克
十月30，2017

我很高兴代表团队宣布Spring Cloud Skipper 1.0 M1的发布。

Skipper是一款轻巧的工具，可让您发现Spring Boot应用程序并在多个Cloud Platform上管理其生命周期。您可以独立使用Skipper，也可以将其与Continuous Integration管道集成在一起，以帮助实施Continuous Deployment的实践。

Skipper 1.0 M1的主要功能包括：

  • 定义可以在其中部署Spring Boot应用程序的多个平台帐户。支持的平台是本地，Cloud Foundry和Kubernetes。
  • 在Mustache模板文件中替代变量，这些变量描述了如何将应用程序部署到平台。
  • 在软件包存储库中搜索现有应用程序。
  • 根据简单的蓝/绿工作流程升级/回滚软件包。
  • 存储已解析的模板文件（也称为“应用程序清单”）的历史记录，这些文件代表针对特定版本已部署到平台的内容的最终描述。
  • 通过独立的交互式Shell或Web API使用。

Spring Web Services已发布3.0.0以获得一般可用性，并发布了一个次要补丁2.4.2。发布。

如前一篇博客文章所述，只要支持Spring Framework 4.x，就会维护2.x系列，但是任何新工作都将在master分支上进行。

以下链接包括相关票证。

Spring REST Docs 2.0.0。RC1

发布
安迪·威尔金森
十月30，2017

谨代表每个贡献者，宣布Spring REST Docs 2.0.0。RC1已发布，可从https://repo.spring.io/milestone/获取 。

什么是新的？

Spring REST Docs 2.0是该项目的最新一代，并且需要Spring Framework 5.0和Java 8。

2.0版本的主要亮点是支持使用Spring WebFlux的WebTestClient记录API。 WebTestClient可以用于使用WebFlux Fn和WebFlux的基于注释的编程模型实现的API的轻量级测试和文档记录。它也可以用来记录可通过HTTP访问的任何API，这以前只能在使用REST Docs的REST保证支持时才能实现。

Spring Framework 5.0.1现在可用

发布
于尔根·霍勒（Juergen Hoeller）
十月24，2017

尊敬的Spring社区，

我很高兴地宣布，Spring Framework 5.0.1 现在已经可用 ，它是我们Spring Framework 5.0产品线中的第一个维护版本。

除了解决用户报告的各种错误之外，我们还应用了较小的重构来清理5.0 GA版本中的意外残留，例如，用于响应式编解码器配置器。

此版本基于Reactor Core 3.1.1将Reactor升级到Bismuth SR3，并在Reactor Netty中进行了必要的修复。此外，AspectJ 1.8.12在运行时为Spring AOP的切入点表达式评估带来了一些令人欢迎的性能改进。
