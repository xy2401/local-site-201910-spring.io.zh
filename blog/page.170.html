用于Apache Hadoop 2.2 M1的Spring发布

发布
托马斯·里斯伯格
2015年4月30日

我们很高兴宣布Spring for Apache Hadoop 2.2 M1里程碑版本。

我们将继续在Spring IO里程碑存储库中提供特定于版本的工件及其各自的传递依赖项：

  • 2.2.0。M1（默认-Apache Hadoop稳定版2.6.0）
  • 2.2.0。M1-phd21（Pivotal HD 2.1）
  • 2.2.0。M1-phd30（Pivotal HD 3.0）
  • 2.2.0。M1-cdh5（Cloudera CDH5）
  • 2.2.0。M1-hdp22（Hortonworks HDP 2.2）

Spring对Apache Hadoop 2.2 M1版本的最重要增强：

  • 现在需要删除对与JDK 6，Java 7或更高版本一起运行的支持
  • HDFS编写器的改进，以支持可同步写入和新的超时选项
  • 添加对Pivotal HD 3.0的支持
  • 更新CLoudera CDH 5以使用版本5.3.3
  • 将Hortonworks HDP 2.2版本更新到2.2.4.0
  • 将Kite SDK更新到1.0版
  • 将Spring项目版本更新为最新版本

Spring XD 1.2 M1和1.1.2发布

发布
马克·波拉克
2015年4月30日

我非常荣幸地代表Spring XD团队宣布Spring XD 1.2的第一个里程碑版本和1.1.2维护版本。

下载链接：

  • 1.1.2。发布： zip
  • 1.2.0。M1： 拉链

您也可以使用brew和rpm安装

JIRA中提供了针对1.1.2修复的问题的完整列表。值得注意的是1.1.2版本提供了PHD 3.0支持。

1.2 M1版本还包括错误修复以及几个新功能和增强功能：

  • PHD 3.0支持
  • MongoDB Source ，来自Abhinav Gandhi的社区贡献
  • HDFS支持的模块注册表
  • Greenplum gpload作为提供的批处理作业。这样可以将CSV文件有效地加载到Greenplum DB / HAWQ中。
  • 遵循gpfdist协议的gpfdist 接收器 。这允许将数据并行流传输到Greenplum DB / HAWQ中。
  • Zookeeper基于分布式队列的流和作业部署。
  • 使用Dead Letter Queue 改进了RabbitMQ的错误处理，并为发布/订阅命名通道提供了持久的队列支持（tap：和topic :)
  • Sqoop集成改进 ，对合并和代码生成命令的支持以及在受保护的Hadoop群集上运行。
  • Kafka消息总线的改进，针对消息总线创建的主题的自定义分区计数。 （模块。[模块名称] .producer.minParitionCount）
  • TupleBuilder和JDBC to HDFS作业的改进的性能特征
  • Spark Streaming集成改进，可靠的接收器支持和错误修复。

SpringOne2GX 2014重放：通过Spring Integration构建高度模块化和可测试的业务系统

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2015年4月28日

记录在SpringOne2GX 2014中。

演讲者：Marius Bogoevici

数据/集成跟踪

幻灯片： http ： //www.slideshare.net/SpringCentral/building-modular-business-applications

就其本质而言，Spring Integration允许构建复杂的业务系统，以聚合多个数据源并编排一组复杂的业务服务。但是复杂的功能不必转化为复杂的设计。实际上，通过强调低耦合，Spring Integration正在促进高度模块化的应用程序设计，在可理解性，可重用性和可测试性方面具有巨大优势。在本课程中，您将学习如何通过将与逻辑相关的组件组合到彼此交互的子系统中来以模块化的方式设计Spring Integration应用程序，这是Spring XD的核心概念，但可以成功应用于任何应用程序中。除了提高抽象水平的好处之外，这种方法还具有其他许多重要好处：首先，此类子系统是可重用的，其次，同样重要的是，可以对它们进行隔离测试。因此，在对可重用性进行了简短讨论之后，本演讲将集中于如何对此类子系统甚至是完整的Spring Integration应用程序进行单元测试，其最终目标是应用诸如行为驱动开发之类的以业务为中心的技术。

SpringOne2GX 2014重放：带有差分同步和JSON补丁的高效客户端-服务器通信

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2015年4月28日

记录在SpringOne2GX 2014中。

演讲者：Brian Cavalier

Web / JavaScript跟踪

幻灯片： http ： //www.slideshare.net/SpringCentral/efficiency-clientserver-communication-with-differential-synchronization-and-json-patch

客户端-服务器的世界已经改变。REST的传统应用不再是最佳选择。我们正在将应用程序释放到一个这样的世界中：即使在断开连接的情况下，用户也希望在所有设备上都具有响应式UI。我们正在部署一个连接延迟，移动无线电使用和电池寿命已成为主要问题的世界。差分同步（DS）是一种算法，即使面对断开的连接，脱机设备等，也可以在N个参与方之间同步数据。通过从客户端到服务器的双向批处理和仅发送更改，它可以更有效地利用连接以及从服务器到客户端。我们将研究如何将其与JSON Patch结合使用，以通过HTTP Patch，WebSocket和STOMP在客户端和服务器之间同步应用程序数据，以及如何将其集成到Spring生态系统中。

在Cloud Foundry中使用Spring Boot绑定到数据服务

工程
戴夫·瑟
2015年4月27日

在本文中，我们将研究如何将Spring Boot应用程序绑定到数据服务（JDBC，NoSQL，消息传递等）以及Cloud Foundry中各种默认和自动行为的来源，并提供有关使用哪些以及将使用哪些的一些指导。在什么条件下保持活跃。Spring Boot提供了许多自动配置和外部绑定功能，其中一些与Cloud Foundry相关，而许多则与Cloud Foundry不相关。Spring Cloud Connectors是一个库，如果您想以编程方式创建自己的组件，则可以在应用程序中使用它，但是它本身并没有做什么“魔术”的事情。最后是Cloud Foundry java buildpack ，它具有“自动重新配置”功能，该功能试图减轻将简单应用程序迁移到云的负担。正确配置诸如JDBC或AMQP或Mongo之类的中间件服务的关键是要了解这些工具各自提供的功能，它们在运行时如何相互影响以及将它们的某些部分打开和关闭。目标应该是从开发人员桌面上的应用程序本地执行到Cloud Foundry中的测试环境的平稳过渡，并最终过渡到Cloud Foundry中的生产（或其他方式），而无需更改源代码或打包，这十二个要素应用指南。
