Spring Cloud Data Flow 1.0.0 M3发布

发布
马克·费舍尔
2016年4月29日

我很高兴代表团队宣布1.0.0。M3版本的Spring Cloud Data Flow 。

在过去的几个月中，我们为各种平台上的数据微服务的整体编排添加了令人兴奋的新功能和改进。我们还进行了一些使开发人员受益的变更，例如为我们发布的所有流和任务应用程序公开Spring Boot Starters 。以下是此版本中的一些重点内容：

  • 为今天也发布的以下数据流服务器实现提供基础：
    • Spring Cloud Data Flow的Cloud Foundry Server 1.0.0。M2
    • Spring Cloud Data Flow的Kubernetes服务器 1.0.0。M2
    • Spring Cloud Data Flow的Apache YARN Server 1.0.0。M2
    • Spring Cloud Data Flow的Apache Mesos Server 1.0.0。M2
  • 在Spring Cloud Deployer Service Provider界面上进行介绍和构建
    • 新的多平台应用程序部署模型从Spring Cloud Data Flow本身中分离出来用于一般用途，包括AppDeployer和TaskLauncher ，分别用于部署长期运行的和短暂的微服务。
    • 改进的应用程序解析策略，支持将应用程序注册表维护为http ， file ， maven ， docker ， 要么hdfs文物
  • 在Spring Cloud Stream 1.0.0的基础上构建。RC3
  • 在Spring Cloud Task 1.0.0的基础上构建。M2
  • 改进了对流和批处理管道的DSL支持
  • 添加了对流和批处理管道的“点击”支持
  • 应用领域
    • 支持从新的Stream Application Starters项目构建的即用型流应用程序 （为Kafka和RabbitMQ绑定器自动生成的应用程序）
    • 支持从新的Task Application Starters项目构建的现成的任务应用程序
    • 添加几个新的现成的流和任务应用程序
    • 通过命令行管理程序和仪表板改进了自定义应用程序注册机制
  • 仪表板
    • 新的和改进的仪表板
    • 添加批处理和任务支持
    • 添加“应用程序”选项卡，以监视和管理现成的和自定义的应用程序
  • Flo for Spring Cloud Data Flow
    • 现代外观，围绕调色板，自动布局，自动链接，画布，节点，节点连接器等进行了UX改进
    • 增加了对可脚本转换处理器的支持，该处理器接受ruby，groovy，python或javascript代码作为运行时计算逻辑
    • 在主管道和分接管道之间添加视觉区别
    • 适应角度样式工具提示
    • 图布局优化
  • 改进的IT和TCK测试
  • 添加新样本
  • 添加新徽标

用于Cloud Foundry 1.0.0的Spring Cloud Data Flow。M2发布

发布
埃里克·博塔德
2016年4月29日

我很高兴代表Spring Cloud Data Flow团队宣布1.0.0。用于Cloud Foundry的Spring Cloud Data Flow的M2版本。

发布
托马斯·里斯伯格
2016年4月29日

我很高兴代表团队宣布1.0.0。适用于Apache Mesos的Spring Cloud Data Flow和适用于Kubernetes的Spring Cloud Data Flow的M2版本。

  • 添加支持以将OOTB和自定义应用程序解析，注册和运行为Docker映像

Spring REST Docs 1.1.0。RC1

工程
安迪·威尔金森
2016年4月28日

遵循1.1.0。M1 ，我很高兴宣布Spring REST Docs 1.1.0。RC1已发布，可从https://repo.spring.io/milestone/获取 。

什么是新的？

HTTPie请求代码段

引入了新的HTTPie请求代码段。与现有的curl请求代码段相似，新代码段包含用于请求的HTTPie命令。感谢Raman Gupta贡献了此新功能。

可重复使用的代码段

现在，可以使用某些通用配置创建代码片段一次，然后重新使用 。这样可以减少在API上记录公共部分时的重复，例如self链接。

网络研讨会：具有Spring Cloud Data Flow的数据微服务

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2016年4月27日

主讲人：Pivotal的Mark Fisher和Mark Pollack
可伸缩数据处理的未来是微服务！基于Spring Boot提供的易于开发和部署以及Spring Cloud的云原生功能，Spring Cloud Stream和Spring Cloud Task项目为微服务流和批处理提供了一个简单而强大的框架。
在更高的抽象层次上，Spring Cloud Data Flow是一个集成的编排层，可为部署和管理由独立微服务组成的复杂数据管道提供高生产力的体验。流和任务是使用DSL抽象定义的，可以通过外壳和Web UI进行管理。此外，可插入的运行时SPI允许Spring Cloud Data Flow在各种分布式运行时平台（如Cloud Foundry，Apache Mesos和Apache YARN）之间协调这些应用程序。
在此网络研讨会中，您将看到Spring Cloud Data Flow的概述，以及从本地集群到远程Cloud的不同平台上的流和批处理应用程序的实时演示，以显示开发人员体验的简便性。

网络研讨会：Spring Cloud Task介绍

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2016年4月27日

演讲者：Pivotal的Michael Minella

云的主要承诺之一就是灵活性。如今，大多数部署到云的应用程序都是长期运行的流程，它们利用云扩展的灵活性。但是计算充满了短暂的任务，这些任务可以启动，完成工作然后终止。这些任务是出色的云用例，因为可以快速调配和回收资源。

在此网络研讨会中，我们将探索Spring Cloud产品组合中的一个新项目Spring Cloud Task，这是一个用于开发和编排短期微服务的新框架。我们将探索各种用例，构建您的第一个任务，并讨论如何使用各种技术来编排任务。最后，我们将窥视Spring Cloud Task项目的路线图。
