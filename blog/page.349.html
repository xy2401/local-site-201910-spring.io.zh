喜欢Spring 2.0的另一个原因：拦截器组合

工程
本·黑尔
2006年4月9日

最近，我正在一个项目中，该项目的Swing客户端通过RMI与服务层进行通信。服务层标记有事务，一切似乎都正常。但是，每当我们在Hibernate DAO层上遇到一个异常时，Spring都会将该异常转换为运行时异常，并将其作为RemoteException一直传播到整个堆栈并贯穿RMI连接。每当反序列化异常时，客户端上都会有一个异常（与RemoteException分开）。决定仅介绍一个方面。子类化ServiceAccessException的任何异常将被传递给客户端，而其他任何异常将被转换为FilteredServiceAccessException （ ServiceAccessException的子类），然后引发该异常。这导致内容丢失，因此我们确保将原始异常记录在可能有用的服务器上，并让客户端显示一个通用对话框，以便用户大致了解发生了什么。

Spring 2.0的JMS改进

工程
本·黑尔
2006年4月9日

随着Spring 1.1的发布，Spring社区获得了对JMS支持的初次体验。这种支持包括异常转换，消息转换和类似于JdbcTemplate的模板类。这种支持还解决了JMS 1.0.2和1.1规范之间的域统一问题。此支持的核心是JmsTemplate类，它是JMS 1.0.2的对应JmsTemplate102 。

与使用原始JMS API进行企业消息传递相比，此支持有了很大的改进。但是它确实有一个缺点。 JmsTemplate仅支持使用JmsTemplate.receive（）方法同步接收消息。这种行为对许多人来说效果很好，但最终的绝大多数用户最终推出了自己的异步使用者实现。简而言之，他们想要EJB 2所谓的消息驱动Bean 。

Spring 2.0中的POJO方面：一个简单的示例

工程
马克·费舍尔
2006年3月22日

尽管这篇文章中的内容非常简单，但实际上它可以使您了解Spring 2.0中一些相当重要的新功能。我希望您只要稍加想象，就可以将在这里看到的内容应用到您自己的琐碎用例中。

我将实际显示2个示例。第一个将使用一个相当简单的记录器：


package example;

import org.apache.commons.logging.Log;
import org.apache.commons.logging.LogFactory;

public class SimpleLogger {

  private static Log log = LogFactory.getLog(SimpleLogger.class);

  public void logOneString(String s) {
    log.info("string=" + s);
  }

  public void logTwoStrings(String s1, String s2) {
    log.info("string1=" + s1 + ",string2=" + s2);
  }
}
