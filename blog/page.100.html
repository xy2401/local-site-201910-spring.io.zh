针对Visual Studio Code发行的Cloud Foundry Manifest编辑器Beta

发布
马丁·利珀特
2017年2月3日

作为支持全球开发人员使用Spring构建应用程序并将这些应用程序部署到Cloud Foundry和PCF的活动的一部分，我们很自豪地宣布我们的第一个beta版本的Cloud Foundry Manifest编辑支持 Visual Studio Code （在macOS，Linux上） x64和Windows）。

为什么使用Visual Studio Code？

Visual Studio Code是一种轻量级的开源代码编辑器，可在macOS，Linux x64和Windows上运行。它基于有关扩展性的有趣架构。Visual Studio Code中对语言的支持被实现为所谓的“语言服务器”。这些语言服务器独立于您使用的编辑器。使用协议（称为语言服务器协议）连接编辑器和语言服务器。即使Visual Studio Code引入了此协议，其他编辑器和IDE仍开始采用此语言服务器协议-例如Eclipse IDE（从4.7版开始）或Eclipse Che作为云IDE。 Sublime Text和Atom等其他轻量级编辑器也可能在不久的将来提供支持。因此，我们可以专注于将Cloud Foundry清单编辑器支持实现为独立的语言服务器，并且可以将此支持添加到您选择的编辑器或IDE中。

Spring Cloud Dalston M1发布

发布
斯宾塞·吉布
一月31，2017

我很荣幸地代表社区宣布， Spring Cloud Dalston Release Train的里程碑1（M1）现已上市。该版本可以在我们的Spring Milestone存储库中找到。您可以查看Dalston 发行说明以获取更多信息 。

Dalston发布系列的重大变化

Spring Cloud Vault Config是一个新项目，可通过Hashicorp Vault为分布式系统中的外部化秘密管理提供客户端支持。

配置服务器

Hashicorp Vault先前已作为后端添加到Config Server，以与基于VCS的后端一起使用。Config Server现在通过Composite模式支持多个后端。这允许组合以前不可能的后端类型，例如：Vault和git或多个git后端。现在支持对AWS Codecommit托管的git存储库的身份验证。

Spring Data Release Ingalls中有什么新功能？

工程
马克·帕鲁奇
一月30，2017

您可能已经看到，我们刚刚宣布了Spring Data发布火车Ingalls的GA发布。由于该发行版包含太多的功能以致无法在发行公告中进行介绍，因此我想使用本文更深入地了解火车上15个模块随附的更改和功能。

家政

发行版依赖项的一个非常根本的变化是升级到Spring Framework 4.3（当前为4.3.6）作为基准。其他依赖项升级主要由基础商店驱动程序和实现的主要版本变化驱动，这些变化需要反映在这些模块公开的API的潜在重大更改中。

Ingalls还附带了一个新的Spring数据模块：Spring Data LDAP。 Spring LDAP项目已经提供了Spring Data信息库支持已经有一段时间了。在遇到一些小故障和不兼容之后，我们决定将LDAP存储库支持移至单独的Spring Data模块中，以使其与发行版更加紧密地对齐。

模块设置的另一个重大变化是Apache Cassandra的Spring Data现在已经成为核心模块，这意味着它已经并且将由Pivotal的Spring Data团队维护。非常感谢以前的核心维护者David Webb和Matthew T.Adams所做的一切。

除了这些非常根本的更改之外，团队还一直在开发一系列新功能：

  • 使用方法句柄在转换子系统中进行属性访问。

  • 支持基于REST负载的XML和JSON投影（通用）

  • 使用Spring Data REST进行跨域资源共享

  • 更多用于数组，算术，日期和集合操作的MongoDB Aggregation Framework运算符。

  • 支持Redis Geo命令。

  • 升级到Cassandra 3.0，在存储库查询方法，用户定义类型，Java 8类型（可选，Stream），JSR-310和ThreeTen Backport中支持查询派生。

  • 支持Javaslang的Option ，存储库查询方法的collection和map类型。

这些是我想在本文其余部分中讨论的内容。
