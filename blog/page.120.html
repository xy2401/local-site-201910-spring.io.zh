Spring Cloud Camden RC1可用

发布
斯宾塞·吉布
九月14，2016

我很高兴地代表团队宣布， Spring Cloud Camden发布系列的发布候选版本1（RC1）现已发布。该版本可以在我们的Spring Milestone存储库中找到。这主要是一个错误修复版本，因为Camden为通用（GA）做准备。您可以查看Camden 发行说明以获取更多信息 。

卡姆登发布系列的重大变化

Spring Cloud Contract是一个新项目，为Spring应用程序中的消费者驱动合同和服务模式提供支持。Spring Cloud Netflix支持通过属性自定义Ribbon组件类，使用新社区维护的OpenFeign ，并对Zuul进行了各种修复和改进。Spring Cloud Consul使用Consul的事件api添加了对Spring Cloud Bus的支持。CLI添加了一个spring cloud命令，以使用单个命令启动各种Spring Cloud服务器。支持的服务器包括Eureka，Config Server和Hystrix仪表板。

网络研讨会：云中的数据微服务

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
九月13，2016

主讲人：马克·波拉克，马克·费舍尔
Spring Cloud Data Flow使您能够为许多常见用例创建数据管道，例如数据提取，实时分析和数据导入/导出。
在本次网络研讨会中，我们将介绍Spring Cloud Data Flow的体系结构，并介绍在多个运行时平台（例如Cloud Foundry，Kubernetes，Apache Mesos和Apache YARN）上长期运行和短期存在的以数据为中心的应用程序的编排功能。
Spring Cloud Data Flow代表了Spring XD的发展，并保留了DSL来定义数据管道以及基于Web的UI设计器，但将组件模型从用于在容器中运行的模块更改为使用Spring Cloud构建的标准Spring Boot应用程序流和Spring Cloud Task API。

SpringOne Platform 2016重放：12因素或云原生应用程序：这对Spring开发人员到底意味着什么？

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
九月12，2016

记录在SpringOne Platform 2016中。
演讲者：托马斯·甘布尔，家得宝
幻灯片： http : //www.slideshare.net/Pivotal/12-factor-or-cloud-native-apps-what-exactly-does-that-mean-for-spring-developers-64853692

您的团队对于开始使用Spring Boot和Cloud Native感到很兴奋，但是您不确定是否已准备好让团队使用本地桌面上的cf push持续交付产品。云原生开发的自由性可以为开发人员带来很大的力量，但这不应使运营和安全团队感到恐惧。我们将讨论如何设置快速而可靠的部署过程，以及将来一些有趣的事情。这些新范例最著名的描述之一是十二因子应用程序（12factor.net），它描述了云原生应用程序的元素。通过Spring框架可以直接满足其中的许多需求，而其他需求则需要其他系统的支持。在本次会议中，我们将研究十二个因素中的每一个，并介绍Spring和Cloud Foundry等平台如何满足它们，在某些情况下，我们甚至建议责任应从Spring转移到平台。最后，您将了解云原生应用程序需要什么，为什么以及如何满足这些要求。

SpringOne Platform 2016重播：Cloud Native Java

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
九月12，2016

记录在SpringOne Platform 2016中。
幻灯片： http ： //www.slideshare.net/SpringCentral/cloud-native-java

“没有必要改变。生存不是强制性的。” -W。爱德华兹·戴明

工作需要花费时间才能流经整个组织，最终要部署到生产中以获取价值。缩短生产时间至关重要。对于许多组织和行业来说，软件是一种竞争优势。组织将他们较大的软件野心分解为较小的，可独立部署的，以功能为中心的批量工作-微服务。为了减少工作地点之间的往返路程，组织尽可能地崩溃或合并，并使其余部分自动化。开发人员和操作变得“发展”，基于云的服务和平台（例如Cloud Foundry）使操作工作自动化，并消除了对ITIL凭单和变更管理委员会的需求。但是，就速度而言，速度是危险的。微服务带来了很少有人准备解决的架构复杂性。在本次演讲中，我们将研究Ticketmaster，阿里巴巴和Netflix等高性能组织如何利用Spring Boot和Spring Cloud来简化这种复杂性。
