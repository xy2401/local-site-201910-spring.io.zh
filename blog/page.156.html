Java EE 7采用的简要更新

新闻与活动
于尔根·霍勒（Juergen Hoeller）
2015年10月6日

提醒您：每当有人通过Java EE 7使用情况栏向您显示调查结果时，尤其是在谈论生产使用情况时，请仔细检查他们的实际要求。Hibernate 4.3的使用是否仅因为它是JPA 2.1（EE 7级别但独立运行）而被视为Java EE 7？Tomcat 8的使用是否仅因为它是Servlet 3.1而算作Java EE 7？当然，运行带有任何此类单独提供程序的Spring应用程序是否算作EE 7？

拿这样的统计数据来衡量：他们通常不要求“本机Java EE 7平台使用”，但最终会以这种方式出现。为什么可以安全地假设他们没有在生产中要求完全使用Java EE 7平台？好吧，请记住我六月起的博客帖子 ，该帖子在2015年底仍然有效。同时，唯一真实的消息是自从IBM在WebSphere的Liberty Profile中发布了对EE 7的支持以来，没有任何新闻。总而言之，对此类调查问题的回答者可能会根据“我正在使用该伞中的一个或多个规范”而不是“我正在使用其预期形式的完整平台”来回答。

2015年10月6日

我们今天很高兴在https://start.spring.io发行新版本的Spring Initializr！

最初是一个小的内部Web应用程序，它生成Spring Boot项目，后来发展到超出我们预期的范围。现在，您可以在Web上，喜欢的IDE（Eclipse STS和IntelliJ IDEA）中甚至通过命令行工具使用Spring Initializr（尝试curl https://start.spring.io ）。

同时，Spring产品组合正在增长，我们从Spring社区收到了很多有用的反馈。由于没有任何数据能胜过实际数据，因此我们改进了该服务，以便在夏季之前将其指标导出到集中式Redis实例。这使我们能够在长时间的活动中（以及与我们在Pivotal Web Services上部署的实例数量无关）保持可靠的统计数据集。
