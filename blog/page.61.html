Spring Framework 5.0.5和4.3.15现在可用

发布
于尔根·霍勒（Juergen Hoeller）
2018年4月3日

尊敬的Spring社区，

我很高兴地宣布，Spring Framework 5.0.5和4.3.15现已推出 ，是另一对优化版本，建议将其作为所有用户的立即升级。我们的5.0.5版本是本周晚些时候即将发布的Spring Boot 2.0.1版本的基础，而Boot.1.5.11将在大约同一时间使用4.3.15。

两种框架版本都具有大量的错误修复，还具有广泛的改进选择，在我们转向Spring Framework 5.1并将5.0.x和4.3.x系列转换为小批量维护模式之前，进行了总结。我们的5.0.5构建也准备好在最近发布的JDK 10上运行，从而允许在JDK 8以及9和10上同时构建和测试该框架。最后但并非最不重要的一点是，5.0.5（以及次要的4.3.15）在启动时对注释自省进行了显着的性能改进。

Spring Boot 2.0中的属性绑定

工程
菲尔·韦伯
2018年3月28日

从Spring Boot的第一个版本开始，就可以通过使用@ConfigurationProperties注解。也可以用不同的形式指定属性名称。例如， person.first-name ， person.firstName和PERSON_FIRSTNAME可以全部互换使用。我们称此功能为“轻松绑定”。

不幸的是，在Spring Boot 1.x中，“松弛绑定”显得有些放松。确切定义绑定规则是什么以及何时可以使用特定格式非常困难。我们还开始获得有关问题的报告，这些问题很难用我们的1.x实施来解决。例如，在Spring Boot 1.x中，无法将项目绑定到java.util.Set 。
