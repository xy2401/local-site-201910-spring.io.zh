Finchley发布火车的显着变化

介绍Spring Cloud功能

工程
马克·费舍尔
2017年7月5日

Spring Cloud Function是一个新项目，具有以下高级目标：

  • 通过功能促进业务逻辑的实现。
  • 将业务逻辑的开发生命周期与任何特定的运行时目标脱钩，以便相同的代码可以作为Web端点，流处理器或任务运行。
  • 支持跨无服务器提供程序的统一编程模型，以及独立运行（本地或在PaaS中）的能力。
  • 在无服务器提供程序上启用Spring Boot功能（自动配置，依赖项注入，指标）。

Spring Data Kay预览

工程
马克·帕鲁奇
2017年6月20日

正如我们刚刚发布了Spring Data发布火车Kay的第四个里程碑版本之后，让我们看一下自第一个里程碑以来火车上的13个模块随附的更改和功能。这篇博客文章涵盖了一系列更改，但并不意味着M2和M4之间的550多个更改是全面的。要获取更改的完整列表，请查看我们针对Kay M1 ， M2 ， M3和M4更改的Jira 。

以下是我们主要变更的精选清单：

  • 反应性支撑的调整

  • 可组合存储库

  • 改进的CRUD方法命名方案

  • 流利的MongoOperations API

  • MongoDB模板API的Kotlin扩展

  • MongoDB整理支持

  • Redis客户端配置

  • Cassandra轻量级事务支持和查询/更新对象

  • Java 9兼容性

  • 升级到Elasticsearch 5.4
