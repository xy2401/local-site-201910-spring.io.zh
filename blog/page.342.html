Spring Framework 2.0.3发布

发布
于尔根·霍勒（Juergen Hoeller）
2007年3月9日

尊敬的Spring社区，

我们很高兴地宣布Spring 2.0.3已发布。

下载 | 说明文件 | 变更日志
 

这是Spring 2.0系列中的错误修正和增强版本，包括大约200（！）完善和修复。此外，此版本还引入了对JSR-166（java.util.concurrent）反向项目的支持和对JCA 1.5的基本支持（ResourceAdapter引导程序，WorkManager支持），以及对IBM WebSphere和Oracle OC4J的扩展支持。

其中值得注意的增强功能包括DefaultMessageListenerContainer的动态缩放功能和经过彻底修改的脚本集成，支持更多风格的Groovy和BeanShell脚本编写，并提供与Spring AOP工具的无缝交互。此版本还包括对AspectJ切入点支持以及范围代理功能的改进。

有关详细信息，请参见更改日志！

干杯，
于尔根

Spring Framework 1.2.9发布

发布
于尔根·霍勒（Juergen Hoeller）
2007年3月9日

尊敬的Spring社区，

我很高兴地宣布Spring 1.2.9终于发布了。下载 | 文件 | 变更日志 。

这是针对仍在使用1.2系列的用户的错误修正和完善版本，旨在作为Spring 1.2.8的嵌入式更新。它包含自1.2.8起报告的所有问题的修复程序，并包括Spring 2.0分支的各种反向移植。

请注意，这是Spring 1.2分支中最后一个计划的发行版。仅在出现严重问题时才提供1.2.9的修补程序版本。我们建议升级到Spring 2.0以便长期使用。

工程
罗德·约翰逊
2007年2月11日

祝贺WebLogic团队发布了通过Java EE 5 CTS 的WebLogic 10预览版 。很高兴看到BEA在使用J2EE 1.4异常之后重新回到了迅速摆脱困境的传统。

对于Spring社区而言，这是一个有趣的消息，因为WebLogic 10在内部使用Pitchfork 。Pitchfork项目是一个由Interface21领导的开源项目，与BEA共同开发，该项目在Spring的顶部实现了EJB 3.0拦截和JSR-250注入。它可在WebLogic 10 EJB容器内部以及服务器的其他部分使用，以满足新的规范要求，但不与WebLogic绑定。Pitchfork可以独立使用（尽管这不是其主要目标），也可以被其他应用程序服务器采用。

为什么叫Acegi？

工程
本·亚历克斯
2007年1月25日

在Rod最近关于接口名称来源的博客， 最近询问Acegi Security重命名的线程以及去年晚些时候我的同事Ben Hale关于博客关于“ Acegi”名称的起源的建议之间，我决定可能是时候这样做了！

首先是发音：“ ah-see-gee”。现在我们已经解决了这个问题，让我们看一下它的来源：


  • 早在1990年代初期，我就开始使用公告栏系统 。我仍然记得使用我的第一个300波特调制解调器，实际上看着字符以与我阅读它们相同的速度出现！无论如何，一个BBS都需要一个名称，我非常喜欢“ Midnight BBS”（或者是“ Lightning BBS”？）。尝试在澳大利亚BBS注册中心注册我的首选姓名后，我接到一个电话，通知该姓名已被使用。我搜索了另一个名称，没有发现冲突，并且更改了所有屏幕（有人记得Avatar吗？）。然后，我用新名称注册了BBS，尽管其他人刚刚用同样的名字殴打了我。大约在澳大利亚，大约有800个BBS。因此，我决定使用保证是原始的内容：英语字母中的字符1、3、5、7和9。Acegi BBS随后出生。

  • 这是BBS的鼎盛时期。它有来自澳大利亚各地的呼叫者，这是我们区号中第一个获得看似无限容量的CD-ROM的BBS。我成为Fidonet的3：624网络协调员，并帮助在整个地区分发了“ echomail”。不久之后，人们开始向“ Acegi BBS”发送支票，因为他们希望提高文件下载限制并访问Fidonet的网络邮件。

  • 因为我需要用“ Acegi”的名义兑现支票，所以在1993年初，我们决定注册一个公司名称。选择Acegi Computer Technology，人们因此高兴地做出了检查。

  • 大约在1995年，我带着工作搬到了悉尼。拨号上网才刚刚开始在澳大利亚公开可用（调制解调器访问每小时仍需花费约5澳元！）。因为BBS从某个特定区域吸引了大多数用户，而我正要离开该区域，所以我将Acegi BBS交给了一个朋友，以继续运行。随后，他也搬家了，并通过BBS转给了另一个朋友。在那之后我迷路了。我使用Google搜索“ Acegi BBS”，甚至发现了1995年的旧唱片 。

  • 到1996年，我需要一家公司来从事IT项目，因此Acegi Technology Pty Limited诞生了。此后一直保持运行状态。

  • 2003年11月左右的某个时候，我写了最终成为Acegi Security的文章。我将其放入一个ZIP文件中，并将其发射到Rod和Juergen。我提议将这个新项目称为“Spring Security ”。但是，他们当时没有时间对其进行全面审查，因此建议我只是将其作为“ _____ Spring安全系统”发布。因此，我在名称前加上了“ Acegi”。截至今天，在Google上搜索“ Acegi”的点击量约为660,000，几乎全部与安全框架有关。

动态数据源路由

工程
马克·费舍尔
2007年1月23日

Spring 2.0.1引入了AbstractRoutingDataSource 。我认为它值得关注，因为（基于客户的常见问题）我有一种直觉，认为有很多“本土”解决方案可以解决这个问题。加上实施起来很简单却容易忽略的事实，现在我有几个理由可以摆脱团队博客的困扰。

一般的想法是DataSource充当中介-可以在运行时根据查找关键字动态确定“真实”数据源。一种潜在的用例是确保标准JTA不支持的特定于事务的隔离级别。为此，Spring提供了一个实现： IsolationLevelDataSourceRouter 。请查阅其JavaDoc以获取包括配置示例的详细描述。

使用AJAX和Spring集成进行Java到JavaScript的编译

工程
本·亚历克斯
2007年1月22日

一段时间以来，我一直对以客户端为中心的基于Web的用户界面感兴趣。这些第四代框架的特征在于它们基于组件的，事件驱动的编程模型，并且专注于完全驻留在客户端上的表示逻辑。以这种方式定位Web浏览器通常需要使用JavaScript或Flash ，这本身就带来了许多独特的挑战。

如果我们可以使用Java编程并自动生成JavaScript或Flash运行时模块，则有可能解决许多挑战。今天实现这一目标的两个著名产品分别是Google Web Toolkit （GWT）和Open Laszlo 。两者都可以通过OSI批准的许可证获得，并且拥有活跃的社区以及它们自己独特的复杂性。一个考虑因素是它们在多大程度上实现了提供针对Web浏览器部署的基于Java的透明开发环境的目标。此注意事项有多个方面，包括IDE支持，调试集成，反射功能，运行时窗口小部件绑定等。当使用诸如Swing和Standard Widget Toolkit （SWT）之类的传统Java技术开发富客户端时，所有这些都是正常的考虑因素。
