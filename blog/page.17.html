我很荣幸地代表Spring Integration团队宣布Spring Integration 5.2.0.M2可用。

Apache Kafka 2.3里程碑2的春天

发布
加里·罗素
五月13，2019

我们很高兴地宣布，Apache Kafka 2.3发行版2.3.0已发布Spring的第二个里程碑。M2。

强调

  • SeekToCurrentErrorHandler可以配置为不重试某些异常。

  • 新的RecoveringDeserializationExceptionHandler可以从Kafka Streams应用程序中失败的反序列化操作中恢复，例如，将失败的消息发送到死信主题。

  • 新的HeaderEnricher可以将变压器添加到Kafka Streams拓扑以添加标头。

  • 新的MessagingTransformer可用于在Kafka Streams拓扑内调用Spring Integration流。

  • 新的AggregatingReplyingKafkaTemplate可以发送记录并等待来自多个使用者的响应。

有关更多信息，请参阅文档中的“新增功能”一章和更改日志。

  • 更改M1的日志

  • 更改M2的日志

此外，针对Apache Kafka的Spring Integration（ spring-integration-kafka ）3.2.0。M2可用；它基于适用于Apache Kafka 2.3和Spring Integration 5.2的Spring。

  • 的KafkaMessageSource消费者现在可以暂停/恢复。

  • 的KafkaMessageSource现在可以将使用者配置为在每次轮询中检索多个记录。

  • XML配置现在可用于所有组件。

有关更多信息，请参见Spring Integration章节 。

Spring Batch 4.2.0。M2现已上市！

发布
Mahmoud Ben Hassine
五月13，2019

我很高兴地宣布Spring Batch 4.2.0。M2现在可以从我们的里程碑资料库中获得 。

什么是新的？

此版本为Apache Kafka添加了新的项目读取器和新的项目写入器：

  • KafkaItemReader可以从同一主题的单个分区或多个分区中读取消息。该阅读器将消息偏移量存储在ExecutionContext支持重启。

可用的Spring Integration AWS 2.2 GA和Spring Cloud Stream Kinesis Binder 1.2 GA

发布
阿尔特姆·比兰（Artem Bilan）
五月13，2019

今天，我很高兴宣布适用于Amazon Web Services扩展版本的Spring Integration的常规可用性2.2.0和适用于AWS Kinesis版本的Spring Cloud Stream Binder 1.2.0 。

这些版本可以从Maven Central，JCenter和我们的版本库中下载 ：

compile "org.springframework.integration:spring-integration-aws:2.2.0.RELEASE"

如果您不使用Kinesis Binder。或通过Binder依赖项：

compile "org.springframework.cloud:spring-cloud-stream-binder-kinesis:1.2.0.RELEASE"

这两个版本的主题都是关于支持Kinesis Client和Kinesis Producer库的新功能。为此， spring-integration-aws附带KclMessageDrivenChannelAdapter和KplMessageHandler实现。目前，实现基于KCL v1.x ，因为并非我们在spring-integration-aws提供AWS SKD v2实施。在Kinesis Client Library调查的基础上， CheckpointMode.periodic已经介绍给两个KclMessageDrivenChannelAdapter和KinesisMessageDrivenChannelAdapter 。
