Spring Cloud Data Flow中的组合功能支持

工程
Ilayaperumal Gopinathan
一月09，2019

Spring Cloud Stream最近添加了一项功能，以将功能定义组合到现有的Spring Cloud Stream应用程序中。在此博客中，我们将看到Spring Cloud Data Flow如何
利用此功能来组成Streaming管道中的功能。

有什么不同？

在Spring Cloud Data Flow中，流数据管道由Spring Cloud Stream应用程序组成。开发人员可以选择现成的流应用程序 ，该应用程序涵盖了许多常见的用例。开发人员还可以使用Spring Cloud Stream框架扩展这些现成的应用程序或创建自定义应用程序。

Spring Cloud Stream 2.1.0 GA的发布包含基于Spring Cloud Function的编程模型的集成，该模型允许将业务逻辑表示为java.util.Function ，一个java.util.Consumer和一个java.util.Supplier ，分别代表Processor ， Sink和Source的角色。有了这种灵活性，Spring Cloud Stream框架现在支持一种简单但功能强大的函数组合方法。在这种情况下，组合可以是将源和处理器组合到一个应用程序中：“新源”。否则，可能是处理器+接收器组合成一个应用程序：“新接收器”。这种灵活性为流应用程序开发人员带来了有趣的新机会。

让我们考虑如何使用三个应用程序创建管道以执行简单的转换，然后看看如何通过使用两个使用功能组合的应用程序将其实现为管道。

流管道具有三个应用程序

对于第一流，

我们将使用现成的http-source ， transform-processor和log-sink应用程序。

第一步，启动Spring Cloud Data Flow local服务器：

java -jar spring-cloud-dataflow-server-local-1.7.3.RELEASE.jar

现在，启动Spring Cloud Data Flow shell ：

java -jar spring-cloud-dataflow-shell-1.7.3.RELEASE.jar

现在，我们注册使用RabbitMQ绑定器的HTTP源，转换器处理器和日志接收器应用程序：

dataflow:>app register --name http --type source --uri https://repo.spring.io/milestone/org/springframework/cloud/stream/app/http-source-rabbit/2.1.0.M2/http-source-rabbit-2.1.0.M2.jar

dataflow:>app register --name transformer --type processor --uri https://repo.spring.io/milestone/org/springframework/cloud/stream/app/transform-processor-rabbit/2.1.0.M2/transform-processor-rabbit-2.1.0.M2.jar

dataflow:>app register --name log --type sink --uri https://repo.spring.io/milestone/org/springframework/cloud/stream/app/log-sink-rabbit/2.1.0.M2/log-sink-rabbit-2.1.0.M2.jar

现在我们可以创建一个没有函数组成的简单流：

dataflow:>stream create hello --definition "http --server.port=9000 | transformer --expression=(\"Hello \"+payload.toString().toUpperCase()) | log"

然后我们可以部署流：

dataflow:>stream deploy hello --properties "deployer.*.local.inheritLogging=true"

dataflow:>http post --data "friend" --target "http://localhost:9000"
POST (text/plain) http://localhost:9000 friend
202 ACCEPTED

您可以在log应用程序中看到以下日志消息：

[sformer.hello-1] log-sink                                 : Hello FRIEND

在此流中，我们将http（源），translator（处理器）和log（sink）应用程序部署为目标平台中的独立应用程序（在本例中为local ）。对于某些用例，对于简单的有效负载转换逻辑，我们可能希望将Processor应用程序与Source或Sink应用程序结合在一起。例如，诸如在源输出数据处屏蔽某些特定用户特定字段之类的转换方案不一定需要部署为单独的独立应用程序。相反，它可以在Source或Sink应用程序中组成。

要将Processor函数组合到Source或Sink应用程序中，我们使用Spring Cloud Stream的函数组合支持。
