Spring Cloud Data Flow中的组合功能支持

工程
Ilayaperumal Gopinathan
一月09，2019

Spring Cloud Stream最近添加了一项功能，以将功能定义组合到现有的Spring Cloud Stream应用程序中。在此博客中，我们将看到Spring Cloud Data Flow如何
利用此功能来组成Streaming管道中的功能。

有什么不同？

在Spring Cloud Data Flow中，流数据管道由Spring Cloud Stream应用程序组成。开发人员可以选择现成的流应用程序 ，该应用程序涵盖了许多常见的用例。开发人员还可以使用Spring Cloud Stream框架扩展这些现成的应用程序或创建自定义应用程序。

Spring Cloud Stream 2.1.0 GA的发布包含基于Spring Cloud Function的编程模型的集成，该模型允许将业务逻辑表示为java.util.Function ， 一种java.util.Consumer和java.util.Supplier ，代表Processor ， Sink和Source分别。有了这种灵活性，Spring Cloud Stream框架现在支持一种简单而强大的功能组合方法。在这种情况下，组合可能是将源和处理器组合成一个应用程序：“新源”。否则，可能是处理器+接收器组合成一个应用程序：“新接收器”。这种灵活性为流应用程序开发人员带来了有趣的新机会。

让我们考虑如何使用三个应用程序创建管道以执行简单的转换，然后看看如何通过使用两个使用功能组合的应用程序将其实现为管道。

流管道具有三个应用程序

对于第一流，

我们将使用开箱即用的http-source ， transform-processor和log-sink应用程序。

第一步，启动Spring Cloud Data Flow local服务器：

java -jar spring-cloud-dataflow-server-local-1.7.3.RELEASE.jar

现在，启动Spring Cloud Data Flow shell ：

java -jar spring-cloud-dataflow-shell-1.7.3.RELEASE.jar

现在，让我们注册使用RabbitMQ绑定器的HTTP源，转换器处理器和日志接收器应用程序：

dataflow:>app register --name http --type source --uri https://repo.spring.io/milestone/org/springframework/cloud/stream/app/http-source-rabbit/2.1.0.M2/http-source-rabbit-2.1.0.M2.jar

dataflow:>app register --name transformer --type processor --uri https://repo.spring.io/milestone/org/springframework/cloud/stream/app/transform-processor-rabbit/2.1.0.M2/transform-processor-rabbit-2.1.0.M2.jar

dataflow:>app register --name log --type sink --uri https://repo.spring.io/milestone/org/springframework/cloud/stream/app/log-sink-rabbit/2.1.0.M2/log-sink-rabbit-2.1.0.M2.jar

现在我们可以创建一个没有函数组成的简单流：

dataflow:>stream create hello --definition "http --server.port=9000 | transformer --expression=(\"Hello \"+payload.toString().toUpperCase()) | log"

然后我们可以部署流：

dataflow:>stream deploy hello --properties "deployer.*.local.inheritLogging=true"

dataflow:>http post --data "friend" --target "http://localhost:9000"
POST (text/plain) http://localhost:9000 friend
202 ACCEPTED

您可以在上看到以下日志消息log应用：

[sformer.hello-1] log-sink                                 : Hello FRIEND

在此流中，我们将http（源），translator（处理器）和log（sink）应用程序部署为目标平台中的独立应用程序（在本例中为local ）。对于某些用例，对于简单的有效负载转换逻辑，我们可能希望拥有Processor应用程序与Source要么Sink应用程序。例如，诸如在源输出数据处屏蔽某些特定用户特定字段之类的转换方案不一定需要部署为单独的独立应用程序。相反，它可以在Source或Sink应用程序中组成。

要将Processor函数组合到Source或Sink应用程序中，我们使用Spring Cloud Stream的函数组合支持。

Spring Cloud Stream中的功能组合支持基于Spring Cloud Function允许注册以下内容的能力java.util.Supplier ， java.util.Consumer和java.util.Function作为 Spring @Bean定义。这些功能@Bean定义可在运行时用于合成。

Spring Cloud Stream引入了一个新属性，名为spring.cloud.stream.function.definition ，它对应于Spring Cloud Function中的功能定义DSL。设置此属性后，所需的功能Bean将在运行时自动链接。

函数组合按以下方式发生：

当Spring Cloud Stream应用程序为类型Source ，在源之后应用组合函数output 。

当Spring Cloud Stream应用程序为类型Sink ，组合函数在接收器之前应用input 。

这样就可以将功能（在Spring Cloud Function DSL中定义）组合到现有的Spring Cloud Stream应用程序中，并随后由Spring Cloud Data Flow在流数据管道中进行编排。

将功能组合到Stream应用程序中

让我们创建并部署一个将前面示例的转换器表达式组成的流Source应用程序本身。变压器逻辑通过使用两个完成java.util.Function实现。

我们将创建一个新的源应用程序，我们将其称为http-transformer这扩展了现成的http源应用程序。可在此处找到新源应用程序的源。

的http-transformer应用程序包含upper和concat功能bean，定义如下：

@SpringBootApplication
@Import(org.springframework.cloud.stream.app.http.source.HttpSourceConfiguration.class)
public class HttpSourceRabbitApplication {

	@Bean
	public Function<String, String> upper() {
		return value -> value.toUpperCase();
	}

	@Bean
	public Function<String, String> concat() {
		return value -> "Hello "+ value;
	}


	public static void main(String[] args) {
		SpringApplication.run(HttpSourceRabbitApplication.class, args);
	}
}

克隆github repo之后 ，您可以使用maven构建应用程序：

cd功能组成/ http转换器
./mvnw清洁包

现在注册http-transformer使用数据流外壳程序

注意

对于以下应用程序注册--uri选项，用适合您系统的值替换工件的目录名称和路径。

dataflow:>app register --name http-transformer --type source --uri file:///Users/igopinathan/dev/git/ilayaperumalg/sandbox/function-composition/http-transformer/target/http-transformer-2.1.0.BUILD-SNAPSHOT.jar

现在让我们创建流：

dataflow:>stream create helloComposed --definition "http-transformer --server.port=9001 | log"

部署流时，我们通过spring.cloud.stream.function.definition属性定义组合功能DSL（如Spring Cloud Function中定义）。在这种情况下，它是：

dataflow:>stream deploy helloComposed --properties "app.http-transformer.spring.cloud.stream.function.definition=upper|concat,deployer.*.local.inheritLogging=true"

上面的部署组成了upper和concat功能 bean 入http源应用程序。

然后我们可以将有效载荷发送到http应用：

dataflow:>http post --data "friend" --target "http://localhost:9001"
> POST (text/plain) http://localhost:9001 friend
> 202 ACCEPTED

然后您可以在log应用为

[helloComposed-1] log-sink                                 : Hello FRIEND

注意

请注意，功能组合支持不适用于现成的Spring Cloud Stream Processor应用程序，因为在需要在现有处理器的应用程序逻辑之前还是之后应用该功能存在歧义。

但是，您可以使用标准的java.util创建使用功能组合的自己的处理器应用程序。函数API，如以下示例所示：

@Configuration
public static class FunctionProcessorConfiguration {

@Bean
public Function<String, String> upperAndConcat() {
return upper().andThen(concat());
}

  @Bean
  public Function<String, String> upper() {
     return value -> value.toUpperCase();
  }

  @Bean
  public Function<String, String> concat() {
     return value -> "Hello "+ value;
  }
}

然后，您需要使用以下属性进行部署：spring.cloud.stream.function.definition=upperAndConcat

Kotlin支持

另一个有趣的功能是Spring Cloud Function支持Kotlin函数的功能组合。这使我们可以将任何Kotlin函数bean添加到可组合的函数中Source要么Sink应用程序。

要查看其工作原理，请使用http-transformer-kotlin-processor来自示例github 存储库的应用程序。

Kotlin功能Bean被配置为处理器。在这里，Kotlin函数bean是transform功能定义如下：

@Bean
open fun transform(): (String) -> String {
   return { "How are you ".plus(it) }
}

此外，这个专案spring-cloud-function-kotlin作为对Kotlin函数应用函数配置支持的依赖项，定义如下：

<dependency>
      <groupId>org.springframework.cloud</groupId>
      <artifactId>spring-cloud-function-kotlin</artifactId>
      <version>2.0.0.RELEASE</version>
    </dependency>

光盘功能组成/ http-transformer-kotlin
./mvnw清洁包

注意

对于以下应用程序注册--uri选项，用适合您系统的值替换工件的目录名称和路径。

dataflow:>app register --name http-transformer-kotlin --type source --uri file:///Users/igopinathan/dev/git/ilayaperumalg/sandbox/function-composition/http-transformer-kotlin/target/http-transformer-kotlin-2.1.0.BUILD-SNAPSHOT.jar

使用此应用程序创建流Source ：

dataflow:>stream create helloComposedKotlin --definition "http-transformer-kotlin --server.port=9002 | log"

就像我们在http-transformer例如，我们可以使用spring.cloud.stream.function.definition属性，以指定任何有效的组合功能DSL来构造功能组合。在这种情况下，让我们将通过Java配置注册的功能bean与来自Kotlin处理器配置的功能bean结合起来。

dataflow:>stream deploy helloComposedKotlin --properties "app.http-transformer-kotlin.spring.cloud.stream.function.definition=upper|transform|concat,deployer.*.local.inheritLogging=true"

在这里，函数名称transform对应于Kotlin函数。

注意：我们可以在Kotlin函数和Java函数之间执行合成，因为Kotlin函数在内部被转换为java.util.Function 。

dataflow:>http post --data "friend" --target "http://localhost:9002"
> POST (text/plain) http://localhost:9002 friend
> 202 ACCEPTED

并且，您可以在log申请为：

[omposedKotlin-1] log-sink               : Hello How are you FRIEND

在此示例中， http-transformer还包含函数的源代码。但是，您可以通过在单独的工件中定义功能Bean，使应用程序更具模块化。然后，您可以通过仅向项目添加maven依赖项并设置spring.cloud.stream.function.definition属性。这样，您就可以将大多数业务逻辑编码为一个函数，并且可以在必要时将其与Source或Sink组合在一起。
