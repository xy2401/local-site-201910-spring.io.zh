Spring Cloud Data Flow 2.2.0发布

发布
珍妮·瓦尔基亚拉赫蒂
2019年7月30日

Spring Cloud Data Flow团队很高兴宣布GA版本2.2.0。

在此GA版本中，Spring Cloud Data Flow团队研究了一些关键功能，包括任务应用程序管理，通过增加跨平台（本地，Kubernetes和Cloud Foundry）的验收测试范围，产品的稳定性，错误修复和增强功能。

任务应用管理

现在可以使用SCDF仪表板和命令行管理程序停止和删除任务执行。
我们增加了对使用千分尺集成的任务应用程序监视的支持。Micrometer集成的核心位于Spring Cloud Task的2.2.0版本中，如果您要尝试Task-metrics和SCDF集成，则这是必需的。

为了方便起见，我们有一个示例应用程序 ，该应用程序基于Spring Boot，Spring Batch和Spring Cloud Task的兼容上游版本构建。通过在SCDF中启动此应用程序，您现在可以使用InfluxDB作为后端来检测指标，并且同样可以通过Grafana仪表板可视化统计信息，如下所示。

访问流/任务应用程序日志

Spring Cloud Data Flow仪表板显示了流和任务应用程序日志。该功能在本地，Kubernetes和Cloud Foundry平台中可用。Spring Cloud Data Flow的仪表板现在可以显示流和任务应用程序日志。此功能由Spring Cloud Data Flow提供的新REST API提供支持。

修补数据库驱动程序

尝试使用专有数据库驱动程序修补Spring Cloud Data Flow的社区成员和客户可以选择在本地克隆和构建项目。
不过，还有另一种选择。现在已为Maven和Gradle用户记录了该过程。

验收测试
我们在支持的平台Local，Pivotal Cloud Foundry，Kubernetes（GKE和PKS）上提高了验收测试的质量。这使我们对继续接受的优质产品充满信心。

开发者食谱
添加了新配方，以帮助开发人员使用Apache Kafka，RabbitMQ和Amazon Kinesis配置和管理流应用程序。在Kubernetes和Cloud Foundry上还有关于文件提取和ETL处理的食谱。参见Spring Cloud Data Flow 网站 。鉴于所有内容编码为降价文件，很容易为社会做贡献，可以随意开始拉请求对话框-我们期待着您的贡献在这里
