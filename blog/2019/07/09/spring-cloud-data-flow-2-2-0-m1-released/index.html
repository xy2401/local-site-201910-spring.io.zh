Spring Cloud Data Flow 2.2.0。M1发布

工程
Ilayaperumal Gopinathan
2019年7月9日

Spring Cloud Data Flow团队很高兴宣布里程碑版本2.2.0。M1。

除了一些增强功能和错误修复之外，以下是该版本的一些重点：

任务指标
指标和监视仍然是Spring Cloud Data Flow的重要主题之一。在2.2中基于流的InfluxDB集成的基础上，我们也添加了对Tasks的支持。Micrometer集成的核心位于Spring Cloud Task的2.2.0版本中，如果您要尝试Task-metrics和SCDF集成，则这是必需的。

为了方便起见，我们有一个示例应用程序 ，该应用程序基于Spring Boot，Spring Batch和Spring Cloud Task的兼容上游版本构建。通过在SCDF中启动此应用程序，您现在可以使用InfluxDB作为后端来检测指标，并且同样可以通过Grafana仪表板可视化统计信息，如下所示。

应用日志
现在可以使用一组REST端点来访问流和任务应用程序的应用程序日志。当前，此功能可用于在Cloud Foundry和Kubernetes平台中运行的应用程序。在即将发布的版本中，我们将具有仪表板访问权限，以查看流/任务应用程序日志以及本地平台支持。

修补数据库驱动程序
尝试使用专有数据库驱动程序修补Spring Cloud Data Flow的社区成员和客户可以选择在本地克隆和构建项目。不过，还有另一种选择。现在已为Maven和Gradle用户记录了该过程。

验收测试
为了进一步提高我们构建的软件的质量并更加信任所发布的产品，我们投入了时间来循环进行和检查验收测试。我们现在继续在本地，Pivotal Cloud Foundry，Kubernetes（GKE）和PKS上进行验收测试。

开发者食谱
现在，最新的Spring Cloud Data Flow 网站上提供了Apache Kafka，RabbitMQ，K8s / CF中的File-ingest和ETL以及Amazon Kinesis配方。鉴于所有内容编码为降价文件，很容易为社会做贡献，可以随意开始拉请求对话框-我们期待着您的贡献在这里
