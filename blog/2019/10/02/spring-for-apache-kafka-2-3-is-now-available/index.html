Apache Kafka 2.3的Spring现在可用

发布
加里·罗素
十月02，2019

我们很高兴地宣布以下版本现已可用。

鼓励所有用户升级到这些版本

  • 2.3.0。发布（ 新 ）

  • 2.2.9。发布（ 维护 ）

2.3.0版本是该项目最新版本的第一个版本。

有关更多信息，请参阅“ 新增功能”一章，但其中有一些重点：

生产中

  • 选择在每个线程中使用生产者，以避免在刷新时阻塞其他线程。

  • 的AggregatingReplyingKafkaTemplate扩展到ReplyingKafkaTemplate提供以汇总来自多个消费者的答复。

消费中

  • 您现在可以添加一个RecordInterceptor在调用侦听器之前修改记录。

  • ConsumerSeekAware现在支持相对搜寻。

  • 现在，您可以指定处理上一个结果之间的延迟poll()并发布下一个poll() 。

  • 使用手册时AckMode例如，您现在可以否定地确认记录，并且容器将执行必要的搜索以重播记录。

  • 千分尺Timer现在支持监视监听器性能。

  • 的SeekToCurrentErrorHandler可以配置为将某些异常视为致命异常，从而禁用重试；您现在也可以添加一个BackOff重新交付失败的交付尝试之间。

  • 的RetryingDeserializer例如，在网络故障访问模式注册表的情况下，提供来重试序列化。

卡夫卡流

  • 的RecoveringDeserializationExceptionHandler现在提供用于处理反序列化异常的代码。

  • 的HeaderEnricherTransformer提供用于在流拓扑定义中添加标头。

  • 的MessagingTransformer提供了，以允许从流定义中调用Spring Integration流。

杂

  • 提供了委托序列化器和反序列化器实现，以允许发送/接收包含不同类型的记录。
