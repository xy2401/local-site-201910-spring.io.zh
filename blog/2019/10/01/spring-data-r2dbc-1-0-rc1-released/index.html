Spring Data R2DBC 1.0 RC1发布

发布
马克·帕鲁奇
十月01，2019

我代表团队和所有贡献者，高兴地宣布，Spring Data R2DBC 1.0的第一个候选发行版已经发布，可以从我们的里程碑资料库中获得。此版本包含60个问题和请求请求。它将其基准升级到R2DBC 0.8 RC1和Spring Data Moore GA。

最值得注意的功能是：

  • ConnectionFactory路由通过AbstractRoutingConnectionFactory 。
  • 通过以下方式初始化模式的实用程序ResourceDatabasePopulator和ScriptUtils 。
  • 通过以下方式传播和重置自动提交和隔离级别控制TransactionDefinition 。
  • 支持实体级转换器。
  • Kotlin扩展，用于通用泛型和协程。
  • 添加可插拔机制来注册方言。
  • API改进。

ConnectionFactory路由

对于每个操作， DatabaseClient获得一个Connection来自ConnectionFactory用于创建DatabaseClient 。

一般的想法是ConnectionFactory充当中介人-而“真实” ConnectionFactory是在运行时根据查找键动态确定的。一种潜在的用例是通过使用多个来满足多租户ConnectionFactory实例指向一个单独的数据库。对于路由ConnectionFactory确定哪个ConnectionFactory要使用它，需要上下文信息。Spring Data R2DBC附带了一个AbstractRoutingConnectionFactory可以返回用于向右查找的查找键ConnectionFactory 。

反应性流程可以使用Project Reactor的内容将上下文信息附加到订阅Context 。

路由的示例实现ConnectionFactory可能看起来像下面的清单：

class RoutingConnectionFactory extends AbstractRoutingConnectionFactory {

    public static final String ROUTING_KEY = "routing-key";

    @Override
    protected Mono<Object> determineCurrentLookupKey() {
        return Mono.subscriberContext().filter(it -> it.hasKey(ROUTING_KEY)).map(it -> it.get(ROUTING_KEY));
    }
}

下面的清单显示了如何初始化连接工厂，查找键和目标之间的映射ConnectionFactory ：

RoutingConnectionFactory connectionFactory = new RoutingConnectionFactory();
Map<String, ConnectionFactory> factories = new HashMap<>();
factories.put("customer1", …);
factories.put("customer2", …);
connectionFactory.setTargetConnectionFactories(factories);
connectionFactory.afterPropertiesSet();

您可以创建一个DatabaseClient通过使用路由ConnectionFactory确定合适的ConnectionFactory对于每个订阅。
路由密钥本身已附加到Context并由RoutingConnectionFactory ，如以下清单所示：

DatabaseClient client = DatabaseClient.create(connectionFactory);

Context routingContext = Context.of(ROUTING_KEY, "customer1");

Flux<User> users = client.execute("SELECT first_name, last_name FROM user")
                .as(User.class)
                .fetch()
                .all()
                .subscriberContext(routingContext);

模式初始化

的org.springframework.data.r2dbc.connectionfactory.init软件包提供了对现有的初始化的支持ConnectionFactory 。有时您可能需要初始化一个实例，该实例在某处的服务器上运行，或者是嵌入式数据库。以下清单显示了如何注册和配置ConnectionFactoryInitializer bean通过以下方式初始化数据库ConnectionFactory 。

@Configuration
public class InitializerConfiguration {

  @Bean
  public ConnectionFactoryInitializer initializer(ConnectionFactory connectionFactory) {

    ConnectionFactoryInitializer initializer = new ConnectionFactoryInitializer();
    initializer.setConnectionFactory(connectionFactory);

    CompositeDatabasePopulator populator = new CompositeDatabasePopulator();
    populator.addPopulators(new ResourceDatabasePopulator(new ClassPathResource("com/foo/sql/db-schema.sql")));
    populator.addPopulators(new ResourceDatabasePopulator(new ClassPathResource("com/foo/sql/test-data1.sql")));
    initializer.setDatabasePopulator(populator);

    return initializer;
  }
}

Spring Boot R2DBC自动配置使用此初始化支持来应用schema.sql和data.sql到配置ConnectionFactory 。

Kotlin扩展

此发行版随附Kotlin扩展，用于在使用Kotlin开发应用程序时惯用Spring Data R2DBC。

Spring Data R2DBC提供了以下扩展：

  • 改进的泛型支持DatabaseClient和Criteria
  • Kotlin Coroutines支持DatabaseClient

检索列表User Java中的对象，通常应编写以下内容：

Flux<User> characters = client.select().from(User.class).fetch().all();

使用Kotlin和Spring Data扩展，您可以编写以下内容：

val users =  client.select().from<User>().fetch().all()

Kotlin协程是Kotlin轻量级线程，允许以命令式风格编写非阻塞代码。
Spring Data R2DBC的Coroutines扩展利用了反应式基础架构来公开您可以使用的功能suspend和桥Flux结果进入科特林的Flow类型。

以下示例显示了协程在Spring Framework中的用法TransactionalOperator自动插入两行：

operator.executeAndAwait {
  client.execute("INSERT INTO person VALUES(:first_name, :last_name)")
        .bind("first_name", "John")
        .bind("last_name", "Doe")
        .await()
  client.execute("INSERT INTO person_events VALUES(:first_name, :last_name, :event_type)")
        .bind("first_name", "John")
        .bind("last_name", "Doe")
        .bind("event_type", "CREATED")
        .await()
}

请注意， suspend方法以await ，作为协程方法的指标。

可插拔方言

到目前为止，R2DBC驱动程序生态系统仅由几个已知的驱动程序组成，但是驱动程序的数量正在增长。为了实现与其他驱动程序的无缝集成，Spring Data R2DBC公开了一个扩展点，用于R2dbcDialect注册。Spring Data R2DBC可以自动发现R2dbcDialect通过注册一个实现的类org.springframework.data.r2dbc.dialect.DialectResolver$R2dbcDialectProvider通过春天的META-INF/spring.factories机制。以下清单显示了如何执行此操作：

public class MyDialectProvider implements DialectResolver.R2dbcDialectProvider {

    @Override
    public Optional<R2dbcDialect> getDialect(ConnectionFactory connectionFactory) {

        if (connectionFactory.getMetadata().getName().equals("my-dastabase")) {
            return Optional.of(…);
        }
        return Optional.empty();
    }
}

弹簧工厂

org.springframework.data.r2dbc.dialect.DialectResolver$R2dbcDialectProvider=com.example.MyDialectProvider

工件坐标

如果您使用Maven，请在您的pom.xml文件升级到Spring Data R2DBC 1.0 RC1：

<dependencies>
  <dependency>
    <groupId>org.springframework.data</groupId>
    <artifactId>spring-data-r2dbc</artifactId>
    <version>1.0.0.RC1</version>
  </dependency>
</dependencies>

<!-- R2DBC 0.8.0.RC1 required -->
<dependencyManagement>
  <dependencies>
    <dependency>
      <groupId>io.r2dbc</groupId>
      <artifactId>r2dbc-bom</artifactId>
      <version>Arabba-RC1</version>
      <type>pom</type>
      <scope>import</scope>
    </dependency>
  </dependencies>
</dependencyManagement>

<repositories>
  <repository>
    <id>spring-milestone</id>
    <url>https://repo.spring.io/milestone</url>
  </repository>
</repositories>
