Apache Geode和Pivotal GemFire 1.1.2的Spring Boot。发布和1.2.0。RC1可用

发布
约翰·布鲁姆
十月03，2019

我很高兴代表Spring，Apache Geode，Pivotal GemFire和Pivotal Cloud Cache社区，宣布发布适用于Apache Geode ，Pivotal GemFire和Pivotal Cloud Cache（PCC）的Spring Boot。 1.1.2.RELEASE以及1.2.0.RC1 。

SBDG 1.1.2.RELEASE建立在Spring Boot之上2.1.9.RELEASE在Maven Central中可用。

SBDG 1.2.0.RC1建立在Spring Boot之上2.2.0.RC1可以在Spring Milestone信息库中找到。

此外，SBDG 1.2.0.RC1可以将这些位包含在通过Spring Initializer通过https://start.spring.io生成的项目中。只需在“ 搜索要添加的依赖项 ”输入字段中键入“ Geode ”，您就会看到“ Spring for Apache Geode ”依赖项作为您可以添加的选项出现。这包括spring-geode-starter使用初始化程序生成项目时，项目中的Maven或Gradle构建文件中的依赖关系。看看这个！

什么是新的

SBDG 1.1.2.RELEASE和1.2.0.RC1修复了自动配置支持中的关键问题，该问题可自动配置GemfireTemplate在Spring上下文中声明为Bean的每个Region实例。请参阅第55期 。

此外，SBDG 1.1.2.RELEASE包括：

  • 升级到Spring Framework 2.1.10。发布

  • 升级到Spring Boot 2.1.9。发布

  • 升级到Spring Data Lovelace-SR11 / 2.1.11。发布

  • 升级到Spring Session Bean-SR8

  • 升级到适用于Apache Geode和Pivotal GemFire 2.1.6的Spring Session。发布。

见SBDG 1.1.2.RELEASE更详细的更改日志 。

SBDG 1.2.0.RC1包括：

  • 有关SBDG的“ 入门 ”的带有指导和示例代码的新示例。该指南将引导您使用* Spring Initializer *创建SBDG应用程序，首先在本地使用Apache Geode，然后在客户端/服务器拓扑中运行，最后使用Pivotal Cloud Cache将应用程序部署到Pivotal Platform环境，而无需进行任何代码或配置更改。

  • 包括一个新的冒烟测试测试套件，以确保SBDG的常用用例和功能正常工作。

  • 升级到Spring Framework 5.2.0。发布

  • 升级到Spring Boot 2.2.0。RC1

  • 升级到Spring Data Moore-RELEASE / 2.2.0。发布

  • 升级到Spring Session Corn-RC1

  • 升级到适用于Apache Geode和Pivotal GemFire 2.2.0的Spring Session。RC1

见SBDG 1.2.0.RC1更详细的更改日志 。

下一步是什么

所有道路都导致SBDG 1.2 GA发布（即1.2.0.RELEASE ），即在10月中旬发布Spring Boot 2.2 GA之后。因此，如果您想在最终GA之前在SBDG 1.2中看到某些内容，那么之间的反馈很重要。

之后，SBDG 1.3将处于开发中，我们计划进行一些项目重组，以更紧密地与Spring Boot的GitHub项目组织保持一致，以使项目查找更直观。

反馈

我们希望收到你的来信！您的反馈意见对于该项目的发展至关重要，绝对会受到欢迎和赞赏。因此，尝试一下新方法，让我们知道您的想法。
