适用于Apache Geode和Pivotal GemFire 1.1.0的Spring Boot。RC1发布！

发布
约翰·布鲁姆
2019年8月2日

我很高兴宣布发布适用于Apache Geode和Pivotal GemFire / Pivotal Cloud Cache （SBDG）的Spring Boot。 1.1.0.RC1 。

这是最终GA之前的2个发布候选版本中的第一个 ，暂定于2019年8月14日（星期三）发布 。请注意SBDG 1.1.0。然后发布。

您获得新的SBDG 1.1.0.RC1来自Spring的libs-milestone 在此处回购，或在您的Maven POM中声明以下依赖项声明：

Maven依赖

  org.springframework.geode
  spring-geode-starter
  1.1.0.RC1

什么是新的

1.1.x发行版的主题再次聚焦于开发人员的生产力，它利用Spring Boot的强大功能结合有效使用Apache Geode的能力。

在企业应用程序体系结构中使用现代的云原生数据访问技术 （和模式）的最简单，最快的入门方法之一就是利用Apache Geode作为Spring的Cache Abstraction中的缓存提供程序 。这是在企业应用程序体系结构中引入诸如Apache Geode之类的现代，可扩展数据堆栈的最小侵入方法。

确实，缓存是一种非常强大的设计模式，可让您更有效地利用系统资源并延长旧数据管理系统（例如RDBMS）的寿命。

在里面1.1.0.RC1发行版中，我们包括一个新的 Sample ，它基于后备缓存模式并向您介绍了内联缓存 。

小费
 当然，如果您尚未这样做，则应该先阅读《 后备缓存 指南》 。它还包括示例代码 。

下一步是什么

由于我们专注于缓存中的主题1.1.x发布系列中，我们将谈论一个更高速缓存的设计模式...... 近缓存在1.1.0完善了对缓存我们的讨论。RC2版本，接下来发布。

最后，在1.1 GA中，我们将讨论缓存的常见用例，HTTP会话状态缓存以及Spring Session如何使其简短工作。

反馈

与往常一样，我们希望听到您的意见。您的反馈对于帮助我们塑造产品以帮助您实现期望的结果非常重要。

同样不要忘记SpringOne Platform 2019指日可待。不要错过！
