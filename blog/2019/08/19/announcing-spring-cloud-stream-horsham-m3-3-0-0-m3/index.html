宣布春季云流霍舍姆。M3（3.0.0。M3）

工程
奥列格·朱拉库斯基（Oleg Zhurakousky）
2019年8月19日

我们很高兴地宣布Spring Cloud Stream Horsham的第三个里程碑。M3（3.0.0。M3）。

注意：Spring Cloud Stream 3.0.0。M1 / M2主要是为了建立与Spring Boot 2.2.x的兼容性。因此不作宣布。

春季云流霍舍姆。M3（3.0.0。M3）模块可在Spring Milestone存储库中使用。

快速亮点：

  • Spring Boot 2.2.x
  • Spring Cloud Function 3.0.0。M2

此里程碑的显着功能和增强功能：

强调函数式编程模型

虽然我们还没有弃用@StreamListener和/或@EnableBinding ，我们正在朝着更简单，更自然的自动配置驱动编程模型发展。

@SpringBootApplication
public class MyApplicationConfiguration {

    @Bean
    public Function<String, String> uppercase() {
        return value -> value.toUpperCase();
    }
}

上面是一个功能完整的spring-cloud-stream应用程序。注意缺少@StreamListener和@EnableBinding 。这是一个事实Function在spring-cloud-stream的上下文中进行引导（因为它在类路径中）有效地告诉我们所有信息， @StreamListener和@EnableBinding之前曾告诉过我们（例如，功能映射到Processor默认目标名称为“输入”和“输出”）。

注意：spring-cloud-function的改进之一是我们不再将非反应性功能自动转换为反应性功能。这意味着您的命令式函数将与任何其他消息处理程序（例如，StreamListener）连接，而反应式函数的连接方式与spring-cloud-stream-reactive模块上的反应式侦听器相同（在另一个博客中有更多介绍），为您提供更大的灵活性。

在接下来的几周中，我们将更新功能支持文档，并提供更多详细信息。

Kafka Streams中的功能支持

Kafka Streams绑定器现在支持基于一流函数的编程模型，您现在可以使用该模型基于java.util.function支持编写您的Kafka Streams应用程序。这进一步减少了应用程序需要编写的样板代码，并使开发人员可以专注于手头的业务逻辑。有关更多详细信息，请访问“ 功能样式”部分以获取更多详细信息。我们计划对所有新功能进行另外的专门撰写。
