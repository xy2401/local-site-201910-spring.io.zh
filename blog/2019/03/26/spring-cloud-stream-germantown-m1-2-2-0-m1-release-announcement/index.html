功能编程的进一步增强

我们添加了其他功能性编程增强和简化功能，不再需要提供@EnableBindingannotation，在某些情况下，您可能不需要spring.cloud.stream.definition`属性。考虑以下：

@SpringBootApplication
public class SimpleFunctionRabbitDemoApplication {

    public static void main(String[] args) throws Exception {
        SpringApplication.run(SimpleFunctionRabbitDemoApplication.class);
    }

    @Bean
    public Function echo() {
        return v -> v;
    }
}

在没有任何其他属性和/或注释的情况下，以上内容是一个功能全面的Spring Cloud Stream应用程序，它基于能够从上述代码确定的框架绑定为Processor 。随后，以下将绑定为Sink ：

@SpringBootApplication
public class SimpleFunctionRabbitDemoApplication {

    public static void main(String[] args) throws Exception {
        SpringApplication.run(SimpleFunctionRabbitDemoApplication.class);
    }

    @Bean
    public Consumer log() {
        return System.out::println;
    }
}

请注意，不再存在@EnableBindingannotation，也不再@EnableBinding spring.cloud.stream.function.definition属性，因为在应用程序上下文中只有一个功能Bean的情况下，框架能够确定意图。如果存在多个，并且/或者如果您希望由多个函数组成函数 ，则仍然可以提供spring.cloud.stream.function.definition属性
