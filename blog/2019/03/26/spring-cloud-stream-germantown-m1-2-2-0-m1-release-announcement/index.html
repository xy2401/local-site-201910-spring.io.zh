功能编程的进一步增强

我们添加了其他功能性编程增强功能和简化功能，不再需要您提供@EnableBinding注解，在某些情况下，您可能不需要spring.cloud.stream.definition`属性。考虑以下：

@SpringBootApplication
public class SimpleFunctionRabbitDemoApplication {

    public static void main(String[] args) throws Exception {
        SpringApplication.run(SimpleFunctionRabbitDemoApplication.class);
    }

    @Bean
    public Function<String, String> echo() {
        return v -> v;
    }
}

在没有任何其他属性和/或注释的情况下，以上内容是功能齐全的Spring Cloud Stream应用程序，绑定为Processor基于能够从上述代码确定的框架。随后，以下内容将被绑定为Sink ：

@SpringBootApplication
public class SimpleFunctionRabbitDemoApplication {

    public static void main(String[] args) throws Exception {
        SpringApplication.run(SimpleFunctionRabbitDemoApplication.class);
    }

    @Bean
    public Consumer<String> log() {
        return System.out::println;
    }
}

注意不再有@EnableBinding注释也不spring.cloud.stream.function.definition假定应用程序上下文中只有一个可用的bean，那么该属性就可以确定框架的意图。如果有多个功能 ，和/或如果您想由多个功能组成 ，您仍然可以提供spring.cloud.stream.function.definition属性

功能性卡夫卡流

遵循我们在Germantown所倡导的主题精神，Kafka Streams活页夹现在还支持使用功能性编程模型编写应用程序。而不是使用StreamListener和SendTo为了组成处理器，您只需定义类型的bean java.util.Function|Consumer活页夹将委派给适当的绑定。该功能仍在 Kafka Streams活页夹中不断发展 ，并在我们逐步突破Germantown的里程碑时，观看此空间以获取更多改进，详细信息和示例应用程序。

每个绑定重试模板

能够按绑定配置RetryTemplate。

大量增强功能和错误修复 。

注意：
如果应用程序是从Spring Initializr创建的，则它们需要在spring-cloud BOM声明之前在maven依赖管理中添加此BOM表片段，否则最终将获得最新快照：

<dependency>
           <groupId>org.springframework.cloud</groupId>
           <artifactId>spring-cloud-stream-dependencies</artifactId>
           <version>Germantown.M1</version>
           <type>pom</type>
           <scope>import</scope>
</dependency>
