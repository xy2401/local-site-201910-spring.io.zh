Spring Initializr：新用户界面

工程
达米安·维特拉克（Damien Vitrac）
三月05，2019

谨代表团队和所有贡献者，宣布Spring Initializr已更新，现在可以在start.spring.io上使用 。

Spring Initializr的简要历史

该项目以最小的HTML形式开始，让您生成带有一些选项的Spring Boot应用程序。它在Spring社区中广受欢迎，因此该团队通过命令行或您最喜欢的IDE添加了新的交互方式。

一段时间后，我们决定进行UI的第一版修订。我们简化了表单，将关键部分放在前面和中间，而默认情况下将其他部分的内容推入了隐藏状态。给定依赖性数量，我们创建了一个搜索框以查找可用的启动器。

现在，该项目是一项受欢迎的服务，甚至是一个库，您都可以根据自己的需要使用和扩展该库。如今，Web UI甚至不是最受欢迎的客户端，但是我们仍然致力于工作和改进服务。

新Web设计背后的原理

让我们看一下更新的设计。

Spring Initializr团队曾多次尝试添加新功能和选项，或解决新问题。我们是否应该生成复杂的构建以解决项目的局限性？我们是否应该在生产中如何构建，分发或运行Spring Boot应用程序方面添加更多意见？我们应该在成熟的Spring Boot入门市场中将其转变吗？

有了这个新修订版，设计通常会（非常需要）刷新，因此我们选择不再显示可用依赖项的完整列表。可以将列表视为解决不良项目元数据的一种方法（ 我们应该 改善它们！ ）或spring.io网站上缺乏发现机制（ 我们应该解决该问题！ ）。

尽管我们认识到它的当前局限性，但我们认为“快速引导刚刚运行的Spring Boot应用程序”仍然是我们服务的核心承诺，开发人员应在创建项目时知道他们要构建哪种类型的应用程序。
不过，我们认为Spring Boot团队可以在帮助开发人员扩展和更新其应用程序方面做更多的工作，因此我们正在努力扩展Spring Initializr的功能（请参阅我们的
前一篇博文 ）。

这个新修订版是朝start.spring.io进行更多更改的新步骤，请继续关注！
