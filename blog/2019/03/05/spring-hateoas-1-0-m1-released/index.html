春季HATEOAS 1.0 M1发布

发布
格雷格·特恩奎斯特
三月05，2019

尊敬的Spring社区，我们很自豪地宣布Spring HATEOAS 1.0的第一个里程碑。春季HATEOAS已经开发了将近7年。从最初对Spring MVC的微小扩展开始，后来便成为Spring Data REST的基础，并已成为Spring MVC应用程序中基于超媒体的API的基本构建块。

我们收到了来自社区的许多高级功能方面的意见，最终决定是时候将这些功能集成到库中了。另外，我们在野外使用该库也获得了大量经验，并认为我们应该抓住机会在1.0版本中反映这些经验。

总结如下：

  • 封装设计和领域模型语言的常规检查。

  • 升级到Java 8和Spring Framework 5.1作为基准。

  • 通过可负担性API（HAL-FORMS，Collection + JSON，UBER）提供高级超媒体支持。

  • 超媒体支持SPI来插入自定义媒体类型实现。

  • Spring WebFlux的LinkBuilder实现。

  • 彻底修改了参考文档 。

让我们详细了解其中一些功能。

包装和类型的大修

在发布1.0之前，我们借此机会重新评估了程序包的结构以及我们在域类型中公开的术语。与我们的0.x版本相比，这导致了许多重大更改。

最根本的变化是Spring HATEOAS不会创建资源。这就是Spring MVC / Spring WebFlux所做的。我们创建超媒体的供应商中立表示 。因此，我们重命名了这些核心类型：

  • ResourceSupport现在是RepresentationModel

  • Resource现在是EntityModel

  • Resources现在是CollectionModel

  • PagedResources现在是PagedModel

副作用是， ResourceAssembler现在是RepresentationModelAssembler ，其方法现在是toModel(…​)和toCollection(…​) 。整个代码库中都有类似的更改，反映了这一更改。

许多API都围绕List的概念List包括RepresentationModel.getLinks() 。现在，我们返回Java Links ，而不是Java列表，该Links已得到显着增强，以使合并，提取和合并链接变得更加容易。核心抽象LinkBuilder ， EntityLinks ， RelProvider和LinkDiscoverer已分别分组为server和client程序包。

声音压倒性吗？我们已经覆盖了您。我们编写了一个脚本 ， 该脚本将尽最大努力将您的大多数代码迁移到新类型并导入语句。它可能无法涵盖所有内容，但应在很大程度上简化迁移过程。

Spring WebFlux支持

此版本中最关键的功能之一是对Spring WebFlux和Reactive编程的支持。这包括：

  • 反应性地建立联系。

  • 向WebFlux端点提供超媒体服务。

  • 支持WebFlux的WebClient消费超媒体。

Spring HATEOAS随WebFluxLinkBuilder一起WebFluxLinkBuilder因此您可以反应性地构建链接。它将自动选择服务器托管的基本URI，并将其与端点的路径合并。

范例1。主动创建链接和报价
import static org.springframework.hateoas.server.reactive.WebFluxLinkBuilder.*;

@GetMapping("/employees")
public Mono>> all() {

  var controller = methodOn(WebFluxEmployeeController.class);

  return Flux.fromIterable(EMPLOYEES.keySet())
    .flatMap(id -> findOne(id))
    .collectList()
    .flatMap(resources -> linkTo(controller.all()).withSelfRel() (1)
      .andAffordance(controller.newEmployee(null)) (2)
      .andAffordance(controller.search(null, null))
      .toMono() (3)
      .map(selfLink -> new CollectionModel<>(resources, selfLink)));
}

  1. 链接到具有Reactor类型的WebFlux端点。

  2. 以域友好的方式添加能力（请参阅下面的更多内容）。

  3. 交出Mono以便您可以执行其他任何Reactor操作。

假设此控制器托管在http://example.com上 ，则希望提供包含指向http://example.com/employees的链接的超媒体文档。

除了以反应方式构建链接之外，当您返回基于RepresentationModel的类型时，无论它是否包装在Reactor类型中，WebFlux端点现在都将呈现超媒体。

新媒体类型

说到HAL-FORM，我们添加了几种新的媒体类型：

  • 哈尔表格

  • 集合+ JSON

  • 优步

但是为什么要停在那里？当我们计划支持其他媒体类型时，它不应阻止您创建自己的媒体。我们引入了一个新的SPI，使您可以编写自己的自定义媒体类型并在Spring HATEOAS中进行注册，以便可以在Spring MVC， RestTemplate bean，Spring WebFlux和WebClient bean中使用它。

Spring Data REST

没有提到Spring Data REST，就无法讨论Spring HATEOAS。

Spring HATEOAS带来的所有这些更改都已在Spring Data REST的最新快照版本中进行了吸收和调整。总体计划是针对Spring Data-发布Moore火车以升级到Spring HATEOAS 1.0。这意味着诸如优惠和新媒体类型之类的内容将成为Spring Data REST的一部分。

Spring Framework 5.1支持

Spring HATEOAS 1.0现在基于Spring Framework 5.1。这意味着将诸如转发标头处理之类的事情委托给Spring Framework进行管理。如果您是应用程序前面的代理服务器正在发送正确的标头，并且您的Spring HATEOAS支持的API突然不重写URI，则需要配置标头处理。

如果您使用的是Spring Boot，这就是您所需要的：

server.use-forward-headers=true

如果您不使用Spring Boot，那么您必须配置以下内容：

@Bean
public FilterRegistrationBean forwardedHeaderFilter() {

  var filter = new FilterRegistrationBean<>();
  filter.setFilter(new ForwardedFilter());

  return filter;
}

有关更多详细信息，请查看Spring Framework的Forwarded头过滤器 。
