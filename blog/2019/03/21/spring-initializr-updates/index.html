Spring Initializr更新

工程
达米安·维特拉克（Damien Vitrac）
2019年3月21日

谨代表团队和所有贡献者，宣布Spring Initializr已更新，现在可以在start.spring.io上使用 。

用户界面更改-听取您的反馈

在上次修订之后，我们收到的主要反馈是有关缺少的依赖项列表。一些社区成员似乎是出于信息目的而查看此列表（不一定使用网站来生成新应用程序），而其他社区成员只是想看看可用的东西。

尽管start.spring.io似乎是一个非常简单的服务，它以简单的HTML表单生成应用程序，但它还有很多其他功能-这一轮反馈告诉我们，我们需要加倍努力进行产品发现和以用户为中心的设计。

我们将依赖项列表重新带回页面，但是这将有助于我们更好地了解网站的使用方式以及如何对其进行改进。我们希望社区能够在该列表中找到先前版本中缺少的内容。

当然，我们修复了一些HTML / CSS问题，并改进了版本不兼容消息，以帮助您选择适当的Spring Boot版本。

建筑

该应用程序仍然由两部分组成：

  • Initializr库 ，其中包含项目生成逻辑和Web API
  • start.spring.io服务实例，其实例的详细信息在生产环境中运行

由于Web UI更改将越来越多地涉及到，因此我们决定将所有特定于Web UI的代码移至start.spring.io项目。希望这将澄清一些事情。如果您对运行与start.spring.io非常相似的服务感兴趣，则现在应该派生start.spring.io项目。如果您想编写自己的Web UI或进行核心更改，那么使用Initializr库是一个很好的起点。

下一步是什么？

在接下来的几周中，我们将进行用户访谈，以更好地了解如何改善网站体验，并更深入地了解用户与服务进行交互的“原因” 。

由于我们无法采访所有用户（非常感谢大家！），因此我们邀请您通过可选的匿名调查留下您的意见和想法。这将使我们能够更全面地了解人们如何使用该服务以及社区的期望和期望。

感谢您在我们继续进行此过程中的反馈和支持。
