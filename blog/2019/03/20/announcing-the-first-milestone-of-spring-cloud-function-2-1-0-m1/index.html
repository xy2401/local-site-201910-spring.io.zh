宣布Spring Cloud Function的第一个里程碑-2.1.0。M1

工程
奥列格·朱拉库斯基（Oleg Zhurakousky）
三月20，2019

我们很高兴地宣布Spring Cloud Function的第一个里程碑-2.1.0。M1。

Spring Cloud Function 2.1.0的各个模块。M1可在Spring Milestone存储库中使用。
此版本包含以下内容：

  • 一些已经往常的bean工厂功能的目录是可利用的功能现在在功能表 （见的一部分戴夫Syer的博客的功能形式的更多详细信息）：
    • 多端点支持 -能够以与传统上下文配置相同的方式映射多个HTTP端点。
    • 功能组合 -组合功能的能力（例如， foo|bar要么foo,bar ）。
  • 在消息功能由非消息组成的情况下，可以通过消息标头传达和保留元信息（例如， Function>, Message>由...组成Function ）。
  • 支持隐式函数组合，其中catalog.lookup("") ， catalog.lookup("|") ， catalog.lookup("foo|")要么catalog.lookup("|bar")是所有有效的查询，可为容器提供足够的信息来确定您的意图。例如， 空字符串仅表示目录中只有一个功能，不应提供任何显式名称。
  • 从远程HTTP端点创建Supplier，可以在其中定义接收器URL （目标）和源URL （源），并将该应用程序用作来自/到HTTP事件的管道。
  • AWS 自定义运行时功能。
  • 其他小的改进和错误修复 。

此外，该项目的主页和文档都进行了重大更新。

Spring Cloud Function 2.1.x依赖于spring-boot2.1.x。
