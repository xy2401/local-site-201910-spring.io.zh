曾经有过更好的时间成为Java开发人员吗？

工程
本威尔科克
三月07，2019

当然，有没有比现在更好的时间成为Java开发人员了？

如今，有很多生产力工具在五年前就令人mind目结舌。

以Spring Boot为例。在Spring网站上阅读此书的许多人可能熟悉Spring Boot。但是，让我们花一点时间来承认它的出色。

几年前，如果您打算在应用程序中使用Spring框架，那么一定要在一天之内完成一些配置工作，然后才能确定。但这不是很好的友好配置，例如，（啊，实际上，对不起，我想不出“友好配置”的示例），它是讨厌的XML配置，附带XSD，eww！

后来，Spring确实提供了更多对开发人员友好的配置选项（例如基于Java的配置），但是当Spring Boot在2013年面世时成功地做到了这一点，这使Spring Framework开发人员免除了许多责任。这在很大程度上要归功于其创新的“自动配置”功能。

使用Spring Boot的“启动器”，您的Spring应用程序可以简单地进行自我配置。您无需再具有平台螺母和螺栓的百科知识即可上手。如果愿意，您仍然可以控制，但这不再是必需的。这极大地拉平了Spring的学习曲线，并极大地解放了Spring开发人员。可以更好地利用阅读文档，配置内容，排除故障所花费的所有这些时间。绝对辉煌！

然后是Spring Boot的高级应用程序包装。

Java的“写一次就可以随处运行”的承诺对于企业Java开发人员来说是一个残酷的笑话。您将花费数小时来捆绑应用程序的WAR文件，弄乱了试图使您的部署描述符正常工作，后来又发现它“仅在您的计算机上运行”。rr！

Spring Boot的可执行JAR打包修复了所有这些废话。通过将Web容器与二进制文件捆绑在一起，您的应用程序可以在兼容JVM的任何地方运行。总的天才。难怪Spring Boot成为世界上最受欢迎的Java Web框架 。

但是，要做到这一点，您需要可构建的Java项目。

坦率地说，开始任何新的Java项目都很麻烦。Maven和Gradle都是出色的构建工具。没有他们，我们无法生存。但是，IDE对它们的支持是零散的，谁想花时间编辑POM并弄清楚Gradle DSL的最新进展？不是我。我太急了生命太短暂了。

因此感谢Spring Initializr 。使用Spring Initializr，您可以立即使用您选择的语言和构建工具下载现成的Spring Boot项目。您甚至不需要浏览器！您可以直接在命令行中使用Spring Initializr，或者curl或Spring CLI 。

严重的是，如果您以前从未尝试过，请立即尝试。这真的很简单：

您只需要一个命令行即可curl和Java。使用第一个简单命令，您可以向Spring Initializr索取一个Zip归档文件，然后可以解压缩该文件，然后就可以使用了。只需将解压缩的项目加载到您喜欢的IDE中，就可以立即开始对其进行编码。

curl https://start.spring.io/starter.zip -d dependencies=web,actuator -d baseDir=demo -o demo.zip
unzip demo.zip

我在这里使用了假定的Linux，但是您可以curl也适用于Windows和Mac。

现在已经下载并解压缩了，让我们运行我们的应用程序，并同时尝试Spring Boot出色的JAR打包。只需键入以下内容即可使用Java VM启动它…

cd demo
./mvnw package # Use mvnw.cmd on Windows
java -jar target/demo-0.0.1-SNAPSHOT.jar

如果您没有Java，请访问https://adoptopenjdk.net/以获取安装程序或安装说明。

而已。Spring Boot将为您启动该应用程序。现在打开您喜欢的浏览器并转到http://localhost:8080/actuator/health ，您将看到一条JSON消息，确认该应用程序处于运行状态并且运行良好。

{"status":"UP"}

与过去相比，这一切都非常简单。

实际上，它是如此简单，我敢打赌一个孩子可以做到。如果您有其中的一个方便，为什么不让他们坐下来立即尝试呢？有什么比使用Spring Boot和Spring Initializr更好的方式开始某人的编码之旅？为什么不让我们知道您的评论方式，或将您的屏幕截图或照片通过井号标签推到@springcentral #SpringFTW 。
