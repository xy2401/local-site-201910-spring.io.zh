反应堆Dy M1现已上市

发布
斯蒂芬·马尔蒂尼（Stephane Maldini）
五月15，2019

第四代反应堆即将到来。我们谨代表团队感谢我们全体社区的巨大反馈。在过去的一年中，我们的反应式阵容显着增长，包括R2DBC和BlockHound 。我们在Java生态系统中的采用看起来是惊人的，并且我们正在与包括Microsoft和Google在内的主要公司合作。我们已经增加了一倍以上我们常规的Gitter观众一些真棒-你猜它- 反应性的讨论，每天发生的事情。最后， 谢尔盖·伊戈罗夫 （ Sergei Egorov）已加入核心团队，我们没有停止扩张的计划！

Dysprosium-M1可在我们的里程碑资料库中找到 。它为工作中的更多变更铺平了道路。值得注意的是，在里程碑阶段结束时，已弃用了2个功能，并评估了它们的删除能力：

  • EventLoopProcessor => TopicProcessor ， WorkQueueProcessor和WaitStrategy ：处理器已移至“ 额外反应堆”以供可选使用。他们需要可用性Unsafe ，鉴于现代JVM的限制，使其变得相当小众。不仅可以在其他地方找到它们，还可以用等价的结构代替它们EmitterProcessor与publishOn和ParallelFlux与runOn 。

  • Kotlin扩展程序被分配一个专用的空间，供进一步实验。随着最新的发展coroutines和inlining ，我们很高兴评估如何利用Kotlin提供更流畅的开发人员体验。我们为此实验设定的目标之一是提供专用的API，使编写具有Kotlin简洁性的自定义生产者和操作员成为可能。

即使进行了这些更改，我们也不会为我们的社区带来重大麻烦，但是我们希望得到您的意见。

更改日志和发行说明：
- 反应堆核心3.3.0。M1
- 反应堆净值0.9.0。M1
- 反应堆插件3.3.0。M1
-BlockHound 1.0.0。M3

运行时性能

我们正在努力改善用户的整体体验，并全面提高我们的运行时效率：
-- reactor-netty 0.9.0.M1/0.8.8对于请求/答复（HTTP）方案，吞吐量提高了40％，对于流传输方案，吞吐量提高了至少20％。更重要的是，已经加强了内存管理，以控制某些报告的内存问题和所需的数量flush操作已大大减少（减少了3倍），这对于CPU使用来说是个好消息。
-对于reactor-core 3.3.0 ，我们正在研究使用基于代理的方法将调试拦截器逐步移出关键路径。

铋锭

期待未来Dysprosium-RELEASE ，我们的反应堆芯3.1.x和反应堆网形0.7.x线将不会再收到补丁。我们鼓励用户更新到Californium发布与Spring Boot 2.1.x和Spring Framework 5.1.x匹配的火车。

如果您已经在使用Spring Boot 2.2.x和Spring Framework 5.2.x ，则需要使用Dysprosium包括反应堆核心3.3.x和反应堆净值0.9.x。
