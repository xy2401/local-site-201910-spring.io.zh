Spring Cloud Data Flow 2.1 GA发布

发布
马克·波拉克
五月21，2019

Spring Cloud Data Flow团队很高兴地宣布Data Flow 2.1的发布。

我们拥有一个全新的网站，其中包含许多新颖的内容，您可以在此处找到我们的入门指南 ，以在Cloud Foundry和Kubernetes以及您的本地计算机上使用。

以下是重点内容：

新的专用数据流网站

数据流团队感到自豪的是，它在包括StackOverflow，Gitter，GitHub，Twitter在内的各种论坛中与社区进行公开交流，甚至在电子邮件和Zoom呼叫中也是如此。

但是，我们意识到，如果我们专注于改进在线文档，我们可以提供更好的体验来回答常见问题，并为使用Data Flow提供更轻松的入门。参考指南不是实现该目标的理想格式，因此我们着手创建一个新网站https://dataflow.spring.io ，该网站可作为学习与Data Flow相关的所有事物的中心。

舵图更新

随着社区的不断关注，我们现在增加了在使用Data Flow的Helm Chart时在RabbitMQ和Apache Kafka之间切换的支持。Kubernetes的安装说明是开始使用Chart的最佳位置。

任务的并行启动节流

利用社区中有关并发任务启动算法的最新反馈，我们重新设计了该方法，以确定当前是否正在运行任务，如果正在运行，则有多少个并发处于运行状态。现在，可以通过查询底层平台（例如，Local，Cloud Foundry和Kubernetes）中Tasks的运行状态来更实际地了解Tasks实际发生的情况。您可以从参考指南中阅读有关新设计和拐角案例的所有信息。

部署程序属性作为下拉列表

越来越多的支持案例表明，开发人员在部署流和任务时很难发现然后提供特定于平台的部署程序属性。我们通过添加新的下拉菜单来解决此问题，该下拉菜单具有根据配置为运行Data Flow的平台自动填充的部署程序属性。当配置了1个或多个目标平台时，情况也是如此，下拉菜单更新以反映目标平台。我们已经为Stream部署实现了此功能，并且在下一版本中将包括对Tasks的支持。

切换到ZenHub

从产品积压的角度来看，我们已经交换和迁移所有GitHub的积压从闲聊到ZenHub（见的原因 ）。如果您有兴趣关注产品发布进度，可以为网络浏览器安装ZenHub插件，并导航至Data Flow GitHub存储库中的ZenHub选项卡。

保持联系

与往常一样，我们欢迎您提供反馈和意见，因此请通过StackOverflow或GitHub或通过Gitter与我们联系。
