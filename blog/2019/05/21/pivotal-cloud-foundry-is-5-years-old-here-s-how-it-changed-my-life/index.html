Pivotal Cloud Foundry成立5岁，这是它如何改变了我的生活...

工程
本威尔科克
五月21，2019

乔什·朗 （ Josh Long）经常说：“生产是Internet上最好的地方。”但是在我以前工作过的地方，开发人员需要与运营，网络和安全部门进行协商，然后他们的代码才能在承诺的范围内到达任何地方。

可以理解的是，这些学科中的每一个似乎都有相同的隐藏议程：变化是有害的。经验告诉我们，变更既困难又容易出错，因此，作为一家公司，我们会规避风险，并谨慎部署。发行很少，发行量很大。有很多可能出错的地方。

尽管如此，开发人员还是每六个月左右将其代码更改捆绑到一个发行版中，编写发行说明，创建一个回滚计划，并等到星期六的午夜才进行部署。部署将手动完成，而不是由开发人员完成，而是由操作人员进行。开发人员严格遵守这种特定的发布方式（毕竟，就像我的朋友Coté所说的那样，我们是“编写所有bug的人”）。

因此，如果制作确实是“ Internet上最好的地方”，那么我只能在周日的凌晨3:00进行代理访问，而且前提是我首先迷住了“更改授权委员会”的位置。

整个过程是毫无回报和令人沮丧的。太难了，我几乎因此而辞职了。前进的道路上有太多障碍。对于任何人来说，交付代码都不是一种有益的体验。您当然无法通过令人兴奋的新想法或新技术进行“实验”。部署太大。反馈循环太长，风险太大。

但是随后发生了令人惊讶的事情……

2013年11月，Pivotal Cloud Foundry出现并改变了一切。通过将数百种DevOps最佳实践整合到易于使用的程序包中。Pivotal Web服务（PWS）可以通过一些简单的命令行说明轻松地打包，部署和运行我们的代码。利用我们的软件，我们可以在几分钟之内，从我们的桌面到生产，只要我们喜欢。

开发人员喜欢它。运营商喜欢它。安全和网络部门很喜欢它。最重要的是，我们的客户喜欢它！他们实际上不能得到足够的东西，但这是另一个故事。看看它有多快 ！！！您可以在2分钟内启动并运行应用程序：

在部署部署之前获得许可的需求已荡然无存。变更授权委员会为确保我们安全而需要的所有内容均已植入平台中。开发人员可以直接与企业合作，每天增加一些价值，并使用自动管道将其推送到云中。因为我们现在得到了即时反馈，所以创新和实验变得容易，安全和受到鼓励。大胜！

支持服务也是如此。您可以将其命名为数据库，消息传递，监视，并全部创建并附加一些简单的命令行说明。Spring Boot甚至自动为您接线。无需等待您的“新数据库请求”通知单就可以将其拖到数据库管理员待办事项列表的顶部。开发人员可以在需要时委托自己的服务。

在短短的5年中，Pivotal Cloud Foundry彻底改变了我的生活。它使我摆脱了发布培训的麻烦，并允许像我这样的开发人员比以往任何时候都更快，更安全，更频繁地将代码发布到用户手中。

因此，感谢Pivotal Cloud Foundry的帮助，如果生产真的比迪士尼乐园更好（（ 乔什·朗 ，2018年）），我将获得头等舱机票！
