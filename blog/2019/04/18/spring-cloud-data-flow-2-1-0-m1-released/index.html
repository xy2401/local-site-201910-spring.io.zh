Spring Cloud Data Flow 2.1.0。M1发布

发布
珍妮·瓦尔基亚拉赫蒂
2019年4月18日

Spring Cloud Data Flow团队很高兴宣布2.1.0 M1里程碑版本的发布。遵循在本地 ， Cloud Foundry和Kubernetes上运行的入门指南。

什么是新的？

SCDF Helm Chart和Apache Kafka

随着社区的持续关注，从v2.0.2开始，我们现在增加了在使用SCDF掌舵表时在RabbitMQ和Apache Kafka之间切换的支持。尽管该图表在Helm信息库中处于PR状态，但是您仍然可以尝试并提供反馈。

Docker Compose和Apache Kafka

为了继续保持与Apache Kafka相同的趋势，以及常规的Apache Kafka活页夹属性，我们现在还增加了对Kafka Streams活页夹属性的支持。同样，我们已改用官方Confluent的Docker映像作为Docker Compose 入门经验的一部分 。

任务的并行启动节流

利用社区中有关并发任务启动算法的最新反馈，我们重新设计了该方法，以确定当前是否正在运行任务，如果正在运行，则有多少个并发处于运行状态。现在，可以通过查询底层平台（例如，Local，Cloud Foundry和Kubernetes）中Tasks的运行状态来更实际地了解Tasks实际发生的情况。您可以从参考指南中阅读有关新设计和拐角案例的所有信息。

部署程序属性作为下拉列表

越来越多的支持案例表明，开发人员在部署流和任务时很难发现然后提供特定于平台的部署程序属性。我们通过添加新的下拉列表来解决此问题，该下拉列表具有根据配置了SCDF的平台自动填充的部署程序属性。当为单个SCDF实例配置了1个或多个平台时，这也适用于方案-下拉菜单将对平台选择做出反应。有了它，不仅可以简化如何派生这些属性及其用途的发现，还可以自动创建SCDF所需的必要前缀，因此在部署流/任务时不必担心。我们将在此版本中针对Streams推出该产品，下一个里程碑还将包括对Tasks的支持。

Spring Cloud Data Flow微站点

SCDF团队感到自豪的是，它在包括StackOverflow，Gitter，GitHub，Twitter在内的各种论坛中与社区进行公开交流，甚至在电子邮件和Zoom呼叫中也是如此。

在使用该模型的多年经验中，我们逐渐了解到仅凭参考指南的形式提供样本并记录所有功能是不够的。在与社区互动的同时，我们倾向于编写更多可行的代码示例，并通过其他示例阐明概念。它发生了太多事情，以至于我们也倾向于经常重复回答相同的问题。

除此之外，在最近的Spring Cloud Stream ， Spring Cloud Task和Spring Cloud Data Flow开发人员调查中，很明显，我们的样本，文档和指南需要一些改进。

我们决心改善这一点！SCDF团队正在研究更加动手的开发人员指南，该指南深入了各种概念，功能，部署选项以及许多其他主题。我们称其为“ Spring Cloud Data Flow Microsite”-一个独立的应用程序，其所有抵押品都可以直接与开发人员联系。2.1.0版本的更大主题只是Microsite-下一个里程碑中的更多主题-敬请期待！

切换到ZenHub

从产品积压的角度来看，我们已经交换和迁移所有GitHub的积压从闲聊到ZenHub（见的原因 ）。如果您有兴趣关注产品发布进度，可以为网络浏览器安装ZenHub插件，并导航到SCDF GitHub存储库中的ZenHub选项卡。

保持联系

与往常一样，我们欢迎您提供反馈和意见，因此请通过StackOverflow或GitHub或通过Gitter与我们联系。
