Spring Cloud Alibaba的预览版本可用：0.9.0、0.2.2和0.1.2

发布
斯宾塞·吉布
2019年4月19日

注意：这是来自我们阿里巴巴朋友的来宾帖子。

嗨！今天，我们非常高兴地宣布，Spring Cloud Alibaba的0.9.0、0.2.2和0.1.2版本现已可用。

发布重点

此GA中包含3个新版本：

  • 版本0.9.0：与Spring Cloud Greenwich兼容
  • 版本0.2.2：与Spring Cloud Finchley兼容
  • 版本0.1.2：与Spring Cloud Edgware兼容

三个版本均添加了四个新模块：

  • Spring Cloud 阿里巴巴前哨祖尔
  • Spring Cloud 阿里巴巴seata
  • spring-cloud-alibaba-dubbo（在版本0.1.2中不可用）
  • Spring Cloud Alibaba 短信

此外，我们在spring-cloud-alibaba-nacos和spring-cloud-alibaba-sentinel中引入了新功能，rocketmq-binder现在已与robotmq-spring集成在一起。

最后，我们修复了先前版本中的一些错误。

注意：Spring Cloud Alibaba项目是由Alibaba维护的社区项目。

Spring Cloud Alibaba Seata

Seata是一个分布式交易组件，由阿里巴巴集团和蚂蚁金服联合开源。Seata能够与当今支持ACID事务的大多数关系数据库集成，从而以有效且非侵入性的方式解决了微服务场景中的分布式事务问题。

在Spring Cloud级别，Seata通过在微服务中传递事务上下文来访问Spring Cloud层：

  • 在Spring MVC环境中支持自动恢复Seata事务上下文
  • 当服务调用者通过RestTemplate或FeignClient调用服务时，支持自动传递Seata事务上下文
  • 支持在速率限制和服务降级方案中将FeingClient与Hystrix或Sentinel一起使用

Spring Cloud Alibaba 达博

Apache Dubbo™（正在孵化）是一种基于Java的高性能开源RPC框架。

Spring Cloud Alibaba与之集成：

  • 基于Dubbo Spring Boot 2.7.1（最低JDK要求：Java 8）
  • 支持所有用于Dubbo服务注册的Spring Cloud注册表实现（包括Alibaba Nacos），并引入了新的Dubbo注册协议：“ ** spring- cloud：// localhost ** ”
  • 扩展了Spring Cloud服务到服务调用的通信协议，这意味着REST不再是唯一的方法，可以选择Dubbo支持的更多通信协议，从而提供更好的性能，例如dubbo，节俭等。为开发人员提供了三种编程模型：
  • 标准Apache Dubbo： @Reference@Service在Java接口上注释
  • Spring Cloud Open Feign： @DubboTransported注释于@FeignClient Java介面
  • Spring Cloud RestTemplate： @DubboTransported注释于@Bean RestTemplate的定义已注释@LoadBalanced
  • 通过使用Dubbo内置的负载平衡器替代Netflix Ribbon，该平衡器提供了一种加权机制，使事情变得更加平稳和稳定。

Spring Cloud Alibaba Nacos

Nacos是一个动态的服务发现，配置管理和服务管理平台，可简化构建云原生应用程序的过程。

Spring Cloud Alibaba Nacos配置

我们更新了Nacos Config如下：

  • 将Nacos Client版本升级到1.0.0
  • 支持spring.cloud.nacos.config.enabled配置以关闭Nacos Config自动注入
  • 更换spring.cloud.nacos.config.enabled与spring.profiles.active

Spring Cloud Alibaba Nacos发现

我们更新了Nacos服务注册，如下所示：

  • 将Nacos Client版本升级到1.0.0。
  • 支持名称空间以隔离数据。
  • 提供NacosWatch引起HeartbeatEvent 。
  • 使@EnableDiscoveryClient可选的注册服务。

Spring Cloud Alibaba 前哨

Sentinel是由阿里巴巴中间件团队开发的一个开源项目，旨在解决流控制问题并确保分布式体系结构中的高可用性。Sentinel以“流量”为切入点，并通过流量控制，断路和系统负载保护帮助您确保服务可靠性。

此版本中的更新包括：
-将Sentinel版本升级到1.5.2
-添加ruleType定点dataSource的属性可修复以下错误：当dataSource首次加载空数据时，规则数据不会动态加载
-修复了@SentinelRestTemplate在某些情况下不起作用
-在中添加JSR 303验证SentinelProperties并添加两个配置项：客户端IP以及是否CommonFilter初始化
-支持自动注入RequestOriginParser
-的优先事项SentinelProtectInterceptor对应于@SentinelRestTemplate成为最高
-- SentinelEndpoint现在显示更多信息，例如应用程序名称，日志目录，日志名称策略，仪表板地址，指标文件大小，客户端IP等
-支持@EnableCircuitBreaker
-优化流量控制和断路处理逻辑RestTemplate和Feign

Spring Cloud Alibaba RocketMQ活页夹

RocketMQ是基于高可用性集群技术的开源分布式消息传递系统。它提供了低延迟和高可用性的消息发布和订阅服务。

此版本中的更新包括：
-与RocketMQ Spring集成并进行重构
-仅通过配置阿里云AK / SK支持与Aliware MQ的连接
-在消息发送和使用错误的情况下优化了处理逻辑
-支持异步发送消息，默认情况下，异步发送消息

Spring Cloud Alibaba 前哨Zuul

Zuul是Netflix的网络网关产品。现在，我们已将其集成到Sentinel中。

此版本中的更新包括：

  • Sentinel现在与Zuul集成在一起并提供SentinelPreFilter ， SentinelPostFilter ， SentinelErrorFilter用于请求拦截
  • 提供的配置前缀spring.cloud.sentinel.zuul.配置前哨过滤器

Spring Cloud阿里云短信

SMS是阿里云提供的消息服务。

Spring Cloud阿里云SMS提供以下功能：
-支持单次和批量短信发送
-支持基于手机号码，日期和消息类型的SMS查询
-通过订阅下行消息来支持对SMS传递状态的查询
-通过订阅上行消息来支持最终用户回复的内容检索

Spring Cloud阿里云SchedulerX

SchedulerX是阿里云提供的作业调度中间件工具。

在此版本中，我们修复了循环引用错误。

如何使用

这些工件可通过使用BOM从Maven Central和Spring Release存储库中获得，如下所示：

<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>org.springframework.cloud</groupId>
            <artifactId>spring-cloud-alibaba-dependencies</artifactId>
            <version>0.9.0.RELEASE</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
