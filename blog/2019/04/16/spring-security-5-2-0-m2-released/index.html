Spring Security 5.2.0。M2发布

发布
乔什·卡明斯
2019年4月16日

我很高兴代表社区宣布Spring Security 5.2.0的发布。M2！此版本包括100多个更新 。您可以在下面找到要点：

OAuth 2.0

gh-6446 -PKCE的客户支持

PKCE不仅适用于本机或基于浏览器的应用程序 ，而且在任何时候我们都希望拥有一个公共客户端。Spring Security 5.2为后端身份验证提供了一种安全的方式，以使其成为公共客户端。

gh-5350 -OpenID Connect RP启动的注销
gh-5465-能够在以下情况下使用对称密钥：JwtDecoder
gh-5397-的能力NimbusReactiveJwtDecoder采取定制处理器
gh-6513和gh-5200-支持资源服务器令牌自省

资源服务器现在支持第二种OAuth 2.0令牌验证策略： 令牌自省 。当资源服务器想要或必须远程验证令牌时，这很方便。

gh-5351-支持资源服务器多租户（仅Servlet）

随着介绍AuthenticationManagerResolver ，对多租户资源服务器的初始支持已经到来。

核心

gh-6494-将关键材料转换为Key实例

Spring Security 5.2简化了将X.509和PKCS＃8密钥材料转换为RSAPublicKey和RSAPrivateKey通过注册实例Converter到ConversionService和PropertyEditor到PropertyEditorRegistry 。您可以在Resource Server静态密钥示例中看到一个示例 。

gh-6774-支持JDK 12
gh-6722-介绍AuthenticationManagerResolver
gh-6546-简介@CurrentSecurityContext用于方法参数

喜欢@AuthenticationPrincipal在此之前， @CurrentSecurityContext与参数解析器一起检索SecurityContext ：

public String hello(@CurrentSecurityContext
        SecurityContext context) {
    return Optional.ofNullable(context.getAuthentication())
            .map(Authentication::getName).orElse("world");
}

// or

public String hello(@CurrentSecurityContext
        (expression="authentication.name") String name) {
    return Optional.ofNullable(name).orElse("world");
}

网页

gh-4187-支持Clear-Site-Data标头
gh-6312-支持HSTS预载
gh-6453-简介CompositeHeaderWriter ，以及其他一些头文件编写的cleanup 。
