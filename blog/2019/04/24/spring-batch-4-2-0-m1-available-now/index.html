Batch Metrics with Micrometer

This release introduces a new feature that lets you monitor your batch jobs by using Micrometer. By default, Spring Batch collects metrics (such as job duration, step duration, item read and write throughput, and others) and registers them in Micrometer’s global metrics registry under the spring.batch prefix. These metrics can be sent to any monitoring system supported by Micrometer.

Spring Batch 4.2.0。M1现已上市

发布
Mahmoud Ben Hassine
2019年4月24日

我很高兴地宣布，Spring Batch 4.2的第一个里程碑已经发布，可以从我们的里程碑存储库中获得 。

此版本的主要功能是使用Micrometer添加了批次指标。

千分尺批处理指标

此版本引入了一项新功能，使您可以使用测微计来监视批处理作业。默认情况下，Spring Batch收集指标（例如作业持续时间，步骤持续时间，项目读写吞吐量等），并将其注册在Micrometer的全局指标注册表中的spring.batch前缀下。这些度量可以发送到Micrometer支持的任何监视系统 。

有关此新功能的更多详细信息，请参考参考文档的“ 监视和指标”部分。

其他更新

除了对测微计的支持之外，此里程碑还包括其他改进：

  • 参考文档已更新为与其他Spring项目相同的样式
  • 依赖关系已更新为Spring项目的最新里程碑：
    Spring Framework 5.2.0.M1 ，Spring Data 2.2.0.M3和Spring Integration 5.2.0.M1
  • 其他一些小的改进和错误修复

有关更改的完整列表，请检查更改日志 。
