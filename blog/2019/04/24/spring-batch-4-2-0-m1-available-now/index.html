Spring Batch 4.2.0。M1现已上市

发布
Mahmoud Ben Hassine
2019年4月24日

我很高兴地宣布，Spring Batch 4.2的第一个里程碑已经发布，可以从我们的里程碑存储库中获得 。

此版本的主要功能是使用Micrometer添加了批次指标。

千分尺批处理指标

此版本引入了一项新功能，使您可以使用测微计来监视批处理作业。默认情况下，Spring Batch收集指标（例如作业持续时间，步骤持续时间，项目读写吞吐量等），并将其注册到Micrometer的全局指标注册表中。 spring.batch字首。这些度量可以发送到Micrometer支持的任何监视系统 。

有关此新功能的更多详细信息，请参考参考文档的“ 监视和指标”部分。

其他更新

除了对测微计的支持之外，此里程碑还包括其他改进：

  • 参考文档已更新为与其他Spring项目相同的样式
  • 依赖关系已更新为Spring项目的最新里程碑：
    春季框架5.2.0.M1 ，春季数据2.2.0.M3和Spring集成5.2.0.M1
  • 其他一些小的改进和错误修复

有关更改的完整列表，请检查更改日志 。

反馈

春季批4.2.0.M1可以通过Spring Boot使用2.2.0.M2 。继续，在start.spring.io上创建您的下一个Spring Batch应用程序！我们还创建了一个新示例，以展示如何使用Micrometer， Prometheus和Grafana监视Spring Batch作业。

我们期待听到您对第一个里程碑的反馈！随时在Twitter上ping @michaelminella或@_benas_或在StackOverflow或Gitter上提问。如果发现任何问题，请在Jira上打开一张票。

下一步是什么？

下一个里程碑将提供对Apache Kafka的支持。我们计划添加一个新的项目读取器和写入器，以从Kafka主题读取数据并将其写入Kafka主题。敬请关注！
