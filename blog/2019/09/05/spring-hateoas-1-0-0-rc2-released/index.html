春季HATEOAS 1.0.0。RC2发布

工程
格雷格·特恩奎斯特
九月05，2019

尊敬的Spring社区，我们很荣幸地宣布Spring HATEOAS 1.0.0的第二个候选版本。自从上一发行版以来，Spring HATEOAS有了两个更大的里程碑和两个RC，取得了长足的进步。

总结如下：

  • 对HAL和HAL-FORMS的国际化支持。

  • 性能提升

  • Spring HATEOS现在基于Spring Framework 5.2和Reactor的Dysprosium版本。这对于支持诸如Spring Data Moore和Spring Boot 2.2之类的下游项目至关重要。

  • 现在支持同时具有超媒体和非超媒体端点的Spring WebFlux应用程序。

  • 改进了大部分参考文档。

  • 创建了Spring HATEOAS Gitter频道 。

  • 修复了各种媒体类型中的问题，以确保它们符合规范定义，并且不会引发意外异常。

让我们详细了解其中一些功能。

国际化

要添加到您的API中最脆弱的功能之一是国际化文本。许多客户端通过解析有效内容来模仿服务器端规则。当服务器希望采用特定于语言环境的文本时，这将为大规模的重大更改做好准备。说服您的客户采用基于链接的方法，这为使用Spring资源包调整链接上显示的文本打开了大门。

HAL为其链接对象定义标题属性。可以通过使用Spring的资源包抽象和名为的资源包来填充这些标题rest-messages这样客户可以直接在其UI中使用它们。该捆绑包将自动设置，并在HAL链接序列化期间使用。

要为HAL链接定义标题，请使用密钥模板_links.$relationName.title如下：

范例1。一个样品rest-messages.properties
_links.cancel.title=Cancel order
_links.payment.title=Proceed to checkout

这将导致以下HAL表示形式：

示例2定义了链接标题的示例HAL文档
{
  "_links" : {
    "cancel" : {
      "href" : "…"
      "title" : "Cancel order"
    },
    "payment" : {
      "href" : "…"
      "title" : "Proceed to checkout"
    }
  }
}

从那里，您可以自由包括rest-messages.properties默认值，然后包括例如rest-messages_de.properties为您的德国客户等

HAL-FORMS既有title和一个prompt属性，供人类食用。看下面rest-messages.properties条目：

范例3。定义HAL-FORMS模板标题和提示
_templates.default.title=Some title (1)
_templates.putEmployee.title=Create employee (2)
Employee._templates.default.title=Create employee  (3)
com.acme.Employee._templates.default.title=Create employee (4)

firstName._prompt=Firstname (5)
Employee.firstName=Firstname (6)
com.acme.Employee.firstName._prompt=Firstname (7)

  1. 使用默认键作为标题的全局定义。

  2. 使用实际可得商品名称作为关键字的标题的全局定义。除非在创建报价时明确定义，否则默认为$httpMethod + $simpleInputTypeName 。

  3. 本地定义的标题将应用于所有命名类型Employee 。

  4. 使用完全限定的类型名称的标题定义。

  5. 所有命名为email的属性都将呈现“ Firstname”，而与声明它们的类型无关。

  6. 类型为Employee的firstName属性将提示“ Firstname”。

  7. com.acme的firstName属性。员工将收到分配“姓氏”的提示。

这样一来，您就可以控制向客户展示他们需要看的内容，并让客户在更改有效负载时不会费力。

性能提升

在社区和我们团队的共同努力下，我们加快了超媒体的渲染速度。这包括缓存关键位，将不必要的对象分配移出热路径，甚至摆脱用于正常控制流的异常。

此外，这个从Java 6开始的项目将继续采用Java 8功能，从而使加速工作变得更加容易。另外，借助通过JMH（Java Microbenchmark线束）进行的负载测试电池，我们已经能够发现许多亟需关注的领域。

Spring Framework 5.2 + Reactor s + Java。下一个

对Spring数据至关重要Moore发布HATSpring已与Spring Framework 5.2和Reactor Dysprosium保持一致。这一切都向Spring Boot 2.2提供 。

同时，我们针对Spring Framework的最新里程碑/ RC以及快照构建每次提交。这样，我们可以快速发现问题并确保它们不会破坏您的利益。

如果您是开拓者并且使用Java的较新版本，那么您应该很高兴知道我们针对LTS版本（AdoptOpenJDK 8和11）和Docker Hub上可用的Java的最新版本（AdoptOpenJDK 12）运行所有提交。这样，如果Java社区即将进行一些重大的API更改，就不会感到惊讶。
