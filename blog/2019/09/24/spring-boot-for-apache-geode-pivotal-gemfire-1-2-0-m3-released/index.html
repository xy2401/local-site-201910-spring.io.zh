适用于Apache Geode和Pivotal GemFire 1.2.0的Spring Boot。M3发布

发布
约翰·布鲁姆
九月24，2019

我很高兴代表Spring和Apache Geode社区宣布发布适用于Apache Geode和Pivotal GemFire （SBDG）的Spring Boot。 1.2.0.M3 。

SBDG 1.2.0.M3继续基于与1.2.0相同的位。M2版本：主要：

  • Spring Framework 5.2.0。RC2

  • Spring Boot 2.2.0。M6

  • Spring Data Moore-RC3

  • 春季会议玉米M4

  • Apache Geode和Pivotal GemFire （STDG）0.0.8的春季测试 。发布

SBDG 1.2.0.M3可以从Spring Milestone存储库以及Spring Initializer获取bit。

什么是新的

现在，我们提供了在无需更改代码或配置的情况下在环境之间切换的支持。

同样，我们对SBDG的主要指令进行了改进，该主要指令是：

  1. 在开源 （ Apache Geode ）和商业 （ Pivotal Cloud Cache （PCC））之间切换。

  2. 在非托管 （独立或外部托管）环境与托管环境（例如Pivotal Platform ）之间切换

  3. 几乎 不需要更改代码或配置 。它只是工作！

注意
 如果需要的话，也可以从托管环境回到非托管环境，从商业环境回到开源。该框架不会锁定您。这是你的选择。

新功能使此功能成为可能@EnableClusterAware批注，可以在您的主菜单上使用@SpringBootApplication类如下：

使用@EnableClusterAware
@SpringBootApplication
@EnableClusterAware
class MySpringBootApacheGeodeClientCacheApplication { ... }

此新注释具有更大的功能。有关更多详细信息，请参阅文档 。

此外，我们还撰写了有关SBDG应用的“ 自动配置 ”以及仍需要什么“ 声明式配置 ” （或显式配置）的新的专门章节。

经常，用户会问SBDG自动配置了什么，我必须自己配置哪些Apache Geode或PCC功能？这些章节回答了这个问题，甚至更多。

此外，一定要检查出部分中的“ 声明配置 ”一章，我们调用“生产力”基于注解。

除了这两个新功能，我们还修复了自动配置自动配置的错误。 GemfireTemplates在某些情况下进入您的应用程序组件不完整和损坏。

感谢Udo Kohlmeyer和Mark Secrist找到并提出了这个问题！

有关更改的完整列表1.2.0.M3 ，请参阅更改日志 。

下一步是什么

总结了SBDG的功能集1.2 。在Spring Boot 2.2 RC1之后，下周将（暂定）发布SBDG 1.2 RC1，这将为SBDG重新设定基础1.2.0.RC1在Spring Boot上 2.2.0.RC1位。

在此版本中，我们还将向后退一步，并从头开始。意思是，我们将专注于`` 入门 ''体验...使用SBDG从头开始使用Spring Boot创建Apache Geode或Pivotal Cloud Cache（PCC）应用程序。

由于Spring Initializer团队，这种体验实际上（现在）实际上是使用Spring Initializer从start.spring.io开始的！

感谢Spring Initializer团队 ，特别感谢Stephane Nicole指导我们完成该过程。非常感激。
