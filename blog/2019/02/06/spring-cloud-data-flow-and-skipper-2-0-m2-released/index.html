Spring Cloud Data Flow和Skipper 2.0 M2发布

发布
马克·波拉克
2019年2月6日

Spring Cloud Data Flow团队很高兴宣布发布2.0 M2数据流。遵循在本地 ， Cloud Foundry和Kubernetes上运行的入门指南 。

手拉手是2.0 M2发布Spring Cloud Skipper 。如果要与数据流分开使用Skipper，则参考指南中的“ 入门”部分是最好的起点。

这是数据流的重点

  • 针对多个平台启动任务

  • 改进指标并监视已部署的应用程序

  • 其他安全角色

  • 仪表板改进

  • 数据库迁移支持

针对多个平台启动任务

启动任务后，您可以指定执行任务的平台。通过Data Flow服务器，您可以配置多个Kubernetes和Cloud Foundry任务平台。当您要针对多个平台协调数据管道，但要从一个中央位置管理所有数据管道时，此功能特别有用。

例如，如果您要在Cloud Foundry上运行数据流org1/space1 ，您可以在中启动任务org2/space2 。如果在本地运行Data Flow Server，则可以指定不同的本地任务平台（例如具有不同的JVM属性），以替代使用任务部署属性。该文档描述了如何为Local ， Cloud Foundry和Kubernetes配置多个任务平台

改进指标并监视已部署的应用程序

Data Flow 2.0引入了一种新的体系结构，用于收集和显示Streams的应用程序指标。删除了Data Flow 1.4中引入的Data Flow Metrics Collector。

这种新的体系结构基于在已部署的应用程序中使用Micrometer库将度量发送到流行的监视系统，然后使用Grafana可视化度量。爱因斯坦发布者入门系列的每个应用程序都包含用于Prometheus，InfluxDB和DataDog的Micrometer库。

我们提供了有关如何开始使用Prometheus和InfluxDB作为笔记本电脑本地运行的Data Flow监视系统的说明。数周后，将在2.0 RC1版本中提供有关CloudFoundry和Kubernetes的说明。还提供了从数据流UI到Grafana仪表板的链接。还提供了两个Grafana仪表板，一个显示了更多以应用程序为中心的视图，另一个显示了更多基于流的视图。这是基于Prometheus的Grafana仪表板的屏幕截图，显示了流中应用程序的消息速率。

应用程序和流的选择器允许您向下导航到感兴趣的特定区域。

其他安全角色。

数据流2.0完全兼容OAuth2 / OpenIDConnect。基于令牌的授权，授予密码的类型验证和LDAP集成是一些选项，并且在UAA作为后端的支持下始终得到支持。

在此里程碑的基础上，我们还添加了新的精细角色来管理流/任务部署操作。粒度使与预期操作的一致性更好，并且客户端工具（Shell和Dashboard）也自动适应该操作。

仪表板改进

  • UI支持针对后端平台集合启动任务。UI / UX的构建具有流和任务启动的一致体验。

  • 基于另一个重要的功能集，现在可以根据数据流中定义的精细角色自动启用/禁用仪表板功能。

  • Grafana启动图标本地集成在仪表板的“流”和“运行时”页面中。借助针对可用的Micrometer后端配置的Data Flow，只需单击一下鼠标，即可监视流管道指标，例如消息速率，错误计数和延迟。

  • 现在支持一个新按钮以回滚到流的先前版本。

  • “分析”标签已被删除，并将在2.0 RC1版本中被基于微米的Grafana仪表板取代。

数据库迁移支持

从Data Flow 1.7迁移到Data Flow 2.0引入了模式更改。现在包括基于迁移的迁移，使您可以在启动2.0数据流服务器时轻松地将现有流/任务定义更新为新格式。

接下来是什么

我们计划在几周内发布2.0 RC1。大多数功能开发已经完成，但是除了上面提到的监视改进之外，我们还将添加支持以获取Task部署的详细历史记录-应用程序属性，部署属性，类似于Streams当今可用的功能。

另请注意，Spring Cloud Data Flow的1.x系列将从2.0 GA发布之日起十二个月后停止维护。暂定于2019年2月发布2.0 GA，因此2020年2月，1.x产品线将达到支持终止的状态。
