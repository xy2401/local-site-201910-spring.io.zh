Spring Cloud Data Flow和Skipper 2.0 RC1发布

发布
马克·波拉克
2019年2月22日

Spring Cloud Data Flow团队很高兴宣布Data Flow 2.0 RC1的发布。遵循在本地 ， Cloud Foundry和Kubernetes上运行的入门指南 。

携手并进的是Spring Cloud Skipper的2.0 RC1版本。如果要与数据流分开使用Skipper，则参考指南中的“ 入门”部分是最好的起点。

这是数据流的重点

  • 改进指标并监控Kubernetes上已部署的应用程序

  • 使用千分尺更新分析

  • 安全改进

  • 仪表板改进

  • 数据库迁移支持

改进指标并监控Kubernetes上已部署的应用程序

Data Flow 2.0引入了一种新的体系结构，用于收集和显示Streams的应用程序指标。删除了Data Flow 1.4中引入的Data Flow Metrics Collector。

这种新的体系结构基于在已部署的应用程序中使用Micrometer库将度量发送到流行的监视系统，然后使用Grafana可视化度量。爱因斯坦发布者入门系列的每个应用程序都包含用于Prometheus，InfluxDB和DataDog的Micrometer库。

在此版本中，我们提供了有关在Kubernetes上运行Data Flow时如何开始使用Prometheus作为监视系统的说明。还提供了两个Grafana仪表板，一个显示了更多以应用程序为中心的视图，另一个显示了更多基于流的视图。这是基于Prometheus的Grafana仪表板的屏幕截图，显示了流中应用程序的消息速率。

应用程序和流的选择器允许您向下导航到感兴趣的特定区域。

使用千分尺更新分析

在Data Flow 2.0中，Redis计数器的使用已由Micrometer库提供的计数器代替。爱因斯坦发布的应用程序入门系列现在提供了计数器处理器和计数器接收器 。twitter Analytics示例应用程序已更新为使用新的计数器实现，并且提供了Grafana仪表板以显示Data Flow Analytics UI先前显示内容的等效视图。

安全改进

服务器使用OAuth 2.0身份验证来保护REST端点。现在提供了文档和示例代码，它们显示了如何配置DataFlowTemplate以与OAuth2.0一起使用 。报告了最近CVE的从属库已更新。

仪表板改进

现在，您可以按日期范围搜索审核跟踪。

数据库迁移支持

使用一套基于Docker的集成测试 ，对所有支持的数据库中的Flyway迁移代码进行了改进和测试 。

接下来是什么

我们推迟了存储2.1任务部署的详细历史记录的功能。计划在2.0 GA发布之前进行各种文档问题和其他测试。2.1的一般主题将集中在文档，入门指南，视频和整体用户体验上。还计划建立一个新的Data Flow站点。

另请注意，Spring Cloud Data Flow的1.x系列将从2.0 GA发布之日起十二个月后停止维护。暂定于2019年2月底发布2.0 GA，因此2020年2月，1.x产品线将达到支持终止的状态。
