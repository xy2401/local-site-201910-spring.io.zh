Introduction

The Spring Cloud Connectors library has been with us since the launch event of Cloud Foundry itself back in 2011. One of the main goals of the connector library and Cloud Foundry’s Java buildpack was to “reduce the initial investment when you want to get started with Cloud Foundry”. The connector library creates the Spring bean definitions required to connect to backing services, like databases, using information contained in the VCAP_SERVICES environment variable. The buildpack then replaces these bean definitions you had in your application with those created by the connector library through a feature called ‘auto-reconfiguration’. java-cfenv简介：用于访问Cloud Foundry Services的新库

发布
马克·波拉克
二月15，2019

介绍

自2011年Cloud Foundry本身启动活动以来，Spring Cloud Connectors库就一直在我们身边。连接器库和Cloud Foundry的Java buildpack的主要目标之一是“减少您想开始使用Cloud Foundry时的初期投资”。连接器库使用VCAP_SERVICES环境变量中包含的信息创建连接到支持服务（如数据库）所需的Spring bean定义。然后，buildpack通过一种称为“自动重新配置”的功能，将连接器库中创建的这些bean定义替换为连接器库创建的那些bean定义。当您将应用程序推送到Cloud Foundry时，您可能已经在日志中看到了该消息...

-----> Downloading Spring Auto Reconfiguration 2.5.0_RELEASE from https://java-buildpack.cloudfoundry.org/auto-reconfiguration/auto-reconfiguration-2.5.0_RELEASE.jar

自动重新配置非常适合入门。但是，当您需要更多控制（例如，更改与DataSource关联的连接池的大小）时，它并不是很好。这需要编写特定于连接器的代码，并且不会像Spring Boot那样公开尽可能多的连接池选项。同样的限制也适用于其他支持服务。

这就提出了一个问题，为什么我们有两种竞争机制来创建服务基础结构bean？我们不能让Spring Boot为我们处理所有这一切吗？这就是创建我们新的java-cfenv库的动机。

Java-Cfenv简介

java-cfenv库的灵感来自Cloud Foundry生态系统其他地方使用的node-cfenv和py-cfenv库。这些库提供了一个简单的API，用于从VCAP_SERVICES环境变量中包含的JSON字符串中检索凭证。我们将首先浏览Java API（我们不会期望它会经常使用），然后展示它如何与Spring和Spring Boot的自动配置功能集成。

核心API包含五类

  • CfEnv负责解析VCAP_SERVICES和VCAP_APPLICATION环境变量的内容。
  • CfApplication提供访问器的内容VCAP_APPLICATION环境变量。
  • CfEnv上的Finder方法返回CfService类的实例。
  • CfService为服务的名称，标签，选项卡和计划以及CfCredentials对象提供访问器。
  • CfCredentials提供了用于获取用户名，密码，主机，端口和URI的访问器。 URI使用UriInfo类表示。

例如，如果您将MySql服务绑定到Cloud Foundry中的应用程序，则VCAP_SERVICES环境变量将包含诸如

使用java-cfenv API时，我们可以使用一些简单的方法调用来获取凭证信息，然后以编程方式创建与数据库的连接。

CfEnv cfEnv = new CfEnv();
CfService cfService = cfEnv.findServiceByName(“mysql”);
String plan = cfService.getPlan(); // 100mb
CfCredentials cfCredentials = cfService.getCredentials();
String password = cfCredentials.getPassword(); // J2BNJYkeXAH9idkG
UriInfo uriInfo = cfCredentials.getUriInfo();
String username = uriInfo.getUsername(); // 8McHri7aKbuTEGCR

findServiceByName方法采用正则表达式来帮助在不同的Cloud Foundry环境之间提供某些可移植性，这些环境可以对服务进行稍微不同的命名。还有其他查找程序方法，可以帮助您从标签和标签等中进行选择。

数据库支持

在前面的示例中，您可能只想获取JSON字段jdbcUrl，以便将其传递到DataSource 。您可以使用API进行此操作

String jdbcUrl = cfCredentials.getString(“jdbcUrl”);

但是，并非Cloud Foundry上的所有数据库服务都提供此方便的字段。实际上，Cloud Foundry上各种数据库服务提供的字段可能相当随意。现有的连接器库已开发出启发式方法来处理这种差异，并且已将此功能移植到新的java-cfenv库中。在CfEnvJdbc类中可用：

CfEnvJdbc cfEnvJdbc = new CfEnvJdbc();
CfJdbcService cfJdbcService = cfEnvJdbc.findJdbcService();
String jdbcUrl = cfJdbcService.getUrl();

如果有多个数据库服务绑定到该应用程序，则findJdbcService方法将引发异常。在这种情况下，可以使用findJdbcServiceByName方法在多个数据库服务中进行选择。

String jdbcUrl1 = cfEnvJdbc.findJdbcServiceByName('mysqlA').getUrl();
String jdbcUrl2 = cfEnvJdbc.findJdbcServiceByName('mysqlB').getUrl();

与Spring一起使用

如果您使用的是Spring而不是Spring Boot，则可以将CfJdbcEnv实例注册为Bean，然后使用Spring Expression Language设置其属性来调用其上的方法。

@Bean
public CfJdbcEnv cfJdbcEnv() {
  return new CfJdbcEnv();
}

然后在属性文件中，访问CfJdbcEnv实例

myDatasourceUrl=#{ cfJdbcEnv.findJdbcService().getUrl() }

与Spring Boot一起使用

大多数Spring Boot用户将不必直接使用java-cfenv API。java-cfenv库包含Spring Boot EnvironmentPostProcessor实现，可自动设置众所周知的Spring Boot属性。这允许Spring Boot的自动配置启动，同时仍然保留通过环境变量或其他优先级更高的环境属性源覆盖值的可能性。

从start.spring.io生成项目后，要做的就是将java-cfenv-boot依赖项手动添加到项目中，并在将应用程序推送到Cloud Foundry时禁用自动重新配置。对于Maven，依赖性为：


  io.pivotal.cfenv
  java-cfenv-boot
  1.0.0.M1

由于这是当前的里程碑版本，因此您需要添加一个里程碑配置，例如：


   spring-milestones
   Spring Milestones
   http://repo.spring.io/libs-milestone-local

要禁用自动重新配置，请在清单文件中使用以下命令或等效命令。

cf set-env  JBP_CONFIG_SPRING_AUTO_RECONFIGURATION '{enabled: false}'

由于自动重新配置还设置了许多应用程序都依赖的cloud配置文件，因此您可能还需要显式设置此配置文件。

cf set-env  SPRING_PROFILES_ACTIVE cloud

如果使用清单，则输入为：

env:
 SPRING_PROFILES_ACTIVE: cloud
 JBP_CONFIG_SPRING_AUTO_RECONFIGURATION: '{enabled: false}'

值得注意的是，在生产中，您可能仍然应该这样做。

要设置连接池属性，你现在可以简单地使用标准的Spring启动性能，如spring.datasource.maxActive=10等更具体的连接池属性。

下一步是什么？

GitHub上提供了有关如何使用java-cfenv库的更多信息。当前版本是1.0.0。M1和GA发布的路径很短，因为本月晚些时候将其合并到Data Flow 2.0 GA版本中。一如既往，我们欢迎您的反馈和贡献，即使它们会导致API更改中断，而该更改也需要在1.0发行后不久纳入Java-cfenv 2.0版本。

java-cfenv的1.0 GA版本将支持Spring Cloud Connectors项目中的所有服务，因为Spring Boot自动配置很好地支持了这些服务。届时，现有的连接器库将进入维护模式。 当然，将解决关键的错误和安全性问题，但不会添加新功能。1.0 GA发行版的java-cfenv将提供将应用程序从Spring Cloud Connectors迁移到java-cfenv的指南。

在项目页面上列出了基于核心连接器项目的其他库。这些扩展项目可以继续使用连接器，但是鼓励维护人员迁移到以引导为中心的方法。
