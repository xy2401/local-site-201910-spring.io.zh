Apache Geode和Pivotal GemFire 1.0.1的Spring Boot。发布＆1.1.0。M2可用

发布
约翰·布鲁姆
2019年6月18日

我很高兴代表Spring和Apache Geode社区宣布发布适用于Apache Geode和Pivotal GemFire （SBDG）的Spring Boot。 1.0.1.RELEASE和1.1.0.M2 。

SBDG的新功能1.0.1.RELEASE

SBDG 1.0.1.RELEASE建立在：

  • Spring Framework 5.0.14。发布

  • Spring Boot 2.0.9。发布

  • Spring Data Kay-SR14

  • Apache Geode和Pivotal GemFire 2.0.9的 Spring 会议。发布

此外，SBDG现在支持配置和引导Locator进程的功能，该进程将禁用自动配置 ，从而无法提供ClientCache实例，默认情况下。

有关更多详细信息，请参见完整的变更日志 。

SBDG的新功能1.1.0.M2

SBDG 1.1.0.M2建立在：

  • Spring Framework 5.1.8。发布

  • Spring Boot 2.1.4。发布

  • Spring Data Lovelace-SR9

  • Apache Geode和Pivotal GemFire 2.1.4的 Spring 会议。发布

此外，SBDG现在包括以下增强功能和新功能：

  • 能够配置和引导Locator进程，禁用默认设置ClientCache 自动配置 。

  • 专门支持内联缓存用例；从这里提供的参考文档中，尤其是将Inline Caching与Spring Data Repository抽象（一种非常强大的组合）联系起来的部分 ，了解更多信息。

  • 添加新模块或Spring Session（例如spring-geode-starter-session ）和 Spring 测试（例如spring-geode-starter-test ）。

有关更多详细信息，请参见完整的变更日志 。

下一步是什么

大部分关注点和重点都转移到了SBDG 1.1，并争取在7月初至7月中旬完成最终的GA。

我们将添加对Pivotal CloudFoundry（PCF），与独立Apache Geode或Pivotal GemFire安装连接的Spring Boot应用程序部署的支持。

当前，SBDG假设在将支持Apache Geode或Pivotal GemFire的Spring Boot应用程序部署到PCF时，这些应用程序将使用Pivotal Cloud Cache（PCC）服务。当然，这并非总是可能的，因此我们计划以您选择的任何能力为您提供支持。

我们强烈建议您在将Spring Boot应用程序部署到PCF时，还要将这些相同的应用程序绑定到高度安全且受管的Pivotal Cloud Cache环境。PCC将运营问题视为非问题。
