Spring Security 5.2.0。M3发布

发布
Eleftheria Stein-Kousathana
六月17，2019

我很高兴代表社区宣布Spring Security 5.2.0的发布。M3！您可以在变更日志和以下重点中找到完整的详细信息：

OAuth 2.0

gh-6727-在Reactive Resource Server中支持多租户
gh-6798-支持不透明令牌中的自定义参数
gh-6239-用于OAuth2 redirectUriTemplate扩展的更精细的变量
gh-6863 -OAuth2登录名具有可配置的身份验证成功处理程序
gh-6832和gh-6849 -JWT和不透明令牌具有可配置的身份验证管理器
gh-6634-在测试中支持模拟JWT

类似于其他请求后处理器， jwt()可用于使用JwtAuthenticationToken建立SecurityContext 。

mockMvc.perform(get("/")
       .with(jwt(jwt -> jwt.claim("scope", "message:read"))));

核心

gh- 6819-添加nohttp来构建

有关nohttp项目的更多信息，请参见此博客文章 。

gh-4469-支持消息表达式中的路径变量
gh- proxyBeanMethods=false配置类是无代理的，并且支持proxyBeanMethods=false

此功能包括对AbstractSecurityWebSocketMessageBrokerConfigurer和GlobalMethodSecurityConfiguration类的重大更改。

为了适应这些更改，扩展AbstractSecurityWebSocketMessageBrokerConfigurer用户（覆盖方法inboundChannelSecurity将需要更新方法签名以匹配以下内容。

@Bean
public ChannelSecurityInterceptor inboundChannelSecurity(
    MessageSecurityMetadataSource messageSecurityMetadataSource) {
// implementation
}

同样，扩展GlobalMethodSecurityConfiguration并覆盖方法methodSecurityInterceptor将需要更新方法签名以匹配以下内容。

@Bean
public MethodInterceptor methodSecurityInterceptor(
    MethodSecurityMetadataSource methodSecurityMetadataSource) {
// implementation
}
