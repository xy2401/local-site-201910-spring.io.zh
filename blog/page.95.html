Spring Cloud Dalston RC1发布

发布
瑞安·巴克斯特（Ryan Baxter）
2017年3月21日

我很荣幸地代表社区宣布， Spring Cloud Dalston发布系列的发布候选版本1（RC1）现已发布。该版本可以在我们的Spring Milestone存储库中找到。您可以查看Dalston 发行说明以获取更多信息 。

Dalston发布系列的重大变化

金库

Spring Cloud Vault Config是一个新项目，可通过Hashicorp Vault为分布式系统中的外部化秘密管理提供客户端支持。

配置服务器

Config Server现在通过Composite模式支持多个后端。这允许组合以前不可能的后端类型，例如：Vault和git或多个git后端。现在支持对AWS Codecommit托管的git存储库的身份验证。在以前的（Camden）版本中，已经将Hashicorp Vault作为后端添加到Config Server，以与基于VCS的后端一起使用。

Spring Cloud Stream 切尔西。RC1发布

发布
马里乌斯·波哥维奇（Marius Bogoevici）
2017年3月16日

我很高兴代表团队宣布发布Spring Cloud Stream切尔西。RC1。 Spring Cloud Stream 切尔西。RC1可在Spring Milestone存储库中使用 ，其功能的详细说明可以在参考文档中找到。有关工件和更改的完整列表的信息，请查阅发行说明 。

有什么新东西？

以下是新版本带来的主要新功能和改进的摘要：

调度功能已添加到StreamListener

我们添加了支持，可以基于基于SpEL的条件将消息调度到在输入通道上注册的多个@StreamListener方法 。这为编写消息驱动的微服务提供了更大的灵活性，尤其是对于DDD / ES / CQRS场景，在DDD / ES / CQRS场景中，可以将不同类型的事件直接分派到其处理方法。

@EnableBinding(Sink.class)
@EnableAutoConfiguration
public static class {

    @StreamListener(target = Sink.INPUT, condition = "headers['type']=='customer'")
    public void handleCustomerEvent(@Payload CustomerEvent customerEvent) {
       // handle the message
    }

    @StreamListener(target = Sink.INPUT, condition = "headers['type']=='order'")
    public void handleOrderEvent(@Payload OrderEvent orderEvent) {
       // handle the message
    }
}

指标

Spring Cloud Stream添加了一个附加模块，该模块支持在专用通道上导出Spring Boot指标。现在，您可以通过简单地在类路径上添加模块并提供参考文档中所述的目标位置来从应用程序中收集度量。默认情况下，该模块导出Spring Integration指标（包括绑定通道指标），但也可以添加其他指标。这为Spring Cloud Stream应用程序中的流量监控提供了一流的支持。

模式改进：搜索和缓存

模式支持的新功能包括模式搜索和客户端级别的缓存 ，后者为序列化支持增加了显着的性能改进。

RabbitMQ自定义基础架构支持

RabbitMQ活页夹现在支持自定义目标的类型及其属性 ，包括支持直接交换和消息的TTL设置。

设置SPI

从此版本开始，Spring Cloud Stream引入了新的置备SPI，抽象了目标代理上目标（主题，交换，队列）的创建和配置。这允许在基础结构管理和活页夹的消息传递方面之间更好地分离关注点。

适用于AWS 1.1.0 M1的Spring Integration Extension

发布
阿尔特姆·比兰（Artem Bilan）
2017年3月9日

我谨代表Spring Integration社区宣布适用于Amazon Web Services版本的Spring Integration Extension的第一个里程碑1.1 。它的伪像是spring-integration-aws.1.1.0.M1 ，可在Milestone存储库中找到 。

当然，首先，非常感谢您，社区的贡献！

此里程碑中包含的功能的一些要点：

运动支持

的KinesisMessageDrivenChannelAdapter和KinesisMessageHandler提供与Amazon Kinesis集成的功能。前者非常简单，可以将数据发送到Kinesis流中。目标的所有信息PutRecordRequest可以根据请求确定Message ：

@Bean
@ServiceActivator(inputChannel = "kinesisSendChannel")
public MessageHandler kinesisMessageHandler() {
    KinesisMessageHandler kinesisMessageHandler =
                new KinesisMessageHandler(amazonKinesis());
    kinesisMessageHandler.setAsyncHandler(asyncHandler());
    kinesisMessageHandler.setStream("my_stream");
    kinesisMessageHandler.
             setPartitionKeyExpressionString("headers[aws_partitionKey]");
    return kinesisMessageHandler;
}

默认情况下，它使用SerializingConverter将请求数据转换为byte[] 。的com.amazonaws.handlers.AsyncHandler可用于异步putRecordAsync()结果反应。

的KinesisMessageDrivenChannelAdapter提供了全面的Kinesis流数据摄取实现，包括sequenceNumber检查点和重新分片支持。的concurrency选项可用于下游流程中的严格订单记录处理。在这种情况下，提供的碎片分布在线程之间。如果未提供并发，则内部ShardConsumer在上执行consumerExecutor直：

@Bean
public KinesisMessageDrivenChannelAdapter kinesisMessageDrivenChannelAdapter() {
    KinesisMessageDrivenChannelAdapter adapter =
            new KinesisMessageDrivenChannelAdapter(amazonKinesis(), STREAM1);
    adapter.setOutputChannel(kinesisChannel());
    adapter.setCheckpointStore(checkpointStore());
    adapter.setCheckpointMode(CheckpointMode.manual);
    adapter.setListenerMode(ListenerMode.batch);
    adapter.setStartTimeout(10000);
    adapter.setDescribeStreamRetries(1);
    adapter.setConcurrency(10);
    return adapter;
}

如果CheckpointMode是manual ， AwsHeaders.CHECKPOINTER消息头填充到每个发出的消息中。它是Checkpointer及其checkpoint()可用于下游流向checkpoint一种sequenceNumber用于分片中的已处理记录。

注意
 Amazon Kinesis Channel Adapters实施完全基于该标准aws-java-sdk-kinesis API并且不使用Kinesis Client Library 。
