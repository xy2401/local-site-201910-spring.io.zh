SpringOne2GX 2013重放：带有Spring Data Neo4j的实际应用程序

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2014年2月11日

在加州圣塔克拉拉的SpringOne2GX 2013上录制。

演讲者：Michael Hunger

Spring Data Neo4j在其2.x版本中是对象图映射框架的成熟实现，它既与Spring框架紧密集成，又通过选择属性图数据模型提供了独特的建模和查询机会。本演讲将介绍几个Neo Technology客户如何将Spring Data Neo4j投入生产。它们来自多个领域-游戏，服务器中心监控，教育，设备管理，能源市场模拟等。基于这些用例，我们讨论如何使用Spring Data Neo4j和Spring Framework实现实际应用程序。首先，我们要讨论他们必须解决的业务问题，以及为什么图数据库非常适合他们。在快速介绍Spring Data Neo4j和Neo4j之后，我们将讨论建模选择以及选择丰富数据模型的好处和影响。从带注释的域实体到强大的存储库抽象的用法，我们将示例域放入代码中。展示如何使用Neo4jTemplate来实现一些较低级别的功能将使核心库的使用更加完善。在此基础上，我们研究了集成方面，如何将Spring Data Neo4j与其他Spring项目（如Spring Data MongoDB，Spring Data REST和Spring Batch）结合。快速演示了如何从（Web）-UI层访问Spring Data Neo4j。谈话将以对未来的展望结束：Neo4j在其2.0版本中添加了一些非常有趣的功能。从节点标签，自动索引和约束到Cypher的事务性HTTP端点。这些更改使得重新思考Spring Data Neo4j的某些方面成为可能，例如类型表示和索引编制。但是，还有一些更大的想法可以改变，例如完全基于Neo4j的Cypher Graph Query Language建立对象图映射。

Spring Data Elasticsearch 1.0 M1发布

发布
奥利弗·德罗博姆
2014年2月11日

我代表BioMed Central的 Mohsin Husen周围的开发团队高兴地宣布Spring Data Elasticsearch社区模块的第一个里程碑。

该模块的发布在Elasticsearch的基础上实现了著名的Spring Data编程模型，其中包括对动态映射（嵌套类型，对象，父子对象），索引和搜索API，方面，突出显示，别名，更类似的支持，扫描和滚动等

从Spring Data的角度来看，此发行版支持模板，存储库，CDI，Java和XML样式的配置。
实体将自动转换为JSON文档并索引到Elasticsearch，可使用模板和存储库查询。
