Spring Integration 5.0 Milestone 1可用

发布
加里·罗素
2016年12月2日

我们很高兴地宣布，Spring Integration 5.0版本的第一个里程碑现已发布。

这是一个新的主要版本，基于Spring Framework 5.0，并且需要Java 8。这是迄今为止最大的更改，但还包括以下内容：

  • 现在， Java DSL已集成到框架本身中。 DSL有一些细微的变化，例如删除了.handleWithAdapter()方法和一些常规Factory类。有关DSL更改的完整讨论，请参见《 迁移指南》 。

  • 升级到Spring Data Kay 。

  • 升级到Spring AMQP 2.0 。

  • DSL中已添加了对TCP / UDP的一流支持。

  • Spring Integration现在基于Reactor 3.0和Messaging Gateway Promise现在必须更改方法以返回Mono 。

  • 您现在可以通过配置中间流程交易TransactionHandleMessageAdvice对于adviceChain邮件注释属性和使用XML配置时的子元素。

春季AMQP 2.0里程碑1上市

发布
加里·罗素
2016年11月30日

我们很高兴地宣布，Spring AMQP 2.0的第一个里程碑已经面世。

有关更改的完整列表；请参阅参考手册和发行说明中 的新增功能 。

以下是此版本的一些亮点

  • 该框架现在完全基于Java 8。提供了几个功能接口，以方便Lambda实施（回调， ReplyingMessageListener等等）。现在需要Java 8。它也基于Spring Framework 5.0 。

  • 该框架使用新的4.0 amqp-client库 。

  • 一个新的DirectMessageListenerContainer现在可以与现有产品一起使用SimpleMessageListenerContainer 。新的容器实际上具有更简单的体系结构，并且直接在amqp-client线程上调用侦听器（因此，名称Direct…​ ）。请参阅选择容器以帮助您确定哪个容器适合您的应用程序。

  • 子类- DirectReplyToMessageLisenerContainer还提供了-，专门用于将RabbitMQ Direct ReplyTo功能用于请求/回复消息传递。现在由RabbitTemplate （默认情况下），而不是像以前那样根据请求/回复创建使用者。消费者被缓存以备重用。

  • 的AsyncRabbitTemplate现在支持使用新容器的直接replyTo；以前，异步模板不支持直接ReplyTo。

  • @RabbitListener批注现在可以配置有错误处理程序，并且可以在使用请求/答复消息传递时将异常发送给发件人。

  • 现在，连接和通道侦听器会收到关闭信号。

  • 您现在可以为Spring AMQP创建的连接分配名称-该名称显示在RabbitMQ管理员UI上。

  • 现在（在入站消息没有消息ID时）可以使用有限的有状态重试（1次重新尝试）。 MissingMessageIdAdvice现在已从框架中删除。需要Spring Retry 1.2才能支持此功能。

  • 由于log4j的EOL，不再提供log4j附加程序（log4j2和logback附加程序仍然可用）。
