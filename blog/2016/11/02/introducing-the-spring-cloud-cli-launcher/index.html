介绍Spring Cloud CLI启动器

工程
斯宾塞·吉布
2016年11月2日

Spring Cloud入门可能令人生畏。如果您看到Josh Long出色地发表了他的Cloud Native Java演示，您将注意到您需要创建多个支持应用程序，然后才能看到您的应用程序在工作。

Spring Cloud CLI启动器

作为从Spring XD到Spring Cloud Dataflow过渡的一部分，提取的项目之一称为Spring Cloud Deployer 。Deployer允许在各种平台上启动应用程序，包括在开发人员计算机上启动。

我们希望简化用户试用Spring Cloud的能力，因此spring cloud命令诞生了。

的spring cloud该命令会自动运行在Spring Cloud环境中运行应用程序所需的一些支持服务器，例如Config Server和Eureka。

安装Spring Boot CLI

第一步是安装Spring Boot CLI （ spring命令）。所需的最低版本是1.4.1.RELEASE 。我自己更喜欢SDKMAN方法 。

$ sdk install springboot
$ spring --version
Spring Boot v1.4.1.RELEASE

安装Spring Cloud CLI

接下来，您需要安装Spring Cloud Launcher CLI（ spring cloud命令）通过Spring Boot CLI。

$ spring install org.springframework.cloud:spring-cloud-cli:1.2.1.RELEASE

默认服务器

如果你跑spring cloud不带任何参数的情况下，将启动Config Server和Eureka。Config Server在其他服务之前启动，以便它可用，以便那些其他服务可以提取其配置。

可用服务器启动

可以启动以下服务器：

$ spring cloud --list
configserver dataflow eureka h2 hystrixdashboard kafka zipkin

要运行一组特定的服务器，只需将它们作为参数列出：

$ spring cloud configserver eureka hystrixdashboard

前面的命令运行Config Server，Eureka和Hystrix仪表板。

用法

$ spring help cloud
spring cloud - Start Spring Cloud services, like Eureka, Config Server, etc.

Option         Description
------         -----------
-d, --debug    Debug logging for the deployer
-l, --list     List the deployables (don't launch
                 anything)
-v, --version  Show the version (don't launch

examples:

    Launch Eureka:
        $ spring cloud eureka

    Launch Config Server and Eureka:
        $ spring cloud configserver eureka

    List deployable apps:
        $ spring cloud --list

    Show version:
        $ spring cloud --version

配置流程

可以使用具有特殊名称和位置的常规Spring Boot外部配置文件来配置每个进程。文件名基于进程名（再次参见spring cloud --list查看完整列表）。例如，可以在名为文件的文件中配置服务器configserver.yml要么configserver.properties 。配置文件的搜索路径是当前目录（ . ）， ./config和$HOME/.spring-cloud/ 。

默认情况下， configserver流程使用native JAR中的配置文件和配置。例如，让我们全局配置Config Server以从GitHub存储库加载属性。内容$HOME/.spring-cloud/configserver.yml可能：

spring:
  profiles:
    active: git
  cloud:
    config:
      server:
        git:
          uri: https://github.com/spring-cloud-samples/config-repo

处理信息

处理 港口 网址
configserver 8888 /info
eureka 8761 不适用
hystrixdashboard 7979 /hystrix
dataflow 9393 /dashboard
kafka 9092 不适用
zipkin 9411 不适用
h2 9095 /

URL列是向Eureka注册的内容（如果正在运行）。这使您可以单击Eureka仪表板中的实例链接。

如果某个进程向Eureka注册，它将使用“进程”列中的值作为spring.application.name 。

的kafka进程还会在端口2181上启动ZooKeeper。Kafka用于Spring Cloud Bus。

的h2数据库可通过以下JDBC URL在端口9096上使用： jdbc:h2:tcp://localhost:9096/./target/test 。H2 URL可以通过spring.datasource.url属性。
