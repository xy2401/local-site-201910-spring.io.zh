Vaadin Spring 1.1-构建可直接连接到Java后端的Web UI

工程
斯特凡·尼科尔（StéphaneNicoll）
2016年11月15日

这篇文章是社区成员Matti Tahvonen（@MattiTahvonen）的来宾帖子，他是Vaadin Ltd的开发倡导者，该公司最初开发了Vaadin Framework，并为其提供商业服务和扩展。

Vaadin的官方Spring集成库的第一个稳定版本已于一年前发布。反馈非常好，许多Spring开发人员一直希望保持安全的“后端”，他们发现了一种使用现有Java技能来构建美观的Web UI的新方法。使用Vaadin Spring，您的UI代码位于Spring托管的bean中，并且与基于Spring的后端的集成变得微不足道。

Vaadin Spring 1.1是第一个功能版本，于上周发布。它提供了一个方便的SpringNavigator （在Spring Boot应用程序中自动配置），使您可以更轻松地为基于Vaadin Spring的较大UI创建“顶级导航”。 Vaadin Spring 1.1.0和先前的1.0.x版本还修复了许多错误，因此现有Vaadin Spring用户肯定应该升级其版本。

如果您是Vaadin的新手，或者想要一些方便的Spring技巧，那么您绝对还应该查看我们最近由StéphaneNicoll（Pivotal）和Matti Tahvonen（Vaadin）主办的联合网络研讨会“ Spring和Vaadin的Java Web应用程序”。除了与Vaadin相关的内容外，网络研讨会还包含许多与devtools相关的工作效率提示，这些信息涉及RESTful服务的使用以及使用Flyway维护数据库结构。我们使用的示例应用程序也可以在github上找到 。

如果您更喜欢“边干边学”，并且对在Spring Web应用程序中使用Vaadin Framework感兴趣，请访问vaadin.com/spring 。
