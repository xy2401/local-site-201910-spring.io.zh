On behalf of the team, I am pleased to announce the GA release of Spring Cloud Data Flow 1.1. Follow the links in the getting started guide to download the local server implementation and shell to create Stream and Tasks.

General highlights of the 1.1 GA Release include:

  • Builds upon Spring Boot 1.4, Spring Cloud Camden SR2, Spring Integration 4.3 and Spring Cloud Task 1.1 release improvements.

  • Adds LDAP, Basic and File based backend authentication

  • Improvements to OAUTH backed authentication

  • LDAP authentication is now supported with SSL

  • Adds a form-based login page for non-OAUTH backend authentication methods such as the LDAP, Basic and File-based options

  • Adds the ability to pass application specific properties via YAML file. This is particularly useful when deploying streams that require many deployment properties to be set.

  • Portable deployment properties for memory, disk and cpu are in place for support across various runtime implementations.

  • Documentation for the core REST API

  • Updated reference documentation showing how to integrate with Spring Boot Admin to visualize server metrics and how to export Stream/Task application metrics to external application monitoring tools.

Spring Cloud Data Flow 1.1 GA发布

发布
马克·波拉克
2016年11月23日

我很高兴代表团队宣布GA发布Spring Cloud Data Flow 1.1。按照入门指南中的链接下载本地服务器实现和Shell，以创建Stream和Tasks。

1.1 GA版本的主要亮点包括：

  • 基于Spring Boot 1.4，Spring Cloud Camden SR2，Spring Integration 4.3和Spring Cloud Task 1.1发行版的改进。

  • 添加基于LDAP ， Basic和File的后端身份验证

  • OAUTH支持的身份验证的改进

  • SSL现在支持LDAP身份验证

  • 为非OAUTH后端身份验证方法（例如LDAP，Basic和基于文件的选项）添加基于表单的登录页面

  • 添加了通过YAML文件传递应用程序特定属性的功能 。当部署需要设置许多部署属性的流时，这特别有用。

  • 内存，磁盘和cpu的可移植部署属性已准备就绪，可在各种运行时实现中提供支持。

  • 核心REST API的文档

  • 更新了参考文档，显示了如何与Spring Boot Admin集成以可视化服务器指标以及如何将Stream / Task应用程序指标导出到外部应用程序监视工具 。

UI重点包括：

  • 流和任务应用启动器的批量导入和注册

  • 对任务定义的批量导入支持。这允许从文件导入任务定义，或者使用Flo-editor添加多个任务定义的功能。

  • 添加Flo可视化表示正在运行的流

  • 为相关流添加视觉表示。此表示形式还包括嵌套的TAP和整个拓扑视图中的下游处理节点。

  • 列表页面现在支持排序

  • 服务器端对流和任务列表页面的搜索支持

有关功能，错误修复和改进的完整列表，请参阅封闭的GitHub 1.1系列问题 。
