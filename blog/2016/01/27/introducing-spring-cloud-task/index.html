介绍Spring Cloud Task

发布
迈克尔·米内拉
一月27，2016

今天，我们很高兴为您介绍Spring产品组合中的一个新项目Spring Cloud Task 。在Spring Cloud项目分组和Spring Cloud Stream的姊妹项目的保护下，Spring Cloud Task旨在为基于Spring Boot的应用程序带来支持短期微服务所需的功能。通过此博客文章，我们将发布该项目的第一个里程碑，并渴望您的投入和反馈。

Spring Cloud Task的目标

在大多数情况下，现代云环境是围绕执行不希望结束的流程（例如Web应用程序或流模块）而设计的。如果这样做，平台认为这是一个故障，通常会重新启动它们。尽管许多平台确实具有某种方法来执行预期将要结束的过程（例如，批处理作业），但是通常不会以消耗性的方式维护该执行的结果。然而，对于任务关键型应用程序，尽管它们的寿命很短，但它们仍然具有长期运行过程中相同的非功能性要求。

尽管此功能在云环境中很有用，但在传统部署模型中也会出现相同的问题。通过像cron这样的调度程序执行Spring Boot应用程序时，能够在应用程序完成后监视其结果可能会很有用。

Spring Cloud Task采用了一种方法，即Spring Boot应用程序可以具有起点和终点，并且仍然是生产级。批处理应用程序只是其中短暂流程可能会有所帮助的一个示例。

入门

在Spring Cloud Task的第一个版本中，我们介绍了配置机制的基础知识以及用于记录执行任务时发生的情况的存储库。那些熟悉Spring Batch及其作业存储库的人应该会熟悉Spring Cloud Task的一般范例。

要熟悉Spring Cloud Task，最简单的方法是编写应用程序。该文档将引导您完成初始任务的创建及其执行。您可以在此处了解更多信息。

我们要去的地方

在即将发布的版本中，任务将是Spring Cloud Data Flow中的新原语，使用户几乎可以将所有Spring Boot应用程序作为短期任务执行。这提供了从简单运行所有内容的灵活性CommandLineRunner以动态方式完成一个完整的批处理作业，从而允许按需消耗资源并在完成后将其返回。即将发布的版本（包括我们计划在Spring Cloud Data Flow达到1.0 GA之前计划的Spring Cloud Task 1.0 GA）将添加支持该工作的功能。
