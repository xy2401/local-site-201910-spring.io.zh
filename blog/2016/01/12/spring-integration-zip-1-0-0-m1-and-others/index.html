春季集成邮编

今天我们很高兴地宣布Spring Integration Zip 1.0.0。现在可以从Milestone存储库中获取M1 。

特别要感谢我们的好朋友Gunnar Hillert ，他发起并开始了这项扩展的工作。

Spring Integration Zip扩展提供了您期望的压缩/解压缩组件。“艰苦”的工作是由AbstractZipTransformer在幕后使用ZeroTurnaround ZIP库的实现。 zt-zip在标准之上提供了方便的高级API java.util.zip包。

借助里程碑1，提供了以下组件：

拉链变压器

该组件的目标是基于java.util.zip压缩入站消息有效负载并生成压缩存档。放气压缩水平。支持以下消息有效负载类型： File ， String ， byte[]要么Iterable这些类型。结果可以返回为File或作为byte[]压缩数据。这是由ZipResultType 。

使用XML组件配置Zip Transformer很容易：

<int-zip:zip-transformer input-channel="input"
                         output-channel="output"
                         result-type="BYTE_ARRAY"
                         compression-level="9"/>

以及Java Config：

@Bean
@Transformer(inputChannel = "input", outputChannel = "output")
public ZipTransformer zipTransformer() {
    ZipTransformer zipTransformer = new ZipTransformer();
    zipTransformer.setCompressionLevel(Deflater.BEST_COMPRESSION);
    zipTransformer.setZipResultType(ZipResultType.BYTE_ARRAY);
    return zipTransformer;
}

解压变压器

解压缩组件中实现的逻辑同样简单明了。对于输入消息有效负载，支持以下类型： File ， String要么byte[]并视为要解压缩的档案。解压缩数据时，还可以指定一个属性expectSingleResult 。如果设置为true并超过1检测到zip条目， MessagingException将被提出。此属性还影响有效负载的返回类型。如果设置为false （默认），则有效负载将为SortedMap如果true ，但是将返回实际的zip条目。

该组件的配置也很简单：

<int-zip:unzip-transformer input-channel="input"
                           output-channel="output"
                           delete-files="true"
                           result-type="FILE"
                           expect-single-result="true"/>

@Bean
@Transformer(inputChannel = "input", outputChannel = "output")
public UnZipTransformer unZipTransformer() {
    UnZipTransformer unZipTransformer = new UnZipTransformer();
    unZipTransformer.setExpectSingleResult(true);
    unZipTransformer.setZipResultType(ZipResultType.FILE);
    unZipTransformer.setWorkDirectory(new File("/usr/tmp/uncompress"));
    unZipTransformer.setDeleteFiles(true);
    return unZipTransformer;
}

解压缩结果分配器

的UnZipResultSplitter可用作下游帮助程序组件，以将每个未压缩的条目生成为单独的消息。的FileHeaders.FILENAME和ZipHeaders.ZIP_ENTRY_PATH每个拆分项目的标题均已填充：

<int:chain input-channel="input" output-channel="out">
    <int-zip:unzip-transformer result-type="BYTE_ARRAY"/>
    <int:splitter>
        <bean class="org.springframework.integration.zip.splitter.UnZipResultSplitter"/>
    </int:splitter>
</int:chain>
