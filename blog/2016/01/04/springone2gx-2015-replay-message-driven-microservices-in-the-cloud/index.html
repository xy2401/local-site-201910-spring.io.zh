SpringOne2GX 2015重播：云中的消息驱动微服务

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2016年1月4日

记录在SpringOne2GX 2015
主讲人：David Syer博士，Mark Pollack博士
幻灯片： http ： //www.slideshare.net/SpringCentral/message-driven-microservices-in-the-cloud

您是否需要构建云原生的微服务来进行分布式流处理和数据集成，但不知道从哪里开始？您所需要的只是代码中的单个注释和POM中的单个依赖项。新的Spring Cloud Stream项目结合了Spring Boot，Spring Integration和Spring Cloud的强大功能，从而实现了这一目标。

简而言之，Spring Cloud Stream支持自动配置Spring Integration功能，例如Kafka，RabbitMQ和Redis的通道适配器。您构建的应用程序基于配置文件驱动的基础，可确保跨运行时环境从便携式计算机到云的可移植性。扩展了这些分布式应用程序后，您将欣赏集成的Spring Cloud功能，例如服务发现和配置管理。

本次会议将在从本地云到远程云的不同平台上进行一系列现场演示，以展示Spring Cloud Stream开发人员体验的简便性。我们还将深入探讨幕后发生的事情。由于该项目本身已经开始了云原生的微服务之旅，因此将有关于该项目如何开始的背景及其与Spring XD的演变的相关性的故事。
第三个平台的特点是，基础设施的流动性强，虚拟服务器的存在和淘汰以及不断变化的工作负载和规模不断扩大，以满足不断变化的需求，因此需要新的设计模式，流程甚至文化。这些新范例最著名的描述之一是十二因子应用程序（12factor.net），它描述了云原生应用程序的元素。通过Spring Framework 可以直接满足其中的许多需求，而其他需求则需要其他系统的支持。在本次会议中，我们将研究十二个因素中的每一个，并介绍Spring和Cloud Foundry等平台如何满足它们，在某些情况下，我们甚至建议责任应从Spring转移到平台。最后，您将了解云原生应用程序需要什么，为什么以及如何满足这些要求。
