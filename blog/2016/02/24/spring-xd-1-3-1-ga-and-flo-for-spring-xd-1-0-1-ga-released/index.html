    发布了Spring XD 1.3.1 GA和适用于Spring XD 1.0.1 GA的Flo

    发布
    马克·波拉克
    2016年2月24日

    今天我们很高兴地宣布Spring XD 1.3.1和Flo for Spring XD 1.0.1的全面上市

    • Spring XD 1.3.1 GA： zip ， brew和rpm 。
    • Spring XD 1.0.1 GA的Flo： zip 。

    以下是一些错误修复和常规改进的要点。有关已修复问题的完整列表，请参考JIRA发行说明 。

    • 支持XD配置文件中的加密
    • 支持Kafka同步或异步生产者
    • 修复嵌入式标头消息转换器缓冲区溢出
    • 更新默认Redis池设置以使活动连接数不受限制
    • 升级到Spring Integration 4.2.5，Spring Framework 4.2.5，Spring AMQP 3.6

    下一个发行版本对基础架构进行了重大的重新设计，并将Spring XD分解为几个独立的项目。在Spring Boot和Spring Cloud功能的基础上，Spring XD被积极地重新设计为Spring Cloud Data Flow –一种用于可组合消息驱动微服务的云原生编程和操作模型。更多的细节可以在博客文章中找到： 介绍春季云数据流和快速浏览一下春天的云数据流 。InfoQ还总结了文章SpringXD中的更改，这些更改被重新构建并重新命名为Spring Cloud Data Flow 。

    反馈非常重要，因此请通过以下方式与问题和评论联系
    * StackOverflow spring-xd标签
    * Spring JIRA或GitHub问题
