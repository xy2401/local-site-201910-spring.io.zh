SpringOne2GX 2015重播：使用内存中的分布式数据构建高度可扩展的Spring应用程序

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2016年2月2日

记录在SpringOne2GX 2015中。
主讲人：约翰·布鲁姆（John Blum）和卢克·香农（Luke Shannon）
数据/集成跟踪
幻灯片： http : //www.slideshare.net/SpringCentral/building-highly-scalable-spring-applications-using-inmemory-data-grids-53086251

构建高度可扩展的，低延迟的应用程序，同时保持一致性，高可用性和弹性，需要一种新的技术。在本演示中，我们介绍了Pivotal GemFire以及开源产品Apache Geode。Apache Geode是一个经过验证的，分布式的内存数据库，具有ACID属性，可以在重负载下处理大量事务数据。Apache Geode为基于Spring的应用程序提供了随需求变化所需的优势，而不会牺牲完整性或最终用户的体验。

通过使用Spring Boot和Spring Data GemFire，我们演示了如何使用Pivotal GemFire / Apache Geode有效地构建高度可扩展的应用程序，从配置和设置开始，然后进入使用Spring Data Commons存储库，具有适当索引的OQL和Spring Data进行持久化和访问数据。 GemFire基于注释的，数据感知的函数执行，基于将业务逻辑带入数据的熟悉的Map-Reduce模式。接下来，我们将扩展这些基本的基本功能，并提供有关分区，并置，直写式/后写式，寄存器兴趣/ CQ，PDX和缓存以及基于JTA的全局事务的高级主题。

最后，我们展示了如何使用Pivotal GemFire / Apache Geode无缝解决其他应用程序问题，从使用Spring的核心Cache Abstraction进行缓存到使用Spring Session进行会话管理。最重要的是，我们介绍了正确测试基于Pivotal GemFire构建的应用程序的技术，以及管理和监视群集的技术。在会议结束之前，与会者应该对有效，高效地构建高度可扩展的应用程序感到满意。
