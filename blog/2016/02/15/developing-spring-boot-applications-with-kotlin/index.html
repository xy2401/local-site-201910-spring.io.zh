使用Kotlin开发Spring Boot应用程序

工程
塞巴斯蒂安·德勒兹（SébastienDeleuze）
2016年2月15日

更新：现已提供全面的Spring Boot + Kotlin教程 。

为了及时发布Kotlin 1.0版本 ，我们正在向https://start.spring.io添加对Kotlin语言的支持，以便更轻松地使用该语言启动新的Spring Boot项目。

这篇博客文章也为我提供了一个解释我为什么觉得这种语言有趣的机会，向您详细展示了一个示例项目并提供了一些技巧。

什么是科特林？

Kotlin是JetBrains创建的一种语言。它运行在JVM之上（但不仅限于此），它是一种面向对象的语言，其中包含许多函数式编程思想。我不会过多介绍Kotlin的所有功能（ PDF ， HTML ），但我想强调我发现最有趣的功能：

  • Kotlin是一种静态类型的语言，但是由于其巧妙的类型推断 ，它使您可以编写与动态语言一样短而具有表现力的代码，并且性能接近纯Java项目。
  • 属性支持
  • 与其他语言相比，相对轻量的标准库
  • 易于学习：Java开发人员可以快速理解大多数语言（这种与Java的快速比较值得一读）
  • Java互操作是一流的关注点，并且很棒
  • 非常适合Android开发
  • 内置的不变性和无效的安全支持
  • 代码易于阅读，编写效率高
  • 允许扩展现有库，而无需继承类或使用任何类型的设计模式（例如Decorator）
  • 不需要分号;-)

您可以在此Kotlin摘要2015博客文章中找到许多有用的链接来提高您对Kotlin的了解。还可以查看这些简单的Kotlin练习 ，以快速了解该语言。

Spring Boot + Kotlin项目样本

Kotlin的设计可以与Java生态系统很好地协作，在我看来，它与Spring Boot具有相同的务实，创新和自觉的思维方式，因此它们可以很好地协作。您可以看一下这个简单的Spring Boot + Spring Data JPA Kotlin项目，以更具体地了解其外观。

由于Kotlin声明类的语法非常简洁，因此可以轻松地编写（和读取）您的域模型。您可以看到Kotlin允许您指定参数默认值，并且在变量/参数名称之后声明类型：

@Entity
class Customer(
	var firstName: String = "",
	var lastName: String = "",
	@Id @GeneratedValue(strategy = GenerationType.AUTO)
	var id: Long = 0
)

您可以在下面看到的Spring MVC REST控制器正在使用构造函数级注入，并且Kotlin中的默认可见性是public因此无需指定它。当函数返回单个表达式时，花括号可以省略，并且在=符号。由于返回类型可以由编译器推断，因此效果更好。

@RestController
class CustomerController (val repository:CustomerRepository) {

	@GetMapping("/")
	fun findAll() = repository.findAll()

	@GetMapping("/{name}")
	fun findByLastName(@PathVariable name:String)
		= repository.findByLastName(name)
}

Spring Data存储库是不言自明的：

interface CustomerRepository : CrudRepository<Customer, Long> {
	fun findByLastName(name: String): List<Customer>
}

由于Kotlin支持顶级功能，因此您可以简单地声明您的应用程序：

@SpringBootApplication
class Application {

	@Bean
	fun init(repository: CustomerRepository) = CommandLineRunner {
		repository.save(Customer("Jack", "Bauer"))
		repository.save(Customer("Chloe", "O'Brian"))
		repository.save(Customer("Kim", "Bauer"))
		repository.save(Customer("David", "Palmer"))
		repository.save(Customer("Michelle", "Dessler"))
	}
}

fun main(args: Array<String>) {
	SpringApplication.run(Application::class.java, *args)
}

您需要使用kotlin-spring自动制作的插件 @Configuration类和其他一些@Service要么@Repository如open因为他们不可能final在春季由于CGLIB代理的使用（Kotlin中的类和方法是final默认情况下不带open修饰符）。使用JDK动态代理的Bean不需要open修饰符。

其他提示

即使Spring Boot和Kotlin可以很好地协同工作，这些额外的技巧也是值得知道的。有关更多详细信息，请参见有关在Spring Boot中改善Kotlin支持的问题 。

数组注释属性

与Java不同，Kotlin当前不允许将数组注释属性指定为单个值，除了value属性，因此请注意，您必须编写@RequestMapping(method = arrayOf(RequestMethod.GET))要么@EnableAutoConfiguration(exclude = arrayOf(Foo::class)) 。

预期在即将发布的Kotlin 1.2中将改善此行为（有关更多详细信息，请参阅此Kotlin问题 ）。Spring Framework 4.3组成的注释像@GetMapping要么@PostMapping也可以提供帮助。

资产注入

以前我们已经看过如何进行构造函数注入，因为这是推荐的方法 （尤其是Kotlin）。如果必须执行属性注入，则必须使用后期初始化的属性，因为通常，必须在构造函数中初始化声明为具有非null类型的原始属性。

@RestController
class CustomerController {

	@Autowired
	lateinit var repository:CustomerRepository

	// ...
}

财产占位符

$在Kotlin中用于字符串插值 ，因此在使用属性占位符时应转义它： @Value("\${some.property}") 。另外，您也可以使用@ConfigurationProperties相反，请参阅此堆栈溢出答案以获取更多详细信息。

杰克逊·科特林模块

如果您使用的是杰克逊，则可能需要添加com.fasterxml.jackson.module:jackson-module-kotlin为了使它能够处理没有默认构造函数或Kotlin集合的数据类。它将由Spring Framework 4.3+自动注册。

试用Java到Kotlin转换器

最后一个技巧是，IntelliJ IDEA中的Java到Kotlin转换器（菜单代码->将Java文件转换为Kotlin文件）在您不知道如何在Kotlin中编写东西时非常有用。因此，不要犹豫，用Java编写某些东西，然后用它来查找Kotlin副本。与Java文档的比较也可以提供一些帮助。
