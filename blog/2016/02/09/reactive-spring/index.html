Reactive Spring

工程
罗森·斯托扬切夫（Rossen Stoyanchev）
2016年2月9日

在2015年的SpringOne2GX上，Juergen 宣布了针对Spring Framework 5的计划 ，该计划主要致力于反应式架构。具体的努力已经在进行中，此后发生了很多事情！

在最基本的级别上，反应式编程模型允许编写非阻塞服务和应用程序。这就要求从写命令式逻辑到异步，非阻塞，功能样式代码的根本转变，尤其是在与外部资源交互时。

响应式Web应用程序

大多数Java Web应用程序都基于Servlet API构建，该API最初是使用同步和阻塞语义创建的。多年来，添加了对异步请求（Servlet 3.0）和非阻塞I / O（Servlet 3.1）的支持。在Spring MVC中，我们发现向现有应用程序添加选择性异步HTTP请求处理是可行的。但是，我们还发现，很难在现有的Web框架和应用程序生态系统中引入非阻塞I / O。这样做需要对核心契约进行非常深刻的改变，这些核心契约需要从阻塞语义转换为异步语义。

Spring MVC持续流行的原因之一是其基于灵活控制器方法签名的直观，基于注释的编程模型。幸运的是，它们可以继续充当反应式Web应用程序的基础。这是Spring Reactive努力的方向，您将找到一个与任何Spring MVC控制器相似但在新的React引擎上运行的TestController ，该引擎具有针对Tomcat，Jetty，Undertow和Netty的集成测试。请注意，该项目将在5月4.3版本发布后不久合并到Spring Framework master分支中，以准备6月发布5.0 M1。

反应性基金会

这项工作的关键要素是Reactive Streams规范 ，该规范的目的是提供“具有无阻塞背压的异步流处理的标准” 。该规范允许在不同的异步组件提供程序之间进行互操作：从HTTP服务器到Web框架，数据库驱动程序等。它将作为java.util.concurrent包含在JDK 9 中。流 。

该规范很小，由4个接口，一些规则和一个TCK组成。但是，要将其公开为API，需要围绕它的基础结构来构成异步逻辑。Spring Reactive使用Reactor Core ，这是一个小型且专注的库，它是要在Reactive Streams上构建的库和框架的基础。对于那些可能熟悉Project Reactor的人来说 ，在本周Reactor Core 2.5 M1发行之前的过去6-8个月中发生了很多事情， Stephane Maldini很快就会提供完整更新。

请继续关注来自不同Spring项目的关于响应式编程的更多博客文章。另外，在美丽的巴塞罗那今年的春季I / O会议上，期待有反应的嗡嗡声！
