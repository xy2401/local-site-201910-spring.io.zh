使用保管库管理数据库秘密

工程
马克·帕鲁奇
2016年8月15日

在上一篇有关使用Vault管理机密的文章中，我向您介绍了Vault以及如何使用通用机密后端存储任意机密。保险柜不仅可以管理秘密数据，例如API密钥，密码和其他类似字符串的敏感数据。今天，我们来看看Vault与数据库，服务和证书的集成。

数据库凭证通常是静态的

对于数据库，获取数据库申请证书的常规工作流程是要求某些操作员或自助服务工具为您提供证书，以便您的应用程序可以登录数据库。此时，凭据被认为是静态的。在迁移数据库或存在安全漏洞的情况下，通常会更改凭据。

使用Spring Cloud Vault，您可以将用户名和密码存储在Vault中，而不是您的应用程序配置中。Spring Config Server也是一个不错的选择。它允许您将加密的机密存储在配置存储库中。您的凭据受到保护。

有一个警告：长期存在的凭证是泄漏的良好目标。凭据泄漏可能会导致意外访问。一些数据库对源主机实施了限制。在某些情况下，数据库用户可以限制为一组主机。该限制阻止了其他主机的访问。尽管如此，每个有权访问允许的计算机的用户和进程都可以使用泄漏的凭据。但是，您如何发现泄漏？如果您的数据泄露给公众或互联网，则可能会发现泄露，但情况并非总是如此。对于其他情况，意外的人可能会读取或更改您的数据，并且可以肯定的是，泄漏会在相当长的一段时间内未被发现。

让我们让凭据短暂存在。

保管库和数据库

Vault附带了与不同系统的各种集成。其中一些与PostgreSQL和MySQL集成为秘密后端。秘密后端可以提供秘密。在这种情况下，秘密后端并不意味着秘密数据存储在PosgreSQL / MySQL中。这意味着Vault可以根据需要创建（和撤消）数据库用户。

要生成数据库凭证，您需要首先设置一个角色。角色控制用于数据库凭据生成的权限上下文。角色定义与凭证关联的权限。如果您运行不同的应用程序，并且每个应用程序需要不同的数据库权限，则该概念很好地适用。
角色还定义了获取凭证的最大租用时间。租约是在保管箱中说出凭据有效的持续时间。凭证过期后，保管箱将从数据库系统中撤消凭证。某些数据库（例如PostgreSQL）已内置了对密码到期的支持。 CREATE ROLE … VALID UNTIL …条款。

Vault和MySQL的第一步

现在如何开始使用Vault和MySQL？

您需要已初始化且未密封的Vault服务器和正在运行的MySQL服务器。请参阅上一篇文章如何设置保险柜。

然后，您可以根据连接详细信息和角色声明来设置MySQL后端：

首先，安装mysql与秘密后端

$ vault mount mysql
Successfully mounted 'mysql' at 'mysql'!

每个安装都与一个MySQL服务器相关。如果要为多个服务器生成凭据，则需要使用不同的路径多次安装后端。

后端安装后，您需要提供控制连接详细信息，其中包括用户名/密码组合以及主机和端口。控制连接用于创建和撤消用户。

$ vault write mysql/config/connection \
	connection_url="root:[email protected](127.0.0.1:3306)/"
Success! Data written to: mysql/config/connection

该角色包含用于用户创建和授权的脚本。如果需要，可以包含多个命令。保险柜将在每一代凭证上执行以下命令：

$ vault write mysql/roles/readonly \
	sql="CREATE USER '{{name}}'@'%' IDENTIFIED BY '{{password}}';GRANT SELECT ON *.* TO '{{name}}'@'%';"
Success! Data written to: mysql/roles/readonly

保险柜现已准备好为readonly角色。您可以通过以下方式获取凭据

$ vault read mysql/creds/readonly
Key            	Value
---            	-----
lease_id       	mysql/creds/readonly/2e7cd1d0-e313-158e-c9a4-1dd3c3277642
lease_duration 	2592000
lease_renewable	true
password       	04cf512a-57f8-d146-9cf5-ec2bd829ca8c
username       	token-10a8b69f-a

并使用mysql控制台应用程序：

$ mysql -h 127.0.0.1 -utoken-10a8b69f-a -p04cf512a-57f8-d146-9cf5-ec2bd829ca8c
Welcome to the MySQL monitor.  Commands end with ; or \g.
Your MySQL connection id is 5
...

现在我们知道Vault / MySQL集成正在运行，我们可以在带有Spring Cloud Vault的Spring Boot应用程序中使用它。

抓住一个Spring Boot项目。 start.spring.io是一个很好的起点。

包括Spring Cloud Vault Starter，数据库依赖项， spring-jdbc和项目中的MySQL驱动程序。
将以下代码添加到您的构建配置文件中。这些行包括所有必需的依赖项。

马文

<dependency>
	<groupId>org.springframework.cloud</groupId>
	<artifactId>spring-cloud-vault-starter-config</artifactId>
	<version>1.0.0.BUILD-SNAPSHOT</version>
</dependency>

<dependency>
	<groupId>org.springframework.cloud</groupId>
	<artifactId>spring-cloud-vault-config-databases</artifactId>
	<version>1.0.0.BUILD-SNAPSHOT</version>
</dependency>

<dependency>
	<groupId>org.springframework</groupId>
	<artifactId>spring-jdbc</artifactId>
</dependency>

<dependency>
	<groupId>mysql</groupId>
	<artifactId>mysql-connector-java</artifactId>
</dependency>

<repositories>
	<repository>
		<id>spring-snapshots</id>
		<name>Spring Snapshots</name>
		<url>https://repo.spring.io/libs-snapshot</url>
		<snapshots>
			<enabled>true</enabled>
		</snapshots>
	</repository>
</repositories>

摇篮

repositories {
	maven {
		url 'https://repo.spring.io/libs-snapshot'
	}
}

dependencies {
	compile("org.springframework.cloud:spring-cloud-vault-starter-config:1.0.0.BUILD-SNAPSHOT")
	compile("org.springframework.cloud:spring-cloud-vault-config-databases:1.0.0.BUILD-SNAPSHOT")
	compile("org.springframework:spring-jdbc:4.3.1.RELEASE")
	compile("mysql:mysql-connector-java:5.1.39")
}

设置配置

Spring Cloud Vault默认使用通用机密后端。在我们的情况下，我们不需要通用后端。所以我们需要禁用generic后端并启用MySQL后端。所有配置都需要在引导程序配置中指定。在本例中，我们使用bootstrap.yml在src/main/resources ：

spring.cloud.vault:
		token: 9a63de21-8af7-311a-9a5a-151b6a0d4795
		scheme: http
		generic:
			enabled: false
		mysql:
			enabled: true
			role: readonly

spring.datasource.url: jdbc:mysql://127.0.0.1:3306

我们不向配置文件添加任何数据库凭据。这些通常是spring.datasource.username和spring.datasource.password 。

的spring.cloud.vault.scheme被设定为http因为我们已经以纯文本HTTP模式启动了Vault（ spring.cloud.vault.scheme默认为https ）。不要在生产中这样做。纯文本使整个秘密故事毫无用处，因为网络上的所有听众都可以看到您的秘密
请注意，示例中使用的令牌是根令牌。

您可以使用以下方法创建新令牌：

$ vault token-create
Key            	Value
---            	-----
token          	728d26ae-53a6-d8b6-d7a0-c5f62238ea55
token_accessor 	2fd7dcba-39d0-04d3-8d6b-096c3529cf14
token_duration 	0
token_renewable	true
token_policies 	[root]

显然，这是您将Vault MySQL与Spring Boot应用程序集成所需要做的全部工作。

让我们快速向您的应用程序中添加一些代码，以便您可以使用真实代码测试集成。


@SpringBootApplication
public class MySqlApplication {
	public static void main(String[] args) {
		SpringApplication.run(MySqlApplication.class, args);
	}

	@Autowired
	DataSource dataSource;

	@PostConstruct
	private void postConstruct() throws Exception {

		try (Connection connection = dataSource.getConnection();
				Statement statement = connection.createStatement()) {
				
			ResultSet resultSet = statement.executeQuery("SELECT CURRENT_USER();");
			resultSet.next();

			System.out.println("Connection works with User: " + resultSet.getString(1));

			resultSet.close();
		}
	}

然后启动您的应用程序。该代码产生如下输出：

Connection works with User: token-...

Spring Cloud Vault在启动时获取数据库凭据，并使用默认属性名称存储这些凭据，以便Spring Boot可以选择它们。

其他服务

保险柜支持其他几种集成。我们已经建立了对Spring Boot中也支持的集成的支持。

您可以使用以下服务和数据库：

  • PostgreSQL和MySQL-生成用于JDBC和Spring Data JPA的凭证
  • Apache Cassandra-生成用于Datastax客户端和Spring Data Cassandra的凭据
  • Hashicorp Consul-生成可与Spring Cloud Consul一起使用的ACL令牌
  • RabbitMQ-生成与Spring AMQP一起使用的凭证
  • Amazon AWS-生成要与Spring Cloud AWS一起使用的访问密钥和秘密密钥

您可以在https://github.com/mp911de/spring-cloud-vault-config-samples的示例存储库中查看完整的示例。
