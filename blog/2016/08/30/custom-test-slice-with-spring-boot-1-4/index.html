使用Spring Boot 1.4的自定义测试片

工程
斯特凡·尼科尔（StéphaneNicoll）
2016年8月30日

Spring Boot 1.4包括对测试支持的重大改进，这些功能之一就是测试切片 。我想借此机会在此博客文章中进一步解释它是什么以及如何轻松创建自己的切片。

测试切片是关于细分ApplicationContext为您的测试创建的。通常，如果要使用以下命令测试控制器MockMvc ，当然您不想打扰数据层。相反，您可能想模拟控制器使用的服务，并验证所有与Web相关的交互是否按预期工作。可以在下面的示例中进行总结：

@RunWith(SpringRunner.class)
@WebMvcTest(UserVehicleController.class)
public class UserVehicleControllerTests {

    @Autowired
    private MockMvc mvc;

    @MockBean
    private UserVehicleService userVehicleService;

    @Test
    public void testExample() throws Exception {
        given(this.userVehicleService.getVehicleDetails("sboot"))
                .willReturn(new VehicleDetails("Honda", "Civic"));
        this.mvc.perform(get("/sboot/vehicle").accept(MediaType.TEXT_PLAIN))
                .andExpect(status().isOk()).andExpect(content().string("Honda Civic"));
    }

}

@WebMvcTest是Spring Boot 1.4中的Web测试片。如果存在，则指示Spring Boot需要一个Web环境，并且仅应实例化指定的控制器。由于它了解测试的性质，因此可以为您做出其他明智的决定（例如，自动配置MockMvc这样剩下的就是注入它）。另外，您的控制器对UserVehicleService因此启动上下文会导致失败，因为ApplicationContext不知道这一点（请记住，只有Web基础架构和UserVehicleController是已知的）。 @MockBean用于这里注册UserVehicleService模拟，以便可以将其透明地注入到控制器中。

现在，让我们看一下实现以更好地了解Spring Boot如何为您管理此实现。我们的第一站是@WebMvcTest （删除@Target和朋友以示敬意）：

@BootstrapWith(WebMvcTestContextBootstrapper.class)
@OverrideAutoConfiguration(enabled = false)
@TypeExcludeFilters(WebMvcTypeExcludeFilter.class)
@AutoConfigureCache
@AutoConfigureWebMvc
@AutoConfigureMockMvc
@ImportAutoConfiguration
public @interface WebMvcTest { ... }

该声明可以分为三个区域：

  • 自动配置自定义
  • 类路径扫描调整
  • 测试引导

自动配置自定义

Spring Boot 1.4现在定义了一个spring-boot-test-autoconfigure提供与测试相关的自动配置的集合的模块。这些自动配置是可组合的，可以帮助您轻松构建自己的基础架构。

回到@WebMvcTest ，我们要做的第一件事是禁用默认的自动配置： OverrideAutoConfiguration做到这一点。由于默认自动配置现已禁用，因此您必须选择加入要包括的相关自动配置。他们三个AutoConfigure批注为我们做到了：它们确保网络环境可用， MockMvc已配置，并且无操作缓存管理器可用。让我们来看看摘录AutoconfigureMockMvc ：

@ImportAutoConfiguration
@PropertyMapping("spring.test.mockmvc")
public @interface AutoConfigureMockMvc {

    boolean addFilters() default true;

    @PropertyMapping("webclient.enabled")
    boolean webClientEnabled() default true;

    ...
}

@ImportAutoConfiguration是一个注释，列出了应包括的自动配置。另外，您可以在META-INF/spring.factories使用键的注释的完全限定名称。这是为AutoConfigureMockMvc ：

org.springframework.boot.test.autoconfigure.web.servlet.AutoConfigureMockMvc=\
org.s.boot.test.autoconfigure.web.servlet.MockMvcAutoConfiguration,\
org.s.boot.test.autoconfigure.web.servlet.MockMvcSecurityAutoConfiguration,\
org.s.boot.test.autoconfigure.web.servlet.MockMvcWebClientAutoConfiguration,\
org.s.boot.test.autoconfigure.web.servlet.MockMvcWebDriverAutoConfiguration

您会明白：每个注释都带来了一些自动配置，您可以根据需要任意组合。您会注意到WebMvcTest也有一个ImportAutoConfiguration但没有条目spring.factories为了它。Spring Boot将扫描所有spring.factories在类路径中，并在必要时合并内容。如果您的模块要添加其他行为， @WebMvcTest （要么@AutoConfigureMockMvc ），只需创建一个META-INF/spring.factories并注册其他自动配置类。您也可以使用@AutoconfigureBefore和@AutoconfigureAfter订购它们。

测试自动配置可以像往常一样配置： @PropertyMapping类级别的注释将注释的属性映射到Environment以便自动配置代码可以提取值并相应地调整配置。我们可以看到webClientEnabled上面的属性在自动配置中透明使用：

@ConditionalOnProperty(prefix = "spring.test.mockmvc.webclient", 
        name = "enabled", matchIfMissing = true)
public class MockMvcWebClientAutoConfiguration { ... }

类路径扫描调整

TypeExcludeFilters是一种调整类路径扫描的方法。如果是@WebMvcTest我们将仅包含某些与网络相关的组件，而忽略其余所有组件 。这是非常强大的功能，因为您可以按常规方式使用类路径扫描，并且仅包含切片所需的内容。

测试引导

最后，新的测试引导程序可确保确定@SpringBootApplication项目中带注释的类（除非指定了一个）。这是一个不错的默认值，因为您不再需要指定它，默认情况下 classpath扫描是正确的 。

创建自己的切片

基于这些知识，创建自己的切片实际上非常简单。这种切片的一个例子可能是新的DataJdbcTest ，类似于DataJpaTest只配置JdbcTemplate并且不使用JPA。如果您想正确地使用代码，请查看github存储库以获取更多详细信息。

我们的第一步是创建@AutoconfigureDataJdbc

package com.example.test.autoconfigure.jdbc;

import java.lang.annotation.*;
import org.springframework.boot.autoconfigure.ImportAutoConfiguration;

@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
@Documented
@ImportAutoConfiguration
public @interface AutoconfigureDataJdbc {
}

并注册存在此注释时要应用的相关自动配置。再次，创建一个META-INF/spring.factories资源：

com.example.test.autoconfigure.jdbc.AutoconfigureDataJdbc=\
org.s.boot.autoconfigure.flyway.FlywayAutoConfiguration,\
org.s.boot.autoconfigure.jdbc.DataSourceAutoConfiguration,\
org.s.boot.autoconfigure.jdbc.DataSourceTransactionManagerAutoConfiguration,\
org.s.boot.autoconfigure.jdbc.JdbcTemplateAutoConfiguration,\
org.s.boot.autoconfigure.liquibase.LiquibaseAutoConfiguration,\
org.s.boot.autoconfigure.transaction.TransactionAutoConfiguration

一旦建立了可重用的基础架构，就可以创建测试切片，并只需指定需要数据库和jdbc即可：

@BootstrapWith(SpringBootTestContextBootstrapper.class)
@OverrideAutoConfiguration(enabled = false)
@TypeExcludeFilters(DataJdbcTypeExcludeFilter.class)
@Transactional
@AutoConfigureCache
@AutoconfigureDataJdbc
@AutoConfigureTestDatabase
@ImportAutoConfiguration
public @interface DataJdbcTest { }

DataJdbcTypeExcludeFilter确保排除所有其他服务，因为这种测试默认情况下不需要您的任何bean。可以改进以允许将服务定义为注释的参数，就像WebMvcTest添加指定的控制器。

完成此操作后，您只需添加注释和JdbcTemplate已使用测试数据库自动为您配置：

@RunWith(SpringRunner.class)
@DataJdbcTest
public class DataJdbcSampleTests {

    @Autowired
    private JdbcTemplate jdbcTemplate;

    ...
}

摘要

Spring Boot 1.4为测试带来了自动配置，并允许您轻松地编写自己的测试注释。在本文中，我们看到了WebMvcTest工作原理，以及如何创建自己的“ jdbc”切片。我们实际上正在考虑在下一个版本中添加该批注，因此请随时提供反馈 ！
