Kubernetes 1.0 GA的Spring Cloud Data Flow发布

发布
托马斯·里斯伯格
2016年7月12日

我们很高兴地宣布发布Spring Cloud Data Flow的Kubernetes 1.0.0。发布。

用于Kubernetes的Spring Cloud Data Flow提供了对在Kubernetes上协调长期运行的（流）和短期的（任务/批处理）数据微服务的支持。

该项目最初是由社区概念化的，我们感谢Florian Rosenberg的早期贡献，最终将其纳入了Kubernetes的正式Spring Cloud Deployer项目。在这个主题的基础上，我们最近碰到了Donovan Muller的博客 ，他在他的经验中扩展了为OpenShift扩展Spring Cloud Deployer项目的经验，为Spring Cloud Data Flow在OpenShift上编排数据微服务铺平了道路。

除了API清理和稳定之外，我们还在此主要版本中研究了以下主题。

  • 以1.0.0为基础。发行Spring Cloud Data Flow Core （shell，UI，REST-API等）
  • 添加功能以支持流分区和缩放：
    • 当前，接收器和处理器的分区和缩放是通过使用多个复制控制器来处理的，每个应用程序实例一个。

    • 通过将Pod副本与单个复制控制器一起使用来处理源的缩放。这使我们能够利用平台负载平衡功能来实现与应用程序的HTTP连接。
  • 使用Google Container Engine针对Kubernetes集群进行流和任务处理的集成和验收测试。使用在Google Cloud Dataproc上配置的Hadoop集群进行其他流测试
  • 将迁移指南和其他文档片段添加到参考文档中
  • 添加项目站点

要开始使用针对Kubernetes的Spring Cloud Data Flow，请遵循参考文档中概述的以下步骤 。

有关更改和改进的完整列表，请参阅1.0.0。版本的问题 。
