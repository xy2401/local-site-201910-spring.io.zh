Spring Boot 1.4发布

发布
菲尔·韦伯
2016年7月28日

我很荣幸地宣布，Spring Boot团队的一个代表以及所有做出贡献的人都已经发布了Spring Boot 1.4.0，现在可以从repo.spring.io ， Maven Central和Bintray获得 。该发行版增加了许多新功能和改进，并以Spring Framework的最新发行版为基础。有关完整的升级说明和“新功能和值得注意的功能”，请参阅发行说明 。

以下是v1.4的一些亮点：

启动失败消息

Spring Boot现在将执行常见启动故障的分析，并提供有用的诊断信息，而不是简单地记录堆栈跟踪。例如，如果发生端口冲突，现在将看到以下消息：

***************************
APPLICATION FAILED TO START
***************************

Description:

Embedded servlet container failed to start. Port 8080 was already in use.

Action:

Identify and stop the process that's listening on port 8080 or configure this application to listen on another port.

基于约定的错误页面

现在，可以通过遵循基于约定的方法来创建给定状态码的自定义错误页面。只需在正确的位置添加静态HTML或模板即可创建映射。例如，要注册自定义404页面，您可以添加src/main/resource/public/error/404.html

扩展数据支持

Spring Boot现在附带对Neo4J ， Couchbase和Redis Spring Data存储库的支持 。此外， Hibernate 5.0现在是默认的JPA提供程序。我们还更新了Elasticsearch集成以支持Jest客户端 。

横幅图片

现在，您可以使用图像文件来呈现ASCII艺术标语。放一个banner.gif ， banner.jpg要么banner.png归档到src/main/resources使其自动转换为ASCII：

测试改进

Spring Boot 1.4包括对测试支持的重大改进。现在以专用方式提供测试类和实用程序spring-boot-test和spring-boot-test-autoconfigure罐子。我们已将AssertJ ， JSONassert和JsonPath依赖项添加到测试启动器，并提供了更简单的统一@SpringBootTest注释，用于Spring的JUnit运行器。

现在，我们还可以自动配置许多测试，这意味着大多数用户将需要较少的测试配置。我们还介绍了专用@JsonTest ， @WebMvcTest ， @RestClientTest和@DataJpaTest批注，使您可以快速测试应用程序的“切片”。

最后，还有全面的Mockto集成。您现在可以轻松地“模拟”或“间谍” Spring Bean：

@RunWith(SpringRunner.class)
@SpringBootTest
public class MyTest {

    @MockBean
    private RemoteService remoteService;

    @Autowired
    private Reverser reverser;

    @Test
    public void exampleTest() {
        // RemoteService has been injected into the reverser bean
        given(this.remoteService.someCall()).willReturn("mock");
        String reverse = reverser.reverseSomeCall();
        assertThat(reverse).isEqualTo("kcom");
    }

}

执行器

执行器/info端点已得到改进，因此您可以轻松贡献其他项目。我们现在支持开箱即用：

  • 从Maven或Gradle插件生成的全部或部分Git信息。
  • 从Spring Boot Maven或Gradle插件生成的构建信息。
  • 来自环境的自定义信息（任何属性开头info.* ）

的/metrics端点也进行了改进，以便可以“合并”和/或“分组”提交。

其他变化

发行说明中还记录了许多其他更改和改进。您还可以找到我们计划在下一版本中删除的不推荐使用的类和方法的列表。
