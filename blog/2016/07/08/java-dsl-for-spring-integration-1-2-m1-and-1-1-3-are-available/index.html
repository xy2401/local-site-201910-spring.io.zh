提供了用于Spring Integration 1.2 M1和1.1.3的Java DSL

发布
阿尔特姆·比兰（Artem Bilan）
2016年7月8日

我很高兴地宣布，用于春季集成的Java DSL 1.2 M1现在可用！

首先，我要感谢所有在StackOverflow上创建问题，提出拉取请求，提供反馈或只是问问题的人。没有社区，我们就不可能成为今天的成功项目！

神器org.springframework.integration:spring-integration-java-dsl:1.2.0.M1可在Milestone仓库中使用 。因此，请尝试一下，不要犹豫，提出GH问题以获取任何反馈！

当前迭代的一些亮点：

Apache Kafka 0.9支持

基于最近发布的Spring for Apache Kafka项目，一个新的Kafka09已经介绍了工厂。这是直接用于生产的Java DSL工厂IntegrationComponentSpec基于Spring for Apache Kafka基础的流利的适配器API构建器。现有的Kafka Apache Kafka的工厂0.8仍然存在向后兼容性的支持。

运行时流注册

在许多情况下，最好根据系统状态甚至在某些业务功能调用期间指定集成流程。甚至在执行后使其变得不稳定。为此， IntegrationFlowContext组件已被引入以使手册IntegrationFlow控制。

@Autowired
private IntegrationFlowContext context;
...

IntegrationFlow myFlow = f -> f
  			.<String, String>transform(String::toUpperCase)
   			.transform("Hello, "::concat);

String flowId = this.context.register(myFlow);
MessagingTemplate messagingTemplate = this.context.messagingTemplateFor(flowId);

assertEquals("Hello, SPRING",
            messagingTemplate.convertSendAndReceive("spring", String.class));

this.context.remove(flowId);

看到IntegrationFlowContext有关更多信息，请参阅JavaDocs。

操作员日志（）

为了方便记录消息旅程的Spring Integration方式（a-la ），新log()已引入运算符。在其下方仅由WireTapChannelInterceptor和LoggingHandler作为订户。它负责记录进入下一个端点的消息：

.filter(...)
.log(LoggingHandler.Level.ERROR, "test.category", m -> m.getHeaders().getId())
.route(...)

在这个例子中id标头将被记录ERROR仅针对通过过滤器且在路由之前的邮件，将其级别设置为“ test.category”。

MessageChannelSpec.wireTap（）

一种.wireTap()引入了流畅的API MessageChannelSpec建设者。现在，目标配置将从Java DSL使用中获得更多收益：

@Bean
public QueueChannelSpec myChannel() {
    return MessageChannels
            .queue()
            .wireTap("loggingFlow.input");
}

@Bean
public IntegrationFlow loggingFlow() {
    return f -> f.log();
}

键入路由器

有了纯Java实例化的bean，它看起来真的很有机，可以将路由键设置为任何所需的类型，并避免转换为String一切。加上轻松的类型，我们可以将其配置为可识别通用的：

@Bean
public IntegrationFlow payloadTypeRouteFlow() {
    return f -> f
          .<Object, Class>route(Object::getClass, m -> m
                         .channelMapping(String.class, "stringsChannel")
                         .channelMapping(Integer.class, "integersChannel"));
}

查看提交历史 1.2.0.M1版本以获取更多信息。

Bug修复

维护版本1.1.3该版本还发布了一些讨厌的错误修复程序，并具有Spring Integration 4.3兼容性。的org.springframework.integration:spring-integration-java-dsl:1.1.3.RELEASE该工件可以在repo.spring.io和Maven Central中找到 。建议所有人升级，尤其是如果您使用JMS工厂和实施的豆类ApplicationListener 。

下一步

我们期待下一个里程碑2 1.2很快，在几个星期内，有了RC并在8月中旬发布。同时我们要吸收spring-integration-java-dsl版本的项目集成到Spring Integration Core中 5.0和Java 8代码库。目前1.2仍将支持该版本，但仅用于错误修复。
