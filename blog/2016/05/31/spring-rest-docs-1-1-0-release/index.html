Spring REST Docs 1.1.0。发布

发布
安迪·威尔金森
2016年5月31日

我很高兴宣布发布Spring REST Docs 1.1.0。RELEASE现在可用于Maven Central，JCenter和我们的版本库 。

强调

放心

作为Spring Framework的轻量级无服务器文档生成的替代方法MockMvc ，您现在可以使用REST Asured来测试和记录RESTful服务。这将Spring REST Docs打开到JVM的所有四个角以及其他地方，使您可以记录可通过HTTP访问的任何内容。

现在，这些示例包括一个使用REST Docs记录，使用Grails构建并使用Spock测试的应用程序 。还提供了使用REST Docs记录使用Ratpack实现的API的第三方示例。感谢Jenn Strater编写了两个样本。

降价促销

添加了对生成Markdown代码片段的支持。Markdown本身不具备Asciidoctor的功能，但是与现有的文档工具链（例如Slate）结合使用时，效果很好。

测试NG

除了对使用JUnit的现有支持之外，您现在还可以将Spring REST Docs与TestNG一起使用 。

可重复使用的代码段

现在，可以使用某些通用配置创建代码片段一次，然后重新使用 。这样可以减少在API上记录公共部分时的重复，例如self链接。感谢Marcel Overdijk的反馈和请求请求，大大改进了此新功能。

轻松的摘要

默认情况下，如果您忽略了文档或尝试文档不存在的内容，Spring REST Docs将导致测试失败。这对于详细的API文档非常有用，但是在记录您要关注请求或响应的特定部分的方案时可能会受到限制。现在，您不必使用显式片段，即可轻松记录链接，请求和响应字段以及请求和路径参数，而不必显式忽略您不想记录的所有内容。

在Spring Boot 1.3中使用Spring REST Docs 1.1

默认情况下，Spring Boot 1.3使用Spring REST Docs 1.0。要使用1.1，您应该在pom.xml或build.gradle中覆盖在Boot的依赖管理中配置的版本：

pom.xml：

<properties>
    <spring-restdocs.version>1.1.0.RELEASE</spring-restdocs.version>
</properties>

build.gradle：

ext['spring-restdocs.version']=1.1.0.RELEASE

谢谢

感谢所有为Spring REST Docs做出贡献并帮助塑造1.1版本的人。我期待您在GitHub ， Stack Overflow和Twitter 上的反馈。
