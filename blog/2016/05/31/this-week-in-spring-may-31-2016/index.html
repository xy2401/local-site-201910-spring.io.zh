Spring 的这个星期-2016年5月31日

工程
乔什·朗（Josh Long）
2016年5月31日

欢迎来到 Spring 本周的另一期！上周有点疯狂，尽管经过一个漫长的宁静周末之后，有很多有趣的内容可以使我们为接下来的一周充满活力。我将在6月16日的聚会上发表讲话，致敬阿姆斯特丹附近的人们，他们很乐意加入我们！和往常一样，我们有很多内容需要介绍，因此让我们开始吧。

  • 请记住，微服务的目标最终是提高速度并投入生产。做连续的交付。其中很大一部分必须是自动将代码从集成和测试迁移到生产中，并且理想情况下必须没有停机。我会推荐Spring Cloud团队成员Marcin Grzejszczak撰写的这个出色的博客， 介绍使用数据库的零延迟部署 。这是极好的资源！
  • 说到micrservices和Spring，Santosh Singh在DZone上做了一个简洁的介绍“ Spring Boot way”（还有其他方法吗？：D）的微服务 。
  • 想更多地了解Kotlin和Spring？想看看为什么大型组织（例如Netflix）将Kotlin与Spring Cloud一起使用 ？查看我们的 Spring 团队成员SébastienDeleuze's spring-kotlin该项目介绍了一些提示和解决方法，以及有用的扩展。
  • Spring RESTDocs负责人Andy Wilkinson刚刚宣布了Spring REST Docs 1.1.0 RELEASE 。该发行版包含很多很棒的东西，包括REST保证集成，Markdown支持，TestNG支持，“可重用”和“轻松”代码段，以及来自Spring REST Docs团队和社区的更多内容。取得成功并赢。
  • Spring Data忍者和Spring Stateflow的负责人Janne Valkealahti刚刚宣布了Spring Statemachine 1.1.0。可以发布
  • Spring REST Docs的负责人Andy Wilkinson并非偶然， 还 刚刚发布了Spring REST Docs 1.0.2，其中包括许多维护更新
  • 我真的在研究Jenkins 2.0，它使连续交付管道的概念成为中心。自然，这个新的Blue Ocean主题重新审视了界面约定，并使事情变得更好！
  • 关于如何在JHipster 3中使用Spring Cloud位的有趣博客
  • 建立边缘服务中的关键组件是路由。在Cloud Foundry中，您有许多选择，可以使用客户端负载平衡或构建更智能的路由试探法和边缘，并使用Cloud Foundry路由服务来处理路由
  • 我通过Spring Cloud挖掘了有关微服务的法语演讲
