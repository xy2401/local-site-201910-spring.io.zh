Spring Cloud Brixton Release Train has reached General Availability. The release is available today and can be found in our Spring Release repository and Maven Central.

We would like to thank many community contributors and testers for their help. There have been many pull requests, issues and conversations that have made Spring Cloud Brixton better.

Highlights

Some of the highlights of the Brixton Release Train are:

  • Spring Boot 1.3.x and Spring Framework 4.2.x support
  • Hashicorp Consul support for service registration/discovery & configuration via Spring Cloud Consul
  • Apache Zookeeper support for service registration/discovery, configuration via Spring Cloud Zookeeper and leader election in Spring Cloud Cluster
  • Distributed tracing through the Spring Cloud Sleuth abstraction with two out of the box implementations: one supporting logging (ideal for log collectors and multiplexers like Logstash and Loggregator) and one supporting Twitter’s Zipkin
  • Netflix Atlas Telemetry System, the next generation Spectator Metrics library and recent versions of Eureka, Ribbon, Hystrix and Feign are available in Spring Cloud Netflix
  • Spring Cloud Bus is now powered by the recently released Spring Cloud Stream
  • Cluster Leadership election and locks via Spring Cloud Cluster
  • Export of Spring Boot metrics to Amazon Cloudwatch, and native support for Amazon RDS

春天的云彩布里克斯顿。可以发布

发布
斯宾塞·吉布
2016年5月11日

我很高兴地代表团队宣布Spring Cloud Brixton Release Train已达到通用。该版本今天就可用，可以在我们的Spring Release存储库和Maven Central中找到 。

我们要感谢许多社区贡献者和测试人员的帮助。有许多请求请求，问题和对话使Spring Cloud Brixton变得更好。

强调

布里克斯顿放行列车的一些亮点是：

  • Spring Boot 1.3.x和Spring Framework 4.2.x支持
  • Hashicorp Consul通过Spring Cloud Consul支持服务注册/发现和配置
  • Apache Zookeeper支持服务注册/发现，通过Spring Cloud Zookeeper进行配置以及Spring Cloud Cluster中的领导者选举
  • 通过Spring Cloud Sleuth抽象进行分布式跟踪，其中有两种现成的实现方式：一种支持日志记录（对于日志收集器和Logstash和Loggregator等多路复用器来说是理想的），另一种支持Twitter的Zipkin
  • Netflix Atlas遥测系统 ，下一代Spectator Metrics库以及Eureka，Ribbon，Hystrix和Feign的最新版本在Spring Cloud Netflix中可用
  • Spring Cloud Bus现在由最近发布的Spring Cloud Stream支持
  • 通过Spring Cloud Cluster进行集群领导者选举和锁定
  • 将Spring Boot指标导出到Amazon Cloudwatch，以及对Amazon RDS的本地支持

以下模块是Brixton的一部分。发布：

  • Spring Cloud AWS 1.1.0。发布
  • Spring Cloud Bus 1.1.0。发布
  • Spring Cloud Cloud Foundry 1.0.0。发布
  • Spring Cloud Cluster 1.0.0。发布
  • Spring Cloud Commons 1.1.0。发布
  • Spring Cloud Config 1.1.0。发布
  • Spring Cloud Consul 1.0.0。发布
  • Spring Cloud Netflix 1.1.0。发布
  • Spring Cloud Security 1.1.0。发布
  • Spring Cloud Sleuth 1.0.0。发布
  • Spring Cloud Stream 1.0.0。发布
  • Spring Cloud Zookeeper 1.0.0。发布

而且，一如既往，我们欢迎您提供反馈：在GitHub ， Stack Overflow或Twitter上 。

笔记：

默认情况下不再创建@LoadBalanced RestTemplate 。有关详细信息，请参见更新的文档 。您需要在应用程序的配置中创建它。例如：

@Configuration
public class MyConfiguration {

    @LoadBalanced
    @Bean
    RestTemplate restTemplate() {
        return new RestTemplate();
    }
}

请注意，要使用的正确BOM表是spring-cloud-dependencies而不是spring-cloud-starter-parent （请参阅下面的入门）。

使用AMQP迁移Spring Cloud Hystrix，涡轮机和总线

过去在Spring AMQP之上实现的Bus，Hystrix和Turbine支持已全部迁移为使用Spring Cloud Stream。旧的工件仍然存在，但已过时，并将在某些时候将其删除。除了应使用spring-cloud-*-amqp工件外，还应使用所需的任何原始功能，以及您选择的流绑定器，例如spring-cloud-netflix-hystrix-stream和spring-cloud-starter-stream-rabbit spring-cloud-netflix-hystrix-amqp 。

天使 布里克斯顿（与AMQP） 布里克斯顿（与卡夫卡）
弹簧云启动总线amqp 弹簧云启动总线amqp Spring-Cloud-Starter-Bus-kafka
弹簧云netflix-hystrix-amqp spring-cloud-netflix-hystrix-stream spring-cloud-starter-stream-rabbit Spring-cloud-netflix-hystrix-stream spring-cloud-starter-stream-kafka
弹簧云启动涡轮amqp 弹簧云启动器涡轮流弹簧云启动器流兔子 Spring-cloud-starter-turbine-stream spring-cloud-starter-turbine-stream卡夫卡

注意：Brixton版本中仍然有spring-cloud-netflix-hystrix-amqp ，但是（错误地）从BOM中省略了它，因此如果您要升级并且不想升级，则需要明确提供一个版本。更改工件。

入门

Brixton入门的最简单方法。RELEASE将前往http://start.spring.io 。如果要使用带有BOM的Maven手动编写构建文件（仅依赖管理）：



  
    org.springframework.cloud
    spring-cloud-dependencies
    Brixton.RELEASE
    pom
    import
  


  
    org.springframework.cloud
    spring-cloud-starter-config
  
  
    org.springframework.cloud
    spring-cloud-starter-eureka
  
  ...

或使用Gradle：

buildscript {
  dependencies {
    classpath("org.springframework.boot:spring-boot-gradle-plugin:1.3.5.RELEASE")
  }
}
repositories {
  maven {
    mavenCentral()
  }
}
apply plugin: 'spring-boot'

dependencyManagement {
  imports {
    mavenBom 'org.springframework.cloud:spring-cloud-dependencies:Brixton.RELEASE'
  }
}

dependencies {
    compile 'org.springframework.cloud:spring-cloud-starter-config'
    compile 'org.springframework.cloud:spring-cloud-starter-eureka'
    //...
}
