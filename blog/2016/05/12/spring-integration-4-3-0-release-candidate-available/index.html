Spring Integration 4.3.0版本候选可用

发布
加里·罗素
2016年5月12日

我很高兴地宣布Spring Integration 4.3.0的可用性。RC1-第一个（也是最后一个）发布候选版本；它可以从Spring里程碑仓库中获得 。GA版本应该在Spring Framework 4.3的GA版本发布之后不久。

4.2.6。RELEASE也可以在春季发行版以及Maven Central中使用。此版本包括一些重要的错误修复 ，建议所有用户升级到该版本。

4.3只是次要发行版，具有一些新功能和对4.2的改进，并且将关闭4.x版本。同时，我们期待在Spring Integration 5.0中加入JVM的Reactive Foundation 。我们尚未确定确切的含义，所以请继续关注！此外，5.0（2017）将包含（当前单独的） Spring Integration Java DSL 。

以下是该版本的一些亮点（其中一些是先前在较早的里程碑中宣布的）：

先前宣布：

  • 一种negate的选项HeaderMapper允许使用逻辑NOT排除请求和/或回复标头! 。 （例如!foo,* -映射所有标头，除了foo ）。

  • Exception的超类映射ErrorMessageExceptionTypeRouter避免对不同继承者进行映射重复。

  • null远程目录支持list()和listNames()列出当前工作目录时，FTP适配器中的SVM功能符合标准的FTP协议支持。

  • 一个选项，用于在Redis队列适配器中更改Redis列表推送/弹出操作的方向。

  • FileWritingMessageHandler文件名中的子路径支持，以允许在本地（例如在解压缩后）恢复目录结构。

  • 升级到Smack-4.1.x。

  • 升级到Spring Amqp 1.6，并支持其新功能，例如延迟消息交换。

  • 现在可以将Spring Integration运行时对象模型与Component Metrics一起公开为graph ，可用于可视化集成应用程序的当前状态。在网络容器中运行时， @EnableIntegrationGraphController注释，以及IntegrationGraphServer bean，创建服务以通过REST协议检索模型和状态，例如JSON表示法：

    {
  "nodes": [
{
  "nodeId": 1,
  "name": "nullChannel",
  "componentType": "channel",
},
{
  "nodeId": 2,
  "name": "errorChannel",
  "componentType": "publish-subscribe-channel",
},
{
  "nodeId": 3,
  "name": "_org.springframework.integration.errorLogger",
  "componentType": "logging-channel-adapter",
  "output": null,
  "input": "errorChannel"
}
  ],
  "links": [
{
  "from": 2,
  "to": 3
}
  ]
}

    沿用Spring Integration的传统，消息通道在此模型中被表示为头等公民（节点），而不仅仅是节点之间的链接。另外，启用统计信息后（ @EnableIntegrationManagement要么 ），对象图中的每个节点都包含这些统计信息（消息计数，响应时间等）。（自从上一个里程碑宣布该图形以来，已经对该图形进行了一些改进，包括：支持链-处理程序列表，

  • 持久的MessageStore现在支持Lazy-Load的算法（默认启用） MessageGroup恢复。当处理具有聚合等操作的大型持久性组时，这会带来显着的性能优势。

  • 服务激活器现在支持async选项。如果服务返回ListenableFuture和async是true ，调用线程将立即释放，并在该线程上（从您的服务中）发送答复消息，以完成将来的任务。在此基础上，我们还提供了AsyncAmqpOutboundGateway和async模式JmsOutboundGateway下游流在回复侦听器容器线程上运行。

  • XMPP适配器现在支持Extensions （XEP）。因此，例如，您可以更轻松地与Google Cloud Messaging （GCM）进行交互：

<int-xmpp:inbound-channel-adapter id="xmppInboundAdapter"
    payload-expression="getExtension('google:mobile:data').json"/>
...
<bean id="gcmExtensionProvider" class="org.jivesoftware.smackx.gcm.provider.GcmExtensionProvider"/>

<int-xmpp:outbound-channel-adapter id="xmppOutboundAdapter"
    extension-provider="gcmExtensionProvider"/>

  • Groovy脚本现在可以使用compile-static要么compiler-configuration以获得改进的性能或为目标AST提供一些特殊的自定义。

  • 的WatchServiceDirectoryScanner已经进行了一些改进。其逻辑已移至FileReadingMessageSource直接通过use-watch-service旗。另外，我们可以指定WatchEventType s在目录树上侦听。还有StandardWatchEventKinds.ENTRY_DELETE事件被视为ResettableFileListFilter.remove()操作，例如从AcceptOnceFilteListFilter 。

  • 通道延迟绑定的其他几种情况已经实现。例如在WireTap ，导致
    MessageChannelSpec.wireTap() Spring Integration Java DSL中的特性。

此RC的新功能：

  • FTP / SFTP入站通道适配器现在可以支持文件内容的直接流传输，而无需复制到本地文件系统。

  • 现在可以将TCP解串器配置为使用缓冲池。

  • 各种次要修复和改进。

该发行版解决了超过125个JIRA问题 。另请参阅“ 新增功能”以获取主要更改的摘要。我们非常感谢该项目的许多外部贡献-谢谢大家！
