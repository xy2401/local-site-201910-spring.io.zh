  • Comprehensive support for Spring Security.

  • Configuration model classes are now a public API.
  • New features in timer based events.
  • New Junction pseudostate.
  • New Exit Point and Entry Point pseudostates.
  • Configuration model verifier.
  • New samples.
  • UI modeling support using Eclipse Papyrus.

Spring Statemachine 1.1.0发布

发布
珍妮·瓦尔基亚拉赫蒂
2016年5月26日

我们很高兴地宣布发布Spring Statemachine 1.1.0 。首先，我要感谢所有为实现这一目标而做出任何贡献的人。可以从Maven Central或从Spring Repository获得工件。

我们在此版本中获得了什么（与1.0.x相比）：

  • 对Spring Security的全面支持。
  • 与`@WithStateMachine'的上下文集成已大大增强。
  • StateContext现在是一流的公民，用户可以与状态机进行交互。
  • 内置的对Redis的支持增强了持久性功能。
  • 新功能有助于持久操作。
  • 配置模型类现在是公共API。
  • 基于计时器的事件的新功能。
  • 新连接点伪状态。
  • 新的退出点和进入点伪状态。
  • 配置模型验证程序。
  • 新样品。
  • 使用Eclipse Papyrus的UI建模支持。

可以像往常一样从changelog中进行所有更改 。

尽管始终通过配置器和构建器接口来完成配置状态机的本机方式，但是如果您看不到“大图”，则可能很难理解此设计任务。状态图下方是来自新示例的示例，该示例演示了状态机可以执行的更复杂的场景。基于Papyrus uml设计器，当您能够将机器视为正确的状态图时，相对容易理解机器的功能。它的用法是可选的，但允许用户将uml模型导入计算机配置。

感谢所有通过Stack Overflow，GitHub Issues或其他渠道做出任何贡献的人。推动变革的动力确实一直是社区及其提出的要求。
