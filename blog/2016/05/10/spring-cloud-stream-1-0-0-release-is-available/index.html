Spring Cloud Stream 1.0.0。可以发布

发布
马里乌斯·波哥维奇（Marius Bogoevici）
2016年5月10日

我很高兴代表团队宣布Spring Cloud Stream 1.0.0的全面可用性。发布。可以在Spring Repository以及Maven Central中找到这些工件。

Spring Cloud Stream的目标是成为一个轻量级框架，该框架基于Spring Boot的易于开发和部署以及Spring Integration的组件模型和集成功能来开发事件驱动的微服务。作为Spring Cloud项目系列的一部分，它特别关注云原生架构。

为了简化流传输和更传统的企业集成的开发，它提供了许多原语和抽象，例如：

  • 不同逻辑应用程序之间的持久发布-订阅语义；
  • 消费者团体支持扩展单个应用程序；
  • 声明性分区支持；
  • 声明式消息转换支持；

上面的所有功能都是通过框架的Binder抽象以中间件不可知的方式实现的，该框架允许不同的实现方式使模型适应通信中间件的本机功能。这使开发人员可以专注于业务抽象，还可以开发可与不同类型的消息传递中间件一起使用的可重用组件。Spring Cloud Stream 1.0.0。RELEASE与用于Apache Kafka和Rabbit MQ的Binders一起提供。可以使用其他实现，例如Gemfire。

完整的功能列表可在参考文档中找到，示例可在此处找到。

从第一次提交开始 ，Spring团队和社区的21位贡献者就添加了506次提交，解决了189个问题。

Spring Cloud Stream具有比其提交历史更悠久的遗产。它的诞生是从面向云服务，面向微服务的角度重新思考Spring XD（它应归功于其当前代码的重要组成部分），更具体地说是在Spring Boot和Spring中重新定义模块和消息总线的概念。云友好型时尚。它提供了机制来编写长期存在的微服务，这些机制可以由Spring Cloud Data Flow在复杂的流传输管道中（与面向任务的同级Spring相对应）在各种平台（Cloud Foundry，Apache YARN，Kubernetes，Apache Mesos）之间进行编排和部署。 云任务 。由于其Spring Integration基础，它为Spring Cloud Stream应用程序的丰富生态系统提供了强大的支持，这些生态系统可与各种类型的中间件和数据存储集成。它还为其他一些Spring Cloud项目提供了支持： Spring Cloud Bus ， Spring Cloud Hystrix和Spring Cloud Sleuth ，他们作为早期采用者提供了非常有价值的输入。
