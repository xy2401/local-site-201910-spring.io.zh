Spring Roo 2.0.0M2发布

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2016年5月18日

我很荣幸地代表DISID Corporation的Spring Roo团队宣布Spring Roo 2.0.0。M2已发布！

这个Spring Roo版本的主要目标是更新代码生成以使用最新的Spring技术：

  • 使用Spring IO Platform来管理依赖版本，并能够使用最新的Spring技术。
  • 更新代码生成以使用最新的Spring框架版本（4.x）
  • 更新代码生成，以在生成的项目中包括Spring Boot。
  • 删除所有基于XML文件生成的配置，并使用Spring Boot自动配置。
  • 删除Active Record数据模型，改用基于存储库（Spring数据）的模型。
  • 默认情况下基于服务层模式生成应用程序体系结构

此外，Spring Roo外壳还提高了其可用性：

  • 更直观的命令，仅提供必要的参数。
  • 用于配置Spring Roo行为的新命令。
  • 多模块项目改进。
  • 使用Spring Roo Shell进行推入式操作。

最后，表示层生成引擎已重构：

  • 同一控制器中允许使用不同的响应类型（JSON，THYMELEAF等）
  • 可以使用THYMELEAF生成视图。
  • 视图层生成引擎基于FreeMarker模板，开发人员可以安装和自定义模板。
  • Dojo已替换为HTML5，CSS3，Bootstrap和jQuery组件。（数据表，Select2等）
  • 新的惊人的Spring Roo响应主题！
  • … 以及更多！

与往常一样，新版本的Spring Roo包含社区报告的一些错误修复和改进。

有关在新的Spring Roo版本中所做的更改的完整详细信息，请参阅
Spring Roo 2.0.0。M2发行说明

此外，Spring Tool Suite（STS）团队正在努力发布STS 3.8.x，其中包括对Roo 2.0.0的Spring IDE Roo支持。M2。

如果要以前安装更新的Roo组件，则可以通过Spring IDE夜间更新站点手动进行安装，但是请记住，您将获得不稳定的STS版本，其中可能包含错误。

感谢所有出色的用户，他们提出了有用的错误报告和建议。
