春季云流布鲁克林。可以发布

发布
马里乌斯·波哥维奇（Marius Bogoevici）
九月23，2016

我很高兴代表团队宣布Spring Cloud Stream布鲁克林发布火车的全面上市。春季云流Brooklyn.RELEASE可在Spring Release存储库和Maven Central中使用。在参考文档中可以找到下面讨论的功能和新功能的详细说明。此处提供发行说明，其中包含有关迁移路径的重要信息。

在高层次上Brooklyn.RELEASE引入了以下新功能：

  • 支持Apache Kafka 0.9和0.10（Apache Kafka 0.8.x用户必须使用Spring Cloud Stream 1.0.x）；
  • 架构注册表支持；
  • 响应式API支持；

自发布候选版本几周以来，我们已经添加了一些改进，我们相信这些改进将有助于框架的可用性。他们是：

  • Spring Cloud Stream应用程序默认显示通道指标-为方便起见，预先包含了必要的依赖关系；
  • 添加了对显示聚合应用程序中内部渠道的指标的支持；
  • 如果应用程序不需要控制主题的创建和配置，对Kafka绑定器的代理依赖就不再需要。
  • 增加了对以编程方式为Kafka绑定程序配置SASL / Kerberos的支持，而无需外部JAAS配置文件；
  • 各种错误修复和文档增强；

我要感谢所有为自己的时间和努力做出贡献的人。特别要感谢社区成员提交拉取请求或花时间测试早期版本，为我们提供了宝贵的反馈。

至于我们的未来计划，它们包括：

  • 新的资料夹：JMS，Google Pub Sub，Amazon Kinesis
  • 支持Kafka Streams
  • 支持完全反应性的粘合剂

与往常一样，您可以在GitHub ， Stack Overflow ， Gitter或Twitter上与我们保持联系。
