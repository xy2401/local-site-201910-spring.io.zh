Spring Cloud Contract 1.0.0。可以发布

发布
马尔辛·格热兹扎克（Marcin Grzejszczak）
九月23，2016

我很高兴代表团队宣布1.0.0版中的Spring Cloud Contract全面上市。发布。您可以从Spring Release或Maven Central下载它。参考文档可在此处获得 。发行说明可在此处获得 。

Spring Cloud Contract的用途是什么？您始终需要有信心将新功能推入分布式系统中的新应用程序或服务中。Spring Cloud Contract项目为Spring应用程序中的消费者驱动的合同和服务模式提供支持，涵盖了用于编写测试，将其作为资产发布，断言生产者和消费者保持合同以进行HTTP和基于消息的交互的一系列选项。

Spring Cloud Contract由三个主要项目组成：

  • Spring Cloud Contract验证程序
  • Spring Cloud Contract WireMock
  • Spring Cloud Contract RestDocs

在高层次上：

Spring Cloud Contract验证程序

它是一个工具，支持基于JVM的应用程序的消费者驱动合同（CDC）开发。它与合同定义语言（DSL）一起提供。合同定义用于产生以下资源：

  • 在用户端进行集成测试时，WireMock将使用JSON存根定义。（测试代码必须仍然是手工编写的，测试数据由Spring Cloud Contract Verifier产生）。
  • 消息传递路线（如果使用的是一条）。我们正在与Spring Integration，Spring Cloud Stream和Apache Camel集成。但是，您可以根据需要设置自己的集成
  • 验收测试（在JUnit或Spock中），用于验证API的服务器端实现是否符合合同（生产者测试）。测试由Spring Cloud Contract Verifier生成。

Spring Cloud Contract Verifier将TDD移至软件架构级别。

Spring Cloud Contract WireMock

通过使用Spring Boot应用程序中嵌入的“环境”服务器，可以将WireMock与其他服务器一起使用。查看样本以获取更多详细信息。

例如，如果您有一个使用Tomcat作为嵌入式服务器的Spring Boot应用程序（默认情况下是spring-boot-starter-web），则只需将spring-cloud-contract-wiremock添加到您的类路径中并添加@AutoConfigureWireMock为了能够在您的测试中使用Wiremock。Wiremock作为存根服务器运行，您可以在测试中使用Java API或通过静态JSON声明来注册存根行为。

Spring Cloud Contract RestDocs

Spring RestDocs可用于使用Spring MockMvc或RestEasy为HTTP API生成文档（例如，asciidoctor格式）。在为API生成文档的同时，还可以使用Spring Cloud Contract WireMock生成WireMock存根。只需编写正常的RestDocs测试用例并使用@AutoConfigureRestDocs在restdocs输出目录中自动具有存根
