提供了适用于Spring Integration 1.2 Release Candidate 1的Java DSL

发布
阿尔特姆·比兰（Artem Bilan）
九月27，2016

亲爱的春季社区，

我很高兴地宣布，用于春季集成的Java DSL 1.2 RC1现在可用。

自从上一个里程碑2开始，我们对我们的新功能有了很好的深刻反馈，并且某些API已被打破以反映实际需求。

像往常一样，非常感谢每个在StackOverflow上创建问题，提出拉取请求，提供反馈或只是问问题的人。

神器org.springframework.integration:spring-integration-java-dsl:1.2.0.RC1可在Milestone仓库中使用 。因此，请尝试一下最后提出任何反馈的GH问题 ！

现在介绍一下1.2 RC1 ：

波勒errorChannel

我们现在可以指定errorChannel在Poller定义上。以前，我们必须配置整个，单独的MessagePublishingErrorHandler ：

e -> e.poller(Pollers.fixedDelay(100)
     .errorChannel(filePollingErrorChannel())

代替：

@Bean
public ErrorHandler filePollingErrorHandler() {
    MessagePublishingErrorHandler errorHandler =
                               new MessagePublishingErrorHandler();
    errorHandler.setDefaultErrorChannel(filePollingErrorChannel());
    return errorHandler;
}
...
e -> e.poller(Pollers.fixedDelay(100)
    .errorHandler(filePollingErrorHandler())

引入后 ，对MessagePublishingErrorHandler在Spring Integration Core中，仅使用Bean名称即可MessageChannel 。

FileSplitter的改进

Files.splitter()规范现在提供了更多选项来配置基础FileSplitter ：

.split(Files.splitter()
    .markers(false)
    .applySequence(true)
    .charset("UTF-8"))

集成流注册

手动流注册过程已得到一些改进：

  • 一个IntegrationFlowRegistrationBuilder从返回IntegrationFlowContext.registration(IntegrationFlow)在注册过程中提供更多选择IntegrationFlow bean和所有相关组件；

  • 一个IntegrationFlowRegistration是由于IntegrationFlowRegistrationBuilder.register()呼叫。这个对象提供了一些有用的IntegrationFlow属性，并控制其生命周期IntegrationFlow与之相关的；

  • 您现在可以添加其他Bean进行注册，这是提供的要求IntegrationFlow 。例如DefaultFtpSessionFactory是由Ftp.outboundAdapter() 。他们被摧毁并从ApplicationContext与IntegrationFlow它们是关联的。

this.integrationFlowContext.registration(myFlow)
        .id("myFlow")
        .autoStartup(false)
        .addBean(new Foo())
        .addBean("bar", new Bar())
        .register();

有关更多信息，请参见Gary Russell的有关动态TCP客户端的惊人样本 。

Apache Kafka支持

我们决定放弃Apache Kafka 0.8.x支持（针对Apache Kafka的Spring Integration 1.3.x ）赞成Apache Kafka 0.9.x和0.10.x 。最近介绍Kafka09已被删除，其内容已移至Kafka厂。现在对这个问题的依赖是Apache Kafka的Spring Integration 2.1.x ，基于Spring for Apache Kafka 1.1.x提供Apache Kafka 0.10.x 。同时一样Kafka DSL工厂和从属组件可以与用于Apache Kafka的以前的Spring Integration一起使用2.0.x类路径上的版本。

阿帕奇·卡夫卡0.8.x可以通过对Apache Kafka的直接Spring集成来实现支持1.3.x适配器的用法。或者您可以简单地复制/粘贴org.springframework.integration.dsl.kafka Java DSL中的软件包 1.1到您的项目中，并继续从流利的Builder API获得收益Kafka厂！

下一步

我们希望版本具有一般可用性1.2在大约一周内。之后spring-integration-java-dsl一定会迁移到Spring Integration Core 5.0和Java 8代码库。目前1.2仍将支持该版本，但仅用于错误修复。
