Spring Cloud Camden SR1可用

发布
马尔辛·格热兹扎克（Marcin Grzejszczak）
2016年10月24日

我很高兴代表团队宣布Spring Cloud Camden Release Train的新服务版本。 Camden.SR1可以在我们的Spring Release存储库或Maven Central中找到。您可以查看Camden 发行说明以获取更多信息 。

卡姆登SR1发布火车的亮点

以下模块构成Camden。SR1：

模组 版
Spring Cloud AWS 1.1.3。发布
Spring Cloud巴士 1.2.1。发布
Spring Cloud Commons 1.1.4。发布
Spring Cloud合约 1.0.1。发布
Spring Cloud Config 1.2.1。发布
Spring Cloud Netflix 1.2.1。发布
Spring Cloud安全性 1.1.3。发布
Spring Cloud侦探 1.0.10。发布
春季云流 布鲁克林发布
春季云任务 1.0.3。发布
春季云动物园管理员 1.0.3。发布

合并的发行培训文档可在此处获得 。

也是Spring Cloud CLI 1.2.1.RELEASE被释放。您可以在此处查看项目页面

Spring Cloud Consul在此版本中未更新，但是版本1.1.1.RELEASE在卡姆登之后不久被释放。SR1，其中包含用户可能感兴趣的几个修复程序 。如果要升级应用程序中使用的Consul版本，则必须在Maven或Gradle项目中手动指定版本号。

而且，一如既往，我们欢迎您提供反馈：在GitHub ， Gitter ， Stack Overflow或Twitter上 。

要开始使用带有BOM的Maven（仅限于依赖管理）：

<dependencyManagement>
  <dependencies>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-dependencies</artifactId>
    <version>Camden.SR1</version>
    <type>pom</type>
    <scope>import</scope>
  </dependencies>
</dependencyManagement>
<dependencies>
  <dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-config</artifactId>
  </dependency>
  <dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-eureka</artifactId>
  </dependency>
  ...
</dependencies>

或使用Gradle：

buildscript {
  dependencies {
    classpath "org.springframework.boot:spring-boot-gradle-plugin:1.4.1.RELEASE"
  }
}

apply plugin: "spring-boot"

dependencyManagement {
  imports {
    mavenBom 'org.springframework.cloud:spring-cloud-dependencies:Camden.SR1'
  }
}

dependencies {
    compile 'org.springframework.cloud:spring-cloud-starter-config'
    compile 'org.springframework.cloud:spring-cloud-starter-eureka'
    ...
}
