春天的这个星期-2016年10月25日

工程
乔什·朗（Josh Long）
2016年10月25日

欢迎来到春季本周的另一期！这周我将在爱荷华州得梅因的草原上。代码会议 。如果您在附近，请停下来打个招呼！

和往常一样，我们有很多内容需要介绍，所以让我们开始吧。

  • Spring Cloud忍者Marcin Grzejszczak刚刚宣布了Spring Cloud Camden SR1。Spring Cloud发布火车真是太美了 -看到它让我想打造一些很棒的东西！它包括Spring Cloud AWS，Bus，Commons，Contract，Config，Netflix，Security，Sleuth，Stream，Task和Zookeeper。
  • Spring Cloud Data Flow联合创始人Mark Pollack博士刚刚宣布了Spring Cloud Data Flow 1.1。M2 。
  • 希望获得帮助以自动化使您的应用程序从git push到生产中成功部署的应用程序？查看新项目Spring Cloud Pipelines ，它是一组shell可以在Pivotal Concourse或Jenkins上移植的脚本。请记住：微服务仅在帮助您实现持续交付的范围内才有用。在连续交付中，您需要一个自动化的管道和自动部署的能力（自然，为此，我们有Pivotal Cloud Foundry）！
  • 我和Trifork的Allard Buijze和我将在2016年11月16日举行有关Spring Boot和Axon Framework的网络研讨会 。如果您对事件驱动的微服务感到好奇，那么本次会议应该给您很多思考。
  • 上周，我通过Spring Tips系列继续了我对Spring生态系统各个方面的热门介绍，并探讨了如何在疯狂的酷Kotlin编程语言中使用Spring。
  • Spring Data和Apache Geode忍者John Blum发布了Spring Data Geode 1.0.0。APACHE-GEODE-INCUBATING-M3除了帮助从Gemfire迁移到Geode代码库外，还具有强大的Java配置支持，可使用Apache Geode。这些东西是炙手可热的，我很尴尬，我以前忽略了它！
  • Spring Cloud Camden具有很多功能，其中之一是对定义和使用服务合同的支持。尽管这篇文章特别与Spring无关，但我认为对于那些想了解消费者驱动的合同和消费者驱动的合同测试动机的人来说，这是一个很好的背景
  • 谈到Kotlin，想学习一些Pivotal Labs的工程人员如何在（有或没有！）情况下使用Kotlin。春天 ？
  • 尽管这个示例应用程序演示了如何根据事件和Spring Cloud构建应用程序，但还是值得一看。
  • 芭芭·达罗（Barb Darrow）在《财富》杂志的这篇文章中做了出色的工作， 阐明了Google Compute Engine为什么是运行Pivotal Cloud Foundry的绝佳去处
  • Lightbend的优秀人员刚刚宣布支持在Pivotal Cloud Foundry上使用功能强大的演员系统Akka 。恭喜团队！
