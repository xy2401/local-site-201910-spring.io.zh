用于Cloud Foundry 1.1M1和1.0.1的Spring Cloud Data Flow发布

发布
埃里克·博塔德
2016年10月6日

用于Cloud Foundry的Spring Cloud Data Flow今天获得了两个新版本，详细说明如下：

1.0.1。GA是维护版本，

  • 以2.0.1为基础。发布Cloud Foundry的Java客户端和3.0.2。释放反应堆

  • 以1.0.1为基础。发布Spring Cloud Data Flow核心

  • 在参考文档中获得一般性说明和改进

  • 修复了与多个ORG +空间组合有关的错误，该错误导致任务启动失败

  • 修复了启动任务时环境变量未正确传播的错误

1.1.0。M1是新开发线的第一个里程碑。除了上面详细介绍的错误修复和依赖项升级以外，它

  • 以1.1.0为基础。Spring Cloud Data Flow核心的M1。在这里查看详细信息

  • 以1.1.0为基础。用于Cloud Foundry的Spring Cloud Deployer的 M1。因此，以下选项现在可用：

  • 增加了对默认应用程序“登台”和“启动”超时的覆盖的支持。这对于资源受限可能导致超时错误的开发机以及非常大的应用程序特别有用

  • 添加了为Cloud Foundry中已部署的应用程序定义自定义路由（或根本没有路由）的功能

  • 增加了完全关闭运行状况检查的功能。这在启动时远程调试已部署的应用程序时特别有用

作为这些版本的一部分，适用于Cloud Foundry 1.0.1的Spring Cloud Deployer。GA和1.1.0。M1也已可用，可由需要使用Spring Cloud Deployer抽象来利用Cloud Foundry的部署功能的任何项目使用。
