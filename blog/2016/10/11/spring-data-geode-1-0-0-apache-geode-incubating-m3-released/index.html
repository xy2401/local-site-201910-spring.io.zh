Spring Data Geode 1.0.0。APACHE-GEODE-INCUBATING-M3发布

工程
约翰·布鲁姆
十月11，2016

相对于Apache Geode M3的发布公告 ，我对Spring和Apache Geode社区的延迟深表歉意，但是我非常高兴并很高兴为Apache Geode 1.0.0孵化发布Spring Data 。M3 。

您可以通过在应用程序Maven或Gradle构建文件中包括以下依赖项来从Maven Central中获取信息：

马文


  org.springframework.data
  spring-data-geode
  1.0.0.APACHE-GEODE-INCUBATING-M3

摇篮

compile 'org.springframework.data:spring-data-geode:1.0.0.APACHE-GEODE-INCUBATING-M3'

包括spring-data-geode依赖关系将传递所有必需的Apache Geode工件，因此您可以立即开始构建使用Apache Geode的Spring应用程序。

什么是新的

尽管Apache Geode M2和M3之间的功能没有太大变化（主要是错误修复 ），但Spring Data Geode确实做了很多更改 。其余所有功能都比其他功能重要。

搭建舞台

由于Apache Geode （以及扩展名为Pivotal GemFire ）的重点一直是提供足够的功能广度和灵活性，以解决各种各样的复杂客户需求和用例 （单击“ 用例 ”），因此很难用户可以尽可能快速地启动并运行。而且，尽管它具有更强的一致性保证，并且通常具有更好的性能，但是在使用替代的，可比的解决方案时，很难与开箱即用的经验相抗衡。

要获得对该问题的更多了解，以及我们如何专注于为Apache Geode和Pivotal GemFire解决该问题，请在8月的Apache Geode Clubhouse会议上查看我的演讲。

问题方案

问题的根源在于Apache Geode缺乏一个自以为是的，一致的编程/配置模型，特别是对于应用程序开发而言。因此，实际上，我们除了Spring Framework之外，不需要再寻找其他任何东西。但是，这仍然留下了“意见”。

Spring再次为我们提供了Spring Boot ，它是多么美丽的工程，如此优雅，如此简单。而且，正如您将在下面看到的那样，这确实是Spring Data Geode上工作的灵感。

闭嘴给我看“密码”…

过去，Spring开发人员必须使用XML或Java config配置Geode，然后使用自举机制（如Gfsh或Spring Boot）启动并运行。这甚至已经在指南中得到了证明。到目前为止，任何一个都没有真正比另一个有明显的优势。

从此里程碑版本开始， Spring Data Geode将包括一组新的注释 ，其形式/功能非常相似，并且与Spring Boot注释互补，但特别适用于SD / Apache Geode，并牢记一个简单的目标……

简直就是开箱即用的体验，以便使用户尽可能快速 ， 轻松地 启动并运行

因此，让我们以XML和Java配置示例为例，再做一次……

@SpringBootApplication
@PeerCacheApplication
@WithReplicateRegions("myDistributedMap")
public class ExampleApplication implements CommandLineRunner {

  public static void main(String[] args) {
    SpringApplication.run(ExampleApplication.class, args);
  }

  @Resource(name = "myDistributedMap")
  private Region myDistributedMap;

  @Override
  public void run(String... strings) throws Exception {
    assertThat(myDistributedMap.put("key", "value")).isNull();
    assertThat(myDistributedMap.get("key")).isEqualTo("value");
    assertThat(myDistributedMap.putIfAbsent("somekey", "somevalue")).isNull();
    assertThat(myDistributedMap.replace("key", "value", "newvalue")).isTrue();
  }
}

挺容易。这个示例非常简单，但是您可能会想到一些应用程序配置（ 例如 ；仅遵循层次结构）很快就变得非常复杂。

目前这是一个预告片，敬请期待。

结论

将有许多不同的注释以一致且健壮的方式解决所有独特的问题，提供了现成的默认值以及许多便利的“启用”功能。这个新的注释配置模型的许多细节仍在开发中，例如适当的示例和文档。

在此期间，我已经开始了参考实现的开始，该参考实现将展示结合使用Spring和Apache Geode的许多特性（例如新的Annotation配置模型 ）。

希望在不久的将来与我们联系。
