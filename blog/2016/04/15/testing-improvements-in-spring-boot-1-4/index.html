测试Spring Boot 1.4中的改进

工程
菲尔·韦伯
2016年4月15日

为Pivotal工作的好处之一是，他们拥有一个强大的敏捷开发部门，称为Pivotal Labs。实验室中的团队大力支持精益和XP软件方法论，例如结对编程和测试驱动的开发。他们对测试的热爱对Spring Boot 1.4产生了特别的影响，因为我们已经开始就可以改进的方面获得大量反馈。这篇博客文章重点介绍了最新的M2版本中已经包含的一些新测试功能。

无弹簧测试

对任何Spring进行单元测试的最简单方法@Component根本不涉及Spring！始终最好尝试构造代码，以便可以直接实例化和测试类。通常，这归结为几件事：

  • 用清晰的关注点分离结构化代码，以便可以对各个部分进行单元测试。TDD是实现此目标的好方法。
  • 使用构造函数注入以确保可以直接实例化对象。不要使用字段注入，因为这只会使您的测试难以编写。

使用Spring Framework 4.3，编写使用构造函数注入的组件非常容易，因为您不再需要使用@Autowired 。只要您只有一个构造函数，Spring就会隐式地将其视为自动装配目标：

@Component
public class MyComponent {
    
    private final SomeService service;

    public MyComponent(SomeService service) {
        this.service = service;
    }

}

测试中MyComponent现在就像直接创建它并调用一些方法一样简单：

@Test
public void testSomeMethod() {
    SomeService service = mock(SomeService.class);
    MyComponent component = new MyComponent(service);
    // setup mock and class component methods
}

Spring Boot 1.3回顾

当然，通常您需要将堆栈往上移一点，并开始编写确实涉及Spring的集成测试。幸运的是，Spring Framework具有spring-test模块在这里提供帮助，不幸的是，在Spring Boot 1.3中有很多不同的方法可以使用它。

您可能正在使用@ContextConfiguration注释与SpringApplicationContextLoader ：

@RunWith(SpringJUnit4ClassRunner.class)
@ContextConfiguration(classes=MyApp.class, loader=SpringApplicationContextLoader.class)
public class MyTest {

    // ...

}

您可能选择了@SpringApplicationConfiguration ：

@RunWith(SpringJUnit4ClassRunner.class)
@SpringApplicationConfiguration(MyApp.class)
public class MyTest {

    // ...

}

您可能将它们中的任何一个与@IntegrationTest ：

@RunWith(SpringJUnit4ClassRunner.class)
@SpringApplicationConfiguration(MyApp.class)
@IntegrationTest
public class MyTest {

    // ...

}

或搭配@WebIntegrationTest （或可能@IntegrationTest + @WebAppConfiguration ）

@RunWith(SpringJUnit4ClassRunner.class)
@SpringApplicationConfiguration(MyApp.class)
@WebIntegrationTest
public class MyTest {

    // ...

}

您还可以加入在随机端口上运行服务器的组合（ @WebIntegrationTest(randomPort=true) ）并添加属性（使用@IntegrationTest("myprop=myvalue")要么@TestPropertySource(properties="myprop=myvalue") ）

有很多选择！

Spring Boot 1.4的简化

使用Spring Boot 1.4，事情应该变得更简单。有一个@SpringBootTest用于常规测试的批注，以及用于测试应用程序切片的一些特殊变体（稍后会详细介绍）。

典型的Spring Boot 1.4集成测试如下所示：

@RunWith(SpringRunner.class)
@SpringBootTest(webEnvironment=WebEnvironment.RANDOM_PORT)
public class MyTest {

    // ...
    
}

这是正在发生的事情的分解：

  • @RunWith(SpringRunner.class)告诉JUnit使用Spring的测试支持运行。 SpringRunner是的新名称SpringJUnit4ClassRunner ，在眼睛上就容易多了。
  • @SpringBootTest说“在Spring Boot的支持下进行引导”（例如，加载application.properties并给我所有的Spring Boot天赋）
  • 的webEnvironment属性允许为测试配置特定的“网络环境”。您可以使用MOCK Servlet环境或运行在RANDOM_PORT或DEFINED_PORT 。
  • 如果要加载特定的配置，可以使用classes的属性@SpringBootTest 。在这个例子中，我们省略了classes这意味着测试将首先尝试加载@Configuration从任何内部类中获取，如果失败，它将搜索您的主要类@SpringBootApplication类。

的@SpringBootTest注释也有一个properties可用于指定应在中定义的任何其他属性的属性Environment 。现在，属性的加载方式与Spring的常规加载方式完全相同@TestPropertySource注解。

下面是一个更具体的示例，它实际上达到了真正的REST端点：

@RunWith(SpringRunner.class)
@SpringBootTest(webEnvironment=WebEnvironment.RANDOM_PORT)
public class MyTest {
    
    @Autowired
    private TestRestTemplate restTemplate;

    @Test
    public void test() {
        this.restTemplate.getForEntity(
            "/{username}/vehicle", String.class, "Phil");
    }

}

注意TestRestTemplate现在可以作为bean使用@SpringBootTest用来。它已预先配置为解析相对路径http://localhost:${local.server.port} 。我们也可以使用@LocalServerPort注解，将服务器正在运行的实际端口注入测试字段。

模拟和间谍

当您开始测试实际系统时，通常会发现模拟特定的bean很有帮助。常见的模拟场景包括模拟运行测试时无法使用的服务，或测试难以在实时系统中触发的失败场景。

使用Spring Boot 1.4，您可以轻松创建一个Mockito模拟，该模拟可以替换现有的bean，或者创建一个新的bean：

@RunWith(SpringRunner.class)
@SpringBootTest(webEnvironment = WebEnvironment.RANDOM_PORT)
public class SampleTestApplicationWebIntegrationTests {

    @Autowired
    private TestRestTemplate restTemplate;

    @MockBean
    private VehicleDetailsService vehicleDetailsService;

    @Before
    public void setup() {
        given(this.vehicleDetailsService.
            getVehicleDetails("123")
        ).willReturn(
            new VehicleDetails("Honda", "Civic"));
    }

    @Test
    public void test() {
        this.restTemplate.getForEntity("/{username}/vehicle", 
            String.class, "sframework");
    }

}

在此示例中，我们是：

  • 为创建一个Mockito模拟VehicleDetailsService 。
  • 注入到ApplicationContext作为豆。
  • 将其注入测试中的字段。
  • 存根行为setup方法。
  • 触发将最终调用模拟的东西。

模拟会在测试之间自动重置。它们也构成了Spring Test使用的缓存键的一部分（因此无需添加@DirtiesContext ）

间谍的工作方式与此类似。只需使用以下注释注释测试字段@SpyBean让间谍将现有的Bean包装在ApplicationContext 。

JSON断言

如果您使用spring-boot-starter-test POM导入测试依赖项，从1.4开始，您将获得出色的AssertJ库。AssertJ提供了一个流畅的断言API，该API取代了JUnit的一些基本功能org.junit.Assert类。如果您之前从未看过，则基本的AssertJ调用看起来像这样：

assertThat(library.getName()).startsWith("Spring").endsWith("Boot");

Spring Boot 1.4提供了扩展的断言，可用于检查JSON编组和解编组。JSON测试器可用于Jackson和Gson。

public class VehicleDetailsJsonTests {

    private JacksonTester<VehicleDetails> json;

    @Before
    public void setup() {
        ObjectMapper objectMapper = new ObjectMapper(); 
        // Possibly configure the mapper
        JacksonTester.initFields(this, objectMapper);
    }

    @Test
    public void serializeJson() {
        VehicleDetails details = 
            new VehicleDetails("Honda", "Civic");

        assertThat(this.json.write(details))
            .isEqualToJson("vehicledetails.json");

        assertThat(this.json.write(details))
            .hasJsonPathStringValue("@.make");

        assertThat(this.json.write(details))
            .extractingJsonPathStringValue("@.make")
            .isEqualTo("Honda");
    }

    @Test
    public void deserializeJson() {
        String content = "{\"make\":\"Ford\",\"model\":\"Focus\"}";

        assertThat(this.json.parse(content))
            .isEqualTo(new VehicleDetails("Ford", "Focus"));

        assertThat(this.json.parseObject(content).getMake())
            .isEqualTo("Ford");
    }

}

JSON比较实际上是使用JSONassert执行的，因此仅需要匹配JSON的逻辑结构。您还可以在上面的示例中看到如何使用JsonPath表达式测试或提取数据。

测试应用程序片

Spring Boot的自动配置功能非常适合配置应用程序需要运行的所有内容。不幸的是，对于测试而言，全自动配置有时可能有点过大。有时您只想配置应用程序的“切片” – Jackson的配置正确吗？我的MVC控制器返回正确的状态代码吗？我的JPA查询会运行吗？

使用Spring Boot 1.4，现在可以轻松测试这些常见方案。我们也使构建仅应用所需的自动配置类的注释变得更加容易。

测试JPA片

要测试应用程序的JPA分片（Hibernate + Spring Data），可以使用@DataJpaTest注解。一种@DataJpaTest将：

  • 配置内存数据库。
  • 自动配置Hibernate，Spring Data和DataSource。
  • 执行@EntityScan 。
  • 开启SQL记录

典型的测试如下所示：

@RunWith(SpringRunner.class)
@DataJpaTest
public class UserRepositoryTests {

    @Autowired
    private TestEntityManager entityManager;

    @Autowired
    private UserRepository repository;

    @Test
    public void findByUsernameShouldReturnUser() {
        this.entityManager.persist(new User("sboot", "123"));
        User user = this.repository.findByUsername("sboot");
        
        assertThat(user.getUsername()).isEqualTo("sboot");
        assertThat(user.getVin()).isEqualTo("123");
    }

}

的TestEntityManager在上面的测试中是由Spring Boot提供的。它是标准JPA的替代方案EntityManager提供编写测试时常用的方法。

测试Spring MVC Slice

要测试应用程序的Spring MVC分片，您可以使用@WebMvcTest注解。这会：

  • 自动配置Spring MVC，Jackson，Gson，消息转换器等
  • 加载相关组件（ @Controller ， @RestController ， @JsonComponent等等）
  • 配置MockMVC

这是一个测试单个控制器的典型示例：

@RunWith(SpringRunner.class)
@WebMvcTest(UserVehicleController.class)
public class UserVehicleControllerTests {

    @Autowired
    private MockMvc mvc;

    @MockBean
    private UserVehicleService userVehicleService;

    @Test
    public void getVehicleShouldReturnMakeAndModel() {
        given(this.userVehicleService.getVehicleDetails("sboot"))
            .willReturn(new VehicleDetails("Honda", "Civic"));

        this.mvc.perform(get("/sboot/vehicle")
            .accept(MediaType.TEXT_PLAIN))
            .andExpect(status().isOk())
            .andExpect(content().string("Honda Civic"));
    }

}

如果您更喜欢HtmlUnit，也可以使用WebClient代替MockMvc 。如果硒更适合您，您可以切换到WebDriver 。

测试JSON切片

如果您需要测试JSON序列化是否按预期工作，则可以使用@JsonTest 。这会：

  • 自动配置Jackson和/或Gson
  • 添加任何Module要么@JsonComponent您定义的bean
  • 触发任何初始化JacksonTester要么GsonTester领域

这是一个例子：

@RunWith(SpringRunner.class)
@JsonTest
public class VehicleDetailsJsonTests {

    private JacksonTester<VehicleDetails> json;

    @Test
    public void serializeJson() {
        VehicleDetails details = new VehicleDetails(
            "Honda", "Civic");

        assertThat(this.json.write(details))
            .extractingJsonPathStringValue("@.make")
            .isEqualTo("Honda");
    }

}
