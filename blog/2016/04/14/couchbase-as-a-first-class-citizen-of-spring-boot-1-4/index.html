Couchbase作为Spring Boot 1.4的头等公民

Spring Boot 1.4.0 MILESTONE 2出来了！现在是时候告诉您有关Spring Boot团队成员与Couchbase Java SDK团队之间的共同努力，以提供Couchbase与Spring Boot的一流集成：）

在Spring Boot 1.4.0 ，Couchbase成为Spring Boot生态系统的一等公民！

Couchbase SDK集成

Spring Boot现在可以直接识别您何时拥有Couchbase SDK在你的classpath中。在这种情况下，它会实例化一个Cluster和一个Bucket bean为您使用自动配置 。

Spring Boot可以选择属性来进一步配置这些核心SDK类，甚至CouchbaseEnvironment ！

可以识别以下引导属性：

spring.couchbase.bootstrap-hosts
spring.couchbase.bucket.name
spring.couchbase.bucket.password

通过以下属性公开环境调整（IO端点，SSL支持和默认同步API超时）：

spring.couchbase.env.endpoints.key-value
spring.couchbase.env.endpoints.query
spring.couchbase.env.endpoints.view

spring.couchbase.env.ssl.enabled
spring.couchbase.env.ssl.key-store
spring.couchbase.env.ssl.key-store-password

spring.couchbase.env.timeouts.connect
spring.couchbase.env.timeouts.key-value
spring.couchbase.env.timeouts.query
spring.couchbase.env.timeouts.view

现在这是⭐️⭐️⭐️️️️️️支持

警告 ：请注意，在上一个里程碑中，引导属性的前缀为“ spring.*data*.couchbase ”，现在变成“ spring.couchbase ”。

Spring Cache的实现

Spring Cache抽象具有一个Couchbase 2.x实现， couchbase-spring-cache 。

现在，当Java SDK和couchbase-spring-cache工件在类路径上。

这与上一节很好地集成在一起，被视为默认的基础存储Bucket缓存是由Spring Boot自动配置的:)

注意 ：缓存实现已从Spring Data Couchbase项目中拉出到其自己的项目（在Couchbase的github存储库中），因此可以与Spring Data分开使用和发布。

提示 ：不要忘记至少配置spring.couchbase.bootstrap-hosts属性。

这个实现CacheManager通过使用每个缓存的名称自动在Couchbase中添加键，从而允许将来自多个缓存的数据存储到同一个Couchbase存储桶中。

的CouchbaseCacheManager将自动重用Bucket由Spring Boot自动配置。通过简单地添加一个属性来声明性地列出缓存名称，相应的缓存将被预加载：

spring.cache.type=couchbase
spring.cache.cache-names=foo,bar

五星级的支持，我们告诉您！

您甚至可以配置它以将不同缓存的数据存储到多个存储桶中。然后可以使用流畅的构建器模式（例如，在CacheManagerCustomizer ）。这个CacheBuilder还可以让您进一步调整缓存，例如设置默认的过期时间：

@Configuration
public class CouchbaseCacheConfiguration {

  private final Cluster cluster;

  //inject the Cluster from Boot core Couchbase support
  public CouchbaseCacheConfiguration(Cluster cluster) {
    this.cluster = cluster;
  }

  @Bean
  public Bucket anotherBucket() {
    return this.cluster.openBucket("another", "secret");
  }

  @Bean
  public CacheManagerCustomizer<CouchbaseCacheManager> cmCustomizer() {
    return c -> {
      c.prepareCache("biz", CacheBuilder
              .newInstance(anotherBucket())
              .withExpirationInMillis(2000));
    };
  }
}

此实现可以使用视图来有选择地清除在同一存储桶中并置的缓存。

最后，它支持根据请求动态创建缓存。要激活它，只需省略声明任何缓存名称，然后使用相关属性激活Couchbase缓存即可：

spring.cache.type=couchbase

这将使用自动配置的Spring Boot Bucket作为所有动态创建的缓存的默认存储区。

Spring数据集成

Spring Data Couchbase 2.1.0 （发行版Hopper）进行了一些修改，使与Spring Boot的集成变得轻而易举。

首先是通常的怀疑：默认情况下，Spring Boot将自动配置Spring Data Couchbase以使用Bucket它创建了。通过将核心SDK配置部分分成一个CouchbaseConfigurer类，而真正特定于Spring Data的是位于新的基类中， AbstractCouchbaseDataConfiguration 。

如果您仅使用Spring Data，则可以继续使用AbstractCouchbaseConfiguration ，现在同时是CouchbaseConfigurer和AbstractCouchbaseDataConfiguration。

但是，如果您使用Spring Boot，它将自动配置一个CouchbaseConfigurer 。您随时可以调整自己的AbstractCouchbaseDataConfiguration并将配置器注入其中。

其他Spring Data Couchbase功能

2.1.0版本还包含一些新功能：

  • 可选自动touch （刷新文档的过期时间）读取（ DATACOUCH-59 ）
  • 排序和分页的改进（ DATACOUCH-211 ， DATACOUCH-214 ）
  • 增加了对审核的支持（例如@CreatedBy注释， DATACOUCH-91 ）
