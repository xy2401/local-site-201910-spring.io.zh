Spring Cloud数据流具有1个流，2个应用程序和3个依赖项

工程
乔什·朗（Josh Long）
2016年4月5日

我只是想在这里注册一个让我昨天微笑的经历：在（几分钟内）使迅速改进的Spring Cloud Data Flow从（Spring Boot）启动（-ers）摆动到服务！

唯一的先决条件是您需要运行Redis实例。我的Redis实例正在运行127.0.0.1而且不需要进一步配置，Spring Boot即可查找并使用它。

我们将使用史诗般的Spring Initializr来简化生成应用程序的工作。还记得那些愚蠢的苹果广告，“那有一个应用程序吗？”没关系， 有一个复选框 ！ 让我们看看您是否像我一样喜欢这种体验！

本地数据流服务器

转到Spring Initializr并选择Local Data Flow Server并命名工件df-server 。这将用于建立本地数据流服务-REST API和一些持久性逻辑-编排和存储有关流和任务的信息。在旧的Spring XD世界中，这称为Spring XD Admin Server 。

在您选择的IDE中打开项目并添加@EnableDataFlowServer到DfServerApplication类：

package com.example;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.cloud.dataflow.server.EnableDataFlowServer;

@EnableDataFlowServer
@SpringBootApplication
public class DfServerApplication {

	public static void main(String[] args) {
		SpringApplication.run(DfServerApplication.class, args);
	}
}

跑mvn spring-boot:run在df-server项目，应用程序将在端口上旋转9393 。

提示 ：看到欢迎的ASCII图稿时，您将知道自己（可能）无家可归！

关于提示的提示 ：所以，这可能不是事实。它可能由于各种原因而失败（例如服务或嵌入式H2数据库的端口冲突），但是高质量的ASCII艺术品已在（我）的研究中证明是治疗性的。

数据流外壳

转到Spring Initializr并选择Data Flow Shell并命名为神器df-shell 。这将用于站立由Spring Shell支持的Data Flow Shell。

数据流外壳程序可以在任何操作系统上运行。它是我们刚刚站起来的Data Flow服务的客户。让我们使用熟悉的管道过滤器DSL和命令来操作服务。我喜欢好的横幅ASCII艺术作品之多未来开发商， 有 （一饮而尽！）这样的事情太好了。默认情况下，春季壳牌和 Spring引导都将尝试发出ASCII旗帜，所以我们将启动春季保持的这一点（这一次！）。在您选择的IDE中打开项目并添加@EnableDataFlowShell到DfShellApplication类，然后配置SpringApplication创建以隐藏Spring Boot标语：

package com.example;

import org.springframework.boot.Banner;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.boot.builder.SpringApplicationBuilder;
import org.springframework.cloud.dataflow.shell.EnableDataFlowShell;

@EnableDataFlowShell
@SpringBootApplication
public class DfShellApplication {

	public static void main(String[] args) {
		new SpringApplicationBuilder(DfShellApplication.class)
				.bannerMode(Banner.Mode.OFF)
				.run(args);
	}
}

跑mvn spring-boot:run在df-shell项目。默认情况下，您应该能够与本地运行的数据流服务器进行交互。通过发出module list命令。您应该看到一张表格，其中包含Spring Cloud Data Flow已经知道的所有内置组件。

记录接收器模块

转到Spring Initializr并选择Stream Redis并命名工件logging-sink 。我们将使用Spring Cloud Stream，它在Spring的MessageChannel Spring Integration中的抽象和组件模型，以简化描述和集成基于消息传递的微服务的工作，以构建用于记录传入消息的自定义模块。然后，我们将使用Spring Cloud Data Flow部署和协调该模块。

Spring Cloud Data Flow是一种以小型Spring Boot驱动的模块描述复杂的集成，批处理和流处理工作负载的强大方法。有几种类型的module s。一种source通常按照固定的时间表生成数据，以供下游组件消耗和处理。一种processor接收数据，对其进行处理，然后将数据写出。一种sink仅接收数据，但不产生任何要发送的信息。这些组件可以很好地组合在一起，以描述任何类型的可能无法停止的工作负载（物联网传感器数据，24/7事件处理，在线交易数据提取和集成方案等）。最终，源通常是Spring Integration入站适配器。处理器通常是任何类型的Spring Integration组件（例如变压器），都可以将数据带入并产生数据。接收器通常是Spring Integration出站适配器。

一种task描述最终将停止的所有工作负载。可能是一个简单的Spring Boot Command Line Runner或春季批Job 。

不过，Spring Cloud Data Flow对Spring Integration没有特别的了解。它只知道Spring Cloud Stream和著名的Spring MessageChannels喜欢input和output 。不管这些通道的终端是什么。Spring Cloud Data Flow也不特别了解Spring Batch。它只知道Spring Cloud Task。

以UNIX的方式sh外壳环境使我们能够通过将数据往返于单个焦点的命令行实用程序来组合任意许多复杂的解决方案stdin和stdout分别，Spring Cloud Data Flow也使我们能够从单独关注的消息传递组件组成任意许多且复杂的解决方案。

Spring Cloud Data Flow已包含很多电池 。我们将开发并安装一个简单的模块来记录事件-在我们的情况下就是时间。值得注意的是，我们将为自己的目的而这样做，但是我们不需要这样做。 Spring Cloud Data Flow已经提供了一个log模块！（和time模块！）

package com.example;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.cloud.stream.annotation.EnableBinding;
import org.springframework.cloud.stream.messaging.Sink;
import org.springframework.integration.annotation.MessageEndpoint;
import org.springframework.integration.annotation.ServiceActivator;
import org.springframework.messaging.handler.annotation.Headers;
import org.springframework.messaging.handler.annotation.Payload;

import java.util.Map;

@EnableBinding(Sink.class)
@SpringBootApplication
public class LoggingSinkApplication {

	@MessageEndpoint
	public static class LoggingMessageEndpoint {

		@ServiceActivator(inputChannel = Sink.INPUT)
		public void logIncomingMessages(
				@Payload String msg,
				@Headers Map<String, Object> headers) {

			System.out.println(msg);
			headers.entrySet().forEach(e ->
					System.out.println(e.getKey() + '=' + e.getValue()));

		}
	}

	public static void main(String[] args) {
		SpringApplication.run(LoggingSinkApplication.class, args);
	}
}

这是一个简单的Spring Cloud Stream绑定。的Sink.class是定义一个MessageChannel input() 。Spring Cloud Stream会将其转换为消息代理的实时命名管道（在本例中为Redis，尽管Spring Cloud Data Flow的默认值可能在未来几个月内更改为RabbitMQ），我们的任何消息传递代码都可以使用。该示例使用Spring Integration在到达时打印出传入的消息数据。让我们先向Data Flow注册我们的自定义模块，然后组成一个流，该流接收包含来自time组件，然后记录结果。

第一， mvn clean install的logging-sink项目，以便可以在本地Maven存储库中解析。Spring Cloud Data Flow使用可插拔策略来解析自定义模块的实例。在我们的示例中，它将尝试在系统本地Maven存储库中解决它们。

下一步

到云！我们正在使用本地数据流服务器。还有其他实现可用于处理诸如Cloud Foundry之类的结构。Cloud Foundry数据流服务器启动应用程序实例，而不是本地Java进程。现在，构建可伸缩的数据提取和处理流就像cf push ..和cf scale -i $MOAR ！

我们仅使用Spring Cloud Data Flow的一些功能！使用Spring Cloud Data Flow协调由Spring Cloud Stream支持的任何数量的基于消息传递的微服务。我建议您看一下一些内置的Spring Cloud Stream模块以获取灵感。
