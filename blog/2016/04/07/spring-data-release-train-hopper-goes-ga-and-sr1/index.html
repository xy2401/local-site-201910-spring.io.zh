    春季数据发布火车Hopper进入GA（和SR1）！

    发布
    奥利弗·德罗博姆
    2016年4月7日

    我很高兴代表整个Spring Data团队宣布Spring Data发布火车Hopper的全面上市。这列火车包括395张固定的车票，最重要的补充是：

    • 将Spring Framework基准升级到4.2（尤其是4.2.5）。
    • 在Spring Data JPA和MongoDB中支持示例查询。
    • 改进了对组合注释的支持。
    • 在Spring Data REST中支持查找类型和嵌套关联
    • Solr（Solr 5.5上的2.0），Neo4j（4.1），Couchbase（Couchbase 2.0上的2.1），Elasticsearch（ES 2.2上的2.0）的主要升级
    • 升级到Querydsl 4。
    • 支持Spring Data Redis的对象映射和存储库。
    • 支持Redis Cluster 。
    • 支持JPA和MongoDB中存储库查询方法的预测。
    • 将Spring Data Envers添加到发布系列中（以前是单独维护的）。

    注意：由于发布过程中的故障，部署到Maven Central的Spring Data JPA工件已损坏。我们立即为整列火车发布了服务版本，以便将建议的版本升级为Hopper-SR1 ！

    在我们精心策划的变更日志中找到更多有关这些功能的链接。在接下来的几天中，将有一篇后续博客文章，其中对添加到版本中的新功能进行了深入的讨论。

    升级到此发行版最简单的方法是将Spring Data BOM与版本结合使用Hopper-SR1 。如果您在后面使用任何具有主要商店升级功能的模块，请准备好也必须对应用程序代码进行一些更改。来自高斯林，如果您恰巧使用Querydsl，同样适用。

    Spring Boot用户必须更加谨慎，但是应该能够升级甚至在1.3版中都没有进行任何重大升级的模块（例如JPA，MongoDB）。如果您使用的是这样做的模块，则必须准备手动升级到存储驱动程序的较新版本并禁用一些自动配置。如有疑问，请查阅Spring Data Examples以获得示例设置。Spring Boot 1.4 M2将提供对Hopper的全面支持。

    团队将在不久的将来的各种会议上忙于介绍Hopper发行火车。JAX 2016将在两周内在德国举行，随后是巴塞罗那的Spring I / O和拉斯维加斯的SpringOne Platform 。如果您想了解更多有关发行列车中最新和最重要的信息，请确保已预订机票。

    下一个发行火车将被命名为Ingalls ，并且应该在SpringOne周围发行。
