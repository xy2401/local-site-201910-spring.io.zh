Spring REST Docs 1.1.0。RC1

工程
安迪·威尔金森
2016年4月28日

遵循1.1.0。M1 ，我很高兴宣布Spring REST Docs 1.1.0。RC1已发布，可从https://repo.spring.io/milestone/获取 。

什么是新的？

HTTPie请求代码段

引入了新的HTTPie请求代码段。与现有的curl请求代码段相似，新代码段包含用于请求的HTTPie命令。感谢Raman Gupta贡献了此新功能。

可重复使用的代码段

现在，可以使用某些通用配置创建代码片段一次，然后重新使用 。这样可以减少在API上记录公共部分时的重复，例如self链接。

轻松的摘要

默认情况下，如果您忽略了文档或尝试文档不存在的内容，Spring REST Docs将导致测试失败。这对于详细的API文档非常有用，但是在记录您要关注请求或响应的特定部分的方案时可能会受到限制。现在，您不必使用显式片段，即可轻松记录链接，请求和响应字段以及请求和路径参数，而不必显式忽略您不想记录的所有内容。

下一步是什么？

我们希望在几周后发布发行版本。请试用发布候选版本，如果您发现任何问题或对最新的改进有任何建议，请告诉我们 。

如果您想了解有关Spring REST Docs和测试驱动文档的更多信息，我将在5月在巴塞罗那举行的Spring / IO会议上谈论它。我很想在那见到你。
