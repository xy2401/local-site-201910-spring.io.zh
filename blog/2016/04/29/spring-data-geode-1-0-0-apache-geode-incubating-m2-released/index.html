Spring Data Geode 1.0.0。APACHE-GEODE-INCUBATING-M2发布

工程
约翰·布鲁姆
2016年4月29日

我很高兴地宣布发布Spring Data Geode 1.0.0。APACHE-GEODE-INCUBATING-M2，为最近宣布的 Apache Geode 1.0.0-incubating提供支持。M2发布。

Spring Data Geode是Spring Data GemFire的特定版本，支持Apache Geode，而不是Pivotal GemFire 。

Spring Data GemFire和Spring Data Geode都基于GitHub中相同的源存储库 ，但是Apache Geode支持位于apache-geode分支中。因此，您不会找到专门用于Apache Geode的单独的Spring Data项目 ，但这没有什么意义，因为您对Pivotal GemFire与Spring Data GemFire结合使用的现有知识将立即有用并可以转移到Apache Geode和Spring Data Geode中 。

去年，我写了一篇博客文章，详细介绍了Spring Data GemFire对Apache Geode 的支持。但是，此后发生了许多变化。

首先， Spring Data GemFire和Apache Geode支持现在有了一个单独的工件， spring-data-geode 。另外，我选择了更合乎逻辑和直观的版本号， 1.0.0.APACHE-GEODE-INCUBATING-M2 。最后，新的工件/版本甚至可以像Apache Geode本身一样在Maven Central中使用 。好极了！

因此，开发人员只需在她的Maven POM中声明以下依赖项即可： build.gradle文件以开始使用Spring开发Apache Geode应用程序…

// Maven POM

<dependency>
    <groupId>org.springframework.data</groupId>
    <artifactId>spring-data-geode</artifactId>
    <version>1.0.0.APACHE-GEODE-INCUBATING-M2</version>
</dependency>

// build.gradle

dependencies {
    compile 'org.springframework.data:spring-data-geode:1.0.0.APACHE-GEODE-INCUBATING-M2'
}

单独的工件将帮助用户使用Spring Data GemFire轻松区分对Apache Geode的支持和对Pivotal GemFire的长期支持。

版本号（ 1.0.0 ）以及版本限定符（ APACHE-GEODE-INCUBATING-M2 ）不仅表明Spring Data Geode和Apache Geode的成熟度，而且彼此对应，还表明支持什么版本的Apache Geode。

什么是新的？

此版本包括对Apache Geode的支持

  • 堆外内存
  • 连续查询（CQ）
  • 多站点WAN拓扑配置

有关Apache Geode的更多信息，请参见网站 。请参考Spring Data GemFire 项目页面以及有关如何使用Apache Geode有效构建Spring应用程序的参考指南 。

下一步是什么？

我计划完成对Apache Geode的Spring Boot Starter以及Spring Session 支持 。

我还对使用基于 Spring Java的配置配置 Pivotal GemFire或Apache Geode时如何改善用户体验有想法。您可以在SGF-492中跟踪我在这项工作上的进展。
