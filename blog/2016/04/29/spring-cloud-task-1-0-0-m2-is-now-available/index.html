Spring Cloud Task 1.0.0。M2现在可用

发布
迈克尔·米内拉
2016年4月29日

我们很高兴地宣布发布Spring Cloud Task 1.0.0。M2。这是Spring Cloud Task项目的第二个里程碑，具有一些令人兴奋的新功能！

什么是新的？

在Spring Cloud Task的M1版本中，我们介绍了任务的概念，即@EnableTask注释，以及微服务具有终结性的一般概念。从那以后我们一直很忙。新增功能的一些重点包括：

  • spring-cloud-task监听器
  • 与Spring Cloud Stream集成
  • 与Spring Batch集成
  • 更多样品

让我们看看其中的每个。

spring-cloud-task监听器

在任务之前，之后（或两者之后）执行逻辑可能会很有用。此版本引入了界面， TaskExecutionListener以及注释@BeforeTask和@AfterTask允许用户定义要在任务执行之前或任务完成后执行的逻辑。

与Spring Cloud Stream集成

将Spring Cloud Task与Spring Cloud Stream集成在一起，为开发人员提供了一个功能强大的工具来编排数据微服务。在此版本的Spring Cloud Task中，我们在两个项目之间添加了一些集成点。

第一个集成点是能够基于通过流发送的消息启动任务的能力。通过使用创建启动应用程序@EnableTaskLauncher注释和正确的依赖关系，您将有一个Spring Cloud Stream Sink可以监听TaskLaunchRequest消息。对于接收到的每个消息，启动器接收器将启动请求的任务。

@SpringBootApplication
@EnableTaskLauncher
public class TaskSinkApplication {

	public static void main(String[] args) {
		SpringApplication.run(TaskSinkApplication.class, args);
	}
}

在撰写本文时，CloudFoundry，Apache Mesos和Kubernetes都可以使用本地和Apache YARN的启动器。

Spring Cloud Task和Spring Cloud Stream之间的另一个集成点是Spring Cloud Task能够将事件发布到Spring Cloud Stream通道的能力。包括spring-cloud-task-stream依赖项（以及您选择的Spring Cloud Stream绑定器）添加到项目中后，任务将在任务的开始和结束时自动发出消息。具体来说，任务将发出TaskExecution在每个这些点上，允许在任务开始，结束以及结果通知上游应用程序。

与Spring Batch集成

使用Spring Cloud Task的常见用例是运行Spring Batch作业。因此，我们在新模块中的两个项目之间添加了一些有趣的集成点spring-cloud-task-batch 。

第一个集成点是通过Spring Cloud Stream通道发出批处理信息消息。与Spring XD容器提供的功能相似，作为任务运行的批处理作业（具有适当的依赖性）可以发出信息性消息，而无需其他配置。以下侦听器发出某种事件：

  • JobExecutionListener
  • StepExecutionListener
  • ChunkListener
  • ItemReadListener
  • ItemProcessListener
  • ItemWriteListener
  • SkipListener

第二个整合点是新的PartitionHandler ， DeployerPartitionHandler 。这个PartitionHandler实现使远程分区批处理作业中的工作人员可以作为任务启动，而不是预先部署并需要侦听工作。这样提供的好处是，只有在工作时才部署和运行工作人员，并且在工作完成后应将其关闭并清理。的DeployerPartitionHandler将任务的启动委托给Spring Cloud Deployer实施，因此支持任务启动的任何部署程序隐式都可以与此新版本一起使用PartitionHandler 。

更多样品

理解事物运作方式的最简单方法是观察它们的运行情况。因此，我们加强了示例模块，以提供此处提到的所有功能以及一些更基本的用例的示例。您可以在Github上查看我们的新样本。

我们从这里去哪里？

在发布RC版本之前，我们还有更多工作要做，应该与Spring Cloud Data Flow的相关版本保持一致。我们将忙于处理Spring Cloud Task中的所有最终问题，并提供Spring Cloud Data Flow所需的其他内容。
