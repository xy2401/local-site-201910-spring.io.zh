引入Spring Cloud Cloud Foundry Service Broker

发布
斯科特·弗雷德里克
2016年6月7日

我很高兴地宣布Spring Cloud系列的最新成员。Spring Cloud Cloud Foundry Service Broker是一个框架，用于为Cloud Foundry平台构建服务代理 。

Cloud Foundry服务经纪人

服务代理提供了一种通过托管服务扩展Cloud Foundry的方法，这些服务可由部署到平台的应用程序使用。托管服务通常向应用程序公开某种资源，例如数据库或其他持久性存储，消息系统或其他支持基础结构。服务代理发布服务和服务计划的目录，管理服务实例的供应和取消供应，并为应用程序消耗资源提供连接详细信息和凭据。服务代理已注册到Cloud Foundry，并使用定义良好的Service Broker REST API与平台进行通信。

Spring Cloud Cloud Foundry Service Broker将完整的服务代理REST API实施为Spring MVC端点。这使服务代理作者可以专注于管理支持资源所需的逻辑，而不必担心实现代理API语义。

一些历史

在发布Spring Boot之后不久， Steve Greenberg开发了spring-boot-cf-service-broker项目并将其贡献给Cloud Foundry社区。在随后的几个月中，许多贡献者帮助添加了功能，修复了错误并与Service Broker API保持同步。不幸的是，由于这是每个参与人员的“附带项目”，因此贡献者无法承诺使该项目保持最新状态。

Spring Cloud团队提出要取得所有权并致力于使项目保持最新状态，Steve亲切地同意允许将该项目移到Spring Cloud的保护下。向Spring Cloud的迁移包括对项目的打包结构和Maven坐标的更改，以及一些需要由服务代理作者实现的接口的清理和合理化。

前进的道路

Spring Cloud Cloud Foundry Service Broker将跟踪服务代理API的演变，并增加对最新代理API功能的支持，从而使开发人员能够快速响应API的更改。当然，欢迎和鼓励人们的贡献。由于与其他Spring Cloud项目之间没有相互依存关系，因此Spring Cloud Cloud Foundry Service Broker将不会成为Spring Cloud 发布培训的一部分 。

已经为该项目发布了一些候选版本。上一个项目的用户已经测试了这些RC，并提供了反馈。通过此介绍，Spring Cloud Cloud Foundry Service Broker将为GA 1.0.0版。发布。
