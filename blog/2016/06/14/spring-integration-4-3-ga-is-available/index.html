Spring Integration 4.3 GA可用

发布
阿尔特姆·比兰（Artem Bilan）
六月14，2016

尊敬的Spring社区，

我很高兴地宣布，现在可以从repo.spring.io以及Maven Central获得Spring Integration Framework 4.3 GA！维护4.2.8版本也已发布。连同内部4.2.7版本一起，它包含一些重要的错误修复 。

除了4.3 RC1公告中提到的功能主题外，GA版本还包括以下几个主要更改：

  • 升级到Spring Framework 4.3 GA，Spring AMQP 1.6 GA，Spring Security 4.1 GA;

  • 的FtpRemoteFileTemplate现在提供ExistsMode选项来控制exists()操作应该起作用；

  • 的FileSplitter让代表FileMarker对象作为JSON字符串（ markersJson选项）以避免额外的spring-integration-file依赖于分布式环境中的消费者应用程序；

  • 一个新的JdbcLockRegistry功能已引入。特别感谢Dave Syer ！

  • 整合Graph已通过通用网关节点进行了改进，并且LinkNode现在有一个type （ input ， output ， error ， discard ， route ）属性，以获得更好的可视化体验；

  • 其他一些小的改进，不仅简单而且（ WebSocketStompSessionManager.connect ） bug修复。

该版本解决了150多个JIRA问题 。另请参阅“ 新增功能”以获取主要更改的摘要。我们非常感谢该项目的许多外部贡献-谢谢大家！

作为一个预告片，与发布无关，而是传奇的Andy Clement的一项新酷功能-用于Spring Integration的Spring Flo 。该项目完全基于我们最新的Integration Graph JSON模型，并允许可视化集成模型并在运行时刷新它：

这项工作仍在进行中，敬请期待！

Spring Integration 4.3设计为所有4.x用户的直接升级。我们将继续支持4.2.x版本，直到今年年底。但是，我们建议您尽早升级到4.3。如先前所暗示的，在一般Spring Framework 4系统要求（JDK 6 +，Servlet 2.5+等）内， 4.3.x系列将享有到2019年的延长支持寿命 。

下一步：Spring Integration 5.0，具有Spring Framework 5.0基础和一些Reactive Streams支持。不要错过今年八月在拉斯维加斯举行的SpringOne Platform会议，来听听我们有关所有新功能和未来计划的消息！
