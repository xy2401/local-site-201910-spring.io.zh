Spring Cloud Data Flow 1.0 RC1发布

发布
马克·波拉克
2016年6月21日

我很高兴代表团队宣布Spring Cloud Data Flow 1.0 RC1的发布。

此发行版中添加了一些令人兴奋的新功能，这些新功能可以延续到今天也发行的其他Data Flow Server实现中。

数据流本地服务器

Cloud Foundry的数据流

Apache YARN的数据流

Kubernetes的数据流

Apache Mesos的数据流

请单击上面的链接，以获取每个运行时平台独有的功能的详细信息。该版本的重点是：

  • Spring Flo现在是一个开源项目，已集成到Spring Cloud Data Flow仪表板中，使您可以在浏览器中直观地设计数据管道 。

  • 批处理和任务事件可以用作YARN和Kubernetes服务器实现的流定义中的源。

  • 通过启用对服务器端点的HTTPS访问并支持基本身份验证和OAuth 2.0身份验证， 提高了安全性 。UI和外壳也已更新，以支持HTTPS和身份验证。

  • 现在，服务器使用RDBMS而不是Redis进行流/任务定义，应用程序注册和作业存储库。仅运行Redis Server才需要分析功能。默认配置使用嵌入式H2实例，但是支持Oracle，SqlServer，MySQL / MariaDB，PostgreSQL，H2和HSQLDB数据库。要使用Oracle和SqlServer，您将需要使用Spring Initializr创建自己的数据流服务器，并添加适当的JDBC驱动程序类。

  • 功能切换可启用/禁用分析，流和任务功能。默认情况下，使用配置启用所有这三个功能。

  • 对于所有服务器，包括“数据流本地服务器”，都不再预先注册应用程序。这有助于使我们开箱即用的应用程序版本与Data Flow Server版本生命周期脱钩。为方便起见，可通过用户友好的公共URL获得最新的流应用程序列表。例如，shell命令

    $ dataflow:>app import --uri http://bit.ly/stream-applications-kafka-maven

    将使用kafka活页夹加载最新的应用程序，并通过我们的maven存储库托管。有关注册应用程序组的更多信息，请参见参考指南。

  • 为了支持对应用程序配置属性的集中和一致的管理，Spring Cloud Data Flow服务器以及Spring Cloud Stream App Starters存储库提供的Spring Cloud Stream应用程序已包含Spring Cloud Config客户端库。当数据流服务器启动时，您还可以将通用的应用程序属性传递给所有流。

  • 您可以使用此版本的Spring Initializr进一步自定义我们的Spring Cloud Stream和Task应用程序启动器。

  • Spring Boot应用程序属性的白名单提供了Shell / UI信息，以显示一组首选的引导属性，以显示代码完成和应用程序信息。

  • 命名更改。现在可以在URL http：// localhost：9393 / dashboard下找到UI。现在，所有对“模块”的引用均已替换为“ app”。

有关功能，错误修复和改进的完整列表，请参阅封闭的1.0.0。RC GitHub问题。
