Spring AMQP 1.6发布

发布
阿尔特姆·比兰（Artem Bilan）
2016年6月1日

我们很高兴地宣布， Spring AMQP 1.6 GA（ 1.6.0.RELEASE ）现在可以在 Spring 发行版以及Maven Central中使用 。

首先，感谢以任何方式为该项目做出贡献的所有人：JIRA，GitHub问题，Pull Request，博客文章和文章，甚至还有简单的StackOverflow问题！

您可以在之前的Spring AMQP 1.6 RC1博客文章中找到完整的功能包。您可以参考参考文档中的新增功能以及整个1.6版本的已关闭JIRA问题 。

尽管如此，我们还是在最后一刻遇到了几个功能，我们也想在这里分享：

  • 现在可以在“ 用户ID”消息属性中填充RabbitTemplate通过userIdExpression 。通常我们可以使用username来自ConnectionFactory ：

<rabbit:template id="rabbitTemplate"
                 connection-factory="connectionFactory"
                 user-id-expression="@connectionFactory.username" />

  • 社区贡献的另一个特点是Builder流利的API Queue和Exchange定义：

@Bean
public Queue fooQueue() {
    return QueueBuilder.nonDurable("foo")
                  .autoDelete()
                  .exclusive()
                  .withArgument("foo", "bar")
                  .build();
}
...
@Bean
public Exchange fooExchange() {
    return ExchangeBuilder.directExchange("foo")
                  .autoDelete()
                  .delayed()
                  .durable()
                  .internal()
                  .withArgument("foo", "bar")
                  .build();;
}

这是最后1.x线释放，尽管有任何维护1.6.x可以发布版本，但仅可以修复错误和进行较小的改进。接下来（2017）是2.0这可能需要Spring 5，并包括一些Reactive Streams工作和Java 8代码库。
