These applications can run independently on variety of runtime platforms including: Cloud Foundry, Apache Yarn, Apache Mesos, Kubernetes, Docker, or even on your laptop.”

春天的这个星期-2016年6月28日

工程
乔什·朗（Josh Long）
2016年6月28日

欢迎来到春季本周的另一期！本周，我将在德国慕尼黑和沃尔夫斯堡与客户进行访问，然后前往韩国首尔，进行客户交流和一些用户群讨论。

这周，我们收到了许多令人振奋的Spring Cloud Data Flow新闻，这与新的Spring Cloud Data Flow RC1相吻合

  • Spring Cloud Data Flow项目的共同负责人好人Pollack博士刚刚宣布了Spring Cloud Data Flow RC1 ，它更新了支持本地部署的部署程序，并支持Apache YARN ， Kubernetes ， Mesos ，当然还有Cloud Foundry ，它是不可思议的集成。 Spring Cloud Flo（用于Spring Cloud Data Flow流的可视化）到Spring Cloud Data Flow仪表板，改进的OAuth支持等等 ！
  • 我真的很喜欢Mark Paluch的出色博客，它回绕了席卷全国的安全机密，Hashicorp Vault以及Spring Cloud Vault的工作原理 ，使人们蒙昧蒙昧。
  • 我喜欢我们自己的Spring Boot联合创始人Phil Webb和我们的云计算专家之一Andrew Clay Shafer之间的讨论！
  • 您是否看过我们的新Spring Cloud Task Starters？ 在页面上：“ Spring Cloud Task Application Starters是独立的可执行应用程序，可用于按需使用案例，例如数据库迁移，机器学习和计划的操作。这些应用程序可以在各种运行时平台上独立运行，这些平台包括：Cloud Foundry，Apache Yarn，Apache Mesos，Kubernetes，Docker，甚至在您的笔记本电脑上。”
  • 我们在JDriven的朋友在创建完全可执行文件方面写了一篇不错的文章.jar使用Spring Boot Maven插件
  • Jakub Narloch在自定义自动配置上撰写了一篇很棒的文章，以其基于Spring Boot的自定义组件模型在各种RxJava调度程序上运行工作
  • Stormpath自己的Micah Silverman撰写了一篇很棒的文章， 介绍了如何在15分钟内将Spring Boot应用程序与Stormpath Authentication连接起来
  • Trevor Page整理了一篇有趣的文章 ，讲述了Spring团队如何使我们成为Java程序员的生活越来越轻松。”我感到我们正在成为粉丝。.:-)
  • 我还没有读过，但是关于Spring Cloud Data Flow的这本书看起来很有趣 ！
  • 在纽约市？与Spring Cloud Data Flow负责人Mark Pollack亲自检查一下Spring Cloud Data Flow的这次聚会。
  • 我们自己的迈克尔·科特（MichaelCoté）为InfoQ撰写了一篇不错的文章， “多云是快车的安全带” 。他的帖子（以及他的演讲和播客！）有见识并且常常很搞笑。

SpringOne平台

我真的很期待今年的SpringOne上平台显示，在八月初的到来。这是构建应用程序的绝佳时机，而SpringOne Platform处于一个独特的位置，可以吸引更大的讨论：为什么我们要＃devops，＃cloud，＃agile或#microservice？参加2016年8月1-4日在美丽的拉斯维加斯的讨论，让我们找出答案！
