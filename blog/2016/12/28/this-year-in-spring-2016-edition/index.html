今年春季-2016年版

工程
乔什·朗（Josh Long）
2016年12月28日

欢迎来到春季特别的本周 ！每年我们都会总结年度亮点。今年没什么不同！这已经是漫长的一年，Pivotal Spring和Cloud Foundry团队似乎每天都在提供更大，更好的技术。如果您一直在关注“春季本周”，那么您无疑会意识到这些变化！即使对于我来说，也很难跟上令人眼花release乱的发行节奏！

因此，以下是我鼓励人们从2016年开始回顾的五件事：

  • Spring Boot ：今年发布了Spring Boot 1.4，该版本又包含对Spring Framework 4.3，Hibernate 5，Couchbase，Neo4J的支持，以及在changelog中讨论的许多其他功能。今年，在Pivotal Cloud Foundry 1.9中，Spring Boot与该平台的集成度更高，它对Spring Boot Actuators有着深入的支持，而Apps Manager控制台中还包含许多其他功能。此外，Spring Boot的下载量从2015年11月的每月240万增加到2016年的每月1020万！Spring Boot是席卷全国的新感觉，您可以参与其中并开始使用Spring Initializr 。

  • 春季云 ：今年看到第二（春季云布里克斯顿） 和第三（Spring Cloud卡姆登）春季云的主要版本，春季云天使之后，登场挤满了新的功能！今年，引入了对消费者驱动的基于合同的测试的支持，分布式跟踪，流处理支持，Apache Zookeeper支持等等。Spring Cloud与Pivotal Cloud Foundry进行了深度集成，并且由于它基于Spring Boot构建，因此可以从Spring Initializr轻松使用 。如果要构建分布式系统，那么您会发现很少有比Spring Cloud更一致，更全面的方法！

  • Spring Cloud Data Flow ：今年发布了可引导的 Spring Cloud Data Flow 1.0。Spring Cloud Data Flow是对Spring XD的以Spring Boot为中心的重新想象。它提供所有相同的支持，但方式与Sprnig Boot规范一致。现在，它就像@EnableDataFlowServer一样简单，可以运行新的Spring Cloud Data Flow服务器！您可以毫无问题地在Cloud Foundry，Mesos，YARN和Kubernetes上部署Spring Cloud Data Flow。它甚至在我们的Cloud Foundry，Pivotal Cloud Foundry 1.9发行版中都得到了深厚的支持。Spring Cloud Data Flow支持批处理数据处理，提取和分析以及流处理工作负载，并且支持整个集群中的这些工作负载的Orchestratino。与往常一样，您可以从Spring Initialzr开始使用Spring Cloud Data Flow。

  • Spring One Platform ：今年是我们有史以来规模最大的Pivotal活动的第一年，SpringOne Platform，数千名代表聚集在拉斯维加斯参加了一次活动，该活动捕捉到了Pivotal端到端应用程序和以平台为中心的推动力。如果您对敏捷，大数据，微服务，开发和持续交付感兴趣，那么没有比Spring One Platform更好的活动了。明年的数字将变得更大，更好，我们将在旧金山的Moscone中心举办2017年版活动 ！

  • Pivotal和Cloud Foundry ：今年我们发布了PCFDev，这是我们以开发者-笔记本电脑为中心的Cloud Foundry发行版。Spring Cloud Services（为Spring Cloud Netflix Eureka注册中心，Spring Cloud Config Server和Spring Cloud Netflix Hystrix仪表板的各种Spring Cloud中间件提供服务代理和集成）达到了1.0。今年，我们宣布并在Microsoft Azure和Google Compute Engine之上发布了Cloud Foundry的稳定集成。今年，宣布了Open Service Broker API，这是Cloud Foundry本身对服务代理机制的标准化；与富士通，谷歌，IBM，Pivotal，红帽和SAP的合作项目使开发人员，ISV和SaaS供应商能够以最直接，最有效的方式向运行在云原生产品（包括Cloud Foundry，OpenShift和Kubernetes）中的应用程序提供服务。可能。我们还宣布了对路由服务的支持，路由服务是可以以编程方式处理平台中路由的扩展平面。今年，我们还发布了Pivotal Cloud Foundry 1.9，该版本通过了解Spring Boot Actuator运行状况端点，指标等等，进一步加强了基于Spring Boot的应用程序与Cloud Foundry之间的集成 。它还在平台本身中引入了更智能的自动缩放器，现在支持延迟以及CPU。它还同时展示了250,000个容器。今年甚至看到了对Microsoft Azure和Google Cloud的多云支持的发布，甚至Google也加入了Cloud Foundry基金会！今年，Pivotal Cloud Foundry 也支持Spring Cloud Data Flow！好东西太多了！

无论如何！我们也可以从社区获得很多东西，所以让我们开始吧！

  • Spring Data负责人Oliver Gierke刚刚宣布了Spring Data Release Train Ingallas RC1 。该版本移至Spring 4.3.5基线，支持将域事件作为来自聚合根的Spring应用程序上下文事件公开，支持功能库Javaslang的Option类型，添加了Spring Data LDAP支持，支持Spring Data REST中的CORS标头等等。拿起碎片，踢轮胎。这次发布的火车以丹•英加尔斯 （ Dan Ingalls ）命名，我有幸在本月初见面 。
  • Spring Boot的共同创始人兼联合负责人Phil Webb刚刚宣布了Spring Boot 1.4.3，其中包含70多个修复，改进和更多内容！
  • Spring Security负责人Rob Winch刚刚宣布了Spring Security 3.2.1、4.1.4和4.2.1的CVE，并具有相应的修复功能
  • Spring Security主管Rob Winch刚刚宣布了Spring LDAP 2.2.1和2.3 。该发行版包含许多新功能，并见证了Spring LDAP向Spring Data伞项目的迁移。
  • Spring Tool Suite负责人Martin Lippert刚刚宣布了Spring Tool Suite 3.8.3
  • Spring Ninja Stephane Nicoll刚刚宣布了Spring Framework 4.3.5、4.2.9和3.2.18
  • 说到Spring Tool Suite，上周我做了另一个Spring Tip，这个技巧是关于Spring Tool Suite的。
  • 真好！谷歌宣布在Stackdriver上支持Zipkin标头
  • 您的负载均衡错了吗？有趣的ACM读物！
  • 我喜欢SpringOne平台上有关使用Spring Cloud从单片架构迁移到微服务的演讲

因此，我谨代表Pivotal和Spring团队祝您新年快乐，最安全！这是2017年！
