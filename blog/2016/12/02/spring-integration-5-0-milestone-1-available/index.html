We are pleased to announce that the first milestone for the 5.0 version of Spring Integration is now available.

This is a new major version, based on Spring Framework 5.0 and requires Java 8; this is the biggest change so far, but the following are also included:

  • The Java DSL is now rolled into the framework itself; there are some minor changes to the DSL, such as the removal of the .handleWithAdapter() methods and some general Factory classes. A complete discussion of the DSL changes can be found in the Migration Guide.

  • Upgrade to Spring Data Kay.

  • Upgrade to Spring AMQP 2.0.

  • First class support for TCP/UDP has been added to the DSL.

  • Spring Integration is now based on Reactor 3.0 and Messaging Gateway Promise methods now have to be changed to return Mono.

  • You can now configure mid-flow transactions via TransactionHandleMessageAdvice for adviceChain Messaging Annotations attribute and sub-element when using XML configuration.

Spring Integration 5.0 Milestone 1可用

发布
加里·罗素
2016年12月2日

我们很高兴地宣布，Spring Integration 5.0版本的第一个里程碑现已发布。

这是一个基于Spring Framework 5.0的新主版本，需要Java 8。这是迄今为止最大的更改，但还包括以下内容：

  • 现在， Java DSL已集成到框架本身中。 DSL有一些细微的变化，例如删除了.handleWithAdapter()方法和一些常规的Factory类。有关DSL更改的完整讨论，请参见《 迁移指南》 。

  • 升级到Spring Data Kay 。

  • 升级到Spring AMQP 2.0 。

  • DSL已添加了对TCP / UDP的一流支持。

  • Spring Integration现在基于Reactor 3.0，并且必须更改Messaging Gateway Promise方法以返回Mono 。

  • 您现在可以通过TransactionHandleMessageAdvice配置adviceChain Messaging Annotations属性和使用XML配置时的子元素。

当然，新一代Spring Integration的另一个目标是增加对Reactive Streams的支持。现在，它由ReactiveChannel和ReactiveConsumer 。而且，当使用ReactiveConsumer时，所有现成的MessageHandler现在都是Reactive Subscriber ，能够处理来自上游MessageChannel包括背压支持。

有关完整信息，请参见参考手册中的“新增功能 ”和JIRA发行说明 。

更多的新功能和改进，当然还有即将到来的里程碑中更多的Reactive Streams支持。
