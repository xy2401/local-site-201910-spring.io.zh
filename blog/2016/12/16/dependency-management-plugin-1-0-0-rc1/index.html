依赖性管理插件1.0.0。RC1

发布
安迪·威尔金森
2016年12月16日

我很高兴宣布1.0.0。依赖性管理插件的 RC1已发布。可从Gradle的插件门户以及Maven Central和Bintray获得。

什么是新的？

该插件已用Java重写，其API已正式化。引入了该API与插件内部之间的明确分隔。这需要进行一些重大更改，但是如果您使用Groovy DSL，则不会受到影响。

转换为Java并对API进行形式化还带来了一些增强：

对Gradle 3的官方支持

以前，该插件是用Groovy编写的，并试图支持Gradle 1、2和3。事实证明，这过于雄心勃勃 。两个主要问题是在三个不同的Groovy运行时（1.8、2.3和2.4）之间的二进制不兼容性，以及在Gradle的三个版本之间的重大更改。为了解决这些问题，Gradle团队的建议是用Java重写该插件，并放弃对Gradle 1.x的支持。此版本正是这样做的，插件的主要代码现在是100％Java，而Gradle 2.9现在是受支持的最低版本。因此，现在正式支持Gradle 3.x，将来更容易支持Gradle的新版本。令人欢迎的副作用是该插件的jar文件大小减少了13％。

对Gradle Script Kotlin的更好支持

以前，该插件的API受其以Groovy编写且其API和DSL以Groovy为中心的事实的影响很大。例如，在许多地方都使用了Groovy闭合，而没有提供非Groovy替代方案。这需要Kotlin脚本来使函数适应闭包，例如：

configure<DependencyManagementExtension> {
    imports(delegateClosureOf<ImportsHandler> {
        mavenBom("io.spring.platform:platform-bom:Athens-SR1")
    })
}

转换为Java突出了问题所在，在Gradle自己的API的带头作用下，1.0重载了许多以前仅使用Groovy的方法Closure也要摇篮Action 。重载的方法使您可以整理一下内容，例如：

configure<DependencyManagementExtension> {
    imports {
        it.mavenBom("io.spring.platform:platform-bom:Athens-SR1")
    }
}

下一步是什么？

随着API的形式化以及插件的版本即将接近1.0，现在是您试用一下的绝佳时机，让我们在GitHub或Gitter上告知您发现或希望看到的任何问题。

到目前为止，感谢您为插件做出的所有反馈和建议。
