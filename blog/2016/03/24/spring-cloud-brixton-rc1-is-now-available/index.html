Spring 的云彩布里克斯顿。RC1现在可用

工程
斯宾塞·吉布
2016年3月24日

我很高兴地代表团队宣布Spring Cloud Brixton Release Train的第一个候选版本已经发布。该里程碑今天可用，可以在我们的Spring Milestone存储库中找到 。

强调

布里克斯顿发行火车的一些亮点是：

  • Spring Boot 1.3.x和Spring 4.2.x支持
  • 通过Spring Cloud Cluster进行集群领导者选举和锁定
  • Hashicorp Consul通过Spring Cloud Consul支持服务注册/发现和配置
  • Apache Zookeeper支持服务注册/发现，通过Spring Cloud Zookeper进行配置以及Spring Cloud Cluster中的领导者选举
  • 通过Spring Cloud Sleuth抽象进行分布式跟踪，其中有两种现成的实现方式：一种支持日志记录（对于日志收集器和Logstash和Loggregator等多路复用器来说是理想的），另一种支持Twitter的Zipkin
  • Netflix Atlas遥测系统和下一代Spectator Metrics库在Spring Cloud Netflix中可用
  • Spring Cloud Bus现在由最近发布的Spring Cloud Stream支持

以下模块是Brixton的一部分。RC1：

  • Spring Cloud Commons 1.1.0。RC2
  • Spring Cloud Config 1.1.0。RC1
  • Spring Cloud Netflix 1.1.0。RC1
  • Spring Cloud Security 1.1.0。RC1
  • Spring Cloud AWS 1.1.0。RC1
  • Spring Cloud Bus 1.1.0。RC1
  • Spring Cloud Cloud Foundry 1.0.0。RC1
  • Spring Cloud Cluster 1.0.0。RC1
  • Spring Cloud Consul 1.0.0。RC1
  • Spring Cloud Sleuth 1.0.0。RC1
  • Spring Cloud Stream 1.0.0。RC1
  • Spring Cloud Zookeeper 1.0.0。RC1

我们期待着布里克斯顿。4月初发布。

而且，一如既往，我们欢迎您提供反馈：在GitHub ， Stack Overflow或Twitter上 。

笔记：

一种@LoadBalancedRestTemplate默认不再创建。有关详细信息，请参见更新的文档 。您需要在应用程序配置中创建它。例如：

@Configuration
public class MyConfiguration {

    @LoadBalanced
    @Bean
    RestTemplate restTemplate() {
        return new RestTemplate();
    }
}

请注意，要使用的正确BOM是spring-cloud-dependencies不spring-cloud-starter-parent （请参见下面的入门）。

入门

Brixton入门的最简单方法。RC1前往http://start.spring.io 。如果要使用带有BOM的Maven手动编写构建文件（仅依赖管理）：

<repositories>
    <repository>
        <id>spring-milestones</id>
        <name>Spring Milestones</name>
        <url>http://repo.spring.io/milestone</url>
        <snapshots>
            <enabled>false</enabled>
        </snapshots>
    </repository>
</repositories>
<dependencyManagement>
  <dependencies>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-dependencies</artifactId>
    <version>Brixton.RC1</version>
    <type>pom</type>
    <scope>import</scope>
  </dependencies>
</dependencyManagement>
<dependencies>
  <dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-config</artifactId>
  </dependency>
  <dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-eureka</artifactId>
  </dependency>
  ...
</dependencies>

或与gradle：

buildscript {
  dependencies {
    classpath "io.spring.gradle:dependency-management-plugin:0.5.6.RELEASE"
  }
}
repositories {
    maven {
        url 'http://repo.spring.io/milestone'
    }
}
apply plugin: "io.spring.dependency-management"

dependencyManagement {
  imports {
    mavenBom 'org.springframework.cloud:spring-cloud-dependencies:Brixton.RC1'
  }
}

dependencies {
    compile 'org.springframework.cloud:spring-cloud-starter-config'
    compile 'org.springframework.cloud:spring-cloud-starter-eureka'
    ...
}
