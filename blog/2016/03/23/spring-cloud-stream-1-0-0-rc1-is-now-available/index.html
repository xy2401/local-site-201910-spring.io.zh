Spring Cloud Stream 1.0.0。RC1现在可用

发布
马里乌斯·波哥维奇（Marius Bogoevici）
2016年3月23日

我很高兴代表团队宣布， Spring Cloud Stream的第一个候选版本已经发布。作为1.0.0之前的最后一个里程碑。RELEASE，它稳定了API，并在工具支持和内容类型管理领域提供了许多新功能和错误修复。以下是最重要更改的重点：

活页夹和绑定属性重组

绑定器和绑定的配置已更改为比Spring Boot配置元数据更友好的模型。现在，所有配置属性（包括特定于绑定的属性）都支持验证和完成（在Spring Tool Suite中）。

通用绑定设置以及绑定程序特定设置的配置属性已更改，请查阅文档以获取详细信息。

注意：目的地的简写形式，例如spring.cloud.stream.bindings.input=someDestination ，不再受支持。标准形式，即spring.cloud.stream.bindings.input.destination=someDestination必须改为使用。

@StreamListener用于Spring Messaging对齐和内容类型处理

除了Spring Integration支持（这是Spring Cloud Stream 1.0.0的一等公民）之外，还提供支持。RC1引入了新的@StreamListener注释为信息调度的方法，基于Spring信息基础设施的参数和内容类型处理（详情点击这里 ）。这样可以更无缝地处理具有已知内容类型的消息（例如带有内容类型标头的经过检验的JSON），并与其他Spring Cloud Stream应用程序以及某些类型的中间件（如Rabbit MQ）甚至不是基于Java的应用程序具有更好的互操作性。在Spring Cloud Stream上。

例如，接收String带有application/json内容类型标题，可以直接转换为Vote POJO。

@EnableBinding(Sink.class)
public class VoteHandler {

  @Autowired
  VotingService votingService;

  @StreamListener(Sink.INPUT)
  public void handle(Vote vote) {
    votingService.record(vote);
  }
}

此外，该版本还对内容类型管理领域的许多错误进行了修复，并改进了文档。完整的更改列表可在Git Hub中找到 。
