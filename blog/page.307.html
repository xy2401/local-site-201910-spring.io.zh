现在可用：SpringSource Tool Suite 2.3.1

发布
亚当·菲茨杰拉德
2010年3月11日

最新版本的SpringSource Tool Suite （STS）现已可用。STS是用于构建Spring，Groovy和Grails驱动的企业应用程序的最佳的Eclipse驱动开发环境。现在可以下载新版本（2.3.1），并使用最新的Eclipse 3.5.2基础。其他功能包括

  • 最新版本的tc Server Developer Edition ，改进了与Spring Insight控制台的集成。
  • 现在可以从项目的类路径加载命名空间处理程序，从而允许使用STS未知的命名空间
  • 显着的性能和内存消耗方面的改进。

SpringSource TC Server 2.0简介

工程
布鲁斯·斯奈德
2010年3月10日

在不久的将来，SpringSource tc Server 2.0™将会发布，其中包括一个新的Spring Edition，这代表了该产品的一些重大变化以及Web应用程序开发的一大进步。这也延续了SpringSource的承诺，即tc Server是Apache Tomcat的100％兼容的即插即用替代品，它提供了构建和运行Spring应用程序的最佳场所，非常适合现代虚拟环境。

基于众所周知的Tomcat服务器，tc Server添加了高级诊断，操作管理和关键任务支持功能。与传统的Java EE应用程序服务器相比，这些功能大大降低了成本和复杂性，同时提高了生产率。SpringSource tc Server现在提供三种不同的版本：Spring Edition，Standard Edition和Developer Edition。这三个版本的tc Server确保开发团队可以轻松访问，并为基于Spring和非基于Spring的应用程序提供了市场领先的企业级生产能力。

Spring Security命名空间的背后

工程
卢克·泰勒
2010年3月6日

通过在Spring Security 2中引入安全模式，使简单安全的应用程序的启动和运行变得更加容易。在较旧的版本中，用户必须分别声明和连接所有实现Bean，从而导致难以理解和维护的庞大而复杂的Spring应用程序上下文文件。学习曲线非常陡峭，我仍然记得，当我在2004年开始从事该项目（当时是Acegi Security）时，花了一些时间才得以解决。从积极的方面来说，对框架的基本构建模块的了解意味着，一旦您设法将工作配置在一起，几乎不可能不对重要的类以及它们如何协同工作有所了解。这些知识反过来又使您处于良好的位置，可以利用自定义的机会，这是使用Spring Security的最大好处之一。

Spring批处理集成

1.0.0的一部分。M2版本是Spring Batch集成模块，最近从Spring Batch迁移而来，并通过Batch Admin获得了新家。Spring Batch Integration中实现或演示了许多Batch-Integration交叉用例。之所以要使用新版本，是因为Batch Admin使用了Batch Integration的许多功能，因此调整这些项目的发布周期更加有意义。
