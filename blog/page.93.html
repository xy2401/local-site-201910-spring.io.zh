Spring Data Kay M2发布

发布

2017年4月13日

我谨代表Spring Data团队宣布发布火车Kay的第二个里程碑。该版本是迈向第二代Spring Data的重要一步。

我们已经将大部分代码库升级到Java 8，现在还包括Optional在方法签名中。这主要影响内部SPI，但也会泄漏到用户代码中，尤其是在CrudRepository 。对反应式Spring Data存储库的支持已扩展到Couchbase（谢谢，Subhashni！），Redis模块现在具有反应式模板API。该版本还支持IsEmpty和IsNotEmpty用于MongoDB和JPA的派生查询和实现。现在，MongoDB中的聚合框架还支持流结果。

2017年4月12日

我很高兴代表社区宣布， Spring Cloud Dalston发行培训的一般可用性（RELEASE）现已开始提供。该版本可以在Maven Central中找到。您可以查看Dalston 发行说明以获取更多信息 。

Dalston发布系列的重大变化

金库

Spring Cloud Vault Config是一个新项目，它通过Hashicorp Vault为分布式系统中的外部化秘密管理提供客户端支持。

配置服务器

Config Server现在通过Composite模式支持多个后端。这允许组合以前不可能的后端类型，例如：Vault和git或多个git后端。现在支持对AWS Codecommit托管的git存储库的身份验证。在以前的（Camden）版本中，Hashicorp Vault已作为后端添加到Config Server，以与基于VCS的后端一起使用。

Spring Integration 5.0 Milestone 3可用

发布
阿尔特姆·比兰（Artem Bilan）
2017年4月5日

Spring Integration小组高兴地宣布，Spring Integration 5.0版本的第三个里程碑（ 5.0.0.M3 ） 现在可用。

此发行版中制作了53个JIRA （以及一些GitHub问题），包括错误修复和许多新功能。自上一个里程碑2以来的一些亮点：

  • Spring Integration Testing Framework的初始实现- @SpringIntegrationTest测试类的注释和MockIntegration工厂帮助您编写集成流程和通道适配器的单元测试。我们打算在GA之前通过更多功能来消除此功能，包括更多的模拟，验证和一些功能。 send-and-receive隔离测试组件的实用程序。欢迎反馈！

  • POJO处理程序方法调用（ @ServiceActivator ， @Transformer等，或现在从XML定义调用的此类方法） InvocableHandlerMethod默认。与ConfigurableCompositeMessageConverter和@Default实用程序，使我们能够基于Content-Type和目标方法参数解析实现条件方法调用方案。要恢复以前基于SpEL的行为， @UseSpelInvoker提供了方法级别的注释。

  • 一个基于WebFlux的WebClientReactiveHttpRequestExecutingMessageHandler提供了实现。与一个ReactiveChannel作为一个outputChannel选项，它为远程HTTP服务使用提供背压方式。

  • （S）FTP（和AWS S3）入站通道适配器现在可以在本地还原文件树。为此，一个新的， Files.walk()基于RecursiveDirectoryScanner介绍。的useWatchService还提供了选项。

  • Web服务网关现在可以交换WebServiceMessage s直接作为入站/出站payload 。这样可以通过直接访问MTOM来提供支持WebServiceMessage属性。的UnmarshallingTransformer现在可以处理MimeMessage作为有效载荷将其解组为带有附件的对象图。

  • 回复产生MessageHandler现在回退到replyChannel回复消息的标题（如果没有） outputChannel要么replyChannel在请求消息标题中。当下一步由当前计算的结果确定时，这允许实施类似业务流程的方案。

春季AMQP 2.0.0里程碑3可用

发布
加里·罗素
2017年4月4日

我们很高兴地宣布，Spring AMQP 2.0版本（2.0.0。M3）现在可用。

此发行版中包含36个JIRA问题 ，包括错误修复和许多新功能：

  • 最初支持无经纪人的集成测试- TestRabbitTemplate在应用程序上下文中发现侦听器容器，并为这些容器调用这些容器的侦听器send()和sendAndReceive()操作。我们打算在GA之前使用更多功能来冲销此功能，包括路由和RabbitMQ支持的各种交换类型的仿真。欢迎反馈。

  • 现在，使用外部事务管理器进行的回滚与本地事务一致。

  • 4.1.x amqp-client库现在是默认库。

  • 的RabbitTemplate现在有一个invoke()允许在同一通道上顺序执行多个操作的方法。这还启用了诸如waitForConfirmsOrDie()将多个消息发送到同一通道时，由基础库提供。

  • null邮件属性不再包含toString()这使得调试日志更易于阅读。

  • @QueueBinding注释现在支持多个路由键，并支持自定义交换类型。
