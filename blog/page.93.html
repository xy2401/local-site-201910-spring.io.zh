Spring Data Kay M2发布

发布
奥利弗·德罗博姆（Oliver Drotbohm）
2017年4月13日

我谨代表Spring Data团队宣布发布火车Kay的第二个里程碑。该发行版是迈向第二代Spring Data的重要一步。

我们已经将大部分代码库升级到Java 8，现在也包含了例如方法签名中的Optional 。这主要影响内部SPI，但也会泄漏到用户代码中，尤其是在CrudRepository 。对反应式Spring Data存储库的支持已扩展到Couchbase（谢谢，Subhashni！），Redis模块现在具有反应式模板API。该版本还提供对IsEmpty和IsNotEmpty支持，以支持派生查询以及对MongoDB和JPA的查询。现在，MongoDB中的聚合框架还支持流结果。

Spring Cloud Dalston发布

发布
斯宾塞·吉布
2017年4月12日

我很高兴代表社区宣布， Spring Cloud Dalston发行培训的一般可用性（RELEASE）现已开始提供。该版本可以在Maven Central中找到。您可以查看Dalston 发行说明以获取更多信息 。

Dalston发布系列的重大变化

金库

Spring Cloud Vault Config是一个新项目，可通过Hashicorp Vault为分布式系统中的外部化秘密管理提供客户端支持。

配置服务器

Config Server现在通过Composite模式支持多个后端。这允许组合以前不可能的后端类型，例如：Vault和git或多个git后端。现在支持对AWS Codecommit托管的git存储库的身份验证。在以前的（Camden）版本中，已经将Hashicorp Vault作为后端添加到Config Server，以与基于VCS的后端一起使用。

适用于Apache Kafka 1.2.0和1.1.4的Spring

发布
加里·罗素
2017年4月7日

我们很高兴地宣布，这些针对Apache Kafka的Spring维护版本现已发布，版本为1.2.0。发布和1.1.4。发布。

这些版本包括一些错误修复和改进，并引入了对KStreams的支持。

它们在功能上是等效的； 1.2.0。RELEASE基于0.10.2.0 kafka-clients jar和1.1.4。RELEASE可以与0.10.0.x和0.10.1.x一起使用。

而1.1.4。RELEASE将在运行时与0.10.2.x客户端库一起使用，嵌入式kafka API中的一些重大更改意味着spring-kafka-test中的嵌入式kafka Junit @Rule无法工作，而1.2.0。为此，需要发布。
