Spring Data REST（CVE-2017-8046）中的安全性问题

工程
奥利弗·德罗博姆
2018年3月6日

去年秋天，发现了一个影响Spring Data REST的安全漏洞。我们修补了受影响的模块并发布了CVE 。我们最近看到了一些与此有关的新闻，这引起了混乱。这是独家新闻：

tl; dr：

  • 存在一个安全漏洞，该漏洞允许在Spring Data REST 2.6.8和3.0.0版之前的任意代码执行。
  • 在以下版本中已修复此漏洞：
    – Spring Data REST 2.6.9（Ingalls SR9，2017年10月27日），包含在Spring Boot 1.5.9（2017年10月28日）中。
    – Spring Data REST 3.0.1（Kay SR1，2017年10月27日），包括在Spring Boot 2.0 M6中（2017年11月6日）
  • CVE最初于2017年9月底发布。我们最初以为我们已经用几天前发布的发行版解决了该问题。随后的反馈表明情况并非如此，该问题最终于2017年10月/ 11月得到解决。遗憾的是，CVE没有更新以反映这一点。团队正在努力确保不再发生这种缺乏更新的情况。
