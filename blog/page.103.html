用于Cloud Foundry 1.1 GA的Spring Cloud Data Flow发布

发布
埃里克·博塔德
一月18，2017

我们很高兴地宣布Cloud Foundry版本1.1.0的Spring Cloud Data Flow全面上市。发布。

除了此发行版所基于的Spring Cloud Data Flow 1.1.2的所有功能之外，Cloud Foundry的具体细节还包括：

  • 内存优化：经过大量的故障排除后，我们很高兴在此版本中提供内存高效的SCDF服务器。通过现在的更改，无论有多少个流/任务部署和并发性，我们都观察到服务器的内存使用量保持在200MB范围内，而不是1G或最终在之前的1.0.0版中崩溃。发布。如果您以前曾遇到过虚假的崩溃，我们强烈建议您升级到1.1.0。立即发布。

  • Maven缓存和磁盘优化：我们改进了Maven缓存的应用和使用方式。Maven缓存的顶部有一个新的LRU（最近最少使用）设计，它使我们可以更加主动地了解缓存的大小，而不是无限期地增长，如果不加以控制，则会导致磁盘空间不足。服务器正在运行的应用程序容器。

  • 任务支持：在最新的PCF 1.9版本中，任务组件从试验功能升级为可用于生产的功能。Spring Cloud Data Flow对Cloud Foundry的任务支持与该开发密切相关，并且在此版本中，我们也从我们这一边删除了“实验”子句。在PCF 1.9上使用时，Spring Cloud Data Flow的任务支持已准备就绪。人们仍然可以在较旧的PCF版本（例如PCF 1.8）上尝试此功能，但是由于较旧的PCF版本被归类为“实验性”，因此既不能保证也不支持该功能。

    在此版本中，任务数据库架构稍有更改。对于从1.0.0升级的人。发行至1.1.0。发布，请查看参考指南中包含的迁移步骤 。

  • 改进的错误处理：在用于Cloud Foundry项目的最新版本的Spring Cloud Deployer的基础上 ，与Cloud Controller REST API的交互已变得更具弹性，在处理偶然的服务器端错误时更加主动。
