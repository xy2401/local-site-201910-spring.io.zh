适用于Apache Geode和Pivotal GemFire 1.1.0的Spring Boot。M1发布！

发布
约翰·布鲁姆
五月7，2019

我很高兴地宣布发布适用于Apache Geode和Pivotal GemFire / PCC （SBDG）的Spring Boot。 1.1.0.M1 。

位（ org.springframework.geode:spring-geode-starter:1.1.0.M1 ）可在Spring libs-milestone存储库中找到。

什么是新的

SBDG 1.1里程碑1（M1）版本包括：

  • 在Spring Framework 5.1，Spring Boot 2.1，Spring Data Lovelace和Spring Session 2.1上重新建立SBDG的基础。

  • 在Apache Geode 1.6.0和Pivotal GemFire 9.5.3上重新建立SBDG的基础。

有关更改的完整列表，请参见changelog 。

下一步是什么

SBDG 1.1 M2将包括对在线缓存的专门支持，这是我们3个单独的缓存模式中的第二个缓存模式[ Look-Aside ， Inline ， Near ]。

谨在此提醒您，Spring的Cache Abstraction已经为后备缓存模式提供了出色的支持，而SBDG（通过SDG的扩展）为使用Apache Geode或Pivotal GemFire / PCC作为Spring Cache中的缓存提供程序提供了一流的支持 。 抽象 。

新 的“后备缓存 示例”也很好地介绍了这一点，其中同时包含指南和示例代码 。

Spring Cloud Open Service Broker 3.0.0发布

发布
罗伊·克拉克森
2019年4月29日

我们很高兴地宣布Spring Cloud Open Service Broker 3.0.0全面上市。该版本引入了从命令式API到响应式API的重大转变，以支持基于响应式的新Web框架Spring WebFlux。 V3.0.0在同时开发的v2.1.x和有特点的奇偶V2.1.2 。展望未来，所有新功能将仅添加到3.x版本中。

v3.0.0中提供的新功能：

  • 继续支持Open Service Broker API v2.14 。
  • 支持Spring Boot 2.1和Spring Framework 5.1
  • 通过新的Reactive API（基于Project Reactor）引入对Spring WebFlux的支持
  • 继续支持Spring MVC
  • 重命名Spring Boot入门程序，因为它现在支持两个Spring Web框架
  • 介绍几个反应式生命周期挂钩，以执行其他操作
