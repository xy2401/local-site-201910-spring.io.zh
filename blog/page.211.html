春天的这个星期-2014年5月27日

工程
乔什·朗（Josh Long）
2014年5月28日

欢迎来到春季本周的另一期！在美国，我们有一个为期3天的周末，这很好。观看一些精彩的技术讲座的好机会！如果您像我一样，那么您的技术讨论已经用光了，很高兴看到我们今天还有很多事情要做！

  1. Grails项目负责人Graeme Rocher刚刚宣布了Grails 2.4 ！新版本之所以令人惊叹，有很多原因，其中最重要的一点是它的许多令人惊叹的功能中的一些功能也易于在Spring Boot中使用。双赢！恭喜，格雷姆和团队。（还要感谢您，发布了另一个出色的版本！）
  2. Spring Boot 1.1.0。M2现在可用 ！新版本增强了对Spring Data Solr，Spring Data Gemfire和整个Spring Data Dijkstra发行版的支持。新版本还提供GSP（Groovy服务器页面）和Velocity作为模板选项，以及对各种库的升级，例如Spring Security 3.2.4和Spring Batch 3.0。该释放火车的移动速度很快，因此请尽速上船！还有很多很棒的东西，所以请查看发行说明。
  3. 6月的网络研讨会在这里！Ramnivas Laddad将于6月3日启动Spring Cloud，以抽象化PaaS服务使其可移植到Spring Cloud中 ； Michael Minella在6月10日发布在Spring Batch 3.0.0中 ； Glenn Renfro在6月17日启动启动的Spring Integration中 。
  4. 该博客介绍了Hibernate内置JDBC日志记录的一些局限性，然后介绍了在简单的Spring / Hibernate应用程序中使用的log4jdbc 。
  5. 我们的朋友Eugen Paraschiv 对Spring Data JPA进行了很好的介绍 。
  6. Vaadin团队的Matti Tahvonen整理了有关使用Spring Data Neo4j，Spring Boot以及我和PetterHolmström所创建的Vaadin4Spring库的精彩介绍。公平地说，由于Petter的缘故，它仅可用：)实际上，如此有用， Matti能够很快将漂亮的东西-Bootiful-组合在一起 。看看这个！应用程序建模（并可视化！）数据存储在Neo4j中，所以它不仅是任何旧的CRUD应用程序，这非常酷！
  7. CloudFoundry忍者詹姆斯拜耳宣布的新枢纽CF 1.2 ，现在支持VMware的混合云服务，最初的审计和自动缩放，新的数据服务（Redis的，MongoDB的，Neo4j的，RiakCS和ElasticSearch都可用！），以及更多 ！
  8. 好奇吗？想更多地了解CloudFoundry，Pivotal的开源Paas？观看这段史诗视频，了解如何使用SpringOne2GX 2013中的BOSH设置自己的 Paas 。并在SprignOne2GX重放中了解其他团队如何做到这一点- 通过CloudFoundry进行自己的自由：私有云体验 。
  9. 数据忍者Thomas Risberg刚刚宣布了适用于Apache Hadoop RC4的新版Spring，太棒了！新版本在Spring YARN集成的基础上进行了改进。当然，YARN是分布式的通用运行时，Hadoop 2.0现在非常特定的映射/归约支持位于该运行时之上。但是，您可以使用YARN进行自己的工作分配。例如，您可能使用YARN来拆分Spring Batch工作负载。在新示例中，Janne Valkealahti演示了一个简单的Spring Boot驱动的Spring YARN组件。现在这东西多么简洁， 真是太神奇了！
  10. 想更多地了解Spring YARN支持背后的疯狂（数据）科学家Janne Valkealahti？查看此Pivotal People个人资料 ！
  11. Spring Data负责人Oliver Gierke很好地介绍了Spring Data Dijkstra （新的Spring Data伞版）中的新功能。此版本包含Java 8 Optional支持，异步存储库方法调用等。
  12. Spring Security主管Rob Winch仅发布了两个小的错误修正版本： Spring LDAP 2.0.2和Spring Security 3.2.4
  13. Spring Web Services负责人Arjen Poutsma 刚刚宣布了Spring Web Services 2.2.0，该版本现在具有Java配置API以及更多功能。
  14. 让我花点时间提醒大家：Java配置无处不在！Spring框架，启动，数据，安全性，MVC，集成，批处理，社交等等，都提供了XML丰富的Java配置集成。对于Boot来说，Java配置是唯一的即用型选项，尽管您当然可以根据需要使用XML。只是..没想到。
  15. Spring Security主管还写了关于使用Spring Security Test来处理Web安全的博客
  16. 想编写自己的Spring Boot启动器吗？在Spring ninja Stephane Nicoll中查看此示例， 了解如何为Spring Boot提供HornetQ 启动器
  17. 最后， Spring Social Google 1.0.0。GA已发布 ！恭喜加布里埃尔，到达这里所需的所有工作。我也喜欢易于使用的示例。
  18. Arnaud Giuliani对在Spring Boot中使用GWT进行了非常酷的介绍 。做得好，阿诺！
  19. Netflix 工程师Tomas Lin在推特上发布了指向REST API的便捷Spring MVC异常处理程序的链接。这是解决问题的一种（精细）方法。另外，建议您看看[使用Spring HATEOAS' VndError （ s ）支持] a）。

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2014年5月27日

演讲者：Glenn Renfro

随着发出信息的设备数量的增加，能够检索该数据并进行相应处理变得越来越重要。为响应此需求，MQTT已成为连接“物联网”的事实上的轻量级传输。话虽如此，您的应用程序如何支持事件，消息传递和调度？使用Spring Boot和Spring Integration，您将看到如何使用调度程序创建应用程序，该调度程序将从Web服务检索数据，通过MQTT清理并发出数据。然后，我们将向您展示如何创建一个使用Spring Boot和Spring Integration编写的应用程序，该应用程序将捕获MQTT事件并记录结果。从此讨论中，您可以了解如何使用这些工具，并在返回办公室后立即将其用于自己的大数据项目。

网络研讨会：将PaaS服务抽象为可通过Spring Cloud移植

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2014年5月27日

演讲者：拉姆尼瓦斯·拉达德（Ramnivas Laddad）

将应用程序开发到云平台需要使用已部署的应用程序环境并连接到服务。新项目Spring Cloud在包括Cloud Foundry和Heroku在内的各种云平台中简化了这些任务。Spring Cloud可以将相同的工件（战争或罐子）部署到多个云环境。它通过Cloud Connector的概念支持多个云，并为Cloud Foundry和Heroku提供了现成的实现，并为其他云平台提供了扩展点。在本讲座中，我们将介绍Spring Cloud项目，展示如何简化配置应用程序以进行云部署，讨论其可扩展性机制，并通过展示该领域的实际示例来充分利用它。
