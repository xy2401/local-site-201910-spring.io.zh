抢先体验：用于Eclipse Indigo的SpringSource工具套件（3.7）

工程
马丁·利珀特
2011年3月25日

几天以来，就可以从Eclipse中下载Eclipse Indigo（3.7）M6a软件包，因此我们有时间允许您在该里程碑版本的顶部使用SpringSource Tool Suite（STS）。 :-)

它只是STS的早期访问版本，但我们设法将Eclipse 3.7的所有部分整合在一起：Eclipse 3.7的AJDT版本，运行3.7的Groovy-Eclipse版本以及SpringSource Tool Suite本身，当然包含Spring IDE，Grails工具和您从STS知道的所有其他不错的功能-现在都在最新的Eclipse Indigo里程碑之上运行。

如何安装



  • 为Java EE开发人员下载并安装Eclipse Indigo（3.7）M6a 。

  • 下载bookmarks.xml并将其导入您的更新站点列表（首选项->安装/更新->可用更新站点->导入）。

  • 转到“安装新软件”，然后选择适用于Eclipse 3.7的SpringSource Tool Suite： http://dist.springsource.com/snapshot/TOOLS/nightly/e3.7 。

  • 选择您要使用和安装的组件。

  • 在Eclipse 3.7上重新启动并使用STS。


可以从组合更新站点获得STS的必要依赖项： http://dist.springsource.com/snapshot/TOOLS/composite/e3.7 ，如果您想手动添加。

我们还在STS的安装说明的“从Nightly Snapshot Update Site 安装 ”部分中提供了3.7的安装说明 。

手动安装


以下是单个更新站点，您可以从中安装STS，AJDT和Groovy-Eclipse for Indigo（3.7）：

  • http://dist.springsource.com/snapshot/TOOLS/nightly/e3.7

  • http://dist.codehaus.org/groovy/distributions/greclipse/snapshot/e3.7/

  • http://download.eclipse.org/tools/ajdt/37/dev/update



反馈


这是不断更新的STS的早期访问版本。如果您发现在Eclipse Indigo（3.7）M6a（或更高版本）安装之上运行STS时遇到问题，请告知我们并将问题报告给： https://issuetracker.springsource.com/browse/STS 。
