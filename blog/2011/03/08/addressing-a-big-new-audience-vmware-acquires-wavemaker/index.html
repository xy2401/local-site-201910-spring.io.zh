VMware announces with this blog the acquisition of WaveMaker, a widely used graphical tool that enables non-expert developers to build web applications quickly.  While WaveMaker is already part of the Spring ecosystem, it will now become an integral part of the Spring family and VMware’s cloud strategy. All of WaveMaker’s staff will be joining VMware.

This acquisition furthers VMware’s cloud application platform strategy by empowering additional developers to build and run modern applications that share information with underlying infrastructure to maximize performance, quality of service and infrastructure utilization.

吸引大量新受众：VMware收购WaveMaker

工程
罗德·约翰逊
2011年3月8日

今天， VMware通过此博客宣布收购WaveMaker ， WaveMaker是一种广泛使用的图形工具，使非专业开发人员能够快速构建Web应用程序。WaveMaker已经成为Spring生态系统的一部分，但现在将成为Spring系列和VMware云战略不可或缺的一部分。WaveMaker的所有员工都将加入VMware。

此次收购通过授权其他开发人员构建和运行与底层基础架构共享信息的现代应用程序，从而进一步提高了VMware的云应用程序平台战略，从而最大限度地提高了性能，服务质量和基础架构利用率。

适合的地方

在过去的8年中，Spring使Java应用程序的构建变得非常容易。Spring的第一批采用者是高级开发人员，他们详细了解了上一代EJB技术并寻求更好的替代方案。在某些情况下，他们自己成为Spring开发人员。下一波是主流业务应用程序开发人员，他们专注于实现业务需求，而没有基础架构方面的专门知识。随着Spring社区进一步发展到数百万个，并且Spring被越来越多的组织要求作为开发解决方案，它逐渐被具有各种技能的开发人员所接受。每个版本的Spring都使应用程序更简单，并减少了所需的代码和配置量。 Grails和Spring Roo的出现也有助于简化各种技能水平的开发人员的日常工作。

对于专业软件开发人员来说，我们一直都有着很棒的故事。但是，并非所有业务应用程序都是由专业开发人员编写的。例如，在金融服务行业工作的任何人都知道商务人员使用Excel电子表格实现了多少功能。特别是在Web应用程序的情况下，业务开发人员可以将许多解决方案从其他组件（例如仪表板）交付或组合在一起。

愿景

我一直认为，我们最终应该提供基于Spring的技术，以使技术含量较低的用户可以构建此类应用程序，从而增加Spring平台的价值，并以最廉价的方式解决许多业务问题。实际上，这意味着在可能实现自动化的区域（例如UI设计）中引入可视化工具来代替编码。做对了，对于某些类的应用，这种自动化甚至可以使专业开发人员受益。

输入WaveMaker。

WaveMaker是一种经过验证的，基于浏览器的可视化工具，用于构建Spring应用程序。

借助WaveMaker，我们可以为编码和非编码开发人员在Spring平台上协作提供独特的机会，从而提供巨大的商业价值。

传统上，这种方法的致命弱点是可视化工具生成的应用程序是开发人员不想手工编辑的，这意味着一旦非技术人员碰壁并需要对应用程序进行进一步的修改，专业的开发人员将无所不用其极从头开始重新实现应用程序。

我们选择WaveMaker是因为有了它，我们可以避免此问题。WaveMaker应用程序是Spring应用程序。展望未来，我们将确保WaveMaker应用程序与Spring的最佳实践和创新保持同步。

像Spring一样，WaveMaker是开源的，并且社区正在增长。在过去的12个月中，采用率急剧上升 ，每月下载量为135K。让我成为第一个欢迎WaveMaker社区加入SpringSource家族并鼓励SpringSource社区看看如何开始使用WaveMaker的人 。

接下来是什么？

我们对未来感到兴奋。您可以期望的一些亮点：


  • 与Spring Roo的紧密结合将确保Spring Roo和WaveMaker应用程序之间的架构一致，并使开发人员可以在三个级别上使用WaveMaker应用程序：传统的纯代码样式；由Spring Roo以代码导向的方式涡轮增压；以及WaveMaker图形环境。这将是一组独特的引人注目的选择。



  • WaveMaker即服务将自然与我们的云计算策略（包括Code2Cloud）契合。

  • 当前，WaveMaker使用数据库表和Web服务端点作为输入来构建应用程序。我们还有很多与Spring合作的优势。我们将通过展示Spring的许多其他功能（例如Spring Integration和Spring Social端点）以及其他云服务来丰富此功能。


对于Spring和VMware云相关技术来说，今年将是丰收的一年。通过收购WaveMaker，我们将能够将我们的新技术的影响带给更大的受众。
