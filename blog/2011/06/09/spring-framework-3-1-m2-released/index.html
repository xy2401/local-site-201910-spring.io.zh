Spring Framework 3.1 M2发布

工程
于尔根·霍勒（Juergen Hoeller）
2011年6月9日

Spring Framework 3.1 M2已于本周发布 ，标志着3.1里程碑阶段的结束。我们现在进入候选发布阶段，准备在7月完成功能完整的RC1，并在9月发布GA版本。

3.1 M2完成了从2月的3.1 M1开始的几个主要主题的工作：

  • 我们已经稳定了环境抽象和环境配置文件机制 。如果您还没有尝试过，那么现在是检查它的好时机！

  • 我们基于Java的应用程序配置方法已从M1中的@Feature方法更改为M2中常规@Configuration类的@ Enable *注释 。

  • 缓存抽象已经过修订，可提供最小的缓存交互SPI。我们的声明性缓存解决方案（@Cacheable等）始终位于其之上。

此外，我们在总体配置中增加了重要的新功能：

  • 对基于Servlet 3.0的初始化程序的丰富支持：我们的新WebApplicationInitializer方法允许在没有web.xml的情况下引导Spring Web应用程序！

  • JPA的新“ packagesToScan”功能，可通过扫描指定的基本软件包来构建默认的持久性单元 - 无需persistence.xml或其他元数据文件。

  • 我们在Spring @MVC中提供的新RequestMappingHandlerAdapter ：基于灵活HandlerMethod抽象的MVC处理的可自定义后端 。

当然，还有很多其他功能可以发现，例如：

  • 我们新的“ c：”命名空间可使用简洁的内联样式方便地按名称指定构造函数参数 。

  • 通过对@Configuration类和环境配置文件的一流测试支持对 TestContext框架进行了全面改革。

  • REST支持在Spring MVC中对URI模板，路径变量处理和内容类型规范进行改进。
