定义虚拟化和云Java的未来

工程
罗德·约翰逊
2011年6月14日

今天，我很自豪地宣布我们的VMware vFabric™应用程序平台的第5版，为云和虚拟化执行环境定义了企业Java的未来。 vFabric通过提供与下一代平台服务配对的现代编程模型，开辟了通往新的现代云架构的道路。这是一条不会因上一代技术的复杂性和复杂性而出现的道路。借助vFabric 5，VMware可以确保企业Java准备就绪，可以应对未来要求苛刻的，数据密集型，可大规模扩展的应用程序的挑战。

vFabric 5继续提供最佳的运行vFabric tc Server的 Spring应用程序的场所，并能够通过vFabric Hyperic以不可思议的智能监控和管理这些生产解决方案。该平台还解决了云计算所面临的技术挑战，支持新的数据管理方法，使我们的应用程序能够通过我们的vFabric GemFire和RabbitMQ技术跨弹性，地理分布的云架构进行扩展。此版本引入了重要的新技术，这些新技术可以利用基础虚拟化环境为Java应用程序提供新的灵活性和控制力

针对vSphere进行了优化

对于已经在运行vSphere的客户，vFabric 5向tc Server添加了新的Java弹性内存（EM4J）功能，从而在应用程序服务器和基础虚拟机之间提供了全新的协调级别。EM4J使用基础的vSphere虚拟化来克服Java静态内存堆的某些限制。

通过在JVM内部使用有助于协调JVM堆管理的内存膨胀技术，EM4J可以与基础虚拟机的内存管理进行通信，以允许共享内存并将内存转移到最需要的位置。这是在不影响JVM执行力的情况下实现的，因此，如果该实例上的工作负载达到峰值，则无需您的应用程序或管理工具就可以执行任何特殊操作，就可以恢复内存。

EM4J允许更高级别的Java服务器整合，并保护应用程序免受工作负载高峰的影响，而不会浪费内存。

运行Spring应用程序的最佳平台

许多Spring开发人员已经熟悉了他们在tc Server Developer Edition中可以使用的Spring Insight技术。Spring Insight提供了代码级跟踪-透明地为Spring应用程序跟踪事务在每个方法和JDBC（数据库）调用中花费的时间。无需修改或设置代码。

vFabric 5现在引入了Spring Insight Operations，它利用了相同的代码级跟踪技术，但是通过汇总视图，下钻和历史比较将来自多个应用程序服务器的信息汇总到一个控制台中。该解决方案旨在最大程度地减少监视开销，因此适用于测试和生产环境。

尽管Spring Insight Operations控制台对于可能需要对生产中的问题进行故障排除的管理员和开发人员都非常有用，但让管理员和操作人员在一个地方进行vFabric监视也很重要。因此，Spring Insight Operations监视不仅可以作为独立控制台使用，还可以通过Hyperic监视解决方案的插件使用，从而实现跨多个来源的数据的单一视图以及集成的警报和控制措施。

包装和定价的简单性和敏捷性

随着应用程序越来越多地部署在虚拟基础架构而不是物理硬件上，我们的客户告诉我们，基于虚拟机的基于平均使用量（而非高峰）的许可方法很有意义。 vFabric 5引入了基于虚拟机的打包和许可模型，该模型允许在VMware vSphere上运行的应用程序基础架构按使用情况进行使用，并为客户提供了灵活性，可以在每个许可的vFabric VM中运行vFabric产品系列中的任何或所有软件。这种方法消除了长期提供应用程序基础架构以适应峰值工作负载的长期需求，并使客户能够重新配置应用程序基础架构而无需复杂的许可问题。这将导致应用程序基础结构可以以开发团队用来构建应用程序的敏捷性发展。来自已配置VM的性能数据可用于迭代地优化应用程序基础结构，并将更多的计算资源专用于最需要关注的层。

还有更多...通过SQL访问GemFire数据结构

公告的一部分还包括一个令人兴奋的新产品，称为vFabric SQLFire。vFabric SQLFire的beta版利用了久经考验的vFabric GemFire基础，可提供内存速度和水平规模的数据，但vFabric SQLFire向该服务添加了熟悉的标准SQL和JDBC接口。测试版下载，文档和讨论可以在这里找到。

借助Cloud Foundry的 最新版本，Spring的新版本以及现在的vFabric 5，我们的Spring和vFabric工程师正在为云和虚拟化执行环境定义企业Java的未来。我鼓励您自己尝试vFabric，并充分利用虚拟机中运行Java的知识。
