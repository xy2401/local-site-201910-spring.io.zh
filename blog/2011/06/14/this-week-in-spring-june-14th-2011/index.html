Spring 的一周：2011年6月14日

工程
乔什·朗（Josh Long）
2011年6月14日

欢迎回到“春季的这一周”的另一期，已经过去了一周！上周这是Spring 3.1 M2 和 vFabric 5的发布 ！那里有很多令人兴奋的话题以及社区新闻，所以让我们开始吧！

  1. 今天，VMware 宣布发布了VMware vFabric 5 ，该应用程序平台为云和虚拟化执行环境定义了企业Java的未来。 vFabric 5包含Spring社区已经熟悉的许多技术，包括tc Server ， Hyperic ， GemFire和RabbitMQ ，但是现在增加了一些新技术。
    • Java弹性内存 （EM4J）：tc Server的一项新功能，可在应用程序服务器和基础虚拟机之间提供全新的协调级别。EM4J使用基础的vSphere虚拟化来克服Java静态内存堆的某些限制。
    • Spring Insight Operations ：利用与Spring Insight项目相同的代码级跟踪技术，但将来自多个应用程序服务器的信息汇总到一个控制台中，并具有汇总视图，下钻和历史比较，可用于生产系统。
    • SQLFire ： vFabric SQLFire利用了久经考验的vFabric GemFire基础，以内存速度和水平规模提供数据，但vFabric SQLFire向该服务添加了熟悉的标准SQL和JDBC接口。

    Rod Johnson在其最新博客中讨论了该版本的所有详细信息。请务必查看最新版本并进行尝试。

  2. Spring核心负责人Juergen Hoeller宣布Spring 3.1.0 M2已发布 ！终于，稳步迈向Spring 3.1 GA的下一步了！新版本的功能与上一版本一样，其中包含许多主要的新功能，包括（但绝对不限于！）改进的Java配置支持，无XML和无麻烦的基于Servlet 3.0的Spring MVC应用程序引导，新Builder JPA和Hibernate的API，还有更多！在此处查看发行公告，并从您选择的构建依赖项管理工具或下载页面中获取内容。
  3. 紧随Spring 3.1发布公告之后， Chris Beams加入了经过改进的Spring 3.1 M2中以Java为中心的配置模型，即使与M1相比也是如此！这些功能真的开始融合在一起，使它成为最流畅，排列合理的版本之一！
  4. Spring 3.1 M2代表了核心Spring以及Spring MVC的显着改进！ Rossen Stoyanchev会介绍很多（确实，您需要阅读详细的博客才能了解一个主意-我什至不用在这里列举所有这些！）Spring MVC 3.1 M2中的新功能，包括Servlet 3.0容器上不包含XML的Web应用程序。看看这个！
  5. SpringSource Tool Suite 2.7.0。M2已发布 ！新版本具有改进的CloudFoundry支持和许多Grails功能，包括对Grails 1.4的支持，支持Grails的重命名类型重构，改进的GSP内容辅助以及对Gradle的支持。看看这个！
  6. 使用Spring Roo？想塑造未来吗？现在是提供意见的好时机！Spring Roo小组希望您填写有关Spring Roo方向的快速调查。
  7. 带有MongoDB支持的Spring Data文档1.0.0。M3，已经发布！ 从公告中
    Spring Data Document 1.0.0中的更改和新功能。M3包括大大改进的映射和转换支持。的MappingMongoConverter现在是MongoTemplate和SimpleMongoConverter已被弃用，将被移除。默认集合名称的概念也已删除，并且MongoTemplate基于用于操作的实体类的集合名称。实体类使用的集合名称默认为以小写字母开头的类名称，但可以使用@Document注解。
  8. Spring Data Graph 1.1.0。支持Neo4j的M1已发布！如公告所述，
    Graph Neo4j模块提供与Neo4j图形数据库的集成。早在2010年，Rod Johnson和Emil Eifrem就关于Spring和Neo4j的集成（包括透明的持久性和跨商店支持）进行了集思广益。在最初的原型之后，它已经在VMware和Neo Technology开发团队之间紧密合作下进一步开发。
    新版本包括依赖项更新，对称为Cypher的Neo4J查询语言的支持，自我关系等等！看看这个！
  9. 使用Spring？是否要构建Swing应用程序，或者至少要构建具有一些数据驱动内容的Swing应用程序并管理与Spring的交互？查看JDAL ，该库具有专用于此的实用程序！根据该网站，JDAL是“基于Hibernate ORM和Spring Framework 构建的，并为您提供了一组核心数据库服务和UI组件，可以通过在Spring上下文配置文件上进行配置来使用它们。”很有意思！毫无疑问，您中的某些人会知道Spring Rich项目 ，该项目最近不太活跃。看到社区填补空白真是太好了！
  10. 使用Spring Security和Active Directory保护应用程序的安全性是可行的，但是直到最近我进行一些快速Google搜索并在SpringSource论坛上找到这篇文章时，这些细节才使我感到困惑。 解决方案的详细说明在响应中详细说明。方便！
  11. 寻找一个比较Spring与Java EE的良好矩阵？一位经理，贝尔吉施·格拉德巴赫（Bergisch Gladbach） 分享了他的观点。我发现非常有趣的一本书，反映了当今越来越多的用户的规范。看看这个！
  12. Loiane Groner的博客描述了如何在JAXB中使用自定义日期 。诚然，该博客不是特定于Spring的，但是使用Spring的JAXB2时可能会遇到此博客Marshaller与不支持相同数据序列化的客户端。这样的一个例子？Spring Android的Simple XML Marshaller默认情况下不支持JAXB2日期！这也是一个不错的博客，可以阅读到其他与Spring相关的，更有用的内容的链接。如果我能早一点知道这一点！我可以想到至少一个可以从此食谱中受益的代码库...
  13. 博客Java-Kata看一下使用Spring MVC，Wicket和GWT 构建基于动态表单的多行Web应用程序 。第一部分介绍代码（本身很有用），然后介绍Spring MVC迭代。好东西，对于那些想知道同一件事的人来说，还有细节。Spring MVC不是像Wicket和GWT这样的面向组件的Web框架，因此很高兴看到它在此复杂任务中仍然表现出色。
