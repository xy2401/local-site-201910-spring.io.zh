Web开发不断发展：Grails 2.0发布了！

工程
格雷姆·罗彻（Graeme Rocher）
2011年12月15日
经过近一年的开发，我们非常高兴地宣布Grails 2.0的GA版本-Web框架的第二个主要版本，正在改变JVM上Web开发的面貌。此版本带来了大大增强的用户体验。从命令行到测试报告，再到Grails为您生成的UI的所有内容都经过了重新思考和重新设计。Grails 2.0中提供的一些令人兴奋的功能包括：
  • 新的控制台UI结合了制表符完成和彩色输出
  • 更好，更可靠的重装机制，从而减少了服务器重启的次数
  • 增强的错误报告和问题诊断
  • 最新最好的库：Groovy 1.8，Spring 3.1，Hibernate 3.6和Servlet 3.0
  • 一个强大的静态资源框架（CSS，Javascript等）
  • 用于链接生成和页面渲染的新API
  • GORM的新功能：分离的条件，何处查询，多个数据源等
  • 用于数据库迁移和逆向工程的标准插件
  • 具有完整GORM仿真功能的新单元测试API
  • …以及数百项较小的改进
“ Grails 2.0的新增功能”中详细介绍了所有这些新功能。 用户指南部分。另外，请务必查看由Peter Ledbrook撰写的Grails 2.0网络研讨会和“ Countdown to Grails 2.0”博客系列：
  • 用户体验
  • 格姆
  • 数据库迁移
  • 单元测试
  • 静态资源
以及所有这些新功能，在Grails 2.0的开发过程中，Grails团队修复了900多个问题，并围绕Grails不断发展插件生态系统 。在Grails 2.0的开发过程中， 源代码以多种重要方式得到了发展。我们通过转移到Gradle作为构建工具来对源代码进行模块化。我们现在使用Artifactory进行仓库管理； Spock已成为我们的事实测试工具；并且我们重写了内部结构以利用Groovy AST转换的优势。通过所有这些更改，Grails的性能将大大改善，并且用户将看到Grails 2.0的好处。除了Grails 2.0的发布之外，我们还有许多其他激动人心的公告，它们将从Cloud开始。

云中的Grails 2.0

对于那些关注云空间的人（谁不是！）您会很高兴知道Grails 2.0已经支持Cloud。 Cloud Foundry -VMware开放的PaaS- 开箱即用地支持Grails 2.0 ，我们已经在努力将Grails网站迁移到cloudfoundry.com ！请查看Peter Ledbrook撰写的精彩博客文章，介绍Grails和Cloud Foundry的一步部署 ，其中展示了如何使用Cloud Foundry插件（适用于Grails）开始Cloud Foundry部署。Heroku的家伙们还宣布了Grails的支持 -支持将Grails应用程序持续部署到Heroku的云平台 。Grails应用程序可以通过简单的“ git push”进行部署，从而触发完整的构建和部署工作流程。要在Heroku上开始使用Grails，请遵循此Grails快速入门指南 。Grails部署到Heroku当前处于公开测试阶段。随着更多支持Grails的云平台的出现，云将在2012年继续成为一个巨大的主题，其中许多平台将基于开源Cloud Foundry。

IDE中的Grails 2.0

对于Eclipse用户，请使用SpringSource Tool Suite（STS）版本2.9.0。M1（2011年12月可用）为Grails 2.0提供了最佳支持。STS 2.8.1版为那些不准备依赖里程碑版本的用户提供了许多基本支持。STS版本可在以下位置找到： http : //www.springsource.com/downloads/sts最新的STS新版本和值得注意的版本可在以下位置找到： http : //download.springsource.com/release/STS/doc/STS-new_and_noteworthy .pdf它描述了Groovy和Grails开发的所有最新增强功能。有关STS可能存在的任何问题的支持论坛，请访问这里： http : //forum.springsource.org/forumdisplay.php? 32-SpringSource-Tool-Suite对于Intellij IDEA用户， JetBrains一直在忙于开发Intellij。上周发布的IDEA 11。Intellij IDEA 11包含许多支持Grails 2.0的新功能，因此请确保在开始使用Grails 2.0和Intellij IDEA之前进行升级。

Grails 2.0和NoSQL

Grails 2.0是Grails的第一个版本，该版本真正抽象了GORM层，因此可以使用GORM的新实现。随着Grails 2.0的发布，我们还很高兴地宣布针对NoSQL数据存储发布了许多与Grails 2.0兼容的插件：
  • 适用于MongoDB 1.0 RC3的GORM
  • 适用于Redis 1.0 M8的GORM
  • 适用于Riak 1.0 M4的GORM
MongoDB插件处于最终发布候选阶段，它基于出色的Spring Data MongoDB项目 ，该项目也以RC形式提供。此外，我们还为希望参与该项目的人们编写了一份开发人员指南 ，说明如何构建GORM实施。Grails用户可以期待即将在2012年发布的有关Neo4j ， Amazon SimpleDB和Cassandra的GORM即将发布的更令人激动的NoSQL公告。
