Spring Roo 1.2.0。M1发布

工程
艾伦·斯图尔特
2011年9月14日

Spring Roo团队很高兴宣布我们的第一个1.2.0里程碑可用。Roo 1.2.0。M1版本继承了Roo的愿景，即可以快速轻松地构建基于Spring的企业Java应用程序。我们已经听取了社区的意见，并在此发行版中包含了一些要求最高的功能：


  • 快十倍（！）我们知道很多人正在使用Roo进行大型项目，因此尽管如此，我们仍继续关注Roo内部，以确保Roo的内部运行平稳快速。通过关注性能分析，文件监视，磁盘I / O，XML模型和双向访问，我们将Roo在大型基准测试项目中的性能提高了10倍。

  • 现在Apache许可了 。在Roo 1.0 / 1.1中，我们使用GPL许可鼓励与社区共享Roo所做的任何改进。为了回应反馈，我们已转移到Apache软件许可2（ ROO-2253 ），因此您可以完全灵活地在商业和非商业项目中使用Roo。这也使Roo与其他大多数Spring项目保持一致。

  • 移至GitHub。 社交编码正在兴起，并且GitHub已成为开放源代码管理的事实上的站点。我们很高兴将Roo的代码库移至GitHub，以便您可以轻松地对其进行分叉，进行更改和提交请求。这将大大简化社区收到新捐款的过程。看看ROO-2708或访问 https://github.com/SpringSource/spring-roo了解更多信息。

  • 存储库层的灵活性 。现在，您可以定义自己的存储库层方法，并充分了解Roo托管的Web层，集成测试和按需数据机制中的选择。对于那些喜欢比Roo 1.0 / 1.1类似Active Record的模型更传统的分层方法的人来说，这是向前迈出的重要一步。当然，支持早期方法的理念仍然存在，因此Roo仍然完全支持在实体中嵌入方便的Active Record风格的方法。您可以在Pizza-Shop示例（请参阅发行版的/ samples目录）和ROO-301中了解有关此新功能的更多信息。

  • 服务层 。我们还决定解决另一个要求很高的社区功能：服务层支持。现在，您可以轻松地添加服务层（通过Roo“服务”命令），并将其用于Roo托管的Web层，集成测试和按需数据功能。自定义服务层也可以与自定义存储库同时使用，从而使您可以完全灵活地构建所需的任何应用程序层。您可以在ROO-340中阅读更多内容，斯特凡·施密特（Stefan Schmidt）即将发布博客，以提供更多详细信息。

  • 灵活的GWT支持 。现在，您可以跳过UI，仅让Roo管理您的RequestFactory类（EntityProxy，RequestContext和Locator）与域模型和服务层的同步。以前，GWT UI脚手架是强制性的，但使用1.2.0。对于M1，您可以更加选择Roo如何与您的项目集成。

  • MongoDB支持 。无论您是Cloud Foundry用户还是Mongo粉丝，我们现在都增加了将实体存储在MongoDB中的支持。 Mongo是一个快速的，无碎片意识的，无模式的文档数据库，它支持map-reduce，非常适合那些希望将NoSQL与Roo一起使用的人。在ROO-2693中了解更多信息。

  • 数据库逆向工程（DBRE）多架构支持 。Roo中最受欢迎的功能之一是DBRE支持，它不仅使您可以从关系模式中构建实体模型，而且还可以在模式更改时自动维护该实体层。这采用模式优先的方法进行开发，我们知道用户在具有数百个表的项目中使用DBRE。作为ROO-1408的一部分，我们增加了对DBRE同时从多个模式中采购其结构的支持，实际上，您甚至可以让不同的模式在不同的Java包中管理实体。

  • 外壳改进 。很多人喜欢Roo的可用性，实际上Shell已经在许多其他VMware产品中找到了使用方法。我们感到高兴的是，JLine 1.0（ ROO-2608 ）中进行了许多次要的修复，从而使Roo外壳（例如，退格到上一行）将终于可以使用！）。您也可以通过“！”直接从Roo Shell中运行操作系统命令。操作员（ ROO-2457 ）和正在启动的shell现在显示最新的社区推文（ ROO-2677 ）。


如您所见，我们对Roo 1.2进行了很多增强，并期待您的反馈。我们在下一个Roo 1.2里程碑中仍有许多令人兴奋的新功能，包括JSF / PrimeFaces （ ROO-516 ）支持和多模块Maven （ ROO-120 ）项目支持。这两个也是要求很高的社区功能。


请记住，这是一个里程碑版本，因此您应该继续将Roo 1.1.5用于生产项目。但是，您可能希望在Roo 1.2.0中探索这些新功能。通过试验包含的示例项目来完成M1。

我们希望您喜欢这个新版本。请通过社区论坛分享您的经验，关注@SpringRoo以获得最新消息，或在推文中使用#springroo

Alan Stewart，项目主管-Spring Roo
