OAuth implementation for Spring Security. Support is provided for the implementation of OAuth providers and OAuth consumers. There is also support for two-legged OAuth (a.k.a. "Signed Fetch") and for OAuth 2.0.

Applying security to an application is not for the faint of heart, and OAuth is no exception. Before you get started, you're going to want to make sure you understand OAuth and the problem it's designed to address. There is good documentation at the OAuth site and a good illustration of how OAuth is applied. You will also want to make sure you understand how Spring and Spring Security work.

欢迎

欢迎使用Spring Security的OAuth！

您可以从其名称中推断出该项目的目的是为Spring Security提供OAuth实现。提供了对OAuth提供者和OAuth使用者实施的支持。还支持两足式OAuth（又名“签名提取”）和OAuth 2.0 。

将安全性应用于应用程序并非出于胆小，OAuth也不例外。在开始之前，您将需要确保您了解OAuth及其旨在解决的问题。OAuth网站上有很好的文档，并且很好地说明了OAuth的应用方式 。您还需要确保您了解Spring和Spring Security的工作方式。

这样，您就可以开始了。您将希望看到OAuth for Spring Security的实际应用，并在用户指南中阅读更详细的说明。

如果您遇到问题或有其他疑问，请使用论坛 。
