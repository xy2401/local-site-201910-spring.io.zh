Spring Roo

发布
克洛伊·杰克逊（Chloe Jackson）
2011年9月12日

即时结果-使Java有趣

Spring Roo是一种轻量级的开发人员工具，可快速轻松地交付即时结果。最重要的是，您可以100％用Java编写代码，并可以重用所有现有的Java知识，技能和经验。您会喜欢它的，而且也有很多乐趣！了解TomTom如何使用Spring Roo 将开发时间减少一半 。

下载Josh Long和Steve Mayzak的书，名为Roo入门 。您也可以按需印刷书籍或从O'Reilly网站上以其他格式购买。

Ken Rimple和Srini Penchikala还编写了权威的Roo参考书，名为Spring Roo in Action，可以从Manning Publications购买。

令人兴奋的功能-GWT。 Solr。云。柔性...

带有JPA的MVC应用程序对Roo来说是一件轻而易举的事，但是它也使您轻松进入下一代技术。我们已经支持一些很酷的功能，例如用于高级下一代用户界面的Google Web Toolkit（GWT），用于搜索服务器集成的Solr和诸如Google App Engine的云部署。另外，我们还具有增量数据库自省，Flex，Spring Integration等功能，并且在开发中更加积极。

SpringSource和Google携手合作，将Spring Roo的开发速度与Google Web Toolkit（GWT）的强大功能相结合，以构建丰富的浏览器应用程序。试用Spring Roo 1.1现在提供的令人兴奋的GWT和Google App Engine支持。

零风险-生产力永不妥协

使用Roo，您仍然可以使用Java编写所有程序。它的创新方法完全在编译时运行，并且与您的IDE完全兼容。另外，由于没有运行时部分，因此采用也被简化并且风险低。从第一天开始，您所有的Java开发人员都将感到舒适并富有成效。Roo确实代表了生产力，而毫不妥协。


为什么选择Spring Roo？

Spring Roo是面向Java开发人员的下一代快速应用程序开发工具。使用Roo，您可以在几分钟内轻松构建完整的Java应用程序。它与其他生产力工具的不同之处在于：

  • 快速的结果
  • 100％ 用Java开发

  • SpringSource认证的架构

生产力不折不扣

Roo的创新方法没有任何妥协：

  • 用Java编写程序！
  • 全面的IDE支持（具有代码辅助，重构和调试等功能）
  • 没有运行时部分，这意味着具有批准要求的企业更容易采用
  • 出色的运行时性能，因为没有运行时内存使用，部署占用空间，控制流调用时间等
  • 由于Roo完全在开发时运行（就像您的IDE一样），因此降低了影响项目的错误风险。
  • 无需锁定Roo，只需几分钟即可将其从项目中删除
  • 借助基于OSGi的附加扩展性，可以轻松扩展新功能
  • 我们可以继续，但是您明白了：Roo不会让您妥协（曾经）:-)

怎么运行的

简而言之，Roo是轻量级的控制台外壳，可在开发项目时加载。尽管Roo Shell可以通过一个快速命令来完成耗时的操作，但大多数情况下，您将忽略Roo并继续在IDE或文本编辑器中开发项目。

 

当您以常规方式编辑代码时，Roo会密切注意您的项目文件，并根据您的操作自动对其进行修改。根据您正在运行的Roo插件，Roo可以帮助您处理不同类型的文件。例如，Roo的JPA插件意味着您可以编写一个非常简单的Hello.java文件，如下所示：

@RooJpaActiveRecord public class Hello {
	private String world;
}

您可能想知道只有一个私有字段的Java类有什么好处。好吧，不多。但是Roo通过使用编译器的“ mixins”功能在编译时（而不是运行时）将额外的东西添加到.class文件中来解决此问题。这意味着Hello.class实际上包含大量有用的成员，您不必自己编写，维护，调试和测试麻烦的成员：

public class Hello {
	private String world {..}
	public String getWorld() {..}
	public void setWorld(String world) {..}
	public Long getId() {..}
	public void setId(Long id) {..}
	public Integer getVersion() {..}
	public void setVersion(Integer version) {..}
	public String toString() {..}
	public void persist() {..}
	public void remove() {..}
	public void flush() {..}
	public static Long countHellos() {..}
	public static Hello findHello(Long id) {..}
	public static List findAllHellos() {..}
        public static List findHelloEntries(int start, int finish) {..} // there are even more methods
}

Roo方法的优点是，您的Hello.java完全没有杂音和精力，但是Hello.class非常有用，并且与您的IDE（代码辅助，调试等）和运行时环境（没有Roo依赖项或其他）完全兼容妥协）。另外，因为它只是一个普通的Java类，所以它在运行时也表现出色，并且在运行时不会消耗额外的内存。Roo完全了解往返，并会在您开发实体时维护mixins。例如，如果添加另一个字段，Roo将立即生成getter / setter并更新toString（）方法。如果您添加@Id字段，Roo将停止产生其标识符字段，而您的标识符字段将具有优先权。如果您自己编写toString（）方法，则将使用您的方法，而Roo将消失。实际上，如果您注释掉toString（）方法，Roo将立即返回。这一切都非常容易（也很有趣！）。

上面没有立即显而易见的更微妙的因素是@ Roo *注释是“源保留”。这意味着它们不会出现在.class文件中，反映出Roo的“零锁定”和“无运行时”策略。另外，通过将Roo生成的mixin源代码的内容直接复制到您的.java文件中，您可以轻松摆脱Roo。实际上，Eclipse中甚至有一个重构命令，它将在几秒钟内为您完成此操作！哦，如果您不想编写Hello.java源文件，则可以只使用两个支持Roo TAB的命令来完成它：“ entity --class Hello”，“ field String world”。就像我们说的那样，这一切都非常容易（也很有趣！）。

Roo的方法为您提供了最好的免费锁定，高性能生产力工具。它只是工作。试试看 ，自己看看。或访问我们的参考指南，以了解更多有关为何选择Roo作为生产力工具的信息。


尝试Spring Roo

开始使用Roo非常容易。只需按照此页面上的步骤操作，您将在几分钟内拥有一个正在运行的环境。

下载

Roo要求您下载并安装：

  •  Java 6 JDK
  •  阿帕奇Maven

要获得Roo，您可以下载其独立的命令行Shell版本，也可以使用SpringSource Tool Suite内置的版本。如果您愿意，也可以下载两者并一起使用：

  •  Spring Roo
  •  SpringSource工具套件

安装

如果您下载了独立的Spring Roo，只需将其解压缩到您选择的位置即可。然后：

  • Windows用户：将％ROO_HOME％\ bin添加到您的路径
  • * nix用户：创建到$ ROO_HOME / bin / roo.sh的符号链接（例如sudo ln -s〜/ spring-roo-1.xx / bin / roo.sh / usr / bin / roo）

 

看视频

如果您想观看安装Roo的过程有多么容易， 该视频将向您展示如何使用。或者，完成以下步骤，然后自己尝试。

第一个项目-Roo Standalone

让我们建立一个项目。在以下步骤中，我们将省略控制台输出。我们也强烈建议您在每个步骤之后键入“提示”，以帮助您学习如何使用外壳程序。请注意，您需要Roo 1.2.0或更高版本才能执行以下命令：

mkdir hello
cd hello
roo
roo> hint
roo> project --topLevelPackage com.foo
roo> jpa setup --provider HIBERNATE --database HYPERSONIC_IN_MEMORY         
roo> entity jpa --class ~.Timer --testAutomatically
roo> field string --fieldName message --notNull
roo> hint web mvc
roo> web mvc setup
roo> web mvc all --package ~.web
roo> selenium test --controller ~.web.TimerController
roo> web gwt setup
roo> web gwt all --proxyPackage ~.client.proxy --requestPackage ~.client.request          
roo> perform tests
roo> quit

哇！现在，您已经拥有一个Web应用程序，其中包含JUnit测试，Selenium测试，MVC前端，Google Web Toolkit前端。只需使用“ mvn gwt：run”来玩GWT客户端，或者使用“ mvn tomcat：run”来启动Tomcat MVC前端并使用http：// localhost：8080 / foo / timers进行查看。

接下来，您可能要键入“ mvn eclipse：eclipse”，然后使用Eclipse的“常规>将现有项目导入工作区”功能。保持Roo Shell运行，并随时在Eclipse中进行编辑。您会看到Roo在运行时会维护文件。如果您在不运行Roo的情况下进行了任何更改，请放心-只需加载Roo，它就会赶上您所做的更改。

第一个项目–通过STS

如果安装了SpringSource Tool Suite，则只需使用File> New> Roo Project。然后输入“ Foo”的“项目名称”和“ com.foo”的“包”。单击“下一步”和“完成”后，您将在STS的底部看到Roo shell加载。然后输入与在“第一个项目（Roo独立）”部分中看到的命令相同的命令，您可以通过WTP轻松运行该应用程序。

Spring Roo社区

Roo社区由使用Roo的数千人组成，从初学者Java开发人员到JavaOne摇滚明星-以及介于两者之间的每个人！Roo社区具有包容性，友好性，并热忱欢迎您的参与。我们不仅会帮助您学习Roo，而且我们也非常乐意为您提供使用技巧，路线图信息和体系结构建议。本页详细介绍了参与Roo社区的方式。我们期待您的回音！

推特

关注@SpringRoo以获取最新的官方新闻和发布公告。如果您对Roo的开发进度和其他有趣的更新感兴趣，我们也建议您关注Roo项目团队的工程师 。

如果您在Roo上发了推文，请在消息中包含#SpringRoo（请不要使用#roo或类似名称）。这将帮助其他人找到您的Tweet，并且可以在此页面和我们的主页上的Twitter小工具中找到它。请让我们（和其他人！）通过这种便捷的方式了解您对Roo的看法。

社区论坛

Roo 社区论坛是与社区其他成员进行交流的主要方式，有关Roo入门，使用建议，体系结构帮助，路线图信息和可能的错误。您还会发现这是一种与Roo核心项目工程师联系的简便方法，他们所有人都活跃在社区论坛上，并乐于与您讨论路线图和详细的Roo内部。

传言

如果您喜欢Roo，请让其他人知道！最简单的方法是只使用@SpringRoo进行Tweet，但是其他方法可以包括内部公司演示，用户小组讨论，会议等。Roo非常高兴演示，因为它非常易于演示，此外，我们还为您提供了许多免费的，定期更新的演示材料（只需在社区论坛上与我们联系 ）。如果您已经介绍过Roo，请告知我们，我们将尝试向您发送一份小小的Roo礼物（感谢SpringSource，我们通常会送出一些好礼物）。

应用展示

不久我们将启动“应用程序展示柜”，向其他人展示人们使用Roo制作的应用程序。如果您在Roo中构建了一个应用程序，并且想提供一个屏幕截图和一两个简短的句子，请通过论坛与我们联系。

问题追踪器

如果您发现了错误或有新功能，请将其记录在我们的问题跟踪器中。我们的问题跟踪器还可用于查找其他人提交的功能，对需求进行评论并为您认为最有用的功能投票。这是我们优先处理增强请求的常用方法。

博客

Roo核心工程师定期在SpringSource Team Blog上发布有关Roo的博客 。您还可以通过遵循Roo类别直接访问与Roo相关的文章。

写代码

如果您想深入了解Roo代码，则第一步是克隆我们的Git存储库并遵循根目录中的readme.txt。我们使用Maven来构建Roo，因此构建自己的发行版非常容易（因为您可能已经安装了Maven才能使用Roo！）。您需要的Git命令是：

git clone git://github.com/SpringSource/spring-roo.git

为Roo编写代码的一种简单方法是开发附加组件。由于Roo使用基于OSGi的附加组件体系结构，因此您可以轻松构建其他人可以找到并安装到其Roo Shell中的附加组件。

如果您想为Roo本身做出贡献（而不是编写外部附加组件），也非常欢迎您这样做。浏览问题跟踪器或在社区论坛上发布消息，概述您感兴趣的工作，我们将很乐意为您提供帮助。
