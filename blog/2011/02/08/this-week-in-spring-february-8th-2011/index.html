Spring 的这个星期：2011年2月8日

工程
乔什·朗（Josh Long）
2011年2月8日

这是Spring社区又一个令人兴奋的一周。社区似乎对新的Tomcat版本，移动客户端，Spring Data和大数据（例如Spring Gemfire，Redis，当然还有Spring Data项目）感到热议。

提醒您：Juergen Hoeller –核心Spring Framework 的负责人，并且以某种或其他方式为所有姊妹Spring项目的贡献者–正在针对 Spring 3.1的新功能提供两个完全相同的网络研讨会- 一个用于北美 ， 另一个用于欧洲 。在两天内（2月10日）！请确保在使用最广泛的Java框架的下一个迭代中注册此免费网络研讨会！

事实证明，这是一个创纪录的活动，它的注册号令人难以置信。尽管还没有Spring 3.1二进制文件（请继续关注！），但是Spring项目的代码始终可以使用 ，因此-如果您像我一样对所有新功能都感到兴奋-您将看到此网络研讨会并在它发布之前就开始使用它。

这周有很多好东西要讲，所以让我们开始吧！

  • Tomcat项目的高级工程师Mark Thomas在2010年的SpringOne社区活动中进行了演讲。现在可以在InfoQ.com上获得该演讲。

  • Spring Batch 2.1.6

    已发布。新版本具有一些修复和功能增强。

  • 本周有两个新版本的Apache Tomcat。首先， Apache Tomcat 7.0.8已发布。 这个新版本包括许多小的更改，包括甚至更好地支持基于Comet的异步应用程序。Apache Tomcat 6.0.32也已经发布，并且主要是维护版本。与往常一样， Tomcat Expert社区站点上对这两个发行版都有很好的报道。

  • Spring Mobile 1.0.0。M3已发布。 Spring Mobile项目是一个旨在促进对移动设备友好的Web应用程序的库。它为Spring Android的 Android设备特定的客户端支持提供了一个不错的服务器端组件。

    鼓励用户检查Spring Mobile git存储库中维护的出色的新示例代码项目。

  • 戈登·狄更斯（Gordon Dickens ）发布了一个关于使用的精彩博客

    Spring的JMS支持（包括JmsTemplate）和Spring核心OXM（一个对象到XML的编组框架）支持将基于JAXB2的消息有效内容编组为XML。好的帖子，它与有关消息传递的Green Beans帖子一起很好地阅读

  • Spring GemFire 1.0.0（最终版）已发布 。此版本提供了与GemFire数据管理和数据网格技术的一流集成。 Spring GemFire在惯用的Spring配置中为GemFire提供了易于使用的配置。另外，Spring Gemfire集成还将在即将到来的Spring 3.1缓存抽象中发挥作用，该抽象将支持GemFire和许多其他数据缓存。
  • Shekhar Gulati发表了一篇精彩的文章， 探讨了Spring Roo和Hades项目，该项目现已成为Spring Data产品组合的一部分。
  • JTeam小组发表了有关使用Spring Android和Spring Mobile构建针对移动客户端的应用程序的博客 。该文章还介绍了Spring Android特定的支持，用于从客户端使用服务器端数据。

  • Spring Framework 的最佳“功能”之一是其社区 。Spring MVC负责人Keith Donald最近在Twitter上发布了指向Gunnar Hillert 博客的链接，其中强调了他的功能请求的故事以及现在成为Spring Mobile M3最新发行版的一部分的途径。

  • Sebastian Pietrowski对最近使用的Spring Data Redis支持的使用进行了引人入胜的介绍。 看看这个！

  • 博客

    贾斯汀·瑞安（Justin Ryan）用迷人的 Spring 写一篇文章BeanDefinitionRegistryPostProcessor实现将模拟自动注入Spring测试。

    该实现与Spring的单元测试支持一起使用。Spring的单元测试支持提供了一种使用Spring注释的一致方法（例如@Autowired ）放在单元测试类中，并通过上下文将那些bean注入，就像单元测试是正确的Spring组件一样。

    该实现使这一步骤更进一步，自动注册您自动装配的bean的模拟对象。

    在Spring Framework 中BeanDefinitionRegistryPostProcessor是经典的延伸BeanFactoryPostProcessor可用于在Spring容器中注册对象，这些容器又可以注册其他元对象，例如BeanFactoryPostProcessor实例。否则，它的工作原理基本相同，但范围更大。
    漂亮的贾斯汀！

  • Ken Rimple在博客上写了更多关于如何一起使用Spring Roo和Spring WebFlow的文章 ，甚至建立了一个简单的示例。精通阅读，并对Spring WebFlow概念进行了很好的介绍。
