适用于MongoDB的GORM：新里程碑，更丰富的体验

工程
格雷姆·罗彻（Graeme Rocher）
2011年2月15日

去年，我们在GORM中引入了对MongoDB的支持 （以及许多其他 GORM 实现 ），并得到了社区的极大欢迎。我们收到了大量反馈，今天我们很高兴地宣布一个新版本（里程碑2），该版本解决了我们收到的一些反馈。

嵌入式文档支持

要求的第一功能是嵌套文档支持，在此版本中，我们对此进行了重大改进。现在，您可以使用GORM中的标准嵌入式映射来嵌入其他域：

class Person {
  String firstName
  String lastName
  Address address
  static embedded = ['address']
}

嵌入式域存储在主Mongo文档中的嵌套文档中。此外，现在还可以将基本类型的列表和映射保留到本机Mongo ArrayList和哈希表中：

class Person {
	List<String> friends
	Map pets
}
...

new Person(friends:['Fred', 'Bob'], pets:[chuck:"Dog", eddie:'Parrot']).save(flush:true)

嵌入的域以及列表和地图都存储在该域的主要Mongo文档中，从而允许使用GORM实施更多常见的MongoDB模式。

地理空间索引和查询

MongoDB具有对地理空间索引和查询的本机支持，现在在GoRM for MongoDB中支持此功能。您可以将列表或地图定义为“地理索引”：

class Hotel {
	String name
	List location
	static mapping = {
		location geoIndex:true
	}
}

然后使用表示纬度和经度的二维列表轻松保存地理数据：

new Hotel(name:"Hilton", location:[50, 50]).save()

另外，也可能更易读，您可以使用包含纬度和经度值的地图：

new Hotel(name:"Hilton", location:[lat: 40.739037, long: 73.992964]).save()

一旦持久化，就可以使用新的findBy * Near语法查询域类：

def h = Hotel.findByLocationNear([50, 60])
assert h.name == 'Hilton'

您还可以使用findBy * WithinBox方法使用绑定查询来定位矩形内的位置：

def box = [[40.73083, -73.99756], [40.741404,  -73.988135]]
def h = Hotel.findByLocationWithinBox(box)

或使用findBy * WithinCircle方法在一个圆内：

def center = [50, 50]
def radius = 10
def h = Hotel.findByLocationWithinCircle([center, radius])

无模式领域模型

MongoDB完全没有架构，这意味着您不限于关系数据库中固定数量的列。GORM for MongoDB现在支持无模式域模型。您可以继续在域模型中指定固定域属性：

class Plant {
    boolean goesInPatch
    String name
}

但是，您现在还可以使用Groovy下标运算符保留其他属性：

def p = new Plant(name:"Pineapple")
p['color'] = 'Yellow'
p['hasLeaves'] = true
p.save()
p = Plant.findByName("Pineapple")

println p['color']
println p['hasLeaves']

还有很多改进，包括自定义索引创建的功能，对示例查询的支持以及对GORM API的更全面的支持。 文档已更新，以涵盖所有这些新功能。让我们知道您的想法，您的反馈非常宝贵。

最后一个插件，如果您在马德里地区，想进一步了解MongoDB的GORM，请参加本周四的Spring IO会议 ，那里将有更多关于Spring，Grails和GORM的讨论。
