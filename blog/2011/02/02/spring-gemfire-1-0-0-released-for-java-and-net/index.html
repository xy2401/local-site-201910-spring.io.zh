针对Java和.NET的Spring GemFire 1.0.0发布

发布
Costin Leau
2011年2月2日

亲爱的春季社区，

我们很高兴地宣布，Spring GemFire 1.0项目的第一个GA版本现已可用于Java和.NET！Spring GemFire项目旨在使使用GemFire作为分布式数据管理平台来构建基于Spring的高度可扩展应用程序变得更加容易。

立即下载 ： 适用于Java的Spring GemFire | NET的Spring GemFire

此版本具有：


  • GemFire基础结构的声明性依赖项注入样式配置（例如Cache，Region，interest等）

  • 广泛的名称空间支持，用于配置所有主要的GemFire组件：缓存，复制，分区和客户端区域等等

  • 异常转换为Spring的可移植DataAccess异常层次结构

  • 模板和回调支持可轻松访问本机API

  • 交易管理支持

  • 弹簧支持的GemFire管理对象布线

  • 自动生成基于非反射的实例化器

  • 对GemFire 6.5的原生支持（6.0以后）

  • 声明式缓存建议（适用于.NET）

通过Spring GemFire，Spring用户在与GemFire交互时应该感到宾至如归，而熟悉GemFire的开发人员将看到Spring容器的优势和灵活性，其强大的AOP集成以及通用的服务抽象。

要了解有关该项目的更多信息，请访问Spring GemFire 主页 。

期待您的反馈！

PS请使用maven.springframework.org作为工件存储库，直到解决影响发布到Maven Central的技术问题为止。
