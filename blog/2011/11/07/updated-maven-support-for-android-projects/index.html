更新了对Android项目的Maven支持

工程
罗伊·克拉克森
2011年11月7日

自从我最初在Spring Android上使用Maven以来，已经有一段时间了，并且工具的状态已经有所改变。随着Android 4.0 版 SDK的最新发布以及Android开发工具（ADT）的修订版14的发布 ，Google进行了多项更改 ，从而影响了第三方Maven插件对构建Android项目的支持。幸运的是，后面的球队的Android Maven插件 ，而Android的配置为M2E一直在努力工作，以支持Android的工具，这些最新的变化。如果您最近尝试运行Spring Android Samples ，那么您可能会注意到这些项目无法构建。在本文中，我将说明在Spring Android Showcase项目中利用最新的Maven插件和Eclipse支持所必需的配置更改。

Android Maven插件

从3.0.0-alpha-11版本开始，Maven Android插件现在有了新名称，现在称为Android Maven插件。最新版本3.0.0-alpha-13修复了Android SDK r14的生成问题。

更新插件

要更新您的插件版本，请对artifactId和版本进行以下更改。


<plugin>
    <groupId>com.jayway.maven.plugins.android.generation2</groupId>
    <artifactId>android-maven-plugin</artifactId>
    <version>3.0.0-alpha-13</version>
    <configuration>
        <sdk>
            <platform>${android-platform}</platform>
        </sdk>
        <deleteConflictingFiles>true</deleteConflictingFiles>
        <undeployBeforeDeploy>true</undeployBeforeDeploy>
    </configuration>
    <extensions>true</extensions>
</plugin>

修改后，您现在可以从命令行运行maven来构建项目。

mvn clean install

要解决此问题，您只需将以下jvmArgument添加到插件声明中即可。这将增加构建可用的内存。


<plugin>
    <groupId>com.jayway.maven.plugins.android.generation2</groupId>
    <artifactId>android-maven-plugin</artifactId>
    <version>3.0.0-alpha-13</version>
    <configuration>
        <sdk>
            <platform>${android-platform}</platform>
        </sdk>
        <dex>
            <jvmArguments>
                <jvmArgument>-Xms256m</jvmArgument>
                <jvmArgument>-Xmx512m</jvmArgument>
            </jvmArguments>
        </dex>
        <deleteConflictingFiles>true</deleteConflictingFiles>
        <undeployBeforeDeploy>true</undeployBeforeDeploy>
    </configuration>
    <extensions>true</extensions>
</plugin>
处理瞬态依赖项时出错

Spring Android 1.0.0。M4是针对Android Rome Feed Reader构建的。此版本的ROME在Maven Central存储库中不可用，因此，要使用它，必须在POM中包含存储库信息。 Spring Android Showcase客户端不包含任何ROME示例，因为它们包含在单独的示例项目中。通常，这意味着您不必包括该库的存储库引用。使用以前版本的Maven Android插件，该项目可以正确构建，而不必包含存储库引用。但是，Android Maven插件的3.0.0-alpha-13版本似乎试图下载所有依赖项，甚至是可选的依赖项。如果不包括ROME存储库，则构建将失败，并显示以下错误。

为了使当前的maven插件成功构建，您需要在Spring Android项目中包括Android ROME存储库。


<repository>
    <id>android-rome-feed-reader-repository</id>
    <name>Android ROME Feed Reader Repository</name>
    <url>https://android-rome-feed-reader.googlecode.com/svn/maven2/releases</url>
</repository>

适用于M2E的Android配置器

自从我上一次在博客中介绍Maven Android插件以来，用于M2E的Android Configurator也进行了更名。以前，它被称为Android开发工具的Maven集成。现在，最新版本的Android Configurator支持Android SDK r14。

安装插件

您可以从Eclipse Marketplace安装配置器。如果您使用的是SpringSource Tool Suite（STS），则可以从仪表板上的“扩展”选项卡安装Eclipse Marketplace。它位于“实用程序”部分下。否则，如果您使用的是其他版本的Eclipse 3.7，则可以从Indigo下载站点获得Eclipse Marketplace。

  1. 安装Eclipse Marketplace客户端。
  2. 在Eclipse Marketplace中，搜索Android Configurator。
  3. 单击安装按钮以安装Android Configurator。
  4. 如果您尚未安装Android ADT，则该配置将安装它。

m2e-Eclipse的Maven集成

需要注意的一件事是，m2e-Maven Integration for Eclipse插件未随配置程序自动安装。如果使用的是STS 2.8.0 ，则已经安装了m2e插件。如果没有，那么您可以按照上述先前的步骤在导入Android项目之前从Eclipse Marketplace安装m2e插件 。

将您的项目导入Eclipse

一旦安装了必需的Eclipse插件，您就可以导入您的Android项目。如果您有一个现有的POM文件，则只需将项目导入为“现有Maven项目”并指向您的POM。如果以前使用过旧版本的Maven插件来处理项目，则需要更新项目。

  1. 在Eclipse Package Explorer中右键单击您的项目。
  2. 选择Maven上下文菜单并更新项目配置...
  3. 确认在“可用的Maven代码库”中选择了您的项目，然后单击“确定”。

结论

Maven插件是用于构建Android项目的强大工具，在社区的帮助下，这些插件将继续发展和成熟。Spring Android样本已更新为可与这些最新插件一起使用，因此请提取最新版本并查看更改。此外， 适用于m2e的Android Maven插件和Android Configurator位于GitHub上，他们欢迎您的反馈和请求！
