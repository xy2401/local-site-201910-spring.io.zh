将Cloud Foundry Services与Spring结合使用：第4部分– Spring配置文件

工程
斯科特·安德鲁斯
2011年11月10日

Spring 3.1增加了对环境的重要新支持。这个新的环境API使得向应用程序公开属性或有条件地加载配置片段变得容易。

在本系列的早期文章中，Ramnivas展示了Cloud Foundry如何无需手动配置即可自动连接到数据库 。当您需要对该过程进行更多控制时（例如，连接到多个数据库），云名称空间会压缩DataSource只需一行XML。云名称空间功能强大，但仅在Cloud Foundry中运行的应用程序支持。使用云名称空间意味着将那部分配置耦合到Cloud Foundry。如果未部署到Cloud Foundry，则应禁用此配置。容器外测试是Spring理念的基本原则，因此如果必须部署应用程序才能运行集成测试，那将是不可接受的。 Spring Profiles解决了这个问题。

Spring中的Profile是配置的一部分，只有在满足特定条件时才会激活。许多应用程序平台都有开发，测试和生产配置的概念。虽然可以以这种方式使用配置文件，但它们也更加灵活。除了“ default ”配置文件，当未定义其他配置文件时激活。配置文件可用于在不同环境中运行时交换配置，或在应用程序中启用可选功能。如何使用它取决于您。

隔离Cloud Foundry特定的配置


将Spring应用程序部署到Cloud Foundry后，“ cloud ”配置文件会自动启用。这为Cloud Foundry特定的应用程序配置提供了一个预定义的便捷位置。云命名空间的所有特定用法都应在云配置文件块内进行，以允许应用程序在Cloud Foundry环境之外运行。


<bean id="mongoTemplate" class="org.springframework.data.mongodb.core.MongoTemplate">
    <constructor-arg ref="mongoDbFactory" />
</bean>
<beans profile="default">
    <mongo:db-factory id="mongoDbFactory" dbname="pwdtest" host="127.0.0.1" port="27017" username="test_user" password="efgh" />
</beans>
<beans profile="cloud">
    <cloud:mongo-db-factory id="mongoDbFactory" />
</beans>

此示例配置显示了Spring的MongoTemplate从两个备用配置的连接工厂中填充。在Cloud Foundry上运行时，将自动配置连接工厂。当不在Cloud Foundry上运行时，将使用到本地运行的MongoDB实例的连接设置手动配置连接工厂。

在Cloud Foundry上启用应用程序功能


我们收到的一个常见问题是：“如何从应用程序发送电子邮件？”不幸的是，SMTP被阻止从CloudFoundry.com上运行的应用程序阻止垃圾邮件和其他滥用。

但是，仍然可以从Cloud Foundry应用程序发送电子邮件。服务提供商（例如SendGrid ）可以代表您通过HTTP Web服务发送电子邮件。当应用程序在您自己的数据中心中运行时，您可能要使用公司的SMTP服务器，然后在CloudFoundry.com上运行时使用SendGrid。这是一个如何创建服务bean以连接到云配置文件中的SendGrid的示例。


<beans profile="cloud">
    <bean name="mailSender" class="example.SendGridMailSender">
        <property name="apiUser" value="[email protected]" />
        <property name="apiKey" value="secureSecret" />
    </bean>
</beans>

云属性

除配置文件外，特定于云的属性现在直接公开到应用程序中，并且可以通过Spring的属性占位符支持来使用。这些属性包括有关应用程序的基本信息，例如其名称和云提供商，以及与该应用程序绑定的所有服务的详细连接信息。这使创建自己的服务连接工厂变得容易。如果出于某种原因，您的应用程序需要c3p0连接池而不是提供的连接池，则很容易做到。


<beans profile="cloud">
    <bean id="c3p0DataSource" class="com.mchange.v2.c3p0.ComboPooledDataSource" destroy-method="close">
        <property name="driverClass" value="com.mysql.jdbc.Driver" />
        <property name="jdbcUrl" value="jdbc:mysql://${cloud.services.mysql.connection.host}:${cloud.services.mysql.connection.port}/${cloud.services.mysql.connection.name}" />
        <property name="user" value="${cloud.services.mysql.connection.username}" />
        <property name="password" value="${cloud.services.mysql.connection.password}" />
    </bean>
</beans>

服务属性通常采用“ cloud.services.{service-name}.connection.{property} ”。可用的特定连接属性取决于服务的类型。

为方便起见，如果将一个给定类型的服务绑定到应用程序，则会根据服务类型而不是服务名称来创建别名。对于MySQL服务，属性将采用“ cloud.services.mysql.connection.{property} ”。

现已自动发售


云配置文件和属性都将自动提供给在Cloud Foundry上运行的所有Spring应用程序。需要使用Spring 3.1（或更高版本）。我们的目标是创建一条无摩擦的路径以在Cloud Foundry上运行，同时仍使在需要时易于控制。最重要的是，我们希望为Spring应用程序提供最大的可移植性，使其在任何地方运行，而不必为每个部署环境手动配置应用程序。
