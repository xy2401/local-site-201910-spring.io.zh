Roo + Cloud Foundry =云中的生产力

工程
詹姆斯·泰瑞尔
2011年4月12日

今天，对于开发人员而言，标志着重要的一天，VMware的开源平台即服务产品Cloud Foundry的公开测试版发布。罗德·约翰逊（Rod Johnson）的博客包含有关此激动人心的公告的许多背景细节， 马克·费舍尔（Mark Fisher）的帖子则首次介绍了该服务以及应用程序在本地环境和云之间移动的便捷程度。正如Rod和Mark强调的那样，今天的公告是关于提高和确保开发人员的生产力。

为了支持这一新服务和平台，我们很高兴地宣布，我们已将Cloud Foundry支持集成到Spring Roo （Spring针对Java开发人员的快速应用程序开发工具）中。现在，您可以将Roo的生产力带到云中，甚至不必离开外壳！有数十种命令可简化Cloud Foundry的工作，当然您可以在几分钟内构建一个新应用程序并将其部署到Cloud Foundry。登录后，就像使用新的“ cloud Foundry deploy”命令一样简单，您已完成。

通过Roo的Cloud Foundry集成，您现在可以从外壳管理应用程序的整个生命周期。除了停止和启动之外，您还可以：通过更改正在运行的实例数和分配的内存来扩展它；查看已部署的应用程序和已调配的服务，以及它们如何相互关联，映射和取消映射URL，查看应用程序的详细统计信息以及绑定和取消绑定服务。借助Roo的聪明的自动完成技术，管理您在云中的存在从未如此简单。

Roo的Cloud Foundry支持使您可以轻松部署基于Roo的Java应用程序中使用的通用技术。例如，您可以将熟悉的MySQL后端数据库和JPA与主流Web前端（例如Spring MVC和Google Web Toolkit（GWT））结合起来。在示例方面，以下是Roo在Cloud Foundry上运行的“ Petclinic”示例的屏幕快照，您可以从roo-petclinic.cloudfoundry.com上进行访问。我们还已在roo-expenses.cloudfoundry.com上发布了GWT费用示例。所有这些样本都可以从Shell或在Spring Tool Suite内的单个Roo命令中轻松生成（“脚本开支.roo”或“脚本诊所.roo”）。

如果您想试用新的Cloud Foundry服务，则首先需要一个Cloud Foundry帐户，但一旦获得该帐户 ，我们就会在参考指南的Cloud Foundry章节中发布详细说明（现在在线，但是快速开始，只需下载1.1.3。释放Roo，在Roo Shell中运行“ cloud Foundry登录”，然后按照提示安装Cloud Foundry支持。这些内容涵盖了如何安装Cloud Foundry附加组件以及如何开始部署应用程序。要按照说明进行操作，您将需要Spring Roo 1.1.3。RELEASE ，我们今天才发布，您可以在这里下载 。

希望您喜欢这项新功能和服务。我们欢迎您在社区论坛上提出反馈和提出问题。
