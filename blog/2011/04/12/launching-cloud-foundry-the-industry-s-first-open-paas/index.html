Cloud Foundry: an open source “Platform as a Service” (PaaS) from VMware allowing easy deployment of applications written using Spring, Rails and other modern frameworks. Cloud Foundry breaks new ground in PaaS through supporting a choice of programming models; a choice of services from VMware and third parties; a choice of clouds for deployment; and being delivered in open source.

Since the creation of Spring, we’ve remained faithful to a few core values: notably, Productivity and Portability. Today, these proven values extend beyond Spring and Java into VMware’s broader cloud computing strategy. Spring makes developers more productive by reducing time spent away from business logic; Cloud Foundry eliminates time spent installing and configuring infrastructure such as application servers and JVMs. Spring facilitates portability between deployment destinations; Cloud Foundry is both a key deployment destination itself and a layer that allows deployment to a choice of clouds, both public and private.

推出业界首个开放式PaaS Cloud Foundry

工程
罗德·约翰逊
2011年4月12日

今天，我们为开发人员带来了令人振奋的消息： Cloud Foundry的发布：VMware的开源“平台即服务”（PaaS），可以轻松部署使用Spring，Rails和其他现代框架编写的应用程序。通过支持多种编程模型，Cloud Foundry在PaaS中取得了新突破。 VMware和第三方的服务选择；选择要部署的云；并以开源形式交付。

自从Spring创立以来，我们一直忠实于以下几个核心价值观：生产力和可移植性。如今，这些经过验证的价值已从Spring和Java扩展到VMware的更广泛的云计算策略 。Spring通过减少花在业务逻辑上的时间来提高开发人员的生产力。 Cloud Foundry消除了安装和配置基础架构（如应用程序服务器和JVM）所花费的时间。 Spring促进了部署目标之间的可移植性； Cloud Foundry本身就是关键的部署目标，也是允许部署到公共和私有云选择的层。

Cloud Foundry将成为Spring开发人员的自然云部署选择。

Java迈出的一大步，而不仅仅是Java

迄今为止，还没有一个强大的Java开放式PaaS目的地。数以百万计的Java开发人员基本上只能在云中自生自灭，而比Ruby开发人员所拥有的选择要弱得多。我们正在改变它。

借助Cloud Foundry，VMware为公共和私有云中的Java应用程序提供了一个方便且引人注目的目的地。

我们使此部署目标易于采用。您将不需要以特殊的方式构建应用程序或使用有限的语言或框架功能子集，也不需要调用特定于Cloud Foundry的API。

我们希望你更多 ，而不是更少的生产。我们将全力以赴，以优化整体体验的生产力。仅仅在空中提供目的地以部署应用程序还不够—考虑开发人员的工作方式，并将部署目的地与他们喜欢使用的高效工具链集成在一起至关重要。Cloud Foundry将与现有的Spring技术（例如Spring Roo和SpringSource Tool Suite）紧密集成，以提供特别有效的Spring体验。

今天的公告对Java来说是个好消息，但不仅仅限于Java。我们希望为开发人员提供服务，无论他们选择哪种语言和框架。因此，Cloud Foundry本质上是跨平台的。它在相同的基础架构上支持Rails和Spring应用程序。

我们还具有node.js支持和SPI，以允许VMware和社区添加对新平台的支持。平台之间共享基础架构和服务将为所有开发人员带来重大利益。

Open PaaS，开源

我为VMware继续加深对开源的承诺感到自豪。Cloud Foundry是一个开源项目。

SpringSource将世界上最大，最重要的开源社区之一带入了VMware系列。我们通过拥抱和领导Rabbit MQ和Redis来提高公司的承诺。今天，我们将Cloud Foundry作为一个开源项目以及一个云服务启动。

开源可以在避免锁定的情况下实现快速发展，这是云计算中的一个重要问题。我们认为，开源是推动Cloud Foundry作为一项技术发展的最佳方法，也是用户和客户的最佳选择。

开放性是VMware实施PaaS方法的关键主题。 Cloud Foundry还以其他三种重要方式开放：


  • 正如我已经提到的，Cloud Foundry对多个框架和开发人员社区开放，它支持Spring， Grails ，Node.js和Ruby on Rails，并且已经在其他框架上进行工作。

  • 对由VMware或其他供应商或社区提供的多种服务开放：最初，我们将提供诸如关系数据库之类的基本服务，但是选择范围会迅速扩大。其他初始选择是MySQL，Redis和MongoDB ，并且RabbitMQ将很快出现。

  • 可以选择以下基础部署目标：Cloud Foundry将在CloudFoundry.com上作为公共云服务提供，并且作为您可以在任何位置运行的软件。我们正在Cloud Foundry层和Spring Framework层拥抱可移植性，并期望Cloud Foundry在EC2和其他IaaS产品之上运行。

部署目的地的选择对Java社区特别重要，因为Java被广泛用于构建企业应用程序，并且许多组织都不愿意在公共云中运行其企业应用程序。有了Cloud Foundry，没关系。您可以根据需要在防火墙后运行该技术，并在拥抱云计算时控制自己的命运。

Cloud Foundry对开放性和选择的拥护在PaaS世界中改变了游戏规则，并有可能使数百万开发人员受益。

卷入

这是一个很大的愿景：开放的PaaS提供高质量的集成开发人员体验。这种体验将继续改善：值得注意的是，随着即将在下个季度推出的Code2Cloud ，云服务弥合了开发人员桌面和云部署之间的鸿沟。

希望您同意这将是一个令人兴奋的旅程。更好的是，您可以一路提供帮助。首先通过注册Beta帐户尝试该服务。接下来，您可能要在自己的环境中而不是在我们的公共云中尝试该软件，从而体验Cloud Foundry承诺的核心可移植性。希望您的兴趣会激起，您将希望下载源代码并为该项目做出贡献。与Spring一样，Cloud Foundry都是根据Apache许可2.0版发行的， Apache许可2.0版是一种广泛采用的许可，可帮助创建强大而多样的社区。

无论您想深入研究启用技术，还是成为一名开发人员的好时机。与往常一样，我希望我们的努力可以帮助您专注于削减代码，交付业务价值以及最重要的是获得乐趣！
