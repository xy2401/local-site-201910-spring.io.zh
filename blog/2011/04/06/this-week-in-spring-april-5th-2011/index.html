春天的一周：2011年4月5日

工程
乔什·朗（Josh Long）
2011年4月6日

今年进展很快！我们已经为Spring用户看到了许多激动人心的新版本，而就在今天，也许是最激动人心的事情出炉了。例如，如果您不是因为不是注册的SpringSource Tool Suite用户而得到的，那么以下是这些要点：

下周二-4月12日-VMware将举办一个网络研讨会- “ Spring到云中！” -带有挑衅性的解释是：“ Spring已经简化了企业Java开发。接下来是云开发。”

该网络研讨会将针对欧洲和北美时区进行。有关详情，请参阅此页面 ， 不要忘记注册！


足够令人兴奋的未来，让我们看一下上周发生的令人兴奋的事情！

  1. 上周，Spring Data项目比以往任何时候都跳得更多！首先，对Spring Data JDBC Extensions项目进行更新，现在为1.0.0。M1，具有对Oracle数据库的特定支持。新版本包括RAC“快速连接故障转移”，流高级队列，本机XML类型和自定义数据源连接准备器。精美易用的软件包中包含许多强大的功能。


    有关更多详细信息，请参见发行公告和项目主页 。

  2. Spring AMQP项目进展很快。最新版本标志着第一个RC（候选版本）导致1.0 GA。 Spring AMQP是Java客户端和AMQP协议的Spring集成层。大多数人将通过SpringSource自己的RabbitMQ消息代理使用此功能，该代理是当今最流行，功能最强大的消息代理。


    在此处查看发布公告 ，并查看项目主页以获取更多信息。

  3. 说到AMQP和高度可扩展的，云友好的RabbitMQ部署（等等，我们不是在谈论这个吗？好吧，我们应该去过！RabbitMQ是使用最广泛的消息代理-即使在像Amazon Web Services这样的云环境中，也有许多替代方案，例如Amazon自己的SQS服务）， Helena Edelson撰写了有关RabbitMQ拓扑性能和可伸缩性的精彩评论 。

    该文档一直以来都非常出色，但是有时很难将所有常见的习惯用法和模式放在一起。这篇文章提供了我肯定希望在一年前进行的RabbitMQ扩展导览！绝对值得一读和收藏。

  4. 想了解一下Spring Integration中的新功能吗？关于开源的好处是，您永远不需要太长时间就感到奇怪。在Spring Integration项目中，适配器和额外的模块在沙盒中开始工作。 通常，这些可以与Spring Integration的现有版本一起使用，而不必总是跟踪最新，最出色的Spring Integration版本。最近几个星期我真的很喜欢两个新模块？Spring Integration MongoDB和Spring Integration Redis。 这两个模块允许您以实用且方便的方式在NoSQL空间中利用这些沉重的选择，作为现有Spring Integration解决方案的附加组件。它们都支持使用后端存储的消息存储，并且Redis在Redis的内置消息传递支持之上还提供了基于Spring Integration的发布/订阅支持！当您在那里时，如果发现需求中未说明的内容，请创建一个帐户并创建项目克隆和反馈。MongoDB和Redis的支持还与Spring Integration GemFire支持和Spring Integration AMQP支持很好地吻合。这些选项为您的应用程序提供了所需的所有灵活性和性能，甚至可以满足您在台式机或云中或其他地方的最苛刻的要求！

  5. Spring Data Graph项目对Neo4j的支持刚刚更新到版本1.0.0。RC1。新版本包括经过重新设计的查询API（用可组合的spring-data-commons仓库替换查找器），对Neo4J-REST的新REST客户端支持，多项性能改进， Neo4jTemplate ，文档更新和小型顶级API重构。



    在这里查看并阅读完整的发行公告，不要忘记注册即将举行的网络研讨会 。


  6. Matt Raible最近更新了一些有关使用Spring MVC的AppFuse Wiki条目，以反映Spring和AppFuse本身的最新版本。签出更新的老歌，但一个好吃的东西！

  7. Sivaprasadreddy Katamreddy发表了一篇关于使用Spring MVC 3和Hibernate构建CRUD示例应用程序的博客。博客文章不遗余力，详细介绍了每个代码工件，包括SQL语句和JPA实体类！不错的工作，以及对那些入门的人的有用指南。读者还可能会喜欢Green Beans的文章“服务层中的Spring入门” ，它谈到了与这篇出色文章相同的目标。
