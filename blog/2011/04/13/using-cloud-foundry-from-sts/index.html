从STS使用Cloud Foundry

工程
克里斯蒂安·杜普依斯（Christian Dupuis）
2011年4月13日

到目前为止，您可能已经听说了Cloud Foundry ，这是昨天宣布的来自VMware的开放式PaaS。如果不是，请确保检查网络广播的记录 。最终，您已经阅读了较早的博客文章，其中介绍了对Cloud Foundry的Spring支持 ， Spring Roo的插件和Grails插件 。

在这篇文章中，我想介绍对Cloud Foundry的基于Eclipse的支持，它使您可以管理云部署，包括服务和服务绑定的配置，应用程序扩展，对文件资源的访问等。

安装适用于Eclipse和STS的Cloud Foundry


有三个选项可将Cloud Foundry插件安装到SpringSource Tool Suite（STS）和纯Eclipse中。我将逐步介绍这些选项，以帮助您快速入门。

通过STS扩展安装进行安装


开始使用Cloud Foundry插件的最简单方法可能是将其安装到预先安装的STS副本中。您应该至少具有版本2.5.1。已安装RELEASE； 2.6.1。SR1更好。可以在下载页面上找到各种受支持的操作系统的STS。

在STS中，选择“帮助>仪表板”，然后单击仪表板底部的扩展选项卡。等待扩展列表加载，然后从“服务器和云”类别中选择“ Cloud Foundry集成”；单击“安装”以启动安装过程。


从这里开始，您可以按照安装向导的步骤进行操作。在此过程中，您需要查看并接受许可协议，然后重新启动STS以完成安装。重新启动STS之后，您就可以连接到Cloud Foundry了。

从Eclipse Marketplace安装


Cloud Foundry插件可以很容易地安装到纯Eclipse安装中。为了确保在安装过程中满足所有依赖性，您应该开始安装“ Eclipse IDE for JEE Developers”软件包。可以从Eclipse下载页面或SpringSource成员分发页面下载该软件包。

通过选择“帮助> Eclipse Marketplace”来启动Eclipse Marketplace客户端，开始安装。在搜索字段中键入“ cloud Foundry”，然后选择“ Go”。这将找到“ Cloud Foundry集成”，您可以通过单击搜索结果中列表旁边的“安装”按钮来进行安装。


同样，现在您只需要按照软件安装向导中的步骤进行操作即可；接受许可条款，最终同意安装未签名的内容，并在被要求这样做时重新启动Eclipse。重新启动Eclipse之后，您就可以连接到Cloud Foundry并从开发环境内部部署第一个应用程序。

从更新站点手动安装


如果您不喜欢或无法使用STS扩展安装机制或Eclipse Marketplace客户端，则可以从以下更新站点手动安装集成插件：

http://dist.springsource.com/milestone/TOOLS/cloud/e3.6

请注意：更新站点不支持目录索引。因此，当使用Web浏览器访问URL时，您将收到“拒绝访问”错误消息。

连接到Cloud Foundry


Cloud Foundry Integration插件将云紧密集成到Web工具项目（WTP）服务器基础架构中。这是从Eclipse部署Java Web应用程序的最普遍的方法。你们中的大多数人可能使用服务器视图来部署到本地Tomcat或tc服务器实例。

使用适用于Eclipse的Cloud Foundry集成，您将继续使用“服务器”视图将应用程序部署到云中。首先，我们需要创建一个新的WTP服务器。该服务器代表您的Cloud Foundry帐户，您将能够连接到Cloud Foundry帐户；非常类似于使用“ vmc目标； vmc登录”。

以下步骤将指导您完成连接到Cloud Foundry的过程：


  • 从“窗口>显示视图...>其他...>服务器”中打开“服务器”视图

  • 右键单击“服务器”视图，然后选择“新建>服务器”

  • 从“ VMware”类别中选择“ Cloud Foundry”，然后单击“下一步”

  • 输入您的帐户信息，然后选择要连接的云。如果您拥有Cloud Foundry帐户，则从URL下拉框中选择“ VMware Cloud Foundry – http://api.cloudfoundry.com” 。您还可以将Cloud Foundry集成与Cloud Foundry的本地安装一起使用。最后按“验证帐户”以检查连接是否可以成功建立。

  • 单击“完成”以关闭“新服务器”向导



完成“新服务器”向导后，您应该在Cloud Foundry的“服务器”视图中看到一个新条目。将自动建立与Cloud Foundry的连接，并且您将在“服务器”视图的Cloud Foundry服务器节点下方看到已部署的应用程序。


您可能会注意到，视图工具栏中的“开始”和“停止”操作被禁用。相反，右键单击Cloud Foundry服务器时，您将在上下文菜单中找到“连接”和“断开连接”操作。

部署应用


现在，将应用程序部署到Cloud Foundry只是将应用程序拖放到“服务器”视图中的Cloud Foundry服务器上的问题。或者，您可以使用服务器上下文菜单中的“添加和删除...”操作。

将应用程序添加到Cloud Foundry之后，即可启动它：在“服务器”视图中右键单击该应用程序，然后选择“启动”。这将打开应用程序部署向导，该向导允许您指定应用程序名称，URL和内存预留。将来我们将添加更多配置选项。

单击“完成”后，插件将上传并在Cloud Foundry上启动应用程序。您可以通过在“服务器”视图中双击应用程序来查看应用程序部署的状态。这将显示“服务器编辑器”，该服务器编辑器已扩展为显示Cloud Foundry的特定信息。

服务提供和绑定

您想要部署的应用程序可能需要访问Cloud Foundry提供的服务。要最初设置服务，可以使用“服务器编辑器”：在“应用程序”选项卡上，从“服务”部分的工具栏中按“添加服务”操作。

将服务配置到您的Cloud Foundry帐户后，您可以将它们绑定到您的应用程序。为此，只需将要绑定到特定应用程序的服务拖放到“服务器编辑器”右侧的“应用程序服务”表中即可。如果您的应用程序需要在启动期间访问服务，并且在没有可用服务的情况下无法启动，请在启动应用程序之前绑定服务。

远程文件访问


用于Eclipse的Cloud Foundry集成提供对远程文件资源的访问，就像“ vmc文件”和“ vmc日志–全部”一样。单击“服务器编辑器”中应用程序详细信息窗格右下方的“远程系统视图”链接。这将调出“远程系统”视图，使您可以直接在Eclipse中浏览文件树并打开文件，例如日志文件。

下一步是什么

在未来几周内，我们将继续改善Cloud Foundry Eclipse的支持。一如既往，我们非常重视社区反馈。因此，请花一些时间来检查插件，并告诉我们您的想法。

如果发现任何问题，请在STS问题跟踪器中提出JIRA。
