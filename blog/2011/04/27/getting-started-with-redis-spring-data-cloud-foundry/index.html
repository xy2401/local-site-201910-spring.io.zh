Redis，Spring Data和Cloud Foundry入门

工程
Costin Leau
2011年4月27日

NoSQL解决方案普及背后的推动力之一是（尤其是）在高负载下的性能。由于它们的数据模型，关键值存储处于领先地位，提供了轻便而灵活的数据处理方式。在本条目中，我将通过一个Spring数据示例（RetwisJ）快速展示使用Spring（Spring Redis）与键值存储（Redis）一起工作以及将应用程序部署到云中 （通过Cloud Foundry）的过程。与世界分享。通过使用Windows作为变更的部署平台，我可以走得更远。

雷迪斯

键值存储中最受欢迎的选择是Redis ，这是一种开源的疯狂的快速数据库，以ANSI C编写，服务器的权重约为200 KB（是， 千字节），而整个软件包的权重约为400 KB（包括命令行客户端）和一些管理实用程序），并且几乎可以在所有主要平台上使用，这也使其成为样本的选择。请注意，除非需要在本地运行示例，否则不需要Redis实例。如果是这样，对于Windows用户（例如作者），您将真正使用（并推荐） 此预打包版本，提供32位和64位版本。

Spring和NoSQL

如果您从Java环境中使用NoSQL，请看一下Spring Data ：它本身不是一个项目，而是一个包含各种新数据访问技术的项目的伞，这些新数据访问技术包括非关系数据库（如Redis或MongoDB），map -减少Java应用程序的框架（例如Hadoop）和基于云的数据服务。Spring Data推广了经典的 Spring值，通过消除API噪声，样板代码和资源管理并提供一致的编程模型来提高开发人员的生产率。它建立在现有Spring功能和项目（例如控制反转，生命周期管理，类型转换，可移植数据访问异常，缓存等）的基础上，因此可以立即添加它，而在其应用程序中的工作量却最小。当然，与其他Spring项目一样，它是开源的，并且在Apache许可下可用。

Spring Data Redis

对于Redis，Spring Data通过Spring Data Redis或简单的Spring Redis项目提供专门的支持。它提供了低级和高级功能，从可移植的Redis客户端抽象（允许通过字面上的一个配置行更改不同的Redis客户端，例如Jedis，Jredis或RJC）到Redis支持的原子集合或计数器或pub-sub支持。项目参考文档非常详细地介绍了这些主题。

YATC RetwisJ-另一个Twitter克隆

RetwisJ源代码（包括此博客中的代码）可以在Spring Data Key Value 示例项目中下载。此外，可在此处找到文档

RetwisJ可以看作是Redis Retwis示例的Java端口：一个简单的Twitter克隆，演示了如何使用Redis灵活的数据模型（例如集合交集 ）替换传统的关系数据库中的昂贵联接。

从表式思维方式迁移到键值关联似乎很困难，但并非如此：相反，将多个值存储在一个键下可以将每个值存储在“相似键”下；实际上，关系本身也可以这样存储。如文档中所述，而不是为用户创建一个表（包含一列用于用户名，一个用于密码等），您可以仅通过Redis哈希（或映射）存储项目，而无需创建要存储的索引条目之间的关系，它可以为其添加单独的，反向或查找键。

以类似的方式，可以通过Redis集映射关注者和被关注者（或目标）之间的关联表。因此，与其执行表联接，不如设置交集。Redis丰富的数据模型（集合，Z集合（或排序集合），列表和哈希）通过Spring Redis很好地映射到Java集合上，该集合在Redis之上提供了相应的java.util实现。这意味着您可以使用众所周知的界面来遍历，查找或修改列表并进行设置，而无需手动发出任何Redis命令。

以机智：

private RedisSet<String> following(String uid) {
   return new DefaultRedisSet<String>(KeyUtils.following(uid), template);
}
      
public Collection<String> commonFollowers(String uid, String targetUid) {
   Set<String> followers = following(uid).
       intersectAndStore(following(targetUid),
       KeyUtils.commonFollowers(uid, targetUid));
   ...
}

对于通用数据访问，中央Spring Redis是RedisTemplate ，它允许从丰富的对象的简单，单行操作，序列化（基于字节数组，基于XML或Jackson的序列化）或消息发布到高级查询功能和数据获取（例如通过Redis SORT / GET模式避免臭名昭著的N + 1问题）。

// adding entries into a hash
BoundHashOperations<String, String, String> userOps = template.boundHashOps(KeyUtils.uid(uid));
userOps.put("name", name);
userOps.put("pass", password);
valueOps.set(KeyUtils.user(name), uid);

有关数据模型和样本的更深入的说明，请参阅RetwisJ 文档 。

连接到Redis

Java用户在连接到Redis时面临的一个常见问题是要使用的客户端 （或驱动程序）。Spring Redis以一致的方式支持三种不同的库（ Jedis ， JRedis和RJC ），因此人们可以在它们之间进行切换而无需重写任何代码。让我们选择Jedis，看看应用程序配置如何：

<beans>
  <context:property-placeholder location="classpath:redis.properties"/>

  <!-- Redis client -->
  <bean id="connectionFactory" class="org.springframework.data.keyvalue.redis.connection.jedis.JedisConnectionFactory"
		p:host-name="${redis.hostname}" p:port="${redis.port}" p:password="${redis.password}"/>

	<bean id="redisTemplate" class="org.springframework.data.keyvalue.redis.core.StringRedisTemplate" 
		p:connection-factory-ref="connectionFactory"/>

	<context:annotation-config />
	<context:component-scan base-package="org.springframework.data.redis.samples"/>

</beans>

非常简单-我们声明驱动程序和Redis属性（在运行时通过redis.properties中的属性占位符替换 ），启用注释配置并扫描类路径以查找bean。

从部署的角度来看，RetwisJ只是一个Web应用程序，一个WAR可以部署在任何（Servlet 2.5）Web容器中，例如Tomcat。它由一个简单的Spring @MVC控制器，一个Repository类和两个域对象组成： Post和User 。以Retwis为例，甚至安全性也将通过Redis而不是容器HttpSession来处理-这不仅显示了键值存储的另一个用例，而且还提高了可伸缩性，因为Redis（以及用户数据）被分发到其他地方。盒子。

要构建它，只需在根文件夹中输入：

./gradlew build

并运行它，将产生的war（ build / libs / retwisj.war ）部署到您选择的Web容器中。

将RetwisJ部署到Cloud Foundry中

专用样本存储库中提供了适用于Cloud Foundry的RetwisJ示例

将上面的WAR部署到Cloud Foundry中非常简单-您只需要一个帐户（您可以在此处 免费注册）。完成后，您可以使用它通过STS（如先前条目中所述 ）或命令行（ vmc ）部署应用程序，这就是我将在本文中使用的。

由于我们正在使用Redis，因此需要用适合Cloud Foundry的本地实例属性替换本地实例属性，这些属性作为环境属性公开。如这些文档中所述 ，通过云名称空间提供了专用的Spring支持。

我们可以使用它，而不是从本地文件中读取Redis属性，而可以从环境中获取它们：

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
		xmlns:p="http://www.springframework.org/schema/p"
		xmlns:context="http://www.springframework.org/schema/context"
		xmlns:cloud="http://schema.cloudfoundry.org/spring"
		xsi:schemaLocation="
			http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd
			http://www.springframework.org/schema/context http://www.springframework.org/schema/context/spring-context.xsd
      http://schema.cloudfoundry.org/spring http://schema.cloudfoundry.org/spring/cloudfoundry-spring.xsd">

  	<cloud:service-properties id="cfoundryEnv" />
  	<context:property-placeholder properties-ref="cfoundryEnv"/>
	  
    <bean id="connectionFactory" class="org.springframework.data.keyvalue.redis.connection.jedis.JedisConnectionFactory"
      p:host-name="${redis.hostname}" p:port="${redis.port}" p:password="${redis.password}"/>

属性名称不必更改-我们可以命名（在这种情况下为redis ）绑定到应用程序的Redis实例，并使用它来最小化本地环境和云之间的更改。

请注意，这只是众多选择中的一种：您可以使用Spring 3.1 配置文件自动进行切换，使用云命名空间自动为您创建RedisConnectionFactory或自动将其连接到您的应用程序。有关可用功能的更多信息，请参见本节 。
