带有Neo4j支持的Spring Data Graph 1.0今天发布

工程
迈克尔·汉格
2011年4月19日

此公告帖子是Neo4j团队的开发人员的来宾贡献，该团队与SpringSource的Spring Data团队密切合作以提供Spring Data Graph集成库。

为了方便地介绍Spring Data Graph，我们将于4月20日在美洲和欧洲的两个便捷时间举办一次由Neo Technology首席执行官Emil Eifrem主持的VMware免费网络研讨会 。[更新：网络研讨会视频现在可以在SpringSourceDev频道的youtube上找到。]

既然Spring即将面世 ， Neo4j和Spring Data团队很高兴，将近一年的工作量已经产生了我们1.0版本的Spring Data Graph库。Spring Data项目旨在将便利的Spring Framework编程模型引入NOSQL数据库。Spring Data Graph支持Spring Data中的图形数据库 ，并包括对Neo4j的第一个受支持的实现。

了解Spring Data Graph的最好方法是使用它，因此我们鼓励您阅读Spring Data Graph指南手册 ，在github上查看示例和源代码 ，并告诉我们您的想法。

Spring Data Graph提供了从任何Spring应用程序中对图形数据库的简单便捷访问。Spring Data Graph最初是SpringSource首席执行官Rod Johnson和Neo Technology首席执行官Emil Eifrem的创意。Rod和Emil共同完善了设计理念，并完善了一些基础技术（如AspectJ）（用于将POJO透明地映射到基础图形）。

Spring Data Graph在JAOO Aarhus 2010上的一个演示中公开首次亮相，当时Rod邀请了Neo Technology的Michael Hunger展示跨商店的持久性作为其SpringSource演示的一部分。10月下旬，在SpringOne会议上举行了几次会议，讨论了Spring Data和Spring Data Graph项目的不同方面。

在过去的一年中，我们定期从社区中获得反馈，并以频繁的迭代发行版作为回应，每个发行版都改进并扩展了库的范围和功能。总之，我们已经建立了对分离实体的支持，图形内类型表示策略，存储库支持和Neo4jTemplate实现。我们还开发了对Spring Roo的附加支持。

以下是Spring Data Graph功能的概述：


  • 支持属性图（通过关系连接的节点，每个都有任意属性）

  • 带注释的POJO实体的透明映射

  • Neo4jTemplate具有便捷的API，异常转换和可选的事务管理

  • 将类型信息保留在图中的不同类型表示策略

  • 动态类型投影（鸭式打字）

  • Spring Data Commons存储库支持

  • 对部分JPA的跨商店支持-图形实体

  • Neo4j对动态字段和通过存储库方法的遍历支持

  • Neo4j索引支持（包括全文和数字范围查询）

  • 支持JSR-303（Bean验证）

  • 对Neo4j服务器的支持

  • 支持在Neo4j Server中作为扩展运行


我们对我们的Spring Data Graph指南手册“ Good Relationships ”感到非常满意，该手册是一本带有叙述性教程和参考的双层书 ，其中包括Rod和Emil的前言。

另一个亮点是社交电影数据库cineasts.net ，它是使用Spring和Spring Data Graph构建真实系统的广泛教程的一部分。cineasts.net应用程序在Spring框架，Spring Data Graph和Neo4j Graph数据库上运行，并使用免费的电影数据并打开themoviedb.org 。

我们的社区也一直在忙于在Spring Data Graph之上构建高效且有趣的系统。例如，代尔夫特工业大学已将Spring Data Graph用于大型复杂的能源市场模拟。

Spring Data Graph是根据Apache软件许可发布的。它的源代码托管在github上 。那里也有几个例子。在springsource.org 上提供了问题跟踪器和论坛 。您可以在Twitter上关注@SpringData和@ Neo4j ，还可以订阅Neo4j邮件列表以获取更多信息。

感谢所有支持Spring Data Graph开发的人员，他们的经验，技能和时间！
