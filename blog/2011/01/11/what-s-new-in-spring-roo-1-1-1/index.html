Spring Roo 1.1.1的新增功能

工程
本·亚历克斯
2011年1月11日

我们很高兴宣布Spring Roo 1.1.1全面上市。这个新版本包含250多个修复和增强功能 ，包括许多性能和功能改进。

与往常一样， SpringSource Tool Suite （STS）2.5.2将很快发布，并包含此新版本的Spring Roo。您还可以下载 Spring Roo 1.1.1的独立版本并立即开始使用。

让我们简要浏览一下Roo 1.1.1中添加的一些更显着的增强。如果您要升级基于Roo的项目，建议您阅读参考指南中包含的特定于版本的升级说明 。下载的ZIP中还包括HTML和PDF格式的参考指南。

性能，占用空间和可用性

Roo团队始终专注于Roo的性能，占用空间和可用性。

在一些常见情况下， Roo的运行速度是以前的两倍 ，因此已经实现了一些显着的性能改进。例如，在同一台计算机上，使用Roo 1.1.0运行“ roo script clinic.roo”来创建Petclinic样本以前花费了25秒，而使用Roo 1.1.1则只花费了11秒。我们还继续分析和增强Roo内部，以在1.1.2中提供更好的性能分析和结果。

可用性已通过多种方式得到增强。我们已经 大大 扩展 了 文档 ，特别是对数据库逆向工程和JDBC驱动程序安装的处理。Roo 1.1.1还包括一些新的Shell功能，这些功能使Roo易于使用。其中包括报告错误（如果向Roo命令提供了未知选项），对命令选项的大小写不敏感的分析，命令期间的更好消息以及更有用的提示 。

我们还稍微减少了Roo的占用空间。Roo 1.1.0发行版ZIP是7.2 Mb，但是Roo 1.1.1现在有了新功能，现在是6.8 Mb。尽管下载量并没有明显减少，但很高兴Roo的每个版本都没有变大，而是朝相反的方向发展。

持久性功能

使用Roo的大多数人都在构建企业Java应用程序。持久性是此类应用程序的关键要求，因此我们继续增强Roo 1.1.1中的持久性功能。

Roo 1.1.1现在更好地支持了云持久性。特别是，此新版本增强了对VMforce的支持 ，包括对Data Nucleus 2.1.3的升级。我们还改进了Google App Engine（GAE）数据存储支持。Roo现在通过其类型间声明（ITD）提供特殊的GAE关系处理 。这意味着您现在可以为GAE关联的对象调用accessor和mutator方法，Roo将在内部管理标识符的持久性和对象检索的语义。

Roo 1.1.0中引入的增量数据库逆向工程（DBRE）功能非常受欢迎，得到了社区的大量反馈和对新增强功能的要求。该功能的质量不断提高，它为用户提供了新的支持，这些功能可让用户指定他们希望包含或排除的表 ，针对DBRE管理的实体的自动集成测试以及对DBRE表之间多个关系的支持。

人们在使用Roo 1.1.0时经常向我们询问的一个问题是，如果我们不随Roo一起提供JDBC驱动程序，如何对数据库进行内部检查。这导致我们发布了一个临时脚本，人们可以使用该脚本来安装通用驱动程序，但是这种方法对于每个可用的JDBC数据库驱动程序都不实用。因此，在Roo 1.1.1中，我们通过Roo广泛的附加机制添加了对自动发现和安装任何JDBC数据库驱动程序的支持。当您首次尝试在Roo 1.1.1中进行DBRE操作时，如果未安装JDBC驱动程序，则会看到一个搜索结果，显示适合您的附件并指示用于安装该附件的命令。最重要的是，社区中的任何人现在都可以为不受支持的数据库生成新的JDBC附加组件，并将其提供给所有Roo用户。我们甚至添加了一个新的附加程序创建器模板，专门用于将JDBC驱动程序包装为适合Roo附加程序发现的形式。您可以在参考指南一章中了解有关此新功能的更多信息。

基于JPA的标准（非云，非DBRE）Roo应用程序还具有一些新的持久性功能。这些包括并发支持多个数据库 ，以及在实体上指定版本字段 。

网络层

我们的两种内置Web技术都在Roo 1.1.1中得到了改进。

自Roo 1.1.0起，对Google Web Toolkit（GWT）的支持已得到实质性增强。原始实现中许多查询的来源之一是如何修改Roo管理的代码。在Roo 1.1.1中，我们现在通过引入仿制ITD模型全面解决了这一要求。很容易确定Roo正在管理的GWT源文件，因为它们采用与常规Roo应用程序相似的文件名约定（即* _Roo_Gwt.java）。这意味着您对文件所做的任何更改都将被 Roo 保留 。我们还增加了对GWT 2.1.1新功能的支持，例如代理实体中的继承。

那些使用Roo构建Spring MVC应用程序的人还将在1.1.1中享受一些新的好处。首先，对JSON REST控制器有更全面的支持。还提供了一个新的集中式ConversionService来替换以前的特定于控制器的转换器，因此，如果您正在开发基于Roo 1.1.0的MVC项目，则应查阅前面提到的升级说明 。增量数据库逆向工程（DBRE）兼容性也得到了改进，特别是通过Web层对复合主键的支持 。我们还进行了内容类型调整，因此您可以更轻松地使用Apache Tiles以外的模板框架，并升级到Spring Security 3.0.5 。

附件发现和安装

Roo 1.1.0中首次引入的附加发现和安装功能一直在不断改进。您会发现现在有更好的附加搜索功能 ，具有内置关键字匹配， 基于排名的排序 ，灵活的行格式，版本兼容性和受信任的开发人员功能。现在，基于分散式PGP的安全性基础结构也与代理服务器后面的那些基础结构更加兼容，并且我们已经将新的附加搜索功能合并到了外壳程序的未知命令解析器和JDBC驱动程序定位器中。所有这些都意味着在Roo中查找和安装新功能的过程更加简单，轻松。

附加或插件模型最困难的问题之一是人们如何确定哪些扩展值得使用。如果许多工具不值得使用，损坏或无法维护，则没有数百个工具扩展。我们决定在Roo中解决此问题，方法是让Roo 自动收集有关人们正在使用哪些公共加载项的匿名信息 ，并将此汇总信息合并到加载项搜索结果中。当然，只有在您同意Roo收集此信息的情况下，才会发生这种情况，并且收集到的信息绝不能用于识别您或您的项目。同样，Roo现在还允许社区提供有关他们正在使用的加载项的可选反馈 （例如，评分和评论），并且此反馈包括在加载项搜索结果中。这些功能提供了一种自动确定附加组件受欢迎程度的高效方法，而无需用户通过访问网站进行投票和评论等带来的不便。

Roo 1.1.1还包括Spring用户代理分析（UAA）支持。UAA启用了上述附加排名数据收集功能，并帮助我们收集有关Roo使用情况的匿名信息。这些信息极大地帮助我们实现了互操作性（因为我们知道人们在项目中一起使用了哪些通用公共技术）并确定了路线图的优先级。此功能默认情况下处于禁用状态，未经您的同意将无法运行。如果您想了解有关UAA及其如何确保您的隐私的更多信息，请参阅UAA常见问题解答 。

附加开发

最后，Roo 1.1.1提供了许多新功能和增强功能，以帮助附加组件开发人员构建附加组件。首先，我们通过“ addon create”命令提供了更完整的示例 。有一些新的实用程序类，用于查找捆绑包中的资源 ，以及显示文本文件和控制过滤器定位 。可以从控制台输出中抑制线程名称，并且对内部类型和更复杂的 Java编译单元创建提供了额外的支持。开发模式还提供许多新功能，例如深度日志记录 ，用于文件系统操作的SHA1代码以及在出现问题时取消自动撤消 （以帮助您查看生成的文件）。元数据层有很多增强功能，包括通过AbstractMemberDiscoveringItdMetadataProvider提供的更好的元数据扫描模型，针对ITD MetadataProviders的更灵活的类级下游解析以及自动无限元数据循环检测和重试功能。最后，我们增强了许多文件系统的内部功能，例如延迟的文件写入 ，甚至延迟的文件删除 ，这有助于通过多次传递创建复杂的ITD元数据。

所有这些增强功能使编写可满足非常复杂的ITD和元数据要求的加载项变得更加容易，但这些加载项却表现出色且易于调试。Roo 1.1.2（并且已经在Git master中）中还引入了显着增强的元数据日志记录 ，如果您正在使用基于ITD的附加组件，则可能会对您感兴趣。

结论

我们已经研究了Spring Roo 1.1.1中包含的250多个修复和增强功能中的一些。无论您是使用Roo构建企业项目，还是为Roo社区构建附加组件，我们都致力于在此新版本中显着改善Java开发体验。

别忘了STS 2.5.2只是几个小时的路程，它将嵌入Roo 1.1.1和其他对开发基于Roo的应用程序有用的技术。您也可以立即下载Roo 1.1.1并立即开始使用。

如果您对此新版本有任何疑问，请在Spring Roo社区论坛上提问。您还可以通过Twitter 关注开发团队成员以获取最新的开发更新，并关注@SpringRoo以获取将来的发行公告。请在与Roo相关的任何推文中使用#springroo ，以便其他人可以轻松看到它们。

我们希望您喜欢这个新版本。
