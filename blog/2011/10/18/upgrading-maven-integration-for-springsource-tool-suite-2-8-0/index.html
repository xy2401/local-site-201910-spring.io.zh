升级SpringSource Tool Suite 2.8.0的Maven集成

工程
安德鲁·艾森伯格
2011年10月18日

更新，11月23日：我们发布了STS 2.8.1，该版本修复了人们遇到的许多m2eclipse问题的安装/卸载。阅读发行公告。 现在，可以轻松升级/降级m2eclipse。STS 2.8.1完全支持m2eclipse的两个版本。

今天，我们发布了SpringSource Tool Suite的 2.8.0版本。 随着更新到Eclipse的靛蓝SR1（其为您带来的Java7完全支持IDE），我们还更新了从旧的和没有维护Maven的整合m2eclipse的版本0.12.x到最新版本M2E 1.0 -这是现在出货的部分Eclipse发布火车。STS 2.8.0软件包附带了m2e 1.0，并预先安装了几个兼容的加载项（项目配置器）。我们还实施了一个迁移助手，可以帮助您将现有项目升级到新的m2e版本。该迁移助手会将所有旧版Maven项目的类路径，构建器和项目性质转换为与m2e 1.0兼容。它不会更新旧的启动配置或项目/工作区设置。这些将需要手动迁移。

不幸的是，升级现有的STS 2.7.x（或更旧版本）安装需要一些额外的工作。首先，按照“ 安装说明”文档中 “更新站点安装”部分的说明进行操作 。更新到2.8.0后，即使尚未升级到m2e v1.0，也会调用m2e迁移助手。最好等到升级到m2e v1.0之后再进行迁移。您的项目不会造成长期损坏，但是在您完成以下所述的升级过程之前，将无法构建它们。

卸载现有的m2e 0.xx组件


  1. 启动您的STS（最好使用空的工作空间，以避免与现有项目混淆
    使用m2eclipse 0.12.x）。

    2. <li>Go to the "About" dialog and click on "Installation Details".</li>

<li>In the 'Installed Software' list that opens, select the <code>org.maven.ide.eclipse.*</code> and the <code>org.sonatype.*</code> entries.  More specifically, there should be five:
    • “ AJDT的Maven集成（可选）”：id = org.maven.ide.eclipse.ajdt.feature.feature.group
    • “ Eclipse的Maven集成（必需）”：id = org.maven.ide.eclipse.feature.feature.group
    • “用于WTP的Maven集成（可选）”：id = org.maven.ide.eclipse.wtp.feature.feature.group
    • “常用Maven项目的项目配置器（临时）”：id = org.maven.ide.eclipse.temporary.mojos.feature.feature.group
    • “ Tycho项目配置器”：id = org.sonatype.tycho.m2e.feature.feature.group

    [caption id =“ attachment_10091” align =“ aligncenter” width =“ 871” caption =“要卸载的Maven功能”] [/字幕]

  2. 按“卸载”

  3. 重新启动STS（同样，最好使用空的工作空间）

安装新的m2e组件
现在，我们将从扩展安装中安装新的m2e 1.0支持，包括许多
与该m2e版本兼容的加载项。我们建议从仪表板安装而不是直接从更新站点安装，因为从仪表板安装将自动安装许多常用的项目配置器（这样您以后就不必这样做了）。


  1. 打开仪表板，然后选择扩展选项卡。

  2. 选择m2e 1.0.100复合安装，然后按“安装”
    [标题id =“ attachment_10094” align =“ aligncenter” width =“ 797”标题=“从仪表板安装Maven”] [/字幕]

  3. 重新启动STS

嗯...我的项目不再建立
重新启动STS后，您可能会看到一些maven项目不再构建，并且在pom.xml文件中有如下错误：
生命周期配置未涵盖插件执行

不要惊慌。从1.0版开始，关于如何在Eclipse / STS中运行Maven插件，m2e变得更加严格。为了使Maven插件作为项目构建的一部分运行或在Eclipse / STS内部进行配置，必须有一个关联的Eclipse插件（称为项目配置器）来管理Maven插件的执行。

有一个简单的解决方法。转到首选项-> Maven->发现，然后单击“ 打开目录 ”。这将允许您浏览所有可用的m2e项目配置器。查找适用于您所看到的错误的配置程序并安装它们。

如果您仍然发现问题或有任何疑问，请在此博客或STS论坛上发表评论。我们将很乐意提供帮助。
