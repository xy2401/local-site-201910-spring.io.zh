春天的一周：2011年5月3日

工程
乔什·朗（Josh Long）
2011年5月4日

已经是五月了！好像昨天我们在敬酒新年的来临……

就像他们说的那样，时间过得开心！今年真是过山车-每天都有令人兴奋的新闻和事件-太多了，一定不能跟上！

  1. 乔恩·布里斯宾（Jon Brisbin）写了一篇史诗般的帖子，介绍了CloudFoundry项目以及您可以在其中使用的许多技术。

    这是今天才提出的，现在是我最喜欢的介绍CloudFoundry的博客之一。就像CloudFoundry本身一样，它是如此雄心勃勃！

  2. 示例的另一个杰作是Costin Leau撰写的博客， Redis和Spring Cloud Foundry入门 。无论您正在做CloudFoundry，Spring Data还是两者都做，这篇文章非常值得阅读。充满活力的二重奏，的确！

  3. European community members can learn more about Spring, Spring Data and Cloud Foundry at the S2G Forum Series: Amsterdam (May 26th) and London (May 31st). There will be tons of great sessions about Spring, Groovy and Grails as well as talks focused specifically on CloudFoundry, Tomcat and Gemfire so be sure to register for the event closest to you. 


    4.

  4. Spring Security 3.1.0。RC2已发布 –标题进行了汇总。查看变更日志的链接。有关Spring Security的更多信息，请继续阅读-还有一些关于Spring Security的非常有趣的帖子！

  5. 春季社交1.0.0。M3发布 。新版本继续了早期版本中提出的核心概念的激动人心的变化。具体来说，此新版本增强了一些社交API的API，扩展了插件机制，并提供了预打包的Spring MVC Controller为您的应用程序增加了提供商登录功能。

  6. 在CloudFoundry博客上， Killian Murphy向社区更新了CloudFoundry项目中的最新信息 。活动非常了不起，而且-从CloudFoundry首次亮相以来的三周内，人们已经在GitHub.com上向GitHub.com提出了对JRuby，PHP和Erlang支持的请求，这证明了这一点。惊人！

  7. 最近，SpringSource进行了有关dm服务器和处女座使用情况的调查。该调查的结果可在此处获得 。

  8. 在这篇文章- 用于本地和CloudFoundry.com服务器的单个Spring应用程序部署中 -David Salter介绍了Spring 3.1的配置文件，以及如何使用它们在本地和云中部署单个。逐步检查它！

  9. Tobias Trelle 引入了RabbitMQ和Spring ， 引导开发人员进行安装和设置。

  10. Rafal Borowiec引入了Spring Security 3.1的增强的安全名称空间配置 。

  11. Steffen Luypaert写了一篇有关从Spring Security 2迁移到Spring Security 3的有趣文章。它着重说明了确保平稳升级所需的一些特定步骤-包括一些可能遇到的非常特殊的情况的“陷阱”。很酷！

  12. CloudFoundry PaaS的透视图。它的主旨– CloudFoundry减少了摩擦并使开发人员能够使用 –应该与已经有机会使用它的众多开发人员产生良好的共鸣。进行检查，然后访问CloudFoundry.com和CloudFoundry.org ，当然可以自己尝试并了解更多！

  13. 人们经常问我关于学习Spring的最佳方法以及如何获得培训以提高Spring技能的问题。我无一例外地向他们推荐了由Spring技术背后的项目负责人和工程师创建的SpringSource培训。培训者也是SpringSource中最有知识的人员-他们具有从概念到API到实现的整个技术堆栈的宏观，鸟瞰和微观视图。因此，很高兴地提醒您，SpringSource刚刚宣布了针对培训和培训计划的新认证。请参阅此认证概述和此计划表以获取更多信息。

    如果您像我一样，并且从一开始就一直在使用和倡导Spring和Spring项目，那么请理解，这次培训很可能还会为您提供一些帮助。我学到了多少知识， 令我震惊 ！
