Groovy-Eclipse中更好的DSL支持

工程
安德鲁·艾森伯格
2011年5月9日

Groovy语言是创建特定领域语言（DSL）的绝佳平台。好的DSL可以使程序更简洁，更具表现力，并使程序员更高效。但是，直到现在，编辑器中的Groovy-Eclipse都不直接支持这些DSL。大量使用DSL时，诸如内容辅助，搜索，悬停和导航之类的标准IDE功能将失去其价值。一段时间以来，可以编写一个Eclipse插件来扩展Groovy-Eclipse，但这是一种重量级的方法，需要对Eclipse API的专门知识。现在，Groovy-Eclipse支持DSL描述符 （DSLD），在Groovy-Eclipse中支持自定义DSL将变得更加容易。

一个简单的例子

考虑一下Joachim Baumann描述的DSL 。他创建了一个简单的DSL以用于远距离工作。使用此DSL，您可以编写如下内容来计算总行驶距离：


3.m + 2.yd + 2.mi - 1.km

这是一个简单而富有表现力的DSL，但是当您在Groovy-Eclipse中将其键入Groovy编辑器时（为简洁起见，假定$url在其他位置定义）：

[标题ID =“ attachment_8774” align =“ aligncenter” width =“ 179”] [/字幕]

您看到的是下划线，没有任何悬停，这意味着编辑器无法静态解析DSL的表达式。使用DSLD，可以教给编辑者这些自定义DSL背后的一些语义，并提供有关悬停的文档：

[标题id =“ attachment_8775” align =“ aligncenter” width =“ 683”] [/字幕]

要为距离DSL创建DSL描述符，您只需要向Groovy项目中添加一个扩展名为.dsld并具有以下内容的文件：


currentType( subType( Number ) ).accept {   
   property name:"m", type:"Distance", 
    doc: """A meter from $url"""
}
