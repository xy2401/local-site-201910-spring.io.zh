适用于Spring开发人员的Micro Cloud Foundry

工程
乔什·朗（Josh Long）
2011年8月24日

今天，VMware团队发布了Micro Cloud Foundry ，这是流行的开源平台即服务的完整本地版本，使开发人员可以在Mac或PC上运行功能全面的云。使用Micro Cloud Foundry，开发人员可以在本地构建端到端的云应用程序，而无须配置中间件的麻烦，同时又可以保留部署位置的选择和扩展其应用程序的能力，而无需更改代码行。

Micro Cloud Foundry当然支持Spring和Java，但也为Scala，Node.js和Ruby提供了运行时环境，因此您可以释放内部的多语言程序员 ！Micro Cloud Foundry还提供了许多服务，例如MongoDB，MySQL和Redis，可以立即使用，而无需进行大量的安装和配置。借助内置的动态DNS支持，开发人员可以在碰巧工作的任何地方（无论是在家，办公室还是咖啡店）运行Micro Cloud Foundry，而无需进行任何重新配置。在Micro Cloud Foundry上创建和测试应用程序之后，您可以轻松部署应用程序，而无需更改到www.cloudfoundry.com或其他Cloud Foundry实例-它实现了跨多种云环境的真正应用程序可移植性。

Micro Cloud Foundry可作为可免费下载的虚拟机映像使用，并且与适用于Mac OS X和VMware Workstation的VMware Fusion和适用于Linux和Windows计算机的VMware Player兼容。它提供了轻松的安装，设置和虚拟机管理，您只需要一个Cloud Foundry帐户即可上手。

Spring和Micro Cloud Foundry入门

我整理了一段视频，以获取，安装，配置然后将Spring应用程序部署到Micro Cloud Foundry。

简而言之，以下是针对Spring Developers的Micro Cloud Foundry入门的五个步骤


  1. 请遵循Micro Cloud Foundry的下载和安装说明 。

  2. 启动您的SpringSource Tool Suite副本并安装Cloud Foundry eclipse支持。您可以通过选择底部的“ 扩展”选项卡从STS仪表板执行此操作。查找并安装Cloud Foundry集成 。它在“ 服务器和云”部分下，但是您也可以使用“ 查找：”字段进行搜索 。当安装提示您重新启动时，请重新启动。您只需一个Eclipse插件即可与任何 Cloud Foundry提供程序一起使用，无论是托管的CloudFoundry.com，本地的Micro Cloud Foundry或任何其他实现。

  3. STS重新启动后，打开“ 服务器”面板。右键单击“ 服务器”面板，然后选择“ 新建”>“服务器” 。在“ 定义新服务器”窗口中，在VMware文件夹下找到Cloud Foundry服务器。通过Cloud Foundry集成，您可以将云视为另一台WTP应用服务器。

  4. 填写主机名 （ 服务器名称） ，它是在步骤1中完成Micro Cloud Foundry实例的配置时应该获得的名称 ，而服务器名只是一个描述性的，有意义的字符串。

  5. Cloud Foundry将提示您在首次使用时进行身份验证。输入您注册时使用的cloudfoundry.com帐户凭证（电子邮件和密码）。当要求您选择URL的类型时，请选择“ 本地云” ，然后填写您在cloudfoundry.com中注册的子域的唯一部分。单击下一步完成安装


增加服务


您可以双击服务器实例以设置新服务（包括MongoDB，Redis，MySQL等的实例）并将它们与您的应用程序关联。在大多数情况下，这足以部署常规的Spring应用程序。Micro Cloud Foundry非常聪明。它将检查您的应用程序，并在您的Spring应用程序上下文中找到它已为您的应用程序预配置的任何对象。因此，例如，如果您已创建数据库javax.sql。在Spring配置中的DataSource中，CF将检测到该情况并尝试将其与绑定到应用程序即服务的正确配置的MySQL DataSource动态连接。例如，这也适用于Redis和MongoDB的相关Spring Data连接工厂。

如果您正在使用即将发布的Spring 3.1版本，则可以利用配置文件功能根据运行定义的环境有条件地启用某些bean定义。与Spring配合使用命名空间可让您精确控制在可能存在歧义的情况下使用哪种预配置服务（也许您有两个与同一应用程序关联的MySQL数据源）。请参阅Mark Fisher在Cloud Foundry上针对Spring Developers的早期文章。

大功告成！现在，像往常一样简单地开发应用程序，但具有在计算机上本地拥有云的所有功能。
