使用Grails的Micro Cloud Foundry

工程
彼得·莱德布鲁克
2011年8月24日

早在4月，VMware就向全球推出了Cloud Foundry，并为Grails开发人员提供了超简单的应用程序部署 。快进了几个月，现在又有了另一块拼图：Micro Cloud Foundry。现在，您可以拥有自己的Cloud Foundry实例以进行测试或任何其他用例。当然，在Grails中使用它非常容易。

那么什么是Micro Cloud Foundry？以下截屏视频为您提供了该产品的简要概述，然后带您完成了下载，安装和配置该产品的过程。最后，您将了解如何将Grails应用程序部署到Micro Cloud Foundry实例而不是cloudfoundry.com：

http://www.youtube.com/watch?v=qSRqIYFmrKg

您可以在Cloud Foundry博客文章中了解有关Micro Cloud Foundry及其设置方法的更多信息 ，如果您是STS用户，请查看Josh Long的博客文章和截屏视频 ，其中包括有关一起使用STS和Micro Cloud Foundry的信息。在这里，我将快速解释一下部署到cloudfoundry.com和Grails的Micro Cloud Foundry实例之间的区别。

在使用自定义cloudfoundry.me子域启动并运行Micro Cloud Foundry VM之后，仅需两个步骤即可将Grails应用程序部署到该子域：

  1. 将用于Grails的最新Cloud Foundry插件安装到您的应用程序中，例如使用grails install-plugin cloud-foundry ；和
  2. 配置Cloud Foundry凭证和目标URL。

就是这样！一旦设置了适当的配置选项，您就可以使用cf-push和cf-update命令直接部署到Micro Cloud Foundry实例。还有那些配置选项？它们构成了Micro Cloud Foundry实例（通常使用vmc register命令创建）上有效用户帐户的用户名和密码，以及该实例的URL。最后一个只是带有“ api”的自定义cloudfoundry.me子域。字首。

可以按优先顺序在BuildConfig.groovy ， $ HOME / .grails / settings.groovy或Config.groovy之一中声明这些选项。例如，您可能在个人settings.groovy文件中拥有cloudfoundry.com凭据，并在BuildConfig.groovy中使用值覆盖它们：

grails.plugin.cloudfoundry.username = "[email protected]"
grails.plugin.cloudfoundry.password = "password"
grails.plugin.cloudfoundry.target = "api.mycompany.cloudfoundry.me"

现在，您执行的每个Cloud Foundry命令将针对配置的Micro Cloud Foundry实例进行操作。有什么会更容易？
