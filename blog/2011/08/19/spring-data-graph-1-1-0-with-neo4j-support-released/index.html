发布了具有Neo4j支持的Spring Data Graph 1.1.0

发布
托马斯·里斯伯格
2011年8月19日

亲爱的 Spring 社区，

我们很高兴地宣布第二版（1.1.0。支持Neo4j的Spring Data Graph项目的RELEASE）现已上市！

在2011年4月首次公开发布Spring Data Graph之后，我们主要关注用户反馈。

通过围绕工具的改进文档和升级的AspectJ版本，我们解决了用户报告的许多AspectJ问题。使用最新的STS和Eclipse，并希望借助Idea11，可以开发出没有红色摆动的Spring Data Graph应用程序。为了进一步简化开发，我们还提供了用于ant / ivy的示例构建脚本和用于gradle的插件。

当然，我们与Neo4j的开发保持同步，目前正在使用Neo4j的最新稳定版本（1.4.1） 。

在Neo4j开发的最后几个月中，改进的查询（ Cypher和Gremlin ）支持是重要的方面之一。因此，我们努力在所有级别上提供支持。现在，可以从Spring数据图存储库，Neo4j模板执行Cypher查询，也可以作为动态字段注释的一部分并通过引入的实体方法执行Cypher查询。Gremlin脚本也是如此。这种新的表达能力有什么可能？让我们来看看。

例如，在存储库中：


	public interface PersonRepository extends GraphRepository, NamedIndexRepository {
	
	    @Query("start team=(%team) match (team)-[:persons]->(member) return member")
	    Iterable findAllTeamMembers(@Param("team") Group team);
	
	    @Query(value = "g.v(team).out('persons')", type = QueryType.Gremlin)
	    Iterable findAllTeamMembersGremlin(@Param("team") Group team);
	}

Neo4j模板API进行了全面的改进，从而产生了更少的，更具针对性的方法。现在，使用更流畅的API实现了高级查询结果处理功能（类型转换，映射，单个结果，处理程序等）。这个新的API可用于所有类型的查询，无论是索引查找，图形遍历，Cypher查询还是Gremlin脚本。 


	template.query("start n=(0) match n-->m return m", null).to(Node.class);

	template.execute("g.v(0).out", null).to(Node.class);

	template.lookup("relationship", "name", "rel1").to(String.class, new PropertyContainerNameConverter()).single();

	template.traverse(referenceNode, traversalDescription).handle(new Handler() {
	            public void handle(Path value) {
	                final String name = (String) value.endNode().getProperty("name", "");
	                resultSet.add(name);
	            }});

REST API包装器也进行了内部更新，并增加了对远程查询Cypher和Gremlin的支持。这使得这两种功能也可用于在远程Neo4j-REST-Server上运行对象图映射和Neo4j模板。

非常感谢社区提供宝贵的反馈，代码贡献和讨论。一如既往，SpringSource和Neo-Technology团队之间的合作非常愉快。

请从Maven Central或SpringSource.org查看当前版本。如果您想讨论Spring Data Graph项目，请确保访问Spring论坛 。我们在github上公开托管该项目，供您分叉，评论和贡献。

我们希望给您一些未来的路线图。

我们将在9月8日举办一个网络研讨会 ，以快速介绍Spring Data Graph。

Spring Data Graph将更名为“ Spring Data Neo4j”，因为它的含义是：“在Spring Framework环境中支持Neo4j Graph数据库”。这已经在改变的包装结构中树立了第一个标志。

我们将重点介绍在没有AspectJ的情况下也可以使用的其他基于映射的实现。另一个主要重点将是远程REST-API，随着PaaS提供者提供的托管Neo4j服务的可用性，远程REST-API变得越来越重要。

Spring Data Graph指南手册将以InfoQ Mini Book的形式发布，并在10月的Spring One会议上以印刷版提供。

Neo Technology将在Spring One上谈论NOSQL，Graph Databases和Spring Data Neo4j。我们也希望为会议贡献一些意想不到的事件和技术。因此，请继续关注。
