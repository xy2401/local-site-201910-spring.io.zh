Spring Cloud for Alibaba 0.2.1发布

发布
斯宾塞·吉布
十二月21，2018

注意：这是来自我们阿里巴巴朋友的来宾帖子。

大家好！今天，我们非常高兴地宣布0.2.1。Spring Cloud Alibaba的发布现已推出。

发布要点：

  • 在Spring Cloud Alibaba中添加了两个新模块：spring-cloud-alibaba-schedulerx和spring-cloud-stream-binder-rocketmq。
  • 在spring-cloud-alibaba-nacos和spring-cloud-alibaba-sentinel中添加了一些新功能。
  • 修复了先前版本中的一些错误。

笔记：

  • Spring Cloud Alibaba项目是由Alibaba维护的社区项目。

  • 0.2.1。RELEASE与Spring Cloud Finchley兼容。此版本还包括0.1.1。发行版，与Spring Cloud Edgware兼容。0.1.1。RELEASE还包含0.2.1中新增的组件和功能。

Spring Cloud Stream Binder RocketMQ

  • 实现Spring Cloud Stream API的消息抽象。
  • 包括对交易消息的支持。
  • 包括支持在使用者端过滤带有标签和SQL表达式的消息，以及有序，并发和广播消息的使用。

Spring Cloud阿里云SchedulerX

  • 提供准确，高度可靠和高度可用的预定作业调度服务，响应时间在几秒钟内。
  • 支持丰富的作业执行模型集，包括独立执行，广播执行和分布式子作业执行。

Spring Cloud阿里巴巴Nacos配置

  • 将Nacos Client版本升级到0.6.1。
  • 添加了对从多个dataid和groupid获取和监听配置以及基于dataid和groupid的组合设置优先级的支持。

Spring Cloud阿里巴巴Nacos发现

  • 将Nacos Client版本升级到0.6.1。
  • 添加了对在Nacos Console上将服务实例设置为不可用状态的支持，以便在发现服务期间自动将服务过滤掉。
  • 添加了对初始化服务发现时忽略本地缓存的支持。

Spring Cloud阿里巴巴前哨

  • 添加了对Feign的支持，使其现在与所有@FeignClient属性兼容，包括fallback和fallbackFactory 。
  • 增加了对参数流控制和集群流控制的支持。
  • 重构了设计ReadableDataSource提供一种更加用户友好的方式来配置持久性Sentinel规则。
  • 优化了Sentinel的后处理机制RestTemplate被降级。
  • 添加了一些与Sentinel配置信息相对应的属性，例如日志目录和日志文件名。
