Spring 的这个星期-2018年7月24日

工程
乔什·朗（Josh Long）
七月25，2018

嗨， Spring 粉丝！欢迎来到 Spring 本周的另一期！本周，我在旧金山的Spring Security负责人Rob Winch的工作室里拍摄了我们新的Spring Security Livelessons视频。有这么多的覆盖，你可以想像，它甚至对我们两个人的一项艰巨的任务！

我还忙于准备在本周的Google Cloud NEXT活动上与我的好友和Googler Tsang进行有关Pivotal和Google Cloud的所有话题。我们将研究可启动的 Spring Cloud GCP项目，更重要的是，今天早晨Google和Pivotal共同宣布的新项目KNative，它是我们在Pivotal上构建和部署无服务器Project Riff的平台功能即服务运行时。如果你周围不要错过哦！

好了，本周我们将涵盖与Pivotal，Google，KNative，Spring等相关的大量内容，所以让我们开始吧！

  • 今天的重大公告：Pivotal在KNative上与Google联手。第一步， KNative将支持我们与Project Riff合作的基础 。
  • Spring Cloud Data Flow负责人Mark Pollack博士刚刚宣布发布Spring Cloud Data Flow 1.6 。新版本包括PCF Scheduler，改进的仪表板，批处理数据库和架构优化等等。看看这个！
  • Spring REST Docs负责人Andy Wilkinson 刚刚宣布了Spring REST Docs 2.0.2
  • Spring REST Docs负责人Andy Wilkinson 刚刚宣布了Spring REST Docs 1.2.5
  • 查看这篇文章，了解在应用程序堆栈中使用Cloud Foundry的CredHub进行秘密管理的更多方法
  • 安迪·威尔金森（Andy Wilkinson）从来没有固步自封，他还 宣布了Gradle依赖性管理插件1.0.6。现在可以从Maven Central，Bintray和Gradle Plugin Portal获得RELEASE 。此维护版本修复了四个问题。
  • Spring 的这个星期-2018年7月17日
  • 该视频演示了BOSH如何轻松部署Kubernetes
  • 我喜欢这个-Spring社区传奇人物Michael Simons研究如何从Kotlin应用程序中支持Spring Boot的配置属性元数据。好东西！
  • 观看此视频以了解Cloud Foundry如何简化Kubernetes
  • Joris Kuipers引入了Spring Cloud AWS中的新支持， 用于将配置存储在AWS Parameter Store中 。
  • 在这篇文章中查看Dan Baskette 在KNative和Pivotal上的文章
  • 我很荣幸再次入选 2018年JAX的20位有影响力者名单 。非常感谢！
  • 真是令人兴奋的一天！在Ryan Morgan的博客文章中了解新宣布的Knative项目对Riff和Spring生态系统的意义
  • https://twitter.com/cloudfoundry/status/1021805217229229230082
  • 终于，猫从书包里走了！KNative在这里，我们一直在与Google合作，使其成为Riff项目的最佳基础。阅读这篇文章以了解更多信息。
  • Reactor Kafka集成有一个新的1.0.1版本。祝贺Oleh Dokuka和团队的其他成员！
  • https://www.linkedin.com/pulse/marathon-digitization-how-flexibility-hiring-fuel-abby-kearns/?published=t
  • 祝贺UAA团队获得OpenID认证 ！当然，UAA是为Cloud Foundry提供支持的身份验证和授权服务。它是基于Spring和Spring Security OAuth的Apache 2许可的开源身份提供者。
  • 进入SpringOne Platform 2018的原因很多，万事达卡企业安全首席顾问张妮（Jenny Zhang）就是其中之一。聆听大规模安全大师的经验将非常有趣。

  • 这是Thomas Darimont的精彩演讲，内容涉及如何使用Redhat的OAuth授权服务Keycloak保护Spring应用程序 。
  • twitter.com
  • 查看Josh Bloch的QCon演讲！Josh是我们在Java平台（如Java Collections API）中理所当然的大量API的创建者，而这次演讲的内容全部是关于他的最新著作《 Effective Java，第三版》的内容。
  • 祝贺Trampoline 团队发布了新版本 ！此版本引入了一个选项，用于定义在组内启动实例之前的延迟，以便能够准备好配置服务器或等待直到缓存可用。
