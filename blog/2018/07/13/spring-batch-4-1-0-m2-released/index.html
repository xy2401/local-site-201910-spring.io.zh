Spring Batch 4.1.0。M2发布

发布
Mahmoud Ben Hassine
七月13，2018

我们很高兴地宣布Spring Batch 4.1.0。现在可以在Github和Pivotal下载存储库上使用M2。非常感谢所有对此版本做出贡献的人！

什么是新的？

以下是此版本的重点：

  • 简化远程分区
  • 添加一个新的JSON项目编写器
  • 添加对使用Bean验证API验证项目的支持

简化远程分区

在4.1.0中。在M1版本中，我们创建了新的API以简化远程组块步骤的配置。在这个里程碑中，我们继续努力通过两个新的构建器来简化远程分区： RemotePartitioningMasterStepBuilder和RemotePartitioningWorkerStepBuilder 。

这些构建器可以自动连接到您的配置类中，如果
@EnableBatchIntegration如以下示例所示：

@Configuration
@EnableBatchProcessing
@EnableBatchIntegration
public class RemotePartitioningAppConfig {

   @Autowired
   private RemotePartitioningMasterStepBuilderFactory masterStepBuilderFactory;

   @Autowired
   private RemotePartitioningWorkerStepBuilderFactory workerStepBuilderFactory;

   @Bean
   public Step masterStep() {
            return this.masterStepBuilderFactory
               .get("masterStep")
               .partitioner("workerStep", partitioner())
               .gridSize(10)
               .outputChannel(outgoingRequestsToWorkers())
               .inputChannel(incomingRepliesFromWorkers())
               .build();
   }

   @Bean
   public Step workerStep() {
            return this.workerStepBuilderFactory
               .get("workerStep")
               .inputChannel(incomingRequestsFromMaster())
               .outputChannel(outgoingRepliesToMaster())
               .chunk(100)
               .reader(itemReader())
               .writer(itemWriter())
               .build();
   }

   // Middleware beans setup omitted
}

这些新构建者负责配置基础结构Bean的繁重工作。
现在，您可以轻松地配置远程分区作业的主步骤和工作步骤。

添加一个新的JSON项目编写器

在上一个里程碑中，我们创建了一个新的项目读取器，以支持读取JSON数据。在这个里程碑中，我们添加了JsonFileItemWriter以及支持构建器以支持写入JSON数据。

Bean验证API支持

此版本带有一个新的ValidatingItemProcessor叫BeanValidatingItemProcessor能够验证使用Bean Validation API注释注释的项目。这个新组件将使Spring Framework或Spring Boot提供的Bean Validation API支持基础结构适应于ItemProcessor在Spring Batch作业的步骤中很有用。
