Spring Cloud Data Flow 1.6 M1和1.5.2发布

发布
马克·波拉克
七月05，2018

Spring Cloud Data Flow团队很高兴宣布1.6 M1版本和1.5.2发布。

对于1.6 M1，请遵循Local Server ， Cloud Foundry和Kubernetes的入门指南 。

对于1.5.2，请遵循Local Server ， Cloud Foundry和Kubernetes的入门指南 。

1.6 M1的改进领域：

  • DSL和部署属性解析

  • 任务执行状态

  • 组成的任务运行器安全

  • 仪表板

  • Kubernetes部署

DSL和部署属性解析

使用自定义参数启动任务是一种在运行时以不同行为影响Task应用程序的好方法。想象一下，影响接受时区作为执行时区特定数据处理参数的批处理作业（作为任务运行）。在此版本中，我们调整了解析逻辑，以将键值对作为值包括在内。感谢社区提供报告，向我们提供反馈以及共享他们的用例。

在查看基于内联与基于属性文件的流和任务定义的属性的解析规则时，社区发现了行为上的差异，我们已记录了该差异以作为一般指导 。

组成的任务运行器安全

随着社区的持续关注，我们增加了支持，以使Composed Task Runner和Data Flow Server之间能够安全访问。我们添加了基本身份验证支持，并将在即将发布的版本中添加Data Flow支持的其他安全选项。

仪表板改进

重新设计了“任务/任务”和“关于”选项卡，以与其余UI部分保持一致。批量操作，分页，布局以及视图的总体外观已实现现代化。

任务选项卡，子选项卡和页面视图之间的路由和导航已更新。您会发现状态管理的改进在列表到详细信息页面之间导航，反之亦然。

Kubernetes部署

进行了一些改进，包括支持使用安全的执行器来部署Boot Apps，因此可以在运行时解决活动性和就绪性探针。

通过自定义的能力Service Account Name现在可以用于每个流/任务部署。对于不同的流/任务部署需要不同的安全权限的情况，这尤其有用。
