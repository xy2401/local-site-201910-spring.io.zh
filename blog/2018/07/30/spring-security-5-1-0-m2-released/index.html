Spring Security 5.1.0。M2发布

发布
罗伯·温奇
七月30，2018

我很高兴代表社区宣布Spring Security 5.1.0的发布。M2。此版本附带100张以上的门票 。

一如既往，我们期待收到您的反馈 ！您可以在下面找到要点：

OAuth2

OAuth2资源服务器

添加了对OAuth2资源服务器的基本支持。请参阅oauth2resourceserver

授权码流程

用户现在可以使用OAuth 2.0授权码授予来获取访问令牌。请参阅authcodegrant示例。

WebClient和OAuth2支持

现在内置了对OAuth2和WebClient支持的支持。支持允许：

  • 将访问令牌添加到请求中

  • 访问令牌过期时自动刷新

  • 解析要使用的访问令牌

例如，在Servlet环境中，您可以这样配置Bean：

现在，您可以通过多种不同方式添加OAuth令牌。如果需要，可以使用Spring MVC支持来解析OAuth2AuthorizedClient 。如果授权服务器返回了刷新令牌，并且访问令牌即将过期，Spring Security将透明地更新访问令牌并提交更新的访问令牌。

WebFlux OAuth2登录支持OIDC

WebFlux应用程序现在可以同时使用OAuth2和OIDC进行身份验证。有关示例，请参见oauth2login-webflux 。

通过OIDC发现创建ClientRegistration的能力

以前，想要与OIDC提供程序集成的用户需要配置所有端点。例如，在Spring Boot中，它看起来像：

春季：安全性：oauth2：客户端：注册：okta：客户端ID：okta-client-id客户端密钥：okta-client-secret提供程序：okta：authorization-uri：https://foo.oktapreview.com/oauth2/ v1 /授权令牌-uri：https://foo.oktapreview.com/oauth2/v1/token用户信息-uri：https://foo.oktapreview.com/oauth2/v1/userinfo用户名属性：sub jwk-set-uri：https：//foo.oktapreview.com/oauth2/v1/keys

现在，用户只需配置发行者uri。例如：

春季：安全性：oauth2：客户端：注册：okta：客户端ID：okta-client-id客户端密钥：okta-client-secret提供程序：okta：issuer-uri：https://foo.oktapreview.com/oauth2/默认/

支持自定义授权请求

通过添加OAuth2AuthorizationRequestResolver添加了对自定义授权请求的支持。例如，如果用户想从请求中添加/删除范围，则他们现在可以轻松地这样做。

现在，单一ClientRegistration在登录时重定向

当使用单个提供程序配置OAuth2 / OIDC登录时，而不是仅显示一个提供程序，默认值已更改为立即重定向到提供程序。

配置改进

现代化默认登录页面

默认的登录页面已更新为html5，看起来更具视觉吸引力。

默认注销页面

由于添加了CSRF注销保护，因此默认应用程序无法注销。现在，如果正在使用默认的登录页面（即未配置任何登录页面），那么还有一个默认的退出页面将显示退出表单。

简化RequestCache配置

用户现在可以通过将默认的@Bean公开为@Bean来配置它。HttpSessionRequestCache已更新为不再要求该值为DefaultRequestCache 。

强化您的应用程序

跨站点跟踪和HTTP动词篡改保护

Spring Security 5.1在Spring Security中添加了跨站点跟踪和HTTP动词篡改保护。

密码编码升级

用户可以实现UserDetailsPasswordService并将其公开为@Bean并且在身份验证成功后，Spring Security的DaoAuthenticationProvider将：

  • 使用新的PasswordEncoder.upgradeEncoding方法检查是否需要更新密码存储机制。例如，如果它是用sha256编码的，那么默认情况下，Spring Security会建议将其升级到BCrypt。

  • 如果密码编码需要升级，它将使用当前的PasswordEncoder对密码进行编码

  • UserDetails和新密码将被传递到UserDetailsPasswordService以便可以使用升级的密码编码进行保存。

依赖关系更新

我们将依赖关系更新为最新的和最大的依赖关系，以确保我们的可传递依赖关系是最新的。
