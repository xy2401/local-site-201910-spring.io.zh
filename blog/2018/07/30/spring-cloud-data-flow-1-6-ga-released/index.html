  • Batch Database Schema and Optimization

Task Scheduling on PCF

We are happy to introduce the native integration of PCF Scheduler in the SCDF for Cloud Foundry implementation!

A typical workflow for batch data processing involves scheduling batch applications. For example, the scheduler system accepts a cron expression and launches the application whenever the expression matches the current time.

Data Flow provides the ability to schedule and unschedule a task definition. The schedule is based on a cron expression. Building upon the PCF Java Client the team has created a portable scheduler interface in the Spring Cloud Scheduler SPI project (Service Provider Interface) and an implementation for PCF, Spring Cloud Scheduler for Cloud Foundry. Spring Cloud Data Flow 1.6 GA发布

发布
马克·波拉克
七月30，2018

Spring Cloud Data Flow团队很高兴宣布1.6.0的发布。遵循本地服务器 ， Cloud Foundry和Kubernetes的入门指南 。

1.6 GA的功能亮点

  • PCF上的任务计划

  • 仪表板即兴

  • Kubernetes支持增强

  • 应用托管工具

  • 组成的任务运行器安全

  • DSL和部署属性解析的改进

  • 批处理数据库架构和优化

PCF上的任务计划

我们很高兴在SCDF中为Cloud Foundry实施引入PCF Scheduler的本机集成！

批处理数据的典型工作流程涉及调度批处理应用程序。例如，调度程序系统接受cron表达式，并在表达式与当前时间匹配时启动应用程序。

数据流提供了计划和取消计划任务定义的功能。该计划基于cron表达式。在PCF Java Client的基础上，团队在Spring Cloud Scheduler SPI项目 （服务提供商接口）中创建了一个可移植的调度程序接口，并为PCF实施了Cloud Foundry的Spring Cloud Scheduler 。仪表板提供了计划和取消计划任务的访问权限，如下面的屏幕快照所示。

仪表板改进

流部署历史记录可从仪表板进行查看。从中心位置查看流的特定于上下文的历史记录很方便；特别是当CI / CD系统连续部署属于该流的新版本的应用程序工件时。

重新设计了“任务/任务”和“关于”选项卡，以与其余UI部分保持一致。批量操作，分页，布局以及视图的总体外观已实现现代化。以前，任务执行状态已存储，但未显示在外壳程序或UI中。现在显示：)

任务选项卡，子选项卡和页面视图之间的路由和导航已更新。您会发现状态管理的改进在列表到详细信息页面之间导航，反之亦然。

SCDF仪表板和Spring Flo堆栈已升级到Angular 6。包括JointJS在内的几个下游依赖项也已更新。尽管测试工具通过各种浏览器运行以进行增量验证，但是如果您在不同的浏览器中发现任何异常，请随时提出问题或在Gitter或StackOverflow中提出。我们感谢您的任何反馈。

Kubernetes部署

进行了一些改进，包括支持使用安全的执行器来部署Boot Apps，以便可以在运行时解决活动性和就绪性探针。

现在，每个流/任务部署都可以传递自定义Service Account Name 。对于不同的流/任务部署需要不同的安全权限的情况，这尤其有用。

应用托管工具

虽然Maven是用于Stream / Task App工件解析的推荐方法，但出于多种原因，某些用户无法使用Maven。我们还听说过客户在无互联网区域中安装SCDF，并且无法通过Maven，HTTP或Docker注册表来解决Stream / Task工件。

为了解决这些问题，我们开发了一个App Hosting Tool，它模仿了一个独立的App存储库，但实际上是一个Spring Boot App，它通过HTTP为App工件提供服务。您可以从此处阅读有关App工具和入门说明的更多信息 。

组成的任务运行器安全

随着社区的不断关注，我们增加了支持，以使Composed Task Runner和Data Flow Server之间能够安全访问。我们添加了基本身份验证支持，并将在即将发布的版本中添加Data Flow支持的其他安全选项。

DSL和部署属性解析

使用自定义参数启动任务是一种在运行时以不同行为影响Task应用程序的好方法。想象一下，影响接受时区作为执行时区特定数据处理参数的批处理作业（作为任务运行）。在此版本中，我们调整了解析逻辑，以将键值对作为值包括在内。感谢社区提供报告，向我们提供反馈以及共享他们的用例。

在查看基于流和任务定义的基于内联属性文件的解析规则和基于属性文件的解析规则时，社区发现了行为上的差异，我们已记录了该差异以作为一般指导 。

批处理数据库架构和优化

感谢社区提供了关于数据库模式的全面验证和反馈。对于MySQL和PostgreSQL有大量任务执行的情况，批处理和任务模式已得到增强。其他数据库的优化正在进行中。

接下来会发生什么...

我们正在制定1.7计划，请查看GitHub了解更多详细信息。但是，重点领域之一是支持Spring Cloud Function编程模型。这也将有助于简化应用程序启动器和自定义业务逻辑的组成。
