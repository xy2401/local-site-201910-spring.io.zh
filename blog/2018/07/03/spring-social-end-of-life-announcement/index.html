Today it is with an optimistic look to the future that I am announcing the end of life for Spring Social, a project that I have had the pleasure to work with over the past eight years.

With the release of Spring Security 5, much of the functionality offered by Spring Social’s connection framework—which I consider the most valuable piece of Spring Social—is now part of Spring Security. In fact, I shared how to use Spring Security’s new support for client-side OAuth2 in a blog article from back in March. And as Spring Security continues to evolve its client-side OAuth support, Spring Social’s connection framework will continue to become more unnecessary. It simply no longer makes sense to have Spring’s OAuth story spread across so many projects—it’s time for a single cohesive OAuth story for Spring developers.

春季社交报废公告

工程
克雷格·沃尔斯
七月03，2018

今天，我以乐观的眼光宣布了Spring Social的生命尽头，在过去的八年中，我很高兴能与这个项目一起工作。

随着Spring Security 5的发布，Spring Social的连接框架（我认为是Spring Social最有价值的部分）所提供的许多功能现在已成为Spring Security的一部分。实际上，我在三月份的博客文章中分享了如何使用Spring Security对客户端OAuth2的新支持。而且，随着Spring Security继续发展其客户端OAuth支持，Spring Social的连接框架将变得越来越不必要。将Spring的OAuth故事分布在如此众多的项目中不再有意义，现在是时候为Spring开发人员提供一个单一的OAuth故事了。

但是，Spring Social的API绑定又如何呢？

笨拙的API绑定

寿命终止公告还涵盖了Spring Social的API绑定，包括Facebook，Twitter和LinkedIn的API绑定。

Spring Social的API绑定最初只是为了展示如何使用连接框架生成的连接。但是后来他们成长了，过着自己的生活。

尽管API绑定通常对于需要与其所服务的API进行交互的应用程序很有用，但它们的体积庞大，复杂且难以实际使用。到目前为止，Facebook API绑定提供了大约180种不同的操作，尽管很少会找到需要这些操作中的一半以上的应用程序。

还已证明很难使API绑定与其所服务的API保持同步。随着API（例如Facebook的Graph API）的不断发展，需要重新审视Spring Social的API绑定（在某些情况下完全用重大更改重写），以保持与各自API的兼容性。考虑到上述API绑定的大小，这需要大量的开发和测试工作-最初只是作为如何编写API绑定的示例进行的。

最后，既然Spring Framework 5向Spring开发人员发布了一个反应式编程模型，那么就需要Spring Social的API绑定的反应式对应物。尽管这肯定是可能的并且可以理解，但是这将需要与现有API绑定的大小成比例的工作，并且基本上会重复非反应式API绑定中提供的大多数操作。这将有效地使API绑定的大小增加一倍，从而使此类绑定的维护更具挑战性。

相反，我是否建议应用程序开发人员编写自己的API绑定实现？这样，您将能够使用与Spring Social提供的庞然大物API绑定相比更加专注于应用程序需求并且更易于维护的API绑定。此外，如果您希望在API绑定中应用反应式类型，欢迎这样做。我在三月份发表的文章中提出了解决此问题的方法。

Sunsetting Spring Social

Spring Social不会立即消失。我打算继续进行一些小错误修复，并根据需要发布维护版本。但是，除非有重大而迫切的要求，否则没有计划进一步发行Spring Social的GA。

从今天开始，Spring Social（以及前面提到的Spring Social API绑定模块）将继续获得12个月的支持，支持期于2019年7月3日结束。

我衷心感谢在过去8年中与Spring Social合作并为之贡献的每一个人。这是一个很有趣的项目，但我感到悲伤的是我宣布它最终消亡。请放心，这不是一个轻易做出的决定。但是，我鼓励您查看Spring Security 5的客户端OAuth支持，并继续与我们合作，提出建议并提出要求，以帮助撰写Spring OAuth故事中的这一新章节。
