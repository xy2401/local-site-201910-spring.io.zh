提供Spring Data Lovelace RC1

发布
马克·帕鲁奇
七月26，2018

我很荣幸代表Spring Data团队宣布Lovelace RC1版本的可用性。第一个候选发行版将在刚刚发布的Spring Framework 5.1 RC1之上，并为即将推出的Spring Boot 2.1 M1发行做准备。

此版本的候选产品已修复194张门票，并且包含以下显着更改：

  • 支持不可变的对象。
  • 升级Querydsl for MongoDB以使用Document API，使您可以发布生命周期事件和参与托管事务。
  • 反应性Slice查询对Apache Cassandra的支持。
  • Apache Cassandra的Kotlin扩展。
  • 反应性SCAN支持Redis。

不可变对象支持允许使用Spring Data真正不可变的对象，而不会由于反射访问而造成干扰。以前，如果不可变对象经过ID生成，审核或乐观锁定，则可能会发生变化。这些类型的操作会改变实体的位置，从而导致实际不变的对象发生变化。

从此发行版开始，不可变对象支持可确保不更改不可变实体。 insert和save方法返回与突变相关联的对象的新实例，例如生成的ID。此更改要求您的代码随着方法返回类型的更改而重新编译。不可变对象的支持需要在应用程序代码中使用相应的数据模型：

  • 使用Kotlin数据类来利用.copy(…)方法
  • 价值对象暴露with...方法

with...方法会创建一个对象的新实例，该对象带有所有先前的属性值，并且具有更改后的with…属性，如以下示例所示：

class Person
{
  private final @Id String id;
  private final String name;
  
  private Person(String id, String name) {
    // …
  }
  
  // …
  
  Person withId(String id) {
   return new Person(id, this.name);
  }
}

龙目岛用户可以使用@Value和@Wither遵循的注释with...图案。

不可变对象支持可用于Spring Data MongoDB，Apache Cassandra，Couchbase，ElasticSearch，Redis，KeyValue，Neo4j（审核）和Spring Data REST模块。由于映射层位于JPA实现内部，因此无法对JPA提供不可变的对象支持。Neo4j不可能完全具有双向关系。Lovelace RC2附带了对JDBC的不可变对象支持。

我们的目标是在8月底发布Lovelace RC2，在9月中旬发布GA，这恰好是今年华盛顿特区的SpringOne平台的发布时间。
