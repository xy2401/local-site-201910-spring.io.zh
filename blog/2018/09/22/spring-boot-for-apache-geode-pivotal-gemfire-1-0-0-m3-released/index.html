适用于Apache Geode和Pivotal GemFire 1.0.0的Spring Boot。M3发布！

工程
约翰·布鲁姆
九月22，2018

我很高兴代表Spring和Apache Geode社区宣布适用于Apache Geode和Pivotal GemFire （SBDG）版本的Spring Boot的第三个里程碑版本1.0.0.M3 。

1.0最终GA的位几乎完成。在最终GA之前，还有1个开发里程碑（RC1）。新的1.0.0.M3可以从Spring libs-milestone存储库中获取位。

什么是新的

SBDG 1.0.0.M3增加了对将Spring Boot的Actuator功能与Apache Geode和Pivotal GemFire（以及扩展为Pivotal Cloud Cache，由Pivotal GemFire支持）一起使用的支持。

具体来说，此支持包括Spring Boot HealthIndicators评估以下GemFire / Geode组件的状态和运行状况：

  • 快取

  • 分布式系统/分布式成员

  • 资源管理器

  • 地区

  • 指标

  • 磁盘存储

  • 连续查询

  • 缓存服务器

  • 泳池

  • AsyncEventQueues

  • 网关发送者

  • 网关接收器

除了基本状态信息外，GemFire / Geode HealthIndicators提供必要的配置元数据详细信息以及GemFire / Geode组件本身的运行时特性/行为。这包括简单的指标，例如缓存区域的命中/未命中计数或客户端和服务器之间的连接负载，等等。

在1个方便的位置提供了“基本”配置元数据详细信息，因此您不必为了查找每个GemFire / Geode的配置而在所有命令行参数，日志文件，属性文件和其他配置文件中进行筛选应用程序体系结构中使用的对象。

有关提供的更多详细信息HealthIndicators 在这里可用。
