OAuth2的简化DSL

最初，Spring Security DSL具有两个oauth：

http
    .oauth2Login()...

和

http
    .oauth2()
        .client()...

之所以有意义，是因为一种身份验证机制，例如formLogin和openidLogin ，而openidLogin一种身份验证机制（ client ， resourceServer和authorizationServer则更像OAuth 2.0个性。

不过，最后，这种分离感觉像是不必要的额外键入，因此我们决定将层次结构弄平，这意味着，从此版本开始，我们现在拥有：

http
    .oauth2Login()...
    .oauth2Client()...
    .oauth2ResourceServer()

与此DSL重构无关的功能或特性更改，只为您减少键入次数。

WebClient扩展

团队一直在努力与WebClient进行合作，我们很高兴宣布适用于Servlet和WebFlux应用程序的一些新的OAuth 2.0 WebClient扩展。通过这些扩展，可以轻松地在计算机之间无缝传输OAuth 2.0授权。

阅读OAuth 2.0 Web客户端主票证中的所有详细信息。

Servlet增强

OAuth2

令牌请求配置

OAuth 2.0旅程的第一部分是规范合规性。该下一发行版为配置Spring Security与扩展或偏离规范的提供程序一起提供了更多支持。

例如，现在可以自定义从客户端到授权服务器的令牌请求 。

资源服务器声明映射

同样，Resource Server附带支持自定义从传入JWT解析的声明集的支持。当应用程序需要添加或删除声明或需要以自定义方式解析声明时，这非常方便：

NimbusJwtDecoderJwkSupport decoder = // ...
decoder.setClaimSetConverter(
    MappedJwtClaimSetConverter
        .withDefaults("custom-date", this::convertToInstant));

更多提供者配置元数据可用

遵循规范的过程也在继续。在此版本中，增加了对收集和提供从OIDC提供程序配置端点返回的任何元数据的支持。

现在， ClientRegistration附带了getConfigurationMetadata ，它与针对特定属性的方法并排，并返回提供程序属性的整个映射。

RestOperations支持

此版本还附带了对各种 端点 的HTTP请求的完全自定义的支持。

这对于配置超时，发现，缓存以及与授权服务器进行通信时充分利用RestTemplate的复杂性非常方便。

其他改进

X.509主体提取器

还添加了通过策略派生X.509主体的支持 。

LDAP自定义环境变量

并且增加了对配置自定义环境变量的支持，这些变量通知了LdapContext的创建 。

WebFlux增强功能

OAuth2资源服务器

在WebFlux端的RC2中添加了最初为基于Servlet的资源服务器发布的一些功能。反应 能力 类似 ，但有一个小但重要的例外。

在Spring Security WebFlux中，更典型的情况是每个请求类型都有一个身份验证管理器。然后，在此版本中，WebFlux附带了Resource Server的身份验证管理器配置 ：

http
    .oauth2ResourceServer()
        .authenticationManager(customAuthenticationManager())

这在应用程序需要将Jwt进行一些自定义转换到一组授权机构的情况下Jwt用。

OAuth2客户端

@RegisteredOAuth2AuthorizedClient注释现在支持 WebFlux中的client_credentials授予 。

重定向到Https

最后但并非最不重要的一点是， https重定向支持已添加到WebFlux，可通过http.redirectToHttps()或直接通过HttpsRedirectWebFilter 。

依赖关系更新

并且我们已经将依赖项更新为最新的和最大的，以确保我们的传递性依赖项是最新的。
