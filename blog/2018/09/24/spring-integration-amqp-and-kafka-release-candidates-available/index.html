Spring Integration，AMQP和Kafka发行候选版本可用

发布
阿尔特姆·比兰（Artem Bilan）
九月24，2018

我很高兴代表Spring Integration团队宣布Spring Messaging项目的候选发布。每个版本都基于最近发布的Spring Framework 5.1 GA ，已经升级了依赖性，并且将成为即将推出的Spring Boot 2.1 M4的一部分 -及时发布Spring One Platform ！

这些项目的工件在Spring Milestone存储库中可用。

请参考What’s New有关每个下文中提到的每个项目的新功能和显着更改的详细信息，请参见《参考手册》中的“第一章”。

春季整合

5.1.0。RC1发行说明。

我想在这里提及一些功能，因为它们尚未记录（我们将在GA发布期间进行此操作）：

  • Java DSL fluxTransform()算子

为了将消息处理传递给Reactive Streams，用于Spring Integration的Java DSL引入了一个中间层fluxTransform() ，例如

IntegrationFlow integrationFlow = f -> f
    .split()
    .<String, String>fluxTransform(flux -> flux
                                .map(Message::getPayload
                                .map(String::toUpperCase))
    .aggregate();

该运算符接受以下形式的参数Function>, ? extends Publisher> 。框架注入FluxMessageChannel在调用此函数之前和之后。它建立一个Flux将请求消息存储到订户上下文，调用Flux.transform()使用提供的功能，如果功能的结果不是Message已经。

  • 反应式轮询

另一项新功能是对一个反应轮询 SourcePollingChannelAdapter 。如果输出通道是的实例FluxMessageChannel ，该框架会将所有轮询选项转换为Flux.generate()根据的结果trigger.nextExecutionTime(triggerContext)并与Mono.delay(duration)实现轮询功能的反应方式。

  • 其他Java DSL更改

我们还向Java DSL添加了其他一些方便的运算符，例如nullChannel()和logAndReply() 。

最后，对Java DSL进行了改进，以改善Kotlin的使用体验，同时又不影响常规Java配置的更改。

适用于Apache Kafka的Spring的Spring集成扩展

3.1.0。RC1发行说明。

基于上述针对Apache Kafka的Spring 2.2.0.RC1和Spring集成5.1.0.RC1 ，提供了一些兼容性修补程序（尤其是Kotlin）和一些次要功能，例如onPartitionsAssignedSeekCallback为了KafkaInboundGateway和KafkaMessageDrivenChannelAdapter 。

有关此扩展的更多信息，请参见《 Spring for Apache Kafka 参考手册 》。
