Spring Integration，AMQP和Kafka发行候选版本可用

发布
阿尔特姆·比兰（Artem Bilan）
九月24，2018

我很高兴代表Spring Integration团队宣布Spring Messaging项目的候选发布。每个版本都基于最近发布的Spring Framework 5.1 GA ，已经升级了依赖性，并且将成为即将推出的Spring Boot 2.1 M4的一部分 -及时发布Spring One Platform ！

这些项目的工件在Spring Milestone存储库中可用。

请参考每本参考手册中的“新增功能What’s New一章，以获取有关下面提到的每个项目的新功能和重要更改器的更多信息。

春季整合

5.1.0。RC1发行说明。

我想在这里提及一些功能，因为它们尚未记录（我们将在GA发布期间进行此操作）：

  • Java DSL fluxTransform()运算符

为了将消息处理传递给Reactive Streams，用于Spring Integration的Java DSL引入了中间的fluxTransform() ，例如

IntegrationFlow integrationFlow = f -> f
    .split()
    .fluxTransform(flux -> flux
                                .map(Message::getPayload
                                .map(String::toUpperCase))
    .aggregate();

该运算符已经接受Function>, ? extends Publisher>. The Framework injects FluxMessageChannel s before and after calling this function. The it builds a Flux to store a request message to the subscriber context, calls Flux.transform() with the provided function and copies headers from the request message if the result from the function is not a Message形式的参数。

  • 反应式轮询

另一个新功能是对SourcePollingChannelAdapter的反应式轮询 。如果输出信道是一个实例FluxMessageChannel ，框架变换所有的轮询选项到Flux.generate()基于所述结果trigger.nextExecutionTime(triggerContext)和与所述的组合Mono.delay(duration) ，以实现轮询功能的反应方式。

  • 其他Java DSL更改

另外，我们还向Java DSL添加了其他一些便捷的运算符，例如nullChannel()和logAndReply() 。

最后，对Java DSL进行了改进，以改善Kotlin的使用体验，同时又不影响常规Java配置的更改。
