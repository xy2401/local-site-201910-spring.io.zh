After a decade of rolling out updates and improvements to the famous Spring Tool Suite and the Spring IDE components for Eclipse, a new era of Spring tooling starts today and we welcome you to the all-new Spring Tools 4. After an intensive period of public betas, we are happy to announce the first official release today.

Spring Tools 4 is a completely new set of tools for working with Spring projects in your favorite IDEs and editors. They are re-built from scratch with the goal in mind to provide the best support for implementing enterprise-grade applications based on Spring and Spring Boot across development environments and lightweight editors. They provide a unique editor-centric experience for working with Spring projects, ranging from smart and Spring-aware content-assist to a unique new way of connecting your source code with and enriching it with detailed information and insights from your running Spring Boot applications. This puts your source code editor side-by-side with your running boot applications.

Spring Tools 4 GA发布

发布
马丁·利珀特
九月25，2018

在对著名的Spring Tool Suite和用于Eclipse的Spring IDE组件进行了十多年的更新和改进之后，Spring工具的新时代从今天开始，我们欢迎您使用全新的Spring Tools 4 。经过密集的公开测试，我们很高兴今天宣布首个正式版本。

Spring Tools 4是一套全新的工具，可以在您喜欢的IDE和编辑器中使用Spring项目。它们是从头开始重建的，其目标是为跨开发环境和轻量级编辑器实现基于Spring和Spring Boot的企业级应用程序提供最佳支持。它们为处理Spring项目提供了以编辑者为中心的独特体验，从智能的和支持Spring的内容助手到独特的新方法，可以将源代码与之连接，并通过运行中的Spring Boot应用程序提供详细的信息和见解，从而丰富其源代码。这使源代码编辑器与正在运行的启动应用程序并排放置。

春季意识

全新的Spring Tools 4可以动态分析您的项目，了解其中的Spring构造，并提供对所有Spring元素的便捷快速浏览。找到特定的请求映射，确切的bean定义或新创建的函数从未像使用新的Spring Tools 4那样容易。

来自实时运行的应用程序的信息

全新的Spring Tools 4引入了一个新概念，即通过实时运行的Spring Boot应用程序的详细信息覆盖您的源代码。在计算机上运行启动应用程序后，该应用程序中的其他信息将直接显示在源代码编辑器中。这里最强大的部分是有关您的bean的详细接线报告。您可以确切地看到（从哪个源）将哪个bean注入了@Autowired注释的依赖项（例如），以及在应用程序中注入了哪个bean。这可以直接在您的源代码中提供对正在运行的应用程序的独特见解。您不必自己收集该数据并手动查找正确的代码行。该工具会自动为您完成操作。再往前走，您也可以从远程Spring Boot应用程序中获取该信息。这使您可以直接在源代码编辑器中查看已部署云的Spring应用程序的bean接线。

与IDE无关

我们知道Spring开发人员喜欢他们的IDE和代码编辑器。我们希望他们继续使用他们喜欢的工具。因此，我们以与IDE无关的方式实现了Spring Tools 4的大部分内容。您可以在以下位置使用它们：

  • Eclipse（ https://www.eclipse.org/downloads或来自https://spring.io/tools的STS4发行版）
  • Visual Studio代码（ https://code.visualstudio.com ）
  • Atom IDE（ https://ide.atom.io ）

这仅仅是开始。我们希望将来会有更多环境加入该聚会。

从头开始重建

为了实现这些全新的Spring Tools 4，我们决定从头开始重写大多数Spring工具-从一开始就牢记这些目标：

  • 让我们成为IDE不可知论者-让每个开发人员继续使用他们喜欢的环境。
  • 从一开始就构建性能-我们讨厌等待IDE或编辑器完成之后才能继续编码。因此，Spring Tools 4的设计绝不会阻止用户，而且不会很快交付结果。
  • 为现代Spring技术而构建-这些工具专注于Spring Boot和注释驱动的Spring应用程序开发。

持续更新

展望未来，Spring Tools 4将不断获得有关错误修复以及新功能和改进功能的更新。这些更新将在所有环境中的安装中自动显示为更新。即使在Eclipse平台版本之间，基于Eclipse的发行版也将不断更新，因此您将始终处于最新状态。

常问问题

旧的STS 3.9.x工具会发生什么？

我们将继续以完整发行版的形式发布STS 3.9.x的更新，直到2019年中，并将更新发行版更新为即将到来的Eclipse版本（2018-09、2018-12及更高版本）。如果您仍然需要Eclipse中旧工具的重要部分，而Spring Tools 4 for Eclipse不支持这些部分，则可以将这些部分作为附加功能安装到Spring Tools 4 Eclipse发行版中，而这些部分将继续接收维护更新，直到2019年中。在2019年中之后，旧的STS 3.9.x工具将不再收到任何维护更新。

Spring Tools 4是否已准备好用于Boot 2.1和Spring Framework 5.1？

是的，Spring Tools 4可以与各种Spring Boot版本（包括1.5.x，2.0.x和2.1.x）以及最新的Spring Framework版本（包括4.x和5.x）一起使用。

Spring Tools 4是否包含Java语言支持？

Eclipse之上的现成的Spring Tools 4发行版包括现成的Eclipse的标准Java语言工具。对于Visual Studio Code，您应该安装Java Extension Pack，它是常规Java语言工具（由RedHat和Eclipse提供）与针对Java for Visual Studio Code的启动，测试和调试支持（来自Microsoft）的组合。对于Atom，还存在一个Java扩展（基于与Visual Studio Code的Java支持相同的代码）。

Spring Tools 4是否支持JDK9，JDK10，JDK11？

是。您可以使用JDK 8/9/10/11运行Spring Tools 4。但是，项目中对Java的语言支持仅限于JDK 8/9/10。对JDK11的完整语言支持将在不久的将来提供更新。

Spring Tools 4是否支持Lombok？

是的，在各种环境中使用Spring Tools 4时，可以在项目中使用Lombok。

我可以在IntelliJ IDEA中安装和使用Spring Tools 4吗？

否。我们实现了一个原型，以将Spring Tools 4与IntelliJ IDEA集成在一起，但是对IntelliJ的语言服务器协议的第三方支持尚不成熟，稳定且功能丰富，不足以实现有意义的Spring Tools版本。 IntelliJ还剩4个。但是，IntelliJ Ultimate Edition中提供了对Spring的出色支持，可以直接使用，如果您更喜欢IntelliJ，可以使用它。

将来还将支持其他哪些IDE和编辑器？

我们还没有确切的计划，但是我们一直在监视语言服务器社区，以寻找可能适合Spring Tools 4的新兴客户和环境。例如，Eclipse Theia是我们目前正在考虑的项目之一。
