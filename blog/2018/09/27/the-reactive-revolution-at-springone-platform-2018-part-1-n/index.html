使用R2DBC进行反应式SQL数据访问

首先，我们宣布了我们的工作，以尝试支持R2DBC的反应式SQL数据访问标准 。 R2DBC还处于初期，但非常令人兴奋。到目前为止，当我们谈论数据访问时，我已经很快提醒人们，尽管他们可以在反应性应用程序中使用JDBC，但他们将扩展交互作用作为自己的责任。由他们来配置更多的线程到Scheduler分配给其中发生任何SQL数据访问的反应性流。这是必需的，因为JDBC是一个从根本上阻止和同步的API。它没有执行IO的功能，并且在活动发生时会给您回电；客户端线程将等待答复。R2DBC提供了另一种选择。这并不是要成为JDBC的包装，而是要支持基于本机反应式SQL数据库驱动程序构建的功能性反应式数据访问。我们有一个SPI层和一个支持PostgreSQL的实现。

让我们看一个例子。为了使它起作用，我去了Spring Initializr并选择了Reactive Web和Lombok 。我确保选择一个SNAPSHOT版本的Spring Boot。本质上 ，您不需要Spring Boot本身的SNAPSHOT，但是您希望Spring Initializr将Spring快照存储库添加到您的构建中，以便您可以解决r2dbc-postgresql依赖性。然后，我（手动！惊恐的事件！）编辑了Maven版本， pom.xml ，并添加了对io.r2dbc:r2dbc-postgresql:1.0.0.BUILD-SNAPSHOT对构建的依赖。

package s1p.r2dbc;

import lombok.AllArgsConstructor;
import lombok.Data;
import lombok.NoArgsConstructor;
import lombok.extern.log4j.Log4j2;

import io.r2dbc.postgresql.PostgresqlConnectionFactory;
import io.r2dbc.postgresql.PostgresqlResult;

import org.springframework.stereotype.Service;
import reactor.core.publisher.Flux;

import org.springframework.beans.factory.annotation.Value;
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.context.annotation.Bean;

import java.net.URI;

@Log4j2
@SpringBootApplication
public class PostgresqlApplication {

  public static void main(String args[]) {
    SpringApplication.run(PostgresqlApplication.class, args);
  }

  @Bean
  PostgresqlConnectionFactory connectionFactory(
      @Value("${spring.datasource.url}") String url) {

    URI uri = URI.create(url);
    String host = uri.getHost();
    String userInfo = uri.getUserInfo();
    String user = userInfo, pw = "";

    if (userInfo.contains(":")) {
      user = userInfo.split(":")[0];
      pw = userInfo.split(":")[1];
    }

    String name = uri.getPath().substring(1);
    PostgresqlConnectionConfiguration configuration = PostgresqlConnectionConfiguration
        .builder() //
        .database(name) //
        .host(host) //
        .username(user) //
        .password(pw) //
        .build();
    return new PostgresqlConnectionFactory(configuration);
  }
}

@Log4j2
@Service
class CustomerService {

  private final ConnectionFactory connectionFactory;

  CustomerService(PostgresqlConnectionFactory pgc) {
    this.connectionFactory = pgc;
  }

  Flux<Result> delete(Long id) {
    return Mono.from(this.connectionFactory.create())
      .flatMapMany(connection -> connection
        .createStatement("DELETE FROM customers where id = $1")
        .bind("$1", id) //
        .execute());
  }

  Flux<Result> create(Long id, String email) {
    return Mono.from(this.connectionFactory.create())
      .flatMapMany(connection -> connection
        .createStatement("INSERT INTO customers(id,email) VALUES($1, $2)")
        .bind("$1", id) //
        .bind("$2", email) //
        .add().execute());
  }

  Flux<Customer> all() {

    return Mono
      .from(this.connectionFactory
        .create())
      .flatMapMany(connection -> Flux.from(
        connection.createStatement("select * from customers").execute())
        .flatMap(result -> result.map((row, rowMetadata) -> new Customer(row.get("id", Long.class),
          row.get("email", String.class)))));
  }

}

@Data
@AllArgsConstructor
@NoArgsConstructor
class Customer {

  private Long id;

  private String email;

}

很酷吧？这些API具有本地反应性，这意味着您可以利用这些API提供给您的重试和合成功能。

RSocket：电抗线协议

我们还首次展示了对RSocket的支持，RSocket是由Netflix员工（后来移居至Facebook）开发的协议。RSocket是一种有线协议，它作为协议本身的一部分，将响应处理的租户浮出水面。Facebook开发了两个RSocket客户端：一个是C ++，另一个是Java。Java RSocket客户端基于Reactor项目！但是，RSocket是一个二进制协议，因此从理论上讲，您也可以使用其他语言来构建客户端。

RSocket是通用数据传输协议。它支持多种消息交换模式或样式，包括但不限于请求-响应，即发即忘，发布-订阅和流式传输。天空是极限！这篇文章不能希望彻底介绍所有选项，所以让我们看一个简单的流示例，它包含两个部分，生产者和消费者。为了使它起作用，我去了Spring Initializr，选择了Lombok我选择了最新（稳定）的Spring Boot版本。在构建文件中， pom.xml ，我添加了两个依赖项： io.rsocket:rsocket-transport-netty:0.11.5和io.rsocket:rsocket-core:0.11.5 。

这里的目标是演示这些API的简单性和灵活性。您可以直接使用它们，也可以像Netifi一样使用它们，作为支持甚至更高级用例的整套基础架构的基础。

这个简单示例的症结在于，制作人将永远永远每秒发出一个新记录！只要宇宙没有经历过热死，这个例子就应该继续下去！

首先，让我们看一个简单的生产者示例。我进行了设置，以便它可以收听ApplicationReadyEvent然后才开始处理请求。我想保持Java进程（从而监听请求）的状态，因此，我有点费解了System.in.read()轮询控制台以获取输入。请记住，此API是非阻塞且异步的！如果我们不帮忙，它将无法使主线程保持活动状态。

package s1p.rsocket.producer;

import io.rsocket.AbstractRSocket;
import io.rsocket.Payload;
import io.rsocket.RSocketFactory;
import io.rsocket.SocketAcceptor;
import io.rsocket.transport.netty.server.TcpServerTransport;
import io.rsocket.util.DefaultPayload;
import lombok.extern.log4j.Log4j2;
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.boot.context.event.ApplicationReadyEvent;
import org.springframework.context.ApplicationListener;
import reactor.core.publisher.Flux;
import reactor.core.publisher.Mono;

import java.io.IOException;
import java.time.Duration;
import java.time.Instant;

@SpringBootApplication
@Log4j2
public class ProducerApplication implements ApplicationListener<ApplicationReadyEvent> {

  public static void main(String[] args) throws IOException {
    SpringApplication.run(ProducerApplication.class, args);
    System.in.read();
  }

  @Override
  public void onApplicationEvent(ApplicationReadyEvent applicationReadyEvent) {
    SocketAcceptor sa = (connectionSetupPayload, rSocket) ->
      Mono.just(new AbstractRSocket() {

        @Override
        public Flux<Payload> requestStream(Payload payload) { // produce a result when asked...
          return Flux
            .interval(Duration.ofMillis(1000))  // ...every one second
            .map(aLong -> DefaultPayload.create("interval: " + aLong));
        }
      });

    RSocketFactory
      .receive()
      .acceptor(sa)
      .transport(TcpServerTransport.create("localhost", 7000))
      .start()
      .onTerminateDetach()
      .subscribe(nettyContextCloseable -> log.info("started the server @ " + Instant.now().toString()));
  }
}

消费者也很简单。它向服务发起请求，获取响应流Publisher从服务中访问并遍历每条记录，将二进制有效负载解压缩为String ，并在到达时对其进行记录。我也在这里用System.in.read()作为保持线程运行的一种方式。

package s1p.rsocket.consumer;

import io.rsocket.Payload;
import io.rsocket.RSocketFactory;
import io.rsocket.transport.netty.client.TcpClientTransport;
import io.rsocket.util.DefaultPayload;
import lombok.extern.log4j.Log4j2;
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.boot.context.event.ApplicationReadyEvent;
import org.springframework.context.ApplicationListener;

import java.io.IOException;

@Log4j2
@SpringBootApplication
public class ConsumerApplication implements ApplicationListener<ApplicationReadyEvent> {

  public static void main(String[] args) throws IOException {
    SpringApplication.run(ConsumerApplication.class, args);
    System.in.read();
  }

  @Override
  public void onApplicationEvent(ApplicationReadyEvent evt) {
    RSocketFactory
      .connect()
      .transport(TcpClientTransport.create("localhost", 7000))
      .start()
      .flatMapMany(socket ->
        socket
          .requestStream(DefaultPayload.create("Hello"))
          .map(Payload::getDataUtf8)
          .doFinally(signal -> socket.dispose())
      )
      .subscribe(name -> log.info("consuming " + name + "."));
  }
}

生产者和消费者都可以连接到localhost:7000然后开始互动。像HTTP一样，RSocket并不真正在乎消息的有效负载是什么。毕竟，它是线路上的二进制数据。

下一步

在SpringOne平台上记录了很多关于这两个主题的大量讨论！他们将尽快上线。（在此期间，您不希望自己在这里吗？） Spring 在这里起着重要的作用，因为它是端到端的反应性。想象一下！您可以构建反应性微服务，以使用RSocket在服务之间进行通信。为什么不使用@Tailable响应式Spring Data MongoDB存储库方法可以通过RSocket对MongoDB数据集和流中的数据进行某种连续查询，这些结果将在到达MongoDB存储区时返回？您也可以做相反的事情。使用刚刚发布的Spring Data Lovelace中的Spring Data MonogDB版本中的新的反应式事务支持，您的RSocket客户端可以流式传输大量数据以写入MongoDB！为什么不使用R2DBC并通过RSocket以流方式返回大量数据。RSocket也是边缘的最佳选择！您可以在例如websockets上执行RSocket。想象一下：HTML5客户端可以直接使用基于RSocket的服务。或者，您可以使用Spring WebFlux创建反应式HTTP服务，该服务依次调用基于RSocket的服务，后者又调用R2DBC。就像他们说的那样，天空可能是无限的和异步的！
