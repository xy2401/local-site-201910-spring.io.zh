Spring Session Bean GA发布

发布
罗伯·温奇
十月31，2018

该帖子由VedranPavić撰写

我很高兴代表社区宣布Spring Session BOM Bean的全面上市。这是基于Spring Session 2.1的第一个发行版，可以很容易地与新发行的Spring Boot 2.1一起使用 。请继续阅读以了解该版本的重点。

相同站点Cookie支持HttpSession积分

Same-Site Cookie是另一种机制， 可帮助开发人员保护其免受Cross-Site Request Forgery的侵害 。我们的DefaultCookieSerializer进行了增强以支持添加SameSite属性赋予Spring Session产生的会话Cookie。的SameSite默认情况下，属性启用为value Lax并且可以使用DefaultCookieSerializer#setSameSite 。

请注意，对WebSession从Spring Framework 5.1开始，它就存在于Spring WebFlux本身中。

HttpSessionBindingListener支持

Spring Session现在支持HttpSessionBindingListener并将正确调用此API实现的回调。这也意味着现在可以使用@PreDestroy在会话范围内的bean上，因为该功能是使用Spring Framework的DestructionCallbackBindingListener这是具体的实现HttpSessionBindingListener 。

自定义序列化Redis WebSession积分

我们的Redis WebSession集成现在可以轻松配置为使用自定义序列化机制。以与Redis类似的方式HttpSession集成，这可以通过提供一个@Bean类型的RedisSerializer
