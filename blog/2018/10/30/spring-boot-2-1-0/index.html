春季启动2.1.0

发布
安迪·威尔金森
十月30，2018

谨代表Spring Boot团队和所有参与的人员，宣布Spring Boot 2.1.0已发布，现在可以从repo.spring.io ， Maven Central和Bintray获得 。此版本增加了大量新功能和改进。有关完整的升级说明以及新的和值得注意的功能，请参阅发行说明 。

2.1新增功能

第三方库升级

我们已尽可能升级到其他第三方库的最新稳定版本。此版本中一些值得注意的依赖项升级包括：

  • 休眠5.3
  • 千分尺1.1
  • Cali反应釜
  • 春天数据Lovelace
  • Spring Framework 5.1
  • 雄猫9
  • 暗影2

性能提升

作为我们不断提高性能的努力的一部分，我们在Spring Boot 2.1中取得了一些重大进展。现在，应用程序启动速度更快，并且消耗的内存更少。这在具有非常严格的内存约束的环境中特别有用。

我们还接受了Spring Framework和Spring Data JPA对异步引导Hibernate的支持。如果您使用Spring Data JPA并设置spring.data.jpa.repositories.bootstrap-mode=deferred ，Hibernate将在单独的线程中引导，而应用程序的其余启动过程将并行进行。

Java 11支持

从Spring Framework 5.1对Java 11的支持开始，Spring Boot 2.1现在还支持Java 11，同时仍与Java 8兼容。

DataSize支持

如果属性需要以字节或类似的方便单位表示大小，则可以公开org.springframework.util.unit.DataSize属性。类似于我们的Duration Spring Boot 2.0中引入了对支持的支持，数据大小支持允许在配置值时指定单位application.properties 。例如， 10MB可以用于10兆字节的值。

执行器端点

Spring Boot 2.1中引入了两个新的Actuator端点：

  • /actuator/caches提供有关应用程序的缓存管理器的信息
  • /actuator/integrationgraph提供Spring Integration组件的图形表示

运行状况端点也得到了增强，以允许请求单个组件的运行状况。例如，请求/actuator/health/db仅执行“ db” HealthIndicator。

指标

除了升级到Micrometer 1.1，还添加了用于导出到AppOptics，Humio和KariosDB的自动配置。指标覆盖范围也得到了改进，包括：

  • 休眠指标
  • Spring Framework的WebClient
  • 卡夫卡消费者指标
  • Log4j2指标
  • Jetty服务器线程池指标
  • 服务器端Jersey HTTP请求指标

其他变化

发行说明中还记录了许多其他更改和改进。您还可以找到我们计划在下一版本中删除的不推荐使用的类和方法的列表。
