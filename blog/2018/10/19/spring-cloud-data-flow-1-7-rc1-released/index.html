The Spring Cloud Data Flow team is pleased to announce the release of 1.7 RC1. Follow the Getting Started guides for Local Server, Cloud Foundry, and Kubernetes.

The RC1 release builds on the core features introduced in 1.7 M1 with a few refinements.

Here are the highlights

  • Improved UI — The UI has a completely new look. The property whitelisting functionality has been refined to not display all application properties by default if the whitelist is empty.

  • Stream Application DSL — This has been integrated into the UI so you can design a data pipeline made out of arbitrary Boot applications.

  • Support for scheduling of tasks on Kubernetes.

  • Audit trail. — Task and Scheduler supported has been added.

  • Concurrent Task Launch Limiting.

  • Stream and Task validation. — Added support for validation of applications registered of type 'app'.

  • Force upgrade for Streams.

  • Added section to customize docker compose to include Skipper. See item 5 in Customizing Spring Cloud Data Flow.

Spring Cloud Data Flow 1.7 RC1发布

发布
马克·波拉克
十月19，2018

Spring Cloud Data Flow团队很高兴宣布1.7 RC1的发布。遵循本地服务器 ， Cloud Foundry和Kubernetes的入门指南 。

RC1版本以1.7 M1中引入的核心功能为基础，并进行了一些改进。

这是亮点

  • 改进的用户界面 -用户界面具有全新的外观。如果白名单为空，则属性白名单功能已经过改进，默认情况下不会显示所有应用程序属性。

  • 流应用程序DSL —已集成到UI中，因此您可以设计由任意Boot应用程序组成的数据管道。

  • 支持Kubernetes上的任务计划 。

  • 审计追踪 。 —已添加支持的任务和计划程序。

  • 并发任务启动限制 。

  • 流和任务验证 。 —添加了对注册为“ app”类型的应用程序的验证的支持。

  • 强制升级Streams 。

  • 添加了用于自定义Docker组成部分以包括Skipper的部分。请参阅“ 定制Spring Cloud Data Flow”中的项目5。
