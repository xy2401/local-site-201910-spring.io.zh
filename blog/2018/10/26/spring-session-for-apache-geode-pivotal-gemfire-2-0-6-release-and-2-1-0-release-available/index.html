Apache Geode / Pivotal GemFire 2.0.6的 Spring 会议。发布和2.1.0。发布可用！

工程
约翰·布鲁姆
十月26，2018

好事成对出现，所以我很高兴宣布Apache Geode和Pivotal GemFire （SSDG）的Spring Session可用2.0.6.RELEASE和2.1.0.RELEASE 。

固态硬盘2.0.6.RELEASE ，基于Spring Framework 5.0.10.RELEASE ， Spring Data Kay-SR11和Spring Session核心2.0.7.RELEASE主要包含依赖项更新。

固态硬盘2.1.0.RELEASE另一方面，已升级到Spring Framework 5.1.1.RELEASE ， Spring Data Lovelace-SR1和Spring Session核心2.1.0.RELEASE ，此外还支持自定义会话过期政策和规则。

自定义会话过期配置

Apache Geode和Pivotal GemFire的 Spring 会议现在提供了一个新的Strategy界面， SessionExpirationPolicy允许用户自定义有关会话应如何终止以及何时终止的规则。

该接口大致定义为：

SessionExpirationPolicy接口
import java.time.Duration;

@FunctionalInterface
interface SessionExpirationPolicy {

  Optional<Duration> determineExpirationTimeout(Session session);

  default ExpirationAction getExpirationAction() {
    return ExpirationAction.INVALIDATE;
  }
}

用户只需要实施determineExpirationTimeout方法，该方法返回一个Optionaljava.time.Duration指示会话将要过期的时间长度。

Duration是可选的，因此，在某些情况下，如果您的自定义策略无法为特定会话确定过期超时，则返回Optional.empty()会向Apache Geode或Pivotal GemFire指示应应用为管理应用程序会话状态的区域配置的默认条目空闲超时到期策略。默认的空闲超时为30分钟，您可以通过设置@EnableGemFireHttpSession批注的maxInactiveIntervalInSeconds属性来更改。

小费
 或者，您可以设置spring.session.data.gemfire.session.expiration.max-inactive-interval-seconds Spring Boot的属性application.properties ，或在Spring容器中声明类型为SpringSessionGemFireConfigurer的 bean，并重写getMaxInactiveIntervalInSeconds（）方法。

然后，您只需声明您的自定义SessionExpirationPolicy实例作为Spring容器中的Bean，然后确保通过设置Apache Geode或Pivotal GemFire的Spring Session知道它sessionExpirationPolicyBeanName的属性@EnableGemFireHttpSession注释，如下所示：

@SpringBootApplication
@EnableGemFireHttpSession(
  sessionExpirationPolicyBeanName = "customExpirationPolicy"
)
class MySpringSessionApplication {

  @Bean
  SessionExpirationPolicy customExpirationPolicy(..) {
    return new CustomExpirationPolicy(..);
  }

  ...
}

因为SessionExpirationPolicy实例是一个bean，就像Spring容器中的任何其他托管bean一样，您可以配置它，但是可以随意注入对其他bean的引用，等等。

小费
 同样，自定义的Bean名称SessionExpirationPolicy可以在Spring Boot中设置application.properties使用spring.session.data.gemfire.session.expiration.bean-name或通过声明一个实例SprinSessionGemFireConfigurer在Spring容器中并覆盖getSessionExpirationPolicyBeanName()方法。

更多细节可以在文档中找到。

固定期限到期

除了允许自定义Session过期策略和规则外，Apache Geode和Pivotal GemFire的Spring Session开箱即用，还提供了此接口的实现以支持固定的持续时间过期。

这在会话应超时的情况下很有用，无论该会话在固定的持续时间（可能为1小时）之后，无论什么情况，只要会话尚未空闲即可。

这个请求实际上是在 Spring Spring Session Issue＃922- “ 绝对会话超时 ”中提出的。

虽然并非所有Spring Session的商店提供者都具有此功能，但Apache Geode或Pivotal GemFire才有可能。

简单启用以下内容：

import org.springframework.session.data.gemfire.expiration.support.FixedTimeoutSessionExpirationPolicy;

@SpringBootApplication
@EnableGemFireHttpSession(
  sessionExpirationPolicyBeanName = "fixedTimeoutExpirationPolicy"
)
class MySpringSessionApplication {

  @Bean
  SessionExpirationPolicy fixedDurationExpirationPolicy() {
    return new FixedTimeoutSessionExpirationPolicyt(Duration.ofMinutes(60L));
  }

  ...
}

当然，适用于Apache Geode和Pivotal GemFire的Spring Session会谨慎地注意Session的空闲超时。这样，只要空闲超时或固定超时时间过去（以较早者为准），会话就会终止。

同样，可以在docs中找到更多详细信息。
