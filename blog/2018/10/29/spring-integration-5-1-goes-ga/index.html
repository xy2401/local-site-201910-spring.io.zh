Spring Integration 5.1正式发布！

发布
阿尔特姆·比兰（Artem Bilan）
十月29，2018

我很荣幸地代表Spring Integration小组宣布5.1.0.RELEASE适用于Spring Integration。

可以从Maven Central，JCenter和我们的发布存储库中下载 ：

compile "org.springframework.integration:spring-integration-core:5.1.0.RELEASE"

首先，我要感谢所有社区成员对框架的持续积极贡献！

除了常规的依赖项升级之外，此版本中还引入了一些值得注意的新功能，这些错误修复和内部性能改进在此处：

BoundRabbitChannelAdvice

为了严格排序消息发布BoundRabbitChannelAdvice可以用作MessageHandler建议允许在同一线程绑定中执行所有下游AMQP操作Channel 。通常与拆分器或其他会导致发送多条消息的机制一起使用：

@Bean
public IntegrationFlow flow(RabbitTemplate template) {
    return IntegrationFlows.from(Gateway.class)
            .split(s -> s.delimiters(",")
                    .advice(new BoundRabbitChannelAdvice(template))
            .transform(String::toUpperCase)
            .handle(Amqp.outboundAdapter(template).routingKey("rk"))
            .get();
}

反应式轮询

如果SourcePollingChannelAdapter要么PollingConsumer配置了outputChannel作为一个FluxMessageChannel ，轮询任务不是由调度程序执行，而是作为Flux.generate()随后Mono.delay()根据trigger.nextExecutionTime() 。通过这种方式，根据需要执行真实源轮询，以尊重由网络传播的下游背压。 FluxMessageChannel 。的PollerMetadata选项对于最终配置保持相同且透明。

Java DSLfluxTransform()

的IntegrationFlowDefinition现在提供了新的fluxTransform()接受一个运算符Function用于转换Flux通过被动方式来处理传入消息。在下面fluxTransform()完全基于反应堆的Flux.transform()运算符，如果函数尚未产生一个请求头，则将请求头复制到答复消息中。该呼叫包含输入和输出FluxMessageChannel内部实例：

IntegrationFlow integrationFlow = f -> f
        .split()
        .fluxTransform(flux -> flux
                .map(Message::getPayload)
                .map(String::toUpperCase))
        .aggregate(a -> a
                .outputProcessor(group -> group
                        .getMessages()
                        .stream()
                        .map(Message::getPayload)
                        .map(String.class::cast)
                        .collect(Collectors.joining(","))))
        .channel(resultChannel);

Java DSL还为方便起见提供了更多的运算符： nullChannel() ， convert(Class)和logAndReply() 。有关更多信息，请参见其Javadocs。

Java函数改进

的java.util.function接口现在充当消息传递端点的一等公民。的Function和Consumer可以直接从@ServiceActivator要么@Transformer定义，例如：

@Bean
@Transformer(inputChannel = "functionServiceChannel")
public Function functionAsService() {
    return String::toUpperCase;
}

的Supplier界面可以简单地与@InboundChannelAdapter注释，或作为ref在一个 ：

@Bean
@InboundChannelAdapter(value = "inputChannel", poller = @Poller(fixedDelay = "1000"))
public Supplier pojoSupplier() {
    return () -> "foo";
}

Kotlin lambdas现在也可以直接在消息传递端点定义中使用：

@Bean
@Transformer(inputChannel = "functionServiceChannel")
fun kotlinFunction(): (String) -> String {
    return { it.toUpperCase() }
}

@Bean
@ServiceActivator(inputChannel = "messageConsumerServiceChannel")
fun kotlinConsumer(): (Message) -> Unit {
    return { print(it) }
}

@Bean
@InboundChannelAdapter(value = "counterChannel",
        poller = [Poller(fixedRate = "10", maxMessagesPerPoll = "1")])
fun kotlinSupplier(): () -> String {
    return { "baz" }
}

千分尺

连同升级到千分尺1.1现在，框架会自动从MeterRegistry 。当我们动态开发时，这变得非常方便IntegrationFlow并在运行时注册并删除它们。

JMX MBean移除

现在，MBean在运行时注册（例如通过动态注册） IntegrationFlow s）在运行时销毁了它们的bean时，会自动从JMX服务器注册中删除它们。

HTTP动态映射

在运行时声明的HTTP（和WebFlux）入站端点（例如，通过动态IntegrationFlow s），现在在HandlerMapping并在销毁其bean时自动将其删除。

Spring Social Twitter支持

由于Spring Social项目即将结束生命 ，我们已经搬迁了spring-integration-twitter模块以在Spring Integration Extensions伞下分隔项目，并且刚刚发布了一个org.springframework.integration:spring-integration-social-twitter:1.0.0.RELEASE也完全基于Spring Integration 5.1 。
