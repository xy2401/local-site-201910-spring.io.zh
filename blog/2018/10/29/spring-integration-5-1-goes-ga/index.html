Spring Integration 5.1正式发布！

发布
阿尔特姆·比兰（Artem Bilan）
十月29，2018

我很高兴代表Spring Integration团队宣布5.1.0.RELEASE 。

可以从Maven Central，JCenter和我们的发行版存储库下载 ：

compile "org.springframework.integration:spring-integration-core:5.1.0.RELEASE"

首先，我要感谢所有社区成员对框架的持续积极贡献！

除了常规的依赖项升级之外，此版本中还引入了一些值得注意的新功能，这些错误修复和内部性能改进在此处：

BoundRabbitChannelAdvice

为了严格地发布消息，可以将BoundRabbitChannelAdvice用作MessageHandler通知，以允许在同一线程绑定Channel执行所有下游AMQP操作。通常与拆分器或其他会导致发送多条消息的机制一起使用：

@Bean
public IntegrationFlow flow(RabbitTemplate template) {
    return IntegrationFlows.from(Gateway.class)
            .split(s -> s.delimiters(",")
                    .advice(new BoundRabbitChannelAdvice(template))
            .transform(String::toUpperCase)
            .handle(Amqp.outboundAdapter(template).routingKey("rk"))
            .get();
}

反应式轮询

如果SourcePollingChannelAdapter或PollingConsumer被配置为与outputChannel作为FluxMessageChannel ，轮询任务不被调度为进行，但Flux.generate()与随后的Mono.delay()基于所述trigger.nextExecutionTime()这样，将按需执行实际源轮询，以尊重FluxMessageChannel传播的下游FluxMessageChannel 。PollerMetadata选项对于最终配置保持相同且透明。

Java DSL fluxTransform()

现在为IntegrationFlowDefinition提供了新的fluxTransform()运算符，该运算符接受用于通过反应方式转换传入消息Flux的Function 。fluxTransform()下面完全基于Reactor的Flux.transform()运算符，如果函数尚未产生请求，则将请求标头复制到回复消息中。该调用在内部包含输入和输出FluxMessageChannel实例：

IntegrationFlow integrationFlow = f -> f
        .split()
        .fluxTransform(flux -> flux
                .map(Message::getPayload)
                .map(String::toUpperCase))
        .aggregate(a -> a
                .outputProcessor(group -> group
                        .getMessages()
                        .stream()
                        .map(Message::getPayload)
                        .map(String.class::cast)
                        .collect(Collectors.joining(","))))
        .channel(resultChannel);

Java DSL还为方便起见提供了更多运算符： nullChannel() ， convert(Class) and logAndReply(). See their Javadocs for more information.

Java Functions Improvements

The java.util.function interfaces now serve as a first class citizens for messaging endpoints. The Function and Consumer可以直接从@ServiceActivator或@Transformer定义中使用，例如：

@Bean
@Transformer(inputChannel = "functionServiceChannel")
public Function functionAsService() {
    return String::toUpperCase;
}

Supplier接口可以简单地与一起使用@InboundChannelAdapterannotation，或作为ref在 ：

@Bean
@InboundChannelAdapter(value = "inputChannel", poller = @Poller(fixedDelay = "1000"))
public Supplier pojoSupplier() {
    return () -> "foo";
}

Kotlin lambdas现在也可以直接在消息传递端点定义中使用：

@Bean
@Transformer(inputChannel = "functionServiceChannel")
fun kotlinFunction(): (String) -> String {
    return { it.toUpperCase() }
}

@Bean
@ServiceActivator(inputChannel = "messageConsumerServiceChannel")
fun kotlinConsumer(): (Message) -> Unit {
    return { print(it) }
}

@Bean
@InboundChannelAdapter(value = "counterChannel",
        poller = [Poller(fixedRate = "10", maxMessagesPerPoll = "1")])
fun kotlinSupplier(): () -> String {
    return { "baz" }
}

千分尺

现在，随着对Micrometer 1.1的升级，该框架会自动从MeterRegistry删除已注册的MeterRegistry 。当我们开发动态IntegrationFlow并在运行时注册并删除它们时，这变得非常方便。

JMX MBean移除

现在，在运行时注册的MBean（例如通过动态IntegrationFlow ）在运行时销毁的MBean时，会自动从JMX服务器注册中删除。

HTTP动态映射

现在，在运行时（例如，通过dynamic IntegrationFlow ）声明的HTTP（和WebFlux）入站端点在HandlerMapping注册了它们的请求映射，并且在销毁了它们的bean时也自动将其删除。

Spring Social Twitter支持

由于Spring Social项目即将结束生命 ，我们将spring-integration-twitter模块移到了Spring Integration Extensions的单独项目下，并刚刚发布了org.springframework.integration:spring-integration-social-twitter:1.0.0.RELEASE为好，这完全基于Spring集成5.1 。
