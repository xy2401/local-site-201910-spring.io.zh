Spring Cloud Data Flow 1.5 RC1发布

发布
马克·波拉克
2018年5月9日

Spring Cloud Data Flow团队很高兴宣布发布1.5.0 RC1 。遵循本地服务器 ， Cloud Foundry和Kubernetes的入门指南 。

以下是重点内容：

总体改进

  • 切换到Hikari连接池并重组代码以使用更少的连接。

  • 底层部署程序库中的几个错误修复。

仪表板

  • 现在可以从Stream Builder中编辑创建/部署的流。可以编辑和重新部署应用程序和部署属性。App版本也可以切换。

  • 新的分页器组件将添加到所有列表页面。每页可能会从20、30、50或100个项目的列表中切换。这进一步简化了批量操作工作流程。

  • 介绍通过Selenium和SauceLabs进行的端到端测试。

应用入门：

  • 新版本- 摄氏温度。SR2

  • 更新了Rabbit源/接收器以在PCF上运行

  • 更新了python应用

Kubernetes服务器

  • 由于Core Workload API使用 GA，因此客户端和群集版本的兼容性得到了改善。例如，针对分区流管道的StatefulSet部署可动态解决版本兼容性-不再需要硬编码的StatefulSet端点。

  • 扩展添加到“ pod”配置中的注释支持，现在还可以将自定义注释添加到“作业”部署中。

  • 现在支持使用自定义活动性和就绪性探针端口进行部署。

查看1.5.0。M1发布博客已将新功能改进添加到1.5代码库中。
