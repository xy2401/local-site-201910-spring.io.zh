Spring Cloud Data Flow 1.5.0发布

发布
马克·波拉克
2018年5月16日

Spring Cloud Data Flow团队很高兴宣布1.5.0 GA发布。遵循本地服务器 ， Cloud Foundry和Kubernetes的入门指南 。

以下是重点内容：

  • 用户界面改进

  • Spring Boot，Spring Cloud Stream 2.0和Spring Cloud Task 2.0支持

  • 更新的应用程序入门

  • 指标改进

  • 组合任务的嵌套拆分

  • Kubernetes的改进

  • 更新的文件提取示例

用户界面改进

我们一直在继续改进仪表板的UI / UX。我们希望您会立即注意到整体较轻的设计。任务选项卡已被重写，以匹配其他选项卡的UX样式。新的分页器组件已添加到所有列表页面。每页可能会从20、30、50或100个项目的列表中切换。这进一步简化了批量操作工作流程。

更新的“ 流构建器”选项卡使部署流定义和更新已部署的流变得容易。您可以编辑应用程序和部署属性，以及更改流中的各个应用程序的版本并重新部署。Data Flow与Skipper的集成可处理升级过程，从而可以轻松回滚，以防升级未按计划进行。Stream Builder选项卡还包括许多优化，包括更好的表单验证和急切的错误报告。试试看！

还进行了大量的重构，以优化代码库并为将来的扩展和功能添加做准备。还添加了使用Selenium和SauceLabs进行的端到端测试。

Spring Boot和Spring Cloud Stream 2.0支持

现在，我们支持部署基于Spring Boot 2.0和基于Spring Cloud Stream 2.0的应用程序。阅读有关度量的部分以获取相关功能。

更新的应用程序入门

GA发行的Stream App Starters的新版本-Celsius 。SR2使用PCF上的Rabbit源/接收器应用程序修复了一个错误，并更新了python应用程序。GA的新版Task App Starters - Clark SR1删除了一些过时的任务，并包括堆肥任务运行程序的新版本。

发布火车Darwin RC1将流应用程序启动程序更新为基于Spring Boot和Spring Cloud Stream 2.0。一个gRPC处理器已添加。导入URL可以在这里找到。

发布火车Dearborn M1将任务应用程序启动程序更新为基于Spring Boot和Spring Cloud Task 2.0。

Spring Cloud Stream应用程序Initializr已更新，以支持定制基于Darwin的应用程序。

指标改进

Spring Cloud Stream Application Starters库提供了几个实用程序类，通过添加用于标识流和Cloud Foundry环境的标签来丰富基于微米的监视。

现在，更新后的Spring Cloud Stream Application Initializr可让您将测微计库添加到基于Boot 1.5和2.0的应用程序中。

两个新的示例应用程序展示了使用带有数据流的测微计。第一个示例使用InfluxDB和Grafana 。第二个示例使用Prometheus和Grafana。

正在进行的另一项非常有趣的工作是使用Promregator项目，以通过Prometheus监视Cloud Foundry上部署的应用程序。请按照以下说明踢轮胎。

Metrics Collector的2.0 RC1版本基于Spring Boot 2.0和Spring Cloud Stream 2.0。Metrics Collector服务器支持从仅包含Boot 1.x或2.x应用程序的流以及包含多个Boot版本的流中收集度量。将捕获吞吐量速率的一致表示，并将其传播到Data Flow的仪表板。

组合任务的嵌套拆分

由于需求旺盛，此版本增加了DSL支持，以解释组合任务中的“嵌套拆分”。Flo Dashboard和Shell工具会自动适应嵌套拆分。

这是Flo仪表板中DSL表达式的外观：

<<extractFromFTP && cleanseFiles || extractFromS3 && splitTransform> && merge || extractfromOracle>

要使用此功能，您必须注册1.1.1.RELEASE SCDF中的组合任务运行程序的版本。

对于Maven用户

maven://org.springframework.cloud.task.app:composedtaskrunner-task:1.1.1.RELEASE

对于Docker用户

docker:springcloudtask/composedtaskrunner-task:1.1.1.RELEASE

Kubernetes的改进

  • 由于Core Workload API使用GA，因此客户端和群集版本的兼容性得到了改善。例如，针对分区流管道的StatefulSet部署可动态解决版本兼容性。

  • 扩展添加到“ pod”配置中的注释支持，现在还可以将自定义注释添加到“作业”部署中。

  • 现在支持使用自定义活动性和就绪性探针端口进行部署。

  • 在将Skipper与Data Flow结合使用时，已经可以将应用程序部署定向到多个平台后端。但是，我们不支持针对多个Kubernetes平台。现在你可以。 :)

  • Kubernetes服务器现在支持按应用程序使用私有Docker注册表 。

更新的文件提取示例

一个常见的用例是检测FTP站点上的新文件，下载它们，然后启动批处理作业。我们为此用例添加了一个新的File Ingest示例 。在接下来的几个月中，我们将继续改进设计和功能。您可以在这里关注 。

其他小物件

  • 越来越多的新问题涉及单独和全局覆盖的能力JAVA_OPTS适用于在Cloud Foundry上运行的应用程序。我们添加了一个deployer属性（ deployer.yourapp.cloudfoundry.javaOpts ）以支持设置此特定环境变量。

  • 切换到Hikari连接池并重组代码以使用更少的连接。

  • 基础部署程序库中的几个错误修复。
