成为Spring Developer从来没有比现在更好

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2018年3月21日

代表Pivotal传播者团队的Mark Heckler张贴了这篇文章。

对于Spring开发人员来说，过去12个月真是令人难以置信。那时，我们已经看到Spring Framework 5.0，Spring Boot 2和Project Reactor成为GA。在每个主要的Spring项目中，我们还提供了新的和更新的组件版本！成为Spring开发人员的最佳时机到了！

使用所有这些新工具和功能，您如何跟上？您如何在最短的时间内从Spring团队开发人员和主要贡献者那里学习最多？当然，请来到SpringOne平台 ！9月24日至27日，我们所有人都将聚集在马里兰州国家港口的盖洛德国家渡假村（华盛顿特区附近），深入研究这些新钻头，并讨论（并展示）如何全速使用它们。 。这个星期将进行现场编码和生动的对话，邀请您参加其中！

我们将在SpringOne平台上探讨的几个最热门的主题是：

  • Project Reactor，使您可以轻松创建可大规模扩展和高性能的分布式应用程序和系统，从而在整个堆栈中最大化无功收益！

  • riff为您的Spring开发人员带来了有时称为“无服务器”功能的功能。了解如何使用riff入门并快速入门...并了解其很酷的递归首字母缩略词（riff是函数）！

  • Spring Framework 5推出时具有Java 8和9支持以及对Kotlin的完全一流的支持，Kotlin是我称其为“ Java ++”的语言，因为它在Java功能上实现了逻辑上的飞跃，并与Java和Spring实现了无缝互操作性。它还将WebFlux + Netty与WebMVC + Tomcat并存，使您可以在命令式和响应式代码和功能之间轻松切换！

  • Spring Boot 2.0充分利用了所有Spring项目的所有优点，甚至还具有更多优点，使您能够快速构建可用于生产JVM的可部署生产的应用程序。由于Spring Boot 2包含Kotlin，Project Reactor，更新的测试印章（包括JUnit 5）等等，因此它比v1.x更加强大！

  • Spring Cloud比以往任何时候都更进一步地路由，流和一般的云优势。Spring Cloud Gateway将智能路由置于首位，提供HTTP / SSE / WebSocket路由，速率限制，金丝雀部署功能等。通过Spring Cloud Stream，您可以使用强大的抽象功能为流应用程序提供超强功能，从而动态地集成流行的消息队列解决方案（例如RabbitMQ和Apache Kafka）并与之交互。

  • Spring Data现在是反应性的！对于公开反应式API的数据库，没有比Spring Data更好的方法来利用（和SCALE）它们。

  • Spring Security也支持Reactor，使您能够以最大的可扩展性保护应用程序。借助新的Spring Security，您可以疯狂地扩展并发用户，而无需牺牲性能或安全性！

  • 我们还将讨论微服务，CI / CD，事件源，实现和保持开发速度，以及更多！您不会想错过任何一个。

准备从（春季）来源开始深度饮用！加入我们的SpringOne平台，学习，共享，发展和增强团队的知识和速度。等不及要在这里见到你！

干杯，
