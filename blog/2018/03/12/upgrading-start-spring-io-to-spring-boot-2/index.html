将start.spring.io升级到Spring Boot 2

工程
斯特凡·尼科尔（StéphaneNicoll）
2018年3月12日

Spring Boot 2是最近发布的，并且Spring Initializr的生产实例（ start.spring.io ）在同一天已升级到Spring Boot 2。

在本文中，我将引导您完成升级Spring Boot的过程1.x应用到Spring Boot 2。

发行说明和迁移指南

好的第一步是通过阅读迁移指南和发行说明来熟悉Spring Boot 2的主要更改。

构建升级

如果您使用的是Maven spring-boot-starter-parent ，您需要注意，一些插件将在升级过程中进行更新。如果您不使用父级，则值得检查您的构建并升级您正在使用的插件。Spring Initializr是使用Maven构建的，因此最简单的方法是进行扫描spring-boot-dependencies并在必要时升级您正在使用的插件。

升级到Spring Boot 2

Initializr具有专用的自动配置功能，可以自动配置服务。它还公开了专用指标，因此它不仅仅限于使用现有指标。第一步是检查由于重命名和重定位而引起的一些基本编译错误。

就Initializr而言，最大的变化是使用微米作为CounterService和GaugeService不再可用。幸运的是，这是很容易解决的编译错误通过注入MeterRegistry 。然后，我们可以使用注册表来检索Counter每个元素增加。

我们当然需要改善与Micrometer的集成，但是这一基本变化足以使我们前进。

属性迁移

我们还没有研究测试套件，但是我们可以给应用程序一个第一印象。在此之前，让我们将属性迁移器添加到构建中

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-properties-migrator</artifactId>
    <scope>runtime</scope>
</dependency>

或者，如果您使用的是Gradle：

runtime("org.springframework.boot:spring-boot-properties-migrator")

如果我们运行该应用程序，它将识别Spring Boot不再管理的属性。如果有替换项，它将临时为您重新映射属性，并显示警告。如果没有替代品，则错误报告将为您提供更多信息。无论哪种方式，都必须更新配置，并在更新配置后删除依赖关系。

在继续之前，最好使用IDE的搜索功能来再次检查您是否没有使用在集成测试中迁移的属性之一。对于Initializr， management.security.enabled被使用并且必须迁移 。

因此，该应用程序现在开始。在继续之前，让我们看一下测试套件。

测试基础架构

作为升级的一部分，测试实用程序也进行了升级（我们主要使用AssertJ和Mockito）。AssertJ有一个有趣的行为更改： containsSequence在一个String停止在序列中允许孔（ containsSubSequence针对该行为而引入）。Mockito升级很轻松（ ArgumentMatcher现在是一个接口，并且导入argThat必须更改）。

我们也在使用PropertiesConfigurationFactory将POJO绑定到Environment 。在2.0中，全新Binder引入了更灵活，更易于使用的API。我们用10行代码完成的绑定可以简化为3行 。

最后，我们的自动配置测试现在可以从ApplicationContextRunner ，而不是创建ApplicationContext并自己管理其生命周期。单独的博客文章中详细介绍了此功能。

现在，我们可以运行测试并意识到有些测试失败。服务的主页在浏览器中不再起作用！某些通过端点执行构建脚本创建的测试也失败了。

胡子前缀更改

模板不再呈现的原因是由于Mustache模板的前缀已更改 。将我们的模板重命名为.mustache足以解决该问题。

Spring MVC路径匹配默认行为更改

第二个问题是我们端点中路径匹配的滥用。允许您创建端点的端点pom.xml文件如下：

@RequestMapping("/pom")
@ResponseBody
public ResponseEntity<byte[]> pom(BasicProjectRequest request) { ... }

希望用户致电/pom.xml这就是我们最初应该映射的内容。在Spring Boot 2中， 匹配默认行为的Spring MVC路径已更改，因此该端点不再匹配。

一个简单的解决方法是：

@RequestMapping(path = { "/pom", "/pom.xml" })
@ResponseBody
public ResponseEntity<byte[]> pom(BasicProjectRequest request) { ... }

重定向为/info

https://start.spring.io/info是一个众所周知的URL，用于检索Spring Initializr当前正在使用的版本映射。通常，它可以让您知道哪个Spring Cloud版本与您当前的Spring Boot版本匹配。

Spring Boot 2将执行器端点移至/actuator并且我们希望遵循默认设置。但这并不意味着我们要破坏用户可能已添加书签的URL。为了获得两全其美的效果，我们可以添加一个简单的重定向：

@Bean
public WebMvcConfigurer initializrWebMvcConfigurer() {
    return new WebMvcConfigurer() {
        @Override
        public void addViewControllers(ViewControllerRegistry registry) {
            registry.addRedirectViewController("/info", "/actuator/info");
        }
    }
}

包起来

最后，将Spring Initializr迁移到Spring Boot 2非常容易。阅读发行说明和迁移指南以事先了解可能影响您的应用程序的更改是关键。自从Spring Boot 2发布以来， start.spring.io每天为世界各地的社区愉快地生成数万个项目。

下一步是将指标正确迁移到Micrometer，放弃层次结构名称，而是使用标签。如果您有兴趣了解更多信息，请订阅＃526 。
