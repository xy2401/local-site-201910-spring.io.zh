使用Spring Security 5与OAuth 2安全服务集成，例如Facebook和GitHub

工程
克雷格·沃尔斯
2018年3月6日

Spring Security 5的主要功能之一是支持编写与由OAuth 2保护的服务集成的应用程序。这包括通过Facebook或GitHub等外部服务登录应用程序的能力。

但是，通过一点点额外的代码，您还可以获得OAuth 2访问令牌，该令牌可用于针对服务的API执行授权的请求。

在本文中，我们将研究如何开发使用Spring Security 5与Facebook集成的Spring Boot应用程序。您可以在https://github.com/habuma/facebook-security5上找到本文的完整代码。

启用OAuth 2登录

假设您希望使应用程序的用户能够使用Facebook登录。使用Spring Security 5，再简单不过了。您需要做的就是将Spring Security的OAuth 2客户端支持添加到项目的构建中，然后配置应用程序的Facebook凭据。

首先，将Spring Security OAuth 2客户端库以及Spring Security入门依赖项添加到Spring Boot项目的构建中：

<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-security</artifactId>
</dependency>
<dependency>
  <groupId>org.springframework.security</groupId>
  <artifactId>spring-security-oauth2-client</artifactId>
</dependency>

然后，您需要配置应用程序的客户端ID和客户端密钥（可以通过在https://developers.facebook.com/在Facebook上注册您的应用程序来获取）。所有OAuth 2客户端的属性都以前缀spring.security.oauth2.client.registration 。专门针对Facebook，您将添加facebook.client-id和facebook-client-secret该前缀下的属性。在项目的application.yml文件，看起来像这样：

spring:
  security:
    oauth2:
      client:
        registration:
          facebook:
            client-id: YOUR CLIENT ID GOES HERE
            client-secret: YOUR CLIENT SECRET GOES HERE

您还可以在属性文件或Spring Boot支持的任何属性源中将这些属性设置为环境变量。当然，您将用自己的应用程序自己的客户端ID和密码替换上面YAML中显示的占位符文本。

设置好OAuth 2客户端依赖项并设置这些属性后，您的应用程序现在将通过Facebook提供身份验证。当您尝试未经事先验证而访问页面时，将显示如下页面：

该页面为您提供了使用任何已配置的OAuth 2客户端登录的机会。就我们的目的而言，Facebook是唯一的选择。

单击Facebook链接后，您将被重定向到Facebook。如果您尚未登录Facebook，则会提示您登录。登录后，并假设您尚未授权此应用程序，将显示一个授权提示，该提示如下所示：

如果您选择继续（通过单击“继续”按钮），您将被重定向回您的应用程序并进行身份验证。（如果选择“取消”，您还将被重定向回该应用程序，但不会成功通过身份验证。）

使用Facebook之类的外部服务进行身份验证是传统应用程序登录的不错选择。但这只是故事的一半。用户登录后，您还可以使用该身份验证来访问远程服务的API上的资源。

访问API资源

使用外部OAuth 2服务成功进行身份验证后， Authentication保存在安全上下文中的对象实际上是OAuth2AuthenticationToken其中，以及来自OAuth2AuthorizedClientService可以向我们提供访问令牌，以针对服务的API进行请求。

的Authentication可以通过多种方式获得，包括通过SecurityContextHolder 。一旦有了Authentication ，您可以将其投射到OAuth2AuthenticationToken 。

Authentication authentication =
    SecurityContextHolder
        .getContext()
        .getAuthentication();

OAuth2AuthenticationToken oauthToken =
    (OAuth2AuthenticationToken) authentication;

会有一个OAuth2AuthorizedClientService在Spring应用程序上下文中自动配置为Bean，因此您只需要将其注入到使用它的任何位置即可。

OAuth2AuthorizedClient client =
    clientService.loadAuthorizedClient(
            oauthToken.getAuthorizedClientRegistrationId(),
            oauthToken.getName());

String accessToken = client.getAccessToken().getTokenValue();

致电loadAuthorizedClient()被赋予客户端的注册ID，这就是在配置中注册客户端凭据的方式-在我们的示例中为“ facebook”。第二个参数是用户的用户名。本质上，我们要求客户端服务加载OAuth2AuthorizedClient给定用户和给定服务。带着OAuth2AuthorizedClient在手中，通过调用来询问访问令牌的值很简单getAccessToken().getTokenValue() 。

我们可以应用这种技术来充实该服务的客户端API绑定。首先，我们将创建一个基本的API绑定类来处理确保访问令牌包含在所有请求中的基本任务：

public abstract class ApiBinding {

  protected RestTemplate restTemplate;

  public ApiBinding(String accessToken) {
    this.restTemplate = new RestTemplate();
    if (accessToken != null) {
      this.restTemplate.getInterceptors()
          .add(getBearerTokenInterceptor(accessToken));
    } else {
      this.restTemplate.getInterceptors().add(getNoTokenInterceptor());
    }
  }

  private ClientHttpRequestInterceptor
              getBearerTokenInterceptor(String accessToken) {
    ClientHttpRequestInterceptor interceptor =
                new ClientHttpRequestInterceptor() {
      @Override
      public ClientHttpResponse intercept(HttpRequest request, byte[] bytes,
                  ClientHttpRequestExecution execution) throws IOException {
        request.getHeaders().add("Authorization", "Bearer " + accessToken);
        return execution.execute(request, bytes);
      }
    };
    return interceptor;
  }

  private ClientHttpRequestInterceptor getNoTokenInterceptor() {
    return new ClientHttpRequestInterceptor() {
      @Override
      public ClientHttpResponse intercept(HttpRequest request, byte[] bytes,
                  ClientHttpRequestExecution execution) throws IOException {
        throw new IllegalStateException(
                "Can't access the API without an access token");
      }
    };
  }

}

最重要的部分ApiBinding类是getBearerTokenInterceptor()为该方法创建请求拦截器的方法RestTemplate以确保给定的访问令牌包含在对API的所有请求中。如果给定的访问令牌是null ，但是，特殊的请求拦截器会抛出一个IllegalStateException甚至不尝试发出API请求。对于大多数要求所有请求都经过授权的API而言，这是可接受的甚至是理想的行为。

现在，我们可以根据ApiBinding基类：

public class Facebook extends ApiBinding {

  private static final String GRAPH_API_BASE_URL =
              "https://graph.facebook.com/v2.12";

  public Facebook(String accessToken) {
    super(accessToken);
  }

  public Profile getProfile() {
    return restTemplate.getForObject(
            GRAPH_API_BASE_URL + "/me", Profile.class);
  }

  public List<Post> getFeed() {
    return restTemplate.getForObject(
            GRAPH_API_BASE_URL + "/me/feed", Feed.class).getData();
  }

}

如您所见， Facebook类很简单。所有OAuth 2细节都在ApiBinding ，因此此类可以专注于发出请求以支持应用程序所需的操作。

现在我们只需要配置一个Facebook豆。该bean将进行请求作用域分析，以允许根据用户的访问令牌创建实例。 Authentication ：

@Configuration
public class SocialConfig {

  @Bean
  @RequestScope
  public Facebook facebook(OAuth2AuthorizedClientService clientService) {
    Authentication authentication =
            SecurityContextHolder.getContext().getAuthentication();
    String accessToken = null;
    if (authentication.getClass()
            .isAssignableFrom(OAuth2AuthenticationToken.class)) {
      OAuth2AuthenticationToken oauthToken =
              (OAuth2AuthenticationToken) authentication;
      String clientRegistrationId =
              oauthToken.getAuthorizedClientRegistrationId();
      if (clientRegistrationId.equals("facebook")) {
        OAuth2AuthorizedClient client = clientService.loadAuthorizedClient(
                    clientRegistrationId, oauthToken.getName());
        accessToken = client.getAccessToken().getTokenValue();
      }
    }
    return new Facebook(accessToken);
  }

}

另外，因为getFeed()的方法Facebook API绑定从用户的供稿中获取数据，我们需要设置spring.security.oauth2.client.registration.facebook.scope在验证用户身份时指定“ user_posts”范围：

spring:
  security:
    oauth2:
      client:
        registration:
          facebook:
            client-id: YOUR CLIENT ID GOES HERE
            client-secret: YOUR CLIENT SECRET GOES HERE
            scope: user_posts

更灵活的API绑定

您可能想知道与Spring Social有什么关系，Spring Social还提供对使用外部服务登录以及Facebook的API绑定的支持。

Spring Social提供以下支持ProviderSignInController和SocialAuthenticationFilter 。这两种实现都利用了ConnectionFactory提供一个ServiceProvider用于外部服务。Spring Social的每个API绑定都必须提供API的特定于实现ConnectionFactory和ServiceProvider 。这限制了Spring Social支持通过其实现的服务登录ConnectionFactory和ServiceProvider可用。

相比之下，Spring Security 5能够通过简单地在配置中提供服务详细信息来支持几乎所有OAuth 2或OpenID Connect服务的登录。Spring Security 5开箱即用，为Facebook，Google，GitHub和Okta提供了基线配置（您只需指定客户端ID和密码）。但是，如果必须与其他服务集成，则只能在应用程序配置中指定该服务的详细信息（例如授权URL）。

对于API绑定，Spring Social的API绑定非常广泛，涵盖了它们所针对的API提供的大部分功能。但是实际上，大多数应用程序只需要Spring Social支持的一部分操作。如果仅需要获取用户的供稿，为什么必须使用提供数百种其他操作的大型API绑定？同样，如果您只关心帖子回复的一个或两个属性，为什么要处理Post与Facebook的Graph API提供的功能全面的对象？在许多情况下，编写自己的API绑定（为您的应用程序量身定制）可能会更容易。

而且，Spring Social的API绑定都使用RestTemplate在幕后。如果您希望使用非阻塞式反应性API绑定，那么您就不走运了。改造API绑定以基于WebClient这是一项艰巨的任务，并且实质上会使这些API绑定的维护增加一倍。

但是，如果您已经开发了自己的API绑定，那么换出就很容易RestTemplate为了反应WebClient ，如图所示ReactiveApiBinding这里：

public abstract class ReactiveApiBinding {
  protected WebClient webClient;

  public ReactiveApiBinding(String accessToken) {
    Builder builder = WebClient.builder();
    if (accessToken != null) {
      builder.defaultHeader("Authorization", "Bearer " + accessToken);
    } else {
      builder.exchangeFunction(
          request -> {
            throw new IllegalStateException(
                    "Can't access the API without an access token");
          });
    }
    this.webClient = builder.build();
  }
}

您甚至可以混搭WebClient和RestTemplate在相同的API绑定中，应用非阻塞WebClient在需要的地方，以及RestTemplate同步请求足够的地方。

摘要

Spring Security 5对OAuth 2的客户端支持提供了通过外部服务登录的功能，以及使用从身份验证获得的令牌使用该服务的API的功能。这只是调和Spring的OAuth故事的第一步，该故事目前分布在多个项目中，例如Spring Social和Spring Security OAuth。

Spring Security的未来版本将在OAuth 2客户端支持的基础上继续改进，并采取措施协调围绕OAuth安全服务器端的Spring故事。实际上，Spring Security 5.1.0当前正在进行的工作旨在使使用API更加容易，从而有效地消除了对API的需求。 ApiBinding类和配置中的许多管道代码Facebook本文中显示的bean。敬请关注！
