Apache Geode / Pivotal GemFire 2.0.5的春季会议。发布和2.1.0。M1发布！

工程
约翰·布鲁姆
八月30，2018

我很高兴代表团队和社区宣布发布Apache Geode和Pivotal GemFire（SSDG）的Spring Session。 2.0.5.RELEASE （ 苹果 ）和2.1.0.M1 （ 豆子 ）。

固态硬盘2.0.5.RELEASE基于春季会议2.0.5.RELEASE ， Spring Data Kay-SR9和Spring框架5.0.8.RELEASE在Maven Central中可用。

固态硬盘2.1.0.M1基于春季会议2.1.0.M2 ， Spring Data Lovelace-RC2和Spring框架5.1.0.RC2可以从Spring libs-milestone获得 。

当使用Apache Geode或Pivotal GemFire甚至Pivotal Cloud Cache（PCC）来管理Spring Boot Web应用程序（HTTP）会话状态时，这两个版本都提供了一种配置Spring Session的新方法。

当前，要启用使用Apache Session Geode或Pivotal GemFire作为提供程序的Spring Session的（HTTP）会话状态管理，您将包括org.springframework.session:spring-session-data-geode要么org.springframework.session:spring-session-data-gemfire在您的Spring Boot应用程序类路径上，然后声明@EnableGemFireHttpSession您的应用程序的1上的注释@Configuration类，如下：

@SpringBootApplication
@EnableGemFireHttpSession(maxInactiveIntervalSeconds=600)
public class MyWebApplication {

  public static void main(String[] args) {
    SpringApplication.run(MyWebApplication.class, args);
  }
...
}

@EnableGemFireHttpSession批注包含多个属性，这些属性可以更改会话管理策略的配置（例如，到期；如上所示），以及Apache Geode或Pivotal GemFire用于有效管理会话状态的数据管理策略和索引。

但是，所有这些属性都必须进行硬编码。如果要在部署期间根据环境更改配置怎么办？也许您想使用Spring配置文件来更改配置。

好了，现在您可以通过2种不同方式之一使用Apache Geode或Pivotal GemFire来动态配置Spring Session。

配置属性

Apache Geode / Pivotal GemFire的Spring Session现在使您能够使用众所周知的已发布属性配置会话管理以及Apache Geode或Pivotal GemFire。

的@EnableGemFireHttpSession注释属性记录了所有众所周知的已发布属性本身。

举例来说，要更改会话过期超时，您可以在Spring Boot中指定以下属性application.properties文件，如下：

#application.properties

spring.session.data.gemfire.session.expiration.max-inactive-interval-seconds=600
...

有关使用属性配置SSDG的更多详细信息，请参见文档 。

使用配置器进行配置

像Spring Web MVC一样WebMvcConfigurer回调接口，用于Apache Geode / Pivotal GemFire的Spring Session提供了SpringSessionGemFireConfigurer回调接口，用于调整Spring Session配置的各个方面。

例如，要设置会话到期超时，只需声明以下内容：

@Configuration
class SpringSessionGemFireConfiguration {

  @Bean
  SpringSessionGemFireConfigurer sessionExpirationConfigurer() {

    return new SpringSessionGemFireConfigurer(
        @Value("${spring.session.timeout:600}") int sessionTimeout) {

      @Override
      public int getMaxInactiveIntervalInSeconds() {
        return sessionTimeout;
      }
    };
  }
  ...
}

如上所示，使用Spring的配置程序甚至可以从其他属性派生配置器的配置@Value注解。您可以将不同的Configurer bean定义与不同的Spring配置文件结合使用，依此类推。

有关使用配置器配置SSDG的更多详细信息，请参见文档 。
