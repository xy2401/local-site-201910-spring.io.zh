适用于AWS 2.0 GA和Spring Cloud Stream Kinesis Binder 1.0 GA的Spring Integration

发布
阿尔特姆·比兰（Artem Bilan）
2018年8月21日

亲爱的春季社区！

我很高兴宣布支持Amazon Web Services的 Spring项目的一般可用性： 适用于AWS的Spring Integration 2.0.0.RELEASE和适用于AWS Kinesis的Spring Cloud Stream Binder 1.0.0.RELEASE 。

这两个工件在Spring Release存储库和Maven Central中都可用，并且可用作依赖项：


    org.springframework.integration
    spring-integration-aws
    2.0.0.RELEASE

...

    org.springframework.cloud
    spring-cloud-stream-binder-kinesis
    1.0.0.RELEASE

的spring-integration-aws使用第二个时，不需要将其包括在内。

此Spring Integration for AWS的主要目标2.0当然，围绕AWS Kinesis的支持是由AWS Kinesis支持产生的，而该支持是由Spring Cloud Stream Binder for AWS Kinesis完成的。特别是我们引入了DynamoDbMetadataStore和DynamoDbLockRegistry支持AWS Kinesis的分布式Binder使用者。同时，按照格式良好的Spring Integration抽象，在将应用程序部署到Amazon云环境中或依赖AWS服务时，这些新组件可用于任何其他分布式任务。例如，AWS S3入站通道适配器可以使用DynamoDbMetadataStore为了S3PersistentAcceptOnceFileListFilter以防止在多个分布式应用程序实例中访问相同的已处理资源。的DynamoDbLockRegistry可以用于应用程序集群中的领导者选举。查看有关的更多信息LockRegistryLeaderInitiator在Spring集成参考手册中 。

除了这些新组件之外，SNS和SQS通道适配器现在还支持将标头映射到消息属性作为后退。看到AbstractMessageAttributesHeaderMapper有关更多详细信息，请参阅Javadocs及其实现。

Kinesis通道适配器也支持标头映射，但是机制完全不同，我们只能将它们与有效负载一起嵌入到记录主体中。看到一个EmbeddedJsonHeadersMessageMapper有关更多详细信息，请参见Spring Integration。当Kinesis Binder使用类似的嵌入技术时， headerMode是embedded （默认值）。

有关适用于AWS的Spring Integration中的新功能的更多信息，请参阅以前的发行公告 。 2.0以及适用于AWS Kinesis的Spring Cloud Stream Binder。

另请参阅Spring Cloud Stream Samples，以获取更多有关如何在云流应用程序中使用Kinesis Binder的想法。

感谢所有以任何可能的方式为这些项目做出贡献的人。

我们期待收到任何反馈，并很高兴在下一部分中发展这些项目，以更好地满足社区需求。
