适用于AWS 2.0.0的Spring Integration。RC1和Spring Cloud Stream Kinesis Binder 1.0.0。RC1

发布
阿尔特姆·比兰（Artem Bilan）
2018年8月1日

亲爱的 Spring 社区！

我很高兴宣布支持Amazon Web Services的Spring项目的第一个候选发布者： Amazon Web Services ： AWS的Spring Integration 2.0.0.RC1和适用于AWS Kinesis的Spring Cloud Stream Binder 1.0.0.RC1 。

这两个构件在Spring Milestone存储库中都可用，并且可以作为Maven依赖项使用：


    org.springframework.integration
    spring-integration-aws
    2.0.0.RC1



    org.springframework.cloud
    spring-cloud-stream-binder-kinesis
    1.0.0.RC1

使用第二个时不需要包括第一个。AWS Kinesis Binder也可传递地包括AWS Kinesis和AWS DynamoDB Java SDK依赖项。现在，还包括AWS DynamoDB Lock Client以供领导选举。

非常感谢社区的贡献和反馈，尤其是在实际项目中测试这些产品的帮助。

这是自先前宣布的里程碑2以来的更改：

  • 一种DynamoDbLockRegistry已添加了基于AWS DynamoDB Lock Client基础的分布式锁定实施。

  • 的KinesisMessageDrivenChannelAdapter现在可以提供LockRegistry并在使用前专有访问分片。这样，我们可以确保组中只有一个使用者从其感兴趣的分片中读取记录。当一个使用者去（收益率领导）时，其他一些使用者可能会获得该分片的锁，并继续从存储的检查点进行消耗。

  • 整个Spring Integration AWS项目中的各个组件中的一些其他错误修复和改进。

  • Kinesis Binder现在公开基础的配置属性DynamoDbLockRegistry用于粘合剂的消费者。

  • Kinesis Binder项目也已完成其他一些修复。

像往常一样，通过适当的交流渠道欢迎任何反馈，功能性想法，批评家，错误报告和问题！

我们的目标是在一周左右的时间内发布GA。
