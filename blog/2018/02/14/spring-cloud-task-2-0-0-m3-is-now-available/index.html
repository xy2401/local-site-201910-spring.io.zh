What’s new?

This release includes upgrades to existing dependencies as well as some exciting new features for users of Spring Cloud Task. From a dependencies perspective, Spring Cloud Task 2.0.0.M3 has been upgraded to use the Spring Boot 2.0.0.RC1 stack as well as Spring Cloud’s Finchley M6 dependencies.

Beyond just a dependency upgrade, there are a number of new features within Spring Cloud Task 2.0.0.M3. Let’s take a look.

Spring Cloud Task 2.0.0。M3现在可用

发布
迈克尔·米内拉
2018年2月14日

我们很高兴地宣布Spring Cloud Task 2.0.0。现在可以在Github和Pivotal下载存储库上使用M3。非常感谢所有对此版本做出贡献的人。

什么是新的？

此版本包括对现有依赖关系的升级，以及针对Spring Cloud Task用户的一些激动人心的新功能。从依赖关系的角度来看，Spring Cloud Task 2.0.0。M3已升级为使用Spring Boot 2.0.0。RC1堆栈以及Spring Cloud的Finchley M6依赖项。

除了依赖关系升级以外，Spring Cloud Task 2.0.0还包含许多新功能。M3。让我们来看看。

限制任务实例

由于Spring Cloud Task不负责编排，因此很难阻止同时执行同一任务的多个实例。但是，这对于诸如数据库迁移之类的事情可能是重要的用例。此版本添加了一个侦听器，该侦听器可用于防止同时执行一个任务的多个实例。通过使用Spring Integration的分布式锁定功能，侦听器尝试在启动时获取锁定。如果成功，则任务继续。如果任务无法获取锁，则任务失败。

批处理作业无法完成任务

Spring Boot提供了用于在启动时执行Spring Batch作业的功能，这非常方便。但是，如果您在Spring Cloud Task范围内有一个作业失败，则该任务仍会被标记为成功，因为Spring Cloud Task无法知道该作业失败了（ JobLauncherCommandLineRunner ）。此版本引入了在任务中的作业失败时将任务标记为失败的功能。您可以通过设置spring.cloud.task.batch.failOnJobFailure财产true 。

更新为重要的默认值

引入Spring Cloud Task后，默认情况下它将关闭ApplicationContext在完成任务时以编程方式进行。之所以将其设置为默认值，是因为当时使用Spring Boot的Acuator时，上下文不会自行关闭。这导致上下文永远存在，没有理由这样做。

此默认值导致了另一个意外问题。Spring Intializr（ https://start.spring.io ）生成的默认项目包括一个引导程序ApplicationContext并验证上下文是否已加载。但是，由于一旦任务完成，Spring Cloud Task将以编程方式关闭上下文，因此该默认测试用例将失败，而无需添加配置属性。

Spring Boot的Acuator不再像以前那样保持打开的上下文，因此我们重新考虑了默认值spring.cloud.task.closecontext.enable 。就是现在false （从true先前）。我们预计这不会影响大多数用户，但是，如果您的应用程序要求以编程方式关闭上下文以关闭，则需要相应地配置此属性。
