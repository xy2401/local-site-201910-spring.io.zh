Spring Integration 5.0.2可用

发布
阿尔特姆·比兰（Artem Bilan）
2018年2月21日

我很高兴代表Spring Integration小组宣布5.0.2.RELEASE提供了Spring Integration的维护版本。

可以从Maven Central，JCenter和我们的发布存储库中下载 ：

compile "org.springframework.integration:spring-integration-core:5.0.2.RELEASE"

与往常一样，我要感谢所有社区成员对框架的持续积极贡献！

随着升级到最新的Spring Framework 5.0.4和反应堆Bismuth-SR6版本，此Spring Integration 支持版本提供了许多错误修复，尤其是针对@ServiceActivator带有收集方法参数LockRegistryLeaderInitiator 。因此，鼓励大家进行升级。

此外，此版本还带来了几个新功能：

  • 千分尺支持收集消息传递组件指标。要启用，只需声明一个MicrometerMetricsFactory应用程序上下文中的bean来覆盖内置指标工厂：

@Bean
public MicrometerMetricsFactory metricsFactory(MeterRegistry meterRegistry) {
    return new MicrometerMetricsFactory(meterRegistry);
}

我们的目的是使MicrometerMetricsFactory接下来的默认5.1版。

  • 一种UdpServerListeningEvent当一个UnicastReceivingChannelAdapter准备通过网络接收UDP包。当配置的端口为零且操作系统选择端口时，此选项很有用。也有助于避免轮询isListening()如果您需要等待开始其他一些过程以连接到套接字。

  • 的IntegrationGraphController现在也已为WebFlux环境注册。

  • 的@GlobalChannelInterceptor s现在可以应用于动态注册MessageChannel豆也一样。与结合使用IntegrationFlowContext当我们注册IntegrationFlow在运行时。

这个版本是最新的Spring Boot 2.0 RC2的基础 。

从这里我们期待着切换master到5.1版本开始使用新功能和有价值的改进！
