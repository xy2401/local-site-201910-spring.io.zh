On behalf of the Spring Integration team I am pleased to announce that the 5.0.2.RELEASE maintenance version for the Spring Integration is available.

It can be downloaded from Maven Central, JCenter, and our release repository:

compile "org.springframework.integration:spring-integration-core:5.0.2.RELEASE"

As usual I would like to thank all community members for their ongoing active contributions to the framework!

Along with upgrades to the latest Spring Framework 5.0.4 and Reactor Bismuth-SR6 versions, this Spring Integration support version provides a number of bug fixes, especially for the @ServiceActivator with a collection method argument and LockRegistryLeaderInitiator. So, everybody is encouraged to upgrade.

Spring Integration 5.0.2可用

发布
阿尔特姆·比兰（Artem Bilan）
2018年2月21日

我很高兴代表Spring Integration团队宣布5.0.2.RELEASE维护版本。

可以从Maven Central，JCenter和我们的发行版存储库下载 ：

compile "org.springframework.integration:spring-integration-core:5.0.2.RELEASE"

与往常一样，我要感谢所有社区成员对框架的持续积极贡献！

除了升级到最新的Spring Framework 5.0.4和Reactor Bismuth-SR6版本之外，该Spring Integration 支持版本还提供了许多错误修复，尤其是对于@ServiceActivator带有一个集合方法参数和LockRegistryLeaderInitiator 。因此，鼓励大家进行升级。

此外，此版本还带来了几个新功能：

  • 千分尺支持收集消息传递组件指标。要启用，只需在应用程序上下文中声明MicrometerMetricsFactory bean即可覆盖内置的度量标准工厂：

@Bean
public MicrometerMetricsFactory metricsFactory(MeterRegistry meterRegistry) {
    return new MicrometerMetricsFactory(meterRegistry);
}

我们的目的是使此MicrometerMetricsFactory在下一个5.1版本中成为默认值。

  • 当UnicastReceivingChannelAdapter准备通过网络接收UDP包时，已添加要发送的UdpServerListeningEvent 。当配置的端口为零且操作系统选择端口时，此选项很有用。如果需要在开始其他进程连接到套接字之前等待，还可以避免轮询isListening() 。

  • 现在， IntegrationGraphController也已为WebFlux环境注册。

  • @GlobalChannelInterceptor也可以应用于动态注册的MessageChannel bean。当我们在运行时注册IntegrationFlow时，这与IntegrationFlowContext结合使用非常有用。

这个版本是最新的Spring Boot 2.0 RC2的基础 。

从这里开始，我们期待将master版本切换到5.1版本，以开始开发新功能和有价值的改进！
