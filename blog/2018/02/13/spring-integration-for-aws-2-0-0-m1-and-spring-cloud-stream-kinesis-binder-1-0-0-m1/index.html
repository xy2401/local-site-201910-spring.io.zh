适用于AWS 2.0.0的Spring Integration。M1和Spring Cloud Stream Kinesis Binder 1.0.0。M1

发布
阿尔特姆·比兰（Artem Bilan）
2018年2月13日

亲爱的春季社区！

我很高兴宣布Amazon Web Services的两个Spring Milestone版本： 适用于AWS的Spring Integration 2.0.0.M1和适用于AWS Kinesis的Spring Cloud Stream Binder 1.0.0.M1 。

这两个里程碑在Spring Milestone存储库中都可用，并且可以作为Maven依赖项使用：


    org.springframework.integration
    spring-integration-aws
    2.0.0.M1



    org.springframework.cloud
    spring-cloud-stream-binder-kinesis
    1.0.0.M1

适用于AWS的Spring Integration

的2.0该版本是适用于AWS框架的下一代Spring Integration。它基于Java 8 ，Spring框架和Spring集成5.0和适用于AWS的Spring Cloud 2.0 。在这一新的文章中，我们改进了所有MessageHandler扩展通用的实现AbstractAwsMessageHandler与AsyncHandler ， failureChannel和ErrorMessageStrategy支持。一套AwsHeaders.RECEIVED_添加了标题，以避免当请求消息从上游AWS Inbound Channel Adapter到达时在出站侧造成混淆。将其与需要相同的入站和出站绑定的Spring Cloud Stream“处理器应用程序”一起使用时，这非常方便。的KinesisMessageDrivenChannelAdapter现在支持通过以下方式进行错误处理errorChannel和一个新的KinesisMessageHeaderErrorMessageStrategy提供有用AwsHeaders.RAW_RECORD中的信息ErrorMessage标头。
