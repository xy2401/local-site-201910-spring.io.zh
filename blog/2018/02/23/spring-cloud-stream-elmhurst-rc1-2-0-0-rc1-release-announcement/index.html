春季云流Elmhurst。RC1 / 2.0.0。RC1发布公告

工程
奥列格·朱拉库斯基（Oleg Zhurakousky）
2018年2月23日

经过漫长而令人兴奋的旅程，我们很高兴地宣布Spring Cloud Stream Elmhurst发布火车的第一个发布候选版本-Elmhurst。RC1 / 2.0.0。RC1。

春季云流Elmhurst 2.0.0。RC1可在Spring Milestone存储库中使用。发行说明包括有关与Spring Boot，Spring Cloud，Spring AMQP和Spring for Apache Kafka的版本兼容性的相关信息。

鉴于这是一个候选版本，以下部分简要概述了功能和改进，不仅包括此版本，还包括整个2.0版，并以即将发布的博客和文档更新的形式提供了详细信息，以准备2.0.0。发布。

新的功能

  • 轮询源可以替代事件驱动的消息使用。
  • 通过@StreamMessageConverterannotation的自定义MessageConverters。
  • New Test Binder-一个由Spring Integration支持的活页夹，用于支持在活页夹目标位置而不是其内部通道进行测试。
  • 用于绑定控制的执行器端点-不仅可以查看现有绑定，还可以管理其生命周期（即停止/启动等）
  • 千分尺，千分尺，千分尺！ -指标支持现在基于千分尺
  • Kafka Streams活页夹的完整改造-详细信息。

改进和增强

  • 改进且一致的内容类型协商
  • StreamListener基础架构增强功能可处理多个目的地
  • 可配置的RetryTemplate
  • 可配置的Web环境-默认情况下是可选的，但根据提供的启动启动器，用户可以选择使用反应性（Netty）还是常规（Tomcat）。
  • 分区现已配置为Spring-在需要围绕分区进行自定义时提供了更大的用户灵活性。
  • 执行器变得可选-
  • 初始化程序（start.spring.io）的改进-创建新的Spring Cloud Stream应用程序时改善了用户体验。

其他各种增强功能和错误修复
