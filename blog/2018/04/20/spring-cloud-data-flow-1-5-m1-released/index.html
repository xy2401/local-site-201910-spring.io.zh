Spring Cloud Data Flow 1.5 M1发布

发布
马克·波拉克
2018年4月20日

Spring Cloud Data Flow团队很高兴宣布发布1.5.0 M1 。遵循本地服务器 ， Cloud Foundry和Kubernetes的入门指南 。

以下是重点内容：

  • 用户界面改进

  • Spring Boot和Spring Cloud Stream 2.0支持

  • 组合任务的嵌套拆分

  • 指标收集器2.0 M1

  • Stream Application Starters达尔文M1发布火车

  • 支持部署到多个Kubernetes集群

用户界面改进

我们一直在继续改进仪表板的UI / UX。您会立即注意到整体更轻的设计。任务选项卡已被重写，以匹配其他选项卡的UX样式。流构建器视图包括许多优化，包括更好的表单验证和急切的错误报告。试试看！

还进行了大量的重构，以优化代码库并为将来的扩展和功能增加做准备。

Spring Boot和Spring Cloud Stream 2.0支持

现在，我们支持部署基于Spring Boot和Spring Cloud Stream 2.0的应用程序。Spring Cloud Stream Application Starters库中的一个小型实用程序类添加了Micrometer度量标准标记，以帮助识别所需监视后端中的流和应用程序。

组合任务和嵌套拆分

由于需求旺盛，此版本增加了DSL支持，以解释组合任务中的“嵌套拆分”。Flo Dashboard和Shell工具会自动适应嵌套拆分。

这是Flo仪表板中DSL表达式的外观：

<<extractFromFTP && cleanseFiles || extractFromS3 && splitTransform> && merge || extractfromOracle>

要使用此功能，您必须注册1.1.1.BUILD-SNAPSHOT SCDF中的组合任务运行程序的版本。

对于Maven用户

maven://org.springframework.cloud.task.app:composedtaskrunner-task:1.1.1.BUILD-SNAPSHOT

对于Docker用户

docker:springcloudtask/composedtaskrunner-task:latest

指标收集器2.0 M1

Metrics Collector的2.0 M1版本基于Spring Boot 2.0和Spring Cloud Stream 2.0。Metrics Collector服务器支持从仅包含Boot 1.x或2.x应用程序的流以及包含多个Boot版本的流中收集度量。将捕获吞吐率的一致表示，并将其传播到SCDF的仪表板。一个示例，该示例在Micrometer的帮助下展示了全面的指标支持，并且一些受支持的后端正在开发中-请继续关注！

流应用启动程序Darwin

发行火车Darwin将应用程序启动程序更新为基于Spring Boot和Spring Cloud Stream 2.0。一个gRPC处理器已添加。导入URL可以在这里找到。

支持部署到多个Kubernetes集群

在将Skipper与Data Flow结合使用时，已经可以将应用程序部署定向到多个平台后端。但是，我们不支持针对多个Kubernetes平台。现在你可以。 :)

其他小物件

越来越多的新问题涉及单独和全局覆盖的能力JAVA_OPTS适用于在Cloud Foundry上运行的应用程序。我们添加了一个部署器属性，例如deployer.yourapp.cloudfoundry.javaOpts支持设置此特定环境变量。

Kubernetes服务器现在支持按应用程序使用私有Docker注册表 。
