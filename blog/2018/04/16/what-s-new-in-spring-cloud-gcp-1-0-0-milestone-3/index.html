Spring Cloud GCP 1.0.0 Milestone 3的新增功能

发布
阿尔特姆·比兰（Artem Bilan）
2018年4月16日

亲爱的 Spring 社区！

Spring Cloud GCP是一组Spring Boot启动器，旨在帮助您直接从Spring Boot利用Google Cloud Platform托管的服务，从而使您可以将Spring Messaging用于Cloud Pub / Sub，将Spring Resource用于Cloud Storage，将Spring Cloud Sleuth用于Stackdriver Trace，带有Cloud SQL的Spring数据等。

我们刚刚发布了Spring Cloud GCP 1.0.0.M3 。查看自上一个里程碑2以来的新功能！

Spring Boot 2.0 GA支持

Spring Cloud GCP 1.0.0.M3现在可以与Spring Boot一起使用2.0.1.RELEASE和Spring Cloud Finchley.M9 。

使用Spring Cloud GCP 1.0.0.M3 ，只需在您的Spring Boot中导入BOM表2.0项目：


    
        
            org.springframework.cloud
            spring-cloud-gcp-dependencies
            1.0.0.M3
            pom
            import

Spring Initializr

现在，您可以在Spring Initializr中找到Spring Cloud GCP！转到http://start.spring.io/并查找与“ GCP”相关的启动器。

例如，要将Cloud Pub / Sub用于消息传递，只需在Initializr中添加GCP消息传递依赖项。该入门程序还包含Spring Integration通道适配器以及Spring Cloud Stream Pub / Sub活页夹。

当前，Initializr菜单中仅提供基本支持，消息传递和存储。请参阅《 Spring Cloud GCP参考》文档以添加其他启动器。我们正在努力为Initializr添加更多功能。

Spring数据扳手

现在，您可以构建由Google Cloud Spanner支持的多区域Spring Boot应用程序-全球分布的，高度一致的关系数据库。使用Spring Data Spanner ，您现在可以注释POJO，以将对象映射到Spanner表。它支持：

  • 扳手交易

  • 分页和排序存储库

  • REST资料库

  • 通过方法名称约定，注释和命名查询进行查询！

@Table(name = "traders")
public class Trader {
        @PrimaryKey
        @Column(name = "trader_id")
        String traderId;

        String firstName;

        String lastName;
}

public interface TraderRepository
                      extends PagingAndSortingRepository {
}

Pivotal Cloud Foundry服务经纪人

如果您使用的是Pivotal Cloud Foundry和Google Cloud Platform Service Broker ，则Spring Boot GCP Starters可以自动使用GCP Service Broker绑定和凭据来配置您的应用程序。支持以下服务：

  • 云储存

  • 云发布/订阅

  • 云扳手

  • 堆栈驱动程序跟踪

如果您不熟悉Cloud Foundry上的Spring Boot ，请查阅Spring Boot Cloud Foundry部署文档。

例如，您可以通过Service Broker设置和绑定Stackdriver Trace服务凭据：

$ cf create-service google-stackdriver-trace default mytrace
$ cf bind-service myapp mytrace

在您的应用程序中，添加Spring Cloud GCP Trace启动器：


    org.springframework.cloud
    spring-cloud-gcp-starter-trace

当您通过此服务绑定将应用程序部署到PCF中时，启动程序将自动从VCAP_SERVICES环境变量。Spring Cloud Trace启动器将自动将分布式跟踪数据转发到Stackdriver Trace。

$ cf push myapp -p path/to/my.jar

Stackdriver Trace是Stackdriver Application Performance Management（APM）工具的一部分。从Stackdriver APM公告博客中了解更多信息。

试试看！

在Spring Cloud GCP GitHub存储库中查看代码示例 ，阅读Spring Cloud GCP参考文档了解更多详细信息，并在代码实验室上进行尝试。最重要的是，让我们知道您的反馈并报告任何问题。
