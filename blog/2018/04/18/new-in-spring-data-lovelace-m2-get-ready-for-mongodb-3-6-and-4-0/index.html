Spring Data Lovelace M2的新功能-准备好使用MongoDB 3.6和4.0。

工程
克里斯多夫·斯特罗布尔
2018年4月18日

在最新的Spring Data Lovelace Milestone 2版本中，MongoDB模块积累了不久将以您的方式出现的新功能。正如您可能在新闻中看到的那样， MongoDB 4将把ACID事务带入文档存储。最新的MongoDB 3.6服务器版本已经随附了这些客户端会话的主要构建块。

您可以在MongoDB参考中阅读有关隔离级别和因果一致性的所有信息。简而言之，会话使您可以按照尊重因果关系的顺序执行操作。

使用Spring Data MongoDB， ClientSession无论是当务之急还是被动的世界，触手可及的地方都已触手可及，因为我们已将它们整合到已经存在的世界中MongoOperations和ReactiveMongoOperations 。为了提供最大的控制权和足够的便利， ClientSession生命周期由您决定，而模板负责将会话正确地传递给驱动程序。以下示例显示了如何创建ClientSession ：

ClientSessionOptions sessionOptions = ClientSessionOptions.builder()
    .causallyConsistent(true)
    .build();

ClientSession session = client.startSession(sessionOptions);

从MongoDB Java驱动程序中获取会话后，只需将其添加到模板中即可检索会话绑定的操作，然后close()会话完成后，如以下示例所示：

MongoOperations sessionBound = template.withSession(session);

Query query = query(where("name").is("Durzo Blint")
Person durzo = sessionBound.findOne(query), Person.class);

Person kylar = new Person("Kylar Stern");
kylar.setMaster(durzo);

sessionBound.insert(kylar);

session.close();

使用反应式编程模型，上面的示例看起来有些不同，因为我们将要获得一个会话Publisher并希望确保在实际订阅之前不要解决它。
因此，与其获得会话范围的ReactiveMongoTemplate ，我们得到了一个实例ReactiveSessionScoped拥有对会话的引用Publisher并提供执行和关闭句柄，以便我们确保关闭ClientSession在里面finally返回的块Flux ，如以下示例所示：

ClientSessionOptions sessionOptions = ClientSessionOptions.builder()
    .causallyConsistent(true)
    .build();

// obtain a session, but do not subscribe to it
Publisher<ClientSession> session = client.startSession(sessionOptions);

template.withSession(session)
    .execute(action -> {

        Query query = query(where("name").is("Durzo Blint"));
        return action.findOne(query, Person.class)
            .flatMap(durzo -> {

                Person kylar = new Person("Kylar Stern");
                kylar.setMaster(durzo);

                return action.insert(kylar);
            });
    }, ClientSession::close) // make sure we close the session when finished
    .subscribe();

到目前为止，还算不错，但是我们没有在本文的开头提到交易吗？

的确，事务将成为MongoDB 4.0的一部分，是的，我们将为此做好准备。如果您想遵循我们的努力来获得交易支持，请确保遵循发展 。快速浏览一下您可以期待的结果。

MongoTransactionManager将成为众所周知的事务支持的门户，允许应用程序使用Spring的托管事务功能。
的MongoTransactionManager将创建并绑定一个ClientSession到线程。在启动，提交或中止事务的同时MongoTemplate自动检测现有ClientSession并对其进行相应的操作，如以下示例所示：

@Configuration
public class Config extends AbstractMongoConfiguration {

  @Bean
  MongoTransactionManager transactionManager(MongoDbFactory dbFactory) {
    return new MongoTransactionManager(dbFactory);
  }

  // ...
}

@Service
public class NightAngel {

  @Autowired PersonRepository repository;

  @Transactional
  public void trainAzoth() {

    Person durzo = repository.findByName("Durzo Blint");

    Person kylar = new Person("Kylar Stern");
    kylar.setMaster(durzo);

    repository.save(kylar);    
  }
}
