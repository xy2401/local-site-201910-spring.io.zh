Spring 的这个星期-2018年4月17日

工程
乔什·朗（Josh Long）
2018年4月17日

嗨，春季粉丝！欢迎来到春季本周的另一期！本周我将在法国巴黎，下周将赴德国美因茨参加DevOne活动的JAX和奥地利林茨。如果您在这些地方中的任何一个，那就不要犹豫，伸出手 ！

我们有很多内容需要介绍，让我们开始吧！

  • Spring Cloud Task和Spring Batch 负责人Michael Minella刚刚宣布了Spring Cloud Task 2.0.0。RC1
  • Artem Bilan着眼于Spring Cloud GCP 1.0.0的新功能。M3发布 。
  • Spring Tool Suite负责人Martin Lippert刚刚宣布了Spring Tool Suite 3.9.4
  • Spring Boot负责人Phil Webb 正在准备一个非常有趣的项目，以确保所有Spring Boot项目的代码格式一致性 。有趣！（而且还算是超级早期。。。）可以认为这与Java和Spring Boot等效gofmt 。
  • Spring Data忍者Mark Paluch刚刚宣布了Spring Data Lovelace M2 。此版本随附所有商店模块的更新，其中包含新功能，改进和错误修复。值得注意的更改包括：MongoDB 3.6支持变更流，$ jsonSchema和客户端会话（为MongoDB事务做准备）；针对Map和Tuple类型的Apache Cassandra映射改进，以及新引入的Lifecycle Callbacks；升级到Apache Solr 7；改进了连接处理，以响应式Redis和静态主/从方式使用；对Redis Cluster的SCAN支持；列出对Spring Data JDBC的支持；通过CDI支持可组合存储库；当我们从SolrJ 6.6.2升级到7.2.1时，将t递增到Spring Data Solr到新的主版本号4.0 M2。
  • 程序化bean定义Kotlin DSL正在进行一些有趣的更改！ 新的DSL支持注入多个bean定义并访问Spring Environment ， 除其他事项外。
  • Springg Security OAuth负责人Joe Grandja刚刚宣布了Security OAuth 2.3.2。释放与Redis的运行时不兼容问题
  • 我们自己的雅库布·皮利蒙（Jakub Pilimon） 在“具有大量领域驱动设计的事件风暴”一文中有一篇不错的文章。
  • Spring Data忍者Mark Paluch分享了有关Spring Data新版本的详细信息-Spring Data Ingalls SR11和Kay SR-包括针对最近宣布的漏洞的修复程序。如果您使用的是Spring Boot 1.5.12和2.0.1，那么它们也会带来相应的Spring Data版本
  • 嗨，春季粉丝！上周，在最新的Spring Tips系列中，我介绍了反应式Cloud Foundry Java客户端
  • 这是2018年Spring开发人员必读书籍的清单。
  • Jenkins X是CloudBees的一项新产品，支持智能连续交付应用程序。在此示例中，它为Spring Boot提供了集成的CI / CD管道
  • Flowbable BPMN引擎在其获得Spring Boot 2.0支持的过程中有一篇不错的文章
  • 这个新项目-Codecentric的Spring Boot的Chaos Monkey-支持基于Spring Boot的服务中的混沌工程原理。它可以帮助您回答测试无法解决的问题，例如：“我们的后备工作会奏效吗？”; “应用程序如何应对网络延迟？”; “如果我们的一项服务中断了怎么办？”;和“服务发现有效，但是我们的客户端负载平衡也有效吗？”如果Spring Boot Chaos Monkey在您的类路径上并使用配置文件名称激活chaos-monkey ，它将自动在您的应用程序中扫描所有带有以下任何Spring注释的类： @Controller ， @RestController ， @Service和@Repository 。然后，您配置不同的“攻击”，这将攻击您的组件并尝试破坏它们。太酷了！
