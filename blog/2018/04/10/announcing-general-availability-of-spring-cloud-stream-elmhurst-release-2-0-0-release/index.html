经过漫长而激动人心的旅程，我们很高兴地宣布Spring Cloud Stream Elmhurst发布火车-Elmhurst的General Availability版本。发布/2.0.0。发布。

新功能，增强功能和改进功能

虽然“ 2.0新增功能”中提供了更多详细信息？ 在更新的参考指南的“部分”中，我们想列举一些项目。

改进的内容类型协商和消息转换

Spring Cloud Stream 2.0包括对内容类型协商功能的全面改进，以解决性能，灵活性以及最重要的一致性问题。最近发布的博客谈到了已完成的工作，期望的工作以及对您有帮助的关键点。
有关更多信息，请参见参考指南的“ 内容类型协商”部分。

用户定义的消息转换器

如上一节所述，简化内容类型协商和消息转换的工作量很大。这项工作的结果是，我们现在支持用户定义的消息转换器（通过Spring的标准org.springframework.messaging.converter.MessageConverters ）注册，以防提供的消息转换功能不够用。您可以使用以下方法注册自定义消息转换器@StreamMessageConverter ， 如下所示：

@Bean
@StreamMessageConverter
public FooBarMessageConverter fooBarMessageConverter() {
	return new FooBarMessageConverter(MimeType.valueOf("foo/bar"));
}

public static class FooBarMessageConverter extends AbstractMessageConverter {
. . .
}

可以在参考指南的“ 用户定义的消息转换器”部分中找到更多信息。

引入轮询消费者

轮询消费者功能使您可以控制消息使用的速率， Gary Russell最近发布的博客提供了其他信息，可以将其简化为两个简单步骤：

  1. 定义一个PollableMessageSource绑定：

    @Input
PollableMessageSource inputSource();

  2. 开始在您的应用程序中使用它：

    @Autowire
PollableMessageSource inputSource;
. . .
inputSource.poll(System.out::println);

可在参考指南的“ 使用轮询的使用者”部分中找到更多信息。

执行器和Web现在是可选的

为了简化封装并在选择Web框架（命令式和Webflux）方面提供更大的灵活性，我们将Spring Boot Actuator和Web都设置为可选依赖项，并且默认情况下不包括它们。
但是，如果需要，可以通过添加以下依赖项元素来将它们添加为应用程序依赖项：


     org.springframework.boot
     spring-boot-starter-web


    org.springframework.boot
    spring-boot-starter-actuator

新的绑定执行器端点

现在，您可以使用新的执行器端点可视化以及控制绑定。

设置management.endpoints.web.exposure.include=bindings引导属性，并通过将它们各自的启动器作为依赖项包含在项目中来启用Web框架和执行器之一（如Actuator和Web Are Now所述 ）

启动您的应用程序并访问以下URL以可视化当前绑定： http：// localhost：8080 / actuator / bindings

从该URL，您可以收到类似于以下内容的输出：

您还可以可视化单个绑定并控制其生命周期（例如停止，开始，暂停和恢复）。可以在用户指南的“ 绑定可视化和控制”部分中找到更多详细信息。

千分尺支持

从Boot 2.0开始，指标支持一直基于Micrometer 。
在Micrometer捕获应用程序指标的同时，Spring Cloud Stream提供了将这些应用程序指标发布到预定义目标（例如Rabbit交换或Kafka主题）的支持。
通过设置spring.cloud.stream.bindings.applicationMetrics.destination=myMetricDestination属性，您正在请求将度量标准消息发布到myMetricDestination 。
发布的消息采用JSON格式，看起来类似于以下内容：

{
  "name": "application",
  "createdTime": "2018-03-23T14:48:12.700Z",
  "properties": {
  },
  "metrics": [
    {

        "tags": [

此信息的使用者之一是Spring Cloud Dataflow及其UI，它显示消息速率以及其他指标数据。有关更多详细信息，请参见参考指南的“ 度量标准发射器”部分。

使用Apache Kafka流进行流传输

我们自己的Soby Chacko所做的出色工作为使用Apache Kafka Streams构建流应用程序提供了增强的支持。值得一提的一些核心功能包括：
-Kafka Streams和基于Kafka通道的绑定之间的互操作性
-多种Kafka Streams类型（例如KStream和KTable ）作为可绑定的组件
-分支支持
-交互式查询支持
- 和更多…

就在我们完成这项新功能的同时，我们无情的同事和朋友Josh Long编写了一个快速演示/教程，演示了使用Kafka Streams和Spring Cloud Stream构建流应用程序的一些技巧。

新的和改进的快速入门指南和示例

这份新的快速入门指南经过专门设计，耗时不超过5分钟，并且几乎不需要或几乎不需要Spring Cloud Stream的知识或经验。它还展示了一些与Spring Cloud Stream 2.0和Spring Boot 2.0相关的Spring Initializr更新。

一旦适应了核心概念，就可以随意冒险并使用我们广泛且不断发展的样本库，这些样本库展示了常规以及反应式流应用程序。

如果您想了解Spring Cloud Stream在Spring Cloud Dataflow中定向数据管道的构建模块中的作用，我们强烈建议克里斯蒂安·特佐洛夫（Christian Tzolov）推荐使用Spring Cloud Stream进行实时对象检测 。
