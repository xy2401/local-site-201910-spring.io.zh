适用于Apache Geode和Pivotal GemFire 1.0.0的Spring Boot。M1发布！

工程
约翰·布鲁姆
六月26，2018

春季问候和Apache Geode / Pivotal GemFire社区-

我很高兴地宣布适用于Apache Geode和Pivotal GemFire的Spring Boot的第一个里程碑版本是1.0.0。M1。

这是一个伟大的新项目，建立在将近两年前在Spring Data中为Apache Geode / Pivotal GemFire启动的工作之上，当时我们引入了基于注解的配置 。

使用自动配置的 Spring Boot的“ 约定优于配置 ”方法，该项目可以将Spring Boot，Apache Geode和Pivotal GemFire应用程序的配置和开发提高到一个新水平，无论是在独立环境还是在托管环境中构建和运行应用程序使用Pivotal Cloud Cache（PCC）时，请使用Pivotal CloudFoundry（PCF ） 。

包含什么

该项目包括对以下功能的自动配置支持：

  1. 后备缓存使用Spring的缓存抽象

  2. 使用Spring数据存储库和Spring事务管理 的记录系统（SOR）

  3. 使用函数实现和执行的分布式计算

  4. 使用连续查询的事件流处理（ESP）

  5. 使用PDX的域对象版本控制/序列化

  6. 安全

    1. 认证/授权

    2. 使用SSL的 TLS。

下一步是什么

里程碑2将使用Apache Geode或Pivotal GemFire来管理您的（HTTP）会话状态， 从而为Spring Session添加自动配置支持 。

在里程碑3中，我们将开始探索自定义的Spring Boot运行状况指示器，以及对由Micrometer驱动的Spring Boot Actuator的支持。

要进一步了解该项目的方向和进展，请查看问题 。

现在就试试

可以从春季获得人工制品libs-milestone Maven资料库：

马文
<repositories>
    <repository>
        <id>spring-milestone</id>
        <url>https://repo.spring.io/libs-milestone</url>
    </repository>
</repositories>
摇篮
repositories {
    maven { url: 'https://repo.spring.io/libs-milestone' }
}

包括spring-geode-starter开发Spring Boot，Apache Geode应用程序或spring-gemfire-starter开发Spring Boot Pivotal GemFire应用程序时需要依赖。这些依赖项可以互换。

马文
<dependencies>
    <dependency>
        <groupId>org.springframework.geode</groupId>
        <artifactId>spring-geode-starter</artifactId>
        <version>1.0.0.M1</version>
    </dependency>
</dependencies
摇篮
dependencies {
  compile 'org.springframework.geode:spring-geode-starter:1.0.0.M1'
}
