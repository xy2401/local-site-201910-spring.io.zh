Spring Cloud Finchley。可以发布

发布
斯宾塞·吉布
六月19，2018

我很高兴代表社区宣布， Spring Cloud Finchley发行培训的一般可用性（RELEASE）现已开始提供。该版本可以在Maven Central中找到。您可以查看Finchley 发行说明以获取更多信息 。

发行培训版本的生命周期公告

卡姆登发布火车已经达到报废状态。Dalston发布列车将于2018年12月达到报废状态。Edgware发行培训将遵循Spring Boot 1.5.x产品线的生命周期终止。

Finchley发布火车的显着变化

Spring Cloud Function和Spring Cloud Gateway是Spring Cloud产品组合的新增功能。该产品组合还与Spring Boot 2.0.x兼容，这是一项巨大的努力。Spring Cloud现在具有Java 8的最低版本。

Spring Cloud Gateway

Spring Cloud Gateway是在Spring Webflux和Reactor Netty之上构建的API网关。它是下一代网关，可以看作是Spring Cloud Netflix Zuul的替代产品。它基于易于定义的谓词提供动态路由。它还提供了适用于每种路由的过滤器，示例包括：路径重写，断路器，添加或删除标头，速率限制和安全性。可以使用属性或包含的对以下内容的支持来定义路线DiscoveryClient （尤里卡，领事和动物园管理员）。

弹簧云功能

Spring Cloud Function是一个具有以下高级目标的项目：

  • 通过功能促进业务逻辑的实现。
  • 将业务逻辑的开发生命周期与任何特定的运行时目标脱钩，以便相同的代码可以作为Web端点，流处理器或任务运行。
  • 支持跨无服务器提供程序的统一编程模型，以及独立运行（本地或在PaaS中）的能力。
  • 在无服务器提供程序上启用Spring Boot功能（自动配置，依赖项注入，指标）。

要开始使用带有BOM的Maven（仅限于依赖管理）：


<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>org.springframework.cloud</groupId>
            <artifactId>spring-cloud-dependencies</artifactId>
            <version>Finchley.RELEASE</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
<dependencies>
    <dependency>
        <groupId>org.springframework.cloud</groupId>
        <artifactId>spring-cloud-starter-config</artifactId>
    </dependency>
    <dependency>
        <groupId>org.springframework.cloud</groupId>
        <artifactId>spring-cloud-starter-netflix-eureka-client</artifactId>
    </dependency>
    ...
</dependencies>

或使用Gradle：

buildscript {
    dependencies {
        classpath "io.spring.gradle:dependency-management-plugin:1.0.5.RELEASE"
    }
}
