响应式 Spring Cloud 之路

工程
乔什·朗（Josh Long）
2018年6月20日

Spring Cloud Finchley GA的新版本中塞满了好东西，代表了向反应式微服务迈进的重要里程碑。我无法列出所有内容，因此请您参考Spencer Gibb的最新烤箱公告。取而代之的是，在本文中，我想重点介绍我们在响应式 Spring Cloud方面所采取的道路。

我们于2017年9月发布了Spring Framework 5 。这是引入新的Reactive编程支持以帮助构建更强大，可扩展的服务的第一个版本。它建立在Pivotal Reactor项目的基础上，该项目是我们的反应式流兼容的反应式运行时。Spring Framework 5还包含了大量的新功能，我也不会尝试全部列出它们，而是选择在此处重点关注响应式支持。什么是反应式编程？为什么这有关系？好吧，当您构建网络服务时，这很重要。

简而言之，Spring中的服务集成基础已得到更新，以完全包含响应式编程。那么，什么是“反应式编程”？响应式编程是一种认识，即一旦您开始通过网络处理更多数据，并通过API调用使IO缓冲区饱和，您就会在执行IO的任何给定请求中花费更多时间。

IO本身并不是问题。IO传统上是阻塞的-线程必须等待InputStream产生新的byte 。 （通常在while循环中， read() byte s的缓冲区）。当线程等待时，不能将其重新用于其他用途。线程很贵！

考虑一下用Java或具有相同线程化方法的任何其他平台实现的传统服务器如何工作。如果您的Web服务器线程池中有100个线程，并且有101个请求到达，那么只有其他一个完成处理其请求后，该最后一个额外的请求才会得到处理。如果其他人可以在第101个请求到达之前完成（从而释放他们垄断的线程），那就太好了！可能不需要反应式编程。如果您可以比新请求到达更快地释放线程，并且花费在这些线程上的时间主要是由于输入/输出，那么就不需要反应式编程。

当您进入微服务，大数据和长期会话（例如在websocket和服务器发送的事件以及任何其他长期服务器端状态）的世界时，您将开始在线。

线程与IO的这种耦合是不必要的。数十年来，您的操作系统一直支持“后台” IO，并通知您何时参与其中。实际上， Java 1.4（从2000年代初开始）支持NIO（通道） ，这为我们提供了这种异步IO机制。

在这个世界上，某人管理着IO，并在需要时回叫代码。如果存在任何延迟，则该线程可以继续执行并处理其他请求。它没有阻止。而不是您的代码将字节从InputStream 拉出，而是将这些字节异步地推送到它。您可以有效地反转与数据源的交互。

许多项目，例如@NetflixOSS的RxJava，@ Pivotal的@ ProjectReactor，Eclipse的@vertx_project和@lightbend的@akkateam，都试图提供一种支持这种新的异步现实的编程模型。Reactive Streams规范具有共同点，而这些共同点正是这些项目支持的。

反应式流规范支持将项目发布给订阅者的Publisher类型。Subscribers在调用其onNextIT)方法时会消费商品。订阅者订阅时，会获得一个Subscription ，它可以用来表示可以处理多少条记录。最后一点是精确指定订户准备处理多少记录的能力，这就是流控制 。Publisher者不能压倒Subscriber 。这促进了稳定性。在反应式编程的情况下，流量控制称为背压 。

最后一个接口Processor是一个桥梁。它实现了Publisher和Subscriber 。Project Reactor支持两种Publisher专长： Flux发出0-N个项目， Mono发出单个项目或不发出任何项目。

这是对IO发生方式的根本性重新思考，因此需要在上面的每个层进行集成。在数据访问层，安全层，引导和微服务层中。

Spring Framework 5还包括一个称为Spring WebFlux的全新的响应式Web运行时（甚至支持Netty项目）。它甚至包括新的功能性反应性端点。早在2016年，我就在此视频中制作了Spring Tips视频！

Spring WebFlux建立在反应流规范的基础上，因此可以与任何其他支持库进行互操作。这是一个Spring Tips视频，其中我演示了如何将反应式Spring Webflux与Lightbend的Akka Streams（和Scala）结合使用。

首先，新的Spring WebFlux组件模型是反应式和异步的。它支持异步情况，例如websocket和服务器发送的事件，其方式与传统上处理同步情况的方式相同。您最终会遇到一种事情。是否想在几纳秒内发送带有10条记录的简短JSON节？使用Publisher ！想要产生服务器发送的事件这是有关服务器发送的事件的Spring Tips视频。

Spring Data Kay通过具有异步IO支持的数据访问技术，通过模板和存储库支持响应式数据访问。这是使用Reactive Spring Data MongoDB的示例。


interface ReservationRepository extends ReactiveMongoRepository<Reservation, String> {

		Flux<Reservation> findByEmail(String email);
}

@Document
@AllArgsConstructor
@NoArgsConstructor
@Data
class Reservation {
		@Id
		private String id;
		private String email;
}

Spring Boot 2将所有这些结合在一起，从而无论您选择使用Spring WebFlux还是Spring MVC，诸如构建REST端点，使用Actuator，管理安全性以及其他所有事情都“正常”。

从代码库更改的角度来看，这也意味着Spring Cloud团队需要进行很多摇摆工作，这就是此版本如此重要的原因。

新的发行版无缝地将反应式编程与现有问题紧密结合在一起：服务注册，发现，安全性，CDC（T）和测试，消息传递，微代理支持，断路器等等。让我们看一些例子。

您可以使用新的反应式WebClient并使用Spring Cloud的DiscoveryClient抽象支持的任何服务注册表（Netflix Eureka，Hashicorp Consul，Apache Zookeeper，Cloud Foundry等）来解析主机。


@Bean

然后，您可以使用该反应式，可WebClient服务注册的WebClient 。在以下示例中， reservation-service是在服务注册表中注册的服务，而不是实际的主机名。


Publisher<String> emails = client
	.get()
	.uri("http://reservation-service/reservations")
	.retrieve()
	.bodyToFlux(Reservation.class)
	.map(Reservation::getEmail);

Spring Cloud Gateway是我们全新的响应式API网关。它建立在Spring的响应式支持之上。毕竟，它的工作是执行客户对下游服务的请求。这是反应式编程的完美用例（和需求）。我也制作了一个Spring Tips视频。

这是一个使用Spring Cloud Gateway将来自:9999/proxy proxy的请求:9999/proxy到服务（通过服务注册表解析并实现负载平衡）和速率限制的示例。（注意：此配置可以存在于Spring Cloud Config Server中的（可刷新）配置中，或者实际上可以用于创建Flux 任何源中Flux ）

此示例将每个经过身份验证的用户限制为每秒100个请求。您不需要Spring Security即可使用网关，但是隐含了它的配置。

@Bean
RouteLocator gateway (RouteLocatorBuilder rlb, RedisRateLimiter rrl) {
  return rlb
    .routes()
    .route( spec ->
      spec
       .path("/rl")
       .flters( fs -> fs
         .requestRateLimiter( c -> c.setRateLimiter( this.redisRateLimiter() ))
         .setPath("/reservations")
       )
       .uri("lb://reservation-service/")
    )
    .build();
}


@Bean // 100 reqs per second, burstable to 150
RedisRateLimiter redisRateLimiter (){
  return new RedisRateLimiter(100, 150);
}

Spring Cloud Function是我们新的功能即服务抽象。它可以将普通功能转换为不同的功能即服务运行时所需的类型。它可以在AWS Lambda，Microsoft Azure以及我们自己的Project Riff中使用 。Project Riff是Apache 2许可的基于Kubernetes的多语言功能即服务运行时。我还制作了有关Spring Cloud Function和Project Riff的Spring Tips视频。

使用它再简单不过了！您需要创建java.util.function.Function 实例。在这种情况下， I和O都可以是Publisher s！

package com.example.uppercase;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.context.annotation.Bean;
import reactor.core.publisher.Flux;

import java.util.function.Function;

@SpringBootApplication
public class UppercaseApplication {

		@Bean
		Function<Flux<String>, Flux<String>> uppercase() {
				return incoming -> incoming.map(String::toUpperCase);
		}

		public static void main(String[] args) {
				SpringApplication.run(UppercaseApplication.class, args);
		}
}
