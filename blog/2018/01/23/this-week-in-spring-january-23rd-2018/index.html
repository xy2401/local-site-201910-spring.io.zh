Spring 的这个星期-2018年1月23日

工程
乔什·朗（Josh Long）
一月23，2018

嗨， Spring 粉丝！欢迎来到 Spring 本周的另一期。这周我在洛杉矶（温暖！）与客户交谈，然后去芝加哥（不温暖！）。本周有很多好东西要报道，让我们开始吧！

  • 在PCF 2.0中，应用程序运行时现在由企业级Kubernetes服务（称为Pivotal容器服务）和新的“功能即服务”产品Pivotal Function Service组成。Pivotal的Jared Ruckle与Jeff和Dormain一同在此新播客中了解PCF 2.0
  • Spring Cloud Sleuth现在与Brave集成在一起 ，这是分布式跟踪的激动人心的时刻。
  • Spring Cloud Vault负责人Mark Paluch 已将Spring Security与Spring Cloud Vault集成
  • Spring Cloud Sleuth团队忙得不可开交！该提交使Spring Cloud Sleuth OpenTracing兼容
  • 支持将测微表（认为是SLF4J，但用于度量标准）与Jersey 2（JAX-RS实现）集成在一起 ！
  • 适用于无服务器的AWS新版本的Java容器支持激活Spring配置文件 。无论如何，这很酷。但是使用Spring Cloud Function和AWS适配器 ;这是更清洁的集成。
  • Spring Boot 在本周的“ Azure Fridays”中支持VS Code 功能
  • 这是构建微服务和云本机应用程序的人们应该读的十本书的有趣清单 。我很高兴这本书名列《 Cloud Native Java 》。
  • PawełUrban撰写了一篇不错的文章，介绍如何使用Logstash通过电子邮件接收Spring Boot错误警报
  • 到目前为止， Spring Boot主管Phillip Webb在推特上发布了有关Spring Boot 2.0令人兴奋的进展的推文 ：迄今已有17周的工作时间，6099次提交和1153个问题得到解决！哇！
  • 这是Java的服务加载器机制的很好看，尤其是Java 9模块中的服务加载器机制是如何工作的。顺便说一下，它还研究了Spring本身如何在类路径上进行组件扫描。
  • Spring 测试负责人和JUnit项目负责人Sam Brannen在即将发布的#JUnit 5.1版本中刚刚通过@RegisterExtension首次支持编程式扩展注册 ！这对于诸如Spring REST Docs之类的Spring项目中的未来集成是一个好兆头。对于应用程序开发人员而言，JUnit 5是令人兴奋的愿景。通过观看Sam的JavaOne 2017演讲来了解有关它的更多信息
  • 这并不是专门与Spring本身相关的， 而是有趣的观察即将发布的JDK 10中不可变和不可修改之间的区别 。看看这个！
  • 这是个老家伙， 但又是个好东西 ：它着眼于何时选择Pivotal Cloud Foundry和/或Kubernetes。
  • Richard Seroter和Jeff Hammond在SpringOne Platform 2017上的演讲介绍了Pivotal Cloud Foundry 2.0
  • Spring Data团队为Spring的JQAssistant项目插件做出了许多小的改进 。JQAssistant支持扫描和验证基于Java的项目。这种新的支持看起来很有希望。
  • 您的监视体系结构应包括什么？Cloud Foundry在《 2018 Dzone #DevOps指南》中对此进行了讨论
  • 休眠ORM 5.3.0。Beta1已发布，支持#JPA 2.2。
  • Mark Pollack博士刚刚宣布了Spring Cloud Skipper 1.0 RC1 。
  • Spring Cloud Contract的波兰语版本看起来非常有趣
  • @pivotal发推文 ：“也许我们应该定期播放“ @starbuxman在哪里？”游戏在2018年？”确实，2018年开始运转，我将尽一切可能带来Spring。在这些城市找到我
  • Spring Cloud忍者Ryan Baxter刚刚宣布Spring Cloud Edgware Service Release 1（SR1）已发布。 。新版本与新版本包装在一起。检查他们！
  • 这篇介绍Spring Boot的文章被评为IBM DeveloperWorks 2017年度热门文章之一
  • Spring Data负责人Oliver Gierke庆祝了首次提交的十周年纪念 ，这些最终将成为Spring Data JPA。生日快乐Spring Data JPA（当时称为“ Hades”）！！
  • 项目Reactor负责人SimonBaslé引入了更好的错误处理，这是Reactor 3.2的一项即将发布的功能， 现已在里程碑版本中提供
  • 查看适用于Apache Geode和Pivotal Gemfire版本的新Spring Session ！
  • 在Linux Foundation提供的 免费 Cloud Foundry课程中接受有关Cloud Foundry的培训
