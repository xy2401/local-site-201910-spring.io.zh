Spring Session 2.0.0发布

发布
罗伯·温奇
一月16，2018

该帖子由VedranPavić撰写

我很高兴代表社区宣布Spring Session 2.0.0的发布。发布 。该版本通过2.0.0演变。M1 ， 2.0.0。M2 ， 2.0.0。M3 ， 2.0.0。M4 ， 2.0.0。M5 ， 2.0.0。RC1 ， 2.0.0。RC2和2.0.0。RELEASE ，关闭超过130个问题并拉取请求。

Spring Session 2.0的新增功能

您可以在参考的“新增功能2.0”部分中找到有关新增功能的重点 。有关详细信息，请参阅上面的变更日志链接。

要求

此发行版移至Java 8和Spring Framework 5.0作为基线要求。整个代码库现在基于Java 8源代码级别。

Spring WebFlux支持

在Spring Framework 5.0中引入反应式编程模型一直是2017年最大的故事之一，我们很高兴为管理Spring WebFlux的管理带来支持WebSession与Redis ReactiveSessionRepository 。

以熟悉的Spring Session方式，支持Redis WebSession支持与以下代码段一样简单：

@EnableRedisWebSession
public class WebSessionConfig {

    @Bean
    public LettuceConnectionFactory redisConnectionFactory() {
        return new LettuceConnectionFactory();
    }

}

Spring Boot用户将很高兴地了解到，自从2.0.0.M6里程碑发布 。

春季会议模块

新版本对项目的模块进行了一些重大更改，因为我们已基于存储库实现将Spring Session拆分为模块。你会找到：

<dependency>
    <groupId>org.springframework.session</groupId>
    <artifactId>spring-session-core</artifactId>
    <version>2.0.0.RELEASE</version>
</dependency>
<dependency>
    <groupId>org.springframework.session</groupId>
    <artifactId>spring-session-data-redis</artifactId>
    <version>2.0.0.RELEASE</version>
</dependency>
<dependency>
    <groupId>org.springframework.session</groupId>
    <artifactId>spring-session-hazelcast</artifactId>
    <version>2.0.0.RELEASE</version>
</dependency>
<dependency>
    <groupId>org.springframework.session</groupId>
    <artifactId>spring-session-jdbc</artifactId>
    <version>2.0.0.RELEASE</version>
</dependency>

作为此更改的一部分，我们从主存储库中删除了一些Spring Session实现。我们现在有Spring Session Data Geode（GemFire）和Spring Session Data MongoDB的子项目。你会找到：

<dependency>
    <groupId>org.springframework.session</groupId>
    <artifactId>spring-session-data-gemfire</artifactId>
    <version>2.0.0.RELEASE</version>
</dependency>
<dependency>
    <groupId>org.springframework.session</groupId>
    <artifactId>spring-session-data-geode</artifactId>
    <version>2.0.0.RELEASE</version>
</dependency>
<dependency>
    <groupId>org.springframework.session</groupId>
    <artifactId>spring-session-data-mongodb</artifactId>
    <version>2.0.0.RELEASE</version>
</dependency>

我们的目标是让核心的Spring Session团队专注于提供新功能，而不需要了解每个数据存储的来龙去脉。这样一来，其他模块的开发就可以完成，而无需来自核心Spring Session团队的审阅工作。
