反应堆铋-SR5出炉了！

工程
斯蒂芬·马尔蒂尼（Stephane Maldini）
一月30，2018

改进，新功能和修补程序降落在您附近的Maven Central镜子中， Bismuth-SR5 材料清单 。Spring Framework 5.0.3和后续版本现已使用此版本Spring Boot 2.0.RC1 ！我们的网站projectreactor.io已更新为最新版本。

堆芯3.1.3

发行说明

质量更新，包括十多个修复程序和几个新功能：新Flux#delaySequence和Signal#getContext访问当前流Context 。

reactor-test还欢迎新功能，包括Context验证设施和StepVerifier#toString实施。

我们现在正在努力3.1.4.RELEASE 。看看我们积压的问题 。

反应堆插件3.1.4

发行说明

在此版本中，我们改进了我们的Retry和Repeat设施计时处理。
我们还推出了首批新的高级缓存帮助器，用于Flux和Mono 。它们将允许轻松的存储实现选择，以及为消耗用户提供预期的行为。

以下是使用任意哈希映射的快速摘要，该示例演示了上游流只订阅一次，而不管多重验证如何：

Map<String, List> data = new HashMap<>();

Flux<Integer> cached = CacheFlux.lookup(data, "foo", Integer.class)
		     .onCacheMissResume(Flux.just(1, 2, 3).log());

StepVerifier.create(cached)
             .expectNext(1, 2, 3)
             .verifyComplete();
//prints 1, 2, 3

StepVerifier.create(cached)
             .expectNext(1, 2, 3)
             .verifyComplete();
//prints nothing

我们现在正在努力3.1.5.RELEASE ，并邀请我们的社区提交有关更多扩展程序或运营商的想法。Oleg Dokuka已经就Flux的IO扩展有了另一个出色的PR

反应堆净值0.7.3

发行说明

一个推荐的更新，当从与Netty自己的线程之一不同的线程发布流主体时，修复了竞争条件。我们现在也支持KQueue作为替代Epoll 。也HttpClient已更新，以在HTTP响应过早完成时发出错误。

我们现在正在努力0.7.4.RELEASE和以下0.8.0 。请注意，下一个主要版本将带来重要的API更改，因此，我们强烈建议您使用适应层而不是公开reactor-netty直接使用API。或简单地使用Spring Framework在客户端和服务器端都做到了这一点。
