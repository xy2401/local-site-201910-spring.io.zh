Spring Security OAuth 2.0.3现在可用

发布
戴夫·瑟
2014年9月1日

Spring Security OAuth 2.0.3现在在所有常规的Maven存储库中都可用。这是一个错误修复版本，没什么大不了的，因此从2.0.x升级应该很轻松（建议这样做）。有些人在让JWT令牌正常工作方面遇到问题，这些问题应该得到解决。唯一值得注意的功能更改是资源服务器（如果配置了@Configuration ）现在将在允许访问受保护资源之前检查客户端和范围的有效性。这意味着可以快速撤消客户端特权，但也可能导致性能下降（因此，缓存ClientDetailsService结果将是推荐的）。

介绍

以前，应用程序可以使用Spring Security在WebSocket应用程序中执行身份验证。之所以有效，是因为Principal的HttpServletRequest将传播到WebSocket会话。

问题是授权仅限于握手。这意味着一旦建立连接，就无法为WebSocket应用程序的授权提供任何粒度。

CVE-2014-3527在Spring Security 3.2.5和3.1.7中修复

发布
罗伯·温奇
2014年8月15日

Spring Security 3.2.5（ 更改日志 ）和3.1.7（ 更改日志 ）已发布，并且在Maven Central中可用。此版本的重要亮点是：

  • 此版本包含针对CVE-2014-3527的修复程序，该修复程序解决了恶意CAS服务在使用代理票证时可能冒充另一个CAS Service的问题。
  • 此版本将cas模块的传递依赖关系更新为cas-client-core，该组件具有针对CVE-2014-4172的修复程序。这个问题不在Spring Security本身中，而是它所依赖的库中。
