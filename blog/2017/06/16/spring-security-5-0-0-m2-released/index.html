Spring Security 5.0.0 M2发布

发布
罗伯·温奇
2017年6月16日

我很高兴代表社区宣布Spring Security 5.0.0 M2的发布。该发行版包含错误修复，新功能，并且基于Spring Framework 5.0.0 RC2。

在Spring Security示例中的hellowebflux和hellowebfluxfn可以找到使用Spring Security保护Spring WebFlux应用程序的完整示例。

该版本的重点包括：

  • 简化的反应式安全配置

  • WebTestClient支持

简化的反应式安全配置

现在，设置最小的反应式安全配置非常容易。加@EnableWebFluxSecurity并提供一个UserDetailsRepository

@EnableWebFluxSecurity
public class SecurityConfig {
    @Bean
    public UserDetailsRepository userDetailsRepository() {
        return new MapUserDetailsRepository(User.withUsername("user")
            .password("password")
            .roles("USER")
            .build()
        );
    }
}

我们加了UserDetailsManagerResourceFactoryBean提供了一种简单的方法来创建UserDetailsRepository使用属性文件。例如：

@Bean
public UserDetailsRepositoryResourceFactoryBean userDetailsService() {
    return UserDetailsRepositoryResourceFactoryBean
                  .fromResourceLocation("classpath:users.properties");
}

和users.properties

user=password,ROLE_USER
admin=password,ROLE_USER,ROLE_ADMIN
注意
 对于非电抗性应用，有一个新的UserDetailsManagerResourceFactoryBean创建一个InMemoryUserDetailsManager从属性文件

WebTestClient支持

此版本还增加了对WebTestClient 。例如，您可以使用以下身份以测试用户身份运行：

ExchangeMutatorWebFilter exchangeMutator = new ExchangeMutatorWebFilter();
WebTestClient mockRest = WebTestClient.bindToApplicationContext(this.context)
    .webFilter(exchangeMutator) // add the exchangeMutator
    .build();

mockRest
        // run only this request with a mocked user
    .filter(exchangeMutator.perClient(withUser()))
    .get()
    .uri("/principal")
    .exchange()
    .expectStatus().isOk();
