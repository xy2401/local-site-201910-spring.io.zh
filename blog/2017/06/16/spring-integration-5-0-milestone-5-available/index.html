Spring Integration 5.0 Milestone 5可用

发布
阿尔特姆·比兰（Artem Bilan）
2017年6月16日

我很荣幸地代表Spring Integration团队宣布Spring Integration 5.0版本的第五个里程碑（ 5.0.0.M5 ） 现在可用。

此发行版包含21个JIRA （以及一些GitHub问题），包括错误修复和许多新功能。自先前宣布里程碑3以来，M4和M5中的一些功能亮点：

  • 的Splitter现在可以处理Java Stream和反应堆Flux有效载荷。如果输出通道是ReactiveStreamsSubscribableChannel ，分裂支持背压。

  • 一种ErrorMessagePublisher与ErrorMessageStrategy已经引入以追求更好的错误处理体验，并具有针对ErrorMessage 。的MessageListenerContainer在Spring Kafka 2.0和Spring AMQP 2.0中提供了它们自己的ErrorMessageStrategy代表原始数据ErrorMessage错误处理流程。

  • 新的MockMessageHandler已被添加到Spring Integration Test框架中，以替换真正的`MessageHandler`用于单元测试：

MessageHandler mockMessageHandler =
         mockMessageHandler()
              .handleNextAndReply(m ->
                     m.getPayload().toString().toUpperCase());

this.mockIntegrationContext
      .substituteMessageHandlerFor("myServiceActivator", mockMessageHandler);

this.pojoServiceChannel.send(new GenericMessage<>("foo"));
receive = this.results.receive(10000);

assertEquals("FOO", receive.getPayload());

  • Java DSL现在通过以下方式提供了更灵活的路由器API： routeByException() ：

@Bean
public IntegrationFlow exceptionTypeRouteFlow() {
    return f -> f
        .routeByException(r -> r
          .channelMapping(IllegalArgumentException.class, "illegalArgumentChannel")
          .channelMapping(RuntimeException.class, "runtimeExceptionChannel")
          .subFlowMapping(MessageHandlingException.class,
                          sf -> sf.channel("messageHandlingExceptionChannel"))
          .defaultOutputChannel("exceptionRouterDefaultChannel"));
}

  • 定制的杰克逊ObjectMapper提供了意识到Message和MessageHeaders到JSON的序列化/反序列化。此功能在允许配置自定义序列化器/反序列化器的那些组件中很有用，例如RedisMessageStore ：

RedisMessageStore store = new RedisMessageStore(redisConnectionFactory);

ObjectMapper mapper = JacksonJsonUtils.messagingAwareMapper();

GenericJackson2JsonRedisSerializer serializer =
                      new GenericJackson2JsonRedisSerializer(mapper);
store.setValueSerializer(serializer);

消息和消息组将以JSON格式存储在Redis中。

还有这个JacksonJsonUtils.messagingAwareMapper()提供一个参数来配置trustedPackages为CVE-2017-4995漏洞提供保护。默认列表为：

java.util
java.lang
org.springframework.messaging.support
org.springframework.integration.support
org.springframework.integration.message
org.springframework.integration.store

可以配置* （星号），表示信任所有软件包。

  • 新的ThreadAffinityClientConnectionFactory被添加到TCP / IP模块以将客户端连接绑定到线程。

  • 的ReactiveChannel已重命名为FluxMessageChannel以更好地反映其实施情况。

我们要感谢一些社区成员对框架的持续积极贡献；扫描提交并给他们一些荣誉！

有关更改的完整列表5.0 ，另请参阅参考手册中的“新增功能”一章。
