Spring Cloud Data Flow 1.2 M3发布

发布
托马斯·里斯伯格
2017年3月22日

我很高兴代表团队宣布发布Spring Cloud Data Flow 1.2的第三个里程碑。

注意：开始使用此新版本的一个好方法是遵循项目页面上的版本列表 ，其中包括下载坐标和参考指南的链接。

1.2 M3版本的亮点：

伴随元数据工件

作为期待已久的功能的一部分，它可以改善对Shell和Dashboard的应用程序属性信息的访问，我们为Stream和Task应用程序引入了一个新的可选工件-我们称之为“伴随元数据工件” 。通过此功能，流和任务应用程序及其属性是基于Docker和基于Maven的应用程序工件的一等公民。

伴随元数据工件是一个轻量级的轻量级jar工件，其中包括属性元数据作为JSON，并且它是为每个现成的应用程序生成的。鉴于此工件的尺寸较小，我们在应用注册时会预取元数据文件。任何下游操作，包括app info : shell命令或仪表板上的加载属性对话框窗口，您将发现应用程序属性的填充效率更高，而不是等待整个uber-jar下载。

引入了具有此功能的新Maven插件 。任何自定义应用程序都可以使用此插件来生成伴随的元数据工件。

注意：向伴随元数据工件注册应用程序不是强制性的，我们仍然保留以前的经验，并且它将继续按预期运行。

环境信息端点

更新了“ / about” RESTful端点，以探测相应的运行时环境以获取详细信息，包括依赖项，版本，功能切换，平台版本等。该端点对于版本兼容性检查和错误报告很有用。可以从仪表板“关于”页面上的“显示详细信息”按钮获得相同的信息，并且可以从该详细信息页面将其复制到剪贴板。

部署程序属性解析改进

在之前的1.2中。在M1版本中，我们在DSL级别上区分了应用程序和部署程序属性的概念。在某些情况下，这种差异会对整体解析行为产生残留的副作用。重新分析了解析器的构造，并解决了不一致问题。

Shell的角色意识

在1.2 M2 / M3中，我们在仪表板中添加了RBAC的基础和支持。现在提供了对Shell进行角色感知的支持。

Spring Cloud Stream应用程序入门培根。RC1

Spring Cloud Stream App入门培根的下一部分。RC1版本已升级。此版本基于Spring Cloud Dalston。RC1和切尔西春季云流。RC1的改进和错误修复。与往常一样，我们为RabbitMQ，Apache Kafka 0.9和Apache Kafka 0.10活页夹实现生成了开箱即用的应用程序。您可以找到bit.ly链接，以从项目站点批量导入开箱即用的应用程序。

Spring Cloud Task App入门贝尔蒙特。M1

推荐的任务应用程序初学者是Spring Cloud Task App Starters Belmont的一部分。M1版本。您可以找到bit.ly链接，以从项目站点批量导入开箱即用的应用程序。

发行说明

请查看发行说明，以获取有关所有已解决问题的更多详细信息，以及指向伴随项目中已解决问题的链接。

下一个里程碑-1.2.0。RC1

进一步增强功能，以支持Docker工件作为Shell，DSL和Dashboard中的一等公民。

编排“批处理作业或任务的组成”的能力正在取得进步。支持Shell / UI的此功能的一组新的DSL原语正在进行中。

计划在即将发布的版本中对流管道进行度量和监视。正在研究单个流应用程序性能的能力，包括其发送/接收速率以及整个管道的聚合指标。

反馈很重要。请在StackOverflow和GitHub上与我们联系，以获取问题和功能要求。我们也欢迎您的贡献！我们对改善Spring Cloud Data Flow生态系统的任何帮助表示赞赏。
