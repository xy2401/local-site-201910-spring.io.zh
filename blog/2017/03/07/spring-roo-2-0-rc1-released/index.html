Spring Roo 2.0 RC1发布

工程
彼得·汉弗莱（Pieter Humphrey）
2017年3月7日

我们荣幸地代表Disid Corp宣布Spring Roo 2.0的第一个候选版本。这是第一个候选版本，从现在开始，我们将致力于社区检测到的改进和错误修复，直到我们能够发布最终版本。我们鼓励大家参与测试2.0.0的过程。RC1版本，并报告JIRA中可能的错误和改进。

RC1版本包括一些改进和Spring Roo 2.0.0中检测到的错误修复。M3版本。此外，它还包括一些重要的新功能：

  • 包括对现有命令的一些改进：
  • 改进了按需数据生成功能，可以在测试命令中使用它。
  • 改进的测试单元命令，现在可以为JPA实体创建功能齐全的测试。
  • 改进的测试集成命令，现在可以为JPA存储库和JSON / Thymeleaf控制器创建集成测试。
  • 改进了现场命令的可读性，消除了不必要的选项并使其他动态可见。
  • 改进的推入方法命令，可以区分名称相同但参数不同的方法。
  • 改进的存储库命令，使开发人员可以将投影用作默认返回类型。

  • 在生成的代码中包括一些改进：

  • 将Spring IO版本更新为Athens-RELEASE 。
  • 将Springlets库版本更新为1.2.0.RC1 。它提供了一些很棒的新功能，这些功能使Spring应用程序的开发更加容易。
  • 使用投影作为存储库方法的默认返回类型。
  • 改进的只读实体管理。
  • 包括新的JSONDeserializers和JSONMixin类。
  • 包括新的LinkFactories类，以轻松地从Thymeleaf获取URL。
  • 为所有生成的方法，构造函数和字段生成默认JavaDoc。
  • 已创建新的DataTables扩展，以对DataTables元素应用高级配置。此扩展是完全可扩展的，因此开发人员将能够轻松自定义其DataTable组件。
  • 添加datatables.mark.js在DataTables组件中标记过滤的文本。
  • 在视图层中添加了并发控制，如果用户要放弃所做的更改并重新加载记录或要应用更改，则可以回答用户。
  • HTML5改进并迁移到Thymeleaf 3。
  • 使用DataTables组件上的DynamicJasper生成报告，允许将数据导出到CSV，PDF和XLS。
  • 使用新增了实体可视化支持--entityFormatExpression和--entityFormatMessage在实体命令中。
  • 增加了对选择应显示详细信息的特定实体的那些视图的支持。
  • 为一对一的构图相关实体添加了新的可视化系统。
  • 添加了对Java枚举常量的多语言支持。
  • 增加了对多行选择和批量删除的支持。

  • 与往常一样，Spring Roo参考指南已更新为所有新功能。

有关新Spring Roo版本中所做更改的完整详细信息，请参阅Spring Roo 2.0.0。RC1发行说明 。

下载新的Spring Roo 2.0.0。RC1版本来自我们的项目页面 。
