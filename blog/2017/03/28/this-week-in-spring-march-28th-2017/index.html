Spring 的这个星期-2017年3月28日

工程
乔什·朗（Josh Long）
2017年3月28日

欢迎来到 Spring 本周的另一期！本周，我在内布拉斯加州的奥马哈，与客户交谈，今晚我将在由奥马哈JUG 和当地奥马哈Spring Users Group共同赞助的活动中作演讲， 谈论反应性Spring 。不要害羞，出来打个招呼 ！
*SébastienDeleuze刚刚介绍了1.0.0。M1 与Kotlin语言使用Project Reactor支持的扩展无关
* Spring REST Docs负责人Andy Wilkinson刚刚宣布了 Spring REST Docs 1.2.0。RC1。立即尝试！对于Spring REST Docs的一个很好的例子，向下滚动，您会从社区成员那里看到一个很好的例子。
* Spring Cloud Data Flow贡献者和全能人类Thomas Risberg
刚刚宣布了Spring Cloud Data Flow 1.2。M3
*上周，继续Spring Tips系列，我研究了将Spring Boot应用程序部署到Servlet容器的各种选项
*超越Spring Boot智能默认值？需要调试到下一个级别吗？加入Hybris的NicolasFränkel，在4月13日（星期四）带领我们进入Spring Boot的行列。
*周三与Michael Plod（InnoQ） 一起注册Spring Boot战略（域驱动）设计网络研讨会还为时不晚
Spring Cloud联合创始人Spencer Gibb再次出席会议，这次他正在从事一个名为Spring Cloud Gateway的新Gateway项目 。这个项目远未完成或最终完成，但现在是个获得成功，努力工作和提供反馈的好时机。斯宾塞一直在努力添加各种很酷的东西， 例如请求节流 。
*您还可以使用Netflix的Zuul做一些非常有趣的事情！如果您想了解更多信息， 请查看不断扩展的样本库
* Jenn Strater在此示例中展示了Spring RESTDocs ， 表现非常出色。
*最后！反对（但当然绝不绝对删除）Java的终结行为的建议
*我的培训课程Cloud Native Java现在可以在Safari上购买了。
*我喜欢Pivotal Conversations的最新一期， 该部分讨论了Pivotal和Google推出的CRE程序 ，以确保Google Cloud Platform上的Pivotal Cloud Foundry提供Google级别的可靠性
* Venkat Rangasamy 在“生产就绪型微服务”上有一篇有趣的文章，讨论但不关注Spring Boot
*我喜欢Federico Paparoni的这篇意大利语帖子，内容涉及使用Spring Boot Actuator进行度量
* Pivotal自己的Richard Seroter着眼于使用单个服务注册中心进行对话以协调.NET和基于Spring Cloud的微服务
*我也很喜欢Federico在意大利语中发布的有关通过Spring Data REST和Spring Data JPA支持CRUD的文章
