Spring Cloud Skipper 1.0 M1发布

发布
马克·波拉克
十月30，2017

我很高兴代表团队宣布Spring Cloud Skipper 1.0 M1的发布。

Skipper是一款轻巧的工具，可让您发现Spring Boot应用程序并在多个Cloud Platform上管理其生命周期。您可以独立使用Skipper或将其与Continuous Integration管道集成在一起，以帮助实施Continuous Deployment的实践。

Skipper 1.0 M1的主要功能包括：

  • 定义可以在其中部署Spring Boot应用程序的多个平台帐户。支持的平台是本地，Cloud Foundry和Kubernetes。
  • 在Mustache模板文件中替代变量，这些变量描述了如何将应用程序部署到平台。
  • 在软件包存储库中搜索现有应用程序。
  • 根据简单的蓝/绿工作流程升级/回滚软件包。
  • 存储已解析的模板文件（也称为“应用程序清单”）的历史记录，这些文件代表针对特定版本已部署到平台的内容的最终描述。
  • 通过独立的交互式Shell或Web API使用。

参考指南中的“ 入门”部分是开始踢轮胎的最佳位置。

该项目的诞生是由于在Spring Cloud Data Flow中提供了持续交付功能。在“数据流”中，创建的数据管道由两个或多个彼此独立发展的微服务应用程序组成。后来认识到，与其创建数据流中的这些功能，不如创建一个更通用的项目Spring Cloud Skipper，以便它也可以成为Spring Cloud Data Flow上下文之外的有用工具。

在短期内，Skipper的主要用例是通过Data Flow部署Spring Cloud Stream应用程序。但是，我们正在寻求支持Cloud Foundry清单的模板并以cf push样式进行部署。
