Spring Batch 4.0.0。M5现在可用

发布
迈克尔·米内拉
十月24，2017

我们很高兴地宣布Spring Batch 4.0.0。现在可以通过Github和Pivotal下载存储库使用M5。这个版本代表了Spring Batch 4.0版本的第五个里程碑。非常感谢对此版本做出的所有贡献。

什么是新的？

该里程碑是先前4.0.0里程碑中列出的工作的延续。具体更新包括：

  • 依赖关系更新
  • FunctionItemProcessor
  • 持续更新文档

依赖关系更新

随着Spring Boot 2依赖关系图的发展，我们一直在更新依赖关系以使其与它们保持一致。此回合包括更新以引入Spring Framework 5的GA版本以及Spring Data Kay。Spring Integration 5.0.0。M7也加入了此版本。

FunctionItemProcessor

随着Java 8和功能接口的引入（从java.util.function软件包），回顾一下这些接口如何映射到Spring Batch是有意义的。这三个关键界面包括Supplier ， Function和Consumer 。此版本提供了ItemProcessor及相关功能，以实现Function作为一个ItemProcessor 。一个例子可能看起来像这样：

...
@Bean
public Function<String, String> upperCaseFunction() {
	return o -> o.toUpperCase();
}

@Bean
public Step step1() {
	return this.stepBuilderFactory.get("step1")
				.<String, String>chunk(10)
				.reader(reader())
				.processor(upperCaseFunction())
				.writer(writer())
				.build();
}
...

请注意，在前面的示例中，实际的Function实现具有零Spring Batch依赖关系。我可以创建一个实际上没有实现的处理器实现ItemProcessor并在我的步骤[1]中将其用作处理器。

从现在开始到Spring Batch的GA，我们将开始研究其他功能接口及其对Spring Batch的适用性。

文档更新

自从Java配置成为规范以来，Spring Batch文档就没有进行大修。此版本开始了文档的过渡以解决此问题。您将在以下页面的顶部看到Java和XML之间的切换。该切换将指示如何配置该页面中的示例（分别通过基于Java的配置或基于XML的配置）。当前实现的页面是：

  1. 春季批处理介绍
  2. 批处理域
  3. 配置和运行作业
  4. 配置步骤
  5. 读者和作家

在下一个版本发布之前，我们将完成此迁移，以使Spring Batch参考文档中的所有示例都可以通过Java配置以及XML配置进行说明。

下一步是什么？

展望未来，我们将总结文档更新。我们还将研究其他功能接口如何适合Spring Batch。最后，我们将了解Spring Framework生态系统的其他最新依赖项，以为GA发布做好准备。

Spring Batch 4.0.0发生了什么。M4？

Spring Batch 4.0.0。M4上周五发布，但存在错误，不建议用于一般消费。4.0.0。M5是建议消耗的最新里程碑。
