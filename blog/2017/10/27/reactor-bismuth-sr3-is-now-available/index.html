反应堆铋-SR3现已上市

发布
斯蒂芬·马尔蒂尼（Stephane Maldini）
十月27，2017

一批修补程序和好东西已经到达，并且由新产品带给您Bismuth-SR3 材料清单 。Spring Framework 5.0.1现在使用了该版本！我们的网站projectreactor.io反映了这些最新更新。

堆芯3.1.1

发行说明

包括20多个修复程序和几个新功能的质量更新：用于从Reactor转换的Kotlin扩展Publisher和Flux#limitRequest运算符替代Flux#take对生产的需求有明确的上限。

我们现在正在努力3.1.2.RELEASE 。请查看我们的问题积压 ，以获取更详细的范围。

反应堆插件3.1.2

发行说明

现在我们有官方Kotlin一些额外运营商的扩展，将来我们将继续添加一些扩展。这个版本的重点是增加了重试 ， 重复和数学扩展 。我们对kotlin扩展提供的可能性感到非常兴奋，并鼓励我们的kotlin用户尝试reactor-extra ！

看看其中一些方便的扩展：

data class User(val age: Int,val name: String)

val userList = listOf(User(18, "bob"),  User(80, "grandpa"), User(1, "baby"))

users.toFlux()
     .retryExponentialBackoff(4, Duration.ofMillis(100))
     .max { a, b -> b.age - a.age }
     .map { it.name }
     .test()
     .expectNext("baby")
     .verifyComplete()

我们现在正在努力3.1.3.RELEASE ，并邀请我们的社区提交有关更多扩展程序或运营商的想法。我们已经对可追溯性，断路器和路由产生了一定的兴趣，并将在未来几周内评估如何确定这些功能的范围。

反应堆净值0.7.1

发行说明

一个建议的更新，修复了可能在特定情况下使我们的连接池停顿的生命周期问题。还有我们的HttpServer现在可以正确处理HEAD请求和匹配的响应内容长度。我们确定并修复了Netty SSL处理中的回归问题 ，该问题会影响4.1.14至4.1.16的版本。如果您的应用程序依赖SSL上的流大型有效负载，请监视下一个4.1.17 Netty版本。

我们现在正在努力0.7.2.RELEASE和以下0.8.0 。由于最终Spring Framework版本的流行，我们仍在进行调查，并且正在处理一系列新的反馈。 0.8旨在以此成熟度为基础，并提供主要的API质量改进。

SR1和SR2发生了什么？

不幸的是，我们两次遭受POM生成问题，尽管这些版本存在于Maven Central中，但它们都为reactor-extra 。请不要使用Bismuth-SR1和Bismuth-SR2 BOM和reactor-extra:3.1.1.RELEASE 。手指越过下一个版本应该是Bismuth-SR4在Spring Framework之前5.0.2 。

再见 ？

如果您对所有新功能和我们的下一个计划感兴趣，整个团队将在Spring One Platform上并计划一个专门的会议 。此外，大多数Spring轨道都将介绍使用Reactor进行反应式编程！
