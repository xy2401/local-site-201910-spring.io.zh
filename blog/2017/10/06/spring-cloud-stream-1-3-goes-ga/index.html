Spring Cloud Stream 1.3正式发布

工程
维尼修斯·卡瓦略（Vinicius Carvalho）
十月06，2017

我们很高兴地宣布Spring Cloud Stream Ditmars发布系列正式发布。

Spring Cloud Stream Ditmars。RELEASE可在Spring Release存储库中使用。发行说明包括有关与Spring Boot，Spring Cloud，Spring AMQP和Spring for Apache Kafka的版本兼容性的相关信息。

以下各节列出了此版本中包含的新功能。

阿帕奇·卡夫卡

Apache Kafka的Kafka流

Spring Cloud Stream自动执行发现和自动配置，它提供了一种简单的方法来促进应用程序业务逻辑与Apache Kafka之间的交互。由于大多数特定于标准消息代理的特定设置都显示为Spring Boot属性，因此易于使用和覆盖以适应业务需求。

为了进一步增强Apache Kafka集成，Kafka Streams集成已被提升为Apache Kafka的活页夹实现中的顶级构造。

由于将Kafka Streams API引入了框架，因此可以将简单和更复杂的数据处理功能构建为Spring Boot应用程序。您需要的是spring-cloud-stream-binder-kstream在类路径中的依赖-就是这么容易上手。此集成包括分区级别的本地状态，以进一步简化事件驱动处理器，数据聚合器，连续查询和转换的求解。

以下截屏视频显示了它的运行情况。

Apache Kafka 0.11

根据客户和社区的普遍需求，我们增加了对Apache Kafka 0.11的支持。尽管今天已支持它，但在此版本中，我们尚未将其切换为默认设置。在即将发布的2.0.0中，默认值为0.11。发布。因此，对于对升级到此版本感兴趣的早期采用者，我们将其作为独立工件发布 。

消费者滞后指标

通过@hekonsek的贡献，增加了新的指标，可以帮助解决滞后的取证问题。这对于涉及是否根据动态流量模式自动缩放用户的情况很有用。

兔子MQ

惰性队列

如果需要减少I / O占用空间，那么RabbitMQ绑定程序实现中现在支持“惰性队列”。同样，还提供了对“惰性队列”的DLQ支持。

弹性重试

RabbitMQ的典型横向扩展设置包括与前面的负载均衡器连接的RabbitMQ节点的集合。在节点发生故障时，要拥抱它并进行恢复，现在有了一个修复程序，可以在重试的帮助下进行最终恢复。

不同的连接工厂

为了避免在繁忙的生产者阻止共享连接时对使用者造成死锁，此版本增加了对非事务生产者单独的连接工厂的支持。

Amazon Kinesis（正在孵化）

Amazon Kinesis的社区驱动的活页夹实施取得了重大进展。但是，考虑到项目的Spring Cloud生态系统的上游依赖性，它无法进入Ditmars发行版。我们希望继续保持这种势头，并在即将到来的2.0.0里程碑中逐步发展运动结合力。我们要感谢社区冠军Peter Oates（ @oatesp ）和Jacob Severson（ @JacobASeverson ） 所做的宝贵时间和质量贡献。

错误处理

Ditmars发布培训重新审视了核心框架中的错误处理支持。我们简化了生产者和消费者级别的异常处理。例如，有一个错误通道（ .errors ）以接收失败消息的副本，以及全局Spring集成errorChannel仍然可用。错误通道的创建，编排和全局表示在此发行系列中。

Spring Cloud Stream 2.0

1.3版本标志着1.x系列和Boot 1.5的最后一次迭代。
下一个主要的发布培训（Elmhurst）包括对几个核心功能的全面检查。正在考虑以下主题：Spring Boot 2.0兼容性，更好的内容类型分辨率，进一步简化的错误处理体验以及更智能的绑定生命周期挂钩，以启动，停止和暂停使用者。它正在积极开发中，我们希望在SpringOne Platform 2017之前达到稳定的发布候选状态。我们对正在进行的工作感到兴奋！

最后，该发行版还包含各种错误修复和改进。有关更多详细信息，请参阅发行说明中的项目级发行标记。
