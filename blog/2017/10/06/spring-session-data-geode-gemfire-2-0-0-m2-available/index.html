春季会议数据Geode / GemFire 2.0.0。M2可用

工程
约翰·布鲁姆
2017年10月6日

问候春季社区！

我很高兴地宣布两个春季会议数据的Geode的为Apache的Geode和春季会议数据的GemFire为枢纽的GemFire的第二个里程碑。

第二个里程碑版本带来了许多可喜的改进，包括：

  • 升级到Spring Framework 5.0.0。发布。

  • 升级到Spring Data Kay-RELEASE。

  • 升级到Spring Session 2.0.0。M4。

  • 升级到Spring Boot 2.0.0。M4。

  • 增加了对Pivotal GemFire和Apache Geode PDX序列化的支持 。

  • 为Pivotal GemFire / Apache Geode的数据序列化和PDX框架引入了新的序列化框架和适配器。

可以使用Maven从Spring的 libs-milestone存储库下载这两种工件。

  
    org.springframework.session
    spring-session-data-geode
    2.0.0.M2

或者，使用Gradle ...

compile 'org.springframework.session:spring-session-data-geode:2.0.0.M2

要将Spring Session与Pivotal GemFire一起使用，只需将工件从spring-session-data-geode切换到spring-session-data-gemfire 。版本保持不变。

从数据序列化到PDX

许多用户一直要求Pivotal GemFire和Apache Geode的Spring Session支持PDX序列化 。鉴于PDX是高度“便携式”的，这是有道理的，如《 参考指南》中进一步说明。

一个PDX的主要好处是，你不需要为枢纽的GemFire或Apache的Geode 春季会议上的服务器的类路径集群中使用客户机/服务器拓扑使用Spring会话在春季启动 ，的GemFire /的Geode缓存客户端应用程序。实际上，您甚至都不需要将应用程序域对象类存储在（HTTP）会话中，并且也可以持久保存到群集中GemFire / Geode服务器的服务器类路径上。

Spring Session Data GemFire / Geode现在默认使用PDX。但是，仅通过切换@EnableGemFireHttpSessionannotation中的bean引用，在数据序列化和PDX之间切换非常容易。

OOTB， Spring会话数据GemFire / Geode代表用户注册2个bean，以实现每种GemFire / Geode序列化策略。

要明确声明要让启用了Spring Boot和Spring Session的应用程序使用PDX序列化，请将@EnableGemFireHttpSessionannotation中的新sessionSerializerBeanName属性设置为GemFireHttpSessionConfiguration.SESSION_PDX_SERIALIZER_BEAN_NAME ...

@SpringBootApplication
@EnableGemFireHttpSession(sessionSerializerBeanName =
    GemFireHttpSessionConfiguration.SESSION_PDX_SERIALIZER_BEAN_NAME
class Application {
  ...
}

要使用数据序列化并利用Delta ，请将sessionSerializerBeanName设置为GemFireHttpSessionConfiguration.SESSION_DATA_SERIALIZER_BEAN_NAME ...

@SpringBootApplication
@EnableGemFireHttpSession(sessionSerializerBeanName =
    GemFireHttpSessionConfiguration.SESSION_DATA_SERIALIZER_BEAN_NAME
class Application {
  ...
}

在参考指南中可以找到有关将PDX与Spring Session和Pivotal GemFire或Apache Geode结合使用的更多详细信息。

序列化框架

Spring Session Data GemFire / Geode引入了一个新的SessionSerializer接口，允许用户自定义Session如何序列化到Pivotal GemFire或Apache Geode。

interface SessionSerializer {

  void serializer(T session, OUT out);

  T deserialize(IN in);

  boolean canSerialize(Class type);
}

用户可以提供此接口的自定义实现，在Spring上下文中将其注册为Bean，将sessionSerializerBeanName属性设置为自定义SessionSerializer Bean定义的名称， Spring会话数据GemFire / Geode会将其适配并注册为PdxSerializer在Pivotal GemFire或Apache Geode中使用。

例如...

@SpringBootApplication
@EnableGemFireHttpSession(sessionSerializerBeanName = "customSerializer")
class Application {

  @Bean
  SessionSerializer customSerializer() {
    // return an instance of a custom `SessionSerializer` implementation
  }
}

这只是冰山一角。在新的序列化框架的参考指南中可以找到更多详细信息。
