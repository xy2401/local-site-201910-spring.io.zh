Spring Integration 5.0 Milestone 6可用

发布
阿尔特姆·比兰（Artem Bilan）
2017年7月25日

我很荣幸地代表Spring Integration团队宣布Spring Integration 5.0版本的里程碑6（ 5.0.0.M6 ） 现在可用。

可以从Milestone存储库下载：

repositories {
    maven { url 'http://repo.spring.io/libs-milestone' }
}
compile "org.springframework.integration:spring-integration-core:5.0.0.M6"

此发行版中包含19个JIRA （以及一些GitHub问题），包括错误修复和许多新功能。该功能的一些重点M6 ，因为先前宣布的里程碑5是 ：

  • 为HTTP服务器端提供了基于Spring WebFlux的反应性通道适配器：

@Bean
public IntegrationFlow sseFlow() {
    return IntegrationFlows
            .from(Http.inboundReactiveGateway("/sse")
                    .requestMapping(m -> m.produces(MediaType.TEXT_EVENT_STREAM_VALUE)))
            .handle((p, h) -> Flux.just("foo", "bar", "baz"))
            .get();
}

该示例演示了如何使用Spring Integration Channel Adapter配置服务器端事件，该适配器完全基于WebFlux基础并按需以反压方式执行集成流程执行。

  • 的IntegrationFlow现在可以从任何界面开始，即使没有@MessagingGateway注解。该注释是在后台使用默认属性合成的， AnnotationGatewayProxyFactoryBean 。这个技巧为我们打开了开始的可能性IntegrationFlow来自java.util.function接口- Function ， Consumer和Supplier 。反过来，这又为我们提供了一种有趣的方法，可以揭露IntegrationFlow使用新的Spring Cloud Function项目作为函数 ：

@Bean
public IntegrationFlow uppercase(MongoOperations mongoOperations) {
    return IntegrationFlows.from(Function.class)
            .handle(MongoDb.outboundGateway(mongoOperations)
                     .queryFunction(msg ->
                             Query.query(Criteria.where("name")
                                     .is(msg.getPayload())))
                     .collectionNameExpression("headers.collection")
                     .expectSingleResult(true)
                     .entityClass(Person.class))
            .get();
}

  • 一个新的JdbcMetadataStore现在可用。此实现是真正基于事务的，并且如果（S）FTP轮询之后的后续流程导致某些错误，则由于事务回滚，因此不会在数据库中存储任何条目。

  • 的preserveTimestamp已修复（S）FTP入站通道适配器的算法，当本地临时副本无法重命名为原始文件时会抛出异常，因为最后一个副本在另一个进程中很忙。

特别感谢一些社区成员对框架的持续积极贡献；扫描提交并给他们一些荣誉！

有关更改的完整列表5.0 ，另请参阅参考手册中的“新增功能”一章。

我们仍希望将几个功能（反应性和测试）包含到当前版本中，请继续关注下一个里程碑！我们预计RC和后续的GA在11月初的10月底-Spring Boot之前2.0 GA ！
