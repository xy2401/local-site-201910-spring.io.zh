春季会议数据Geode / GemFire 2.0.0。M1现在可用。

工程
约翰·布鲁姆
2017年7月27日

问候春天社区！

我很高兴地宣布两个春季会议数据的Geode的为Apache的Geode和春季会议会话数据的GemFire为枢纽的GemFire的第一个里程碑版本。

两种工件都可以从Spring的 libs-milestone使用Maven的存储库...

  
    org.springframework.session
    spring-session-data-geode
    2.0.0.M1

或与Gradle一起 ...

  compile 'org.springframework.session:spring-session-data-geode:2.0.0.M1'

要将Spring Session与Pivotal GemFire一起使用，只需将工件从spring-session-data-geode至spring-session-data-gemfire 。

背景故事

不久前 ， Spring Session Core的创建者和项目负责人@rob_winch通过将以前支持的数据存储分成单独的模块来重组了Spring Session项目。

进行此拆分的主要驱动力是使受支持的Spring Session数据存储提供程序能够根据社区和客户的需求独立地发展。

好吧， Pivotal GemFire是与核心Spring Session项目分离的受支持数据存储之一。除了Pivotal GemFire之外 ，还正在开发对Apache Geode的支持。

因此，今天发布了受支持的Spring Session数据存储，现在在2.0版本中，除了Redis，MongoDB，JDBC和Hazelcast外，现在还正式包括Apache Geode和Pivotal GemFire 。

在发行中

Spring Session Data Geode和Spring Session Data GemFire都包括以下改进...

  • 与Spring Framework 5.0.0兼容。RC3。

  • 与Spring Boot 2.0.0一起运行。M2。

  • 基于Spring Session core 2.0.0。M3 。

  • 基于Spring Data Kay RC2 ，其中包括_Spring Data Geode 2.0.0。RC2和_Spring数据GemFire 2.0.0。RC2。

  • 分别基于Apache Geode 1.2.0和Pivotal GemFire 9.1.0 。

  • 修订文件 。

  • 使用Spring Data Geode / GemFire 新的基于注解的配置模型，大大简化了配置。请参阅样本 。

例如，建立一个春天启动的应用程序，使用Spring会话数据的Geode管理HttpSession开发商需要做的就是...

@SpringBootApplication
@ClientCacheApplication
@EnableGemFireHttpSession
class MyApplication {

  public static void main(String[] args) {
    SpringApplication.run(MyApplication.class, args);
  }
}

两者都是Spring Data Geode的 @ClientCacheApplication和Spring Session Data Geode的 @EnableGemFireHttpSession批注是将Spring Boot应用程序转变为能够通过将数据分发到Geode服务器集群以快速，可靠（复制）访问来管理HTTP会话状态的Geode缓存客户端所需的全部注释，从而确保用户对应用程序的体验不间断和一流。

结合Pivotal Cloud Foundry的功能， Pivotal Cloud Cache磁贴和新的 会话状态缓存计划，您便拥有成功的秘诀！
