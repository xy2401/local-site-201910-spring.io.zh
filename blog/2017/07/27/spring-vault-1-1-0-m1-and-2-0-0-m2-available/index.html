Spring Vault 1.1.0 M1和2.0.0 M2可用

发布
马克·帕鲁奇
2017年7月27日

我谨代表社区宣布两个新的Spring Vault里程碑：1.1.0 M1和2.0.0 M2。版本1.1.0 M1附带完成的24张凭单，而版本2.0.0 M2附带7个已解决的凭单，可从里程碑资料库中获得。

从以前的版本开始，这些功能已成为当前的里程碑：

  • 通过AWS IAM使用保管库登录

  • 根据通用密钥的租赁期限进行轮换

  • 介绍VaultEndpointProvider动态配置端点

此外，2.0.0 M2随附：

  • 身份验证DSL声明身份验证流

  • 基于Spring Framework 5的响应式支持WebClient和项目反应堆

  • 通过添加增强的工具支持@NonNullApi和@Nullable批注将由您最喜欢的IDE提取

认证DSL

身份验证步骤提供了常见身份验证活动的可重用性。通过创建的步骤AuthenticationSteps以功能样式描述认证流程，将实际的认证执行留给特定的执行者。

// Static token use
AuthenticationSteps.just(VaultToken.of(…));

// AppRole authentication
AuthenticationSteps.fromSupplier(

    // Construct login body
    () -> Collections.singletonMap("role_id", options.getRoleId()))

    // post the payload to Vault to log in
    .login("auth/{mount}/login", options.getPath());

身份验证流要求执行者执行实际的登录。根据支持的执行模型，我们提供了两个执行器：

  • AuthenticationStepsExecutor作为同步的替代品ClientAuthentication 。

  • AuthenticationStepsOperator用于响应式执行。

ClientAuthentication带有静态工厂方法来创建AuthenticationSteps对于特定于身份验证的选项：

CubbyholeAuthenticationOptions options = …
RestOperations restOperations = …

AuthenticationSteps steps = CubbyholeAuthentication.createAuthenticationSteps(options);

AuthenticationStepsExecutor executor = new AuthenticationStepsExecutor(steps, restOperations);

VaultToken token = executor.login();

身份验证DSL是反应式Vault客户端将身份验证步骤与其实际执行脱钩的先决条件。

反应性 Vault 客户端

Spring Vault的反应式客户端支持建立在Authentication DSL和Spring Framework 5的基础之上WebClient 。反应性支持涵盖了通过以下方式进行的读/写/列出和删除操作ReactiveVaultTemplate和缓存VaultToken会话管理的供应商。您几乎可以使用所有身份验证机制来获取Vault令牌。以下机制实现AuthenticationStepsFactory并提供AuthenticationSteps对于非阻塞登录：

  • AppId

  • AppRole

  • AWS-EC2

  • 客户证书

  • 衣帽间

  • 静态令牌

创建一个配置类以开始使用响应式支持：

public class Foo extends AbstractReactiveVaultConfiguration {

	@Override
	public VaultEndpoint vaultEndpoint() {
		return VaultEndpoint.from(URI.create("https://localhost:8200"));
	}

	@Override
	public ClientAuthentication clientAuthentication() {
		return new AwsEc2Authentication(restOperations());
	}
}

配置支持AbstractReactiveVaultConfiguration调整实现的身份验证AuthenticationStepsFactory响应式身份验证方法，并配置命令式和响应式客户端。会话不会在两个客户端之间共享，但这是未来的任务。Ypu可以使用ReactiveVaultTemplate独立：

reactiveVaultTemplate.write("secret/mykey", Collections.singletonMap("hello", "world"))
		.thenMany(vaultOperations.list("secret"))
		.subscribe(item -> System.out.println(item));

或在反应式运行时基础架构（例如Spring WebFlux）中：

@RestController
public class SecretsController {

  private final ReactiveVaultOperations operations;

  // Constructor omitted for brevity

  @GetMapping("secrets")
  Flux<List> listSecrets() {
    return vaultOperations.list("secret");
  }
}
