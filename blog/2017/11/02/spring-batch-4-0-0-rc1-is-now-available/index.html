Spring Batch 4.0.0。RC1现在可用

发布
迈克尔·米内拉
2017年11月2日

我们很高兴地宣布Spring Batch 4.0.0。RC1现在可以通过Github和Pivotal下载存储库获得。这个版本代表了Spring Batch 4.0版本的第一个候选版本。非常感谢对此版本做出的所有贡献。

什么是新的？

此版本表示4.0.0行上新功能的完成。Spring Batch 4的新功能包括：

  • 更新基准
  • 开箱即用的组件的新构建器API
  • 更新，Java配置友好，文档

更新基准

Spring Batch 4.0借此机会重置了项目的基准。考虑到这一点，我们将最低Java版本升级到Java 8。尽管这对用户应该是透明的，但它使我们能够利用尚未使用的新语言功能。

除了Java版本升级外，我们还升级了所有依赖项。这包括要求Spring Framework 5及其产品组合的所有相关版本。Spring Batch 4已经做到最好使依赖项与Spring Boot 2保持一致。

此发行版的特定更新包括Spring Integration的5.0.0。RC1和其他次要更新。

新的Builder API

由于这是自Spring Boot发行以来的第一个主要版本发行，因此这是我们第一次能够花一点时间来全面地看一下Java配置故事。尽管Spring Batch在Spring Boot之前具有Java配置选项，但是配置许多现成的组件需要大量冗长的配置。一个例子是FlatFileItemReader 。配置此阅读器以前需要数十行XML或Java配置，现在可以在10行左右进行配置。此版本包括所有ItemReader和ItemWriter Spring Batch提供的实现。

更新的参考文档

过去，Spring Batch的文档在配置示例中非常注重XML。在Spring Batch 4.0版本中，我们从头开始对文档进行了修订。参考文档中每页的顶部是用于Java或XML的切换开关。此切换指示整个页面中呈现的配置形式。如果选择了Java，则将使用Java配置来显示所有示例。如果选择XML，则显示基于XML的示例。我们希望这将帮助用户更好地浏览框架内的配置选项。

下一步是什么？

如前所述，此版本代表最终版本，其中包含针对4.0.0版本的新功能。从4.0.0开始。RC1，当我们进入GA时，我们将研究错误并进一步调整文档以提供更好的总体体验。
