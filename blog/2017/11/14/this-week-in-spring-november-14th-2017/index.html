Spring 的这个星期-2017年11月14日

工程
乔什·朗（Josh Long）
2017年11月14日

嗨， Spring 粉丝！欢迎来到 Spring 本周的另一期！本周，我在摩洛哥卡萨布兰卡参加令人惊叹的Devoxx摩洛哥活动，与北非枢纽的开发人员交谈。

您可以相信已经是11月中旬了吗？您是否可以相信距加利福尼亚州旧金山的SpringOne平台盛会还不到一个月的时间？希望你走！

现在，事不宜迟，让我们开始吧。

  • Spring 消息传递忍者Oleg Zhurakousky刚刚宣布了Spring Cloud Stream Elmhurst。M3 / 2.0.0。M3 。新版本提高了内容类型的分辨率，支持Apache Kafka 0.11+，并引入了许多不错的示例。
  • Spring Data Geode / GemFire负责人John Blum 刚刚宣布了Spring Data Geode / GemFire 2.0.0。RC1 。新版本更新了依赖关系，建立在Java 8（引入Java 8类型和lambda）的基础上，并引入了新的示例。
  • 在上周的Spring Tips系列中， 我回顾了Kotlin和Spring ，着眼于Spring中可用的某些DSL及其用法。这已经很久了，很高兴看到事情发生了！
  • Spring消息传递忍者Artem Bilan 刚刚宣布了针对Apache Kafka 2.1RC1和2.0.1和1.3.1的Spring，该版本主要引入了对Apache Kafka 1.0的支持。
  • Spring Cloud Task的贡献者Glenn Renfro刚刚宣布了Spring Cloud Task 2.0.0。M2 。这个版本更新了代码，使其与Spring Boot 2.0.0兼容。M5（以及所有其他依赖它的下游依赖项）。
  • 这个由Ben Ooms撰写的Devoxx BE关于消费者驱动的合同测试与Spring Cloud Contract的讨论看起来很有趣
  • 我碰巧知道，Keycloak贡献者Sebastien Blanc 关于使用Spring Boot和RedHat的Keycloak OAuth身份提供者的演讲值得关注。
  • Spring Framework和Spring联合创始人Juergen Hoeller在比利时Devoxx上作了精彩演讲， 介绍了Spring Framework 5的主题和趋势 。
  • 我还和我的好友Mark Heckler 一起在Devoxx BE上有趣地演示了如何构建响应式Spring应用程序 。
  • Spring团队传奇人物Thomas Risberg 关于如何在Kubernetes上构建Spring Boot应用程序的演讲无疑值得关注。
  • 在Devoxx BE上，与我的好友和Okta开发者倡导者Matt Raible一起进行的名为“ 构建云原生渐进式Web应用程序”的演讲非常有趣。在本文中，我们着眼于使用Spring Boot，Spring Cloud和Kotlin构建强大的基于微服务的后端，然后构建基于TypeScript的Angular客户端与该微服务系统进行通信。
  • 这看起来很有趣-这是Arne Vandamme和Steven Gentens的Devoxx BE演讲，内容是使用称为Across Platform（带有Spring）的东西构建Web应用程序。我已经收藏了！
  • 您绝对应该查看Spring Framework负责人Juergen Hoeller关于Spring Framework 5隐藏的宝石的演讲
  • 我喜欢 Kotlin和Spring NinjaSébastienDeleuze进行的Devoxx BE演讲，很好， Spring Boot和Kotlin 。
  • Spring团队成员Stephane Nicoll和Brian Clozel在Devoxx BE上进行的有关构建Spring Boot 2.0 Web应用程序的演讲是必须注意的！
  • 我真的很喜欢Jisha Abubaker在Google开发人员关系工程团队上进行的Devoxx BE演讲。在其中，她着眼于使用一些Google Cloud Spring Boot启动器 。
  • 这篇文章很有趣-Hibernate Developer Advocate Vlad Mihalcea着眼于如何模拟Spring Data的@CreatedBy和@LastModifiedBy与直冬眠 。有趣..但是为什么不只使用Hibernate 和 Spring Data JPA？那里不是更自然的一对:)
  • 这与Spring无关，但我很荣幸被列入Twitter应当遵循的二十位Java开发人员的名单 。如果您想关注我 ，我将不胜感激，但是您绝对应该关注该名单上的其他人。他们中的大多数是朋友，我不能推荐他们。
