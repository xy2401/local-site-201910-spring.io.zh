Spring Cloud Edgware。可发布

发布
斯宾塞·吉布
2017年11月27日

我很高兴代表团队和社区宣布， Spring Cloud Edgware发行培训的一般可用性（RELEASE） 已于今天发布。该版本可以在Maven Central中找到。您可以查看Edgware 发行说明以获取更多信息 。

Edgware发布系列中的显着变化

spring-cloud-bus

更新以允许Bus客户端在独立程序包中实现远程事件。

春季云任务

查看博客文章 ，宣布任务1.2.0。

Spring Cloud Commons

重试请求时增加对退避策略的支持。

Spring Cloud Stream

有关更多信息，请参见Ditmars发行公告 。

Spring Cloud Sleuth

  • 添加RestTemplateBuilder支持
  • Zipkin 2现在是默认设置
  • spring-cloud-starter-zipkin现在使用Zipkin 2和spring-cloud-starter-zipkin-legacy使用Zipkin 1。
  • 现在，您可以通过将服务ID设置为Zipkin URL来通过Service Discovery找到Zipkin。spring.zipkin.baseUrl: http://zipkinserver/
  • @Async方法可以使用@SpanName更改默认跨度名称

Spring Cloud安全性

增加了对重试使用的支持UserInfoRestTemplate 。

Spring Cloud合约

  • WireMock升至2.11.0并打开并启用了详细模式
  • 引入了添加自定义WireMock扩展的信息
  • 正文响应验证中允许引用网址段
  • 添加了对合同中引用文件内容的支持
  • 添加了将映射转储到文件的选项
  • 保证放心升级到3.0
  • 引入了影响插件的回归，但已在其中修复1.2.1 。对于插件，请使用1.2.1.RELEASE

Spring Cloud Vault

  • 支持Kubernetes和AWS IAM（需要AWS Java SDK）身份验证方法。
  • 使用Vault发现DiscoveryClient在引导上下文中。
  • 配置PropertySourceLocator自定义上下文路径。由Spring Cloud Vault Connector使用，用于在Cloud Foundry上通过HashiCorp的Vault Service Broker运行应用程序。
  • TTL到期后刷新通用机密。
  • 升级到Spring Vault 1.1

Spring Cloud Netflix

重试请求时增加对退避策略的支持。

使用以下工具构建的可选Eureka客户端RestTemplate而不是泽西岛。

@EnableDiscoveryClient现在是可选的。包含spring-cloud-starter-netflix将自动假定该应用程序应该注册并成为发现客户端。禁用自动注册设置spring.cloud.service-registry.auto-registration.enabled=false 。Hystrix和Eureka Server现在通过Spring Cloud Contract进行了测试。

spring-cloud-consul

增加了与最新的Hashicorp Consul 1.0版本的兼容性。

Consul DiscoveryClient现在支持数据中心参数。现在支持带有自签名证书的HTTPS检查。

Spring Cloud Config

新的JDBC EnvironmentRepository已创建。

与往常一样，我们欢迎在GitHub ， Gitter ， Stack Overflow或Twitter上提供反馈。

要开始使用带有BOM的Maven（仅限于依赖管理）：


<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>org.springframework.cloud</groupId>
            <artifactId>spring-cloud-dependencies</artifactId>
            <version>Edgware.RELEASE</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
<dependencies>
    <dependency>
        <groupId>org.springframework.cloud</groupId>
        <artifactId>spring-cloud-starter-config</artifactId>
    </dependency>
    <dependency>
        <groupId>org.springframework.cloud</groupId>
        <artifactId>spring-cloud-starter-eureka</artifactId>
    </dependency>
    ...
</dependencies>

或使用Gradle：

buildscript {
    dependencies {
        classpath "io.spring.gradle:dependency-management-plugin:1.0.2.RELEASE"
    }
}



apply plugin: "io.spring.dependency-management"

dependencyManagement {
    imports {
        mavenBom 'org.springframework.cloud:spring-cloud-dependencies:Edgware.RELEASE'
    }
}

dependencies {
    compile 'org.springframework.cloud:spring-cloud-starter-config'
    compile 'org.springframework.cloud:spring-cloud-starter-eureka'
    ...
}
