春季会议数据Geode / GemFire 2.0.0。RC1可用

工程
约翰·布鲁姆
2017年11月7日

我很荣幸代表Spring和Apache Geode / Pivotal GemFire社区宣布发布Spring Session Data Geode和Spring Session Data GemFire 2.0.0。RC1版本。

两种版本均基于：

  1. 春季框架 5.0.1.RELEASE

  2. Spring Security 2.0.0.RC1

  3. Spring Data Kay-RC1

  4. Spring Session核心2.0.0.RC1

  5. Spring Boot 2.0.0.M5

可以从Spring获取工件libs-milestone ：

马文

  
    spring-libs-milestone
    https://repo.spring.io/libs-milestone
  



  
    org.springframework.session
    spring-session-data-geode
    2.0.0.RC1
摇篮
repositories {
  maven { url "https://repo.spring.io/libs-milestone" }
}

dependencies {
  compile 'org.springframework.session:spring-session-data-geode:2.0.0.RC1'
}

什么是新的！

此版本建立在2.0.0.M2通过应用Java 8类型发布（例如Stream ）和Lambda表达式来简化代码库。

另外，我包括一个新示例 ，该示例演示如何使用由Apache Geode（或Pivotal GemFire）支持的Spring Session来构建使用“ session ”或“ request ”作用域代理Bean的Spring Boot Web应用程序。

此示例证明，即使Spring Session替换了基础Web容器的基础结构，在Spring容器中定义的“ 会话 ”范围的代理Bean仍然可以正确管理。 HttpSession由Spring Session提供的使用备用数据存储的实现，例如Apache Geode。

该示例基于StackOverflow中的 发布 。

下一步是什么？

一旦Spring Session核心进入GA，我就可以发布Spring Session Data GemFire / Geode 2.0 GA。

一如既往地欢迎并赞赏对此版本的反馈。
