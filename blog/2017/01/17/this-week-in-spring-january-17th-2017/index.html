Remember, at the end of the day, the most important metric for organizational agility isn’t cloud computing or microservices, it’s the ability to continuously (quickly and safely) deliver value to the customer.

工程
乔什·朗（Josh Long）
一月17，2017

欢迎来到春季本周的另一期！这个星期我们有很多事情要做-比平常还要多 ！所以，让我们开始吧！

开始为春季的下一季整理主题提示：您希望看到哪些讨论？

—龙之春，Jo（@starbuxman） 2017年1月13日

由于我们自己的彼得·汉弗莱（Pieter Humphrey），本周网上有很多很棒的SpringOne Platform 2016重播！

  • 千万不要错过斯特凡马尔蒂尼的SpringOne上平台（S1P）2016通话将反应器3.0
  • 或塞巴斯蒂安·德勒兹（Sebastien Deleuze）和斯蒂芬·马尔迪尼（ Stephane Maldini）的本S1P 2016演讲， 针对JVM的反应式编程选项 （包括Reactor和RxJava等）
  • 雅虎！ 马尔辛-只是啾啾一些令人振奋的消息： 条约的支持已经被添加到Spring Cloud合同
  • 宪明辰巳（来自Capital One公司）给了一个很好S1P 2016的谈话上使用Spring云操作系统高速组织
  • Google的Ray Tsang和Bret McGowen一起为Spring Developers使用gRPC进行了精彩的演讲
  • 在本次S1P 2016演讲中，Ben Hale，Paul Harris和Stephane Maldini介绍了构建反应式API的一些细微差别
  • Ryan Baxter刚刚宣布了Spring Cloud Camden。SR4
  • Glenn Renfro刚刚宣布了Spring Cloud Task 1.1.2。发布
  • 说到Spring Cloud Contract，我真的很喜欢epages的Mathias撰写的有关使用Spring Cloud Contract构建和测试事件驱动的微服务的文章。
  • 我喜欢我们自己的Toshiaki Maki详尽的存储库，该存储库演示了如何将Spring Boot与大量不同的SQL持久性技术一起使用 。很棒的资源！收藏这个！
  • 说到持久性选项，我真的很喜欢社区英雄Michael Simmons在JOOQ和Spring Boot上的演讲
  • Hortonworks的Tim Spann在将Spring Boot与Apache Phoenix或Apache Hive结合使用时获得了一些不错的文章
  • 马丁·韦斯特加德·拉森（Martin Westergaard Lassen）在他最喜欢的七张专辑上合了个好文章（啊！六..：D） Spring Boot的功能
  • 我真的很喜欢Apigee的Alan Ho的这篇帖子，原因是组织散布了单片定时炸弹 ：“如果单片堆栈不走，那么好的开发人员就会这样做”
  • 雅虎！只是开源了他们称为连续螺丝刀的连续交付平台 。这与Spring或Pivotal几乎没有关系，甚至没有关系，但这是不断扩展的持续交付工具生态系统中的另一个竞争者，以及Pivotal的Concourse，Netlfix Spinnaker（Pivotal，Google，Microsoft和Netflix都致力于其中）和贡献），詹金斯等等。请记住，归根结底，组织敏捷性最重要的指标不是云计算或微服务，而是持续（快速，安全）为客户提供价值的能力。
  • 这太酷了！现在您可以将Apache Camel与Spring Cloud Stream一起使用
  • 这是我的内心深爱的话题：使用Spring Boot Slack机器人通知人们咖啡的供应情况
  • 欢呼！基于Spring Cloud Ribbon的负载平衡现在支持Cloud Foundry外科手术路由
  • 是否想从.NET应用程序中使用Spring Cloud Services？在Steeltoe上查看此帖子
  • 埃文·布朗（Evan Brown）整理了一个简洁的博客，介绍了有关如何在Google Cloud Platform上部署Pivotal Cloud Foundry的演讲的重播。查看有关BOSH和Google Cloud Platform的Cloud Foundry基础知识的演讲，第pt页。1个
  • 是否想了解如何将Spring Cloud Security（及其对客户端的OAuth支持）与RedHat的Keycloak结合使用？通过RedHat的Kamesh来查看此博客
