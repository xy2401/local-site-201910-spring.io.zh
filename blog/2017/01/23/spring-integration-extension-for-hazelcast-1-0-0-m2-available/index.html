适用于Hazelcast 1.0.0 M2的Spring Integration Extension

发布
阿尔特姆·比兰（Artem Bilan）
一月23，2017

我谨代表Spring Integration社区宣布Hazelcast的Spring Integration Extension的第二个里程碑，其工件是spring-integration-hazelcast.1.0.0.M2 ，可在Milestone存储库中找到 。

该项目已经存在了一段时间，自从上一个里程碑1以来，没有进行太多活动，并且直到最近它才获得了足够的社区关注，因此有必要发布。因此，首先，非常感谢您社区！

此里程碑中包含的功能的一些要点：

榛果领袖选举

如果您需要选举一个领导者（例如，对于只有一个节点应该接收消息的高可用性消息使用者），您只需要创建一个LeaderInitiator即可：

@Bean
public HazelcastInstance hazelcastInstance() {
    return Hazelcast.newHazelcastInstance();
}

@Bean
public LeaderInitiator initiator() {
    return new LeaderInitiator(hazelcastInstance());
}

然后，当节点被选为领导者时，它将向所有应用程序侦听器发送OnGrantedEvent。有关如何使用这些事件来控制消息传递端点的更多信息，请参见《 Spring Integration参考手册 》。

Hazelcast编写MessageHandler

的HazelcastCacheWritingMessageHandler现在可以解决DistributedObject在运行时针对传入Message通过cacheExpression属性。支持的类型是IMap ， MultiMap ， ITopic ， ISet要么IList ：

@Bean
public IMap<Integer, HazelcastIntegrationTestUser> distMap() {
    return testHzInstance().getMap("Distributed_Map1");
}

@Bean
@ServiceActivator(inputChannel = "distMapChannel")
public HazelcastCacheWritingMessageHandler hazelcastMessageHandler() {
    HazelcastCacheWritingMessageHandler handler =
                   new HazelcastCacheWritingMessageHandler();
    handler.setDistributedObject(distMap());
    handler.setKeyExpression(PARSER.parseExpression("payload.id"));
    handler.setExtractPayload(true);
    return handler;
}

Hazelcast群集监视器入站通道适配器

的HazelcastClusterMonitorMessageProducer允许侦听在集群上执行的修改。这是一个事件驱动的通道适配器，它侦听相关的成员资格，分布式对象，迁移，生命周期和客户端事件：

<int-hazelcast:cm-inbound-channel-adapter
                 channel="monitorChannel"
                 hazelcast-instance="instance"
                 monitor-types="MEMBERSHIP, DISTRIBUTED_OBJECT" />

<bean id="instance" class="com.hazelcast.core.Hazelcast"
            factory-method="newHazelcastInstance">
    <constructor-arg>
        <bean class="com.hazelcast.config.Config" />
    </constructor-arg>
</bean>
