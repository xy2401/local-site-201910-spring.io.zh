春天的这个星期-2017年1月10日

工程
乔什·朗（Josh Long）
一月10，2017

欢迎来到春季本周的另一期！本周，我将在旧金山和纽约与开发人员进行交谈，并为O'Reilly的《 Cloud Native Java》一书的最终编辑作准备（几乎在那里！）。此外，行销忍者Pieter Humphrey还将加强S1P 2016的播放 ，因此请系紧安全带并查看S1P 2016 YouTube播放列表中的最新版本。这周我们有很多工作要做，让我们开始吧！

  • Spring Cloud Task忍者Glen Renfro刚刚发布了Spring Cloud Task 1.1.1 ，该版本具有快速跟踪，仅错误修复的1.1.2版本 。
  • 我们最新最好的Spring Boot团队成员Madhura Bhave刚刚宣布了Spring Boot 1.5.0。RC1
  • Spring Integration专家Artem Bilan刚刚宣布了Spring Integration 5.0。M2
  • Spring Batch和Spring Cloud Task负责人Michael Minella刚刚宣布了Spring Batch 4.0 ，它具有很多优点，尤其是我永远想要的功能，这是构建更复杂实例的简便方法ItemReader和ItemWriter实例，包括FlatFileItemReader ！这是一个很长的时间，我迫不及待想要发布这个版本。这完善了Spring Batch的史诗般的Java配置故事。
  • Spring IO Platform负责人Andy Wilkinson刚刚宣布了Spring IO Platform Athens SR2 ，它带来了Spring Boot，Spring Retry，Spring AMQP等的新版本。
  • Spring Framework 忍者和Kotlin亲戚Sebastien Deleuze刚刚在博客上发表了一些新的改进，以支持Spring Framework 5中的Kotlin语言
  • 致力于安全性和服务差异化的微服务开发人员不会想错过Spencer Gibb的S1P 2016重播：Spring Cloud with Consul和Vault 。如果您是Vault的新手 ，那么Mark Paluch会为您介绍上周春季发布的S1P重播 。
  • 赶上Spring Data负责人Oliver Gierke，他将在S1P 2016上提供Spring Data REST的深入研究 ，现在在youtube上！
  • Shin Tanimoto的S1P 2016会议提供了可视化由Spring Boot应用程序使用Elasticsearch，Kibana和Spring Cloud Sleuth构建的微服务的技术。
  • Apache Geode和Pivotal Gemfire的发烧友不会想念来自Shoo 约翰·布鲁姆 （ John Blum）和卢克·香农 （ Luke Shannon）的这个大师班，该班关于2016年S1P的Spring Data和实际的内存数据管理 。
  • 在Cloud Foundry上使用Spring Boot？Scott Frederick 在S1P 2016上的演讲中向您展示了如何像老板一样使用自动配置和Java Buildpack来访问平台和非平台服务。
  • 您是Apache Cassandra用户吗？Spring Data专家Mark Paluch，John Blum 在本次S1P 2016重播中谈论了社区模块成为一等项目，即Spring Data Cassandra 。
  • Pivotal自己的Ben Wilcock整理了一个非常不错的示例项目，展示了如何在Pivotal Cloud Foundry中使用Spring Cloud Services
  • 他还整理了Spring Cloud Contracts的便捷演示
  • 我们自己的Toshiaki Maki Making撰写了一篇很好的文章，介绍了Spring Boot 1.5中的一些细节以及它们如何与Cloud Foundry的Pivotal Apps Manager集成
  • Daniel Olszewski汇集了快速轻松地了解如何为Spring Cache抽象配置生存时间值
  • 我喜欢这篇关于使用Spring Boot构建REST服务的德语文章
  • Jose Praveen 很好地演示了如何将H2数据库与Spring Boot应用程序集成
  • 我喜欢Sam Chu的中文帖子， 内容是在Spring应用程序中建立多个MongoDB连接
  • 这是在REST API中如何验证JSON负载的深思熟虑方法
