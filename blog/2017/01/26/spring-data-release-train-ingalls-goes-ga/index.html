Spring Data 发布列车Ingalls登上GA

发布
奥利弗·德罗博姆
一月26，2017

我谨代表Spring Data团队宣布我们的Ingalls发布系列的一般可用性。经过9个月的开发，此发行版不仅包含大量新功能，改进和错误修复，而且标志着Spring Data分支的结束，该分支已帮助开发人员解决持久性问题超过5年，甚至8年。如果算上某个模块的前身。

该版本共发行了630张以上的固定票。以下是该版本附带的最重要功能的简短说明：

  • 升级到Spring 4.3作为Spring Framework基准。
  • 将Spring Data LDAP（Spring LDAP的与Spring Data相关的位）添加到发行版中。
  • 将Apache Cassandra的Spring Data提升到Core模块，对其进行彻底的检修并将其升级到Cassandra 3。用户定义类型，支持Java 8 Stream和Optional等等
  • 通过避免使用方法句柄支持ASM生成的代码进行反射，从而提高了对象到存储映射的性能。
  • 适用于MongoDB的更多聚合框架。
  • 改进了JPA中的流执行验证（报告了丢失的周围事务）。
  • 支持将XML和JSON有效负载映射到投影接口。
  • 我们的示例查询支持的任意匹配模式。
  • 支持Redis Geo命令。
  • Spring Data REST中的CORS支持。
  • 支持Javaslang集合和映射类型作为存储库查询方法返回类型。

我们将在两天内发布一篇博客文章，为这些功能提供更多信息。在我们的发行培训Wiki中找到更完整但精选的更改列表。该团队现在的重点是转移到我们的下一代Spring Data，从即将于2017年第二季度在Google Analytics（分析）上发布的Kay发布火车开始。注意即将发布的另一个里程碑版本。反过来，这意味着Ingalls将包含在即将到来的Spring Boot 1.5发行版和基于Spring Framework 4的发行版系列中，这些版本将在主要维护中进行。Hopper即将淘汰，仅看到Spring Boot 1.4发行版的偶尔更新。
