Spring Cloud Stream 布鲁克林。SR2和切尔西。M1发布

发布
马里乌斯·波哥维奇（Marius Bogoevici）
一月20，2017

我很高兴代表社区宣布发布Spring Cloud Stream Brooklyn。SR2和切尔西。M1发行列车。作为布鲁克林发行火车下一期的一部分，布鲁克林。SR2添加了许多新功能和修复。下一班火车切尔西。M1建立在Spring Boot 1.5之上，并且包含以下列出的核心改进。

有什么新东西？

改进区域的一些亮点：

  • Schema Registry支持：修复了与使用Schema Registry服务器有关的问题，尤其是对MySQL和Postgresql的支持；
  • 聚合应用程序：改进了对通过名称空间进行属性传播的支持；
  • 反应性支持 ：将声明性（例如，反应性）处理程序支持与命令性模式对齐，这使无缝采用功能性编程模型更加容易。特别是，这增加了对以下惯用语的支持：
@StreamListener(Processor.INPUT)
@SendTo(Processor.OUTPUT)
public Flux<String> uppercaseTransformer(Flux<String> input) {
  return input.map(String::toUpperCase);
}
  • Apache Kafka活页夹和核心 ：在核心中添加了对活页夹本机编组解组的支持，对于Kafka活页夹，它支持直接使用序列化器/反序列化器（包括基于Confluent Schema Registry的反序列化器）；
  • 核心 ：切尔西。M1发布系列引入了内部更改，该更改允许绑定过程在除Spring Integration消息通道之外的其他实体上运行，这是Kafka Streams（ KStream ）和本机响应式支持的先决条件。

可以找到针对核心 ， Rabbit MQ绑定程序和Apache Kafka绑定程序的发行培训组件修复程序的完整列表。

接下来是什么？

  • 添加了JMS粘合剂支持（针对切尔西。M2）和ActiveMQ的现成实现。 Solace和IBM MQ的其他实现将作为社区贡献提供，但出于许可的原因，不会作为工件发布。
  • 支持KStream绑定作为实验功能；
  • Rabbit MQ活页夹改进，包括对用户基础结构，优先级队列和TTL的支持。
