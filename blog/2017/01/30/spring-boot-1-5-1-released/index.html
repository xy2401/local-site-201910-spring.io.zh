Spring Boot 1.5.1发布

发布
安迪·威尔金森
一月30，2017

我很荣幸地代表Spring Boot团队以及所有贡献者，宣布Spring Boot 1.5.1已发布，现在可以从repo.spring.io ， Maven Central和Bintray获得 。此版本增加了大量新功能和改进。有关完整的升级说明和“新功能和值得注意的功能”，请参阅发行说明 。

1.5的新功能

Apache Kafka支持

Spring Boot 1.5通过spring-kafka项目包括对Apache Kafka的自动配置支持。要使用Kafka，只需添加spring-kafka依赖关系并配置适当的spring.kafka.*应用程序属性。

Cloud Foundry执行器扩展

Spring Boot的执行器模块现在包括附加支持，当您将其部署到兼容的Cloud Foundry实例时，该支持即被激活。的/cloudfoundryapplication路径为所有人提供了替代的安全路径NamedMvcEndpoint豆子。

Cloud Foundry管理UI可以利用端点来显示其他执行器信息。例如，Pivotal Cloud Foundry在应用程序状态旁边显示健康信息。有关更多详细信息，请参阅有关Pivotal Cloud Foundry 1.9的博客文章 。

Spring Data 摘要

Spring Boot 1.5附带了最近发布的Spring Data Ingalls。请参阅公告博客文章以了解其所有新功能。

LDAP支持

Spring Boot现在为任何兼容的LDAP服务器提供自动配置，并从UnboundID支持嵌入式内存中LDAP服务器。请参阅文档以获取更多详细信息。

记录器端点

新的执行器记录器端点可让您即时查看和更改应用程序记录级别。JMX和MVC端点均可用。例如，要更改MVC端点的日志记录级别，可以使用以下JSON向/loggers/com.yourcorp.application发布POST：

{
  "configuredLevel": "DEBUG"
}

要使用JMX端点更新记录器，您可以使用setLogLevel操作。请参阅文档以获取更多详细信息。

其他变化

发行说明中还记录了许多其他更改和改进。您还可以找到我们计划在下一版本中删除的不推荐使用的类和方法的列表。
