Spring Integration 5.0 Milestone 2可用

发布
阿尔特姆·比兰（Artem Bilan）
一月05，2017

我谨代表Spring Integration团队宣布Spring Integration 5.0的第二个里程碑，该里程碑可在Milestone存储库中找到 。

自上一个里程碑以来，此版本的一些亮点。

当然，首先，非常感谢您，社区的贡献！

MongoDb的改进

  • MongoDbOutboundGateway -用于对集合执行查询或任何任意操作

  • 对MongoDB组件的初始Java DSL支持

  • MongoDb组件现在可以使用org.springframework.data.mongodb.core.query.Query表达式中的API

@Bean
public IntegrationFlow mongoDbGatewayFlow() {
    return f -> f
        .handle(MongoDb.outboundGateway(this.mongoTemplate)
                            .collectionCallback(MongoCollection::count)
                            .collectionNameFunction(m ->
                                           m.getHeaders().get("collection")));
}

@MessagingGateway和Java DSL

Java DSL IntegrationFlow现在可以从标有的界面开始@MessagingGateway ，并且目标代理Bean上的所有方法调用都将执行发送Message到下游IntegrationFlow 。这可以让您省略@IntegrationComponentScan和额外的频道配置。例如，控制总线组件的简单网关：

@MessagingGateway
public interface ControlBusGateway {

    void send(String command);

}
...
@Bean
public IntegrationFlow controlBusFlow() {
    return IntegrationFlows.from(ControlBusGateway.class)
                 .controlBus()
                 .get();
}

反应式流支持

当然，还有一些来自Reactive Streams主题的新闻。

的MessageChannelReactiveUtils可以用来适应任何MessageChannel到org.reactivestreams.Publisher 。当您想通过一侧的松散耦合方式和另一侧的反作用背压“流入”上游数据时，此功能很有用：

@Autowired
private PollableChannel queueChannel;
...
Flux.from(MessageChannelReactiveUtils.toPublisher(this.queueChannel))
                   .map(Message::getPayload)
                   .map(String::toUpperCase)
                   .doOnNext(results::add)
                   .subscribe(v -> done.countDown());

现在在现有技术中使用这种技术IntegrationFlowDefinition.toReactivePublisher() ：

@Bean
public Publisher> pollableReactiveFlow() {
    return IntegrationFlows
             .from("inputChannel")
             .split(s -> s.delimiters(","))
             .transform(Integer::parseInt)
             .channel(MessageChannels.queue())
             .toReactivePublisher();
}
...
@Autowired
@Qualifier("pollableReactiveFlow")
private Publisher> pollablePublisher;

的ReactiveChannel现在有能力订阅上游Publisher与常规（但背压）一起send(Message)实施。这使我们可以引入诸如启动IntegrationFlow来自Publisher ：

Flux> messageFlux = Flux.just("1,2,3,4")
        .map(v -> v.split(","))
        .flatMapIterable(Arrays::asList)
        .map(Integer::parseInt)
        .map(GenericMessage::new);

QueueChannel resultChannel = new QueueChannel();

IntegrationFlow integrationFlow =
        IntegrationFlows.from(messageFlux)
                .transform(p -> p * 2)
                .channel(resultChannel)
                .get();

this.integrationFlowContext.registration(integrationFlow)
        .register();

如此明智和放置ReactiveChannel在端点之间（ MessageChannels.reactive() ），我们可以同时兼顾集成和反应世界！
