Spring Tools 4-公开测试版发布

发布
马丁·利珀特
2017年12月4日

从今天开始，我们进入了Spring的工具领域的新篇章，并在事实上的标准Spring Boot之上构建了企业应用程序的开发。通过发布下一代Spring工具的公开测试版，我们可以让全球的开发人员预览下一步的发展，以改善他们在Spring和Spring Boot方面的开发人员体验。

春季意识

全新的Spring Tools 4可以动态分析您的项目，了解其中的Spring构造，并提供对所有Spring元素的便捷快速浏览。找到特定的请求映射，确切的bean定义或新创建的函数从未像使用新的Spring Tools 4那样容易。

来自实时运行的应用程序的信息

全新的Spring Tools 4引入了一个新概念，即通过实时运行的Spring Boot应用程序的详细信息覆盖您的源代码。在计算机上运行启动应用程序后，此应用程序中的其他信息就会直接显示在源代码编辑器中。这样，您可以查看（并直接跳至）可用的请求映射。但是，最有效的部分是有关您的bean的详细接线报告。您可以确切地看到（从哪个源）将哪个bean注入到@Autowired注释的依赖项中（例如），以及在应用程序中将bean注入的位置。这可以直接在您的源代码中提供对正在运行的应用程序的独特见解。您不必自己收集该数据并手动查找正确的代码行。该工具会自动为您完成操作。

与IDE无关

我们知道Spring开发人员喜欢他们的IDE和代码编辑器。我们希望他们继续使用他们喜欢的工具。因此，我们以与IDE无关的方式实现了Spring Tools 4的大部分内容。您可以在以下位置使用它们：

  • Eclipse （或照常下载Eclipse的即用型发行版）
  • Visual Studio程式码
  • Atom IDE

这仅仅是开始。我们希望将来会有更多环境加入该聚会。

从头开始重建

为了实现这些全新的Spring Tools 4，我们决定从头开始重写大多数Spring工具-从一开始就牢记这些目标：

  • 让我们成为IDE不可知论者 -让每个开发人员继续使用他们喜欢的环境。
  • 从一开始就构建性能 -我们讨厌等待IDE或编辑器完成之后才能继续编码。因此，Spring Tools 4的设计绝不会阻止用户，而且不会很快交付结果。
  • 为现代Spring技术而构建 -这些工具专注于Spring Boot和注释驱动的Spring应用程序开发。

常问问题

Spring Tools 4何时会成为GA？

您可以立即下载公共测试版。我们将在相对较短的时间内发布公开测试版的更新。首个正式版本将于2018年年中正式发布。

旧的STS 3.9.x工具会发生什么？

我们将继续以完整发行版的形式发布STS 3.9.x的更新，直到2018年中，并将发行版更新为即将到来的Eclipse Photon（4.8）版本。如果您仍然需要Eclipse中旧工具的重要部分，而Spring Tools 4 for Eclipse不支持这些部分，则可以将这些部分作为附加功能安装到Spring Tools 4 Eclipse发行版中，而这些部分将继续接收维护更新一段时间。

这与Boot 1.5和Boot 2.0兼容吗？

是的，公开测试版已经与Spring Boot 1.5.x和Spring Boot 2.0兼容。

Spring Tools 4是否包含Java语言支持？

Spring Tools 4本身不包括对Java编程的任何特定支持，例如标准Java代码完成等。但是，我们建议对各种环境使用现有的Java语言支持。Spring Tools 4（又称Spring Tool Suite 4）基于Eclipse的发行版包括Eclipse Java支持。对于Visual Studio Code，您应该安装Java Extension Pack，它是常规Java语言工具（由RedHat和Eclipse提供）与针对Java for Visual Studio Code的启动和调试支持（来自Microsoft）的组合。对于Atom，还存在一个Java扩展（基于与Visual Studio Code的Java支持相同的代码）。

Spring Tools 4是否支持JDK9？

您可以在JDK9 JVM上运行公共beta版本，并且可以在Eclipse，Visual Studio Code和Atom中使用现有的支持JDK9的Java语言工具。但是Spring Tools 4本身还不支持依赖JDK9语言语法的项目。在即将到来的测试版周期中，将对此进行处理。
