今年春季-2017

工程
乔什·朗（Josh Long）
2017年12月27日

嗨，春季粉丝！欢迎来到春季本周的另一期，尽管非常特别！这是2018年之前的最后一期，因此，我们将像在今年的最后一期中一样，回顾春季和其生态系统非常令人振奋的一年，借此机会进行回顾。

让我代表Spring和更大的Pivotal团队，第一个祝您和您衷心快乐，新年快乐！首先让我们看一下定义2017年的一些主要趋势。我们有很多要讲的！

  • Spring Framework 5和响应式编程 -今年我们发布了Spring Framework 5.0。这个版本是巨大的 -具有很多新功能，包括对Java EE 8，Java SE 9的支持，以及-当然-基于Project Reactor的反应式编程以及称为Spring WebFlux的完全反应性的新Web运行时。Web运行时支持类似Spring MVC的组件模型，新的处理程序模型（称为功能反应式端点）和新的完全反应式HTTP客户端，称为WebClient 。Spring Framework 5的反应式支持，虽然令人印象深刻，但仅仅是开始。Spring Framework 5.0是Spring Data Kay中对反应性数据访问支持，Spring Security 5.0中的反应性安全集成以及Spring Cloud Stream中的反应性消息传递的基础。所有这些都将出现在Spring Boot 2.0中，这反过来又为Spring Cloud Finchley奠定了基础。两者都将在2018年以GA形式首次亮相。我们的响应式编程支持的发布与我们在Spring Framework 5中首次亮相的一流的Kotlin支持非常吻合，并且已经扩展到其他许多项目，尤其是全新的Spring Cloud Gateway项目，该项目也将在Spring Cloud Finchley中首次亮相。
  • PCF 2.0 - Pivotal Cloud Foundry 2.0已发布！ 。对我们来说，这是一个很大的释放。它标志着道路上的重要分支。我们将PCF视为支持功能/无服务器，容器和高阶应用程序的通用云。我们将在称为Pivotal Function Service （PFS）的项目中使用Project Riff支持无服务器和功能。我们在Pivotal Container Service（PKS）中支持容器-非常适合有状态工作负载（如Apache Spark或ElasticSearch）。我们使用Pivotal Application Service （PAS）支持无状态的12要素应用程序。我喜欢CTO Onsi Fakhouri的PCF 2.0 。
  • SpringOne Platform 2017-今年的开发者大会比以往任何时候都更大，更好！在这个节目中，我们有近3,000人参加。我们让每个部门的用户讲述他们的生产历程。查看SpringOne Platform 2017 YouTube播放列表中的所有技术讲座和推荐，在其中您可以找到数百个视频，尽情享受狂欢 。
  • 功能即服务或无服务器 -今年，我们认真地进入了无服务器竞争，首先是Spring Cloud Function ，然后是Project Riff ，它将成为Pivotal Cloud Foundry 2.0中Pivotal Function Service的基础。
  • Pivotal Container Service（PKS） -去年我们知道，当我们与Google合作宣布使用BOSH作为Kubo项目的一部分来管理Kubernetes的支持时，它将引起共鸣。今年，随着Pivotal Container Service（PKS）的出现，合作伙伴关系和增长迅猛 。

现在，让我们来看一下2017年最后一周的一些情况！

  • Spring Tool Suite的负责人Martin Lippert 刚刚宣布了Spring Tool Suite 3.9.2 ，其中包括对Eclipse Oxygen 2的更新，对JDK9和JUnit 5的新支持，并且它支持安装STS 4 Public Beta功能作为早期访问的附件。
  • Pivotal Cloud Foundry 2.0刚刚发布 。PCF 2.0将Cloud Foundry定位为支持不同类型工作负载的统一平台。如果要运行容器，功能或应用程序，则可以使用PCF。
  • Spring Security负责人Rob Winch 刚刚宣布了Spring Security SAML 1.0.3
  • Spring Cloud Finchley的最新更改将使用响应式自动配置服务注册感知负载平衡WebClient在Spring WebFlux中，就像Spring Cloud一直为RestTemplate在Spring MVC世界中。
  • 说到IBM DeveloperWorks，我不能相信我没有将这次综述包含在Spring Framework 5的所有最新和最出色的文章中 。
  • IntelliJ IDEA 2017.3 为Spring和Spring Boot应用程序提供了许多新功能。现在，它具有对Spring Boot 2.0 Actuator端点的内置支持。现在，它能够检测配置了Spring Boot的Spring MVC上下文，并且现在已初步支持基于Spring Cloud的bootstrap.properties 。
  • 这是Okta的Matt Raible的老话，但是关于如何使用Spring Cloud构建微服务应用程序的好东西 。
  • Spring Framework 5.0 与许多Java EE 8 API很好地集成在一起 ，包括JSON Binding API。 这是IBM的DeveloperWorks上一篇不错的文章，内容涉及JSON Binding API的用法 。
  • 迈克尔·普洛德（MichaelPlöd）在美丽的巴塞罗那举行的Spring I / O 2016大会上的这篇演讲很好地探讨了Spring的高级缓存用例 。这是个老歌，但最近又出现了。
  • 我喜欢这个视频，它介绍了Atomist联合创始人Rod Johnson和Christian Dupuis的Atomist 。
  • 关键的忍者卢克·香农（Luke Shannon）发了一条推文，介绍了在使用Linux时如何使Lombok与Beta Spring Tool Suite 4版本一起使用。
