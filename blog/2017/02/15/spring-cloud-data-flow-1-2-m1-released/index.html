Spring Cloud Data Flow 1.2 M1发布

发布
托马斯·里斯伯格
2017年2月15日

我很高兴代表团队宣布发布Spring Cloud Data Flow 1.2的第一个里程碑。

注意：开始使用此新版本的一个好方法是遵循项目页面上的版本列表 ，其中包括下载坐标和参考指南的链接。

在过去的几周中，我们为数据微服务的整体编排添加了新功能并进行了改进。1.2.0中包含以下新功能。M1版本：

核心

  • 为部署属性引入专用前缀。使用部署程序属性非常简单deployer..xxx相对于app..spring.cloud.deployer.xxx
  • 引入新的REST-API控制器和外壳程序支持以清理任务执行
  • 基础工作，以巩固在任务部署和任务执行之间使用控制器
  • 合并REST-API调用跟踪和返回代码以保持一致性
  • 添加基于角色的访问控制，以定义谁有权创建，部署，销毁或查看流/任务。这可以与支持的身份验证方法无缝协作。
  • 针对“流列表”操作的性能优化。新引入的方法不是对与流相关的每个应用程序单独调用，而是MultiStateAppDeployer SPI操作按流调用一个查询，该查询在单个网络调用中查询所有应用程序状态
  • 改进了“流列表”操作的错误报告

仪表板

  • 在仪表板的“关于”选项卡中添加方便的选项，以下载兼容的Shell应用程序
  • 在仪表板中的任务和批处理作业之间添加连接。可以从“任务列表”页面访问批处理作业“详细信息视图”，同样，可以从“批次列表”页面访问任务“详细信息视图”。
  • 将基于角色的访问控制集成添加到仪表板

开箱即用的流应用程序

  • 添加了以下新应用程序，并计划在即将发布的Bacon发布系列中发布它：

    • MongoDB接收器

    • PGCopy接收器

    • 聚合处理器

    • Header-enricher处理器

  • 在app-starters项目中为核心应用程序生成框架添加改进，从而可以有选择地升级相关发行版本。我们可以在每个应用程序级别独立升级Spring Boot，Spring Integration或任何其他依赖项，并更轻松地生成kafka，rabbitmq或任何其他基于活页夹的应用程序。

查看1.2.0。M1（核心） / 1.2.0。M1（ui）发布标记可了解有关增量改进的更多信息。

下一个里程碑

  • 包括核心基础工作以在外壳，dsl和UI中以一流的公民身份支持Docker工件。
  • 编排“批处理作业或任务的组成”的能力正在取得进步。支持Shell / UI的一组新的DSL原语也在开发中。
  • 为了支持“应用程序分组”功能，正在进行围绕控制器，dsl和REST-API的核心结构的重大重构。除了能够编排Spring Cloud Stream或Spring Cloud Task应用程序的能力之外，这种新模型还允许对任何Spring Boot应用程序进行编排。将有一个定义应用程序组的选项，并且可以用“标签”标记这些“组”，因此在“标签”级别执行组操作（如组部署或组销毁）将变得容易。例如，流是“组”的一种特殊化，其中包括源，应用程序和接收器类型的应用程序。
  • 我们设想也将进一步发展“应用程序分组”功能来流版本控制。敬请关注！

来自Spring Cloud Data Flow团队的一些人下周将参加DevNexus 。请考虑参加这些会议，以了解有关这些功能的更多信息。
