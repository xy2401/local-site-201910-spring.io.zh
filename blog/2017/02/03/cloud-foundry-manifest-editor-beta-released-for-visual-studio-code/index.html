Cloud Foundry Manifest editing support for Visual Studio Code (on macOS, Linux x64, and Windows).

Why Visual Studio Code?

Visual Studio Code is a lightweight and open-source code editor that runs on macOS, Linux x64, and Windows. It is based on an interesting architecture with regards to extensibility. Support for languages in Visual Studio Code gets implemented as so called “language servers”. Those language servers are independent of the editor that you use. The editor and the language server are connected using a protocol (called the language server protocol). Even though Visual Studio Code introduced this protocol, other editors and IDEs started to adopt this language server protocol - like the Eclipse IDE (from version 4.7 on) or Eclipse Che as a cloud IDE. Other lightweight editors like Sublime Text and Atom will likely offer support in the near future, too. As a result, we can focus on implementing the Cloud Foundry manifest editor support as an independent language server and you can add this support to the editor or IDE of your choice.

针对Visual Studio Code发行的Cloud Foundry Manifest编辑器Beta

发布
马丁·利珀特
2017年2月3日

作为支持全球开发人员使用Spring构建应用程序并将这些应用程序部署到Cloud Foundry和PCF的活动的一部分，我们很自豪地宣布我们的第一个beta版本的Cloud Foundry Manifest编辑支持 Visual Studio Code （在macOS，Linux上） x64和Windows）。

为什么使用Visual Studio Code？

Visual Studio Code是一种轻量级的开源代码编辑器，可在macOS，Linux x64和Windows上运行。它基于有关扩展性的有趣架构。Visual Studio Code中对语言的支持被实现为所谓的“语言服务器”。这些语言服务器独立于您使用的编辑器。使用协议（称为语言服务器协议）连接编辑器和语言服务器。即使Visual Studio Code引入了此协议，其他编辑器和IDE仍开始采用此语言服务器协议-例如Eclipse IDE（从4.7版开始）或Eclipse Che作为云IDE。 Sublime Text和Atom等其他轻量级编辑器也可能在不久的将来提供支持。因此，我们可以专注于将Cloud Foundry清单编辑器支持实现为独立的语言服务器，并且可以将此支持添加到您选择的编辑器或IDE中。

Visual Studio Code引入了此协议，并为此提供了最好的支持-包括这些扩展的理想市场，这些扩展允许安装和使用它们的过程轻松流畅。因此，作为第一步，我们的Cloud Foundry清单文件的语言服务器现已在该市场上可用。打开Visual Studio Code的安装，转到扩展部分，然后搜索“清单”。将显示Cloud Foundry Manifest编辑器支持。

使用清单文件

安装后，它可以帮助您使用Cloud Foundry清单文件。它为属性和某些值提供内容辅助，验证属性键和值（以避免在部署应用程序时出现拼写错误和意外情况），并提供广泛的悬停帮助。

内容助手还可以提供帮助，帮助您输入某些值，具体取决于您使用的特定Cloud Foundry目标。更具体地说，该工具会推断可用的构建包以及您在Cloud Foundry目标上拥有的服务实例列表。一旦获得该信息，便可以在Cloud Foundry清单文件中轻松定义构建包或服务。

为了获得必要的信息，该工具使用CF CLI配置及其访问令牌。因此，您在CF CLI中配置的任何内容（例如，API端点+空间）都将自动用于按需收集有关buildpack和服务的信息（只要您在CLI上登录到CF目标）。

外表

这标志着我们迈向以IDE无关的方式实现工具的第一步，采用了Visual Studio Code的语言服务器协议-除了我们在基于Eclipse的Spring Tool Suite上的工作以及对Jetbrains的持续支持以改善其出色的Spring工具之外在IntelliJ中更进一步。由于这只是我们迈出的第一步，因此在2017年还会有更多的变化，例如围绕Spring和Spring Boot推出的新语言服务器，清单编辑支持的改进版本以及可供编辑器和IDE插入的更多选项。这些选项之一将保留为Spring Tool Suite，而其他（轻量级）编辑器和IDE也将以这种方式获得出色的Spring和Spring Boot工具。敬请关注。
