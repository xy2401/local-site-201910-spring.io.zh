春天的这个星期-2017年2月7日

工程
乔什·朗（Josh Long）
2017年2月7日

欢迎来到春季本周的另一期！这周我们有很多工作要做，让我们开始吧。

  • 弗雷德·梅洛（Fred Melo）是Pivotal最聪明的人之一，当他谈论数据时，人们会倾听。这是您观看他的SpringOne平台有关数据微服务的演讲的机会
  • RabbitMQ团队一直很忙！RMQ Java客户端在度量，日志记录和Java NIO等方面具有一些非常好的新功能（不间断）。
  • Rabbit团队在RMQ Java Client 4.0中扩展了对指标的支持之后，在博客中发布了有关使用JMX，Spring Boot Actuator和Coda Hale出色的DropWizard指标库的博客。
  • 我喜欢这次来自Confluent的Joe Kutner的SpringOne Platform演讲， 我无法相信这不是一个队列：将Apache Kafka与Spring结合使用 。它着眼于Apache Kafka，这是当今最强大的云原生消息队列之一-Apache Kafka-带有Spring。
  • 查看Spring Integration和消息传递专家Gary Russell的SpringOne Platform在Apache Kafka的Spring中查看
  • 查看Spring和Spring，为Apache Hadoop贡献者Thomas Risberg看用Spring构建大数据管道
  • 我们的朋友俊明希已经建立了一个非常酷 bean 的Graphviz的可视化应用程序中 ，根据访问的/beansviz端点。
  • Spring Cloud贡献者Vinicius Carvalho在SpringOne Platform上关于演化和弹性微服务的演讲值得关注
  • Spring Integration Java DSL负责人Artem Bilan刚刚宣布了适用于Apache Kafka 1.1.3的Spring。
  • Spring Cloud忍者Marcin Grzejszczak 刚刚宣布了Spring Cloud Camden SR5
  • Spring Tool Suite背后的团队正在为Spring Tool Suite 4编写一些令人惊奇的东西，其中很大一部分是从STS中提取的插件作为语言服务器，可以与各种其他IDE一起使用，包括Atom和Microsoft VS代码。作为该计划的一部分， 该团队刚刚发布了Cloud Foundry manifest.yml编辑
  • Spring IO Platform负责人Andy Wilkinson刚刚宣布了Spring IO Platform Brussels RC1
  • 安迪（Andy） 还没有宣布自己的桂冠，他还宣布了Spring IO Platform Athens SR3
  • 您将在本月晚些时候参加在乔治亚州亚特兰大举行的神话般的DevNexus活动吗？Spring团队也将在那里！
  • 似乎我错过了一月份的好时光！查阅Muhammad Noor的关于使用Spring Cloud和基于服务构建基于响应的微服务的文章。
    反应堆
  • Rashidi Zin提出了一个很好的示例，该示例使用Spring Data支持实体审核 。非常酷的拉希迪！
  • Bruno Krebs撰写了一篇非常酷的文章，介绍如何通过测试驱动的开发来构建REST API（自然而然是Spring Boot）
  • Mario-Leander Reimer在OOP17大会上的云原生研讨会看起来非常有趣！我希望我去过那里。但是对于那些不是我们的人， 至少有他的Git存储库 。
  • 说到DevNexus，这个老套但又不错的东西，“ 保持干净：来自Open Boot 和How to Polish的经验” ，来自Spring Boot联合负责人Phil Webb，如果您还没有看的话，绝对是一个不错的手表
  • 严格来讲，与Spring无关，但持续交付是微服务，敏捷，TDD和我们今天理所当然的许多其他实践背后的宏大统一理论，几乎没有人比Chris Farley更好地介绍其背后的原理
  • 这篇文章与Spring本身关系不大，但是它对JVM中对性能的影响没有被覆盖是非常有趣的#hashCode在某些情况下
