Spring Framework 5.0 M5更新

工程
罗森·斯托扬切夫（Rossen Stoyanchev）
2017年2月23日

Spring Framework 5.0的第5个也是最后一个里程碑的更新…

Spring MVC和Spring WebFlux

Spring MVC的名称既广为人知，也被广泛使用，但可能会令您感到惊讶的是，没有实际的项目或使用该名称的独立发行版。而是它是Spring Framework发行版中的一个模块，称为spring-webmvc 。这是另一个琐事问题。您知道模块中的顶级软件包不包含“ mvc”功能吗？而是被称为org.springframework.web.servlet 。实际上，这些是我们不必记住的细节。重要的是，我们有一个简短易记的名称来引用基于Spring的Servlet堆栈的 Web框架。

Spring的反应式堆栈 Web框架是5.0中的新功能，它是完全反应性且无阻塞的。它适用于具有少量线程的事件循环样式处理。它在Servlet容器（Tomcat，Jetty，Servlet 3.1+）上受支持，但在非Servlet运行时（Netty，Undertow）上也受支持，因为该堆栈的通用基础不是Servlet API，而是基于反应式流和反应堆项目。如果您想知道，Servlet 3.1是否能够无阻塞地进行I / O？是的，是的，我们支持在Servlet 3.1容器上运行，但是其余Servlet API是命令式的，不能在反应性，非阻塞式堆栈中使用。

到目前为止，我们还没有为反应式Web堆栈提供专用名称，该名称同时支持Spring MVC注释（即@Controller ， @RequestMapping ）和新的函数式编程模型，这使得讨论和清楚对比编程模型和堆栈成为一个挑战。在我们的里程碑5中， spring-web-reactive模块已重命名为spring-webflux -从Spring Web Reactive API核心的Flux反应类型中汲取灵感和简洁性 ，而我们的较低层反应HTTP抽象则存在于公共环境中spring-web模块。所以我们现在有了spring-webmvc和spring-webflux并排称为Spring（Web）MVC和Spring WebFlux的模块 。如果您想知道：新模块中的顶级软件包是org.springframework.web.reactive 。

现在，要获取此空间中的其他一些关键更新…

网络客户端

的WebClient是反应性，无阻碍性的替代品RestTemplate为反应式和Servlet堆栈应用程序增加价值。它使处理异步和流方案变得毫不费力。

在M5中，我们进行了重大改进，消除了静态导入来指定请求详细信息和执行交换的需要：

WebClient webClient = WebClient.create();

Mono<Person> person = webClient.get()
        .uri("http://localhost:8080/persons/42")
        .accept(MediaType.APPLICATION_JSON)
        .exchange()
        .then(response -> response.bodyToMono(Person.class));

如果所有请求都有一个通用基本URL，则可以将其预先配置一次：

WebClient webClient = WebClient.create("http://localhost:8080");

Mono<Person> person = webClient.get()
        .uri("/persons/{id}", 42)
        .accept(MediaType.APPLICATION_JSON)
        .exchange()
        .then(response -> response.bodyToMono(Person.class));

也可以通过UriBuilder ：

Mono<Person> person = webClient.get()
        .uri(builder -> builder.path("/persons/{id}").build("42"))
        .accept(MediaType.APPLICATION_JSON)
        .exchange()
        .then(response -> response.bodyToMono(Person.class));

WebTestClient

新的WebTestClient在里面spring-test模块构成了Spring WebFlux集成测试支持的基础。它包装一个WebClient并提供用于集成测试的API。相似MockMvc来自Spring MVC Test的新测试客户端不需要实际运行的服务器，并且可以使用模拟请求和响应直接绑定到WebFlux服务器基础结构：

WebTestClient client = WebTestClient
        .bindToController(new PersonController())
        .build();

client.get().uri("/persons/42")
        .exchange()
        .expectStatus().isOk()
        .expectHeader().contentType(MediaType.APPLICATION_JSON_UTF8)
        .expectBody(Person.class).value().isEqualTo(new Person("John"));

但是，新的测试客户端也可以在实时服务器上运行：

WebTestClient client = WebTestClient
        .bindToServer().baseUrl("http://localhost:8080")
        .build();

// Same test case...

流也很容易测试，可能使用Reactor StepVerifier ：

FluxExchangeResult<Person> result = client.get().uri("/persons")
        .accept(TEXT_EVENT_STREAM)
        .exchange()
        .expectStatus().isOk()
        .expectHeader().contentType(TEXT_EVENT_STREAM)
        .expectBody(Person.class)
        .returnResult();

StepVerifier.create(result.getResponseBody())
        .expectNext(new Person("Jane"), new Person("Jason"))
        .expectNextCount(3)
        .consumeNextWith(p -> assertEquals("John", p.getName()))
        .thenCancel()
        .verify();

路径模式解析器

M5版本增加了新的PathPatternParser替代AntPathMatcher可以使用更有效的请求映射解析模式表示形式，并支持非常方便的操作"{*foo}" URI变量语法，用于捕获模式结尾处的任意数量的段。

第一步，新ParsingPathMatcher实施已经允许新PathPatternParser可以轻松插入Spring MVC和Spring WebFlux映射。迈向RC1的下一步，目标是拥有一个模式注册中心，以与请求路径的已解析表示形式进行匹配。

服务器发送的事件和JSON流

使用Spring WebFlux轻松支持流传输：

@GetMapping(path = "/persons", produces = "text/event-stream")
Flux<Person> getPersons() {
    return this.repository.getPersons();
}

上面的代码会将数据流化为服务器发送事件（SSE）格式的浏览器：

data: {"name":"Jane"}
data: {"name":"John"}
...

但是，应针对以下情况进行处理：

@GetMapping("/persons")
Flux<Person> getPersons() {
    return this.repository.getPersons();
}

我们可以流式传输单个JSON对象，但是从整体上来看这不是一个有效的JSON文档，并且浏览器客户端除了使用Server-Sent Events或WebSocket之外，无法使用流。

默认Flux将产生一个JSON数组：

[{"name":"Jane"},{"name":"John"},...]

非浏览器客户，例如WebClient可以请求请求内容类型application/stream+json并且响应将是类似于服务器发送事件的JSON对象流，但没有额外的格式：

{"name":"Jane"}
{"name":"John"}
...
