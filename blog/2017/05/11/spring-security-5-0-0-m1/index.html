Spring Security 5.0.0 M1

发布
罗伯·温奇
2017年5月11日

我很高兴代表社区宣布Spring Security 5.0.0 M1的发布。该发行版包含错误修复，新功能，并且基于Spring Framework 5.0.0 RC1。该版本的重点包括：

  • 对响应式Web应用程序的初始支持
  • 对OAuth 2.0和OpenID Connect 1.0的新支持

对响应式Web应用程序的初始支持

遵循Spring Framework 5.0的主要主题之一，Spring Security 5.0将在Spring的响应式支持的基础上增加对Reactive应用程序的支持。第一个里程碑是将主要基础设施部署到位。

可以在hellowebflux的Spring Security示例中找到使用Spring Security保护Spring WebFlux应用程序的完整示例。

下面是该代码的快速摘要：

@Bean
WebFilter springSecurityFilterChain(ReactiveAuthenticationManager manager) {
  HttpSecurity http = http();
  http.authenticationManager(manager);
  http.httpBasic();

  AuthorizeExchangeBuilder authorize = http.authorizeExchange();
  authorize.antMatchers("/admin/**").hasRole("ADMIN");
  authorize.antMatchers("/users/{user}/**").access(this::currentUserMatchesPath);
  authorize.anyExchange().authenticated();
  return http.build();
}

如果您使用过Spring Security的基于Web的授权支持，则代码看起来应该很熟悉。

要注意的一件事是在反应性应用中access允许注入接口，而不是String 。这允许将lambda或方法引用用于自定义授权逻辑。例如， currentUserMatchesPath看起来像这样：

Mono<AuthorizationDecision> currentUserMatchesPath(
    Mono<Authentication> authentication, AuthorizationContext context) {
  return authentication
    .map( a -> context.getVariables().get("user").equals(a.getName()))
    .map( granted -> new AuthorizationDecision(granted));
}

显然，我们可以只使用lambda而不是方法引用，但这更易于阅读（尤其是当可能有更多规则时）。

对OAuth 2.0和OpenID Connect 1.0的新支持

我们非常高兴在Spring Security 5中引入对OAuth 2.0授权框架和OpenID Connect 1.0的新支持。M1中的最初支持主要集中在OAuth客户端角色上，提供了针对OAuth 2.0提供程序（例如，Facebook）或OpenID Connect 1.0提供程序（例如，Google）对最终用户进行身份验证的功能。

OAuth 2.0登录功能实质上实现了用例“用Google登录”或“用Facebook登录”，并通过利用OAuth 2.0授权框架中指定的授权代码授予流程来实现。

开始学习如何使用OAuth 2.0登录的最佳位置是遵循指南和相关示例 。该指南将引导您通过Google，GitHub，Facebook和Okta设置OAuth 2.0登录 。

OAuth在Spring Security中的未来

当前在以下Spring项目中扩展了对OAuth的支持：

  • Spring Security OAuth
  • 春季社交
  • Spring Cloud安全性
  • Spring Boot的OAuth2 SSO支持

借助Spring Security 5，未来的主要方向是建立对
OAuth 2.0授权框架 ， OpenID Connect 1.0 ， JWT和JOSE（ JWS / JWE / JWK ） 均已安装到Spring Security中。我们的目标是让Spring Security “容纳”较低层协议流的核心逻辑，例如“授权代码授予”流，然后由需要它的各个Spring项目重用，例如Spring Social。 。Spring Security OAuth中已经实现了许多协议流逻辑，但是，由于我们将其合并到Spring Security中，因此决定进行重写。话虽如此，Spring Security OAuth项目处于维护模式，因为我们正在限制新功能，并且只关注错误修复和较小的增强。我们今后的工作将集中于在Spring Security内建立支持。

对OAuth 2.0和OpenID Connect 1.0的新支持的最终目标是在固有复杂的低级协议流之上提供易于使用的抽象。为了快速行动并保持最新状态，我们将Nimbus OAuth 2.0和OIDC SDK用于我们的内部实现。作为进行重写之前的关键的初步准备，我们研究并评估了开放源代码社区中可用的现有OAuth 2.0和OpenID Connect 1.0 Java库，并在此处记录了我们的发现。经过此评估阶段，我们发现Nimbus OAuth 2.0和OIDC SDK是当今可用的最成熟，最全面的Java库，因为它为所有相关规范提供了广泛的支持。

在即将发布的5.0.0 M2版本中，我们计划进一步支持基于响应的安全性以及OAuth 2.0和OpenID Connect 1.0。对于这些新功能的任何反馈，我们将不胜感激。如果您有任何优先事项，请提出问题 。
