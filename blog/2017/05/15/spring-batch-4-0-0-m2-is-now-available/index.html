Spring Batch 4.0.0。M2现在可用

发布
迈克尔·米内拉
2017年5月15日

我们很高兴地宣布Spring Batch 4.0.0。现在可以通过Github和Pivotal下载存储库使用M2。这个版本代表了Spring Batch 4.0版本的第二个里程碑。非常感谢对此版本做出的所有贡献。

什么是新的？

这个里程碑延续了Spring Batch 4.0.0中的工作。M1。具体更新包括：

  • 依赖关系更新
  • 持续的Java配置改进

依赖关系更新

Spring Batch 4将是Spring Boot 2中包含的版本，因此我们花时间来同步我们的依赖项。这样一来，此版本就支持升级到Spring Data Kay发行培训和Spring Integration 5的最新里程碑。

除了清理依赖项，我们还对折旧进行了一些清理。为了支持MyBatis提供的本机支持，已删除了对基于iBatis的读写器的支持。

Java配置

继续改善Java配置功能的故事始于里程碑4.0.0。M2包括以下构建器：

  • StaxEventItemReaderBuilder
  • StaxEventItemWriterBuilder
  • JdbcPagingItemReaderBuilder
  • HibernateCusrorItemReaderBuilder
  • HibernateItemWriterBuilder
  • HibernatePagingItemReaderBuilder
  • JpaPagingItemReaderBuilder
  • SingleItemPeekableItemReaderBuilder
  • SimpleMailMessageItemWriterBuilder
  • SynchronizedItemStreamReaderBuilder
  • CompositeItemWriterBuilder
  • ClassifierCompositeItemWriterBuilder
  • JmsItemReaderBuilder
  • JmsItemWriterBuilder
  • MultiResourceItemReaderBuilder
  • MultiResourceItemWriterBuilder

所有这些现在都可用。而且我们还没有完成所有ItemReader和ItemWriter实现！我们会及时整理它们，以备下一个里程碑。

下一步是什么？

展望下一个版本，我们计划为此版本打包构建器。我们还将更新文档，使其对Java配置更加友好。在Spring Boot 2之前将Spring Batch 4引入GA，以寻找未来几个月的里程碑。

你怎么看？

我们期待您在Jira ， StackOverflow中对这些新功能的反馈，或者通过Twitter @michaelminella直接给我！
