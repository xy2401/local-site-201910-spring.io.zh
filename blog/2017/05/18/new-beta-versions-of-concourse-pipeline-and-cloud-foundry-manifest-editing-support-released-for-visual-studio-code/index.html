针对Visual Studio Code发布了新的Beta版本的Concourse管道和Cloud Foundry清单编辑支持

发布
马丁·利珀特
2017年5月18日

早在2017年2月，我们就开始使用第一个Beta版Cloud Foundry清单编辑支持引入与IDE无关的新工具支持 。如所承诺的，我们将继续改进版本的Cloud Foundry清单编辑对Visual Studio Code的支持，以及对编辑Concourse任务和管道定义的全新支持（也作为Visual Studio Code的扩展），以继续这一旅程。这标志着我们迈出了第二步，即通过与Visual Studio Code兼容的语言服务器协议，以与IDE无关的方式实现工具。

我们还将继续基于Eclipse的Spring Tool Suite的工作，计划于2017年7月对其进行下一次更新（STS 3.9.0将基于Eclipse Oxygen版本），同时将在下一代Spring Tooling中进行开发。与IDE无关的并行方式。我们在此处介绍的那些Visual Studio Code扩展可能与此无关，但它们只是我们正在研究的与IDE无关的Spring工具的更广泛理解和重要小步骤的一部分。

新的Concourse管道编辑支持

大堂CI是一个开源的持续集成和部署工具，它正在枢纽内部使用的各种团体和项目以及枢纽外被越来越多的用户和公司的。在Concourse中，使用三个核心概念（任务，资源和作业）定义构建和部署管道，而docker是创建和运行这些构建环境的核心元素。看看，作为其他CI系统的替代品，绝对值得探索。

基于Concourse的构建的核心部分是管道定义，通常在pipeline.yml文件中进行描述。使用我们为Visual Studio Code构建的编辑支持，使用这些基于YML的管道文件现在变得更加容易。看一下截屏视频，看看实际使用的工具：

更新了Cloud Foundry清单编辑支持

我们还更新了Cloud Foundry清单编辑支持。以下是此Visual Studio Code更新版本的详细信息：

  • 增加了对域和堆栈的动态内容辅助的支持
  • 增加了对路由的支持，包括对路由内部域（包括TCP路由）的内容辅助
  • 增加了对新的运行状况检查类型端点属性的支持
  • 增加了对不赞成使用的运行状况检查类型的“无”值的支持（包括将“不赞成使用的值”替换为“过程”的快速修复）
  • 添加了有关用于动态内容辅助值的CF端点的详细信息
  • 内容助手现在显示嵌套属性的建议

外表

2017年还会有更多的变化，例如围绕Spring和Spring Boot的新语言服务器，以及供编辑器和IDE插入的更多选项。这些选项之一将保留为Spring Tool Suite，而其他（轻量级）编辑器和IDE也将以这种方式获得出色的Spring和Spring Boot工具。敬请关注。
