没有像生产这样的地方。SpringOne Platform将阐明路径。

新闻与活动
乔什·朗（Josh Long）
2017年8月8日

嗨，春季粉丝！

我一年中最喜欢的时间快到了！在北半球寒冷的十二月，人们团聚的那个特殊时刻，人们可以自由无条件地互相赠予；当人们与他们欣赏的晚餐一起进行欢乐的讨论（有时是激烈的辩论）时；当人们彼此开放思想时；当我们回顾过去的一年，并期待明天，下周和下一年；当我们留下美好的回忆时，我们将永远珍惜。就是那个特别的奉献，开放的时刻。不过，我不需要告诉您这个特别的时间！当然，我说的是SpringOne平台一年中的时间！ （我还能说什么？）

我个人几乎无法抑制我的激动！SpringOne平台是我们云原生社区的首要活动。这是一个庆祝开放精神的机会（开放源代码，开放扩展框架，开放平台，开放服务经纪人，开放跟踪；您说的是我们已经开放了它！）在关键社区。

今年的SpringOne平台有望成为一个令人惊叹的展览，里面充满了大创意和更大的代码。最终，我们在Spring团队或Pivotal的任何团队中关心的是帮助组织构建更好，更安全，更快的软件。我们关心的是帮助他们尽快从概念到客户，并帮助他们尽快重复这一周期。我们关心生产。您会发现，生产是不断给予的礼物。

今年见过的Pivotal及其生态系统提供了许多旨在帮助维护和加速这一旅程的事物。SpringOne Platform是了解这些以及其他所有云原生内容的最佳场所。以下是最让我兴奋的一些事情：

  • 功能即服务：Spring Cloud Function将功能即服务范例带到了Spring。
  • Kubo：由BOSH管理的Kubernetes与Cloud Foundry集成。Kubernetes是一种管理容器的强大方法，但是必须有人对其进行操作。我们有。
  • 响应式编程：Spring Framework 5随处可见，并带有响应式运行时和下一代Spring Data，Spring Security，Spring Boot和Spring Cloud的基线。
  • 开放服务代理：用于Cloud Foundry和Kubernetes的有状态平台服务的标准化模型。
  • Spring Boot：Spring Boot仍然是开发人员工具箱中的一个非常有用的工具。找出您错过的内容以及Spring Boot 2.0中的内容。
  • 流处理和分析：随着我们的扩展，数据工作负载的大小也会随之增加。了解如何使用Spring Cloud Stream和Spring Cloud Data Flow通过RabbitMQ和Apache Kafka等消息传递技术对数据进行切片和切块，集成和分析。
  • 微服务：某些定律，例如引力和康韦定律，是无法打破的。通过使用微服务和Spring Cloud优化代码来优化组织。
  • 连续交付：了解结对编程，测试，使用Pivotal Tracker进行项目演变，使用Concourse进行连续交付等更多信息。

希望您能与我和Pivotal团队的其他成员以及各种规模和组织的组织一起参加今年12月在我的家乡旧金山举行的SpringOne Platform ，我们将一起进行生产之旅！
