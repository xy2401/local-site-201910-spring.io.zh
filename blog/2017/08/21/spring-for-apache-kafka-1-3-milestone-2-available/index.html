适用于Apache Kafka 1.3里程碑2的Spring

发布
阿尔特姆·比兰（Artem Bilan）
2017年8月21日

我们很高兴宣布Apache Kafka 1.3版本的Spring Milestone 2可用。

可以从Milestone存储库下载：

repositories {
    maven { url 'http://repo.spring.io/libs-milestone' }
}
compile "org.springframework.kafka:spring-kafka:1.3.0.M1"

（里程碑1出现了阻止程序问题）。

从功能上讲，1.3.x行提供了2.0行的子集（需要Spring Framework 5.0和Java 8），但提供了对Java 7项目和Spring Framework 4.3.x 。

它主要旨在提供对0.11.0.0 Apache Kafka客户端功能的早期访问，但是这里是一个摘要：

  • Apache Kafka 0.11.0.0客户端;

  • KIP-62促进了改进的侦听器容器线程模型；

  • 通过新的Apache Kafka基础，我们通过新的executeInTransaction()操作在KafkaTemplate引入了KafkaTransactionManager和本地事务支持。

  • KafkaListenerContainer现在可以与TransactionManager一起提供，以执行具有事务边界的轮询和偏移提交，并使Kafka事务与某些外部事务管理器同步。

  • 还提供了对新的Kafka标Headers功能的支持。现在，您可以发送和接收带有任意头的消息，例如，使用一些correlationKey头构建请求-回复方案；

  • KafkaEmbedded JUnit类/ Spring Bean现在可以提供任何任意brokerProperties ；

  • 通过扫描应用程序上下文中的NewTopic bean，可以使用KafkaAdmin自动创建主题。

另外，针对Apache Kafka 2.3.0的 Spring Integration扩展。M2可用于合并Spring for Apache Kafka 1.3基础。特别是KafkaProducerMessageHandler可以与被供给KafkaHeaderMapper ，其中DefaultKafkaHeaderMapper默认情况下使用，如果杰克逊JSON处理器存在于类路径。如果还包括对错误处理的增强。

spring-integration-kafke 2.2.0。M1也可用于spring-kafka1.2.x。它包括上述增强的错误处理。

有关spring-kafka，spring-integration-kafka和kafka-clients版本兼容性的完整矩阵，请参见项目页面 。

感谢所有社区成员的反馈和贡献！

预计1.3.0版本将与2.0.0.RELEASE
