Spring 的这个星期-2017年8月1日

工程
乔什·朗（Josh Long）
2017年8月1日

嗨， Spring 粉丝！欢迎来到 Spring 本周的另一期！本周，我在田纳西州的纳什维尔，参加有关Reactive Spring 的聚会演讲 ，然后去堪萨斯州堪萨斯城，重播同样的演讲。两次我都会被朋友马克·赫克勒（Mark Heckler）所吸引。如果您在任何一个城市中，请不要错过！

我正在处理Spring Tips的下一部分：您想看到哪些内容？让我在Twitter上@SpringTipsLive知道 。

这周我们要看的东西确实很多，所以让我们开始吧！

  • Spring消息传递和集成大师Artem Bilan刚刚宣布了适用于Apache Kafka 2.0 M3的Spring 。新版本包括（其中包括）新的KafkaTransactionManager并根据KakfaTemplate 。
  • Spring Session 2.0 M3发布 -Spring Security和Session忍者Joe Grandja刚刚宣布了Spring Session 2.0。M3。这个新版本增加了对Spring WebFlux及其支持WebSession和反应式编程。
  • 克里斯托弗·斯特罗布（Christoph Strobl） 也不为所动， 还宣布发布了Spring Data Kay RC1和RC2 。这些发行版包括适用于Apache Geode的Spring Data，响应式调整，Cassandra v3.0驱动程序升级等等，其中包括120多个已修复的票证！
  • Christoph Strobl刚刚宣布刚刚宣布了一个大规模的Spring Data Ingalls SR5版本，修复了40多个问题。
  • Spring Integration 5.0里程碑6可用 -Spring Integration忍者Artem Bilan刚刚宣布了Spring Integration 5.0M6。该版本包括对响应式HTTP通道适配器的支持，与Spring Cloud Function的便捷集成，新的JDBC元数据存储等。
  • Spring Data忍者Thomas Risberg宣布了Spring Cloud Data Flow 1.2.3维护版本，其中包括OSS许可的SQL Server JDBC驱动程序和改进的参考文档。
  • Spring Boot和Cloud Foundry的忍者Madhura Bhave刚刚宣布了Spring Boot 2.0.0。M3引入Spring Framework 5RC3，并关闭了100多张票！立即获取位！
  • Spring Batch主管Michael Minella刚刚宣布了Spring Batch 4.0.0。M3 。这个新版本进一步改进了Java Configuration构建器，以适应各种ItemReader和ItemWriter s。它还将终止Spring Batch Admin。您应该改为使用Spring Cloud Data Flow。
  • 欢呼！Spring Session Data Geode 2.0.0。M1现在可用。这是一个基于Spring Data Geode（也支持Pivotal的Gemfire产品）的Spring Session集成。Spring Data Geode的@ClientCacheApplication和Spring Session Data Geode的@EnableGemFireHttpSession批注将您的Spring Boot应用程序变成一个Geode缓存客户端，该客户端可以通过将数据分发到Geode服务器集群来管理HTTP会话状态，以实现快速，可靠（复制）的访问。Spring Data传奇人物John Blum和团队的工作做得很好！这是一个很酷的版本。
  • Spring Vault负责人Mark Paluch发布了Spring Vault 1.1.0。M1和2.0.0。M2 。这两个新版本都包括AWS IAM Vault登录，基于通用密钥的租约期限的轮换以及对VaultEndpointProvider动态配置端点。此外，2.0.0。M2包括身份验证DSL，响应式Spring Framework 5集成以及增强的可空性支持。
  • Spring Boot 1.5.6现在可用-Spring Boot 1.5.5发行版中存在一个小问题，因此请改用1.5.6。
  • Spring Session MongoDB 2.0.0。M2发布 - Spring 传奇人物Greg Turnquist刚刚宣布了Spring Sessions MongoDB 2.0.0。M2。这个新版本引入了新的依赖关系，并与Spring Framework 5的响应式支持集成。
  • Spring Security OAuth忍者Joe Grandja 刚刚宣布了Spring Security OAuth 2.2.0.RELEASE 。这是一个打包的版本，所以祝贺Spring Security团队做得很好！
  • Spring Framework 5 Kotlin API的功能方式 -SébastienDeleuze着眼于Spring Framework 5对Kotlin语言的本机支持中的许多不错的改进。这里有太多爱！某些工作（例如关注可空性检查）也通过使用Spring改善了Java开发人员的生活，并改善了他们的生活。
  • 尽管我们向任何想要管理和询问Spring Batch作业的人推荐Spring Cloud Data Flow，但您仍然可以使用Spring Batch Admin，甚至可以在Spring Boot环境中运行它，如Joris Kuipers的示例所示 。
  • ADTMag 对最近的Spring Framework 5.0RC3版本有很好的了解 。
  • Hootsuite的Kotlin时代的黎明–代码@ Hootsuite -Hootsuite谈论了他们对Kotlin语言的使用。在本文中，他们将探讨他们的动机以及将其用于Android开发时遇到的问题。
  • Verizon的Josh Stone谈到大规模运行Cloud Foundry的经验
  • 我的朋友哈里·张 （ Harry Zhang）指出 ，容器存储接口将是一个非常有前途的项目，因为它将涉及Cloud Foundry，Kubernetes，Mesos和Docker的参与者。
  • 由于兴趣的减弱，Coding Dojo显然已经放弃了Ruby on Rails的训练营， 现在正在运行Spring研讨会 。我个人最近几年已经看到了这种效果。我认为这说明了两点：a）Ruby on Rails对业界来说是一个了不起的福音b）如果人们选择使用它代替Ruby on Rails，那么Spring必须做得正确！为什么不自己尝试呢 ？
  • Codecentric的Johannes Edmeier分享了一个好消息：Codecentric Spring Boot Admin Server的下载量已超过600万 ！这是非常有希望的消息！在此处了解有关基于Spring Boot的应用程序的各种仪表板选项的更多信息
  • 为什么PCF是Spring Boot的最佳平台 -Pivotal自己的Toshiaki Maki汇总了为何Cloud Foundry（尤其是Pivotal的Cloud Foundry）对于使用Spring的开发人员来说是最佳体验 。
  • Michael Kops有一篇很好的文章，介绍了如何通过Web测试框架Karate测试基于Spring Boot的REST API 。
  • 慈善专业学院有一篇很棒的文章， “操作：现在是每个人的工作！” 敦促软件开发人员采用注重运营的思维。如果我们作为一个社区要转向更高阶的系统和可交付成果，那么这一点至关重要。
  • Dan Vega 在为Spring MVC端点配置多个HTTP请求映射方面有一篇不错的文章。
  • Grails项目负责人Graeme Rocher 刚刚宣布了Grails 3.3。GA 。看看这个！
  • 是否需要功能全面的示例Spring Cloud应用程序？您可能会喜欢Benjamin Wilms制作的这个富有弹性的演示 。
  • Spring的创建者Rod Johnson着眼于他的公司Atomist的技术，该技术可自动执行并大大简化项目初始化 。Atomist不仅关注代码生成，还关注云平台集成，生命周期，存储库，Slack集成等基础架构。
