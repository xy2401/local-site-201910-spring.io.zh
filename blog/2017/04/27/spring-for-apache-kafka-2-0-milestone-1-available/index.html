适用于Apache Kafka 2.0里程碑1的Spring

发布
加里·罗素
2017年4月27日

我们非常高兴地宣布Apache Kafka 2.0发行版2.0.0的Spring的第一个里程碑可用。M1 。

2.0系列中的重要新功能包括：

  • 支持时间戳KafkaTemplate 。

  • 寻求话题的开始/结束。

  • KIP-62促进了新的线程模型-现在，消费者客户端不再依赖poll()被频繁调用的线程模型要简单得多。避免需要内部pause() / resume()处理。现在总是在使用者线程上调用侦听器。反过来，这促进了：

  • ConsumerAwareMessageListener （和BatchConsumerAwareMessageListener ），以便侦听器实现可以访问Consumer对象执行诸如pause() ， resume() ， metrics()等等

  • @KafkaListener POJO方法现在可以用注释@SendTo将方法结果发送到其他主题。

@KafkaListener(id = "replyingListener", topics = "inTopic")
@SendTo("replyTopic")
public String replyingListener(String in) {
	return in.toUpperCase();
}

有关更多信息，请参见使用@SendTo转发侦听器结果 。

  • @KafkaListener注释现在有一个errorHandler属性，允许为每个配置自定义错误处理程序。以前，您必须为每个容器使用不同的容器工厂。

  • 嵌入式kafka服务器JUnit @Rule在spring-kafka-test现在可以作为Spring Bean提供（并自动连接到您的测试中）。为了进一步简化，框架可以为您自动声明Bean。只需添加@EmbeddedKafka参加您的测试班。有关更多信息，请参见@EmbeddedKafka注释 。

此外，Spring Integration Kafka扩展的第一个3.0里程碑可用（3.0.0。M1）基于此spring-kafka里程碑。

Spring里程碑存储库中提供了这两个项目的里程碑版本 。

我们非常感谢您迄今收到的反馈（和贡献）。

注意
 不幸的是，spring-kafka pom对传递有不正确的依赖spring-messaging-5.0.0.BUILD-SNAPSHOT 。我们不建议在生产中使用里程碑，但是为了提供一个稳定的测试平台，您应该覆盖此依赖关系以5.0.0.M5 。
