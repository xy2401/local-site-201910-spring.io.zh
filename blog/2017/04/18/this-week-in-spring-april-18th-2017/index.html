Spring 的这个星期-2017年4月18日

工程
乔什·朗（Josh Long）
2017年4月18日

嗨，春季粉丝！欢迎来到春季本周的另一期！本周，我在佐治亚州亚特兰大市，亚特兰大DevOps日和波士顿举行了敏捷技术联盟会议 。如果您愿意，我很想见您-打个招呼 ！

在即将到来的春季芝加哥 ， 纽约和亚特兰大活动中，不要错过我，春季团队成员和社区发言人！

和往常一样，我们有很多内容需要介绍，让我们开始吧！

  • Spring Framework 5.0中的WebFlux功能反应式支持真的很受欢迎！我特别喜欢这篇名为《 Spring ：下一个微框架》的文章。
  • Baeldung在介绍WebFlux功能反应性端点方面有出色的文章
  • 乔纳森·波伦斯坦（Jonathan Borenstein）进行了一个简单的演示， 探讨了如何使用Spring Framework 5.0构建功能性REST服务。
  • 欢呼！Spring Cloud Dalston版本现已发布！ 拿出来试试吧！此版本包含了许多很棒的新东西！
  • Spring Cloud Task忍者Glenn Renfro 刚刚宣布了Spring Cloud Task 1.2.0。RC1
  • Spring Cloud Stream负责人Marius Bogoevici 刚刚宣布了Spring Cloud Stream切尔西GA
  • Spring Data负责人Oliver Gierke刚刚宣布了Spring Data Kay M2，其中包含许多改进和错误修正！趁热拿到！
  • 上周我结束了Spring Tips系列的第二季 。继续阅读，回顾最近11集以及更多内容！
  • Spring Cloud Vault负责人Mark Paluch刚刚宣布了Spring Cloud Vault 1.0
  • 我与我的朋友和同事播客迈克尔·科特上实现微服务与DEVOPS 。或者其他的东西。
  • 定期提醒SDKMAN！ 很棒，您可以使用它来安装Spring Boot：sdk install springboot
  • 使用JVM？（我当然希望如此！）您可能会关心关于弃用的讨论java.lang.Object#finalize
  • 切线：ZeroTurnaround推出了一个很棒的备忘单，其中涵盖了一些技巧和窍门git去年
  • Java Pub House播客对Spring Batch和Spring Cloud Task Lead Michael Minella进行了很好的采访
  • ŁukaszMonkiewicz 在Spring Boot上根据请求验证整理了一篇不错的文章
  • 友善的提醒：微服务，云计算，敏捷以及您将听到的Pivotal谈论的其他一切背后的目标最终是实现持续交付，此处对一些租户进行了详细说明
  • 来自Adobe的Lars Trieloff撰写了一篇非常酷的文章，讨论了REST与Webhooks之类的技术之间的范式转变，您仍在进行REST，但它们变得更像是消息传递体系结构中的主题
  • 我喜欢这篇文章RabbitMQ：Windows Server的我的替代ServiceBus
  • 说到RabbitMQ：您是否看到过新的RabbitMQ网站？是炸药！
  • Pivotal Cloud Foundry现在具有内置支持，可实现持久持久性
  • 我喜欢Neoj首席科学家Jim Webber 关于Neo4j如何实现集群共识的介绍的这篇文章
  • Cloud Technology Partners播客对Pivotal的Richard Seroter进行了精彩的采访， 探讨了Cloud Foundry的未来
  • JaxEnter在微服务趋势方面进行了很棒的德语调查]（ https://jaxenter.de/microservices-trends-55937 ）
  • Baeldung也有使用大后的@JsonComponent使用Spring Boot注释
