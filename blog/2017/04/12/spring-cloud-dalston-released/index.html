Spring Cloud Dalston发布

发布
斯宾塞·吉布
2017年4月12日

我很高兴代表社区宣布， Spring Cloud Dalston发行培训的一般可用性（RELEASE）现已开始提供。该版本可以在Maven Central中找到。您可以查看Dalston 发行说明以获取更多信息 。

Dalston发布系列的重大变化

金库

Spring Cloud Vault Config是一个新项目，可通过Hashicorp Vault为分布式系统中的外部化秘密管理提供客户端支持。

配置服务器

Config Server现在通过Composite模式支持多个后端。这允许组合以前不可能的后端类型，例如：Vault和git或多个git后端。现在支持对AWS Codecommit托管的git存储库的身份验证。在以前的（Camden）版本中，已经将Hashicorp Vault作为后端添加到Config Server，以与基于VCS的后端一起使用。

Spring Cloud Commons

自从核心Spring Cloud模块（Commons，Config，Netflix，Bus）开始以来， DiscoveryClient已自动将运行中的客户端注册到发现服务器。这是最初的Netflix Eureka实施的副作用。现在，服务注册和注册服务的发现是单独的关注点。现在有一个ServiceRegistry Spring Cloud Netflix，Consul和Zookeeper都实现的界面。默认情况下，客户端仍会自动注册，但是可以通过autoRegister的属性@EnableDiscoveryClient或通过属性动态地这还允许以编程方式注册每个JVM的多个服务。寻找将来的博客文章以获取更多详细信息。基于配置DiscoveryClient也可以。

支持@LoadBalancedAsyncRestTemplate由社区贡献。

侦探

对通用上下文传播系统的支持已添加到Sleuth。它被称为“行李”，它允许使用Sleuth已经建立的工具通过边界（例如HTTP或消息传递）传递任意属性。

另一个功能是基于注释的跨度创建以及日志和标签的添加。这不仅生成更少的样板代码，而且在使用在运行时生成实现的框架（例如Spring Data）时也很有用。

合同

合同得到了增强，以支持合同输入和输出的更大灵活性。与阅读Groovy DSL相比，增加了协议支持来读取合同。使合同更具可插拔性，允许自定义DSL格式，测试和存根生成以及存根运行器实现。

领事

除了支持新的ServiceRegistry API，Spring Cloud Consul支持Consul功能以注销僵尸服务（即，在运行状况检查中未经过可配置的时间的服务将从Consul中删除）。

其他

Dalston基于Spring Boot 1.5.x，这是最低要求的版本。其他更改包括对各种库的升级，错误修复，文档和完善。非常感谢所有社区对请求请求和问题的贡献。

通过BOM使用Maven入门（仅依赖管理）

<dependencyManagement>
  <dependencies>
    <dependency>
      <groupId>org.springframework.cloud</groupId>
      <artifactId>spring-cloud-dependencies</artifactId>
      <version>Dalston.RELEASE</version>
      <type>pom</type>
      <scope>import</scope>
    </dependency>
  </dependencies>
</dependencyManagement>
<dependencies>
  <dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-config</artifactId>
  </dependency>
  <dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-eureka</artifactId>
  </dependency>
  ...
</dependencies>

或使用Gradle：

buildscript {
  dependencies {
    classpath "io.spring.gradle:dependency-management-plugin:1.0.0.RELEASE"
  }
}
apply plugin: "io.spring.dependency-management"

dependencyManagement {
  imports {
    mavenBom 'org.springframework.cloud:spring-cloud-dependencies:Dalston.RELEASE'
  }
}

dependencies {
    compile 'org.springframework.cloud:spring-cloud-starter-config'
    compile 'org.springframework.cloud:spring-cloud-starter-eureka'
    ...
}
