Spring Boot的新Gradle插件

工程
安迪·威尔金森
2017年4月5日

Spring Boot 2.0 M1的主要主题之一是对其Gradle插件进行了一系列重大改进 。其中的许多改进都已合并，并且可以在最新的Spring Boot快照中使用。距离Spring Boot 2.0.0还有几周的时间。M1将于5月初发布，我们很乐意在此之前听到您对新插件的早期反馈。

您可以在其参考和API文档中阅读有关该插件功能的更多信息。

尝试新插件

新的插件需要Gradle 3.4或更高版本。与Spring Boot的其余部分一样，快照也会发布到https://repo.spring.io/libs-snapshot 。使用快照的最简单方法是在start.spring.io上创建一个新的Gradle项目，然后选择Spring Boot 2.0快照。

或者，您的build.gradle应该看起来像这样：

buildscript {
    ext {
        springBootVersion = '2.0.0.BUILD-SNAPSHOT'
    }
    repositories {
        mavenCentral()
        maven { url 'https://repo.spring.io/libs-snapshot' }
    }
    dependencies {
        classpath "org.springframework.boot:spring-boot-gradle-plugin:${springBootVersion}"
    }
}

apply plugin: 'java'
apply plugin: 'org.springframework.boot'
apply plugin: 'io.spring.dependency-management'

构建可执行的jar和Wars

的bootRepackage任务已被替换为bootJar和bootWar分别用于构建可执行jar和war的任务。两项任务都扩展了其等效的标准Gradle jar或war任务，使您可以访问所有常规配置选项和行为。

依赖管理

Spring Boot的Gradle插件不再自动应用依赖项管理插件 。相反，Spring Boot的插件现在可以通过导入正确版本的spring-boot-dependencies宝这使您可以更好地控制如何以及何时配置依赖项管理。对于大多数应用程序而言，应用依赖管理插件就足够了：

apply plugin: 'io.spring.dependency-management'

请注意，依赖项管理插件仍然是spring-boot-gradle-plugin因此无需将其列为classpath对你的依赖buildscript组态

提供反馈

就像我上面说的，Spring Boot 2.0.0还有几周的时间。M1已发布，因此现在是试用新插件的绝佳机会，然后请告诉我们您的想法。

该插件仍在开发中。新版本使许多以前版本无法实现的事情成为可能。而且，以前可能发生的一些事情可能不再可能，或者可能需要其他方法。特别是，故意将插件的DSL剥离了，以仅涵盖我们确定想要的功能。
