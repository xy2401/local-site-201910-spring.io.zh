宣布Reactor Core 3.1的第一个候选版本

发布
西蒙·巴斯莱（SimonBaslé）
2017年9月18日

我很荣幸代表反应堆小组宣布反应堆堆芯3.1.0.RC1已被释放🎉。这是月底向GA发布3.1迈出的最后一步，长期支持版本将支持Spring 5 ！

它还由各种发行版作为补充，所有发行版都捆绑在一起Bismuth-M4发布培训和BOM：

  • reactor-test ， reactor-extra ， reactor-adapter和reactor-logback都切入3.1.0.RC1
  • reactor-netty在新版本中看到了重大更新和错误修复0.7.0.M2里程碑
  • reactor-kafka已在其发布1.0.0.M4里程碑

为了得到这个版本中，最好的办法就是使用BOM，作为参考指南中所述这里 。确保阅读有关里程碑的部分（将其转换为Bismuth-M4 ）。

反应堆堆芯的重要变化

由于自MILESTONE 1以来我们没有发布太多公告，因此让我们更深入地了解一下reactor-core 3.1.0，其中更改最多。

我们保留在3.1.0之前更改API的选项。发布，这使我们能够从社区和Spring Framework团队收集大量反馈。因此，我们能够完善该API并提出我们认为100％适合于Spring Framework 5.0长期使用的内容。

自从Reactor 3.1.0的第一个MILESTONE以来，这些更改就逐步进行了，您可以访问Github上的发行说明以获取完整的更改日志，尤其是如果您自从3.0.7.RELEASE 。

以下是这些更改的详尽介绍：

助焊剂和Mono API的更好对齐

  • Mono和Flux错误处理运算符已进行了调整。（M1）
  • Mono#flatMap已经成为flatMapMany和Mono.then（ Mono then(Function other) ） 就是现在flatMap 。（M1）
  • 元组返回Mono.when静态方法和Mono#and现在zip和zipWith ，与他们的一致Flux同行。注意when和and 3.1.0中仍然存在这些方法。RC1，但它们仅针对只对完成几个感兴趣的情况Publisher对象（表示为Mono返回类型）。（RC1）
  • 的Flux#firstEmitting现在称为静态方法first和Flux#firstEmittingWith现在调用实例方法or 。（RC1）

更好的操作员一致性

  • 一种*When后缀用于带有伴侣的运算符变体Publisher或异步执行与未添加后缀的任务相同的任务。（M1）
  • 所有delayError删除了参数变体，以便添加后缀为DelayError （例如flatMapDelayError ）。（M1）
  • 所有*Millis变体（以long持续时间）已被删除， Duration基于变体。（M1）

这些更改通常具有消除关于哪个替代项的任何歧义的额外好处。
使用lambda时适用的方法，尤其是像Kotlin这样的语言。

单声道专用API

Mono有几个API用于处理任务。他们基本上忽略了元素（如果有的话），只传播onComplete信号。

在RC1之前，这些API还将具有一些覆盖，这些覆盖将不会专注于此任务完成用例，从而带来一些混乱。自RC1起，这些变体均已重命名（例如Mono and(Mono other)已重命名zipWith ）。

保留的面向任务完成的变体可以通过以下事实发现： Mono 。他们现在可以与任何Publisher ，而不仅仅是Monos：

  • when静态方法立即触发几个提供的任务，并等待所有任务完成
  • and从a调用时执行相同的操作Mono ，提供一个Publisher
  • then是关于忘记源的元素而仅表示其完成

其他变化

  • reactor-test来源已移至与reactor-core现在工件在io.projectreactor Maven中的groupId。（M2）
  • Hooks累积（M3），可以使用键（RC1）进行部分设置/重置
  • WorkQueueProcessor和TopicProcessor有很多构造选项，现在表示为Builder （M2）
  • Processor Reactor中的实现已被大量重做，并且外观Sink已创建界面以简化与处理器的直接交互。使用Processor#sink()以获得并保留对此外观的参考。（M3）
  • 一种Flux / Mono可以用多个标签tag注释或name注解。（M3）
  • 现在可以将Context订阅反应式序列后，使用subscriberContext(Context) 。该上下文与订阅信号一起在上游传播。（RC1）

从这里的路

接下来的两周将是关于准备发布GA的全部内容，除了文档和一些新的操作员之外，我们预计不会对此进行任何其他更改。

但这也是您尝试更改并提供反馈的最后机会 ！因此，请尽快提供帮助，并提供任何反馈（在GitHub上或通过Gitter chat ）返回给我们。
