Spring Cloud Data Flow 1.3.0。M2发布

发布
格伦·伦弗罗（Glenn Renfro）
2017年9月18日

我们很高兴宣布1.3.0。M2版本的Spring Cloud Data Flow及其相关项目生态系统。

本地服务器： 入门指南 。

仪表板/浮动

在Dashboard / Flo的1.3版本的第二部分中，我们介绍了支持流和任务/批处理操作的核心功能。

继续基于Angular4的基础架构升级 ，流和任务/批处理工作流现在包括现代外观，并且还改进了可用性。

文档 ， 测试覆盖率和webpack捆绑分析器都进行了重大改进和添加。

扇入和扇出

由于社区，客户和现场的普遍需求，此版本包括对数据管道的扇入和扇出可视表示的支持。下图显示了扇动工作：

同样，还有一个选项可以在拓扑结构内切换主要流。只需单击一下即可-就这么简单！

直接命名通道控件

Spring Cloud Data Flow始终包括对与生产者，使用者或两者同时与命名目的地交互的数据管道的支持。此版本增加了与它们进行可视化交互的能力，并使构建复杂的拓扑变得更加容易。连接到同一目标的生产者和消费者的组合可以有“ n”个，这对于涉及多种数据源和目标的体系结构而言非常强大。下图显示了复杂拓扑的示例：

App Registry和Maven更新政策

从公共或私有Maven工件解析Maven工件的用户现在可以利用“ update-policy ”功能。您可以使用此功能来覆盖和刷新Spring Cloud Data Flow的内部Maven缓存。例如，在开发中，您可以通过设置持续解析Maven工件的SNAPSHOT版本update-policy=always ，这将强制下载DSL /仪表板中正在使用的流或批处理/任务应用程序的最新版本。

安全注意事项

鉴于Spring Cloud Data Flow 提供了传统的OAuth安全支持 ，并要求对其配套服务器（例如spring-cloud / spring-cloud-dataflow-metrics-collector ， spring-cloud-task-app-starters /由composite-task-runner和即将发布的spring-cloud / spring-cloud-skipper组成 ，我们将通用的安全基础结构提取到了一个独立的库中。spring-cloud / spring-cloud-common-security-config库将在以后的版本中在伴侣服务器中重用。

外壳改进

此版本为流和任务/批处理名称以及其他元数据添加了“自动补全”。不用再猜测了-一切都在TAB之外！请查看以下截屏视频，以了解有关高级Shell功能，技巧和窍门的更多信息：

版本兼容性

此版本带来了Spring Boot 1.5.7的兼容性，并且基础Spring Cloud基础结构已更新为Dalston。SR3。有关更多详细信息，请查阅1.3.0 M2发行说明 。

展望未来，我们的目标是1.3.0 M3，然后是候选版本，然后是2017年10月的一般可用性版本。
