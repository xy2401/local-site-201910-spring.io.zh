Spring Shell 2 is a complete rewrite of Spring Shell, leveraging newer components (such as JLine 3) and applying better modularization. Spring Shell 2 is also built with Spring Boot in mind, making use of auto configuration and other boot features.

The internal architecture now uses a pluggable model to discover what methods to turn into commands, how to parse user input into argument values and how to handle return values. This is very similar to the approach taken by Spring MVC for example and allows extensions of the framework in ways that were not previously possible. Users of Spring Shell would typically not care though, only dealing with the new “standard” command API to define their commands.

推出Spring Shell 2.0M1！

发布
埃里克·博塔德
2017年9月18日

我们很高兴地宣布Spring Shell 2.x的第一个发行里程碑！

经过两年的开发， Spring Shell 2完全重写了Spring Shell，利用了更新的组件（例如JLine 3）并应用了更好的模块化。Spring Shell 2在构建时也考虑了Spring Boot，并利用了自动配置和其他启动功能。

内部体系结构现在使用可插入模型来发现哪些方法可以转换为命令，如何将用户输入解析为参数值以及如何处理返回值。例如，这与Spring MVC所采用的方法非常相似，并且允许以以前不可能的方式扩展框架。不过，Spring Shell的用户通常不会在乎，仅处理新的“标准” 命令API即可定义其命令。

功能亮点

下面的视频中突出显示了一些新功能（请确保全屏运行），但是这里有一个更详尽的摘要：

  • 一项长期要求的功能，现在支持位置参数 （甚至可以在某种程度上混合使用）
  • 使用标准的Spring转换API可以重用许多现有的转换器，而Shell 1有其自己的专用方法
  • 在JLine 3之上进行构建可带来不错的用户体验增强，例如使用TAB在完成建议中导航
  • 与bean验证API的集成使命令更具表现力
  • 新的help 命令比以前更有用，类似于命令的输出man页
  • 接受集合或数组值的命令参数现在可以指定非默认Arity
  • 另一个长期要求的功能，现在支持多行命令
  • 仍然支持打开和关闭命令，现在具有更自然的编程模型，并且能够解释当前为何不可用命令

让我们来看看它！

下面的简短视频使用具有以下签名的示例命令显示了某些功能：

public String addItem(

 	@ShellOption(help = "unit price") @Min(0) float price,
 	@ShellOption(help = "the number of items to add", defaultValue = "1") @Range(min = 1, max = 15) int qty
 ) { ... }

@ShellMethod("Submit the order for processing.")
public List<?> submitOrder() { ... }

下一步是什么

这只是充满新功能的第一个里程碑。在接下来的几周中，我们计划修复一些错误，完成编写全面的参考文档并重构一些内部组件，从而最终实现2.0.0。发布Spring一平台 。

同时，请尝试并提供反馈！
