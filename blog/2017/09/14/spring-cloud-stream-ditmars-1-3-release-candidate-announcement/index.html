Spring Cloud Stream Ditmars / 1.3版本候选公告

发布
加里·罗素
2017年9月14日

我们很高兴地宣布，候选发行版Spring Cloud Stream Ditmars。RC1可在Spring Milestone存储库中使用。发行说明包括有关与Spring Boot，Spring Cloud，Spring AMQP和Spring for Apache Kafka的版本兼容性的相关信息。

Apache Kafka的Kafka流

此版本的目标是推广Apache Kafka活页夹实现中作为顶级项目的Apache Kafka支持的Kafka Streams 。通过将Apache Kafka的Kafka Streams定位为一流公民，开发人员现在可以通过在绑定级别利用Kafka Streams API来构建Spring Cloud Stream应用程序，因此实现实时数据处理业务要简单得多。功能。开发人员可以专注于功能，而不是基础架构和消息传递中间件配置细节，Spring Cloud Stream通过即开即用地解决了这些问题。除了编写标准的流处理应用程序之外，用于Kafka Streams的Spring Cloud Stream绑定器还使开发人员能够编写利用Kafka Streams的交互式查询功能的应用程序。查看样本以获取更多详细信息。

Apache Kafka 0.11

受到客户和社区的普遍需求，我们很高兴宣布Apache Kafka 0.11对Spring Cloud Stream的支持。鉴于基础项目（适用于Apache Kafka的Spring和适用于Spring Integration Kafka的Spring）在其下一个主要版本中将更明确地添加0.11支持，因此该团队通过将其反向移植到Spring Cloud Stream的预览版中，确保将其提供0.11。次要版本。因此，有一个0.11分支明确添加了对Apache Kafka 0.11发行版的支持。对于有兴趣升级到此版本的早期采用者，我们将其作为独立的工件发布 。类似地， kstream绑定器存在0.11的伪像 。

注意
 提醒一下，Apache Kafka活页夹实现程序的Spring Cloud Stream 2.0版本将默认为Apache Kafka 0.11，并且它将是该下一个主要版本中唯一支持的版本。Spring Cloud Stream的1.3.x版本行将支持较旧的Apache Kafka版本。

错误处理

为了进一步改善错误处理支持，我们简化了生产者级别和消费者级别的异常处理 。错误通道的创建和编排以及错误通道的全局表示也已在此发行版中重新实现。

除此之外，该版本还包含各种错误修复和改进。有关更多详细信息，请参阅发行说明中的项目级别发行标记。

对于下一步，我们计划在本月底之前发布Spring Cloud Stream Ditmars发布系列的一般可用性。与往常一样，我们欢迎您提供反馈和意见，因此请通过Stackoverflow或GitHub或通过Gitter与我们联系。

有关更多信息，请参见项目页面 。
