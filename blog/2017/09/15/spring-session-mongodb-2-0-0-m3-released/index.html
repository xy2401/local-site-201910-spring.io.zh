Spring Session MongoDB 2.0.0。M3发布

工程
格雷格·特恩奎斯特
2017年9月15日

亲爱的春季社区，

Spring Session MongoDB 2.0.0。M3被释放。它基于：

  • Spring Session 2.0.0。M4

  • Spring Data Kay-RC3

  • 铋反应堆-M4

  • Spring Framework 5.0.0。RC4

在此版本中，添加了一些新功能，以简化与Spring WebFlux应用程序的结合使用。

@EnableMongoWebSession
public class SpringWebFluxConfig {

}

您所要做的就是套用@EnableMongoWebSession任何您的Spring配置类以激活MongoDB的会话支持。此外，您必须提供一个ReactorMongoOperations Spring bean，但是如果您使用的是Spring Boot的spring-boot-starter-data-mongodb-reactive入门，已经提供了。

@EnableMongoWebSession具有设置整个会话范围的替代功能，包括maxInactiveIntervalInSeconds和collectionName ，以匹配与基于servlet的功能相同的功能@EnableMongoHttpSession注解。

注意
 @EnableMongoWebSession本身适用于弹簧@Configuration注释，因此无需将相同的注释应用于您自己的配置类。

默认情况下，Spring Session MongoDB使用基于JDK的序列化策略，并且经过验证可支持将安全性上下文详细信息写到MongoDB中，从而使其似乎无法与Spring Security集成。

如最近的Spring Session博客文章提到的 ，Spring Session MongoDB（通过Spring Session和Spring WebFlux）利用基于cookie的会话处理。这意味着在创建新会话时，该应用将向客户端发送回set-cookie SESSION=…​响应标头，还检查传入的Web请求中是否有一个名为SESSION 。但是使用Spring Framework 5.0.0。M4，现在有机会从cookie切换到基于标头的策略。

要从cookie更改为标题，只需将此Spring bean添加到您的配置中：

@EnableMongoWebSession
public class SpringWebFluxConfig {

	@Bean
	HeaderWebSessionIdResolver headerBasedSessionIdResolver() {
	    return new HeaderWebSessionIdResolver();
	}
}

创建新会话时， HeaderWebSessionIdResolver会产生一个SESSION=…​响应标头客户端可以接收。传入的Web请求将被解析为SESSION=…​请求标头。

也可以更改标题的名称，如下所示：

@EnableMongoWebSession
public class SpringWebFluxConfig {

	@Bean
	HeaderWebSessionIdResolver headerBasedSessionIdResolver() {
	    HeaderWebSessionIdResolver resolver = new HeaderWebSessionIdResolver();
	    resolver.setHeaderName("MyCustomHeaderName"); // Use this instead of SESSION for the header
	    return resolver;
	}
}

有了这些，您就可以使用Spring Session MongoDB进行会话管理，并根据需要进行调整！

重要
 过去，如果Spring Session MongoDB在类路径上检测到Jackson，它将使用基于Jackson的序列化。由于我们已经切换到了基于JDK的序列化，并且没有完全基于类路径设置进行跳转，因此建议所有正在运行的应用程序都必须删除现有会话并重新初始化。

同时，如果您访问项目站点，获取坐标并在您的Spring Boot应用程序中包含版本号 ，今天就可以获取。
