Spring Data Neo4j 2.1.0 Release Candidate 4发布

发布
迈克尔·汉格
2012年9月18日

在Spring Data GA发布培训即将到来之前，我们希望收集有关Spring Data - Neo4j中的修复和更新的一些反馈。因此，我们发布了Release Candidate 4。

这里是对该候选版本所做的更改的快速概述，虽然比我们预期的要长得多，但仍然非常有用。

2.1.0版中的更改。RC4（2012-08-19）

  • DATAGRAPH-296更新为Neo4j 1.8的稳定版本。更改API时采用RC1，空间，cypher-dsl和java-rest-binding
  • DATAGRAPH-285添加用于保存和删除的应用程序事件
  • DATAGRAPH-263，DATAGRAPH-212，DATAGRAPH-272，DATAGRAPH-147重构了派生查询的创建，增加了对多个索引字段和所有查询关键字的支持
  • DATAGRAPH-294导出的用于数字索引值的查找器方法
  • DATAGRAPH-293通过graph-id查找对象
  • DATAGRAPH-275 TypeRepresentationStrategyFactory在空图上的非正常回退
  • DATAGRAPH-246允许实体作为派生查找器的参数
  • DATAGRAPH-281添加了对枚举和日期的支持，作为Cypher的参数

Spring工具套件和Groovy / Grails工具套件3.1.0。M1发布

发布
马丁·利珀特
2012年9月14日

亲爱的春季社区，

我很高兴地宣布第一个里程碑版本3.1.0。Spring工具套件（STS）和Groovy / Grails工具套件（GGTS）的M1。

此里程碑的重点包括：

  • 现在，我们提供基于Eclipse 3.8和Eclipse 4.2的发行版
  • Mylyn更新至3.8.1
  • Mac应用程序现已在OSX 10.8 Mountain Lion中签署了Gatekeeper
  • Maven对Grails项目的支持

我们的一些用户报告了STS 3.0.0附带的新Eclipse Juno 4.2平台的一般性能问题。这些问题中的大多数似乎与基础UI平台及其在各种插件和扩展中的用法有关。由于我们的目标是提供最佳的用户体验，因此我们决定为STS 3.1.0提供两个单独的发行版。M1：一种是在Eclipse Juno 3.8之上构建的，另一种是在Eclipse Juno 4.2之上构建的。使用基于4.2的版本有困难的人应该尝试使用基于3.8的发行版。

欢迎来到春季本周的另一个装置！我将去挪威奥斯陆参加JavaZone会议，与人们讨论如何在Cloud Foundry上使用Spring Integration和Spring Batch 。再次，这是一个自然的用例：Cloud Foundry使其易于扩展以处理最大的工作负载，而Spring Integration和Spring Batch（大概位于RabbitMQ之上）负责管道工作，并在整个工作范围内进行繁重的工作负载分配集群。

 <P>Wrapping up from last week's Cloud Foundry   Open Tour - India, touring <a href="http://opentour.cloudfoundry.com/2012/bangalore">Bangalore</a>, and <a href="http://opentour.cloudfoundry.com/2012/pune">Pune</a>, I got great feedback about using Spring MVC for multi-client applications   with REST. We looked at deploying those applications to <a href="http://cloudfoundry.org">Cloud Foundry</a>, the open-source PaaS from   VMware, where you can control the range of channels that your Spring MVC/REST   architecture can deliver to. It's been a truly exciting time. Cloud Foundry   gives developers a platform to deploy the Spring applications they want to   build, how they want to build them, without having to worry about infrastructure   and middleware concerns. </p>
