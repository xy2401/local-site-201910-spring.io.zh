发布了Spring Data Hopper SR4和Gosling SR6

发布
奥利弗·德罗博姆
2016年10月3日

我谨代表整个团队宣布Spring Data发布火车有两个服务版本可用：Hopper SR4和Gosling SR6。建议使用这些版本的升级，因为它们包含Spring Data JPA中CVE的修复程序 。

Spring Boot 1.3.8和1.4.1用户可以直接使用Hopper SR3和Gosling SR5。这些版本也已经包含CVE的修复程序，但不幸的是破坏了Java 6的兼容性。要重新实例化后者，请通过设置spring-data-releasetrain.version属性设置为相应的值。非引导用户只需升级到新的BOM 。与往常一样，下一个Spring Boot版本将在依赖项升级中获得改进的修复程序。

Spring Statemachine 1.2.0版。M1发布

发布
珍妮·瓦尔基亚拉赫蒂
九月29，2016

我们很高兴地宣布Spring Statemachine 1.2.0的第一个里程碑版本。M1 。可以从Spring Repository获得工件。

我们达到的第一个里程碑是：

  • 通常的错误修复和小的增强
  • 支持UML子机
  • 新的存储库抽象将计算机配置保留在外部存储库中
  • 对国家行动的新支持。
  • 新的过渡错误操作概念

从一开始就支持复杂的状态层次结构，但是麻烦的是，必须在相同的状态配置中定义所有内容。新的子机支持增加了一种功能，该功能可以将状态与其他机器关联，从而允许一定程度的重用，并使配置更加简洁。
