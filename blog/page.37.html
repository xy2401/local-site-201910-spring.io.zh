Spring CredHub 2.0.0。M1发布

工程
斯科特·弗雷德里克
十一月5，2018

我很高兴地宣布Spring CredHub 2.0.0的第一个里程碑版本，现在可以在Spring里程碑存储库中使用 。

CredHub

CredHub为Cloud Foundry平台提供集中的凭据管理。CredHub实现了HTTP API以安全地存储，生成，检索和删除各种类型的凭据。

Spring CredHub为CredHub API提供了Java绑定，可轻松将Spring应用程序与CredHub集成。

什么是新的

2.0.0版本带来了对CredHub服务器版本1.x和2.x以及CredHub v1和v2 API的支持。此版本还提供与Spring Framework 5和Spring Boot 2的兼容性。Spring CredHub 2.0.0的GA版本将为现有接口添加一个反应性替代方案。

Spring Security OAuth2自动配置2.0.6和2.1.0发布

发布

十月31，2018

我很高兴代表社区宣布Spring Security OAuth2启动自动配置 2.0.6和2.1.0。

两种版本均主要提供错误修复和依赖版本更新以及一些小的改进。特别值得注意的是，这些与Spring Boot的最新 版本保持一致。

  oauth2:
    authorization:
      jwt:
        key-value: ${PRIVATE_KEY}

Spring Session Bean GA发布

发布
罗伯·温奇
十月31，2018

该帖子由VedranPavić撰写

我很高兴代表社区宣布Spring Session BOM Bean的全面上市。这是基于Spring Session 2.1的第一个发行版，可以很容易地与新发行的Spring Boot 2.1一起使用 。请继续阅读以了解该版本的重点。

相同站点Cookie支持HttpSession积分

Same-Site Cookie是另一种机制， 可帮助开发人员保护其免受Cross-Site Request Forgery的侵害 。我们的DefaultCookieSerializer进行了增强以支持添加SameSite属性赋予Spring Session产生的会话Cookie。的SameSite默认情况下，属性启用为value Lax并且可以使用DefaultCookieSerializer#setSameSite 。

请注意，对WebSession从Spring Framework 5.1开始，它就存在于Spring WebFlux本身中。
