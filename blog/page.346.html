Spring LDAP 1.1发布

发布
马蒂亚斯·阿瑟森（Mattias Arthursson）
2006年8月28日

亲爱的春季社区，

我们很高兴地宣布，Spring LDAP 1.1版已经发布。这是该库的第一个发行版，作为Spring Framework子项目。下载 | 说明文件 | API

关于Spring LDAP
Spring LDAP是一个Java库，用于基于Spring的JdbcTemplate模式简化LDAP操作。该框架减轻了用户查找和关闭上下文，遍历NamingEnumerations，编码/解码值和过滤器等的负担。

LdapTemplate类封装了传统LDAP编程中涉及的所有管道工作，例如创建，遍历NamingEnumerations，处理Exception和清理资源。这使程序员可以处理重要的事情-在哪里查找数据（DN和筛选器）以及如何处理数据（与域对象进行映射，绑定，修改，取消绑定等），方法与JdbcTemplate减轻负担的方式相同除了实际的SQL之外的所有程序员，以及数据如何映射到域模型。

验证逻辑（和我的第一篇文章！）

工程
科林·耶茨
2006年8月25日

大家好！

自从我上个月加入Interface21以来，这是我的第一篇帖子。我以前的博客现已正式弃用，我将不再进行更新。

那么我的第一篇文章的主题是什么（除了自我介绍之外）？验证逻辑。这不会是如何在Spring框架中执行验证的演练，而是将讨论我的一个特殊的bug :)

特别是，我想确切地讨论验证逻辑应包含的内容。这似乎是一个明智的选择。 “验证指定数据的逻辑”。好吧，这很容易理解，但请继续阅读:)。
如您所知，Spring框架通过Errors和Validator接口为您的验证提供了一个不错的抽象层。特别是在Validator中，您可以将业务特定的验证规则应用于填充的域对象。Spring出色的绑定支持负责根据某些输入来更新您的域模型，验证程序负责确保填充的域模型在语义上是正确的。
那我的臭虫熊是什么？我一次又一次地跨应用程序运行，这些应用程序允许验证逻辑从验证器中滴入控制器（对于Web应用程序），或更糟糕的是进入中间层。在人们开始提出问题之前；我并不是说验证不属于中间层，而是说验证器是放置验证逻辑的地方！
