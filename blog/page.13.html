发布
罗伯·温奇
2019年6月18日

该帖子由VedranPavić撰写

我很高兴代表社区宣布Spring Session的发布Corn-M2和Bean-SR6 。这些版本将由Spring Boot挑选2.2.0.M4和2.1.6.RELEASE ， 分别。

春季会议Corn-M2

的Corn-M2发布基于：

  • Spring Session核心模块2.2.0.M2

  • 春季会议数据Geode2.2.0.M2

  • 春季会议数据MongoDB2.2.0.M3

春季会议的一些亮点2.2.0.M2是：

  • 基于Redis的简单实现SessionRepository

  • 重做@Configuration类现在与proxyBeanMethods=false

  • 将项目测试迁移到JUnit 5

  • 简化的项目结构

这些版本的完整详细信息可以在changelog中找到。

SimpleRedisOperationsSessionRepository

该版本的最大亮点是基于Redis的新的，简单的实现。 SessionRepository它是众所周知的替代品RedisOperationsSessionRepository 。

原本的RedisOperationsSessionRepository ，位于核心之上SessionRepository功能，为会话事件提供支持（已转换为HttpSessionEvent实例）并实现FindByIndexNameSessionRepository （允许检索给定主体的会话）。对这两个功能的支持需要付出一定的代价，因为在Redis中需要如何保留会话存在一些复杂性。

对于许多应用程序来说，对会话事件和主体索引的支持不是必需的，这是在Windows 2000中提供替代方法的主要动机。 SimpleRedisOperationsSessionRepository 。新的SessionRepository尚未在Spring Session的配置基础结构中提供一流的支持，因此可以按以下方式进行配置：

@EnableSpringHttpSession
public class RedisSessionConfiguration {

    @Autowired
    private RedisConnectionFactory redisConnectionFactory;

    @Bean
    public RedisOperations<String, Object> sessionRedisOperations() {
        RedisTemplate<String, Object> redisTemplate = new RedisTemplate<>();
        redisTemplate.setConnectionFactory(this.redisConnectionFactory);
        redisTemplate.setKeySerializer(new StringRedisSerializer());
        redisTemplate.setHashKeySerializer(new StringRedisSerializer());
        return redisTemplate;
    }

    @Bean
    public SimpleRedisOperationsSessionRepository sessionRepository(
            RedisOperations<String, Object> sessionRedisOperations) {
        return new SimpleRedisOperationsSessionRepository(sessionRedisOperations);
    }

}

考虑给予Corn-M2释放并SimpleRedisOperationsSessionRepository旋转一下，让我们知道您的反馈！

2019年6月18日

微服务架构很棒，但是随着您的应用程序编程接口（API）开始增长，与它们的维护相关的挑战也随之增加。

例如，随着现有API的成熟和添加新功能，它将需要客户及其陪伴。当API的详细信息发生更改时，客户需要进行调整才能使用这些更改。此过程需要时间，并且确实会减慢API的开发速度并干扰您快速迭代的能力。

提供多种API会带来一系列挑战。您如何将请求和响应路由到正确的API？您如何处理任何邮件差异？当端点可以移动时，如何为客户提供支持？
