Spring LDAP 1.2.1发布

发布
乌尔里克·桑德伯格
2007年12月8日

亲爱的 Spring 社区，

我们很高兴地宣布，Spring LDAP版本1.2.1已发布。这是一个更新版本，其中添加了新的池库并修复了1.2中的一些问题。下载 | 变更日志

更重要的变化的摘要：

  • 添加了池库，该库具有比内置池更灵活的连接验证和更好的配置。非常感谢Eric Dalquist的贡献。（ LDAP-85 ）

  • 修复了SimpleLdapTemplate中错误的目标方法被调用的错误。（ LDAP-93 ）
  • 使AbstractContextSource中的createContext受保护，而不是私有包装。（ LDAP-94 ）

Spring Framework 2.5发布

发布
于尔根·霍勒（Juergen Hoeller）
2007年11月19日

亲爱的 Spring 社区，
 
我们很高兴地宣布，Spring Framework 2.5最终版本现已发布。

下载 | 技术支持 说明文件 | 变更日志 

Spring 2.5通过许多令人兴奋的新功能增强了Spring 2.0，其中包括：

  • 完全支持Java 6和Java EE 5（JDBC 4.0，JTA 1.1，JavaMail 1.4，JAX-WS 2.0）
  • 功能齐全的注释驱动的依赖项注入，包括对“限定符”的支持
  • 支持在类路径中自动检测应用程序组件并将其自动配置为Spring托管对象
  • AspectJ切入点表达式中的新bean名称切入点元素
  • 内置对基于LoadTimeWeaver抽象的AspectJ加载时编织的支持
  • 新的XML配置名称空间“ context”和“ jms”，以提供最大的便利
  • 完全修订的集成测试框架，具有对JUnit 4和TestNG的一流支持
  • 一个新的基于注释的Spring MVC控制器模型，支持Servlet和Portlet环境
  • 扩展的SimpleJdbcTemplate功能，包括对命名SQL参数的支持
  • 官方认证的WebSphere支持
  • 将Spring Framework jars包装为兼容OSGi的捆绑包
  • 可以将Spring ApplicationContext作为JCA RAR文件部署，用于无头应用程序模块
  • JCA 1.5消息端点管理，用于Spring管理的JMS和CCI消息侦听器

Interface21成为SpringSource

工程
罗德·约翰逊
2007年11月19日

我们正在更改名称。本周，Interface21将成为SpringSource。

在我们建立公司的过程中，Interface21因其卓越的产品，思想领导力，杰出的人才，敬业精神以及优质的支持和服务而赢得了声誉。随着我们继续交付所有这些产品，我们相信更改我们的名称将有助于我们的公司将它们带给更广泛的受众。

当我2004年创立Interface21时，我不得不选择一个名字。我相信Spring是企业Java的未来，“ Interface21”反映了这些感觉-21世纪的框架。现在我们已经进入21世纪。事实证明，Spring比我想象的要成功得多，并且已经成为企业Java的事实上的标准。在.NET平台上，它也越来越受欢迎。全球数以百万计的人已经下载了Spring Portfolio产品。所有主要的Spring提交者都在这里工作。

Spring Web Flow 2.0愿景

工程
基思·唐纳德
2007年11月15日

Spring Web Flow 2.0 M2刚刚发布 。我对此发布感到特别兴奋，因为它为实现我们对社区未来的大胆愿景奠定了基础。在这篇文章中，我将解释该愿景是什么，以及该基础将实现什么。我还将详细介绍Web Flow 2.0的体系结构，并将其与1.0版本进行比较。

Spring Web Flow 2.0愿景


*更新：自从2007年 Spring 体验以来，考虑了Spring社区的大量反馈之后，上述愿景已于1/11/08更新。根据这些反馈，计划于2008年3月发布的Spring Web Flow 2.0将继续专注于Web应用程序中受控导航和应用程序事务的编排，可在基于动作的MVC环境和JavaServerFaces中用作Spring MVC的补充。在基于组件的环境中。当与JSF一起使用时，Spring Web Flow 2.0可以作为“黑匣子”为整个基于JSF的Web应用程序提供动力，或者可以与实现自由导航要求的标准JSF导航处理程序混合使用。 因此，2.0将是 演化版本增加了对JSF和Ajax的一流支持，支持Java 1.4+，并提供与SWF 1.0流定义语言的完全向后兼容性。
