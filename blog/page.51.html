1.6 M1的改进领域：

  • DSL和部署属性解析

  • 任务执行状态

  • 组成的任务运行器安全

  • 仪表板

DSL和部署属性解析

使用自定义参数启动任务是一种在运行时以不同行为影响Task应用程序的好方法。想象一下，影响接受时区作为执行时区特定数据处理参数的批处理作业（作为任务运行）。在此版本中，我们调整了解析逻辑，以将键值对作为值包括在内。感谢社区提供报告，向我们提供反馈以及共享他们的用例。

在查看基于内联与基于属性文件的流和任务定义的属性的解析规则时，社区发现了行为上的差异，我们已记录了该差异以作为一般指导 。

春季社交报废公告

工程
克雷格·沃尔斯
七月3，2018

今天，我对未来充满乐观，我宣布Spring Social的生命已结束，在过去的八年中，我很高兴能与这个项目一起工作。

随着Spring Security 5的发布，Spring Social的连接框架（我认为是Spring Social最有价值的部分）所提供的许多功能现在已成为Spring Security的一部分。实际上，我在三月份的博客文章中分享了如何使用Spring Security对客户端OAuth2的新支持。而且，随着Spring Security继续发展其客户端OAuth支持，Spring Social的连接框架将变得越来越不必要。将Spring的OAuth故事散布在如此多的项目中不再有意义，现在是时候为Spring开发人员提供一个单一的OAuth故事了。

在撰写本文时，MongoDB多文档事务在单个副本集上受支持，感觉就像您可能从关系数据库中熟悉的事务一样。查看驱动程序API，您会立即感到宾至如归：

try (ClientSession session = client.startSession()) {

    session.startTransaction();

    try {

        collection.insertOne(session, documentOne);
        collection.insertOne(session, documentTwo);

        session.commitTransaction();

    } catch (Exception e) {
        session.abortTransaction();
    }
}
