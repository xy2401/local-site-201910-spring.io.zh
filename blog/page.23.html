Spring Tools 4.2.0发布

发布
马丁·利珀特
2019年3月29日

亲爱的春季社区，

我很高兴地宣布适用于Eclipse，Visual Studio Code和Atom的Spring Tools 4的4.2.0版本。

此版本的重点包括：

  • （Spring Boot）新增功能：允许为LSP进程“ PropertiesLauncher”配置虚拟机参数（ ＃211 ）
  • （Spring Boot）性能：通过跨语言服务器启动缓存创建的符号，大大改善了符号索引基础结构的性能
  • （Spring Boot）性能：用与JDT（语言服务器）的通信代替了内部类型索引，以节省时间和内存开销来保持我们自己的类型索引
  • （Eclipse）新：发行版更新到Eclipse 2019-03
  • （Eclipse）新增功能：现在默认在Spring Boot启动配置中启用Spring JMX支持
  • 各种其他错误修复和改进

适用于Apache Geode和Pivotal GemFire 1.0.0的Spring Boot。M4发布！

发布
约翰·布鲁姆
三月22，2019

我荣幸地代表Spring，Apache Geode和Pivotal GemFire社区宣布发布适用于Apache Geode和Pivotal GemFire的Spring Boot （SBDG）。 1.0.0.M4 。

对于@starbuxman表示特别感谢，感谢他在SpringOne Platform 2018之后的宝贵反馈。而且，如果您还没有在Apache上观看Josh的Spring Tip 视频 ，请查看。

什么是新的

在一些次要更新中，里程碑4添加了以下新功能：

  • 提供了Spring Boot的配置处理器和SBDG的外部配置支持@ConfigurationProperties类（有关更多详细信息，请参见此处 ）。在STS或IJ Enterprise附加产品内部，开发人员将具有自动完成和内容辅助的附加便利。

  • GemFire / Geode节点现在可以使用spring.application.name属性。

  • GemFire / Geode Spring Boot启动程序现在包括Spring Shell作为运行时依赖项，在启用和使用GemFire / Geode的管理服务时非常方便。

  • 提供了针对Spring Cloud Commons中问题的解决方法。

有关更改的完整列表，请参见changelog 。

您可以从Spring获取钻头libs-milestone资料库：

马文
<repositories>
    <repository>
        <id>spring-libs-milestone</id>
        <name>Spring Milestone Maven Repository</name>
        <url>https://repo.spring.io/libs-milestone</url>
    </repository>
</repositories>

<dependencies>
    <dependency>
        <groupId>org.springframework.geode</groupId>
        <artifactId>spring-geode-starter</artifactId>
        <version>1.0.0.M4</version>
    </dependency>
</dependencies>
摇篮
repository {
  maven { url "https://repo.spring.io/libs-milestone" }
}

dependencies {
  compile 'org.springframework.geode:spring-geode-starter:1.0.0.M4'
}

要简单地从Apache Geode切换到Pivotal GemFire，请将Spring Boot启动器更改为spring-geode-starter至spring-gemfire-starter 。

2019年3月21日

谨代表团队和所有贡献者，宣布Spring Initializr已更新，现在可以在start.spring.io上使用 。

用户界面更改-听取您的反馈

在上次修订之后，我们收到的主要反馈是关于缺少依赖项列表。一些社区成员似乎是出于信息目的而查看此列表（不一定使用网站来生成新应用程序），而其他社区成员只是想看看可用的东西。
