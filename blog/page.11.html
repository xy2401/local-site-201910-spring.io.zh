适用于Apache Geode和Pivotal GemFire 1.1.0的Spring Boot。M3发布！

发布
约翰·布鲁姆
2019年7月3日

我很荣幸代表Spring，Apache Geode和Pivotal社区宣布发布适用于Apache Geode和Pivotal GemFire的Spring Boot。 1.1.0.M3 。

什么是新的

此版本的主题是增加对混合云部署的支持。

例如，也许您想在Pivotal CloudFoundry （PCF）上推并运行Spring Boot，Apache Geode或Pivotal GemFire应用程序，但将这些应用程序连接到外部管理的独立Apache Geode或Pivotal GemFire集群。现在，SBDG允许您执行此操作。

从技术上讲，SBDG利用PCF中称为CUPS的功能，或创建用户提供的服务 。通过定义自己的服务描述符，您可以将Spring Boot应用程序连接到外部管理的服务，例如数据库，消息队列，甚至是内存中的数据网格和缓存，例如Apache Geode或Pivotal GemFire 。

通过遵循规定的服务描述符格式，可以轻松地在需要时切换到平台管理的服务，例如Pivotal Cloud Cache 。

确实，我们相信这对于用户进入PCF这样的Cloud-Native平台是一个重要的垫脚石，在该平台上，他们仍在传统架构上进行了大量投资。

我们强烈建议为您的应用程序和微服务缓存需求使用诸如Pivotal Cloud Cache之类的托管服务，但是我们也意识到，进行此切换并不像翻转该切换那样容易。

我们想帮助您到达那里。毕竟， 云原生（数据）模式对于现代应用程序架构至关重要，尤其是在谈论诸如高可用性和弹性以及有效利用计算资源之类的内容时。

您可以在参考文档中了解有关此新支持的更多信息。

除了支持混合云，我们还添加了：

  • 自动配置日志。

  • 当多个实例绑定到您的Spring Boot应用程序时，支持针对特定的Pivotal Cloud Cache（PCC）服务实例。

  • 升级到Spring Boot2.1.6.RELEASE.

  • 升级到Apache Geode和Pivotal GemFire （STDG）的Spring Test 0.0.5.RELEASE 。

请参阅更改日志以获取更多详细信息。
