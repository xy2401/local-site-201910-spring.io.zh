Spring Security OAuth2-客户端身份验证问题

工程
乔·格兰贾
2016年8月31日

最近报告了问题＃808 ，该问题允许用户通过客户端身份验证并通过client_credentials或密码授予流获取访问令牌。

当客户端和用户具有相同的标识符（clientId和用户名）时，就会发生这种独特的情况。用户的凭据在client_credentials或密码授予流程中用于客户端身份验证，并且成功获取了具有客户端权限的访问令牌。

修复

此错误已在1ed986a中修复，并在2.0.11中发布。发布。

如果您使用的是基于Java的配置，请更新至2.0.11。发布。

但是，如果您使用的是基于XML的配置，请执行以下操作：

  • 更新至2.0.11。发布

  • 查看此JUnit测试及其相关的XML配置，以确保AuthenticationManager用于客户端身份验证和AuthenticationManager用于用户身份验证的设置与您的配置相同。

  • 作为预防步骤，确保你的XML配置不设置相同，在此JUnit测试和相关的XML配置 ，因为它证明了原来的问题。

用于Mesos 1.0 RC2的Spring Cloud Data Flow发布

发布
托马斯·里斯伯格
2016年8月26日

我们很高兴宣布1.0.0。RC2是Mesos的Spring Cloud Data Flow的候选版本，这是一个团队的工作，其中包含了许多新功能。

此发行候选版本以Spring Cloud Data Flow的最新1.0 GA版本为基础。一些重点包括：

  • 现在，我们在Marathon（Mesos的容器编排平台）上将Spring Cloud Data Flow Server作为Docker映像运行。
  • 此版本增加了功能来支持流分区和扩展
    • 当前，接收器和处理器的分区和缩放是通过使用多个应用程序部署来处理的，每个应用程序实例一个，由名称后附加的索引标识。

    • 通过使用其他应用程序实例来处理源扩展。
  • 现在，可以使用Marathon 应用程序组来部署流，因此更容易识别组成流的不同应用。
  • 我们增加了对使用Chronos（Mesos的容错作业调度程序）启动任务的支持。
