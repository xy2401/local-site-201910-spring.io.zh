2014年5月22日

我很高兴地宣布Spring Web Services 2.2.0。RELEASE已发布！这是2.2发布周期中的第一个版本。2.2中的主要新功能是引入了对Spring-WS的代码配置支持。这意味着您现在可以使用简单的配置Spring-WS @EnableWs注解。例如：

@Configuration
@EnableWs
@ComponentScan(basePackageClasses = { MyConfiguration.class })
public class MyWsConfiguration {

  // @Beans go here
}

有关此主题的更多信息，请参考@EnableWs的javadoc 。您还可以在更新的参考文档中阅读有关此新功能的更多信息。要查看更改的完整列表，请参阅changelog 。

Grails 2.4发布

发布
格雷姆·罗彻（Graeme Rocher）
2014年5月21日

今天我们很高兴地宣布Grails 2.4的发布。这是2.x版本中的最终版本，其中包括许多重要更新。

最重要的是，Grails 2.4.x附带了Spring IO Platform的关键部分，包括Groovy 2.3和Spring 4.0.5 。

发布要点

除了包含最新版本的Spring和Groovy之外，此版本中还包含许多关键的新功能，包括：

  • 独立的GORM和GSP-是的， Hibernate 4的 GORM和MongoDB的GORM都可以在Spring Boot应用程序或Groovy脚本的Grails之外使用。
  • 静态编译 -使用Groovy静态编译功能的扩展，现在可以静态编译与Grails框架的许多交互，包括动态查找器。
  • Asset Pipeline插件 -Grails 2.4用新的Asset Pipeline插件替换了以前的Resources插件，用于管理静态资产（CSS，JavaScript等）。
  • GORM子查询 -GORM已得到增强，并为相关子查询提供了广泛的支持。
  • 新的Maven插件 -Maven插件已被重写为使用Aether进行依赖关系解析，现在可与多个版本的Grails一起使用。

Spring 的这个星期-2014年5月20日

欢迎回到春季本周的另一期！本周，我在波兰的克拉科夫参加Geecon ，这是波兰的开发者大会，当然，我将在此与开发者谈论Spring。 （也许还有Spring）。如果您在附近，找到我，我会穿着那件巨大的Spring leaf T恤！ :)

除此之外，还有很多事情要做，让我们开始吧！

  • 为了为即将到来的Spring IO Platform做准备，Spring Data发布火车Dijkstra已发布！这是一个巨大的发行系列，其中包括：JPA，MongoDB，Neo4J，Apache Solr，Couchbase，Cassandra，Elasticsearch，Gemfire，Redis和Data REST！祝贺Spring数据团队 ！
  • Spring Boot 1.1.0 M1现在可用，并引入了MongoDB和Gemfire支持，以及改进的执行器指标和运行状况端点。
  • Spring Framework 3.2.x和4.0.x版本的最终维护版本现已发布 ！
  • Spring Integration忍者（Rockstar！）Artem Bilan撰写了一篇不错的文章，介绍了在新兴的Spring Integration Java配置DSL中所有令人惊叹的Java配置支持，它基于基本的@EnableIntegration刚刚发布的Spring Integration 4.0中提供了支持。您应该阅读该帖子。说真的我想摘录这个惊人的代码片段来窃取Artem的风头：

    java @Bean IntegrationFlow helloWorldFlow() { return IntegrationFlows.from("helloWorldInput") .filter("World"::equals) .transform("Hello "::concat) .handle(System.out::println) .get(); }是的这是一个处理输入消息的Spring Integration流， filter是他们， transform s，然后将它们提供给escape-hatch方法， handle ，使开发人员可以将任何行为插入混合。请记住，您可以对此进行任何更改-包括从何处获取消息以及向何处写入消息。实际上，一个流的输出可能是另一个流的输入。恭喜，Spring Integration团队！另外，请务必检查启动网络研讨会重播 ！

  • 我知道我这个上周提到的，但它是如此值得再读取！Groovy 2.3.0来了 ！（哈！知道了！本周的链接指向同一作者关于Groovy 2.3.0发行版的不同文章！但是，无论如何，您不高兴吗？快点，Groovy，快点！
  • 我的好朋友Pieter Humphrey 在Spring XD上做了一个很好的介绍性截屏视频，展示了如何启动和运行流处理，以及如何在不到7分钟的时间内将其连接到分析仪表板。XD使用看似简单的DSL（特定于域的语言）并且没有Java代码-使用Hadoop从未如此简单。
