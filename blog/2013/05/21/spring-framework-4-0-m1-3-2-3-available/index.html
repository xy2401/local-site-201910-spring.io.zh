Spring Framework 4.0 M1和3.2.3可用

工程
于尔根·霍勒（Juergen Hoeller）
2013年5月21日

尊敬的Spring社区，

我很荣幸地宣布迈向Spring Framework 4.0的第一个里程碑 ，这是我们在以下几个关键主题上所做的工作的第一部分：


  • Java SE 8 / OpenJDK 8支持的第一波（以下详细信息）
  • 最初对JMS 2.0，JPA 2.1，Bean验证1.1和JSR-236并发的支持 （来自EE 7总括）
  • Bean定义的通用条件机制的第一个切入点（@Profile的@Conditional）
  • Spring的WebSocket终结点模型 （Rossen稍后将对其进行详细介绍）

关于Java SE 8，我们在框架的多个方面支持OpenJDK 8 build 88+：


  • 在组件扫描和子类生成中支持1.8字节码格式
  • 针对Spring回调接口的lambda表达式和方法引用
  • 用于Spring数据绑定和格式化的JSR-310 Date-Time值类型
  • 初步支持JDK 8 参数名称发现机制

注意：
Spring的Java 8支持正在进行中，正在跟踪OpenJDK 8快照，这些快照将在9月的JDK 8 Developer Preview日期 （请参阅参考资料）中找到。 http://openjdk.java.net/projects/jdk8/ ）。我们希望在那时左右发布相应的Spring Framework 4.0发行候选版本，该候选版本将在SpringOne（9月9日至12日；请参见 http://springone2gx.com ）。

与4.0 M1一起，我们发布了Spring Framework 3.2.3 ，其中包含针对最近报告的问题的修复程序，同时还包含OpenJDK 8运行时支持。Spring Framework 3.2.x将支持针对根据JDK 7 （带有-target 1.7）或更早版本编译的应用程序在JDK 8运行时上进行部署 。请注意，它不支持JDK 8的字节码格式（目标值为1.8，这是lambda所需的）。为此，请升级到Spring Framework 4.0。

享受您在OpenJDK 8上使用Spring的初次体验，并让我们知道它如何为您工作！

干杯，
于尔根
