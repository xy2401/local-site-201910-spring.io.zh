通过拉取请求为Spring Boot做出贡献

工程
格雷格·特恩奎斯特
2013年9月20日

万一您错过了今年的SpringOne 2GX会议，最热门的主题之一就是Spring Boot的发布 。Dave Syer展示了如何使用适合单个推文的代码快速创建Spring MVC应用程序。在此博客文章中，我将剥开Spring Boot的封面，并通过组合拉取请求向您展示它的工作方式。

自动配置

Spring Boot具有强大的自动配置功能。当它在类路径上检测到某些东西时，它将自动创建bean。但是它尚不具备的一项功能是对Spring JMS的支持。我需要该功能！

第一步是编写一个自动配置类：

package org.springframework.boot.autoconfigure.jms;

. . .some import statements. . .

@Configuration
@ConditionalOnClass(JmsTemplate.class)
public class JmsTemplateAutoConfiguration {

	@Configuration
	@ConditionalOnMissingBean(JmsTemplate.class)
	protected static class JmsTemplateCreator {

		@Autowired
		ConnectionFactory connectionFactory;

		@Bean
		public JmsTemplate jmsTemplate() {
			JmsTemplate jmsTemplate = new JmsTemplate(connectionFactory);
			jmsTemplate.setPubSubDomain(true);
			return jmsTemplate;
		}

	}

	@Configuration
	@ConditionalOnClass(ActiveMQConnectionFactory.class)
	@ConditionalOnMissingBean(ConnectionFactory.class)
	protected static class ActiveMQConnectionFactoryCreator {
		@Bean
		ConnectionFactory connectionFactory() {
			return new ActiveMQConnectionFactory("vm://localhost");
		}
	}

}

我的Spring JMS自动配置类标记有Spring的@Configurationannotation，将其标记为要拾取并放入应用程序上下文的Spring bean的源。它利用Spring 4的@Conditionalannotation，将其限制为仅在JmsTemplate在类路径上时才添加这组bean。这是一个标志，表明spring-jms在类路径上。完善！

我的新类有两个内部类，它们也标记为Spring Java Configuration和附加条件。这使整合所有我的配置需求变得容易，以自动化Spring JMS配置。

  • JmsTemplateCreator创建一个JmsTemplate 。它仅在其他地方尚未定义JmsTemplate才有效。这就是Spring Boot如何创建JmsTemplate的看法，但是如果您提供自己的JmsTemplate ，它将很快退出。
  • ActiveMQConnectionFactoryCreator创建一个ActiveMQConnectionFactory ，但ActiveMQConnectionFactory是它在类路径上检测到ActiveMQ，并且在所有Spring Bean之间都没有定义其他ConnectionFactory 。该工厂对于创建JmsTemplate是必需的。它设置为内存模式，这意味着您甚至不需要安装独立的代理即可开始使用JMS。但是您可以轻松地替换自己的ConnectionFactory ，并且无论哪种方式，Spring Boot都会自动将其连接到JmsTemplate 。

如果我没有正确注册新的JmsTemplateAutoConfiguration那么所有这些自动配置都将一事无成。为此，我将FQDN添加到Spring Boot的spring.factories文件中。

. . .
org.springframework.boot.autoconfigure.jms.JmsTemplateAutoConfiguration,\
. . .

当然，没有一些自动化的单元测试，拉取请求是不完整的。我不会将我编写的所有测试都放在此博客条目中，但是您可以检查通过我的pull request提交的测试 。在提交请求之前，请准备好编写自己的测试电池！

这就是向Spring Boot添加自动配置的全部内容！没那么复杂。实际上，您可以浏览现有的自动配置类以获取更多示例。

Spring Boot的Groovy支持

Spring Boot最大的特点之一就是它对Groovy的强大支持，引起了众多关注。在主题演讲中赢得了很多掌声，第二天在Dave和Phil的演讲中被吃光了。万一您错过了它，以下是Spring Boot REST服务Dave Syer演示的内容：

@RestController
class ThisWillActuallyRun {
    @RequestMapping("/")
    String home() {
        "Hello World!"
    }
}

将代码放入app.groovy之后 ，Dave通过键入以下内容启动了它：
sh $ spring run app.groovy

Spring Boot的命令行工具使用嵌入式Groovy编译器，并查看所有符号（如RestController ）。然后，它会自动添加@Grab和import语句。它实质上将前面的片段扩展为：

import org.springframework.web.bind.annotation.*
import org.springframework.web.servlet.config.annotation.*
import org.springframework.web.servlet.*
import org.springframework.web.servlet.handler.*
import org.springframework.http.*
static import org.springframework.boot.cli.template.GroovyTemplate.template
import org.springframework.boot.cli.compiler.autoconfigure.WebConfiguration

@RestController
class ThisWillActuallyRun {
    @RequestMapping("/")
    String home() {
        "Hello World!"
    }
    
	public static void main(String[] args) {
		SpringApplication.run(ThisWillActuallyRun, args)
	}
}

添加您自己的Groovy集成

要添加Spring JMS支持，我需要向Boot的CLI添加类似的自动配置，以便每当有人使用JmsTemplate ， DefaultMessageListenerContainer或SimpleMessageListenerContainer ，它将添加正确的位。

在编写该代码之前，我首先编写了一个简单的Groovy脚本，该脚本在jms.groovy中使用Spring JMS：

package org.test

@Grab("org.apache.activemq:activemq-all:5.2.0")

import java.util.concurrent.CountDownLatch

@Configuration
@Log
class JmsExample implements CommandLineRunner {

	private CountDownLatch latch = new CountDownLatch(1)

	@Autowired
	JmsTemplate jmsTemplate

	@Bean
	DefaultMessageListenerContainer jmsListener(ConnectionFactory connectionFactory) {
		new DefaultMessageListenerContainer([
			connectionFactory: connectionFactory,
			destinationName: "spring-boot",
			pubSubDomain: true,
			messageListener: new MessageListenerAdapter(new Receiver(latch:latch)) {{
				defaultListenerMethod = "receive"
			}}
		])
	}

	void run(String... args) {	
		def messageCreator = { session ->
			session.createObjectMessage("Greetings from Spring Boot via ActiveMQ")
		} as MessageCreator
		log.info "Sending JMS message..."
		jmsTemplate.send("spring-boot", messageCreator)
		latch.await()
	}

}

@Log
class Receiver {
	CountDownLatch latch

    def receive(String message) {
        log.info "Received ${message}"
        latch.countDown()
    }
}

这个测试脚本期望Spring Boot自动提供JmsTemplate和ConnectionFactory 。请注意，除了拉入activemq-all之外，没有任何import语句或任何@Grab 。它使用Spring Boot的CommandLineRunner接口启动run()方法，该方法进而通过JmsTemplate发送一条消息。然后，它使用CountDownLatch等待来自使用者的信号。

在另一端是DefaultMessageListener ，它在收到消息后递减计数。为了从Spring Boot的测试套件中调用脚本，我向SampleIntegrationTests添加了以下测试方法以调用jms.groovy ：

	@Test
	public void jmsSample() throws Exception {
		start("samples/jms.groovy");
		String output = this.outputCapture.getOutputAndRelease();
		assertTrue("Wrong output: " + output,
				output.contains("Received Greetings from Spring Boot via ActiveMQ"));
		FileUtil.forceDelete(new File("activemq-data")); // cleanup ActiveMQ cruft
	}

为了测试我的新补丁，我发现运行特定测试要容易得多。这肯定加快了速度。

$ mvn clean -Dtest=SampleIntegrationTests#jmsSample test

注意：我必须先运行mvn -DskipTests install才能将新的JMS自动配置功能部署到本地maven存储库中。

由于我尚未编写任何Groovy自动配置，因此测试将失败。是时候编写CLI自动配置了！

package org.springframework.boot.cli.compiler.autoconfigure;

. . .import statements. . .

public class JmsCompilerAutoConfiguration extends CompilerAutoConfiguration {

	@Override
	public boolean matches(ClassNode classNode) {
		return AstUtils.hasAtLeastOneFieldOrMethod(classNode, "JmsTemplate",
				"DefaultMessageListenerContainer", "SimpleMessageListenerContainer");
	}

	@Override
	public void applyDependencies(DependencyCustomizer dependencies)
			throws CompilationFailedException {
		dependencies.add("org.springframework", "spring-jms",
				dependencies.getProperty("spring.version")).add(
				"org.apache.geronimo.specs", "geronimo-jms_1.1_spec", "1.1");

	}

	@Override
	public void applyImports(ImportCustomizer imports) throws CompilationFailedException {
		imports.addStarImports("javax.jms", "org.springframework.jms.core",
				"org.springframework.jms.listener",
				"org.springframework.jms.listener.adapter");
	}

}

这些回调挂钩使与Spring Boot的CLI工具集成变得非常容易。
matches()允许您定义哪些符号触发此行为。为此，如果存在JmsTemplate ， DefaultMessageListenerContainer或SimpleMessageListenerContainer ，它将触发操作。
applyDependencies()确切指定要通过Maven坐标将哪些库添加到类路径。这类似于向应用程序添加@Grabannotation。为此，我们需要JmsTemplate spring-jms和JMS API规范类的geronimo-jms 。
applyImports()将import语句添加到代码顶部。我基本上从自动配置测试代码中查看了Spring JMS导入，并在此处添加了它们。

这次，如果您运行测试套件，它将通过。

$ mvn clean -Dtest=SampleIntegrationTests#jmsSample test
