Spring Data Babbage发布火车上线

发布
奥利弗·德罗博姆
2013年9月10日

尊敬的Spring社区，我们正好赶在SpringOne 2013推出之际，我们很高兴地宣布推出名为Babbage的GA版本的Spring Data发布火车代码。该版本结束了对Spring Data Core， JPA ， MongoDB和Neo4j模块的大量新功能和改进的开发，并将作为即将发布的一系列社区模块的基础。这是该版本的最重要功能。

常规升级/ Spring Data Commons：

  • Spring Data Core中的Spring 4和JDK 8兼容性改进可泄漏到所有模块中
  • 扩展的Pageable / Page蜜蜂
  • 用于Spring MVC的高级Web和超媒体集成
  • 支持count…By在存储库的派生查询方法名称中
  • 升级到Querydsl 3.x时间轴
  • 改进了所有模块的CDI扩展

Spring Data JPA

  • 在手动定义的查询中支持SpEL表达式
  • 支持TemporalType查询参数绑定

Spring Data MongoDB

  • 支持MongoDB聚合框架
  • FieldNamingStrategy带有驼峰式缩写的交钥匙选项

Spring Data Neo4j

  • 升级到Neo4j的最新版本
  • 持久性元数据子系统中的性能改进

有关发行内容的更多信息，请查看Babbage Wiki页面或运行JIRA查询，其中列出了此发行系列中修复的所有200张以上的票证。

本发行版结束了过去6个月的工作，对于使社区参与发行的数量增加，我们尤其感到高兴。该培训将成为即将编排的Spring IO Platform版本的一部分，并为即将发布的Solr，Couchbase和Elasticsearch的Spring Data社区模块奠定基础。
