脚本编辑器0.3版本可用

工程
安迪·克莱门特
2013年1月8日

10月，第一个公开版本的脚本代码编辑器发布了，这是一个使用Node.js本地服务器的基于浏览器的编辑器，着重于提供出色的JavaScript编辑体验。今天，第一个更新可用，版本为0.3。

有关最新功能的完整发行说明，请参见此处 。这儿是一些精彩片段：

 

可安装NPM

现在，可以使用Windows，Linux或Mac上的节点包管理器（ npm ）来简单地安装脚本：

npm install -g scripted

（在Mac / Linux上可能需要'sudo'前缀）。安装后，“ scr ”命令可立即用于启动脚本。为了使npm安装更趋完善，启动脚本进行了一些大修，尤其是在Windows上。使用npm进行安装是试用脚本的一种非常简单的方法。

 

按键绑定

脚本化是关于速度和快速遍历代码的全部。为此，Scripted大量使用键盘快捷键，但是显然并非所有人都同意这些快捷键应该是什么！在此版本中，脚本键绑定现在是完全可配置的。在帮助面板（使用F1打开）中，单击键绑定以进行任何更改。自定义的键绑定保留在用户主目录中的文件中。

 

代码理解

脚本化的重点是JavaScript编辑。基于以前版本中对JavaScript的基本理解，Scripted现在可以了解模块化JavaScript应用程序的更多配置习惯。现在，使用AMD / CommonJS时，Scripted可以更好地发现项目在何处提供模块配置，在require配置调用中处理配置选项（例如“ packages ”）以及查找模块引用。对于使用curl资源加载器而不是RequireJS的项目，还提供了一些基本支持。

最重要的是，推理引擎现在也可以完成更彻底的工作。通过使用可以推断对象的形状（对象上的有效属性），并可以推断数组元素的类型。与依赖关系分析器一起使用，推理程序也开始更好地处理全局依赖关系。在这些设置中，项目可能没有使用模块系统，而是通过HTML文件中的脚本标签有效地将JavaScript文件合并在一起-脚本化将识别这种情况并理解文件之间的引用。
 

导航和内容协助

通过更好地理解代码结构，导航更加顺畅，内容帮助得到改善。现在可以通过按Cmd （在Mac上）或Ctrl （在Windows / Linux上）并单击标识符或模块引用来进行导航。这将带您到该元素或模块定义的声明。

为了完成导航故事，已使Scripted中的历史记录支持更加智能，并且现在包括有关当前选择，滚动位置和侧面板中打开的任何文件的信息-这样可确保在使用浏览器浏览历史记录时向前/向后状态编辑器的/ layout保留得更好。

 

代码验证

以前，JSLint是首选，但由于受欢迎的需求，Scripted现在已切换到JSHint 。配置选项相似，并且它也了解JSLint样式代码注释，因此无需立即更新JS代码中任何现有的JSLint配置注释。有关配置和选项的更多信息，请参见脚本发行说明和JSHint文档 。脚本编制者还将注意到任何提供的.jshintrc文件以及从此处获取的皮棉短绒配置。

 
这只是其中的几个亮点。有关这些功能的更多详细信息，并了解更多的增强功能，请参见发行说明 。
 

下一步是什么？

在下一版本中，正在研究的一些功能区域是：

  • 可扩展性。插入您自己的linter（或其他度量标准）计算工具。
  • 利用侧板。这不只是为了托管第二个编辑器……
  • 编辑器主题。就像键绑定是个人事物一样，颜色也是如此。当Scripted移至Orion编辑器的最新版本时，它将获得对更多可配置编辑器的支持。
  • 调试器集成。

当然，这也进一步提高了Scripted对JavaScript项目的理解。
