下一站：Spring Framework 4.0

工程
于尔根·霍勒（Juergen Hoeller）
2013年1月16日

尊敬的Spring社区，

我很高兴地宣布，核心框架的下一个迭代版本将是Spring Framework 4.0 ！

当前的3.2代是3.x系列的自然结论，基于Java的配置和REST已成为Java SE 7和Servlet 3.0支持之后的最新关注领域。

对于Spring Framework 4.0，我们的重点是2013年及以后的新兴企业主题：


  • 对基于Java SE 8的Spring应用程序的一流支持 ：
    语言功能，例如lambda表达式； API，例如JSR-310日期和时间
  • 使用Groovy 2配置和实现Spring风格的应用程序 ：
    基于Groovy的bean定义； Groovy是整个应用程序的首选语言
  • 支持关键的Java EE 7技术 ：
    包括JMS 2.0，JPA 2.1，Bean验证1.1，Servlet 3.1和JCache
  • 启用WebSocket风格的应用程序体系结构 ：
    支持符合JSR-356的运行时和相关技术
  • 应用程序内的细粒度事件和消息传递 ：
    基于我们现有的应用程序事件和消息侦听器机制
  • 修剪和依赖项升级 ：
    删除不推荐使用的功能；提高对Java 6+等的最小依赖

基于Spring Framework 3.2的发展动力和准备工作，我们计划再进行一年的迭代，并在2013年底之前达到4.0 GA。

我们将密切跟踪OpenJDK 8的进度，最早在4月实现第一个Spring Framework 4.0里程碑。随着我们与M1的距离越来越近，详细信息将在后续的博客文章中介绍。

让我们知道您在以上内容，特定的体系结构要求，计划采用即将用于Spring的技术中是否有任何特别的偏爱，…

干杯，
于尔根

PS： 不要错过我们即将在星期四举行的网络研讨会 ！
