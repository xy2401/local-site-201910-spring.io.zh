SpringOne 2GX 2012重播：监视和管理Spring集成，使用Spring Hadoop构建大数据管道

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2013年2月19日


在本演示中，我们将讨论用于管理和监视使用Spring Integration的应用程序的选项。它将全面概述Spring Integration提供的对JMX的广泛支持，包括提供对Spring Integration内部的访问以及创建JMX客户端以与本地和远程MBeanServer交互的方面。


此外，我们将展示如何使用Spring Integration插件用于Spring Insight，以深入研究Spring Integration流处理以检查应用程序性能。


主题包括：



  • 使用Integration MBean Exporter及其注册的MBean来分析消息传递端点和通道。

  • 将Integration MBean Exporter本身导出为MBean，以获得对其属性和操作的访问权限。

  • 使用控制总线启动和停止端点。

  • 使用Spring Integration的Spring Insight插件可以实时查看应用程序及其性能。

  • 启用和使用消息历史记录

  • 使用Spring Integration 2.2中可用的有序关闭机制。

  • 使用JMX端点（具有本地和远程MBeanServer）来监视属性。调用操作，发布通知并接收通知。

如何使用OSS为Hadoop构建大数据管道

Hadoop不是孤岛。为了提供完整的大数据解决方案，需要开发一种数据管道，该管道整合并协调了许多不同的技术。以Hadoop为重点的数据管道不仅需要协调多个Hadoop作业（MapReduce，Hive，Pig或Cascading）的运行，而且还包括实时数据采集以及对提取到关系/ NoSQL数据库或专用分析工具中的简化数据集的分析。引擎。

本次会议探讨了大数据管道的体系结构，未来的挑战以及如何使用开源软件（例如Apache Hadoop，Hive，Pig，Spring Hadoop，批处理和集成）构建可管理且强大的解决方案。
