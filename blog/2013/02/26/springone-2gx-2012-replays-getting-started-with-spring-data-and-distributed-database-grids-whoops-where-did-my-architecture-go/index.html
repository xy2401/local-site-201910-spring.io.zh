Spring数据和分布式数据库网格入门



如今，替代数据持久性方法风行一时。但是，将我们的技能和遗留应用程序转换为这些新的有前途的技术可能会出现问题。Spring Data是持久性增长的令人兴奋的解决方案。它带来了Spring Framework的灵活性和熟悉性，并增加了Repositories的概念，使开发人员可以使用熟悉的方法（如保存，更新，删除和动态查找器）编写程序。


在本演示中，我们将介绍用于GemFire的Spring Data以及它如何利用您现有的Spring Framework技术来创建通用的Spring样式的界面，这将使其更有效地过渡到分布式数据网格（例如GemFire）。

哎呀！我的建筑去了哪里？

当应用程序变得更大时，模块化成为有关可维护性的关键方面。早期做出的设计决策很难在代码库中发现，模块间的依赖关系增长很多。该演讲介绍了将逻辑体系结构连接到代码库的方法和方法。除了讨论通用代码组织的模式和最佳实践之外，我们还讨论了用于为Java应用程序打下坚实基础的包结构，以及Spring可以帮助创建松散耦合的组件和扩展应用程序的专用点的范围。
