Apache Hadoop 1.0的Spring正式发布

工程
Costin Leau
2013年2月26日

我们很高兴地宣布，针对Apache Hadoop的Spring的第一个GA 版本 （1.0），距离其第一个里程碑版本的发布已经差不多一年了。
在此期间，我们收集了您的大量反馈意见，以制定路线图，因此，感谢社区中所有提供帮助的人！尽管一年中增加了新功能，但Spring for Apache Hadoop的目标保持不变，以简化基于Hadoop的应用程序的开发。

立即下载 ，或在此处查看Maven工件。

简化的编程模型和一致性


我们观察到的是，使用Hadoop随附的标准即装即用工具，您很容易最终获得Hadoop应用程序，这些应用程序的结构很差，这些命令行实用程序，脚本和代码片段组合在一起。Hadoop生态系统中各种项目的起源不同，例如Hive和Pig专注于声明性用法，或者以编程角度关注Cascading和HBase ，导致了不同的配置和API设计方法。

 

用于Apache Hadoop的Spring在广泛的Hadoop生态系统项目中提供了一致的编程和配置模型：该框架不决定要使用什么，而是包含并增强了您的技术堆栈，并忠于Spring的核心原则。

 

Spring熟悉的Template API设计模式应用于Hadoop，其结果是诸如HBaseTemplate ， HiveTemplate和PigTemplate的帮助器类。这带来了熟悉的Spring数据访问模板功能，例如转换为Spring的可移植数据访问异常层次结构，对基础资源的线程安全访问以及轻量级对象映射功能。
通过Spring Framework的出色Java配置，无论是否进行其他配置，都可以自由使用以Java为中心的API，例如Cascading。

从小处开始并根据需要成长


过去一年中出现的另一个主题是鼓励您从小处着手并发展为复杂解决方案的方法。引进各类的亚军类允许执行蜂巢 ， 猪脚本，香草的Map / Reduce或流工作 ，层叠流动，而且预调用和后仿制基于JVM的脚本通过熟悉JDK全部可赎回的合同。您可以根据需要混合和匹配跑步者，但是随着复杂性的增加，可以轻松升级到Spring Batch ，从而可以以有状态的方式协调多个步骤并使用REST API进行管理。 Spring Batch丰富的功能可用于处理大文件的ETL处理，可直接转换为Hadoop用例，以从HDFS提取和导出文件。将Spring Hadoop与Spring Integration结合使用，可以对事件流进行丰富的处理，这些事件流可以在从HDFS或其他存储（例如NOSQL存储）读取和写入之前进行转换，丰富，过滤， Spring Data为此提供了充分的支持。
我们已经讨论过各种方案示例应用程序 （无需编译他们，他们已经被编译并随时可以下载），以补充全面的用户文档（它甚至包括一节关于如何开始使用Spring盯着为Apache Hadoop的使用Amazon的Elastic MapReduce服务）。
另外，作为示例的伴侣，可以使用最新的Spring Data book [1]以获得可以使用Spring技术，Hadoop和NOSQL实现的完整功能集。

 

可移植性


每天都会针对各种Hadoop 1.x发行版（例如香草Apache Hadoop，Cloudera CDH3和CDH4，Greenplum HD）对Apache Hadoop的Spring进行测试 ：无论您的Hadoop环境如何，我们都要确保SHDP能够可靠地工作。我们正在积极努力改善用户体验-Greenplum HD发行版中提供了适用于Apache Hadoop的Spring。
我们密切关注Hadoop 2.x的开发，并努力在不久的将来为其提供支持。

 

如果您将Spring用于Apache Hadoop，我们将很高兴收到您的来信。请参加我们的调查并分享您的经验。

 

 

一如既往，我们期待您的反馈！

 


[1] Spring Data图书销售的作者版权费捐赠给了Creative Commons组织。
