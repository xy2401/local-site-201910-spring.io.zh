通过Spring Boot的属性支持增强您的应用程序

工程
格雷格·特恩奎斯特
2013年10月30日

Spring Boot仍在继续发展。上个月，我写了一个有请求请求的Contributing to Spring Boot 。我剥离了Spring Boot的各层，以演示其令人难以置信的自动配置功能和CLI支持。

在本文中，我想深入探讨Spring Boot对属性的惊人支持。属性是很小的东西，不是很明显，但是可以以非常实用的方式快速增强您的应用程序。在本文中，我将逐步介绍如何在上一篇博客文章中编写的Spring JMS支持中添加属性支持 。

什么是属性？

属性本质上是外部化应用程序设置的方法。您可能会在应用程序中烘烤一些特定的信息，但是由于多种原因，您希望稍后进行更改。

  • 您的默认配置是基于生产的（主机名，端口等），但是您需要在测试床上用不同的主机名覆盖它。
  • 您将应用程序配置为使用池，但要调整池大小。
  • 您需要为应用程序提供一个“超级秘密”密钥，也许是一个OAuth密钥，并且您不想在已发布的应用程序中放入默认密钥。

所有这些用例都在请求不同的方法来启动应用程序时提供自定义设置。春季靴救助！

Spring JMS和属性

抽象用例就足够了。让我们看一下Spring Boot如何提供属性支持的真实示例！我们将更深入地研究Spring JMS支持 。

@Configuration
@ConditionalOnClass({ JmsTemplate.class, ConnectionFactory.class })
@EnableConfigurationProperties(JmsTemplateProperties.class)
public class JmsTemplateAutoConfiguration {

	@Autowired
	private JmsTemplateProperties config;
	. . .
	@ConfigurationProperties(name = "spring.jms")
	public static class JmsTemplateProperties {

		private boolean pubSubDomain = true;

		public boolean isPubSubDomain() {
			return this.pubSubDomain;
		}

		public void setPubSubDomain(boolean pubSubDomain) {
			this.pubSubDomain = pubSubDomain;
		}

	}
	. . .

来自Spring Boot的这段代码JmsTemplatAutoConfiguration显示了几个关键组件。
-@EnableConfigurationProperties利用JmsTemplateProperties作为属性的来源，并使它们可用于整个类。
-@ConfigurationProperties（name =“ spring.jms”）将JmsTemplateProperties类标记为它将成为属性的持有者。使用“ spring.jms”作为前缀，该类的每个属性都将成为目标属性。

我要做的就是创建一个application.properties文件并为其分配值。

spring.jms.pubSubDomain=true

在同一文件的最下方，还具有内置连接工厂的属性。

	@Configuration
	@ConditionalOnClass(ActiveMQConnectionFactory.class)
	@ConditionalOnMissingBean(ConnectionFactory.class)
	@EnableConfigurationProperties(ActiveMQConnectionFactoryProperties.class)
	protected static class ActiveMQConnectionFactoryCreator {

		@Autowired
		private ActiveMQConnectionFactoryProperties config;

		@Bean
		ConnectionFactory jmsConnectionFactory() {
			if (this.config.isPooled()) {
				PooledConnectionFactory pool = new PooledConnectionFactory();
				pool.setConnectionFactory(new ActiveMQConnectionFactory(this.config
						.getBrokerURL()));
				return pool;
			}
			else {
				return new ActiveMQConnectionFactory(this.config.getBrokerURL());
			}
		}

	}

	@ConfigurationProperties(name = "spring.activemq")
	public static class ActiveMQConnectionFactoryProperties {

		private String brokerURL = "tcp://localhost:61616";

		private boolean inMemory = true;

		private boolean pooled = false;

		// Will override brokerURL if inMemory is set to true
		public String getBrokerURL() {
			if (this.inMemory) {
				return "vm://localhost";
			}
			else {
				return this.brokerURL;
			}
		}
    . . .

默认情况下，除非您提供自己的，否则Spring Boot将创建一个ActiveMQ连接工厂。
-- ActiveMQConnectionFactoryCreator被标记为@EnableConfigurationProperties 。
-我们可以覆盖brokerURL ， inMemory和pooled 。

在ActiveMQConnectionFactoryCreator ，如果设置了pooled ，它将创建一个PooledConnectionFactory 。如果设置了inMemory ，它将使用* vm：// localhost * 。但是将其翻转为false将导致连接工厂切换到使用brokerURL。

到目前为止，这些属性已通过其各自的getter访问。但这不是唯一的方法。春天带有强大的力量@Value注释以从多个属性（包括属性）中注入数据。

因此，这就是我为Spring Boot 贡献Spring JMS属性支持所做的大部分工作。

让我们看一下使用Spring Boot的属性支持的更多方法。

超级秘密GitHub oauth数据

我一直在研究一个小型应用程序，以快速列出所有未解决的问题以及各种入门指南 。它使用Spring Social GitHub来查询问题。要获取我需要的数据，它功能强大GitHubTemplate需要一个oauth密钥。

@Controller
@Log
class IssueAggregator implements CommandLineRunner {

	/**
	 * This needs to be supplied by application.properties, a file NOT to be put under source control
	 */
	@Value('${token}')
	String githubToken
	
	@Bean
	GitHubTemplate githubTemplate() {
		new GitHubTemplate(githubToken)
	}
	. . .

你可以猜到githubToken由填充@Value('${token}') 。如果仔细观察，您会发现没有默认值。那是因为我不会提供一个。如注释中所述，需要此应用程序副本的任何人都必须提供自己的秘密oauth密钥。

注意：这是Groovy代码，但是相同的注释也适用于Java（您只需要双引号）。Spring Boot使得利用Spring Framework的属性支持非常容易。

用更多属性覆盖属性

这还不是全部。我可以在分发的JAR中嵌入一个application.properties文件，提供默认设置。

但是，如果我需要重写任何内容，我所要做的就是创建另一个 application.properties文件并将其暂存到JAR文件旁边。Spring Boot将首先读取内部application.properties文件，然后自动查找外部文件。

最后，它还将读取使用Java提供的属性-Dspring.activemq.inMemory=false命令行指令。这提供了覆盖事物的第三种方法。

PS对于我的Windows兄弟，Spring Boot通过映射诸如以下内容提供了额外的特殊支持： SPRING_ACTIVEMQ_INMEMORY ， spring-activemq-inmemory和springActivemqInmemory到同一目标： spring.activemq.inMemory 。这有助于处理特定于平台的问题，例如环境变量不支持句点。

摘要

希望您对Spring Boot的惊人能力感到震惊，该能力通过属性设置使我们的应用程序灵活且可配置。这个工具范围可能很小，但是很有意义。
