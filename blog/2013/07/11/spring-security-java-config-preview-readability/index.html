Spring Security Java配置预览：可读性

工程
罗伯·温奇
2013年7月11日

在本文中，我将讨论如何使您的Spring Security Java配置更具可读性。该帖子旨在详细说明Spring Security Java Config Preview：Web Security中的内容：

通过格式化我们的Java配置代码，它更容易阅读。可以类似XML名称空间的方式读取它，其中“ and（）”表示有选择地关闭XML元素。

缩进

Spring Security的Java配置的缩进确实会影响其可读性。通常，缩进优先使用，如项目符号列表。

有关更具体的示例，请看以下代码：


http
    // #1
    .formLogin()
        // #2
        .loginPage("/login")
        .failureUrl("/login?error")
        // #3
        .and()
    // #4
    .authorizeRequests()
        // #5
        .antMatchers("/signup","/about").permitAll()
        .antMatchers("/admin/**").hasRole("ADMIN")
        .anyRequest().authenticated();
  • ＃1 formLogin更新http对象本身。的缩进formLogin从的增量http （就像他们的方式从缩进 ）
  • ＃2 loginPage和failureUrl更新formLogin组态。例如， loginPage确定如果需要登录，Spring Security将重定向到何处。因此，每个人都是formLogin 。
  • ＃3 and表示我们已经完成了父配置（在这种情况下， formLogin ）。这也意味着下一行将使缩进减少一个。查看配置时，您可以将其读取为http配置有formLogin 和 authorizeRequests 。如果我们没有其他配置， and没有必要。
  • ＃4我们减少缩进authorizeRequests因为它与基于表单的登录无关。相反，其目的是限制对各种URL的访问。
  • 每个＃5 antMatchers和anyRequest修改的授权要求authorizeRequests 。这就是为什么每个人都是authorizeRequests

IDE格式化程序

缩进可能会导致代码格式化程序出现问题。许多IDE允许您禁用带有注释的选定代码块的格式。例如，在STS / Eclipse中，可以使用@formatter：off和@formatter：on的注释来关闭和打开代码格式。一个例子如下所示：


// @formatter:off
http
    .formLogin()
        .loginPage("/login")
        .failureUrl("/login?error")
        .and()
    .authorizeRequests()
        .antMatchers("/signup","/about").permitAll()
        .antMatchers("/admin/**").hasRole("ADMIN")
        .anyRequest().authenticated();
// @formatter:on

为了使此功能正常工作，请确保已启用它：

  • 导航至首选项-> Java->代码样式->格式化程序
  • 点击编辑按钮
  • 选择关闭/打开标签选项卡
  • 确保选择了启用关闭/打开标签
  • 您也可以在此处更改用于禁用和启用格式设置的字符串。
  • 点击确定

与XML命名空间的比较

缩进还有助于我们将Java配置与XML名称空间配置相关联。这并不总是正确的，但确实有帮助。让我们将我们的配置与下面的相关XML配置进行比较。


http
    .formLogin()
        .loginPage("/login")
        .failureUrl("/login?error")
        .and()
    .authorizeRequests()
        .antMatchers("/signup","/about").permitAll()
        .antMatchers("/admin/**").hasRole("ADMIN")
        .anyRequest().authenticated();

相关但非等效的XML配置如下所示。请注意，在这些配置之间，Spring Security的行为方式之间的差异是由于Java配置和XML配置之间的默认值不同。


<http use-expressions="true">
  <form-login
      login-page="/login"
      authentication-failure-url="/login?error"
  /> <!-- similar to and() -->
  <intercept-url pattern="/signup" access="permitAll"/>
  <intercept-url pattern="/about" access="permitAll"/>
  <intercept-url pattern="/**" access="hasRole('ROLE_USER')"/>
</http>
  • 首先要注意的是http和非常相似。一个区别是Java配置使用authorizeRequests指定use-expressions="true"
  • formLogin和非常相似。每个孩子formLogin是的XML属性 。根据我们对缩进的解释，相似之处是合乎逻辑的，因为XML属性会修改XML元素。
  • 的and()下formLogin与结束XML元素非常相似。
  • 每个孩子authorizeRequests每个都相似 ，除了Java配置以不同的方式指定了require-channel之外，这在许多情况下有助于减少配置。

摘要

现在，您应该知道如何一致地缩进您的Spring Security Java配置。这样，您的代码将更具可读性，并且更容易与XML配置等效项相互转换。
