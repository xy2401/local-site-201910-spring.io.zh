Spring Security Java Config预览：简介

工程
罗伯·温奇
2013年7月2日

昨天，我宣布了Spring Security Java Configuration支持的发布以及Spring Security 3.2.0的发布。M2包含Java配置支持。

Spring Security的Java配置支持旨在完全替代XML名称空间配置 。它还被设计为可扩展的，因此Spring Security的扩展项目可以与Java Configuration支持一起很好地工作。

在由五 部分组成的 Spring Security Java Configuration 博客 系列的第一篇文章中，我讨论了Spring Security Java Configuration项目的后勤工作。

“所需版本”
无论您决定如何与Spring Security集成，确保您使用的是Spring 3.2.3都是很重要的。RELEASE +以确保您避免使用SPR-10546 。

可用性

在开始之前，我想谈谈可以找到Spring Security的Java配置的两个模块。

Spring Security 3.2.0中的可用性。M2 +

Spring Security Java配置已被复制到Spring Security 3.2.0中。M2 +代码库。这意味着如果您使用的是Spring Security 3.2.0。M2 +，您应确保在类路径上具有spring-security-config jar。例如，您的Maven pom.xml中可能包含以下条目：



  org.springframework.security
  spring-security-config
  3.2.0.M2

…

  repository.springsource.milestone
  SpringSource Milestone Repository
  http://repo.springsource.org/milestone

“未来可用性”
当前没有计划提供对spring-security-javaconfig的任何更新。相反，将鼓励用户在Spring Security 3.2发布时进行更新。

Spring Security 3.1.4的可用性。发行+

为了鼓励用户尝试使用Spring Security Java Configuration，它也可以作为称为spring-security-javaconfig的独立模块使用。例如，您的Maven pom.xml中可能包含以下条目：



  org.springframework.security
  spring-security-javaconfig
  1.0.0.M1

…

  repository.springsource.milestone
  SpringSource Milestone Repository
  http://repo.springsource.org/milestone
