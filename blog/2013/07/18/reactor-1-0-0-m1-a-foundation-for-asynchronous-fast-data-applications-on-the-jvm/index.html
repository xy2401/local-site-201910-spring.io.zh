反应器1.0.0。M1-JVM上异步快速数据应用程序的基础

工程
乔恩·布里斯宾
2013年7月18日

我很高兴宣布Project Reactor的第一个里程碑版本！Project Reactor是用于在JVM上构建异步FastData应用程序的基础框架。 Reactor 1.0.0中的一些优点。M1包括：反应式合成助手Stream和Promise，TcpServer和TcpClient以及Groovy和Spring支持。受响应式扩展，RxJava，新的JDK 8 Stream API（以及Scala等）的启发，这些Composable使协调异步任务变得非常简单。它们支持使用Consumers的传统回调样式编程，但是它们还提供了简洁的组合API，其使用的方法如map(Function fn) ， filter(Predicate p) ， batch(int size)和更多。

反应堆解决什么问题？

从头开始，Reactor被设计为一个高性能，大规模的平台，用于构建下一代大数据应用程序。当将应用程序扩展到成百上千，甚至数百万个用户时，异步体系结构显然优于按请求线程体系结构。Reactor的异步基础为每秒处理数万或数十万甚至数百万个事件的大数据应用程序提供了坚实的基础。它提供了将异步任务链接在一起的简单工具，并使执行这些任务像调用单个方法一样容易。

使用Reactor进行合成

当数据异步流经应用程序时，流是一种处理数据的简便方法。在Reactor中，流实际上有两个部分：递延者（即发布者）和实际流（即消费者）。您可以在Stream上分配处理程序以结合使用组合方法和简单的回调来处理数据。

在将应用程序中的数据转换并过滤后，再将其放入队列中，以便使用Stream和JDK Lambda进行进一步处理，它看起来像这样：


// Create Environment in which Reactors operate
Environment env = new Environment();

// Create a Stream using the high-speed LMAX Disruptor RingBuffer
Deferred<Trade, Stream<Trade>> incoming = Streams.<Trade>defer()
		.env(env)
		.dispatcher(Environment.RING_BUFFER)
		.get();

// Work with the incoming trades
Stream<Trade> trades = incoming.compose();
Stream<Order> orders = trades.map(trade -> tradeService.placeTrade(trade));

// Filter out large orders from small
Stream<Order> highPriority = orders.filter(order -> order.getSize() >= 1000);
Stream<Order> lowPriority = orders.filter(order -> order.getSize() < 1000);

// Consume the orders in different ways
highPriority.consume(order -> orderService.executeNow(order));
lowPriority.consume(order -> orderService.executeLater(order));

TCP支持

M1还包括一个易于使用的TCP客户端和服务器。由超快的Netty网络库提供支持，Reactor驱动的syslog服务器可以在服务器级硬件上每秒接收约100万条消息。Reactor TCP支持包括一个简单的编解码器工具，该工具可轻松扩展到内核中提供的默认编解码器集之外，并通过使用Reactor的Buffer类设计为轻量级，该类提供诸如对数据的极高效视图以及大量帮助程序之类的东西。用于标准Java NIO ByteBuffers的方法-但无需直接处理ByteBuffer。

Reactor的TCP支持与JSON一起提供。创建使用JSON作为协议的基于TCP的RPC服务器很简单：


TcpServer<Pojo, Pojo> server = new TcpServerSpec<Pojo, Pojo>(NettyTcpServer.class)
		.env(env)
		.codec(new JsonCodec<>(Pojo.class))
		.consume(conn -> {
			conn.consume(data -> {
				// handle incoming data
			});
		})
		.get()
		.start();

Groovy和Spring支持

Reactor M1还提供了很棒的Groovy支持。它提供了一些帮助，使使用Closures的事件变得非常简洁。不用说，用Groovy编写Reactor事件处理非常容易。使用Closures处理Reactor使异步代码真正可读！


def env = new Environment()

// Create Reactor using default RingBuffer Dispatcher
def reactor = Reactors.reactor().env(env).get()

reactor.on('topic') { String s ->
	// handle data
}

// Publish an event to a topic
r1.notify 'topic', 'Hello World!'

Reactor M1还包括Spring支持，使编写事件驱动的POJO像MVC Controller一样容易。通过使用@On注解，通过组件扫描拾取的bean可以自动连接到Reactor并通知事件。

一个基于JavaConfig的简单Spring配置可能如下所示：


public class HandlerBean {
	@On(reactor = "@rootReactor", selector = '$("test")')
	public void handleTest() {
		// event 'test' was fired
	}
}

@Configuration
public class AnnotatedHandlerConfig {

	@Bean
	public Environment env() {
		return new Environment();
	}

	@Bean
	public Reactor rootReactor() {
		return env().getRootReactor();
	}
}

只需将您的Reactor注入服务层，并在事件准备就绪时，使用notify()方法。

工件，来源和文档

Maven工件在SpringSource Artifactory存储库中可用。在Gradle项目中，您可以像这样将Reactor拉入：


ext {
  reactorVersion = '1.0.0.M1'
}

repositories {
	maven { url 'http://repo.springsource.org/libs-milestone' }
  mavenCentral()
}

dependencies {
  // Reactor Core
  compile 'org.projectreactor:reactor-core:$reactorVersion'
}
