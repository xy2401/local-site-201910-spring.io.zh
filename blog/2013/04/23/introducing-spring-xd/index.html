介绍Spring XD

工程
马克·费舍尔
2013年4月23日

今天，我们正式启动了一个名为Spring XD的新计划，其主题是“解决大数据复杂性” 1 。

在过去的几年中，Spring Data团队异常忙碌，不仅为NoSQL数据存储提供支持，而且简化了Hadoop的开发经验。通过创建Spring for Apache Hadoop项目 ，我们为Hive和Pig等Hadoop生态系统项目提供了丰富的配置模型和一致的编程模型，从而使开发Hadoop应用程序变得更加容易。正如Spring用户所期望的，人们可以：


  1. 将MapReduce作业配置为容器管理对象并运行。

  2. 为HDFS，HBase，Pig和Hive使用模板帮助程序类，以从应用程序中删除样板代码。

用于Apache Hadoop的Spring为构建Hadoop应用程序提供了坚实的基础。Spring XD建立在这些基础资产的基础上，进一步简化了创建实际大数据解决方案的过程。具体来说，Spring XD解决了常见的大数据用例，例如：

  1. 高吞吐量将来自各种输入源的数据分散到HDFS中。
  2. 摄取时进行实时分析，例如收集指标和计数值。
  3. 通过将与标准企业系统（例如RDBMS）和Hadoop操作（例如MapReduce，HDFS，Pig，Hive或Cascading）的交互结合在一起的批处理作业来进行Hadoop工作流管理。
  4. 高吞吐量数据导出，例如从HDFS导出到RDBMS或NoSQL数据库。

Spring Data本书涵盖了其中几个用例，该书的示例代码可在我们的GitHub存储库中找到 。这些示例除了基于Spring for Apache Hadoop项目之外，还基于Spring Batch和Spring Integration构建。

在管理事件驱动的数据摄取流时， Spring Integration提供了一个行之有效的模型，其灵感来自完善的企业集成模式 。同样， Spring Batch是用于管理工作流的强大解决方案，对诸如工作状态管理和重试/重新启动功能等最重要的要求提供了强大的支持，并且是JSR-352的基础。

从书本示例开始扩展框架以支持大数据用例，但是对于Spring XD，我们旨在将这种支持提高到另一个层次。首先，我们将提供一个涵盖上面列出的四个用例类别的一致模型。那些具有Spring经验的人将立即熟悉该模型。其次，随着Spring XD的发展，我们将远远超出API层，以提供现成的可执行服务器，可插拔模块系统，用于在Hadoop集群内外分配数据收集实例的简单模型，以及更多功能。 。
