Pivotal Initiative的春季和开源

工程
阿德里安·科耶尔（Adrian Colyer）
2013年4月3日

到目前为止，您可能已经听说过Pivotal 。我想在这里花点时间解释一下这对Spring的意义，并向您介绍我们2013年的一些计划。

万一您错过了它，以下是Pivotal的基本背景：在Paul Maritz的带领下，该计划将EMC和VMware的关键人员和项目联合起来， 将“消费级”技术引入企业 。我们看到新一代应用程序不断涌现，这些应用程序由新的数据结构提供支持，并将根据客户的身份，所在的位置以及当前的工作状况与客户互动并为其提供服务。这些应用程序将以前所未有的速度构建，部署和扩展。他们将存储，管理和交付大型数据集的价值，并在以开源技术为基础的一个统一平台上完成所有这些工作。

…现在我们回到定期播放的节目。

我们的Spring团队是Pivotal的一部分，而Spring与Pivotal HD和Cloud Foundry一起是Pivotal战略的核心。Spring 是企业中的主导力量，我们为2013年制定了非常强大的路线图，包括但不限于发布Spring Framework 4。这是我们历史上第三次增加Spring的主要版本，这不是我们轻率做的事情。您有充分的理由期望我们取得伟大的成就，我们无意让您失望！

作为Pivotal的一部分，Spring将继续由Spring项目负责人推动：Juergen Hoeller，Chris Beams，Mark Fisher，Mark Pollack以及您多年来认识并信任的许多其他人。他们的经验，丰富的技术知识和创新思维将继续指导Spring的发展。

2013年春季的重点领域包括促进现代应用程序体系结构，解决大数据应用程序的复杂性以及改善端到端Spring开发人员的体验。Spring Framework 4.0路线图将确保您可以充分利用Java生态系统中的最新开发。特别是对于Java SE 8，就好像它是为Spring制作的一样！Lambda表达式可与我们的模板和回调接口长期方法完美配合。例如，这是古老的JDBC模板在lambda上的外观：

Person person = jdbcTemplate.queryForObject(
 “select name, age from person where id = 42”,
 (rs, rowNum) -> {
 return new Person(
 rs.getString(“name”),
 rs.getInt(“age”)
 );
 });

我们还将研究异步Web使用模式和基于非servlet的运行时支持。作为改善端到端开发人员体验的一部分，我们计划使我们的文档，示例和网站完全更新，以专注于企业应用程序开发的最佳实践。期望看到更多的面向任务而不是面向项目的材料。

在解决大数据应用程序复杂性方面，我们将基于Spring在这些领域已经取得的巨大进步 。随着基于Hadoop的数据结构在企业中不断涌现，我们旨在为大数据应用程序开发提供统一，简化的体验，包括分布式数据提取和实时分析，Hadoop工作流程编排以及从HDFS到关系和非关系数据的导出。关系商店。在接下来的几个月中，您会从我们这里听到更多信息-Apache Hadoop 1.0的春季仅仅是开始。适用于Apache Hadoop的Spring可与所有主要的Hadoop发行版一起使用-Hortonworks，Cloudera，MapR和我们自己的Pivotal HD Community Edition，并且已完全集成在商业Pivotal HD中。

Groovy和Grails是Spring扩展家族的一部分。如果您喜欢Java SE 8的外观，但是担心您的组织将其用作生产运行时可能要花一些时间，那么有个好消息：Groovy语言已经提供了闭包和函数式编程样式等现代功能，以及可选功能。静态类型，并且在JRE 6和7上完全受支持！那些寻求提高生产率，具有表现力的DSL支持以及与Java语言和库的一流集成的人正越来越多地使用Groovy。这些功能还使开发异步和并行应用程序变得更加简单。Spring Framework 4的发展路线图上，与Groovy的紧密集成是可行的，即使您确实可以访问SE 8，Groovy还是Java的一个很好的补充。

Grails 3.0将把Grails与传统的应用程序服务器分开， 并扩展Grails的覆盖范围，以允许开发轻量级的异步应用程序 。Grails的持久性技术GORM也已经超越传统的关系数据库而发展，现在可以使用NoSQL数据库的实现。随着企业数据结构的发展，GORM将继续成为我们的一项重要技术。

开放平台即服务项目Cloud Foundry也是Pivotal Initiative的一部分。我们将确保在Cloud Foundry中对Spring和Grails应用程序提供一流的支持-无论是在公共云中还是在企业内部。有关作为Pivotal Initiative一部分的Cloud Foundry的更多详细信息，请查看Cloud Foundry Blog 。RabbitMQ ， Redis和我们对Apache Tomcat的贡献者也都来了！
