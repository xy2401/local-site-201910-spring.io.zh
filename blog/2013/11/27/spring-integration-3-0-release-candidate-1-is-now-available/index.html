Spring Integration 3.0 Release Candidate 1现在可用

发布
加里·罗素
2013年11月27日

我们非常高兴地宣布Spring Integration 3.0.0。RC1现在在Spring Milestone Repo中可用

该版本默认使用Spring Framework 3.2.5，但也与Spring 3.1.x和4.0.x兼容。

此处显示了自里程碑3以来的更改的发行说明。

首先，关于Spring Framework 4支持的说明

Spring Framework 4.0有了新的spring-messaging包含许多来自Spring Integration的关键抽象的模块。

Spring Integration 3尽管与Spring Framework 4兼容，但并不依赖于它，并且不使用这些类，因此它也可以与Spring Framework 3.2和3.1一起使用。

我们计划在2014年初发布Spring Integration 4，它将与Spring Framework 4完全兼容（并要求与之兼容）。 spring-messaging模块为基础。Spring Framework 4发布后不久将提供一个里程碑。

3.0新增功能

可以在参考文档中找到3.0版本的完整“新增功能”，但是这里有一些要点……

新的和修改的端点

  • syslog入站适配器（UDP和TCP）
<int-syslog:inbound-channel-adapter id="foobar" channel="foo" port="1514" />
  • 一个JMX入站适配器，它轮询多个_MBean_s /属性。 
<jmx:tree-polling-channel-adapter id="jmxtree"
		channel="mbeanData"
		auto-startup="false"
		query-name="java.lang:type=Runtime"
		query-expression="*:type=*">
	<si:poller max-messages-per-poll="1" fixed-rate="5000"/>
</jmx:tree-polling-channel-adapter>
  • 文件尾部入站适配器
<int-file:tail-inbound-channel-adapter id="tailer"
	channel="lines"
	file="/tmp/baz" />
  • 脚本化入站适配器
  • Content Enricher现在可以更新标题和有效负载
  • HTTP入站端点现在利用Spring MVC 3.1 RequestMapping-一个端点现在可以支持多个路径
  • Redis队列入站和出站适配器
  • _Gateway_s现在支持在所有网关方法之间设置通用标头（包括调用的Method ）
<int:gateway id="myGateway"
		service-interface="foo.Bar"
		default-request-channel="requestChannel">
	<int:default-header name="calledMethod"
                        expression="#gatewayMethod.name"/>
</int:gateway>
  • 可以将FTP / SFTP入站适配器配置为保留远程时间戳
  • FTP / SFTP网关现在支持mv （改名）， put和mput命令; ls ， mget和mput现在支持递归-允许目录树的传输。
  • JPA端点现在可以存储多个（_Iterable_）实体；此外，还支持刷新和清除操作
  • 内的元素现在可以有一个id ，使它们可以作为bean和/或_MBean_s使用
  • 现在有一种机制允许通过TCP传输选定的_MessageHeader_s
  • TCP连接事件入站适配器-现在，在打开，关闭连接或发生异常时发出_ApplicationEvent_s
<int-ip:tcp-connection-event-inbound-channel-adapter channel="events" />

总体改进

  • 标头通道注册表 -序列化消息时，现在可以在注册表中保存标头通道（_reply，error_），并且在发送回复时如有必要，将引用该注册表。这可以用来避免No output-channel or replyChannel header available _ChannelResolutionException_s。
  • 可以注册SpEL函数并将其提供给所有消息传递SpEL表达式
<int:spel-function id="foo"
	class="foo.MyUtils" method="evaluate(java.lang.String, java.lang.Object)"/>

<int:transformer input-channel="in" output-channel="out"
		 expression="#foo(headers.bar, payload)" />

提供了两个内置函数： #jsonPath和#xpath 。

自定义_PropertyAccessor_s也可以添加到评估上下文中。

  • Redis元数据存储 -用于存储提要和Twitter适配器的状态。
  • 可配置的MongoDb消息存储 -现在可以使用更灵活的消息存储
  • 内联脚本现在可以具有变量
  • 持久性AcceptOnceFileListFilter现在可用，可以防止系统重新启动时出现重复。

结论
