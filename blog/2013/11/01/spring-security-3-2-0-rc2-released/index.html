Spring Security 3.2.0。RC2发布

发布
罗伯·温奇
2013年11月1日

Spring Security 3.2.0。RC2现在可以从SpringSource存储库（位于http://repo.springsource.org）获得 。请参阅此处 ，以获取有关通过Maven解决这些工件的快速教程。

什么是新的？

该版本解决了80多个问题 。此版本的大多数问题是修复错误，添加文档以及将我们的文档转换为Asciidoctor 。

请参阅最近添加的Spring Security 3.2新增功能，以找到此发行版的所有要点。您会发现有一些不错的功能，我还没有写过。确保点击链接以查看和参考中的详细信息。

应该强调对Java配置支持的一些非被动更改。

registerAuthentication->配置

第一个更改是WebSecurityConfigurerAdapter和GlobalMethodSecurityConfiguration上的registerAuthentication(AuthenticationManagerBuilder)方法已更改为configure(AuthenticationManagerBuilder) 。这样可以更好地与Spring Security配置命名约定保持一致，并减小方法签名的大小，以节省键入的时间。

ObjectPostProcessor。QUIESENT_POSTPROCESSOR已删除

第二个变化是，如果要创建自己的AuthenticationManagerBuilder实例，则可能传入了ObjectPostProcessor.QUIESENT_POSTPROCESSOR进入其中。这可能会导致严重的问题（内存泄漏等）。因此，使用ObjectPostProcessor.QUIESENT_POSTPROCESSOR已被删除。相反，用户应使用在使用@EnableWebSecurity或@EnableGlobalMethodSecurity时可用的Global AuthenticationManagerBuilder 。例如，如果要配置全局身份验证（即，您只有一个AuthenticationManager），则应自动连接AuthenticationMangerBuilder：

@Autowired
public void configureGlobal(AuthenticationManagerBuilder auth) {
   // ... configure it ...
}

如果希望身份验证的范围对特定的WebSecurityConfigurerAdapter私有，则可以重写configure(AuthenticationManagerBuilder)方法。

反馈请

一如既往地保持反馈。如有疑问，请使用spring-security标签发布到stackoverflow 。如果发现问题，请在我们的JIRA中进行记录 。我们计划在12月初发布最终版本，因此请确保在尝试此版本之前尝试一下Spring Security 3.2！
