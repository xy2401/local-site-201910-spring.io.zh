Spring Data发布火车Babbage的第一个里程碑到达

工程
奥利弗·德罗博姆
2013年6月10日

我很高兴地宣布名为Babbage的Spring Data发布系列的第一个服务里程碑发布。它包括以下模块：


  • Spring Data Commons 1.6 M1-更新日志

  • Spring Data JPA 1.4 M1- 变更日志

  • Spring Data MongoDB 1.3 M1-更新日志

  • Spring Data Neo4j 2.3 M1-更新日志

第一个里程碑包括许多新功能以及Arora服务版本中已经发布的所有错误修复。

快速浏览发行版

此版本的大多数更改都已纳入Spring Data Commons，以为下一代Spring Data项目奠定坚实的基础，并确保该基础快速成熟。该火车站发布的其他模块已适应这些变化，因此也从中受益。

我们已经升级到Querydsl 3.x API，以适应其主要版本中引入的更改。存储库抽象增加了对忽略大小写的排序的支持，以及count…By…(…)派生查询的投影。我们还对映射元数据实现进行了重大的性能改进，因此，尤其是MongoDB和Neo4j模块的映射操作性能应提高约20％。

分页和网络支持的全面改进工作尤其是与Spring HATEOAS结合使用时，还需要进行大量工作。如您在参考文档中所见，为您创建分页的资源表示形式Spring MVC控制器从未如此简单。Spring Data Commons中的更改通过对CDI集成的一些改进以及对CDI的迁移而得以完善。 ChainedTransactionManager从Spring Data Neo4j到核心模块。

在Spring Data MongoDB中，我们增加了对通过全局策略自定义字段名称的支持，并提供了CamelCaseAbbreviatingFieldNamingStrategy盒子外面。我们引入了XML名称空间元素MongoTemplate和GridFsTemplate ，增加了对用于索引的背景属性的支持，现在还支持DBRef在Map价值观。Neo4j模块为最新的Neo4j和Cypher版本带来了更新。

外表

发布火车的下一个站点将在商店模块中包含更多新功能。我们目前正在努力支持MongoDB中的聚合框架，针对Neo4j模块的CDI集成等等。我们还将让Spring Data REST加入到发布火车中，以及一些（新的）社区模块。要全面了解已发货和即将发货的产品，请查看此Wiki页面 。我们的JIRA始终非常欢迎您提出建议，反馈和功能要求。
