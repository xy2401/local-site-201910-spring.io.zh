支持Spring数据

Spring Data是一个项目的集合，旨在使开发使用新数据访问技术（例如非关系（NoSQL）数据库）的Spring支持的应用程序更加容易。基于公开存储库对象的模型，Spring Data允许应用程序在许多新平台上以简单一致的方式访问数据。Spring Batch 2.2.0。RELEASE提供ItemReader为Neo4j的和MongoDB实现以及ItemWriter impelementaions为Neo4j的，MongoDB的和的GemFire。我们还创建了RepositoryItemReader和RepositoryItemWriter 。这些实现中的每一个都分别包装PagingAndSortingRepository和CrudRepository任何自定义实现。

Java配置

通过Spring Batch 2.2.0加入大多数其他主要的Spring项目。发布，您将能够通过Java config配置批处理作业。@EnableBatchProcessingannotation不仅提供对可用于构建批处理作业的构建器的访问，而且还提供了无需附加配置即可自动装配许多有用对象（ JobRepository ， JobLauncher ， JobRegistry ， PlatformTransactionManager ）的功能。以下是通过新的Java配置配置的示例作业。

@Configuration
 @EnableBatchProcessing
 @Import(DataSourceCnfiguration.class)
 public class AppConfig {

    @Autowired
    private JobBuilderFactory jobs;

    @Bean
    public Job job() {
        return jobs.get("myJob").start(step1()).next(step2()).build();
    }

    @Bean
    protected Step step1() {
       ...
    }

        @Bean
    protected Step step2() {
     ...
    }
 }

非识别作业参数

通常需要使用运行具有相同参数的批处理作业。从Spring Batch 2.2.0开始。释放，Spring Batch现在通过允许作业接受非标识性作业参数（对创建新JobInstance没有贡献的参数）来支持此用例。此更新确实需要代码更改以及对Spring Batch作业存储库使用的基础数据库架构的更新。幸运的是，我们提供了一个迁移脚本来帮助进行过渡。您可以在《 入门指南》中阅读有关迁移脚本的详细信息。

AMQP支持

利用Spring AMQP项目Spring 2.2.0。RELEASE提供对AMQP端点的读写支持。

SQLFire支持

Spring Batch的早期版本为作业存储库中要使用的数据库提供了许多选项。随着Spring Batch 2.2.0的发布。发布，我们添加了对SQLFire的支持，这是您存储作业存储库数据的另一种选择。

依赖升级

为了保持Spring Batch的依赖关系为最新工作的一部分，我们更新了Batch以支持Spring 3.2.x（最低支持级别为3.1.2）以及Hibernate 4（在基于Hibernate的ItemReaders和ItemWriters中） ）。

其他更新和修复

除了所有新功能，我们还解决了许多错误并提供了许多其他改进。在Spring Batch 2.2.0之间进行了更改的完整列表。RELEASE和您当前的Spring Batch版本可以在changelog中找到。
