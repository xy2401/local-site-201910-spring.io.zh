Spring 本周-2013年8月6日

工程
乔什·朗（Josh Long）
2013年8月7日

欢迎回到 Spring 本周的另一期。8月1日，我在SpringSource庆祝我的三年级。我继续享受自己的生活，其中很大一部分是我与您（有史以来最令人惊叹的社区）的互动。谢谢大家。

您是否已预订SpringOne2GX的票？今年的表演很特别。在作为Spring Developer Advocate的工作中，我在世界各地的许多会议上发表演讲。询问任何开发人员，他们都会确认大数据（和Hadoop），响应式Web应用程序，REST，移动应用程序开发和云计算是当今炙手可热的热门话题。Pivotal和Spring支持当今的开发人员，而我估计SpringOne2GX的议程代表了当今开发人员的内容的完美融合。查看议程 。最近，我们刚刚添加了有关大数据以及OAuth的REST服务安全性的讲座。这将是我们在Pivotal之下的第一场展览，它是您可以与在SpringSource和社区中关注您所关注的技术的开发人员交谈的唯一场所。如您所知，我们刚刚宣布了Cloud Foundry会议， Platform和SpringOne2GX全票证持有人可以免费注册该节目-该节目与两天前与SpringOne2GX相同。如果我每年只需要为一个节目付费，那将是那个节目。快点早鸟价在这个星期五到期！

  1. 重要新闻 ：Phil Webb和David Syer博士刚刚宣布了Spring Boot ，它简化了Spring应用程序的开发。Spring Boot在Spring之上提供了一个自以为是的层，这样做使以最小的麻烦启动和运行应用程序变得非常简单。严重的是，这些东西会让您大吃一惊。在您阅读了这篇简短而甜美的文章之前，请不要再继续阅读！放手一搏，一定要让我们知道您的经历 ！
  2. Spring Framework 3.2.4维护版本现已发布 ，其中包含针对SpringOXM的重要安全修复程序。
  3. Spring Data Redis负责人和忍者Jennifer Hickey刚刚宣布了两个Spring Data版本的可用性。Spring Data Redis 1.1，M2，具有许多新功能，包括增强的数据流水线，Redis 2.6脚本等等。Spring Data Redis 1.0.6也可用，并且具有错误修复和较小的改进。
  4. Spring Mobile和Android主管Roy Clarkson刚刚宣布了Spring Mobile 1.1.0。RC1 ，改进了Spring Mobile中的设备检测和视图分辨率。Roy还宣布了Spring Mobile稳定版1.0.2的新版本，该版本具有类似的改进，但有一些反向移植。
  5. Spring Data忍者Oliver Gierke刚刚宣布， Spring Data Babbage的最终发布候选版现已发布。此版本以Charles Babbage命名。此版本具有对MongoDB聚合框架的支持，并改进了多态查询的执行，支持在JPA手动定义的查询中使用SpEL表达式，改进了使用@IdClass的实体处理， countBy(..)方法Neo4j的资料库， 等等 。
  6. 现在可以在线获取不带Lamba的网络研讨会功能编程和带有Cucumber for Automation的Spring的回放。请务必检查一下！
  7. 几周前，我们的朋友Johnathan Mark Smith编写了一个视频，介绍了如何使用Spring Data MongoDB和Java配置 。看看这个！而且，如果您要制作很棒的视频，请随时分享。我也想在 Spring 的“本周”上发布它们！
  8. 当我看到Reactor项目负责人Jonathan Brisbin的一条推文时，我笑了笑，他在推文中说：“处理器吞吐量：笔记本电脑上为90M ops /sec。1个线程+ @LMAX Disruptor。不是#fastdata，而是#uberfastdata”，然后链接到代码中的测试用例 。不用说， Reactor将会使事情大摇大摆！（当然，在SpringOne2GX上，我们还将在Reactor上有更多内容。
  9. Tomcat的最新版本， Apache Tomcat 8.0.0-RC1（alpha）现已发布！有很多新功能。值得注意的是，Tomcat 8将成为第一个支持JSR 356 Web套接字的Tomcat。这是对Spring 4 最近宣布的Web套接字支持的完美补充。
  10. Mohan Srihari Kantipudi撰写了一篇有关Spring基本REST功能的精彩文章
  11. 我喜欢Gregor Riegler在Spring Loaded上的帖子，这是开源中最秘密的信息。Spring Loaded是一个Java代理，可让您在处理代码时重新加载代码（无需重新部署！）。这是一个非常酷的职位，我希望你会考虑使用Spring 加载了。
