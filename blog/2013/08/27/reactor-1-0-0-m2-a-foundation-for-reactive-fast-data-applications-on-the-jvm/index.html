反应器1.0.0。M2 – JVM上的快速反应数据应用程序的基础

工程
乔恩·布里斯宾
2013年8月27日

我很高兴宣布Reactor即将迈向1.0的第二个里程碑版本！适用于Reactor 1.0.0的Maven工件。M2在常规的里程碑存储库中可用。

什么是反应堆？

Reactor是用于在JVM上构建高吞吐量，异步，反应式应用程序的基础框架。它提供了Selector风格的主题匹配功能，用于事件路由，动态消费者分配，超快速任务处理器以及反应性Stream和Promise API，用于异步处理数据和协调异步任务。它通过提供语言扩展来提供全面的Groovy语言支持，从而使在Groovy中编写Reactor应用程序变得非常精明。它还具有易于使用的Spring支持，可自动将带注释的POJO连接到Reactor。

此版本中有什么内容？

这第二个里程碑包括许多错误修正和一些令人兴奋的新功能。反应堆现在包括一个Processor抽象，它是基于LMAX Disruptor RingBuffer的高度优化的任务处理器。它使用Reactor的通用抽象来配置RingBuffer，并允许您使用Reactor的通用API而不是Disruptor专用的API。根据设计，它也跳过了Reactor提供的Selector匹配和动态的Consumer分配，以便尽其所能减少吞吐量的每一个下降。MacBook Pro上的轶事基准表明，处理器每秒可通过管道发送约100,000,000个事件。是的，您没看错：每秒1 亿 ！

1.0.0。M2在Reactor API中还包括一个小而重要的新功能，该功能可优化Reactor中的事件发布，以使吞吐量提高约30-50％。因为它准备了反应堆的优化的用户列表，所以它并不适合所有情况，但是对于每秒额外的 1000万个事件，这是一个很棒的新功能。

优化发布

Reactor强大的方面之一是Selector匹配主题pub / sub。它使您可以轻松地使用主题，匿名对象，可分配的类型层次结构，URI路径匹配或正则表达式（或任何其他类型的Selector匹配，如果您实现自己的特定于域的Selector），则将处理程序分配给事件。但是许多应用程序可以在启动时分配其处理程序，这意味着可以优化这些消费者的路径，以进行有效的事件发布。新的反应堆方法prepare(Object)允许您预先选择“使用者”作为密钥。它返回消费者本身，事件发布者可以使用它来有效地通知新事件。

// Create Environment in which Reactors operate
Environment env = new Environment();
Reactor reactor = Reactors.reactor().env(env).get();

reactor.on($("say.hello"), new Consumer<Event<String>>() {
	public void accept(Event<String> ev) {
		System.out.println("Hello " + ev.getData() + "!");
	}
});

Consumer<Event<String>> sayHello = reactor.prepare("say.hello");
for(String name : listOfNames) {
	sayHello.accept(name);
}

RingBuffer任务处理器

反应器1.0.0。M2包括Processor抽象。它是一个由LMAX Disruptor RingBuffer支持的简单任务处理器，旨在将其无缝集成到Reactor中使用的反应式API中，因此它使用诸如Supplier和Consumer之类的通用抽象。可以在单个表达式中创建完全配置的处理器，而使用Java 8 lambda则更为简洁：

Processor<Message> proc = new ProcessorSpec<Message>()
	.dataSupplier({ return new Message(); })
	.consume({ msg -> // handle the updated Message object })
	.get();

的Processor提供了两种与底层RingBuffer进行交互的方式。单操作模式通过请求一个Operation来自的对象Processor通过调用prepare()方法。的Operation有个get()方法，以访问创建RingBuffer时已填充的预分配事件对象。可以使用新数据更新此对象的成员。准备发布操作并触发事件处理程序时，只需调用操作的commit()方法。

public class Message {
	int type;
	Buffer buffer;
}

@Autowired
Processor<Message> proc;

public void handle(Buffer buff) {
	Operation<Message> op = proc.prepare();

	op.get().type = buff.readInt();
	op.get().buffer = buff;

	op.commit();
}

如果您可以处理大量数据，则Processor提供一个batch(int, Consumer ) 允许您指定批量大小并以形式传递变量的方法Consumer更新每个事件的数据是谁的工作。如果批处理大小大于基础RingBuffer的大小，则将隐式刷新该批处理，否则将延迟发布步骤，直到达到批处理大小为止。这通常会提高吞吐量和效率。

public class Message {
	int type;
	Buffer buffer;
}

@Autowired
Processor<Message> proc;

public void handle(List<Buffer> buffs) {

	proc.batch(buffs.size(), new Consumer<Message>() {
		ListIterator<Buffer> it = buffs.listIterator();

		public void accept(Message msg) {
			Buffer next = it.next();

			msg.type = next.readInt();
			msg.buffer = next;
		}
	});

}

Maven工件

ext {
	reactorVersion = '1.0.0.M2'
}

repositories {
	mavenCentral()
	maven { url 'http://repo.springsource.org/libs-release' }
	maven { url 'http://repo.springsource.org/libs-milestone' }
}

dependencies {
	// Reactor core
	compile "org.projectreactor:reactor-core:$reactorVersion"

	// Reactor Groovy support
	compile "org.projectreactor:reactor-groovy:$reactorVersion"

	// Reactor TCP client/server
	compile "org.projectreactor:reactor-tcp:$reactorVersion"

	// Reactor Spring support
	compile "org.projectreactor:reactor-spring:$reactorVersion"
}
