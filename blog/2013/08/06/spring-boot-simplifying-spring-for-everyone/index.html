Spring Boot –为所有人简化春天

工程
菲尔·韦伯
2013年8月6日

（此博客文章由Phil Webb和Dave Syer共同撰写）。

我们很高兴地宣布一个名为Spring Boot的新项目的第一个里程碑版本。

Spring Boot旨在简化创建具有Spring动力的生产级应用程序和服务的过程。它从Spring平台的角度出发，以便新老用户都能快速找到所需的信息。你可以用它来创建可以使用启动独立的Java应用程序‘java -jar’或更传统的WAR部署。我们还提供了运行“ spring脚本”的命令行工具。

下图显示了Spring Boot作为对更大的Spring生态系统的关注点。它为用户提供了一个较小的表面积，可以从Spring的其余部分获取并提取价值：

Spring Boot的主要目标是：

  • 为所有Spring开发提供根本上更快且可广泛访问的“入门”体验
  • 开箱即用，但是随着需求开始偏离默认值，您很快就可以摆脱困境
  • 提供大类项目通用的一系列非功能性功能（例如，嵌入式服务器，安全性，指标，运行状况检查，外部化配置）

春天开机不生成代码，也绝对对XML配置没有要求。

春季剧本

Spring Boot附带一个小型命令行应用程序，可用于运行“ spring脚本”。Spring脚本是用Groovy编写的，这意味着您具有类似Java的熟悉语法，而没有太多样板代码。通过查看您编写脚本的方式，我们能够推断出很多信息。例如，这是一个简单的Web应用程序：

@Controller
class ThisWillActuallyRun {

    @RequestMapping("/")
    @ResponseBody
    String home() {
        return "Hello World!"
    }

}

当您使用‘spring run webapp.groovy’运行此应用程序时，发生了很多事情：

  • 您的脚本通过常见的‘import’语句得到了增强，可节省您键入的时间
  • 我们识别@ResponseBodyannotation并下载适当的Spring JAR
  • 我们自动创建Spring @Configuration ，否则您需要编写
  • 我们启动一个嵌入式Servlet容器并处理端口8080上的传入请求

命令行工具可以识别许多不同类型的Spring应用程序，包括Web，批处理和集成。GitHub存储库中提供了许多示例 。

Java的Spring Boot

您无需使用命令行工具或编写Groovy代码即可获得Spring Boot的好处。我们也有一流的Java支持。例如，这是用Java编写的同一应用程序：

import org.springframework.boot.*;
import org.springframework.boot.autoconfigure.*;
import org.springframework.stereotype.*;
import org.springframework.web.bind.annotation.*;

@Controller
@EnableAutoConfiguration
public class SampleController {

    @RequestMapping("/")
    @ResponseBody
    String home() {
        return "Hello World!";
    }

    public static void main(String[] args) throws Exception {
        SpringApplication.run(SampleController.class, args);
    }
}

除了import语句，此示例与早期的Groovy脚本之间的主要区别在于调用SpringApplication和@EnableAutoConfigurationannotation的main()方法。

显然，对于Java，您还需要一个构建系统来编译和打包代码。我们提供了许多方便的“入门” POM，您可以将它们与Maven ， Gradle或Ant + Ivy一起使用，以快速获取适当的依赖项。例如，上面的应用程序仅需要对spring-boot-starter-web模块的单个依赖项。

我们还提供Maven和Gradle插件，使您可以打包一个完全自包含的“胖罐”，该胖罐可以从命令行启动：

准备生产

Spring Boot还包含将应用程序推入生产时经常需要的有用功能。我们可以自动提供Web终结点，可用于监视应用程序的运行状况，提供基本指标或用于分析生产问题（例如线程死锁）。我们还提供了一个新的@ConfigurationPropertiesannotation，可用于外部化应用程序配置（完整支持JSR-303 @Validannotation）。
