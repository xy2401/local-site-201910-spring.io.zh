Spring XD 1.0里程碑2发布

发布
马克·波拉克
2013年8月14日

今天，我们很高兴地宣布Spring XD的1.0 M2版本（ 下载 ）Spring XD是一个统一的，分布式的和可扩展的系统，用于数据提取，实时分析，批处理和数据导出。该项目的目标是简化大数据应用程序的开发。

Spring XD的第二个里程碑版本引入了几个新功能，这些功能使提取和处理实时数据流以及协调基于Hadoop的批处理作业变得更加容易。在此博客文章中，我们将介绍

  • 贝壳
  • 新货源，汇和运输
  • DSL改进
  • 批处理作业

贝壳

最引人注目的新功能是交互式外壳程序的引入。该外壳程序为您提供了一种轻松的方法来创建新的流和作业，查看指标，与Hadoop交互等等。作为shell的介绍，我将重做M1博客文章中的一些示例。

以单节点模式（$ XD_HOME / bin / xd-singlenode）启动Spring XD，然后在单独的窗口中启动Shell。下面的示例演示如何创建一个简单的流，该流将通过http发布的数据写入文件。注意，shell提供了命令的制表符完成提示。

$bin>./xd-shell
Welcome to the Spring XD shell. For assistance hit TAB or type "help".

xd:>stream create --name httpStream --definition "http | file"

xd:>tap create --name httpTap --definition "tap httpStream | counter"

xd:>http post --target http://localhost:9000 --data "helloworld"

您可以列出所有流和水龙头以验证它们是否已创建

xd:>stream list
Stream Name  Stream Definition
-----------  -----------------
httpStream   http | file

xd:>tap list
Tap Name  Stream Name  Tap Definition
--------  -----------  ------------------------
httpTap   httpStream   tap httpStream | counter

如果您检查文件，位于目录中/tmp/xd/output/httpStream.out ，您将看到世界您好消息。

xd:>! cat /tmp/xd/output/httpStream.out

The httpTap is simply counting messages.  To see the name of the counter created and its value, use the counter shell command

xd:>counter list
Counter name
------------
httpTap

xd:>counter display --name httpTap
1

在单节点模式下，计数器位于内存中，但也支持Redis，这是不使用单节点模式时的默认值。您可以通过以下方式启用Redis支持： –analytics redis命令行参数。

要创建将数据存储在Hadoop中的Twitter流，并创建推文中＃标签频率的实时计数器，请运行以下命令。注意，要获取consumerKey和consumerSecret，您需要注册一个Twitter应用程序。如果尚未设置，则可以在Twitter Developers站点上创建一个应用程序以获取这些凭据。

xd:> stream create bieberStream --definition "twittersearch --consumerKey=<your-key> --consumerSecret=<your-secret> --query=bieber | hdfs"

xd:> tap create --name bieberHashTap --definition "tap bieberStream | field-value-counter --fieldName=entities.hashTags.text --counterName=bieberHashCount"

xd:> hadoop config fs --namenode hdfs://localhost:8020

xd:> hadoop fs cat /xd/bieberStream/bieberStream-0.log

在计数器主题上，引入了一种新的汇总计数器类型，该类型将消息中字段的计数汇总到每年，每月，每天，小时和分钟的时间段中。

DSL改进

到目前为止，所示的流处理管道是线性的，但是经常需要支持更复杂的流。为了开始解决这种情况，在M2中引入了命名通道。可以使用命名通道来代替源模块或接收器模块。与unix主题保持一致，从/向特定通道获取/接收数据使用“>”字符，并且名称以“：”为前缀。

这是一个示例，显示了如何使用命名通道共享由不同输入源驱动的数据管道。

xd:>stream create out --definition ":foo > file --name=demo"
xd:>stream create in1 --definition "http > :foo"
xd:>stream create in2 --definition "time > :foo"
xd:>http post --target  http://localhost:9000 --data "hello"

查看输出文件

xd:>! cat /tmp/xd/output/demo.out

您会看到单词“ hello”与时间戳记值混合在一起。计划将消息扇出到多个流，以及支持基于消息内容将消息路由到不同的流，以用于将来的里程碑版本。

新货源，水槽和运输工具

此版本提供了几个其他的源和接收器模块

  • MQTT
  • 兔子MQ
  • Splunk
  • JDBC

还要注意的是，我们添加了对4个Hadoop版本的支持

  • hadoop10-Apache Hadoop 1.0.4（默认）
  • hadoop11-Apache Hadoop 1.1.2和Hortonworks Data Platform 1.3
  • hadoop20-Apache Hadoop 2.0.5-alpha
  • phd1-Pivotal HD 1.0

您可以通过传递命令行选项来选择启动XDContainer时要使用的特定分发罐。 –hadoopDistribution 。您还应该能够使用其他Hadoop发行版，例如Hadoop1.2.x。在以后的版本中，我们将为其他发行版添加显式选项。值得注意的是，在示例存储库中有一个示例显示了如何将Spring XD与Pivotal HD的HAWQ功能一起使用。

M1版本提供了本地和Redis队列支持的传输，用于模块之间的通信，如DSL中的管道符号所示。M2版本提供了对Rabbit支持的传输的支持，使您可以利用功能齐全的消息代理来获取流。

工作和触发器

可以使用Spring XD执行批处理作业，并设置触发器以启动这些作业。作为示例，我们可以在Hadoop中重用经典的单词计数示例，以提供具有两个步骤的简单工作流程流程。第一步是将文件复制到HDFS中，第二步是运行wordcount MapReduce作业。

要运行该示例，请克隆spring-xd-samples存储库并构建示例batch-wordcount。然后如下所示复制jar，config和数据文件。

$ cd batch-wordcount
$ mvn clean assembly:assembly
$ cp target/batch-wordcount-1.0.0.BUILD-SNAPSHOT-bin/modules/job/* $XD_HOME/modules/job
$ cp target/batch-wordcount-1.0.0.BUILD-SNAPSHOT-bin/lib/* $XD_HOME/lib
$ cp target/batch-wordcount-1.0.0.BUILD-SNAPSHOT-bin/nietzsche-chapter-1.txt /tmp

现在停止并以单节点模式（$ XD_HOME / bin / xd-singlenode）重新启动Spring XD。然后在shell中执行以下命令

xd:> job create --name wordCountJob --definition "wordcount"

或者，也可以指定计划作业执行的cron表达式。您可以通过查看map reduce作业的输出来验证结果

xd:> hadoop config fs --namenode hdfs://localhost:8020
xd:> hadoop fs cat /count/out/part-r-00000

还支持在工作流程中执行其他步骤，例如执行Hive或Pig脚本。要编写这些类型的工作流，请参阅Spring for Apache Hadoop参考指南。还支持基于非Hadoop的步骤。

还有更多

下一个版本的主要主题是通过合并Spring Batch Admin项目的组件来公开更多针对批处理作业的管理功能。通过将消息发送到命名通道以及从命名通道接收作业状态通知，您将能够触发批处理作业。例如，这将使您可以轻松地基于数据可用性来设置批处理作业的触发

file --dir "/data/inbound" | jobParameterCreator > :wordCountJob

当目录中的文件可用时，将启动wordcount批处理作业/data/inbound通过向命名频道：wordCountJob发送消息。作为批处理作业执行，数据流将可为你消费有关数据消息JobExecution ， StepExecution等。

:wordCountJob.notifications > filter --expression "payload.status.equals('COMPLETED')" | email --address "[email protected]"

使用通道在流和作业之间交换数据是您可以开始了解Spring XD如何采取步骤统一流和批处理两个域的领域之一。敬请关注！
