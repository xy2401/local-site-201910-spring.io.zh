Spring has had pretty good Java generics support for a while. For example, with version 3.2 you can easily inject all beans of a specific type into a generic List just by using the @Autowired annotation:

@Autowired
private List beans; 
// all beans that extends MyType will be injected

Spring’s conversion service, binding system and Web MVC framework are all ‘generic aware’ and there are also the handy GenericCollectionTypeResolver and GenericTypeResolver utility classes that you can use in your own code.

Spring Framework 4.0和Java泛型

工程
菲尔·韦伯
2013年12月3日

随着Spring Framework 4.0的最新RC2版本的发布 ；以及将于今年年底之前发布的GA，以下是一些预告片，如果您使用Java泛型类型，这些更改应该可以改善您的生活。

Spring已经有相当不错的Java泛型支持一段时间了。例如，在3.2版中，您可以轻松地将特定类型的所有bean注入泛型List只需使用@Autowired注解：

@Autowired
private List beans; 
// all beans that extends MyType will be injected

Spring的转换服务，绑定系统和Web MVC框架都是“通用的”，并且也很方便GenericCollectionTypeResolver和GenericTypeResolver您可以在自己的代码中使用的实用程序类。

不幸的是，Spring早期版本的一个问题是您不能使用泛型作为限定符的形式。例如，您可能具有以下配置：

@Configuration
public class MyConfiguration {

	@Bean
	public Store stringStore() {
		return new StringStore();
	}

	@Bean
	public Store integerStore() {
		return new IntegerStore();
	}

}

如果您尝试@AutowireStore在Spring 3.2中，您将获得以下异常：

“未定义[Store]类型的合格Bean：预期的单个匹配Bean，但找到2：stringStore，integerStore”

Spring在这里告诉您，它发现了两个实现Store并且无法区分它们。

一种可能的解决方法是使用混凝土StringStore和IntegerStore直接类，而不是使用泛型Store接口。但是，如果您的实现类是“私有”的，或者如果它们是使用代理动态创建的，则这可能是不可能的（就像在Spring Data中那样） Repository ）。

另一种选择是使用@Qualifier都在@Bean方法和@Autowire字段为Spring提供有关您实际需要哪个Bean的提示。

幸运的是，从Spring Framework 4.0开始，不再需要这种解决方法。Spring 4.0会自动将泛型视为一种形式@Qualifier 。例如：

@Autowired
private Store s1; // Injects the stringStore bean

@Autowired
private Store s2; // Injects the integerStore bean

注入列表时，可以使用嵌套泛型事件：

// Inject all Store beans as long as they have an  generic
// Store beans will not appear in this list
@Autowired
private List> s;

在幕后，新的ResolvableType类提供了实际使用泛型类型的逻辑。您可以自己使用它来轻松浏览和解析类型信息。多数方法ResolvableType自己会返回一个ResolvableType ， 例如：

// Assuming 'field' refers to 's' above
ResolvableType t1 = ResolvableType.forField(field); // List> 
ResolvableType t2 = t1.getGeneric(); // Store
ResolvableType t3 = t2.getGeneric(); // Integer
Class c = t3.resolve(); // Integer.class

// or more succinctly
Class c = ResolvableType.forField(field).resolveGeneric(0, 0);

这是另一个示例，显示了如果您获得超类型或已实现的接口，则如何保留通用信息。

// Assuming 'm' refers to a method that returns MultiValueMap 
ResolvableType t1 = ResolvableType.forMethodReturnType(m); // MultiValueMap 
ResolvableType t2 = t1.asMap(); // Map>
ResolvableType t3 = t2.getGeneric(1); // List
ResolvableType t4 = t3.getGeneric(); // String
Class c = t4.resolve(); // String.class

// or more succinctly
Class c = ResolvableType.forMethodReturnType(m).asMap().resolveGeneric(1, 0);

Spring 4.0将于12月12日发布，但是如果您今天想对通用支持进行一下尝试，可以尝试在我们的里程碑资料库中找到最新的4.0版本候选版本 。
