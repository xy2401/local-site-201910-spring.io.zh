宣布Spring Framework 4.0 GA发行

发布
阿德里安·科耶尔（Adrian Colyer）
2013年12月12日

在过去的十年中，Spring Framework重新发明了企业Java，成为企业Java中占主导地位的编程模型。今天，我们发布了Spring Framework 4.0，这是Spring的全新主要版本，使Spring处于现代Java开发的最前沿。与即将面世的Spring IO平台的其余部分一起，Spring Framework 4.0的定位是实现基于JVM的下一个十年的创新，以响应并设定开发人员生产力，大数据，云，REST和微服务体系结构中的趋势。

Spring Framework 4.0与Java 8完美配合，还允许应用程序部分或全部用Groovy（JVM上的一种简洁的动态语言）编写。 Spring Framework 4.0还为诸如Spring Boot之类的Spring IO平台的新进展提供了基础。Spring Boot几乎完全消除了初始配置工作。像这样的开发人员生产力选项通常与单一用途的框架相关联，这些框架的范围受到限制，并且根本无法提供Spring提供的广度，深度和经实践检验的生产使用年限。

Spring IO的主要目标是通过Spring Boot等组件使Java成为开发人员生产力的领导者。爱德华·希亚特（Edward Hieatt）在SpringOne 2GX 2013大会的主题演讲中对Boot进行了一些精彩评论：“ Pivotal Labs设定了开发人员的幸福感和样板代码的不容忍性的很高标准，从历史上讲，它做了很多Ruby on Rails开发，我们将Spring Framework 4.0与Spring Boot结合使用的Ruby on Rails开发人员对它的生产力和在单个编程模型上解决各种场景的能力印象深刻。”Edward Hieatt是Pivotal Labs的首席运营官，他使用Spring Framework 4.0和Spring Boot等技术构建了spring.io网站。

现代化

Spring Framework 4.0现在基于Java 8的预发布版本提供对Java 8的一等支持，并且一旦OpenJDK 8在2014年3月成为GA即可立即投入生产。Spring Framework 4.0不仅可以在Java 8上运行，它还使利用新语言功能（如Lambda，方法引用，JSR-310日期和时间以及可重复的注释）变得非常容易和自然。该框架也已更新，可以与流行的开源库（如Hibernate，Quartz和EhCache）的最新版本一起使用。

新的应用架构

诸如亚马逊，谷歌，Facebook等许多行业领导者已经放弃了单片Java架构，转而使用微服务架构（MSA）和REST-通常将表示逻辑转移到客户端层。使用RESTful原理 分解应用程序的这一重要趋势仅在概念上类似于传统的SOA，但我们同意那些认为它应该被称为不同的观点。

詹姆斯·刘易斯（James Lewis）写了一篇受欢迎的InfoQ 演讲 ，他在演讲中阐明了我们同意的微服务的一些关键特征：

  • 每个应用程序只能做一件事
  • 足够小以适合您的头部
  • 足够小，可以将它们扔掉
  • 嵌入式Web容器
  • 打包为单个可执行jar
  • 使用HTTP和HATEOAS解耦服务
  • 每个应用都会公开有关其自身的指标

Spring核心框架在直接和基础支持下，正在展望这一微服务的未来。REST是整个Spring的一等公民，体现在Spring MVC的核心4.0框架以及Spring IO平台组件（如Spring Boot，Spring HATEOAS，Spring Security，Spring Social和Spring Data）中。借助Spring Framework 4.0，开发人员可以使用新的非阻塞性AsyncRestTemplate以及Java语言功能（例如Futures）来创建更多响应式事件驱动的REST服务。

基于Spring Framework 4.0的@Conditional bean定义基础架构的Spring Boot为基于REST的微服务提供了“无容器”（嵌入式）运行时，除了其他轻量级功能外，还可以打包为单个可执行JAR。例如，公开本身的指标。

Spring Framework 4.0对HTML5 / WebSocket RFC 6455 / JSR-356的领先支持还启用了反应式，事件驱动的应用程序。Spring的WebSocket支持提供了当今网络拓扑所需的基本协议回退机制。更重要的是，它为构建用于Web应用程序的WebSocket样式的消息传递体系结构提供了基础。该基础集成了STOMP，Spring MVC和轻量级客户端消息代理（例如msgs.js）或成熟的服务器端消息代理（例如RabbitMQ）。而且，Spring Framework 4.0现在包含了来自Spring Integration的核心消息和通道抽象，以支持服务器端消息传递。

揭穿神话

许多不熟悉最新版本的人将Spring与XML紧密联系在一起，但是长期以来，它一直提供丰富的，注释驱动的编程模型。该模型在Spring Framework 3中引入，并在Spring Framework 4中进一步完善。从两个版本来看，该模型提供的功能如下：

  • 通过合成定制注释
  • 配置类和工厂方法
  • Spring Expression Language
  • 灵活的MVC / REST控制器样式
  • 声明式验证和格式
  • 声明式调度和缓存

此外，Java配置在Spring项目中得到广泛支持，包括核心框架，作为XML和注释驱动配置的另一种选择。

与JCP合作

Spring的Java EE支持方法体现在Spring Framework 4.0在配置，WebSocket，DI，Batch等方面的创新。对于相关的Java EE规范，Spring在JCP中进行协作以进行定义，然后努力超越相关的JSR规范。Pivotal一直是JCP围绕Java EE 7关键功能（如WebSocket）的积极合作者，并直接通过Spring Batch启发了批处理JSR。Java EE 7还对Spring支持的JMS，JTA，JPA和Bean验证规范进行了一些修订。

Spring的发展远远超出了Spring核心框架中要解决的内容。需要一个模块化的微平台来解决未来的架构，趋势和技术。可以启用新功能，但占用的空间却很小的东西。Spring Framework 4.0是Spring IO平台的关键元素。Spring IO平台的其他组件补充了Spring Framework 4.0，因为它们解决了Hadoop和NoSQL，批处理，企业集成模式以及响应/事件驱动的应用程序开发方面的现代趋势。这些选项允许仅使用作业所需的依赖项来开发微服务，同时保持一致但简单的POJO编程模型。

Spring Framework 4.0是开源的，根据Apache 2.0许可免费提供。我们建议您使用Maven或Gradle，并且可以从spring.io网站获取坐标。
