Spring Data MongoDB 1.4 M1中的新增功能

工程
托马斯·达里蒙特
2013年12月4日

作为Spring Data发布火车的第一个里程碑，Codd现在已经降温了一些，我想强调一下MongoDB模块随附的一些新功能。

SpEL支持聚合框架中的预测

有时，在MongoDB聚合框架投影中定义算术表达式可能会非常复杂。

假设订单总价的一部分是总价，可以使用以下公式有效地计算出总价： (netPrice * discountRate + fixedCharge) * taxRate 。折现率为0.8，固定费用为1.2，税率为1.19，根据DBObject用MongoDB聚合框架对该公式进行编码如下所示：

{ "aggregate": "product",
  "pipeline": [
    { "$project": {
        "name": 1,
        "netPrice": 1,
        "grossPrice": {
          "$multiply": [ 
            { "$add": [ { "$multiply" : [ "$netPrice", 0.8 ] }, 1.2 ] }, 1.19
          ]
        }
      }
    }
  ]
}

借助我们对在适当的MongoDB投影表达式中转换SpEL表达式的新支持，这将变得更加容易，因为您可以按原样有效地使用源公式：

import static org.springframework.data.mongodb.core.aggregation.Aggregation.*;
 
double discountRate = 0.8;
double fixedCharge = 1.2;
double taxRate = 1.19;
 
TypedAggregation<Product> agg = newAggregation(Product.class,
  project("name", "netPrice")
    .andExpression("(netPrice * [0]  + [1]) * [2]", 
      discountRate, fixedCharge, taxRate)
    .as("grossPrice")
);
 
AggregationResults<DBObject> result = mongoTemplate.aggregate(agg, DBObject.class);
List<DBObject> resultList = result.getMappedResults();

在幕后，我们将SpEL表达式的已解析AST（抽象语法树）转换为适当的MongoDB聚合框架表达式。请注意，我们通过使用数组索引运算符来引用先前声明的变量，该数组索引运算符引用了varargs Object数组，即….andExpression(…)作为第二个参数。您可以在的单元测试中找到其他用法示例SpelExpressionTransformer 。

使用Java Config审计配置

在此版本中，我们克服了Spring Data模块的最后一个功能，该功能严格要求XML配置-审计。
如果要在应用程序中使用审核，那么现在要做的就是使用新的@EnableMongoAuditing注释（或分别相当于JPA的注释）：

@Configuration
@EnableMongoAuditing
@EnableMongoRepositories
class Config {
 
  @Bean
  public MongoOperations mongoTemplate() throws Exception {
    MongoClient client = new MongoClient();
    return new MongoTemplate(new SimpleMongoDbFactory(client, "database"));
  }

  @Bean
  public MongoMappingContext mappingContext() {
    return new MongoMappingContext();
  }

  @Bean
  public AuditorAware<BusinessEntity> auditorProvider() {
    return new MongoAuditorProvider<BusinessEntity>();
  }
}

通过以下方式启用的基础架构@EnableMongoAuditing会自动拿起AuditorAware实例在ApplicationContext 。如果您只想在实体上设置创建和修改日期，则无需声明AuditorAware豆。

重新定义CRUD方法的行为

方法中定义CrudRepository通常由提供必要行为的商店特定类实现。但是，您可能希望通过简单的查询执行来覆盖这些方法的执行。现在，您可以使用注释任何CRUD方法@Query注释，用于定义应执行的查询表达式。对于基于MongoDB的存储库，其外观如下：

interface PersonRepository extends MongoRepository<Person, String> {

  @Query("{ 'username' : { $nin : [ 'admin' ] }}")
  List<Person> findAll();
}

该机制可用于支持存储库抽象的所有模块。

支持惰性DBRef

到目前为止，绑定到MongoDB DBRef的域模型属性已急切加载，如果实体之间具有双向DBRef关系，则会造成一些麻烦。您现在可以设置一个lazy上的属性@DBRef注释以声明要延迟解决的字段。如果我们在加载文档时检测到此类字段，则会为给定对象生成一个代理，并在调用该对象的任何方法时对其进行解析（从java.lang.Object ）。

class User{
    
  @DBRef(lazy = true) List<User> fans;
  // …
}
