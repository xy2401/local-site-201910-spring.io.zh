今年 Spring -2013年12月31日

工程
乔什·朗（Josh Long）
2013年12月31日

新年快乐！欢迎回到 Spring 本周的今年最后一期！

我们将像往常一样做一些新闻，然后回顾一下去年关于Spring的新闻，其中确实有很多新闻！

  1. 我们知道也一直在研究Spring Boot的Matt Raible 刚刚为InfoQ撰写了我们刚刚发布的Spring 4 。如果您有时间，绝对值得一读！
  • XavierPadró的一篇不错的文章介绍了如何使用RequestHandlerRetryAdvice 。
  • 关于Spring的Stack Overflow有一个很好的讨论CrudRepository与更具体的JPA JpaRepository Spring Data负责人Oliver Gierke的精彩回答
  • 肯·布莱尔（Ken Blair）很好地介绍了使用Orika bean映射框架和Spring进行bean映射的方法
  • Groovy ninja Guillaume LaForge已开始对Groovy社区中的所有最新事件进行汇总，我也想邀请本专栏的读者阅读该系列文章
  • Viral Patel又回来了，这次有一篇关于使用Spring MVC处理请求标头的文章
  • Norris Shelton，Jr.撰写了一篇不错的文章，内容涉及使用Spring MVC从JSON服务呈现验证消息。
  • Intertech博客上的Jim White的帖子介绍了Spring 4的泛型感知依赖注入 。
  • Lalit Jha在如何将Play与Spring集成以进行依赖注入方面写了一篇不错的文章。老实说，我想我已经看到了不同的替代方法，因此显然有很多方法可以解决此问题。尽管如此，有趣的阅读！
  • 微软对在Windows Azure上运行Spring应用程序做了一点说明。我不一定推荐Azure，但是如果您要采用这种方法，…:)
  • numberformat博客上有一篇关于使用Spring Batch的Java Configuration API的不错的文章（标题为带有批注的spring batch 3.0.x的3 r ）。绝对值得一读！
  • Way2Java博客上有一篇关于使用Spring AOP的掷后建议的不错的文章。
  • 郑东平（我也假设这也是他的博客，尽管我只在@author代码中的属性…否则，此博客将在用户dempe之下进行汇总 ），以纯代码方式介绍如何使用Spring Data MongoDB的映射支持 。博客的其余部分为中文，因此我仅依靠Google Translate可以给我的东西。看起来不错。
  • xinhuaxuan和前一个条目都在相同的Chinese Blogs网站下，汇集了一篇有关使用Spring的不错的文章@Required注释以触发启动时检查是否存在依赖项。

回顾过去一年并试图对其进行总结非常困难！相信我，我已经尝试过并且尝试过 ！今年，Spring团队加入了GoPivotal ，这反过来又使所有相关人员为之疯狂！短期而言，以下是本周 Spring 本周引起我们注意的一些内容：

  1. 我们推出了Spring 4！ 此功能显着改善了REST支持，Java EE 7和Java 8支持，websockets支持以及许多其他功能。
  • 3..2…1…抬起！我们于2013年3月底加入了GoPivotal计划 ！（并在接下来的几个月中正式在其他国家/地区）
  • 在我们的年度SpringOne2GX大会上，我们在网络上发布了Spring的新家。IO ！（如果您正在阅读此博客，您可能已经知道了！：D）
  • Cloud Foundry v2已发布
  • 我们推出了Pivotal One ，这是一个全面的平台，具有Pivotal HD和Cloud Foundry以及流行的基础架构（如RabbitMQ ， Redis等），当然Spring也是所有功能的核心。
  • 当然，今年对常见的嫌疑人发布了许多精美的更新，包括Spring Social ， Spring Integration ， Spring Batch ， Spring Data ， Spring Security和Spring for Hadoop 。值得注意的是：Spring Batch获得了新的Java Configuration API，并看到其设计是刚刚发布的Batch标准的基础 。Spring Data向发布系列添加了更多项目。Spring Integration 3.0刚刚附带了许多新适配器和改进功能。Spring Security的Java配置API和开放式Web应用程序安全性的大量改进都在今年问世。
  • Spring Boot ，它对Spring平台和第三方库有一个坚定的看法，因此您可以以最小的麻烦开始，并在年中发布，从而获得好评和评论。Spring Boot在Java开发商和其他社区的开发商中引起了业界的广泛关注，他们一致认为：Boot具有生产力。
  • Spring XD提供了用于构建大数据感知应用程序的交钥匙设备。建立在Spring Batcha和Spring Integration之上的Spring XD是一个运行时 ，可以处理诸如自行扩展作业之类的常见事情。
  • 而且，当然，如果不提及Pivotal HD （我们的企业级Hadoop发行版），那么对Pivotal提出的关于大数据技术的任何讨论都是不完整的。当然，Pivotal HD与Spring for Hadoop配合良好，是Spring XD中支持的在线处理的自然数据仓库补充。
  • 尽管此列表可能会继续引起恶心，但2013年最令人惊讶的方面是您，这个社区：全世界，喧嚣而自豪，您的代码，博客，文章，视频，会议上的演讲，推文和互动推动了本周的发展通过这一切 Spring 。谢谢，继续前进！

我确定我代表所有Spring团队祝您新年快乐，最健康和最安全（如果您还没有跨过的话）！我们将于2014年再见！
