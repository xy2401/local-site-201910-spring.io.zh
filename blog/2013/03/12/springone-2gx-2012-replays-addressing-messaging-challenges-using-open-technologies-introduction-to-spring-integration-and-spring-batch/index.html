SpringOne 2GX 2012重播：使用开放技术应对消息挑战，Spring集成和Spring Batch简介

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2013年3月12日

使用开放技术应对消息传递挑战



对于现代应用程序，许多企业都面临着现代应用程序所面临的一些新的消息传递挑战，例如消息传递层的水平可伸缩性，异构消息传递系统和访问方法以及极端的事务处理。本演示文稿/演示将介绍企业如何通过使用Spring和RabbitMQ技术克服这些消息传递挑战。


Tom将为AMQP建立一个案例，说明SpringSource如何通过Spring AMQP和Spring Integration提供AMQP支持，并说明RabbitMQ是一种现代消息传递解决方案，该解决方案提供可靠，高度可用，可扩展的便携式消息传递系统，并具有可预测且一致的吞吐量和延迟。 ，并演示如何将Spring Integration和RabbitMQ逐步引入标准的Spring Web应用程序中。

Spring Integration和Spring Batch简介

在本课程中，您将了解Spring Integration和Spring Batch的全部含义，它们之间的区别，共性以及如何一起使用Spring Batch和Spring Integration。

正如同名的极具影响力的书中所述，我们将简要概述企业集成模式（EIP）。然后，基于这些模式，我们将看到Spring Integration如何实现消息驱动的应用程序的开发。这不仅使您可以模块化新的或现有的应用程序，而且还可以轻松地与外部系统集成。

本课程还将介绍Spring Batch。Spring Batch满足了任何批处理过程的需求，无论是大型金融机构中的复杂计算还是许多软件开发项目中存在的简单数据迁移任务。我们将介绍什么是Spring Batch，Spring如何处理批处理的概念以及Spring如何处理扩展批处理流程以能够处理任何数据量。

您还将看到Spring Integration和Spring Batch如何最大限度地重用核心Spring Framework提供的集成支持。除了提供可靠的，经过验证的基础之外，这还为所有已经熟悉Spring的开发人员大大拓宽了学习曲线。
