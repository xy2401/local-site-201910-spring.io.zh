Spring Data GemFire 1.3.0发布

工程
大卫·图兰斯基（David Turanski）
2013年3月14日

我很高兴宣布Spring Data GemFire 1.3.0的GA版本。除了许多小的错误修复和增强功能之外，此版本还包括一些值得注意的新功能，这些功能使使用GemFire编写Java应用程序变得更加容易：

功能注释支持


GemFire通过提供用于远程功能执行的框架，提供了“将代码带入数据中”的功能。为了保持Spring的核心价值，Spring Data GemFire隐藏了注册和执行远程功能所需的样板代码，从而使您可以编写POJO并专注于应用程序逻辑。有关详细信息，请参见《 Spring Data GemFire参考指南》中的“ 函数执行的注释支持”一章。

与GemFire数据源的简化连接

GemFire提供了许多选项来调整其连接池的性能，并配置如何同步管理本地数据。Spring Data GemFire命名空间支持所有这些选项，但是许多应用程序是仅需要对GemFire数据网格进行读/写访问的客户端。对于此类应用程序，现在可以作为客户端连接到GemFire，而无需显式配置池或客户端区域：


 <gfe-data:datasource>
        <gfe-data:locator host="${host}" port="${port}"/>
 </gfe-data:datasource>

上面的配置将为服务器上所有可用区域创建一个客户端缓存，池和代理客户端区域，并使用合理的默认值，并将它们注册为Spring Bean。

JSON支持

GemFire 7.0可以存储具有完整查询支持的JSON。通常，这要求应用程序使用JSONFormatter在GemFire的内部格式与JSON字符串之间进行转换。Spring Data GemFire现在提供了一个针对选定区域自动执行此转换的选项，这是使用Jackson的ObjectMapper从Object到JSON的一种转换方式。此功能使用Spring AOP来拦截Region和GemFireTemplate上的适当操作。有关详细信息，请参见《 Spring Data GemFire参考指南 》。
