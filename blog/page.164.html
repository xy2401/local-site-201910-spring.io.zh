Spring Boot 1.3中的DevTools

工程
菲尔·韦伯
2015年6月17日

Spring Boot 1.3将附带一个名为的全新模块spring-boot-devtools 。该模块的目的是尝试和改进在Spring Boot应用程序上工作时的开发时间体验。

要使用该模块，您只需要将其作为依赖项添加到您的Maven POM中：

<dependencies>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-devtools</artifactId>
    </dependency>
</dependencies>

或您的Gradle构建文件：

dependencies {
    compile("org.springframework.boot:spring-boot-devtools")
}

Spring Boot 1.3中的缓存自动配置

工程
斯特凡·尼科尔（StéphaneNicoll）
2015年6月15日

在过去的一年中，我们通过支持JSR-107（JCache）注释和更好的声明性模型来共享或外部化通用设置 ，大大改善了缓存抽象。在Spring Boot 1.3中，我们现在为其提供全面的自动配置。

简而言之，缓存抽象将缓存应用于方法，从而根据缓存中可用的信息减少执行次数。缓存逻辑是透明应用的：仅当指定了以下方法时，才会调用以下方法ISBN还不存在books缓存。在调用该方法时缺少Book ，则缓存将透明更新，因此进一步的调用不会再次调用该方法。

Spring Data GemFire支持Apache Geode

发布
约翰·布鲁姆
2015年6月12日

我很高兴地宣布， Spring Data GemFire现在已支持Apache Geode 。

什么是Apache Geode ？

简而言之， Apache Geode是Pivotal GemFire的 开源核心。作为BDS开源工作的一部分，Peotal将Geode提交给Apache软件基金会之后，Geode最近被接纳为Apache孵化器 。

从技术上讲， Apace Geode是一个内存中的分布式数据库（又名IMDG），使新的以及现有的Spring / Java应用程序可以在云规模上以高可用性和可预测的延迟运行，而不会牺牲一致性。应用程序能够实时处理和分析大数据 ，以实现有意义的，有意义的业务结果。

Spring Boot 1.3.0。M1现在可用

发布
菲尔·韦伯
2015年6月12日

我很高兴地宣布Spring Boot v1.3.0。M1现在可以从Spring里程碑存储库中获得 。这个版本建立在Spring Framework 4.2.0之上。RC1并在Spring Boot 1.2上提供了许多改进和新功能 。重点包括：

  • 一个新的spring-boot-devtools支持自动重启，LiveReload和远程更新的模块。
  • 对EhCache，Hazelcast，Infinispan，Redis，Guava或任何兼容的JSR-107（JCache）实现的广泛缓存支持。缓存度量标准信息现在也通过执行器公开（当基础技术支持时）。
  • 改进的指标支持，包括导出和聚合功能，以及使用Java 8时的性能大幅提升。
  • 用于基于Linux的操作系统的完全可执行的JAR，包括“服务”支持。现在，将启动作为Linux服务启动应用程序就像键入命令一样简单： sudo ln -s /var/myapp/myapp.jar /etc/init.d/myapp 。
  • 其他OAuth2支持，包括@EnableAuthorizationServer ， @EnableResourceServer和@EnableOAuth2Client注释。
  • 新HealthEndpoints适用于Elasticsearch，Email和JMS。
  • 马文start和stop帮助集成测试的目标。
