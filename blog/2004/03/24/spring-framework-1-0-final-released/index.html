Spring Framework 1.0 Final发布

发布
托马斯·里斯伯格
2004年3月24日

我们很高兴地宣布 Spring
Framework 1.0最终版
刚刚被释放。

1。范围


Spring 1.0是一个完整的Java / J2EE应用程序框架，涵盖了
以下功能：

  • 最复杂的轻量级
    当今可用的容器，具有各种口味的塞特犬和
    构造函数注入

  • 基于AOP的拦截框架
    与核心容器集成的AOP Alliance接口

  • JNDI支持类，允许
    使用JNDI定位的对象轻松连接Spring管理的bean

  • 应用程序上下文概念，
    提供资源加载和消息访问抽象

  • 通用交易管理
    可插拔策略，支持声明式和程序化
    划界

  • 支持源级元数据，
    将Commons Attributes作为默认实现（例如，用于交易）
    属性）

  • 通用的DAO支持，提供了
    通用数据访问异常层次结构，可用于任何数据访问
    战略

  • 简化的JDBC抽象
    资源和错误处理，还包括BLOB / CLOB支持

  • 休眠支持，提供
    SessionFactory管理和事务范围的ThreadLocal会话

  • JDO 1.0和
    iBATIS SQL Maps 1.3 / 2.0，与Spring的事务集成
    管理

  • 邮件发件人抽象，带有
    对JavaMail的特殊支持，包括文件的便捷处理
    附件

  • 计划支持Quartz和
    计时器，可轻松调用Spring托管Bean的方法

  • 远程支持RMI，JAX-RPC
    和Caucho的粗麻布/粗麻布，可轻松暴露 Spring 管理的 Bean type

  • 便利班
    并实现本地和远程EJB

  • Web应用程序上下文，用于
    在Web环境中加载Spring应用程序上下文

  • 内置的灵活的Web MVC框架
    在策略界面上并与各种视图技术集成


的独特优势
Spring 是申请的能力
陈述式
使用JTA或本地交易策略向任何POJO进行交易：
这允许在任何情况下都具有轻量级的事务业务对象
某种环境，例如在普通Tomcat上运行的Web应用程序中。
Spring的事务管理还能够管理关联
像Hibernate Sessions这样的资源，避免了自定义的负担
ThreadLocal会话。


建立在
资源管理基础架构
Spring的HibernateTemplate大大简化了实现
基于Hibernate的DAO，将典型的数据访问操作减少到
单一陈述。JDBC的便利程度相似
以Spring的JdbcTemplate的形式提供，对于iBATIS SQL Maps 1.3 / 2.0
分别以SqlMapTemplate和SqlMapClientTemplate的形式。


一个重要的
Spring 的特点是
它的功能可以单独使用，而无需采用
完全基于Spring的体系结构。此外，
Spring管理的中间层及其提供的所有功能可以是
在任何类型的环境中都可以重用，无论是J2EE Web应用程序还是
Spring Web MVC，Struts，WebWork或Tapestry Web层或独立
带有Swing用户界面的应用程序。

用途


Spring发行版附带了许多示例应用程序。的

    意见

3。升级中


从Spring 1.0里程碑或候选版本升级的用户，
请查看变更日志 ;
有
在细节上做了很多改进。以来的变化
1.0 RC2是：

  • AOP支持升级到AOP
    联盟1.0

  • 更复杂的处理
    BeanWrapperImpl中的索引和映射属性

  • 新的ResourceLoader界面，
    由ApplicationContext接口扩展

  • ReloadableResourceBundleMessage
    支持可配置的字符编码

  • MimeMessageHelper支持
    可配置的字符编码

  • JdbcTemplate具有新的泛型
    “执行”方法和改进的“查询”方法

  • iBATIS SQL Maps 2.0支持
    升级到SQL Maps 2.0 RC1

  • 增加了对FreeMarker 2.3的支持


请注意以下有关AOP的升级问题
支持：

  • 你必须更新你的
    aopalliance.jar

  • AdvisorAutoProxyCreator已重命名
    到DefaultAdvisorAutoProxyCreator

  • TransactionAttributeSourceTransactionAroundAdvisor
    被重命名为
    TransactionAttributeSourceAdvisor

  • 自定义Advisor实施：
    getAdvice（）现在返回org.aopalliance.aop。建议而不是对象

  • 如果您实施
    org.springframework.aop。MethodAfterReturningAdvice，替换为
    AfterReturningAdvice（方法签名不变）
