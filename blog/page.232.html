Spring Framework 4.0和Java泛型

工程
菲尔·韦伯
2013年12月3日

随着Spring Framework 4.0的最新RC2版本的发布 ；以及将于今年年底之前发布的GA，以下是一些预告片，如果您使用Java泛型类型，这些更改应该可以改善您的生活。

Spring已经有相当不错的Java泛型支持一段时间了。例如，在3.2版中，您可以轻松地将特定类型的所有bean注入泛型List只需使用@Autowired注解：

@Autowired
private List<MyType> beans; 
// all beans that extends MyType will be injected

Spring的转换服务，绑定系统和Web MVC框架都是“通用的”，并且也很方便GenericCollectionTypeResolver和GenericTypeResolver您可以在自己的代码中使用的实用程序类。

Spring Integration 3.0 Release Candidate 1现在可用

发布
加里·罗素
2013年11月27日

我们非常高兴地宣布Spring Integration 3.0.0。RC1现在在Spring Milestone Repo中可用

该版本默认使用Spring Framework 3.2.5，但也与Spring 3.1.x和4.0.x兼容。

此处显示了自里程碑3以来的更改的发行说明。

首先，关于Spring Framework 4支持的说明

Spring Framework 4.0有了新的spring-messaging包含许多来自Spring Integration的关键抽象的模块。

Spring Integration 3尽管与Spring Framework 4兼容，但并不依赖于它，并且不使用这些类，因此它也可以与Spring Framework 3.2和3.1一起使用。

我们计划在2014年初发布Spring Integration 4，它将与Spring Framework 4完全兼容（并要求与之兼容）。 spring-messaging模块为基础。Spring Framework 4发布后不久将提供一个里程碑。
