Spring Web Flow 1.0 RC3发布

发布
基思·唐纳德
2006年6月26日

 

亲爱的春季社区，

我们很高兴地宣布， Spring Web Flow （SWF）1.0 RC3（候选版本3）已经发布。下载 | 说明文件 | 变更日志

此稳定版本包含错误修复和较小的改进。目前，我们希望这是即将到来的Spring Web Flow 1.0 final之前的最后一个候选版本。值得注意的包括……

值得改进

改进的流异常层次结构。  总体而言，异常层次结构现在可以更清楚地组织从流定义访问（FlowLocatorException）到运行时流执行（FlowExecutionException）和执行持久性（FlowExecutionRepositoryException）的故障类别。

 

改进了在Portlet环境中使用BACK按钮的支持。  结合基于连续性的存储库，SWF支持在Portlet环境中充分使用浏览器导航按钮（后退，刷新），同时使流程保持活动状态。当流执行终止时，浏览器刷新将触发自动启动新的执行。

更简单的默认JSF视图映射。现在，默认情况下，SWF视图名称应与JSF视图ID完全对应。您将在Sellitem-jsf示例中看到此默认设置-请注意如何像标准JSF导航处理程序一样引用视图，例如“ /priceAndItemCountForm.jsp”。这自然支持其他JSF视图技术（例如Facelets），并且对于JSF开发人员来说是更自然的默认选择。如果需要自定义映射SWF到JSF视图映射，请将自定义ViewIdMapper插入FlowPhaseListener。

简化示例应用程序 。我们简化了文件上传，flowlauncher，birthdate和sellitem示例，以利用Spring Web Flow的最新功能和最佳实践。

潜在的用户影响变化

有了1.0 RC3，在通往1.0最终版的道路上可能会有一些可能影响用户的更改。以下部分将对它们进行注释：

为简单起见，删除了FormAction属性“ validateUsingValidatorMethod” 。经验表明，此属性使新用户感到困惑。作为更好的替代方法，无需验证即可执行数据绑定操作，只需从流定义中调用“绑定”操作方法即可。当您需要零碎的验证器验证时，只需将“ validatorMethod”属性设置为特定验证器方法来调用“ bindAndValidate”或“ validate”。有关示例，请参见sellitem。

StateExceptionHandler被重命名为“ FlowExecutionExceptionHandler”。 这会影响自定义处理程序的实现。要升级，只需实现新接口即可；签名在逻辑上是相同的。

FlowExecutorImpl的“ redirectOnPause”属性现在接受布尔值，而不是RedirectType枚举。  从1.0 RC2开始，SWF中只有一种重定向类型-流执行重定向。每次通过进入视图状态而使流程执行暂停时，将此标志设置为“ true”都会触发该标志。有关示例，请参见Numberguess或sellitem。
——–

开始使用Spring Web Flow的最佳方法之一是查看并演练示例应用程序。我们建议您检查所有样品，并从一开始就根据需要补充参考手册材料。1.0 RC3版本附带了十个示例应用程序，每个示例程序都展示了一套独特的产品功能。这些样本是：

1。电话簿-演示大多数功能（包括子流）的原始样本
2。Sellitem-演示具有条件转换，流执行重定向，对话范围和延续的向导
3。Flowlauncher-演示启动和恢复流程的所有可能方法
4。项目清单-演示REST风格的URL和内联流
5，Shippingrate-与Ajax技术一起展示Spring Web Flow（感谢Steven Devijver）
6。NumberGuess-演示有状态Bean和“单键”流执行重定向。
7。生日-演示Struts集成
8。Fileupload-演示分段文件上传
9。电话簿-Portlet-Portlet环境中的电话簿样本
（注意流定义如何不变）
10。Sellitem-JSF-JSF环境中的Sellitem示例
