Spring Web Flow 1.0 RC2发布

发布
基思·唐纳德
2006年6月10日


 

亲爱的春季社区，

我们很高兴地宣布，已经发布了Spring Web Flow （SWF）1.0 RC2（候选发布版2）。下载 | 说明文件 | 变更日志

此版本包含错误修复和较小的改进。新的和值得注意的1.0 RC2包括…

新的和值得

支持以可配置的方式传递来自其调用环境的新启动的流执行输入。  默认情况下，所有请求参数都作为输入公开。然后，流程可以选择使用其输入映射器将此输入映射到其自己的本地范围。该映射器为流定义了输入契约，该契约是一致的，而不管该流是以顶级流还是子流启动。

考虑以下请求URL作为示例：

http：// localhost：8080 / flights / search-flow？flightNumber = 12345

默认情况下，访问此URL时，后备FlowExecutor会将“ flightNumber”请求参数放入“输入映射”中。然后，将输入映射传递给“搜索流”的新执行。

要自定义流执行输入映射填充，例如，从请求路径或某些其他外部源中提取属性，请配置“ FlowExecutorImpl.inputMapper”属性。

在JSR168 Portlet环境中支持流执行和外部重定向。  结合基于延续的存储库，可以在Portlet环境中使用浏览器导航按钮（后退，刷新）。同样在Portlet环境中，我们现在公开了一个“ globalSessionMap”属性，用于访问Portlet会话APPLICATION_SCOPE中的属性。

一个名为“ SingleKeyFlowExecutionRepositoryFactory”的新存储库工厂。 该实现为每个持久流执行生成单个唯一标识符。这对于实现1.0 EA“会话重定向”语义非常有用-在每次POST之后，REDIRECT-GET都会命中稳定的“流执行URL”，该URL嵌入了恒定的流执行密钥。有关说明，请参见NumberGuess示例。

引入独立的“对话”子系统 ，提供的流执行存储库实现将委派给该子系统 ，以划分管理流执行状态的逻辑对话。该对话子系统与Spring Web的其余部分完全脱钩
Flow在SWF之外可用，并且随着时间的推移可能会演变成它自己的独立模块。 中央服务接口包括：

当启动新的流程执行并且需要将其保留到一个请求之外时，存储库将调用“ beginConversation”以启动新的逻辑
对话并在对话范围内放置属性以跟踪执行状态。同样，当流程执行结束时，主导会话也将结束，并且所有分配的状态都将被清除。

将来，我们希望在此系统中提供强大的功能，包括通过JMX进行对话监视和管理以及对话历史记录和统计信息。我们还希望证明其适用于Spring Web Flow以外的其他环境。特别感谢Juergen Hoeller和Ben Hale在设计此便携式对话服务抽象时所提供的帮助。


潜在的用户影响变化

有了1.0 RC2，通往1.0最终版的过程中会有一些可能会影响用户的更改。以下部分将对它们进行注释：

在spring-webflow-dtd中， 我们重命名了“ resultName”和“ resultScope” 元素属性分别为'result-name'和'result-scope' ，以与其他属性和元素名称保持一致。

为简单起见，删除了FormAction属性“ bindOnSetupForm”和“ validateOnBinding” 。经验表明，这些属性很少使用，并且已经使新用户感到困惑。作为更好的选择，要在进入视图状态之前执行数据绑定操作，只需从流定义中调用“绑定”操作方法即可。要计算是否应为bindAndValidate尝试进行验证，请覆盖单个“ validationEnabled（RequestContext）”钩子。

FormAction“ exposeFormObject”操作方法已删除。  只需使用首选的“ setupForm”即可。

FlowExecutionRepository和FlowExecutor SPI接口得到了简化。  现在，更多逻辑被封装在FlowExecutionRepository后面，包括生成的FlowExecutionKeys的结构和格式。此外，FlowExecutionRepository现在严格负责管理持久流执行，而仅此而已。SWF核心不再知道“对话”的其他概念。这意味着几件事：

  • 整个存储库界面更简单，从而使使用自定义FlowExecutionKeys创建自定义FlowExecutionRepositories更加容易。
  • SWF核心词典更强大：流执行程序调用流执行以执行流。超出一个请求而保持活动状态的执行将持久保存到存储库中。
  • 默认存储库实现选择委派给不同的“会话子系统”以跟踪由执行系统驱动的会话状态，但是对此系统的依赖关系已完全封装，并且是可选的。

FlowExecutor接口（SWF的入口点）也已简化为调用方。现在，它封装了复杂内部类型的知识，例如
作为EventIds和FlowExecutionKeys，因此总体上更易于使用。

同样， 删除了对显式“ conversationRedirect”的支持 。这表示删除了“ conversationRedirect：”
“视图”前缀和“ CONVERSATION” RedirectType。要在1.0 RC2中实现相同的逻辑重定向语义，只需将FlowExecutor配置为redirectOnPause类型为FLOW_EXECUTION且将它的repositoryFactory设置为
SingleKeyFlowExecutionRepositoryFactory。

Spring Web Flow 1.0 RC2进一步完善了参考手册，提供了50页有关SWF使用的信息。该手册以HTML和PDF格式在线提供。

开始使用Spring Web Flow的最佳方法之一是查看并演练示例应用程序。我们建议您检查所有样品，并从一开始就根据需要补充参考手册材料。1.0 RC2发行版附带了十个示例应用程序，每个示例都展示了一组独特的产品功能。这些样本是：

1。电话簿-演示大多数功能（包括子流）的原始样本
2。Sellitem-演示具有条件转换，流执行重定向，对话范围和延续的向导
3。Flowlauncher-演示启动和恢复流程的所有可能方法
4。项目清单-演示REST风格的URL和内联流
5，Shippingrate-与Ajax一起展示Spring Web Flow
技术（感谢Steven Devijver）
6。NumberGuess-演示有状态的bean和“单键”流程
执行重定向。
7。生日-演示Struts集成
8。Fileupload-演示分段文件上传
9。电话簿-Portlet-Portlet环境中的电话簿样本
（注意流定义如何不变）
10。Sellitem-JSF-JSF环境中的Sellitem示例

要一步构建用于部署的示例应用程序，只需提取发行档案，访问projects / spring-webflow / build-spring-webflow目录并执行“ ant dist”目标。请参阅发行版readme.txt和
请参阅projects / spring-webflow / spring-webflow-samples / readme.txt，以分别获取有关发行档案内容和样本的更多信息。
所有示例项目都是可直接导入Eclipse的Spring IDE项目。
