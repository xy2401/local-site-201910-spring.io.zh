Spring Web Flow 1.0 RC1发布

发布
基思·唐纳德
2006年5月3日

亲爱的春季社区，

我们非常高兴地宣布，已经发布了Spring Web Flow（SWF）1.0 RC1（候选发布版本1）。下载它 。

经过一年多的艰苦开发工作，500个论坛用户的 29,000次下载，3,700个帖子， 一本书以及许多 社区 驱动的 文章 ，此发行版提供了Spring Web Flow的第一个1.0版本候选。考虑到适合生产使用，此版本巩固了1.0 API，它将在整个1.x系列中保持向后兼容。

对于支持我们产品的Spring Web Flow的早期采用者和拥护者，谢谢。由于您的反馈和支持，Spring Web Flow提供了当今可用的最具创新性和功能最强大的控制器引擎之一。

Spring Web Flow团队希望在1.0最终版本之前再发布一个候选版本。1.0 RC1中的新功能和值得注意的功能包括…

新的和值得

改进了对管理有状态业务组件的支持。  Spring Web Flow现在采用几种技术来管理有状态的中间层组件的实例，这些实例在任务执行过程中执行业务逻辑。在大多数情况下，状态管理是完全透明的 。您只需将应用程序状态存储在实例变量中，Spring Web Flow就会以线程安全的方式在对话中处理该状态的作用域。

简单的NumberGuess示例应用程序说明了这种支持，其中由流程管理的“游戏”组件执行游戏业务逻辑的执行。组件本身对Spring Web Flow API的依赖性为零。

HigherLowerGame实现也与Spring和Spring Web Flow API完全分离。

对流变量的增强支持，在流启动时自动创建。 流变量值甚至可以从支持bean工厂获得，这得益于那里的完全依赖注入。变量的确切范围是可配置的。

一种新的流程执行重定向响应类型，用于重定向到唯一的“流程执行URL”。  访问流执行URL将在先前输入的ViewState处刷新Flow，从而允许从那里继续。对话处于活动状态时，URL保持有效，并且继续点保持有效。这样可以完全使用后退，下一步，刷新和新窗口按钮，而无需页面缓存。

优化了状态异常处理，并为过渡执行状态异常处理程序提供了方便的支持。 现在，核心过渡元素支持on-exception属性，该属性会在发生异常时驱动向新状态的过渡。例如：

 

…如果placeOrder方法引发AccountException，则将流程转换为editAccount状态。

流属性映射支持的改进。 现在可以为每个流配置一个输入映射器，以映射由启动该流的调用者提供的输入。流还可以配置有输出映射器，以将返回值公开给终止流的调用方。这些增强功能使流可以作为顶级流和子流重用，而无需更改，因为两种情况下输入和输出属性都一致地映射。

支持动态视图名称和目标状态表达式 ，从而可以方便地分别基于运行时对ViewState的逻辑视图名称和转换的目标状态进行计算。

增强的JSF集成。  JSF集成现在支持逻辑重定向响应类型，包括FlowExecutionRedirect，ConversationRedirect，FlowRedirect和ExternalRedirect。此外，对FlowPhaseListener的增强允许启动和刷新流执行，而无需执行正式的导航步骤。预计1.0 RC2将在Portlet环境中进一步为JSF提供JSF便利性和官方支持。

其他资源和从哪里开始

Spring Web Flow 1.0 RC1进一步完善了参考手册，提供了有关SWF使用情况的50个重点页面。该手册以HTML和PDF格式在线提供。

开始使用Spring Web Flow的最佳方法之一是查看并演练示例应用程序。我们建议您检查所有样品，并从一开始就根据需要补充参考手册材料。九（9）个示例应用程序附带了1.0 RC1版本，每个示例程序都展示了一组独特的产品功能。这些样本是：

1。电话簿 -演示大多数功能（包括子流）的原始示例。
2。Sellitem-演示具有条件转换，流执行重定向，对话范围和继续的向导。
3。Flowlauncher-演示启动和恢复流程的所有可能方法。
4。 项目列表 -演示REST风格的URL，会话重定向到可刷新的会话URL以及内联流。
5，Shippingrate-与Ajax技术一起展示Spring Web Flow（感谢Steven Devijver）
5，NumberGuess-演示状态豆
6。生日 -演示Struts集成。
7。Fileupload-演示分段文件上传。
8。Phonebook-Portlet -Portlet环境中的电话簿示例（请注意流定义如何保持不变）
9。Sellitem-JSF -JSF环境中的Sellitem示例（注意流程定义如何更改）

要一步构建用于部署的示例应用程序，只需提取发行档案，访问projects / build-spring-webflow目录并执行ant dist目标。请参阅发行版readme.txt和projects / spring-webflow-samples / readme.txt分别获得有关发行档案内容和示例的更多信息。现在，所有示例项目都是可直接导入Eclipse的Spring IDE项目。  请注意Spring IDE团队即将推出的Spring IDE图形Web流程编辑器。
