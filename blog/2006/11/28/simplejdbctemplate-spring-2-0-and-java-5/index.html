SimpleJdbcTemplate：Spring 2.0和Java 5

工程
本·黑尔
2006年11月28日

在进行“春季体验”之前，我一直很忙，但是我注意到Rod在博客方面非常活跃。所以在今天的空余时间里，我决定在机场和飞机上做一些博客工作。

Spring社区中最大的平衡行为之一是确保在创新的同时保持向后兼容。该创新的一部分是利用Java的更高版本（例如Java 5）中的新功能和构造。从1.2.x分支开始，我们已经通过@Transactional批注和基于@ManagedResource批注的JMX自动检测看到了其中的一些内容。最后，这些都是很棒的功能，并且大大简化了开发（至少无论如何是我的），但是它们实际上相当于将元数据移入代码中。我们没有看到的是API的实际简化。

在Spring 2.0中，情况发生了变化。我们看到某些API不仅使用注释，还利用了其他功能。实际上，有一个很好的示例，其中我们看到了几乎所有使用的Java 5的新语言功能（自动装箱，可变参数，泛型），即SimpleJdbcTemplate 。SimpleJdbcTemplate并不是标准JdbcTemplate的替代品，而是使用Java 5来简化某些常见任务。

自动装箱

严格来说，自动装箱不是我们对API所做的事情，但是可以使使用JdbcTemplate更好。因此，例如，在过去您必须将原始值装箱的地方……


public int getLargeAccountCount(double value, int type) {
    return getSimpleJdbcTemplate().queryForInt(
        "select count(*) from accounts where balance > ? and type = ?",
        new Object[] { new Double(value), new Integer(type) });
}

……您不再需要。


public int getLargeAccountCount(double value, int type) {
    return getSimpleJdbcTemplate().queryForInt(
        "select count(*) from accounts where balance > ? and type = ?",
        new Object[] { value, type });
}

在这个简单的示例中，它并没有太大的区别，但是您可以想象，在具有多个绑定变量的复杂SQL查询中，装箱会占用相当多的空间。

瓦拉格斯

如果我们看一下原始示例，就会看到Java 5可以帮助我们删除的另一段样板代码。绑定参数作为对象数组传递，但是Java 5将允许我们仅指定一个以逗号分隔的对象列表，并将其转换回所需的对象数组。

因此，如果我们从原始示例开始...


public int getLargeAccountCount(double value, int type) {
    return getSimpleJdbcTemplate().queryForInt(
        "select count(*) from accounts where balance > ? and type = ?",
        new Object[] { new Double(value), new Integer(type) });
}

…并结合了自动装箱和可变参数，事情开始变得越来越短。


public int getLargeAccountCount(double value, int type) {
    return getSimpleJdbcTemplate().queryForInt(
        "select count(*) from accounts where balance > ? and type = ?",
        value, type);
}

泛型

我们在SimpleJdbcTemplate中看到的最后一个改进是引入了泛型。使用JdbcTemplate时，对我来说最大的痛苦之一是，每当我执行queryForObject（）时，我都必须进行手动转换，而得不到我的IDE或编译器的很多帮助。


public Account getAccount(long id) {
    return (Account) getJdbcTemplate().queryForObject(
        "select * from accounts where id = ?",
        new Object[] { new Long(id) }, new RowMapper() {

            public Object mapRow(ResultSet rs, int rowNum)
                    throws SQLException {
                String accountNumber = rs.getString("account_number");
                int balance = rs.getInt("balance");
                return new Account(accountNumber, balance);
            }
        });
}

在Spring 2.0中，我们发布了SimpleJdbcTemplate的配套产品ParameterizedRowMapper 。当两者一起使用时，它实际上创建了一个非常漂亮的小系统，它根本不需要强制转换，并且让您的IDE和编译器进行强类型检查。


public Account getAccount(long id) {
    return getSimpleJdbcTemplate().queryForObject(
        "select * from accounts where id = ?",
        new ParameterizedRowMapper<Account>() {

            public Account mapRow(ResultSet rs, int rowNum)
                    throws SQLException {
                String accountNumber = rs.getString("account_number");
                int balance = rs.getInt("balance");
                return new Account(accountNumber, balance);
            }
        }, id);
}

需要记住的一件事是， SimpleJdbcTemplate没有JdbcTemplate拥有的所有方法。它甚至没有扩展JdbcTemplate ，而是可以提供对JdbcTemplate的引用。SimpleJdbcTemplate的目标是在利用Java 5的同时简化一些常见行为的使用。

最后，这并不是革命性的。就像Rod的早期文章一样 ，这只是语法糖。但这是Spring拥抱Java 5及更高版本中的新功能的一个示例。
