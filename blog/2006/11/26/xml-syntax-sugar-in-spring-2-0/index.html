Spring 2.0中的XML语法糖

工程
罗德·约翰逊
2006年11月26日

如果您遵循10月的Spring 2.0版本，您将知道XML扩展名称空间是其中的一大新功能：定义生成XML元数据的新XML元素和属性的功能，并且可以与常规bean定义一起使用。这提供了一个有价值的新扩展点，并使Spring配置更易于执行许多重复任务，并且功能更强大。

但是，您可能还没有注意到一点点甜蜜的语法糖-可能是因为Spring团队中没有人告诉您……已经答应了一段时间，我将以代码为中心接下来是博客。

转向XML模式以允许扩展名称空间也允许使用属性而不是子元素的一些捷径。这些属性没有经过验证，但是因为我们使用的是XML模式而不是DTD，所以我们仍然可以保留所有其他验证。由于属性名称是属性名称，因此XML验证不会添加任何内容。这是基于Java而不是XML结构的验证问题。
想象下面的Java对象，它具有两个简单的属性以及对关联对象的依赖关系：


public class Person {
	
   private int age;
	
   private String name;
	
   private House house;

   public void setAge(int age) {
      this.age = age;
   }

   public void setName(String name) {
      this.name = name;
   }
	
   public void setHouse(House house) {
      this.house = house;
   }
}

可以在Spring 2.0中使用XML进行如下配置：


<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
   xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
   xmlns:p="http://www.springframework.org/schema/p"
   xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd">

<bean class="com.interface21.spring2.ioc.Person"
   p:name="Tony"
   p:age="53"
   p:house-ref="number10"
/>
	
<bean class="com.interface21.spring2.ioc.House"
   id="number10"
   p:name="10 Downing Street"
/>
	
</beans>

注意如何使用属性而不是元素来提供属性。这可以通过特殊命名空间“ p”的魔力实现。这是一个未经验证的特殊名称空间，但旨在允许使用名称与Java属性名称匹配的属性。

对于简单类型，我们只需在“ p”名称空间中使用属性名称，就像在“ p：name”中一样。在注入对其他Spring bean的引用时，请使用“ -ref”后缀，如“ p：house-ref”中所示。

当您要使用自动装配时，此快捷语法特别引人注目。例如，考虑以下变体：


<bean class="com.interface21.spring2.ioc.Person"
   autowire="byType"
   p:name="Tony"
   p:age="53"
/>

这里我们没有设置“ house”属性，因为自动装配可以解决这个问题。您甚至可以在元素级别使用default-autowire 来对整个文件进行自动装配。

以下是Spring 1.0或1.1用法的摘录，说明了最近两个主要发行版（1.2和2.0）中，Spring的配置已减少了尖括号的最小数量：


<bean class="com.interface21.spring2.ioc.Person">
   <property name="name"><value>"Tony"</value></property>
   <property name="age"><value>"53"</value></property>		
   <property name="house"><ref local=&rdquo;number10&rdquo; /></property>
</bean>

在Spring 1.2中，我们引入了“值”和“ ref”属性，而不是在大多数情况下都需要子元素，而在Spring 2.0中，现在可以使用纯属性和简单属性。

当然，传统语法仍然有效-Spring 2.0只是添加了其他选项。当属性值复杂且不合法或不可读为属性值时，请使用传统的手抄写形式。而且，当然，不需要重写现有的配置文件，因为Spring 2.0总体上完全向后兼容。

将配置示例放入演示文稿时，新语法特别好。这使我想起了，我今天的真正工作是整理我的Spring Experience幻灯片...希望在那里见到你！
