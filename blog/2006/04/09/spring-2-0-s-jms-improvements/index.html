Spring 2.0的JMS改进

工程
本·黑尔
2006年4月9日

随着Spring 1.1的发布，Spring社区获得了对JMS支持的首次体验。这种支持包括异常转换，消息转换和类似于JdbcTemplate的模板类。这种支持还解决了JMS 1.0.2和1.1规范之间的域统一问题。该支持的核心是JmsTemplate类，它是JMS 1.0.2的对应JmsTemplate102 。

与使用原始JMS API进行企业消息传递相比，此支持有了很大的改进。但是它确实有一个缺点。 JmsTemplate仅支持使用JmsTemplate.receive（）方法同步接收消息。这种行为对许多人来说效果很好，但是最终的绝大多数用户最终推出了自己的异步使用者实现。简而言之，他们想要EJB 2所谓的消息驱动Bean 。

但是用户将不再没有它。随着2.0M1版本和更高版本的最终2.0版本的发布，已经添加了对JMS消息的异步接收的本机支持。JmsTemplate仍然一直用于发送消息，但现在它已与AbstractMessageListenerContainer的子类（例如DefaultMessageListenerContainer ， SimpleMessageListenerContainer和ServerSessionMessageListener）结合在一起 。

让我们看一下如何使用这些MessageListenerContainer 。第一步是创建一个可以接收消息的类。为此，必须创建一个实现MessageListener接口的类。


package jmsexample;

import javax.jms.JMSException;
import javax.jms.Message;
import javax.jms.MessageListener;
import javax.jms.TextMessage;

public class ExampleListener implements MessageListener {

	public void onMessage(Message message) {
		if (message instanceof TextMessage) {
			try {
				System.out.println(((TextMessage)message).getText());
			} catch (JMSException e) {
				throw new RuntimeException(e);
			}
		} else {
			throw new IllegalArgumentException(
					"Message must be of type TestMessage");
		}
	}

}

一旦有了这些，就需要一个消息生成器。该代码与Spring 2.0之前的代码相同，因此，如果您已经有执行此操作的代码，则不需要进行任何更改。


package jmsexample;

import org.springframework.jms.core.JmsTemplate;

public class ExampleProducer {

	private JmsTemplate jmsTemplate;

	public ExampleProducer(JmsTemplate jmsTemplate) {
		this.jmsTemplate = jmsTemplate;
	}

	public void sendMessage() {
		jmsTemplate.convertAndSend("Example Message");
	}

}

接下来，您需要配置上下文以创建一个MessageListenerContainer ，该消息将消息路由到该bean。您会注意到，在此示例中，我正在使用ActiveMQ实现类。这恰好是许多JMS实现之一，并且恰好是我最熟悉的实现。

我现在暂时跳过它，但是显然您需要启动一个MQ，并且需要一个引导您的上下文的main方法。我已经从该示例中添加了项目的存档，以便您可以在需要时看到其余的代码。

最后，您只需要运行您的应用程序并查看输出。

Example Message

需要注意的一件事是，到目前为止，我们一直在处理单个使用者线程的异步接收。使用MessageListenerContainer的并发消费者属性，可以对使用者使用多线程（请记住，您仍然必须使它们成为无状态或线程安全的）。

从我自己的痛苦经历中，我想指出的一件事是，确保不要将并发使用者与Topic一起使用。请记住，在JMS主题中，所有消息都会传递给该主题的所有使用者。这意味着，如果您同时有某个主题的使用者，那么他们所有人都会收到相同的消息；通常您要避免的事情。但是，如果您使用队列，则显然这将以循环方式将每个新消息分发给使用者。

所以你有它。它不是很浮华，可能与您在某些时候写的东西非常相似，但是现在您所要做的就是使用它，而不必维护它。我还要说，这只是冰山一角。MessageListenerContainer具有参与事务，使用自定义线程池（如应用服务器随附的线程池 ）和新的Spring TaskExecutor抽象功能的功能，甚至可以将本机JMS会话公开给使用者。但是，所有这些都是每个帖子的主题。
