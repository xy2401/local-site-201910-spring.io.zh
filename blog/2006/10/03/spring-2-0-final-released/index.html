Spring 2.0 Final发布

发布
罗德·约翰逊
2006年10月3日

我们很高兴地宣布，人们期待已久的Spring Framework 2.0版最终版本现已发布。

下载 | 说明文件 | 变更日志 

作为领先的全栈Java / Java EE应用程序框架，Spring为许多项目带来了显着的收益，在降低开发工作量和成本的同时，还提高了测试覆盖率和质量。

经过9个月的积极开发，该稳定的生产级版本才发布。在此短时间内，Spring 2.x系列已经非常成熟，得益于9个里程碑发行版中的150,000多个早期访问下载，解决了750多个JIRA问题，其中有50个引入了主要的新功能。

什么是新的？

我们相信，三个属性可以反映出用户在Spring 2.0系列中所期望的： 简单 ， 强大和可靠 。


2.0版为框架的整体使用模型带来了新的重大简化。正如我们现有的用户所知道的那样，Spring的核心是Bean容器 ， 该 容器 驱动 Java和Java EE应用程序的配置 。在2.0版中，通过引入自定义Bean配置方言简化了许多常见的配置任务。这对您意味着什么？

这意味着您现在可以：

  • 使您的业务服务在一行配置代码中具有事务性。
  • 在JNDI中以一行配置代码查找对象。
  • 通过一行配置代码将外部化的属性公开给您的服务。
  • 使用单个注释将一致的异常处理策略应用于数据访问对象。
  • 通过定义单个配置标签来调用无状态远程EJB。没有更多的自定义服务定位器或业务代表。
在框架的各个模块中继续进行简化，从而使您能够：
  • 用一行代码编写参数化的JDBC查询。
  • 部署Spring MVC控制器时， 在配置上应用约定 。
  • 使用Spring JSP标记来减少开发输入表单时的键入。


Spring 2.0的主要目标是使常见任务变得更容易。2.0版还打开了激动人心的新门，以优雅的方式解决难题。在2.0中，您可以：

  • 使用AspectJ的简洁切入点表达语言将自定义行为编织到程序执行的多个点。
  • 接收具有事务和线程安全保证的异步JMS消息。现场观看
  • 为您的应用程序开发自己的Bean配置方言 。
  • 以线程安全的方式从自定义范围（如“请求”和“会话”范围）注入对象。
  • 从Java应用程序中调用Groovy，Beanshell和JRuby脚本。
  • 通过复杂的线程和池选项安排任务以异步运行。

2.0版基于Spring 1.x设置的基础。此新版本提供了主要的新功能，同时尽可能保留了向后兼容性。

自2004年3月发布以来，Spring 1.x的下载量已超过一百万 ，使事实上的普通标准Java对象（POJO）开发复杂的应用程序成为可能。2.x系列基于此公认的最佳实践，可提供新的简化功能和强大功能，同时保持与已建立的Spring 1.x系列的完全兼容性。用户可以期望他们的升级非常简单；在大多数情况下，只需用Spring 2.0中包含的文件替换1.2.8 JAR文件即可。

享受，谢谢

Spring 2.0代表了过去一年中许多人的累积努力。从Interface21的主要开发人员Juergen，Rob，Rick和Costin到我们的支持合作伙伴BEA和Oracle，再到社区中许多贡献创新，补丁，文档，错误报告和测试的人-充满了很多血汗和这里的眼泪。我们衷心希望您能发现这个新版本给我们带来极大的乐趣。尽情享受，放心：工作不止于此。

真诚的

The Spring Team
