Interface21 / Spring在Oracle OpenWorld 2006上

工程
内兰·乔克西（Neelan Choksi）
2006年10月23日

罗德和我在旧金山举行的Oracle OpenWorld 2006上在这里。这确实是一个场景。我无法确定“哈伯”或“令人印象深刻”是否是描述它的正确方法，但数字并非谎言。他们预计将有45,000人参加会议，旧金山市认为该展会将为该市带来50-60M美元的收入。显然，不允许任何Oracle员工住在机场北部的酒店。更令人震惊的事情之一是，这座城市在会议的幕后封锁了莫斯康尼前面的霍华德街。

在今天上午， Oracle 高级副总裁Thomas Kurian所作的Oracle Develop主题演讲中，Spring是当务之急 。Thomas在主题演讲中邀请Rod上台，并介绍Spring是什么以及Spring 2.0的新功能。Rod还谈到了Interface21与Oracle之间的合作，重点介绍了：


  • Spring 2.0随Toplink Essentials一起发布，这是Oracle的JPA参考实现，以及Interface21如何与Oracle的Mike 3 Keith合作，EJB 3共同规范使这一目标得以实现。

  • JDeveloper中的许多功能可帮助创作Spring应用程序

  • Spring和Oracle应用服务器之间的许多集成（对于那些想知道的人，尚未宣布对应用服务器的Java EE 5支持的路线图）

  • Oracle积极参与的Spring OSGI主导项目

  • 服务组件体系结构（SCA） -SCA是Oracle SOA解决方案使用Spring的重要组成部分

甲骨文在星期四发布了他们的SOA套件的最新版本。它基于Spring。

Thomas还在12:45强调了Rod的主题演讲-“ Spring 2.0的新功能和新功能”。

另外，Oracle今天宣布了Oracle Developer Depot的开发 。除了Oracle技术网之外 ，它还有一些Spring示例，并且看起来很容易在您的计算机上运行示例。它负责安装模式，数据源等所有工作。ODD / OTN具有RSS提要-因此，您可以通过新闻阅读器轻松查看新功能。

Oracle Developer Depot实际上是在Spring上构建的，这很酷。

因此，总而言之，在Oracle OpenWorld展会上，Interface21和Spring显然是这里的重点。
