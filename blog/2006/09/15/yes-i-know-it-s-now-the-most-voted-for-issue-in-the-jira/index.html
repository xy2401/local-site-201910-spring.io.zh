是的，我知道它是JIRA中投票最多的问题！

工程
本·黑尔
2006年9月15日

你能猜出这是什么？如果您猜到了Spring的Maven捆绑包/版本 ，您将获胜。快来12月在“ Spring 体验”中见我，我们将分享一份冰镇饮料作为您的奖励。

米亚·库尔帕（Mea Culpa）


前世，我在配置管理和构建系统方面做了很多工作。当我在Interface21开始工作时，我立即自愿提供了尽可能多的构建基础设施帮助。我知道的下一件事，就是在Spring和Spring Web Flow中分配了与Maven相关的每张票证。然后，由于我对JIRA缺乏冷静的态度，我把这个特殊问题解决了。在过去的几个月里，事情一直在前进与问候 Spring 和Maven;我应该在JIRA中发布的内容，而没有。在过去的几天中，这导致JIRA主要问题中的一些合理的宣泄。因此，为了使每个人都快上一步，这里是当前状态。

状态


对于那些在过去几个月中对Spring 2.0和Maven感到绝望的人，他们的时间不会更长。Spring社区决定将所有Spring项目逐步转换为Maven。您可能知道Acegi已经使用Maven很长时间了。最近， Spring-WS项目也进行了转换。我已经亲自设计了Spring Web Flow的转换原型，并且普遍同意Spring也将继续发展。

也就是说，现在还不是庆祝的时间。转换最后两个项目（Spring和Spring Web Flow）是一项艰巨的任务（如果您不相信我，可以看看Maven的“更好的构建” ）。因此，对于真正的主要2.0和1.0版本来说，转换并不是我们真正想要的。我可以告诉您的是，转换是在发行后预定的目标。

那有什么计划？


实际上，这非常简单。对于Spring和Spring Web Flow，我将在接下来的几周内通过依赖项列表手动构建POM。目前，计划将这些POM与Spring和Spring Web Flow的最终版本一起发布。发布之后，在Arjen的帮助下，我将协助Juergen和Keith将其源代码树转换为Maven版本，并使这些版本在Continuum中运行。

那用户，我同时可以做什么？


首先要有耐心。我一直以来对此问题一直保持沉默，这是我的错，希望这篇文章可以使我们的思考过程更加透明。其次，我很乐意帮助测试。我的计划是在工作时将试用的POM签入CVS和SVN，并在JIRA问题上宣布已进行了更改。您可能会猜到，手动创建POM容易出错（Maven构建的驱动因素之一），所以我很乐意为测试它们提供一些帮助。JIRA中的评论， 论坛中的帖子以及开发人员邮件列表中的帖子都是反馈的好途径。

您还想知道什么吗？


再次，我希望这有助于减轻社区中的一些挫败感，并使您对我们的未来发展方向有所了解。当然，欢迎欢迎对此当前计划发表任何评论（这里的评论将是一个不错的选择），但是我也想知道您是否还有其他问题或让您感到沮丧的问题。如果您在下面留下问题或直接给我发送电子邮件，我会尽力为您提供一个很好的答案，然后将其发布在博客上，或者通过电子邮件将其发送给您。
