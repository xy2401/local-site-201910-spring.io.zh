消息驱动的POJO！

工程
马克·费舍尔
2006年8月11日

在Spring 2.0的所有新功能和改进中，我必须承认消息驱动POJO是我个人的最爱之一。我感觉很多其他Spring用户也会有同样的感觉。

在这里，我提供了一个简短的介绍。还有更多要显示的内容，我将在其他文章中继续介绍。目前，这应该为您提供足够的信息，以使您可以使用一些真正基于POJO的异步JMS并开始运行它！希望您像我一样对此感到兴奋；）

先决条件：

您将在类路径上需要以下JAR文件。我还列出了我正在使用的版本（任何spring-2.x版本都可以。实际上，我只是在大约2分钟前将RC3放到了那里）：

示例域

我在这里故意使事情变得简单-主要目标是演示各个部分如何组合在一起。我要指出的最重要的事情之一是，我的“域”中的这些类是POJO。您将根本看不到任何Spring或JMS依赖项。

最终，我们将接受来自用户的输入（通过stdin输入“名称”），并将其转换为针对某些未指定事件的“注册请求”。该消息将异步发送，但是我们将有另一个队列来处理回复。然后，ReplyNotifier将把确认（或“未确认”消息）写入标准输出。

顺便说一下，我正在“ blog.mdp”包中创建所有这些类。
第一类是RegistrationRequest ：


package blog.mdp;

import java.io.Serializable;

public class RegistrationRequest implements Serializable {

    private static final long serialVersionUID = -6097635701783502292L;

    private String name;
	
    public RegistrationRequest(String name) {
        this.name = name;
    }
	
    public String getName() {
        return name;
    }
}

接下来是RegistrationReply ：


package blog.mdp;

import java.io.Serializable;

public class RegistrationReply implements Serializable {

    private static final long serialVersionUID = -2119692510721245260L;

    private String name;
    private int confirmationId;
	
    public RegistrationReply(String name, int confirmationId) {
        this.name = name;
        this.confirmationId = confirmationId;
    }
	
    public String toString() {
        return (confirmationId >= 0) 
                ? name + ": Confirmed #" + confirmationId 
                : name + ": Not Confirmed";
    }
}

和RegistrationService ：


package blog.mdp;

import java.util.HashMap;
import java.util.Map;

public class RegistrationService {
	
    private Map registrations = new HashMap();
    private int counter = 100;

    public RegistrationReply processRequest(RegistrationRequest request) {
        int id = counter++;
        if (id % 5 == 0) {
            id = -1;
        }
        else {
            registrations.put(new Integer(id), request);
        }
        return new RegistrationReply(request.getName(), id);
    }
}

如您所见，这仅是一个示例。实际上，注册地图可能会做一些事情。另外，您会看到20％的注册尝试都将被拒绝（给定-1 ConfirmationId）-这不是处理注册请求的非常实用的方法，但是它将为答复消息提供一些变化。同样，重要的是该服务类与Spring或JMS没有关系。但是，正如您稍后将看到的那样，它将处理通过JMS发送的消息的有效负载。换句话说，此RegistrationService 是 消息驱动的POJO 。

最后，创建一个简单的类来记录回复消息：


package blog.mdp;

public class ReplyNotifier {

    public void notify(RegistrationReply reply) {
        System.out.println(reply);
    }
}

配置消息驱动的POJO

现在最重要的部分。我们如何使用Spring配置POJO服务，以便它将接收JMS消息？答案以2种bean定义的形式出现（嗯，如果算上服务本身，则为3种）。在下一个bean定义文件中，请注意“容器”，它实际上接收消息并启用异步监听器 。容器需要知道connectionFactory以及从其接收消息的目的地 。有多种类型的容器可用，但这超出了本博客的范围。阅读参考文档以获取更多信息： 消息侦听器容器 。

在这种情况下，“侦听器”是Spring的MessageListenerAdapter的一个实例。它具有对委托的引用（POJO服务）和处理程序方法的名称。在这种情况下，我们还提供了defaultResponseDestination 。对于返回空隙的方法，您显然不需要这样做。另外（可能更可能在生产应用程序中），您可以省略此设置，而改为设置传入的JMS Message的“ reply-to”属性。

配置客户端

在“客户端”端，我们将发送注册请求并记录回复。首先，我将列出bean的定义。在上一节之后，您应该了解“容器”和“侦听器”的作用。在这种情况下， 委托是ReplyNotifier，并且由于其返回类型为void，因此它本身不会发送答复（因此，不存在“ defaultResponseDestination”属性）。我已将此文件命名为“ client-context.xml”：

那里定义了另一个bean-Spring的“ jmsTemplate”实例。我们将使用它来将注册请求消息发送到其defaultDestination 。使用Spring提供的简单的convertAndSend（..）方法，发送JMS消息变得很简单。我创建了一个接受用户输入的类，然后使用此“ jmsTemplate”发送消息：


package blog.mdp;

import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;

import org.springframework.context.ApplicationContext;
import org.springframework.context.support.ClassPathXmlApplicationContext;
import org.springframework.jms.core.JmsTemplate;

public class RegistrationConsole {
	
    public static void main(String[] args) throws IOException {
        ApplicationContext context = new ClassPathXmlApplicationContext("/blog/mdp/client-context.xml");
        JmsTemplate jmsTemplate = (JmsTemplate) context.getBean("jmsTemplate");
		
        BufferedReader reader = new BufferedReader(new InputStreamReader(System.in));		
		
        for (;;) {
            System.out.print("To Register, Enter Name: ");
            String name = reader.readLine();
            RegistrationRequest request = new RegistrationRequest(name);
            jmsTemplate.convertAndSend(request);
        }
    }
}

运行示例

现在是有趣的部分。启动ActiveMQ代理（如“设置环境”一节中简要讨论的）。运行RegistrationServiceRunner的main（..）方法。运行RegistrationConsole的main（..）方法。输入名称，您将在同一控制台中看到答复。
