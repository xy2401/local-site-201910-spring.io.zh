Spring 2.0 Release Candidate 3发布

发布
于尔根·霍勒（Juergen Hoeller）
2006年8月11日

尊敬的Spring社区，

我们很高兴地宣布Spring 2.0 RC3已发布。 下载 | 说明文件 | 变更日志

根据我们收到的针对先前发行候选版本的宝贵用户反馈，此第三发行候选版本包括许多改进。在此版本中，Spring 2.0 final即将到来。

最重要的改进包括：

  • 相对于按类型检测特殊bean（例如PropertyPlaceholderConfigurers）， 已经为default-lazy-init =“ true”恢复了Spring 1.2兼容性 。此外，还对惰性类加载进行了重新设计，以允许在类名等中使用占位符。对于特殊的ApplicationContext，仍然可以强制执行严格的惰性类加载。
  • 基于@Repository批注的持久性异常转换现在也适用于Hibernate3，JDO和TopLink ，而不仅仅是JPA。现在，异常转换尽可能地基于底层ORM工具的本机异常，而特定于Spring的SQLException转换仅在明确指定时才适用。
  • 在我们的JMS支持中，DefaultMessageListenerContainer具有完善的资源处理功能 （同样适用于JBoss 4.0），并且能够从断开的Connection或Destination中恢复。现在，可以完全配置JMS资源的缓存，对于XA和非XA场景都具有明智的默认设置。此外，JmsTemplate在JTA事务中重用缓存的JMS资源。
  • Servlet和Portlet Web MVC现在支持通用的WebRequestInterceptor抽象 ，它允许View拦截器中的Open Session / EntityManager / etc在Servlet和Portlet环境中重用。结果，所有这些特定于Portlet的拦截器都已被放弃，而使用了新的通用拦截器（OpenSessionInViewInterceptor等）。

当然，在细节方面还有许多其他建议。有关详细信息，请参见更改日志文件（以及JIRA中的更改日志）。

让我们知道您在使用RC3时可能遇到的其他问题。Spring 2.0最终版本现在指日可待。

Juergen Hoeller，
领导Spring框架开发
