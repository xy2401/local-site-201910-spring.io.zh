在Spring中使用JPA而不引用Spring

工程
本·黑尔
2006年8月7日

Spring 2.0增加了对JPA数据访问标准的支持，以及人们期望的所有标准Spring支持类 。马克·费舍尔（Mark Fisher）在如何使用此新支持方面有出色的文章 。但是，我们不断遇到的问题之一就是为什么要使用Spring类（ JpaTemplate ）来访问EntityManager 。这个问题的最佳答案在于JpaTemplate提供的增值。除了提供作为Spring数据访问特征的单线便捷方法之外 ，它还提供自动参与事务以及从PersistenceException到Spring DataAccessException层次结构的转换的功能。

但是我还是不想使用JpaTemplate

很好，因为您不必牺牲Spring的力量。具体来说，无需针对Spring类进行编码即可获得两个最大的优势（事务参与和异常转换）。实际上，Spring实际上对纯API DAO具有广泛的支持。

交易参与

Spring的声明式事务管理的好处之一是，您不必在代码中引用事务结构。因此，如果您希望自动参与交易，则只需要几个bean定义。

JpaTransactionManager负责创建EntityManager的打开事务并将其绑定到当前线程上下文。的只是告诉Spring将事务建议放在任何具有@Transactionalannotation的类或方法上。现在，您只需编写主线DAO逻辑，而不必担心事务语义。


public Collection loadProductsByCategory(String category) {
    return entityManager.createQuery("from Product p where p.category = :category")
        .setParameter("category", category).getResultList();
}

例外翻译

如果您想要Spring的异常翻译，您也可以获取。所有需要发生的就是在类上引入@Repositoryannotation。这个（实际上是次要的）Spring注释只是告诉Spring容器，该类是一个持久性存储库，需要对其进行异常转换。要获取异常转换，需要一个简单的bean定义。

很好，但是如何获得EnityManager？

这实际上可能是最酷的部分。基本上，通过添加@PersistenceContext JPAannotation，您将完全按照不使用Spring的方式定义DAO。


public class ProductDaoImpl implements ProductDao {

    private EntityManager entityManager;

    @PersistenceContext
    public void setEntityManager(EntityManager entityManager) {
        this. entityManager = entityManager;
    }

    public Collection loadProductsByCategory(String category) {
        return entityManager.createQuery("from Product p where p.category = :category")
            .setParameter("category", category).getResultList();
    }
}

通过添加单个bean定义，Spring容器将充当JPA容器，并从EntityManagerFactory注入EnitityManager 。
