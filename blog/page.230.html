Spring Integration 4.0 Milestone 2现在可用

发布
加里·罗素
2013年12月18日

继最近的3.0.0。发布 ，我们很高兴地宣布，Spring Integration 4.0的第二个（第一个公开）里程碑现已发布。

如3.0版本候选公告中所述，最新的Spring Integration 3.0版本与Spring Framework 4.0完全兼容，但不使用spring-messaging模块。这使得Spring Integration 3.0可以与Spring Framework的早期版本一起使用。

以此里程碑为基础的4.0流将所有核心Spring Integration消息传递抽象替换为spring-messaging模块中的抽象。3.0之后不久又发布另一个主要版本的原因是基于这样一个事实，即直接在其代码中使用受影响的SI类的现有应用程序将需要转换为Spring Framework抽象。在大多数情况下，这仅意味着import语句中的软件包更改，但是《 迁移指南》中提供了完整的详细信息。不直接引用类并使用名称空间支持所有配置的用户将不会受到影响。

网络研讨会重播：Reactor进入GA

新闻与活动
克洛伊·杰克逊（Chloe Jackson）
2013年12月17日

演讲者：乔恩·布里斯宾

幻灯片：www.slideshare.net/jbrisbin

Reactor是一个简洁而强大的基础库，用于在JVM上构建响应式，快速数据应用程序。尽管它是Spring IO Platform的一部分，但核心Reactor库并不依赖于Spring。在核心库之上，通过高速处理器抽象直接支持Disruptor，该抽象通过RingBuffer提供了Reactor API，通过灵活的PersistentQueue抽象对高性能JavaChronicle持久消息传递库提供了一流的支持， Groovy闭包和@CompileStatic的类支持，基于Netty 4.0的高性能TCP客户端和服务器支持，强大的基于注释的Spring支持等等。与Jon Brisbin一起参加该活动，以介绍Reactor的第一个主要GA版本，并了解如何使用Reactor的Promise和Stream API来消除异步的，事件驱动的应用程序代码固有的复杂性。
