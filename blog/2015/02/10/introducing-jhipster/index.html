JHipster简介

工程
乔什·朗（Josh Long）
2015年2月10日

这篇文章是社区成员Julien Dubois（@juliendubois）的来宾帖子， Julien Dubois是SpringSource的前雇员，现为Ippon Technologies工作，并且是JHipster项目的创建者。谢谢朱利安！我希望看到更多这些来宾帖子，因此-和往常一样-不要犹豫ping我（@starbuxman） ！ -乔什

介绍

JHipster或“ Java Hipster”是一个方便的应用程序生成器，它将为您创建一个Spring Boot（这是Java部分）和AngularJS（这是时髦部分）应用程序。

在很短的时间内， JHipster在Github上变得非常流行 ，并且在在线杂志（如InfoQ ，Infoworld或SD Times）以及世界各地的会议（巴黎，伦敦，蒙特利尔，奥马哈，台北）中都得到了推荐。 ，里士满，法兰克福！

JHipster致力于使用广泛的Spring技术，通过Java后端生成高质量的应用程序。 Spring Boot ， Spring Security ， Spring Data ，Spring MVC（为websockets，REST和MVC提供框架）等Angular.js前端和一套预配置的开发工具，例如Yeoman ， Maven ， Gradle ， Grunt ， Gulp.js和Bower 。JHipster创建一个完全配置的Spring Boot应用程序，该应用程序带有一组用于用户管理，监视和日志记录的预定义屏幕。生成的Spring Boot应用程序专门为使Angular.js的使用更加顺畅而定制。

JHipster为您提供了更新，管理和打包生成的应用程序的工具。跑mvn package -Pprod触发使用Spring Boot Maven插件创建单个可执行文件的Maven构建.war文件以及Grunt或Gulp.js任务来测试，最小化和优化JavaScript，HTML和CSS代码。

入门

JHipster使用Node.js和Yeoman生成应用程序，并使用Java和Maven（或Gradle）运行生成的应用程序。您可以在在线文档中找到更新的入门指南。

观看以下视频，了解如何创建JHipster应用程序以及如何使用提供的工具集来修改和改进生成的代码。

生成的Spring Boot应用程序演练

生成的应用程序可能会完全不同，具体取决于创建时选择的选项。例如，如果您选择MongoDB代替MySQL ，您的Spring Data存储库将有所不同。如果您选择“ Java 8”而不是Java 7 ，您将在生成的代码中包含lambda表达式。无论您选择什么，生成的应用程序都将共享相似的体系结构和基本代码库。

让我们看一下生成的应用程序的一些关键功能：

  • src/main/java目录在目录中具有Spring Boot配置类config包。JHipster使用Spring的Java配置，没有XML配置。
  • JPA实体或MongoDB文档类在domain包。JPA实体配置有缓存和自动生成的主键。如果使用JHipster生成JPA实体，则它可以创建一对多和多对多关系。
  • （小型）Spring Data存储库中的repository包。
  • 您也可以选择定期@Service - bean 在service层。这些服务通常是事务性和受保护的业务对象。
  • REST端点位于web.rest包，使用Spring MVC的REST支持创建。
  • JHipster还生成Liquibase changelog文件来处理数据库更新。添加新实体将创建特定的架构更新，该更新将进行版本控制，并在重新启动应用程序时执行。
  • 使用Spring的Test Context支持进行集成测试。
  • JHipster在此Spring应用程序的顶部创建了一个完全可用的AngularJS前端，并带有CRUD屏幕来管理您生成的实体。

用JHipster开发

JHipster是一个很好的应用程序启动器，但是一旦生成了应用程序并添加了一些CRUD Angular。在JPA实体顶部的JS屏幕上，您将需要进行编码。

JHipster项目最终只是基于Spring Boot的简单Maven和Gradle项目，可以导入任何了解Maven（或Gradle）和Java的IDE中。

JHipster为您提供了用于构建和更新应用程序的命令行工具。Grunt或Gulp.js可帮助您构建前端。例如，跑步grunt serve将启动由Grunt管理的浏览器，当您的HTML，JavaScript或CSS代码发生任何更改时，该浏览器都会自动重新加载。Bower允许您升级或安装JavaScript或CSS库。例如，跑步bower install ng-table将会安装ng-table 。

您可以使用Spring Boot Maven插件（或Gradle插件）从命令行运行Spring Boot应用程序： mvn spring-boot:run 。您可以使用Maven Liquibase diff自动生成Liquibase变更日志的目标。跑mvn liquibase:diff将会针对您当前的（已修改的）JPA代码检查您现有的数据库架构，并生成必要的Liquibase changelog文件，以便自动更新您的数据库架构。

您可以使用Docker设置您的开发箱。对于某些高级用户（尤其是Linux用户）来说，这可能会更容易，并且如果您拥有一组开发人员，则将使您拥有一致的工作环境。具体的jhipster-docker为此已创建项目。有关更多详细信息，请查看有关安装的JHipster文档页面 。

管理生产中的应用程序

JHipster应用程序具有默认配置admin有权访问多个管理特定屏幕的用户。三个最有趣的屏幕是API屏幕（由Swagger生成），指标屏幕和日志记录屏幕。API屏幕记录了Spring Boot应用程序的REST API，并简化了使用Angular与前端开发人员的合作。JS。 指标屏幕使用Dropwizard指标，并为您提供有关应用程序性能的详细信息，包括Spring bean方法的性能。日志记录屏幕使用Logback，并允许您在运行时更改日志记录级别。例如，您可以更改Spring Security的日志记录级别，以获取有关您的安全配置的详细信息。

JHipster有一个development和一个production模式。的production模式可为您提供精简和优化的前端，GZip压缩，HTTP缓存等。为了测试production模式，使用prod配置文件（适用于Maven或Gradle）： mvn spring-boot:run -Pprod 。使用Chrome开发者工具，您可以对生成的应用程序进行审核，以检查性能。

部署到Cloud Foundry

JHipster拥有Cloud Foundry sub generator这样可以轻松地将应用程序部署到Cloud Foundry： yo jhipster:cloudfoundry 。这将添加一个Cloud Foundry应用程序，创建一个数据库（支持MySQL，PostgreSQL和MongoDB），然后将该数据库绑定到该应用程序并上传（ cf push ）您的应用程序到Cloud Foundry。
