Spring Integration Kafka扩展1.0。GA可用

发布
阿尔特姆·比兰（Artem Bilan）
2015年2月9日

尊敬的Spring社区，

我们很高兴地宣布，Spring Integration Kafka 1.0 GA扩展现已可用，它为Apache Kafka提供了熟悉的Spring Integration端点。
与往常一样，将发布存储库与Maven或Gradle一起使用
compile "org.springframework.integration:spring-integration-kafka:1.0.0.RELEASE"
或下载发行档案 ，以进行试用。

首先，感谢所有为该项目做出贡献的人-特别感谢该项目的创始人Soby Chacko ，他实施了基础架构，以及基于高级消费者的消息源和生产者，还感谢Marius Bogoevici ，他了解了消息驱动型使用者的简单使用者API的复杂性。

总览

这个项目完全基于Apache Kafka （0.8.1.1
版本）和Spring Integration基础（4.0.5。发布版本）。
我们提供了几个抽象，例如Configuration ， ConnectionFactory ， KafkaMessageListenerContainer ， KafkaConsumerContext / KafkaProducerContext ， KafkaMessage等，以遵循Spring的去耦和易用性原则。基于这些抽象，我们提供了高级API，例如KafkaMessageDrivenChannelAdapter ， KafkaHighLevelConsumerMessageSource和KafkaProducerMessageHandler ，它们是Spring Integration的适配器。还提供了XML配置支持。

KafkaHighLevelConsumerMessageSource

向Kafka高级消费者展示了和使用KafkaStream poll来自Kafka主题的消息。如果消息源的多个实例并行运行，它的主要优点是使用简单并且可以平衡使用者之间的分区。

KafkaMessageDrivenChannelAdapter

Kafka Simple Consumer出现了并基于众所周知的ListenerContainer抽象KafkaMessageListenerContainer （类似于Spring AMQP SimpleMessageListenerContainer或Spring JMS DefaultMessageListenerContainer ）：

@Bean
public Configuration zkConfiguration() {
   return new ZookeeperConfiguration(new ZookeeperConnect());
}

@Bean
public ConnectionFactory kafkaConnectionFactory() {
   return new DefaultConnectionFactory(zkConfiguration());
}

@Bean
public MessageProducer kafkaMessageDrivenChannelAdapter() {
   KafkaMessageDrivenChannelAdapter adapter = 
            new KafkaMessageDrivenChannelAdapter(
   		new KafkaMessageListenerContainer(kafkaConnectionFactory(),    
                                       "topic1", "topic2"));
   adapter.setOutputChannel(inputChannel());
   return adapter;
}

该组件的主要优点是可以更好地控制一个组件侦听的分区（可配置）以及起始偏移量（例如在需要重播主题的任何地方）。同样，也可以使用更丰富的偏移管理和错误处理策略。

侦听任务的结果是一个Message ，该Message具有基于Kafka消息的payload ，以及带有来自KafkaHeaders键的其他headers 。顺序保留在分区内。

这两个适配器可以配置kafka.serializer.Decoder 。Spring Integration Kafka提供了现成的Avro Encoder/Decoder实现。

此外，Spring集成卡夫卡介绍OffsetManager抽象得到处理卡夫卡话题偏移，这是不可用High Level Consumer 。介绍了MetadataStoreOffsetManager和KafkaTopicOffsetManager 。必须将OffsetManager注入到KafkaMessageListenerContainer 。默认情况下
MetadataStoreOffsetManager ，由Spring Integration Core的SimpleMetadataStore支持。
