Spring Integration Kafka扩展1.0。GA可用

发布
阿尔特姆·比兰（Artem Bilan）
2015年2月9日

尊敬的Spring社区，

我们很高兴地宣布，Spring Integration Kafka 1.0 GA扩展现已可用，它为Apache Kafka提供了熟悉的Spring Integration端点。
与往常一样，将发布存储库与Maven或Gradle一起使用
compile "org.springframework.integration:spring-integration-kafka:1.0.0.RELEASE"
或下载发行档案 ，以进行试用。

首先，感谢所有为该项目做出贡献的人-特别感谢该项目的创始人Soby Chacko ，他实施了基础架构，以及基于高级消费者的消息源和生产者，还感谢Marius Bogoevici ，他了解了消息驱动的使用者的简单使用者API的复杂性。

总览

这个项目完全基于Apache Kafka （0.8.1.1
版本）和Spring Integration基础（4.0.5。发布版本）。
我们提供了几种抽象Configuration ， ConnectionFactory ， KafkaMessageListenerContainer ， KafkaConsumerContext / KafkaProducerContext ， KafkaMessage等遵循Spring的去耦和易用性原则。基于这些抽象，我们提供了高级API，例如KafkaMessageDrivenChannelAdapter ， KafkaHighLevelConsumerMessageSource和KafkaProducerMessageHandler就Spring Integration而言，它们是适配器。还提供了XML配置支持。

KafkaHighLevelConsumerMessageSource

向Kafka高级消费者展示了和至poll来自Kafka主题的消息，使用KafkaStream 。如果消息源的多个实例并行运行，它的主要优点是使用简单并且可以平衡使用者之间的分区。

典型配置如下所示：

<int-kafka:inbound-channel-adapter kafka-consumer-context-ref="consumerContext"
                                    channel="inputFromKafka">
       <int:poller fixed-delay="10"/>
</int-kafka:inbound-channel-adapter>

<int-kafka:consumer-context id="consumerContext"
                            consumer-timeout="4000"
                            zookeeper-connect="zookeeperConnect">
     <int-kafka:consumer-configurations>
           <int-kafka:consumer-configuration group-id="default"
                                  value-decoder="valueDecoder"
                                  key-decoder="valueDecoder"
                                  max-messages="5000">
                  <int-kafka:topic id="test1" streams="4"/>
                  <int-kafka:topic id="test2" streams="4"/>
           </int-kafka:consumer-configuration>
    </int-kafka:consumer-configurations>
</int-kafka:consumer-context>

如您所见， 需要参考zookeeperConnect 。它是一个简单的bean，代表与Zookeeper集成的连接：

<int-kafka:zookeeper-connect id="zookeeperConnect" 
                          zk-connect="host1,host2,host3:2182"
                          zk-connection-timeout="6000"
                          zk-session-timeout="6000"
                          zk-sync-time="2000"/>

的KafkaHighLevelConsumerMessageSource产生Message与Map>>payload ，就像“按主题划分的Kafka消息”。

KafkaMessageDrivenChannelAdapter

Kafka Simple Consumer出现了并基于众所周知的ListenerContainer抽象- KafkaMessageListenerContainer （类似于Spring AMQP SimpleMessageListenerContainer或Spring JMS DefaultMessageListenerContainer ）：

@Bean
public Configuration zkConfiguration() {
   return new ZookeeperConfiguration(new ZookeeperConnect());
}

@Bean
public ConnectionFactory kafkaConnectionFactory() {
   return new DefaultConnectionFactory(zkConfiguration());
}

@Bean
public MessageProducer kafkaMessageDrivenChannelAdapter() {
   KafkaMessageDrivenChannelAdapter adapter = 
            new KafkaMessageDrivenChannelAdapter(
   		new KafkaMessageListenerContainer(kafkaConnectionFactory(),    
                                       "topic1", "topic2"));
   adapter.setOutputChannel(inputChannel());
   return adapter;
}

该组件的主要优点是可以更好地控制一个组件侦听的分区（可配置），以及起始偏移量（例如在需要重播主题的任何地方）。同样，也可以使用更丰富的偏移管理和错误处理策略。

聆听任务的结果是一个Message与payload基于Kafka消息和其他headers与来自KafkaHeaders 。顺序保留在分区内。

两个适配器都可以配置为kafka.serializer.Decoder卡夫卡消息以及卡夫卡消息密钥。Spring Integration Kafka提供开箱即用的Avro Encoder/Decoder实现。

此外，Spring Integration Kafka还引入了OffsetManager卡夫卡主题偏移处理无法使用的抽象方法High Level Consumer 。的MetadataStoreOffsetManager和KafkaTopicOffsetManager被提出。的OffsetManager必须注入到KafkaMessageListenerContainer 。默认情况下
MetadataStoreOffsetManager使用，由SimpleMetadataStore来自Spring Integration Core。

KafkaProducerMessageHandler

卡夫卡制片人代表和对。最后一个利用目标Kafka的配置Producer ，由KafkaHeaders.TOPIC来自MessageHeaders或通过topic-expression在并与已配置topic在选项子元素：

<int-kafka:producer-context id="kafkaProducerContext">
	<int-kafka:producer-configurations>
		<int-kafka:producer-configuration broker-list="localhost:9092"
					key-class-type="java.lang.String"
					value-class-type="java.lang.String"
					topic="test1"
					value-encoder="kafkaEncoder"
					key-encoder="kafkaEncoder"
					compression-codec="default"/>
		<int-kafka:producer-configuration broker-list="localhost:9092"
					topic="test2"
					compression-codec="default"/>
	</int-kafka:producer-configurations>
</int-kafka:producer-context>

如您所见，有足够的选项可以调整目标Producer ，以及每个Producer可以由特定的支持broker-list 。如果只有单身存在，您可以向任何人发送消息topic ，例如基于Message上下文标头。

Spring XD将这些适配器用作Kafka Source和Sink。此外，它还提供了KafkaMessageBus 。这些功能将很快在Spring XD 1.1 RELEASE中提供。

同样，当我们完成本发行版时， Apache Kafka 0.8.2也收到了其最终发行版。我们很高兴向团队表示祝贺，我们将在不久的将来合并新功能-这仅仅是在春季应用程序中为Kafka提供首要支持的开始！
