用于Apache Hadoop 2.1的Spring发布

发布
托马斯·里斯伯格
2015年2月9日

大约六个月前，我们开始为Apache Hadoop开发Spring 2.1版本。现在，我们很高兴地宣布2.1.0版已全面上市。

从Spring for Apache Hadoop 2.1版本开始，我们现在仅支持Hadoop 2.0 API，不再提供与较早Hadoop v1发行版的向后兼容性。如果您需要对较早Hadoop版本的支持，请使用Spring for Apache Hadoop的2.0.4或1.1.0版本。

2.1版的主要新功能包括：

配置和启动支持 ：

  • 新的@Configuration更改和对Boot自动配置功能的改进。在boot-fsshell DemoApplication示例应用程序中可以看到这种支持的一个很好的例子。
@SpringBootApplication
public class DemoApplication implements CommandLineRunner {

	@Autowired
	private FsShell shell;

	@Override
	public void run(String... args) {
		for (FileStatus s : shell.lsr("/tmp")) {
			System.out.println("> " + s.getPath());
		}
	}

	public static void main(String[] args) {
		SpringApplication.run(DemoApplication.class, args);
	}
}

有关完整的示例，请访问https://github.com/spring-projects/spring-hadoop-samples/blob/master/boot/boot-fsshell

店铺 ：

  • 在HDFS存储编写器中增加了对追加模式的支持。
  • Kite SDK数据集支持已更新至0.17.0。这意味着对该API进行了一些更改。现在，除了basePath外，还必须使用名称空间。现在，DatasetTemplate还使用ViewCallbacks代替分区表达式来查询数据。

纱 ：

  • 在Spring YARN中支持容器分组和集群，这带来了在单个YARN应用程序中运行多种容器类型的功能。
  • 用于提交的应用程序的新REST API和具有新客户端命令和命令行外壳的改进的应用程序模型。
  • 要查看这些功能的示例，请查看yarn-store-groups 示例应用程序或在YARN上运行的Spring XD实现。

我们将继续更新对最新Hadoop版本的支持。现在，我们为以下发行版提供“风味”版本：

  • Apache Hadoop 2.4.1（2.1.0。发布-hadoop24）
  • Apache Hadoop 2.5.2（2.1.0。发行-hadoop25）
  • Apache Hadoop 2.6.0（2.1.0。发布）
  • Pivotal HD 2.1（2.1.0。释放-phd21）
  • Cloudera CDH5 5.3.0（2.1.0。发行-cdh5）
  • Hortonworks HDP 2.2（2.1.0。发布-hdp22）

现在默认的发行版本是Apache Hadoop 2.6.0。

向前走

随着Hadoop生态系统的快速发展，我们希望更频繁的发布将有助于我们跟上发展。对于下一个版本，我们计划添加以下内容：

  • 更好的Java配置支持。
  • 为Hiveserver2添加更好的支持，包括批处理任务集。
  • 对批处理Tasklet的基本支持，以运行Spark应用程序。
  • 跨不同模块提供更好的启动支持。
  • 改进的安全支持（即YARN Boot CLI交互等）。
  • 增强了与Spring-cloud组件（即spring-cloud-cluster）的无缝集成。

请通过JIRA问题或GitHub问题提供反馈和功能请求（请参阅项目页面以获取链接）。
