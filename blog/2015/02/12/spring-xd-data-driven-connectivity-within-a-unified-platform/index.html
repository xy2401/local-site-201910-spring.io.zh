Spring XD：统一平台内的数据驱动连接

发布
萨比·阿南丹（Sabby Anandan）
2015年2月12日

无论您是在家中，办公室还是在途中，连接都是常态。这是我们所有人都期望并依赖的日常生活的一部分。人与信息之间的连通性全都与数据的移动和分析有关：数据提供见解，而这些见解必须越来越多地为用户提供即时结果。这种始终在线，始终可用的连接水平带来了许多挑战。数据类型，数据格式和数据量是动态的，数据生成代理也是如此。

Spring XD在统一平台中解决了这些众多挑战。无论是通过连续数据流还是基于计划的数据移动，Spring XD都提供了端到端数据流水线功能，可以在需要的位置和方式使用，处理，分析和暂存数据。

在1.1 GA版本中，Spring XD通过添加对项目Reactor ， RxJava和Spark Streaming的支持来适应功能流处理。除了当前的Spring Batch和基于Hadoop的作业（MR / Hive / Pig）之外，我们还增加了对Sqoop和Spark批处理作业的支持。认识到Python在大数据应用程序中的重要性，我们还添加了Python处理器和接收器模块以与Spring XD流集成。这使用户能够为用例选择合适的工具，从而使他们能够利用本机API来构建复杂的数据处理管道。

与其他流处理框架不同，Spring XD的DSL（特定于域的语言）消除了编码要求。不用设置项目，IDE，构建脚本或捆绑，而使用高级配置DSL 。Spring XD团队继续关注开发人员的生产力，该生产力为平台带来了许多开发人员友好的附加组件和示例 。流是流水线中的一系列处理单元，可以通过DSL进行编排。内置的Admin UI可用于远程监视和管理流，批处理作业和集群。

Spring XD 1.1 GA提供了数以千计的数据管道组合，可以直接使用它们构建，而无需编码。与其他流处理框架不同，此功能由企业信任的Spring Integration项目支持，该项目被公认为是新兴标准和市场份额的领导者 。

Spring XD的核心是易于扩展的支持。1.1 GA版本增加了一种开发和贡献定制模块的新方法。借助Spring Boot的模块打包插件，您现在可以选择在Maven或Gradle中进行选择以捆绑您的自定义模块。通过包括自定义模块所需的依赖关系，您可以通过REST-API部署打包的uber-jar。这不需要降低运行时间，也不必担心当前正在运行的管道。REST-API简化了贡献，因为已安装的自定义模块可立即使用并可以使用。

由于其可插拔的体系结构，Spring XD继续与框架集成以简化大数据应用程序开发。在此最新版本中，您可以使用Kafka主题（ source-adapter ）或写入Kafka主题（ sink-adapter ），也可以将其插入Kafka作为消息总线。这使Spring XD可以处理业务流程，使您可以专注于业务逻辑。

通过全新的Kafka Client API ，用户现在可以从新选项中进行选择，例如在流级别进行分区，控制偏移量，批处理和可靠的数据重新处理。

1.1 GA版本还与顶级Apache项目Spark集成。通过将现成的Spark用作批处理作业 ，或将Spark Streaming用作处理器模块 ，可以使用本机Spark API创建数据管道。对于已经使用Spark的人来说，这是熟悉的开发经验。同时，Spark的任何新手都可以使用模块化和分布式容错运行时，立即利用数十个输入和输出适配器，而只专注于业务需求。为了解决单点故障，Spring XD通过重新启动Spark Streaming驱动程序以从故障方案中恢复来提供附加值。

随着Spring XD 1.1 GA的发布，没有理由开发大数据应用程序既费时又复杂。Spring XD的目标是消除应用程序开发和大数据之间的障碍，并缩短从数据提取到洞察的周转时间，从而实现真正的数据驱动型应用程序。在将来的版本中，我们将继续提高开发人员的工作效率，同时保留Spring XD的核心作为开放和可扩展的运行时。
