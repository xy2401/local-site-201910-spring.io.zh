今年春季-2015年12月29日

工程
乔什·朗（Josh Long）
2015年12月29日

欢迎来到春季本周的另一期，其中是2016年
指日可待，我们将看看上周的一些有趣新闻
我们会花一些时间来审查另一个惊人一年的 Spring 和关键的生态系统。

现在，让我们看一下最近一周以来最新的最简短的清单：

  • 从假期开始，有没有Amazon.com礼品卡在您的口袋里烧洞？
    好吧，不用担心，我已经为您准备了东西！终于， 春季传奇人物克雷格·沃兹 （ Craig Walls）的史诗
    Spring Boot in Action可用！趁热拿到！冲啊！
  • Netflix的Genie ，它是作业调度和资源管理层
    基于Spring Boot的大数据作业最近才从Swagger切换到Spring REST Docs
  • Christoph Burmeister汇总了使用Spring Boot构建全栈Web应用程序的介绍
  • 此葡萄牙语语言帖子介绍了如何使用Spring Tool Suite和Spring Boot启动新项目
  • 这篇西班牙语文章介绍了如何通过Thymeleaf视图来站起来的Spring Boot Web应用程序，该应用程序由Spring Security保护
  • 此中文帖子还介绍了如何使用Spring Boot入门
  • 我们的朋友Petri Kainulainen又回来了，这一次是关于使用Spring Boot 生成 .pdf炮轰到wkhtmltopdf
  • Jakub Narloch有一篇关于使用Spring Cloud Zuul创建Trie匹配器的不错的文章
  • SimonBaslé刚刚宣布了Spring Data Couchbase 2.0的第一个候选版本 ！
    尽您所能，撬起轮胎并踢轮胎！在Spring Boot中支持Spring Data Couchbase的相应PR 也已更新
  • 查看这篇文章， 学习如何使用社区项目Spring Data JDBC。

保持事物的透视很有帮助；尽管每个星期都为我们带来了一些有趣的东西，但在《 今年的 Spring 》中 ，有些事情是最重要的，值得在此进行修改。

  • 在今年4月， 我们发布了Spring Cloud 1.0 ，该版本支持服务注册和发现，客户端负载平衡，断路器，微代理等 。Spring Cloud建立在Spring Boot的基础上，并被Netflix，TicketMaster，阿里巴巴等公司全心全意地（公开地）拥抱。
  • 在SpringOne2GX 2015之前，我们宣布了Spring Cloud Data Flow ，它将Spring XD中的技术分解为基于Spring Boot的小型微服务和自动配置。Spring Cloud Data Flow建立在Spring Cloud Stream，Spring Data和Spring Batch之上，可轻松组成和控制大数据作业和流。
  • Spring Boot 1.3在2014年11月发布Spring Boot 1.2大约一年后发布。该发行版包含各种新功能，包括众多新的自动配置，强大的支持开发时便利性（例如实时重新加载），当然还有令人惊叹的彩色横幅ASCII图稿 ！
  • 11月，我们发布了期待已久的Pivotal Cloud Foundry 1.6 ，它基于对Cloud Foundry OSS代码的最新和最大改进，包括支持运行基于Windows和Docker的工作负载的Diego Scheduler，并添加了功能强大的Spring Cloud Services这使得诸如服务注册表和配置服务之类的独立基础架构变得非常简单。
  • 我们宣布了MicroPCF ，这是获得类似于本地节点Pivotal Cloud Foundry的体验的最简单方法，开发人员和管理员都熟悉

感谢您的收看，别忘了下周再次加入我们，我们将在新的一年拉开序幕，并标志着本周的春季迎来五年！

代表春季和关键团队，新年快乐！
