Spring AMQP 1.4.5。发布和1.5.0。M1可用

发布
加里·罗素
2015年5月8日

我们很高兴地宣布，Spring AMQP（1.4.5。RELEASE）现在可用。自1.4.4以来，它仅包含一些错误修复 。

此外，还提供了1.5版本的第一个里程碑版本（1.5.0。M1）。一个候选版本将在一个月左右的时间内提供，预计将于今年夏季晚些时候发布。

此版本的主要功能突出显示如下：

班级@RabbitListener

现在可以定义@RabbitListener类级别的注释。与方法级别一起@RabbitHandler注解，这允许单个侦听器根据有效负载类型调用类中的不同方法。例如：

@RabbitListener(queues="orders, shipments")
public class MultiListenerBean {

    @RabbitHandler
    public OrderConfirmation handleOrder(Order order) {
        ...    
    }

    @RabbitHandler
    public Invoice ship(Shipment shipment) {
        ...
    }

}

自动声明队列和绑定@RabbitListener

现在可以在使用时自动声明队列，交换和绑定@RabbitListener注解。

@RabbitListener(bindings = @QueueBinding(
	value = @Queue(value = "auto.declare", autoDelete = "true"),
	exchange = @Exchange(value = "auto.exch", autoDelete = "true"),
	key = "auto.rk"))
public String handleWithDeclare(String foo) {
	return foo.toUpperCase();
}

在此示例中，队列auto.declare被宣布并有义务交换auto.exch带路由键auto.rk 。交易所也被宣布。

以前， Queue ， Exchange和Binding必须声明为单独@Bean s。

可配置的交换/路由密钥以进行答复

以前，在将请求/回复消息与RabbitTemplate ，答复被路由到默认交换，并使用队列名进行路由。现在可以提供一个reply-address与形式exchange/routingKey使用特定的交换和路由密钥进行路由。

RabbitManagementTemplate

的spring-erlang不再提供jar。鼓励用户改用RabbitMQ REST API。RabbitMQ团队正在为REST API进行Java绑定，而Spring AMQP现在使用熟悉的API为该API提供了轻量级包装Queue ， Exchange和Binding域对象。

结论

有关新功能的完整详细信息，请参见参考手册的“新增功能”部分和JIRA发行说明 。

鼓励用户在GA发布之前尝试这些新功能，并使用常规渠道提供反馈（ spring-amqp堆栈溢出和JIRA标签）。

请参阅项目页面以获取文档和下载的链接。
