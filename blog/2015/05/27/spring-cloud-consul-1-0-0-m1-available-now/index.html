Spring Cloud Consul 1.0.0。M1现在可用

发布
斯宾塞·吉布
2015年5月27日

Consul是用于发现和配置基础结构中的服务的系统。它是由Hashicorp建造的， Hashicorp是创建Vagrant和Packer的聪明人。领事提供所有服务，例如服务发现，运行状况检查，密钥/值存储，同时开箱即用地支持多个数据中心。

Spring Cloud Consul旨在将所有这些功能引入Spring Cloud生态系统。该项目已达到第一个里程碑，可在repo.spring.io存储库中找到新鲜的jar。Spring Cloud Consul提供以下功能：

  • Spring Cloud Consul发现：Spring Cloud Commons的实现DiscoveryClient 。服务注册和发现通过Consul HTTP API执行 。

  • Spring Cloud Consul Config：通过Consul Key / Value API进行分布式配置。这的行为类似于Spring Cloud Config Client ，但由分布式Consul KV Store支持。

  • Spring Cloud Consul总线：一种事件总线，用于将服务和服务实例与分布式消息传递链接在一起。对于在群集中传播状态更改（例如配置更改事件）很有用。这是使用Consul Event API实现的 。

  • Spring Cloud Consul UI： Consul Web UI的嵌入式版本。

以上所有均具有1.0.0。M1版本标签，开始时，请在您的产品中包含以下工件pom.xml ：

<dependency>
  <groupId>org.springframework.cloud</groupId>
  <artifactId>spring-cloud-consul-config</artifactId>
  <version>1.0.0.M1</version>
</dependency>
<dependency>
  <groupId>org.springframework.cloud</groupId>
  <artifactId>spring-cloud-consul-discovery</artifactId>
  <version>1.0.0.M1</version>
</dependency>
<dependency>
  <groupId>org.springframework.cloud</groupId>
  <artifactId>spring-cloud-consul-bus</artifactId>
  <version>1.0.0.M1</version>
</dependency>
<dependency>
  <groupId>org.springframework.cloud</groupId>
  <artifactId>spring-cloud-consul-ui</artifactId>
  <version>1.0.0.M1</version>
</dependency>

请遵循文档以确保Consul已安装并正在运行。

一个示例应用程序可能类似于以下内容：

@SpringBootApplication
@EnableDiscoveryClient
@EnableConsulUi
@RestController
public class SampleApp2 {

  @RequestMapping("/")
  public String hello() {
    return "Hello World";
  }

  public static void main(String[] args) {
    SpringApplication.run(SampleApplication.class, args);
  }
}

运行您的应用后，点击http://localhost:8080/ui看领事ui。

该代码托管在github上，非常欢迎社区贡献，因此请继续阅读并查看。在spring-cloud-consul-sample中有一个示例 。自述文件中包含运行样本的说明。
