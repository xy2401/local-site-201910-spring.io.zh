SpringOne2GX 2014重放：内存中的大数据

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2015年3月24日

记录在SpringOne2GX 2014中。内存中的大数据

演讲者：约翰·戴维斯（C24）

大数据轨道

幻灯片： http ： //www.slideshare.net/SpringCentral/spring-one2gx-2014-john-davies-41130273

好的，所以每个人都在关注大数据，但他们通常谈论的是持久性，磁盘或更近的SSD，内存又如何？我们可以简单地添加几个TB的RAM，但即使是每GB 100美元，也要花费很多。如果我们可以将数据大小减少50倍并有效地将成本RAM降低到磁盘成本，该怎么办？保留Spring Integration，Spring Batch，GemFire内存缓存，RabbitMQ消息传递，但将数据缩减为二进制（是位和字节）而不是对象。更少的垃圾，更少的网络开销，相同的API，但内存中的大数据。John将展示一个消耗7.4kB XML消息的Spring工作流，将其绑定到25kB Java，但每个消息仅存储450个字节，并在笔记本电脑上存储1000万个派生合同。
