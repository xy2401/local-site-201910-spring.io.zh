Spring Cloud 1.0.0现在可用

发布
戴夫·瑟
2015年3月4日

Spring Cloud 1.0.0。现在可以在Maven Central（和repo.spring.io）中使用RELEASE。自RC3以来，变化不大，但是由于社区的参与，我们确实发现了一些错误，因此感谢所有迄今为止尝试过的人。这提醒了Spring Cloud的目标：

Spring Cloud为开发人员提供了工具，可快速构建分布式系统中的一些常见模式（例如，配置管理，服务发现，断路器，智能路由，微代理，控制总线，一次性令牌，全局锁，领导选举，分布式会话，群集状态）。分布式系统的协调导致样板式样，并且使用Spring Cloud开发人员可以快速站起来实现这些样板的服务和应用程序。它们将在任何分布式环境中都能很好地工作，包括开发人员自己的笔记本电脑，裸机数据中心以及诸如Cloud Foundry之类的托管平台。

以及1.0内容的快速摘要：

  • 带有git后端的配置服务器
  • 配置客户端具有刷新和加密/解密以及Spring应用程序生命周期中的引导阶段
  • 负载均衡器，服务发现和断路器的通用抽象
  • 安全集成：声明式SSO和代理身份验证策略
  • Eureka用于服务发现：客户端和服务器（带有UI）
  • 适用于非JVM客户端的Eureka Sidecar
  • 使用Zuul的自动反向代理
  • Zuul过滤器的Spring配置模型
  • 功能区负载均衡器的弹簧配置模型
  • 虚假的声明式Web服务客户端（具有Ribbon集成）
  • RestTemplate与Ribbon集成
  • 断路器的Hystrix：客户端和仪表板（UI）
  • 涡轮机，用于断路器汇聚，通过HTTP进行推送和通过AMQP进行推送
  • AWS集成：关系数据库，消息传递和EC2元数据
  • AMQP总线用于一组定义的操作事件，例如配置更改
  • 支持以上大多数功能的Groovy CLI

在http://cloud.spring.io和Github中找到更多信息。
