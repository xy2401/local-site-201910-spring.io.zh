Couchbase和Spring Data Couchbase入门

工程
乔什·朗（Josh Long）
2015年3月16日

该博客的灵感来自于Couchbase的开发人员倡导者Laurent Doguin的讲话，我去年在Couchbase Connect上发表了这一讲话。梅西·洛朗！

这是Spring Data Couchbase集成的演示。在项目页面中 ，Spring Data Couchbase是：

Spring Data Couchbase项目提供了与Couchbase Server数据库的集成。Spring Data Couchbase的关键功能区域是一个以POJO为中心的模型，用于与Couchbase桶进行交互并轻松编写存储库样式的数据访问层。

什么是Couchbase？

Couchbase是具有真正水平缩放功能的分布式数据存储。我喜欢将它看作Redis和MongoDB的混合体：您使用通过其键访问的文档。有许多适用于所有语言的客户端API。如果您将Couchbase用于后端并使用JVM，那么您会喜欢Spring Data Couchbase的 。项目主页上的项目符号最好地列举了其许多功能：

  • Spring配置支持使用基于Java的@Configuration类或Couchbase驱动程序的XML名称空间。
  • CouchbaseTemplate帮助程序类，可提高执行常见Couchbase操作的效率。包括文档和POJO之间的集成对象映射。
  • 将异常转换为Spring的可移植数据访问异常层次结构。
  • 与Spring的转换服务集成的功能丰富的对象映射。
  • 基于注释的映射元数据，但可扩展以支持其他元数据格式。
  • Repository接口的自动实现，包括对自定义finder方法的支持（由Couchbase Views支持）。
  • JMX管理和监视
  • 透明的@Cacheable支持可缓存任何需要高性能访问的对象。

运行Couchbase

使用Vagrant在本地运行Couchbase

如果您（自然）还没有安装Couchbase，则需要安装。Michael Nitschinger（ @daschl ，也是Spring Data Couchbase项目的负责人）在博客中介绍了如何在此处启动并运行一个简单的4节点Vagrant集群 。我已经在vagrant目录中复制了他的示例。要使用它，你需要安装虚拟盒和流浪 ，当然，后来干脆跑vagrant up在vagrant目录。为了获得此配置脚本的最新版本，我去了Michael的GitHub vagrants项目 ，发现除了此示例之外，还有许多其他的Vagrant脚本可用 。我在此代码的项目目录中有一个指向该子模块的子模块，但请确保参考最新和最新的子模块。为了使所有内容都在我的计算机上运行，我选择了Couchbase 3.0.2的Ubuntu 12安装 。您可以通过在启动之前配置VAGRANT_NODES环境变量来更改启动多少节点：

VAGRANT_NODES=2 vagrant up

您需要在初始设置时管理和配置Couchbase。将浏览器指向每个节点的正确IP。有关确定IP的规则，请README 。就我而言，管理界面位于192.168.105.101:8091和192.168.105.102:8091 。有关此过程的更多信息，建议您遵循此处的指南以获取详细信息。

这是我的方法。我在第一个节点上点击了管理界面，并创建了一个新集群。我使用admin作为用户名，使用password作为密码。在所有后续管理页面上，我仅通过将节点指向192.168.105.101并使用上述admin凭据就可以加入现有集群。加入所有节点后，在“ 服务器节点”面板中寻找“ Rebalance按钮并触发集群重新平衡。

如果您完成了Vagrant集群的操作，则可以使用vagrant halt命令将其彻底关闭。vagrant suspend也非常方便，它可以保存节点的状态，而不是完全关闭它们。

如果要从命令行管理Couchbase集群，则有方便的couchbase-cli 。您可以简单地使用vagrant ssh命令进入每个节点（通过它们的节点名： node1 ， node2等）。在那里，您可以运行集群配置命令。例如， server-list命令将枚举群集节点。

/opt/couchbase/bin/couchbase-cli server-list -c 192.168.56.101 -u admin -p password

使用以下方法很容易触发重新平衡 ：

/opt/couchbase/bin/couchbase-cli rebalance -c 192.168.56.101 -u admin -p password

云和Cloud Foundry中的Couchbase

Couchbase适合在云中使用。它是水平可伸缩的（例如Gemfire或Cassandra），没有单点故障。它不使用主从系统或主动/被动系统。有几种方法可以在应用程序运行的地方启动和运行它。如果您正在运行Cloud Foundry安装，则可以安装Cumulogic Service Broker ，然后让您的Cloud Foundry安装与本身可以管理Couchbase实例的Cumulogic平台进行对话。服务代理是集成代码的一部分，在这种情况下，它可以教Cloud Foundry如何配置，销毁以及通常与托管服务（例如Couchbase）进行交互。

获取Facebook访问令牌

您还需要一个Facebook访问令牌。最简单的方法是转到Facebook Developer Portal ，创建一个新应用程序，然后获取应用程序ID和应用程序密码。取这两个值，并用派克字符（ | ）将它们连接起来。这样，您将具有以下形式： appID|appSecret 。该示例应用程序使用Spring的Environment机制来解析facebook.accessToken密钥。您可以在src/main/resources/application.properties文件中或使用其他任何受支持的Spring Boot属性解析机制为其提供值。您甚至可以将值提供为-D参数： -Dfacebook.accessToken=...|...

向Spring Data Couchbase讲述我们的集群

Couchbase中的数据存储在存储桶中 。从逻辑上讲，它与SQL RDBMS中的数据库相同。它通常跨节点复制，并具有自己的配置。我们将使用默认存储桶，但是创建更多存储桶很容易。

让我们看一下使用Spring Data Couchbase所需的基本配置（在这种情况下，就Spring Boot应用程序而言）：


@SpringBootApplication
@EnableScheduling
@EnableCaching
public class Application {

  @EnableCouchbaseRepositories
	@Configuration
	static class CouchbaseConfiguration extends AbstractCouchbaseConfiguration {

		@Value("${couchbase.cluster.bucket}")
		private String bucketName;

		@Value("${couchbase.cluster.password}")
		private String password;

		@Value("${couchbase.cluster.ip}")
		private String ip;

		@Override
		protected List bootstrapHosts() {
			return Arrays.asList(this.ip);
		}

		@Override
		protected String getBucketName() {
			return this.bucketName;
		}

		@Override
		protected String getBucketPassword() {
			return this.password;
		}
	}

  // more beans

}

Spring Data Couchbase存储库

Spring Data提供了存储库的概念-处理典型数据访问逻辑并提供基于约定的查询的对象。它们可用于将POJO映射到后备数据存储中的数据。

我们的示例仅存储了从Facebook的Places API读取的企业信息。为了实现这一点，我们创建了一个简单的Place实体，Spring Data Couchbase存储库将知道如何持久存储：


@Document(expiry = 0)
class Place {

	@Id
	private String id;

	@Field
	private Location location;

	@Field
	@NotNull
	private String name;

	@Field
	private String affilitation, category, description, about;

	@Field
	private Date insertionDate;
  // .. getters, constructors, toString, etc
}

Place实体引用的另一个实体Location基本上相同。

对于Spring Data Couchbase，存储库查找器方法映射到Couchbase服务器中的视图 -用JavaScript编写的查询。您需要在Couchbase服务器上设置视图。转到任何Couchbase服务器的管理控制台并访问“ 视图”屏幕，然后单击“ 创建开发视图”并将其命名为place ，因为我们的实体将是demo.Place （默认情况下，开发视图名称从实体的类名称改编而来）。

我们将创建两个视图，这是所有Spring Data Couchbase POJO必需的通用all ，以及将用于驱动存储库的findByName finder方法的byName视图。尽管您可以使用finder方法声明中的@Viewannotation覆盖使用哪个视图，但该映射是@View进行的。
