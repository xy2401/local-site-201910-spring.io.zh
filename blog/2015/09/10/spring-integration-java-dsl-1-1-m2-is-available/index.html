Spring Integration Java DSL 1.1 M2可用

发布
阿尔特姆·比兰（Artem Bilan）
2015年9月10日

我们很高兴地宣布， Milestone存储库现已提供Spring Integration Java DSL的Milestone 2：

对于Gradle，请使用以下命令：

compile "org.springframework.integration:spring-integration-java-dsl:1.1.0.M2"

对于Maven来说：


     org.springframework.integration
     spring-integration-java-dsl
     1.1.0.M2

自Milestone 1以来，没有太多新功能，但以下是需要注意的最重要的事项：

Spring Integration 4.2基础

Spring Integration Java DSL 1.1现在基于Spring Integration 4.2 ，成为即将推出的Spring IO 2.0的一部分 。Java DSL的1.1版本不再与Spring Integration 4.1.x兼容。

Spring Integration Kafka 1.2.x升级

我们现在提供Kafka Spring Integration Kafka适配器的命名空间工厂。新的Spring Integration Kafka 1.2 版本以及对Apache Kafka的升级0.8.2带来了许多重大变化。因此，此Java DSL Milestone 2 of 1.1不再分别与Spring Integration Kafka和Apache Kafka的先前版本兼容。

HTTP命名空间工厂

现在，Spring Integration HTTP Adapter涵盖了它们的特定功能Http命名空间工厂。请满足我们经典的HTTP代理示例，但是已经在Java DSL上：

@Bean
public IntegrationFlow httpProxyFlow() {
    return IntegrationFlows
        .from((MessagingGateways g) ->
               g.httpGateway("/proxy")
                     .requestMapping(r -> r.params("name"))
                     .payloadFunction(httpEntity ->
                           ((ServletRequestAttributes) RequestContextHolder
                                                     .currentRequestAttributes())
						.getRequest()
						.getQueryString()))
        .handleWithAdapter(a ->
              a.httpGateway(m ->
                   String.format("http://target/service?%s", m.getPayload()))
                              .expectedResponseType(String.class))
        .get();
}

反应式流支持

我们很高兴地宣布，Spring Integration开始提供与Reactive Streams的集成并为Publisher直接在Java DSL中显示：

@Bean
public Publisher> reactiveFlow() {
	return IntegrationFlows
		.from(() -> new GenericMessage<>("a,b,c,d,e,f"))
		.split(String.class, p -> p.split(","))
		.toReactivePublisher();
}

如您所见，从Spring Integration世界转到Reactive Streams世界很简单。的org.reactivestreams.Publisher之后可以将bean用作Reactive程序的事件源，例如，使用Project Reactor实现：

@Autowired
@Qualifier("reactiveFlow")
private Publisher> publisher;
....
List results = new ArrayList<>();
CountDownLatch latch = new CountDownLatch(6);
Streams.wrap(this.publisher)
	.map(m -> m.getPayload().toUpperCase())
	.consume(p -> {
			results.add(p);
			latch.countDown();
		});

另一个方向-从响应流到Spring集成-也很简单（从Reactor的角度来看）：

.consume(v -> messagingTemplate().convertAndSend(reactorStreamResult(), v));

我们会考虑一些org.reactivestreams.Subscriber的选项MessageChannel在将来的版本中实施。

其他变化

为了使DSL更流畅，方便和灵活，还进行了其他重要的重构和弃用，例如：

  • 的Files.splitter()的工厂方法FileSplitter零件;
  • 的(S)Ftp.outboundGateway()对于新MessageSessionCallback （Spring Integration 4.2的功能）；
  • 重构和改进.aggregate()和.resequence() EIP方法等
