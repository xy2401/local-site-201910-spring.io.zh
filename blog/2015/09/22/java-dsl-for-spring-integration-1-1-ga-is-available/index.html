适用于Spring Integration 1.1 GA的Java DSL

工程
阿尔特姆·比兰（Artem Bilan）
2015年9月22日

亲爱的春季社区！

我很荣幸地代表Spring Integration团队宣布，可以从Release Repository和Maven Central获得Spring Integration Java DSL 1.1 GA。

对于Gradle，请使用以下依赖项：

'org.springframework.integration:spring-integration-java-dsl:1.1.0.RELEASE'

对于Maven来说：


     org.springframework.integration
     spring-integration-java-dsl
     1.1.0.RELEASE

首先，非常感谢上周在SpringOne 2GX 2015上访问我的演讲的所有人。感谢您收到的所有反馈。

因此，我们能够在GA和以前的Milestone之间添加一些小的修订。

正如我在演讲中所承诺的，我们为Spring Integration Core模块中的简单适配器添加了DSL定义。

让我在这里再次演示一下，并解释该解决方案基于什么来与将来的贡献者共享一些挂钩：

@Bean
public IntegrationFlow feedFlow() {
return IntegrationFlows
              .from(s -> s.feed(this.feedUrl, "feedTest")
                                    .feedFetcher(new FileUrlFeedFetcher())
                                    .metadataStore(metadataStore()),
                             e -> e.poller(p -> p.fixedDelay(100)))
              .channel(c -> c.queue("entries"))
              .get();
}
  • 它是一个IntegrationFlow定义从inbound-channel-adapter为了FeedEntryMessageSource 。
  • 第一个Lambda .from()方法是MessageSources仅委托给目标命名空间工厂的工厂：
public FeedEntryMessageSourceSpec feed(URL feedUrl, String metadataKey) {
	return Feed.inboundAdapter(feedUrl, metadataKey);
}

它的存在是为了方便使用-从IDE自动完成功能中获益。

  • 在这种情况下，我们委托给org.springframework.integration.dsl.feed.Feed具有非常短的源代码的命名空间工厂：
public abstract class Feed {

	public static FeedEntryMessageSourceSpec inboundAdapter(URL feedUrl, 
                                      String metadataKey) {
		return new FeedEntryMessageSourceSpec(feedUrl, metadataKey);
	}

}
  • 的FeedEntryMessageSourceSpec是MessageSourceSpec具有一个Builder模式的实现FeedEntryMessageSource 。

Spring Integration Java DSL中的所有其他组件都是以相同的方式构建的，我希望它看起来足够简单，可以从社区中获得越来越多的贡献，以解决您在Framework中对特定适配器的支持。

请继续关注Spring Framework 5基础以及与Java 8的更有机集成。
