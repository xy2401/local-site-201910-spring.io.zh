Spring Cloud Brixton发布火车的第一个里程碑已经发布

发布
斯宾塞·吉布
2015年9月16日

我很高兴代表Spring Cloud团队宣布Spring Cloud Brixton发布火车的第一个里程碑。该里程碑今天可用，可以在我们的Spring Milestone存储库中找到 。我们进行了许多增强和错误修复，其中一些重点包括：

  • Spring Boot 1.3.x和Spring 4.2.x支持
  • 集群领导者选举和锁定
  • Hashicorp Consul支持服务注册/发现，配置和总线
  • Apache Zookeeper支持服务注册/发现，配置和领导者选举
  • 莱迪思对服务注册/发现的支持
  • 分布式跟踪支持

以下是Brixton中的新模块。M1：

  • Spring Cloud for Cloud Foundry 1.0.0。M1
  • Spring Cloud Cluster 1.0.0。M1
  • Spring Cloud Consul 1.0.0。M2
  • Spring Cloud Lattice 1.0.0。M1
  • Spring Cloud Sleuth 1.0.0。M1
  • Spring Cloud Zookeeper 1.0.0。M1

以下模块已更新：

  • Spring Cloud Commons 1.1.0。M1
  • Spring Cloud Config 1.1.0。M1
  • Spring Cloud Netflix 1.1.0。M1
  • Spring Cloud Security 1.1.0。M1

Spring Cloud AWS和Spring Cloud Bus在此里程碑中未获得新版本。

要开始使用带有父POM的Maven，请执行以下操作：

<parent>
  <groupId>org.springframework.cloud</groupId>
  <artifactId>spring-cloud-starter-parent</artifactId>
  <version>Brixton.M1</version>
</parent>
<repositories>
    <repository>
        <id>spring-milestones</id>
        <name>Spring Milestones</name>
        <url>http://repo.spring.io/milestone</url>
        <snapshots>
            <enabled>false</enabled>
        </snapshots>
    </repository>
</repositories>

或作为BOM（仅依赖关系管理）

<repositories>
    <repository>
        <id>spring-milestones</id>
        <name>Spring Milestones</name>
        <url>http://repo.spring.io/milestone</url>
        <snapshots>
            <enabled>false</enabled>
        </snapshots>
    </repository>
</repositories>
<dependencyManagement>
  <dependencies>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-parent</artifactId>
    <version>Brixton.M1</version>
    <type>pom</type>
    <scope>import</scope>
  </dependencies>
</dependencyManagement>
<dependencies>
  <dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-config</artifactId>
  </dependency>
  <dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-eureka</artifactId>
  </dependency>
  ...
</dependencies>

或与gradle：

buildscript {
  dependencies {
    classpath "io.spring.gradle:dependency-management-plugin:0.5.3.RELEASE"
  }

    }

  }

    ...
}
