莱迪思和Spring Cloud：云原生Spring应用程序的弹性子结构

工程
马特·斯汀（Matt Stine）
2015年4月6日

我们相信， 云原生应用程序架构的开发是企业应用程序开发的下一个巨大发展阶段。这些架构结合了十二要素应用程序 ， 微服务 ， 自助式敏捷基础架构 ， 基于API的协作和抗脆弱性等元素。所有这些要素使我们能够在不断为客户提供业务价值的同时，快速而安全地移动。

Spring团队的目标一直是打赢Java复杂性的战争 ，现在我们正在与Cloud Foundry团队的同事合作，以消除云原生应用程序的开发和操作中的复杂性。

我们从Spring Cloud开始这项工作， Spring Cloud是一个伞形项目，它使云原生应用程序体系结构具有与Spring Boot相同的简单性和生产力。分布式系统的协调可以通过应用许多特征鲜明的样板来完成。使用Spring Cloud，开发人员可以快速支持实现这些模式的服务和应用程序。这些模式中的许多模式都是通过包装NetflixOSS上久经考验的组件而提供的 。

您可以在任何地方开发和运行Spring Cloud应用程序，包括您的笔记本电脑，裸机数据中心基础架构或AWS或Google Cloud等云基础架构。但是，为了获得最大的效率，云原生应用程序需要一个云原生应用程序平台。我们相信Cloud Foundry是与Spring Cloud结合使用的平台，它为构建和运行云原生应用程序提供了最佳的必要子结构。

从历史上看，在笔记本电脑上运行Cloud Foundry一直是一个挑战。这就是为什么我们很高兴向您介绍莱迪思 。Lattice是一个云原生应用程序平台，使您能够使用Docker等解决方案在容器中运行应用程序。莱迪思的功能包括：

  • 集群调度
  • HTTP负载均衡
  • 日志汇总
  • 健康管理

莱迪思通过打包在Cloud Foundry弹性运行时中找到的组件的子集来做到这一点。结果是适合于快速应用程序开发的开放式单租户环境。使用莱迪思开发的应用程序应原样迁移到完整的Cloud Foundry部署。

为了帮助您开始使用莱迪思，我们发布了两个入门指南：

  • 带有Docker的Spring Boot ：Lattice当前支持打包为Docker映像的运行应用程序，因此本指南可帮助您使用Maven或Gradle从Spring Boot应用程序构建Docker映像。
  • Spring Cloud + Lattice ：本指南将引导您完成设置Lattice的工作，然后向其部署Spring Boot应用程序。按照Docker指南打包的应用程序应该可以轻松地在Lattice上运行。

第二本指南重点介绍了我们在Spring Cloud和Lattice之间的首次直接集成，即spring-cloud-lattice ，这是Spring Cloud的实现DiscoveryClient由Lattice的API支持。这种实现方式使我们可以直接从莱迪思执行服务发现 ，而无需任何其他依赖。如果您已经开始通过Spring Cloud利用Netflix Ribbon或Zuul，那么现在无需部署Eureka服务器就可以使用这两者！Spring Cloud Lattice当前是具有alpha质量的预览版本。

从Cloud Foundry和Spring团队之间不断加深的协作中，您将看到这种集成只是其中的许多。例如，Spring XD团队一直在努力构建一个服务提供者接口，该接口允许XD将Lattice视为运行时层，并将流模块部署为容器。

我们的目标是为您（企业Java开发人员）提供理想的本地开发和云操作环境，您需要利用它们来利用云原生应用程序的功能。我们期待在2015年9月的SpringOne 2GX上向您展示更多伟大的事物。敬请关注！
