对Java 7（甚至Java 6）的持续支持

工程
于尔根·霍勒（Juergen Hoeller）
2015年4月1日

您可能已经听说过Java 7的公共更新即将结束，而Java 7更新80的发布将在几天之内完成。请参阅以下Oracle页面以获取参考：

https://blogs.oracle.com/java-platform-group/entry/future_updates_of_java_7
http://www.oracle.com/technetwork/java/eol-135779.html

让我们澄清一下它的真正含义： 公开更新的结束，绝不是生命的终结（EOL）。通过上面的第二个链接，向下滚动到“ Oracle Java SE支持路线图”部分，您将找到实际的停产日期：Java 6为2018年12月，Java 7为2022年7月。 是的，从现在起Java 6还要再花3.5年，甚至还要再花7年（！）对于Java 7。

IBM的EOL时间框架是相似的，尽管在那里预计会有更长的Java 6支持时间轴。通常值得注意的是，Oracle的任何更新结束声明对于IBM用户而言都是完全不相关的： IBM JDK是一种独立产品，近年来恰好基于同一开放源代码项目（OpenJDK）。

因此，请放心，如果您最近发现有关Java 7停止使用的任何声明，请多加盐分。公开更新的结束只是Oracle开始收取维护版本费用的时间点。 Java 7还有7年的历史了。

如果您想了解Spring的观点，我们设计了Spring Framework 4.x以在同一代框架中支持Java 6、7和8。这是一个涵盖三代Java的统一编程模型，使您可以按照自己的进度进行升级，同时最大程度地利用当前使用的JDK！

提示是，在JDK 8上使用时，Spring Framework 4.x看起来像是一个基于Java 8的框架。使用相同的框架版本和框架jar，您还将获得有关JDK 6和7的良好体验： 您将仅受Java语言和API级别的限制，但在您触手可及的地方找到相同的2015年代框架功能。

请继续关注有关我们如何在代码库中支持多代JDK的后续博客文章……

于尔根

PS：请注意，只有Java 7和8支持Spring IO平台 ，包括其范围广泛的依赖关系。同时，我们的许多Spring产品组合项目仍然与Java 6…和基于JDK 6的应用程序服务器兼容，例如WebSphere 7.0 / 8.0。
