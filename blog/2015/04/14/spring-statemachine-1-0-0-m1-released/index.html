Spring Statemachine 1.0.0。M1发布

发布
珍妮·瓦尔基亚拉赫蒂
2015年4月14日

我们很高兴地宣布Spring Statemachine 1.0.0的第一个里程碑版本。M1 。此博客还充当此新项目的介绍。

虽然写一篇博客文章来介绍新闻界热切关注的最新尖端技术会很不错，但是有限状态机的时间比这篇文章的读者要长（祖父，您现在可以放下手）。有先后有来的技术，然后有这些基本技术之所以存在，仅仅是因为它们被证明可以工作。

每个程序员很可能已经在其应用程序中使用了某种状态的概念。当应用程序基于该布尔标志做出决策时，拥有一个简单的布尔标志本身已经是一个简单的状态机。然后，程序员会抛出更多的标志，枚举和一堆if / else / break结构，这在简单的用例中就可以了。

烘焙状态和应用程序逻辑一起存在的问题是，随着时间的流逝，当应用程序变得更复杂时，代码通常开始看起来像是一盘满是意大利面条的盘子，并且每当程序员更改一个部分时，其他一些部分就会损坏。

Spring Statemachine是一个框架，供应用程序开发人员在Spring应用程序中使用Statemachine概念，并旨在提供以下功能：

  • 易于使用的单层状态机，用于简单的用例。
  • 分层状态机结构可简化复杂的状态配置。
  • 状态机区域提供甚至更复杂的状态配置。
  • 触发器，过渡，防护和操作的使用。
  • 键入安全配置适配器。
  • 状态机事件监听器。
  • Spring IOC集成，将bean与状态机关联。

在可能异步发生的应用程序中，正确的状态处理非常重要。这在容易遇到并发问题的云环境中尤其重要。Spring Statemachine是状态处理的另一种形式，即Spring Cloud Cluster项目旨在提供其功能，例如分布式锁定和领导者选举 。

项目页面包含一个简单的示例，以了解Spring Statemachine的工作原理。我们还制作了一些样本 ，这些样本在我们的参考文档中有更详细的说明。这些示例与Spring Boot打包在一起，并使用Spring Shell来启动用于与状态机进行交互的交互式Shell。玩游戏的好方法，无需重新编译即可查看不同的行为。

我们很乐意通过参加Spring Statemachine Project或在GitHub上简单地发布问题或功能请求来回听人们的想法。
