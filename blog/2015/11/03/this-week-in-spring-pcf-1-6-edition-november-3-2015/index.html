春季本周（PCF 1.6版！）-2015年11月3日

工程
乔什·朗（Josh Long）
2015年11月3日

欢迎来到春季本周的另一个装置！本周，我将在保加利亚的索非亚参加我最喜欢的节目之一，即史诗般的Java2Days活动 ！

上周看到了发布（_finally！_） Pivotal Cloud Foundry 1.6 ，其中包含超过一年半的高度期望功能和繁重工作。新版本包括，在许多其他事情 ，对平台管理GitLab，JFrog Artifactory的，和CloudBees的詹金斯CI支持;支持微服务基础架构，包括Spring Cloud Config Server和Spring Cloud Eureka；并支持Docker容器映像和.NET应用程序；并支持在Microsoft Azure上运行 。该发行版包含各种功能，这些功能针对将软件连续安全地交付到生产环境进行了优化，我竭诚鼓励您针对您的环境进行研究。PCF基于Cloud Foundry 。Cloud Foundry是一种平台即服务，可支持服务和集成，以可预测的安全方式支持开发和部署应用程序。它是高性能软件组织及其美德的缩影。

和往常一样，我们有很多内容需要介绍，让我们开始吧！

  • InfoQ还很好地编写了PCF 1.6中的一些新功能
  • 然后Pieter Humphrey和Matt Stine提供了Pivotal Cloud Foundry中一些特定于Spring Cloud Services的支持的演练
  • 伊恩·安德鲁斯（Ian Andrews） 通过Pivotal Cloud Foundry 1.6的公告博客为舞台做准备
  • Spring Security主管Rob Winch刚刚宣布了Spring Security 3.2.9。发布和Spring Security 4.0.3。RELEASE ，均包含较小的改进和安全修复程序
  • Spring IO Platform负责人Andy Wilkinson 刚刚宣布了Spring IO Platform 2.0.0。RC1
  • Spring Ninja Greg Turnquist 在将Spring Data REST与React.js结合使用的系列文章中的史诗般的第五篇文章探讨了最新一期的安全性。
  • Spring Cloud联合创始人Spencer Gibb不久前在发布通知时发表了一篇不错的文章，以便当Spring Cloud管理的Git存储库中发生某些更改时，可以在各种配置客户端中重新加载属性。
  • 我们的朋友Jakub Kubrynski在2014年撰写的这个博客很好地解释了如何在Spring Boot应用程序中包含New Relic 。去年我们看到了这篇文章，但是本周有人问我类似的问题，值得人们注意。
  • Ashish Datta有一篇关于在Spring Security中使用自定义授权令牌的不错的文章。我可能会补充说，您应该考虑使用Spring Security OAuth作为此基本机制，并且，如果您决定在OAuth已经提供的更多功能上进行分层，则所有这些都已经内置并且与前言兼容。
  • Pivotal自己的拥护者Mark Heckler 在最近加入Pivotal的第一印象中就建立了一个不错的博客
  • 我的Pivotal Labs朋友雅克·切斯特（Jacques Chester）帮助我找到了一个关于从2012年开始持续交付流程中使用Pivotal Tracker的出色博客 。这不是特定于Spring的，但我认为这对于管理复杂项目的人非常有帮助，这绝对是我将在我即将出版的书《 Cloud Native Java》中处理的内容。
