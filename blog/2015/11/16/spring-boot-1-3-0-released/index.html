Spring Boot 1.3.0发布

发布
菲尔·韦伯
2015年11月16日

我很荣幸地宣布，Spring Boot团队的一个代表以及所有做出贡献的人，已经发布了Spring Boot 1.3.0，现在可以从repo.spring.io ， Maven Central和Bintray中获得 。该发行版增加了许多新功能和改进，并以最新版本的Spring Framework为基础。有关完整的升级说明和“新功能和值得注意的功能”，请参阅发行说明 。

以下是v1.3的一些亮点：

开发者工具

一个新的spring-boot-devtools添加了模块，旨在改善开发时间体验。该模块提供：

  • 明智的属性默认值（例如，禁用模板缓存）
  • 自动应用程序重启
  • LiveReload支持
  • 远程开发支持（包括通过HTTP隧道的远程更新和远程调试）。
  • 重启后的持久HTTP会话

如果您有10分钟的空闲时间，请看以下简短的介绍性视频 。

缓存自动配置

现在，我们为EhCache，Hazelcast，Infinispan，JCache（JSR 107）实现，Redis和Guava提供自动配置。

此外，还支持基于简单Map的内存中缓存

完全可执行的JAR和服务支持

现在，Maven和Gradle插件可以为Linux / Unix生成完全可执行的存档，只需键入以下内容即可运行：

$ ./myapp.jar

更好的是， init.d要么systemd服务。要安装为init.d服务只是创建一个符号链接：

$ sudo link -s /var/myapp/myapp.jar /etc/init.d/myapp

彩色横幅

支持@ WebServlet，@ WebFilter和@WebListener批注

使用嵌入式Servlet容器时，自动注册@WebServlet ， @WebFilter和@WebListener现在可以使用以下方式启用带注释的类@ServletComponentScan 。

其他自动配置

现在提供了开箱即用的自动配置，用于：

  • 卡桑德拉
  • OAuth2
  • Spring Session
  • OO
  • SendGrid
  • 阿耳emi弥斯

执行器指标

指标支持已扩展为支持导出和聚合。此外，特定于Java 8 GaugeService和CounterService现在提供（并在可能时使用）实现方式，以提高性能。

更新端点和健康指标

现在，我们包括/logfile ， /flyway和/liquibase执行器端点以及对Elasticsearch，Email和JMS运行状况指示器的支持。

其他变化

发行说明中还记录了许多其他更改和改进。您还可以找到我们计划在下一版本中删除的不推荐使用的类和方法的列表。
