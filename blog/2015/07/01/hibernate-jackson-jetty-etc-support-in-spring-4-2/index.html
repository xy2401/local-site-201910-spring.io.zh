在Spring 4.2中支持Hibernate，Jackson，Jetty等

发布
于尔根·霍勒（Juergen Hoeller）
2015年7月1日

众所周知，Spring积极支持最新的常见开源项目版本，例如Hibernate和Jackson ，还支持常见的服务器引擎，例如Tomcat和Jetty 。我们通常以向后兼容的方式执行此操作，同时通过反射适应或通过单独的支持包同时支持较旧的版本。这允许应用程序有选择地决定升级，例如在保留现有Hibernate 3投资的同时升级到最新的Spring和Jackson版本。

随着即将发布的Spring Framework 4.2，我们正借此机会支持相当多的新开源项目版本列表，其中包括一些相当主要的版本：

  • 休眠ORM 5.0
  • 休眠验证器5.2
  • Undertow 1.2 / WildFly 9
  • 杰克逊2.6
  • 码头9.3
  • 反应堆2.0
  • SockJS 1.0最终版
  • Moneta 1.0（JSR-354货币和货币参考实现）

虽然上述支持已经在Spring Framework 4.2 RC中提供，但我们正在努力的最终目标当然是将4.2 GA计划在7月30日进行 。
在这一点上，我们热切地等待将Hibernate ORM 5.0和Hibernate Validator 5.2转换为GA（它们当前都在RC1上），以及WildFly 9.0（当前在RC2上）和Jackson 2.6（当前在RC3上）。时间紧迫…我们将于7月30日发布自己的4.2 GA，以继续支持最新的候选版本，并在必要时将所有剩余的GA支持扩展到4.2.1中。

如果您想尝试使用某些当前发行版的Spring，请让我们知道它的进展。现在是向Spring Framework 4.2 GA提供此类反馈的绝佳时机！

更新：WildFly 9.0现在是最终版本。如果您有机会在其上测试Spring Framework 4.2 RC2，请告诉我们4.2 RC3的发布时间（7月15日）！同时也提供了Hibernate ORM 5.0 RC2，与此同时我们也提供了4.2 RC3。

PS：
请注意，即使使用Spring Framework 4.2，您当然也可以继续使用Hibernate ORM 3.6+和Hibernate Validator 4.3+。特别是迁移到Hibernate ORM 5.0可能会影响您的设置，因此我们仅建议您在应用程序的主要版本中进行推荐，而Spring Framework 4.2本身被设计为直接升级路径，对现有代码没有影响因此，立即向所有用户推荐。
