Spring Framework 4.2正式发布

发布
于尔根·霍勒（Juergen Hoeller）
2015年7月31日

尊敬的Spring社区，

我很高兴地宣布，现在可以从repo.spring.io以及Maven Central总体上获得Spring Framework 4.2！这是4.x产品线中的一项功能发行版，与Java 6和7以及Java 8兼容，着重于核心改进和现代Web功能：

  • Java 8默认方法上的注释检测（例如， @Bean ）
  • 基于注释的应用程序事件（ @EventListener ）
  • 对注释属性别名的一流支持（ @AliasFor ）
  • 用于直接字段绑定的完整嵌套路径处理
  • JSR-354货币和货币的数据绑定和转换
  • 与Hibernate ORM 5.0集成（本机和通过JPA）
  • 通过JSR-223（JRuby，JavaScript）的基于标准的bean脚本
  • 基于JSR-223的Web视图（重点是Nashorn上的JavaScript）
  • 对CORS和声明性HTTP缓存的丰富支持
  • 对HTTP流和服务器发送事件的一流支持
  • CompletableFuture用于处理程序方法和@Async方法
  • 支持杰克逊的@JsonView在STOMP端点方法上
  • 通过TCP和WebSocket通道使用的STOMP客户端
  • MockMvc HtmlUnit集成可轻松对网页进行本地测试
  • 集成测试也可以使用JUnit规则执行
  • 与往常一样，在细节上还有许多进一步的改进。

请参阅参考文档中的Spring Framework 4.2中的新功能和增强功能，以获取更详细的概述。请继续关注基于4.2的Spring Boot 1.3 ，预计将在9月份发布SpringOne候选版本。

请注意，建议所有4.x用户升级Spring Framework 4.2，并立即取代4.1行。最终的4.1.x维护版本预计在12月发布；对于任何剩余的问题，您此时必须升级到4.2.x。

我们的下一个功能版本将是Spring Framework 4.3 ，预计将于2016年3月发布。这将是一般Spring 4系统要求（Java 6 +，Servlet 2.5+）中的最后一代，并准备将4.3.x的支持寿命延长至2019年。

干杯，
于尔根
