Spring Roo 2.0.0。M1重构附加组件，协作结构

发布
彼得·汉弗莱（Pieter Humphrey）
2015年7月20日

我很荣幸代表DISID Corporation的Spring Roo团队宣布Spring Roo 2.0.0。M1已经发布！

Spring Roo 2.0的第一个发行版是Spring Roo项目的大型重构。我们已将Roo移至一个更小，更易于维护的项目：

  • OSGi容器已升级到OSGi R5。
  • Spring Roo定义了其运行时。Roo运行时包含执行插件所需的所有内容，并且不包含任何插件。通过不包含任何附加组件，可以更轻松地保持Roo运行时更加稳定和向后兼容。
  • 今天，Spring Roo集中在Spring技术上-像GWT和JSF这样的插件已经移到他们自己的项目中，以便由Roo社区维护。
  • 用户指南已迁移到ASCIIdoc。

如果您的组织希望通过维护这些附加组件为Roo项目做出贡献，请联系：disid.com上的springroo。要了解更多信息，请查看Spring Roo社区插件的源代码。

此版本还提高了可扩展性，并增加了围绕Spring Roo项目进行协作的潜力：

  • Roo市场已创建为共享，查找和跟踪第三方插件的地方
  • Roo提供了一种将一组插件打包和分发在一起的新方法：“ Roo Addon Suite”。Roo Addon Suite基于OSGi R5子系统，可提供真正方便的部署模型，而不会损害Roo的模块化。

与往常一样，新版本的Spring Roo包含社区报告的一些错误修复和改进。

有关新Spring Roo版本中所做更改的完整详细信息，请参阅Spring Roo 2.0.0。M1发行说明

感谢我们所有出色的用户，他们提出了有用的错误报告和建议。目前，我们正在开发Spring Roo 2.0.0。M2将包括以下改进：

  • 更新代码生成以使用最新版本：Spring Framework 4，从Spring Framework 依赖关系转移到Spring IO Platform
  • 用户管理的POM [ ROO-3465 ]，[ ROO-3478 ]
  • 新版Web UI的第一版[ ROO-3629 ]

此外， Spring Tool Suite （STS）团队正在努力发布STS 3.7.0，其中将包括针对Roo 2.0.0的Spring IDE Roo支持。如果要以前安装更新的Roo组件，则可以通过Spring IDE每晚更新站点手动进行安装。
