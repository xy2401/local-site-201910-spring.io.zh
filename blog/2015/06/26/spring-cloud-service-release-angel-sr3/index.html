Spring Cloud服务发布：Angel。SR3

发布
戴夫·瑟
2015年6月26日

Spring Cloud的最新版本是“ Angel。SR3”，并且现在可以从常规存储库中使用它（请注意新的命名约定，下面有更多详细信息）。现在大多数组件的版本为1.0.2。释放，但spring-cloud-netflix为1.0.3。发布。起跑器和cli以及1.0.3。发布，反映出他们对Netflix的依赖。天使。SR3是Angel的错误修正版本。鼓励SR2（以前称为1.0.2）和所有现有项目进行升级以加以利用。

有一些新功能，其中最大的是入门级父项目的新命名约定（请参见下文）。重点包括：

  • 支持git标签中的斜线（通过HTTP资源路径中的特殊字符序列）

  • Config Server启动时可选的git初始化

  • 配置客户端自行处理解密的其他选项，包括能够解密bootstrap.yml属性。

  • 新的RSA和AES算法的其他加密配置选项

  • 用户可以实施TextEncryptorLocator策略进行密钥旋转的策略，以动态查找密钥。

发布火车

Spring Cloud是一个总括项目，由原则上具有不同发布节奏的独立项目组成。为了管理项目组合，将发布BOM（物料清单），并带有对单个项目的精选依赖关系集（请参见下文）。发行列车使用名称而不是版本，以避免与子项目混淆。名称是按字母顺序排列的（因此您可以按时间顺序对其进行排序），带有伦敦地铁站的名称（“ Angel”是第一个发行版，“ Brixton”是第二个发行版）。当单个项目的点发布累积到一定数量时，或者如果其中一个关键错误需要所有人使用，则发布培训将推出名称以“。”结尾的“服务发布”。SRX”，其中“ X”是数字。

此发行版是此发行版的新事物（天使。SR3），但较旧的版本已重新发布（相同的工件，不同版本）为“ Angel，[RELEASE，SR1，SR2]”，分别对应于1.0.0、1.0.1和1.0.2。实际上，发行系列标签仅在一个工件中明确使用：“ spring-cloud-starter-parent”（所有其他工件具有与其父项目绑定的常规“ Spring”发行标签）。入门级父级可以用作物料清单以进行依赖关系管理或用作父级POM（仅适用于Maven）。使用具有客户端和POM的Maven的最新版本的配置客户端和eureka（更改工件ID以引入其他启动器）的示例：


  org.springframework.cloud
  spring-cloud-starter-parent
  Angel.SR3


  
    org.springframework.cloud
      spring-cloud-starter-config
  
  
    org.springframework.cloud
    spring-cloud-starter-eureka
  
  ...

或作为BOM（仅依赖管理）：


  
    org.springframework.cloud
    spring-cloud-starter-parent
    Angel.SR3
    pom
    import
  


  
    org.springframework.cloud
    spring-cloud-starter-config
  
  
    org.springframework.cloud
    spring-cloud-starter-eureka
  
  ...

或与gradle：

buildscript {
  dependencies {
    classpath "io.spring.gradle:dependency-management-plugin:0.4.0.RELEASE"
  }
}

apply plugin: "io.spring.dependency-management"

dependencyManagement {
  imports {
    mavenBom 'org.springframework.cloud:spring-cloud:Angel.SR3'
  }
}

dependencies {
    compile 'org.springframework.cloud:spring-cloud-starter-config'
    compile 'org.springframework.cloud:spring-cloud-starter-eureka'
    ...
}
