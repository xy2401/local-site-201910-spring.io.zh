春季指南移至Java 8

工程
格雷格·特恩奎斯特
2015年6月17日

也许您最近注意到了最近的一些文章？

  • Spring 5升级到Java 8+
  • 大量采用Spring 4和Java 8
  • http://start.spring.io/默认使用Java 8作为版本
  • Spring Boot的1.3.0。推荐新应用的M1参考文档从Java 8开始。

好的，后两篇不是文章，而是由Java 8的迅速采用推动的。Java 8已被开发社区FAST所采用。在Spring团队中，我们坚信将Java 8用于新应用程序。为了支持这一点并提高您自己的移动能力，我们刚刚将所有Spring Spring Started Guides Java 8更新。

拥挤的优势清单

  • Java SE的最新和最大稳定版本。 （功能，性能等。）
  • 如果需要，这些指南的内容很容易向后移植。甚至lambda都能很好地映射回单方法接口。
  • 在您的初学者项目中挑选一个指南，将使您开始扎实，现代化的设置。

在当今时代，随着对Java 7的公共更新的结束，开发人员似乎不太可能无法轻松地使用Java 8 SDK并随意旋转我们的代码。
