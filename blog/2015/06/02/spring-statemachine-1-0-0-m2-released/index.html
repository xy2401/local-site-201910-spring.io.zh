Spring Statemachine 1.0.0。M2发布

发布
珍妮·瓦尔基亚拉赫蒂
2015年6月2日

我们很高兴地宣布Spring Statemachine 1.0.0的第二个里程碑版本。M2。

有了这个里程碑，我们专注于区域和伪状态中的特征。区域为并行执行状态机提供了更复杂的功能，而伪状态将为处理初始，终止，历史，选择，派生和联接状态带来更多功能。

可能会问什么是区域以及它如何与状态机一起工作。状态机是正交区域由多个独立状态机组成的区域的特殊化。这是一个很难理解的概念，因此我们做了一个示例来演示此功能。

状态图上方显示了在工作流期间如何执行一组任务的通常概念。执行多个任务意味着，如果这些任务之一将失败，则除非修复了失败原因，否则无法再次执行任务。这些故障可以自动修复，也可以将状态机置于需要用户级别干预的手动状态。只有在此之后，状态机才能返回到其主要状态，在该状态下可以再次执行所有任务。

状态机中一个有用的概念是历史状态。该伪状态正在记录状态机的最后一个活动状态，返回该状态将恢复活动状态配置。其概念如下所示（我们知道，当电源恢复时，洗衣机能够从其最后状态继续运行），并且可以从参考文档中找到更详细的描述。

我们从这里去哪里？在下一个里程碑中，我们将解决以下功能：

  • 更好的稳定性和速度
  • 放宽对状态和事件使用枚举的要求。
  • 为状态机执行添加错误处理。
  • 为Spring SpEL的行动和后卫提供更好的支持。
  • 状态机的更好的编程实例化。
  • 使用配方简化状态机使用的新概念。上面的Tasks示例是一个很好的示例，我们可以为通常的状态机用例提供配方。
