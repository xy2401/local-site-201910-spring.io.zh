In a nutshell, the cache abstraction applies caching to methods, thus reducing the number of executions based on the information available in the cache. The caching logic is applied transparently: the method below will only be invoked if the specified ISBN is not already present in the books cache. Upon calling that method for a missing Book, the caches will be updated transparently so that a further call does not invoke the method again.

Spring Boot 1.3中的缓存自动配置

工程
斯特凡·尼科尔（StéphaneNicoll）
2015年6月15日

在过去的一年中，我们通过支持JSR-107（JCache）注释和更好的声明性模型来共享或外部化通用设置 ，大大改善了缓存抽象。在Spring Boot 1.3中，我们现在为其提供全面的自动配置。

简而言之，缓存抽象将缓存应用于方法，从而根据缓存中可用的信息减少执行次数。缓存逻辑是透明应用的：仅当books缓存中没有指定的ISBN时，才会调用以下方法。在为缺少的Book调用该方法后，缓存将透明更新，以便再次调用不会再次调用该方法。

@Cacheable("books")
public Book findBook(ISBN isbn) {...}

要实际启用对这些annotation的处理，基本设置需要设置@EnableCaching并定义CacheManager bean。

@SpringBootApplication
@EnableCaching
public class MyApp {
    @Bean
    public CacheManager cacheManager() { ...}

    ...
}

我们想尝试减少最后一步，因此您要做的就是在Spring Boot应用程序上添加@EnableCaching就可以了。从Spring Boot 1.3开始，这已经成为现实。M1！Spring Boot为任何符合JSR-107的提供商提供自动配置，还为EhCache，Hazelcast，Infinispan，Redis和Guava提供自动配置。如果使用ConcurrentHashMap都不可用，我们还将提供非常基本的支持。

Spring Boot正在检查您的项目以推断正确的默认值。例如，如果您在类路径的根目录下有一个ehcache.xml并且存在EhCache 2.x，那么我们将使用该配置文件自动创建EhCacheCacheManager （除非另有指定）。我们还提供了对配置选项的广泛支持：只需抓住Spring Boot 1.3。M1并在IDE中键入spring.cache以列出可用选项。

如果底层提供程序支持，Spring Boot可以在启动时创建缓存。如果您使用的是JCache，它还可以为您提供javax.cache.CacheManager
