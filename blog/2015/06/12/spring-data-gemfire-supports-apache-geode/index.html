What is Apache Geode?

In a nutshell, Apache Geode is the open source core of Pivotal GemFire. Geode was recently accepted into the Apache incubator after being submitted by Pivotal to the Apache Software Foundation as part of the BDS open sourcing effort.

Technically, Apace Geode is an in-memory, distributed database (a.k.a. IMDG) enabling new as well as existing Spring/Java applications to operate at cloud scale with high availability and predictable latency without sacrificing consistency. Applications are able to transact and analyze Big Data in realtime to achieve meaningful and impactful business results.

Spring Data GemFire支持Apache Geode

发布
约翰·布鲁姆
2015年6月12日

我很高兴地宣布， Spring Data GemFire现在已支持Apache Geode 。

什么是Apache Geode ？

简而言之， Apache Geode是Pivotal GemFire的 开源核心。作为BDS开源工作的一部分，Peotal将Geode提交给Apache软件基金会之后，Geode最近被接纳为Apache孵化器 。

从技术上讲， Apace Geode是一个内存中的分布式数据库（又名IMDG），使新的以及现有的Spring / Java应用程序可以在云规模上以高可用性和可预测的延迟运行，而不会牺牲一致性。应用程序能够实时处理和分析大数据 ，以实现有意义且有意义的业务结果。

尽管这听起来可能很复杂，但是Spring Data GemFire兑现了Spring的承诺，即使开发人员能够使用Apache Geode或Pivotal GemFire有效地使用Spring全面而强大的编程模型来构建高度可扩展的应用程序，从而消除复杂性。

开始使用

要开始使用Apache Geode构建Spring应用程序，只需在项目Gradle构建文件或Maven POM中包括Spring Data GemFire依赖项，就像这样……

build.gradle
dependencies {
    compile "org.springframework.data:spring-data-gemfire:1.7.0.APACHE-GEODE-EA-SNAPSHOT"
}
pom.xml
<dependency>
    <groupId>org.springframework.data</groupId>
    <artifactId>spring-data-gemfire</artifactId>
    <version>1.7.0.APACHE-GEODE-EA-SNAPSHOT</version>
    <scope>compile</scope>
</dependency>

Spring Data GemFire工件在春季可用libs-snapshot储存库...

<repository>
  <id>spring-libs-snapshot</id>
  <name>Spring Maven libs-snapshot Repository</name>
  <url>https://repo.spring.io/libs-snapshot</url>
</repository>

1.7.0版与提供Apache Geode支持的SDG的第一个版本相吻合，并且包含SDG 1.7版中的所有最新开发。

APACHE-GEODE-EA-SNAPSHOT版本限定符将Apache Geode的SDG构建与Pivotal GemFire 1.7.0 -BUILD-SNAPSHOT的 SDG构建区分开来。

那么，当传统的Spring发行公告围绕里程碑，发行候选和最终GA时，为什么还要宣布“类似快照”的发行呢？

主要是因为Apache Geode还没有正式发布。但是更重要的是，我们希望给开发人员一个机会，使用Spring Data GemFire在基于Spring的应用程序中试用Apache Geode ，以便尽快获得反馈。

虽然具有Apache Geode支持的Spring Data GemFire版本足够稳定，可以用于开发目的，但尚未完全集成到该版本中。 master给出了Apache Geode和Pivotal GemFire之间的技术差异。本质上， Pivotal GemFire必须与Apache Geode融合以在同一个Spring Data GemFire分支上同时支持两者。

如果您熟悉使用Spring Data GemFire开发Pivotal GemFire应用程序，那么您已经知道如何使用Apache Geode 。如果没有， 春数据的GemFire 参考指南和实例开始的地方因为是一个很好的指南上spring.io 。特别是，请检出使用GemFire访问数据和使用GemFire 缓存数据 ，或使用REST访问GemFire数据 。

试试看。如果您有任何反馈意见，我欢迎您在JIRA和StackOverflow中提出您的想法和问题，以及通过在GitHub上提交PR进行任何贡献。

参与其中

除了使用Apache Geode构建Spring应用程序外，现在是加入Geode社区并为不断发展的Apache项目做出贡献的绝佳时机。您可以通过多种方式参与 。

结论

阿帕奇的Geode是一个ASF贡献显著给出的唯一拥有 举足轻重的GemFire在Apache中的Geode不可用的是连续查询（CQS）和多站点， 广域网复制 。为了更好地了解Apache Geode ，请查看“ 入门”页面。

如果您在该地区，则还可以通过在2015年6月24日（星期三）下午6点30分在加拿大安大略省多伦多市Pivotal Labs办公室的这次聚会中参加有关Apache Geode的会议来了解更多信息。

最后， Luke Shannon和我还将在今年的SpringOne上的“利用内存中的分布式数据网格构建高度可扩展的Spring应用程序”会议中讨论Apache Geode和Spring Data GemFire 。
