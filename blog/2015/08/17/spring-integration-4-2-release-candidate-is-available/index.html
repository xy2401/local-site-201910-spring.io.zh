Spring Integration 4.2版本候选可用

发布
加里·罗素
2015年8月17日

我们很高兴地宣布，Spring Integration 4.2（ 4.2.0.RC1 ）现在可以在春季里程碑资料库中使用 。

此版本的功能包括：

安全上下文传播

现在可以在整个环境中传播安全上下文ExecutorChannel和QueueChannel s。该框架负责在下游流程完成时清除继承的安全上下文。

STOMP客户端通道适配器

现在可以使用基于Spring Framework 4.2支持的STOMP客户端通道适配器（里程碑1中宣布的初步工作现在已完成）。

指标

MessageChannel ， MessageSource和MessageHandler现在可以独立于JMX启用指标。

文件扫描仪

通过对文件系统事件做出反应，而不是在每次轮询中扫描整个树，可以使用新的目录扫描程序来更有效地扫描目录层次结构。（需要Java 7）

事件通道适配器

事件通道适配器现在支持新的POJO ApplicationEvent Spring Framework 4.2中的支持。

工艺障碍

引入了一个新组件，允许暂停集成流，直到发生一些异步事件为止。（ 示例应用即将推出 ，我们等待RabbitMQ发布者确认后返回HTTP响应）。

上次修改的文件列表过滤器

轮询目录的一个常见问题是文件生产者是否“就地”创建文件。可以在仍在写入文件的同时拾取该文件。通常建议使用临时名称创建文件，并在完成后重命名。但是，如果无法做到这一点，则此新筛选器将阻止处理文件，直到一段时间未对其进行修改为止。

编解码器

引入了编解码器抽象（包括Kryo实现）作为Java序列化的替代方法。

JMS共享订阅

命名空间现在支持JMS 2.0共享订阅，从而简化了使用XML时的配置。

（S）FTP改进

入站通道适配器现在可以将表达式用于远程目录。

SOAP动作传播

现在，使用Spring Integration来实现Web服务代理更加容易。

以前的里程碑

关于里程碑1和里程碑2版本中已经宣布的功能的提醒：

  • 对框架中的JMX支持进行了重大改进，在启用JMX时提供了性能和其他改进
  • mongodb元数据存储
  • @SecuredChannel注解
  • 条件轮询器-可以根据当前轮询的结果（或在执行之前）动态调整轮询间隔
  • http入站网关中的跨源资源共享（CORS）支持
  • Zookeeper支持领导力，锁和元数据
  • CompletableFuture网关方法返回值
  • 聚合器改进
  • （S）FTP改进
  • 已经发现并纠正了一些性能瓶颈。
