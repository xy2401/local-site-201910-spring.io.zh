即将在2016年推出：Spring Framework 4.3和5.0

发布
于尔根·霍勒（Juergen Hoeller）
2015年8月3日

作为Spring Framework 4.2 GA公告的后续以及我先前关于Spring Framework 5系统需求的帖子，我想借此机会总结一下我们明年的核心框架计划：

Spring Framework 4.3

首先发布的是2016年第二季度的Spring Framework 4.3，作为4.x系列的最终功能发布。
如前所述，这仍将针对一般的Spring 4系统要求（Java 6 +，Servlet 2.5+）而设计，并为延长4.3.x的支持寿命做好准备，直到2019年，尤其是针对JDK 6和7的用户，以及应用程序部署到JDK 8上的旧版应用程序服务器。

我们4.3计划的关键部分是有选择地升级我们的第三方依赖项安排：例如，完全拥抱JUnit 4.12 ， Jackson 2.5+ API ，新的JasperReports Exporter API ……以使我们相应的集成类与子孙后代向前兼容这些库（通过避免使用不建议使用的类型和方法）。

当然，也有机会在核心配置模型中以及跨各种消息传递端点变体来微调最近的框架功能。在JIRA上查看当前的4.3待办事项列表以获取候选改进列表。确保通过对JIRA的评论和投票表达对特定问题重要性的意见！

Spring Framework 5.0

同时，我们计划在2016年第四季度推出5.0 GA版本的Spring Framework 5。关键一步是需要Java 8+，这使我们能够将Java 8语言级别应用于整个框架代码库。当前，我们可以在您的应用程序组件中自动适应许多Java 8构造，但目前还不能在核心框架代码库中自己使用它们。特别是，我们还不能在核心接口中公开它们。在这方面，5.0将是一个重要的推动者。

尽管最低要求是Java 8，但Spring Framework 5.0将从一开始就将在JDK 9上构建，为下一代JDK提供全面支持：包括新的HTTP客户端API ， 并发增强等。我们还将跟踪计划的JSR-330（Java依赖注入）修订版，并为EE 8级规范（如Servlet 4.0和JMS 2.1 ）提供早期支持（在即将到来的预览中，它是可行的）。

我们在JIRA上的5.0待办事项列表已经表明了各个关键步骤。这仍处于早期计划阶段，但是仍然可以在这里发表您的意见……
