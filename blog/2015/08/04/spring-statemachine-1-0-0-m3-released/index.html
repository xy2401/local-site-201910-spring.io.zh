Spring Statemachine 1.0.0。M3发布

工程
珍妮·瓦尔基亚拉赫蒂
2015年8月4日

我们很高兴地宣布Spring Statemachine 1.0.0的第三个里程碑版本。M3。

自上一个里程碑以来，发生了很多事情，到达这里花费了我们最初估计的时间。现在我们在这里，让我们对其进行破解，看看在此版本中我们拥有哪些新功能。

除了常见的错误修复以外，这里还列出了一些主要的新功能：

  • 分布式状态机
  • 持久化状态机上下文
  • 放松使用枚举作为状态和事件
  • 状态机的程序化实例化
  • 新配方模块
  • 新样本（持久性，动物园管理员，网络）

分布式状态机

Distributed state是一项令人兴奋的新功能，旨在跨越jvm的边界跨越状态机概念。让我们再想一想，如果单个jvm上运行着一个状态机，那么如果一个jvm上的状态更改可以自动在另一个jvm上的其他参与状态机上触发状态更改，那不是很好。

自然地，分布式状态的概念是相对复杂的主题，因此，在此里程碑版本中，我们提供此功能作为技术预览，以改变其玩法和使用体验。

我们确实有了一个新的抽象概念，即如何将任意状态机和jmv连接到分布式状态机集合中，以从本地状态机外部协调状态更改。此抽象的第一个实现基于Zookeeper 。我们围绕这个主题有两个样本， zookeeper和web样品。

在此之上，我们有一个相对不错的示例，您可以从我们的参考文档中或稍后在此博客文章中阅读有关它的更多信息。

持久状态机

社区命令要求保留状态机上下文，以简化如何使用简单状态机实例（即，用于更新数据库中的表）的概念。本质上，此功能还用于distributed state machine但也允许用户重置state machine状态，以便使用任意存储库更新而无需为每个操作构建新的状态机实例。构建新状态机实例的问题在于，实例化是一项相对繁重的操作，因此能够重置状态机将是光波操作。

此外，围绕此功能，我们有一个新interceptor该功能允许用户与外部系统进行通信，如果无法将某些内容持久存储到外部存储中，则停止状态转换逻辑。

用于构建状态机的构建器模式

尽管将枚举用作状态和事件类型是一个很好的概念，因为它为您提供了可能的组合的编译时保证，但它限制了您精确地编译时间。您无需进行任何更改即可构建状态机，该状态机可动态构建状态和事件。

放宽了使用的要求enums现在可以使用strings作为状态和事件。这项要求也是社区提出的多项要求之一，可能也是最佳功能要求之一。

现在，我们能够脱离枚举，这为实现真正builder pattern它允许按需构建状态机实例。我们仍然使用与注解配置模型相同的熟悉界面，但是无需在编译时进行设置，而是可以按需动态构建所有内容。

下面显示了此示例的简化示例，其中未配置转换：

StateMachine<String, String> buildMachine() throws Exception {
  Builder<String, String> builder = StateMachineBuilder.builder();
  builder.configureStates()
    .withStates()
      .initial("SI")
      .end("SF")
      .states(new HashSet<String>(Arrays.asList("S1","S2","S3","S4")));
  return builder.build();
}

菜谱

在某些春季项目中，我们开始使用“ recipe 。Zookeeper和Curator熟悉配方的概念，其中基本系统的正确使用可能会有些困难。人们有类似的用例，复制粘贴的代码成为一种常用的方式，这些常用的用例被转移到配方中。

我们只是将相同的概念纳入Spring Statemachine通过开始添加常见用例的配方。前两个食谱是tasks和persist 。

您可以从参考文档中找到有关这些的更多信息。

样品

样品始终是了解事物使用方法的基础。我们有三个新样本， zookeeper和web这是分布式状态机的样本， persist是一个示例，显示了如何通过状态机处理任意数据库更改。

Web示例实际上是一个稍微复杂的示例，因为我们添加了一个UI层，该UI层使用JS前端，并通过websocket与后端服务器进行通信以通知状态机状态更改。状态机正在内部运行Spring Boot应用程序和用户界面会根据状态机的更改自动更新。

我们从这里去哪里？目前，我们希望该版本是最后一个里程碑，而下一个版本应该是第一个版本。我们大部分功能都是完整的，并且在添加新的重要功能方面，我们将进入代码冻结状态。进入RC阶段的主题是修复更多错误，使内核尽可能稳定，尤其是使分布式状态稳定。
