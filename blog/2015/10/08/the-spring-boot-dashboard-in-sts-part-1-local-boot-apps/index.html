STS中的Spring Boot仪表板-第1部分：本地启动应用程序

工程
马丁·利珀特
十月08，2015

欢迎春季社区，

这是向您介绍STS中新的Spring Boot Dashboard的三个博客条目中的第一个。您将看到如何在IDE中使用Spring Boot仪表板，了解它提供的各种功能，以及在基于Spring Boot开发应用程序时如何帮助您。由于Spring Boot Dashboard是在最近发布的STS 3.7.1中引入的，因此您将需要该版本来跟随本博客系列。如果尚未安装该软件，请转到http://spring.io/tools/并获取其副本。

介绍

现在，Spring Boot是一项广泛采用的技术，可简化为企业实施Spring应用程序时的工作。特别适合创建基于微服务的应用程序，它改变了我们对小型且易于配置的Spring应用程序的思考方式。与Spring Cloud一起，它为真正的云原生应用程序打开了大门，这些应用程序可能由许多微服务组成。

我们架构思想中的这种巨大转变也必须反映在我们的IDE中。我们现有的大多数传统Java IDE不仅是在云原生之前的时代发明的，而且是在云预制之前，并且在某种程度上是在预分布式编程时代发明的。Spring Tool Suite及其基础Eclipse IDE也是如此。现在是时候将这些IDE推入云原生时代了。这是Spring Boot Dashboard的起点。它的重点是帮助您处理可能数量更多的基于Spring-Boot的微服务应用程序，并在IDE中处理这些项目时使您作为开发人员的生活变得更加轻松。

Spring Boot仪表板

Spring Boot Dashboard是从STS / Eclipse的一个附加视图开始的，它是从小规模逐步开始的。您可以通过按下主工具栏中的Spring Boot按钮来打开它。它打开一个与您的工作空间项目同步的简单视图，显示您的工作空间中那些属于Spring Boot项目的项目。

Boot Dashboard的主要目的是使您能够快速访问每天，每小时或更频繁地执行的操作。通过在启动仪表板中选择启动应用程序并在工具栏中按“运行”或“调试”操作，可以快速启动启动应用程序（以运行或调试模式）。没有比这更快的启动启动应用程序的方法。而且由于您可能要更改代码并希望重新启动应用程序，因此该操作使您可以重新启动启动应用程序（如果已运行）。更改代码，在启动仪表板中按“运行”按钮，启动应用程序将停止并为您重新启动。同样，一个简单的一键式操作。

引导仪表板中的这些操作（以及更多操作）的优点在于，它们可以同时对单个或多个项目进行操作。如果要同时启动或停止一堆启动应用程序（例如，启动一组协作服务），只需在仪表板上将它们全部选中，然后按“运行”即可。而已。

在项目中采用Spring Boot 1.3后，您会注意到Boot Dashboard中显示了其他东西。启动启动应用程序后，启动仪表板会直观地显示该应用程序正在启动-并区分其启动阶段（VM正在运行，但应用程序仍在初始化）与真正的运行状态并可供使用。进度图标表示应用程序仍在启动，一旦应用程序完全初始化并运行，就会显示绿色的“向上”图标。在显示“服务器启动”消息或类似内容之前，您无需观察日志输出。引导仪表板图标将通知您。

应用程序运行后，其监听的端口将显示在启动仪表板中-为方便起见。您不必再扫描日志输出即可了解应用程序正在使用哪个端口。仪表板会自动显示该信息。而且，如果您想跳至正在运行的启动应用程序的控制台输出，也可以快速执行此操作。

通常，不再需要了解正在运行的应用程序的端口-因为您不必自己打开该应用程序的浏览器选项卡。双击启动仪表板中的项目，它将为您打开一个新的浏览器选项卡-直接导航至应用程序的默认URL。

默认情况下，这将打开Eclipse / STS内部浏览器视图。如果您喜欢使用外部浏览器，则可以在首选项中进行设置。

如果此默认URL不是您默认要访问的URL，则可以对其进行自定义。打开属性视图，在启动仪表盘中选择项目，然后输入所需的默认URL扩展名。双击将打开该应用程序的默认URL及其自定义的扩展名。

如果不确定您的应用程序定义了什么URL扩展，可以在属性视图中选择“请求映射”选项卡。列出了正在运行的应用程序的所有请求映射-顶部是您自定义的映射，底部是列表中的库。双击URL扩展名将打开该扩展名的浏览器，双击代码指针可在编辑器中打开项目的相应文件，并跳至定义请求映射的行。

您的工作空间中拥有的微服务项目越多，您可能无法同时处理所有这些项目。您想关注一个子集。例如，在Eclipse / STS中，您可能已经创建了一个工作集。引导仪表板允许您使用任意数量的标签（同样在属性视图中）为项目添加标签。然后，可以在过滤器框（位于启动仪表板顶部）中使用这些标签，以减少启动仪表板中显示的项目数量。

外表
