Java EE 7采用的简要更新

新闻与活动
于尔根·霍勒（Juergen Hoeller）
2015年10月6日

提醒您：每当有人通过Java EE 7使用情况栏向您显示调查结果时，尤其是在谈论生产使用情况时，请仔细检查他们的实际要求。Hibernate 4.3的使用是否仅因为它是JPA 2.1（EE 7级别但独立运行）而被视为Java EE 7？Tomcat 8的使用是否仅因为它是Servlet 3.1而算作Java EE 7？当然，运行带有任何此类单独提供程序的Spring应用程序是否算作EE 7？

拿这样的统计数据来衡量：他们通常不要求“本机Java EE 7平台使用”，但最终会以这种方式出现。为什么可以安全地假设他们没有在生产中要求完全使用Java EE 7平台？好吧，请记住我六月起的博客帖子 ，该帖子在2015年底仍然有效。同时，唯一真实的消息是自从IBM在WebSphere的Liberty Profile中发布了对EE 7的支持以来，没有任何新闻。总而言之，对此类调查问题的回答者可能会基于“我正在使用该伞中的一个或多个规范”而不是“我正在使用其预期形式的完整平台”来回答。

FWIW，我们决定将Spring Framework 5系统要求不仅提高到JDK 8+，而且提高到JPA 2.1+和Bean Validation 1.1+。毕竟，到2016年第4季度（我们的春季5 GA目标），这些规格已经存在3.5年了，因此我们愿意在那时要求它们。而且，我们仍将支持Servlet 3.0+，因此混有JPA 2.1的EE 6基准服务器将是可以的。

为避免任何误解：自2013年以来，Spring 4.x一直提供对JPA 2.1和其他EE 7级别规范的全面支持，并与我们的JPA 2.0 / EE 6支持并列。上面只是指到2016年底为止需要Spring Framework 5.0之前的JPA 2.1+，使其与仍受JPA 2.0约束的基础服务器基础结构不兼容。
