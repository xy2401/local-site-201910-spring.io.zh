工程

2012年12月10日

为什么选择Spring Scala？

Spring Scala项目的目的只是为了使在Scala中使用Spring框架更加容易 。我们相信，有很多Spring用户想试用Scala，但又不想将他们对Spring的体验抛在脑后。这个项目是为那些人准备的。

发布

2012年12月6日

亲爱的春季社区，

我们很高兴地宣布发布适用于Android 1.0.1的Spring ！

Spring for Android是Spring Framework的扩展，旨在简化本机Android应用程序的开发。

此版本通过新的MappingJackson2HttpMessageConverter在RestTemplate中添加了对Jackson 2.x的支持。它还解决了RestTemplate和Android Jelly Bean的一些错误和兼容性问题。有关更多信息，请参见变更日志和参考手册 。

Spring工具套件和Groovy / Grails工具套件3.2.0。M1发布

发布
马丁·利珀特
2012年12月3日

亲爱的春季社区，

我很高兴宣布第一个里程碑版本3.2.0。Spring工具套件（STS）和Groovy / Grails工具套件（GGTS）的M1。

此里程碑版本的亮点包括：

  • 整体绩效改善
  • 更新至tc Server 2.8.1
  • Live Beans Graph功能的改进
  • 更新了Spring Integration 2.2
  • 使用AJDT的AspectJ项目的编译时间得到了改善（减少了！）

这两种工具套件均基于2012年9月发布的最新Eclipse Juno SR1。一旦可以使用Eclipse Juno SR2发行版上的性能工作，我们就将其更新为该版本。同时，我们建议在Eclipse 3.8.1之上使用STS和GGTS版本。

SpringOne 2GX 2012 Web Track：使用Spring 3，Javascript中的IOC设计REST-ful API

新闻与活动
彼得·汉弗莱（Pieter Humphrey）
2012年11月30日

 

使用Spring 3设计REST-ful API



在创建VAS的过程中，我们做了很多工作来设计有用的REST API。


REST通常是一个非常松散的主体集合，可以用多种方法来解释，因此本演讲将描述一个REST-ful API应该是什么样的更具体的想法。


此外，该API的实现是使用Spring 3中的许多新功能完成的，很好地展示了其提供的功能。










关于本·黑尔



Ben Hale是Springsource的高级软件工程师，也是SpringSource dm Server项目的核心开发人员。Ben专门从事中间件开发，使用OSGi和面向方面的编程等技术，并指导Spring和SpringSource产品组合中所有产品的构建和发布过程。


他的兴趣包括中间层体系结构以及有效的构建和发布管理策略。


在加入SpringSource之前，Ben在电信行业的大型企业管理应用程序的体系结构和开发中花费了数年的领导团队。
