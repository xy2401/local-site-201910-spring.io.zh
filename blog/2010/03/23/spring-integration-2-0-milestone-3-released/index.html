Spring Integration 2.0 Milestone 3发布

工程
马克·费舍尔
2010年3月23日

上周发布了Spring Integration 2.0的第三个开发里程碑（ 在此处下载）。

如果您阅读了Oleg的最新博客 ，您已经知道我们已经开始基于“贷款经纪人”示例进行新的参考示例实现，该示例在Gregor Hohpe和Bobby Woolf的“ 企业集成模式”书中扮演着重要角色。

我只想发布一个简短的博客，列出此版本中的其他一些新增功能和改进功能。您可能会在接下来的几周内看到涉及这些功能的更详细的博客文章。我包括一些指向文档，示例和测试代码的链接，以便您至少可以快速了解这些功能的发展方式。

TCP / UDP的入站和出站通道适配器

在这里阅读有关它们的信息 。

用于JMX的通道适配器

这组适配器为JMX通知（侦听或发布），MBean属性轮询和MBean操作调用提供支持。此外，除了提供JMX支持之外，我们现在还可以初步实现Control Bus （我们的贷款经纪人示例最终将得到扩展，以显示Control Bus的运行方式，如EIP第12章所述）。在这里阅读更多。

JDBC的入站通道适配器

文档正在编写中，但是您可以在测试用例中看到一些示例。我们还开始了基于JDBC的MessageStore实现的工作，可以在需要持久性的框架中的许多组件（例如聚合器）中使用它。MessageStore接口本身仍在不断发展，但我们正在尝试使其尽可能简单，因为将来可能会考虑许多不同的存储选项。

可配置类型转换

Spring Integration现在支持可配置的Spring 3.0 ConversionService实例。类型为ConversionService的名为“ integrationConversionService”的bean将用于处理程序方法调用（服务激活器，转换器，路由器，分离器等）。换句话说，如果传递的消息的负载类型为Foo，但是调用的方法期望使用Bar类型的实例，则只要您配置的ConversionService包含支持从Foo进行转换的Converter，它就可以自动转换。去吧。这是一些测试代码和相应的配置 。

入站Web服务样本

我们添加了一个新示例，演示了入站Web服务支持。与所有示例一样，它可以通过Maven进行构建，如发行版中随附的示例的README.txt中所述。

请继续关注更多博客文章。我们非常期待您在JIRA和/或论坛向2.0版本发布时的反馈！
