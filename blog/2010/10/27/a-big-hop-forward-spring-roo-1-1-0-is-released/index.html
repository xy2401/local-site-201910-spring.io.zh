大跃进：Spring Roo 1.1.0发布了！

工程
本·亚历克斯
2010年10月27日

经过十多个月的开发和近900项个人改进， Spring Roo 1.1.0已发布（ 在此处下载 ）！与Spring Roo 1.1.0 GA发行版同时， Google Web Toolkit ， SpringSource Tool Suite ， AspectJ和AJDT团队已经完成了对GA发行版的支持，因此您可以一起使用这些工具的最新版本。

我们在Spring Roo 1.1.0中引入了许多新功能，因此很难决定要突出显示的内容。尽管如此，让我们简要浏览一下为您的Java编程乐趣而添加的一些优点。

增量数据库逆向工程

现在可以对现有的关系数据库进行反向工程，并自动创建具有相应字段的Roo实体。但是，使用JPA工具长期以来不可能做到这一点吗？是的，一点没错。关键的区别是Roo的数据库逆向工程是增量式的 。这意味着Roo对数据库进行反向工程时，会将发现的所有字段放入类型间声明（ITD）中。这与Roo的其余部分保持一致，并使Roo可以轻松地进行逆向工程实体的往返维护。特别是，您可以重复重新检查数据库以识别任何更改，同时确保保留在.java源代码中编写的任何代码。Roo甚至会删除不再存在的实体（当然，除非您已要求Roo不这样做），Roo会自动处理复杂的情况，例如复合主键（完成标识符类的创建和维护）。最重要的是，Roo的逆向工程功能可轻松完成，而不会问您一个问题。这项新功能将Roo可用性经验带入了一项任务，该任务迄今为止一直很乏味，耗时且产生混乱的.java文件，这些文件无法提供增量刷新功能。

增量逆向工程是我们对Roo提出的最受欢迎的功能请求，赢得了大约129个社区投票 。在1.1.0开发周期中，它已经进行了大量测试，我们发现社区的反馈非常宝贵。我们甚至使人们成功地逆转了超过440个表的工程模式，并且我们对Roo进行了调整，以在这种情况下提供可靠的性能。

要生成一个新项目并对关系数据库进行逆向工程，只需要三个无问题的命令（所有这些命令都提供了Roo的常规制表符帮助您节省键入内容）

project --topLevelPackage com.foo.rootest
persistence setup --provider HIBERNATE --database POSTGRES --userName rootest --password rootest
database reverse engineer --package ~.domain --schema dbretest

当然，您可以根据需要多次重复“数据库反向工程”命令，Roo将仅使用对模式所做的更改来仔细地更新您的实体。如果重复执行该命令，则无需重复“ package”或“ schema”选项，因为Roo会记住初始命令中的选项。

对于Roo需要连接到数据库以进行自省的JDBC驱动程序而言，我们已经为Roo发行版的一些常见生产开源数据库（例如Postgres，MySql）提供了驱动程序。如果您有另一个数据库，只需键入“ script install-jdbc-drivers.roo ”并重新启动Roo即可访问Oracle，Microsoft SQL Server，HSQLDB，H2，Derby等。

Spring MVC增强功能

我们结合了您对Roo 1.0的反馈，并显着增强了Roo 1.1.0中可用的Spring MVC功能。例如，现在可以自动往返JSPX文件。这意味着您现在可以随时随地以任何方式编辑脚手架的JSPX文件，Roo会自动检测到您所做的更改。如果Roo出于某种原因需要编辑文件（例如，您已经向实体添加了新字段，删除了一个字段等），那么Roo将会自动安全地将其更改与您的任何更改合并。如果您对给定的HTML元素进行了任何编辑，则Roo始终会将您的更改保留在原处，并且它们将具有优先级-就像您期望的那样。

对JSPX往返支持的一个相关改进是使用标记库封装标记创建的更为优雅的用法。这意味着Roo 1.0中的200行JSPX文件现在在Roo 1.1.0中仅12行。标记的重复现已移至基于JSPX的标记中，使您可以轻松进行影响应用程序中所有页面的编辑。因此，您现在可以更轻松地维护，自动往返以及简洁，集中，整洁的JSPX页面！您可以在我们的参考指南的“ Web MVC加载项”部分中了解有关所有这些工作原理的更多信息。

Spring MVC支持的其他增强功能包括取消了URL重写过滤器，因为此功能现已被引入Spring本身。现在，可以使用“ web mvc install language ”命令从第三方加载项安装额外的本地化包。这一点特别有用，因为它允许外部开发人员维护本地化（我们甚至创建了一个加载项创建命令来简化这些本地化加载项的制作 ）。

Spring MVC支架现在还支持嵌入来自16个社交媒体站点的内容。使用新的“ web mvc embed ”命令就像从YouTube，Google Video，Vimeo，Viddler，Screenr，UStream，LiveStream，Flickr，Picasa，Google Presentation，Scribd，SlideShare，Google Maps，Google Wave，Twitter和Wikinvest。

对JSON MVC功能的这些广泛改进是自动JSON REST支持 。要查看实际效果，请运行“ script clinic.roo”命令，然后运行“ json all”命令，使用Web UI创建所有者，然后使用此命令列出所有所有者： curl -v -H ‘Accept: application/json’ -X GET http://localhost:8080/petclinic/owners/ 。

Google Web工具包（GWT）2.1

我们从2009年末开始与Google合作 ，我们的工程团队一直紧密合作，以提供一套集成的开发人员技术。在2010年5月的Google I / O第1天主题演讲中，我们展示了我们在Spring Roo和Google Web Toolkit互操作性方面的早期工作。我们将继续增强这种互操作性，现在您可以使用Roo来构建完整的GWT应用程序，这些应用程序使用最新的GWT 2.1功能，例如RequestFactory ， MVP框架 ， 数据绑定小部件等。

与这些功能相关的是SpringSource Tool Suite 2.5.0中新的Google插件支持。SpringSource Tool Suite不仅具有广泛的Spring Roo 1.1.0支持（例如嵌入式Roo Shell），而且还通过包含Google Eclipse插件为 Google Web Toolkit开发提供了现成的支持。

您可以使用我们的费用示例试用最新的GWT支持。您可以在Spring Roo下载的/ samples目录中找到它。或者您可以简单地创建项目并通过以下命令运行它

mkdir expenses
cd expenses
roo
roo> script expenses.roo
roo> quit
mvn gwt:run

Google App Engine（GAE）

另外，作为与Google的上述合作的一部分，我们确保可以构建Roo应用程序以将其部署到Google App Engine中 。GAE提供了一种可扩展的方法来构建应用程序，并包括免费使用配额和Google App Engine for Business之类的选项 。

SpringSource Tool Suite中也进行了增强，以更好地支持Roo创建的Google App Engine项目。例如，STS管理Data Nucleus 1增强程序插件的生命周期，这对于GAE持久性是必需的。所有这些都与对m2eclipse的内置支持集成在一起，这是在基于Eclipse的IDE（例如STS）中管理Roo项目的典型方式。

您可以使用expenses-gae.roo示例脚本试用最新的GAE支持 。再一次，您可以在Roo的/ samples分发目录中找到它。您将在即将发布的Roo 1.1.1中看到对GAE支持的进一步改进 。

Apache Solr搜索服务器

Apache Solr是基于Apache Lucene的搜索服务器。Solr允许您有一个专用的搜索服务器来为您的对象建立索引，并通过REST API（也支持JSONP）查询它们。

Roo 1.1.0添加了Solr集成，从而确保每当创建，更新或删除实体时，都会自动通知Solr搜索服务器。它还会向您的实体添加各种与Solr相关的便捷方法，例如search(String)该方法将自动向您的Solr服务器调度一个自由文本搜索查询，并找到匹配的实体。还通过Roo ITD向实体添加了其他类似的与搜索和索引管理相关的方法。

要了解如何设置Solr服务器并在基于Roo的应用程序中使用它，只需访问参考指南的Solr部分 。

JSON支持

由于许多原因， JavaScript Object Notation（JSON）是一种有吸引力的序列化格式，例如其优化的浏览器“评估”支持，简便的人类可读编码格式以及显着的跨平台库支持。

在Roo 1.1.0中，我们增加了对允许将任何Java对象轻松地与JSON表示形式相互转换的支持 。新方法，例如toJson() ， fromJson(String)通过Roo ITD自动管理与数组相关的JSON处理方法。只需在Java类中添加@RooJson批注，即可自动获得此支持。

可序列化的附件

大多数Java开发人员都会经历乏味的任务，即维护serialVersionUID您的java.io。可序列化的类。

使用Roo 1.1.0，您无需担心序列化警告。只需在您的Java类上弹出@RooSerializable，就不用管它了。Roo将为您实现Serializable，并代表您自动维护版本UID。

图书馆更新

当然，我们会保持最新的库版本，因此您基于Roo的项目始终可以访问最新功能和错误修复。现在，基于Roo的项目默认使用的新版本

  • JPA 2（以及对JPA实现的相应更新）
  • 春季框架
  • 春季安全
  • Spring Web Flow
  • Log4J
  • Apache ActiveMQ
  • AspectJ
  • GWT

易用性

我们一直在不断改进可用性，以使您的开发变得轻松有趣。我们最新的可用性增强功能包括

  • 控制台消息现在变得更有意义 ，例如，告诉您正在将哪些依赖项添加到pom.xml中
  • 现在，命令选项的大小写不区分大小写 ，从而确保您可以轻松键入（当然，如果使用TAB，大小写将始终是正确的）
  • 现在，我们尽可能使用来自Maven Central的依赖项标识符（不再像Roo 1.0那样使用企业捆绑存储库ID）
  • 现在，升级变得更加容易，因为Roo会自动将Roo注释JAR更新到当前Roo版本
  • 在Roo命令窗口（支持操作系统；我们建议使用任何Linux，Windows或OSX的iTerm Shell以获得最佳体验）顶部的Roo命令窗口顶部，有一个新的“ 闪存区域 ”，用于显示长期运行状态的状态更新。
  • 操作系统的窗口标题将自动显示Roo版本或项目标识符（目前Windows上尚不可用，但很快就会显示）
  • 现在记录与OSGi相关的消息，以便于调试
  • 现在，注释可以按照配置文件和Shell脚本约定以“ ＃ ”开头
  • 现在为“ 提示 ”主题提供TAB辅助
  • 现在，AspectJ ITD文件中提醒您不要对其进行编辑
  • Cygwin和本机Windows启动脚本的增强的Windows支持

Roo现在基于OSGi构建

在Roo 1.0和1.1.0之间的一个主要但几乎无法区分的变化是向OSGi基础的过渡。我们使用Apache Felix作为我们的OSGi框架，并使用服务组件运行时 （SCR）进行组件管理，并使用OSGi Bundle Repository （OBR）进行包解析。这些重要的变化发生了，因此我们可以确保Roo的附加基础结构将基于模块化的，经过验证的，可解决远程依赖项的类路径管理模型。诸如Eclipse之类的现代IDE也是基于OSGi构建的，因此这种用于工具模块化和可扩展性的方法已得到很好的建立。

与此更改相关，我们需要“包装” Roo本身需要运行的许多依赖项。因此，我们建立了一个包装模块来满足这一要求，同时还为Roo附加组件开发人员提供了该技术的示例。

那些对Roo的OSGi内部结构感兴趣的人可以通过键入“ osgi”并按TAB进行交互。 Roo现在包含许多新的OSGi命令。与普通Roo用户相反，大多数这些都是附加开发人员感兴趣的。您甚至可以通过键入“ osgi framework command ”，然后输入普通的Felix命令，直接将命令分发到Felix运行时。

附加组件创建，发布，RooBot和PGP签名安全性

我们添加OSGi支持的原因之一是，我们可以支持围绕Roo开发的第三方加载项社区。编写加载项是一回事，但社区还需要一种简单的方法来找到感兴趣的加载项并进行安装。您还需要一个可靠且可靠的安全模型，以便只有您信任的开发人员才能编写安装在计算机上的加载项。您还需要一种让插件开发人员能够在他们喜欢的任何地方开发和托管插件的方法，同时仍然为他们提供一些合理的默认值，以便他们可以快速获得结果。

为了帮助人们快速开始开发附件，我们在Roo中添加了附件创建器。只需使用新的“ addon create ”命令。该命令不仅会为您创建一个附加框架，还将为在Google Code上的开箱即用托管配置它，并确保在使用时自动更新项目的OSGi捆绑存储库 （OBR）文件部署目标。OBR文件甚至使用“ httppgp：//”作为协议，因此您的附件与我们基于分散式PGP的安全基础结构联系在一起。因此，您可以在几分钟之内从一个空目录转到正确承载的，经OBR列出的，与安全模型兼容的加载项。

当然，您可以将OBR存储库URL分配给您的朋友和粉丝，他们可以使用“ osgi obr url add”命令添加它。但是，还有更好的方法。只需将包含OBR存储库URL的电子邮件发送到s2-roobot@vmware.com即可 。然后，我们的RooBot服务器将访问您的OBR文件并定期对其进行监视以进行更改。然后， 它将使用有效的，与安全模型兼容的加载项更新http://spring-roo-repository.springsource.org上的roobot.xml文件。然后，所有Roo实例都将发现您的加载项，因为它们在运行时将下载roobot.xml的副本。

Roo用户然后只需键入“ addon install ”命令即可安装您的加载项。TAB协助甚至可以完成您的加载项名称。他们按下ENTER键后，您的OBR文件将被添加到Roo安装中，并尝试下载。但是，由于使用了httppgp：//协议，因此Roo仅在用户信任您的PGP密钥 （或如果他们信任所有人的情况下键入“ pgp自动信任”）时才允许下载完成。

Roo用户可以发现您的加载项的另一种方法是简单地键入您的加载项提供的命令。Roo会自动列出提供该命令的所有已知加载项 。还有一个“插件列表”命令可以帮助人们找到您的插件。

在Roo 1.1.0中，编写附加组件也变得更加容易。我们添加了新的“ Joey”基础架构和更好的抽象，以帮助您编辑通用文件 。

结论

如果您不熟悉Spring Roo，并且想利用所有这些功能，只需下载Roo并按照参考指南的介绍性章节进行操作即可。我们还为那些想要将其项目升级到新版本的现有用户准备了升级说明 。您可能还想考虑下载SpringSource Tool Suite 2.5.0 （STS），因为它具有许多Roo和Google特定的功能，这将使您的开发工作变得更加轻松。确实，如果您下载STS，则可以跳过Roo的下载，因为STS 2.5.0为方便起见嵌入了Roo 1.1.0。

下个月，我们将发布Roo 1.1.1。目前，我们正在完成许多改进，包括Roo 1.1.1，因此请稍后再回来查看。在Twitter上关注@SpringRoo和工程团队是最容易了解开发和新版本的方法。我们也鼓励您参加Spring Roo论坛，以获取有关Roo的建议和问题。在论坛上，您将获得友好的Roo社区的帮助，以及Roo和GWT团队的工程师。

我们希望您喜欢使用Spring Roo 1.1.0就像我们乐于使用它一样。
