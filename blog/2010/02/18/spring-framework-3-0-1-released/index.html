Spring Framework 3.0.1发布

工程
于尔根·霍勒（Juergen Hoeller）
2010年2月18日

在收集了宝贵的反馈意见两个月之后，我很高兴宣布第一个Spring 3.0维护版本- 解决了170多个已报告的问题 。从我们的下载页面获取它。

由于相当多的用户要求分发依赖项（作为通过Maven或Ivy抓取依赖项的替代方法），我们现在提供第三次下载：包含具有常见第三方jar文件的Ivy存储库。请注意，核心框架与依赖项是分开的。后者只是为了额外的便利而提供，并不构成框架分发的固有部分。您当然可以继续使用所选第三方库的任何受支持版本 。

在谈论第三方库版本时，Spring 3.0.1包括对两个重要的新第三方发行版的支持： Tiles 2.2 （最新发行版为2.2.1）和Hibernate 3.5 （最新发行版为3.5 CR1）。您可能分别继续使用Tiles 2.1和Hibernate 3.2 / 3.3。我们只是在积极跟踪这些库的新兴下一代，以便您可以选择随时升级。仅供参考，与Spring 3.0的JPA 2.0支持一致，Hibernate 3.5是实现JPA 2.0的第一代Hibernate 。

Spring 3.0.1引入了一些核心改进，以解决针对3.0 GA报告的限制：


  • 已对ApplicationListener检测进行了修改，以提高鲁棒性和一致性，例如，相对于代理和工厂方法，尤其是对于Spring 3.0通过使用泛型声明特定事件的支持。
  • 基于切入点的代理，以及EntityManager代理和@Transactional代理现在都可以完全序列化 。这完成了3.0 GA中的BeanFactory序列化功能，尤其是在Web应用程序环境中使用时。
  • 诸如FactoryBean和HttpMessageConverter之类的通用接口现在已经放宽了Class参数的声明，以便在实践中更方便地使用。（如果您使用泛型来实现这些接口，则在某些情况下可能必须调整源签名；二进制兼容性不应该受到影响。）
  • Spring的JdbcTemplate 具有充分利用varargs的重载查询方法 ，如先前从SimpleJdbcTemplate类所知。对于大多数查询用例，这会将标准JdbcTemplate类转变为同样方便的基于Java 5的委托。

我们建议升级到Spring 3.0.1，以获得最佳的Spring 3 API体验！

另外，3.0.1还为Spring的JSP标记库引入了一个附加功能：该标记允许评估SpEL（Spring EL）表达式并将其结果嵌入到通过Spring 3.0的新格式化系统正确格式化的JSP页面中。这基本上是JSTL的Spring变体，具有现成的集成功能。请注意最新的示例应用程序，这些示例演示了如何使用Spring 3.0的Web功能集-指日可待！

快速更新（2月19日）：由于OSGi清单的意外损坏，我们发布了3.0.1。更新（称为“ 3.0.1。出于OSGi命名约定的原因，请使用“ RELEASE-A”）。如果要在OSGi环境中使用3.0.1，请确保使用那些经过修订的工件。

PS：3月18日在慕尼黑举行的SpringSource S2G论坛（今年欧洲在S2G会议系列中的一部分）是一个很好的机会，可以了解有关Spring 3的最新功能以及整个SpringSource产品组合的最新版本。到时候那里见！
