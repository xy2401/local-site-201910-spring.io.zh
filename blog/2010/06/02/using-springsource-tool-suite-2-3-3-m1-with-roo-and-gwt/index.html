使用SpringSource Tool Suite 2.3.3。带有Roo和GWT的M1

工程
克里斯蒂安·杜普依斯（Christian Dupuis）
2010年6月2日

到目前为止，大多数人可能已经听说过有关Spring，Roo，STS和GWT的Google I / O公告。 Ben和Rod最近在各自的博客 文章中对此进行了介绍。如果您错过主题演讲，我强烈建议您在YouTube上观看录音以赶上进度（相关部分从录音的1:09:00开始）。

今天，我想提供一些详细的步骤和说明，说明如何使用Roo和STS创建第一个GWT应用程序。

安装

在启动Roo shell并开始键入命令之前，我们需要下载并安装所有必备组件。尽管构建可在云上运行的高级单页应用程序涉及很多组件，但实际上您只需要下载STS 2.3.3。适用于您的操作系统的M1软件包。STS安装包括Roo 1.1.0。M1，具有Spring Insight的tc Server Developer Edition（需要与Speed Tracer集成），Maven 2.2和对Google Eclipse插件（GPE）的访问。

  1. 下载STS 2.3.3。M1来自springsource.com 。
  2. 运行安装程序或解压缩下载的档案。请遵循安装说明 。
  3. 启动STS并调出仪表板（“帮助->仪表板”）。
  4. 从控制台的“扩展名”标签中安装“ Google Eclipse插件”和“ DataNucleus插件”。
  5. 出现提示时重新启动。

创建GWT脚手架应用程序

安装STS后，我们准备启动Roo并创建我们的应用程序。如果要查看在主题演讲中使用的应用程序，请执行以下步骤：

  1. 在操作系统命令提示符下，创建一个新的空目录并将其更改。
  2. 启动Roo 1.1.0。通过键入“ roo”并按Enter键来M1。查看启动时Roo在shell横幅中显示的版本号。它应显示为“ 1.1.0。M1 [rev 3a0b8a3]”。为方便起见，Roo已与STS一起安装在同一文件夹中。
  3. 输入“脚本开支.roo”。这将运行提供的Roo脚本并创建Ben在主题演讲期间使用的基本费用跟踪示例应用程序。
  4. 输入“ quit”退出Roo shell。

到目前为止，您已经可以正常使用Spring＆GWT 2.1应用程序。随意浏览Roo为您制作的代码。如果您想了解有关此GWT应用程序基础架构的更多信息，建议您使用Ray Ryan的Google I / O会话 。查看会话记录和可从I / O端获取的幻灯片。

运行应用程序

感谢Roo，您可以轻松地在本地运行生成的应用程序。如果您的系统上安装了Maven，则只需在操作系统命令提示符下键入“ mvn gwt：run”。只要确保您仍与创建项目时所在的目录相同即可。这将启动GWT开发模式，从中可以轻松在浏览器中加载GWT应用程序。为了使用GWT开发模式，您需要安装相关的浏览器插件。如果尚未安装插件，浏览器将提示您安装插件。

将项目导入STS

通过与Google的Google Eclipse插件（GPE）团队合作，我们增强了STS，以便您可以轻松地将创建的Roo项目导入到Eclipse环境中。STS将自动配置您的项目，以便它可以与内置的Maven和GWT支持一起使用，而无需离开您的IDE。

从STS使用“文件->导入…-> Maven->现有Maven项目”导入项目。这将自动配置所有必需的项目设置，安装类路径容器并启动Roo Shell。

从STS内部运行应用程序

使用Google的GWT工具，现在可以很容易地从STS中启动应用程序。这使您可以调试服务器端代码，但更有趣的是，您还可以调试前端GWT Java代码。从项目的上下文菜单中选择“调试为-> Web应用程序”。请选择“ ApplicationScaffold.html” html页面开始，然后单击“确定”。只需确认“ WAR目录选择”对话框即可接受默认设置。

现在，在“控制台”视图中，您可以看到嵌入式GWT开发模式的启动。此外，还将显示GPE的“开发模式”视图，使您可以轻松访问已启动的应用程序。请参见下面的屏幕截图。

此时，可以随意添加断点并使用Eclipse调试器来熟悉所生成的应用程序。

为了尝试Roo强大的往返支持，请打开“ Employee” Java源文件并添加一个新字段。例如，添加“私有字符串注释”字段并保存文件。请注意，Roo将如何立即更新GWT应用程序，以将新字段合并到UI和后端代码中。您可以从“ Roo Shell”视图中观察到这一点。注意，STS集成包括Roo编辑的文件的链接，因此您可以打开它们并查看更改。

如果此时仍在运行开发模式服务器，请打开“开发模式”视图，然后单击视图工具栏中的“刷新”图标。深吸一口气（大约2-3秒）后，返回浏览器并单击“刷新”。导航到“雇员”屏幕，并注意UI上新添加的字段。您可以通过在“评论”字段中创建新的员工来验证服务器代码是否已完全刷新（包括适当的持久性）。

部署到Google App Engine

我们已经收到了很多有关如何将支架式应用程序部署到Google App Engine的问题。以及它是否在GAE上运行。当然可以，可以在这里进行测试。

部署非常简单，可以从STS内或从命令行完成。在以下部分中，我将展示如何从命令行部署到GAE。

  1. 首先，您可能想从示例应用程序使用的内存内Hypersonic数据库切换到基于BigTable的Google数据存储。为此，请从命令行启动Roo shell，然后执行以下命令：“持久性设置--database GOOGLE_APP_ENGINE --provider DATANUCLEUS”。
  2. 接下来，我们需要指定GAE应用程序名称。可以通过编辑“ src / main / webapp / WEB-INF / appengine-web.xml”文件来完成。请将应用程序名称设置为您GAE帐户中的现有应用程序。或者，通过“ --applicationId”选项键入“持久性设置”命令时，您可能已经指定了应用程序名称。
  3. 部署到GAE现在只需运行“ mvn gwt：compile gae：deploy -DskipTests”（仅在Roo 1.1.0中才需要“ -DskipTests”）。M1。该问题已得到修复 ，对于Roo 1.1.0而言则不是必需的。M2）。

我希望这些指示和步骤会有所帮助，并会鼓励更多的人尝试Roo对GWT的支持。请记住，GWT，Roo和STS处于里程碑1阶段，因此您实际上可能遇到限制或意外问题。在这种情况下，请在Spring社区Roo或STS论坛上发布。我们，SpringSource和Google工程师很高兴为您提供帮助，并期待您的反馈。
