Spring Framework 3.0.3发布

工程
于尔根·霍勒（Juergen Hoeller）
2010年6月15日

经过数周的微调和社区反馈后，Spring Framework 3.0.3现已可用。此版本修复了针对Spring 3.0.2报告的一百多个小问题，尤其是在JSP标记库和Portlet会话处理以及ConversionService详细信息中。此版本再次赶上了最近的第三方版本：OpenJPA 2.0 final，Hibernate 3.5.2和JBoss 6.0.0 M3，现在与Spring 3结合完全支持所有这些版本。

请注意，与此同时，所有主要的持久性提供程序都发布了具有JPA 2.0支持的GA版本，甚至包括用于WebSphere 7的JPA 2.0功能包。显然，这表明JPA 2.0即将成为主流……如果您还没有尝试的话，可以尝试一下！当然，Spring 3很高兴与服务器提供的JPA 2.0 EntityManagerFactory（例如在WebSphere环境中）一起工作，同时还能够将嵌入式JPA 2.0的全部功能带到Tomcat和独立环境中。（与此同时，Spring 3也会在运行时遇到的情况（例如在Java EE 5服务器上）自动适应JPA 1.0提供程序。）

最后，请注意，由于那里大量许可的法律影响，我们不再提供依赖项下载。获得与Spring一起使用的第三方库的推荐方法是Maven / Ivy；您还可以下载所选的第三方发行版（例如Hibernate 3.5.2），然后从那里获取jar。请注意，除非您愿意，否则没有理由升级第三方库：最简单的解决方案是继续使用您知道和信任的版本。
