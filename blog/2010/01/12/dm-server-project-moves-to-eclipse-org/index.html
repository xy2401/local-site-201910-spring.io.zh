dm Server项目移至Eclipse.org

工程
阿德里安·科耶尔（Adrian Colyer）
2010年1月12日

今天，我们将发布DM服务器的2.0版。这代表了该项目以及整个OSGi企业应用程序开发的重要里程碑。我很高兴现在能够揭示dm Server历程的下一步：我们已经在Eclipse.org上提交了一份提案，以继续开发dm Server，这是Eclipse RT顶级项目的一部分。该项目的Eclipse昵称是Virgo。

快速链接：


  • Eclipse.org上的处女座提案文档

  • 处女座社区论坛


这对dm Server用户意味着什么？


迁移到Eclipse.org对于dm Server用户具有许多实际意义：

  • 项目托管，主页，论坛和下载将全部移至Eclipse.org基础结构

  • 该许可证将从当前（主要是GPL）许可证更改为Eclipse Public License（ EPL ）

  • 其他组织和社区成员更容易参与处女座的持续发展


许可证变更和Eclipse.org上的社区托管的结合为更广泛的用户和开发人员开放了代码库。

dm Server的后续发行版将从Eclipse.org开发和发行。

为什么SpringSource进行此更改？


dm服务器代表了大量的知识产权（IP），并且已经进行了2年以上的全职开发。为什么SpringSource会将这个项目移至Eclipse.org？

我们着眼于使模块化应用程序开发成为企业开发人员的现实。在过去的3-4年中，我们与许多苦苦挣扎于模块化问题的最终用户和客户进行了交谈，我们知道真正需要实用的解决方案。我们经常听到以下问题：


  • 我有大量由Spring管理的组件，如何最好地构造Spring配置以使其更易于管理？

  • 我如何最好地组织一个大型（“大型”的定义可能会有很大不同）开发团队，以有效地在大型项目上合作？

  • 如何在代码库的给定区域内最小化/隔离更改的影响？

  • 将更新部署到生产中的应用程序时，如何最大程度地减少更改的影响？


如今，dm服务器为基于OSGi和Spring动态模块 （现已标准化为OSGi蓝图服务 ）编程模型的模块化企业应用程序开发提供了先进的服务器平台 。Enterprise OSGi和dm Server取得了长足的进步。但是可以说，在企业应用程序开发中采用OSGi并非没有代价。像许多新技术一样，必须进行初始投资，该投资将随着时间的流逝而收回。哈尔·希尔德布兰德（Hal Hildebrand）在最近发表的有关OSGi价值主张的博客文章中很好地把握了当前情况。

企业OSGi和dm Server引起了很多兴趣和创新。在早期采用者中，这种兴趣是最强烈的，并且其要求与OSGi服务平台的动态模块化本质非常匹配。但是对于一个主流开发团队来说，他们只是想尽快构建企业应用程序，而又尽量避免麻烦，因此目前采用企业OSGi所带来的成本可能会超过短期收益 。在企业OSGi成为主流企业 应用程序开发的实际方法之前，需要解决这种情况。请注意，我在这里谈论的是企业应用程序开发；如果您正在编写基础结构软件，并且需要创建“无堆栈堆栈”（ Kirk Knoerschild ， James Governor ），则OSGi已经是事实上的方法，并且已由dm Server和相关的dm内核子项目完全支持。

在SpringSource，我们知道开源开发和社区参与可以在不断发展的简单实用的解决方案中发挥巨大作用，这些解决方案使技术能够从早期采用者过渡到主流用法。我们知道，因为这是我们成功采取了多次的途径。在Eclipse.org上创建Virgo项目时，我们试图加速dm Server和企业OSGi沿这条路径的发展。

为什么选择Eclipse.org？


dm服务器建立在Eclipse Equinox上并与之紧密集成，Eclipse Equinox是OSGi服务平台的参考实现。此外，最近在Eclipse.org上提出的Gemini项目将托管Virgo支持的许多关键企业OSGi规范的参考实现，包括SpringSource领导的Gemini Web Container和Gemini Blueprint Service项目。dm Server开发工具也全部基于Eclipse。作为Eclipse RT项目，Virgo将受益于与Eclipse.org上的OSGi社区的密切合作，多方参与对其不断发展的贡献的能力以及Eclipse用户群的反馈以帮助塑造未来的方向。

从Eclipse.org的角度来看，Eclipse Foundation的第一大战略目标已在最近的《执行董事报告》中向成员展示（ http://www.eclipse.org/membership/slides09q4.pdf ）旨在“将Eclipse运行时技术建立为领先的开源运行时平台 。”在Eclipse.org上创建处女座项目是实现该目标的重要一步。

在哪里可以找到更多？


处女座提案文档本身是查找贡献细节，它与现有Eclipse.org项目的关系以及相关开发工具的最新信息的最佳信息来源。处女座社区论坛提供了贡献和互动的机会。
