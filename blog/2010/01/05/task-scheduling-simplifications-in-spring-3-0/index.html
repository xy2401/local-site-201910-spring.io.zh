Spring 3.0中的任务计划简化

工程
马克·费舍尔
2010年1月5日

延续Keith和Chris提出的Spring 3.0“简化系列”，我想快速概述一下Spring 3.0支持的调度和任务执行中的简化。

我将逐步介绍一个基本的示例应用程序 ，您可以从spring-samples Subversion存储库中检出该应用程序 。它设计得尽可能简单，同时展示了Spring 3.0中注释驱动和基于XML的计划任务的方法。

让我们从注释驱动的方法开始。您可以通过AnnotationDemo中的main（）方法直接运行它。如果您看一看，您会发现它不过是Spring ApplicationContext的引导程序：


public static void main(String[] args) {
    new ClassPathXmlApplicationContext("config.xml", AnnotationDemo.class);
}

没有其他必要的原因是ApplicationContext包含一个“活动”组件，我们很快就会看到。由于该组件，main（）方法将不会退出。config.xml也是最小的，仅包含两个元素：


<context:component-scan base-package="org/springframework/samples/task/basic/annotation"/>

<task:annotation-driven/>

“ component-scan”元素指向包含“ beans”的包。其中有两个：ScheduledProcessor和AsyncWorker。我们将暂时看一下这些内容，但首先看一下“注释驱动”元素。那是Spring 3.0中的新功能，它驱动两个注释：@Scheduled和@Async。您可以分别使用“ scheduler”和“ executor”属性提供对Spring TaskScheduler和TaskExecutor的引用，但是对于本示例，我们将仅依赖默认值。

ScheduledProcessor在方法上包含@Scheduled批注，因此是我上面提到的“活动”组件。由于配置中存在“注释驱动”元素，因此该方法将在Spring TaskScheduler实例中注册，该实例将以30秒的固定延迟定期执行该方法。


@Service
public class ScheduledProcessor implements Processor {

    private final AtomicInteger counter = new AtomicInteger();

    @Autowired
    private Worker worker;

    @Scheduled(fixedDelay = 30000)
    public void process() {
        System.out.println("processing next 10 at " + new Date());
        for (int i = 0; i < 10; i++) {
            worker.work(counter.incrementAndGet());
        }
    }
}

如您在前面的代码摘录中所看到的，Worker在循环中由ScheduledProcessor调用。但是，AsyncWorker实现在其work（..）方法上包含@Async注释，由于配置中的“注释驱动”元素，该元素将被包装在代理中，以便该方法实际上由TaskExecutor调用实例。为了验证这一点，当前线程名称显示在该方法中。同样，为了澄清工作是同时执行的，调用sleep（..）来模拟耗时的工作。


@Component
public class AsyncWorker implements Worker {

    @Async
    public void work(int i) {
        String threadName = Thread.currentThread().getName(); 
        System.out.println("   " + threadName + " beginning work on " + i);
        try {
            Thread.sleep(5000); // simulates work
        }
        catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
        System.out.println("   " + threadName + " completed work on " + i);
    }
}

关于该输出，有一些注意事项。首先，一次处理10行将每30秒重复一次（由于@Scheduled）。其次，工作项由不同的线程并发处理（由于@Async）。最后一条“开始工作”消息和第一条“完成工作”消息之间应该有大约5秒钟的停顿。如果所有工人都在一个线程中运行，我们将看到连续的开始/完成对，而整个过程将花费大约50秒。当然，在博客条目中无法捕获时间方面，因此，您确实应该自己下载并运行示例以获取全部效果（可以将项目直接导入到SpringSource Tool Suite或具有Maven支持的另一个基于Eclipse的环境中） 。

我要显示的最后一件事是@Scheduled注释的基于XML的替代方法。该示例包括另一个类SimpleProcessor，该类的process（）方法不包含@Scheduled：


@Service
public class SimpleProcessor implements Processor {

    private final AtomicInteger counter = new AtomicInteger();

    public void process() {
        System.out.println("processing next 10 at " + new Date());
        for (int i = 0; i < 10; i++) {
            System.out.println("   processing " + counter.incrementAndGet());
        }
    }
}

XML仅比注释驱动的版本更为冗长，因为Spring 3.0现在提供了一个“任务”名称空间来保持配置的简洁。


<context:component-scan base-package="org/springframework/samples/task/basic/xml"/>

<task:scheduled-tasks>
    <task:scheduled ref="simpleProcessor" method="process" cron="3/10 * * * * ?"/>
</task:scheduled-tasks>

如果在XmlDemo中运行main（）方法，您将看到该进程每10秒执行一次。对于变体，此函数使用cron表达式而不是简单的固定延迟。结果，您会发现计时基于3秒的偏移量（：13，：23等），但是cron表达式当然比这更强大（我只是不想创建示例那只会在特定的日子或一天中的某些时间运行）。值得指出的是，Spring 3.0本身直接包含对基于cron的调度的支持。

请务必查看《 Spring 3.0 参考手册》中的“ 任务执行和计划”一章，以了解有关新TaskScheduler抽象和触发器策略的更多信息，这些策略为您在此处看到的内容提供了基础。该参考手册还讨论了由“任务”名称空间提供的其他元素，这些元素用于使用特定线程池设置来配置TaskScheduler和TaskExecutor实例。

我希望这篇文章对这些新功能有所帮助。请继续关注SpringSource团队博客，以获取更多与Spring 3.0相关的内容。
