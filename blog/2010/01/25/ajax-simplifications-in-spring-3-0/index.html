Spring 3.0中的Ajax简化

工程
基思·唐纳德
2010年1月25日

在上一篇文章中 ，我向您介绍了Spring 3中针对Web应用程序开发的一些增强功能。你们中的许多人都对关注Ajax远程处理的后续文章感兴趣。Spring 3在这方面提供了很多优势。继续阅读，我将带您逐步学习。

Spring和Ajax概述

出于本文的目的，当我说Ajax时，我是在谈论Web浏览器使用JavaScript与Web服务器异步通信的能力。在服务器端，Spring提供了用于定义Web服务（包括JavaScript客户端使用的服务）的编程模型。在客户端，这些天也没有人推出自己的Ajax框架。大多数使用已建立的JavaScript框架，例如jQuery或Dojo 。

对Ajax客户端的支持

在版本3之前，Spring尚未提供对Ajax远程处理的支持。但这并没有阻止我们的用户扩展Spring以获取它，或者自己集成其他选项。有些人使用了DWR，尤其是在JavaScript框架兴起之前。最近，以JSON作为数据交换格式的REST风格远程处理变得越来越流行，尤其是因为jQuery和co。 使它变得如此容易 。

现在Spring 3已经发布，作为Spring MVC的一部分，提供了对使用JSON进行Ajax远程处理的官方支持。这包括使用Spring MVC @Controller编程模型生成JSON响应和绑定JSON请求的支持。在本文中，我将集中于使用这种支持来实现几个Ajax用例。就像我的上一篇文章一样，我将通过逐步介绍您可以自己进行实验的示例应用程序来完成此操作。

MVC Ajax示例

mvc-ajax旨在说明Spring MVC的JSON支持。该项目在我们的spring-samples Subversion存储库中可用，可以用Maven构建，并且可以导入到STS / Eclipse中。 mvc-ajax与我上一篇文章中介绍的mvc-basic项目具有相同的结构。实际上，两者之间的Spring配置是相同的。

通过将项目部署到servlet容器并访问位于localhost：8080 / mvc-ajax的欢迎页面来开始审核。由于我使用STS，因此首先将项目导入IDE，然后将其部署到内置的Tomcat / tc-server实例中。

从服务器获取JSON

在欢迎页面上，激活“ Ajax @Controller Example”链接。您将看到一个用于创建新帐户的表格。当您从“名称”字段中跳出时，浏览器将询问服务器您刚才输入的名称是否可用。如果不是这样，将显示一条错误消息，并且在您输入可用名称之前，该表单将保持禁用状态。处理此问题的客户端JavaScript位于/WEB-INF/views/account/createForm.jsp中，如下所示：


$(document).ready(function() {
    // check name availability on focus lost
    $('#name').blur(function() {
        checkAvailability();
    });
});

function checkAvailability() {
    $.getJSON("account/availability", { name: $('#name').val() }, function(availability) {
        if (availability.available) {
            fieldValidated("name", { valid : true });
        } else {
            fieldValidated("name", { valid : false,
                message : $('#name').val() + " is not available, try " + availability.suggestions });
        }
    });
}

这里没有特定于Spring的内容，只有标准的jQuery JavaScript。

在服务器端，用于帐户/可用性资源的控制器是标准Java，带有一些Spring MVC注释：


@RequestMapping(value="/availability", method=RequestMethod.GET)
public @ResponseBody AvailabilityStatus getAvailability(@RequestParam String name) {
    for (Account a : accounts.values()) {
        if (a.getName().equals(name)) {
            return AvailabilityStatus.notAvailable(name);
        }
    }
    return AvailabilityStatus.available();
}

AvailabilityStatus是一个简单的Java值对象，具有两个属性：可用性标志，用于告知客户端用户名是否可用，以及一系列替代项，用于提示您所需的名称是否不可用。@ResponseBody注释指示Spring MVC将AvailabilityStatus序列化到客户端。Spring MVC自动序列化为JSON，因为客户端接受该内容类型。

在幕后 ，Spring MVC委托给HttpMessageConverter来执行序列化。在这种情况下，Spring MVC会调用在Jackson JSON处理器上构建的MappingJacksonHttpMessageConverter 。当您在类路径中使用Jackson时使用mvc：annotation驱动的配置元素时，将自动启用此实现。

太酷了吧？尝试创建一个帐户，然后创建另一个具有相同名称的帐户。您应该看到一条错误消息，提示您使用备用名称。打开Firefox的Firebug或Safari的Web检查器，以调试异步交互。

将JSON发布到服务器

Spring MVC还提供了将JSON发送到服务器的支持。我发现这种需求不那么普遍，仅因为发布表单参数通常就足够了。尽管如此， JSON提供了一种灵活的数据交换格式，更丰富的JavaScript客户端可以方便地使用它。在这些情况下，将JSON映射到服务器端Java对象进行处理的功能可能是有用的功能。

在该示例中，JavaScript事件处理程序拦截表单提交事件，并将表单数据作为JSON发布。服务器返回新创建的帐户的ID，然后将其用于显示模式确认对话框：


$("#account").submit(function() {
    var account = $(this).serializeObject();
    $.postJSON("account", account, function(data) {
        $("#assignedId").val(data.id);
        showPopup();
    });
    return false;
});

在服务器端，Controller是带有Spring MVC注释的更标准的Java：


@RequestMapping(method=RequestMethod.POST)
public @ResponseBody Map create(@RequestBody Account account, HttpServletResponse response) {
    Set> failures = validator.validate(account);
    if (!failures.isEmpty()) {
        response.setStatus(HttpServletResponse.SC_BAD_REQUEST);
        return validationMessages(failures);
    } else {
        accounts.put(account.assignId(), account);
        return Collections.singletonMap("id", account.getId());
    }
}

在这里，@RequestBody注释指示Spring MVC将HTTP请求的主体映射到Account对象。Spring MVC知道可以从JSON进行映射，因为客户端将请求的内容类型设置为application / json。

create方法还将验证Account对象。如果存在验证错误，则返回带有错误消息的HTTP 400，否则返回带有分配的帐户ID的HTTP 200。

摘要

Spring 3通过Spring MVC模块提供了对JSON的一流Ajax支持。这包括使用Spring MVC @Controller编程模型结合Jackson JSON处理器来支持生成JSON响应和绑定JSON请求。在本文中，我向您展示了此支持的工作方式。我希望您觉得这篇文章有用，并希望听到更多将Spring 3用于您自己的应用程序的经验！
