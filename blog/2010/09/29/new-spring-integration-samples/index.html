新的Spring集成示例

工程
奥列格·朱拉库斯基（Oleg Zhurakousky）
2010年9月29日

根据您对更多Spring Integration样本和更多可用结构的压倒性要求，我们很高兴宣布以下消息：

从当前版本的Spring Integration开始，这些示例将不再包含在Spring Integration发行版中。取而代之的是，我们已切换到一种更简单的协作模型，该模型应促进更好的社区参与和社区贡献。现在，样本具有专用的Git SCM存储库和专用的JIRA问题跟踪系统。样本开发也将具有其自己的生命周期，该生命周期不依赖于框架版本的生命周期，尽管出于兼容性方面的考虑，仍将为每个主要版本标记存储库。对社区的最大好处是我们现在可以添加更多示例，并立即将它们提供给您，而无需等待框架的发布。拥有自己的JIRA而不绑定到实际框架也是一个很大的好处。现在，您可以在专用的地方建议样品并报告现有样品的问题。或者，您也可能希望通过JIRA将样本作为附件提交给我们，如果我们相信您的样本会增加价值，那么我们很乐意将其添加到我们的样本存储库中，以功劳于作者。

从哪里获得样品


要监视样本开发并获取有关存储库的更多信息，您可以访问以下URL： http://git.springsource.org/spring-integration/samples 。我们还将Git SCM用作样本存储库。有关Git SCM的更多信息，请访问其网站： http://git-scm.com/

CLONE样本存储库。 （对于不熟悉Git的人，这在某种程度上相当于结帐。）

这是您应该经历的第一步。您必须在计算机上安装Git客户端。有许多基于GUI的产品可用于许多平台。简单的Google搜索将使您可以找到它们。要从命令行克隆样本存储库：

mkdir spring-integration-samples
cd弹簧集成样本
git克隆git：//git.springsource.org/spring-integration/samples.git


这就是您需要做的。现在，您已经克隆了整个样本存储库。由于样本存储库是实时存储库，因此您可能需要执行定期更新以获取新样本以及对现有样本的更新。要获取更新，请使用git PULL命令：

git pull

追踪样本


如前所述，Spring Integration示例具有专用的JIRA问题跟踪系统。要提交新的样品申请或提交实际样品（作为附件），请访问我们的JIRA问题跟踪系统： https://jira.springframework.org/browse/INTSAMPLES

样本结构

样本的结构也发生了变化。有了更多样品的计划，我们意识到一些样品的目标与其他目标不同。尽管它们都具有向您展示如何应用和使用Spring Integration框架的共同目标，但它们还涉及某些样本应集中于技术用例的示例，而其他样本涉及业务用例和某些示例的相关领域。展示了可用于解决某些情况（技术和业务）的各种技术。样本分类将使我们能够根据每个样本所解决的问题更好地组织它们，同时为您提供一种更简单的方法来找到合适的样本。

当前有4个类别。在样本存储库中，每个类别都有自己的目录，该目录以类别名称命名：

基本知识
（样本/基本）
这是一个入门的好地方。这里的示例出于技术动机，并演示了有关配置和代码的最低要求，以通过向您介绍Spring Integration的基本概念，API和配置以及企业集成模式（EIP），帮助您快速入门。例如;如果您正在寻找有关如何实现和将Service Activator连接到通道或如何使用Messaging Gateway进行消息交换的答案，或者如何开始使用MAIL或TCP / UDP模块等，这是正确的选择找到一个好的样本。最重要的是，这是一个入门的好地方。
中间
（样品/中间体）
该类别面向的是已经熟悉Spring Integration框架（过去入门）但在解决更高级的技术问题（一旦切换到消息传递体系结构后可能会解决）的同时，需要更多指导的开发人员。例如;如果您正在寻找有关如何在各种消息交换方案中处理错误或如何为某些消息可能永远不会到达以进行聚合等的情况正确配置Aggregator的答案。以及超出特定组件的基本实现和配置并解决“您还可以用它做什么”类型的任何其他问题，这将是查找此类样本的正确位置。
高级
（样本/高级）
该类别针对的是非常熟悉Spring Integration框架但希望通过使用Spring Integration公共API对其进行扩展以解决特定自定义需求的开发者。例如;如果您正在寻找展示如何实现自定义Channel或Consumer（基于事件或基于轮询）的示例，或者您试图找出在Spring Integration Bean解析器之上实现自定义Bean解析器的最合适方法？为自定义组件实现自定义名称空间时，这将是正确的位置。在这里您还可以找到有助于适配器开发的示例。Spring Integration附带了一个广泛的适配器库，可让您使用Spring Integration消息传递框架连接远程系统。但是，您可能需要与核心框架未提供适配器的系统集成。因此，您必须实现自己的。
此类别将包括向您显示如何执行操作的示例。
应用领域
（样本/应用程序）
此类别的目标读者是对消息传递体系结构， EIP有深入了解并且对Spring和Spring Integration框架有较高了解的开发人员和架构师，他们正在寻找解决特定业务问题的示例。换句话说，样本的重点是业务用例，以及如何通过Messaging Architecture和Spring Integration解决它们。例如;如果您有兴趣了解如何通过Spring Integration实施和自动化贷款经纪人或旅行社流程，那么这里将是查找此类样本的正确位置。

Spring Integration样本还带有主要版本。因此，要签出特定于2.0.0的版本。释放所有您需要做的就是克隆存储库“ git checkout v2.0.0”后执行以下Git命令。发布”。
记得！Spring Integration是一个社区驱动的框架，因此社区参与很重要。其中包括样本，所以如果您找不到想要的东西，请告诉我们。当然，也欢迎反馈！
