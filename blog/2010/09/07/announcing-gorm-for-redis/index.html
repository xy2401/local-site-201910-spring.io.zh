宣布Redis GORM

工程
格雷姆·罗彻（Graeme Rocher）
2010年9月7日

受云平台兴起启发的主要运动之一是对传统关系数据库的替代存储技术的兴趣日益浓厚。Redis是一种这样的存储机制，这是一种高性能的键/值存储，在未来的云基础架构中扮演着重要的角色。

在Grails社区中，人们对Redis表现出了极大的兴趣，在Redis 2.0.0版本发布之后，我们很高兴地宣布，Redis Milestone 1可以使用GORM 。

什么是Redis GORM？非常简单，它允许Grails开发人员习惯于使用诸如动态查找器 ， 条件和命名查询之类的功能 ，以利用Redis作为其基础存储而不是Hibernate。

Redis启动并运行后 ，可以从Grails命令行安装GORM for Redis：

grails install-plugin redis

如果您卸载了Hibernate插件，则grails-app / domain中的所有域类都将变为Redis实体，否则，您可以使用以下命令将单个实体指定为Redis可持久的：

static mapWith = "redis"

准备好Redis域类后，就可以继续使用常规的GORM API方法，例如save（），delete（），动态查找器等。

如果要构建自己的索引，还包括一个较低级别的API ：

def theSimpsons = Person.findAllByLastName("Simpson")
def list = redis.list("the.simpsons")
theSimpsons.each { list << it.id }

然后查询索引：

def theSimpsons = redis.entities(Person, "the.simpsons")
theSimpsons.each {
    println it.firstName
}

有关使用较低级别的API和其他功能的更多信息，请参见完整的文档 。还有一个示例应用程序可供下载。

Redis插件本身建立在无关紧要的代码库上，该代码库被设计为NoSQL存储之上的抽象。我们还为其他替代商店（例如Cassandra和JCR）提供了正在进行的支持。

该项目是开源的，我们有兴趣与NoSQL社区合作以建立类似于GORM for Redis中的支持。有一个GORM TCK，您可以将其与某个实现进行对比以测试兼容性，我们将继续扩展该覆盖范围，以涵盖尽可能多的GORM功能。

如果您有兴趣帮助我们完成任务，请给我们一个无关紧要的邮件列表 。

这是一个里程碑版本，因此，在我们接近里程碑2和不可避免的最终版本时，我们很想听听您的反馈。
