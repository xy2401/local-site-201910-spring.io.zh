将RabbitMQ与Spring Python混合

工程
格雷格·特恩奎斯特
2010年9月20日

RabbitMQ是基于高级消息队列协议（AMQP）的功能强大的消息代理。在较早的文章中，我们研究了构建Python股票行情收录器程序的方法 。我们比较了使用RabbitMQ的pika和py-amqplib进行比较 ，以及如何以最小的更改轻松地从一个过渡到另一个。

在本文中，我们将展示Spring Python如何轻松使用pika。Spring Python是使用Python语言实现的Spring概念的实现。它包括许多功能，例如依赖项注入 。如果我们选择在Python中构建类似Spring的应用程序，则很容易利用pika。

任何Spring应用程序的核心都是一组蓝图或应用程序上下文。对于Spring Python和Spring Java，这是相同的。

就像您确定的那样，此上下文不是用XML编写的。相反，它是用纯Python编码的。这非常类似于Spring Java Config（此后已引入到核心Spring Framework中）。这定义了发布者和订阅者。

  • 它们都配置有延迟初始化，因此我们可以使用一个上下文来支持两个启动脚本。要创建股票报价器，我们需要创建相同的股票报价器应用程序，仅使用Spring Python容器。
  • 它们都配置了作用域。原型，在某种情况下，我们想获取多个副本。

我们还需要一个脚本来使用Spring Python的容器来启动订阅这些消息的经纪行。

下图显示了两个终端窗口。后台窗口显示股票行情应用程序正在运行，吐出股票报价。如果仔细观察，您会发现最后一个参数是一条标识为290的消息。前景窗口显示经纪行订阅股票报价。它刚收到消息290，然后打印出其当前净值。

因为我们使用Spring Python注入了基于pika的RabbitMQ客户端，所以我们可以轻松地将其切换到其他客户端。如上一篇博客文章中所指出的那样，这有助于我们避免锁定，而是根据重要的方面（技术优势，供应商支持和业务需求）选择解决方案。在开发RabbitMQ经纪人的同一个团队的支持下，Pika已证明自己是一个活跃的项目。从py-amqplib迁移到它很容易，并且支持我们正在使用的任何工具，包括我们刚刚看到的Spring Python。如果将来需要，将来很容易迁移到另一个库。
