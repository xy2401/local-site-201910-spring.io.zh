Hyperic通过版本4.4中的vCenter API扩展了对vSphere的支持

工程
查尔斯·李
2010年8月6日

SpringSource Hyperic 4.4于本周早些时候发布，通过与VMware vCenter集成带来对VMware虚拟化环境的增强管理。Hyperic长期以来一直支持VMware技术（即VMware Infrastructure或VI），但VI Java API的可用性（ http://vijava.sf.net ）使我们的工作更加轻松，因为强大的SDK简化了Java应用程序与VI的交互方式。通过采用VI Java API，我们能够轻松跟上VMware从VI到VMware vSphere的发展，这就是众所周知的。这也使我们能够直接与ESX和ESXi服务器通信，而无需随着这些技术的发展而进行区分或修改。随着VMware成为数据中心基础架构中越来越大的组件，当它作为较大基础架构的成员时，将每个ESX和ESXi服务器作为一个单独的实体进行管理似乎变得不完整。实际上，VMware在物理硬件资源上采用了覆盖技术，从而在称为vSphere的整体环境中将它们绑定在一起，从而向前发展。VMware vSphere的管理由最初名为VirtualCenter的软件处理，现在称为vCenter。作为VMware系列的一部分，我们开始意识到，我们对VMware的支持需要超越简单地利用易于使用的SDK来发展。当我们从新的VMware同事那里了解到有关最新vSphere技术的更多信息时，我们意识到我们需要重新考虑我们如何支持vSphere环境的监视和性能管理。Hyperic 4.4是Hyperic的VIM插件重新架构的结果，为了支持新的vSphere插件而弃用了Hyperic。使用VI Java调用vCenter API更有意义，vCenter API提供的功能比ESX或ESXi多得多，并且可以查询vCenter Server上聚集的数据。当然，将多个vSphere连接合并到一个连接中显然有好处。此外，在vCenter中执行和跟踪vMotion具有较新的功能，而您在其他地方则无法做到。通过vCenter的vSphere集成的新级别在Hyperic 4.4中带来了许多新的优点和功能，其中包括：




  • vSphere自动发现所有vSphere ESX / ESXi主机，VM及其层次结构。以这种方式发现的vSphere资源会自动映射到Hyperic中，并且在虚拟机中安装Hyperic代理后，VM会自动与Hyperic平台配对，从而提供对虚拟基础架构内部运行的应用程序的深度管理。

  • 全新的虚拟化应用程序基础结构视图，可为应用程序操作员提供其vSphere环境的分层视图，以便他们可以更快速地查明并防止应用程序性能问题，无论问题在物理，虚拟或云基础结构中的位置如何。

  • 警报现在可以识别虚拟化，并可以防止对已关闭电源，挂起或删除的VM发出错误警报。



Hyperic 4.4标志着第一次创建了一种工具，该工具可让虚拟基础架构管理员和应用程序操作员共同协作以管理应用程序性能。请访问Hyperic了解更多信息并下载试用版。
