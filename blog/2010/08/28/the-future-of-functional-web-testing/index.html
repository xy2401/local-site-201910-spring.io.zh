功能性网络测试的未来？

工程
彼得·莱德布鲁克
2010年8月28日

Groovy社区是一个富有成果的社区，这意味着有许多框架，库和工具可简化您的生活。测试领域似乎特别肥沃，我最近一直在研究一些工具，这些工具结合使用时，可以保证编写功能性Web测试时生产力发生重大变化。

尽管我通常将重点放在Grails上，但是您不必使用Grails来获得这些工具的好处：它们可以与任何Web应用程序一起使用，并且可以与任何基于Java的项目/构建很好地集成。碰巧的是，它们都具有相关的插件，这些插件使从Grails使用它们变得非常简单。

我要谈的第一个工具是Spock 。它基于行为驱动开发 （BDD）范例，该范例将重点从测试本身转移到根据预期行为来考虑代码。您编写的测试用例将作为规范阅读，这不仅使它们更易于阅读和理解，而且易于编写。您甚至可以将Spock集成到任何Java项目中，并从IDE的规范中运行（只要IDE具有Groovy支持，三个主要的功能就可以）。

第二个工具甚至更新。它称为Geb ，它使用WebDriver通过真实的浏览器或HtmlUnit库测试Web应用程序。让Geb脱颖而出的是用于查询HTML页面的类似jQuery的语法及其对Page Object模式的内置支持。

那么，为什么我认为这些构成了成功的组合？因为它们使编写功能性的Web测试变得尽可能容易！让我们看一下其中两个。

一个简单的例子

假设您有一个要测试的简单登录页面。它接受用户名和密码，并带有“登录”按钮。HTML看起来像：

<html> 
<head> 
  <title>Login</title>
</head> 
<body>
  <form action="/wildcard-realm/auth/signIn" method="post" > 
    <input type="hidden" name="targetUri" value="" /> 
    <table> 
      <tbody> 
        <tr> 
          <td>Username:</td> 
          <td><input type="text" name="username" value="" /></td> 
        </tr> 
        <tr> 
          <td>Password:</td> 
          <td><input type="password" name="password" value="" /></td> 
        </tr> 
        <tr> 
          <td>Remember me?:</td> 
          <td>
            <input type="hidden" name="_rememberMe" />
            <input type="checkbox" name="rememberMe" id="rememberMe"  />
          </td> 
        </tr> 
        <tr> 
          <td /> 
          <td><input type="submit" value="Sign in" /></td> 
        </tr> 
      </tbody> 
    </table> 
  </form>
</body> 
</html>

现在看一下以下Spock规范，并尝试找出要测试的行为：

import geb.spock.GebReportingSpec
import pages.*
 
class MySpec extends GebReportingSpec {
    String getBaseUrl() { "http://localhost:8080/wildcard-realm" }
    File getReportDir() { new File("target/reports/geb") }

    def "Test invalid password"() {
        given: "I'm at the login page"
        to LoginPage

        when: "I enter an invalid password for 'admin'"
        loginForm.username = "admin"
        loginForm.password = "sdfkjhk"
        signIn.click()

        then: "I'm redirected back to the login page with the password field empty and an error message"
        at LoginPage
        loginForm.username == "admin"
        !loginForm.password
        message.text() == "Invalid username and/or password"
    }

    def "Test valid login"() {
        given: "I'm at the login page"
        to LoginPage

        when: "I enter a valid username and password"
        loginForm.username = "admin"
        loginForm.password = "admin"
        signIn.click(HomePage)

        then: "I'm redirected to the home page, which displays my username"
        at HomePage
        $().text().contains("Welcome back admin!")
    }
}

我不了解您，但是我发现制定测试要尝试的工作非常容易。即使您现在不知道变量来自何处，也可以有效地阅读自然语言规范。这种易于理解的功能是Spock等BDD工具的一大优势。

让我们更详细地看一下规范。每个测试方法（或Spock喜欢称呼它们的“功能”方法）分为几个部分。第一个给定 ，包含任何设置代码，并为您提供测试的开始状态。然后，您声明一个when块，无论您要测试什么，它都会启动某种行为，例如，通过提交表单。最后，您需要在then块中检查刺激的结果，其中包含您需要完全验证预期行为的条件。与JUnit测试不同，您无需在then节中使用显式断言，因为每个表达式都是隐式断言。

这是一个简单的概念，但是一旦习惯了编写规范，就会发现Spock使编写测试变得更加容易。这是我无法真正解释的东西。我最好的猜测是，语法和结构与您在脑海中制定测试的方式相匹配，因此在考虑要测试什么和编写物理测试用例之间几乎没有什么阻力。但是，我敦促您尝试一下，而不是信守承诺。您可以将Spock用于单元测试和功能测试，因此很容易使用。

测试中的几乎所有其他内容都是Geb，包括to（）和at（）方法。两者都在页面对象上工作，您必须自己编写。幸运的是，从LoginPage类可以看出，这很简单：

package pages

import geb.Page

class LoginPage extends Page {
    static url = "auth/login"

    static at = { title == "Login" }

    static content = {
        loginForm { $("form") }
        message { $("div.message") }
        signIn { $("input", value: "Sign in") }
    }
}

让我们分别查看此类中的静态属性：


  • url-页面的相对URL； to（）方法使用它来确定将HTTP请求发送到哪个URL。

  • at-一种闭包，指示当前页面是否为该页面-由at（）方法调用；它应该返回一个布尔值，但是您也可以包含断言。

  • content-页面内容的描述，可以轻松访问此处声明的部分。

因此，在上面的示例中，您可以看到登录页面的相对URL为“ auth / login”。相对于什么？到测试的baseUrl 。确定当前页面是否为登录页面，只需检查at闭包中页面的标题是否为“ Login”。最终， 内容块提供了直接访问登录表单（这是页面上唯一的表单），信息/错误消息“ div”和“登录”按钮的全部-通过Geb的$（）方法。

如果回头看一下测试，您会发现我能够访问内容元素（例如loginForm） ，就好像它们是测试的属性一样。 Geb的此功能允许进行非常简洁和自描述的测试，但更重要的是，它可以促进代码重用。想象一下，您的HTML页面已更改，并且其中一个表达式不再与您想要的表达式匹配。如果您不使用页面对象，则必须执行可能不可靠的全局搜索和替换。改为在页面对象中更改该引用有多好！

$（）函数不仅限于内容块-如果需要，您可以直接从测试代码中使用它。考虑以下测试：

    ...
    def "Test authentication redirect with query string"() {
        when: "I access the book list page with a sort query string"
        login "admin", "admin", BookListPage, [sort: 'title', order: 'desc']

        then: "The list of books is displayed in the correct order"
        at BookListPage
        $("tbody tr").size() == 3
        $("tbody tr")*.find("td", 1)*.text() == [ "Misery", "Guns, Germs, and Steel", "Colossus" ]
    }
    ...
    /**
     * Logs into the application either via a target page that requires
     * authentication or by directly requesting the login page.
     */
    private login(username, password, targetPage = null, params = [:]) {
        if (targetPage) {
            to([*:params], targetPage)
            page LoginPage
        }
        else {
            to LoginPage
        }

        loginForm.username = username
        loginForm.password = password

        if (targetPage) signIn.click(targetPage)
        else signIn.click(HomePage)
    }
    ...

类似jQuery的语法可以非常轻松地匹配HTML页面中的元素并提取属性值和内容。Geb手册以及Geb的许多其他功能都很好地涵盖了它的语法。

前面的示例还演示了如何将测试中的代码分解为可重用的方法。因为Spock的语法一开始是很陌生的，所以您可能认为这不可能。但是规范最后是类，因此您可以将其视为类。

我可以继续讨论Spock和Geb的功能以及如何使用它们，但是本文不是教程。使您更感兴趣的是品尝者。如果要查看我从上面摘录的摘录的完整Spock规范，请查看它的源代码以及相关的页面对象 。

你在等什么？

目前，Spock和Geb都是非常年轻的技术（都没有达到1.0），但是它们已经足够远，人们可以在他们的项目中积极使用它们。即使在这个阶段，它们也提供了一个引人注目的案例：相对容易编写和理解的功能性Web测试。

这不是小事。自动化功能测试对于确保Web应用程序的正常运行至关重要，但是当前的方法（至少在Java Universe中）通常比较笨拙，并且不利于编写这些测试。因此，团队最终依赖于手动测试，而这永远不会为您提供真正需要的覆盖范围的可靠性。

都是玫瑰花吗？当然不是。但是我们这里有两个工具，它们使应该易于测试的东西，实际上易于测试，这在功能性网络测试世界中绝非易事。随着您需要测试的页面变得更加复杂，他们将继续为您提供支持。也许最大的问题是处理Javascript并从测试中触发DOM事件，但是即使在那儿，您也会发现Geb正在迅速发展以帮助您。

我什至没有提到Spock的内置模拟框架或它对数据驱动测试的支持（请查看项目文档中的where子句）。甚至其断言的输出也是一个很大的好处：


dateService.getMonthString(new Date().updated(month: month)) == expected

| June | | 5 | July
| | | false

| | | Ju(ne)

在Geb前端，看一下模块功能，它使您可以将页面对象分解为可重用的部分。非常适合复杂的页面。并且由于它在后台使用WebDriver，因此您可以使用真实的浏览器或以无头模式（通过HtmlUnit）运行测试。

最后，让我重申这些工具在Java项目中与Groovy或Grails一样好用。仅仅因为您的Web应用程序是用Java编写的，并不意味着您的测试也必须如此。还应了解，Spock可用于任何类型的Java项目-进行单元，集成和/或功能测试。

如果我了解到一件事，那就是编写测试必须尽可能简单，否则就不会编写。这就是为什么我认为Geb和Spock是测试领域中的重要发展，值得研究。
