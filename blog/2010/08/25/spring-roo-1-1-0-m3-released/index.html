Spring Roo 1.1.0。M3发布

工程
本·亚历克斯
2010年8月25日

更新： SpringSource Tool Suite 3.5.0。现在可以下载 M3。它包括最新的Spring Roo 1.1.0。M3和GWT 2.1.0。M3支持。

社区中的许多成员一直在等待Spring Roo 1.1.0。M3，我很高兴地宣布它现已可用（ 在此处下载 ）。自里程碑2以来，此版本合并了150多项新功能，增强功能和其他改进功能，使1.1.0里程碑系列的累积总数达到了500个问题。Spring Roo 1.1.0中的许多改进。M3反映了我们与Spring Framework，Google Web Toolkit，AspectJ和SpringSource Tool Suite团队的持续紧密合作，您可以在一些主要的新功能中看到结果：


  • 增量数据库逆向工程 （或众所周知的办公室DBRE）：在Spring Roo项目的历史上，这是Jira票证最高的投票 ，大约有130张独立票。这项新功能可以对现有的关系数据库进行内部检查并发出一系列JPA实体。但是，与传统的数据库逆向工程工具不同，DBRE可以在同一项目上重复运行，而不会产生不希望的副作用（因此，功能名称中的术语“递增”）。自省的所有成员都存储在AspectJ ITD中，这与Roo的其余部分一致。这样可以使您的.java文件整洁有序，并可以进行意图揭示（就像它们应该那样）。自省现在运行良好（即使对于复杂的情况，例如复合主键和所有基数类型），它也可以在Roo 1.1.0中以正常Java样式以编程方式使用生成的实体。M4我们将为自省数据库添加Web层支持。

  • 最新的Google Web工具包2.1.0。M3功能：我们与GWT的合作使Roo可以生产使用GWT 2.1.0中最新功能的应用程序。M3（今天也发布了）。备受期待的新功能之一是RequestFactory基础结构下的关系支持。您可以通过以下命令来进行尝试：使用“ scriptburst.roo”命令，然后使用“ mvn gwt：run”并通过GWT UI创建相关的员工对象。您可以阅读有关新GWT 2.1.0的更多信息。G3博客条目中的M3功能。

  • 自动JSON支持，包括在Spring MVC应用程序中： Spring Roo产生的所有域对象现在都将包括与JSON相关的自动方法。这些允许您在JSON表示形式和对象之间双向转换。我们以普通Roo风格通过ITD引入了这些与JSON相关的新方法。Roo管理的Spring MVC控制器现在也可以自动使用这些方法。要查看实际效果，请使用“ script clinic.roo”脚本，然后使用“ mvn tomcat：run”，通过网络用户界面创建新的所有者，然后尝试执行curl命令，例如curl -v -H ‘Accept: application/json’ -X GET http://localhost:8080/petclinic/owners/ 。

  • 16种社交媒体嵌入功能：现在可以使用一个Roo命令将16个社交媒体网站上的内容添加到基于Spring MVC的前端中！这包括YouTube，Google Video，Vimeo，Viddler，Screenr，UStream，LiveStream，Flikr，Picasa，Google Presentations，Scribd，SlideShare，Google Maps，Google Wave，Twitter和Wikinvest。所有这些都可以通过新的“ web mvc embed ”命令获得。

  • 库升级：我们已经更新了Roo在项目中自动使用的库。一些值得注意的更新包括转移到Spring Framework 3.0.4 （通过消除URL重写过滤器增强了资源服务）和Spring Web Flow 2.1.1 。

  • 自动Roo注释JAR升级：现在，当您升级到Roo的新版本时，无需编辑项目的pom.xml即可更改Roo注释库的版本。袋鼠会照顾这将自动为您服务。

  • 可用性增强：我们现在添加了许多新功能来提高可用性。您可能会注意到的一项是屏幕顶部的新“即显信息 ”区域。这将显示长时间运行的操作的进度（您可以尝试使用“ flash test”命令进行快速演示）。我们建议您使用Linux，Windows或iTerm （对于OSX用户）以获得最佳的Flash消息体验。其他一些鲜为人知的改进包括更好的日志记录 ，支持“＃”注释 ，支持“ hint ”命令的选项卡辅助功能，AspectJ ITD顶部的注释（ 提醒您不要对其进行编辑）以及许多其他功能。

与往常一样，《 Spring Roo参考指南》包括针对已经使用Roo的人员的升级说明，以及针对有兴趣了解更多内容的人员的背景阅读和入门教程。该参考指南可以在线查看 ，并且它也包含在下载的 ZIP中。您还可以通过参加SpringOne ， JavaOne ， Devoxx ， Google Developer Days或YOW来了解有关Spring Roo的更多信息！未来几个月的2010年 。

Spring Roo的下一版本将是1.1.0。M4。这将是该系列的最后一个里程碑，我们预计将达到1.1.0。在2010年10月中旬发布。为了方便起见，我们在吉拉保留了完整的路线图日期 。

与往常一样，我们欢迎您对这个新版本发表评论。您可以在Twitter上关注@SpringRoo和工程团队 ，通过社区论坛讨论Roo或通过上述会议之一来访问我们。我们希望您喜欢Spring Roo的新版本。
