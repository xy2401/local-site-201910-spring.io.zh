Spring Android和Maven（第1部分）

工程
罗伊·克拉克森
2010年12月17日

我们最近宣布了Spring Android的M1版本 ，随着该版本的发布，出现了一些有关如何使用Spring Android Rest Template和Spring Android Commons Logging库构建Android应用程序的问题。Google提供了几种用于编译Android应用程序的方法，包括SDK命令行工具和Eclipse的ADT（Android开发工具）插件。不幸的是，这些方法都不包含集成的依赖性管理支持。

总览

作为Java开发人员，我们开始欣赏Maven和Gradle之类的用于管理外部依赖项的工具。传统Java应用程序在JVM中运行，而Android应用程序在Dalvik虚拟机上运行。Dalvik VM执行Dalvik Executable（.dex）格式的文件。它运行由Java语言编译器编译的类，这些类已转换为.dex格式。如果构建工具能够编译具有依赖项的兼容Android应用程序，则需要支持该过程。

在Android应用程序中包括外部库的方法基本上有两种。第一种是将jar手动复制到项目内的libs目录中，并在Eclipse中更新类路径。这是最简单的解决方案，也是ADT插件最支持的解决方案。缺点是您必须手动管理依赖项。或者，可以使用第三方插件（例如Maven Android插件）来自动包括Maven存储库中的依赖项。

在本文中，我将逐步介绍如何使用Android命令行工具，Maven，Maven Android插件和Android Maven工件来编译利用Spring Android库的示例应用程序，并将其部署到Android仿真器。配置Maven之后，可以轻松地创建构建，将其部署到仿真器，运行测试以及打包应用程序以部署到Android Market。在运行示例代码之前，我们将首先在pom.xml中突出显示必要的配置设置。下面列出了此示例中使用的组件。

  • SpringSource工具套件2.5.2（Eclipse 3.6.1）
  • Android SDK修订版9
  • Maven Android插件2.8.4

Maven配置

本节介绍了使用Spring Android和Maven Android插件进行开发所需的pom.xml部分。

Maven Android插件

为了使用Maven构建Android应用程序，您需要在pom.xml文件中配置Maven Android插件。Android应用程序以apk文件而不是jar的形式部署到设备上。您必须在包装配置中指定此名称。


<packaging>apk</packaging>

要在构建任务中配置Maven Android和Maven编译器插件，请将sdk平台设置为所需级别。在此示例中，将其设置为9，对应于Android版本2.3.1。模拟器avd值是您在AVD管理器中定义的AVD（Android虚拟设备）的名称。在这种情况下，AVD的名称为“ 9”，但只要与创建AVD时指定的名称匹配，就可以随意命名。这是构建和运行Android应用程序所需的插件的基本配置。可以包含其他参数以获得更多功能。


<build>
	<sourceDirectory>src</sourceDirectory>
	<finalName>${project.artifactId}</finalName>
	<plugins>
		<plugin>
			<groupId>com.jayway.maven.plugins.android.generation2</groupId>
			<artifactId>maven-android-plugin</artifactId>
			<version>2.8.4</version>
			<configuration>
				<sdk>
					<platform>9</platform>
				</sdk>
				<emulator>
					<avd>9</avd>
				</emulator>
				<deleteConflictingFiles>true</deleteConflictingFiles>
				<undeployBeforeDeploy>true</undeployBeforeDeploy>
			</configuration>
			<extensions>true</extensions>
		</plugin>
		<plugin>
			<artifactId>maven-compiler-plugin</artifactId>
			<version>2.3.2</version>
		</plugin>
	</plugins>
</build>

依存关系

通过Android for Maven项目的努力，已经构建了Android工件并将其发布到Maven存储库。Google阻止了将官方Android jar上载到Maven，因此，第三方Android for Maven项目开始提供可与API兼容的Android工件，该工件可以上载到Maven存储库。现在，Maven存储库中提供了每个主要Android版本的工件。但是，这些功能不起作用，仅提供API的存根实现。所有类中的所有方法都将引发运行时异常。由于Android应用程序在设备上运行，因此它将永远不会使用这些库来执行，但是API的兼容性使应用程序可以像真实库一样进行编译。可以在此处找到更多信息。

要编译具有依赖项的Android应用程序，您需要包括要为应用程序定位的Android版本。如前所述，我们正在使用级别9，它对应于版本2.3.1。检查Maven存储库中的可用版本。您必须将android依赖项范围设置为提供 ，否则Maven会尝试将Android jar库包含到您的apk中。


<dependency>
	<groupId>com.google.android</groupId>
	<artifactId>android</artifactId>
	<version>2.3.1</version>
	<scope>provided</scope>
</dependency>

通过添加以下依赖项，针对Spring Android Rest Template的最新里程碑版本进行编译。


<dependency>
	<groupId>org.springframework.android</groupId>
	<artifactId>spring-android-rest-template</artifactId>
	<version>1.0.0.M2</version>
</dependency>

包括快照和里程碑构建的存储库，以在您的应用中使用任何Spring Android库的最新构建或里程碑版本。


<repositories>
	<!-- For testing against latest Spring snapshots -->
	<repository>
		<id>org.springframework.maven.snapshot</id>
		<name>Spring Maven Snapshot Repository</name>
		<url>http://maven.springframework.org/snapshot</url>
		<releases><enabled>false</enabled></releases>
		<snapshots><enabled>true</enabled></snapshots>
	</repository>
	<!-- For developing against latest Spring milestones -->
	<repository>
		<id>org.springframework.maven.milestone</id>
		<name>Spring Maven Milestone Repository</name>
		<url>http://maven.springframework.org/milestone</url>
		<snapshots><enabled>false</enabled></snapshots>
	</repository>
</repositories>

开发环境

开发Android应用程序需要Android SDK。如前所述，Google提供了命令行工具以及用于构建Android应用程序的Eclipse插件，但是您不仅限于这些选项。其他IDE也为构建Android应用程序提供支持。Maven Android插件利用Android SDK命令行工具来编译应用程序并将其部署到仿真器，因此无需单独的IDE设置或配置。

可以在Android网站上找到有关下载和安装Android SDK的说明。请注意， Android SDK修订版8发行版更改了某些工具的位置。除tools目录外，还必须将platform-tools目录添加到路径中。

例如，在Mac上，.bash_profile可能如下所示。

export ANDROID_HOME=~/android-sdk-mac_x86
export PATH=${PATH}:$ANDROID_HOME/tools:$ANDROID_HOME/platform-tools

配置Android虚拟设备

要运行Android应用，您必须配置一个Android虚拟设备 （AVD）。AVD是仿真器选项的配置。换句话说，您正在定义运行仿真器时要使用的设置。您可以保存带有名称的配置，并在以后使用。您还可以定义多个AVD，以针对不同的Android版本或硬件配置进行测试。

示例Android客户端应用程序随附的pom.xml文件指定名称为“ 9”的AVD。为了使Maven能够部署Android应用，您必须使用相同的名称配置一个AVD。这是特别令人感兴趣的，因为所有开发人员机器都需要配置相同的AVD，因为pom.xml通常致力于源代码控制。

  1. 在命令行中，输入android并按回车键。这将打开“ Android SDK和AVD Manager”窗口。
  2. 在左侧栏中选择虚拟设备 ，然后单击新建...按钮
  3. 在名称字段中输入9
  4. 在目标选择器中选择Android 2.3.1- API Level 9
  5. 点击创建AVD以完成

样品申请

我们已经为Spring Mobile项目建立了一个示例存储库。在命令提示符下，使用以下命令将存储库克隆到本地计算机。

$ git clone git://git.springsource.org/spring-mobile/samples.git spring-mobile-samples

启动服务器

如果您想运行示例应用程序的服务器组件，以查看Android客户端和Spring MVC网站之间的交互，最简单的方法是从STS IDE中进行。导航到spring-android-showcase目录。有两个目录，一个用于客户端Android应用程序的“客户端”和一个用于Spring MVC服务器应用程序的“服务器”。客户端应用程序向服务器发出网络请求以说明RestTemplate的功能，因此必须运行服务器才能使客户端正常运行。

  1. 在STS中，选择文件->导入...
  2. 在“ 常规”文件夹中，选择“ 现有项目到工作区” ，然后单击“ 下一步”。
  3. 单击浏览并导航到spring-android-showcase目录的Server目录。
  4. 单击打开将项目添加到您的工作区
  5. 在“ 程序包资源管理器”视图中突出显示spring-android-showcase-server项目，然后将其拖到“ 服务器”视图中的SpringSource tc Server Developer Edition中以部署Web应用程序
  6. 最后，单击“ 运行”按钮以启动服务器

运行Android客户端

要构建客户端应用程序，请从命令行输入以下命令

$ mvn clean install

输入以下命令以启动Android模拟器。Maven尝试使用pom.xml中配置的名称启动AVD，这就是为什么该名称需要与您创建的实际AVD名称匹配的原因。

$ mvn android:emulator-start

最后，使用以下命令将应用程序部署到仿真器。

$ mvn android:deploy

该应用程序作为S2Android Showcase部署到模拟器。在运行该应用程序之前，请启动Android日志查看器以查看该应用程序的活动。在进行开发时，您将花费大量时间使用Android日志。要查看日志，请在命令提示符处执行以下命令。

$ adb logcat

恭喜你！现在，您已经构建并部署了具有Maven托管依赖项的Android应用。

结论

将Maven添加到您的Android开发过程中会增加额外的复杂性，但是它提供了编译包含Maven存储库中的外部依赖项的Android应用程序的功能。没有它，您将必须手动下载编译和运行应用程序所需的依赖项。我们已经展示了将其用于依赖项管理与Spring Android库一起构建的好处。在第2部分中，我将介绍Maven Integration for Android开发工具插件和Eclipse的ADT（Android开发工具）插件，介绍Eclipse中的Android开发 。m2eclipse和ADT插件都不支持构建具有Maven依赖关系的Android应用程序。适用于Android开发工具的Maven集成插件提供了通往Maven Android插件的桥梁，从而在Eclipse for Android项目中启用了Maven依赖项管理。如果您喜欢使用Eclipse进行开发，我将在下一篇文章中讨论如何使用这些插件。
