Grails 1.3发布

工程
格雷姆·罗彻（Graeme Rocher）
2010年5月11日

我很高兴地宣布Grails 1.3即将发布！Grails版本的追随者会记得，自从Grails的最后一个主要版本（Grails 1.2于12月发布）以来已经不多了，但是由于需要在开发人员手中获得最新最好的Groovy版本，Grails 1.3可以被视为Groovy 1.7发行版 。Groovy团队做了出色的工作，为Groovy语言带来了各种出色的新功能，例如匿名的内部/嵌套类，AST生成器和power asserts （我个人的最爱），所有这些现在都可以在Grails应用程序中使用。

除了Groovy 1.7的所有优点之外，还有许多重要的新功能可以进一步推进Grails平台，包括：


  • JUnit 4支持

  • Maven存储库对Grails插件的支持

  • 声明性插件依赖性

  • GORM中的脏物检查

  • 链接命名条件

  • 应用范围广泛的Sitemesh布局


在接下来的几天/几周内，将有后续文章详细介绍所有这些功能。现在，您可以参考以下链接以获取有关该发行版的更多信息：

  • 发行说明

  • 下载

  • 文献资料


随着Grails 1.3的发布，我们将继续分发Grails的1.2.x和1.3.x分支的维护版本，同时开发主要的新插件。其中的第一个已经随着针对Grails的Spring Security Core插件的发布而实现，这是一个基于Spring Security的新模块化插件，带有可插入的身份验证层，用于Open ID ，OAuth等。

Grails的用户可以在接下来的几个月中在持久性（NoSQL），云，性能监控和消息传递领域期待令人兴奋的新插件版本。说到插件，Grails现在总共有408个（并且还在不断增加）插件 ，并且我们正在意识到一个不断发展的生态系统。网站周围的插件网站基础结构旨在进行改进，这些改进将继续使社区蓬勃发展。

除插件外，我们还将正式开始Grails 2.0的规划过程，这将是该框架的一个主要新版本，重点是插件运行时模块化，数据库反向工程/迁移和构建时间/可用性改进。更多详细信息即将发布。

在此期间，请尽情享受发布，并感谢为发布过程做出贡献的所有人。

PS：如果您想亲自听我说Grails 1.3的新功能，请确保参加伦敦S2G论坛上有关“ Grails 1.3的新功能”的演讲 。
