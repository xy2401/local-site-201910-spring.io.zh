向开放式PaaS前进

工程
罗德·约翰逊
2010年5月19日

几周前， 我发了一条推文 ，令人难以置信的是，SpringSource在VMware内部的执行速度比一家初创公司快。今天，我们宣布了另一个令人振奋的发展。

继我们与SaaS领导者salesforce.com 达成 VMforce合作伙伴关系之后，我们今天宣布VMware和Google之间围绕Spring编程模型以及SpringSource IDE和RAD工具进行合作。今天的公告使Spring成为Google App Engine的首选编程模型。这是Spring作为Java最佳和最可移植的编程模型的巨大认可，并为Spring开发人员提供了新的部署机会。今天在Google I / O上的主题演讲中的演示展示了SpringSource和Google工程师数月合作的结果-无论使开发人员希望将其部署在何处，其中的大多数都使Spring开发人员受益。亮点：Spring与Google Web Toolkit （GWT）之间的创新，紧密集成，提供了以惊人的速度构建丰富的应用程序的能力；轻松将Spring应用程序定位到App Engine的能力； Spring Insight与Google Speed Tracer之间的引人注目的集成，可洞悉从浏览器到数据库的Spring应用程序的性能；并将所有这些功能与SpringSource Tool Suite紧密集成，以提供优美的生产体验。

目标

在深入研究该技术之前，我想对目标进行思考。该公告强调了Spring的两个核心价值，它们将继续指导我们前进： 生产力和可移植性 。

生产率

Spring的采用归功于它能够显着提高Java生产力的能力。SpringSource继续在提高生产力上投入大量资金，今天的公告以SpringSource产品组合的两大亮点：SpringSource Tool Suite（STS）和Spring Roo的形式，突出了一些好处。在一起，它们为Java开发人员提供了最佳的生产力体验，令人振奋的是Google对此的认可。

  • STS（我们的免费Eclipse发行版）是开发Spring应用程序的最佳环境。我们在这里的目标是雄心勃勃的：一次下载即可提供构建最佳生产力的Java应用程序所需的全部内容。
  • Spring Roo是一个Spring项目，可帮助您比您想象的更快地构建Spring应用程序。您可能已经听说过Roo-如果您是Java开发人员并且还没有尝试过，现在是时候了。这完全是构建高质量Java应用程序的最快方法。简而言之，Spring Roo是一个交互式的往返代码生成器，可确保您只需要编写真正能增加价值的Java代码行，而Roo则负责创建和维护构建脚本。持久性映射；引导程序配置； JavaBean方法；与典型的代码生成器不同，Roo适用于希望编写代码的开发人员-确保他们只需要编写正确的代码即可。它的往返支持扩展到UI层，根据用户代码更改来更新托管工件。

尽管我们尊重Spring社区中的许多人希望从细粒度的选择中受益的事实，但我们坚信，诸如Spring Roo之类的自以为是的工具在生产率方面的优势将变得越来越重要。（尽管Roo面向希望使用Java编写代码的开发人员，但Grails却迎合了希望使用Groovy进行编程的人。这里需要特别注意的是Grails和Groovy社区已经在Google App Engine上展示了其技术的价值。）

生产力还与集成有关：使所有内容协同工作。通过SpringSource / Google合作，您将能够下载STS ，使用一个简单的对话框创建一个新的Spring Roo项目，并使用Roo Shell帮助您构建完整的应用程序。在几分钟之内，您可以创建一个真实的应用程序（包括数据访问），并使它在云中运行。借助与STS集成的Roo双向旅行，您可以向实体添加新字段，请参阅Roo更改GWT脚手架应用程序，在GPE的DevMode中按Refresh，切换回浏览器，然后查看包括数据库更改在内的整个应用程序在2后可用刷新时间为-3秒，无需重新启动服务器。

可移植性

Spring的核心价值一直是其使开发人员能够编写业务逻辑的能力，该逻辑不仅更简单，而且与部署环境完全脱钩。在春季的早期，这使开发人员能够编写可在不同服务器（例如WebSphere或Tomcat）之间移植的应用程序，从而避免了EJB或JTA之类的特定于环境的API（并为其提供了替代品）。如今，部署环境之间的选择涵盖了传统数据中心和云之间的选择。Spring可移植性正成为Java开发人员向云计算发展的最重要的推动力。

如今，Spring的可移植性是实现我们对Open PaaS愿景的关键：正如VMware CTO Steve Herrod所述，“专注于一个特别重要的选择……在其中部署和运行应用程序。”令人高兴的是，VMware拥抱Spring赖以成功的核心价值观。也很高兴看到Google分享对可移植性的承诺。

技术概述

我已经提到了Spring Roo和STS的作用。其他关键技术要素包括GWT和SpringSource以及Google性能分析工具。不久，我们从事这项工作的工程师将以更详细的技术细节以及新的Roo 1.1.0涵盖这些细节。M1和STS 2.3.3。M1版本可让您自己探索-但这里是您可以期待的快速指南。

GWT

Google Web Toolkit是一项强大的技术，用于开发丰富的Internet应用程序，它使用AJAX和HTML5等现代技术在计算机和非传统客户端（例如智能手机）上创建最引人注目的最终用户体验。随着UI需求的变化以及非传统客户端变得越来越重要，GWT对Java开发人员而言将变得越来越重要。

今天，我们展示了Spring，Spring Roo和GWT之间的深度集成-SpringSource和Google工程师之间数月合作的结晶，并回答了Spring社区最频繁的要求之一。这种集成使GWT应用程序比以往任何时候都更容易开发，并为Spring开发人员提供了引人注目的新UI选择。GWT是开源的，并且不受部署环境的影响，因此该集成将使所有Spring用户受益，无论他们是否打算将其应用程序定位到Google App Engine。凭借对领域模型的了解，Spring Roo是GWT的天作之合。通过今天的集成，Spring Roo可以生成和往返GWT接口，从而使开发人员在整个应用程序生命周期中生产率更高；并支持从Roo Shell轻松部署到App Engine。

我们还通过STS扩展安装机制添加了易于安装的Google Eclipse插件（GPE）。GPE支持包含GWT的工具。DevMode并部署到App Engine。

Spring Insight / Google Speed Tracer

Spring Insight是一项技术，可让开发人员深入了解其应用程序的性能，而无需编写任何特殊代码或设置任何配置。Insight使用高效的AspectJ工具及其对Spring Framework 的了解（以及因此了解应用程序结构的能力）来显示有关应用程序性能的详细信息，包括在何处花费时间。在SpringSource tc Server中可以使用Insight。 （tc Server Developer Edition是免费的，并且包含在STS中。

Google Speed Tracer是Chrome扩展程序，可显示您的应用程序在哪里花费时间。这包括由JavaScript解析和执行，布局，CSS渲染，DOM事件处理，资源加载等引起的问题。

今天，我们展示了两种技术之间的集成。这使Speed Tracer用户可以看到无缝插入其Speed Tracer屏幕的Insight跟踪，并向下钻取到特定的服务器端操作，例如JDBC查询。Speed Tracer和Insight的结合是前所未有的且功能强大。第一次，有可能看到浏览器和后端服务器进行协调，以从整体角度准确显示请求所做的工作。

这一切意味着什么

几个星期有什么不同！在发布VMforce和今天发布之前，Java开发人员还没有一个PaaS目的地，可以轻松地将其应用程序部署到该目的地。这是一个重要的鸿沟，有可能威胁到Java的长期发展。我很高兴VMware / SpringSource领导这项工作以填补这一空白。

今天的公告对于Spring开发人员来说是个好消息。春季技能从未比现在更有价值。春季生态系统的范围不断扩大；春天的核心价值观继续指导着它的发展。 Spring致力于发展Java社区，并将其领导力带入云计算。无论您是要在现有数据中心，私有Java云还是公共云中部署Java应用程序，Spring都是最佳的编程模型。我很兴奋，希望你也是如此。
