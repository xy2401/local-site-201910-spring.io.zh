Spring Roo 1.1.0。M1发布

工程
本·亚历克斯
2010年5月19日

我很高兴地宣布，我们刚刚发布了Spring Roo 1.1.0。M1。Spring Roo是Java开发人员以Java编程语言构建基于Spring的应用程序的最快方法。使用Roo 1.1.0。M1可以构建有效的Web应用程序-配有Google Web Toolkit （GWT）前端-只需200次按键！像往常一样，我们同时发布了新版本的SpringSource Tool Suite （STS 2.3.3）。M1），针对最新和最出色的Roo好东西进行了优化！

自三个月前的1.0.2版发布以来，已有200多个修复程序，增强功能和新功能 。一些亮点包括：

  • Spring MVC集成的重大改进：这些改进意味着Roo 1.0.2中的130行JSPX文件现在只有12行。更好的是，现在有完整的往返。Roo可以自动检测托管JSPX文件中所做的更改，并确保它保留您的更改。这是Web层往返过程中的一项非常重大的改进，但并不仅限于此！我们还增加了直接在JSPX中编写新标签，自动工作流程服务，更丰富的默认脚手架和更好的扩展点的功能。
  • Google Web Toolkit （GWT）支持：我一直是GWT的忠实拥护者，我很高兴我们现在开始在Roo中获得GWT全面支持的旅程。对于不习惯GWT的用户来说，这是一项令人印象深刻的技术，它使您可以编写Java代码，并将其自动编译为经过高度优化的JavaScript以供浏览器执行（而无需特殊的插件）。这是编写具有丰富外观，易于开发和维护的UI代码的快速Web应用程序的好方法。Java的一流开发经验的GWT理念也与Java的一流开发经验的Roo理念完全一致，因此很自然地添加了这种集成。新的“ gwt setup”命令将为您提供一个具有脚手架桌面和移动设备入口点的可运行Web应用程序，并在您修改实体时进行往返更改。它甚至以最新的GWT 2.1功能为基础，例如新的绑定小部件和远程支持。看看吧，玩得开心！
  • Google App Engine （GAE）支持：最近，我们一直在忙于在Spring平台生态系统中添加对越来越多的云平台的支持。在此新版的Roo中，我很高兴地分享我们现在对Google App Engine的支持。这样，您现在可以使用“持久性设置--provider DATANUCLEUS-数据库GOOGLE_APP_ENGINE” Roo命令，并自动接收与GAE兼容的完整配置。部署也非常容易，只需“ mvn gae：deploy”即可。加上STS 2.3.3。M1可从“扩展”选项卡中与Google Eclipse插件（GPE）完全集成，从而使您的Roo-STS-GAE-GWT体验极为流畅。
  • Apache Solr Search Server：一项令人兴奋的新功能是通过Solr对实体进行自动索引。这使您可以使用专用的搜索服务器为您的对象建立索引，并通过REST API（也支持JSONP）查询它们。
  • Maven Central依赖关系：我们很高兴响应社区的偏好，即Roo生产的项目使用Maven Central组ID，工件ID和版本号。
  • Java Persistence API（JPA）2支持： Roo 1.1.0将我们的默认依赖项更改为最新发布的Hibernate和OpenJPA版本。
  • 复合键支持： Roo 1.1.0的新增功能是定义复合键的功能。这是我们数据库逆向工程功能的先驱。
  • 可序列化的附件：一个可爱的新附件是可序列化的附件。这意味着Roo将自动为您的实体计算正确的serialVersionUID。
  • OSGi基础：我们已经更改了Roo基础结构，因此它完全基于OSGi。因此，当您在命令行（或通过SpringSource Tool Suite）启动“ roo”时，OSGi容器将启动并加载所有Roo捆绑软件。这意味着在不久的将来，我们将以一种更为简单的方式来支持复杂的附件。
  • Cygwin支持：我们已经确保Roo在Cygwin上运行，既是“ roo”独立客户端，又是“ roo-dev”开发脚本，适用于从源构建的人员。当然，Roo在Windows独立版本以及Mac和（我最喜欢的）Linux上仍然可以完美地运行。
  • 改进的控制台消息： Roo现在将在您的项目上工作时输出更多有意义的控制台消息。例如，您可以看到正在将哪些依赖项添加到pom.xml中。
  • 最新的Spring平台项目和公共依赖项：项目现在使用较新版本的Spring平台和公共依赖项，例如Log4J，ActiveMQ等。AspectJ1.6.9是一个特别值得注意的增强功能，它现在可以使用ITD成员编译甚至更好的.class文件。
  • 大量改进使附加组件开发更加容易：现在编写Roo附加组件更加容易。我们已经在流行的组件（例如MVC附加组件）中添加了新的挂钩，以实现自定义；新的抽象结构简化了与常见文件（如web.xml）的交互；更轻松的增量文件更改；以及许多其他小改进。

那么在过去三个月里，Roo发生了什么？有点：

  • 我们已经从Subversion转到了Git（不用担心，我们保留了所有版本历史记录，标签等）
  • 崭新的网站
  • 采用了新的Spring Roo徽标
  • 创建了@SpringRoo Twitter ID，人们可以关注@Roo新闻并找到Roo工程师的Twitter ID的便捷列表
  • 赠送大量免费袋鼠
  • 举办了一系列网络研讨会，有800多人参加
  • 许多会议和水罐中的特色Roo

Roo 1.1.0是一个里程碑版本。M1不适用于关键任务。在接下来的几个月中，我们将在1.1.0版本中添加许多其他新功能，包括：

  • 增量数据库逆向工程
  • 用户界面对新Solr和复合键功能的支持
  • Spring MVC UI的更多改进，例如更深入的JavaScript库集成
  • 迁移到基于OBR的模型以进行附加发现，解析，下载和安装
  • Roo本身的性能调优
  • 文献资料
  • 蚂蚁/常春藤（视时间而定）

这已经是一篇很长的文章，所以我会推迟撰写有关如何在以后的文章中使用其中一些新功能的文章（更新：我的同事Christian Dupuis和Jon Travis在博客中发表了有关使用STS进行新的GWT项目和Spring的文章。 Insight Google改进 ）。同时，我希望您喜欢下载Roo 1.1.0。M1并尝试一下。我们很乐意收到您的来信-只在您的推文中加入@SpringRoo或访问社区论坛 。玩得开心！
