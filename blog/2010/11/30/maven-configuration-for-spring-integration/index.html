用于Spring集成的Maven配置

发布
马克·费舍尔
2010年11月30日

依存关系

在您的POM部分中添加以下内容：


   org.springframework.integration
   Spring 整合芯
   2.2.3。发布

这将使“ spring-integration-core”模块可用于您的项目（“ core”包括Messaging API和Enterprise Integration Patterns支持）。如果要使用任何适配器或对XML，Groovy和/或Spring Security的支持，则可以添加以下一项或多项作为“ artifactId”值：

  • Spring Integration 活动
  • Spring 整合进给
  • Spring 集成文件
  • Spring 集成FTP
  • Spring 集成槽
  • Spring 集成-http
  • Spring 集成ip
  • Spring 集成jdbc
  • Spring 整合
  • Spring 集成jmx
  • Spring 整合邮件
  • Spring 整合
  • Spring 整合安全
  • Spring 集成-sftp
  • Spring 整合流
  • Spring 整合测试
  • Spring 整合推子
  • Spring 整合ws
  • Spring 集成XML
  • Spring 集成xmpp

注意：上面列出的任何模块都将“核心”模块作为传递依赖项引入，因此，如果您使用该列表中的至少一个模块，则无需将其包含在POM中。另外，某些依赖项（例如用于JSON映射的Jackson）不会作为传递性依赖项引入，因此您需要将它们添加到POM中（如果使用的是诸如gradle之类的其他构建系统，这也适用）。

储存库

这些工件可以通过Maven中央存储库获得 ，但是您也可以在POM的部分中添加SpringSource存储库：


   repository.springframework.maven.release
   Spring Framework Maven版本库
   http://maven.springframework.org/release

如果要使用“里程碑”或“快照”依赖项来代替或依赖于“发布”依赖项，则可以分别提供以下URL：


  • http://maven.springframework.org/milestone

  • http://maven.springframework.org/snapshot
