vFabric tc Server 2.1

Updates to vFabric tc Server continue to make it the best place to build and run Spring applications: 


  • Spring Insight

    Included with tc Server 2.1, Spring Insight delivers real-time visibility into application behavior and performance for Spring applications during development and pre-production.  Deep real-time visibility into application behavior can now be persisted to view information spanning a previous time range.  Improvements to administration and navigation make it easy to operate and surface useful information more intuitively.

  • Enhanced templating

    A new built-in template mechanism simplifies the configuration of tc Server with the vFabric GemFire HTTP Session Management and Hibernate Cache Modules.  This allows vFabric GemFire to automatically launch with tc Server 2.1 – no application code changes or other manual intervention is required.

vFabric Cloud应用程序平台更新

工程
亚当·菲茨杰拉德
2010年11月30日

我们最近发布了vFabric Cloud Application Platform几个组件的更新。正如罗德·约翰逊（Rod Johnson）在原始公告中提到的那样，vFabric旨在快速交付可即时扩展和云移植的下一代应用程序。这些最新更新进一步增强了运行时应用程序服务之间的集成，并改善了现代应用程序的性能，可伸缩性，监视和可管理性。

vFabric tc服务器2.1

vFabric tc Server的更新继续使其成为构建和运行Spring应用程序的最佳场所：


  • 春季见解

    Spring Insight包含在tc Server 2.1中，可在开发和预生产期间实时了解Spring应用程序的应用程序行为和性能。现在可以持久保留对应用程序行为的深入实时可见性，以查看跨越先前时间范围的信息。管理和导航方面的改进使其易于操作，并且更直观地显示有用的信息。

  • 增强模板

    新的内置模板机制通过vFabric GemFire HTTP会话管理和休眠缓存模块简化了tc Server的配置。这使vFabric GemFire可以随tc Server 2.1自动启动-无需更改应用程序代码或进行其他手动干预。

vFabric GemFire 6.5

vFabric GemFire是用于现代应用程序的分布式数据管理平台，可提供动态可伸缩性，超高性能和类似数据库的持久性。两个新模块简化了GemFire与tc Server和Hibernate的集成，为现代应用程序提供了复杂的数据管理。Jags Ramnarayan在他的最新博客文章中对细节进行了详细介绍，但我们将在此处提供快速概述。


  • vFabric GemFire HTTP会话管理模块

    vFabric GemFire HTTP会话管理模块可以分载和管理tc Server或Tomcat服务器的HTTP会话状态。预配置并通过tc Server自动启动的GemFire HTTP会话管理可提供高性能和可伸缩性。这对于处理许多请求且需要扩展以满足需求的Web应用程序特别有用。


    • 将会话管理与tc Server或Tomcat Servlet / JSP容器分离

    • 独立扩展应用程序服务器和HTTP会话处理

    • 处理非常大的会话而不会丢失会话

    • 易于通过新的模板机制连接到tc Server或通过较小的配置修改即可连接到Tomcat



  • vFabric GemFire休眠缓存模块

    vFabric GemFire Hibernate缓存模块为Hibernate提供了快速，可扩展的分布式L2缓存。Swapnil Bawaskar在他的最新博客文章中介绍了详细信息。

    • 显着提高Hibernate性能

    • 减少到数据库服务器的网络流量（和潜在的瓶颈）

    • 获得GemFire的所有企业级功能

    • 可扩展性–在整个集群中分区数据

    • 开发人员生产力和云规模部署



vFabric Hyperic 4.5

vFabric Hyperic是vFabric Cloud应用程序平台的应用程序管理和监视组件，它提供了对整个虚拟化应用程序堆栈的完整，连续的可见性。詹妮弗·希基（Jennifer Hickey）最近描述了Hyperic 4.5版本的内部细节，但还添加了两个重要的新监视功能。


  • GemFire监控

    Hyperic 4.5引入了一个新插件，使应用程序运营团队能够确保GemFire可用和高性能，并为他们提供快速修复故障并报告SLA所需的信息，从而使他们能够更好地管理GemFire的性能。


    • 监视JMX代理，高速缓存服务器，应用程序对等方和网关集线器

    • 总部GemFire视图：GemFire指标的实时视图。

    • 监控GemFire事件–加入/离开GemFire系统，崩溃，日志条目等



  • RabbitMQ监控

    Hyperic 4.5引入了一个新插件，使应用程序运营团队能够确保RabbitMQ可用并具有高性能，并为他们提供快速修复故障并报告SLA所需的信息，从而使他们能够更好地管理RabbitMQ的性能。

    • 监视RabbitMQ AMQP代理，交换，队列和虚拟主机以及日志

    • 观察RabbitMQ的可用性，错误，CPU，线程，连接和队列。



在通往云的道路上迈出一步

vFabric云应用程序平台通过现代编程模型和下一代平台服务的支持，提供了通往云计算架构的清晰创新路径。通过这些新更新，现在是试用vFabric tc Server ， GemFire和Hyperic的绝佳时机。
