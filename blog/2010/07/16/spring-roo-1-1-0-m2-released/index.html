Spring Roo 1.1.0。M2发布

工程
本·亚历克斯
2010年7月16日

我很高兴地宣布，我们刚刚发布了Spring Roo 1.1.0。M2（ 在此处下载 ）。我们还同时发布了SpringSource Tool Suite 2.3.3。M2，提供了与此新Spring Roo版本的最新集成。对于生产用途，我们建议您继续使用Spring Roo 1.0.2，尽管我们知道很多人也很高兴使用Roo 1.1.0开发版本。

什么是新的？

那么，新的Spring Roo 1.1.0 Milestone 2中包含什么？好吧，自里程碑1以来有140项修复，改进和增强功能 。一些亮点包括：

  • 自里程碑1以来，性能提高了三倍
  • 升级到Spring Framework 3.0.3和GWT 2.1.0。M2 ， AspectJ 1.6.10。M1 ， OpenJPA 2.0.0 ， Log4J 1.2.16等
  • 更好的GWT支持，例如取消了GWT 2.1“ bikeshed ” JAR
  • 大大改进了Windows启动脚本
  • 改进的外壳可用性，例如不区分大小写的选项匹配
  • 一个新的“ web mvc view ”命令可以简化MVC视图的创建
  • 通过“ web mvc安装语言 ”命令支持用户定义的Web语言环境
  • 额外的按需数据功能
  • Google App Engine 集成测试支持
  • 帮助您轻松创建简单，高级和国际化附加组件的命令
  • 大量的附加基础架构改进（请参见下文）

更多附加发现和管理功能

Spring Roo 1.1.0。M2还具有大量增强功能，以准备最重要的附加发现和管理功能。Roo团队致力于帮助Roo建立一个充满活力的附加生态系统，这将使许多希望Roo核心发行版中未包含功能的用户受益。

作为这项工作的一部分，Roo现在会从我们的新存储库中自动下载OBR索引， http://spring-roo-repository.springsource.org/repository.xml 。作为Roo用户，您不太可能会看到很多OBR基础结构（因为它确实起作用！），但是很高兴知道OBR为Roo提供了OSGi标准化的方法来发现可用的加载项，解决其依赖项，下载它们到您的Roo环境中，并自动启动它们。这意味着Roo用户可以从OSGi的行业实力，标准化的插件管理方法中受益，而且随着Roo社区可以使用越来越复杂的插件，我们也不会遇到障碍。

Roo还通过将OBR元数据直接集成到Shell命令解析中来扩展了OBR功能。这意味着，如果您键入的命令是当前已卸载附件的一部分，那么Roo会自动提示您哪些附件可以提供该命令！我很高兴地承认这一特殊功能是我使用Ubuntu启发的，因为仅键入任意随机命令并让系统通知您键入该命令以安装该命令非常有用。

虽然这种新的OBR支持和shell命令发现当然非常好，但我们也想确保我们可以为Roo社区提供方便的附加主机。但是正确执行此操作意味着我们需要仔细考虑安全隐患。

我们通过在Roo 1.1.0中添加广泛的PGP密钥管理功能来解决附加安全性。M2。这些可以让您确切地指出您信任哪些开发人员来签署Roo将下载到您计算机上的软件。现在，Roo本身也在每个发行版中均已签署PGP。为了支持这些功能，我们在Roo中引入了一个名为httppgp：//的新协议处理程序。这告诉Roo给定的HTTP URL也具有可用的PGP装甲分离签名。

通过要求所有插件都具有PGP签名，我们可以方便，安全地为社区托管所有Roo插件。由您决定是否信任给定的PGP密钥，如果不信任该密钥，Roo将拒绝甚至花费时间下载httppgp：//资源。我们还实现了该架构，因此我们无需更改Apache Felix的OBR基础架构，也无需强迫每个人使用我们的中央附加托管功能。这意味着我们将受益于Felix较新版本中对OBR的未来改进，此外，您也可以在自己的OBR repository.xml文件中使用httppgp：// URL。Roo的方法还意味着您还可以使用独立的PGP工具（例如GnuPG）执行与签名相关的操作，以独立验证Roo的正确操作。如您所知，我对新的PGP OBR基础结构感到非常高兴（但我想我那令人讨厌的Acegi Security / Spring Security过去正在闪闪发光！）。

当我们宣布里程碑3时，我将更多地讨论这种新基础架构的工作方式，因为它将结合完整的附加最终用户界面。另外，我们还将引入新的后端工具，该工具可帮助自动将附加出版物发布到中央存储库（称为RooBot）。如果您不能等待新的最终用户界面，请尝试使用“ pgp list trust keys”命令，以查看Roo默认情况下信任哪些密钥。也可以尝试在Roo提示符下键入“ welcome property”，以查看Roo如何建议当前提供“ welcome property”命令的加载项。如果您确实希望看到httppgp：//基础结构正在运行，请尝试不信任我的PGP密钥（pgp untrust –keyId 00B5050F ），然后尝试安装欢迎加载项（osgi obr start –bundleSymbolicName org.springframework.samples.roo。欢迎）。

1.1.0路线图

尽管Roo 1.1.0中具有所有巧妙的新功能。M2，我们也一直在忙于准备其他尚未加入Milestone 2版本的功能。

增量数据库逆向工程是许多人迫切等待的功能。现在，此功能请求获得了113票赞成，并且该实现已经支持Java编译单元生命周期管理，增量字段维护，复合主键和大多数关系类型（1：M，M：M，1：1）。我们预计该功能将包含在Roo 1.1.0中。M3，但如果您希望生活在最前沿，则可以通过从Git 构建Roo进行试验。

我们一直在研究的其他有趣功能包括Op4J插件 ，非常好的Git自动插件和一些Web嵌入命令。Op4J附加组件使您可以显着减少所需编写的代码量，而不会失去Java的类型安全性，代码辅助和性能优势。自动的Git插件将您执行的每个Roo命令转换为一个单独的Git提交，这对于查看Roo的工作并能够选择性地回滚等非常有用。Web嵌入命令非常好，它允许您合并YouTube之类的资源仅需一个Roo命令即可将视频，SlideShare演示文稿，Google Maps，Flikr照片和类似媒体添加到您的Web视图中（目前支持15种不同的资源类型，涵盖了从照片共享到财务更新以及介于两者之间的所有内容）。

如果您对我们的发布日期感到好奇，可以访问以下网址进行跟踪 http://tinyurl.com/rooroadmap 。当然，这些日期会随着我们的开发进度而修改（相关项目也会同样地修改其日期），但是它们为您提供了很好的指示。
