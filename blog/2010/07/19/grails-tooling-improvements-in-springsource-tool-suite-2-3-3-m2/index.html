SpringSource Tool Suite 2.3.3 M2中的Grails工具改进

工程
安迪·克莱门特
2010年7月19日

最近发布的STS 2.3.3 M2对Grails对Eclipse的支持引入了一系列增强功能。在本文中，我将讨论如果您尝试一下将发现什么（在这里获取： SpringSource Tool Suite ）。

Groovy-Eclipse


Grails工具实际上是在Groovy-Eclipse支持的基础上扩展的，该支持由SpringSource的同一团队开发。今年早些时候，Groovy-Eclipse工具在Eclipse-Con 2010上获得了“最佳开源开发人员生产力工具”奖。您可以在此处阅读有关最新Groovy-Eclipse 2.0.2版本中最新增强功能的更多信息： New和Noteworthy 。我在这里不打算重点介绍Groovy-Eclipse，但是该版本的一些重点是：

  • 重构支持：现在支持提取方法，提取常量，提取局部变量
  • 代码格式和缩进方面的改进

现在，到Grails！

获得新的视角


现在有了新的Grails透视图，可以更好地组织我们一直添加到STS中的视图和小部件。您可以以常规方式打开窗口 （ Window> OpenPerspective> Grails ）。在新的角度来看，首先要注意的是Eclipse项目浏览器是打开的，而不是软件包浏览器。项目浏览器正在使用自定义内容提供程序，该提供程序显示了grails开发人员应该更熟悉的grails项目视图：



对于相似实体组：域对象，控制器，视图等，有新的标签和图标。但是最大的变化是新的plugins文件夹。以前，在使用grails项目时，插件依赖项很难在UI中看到，它们被隐藏在classpath容器中，并通过一些Eclipse链接的源文件夹条目隐藏。现在，在项目浏览器中，应用程序依赖项变得更加明显。

另外，与插件有关，我们还有一个新的Grails插件项目向导。这与Grails项目向导非常相似，但是它不会运行create-app ，而是会运行create-plugin命令。有了这个新向导和对本地（inplace）插件的新支持，遵循面向插件的体系结构开发应用程序要容易得多。

这是使用本地/就地插件配置的示例：

在该快照中，您可以看到两个Eclipse项目。主grails应用程序“ MyFirstGrailsApp”和插件项目“ MyFirstGrailsPlugin”。通过在主应用程序中编辑BuildConfig.groovy，已经在插件项目上表达了依赖性：

grails.plugin.location.MyFirstGrailsPlugin="../MyFirstGrailsPlugin"

在指定该依赖关系并针对该应用程序运行“刷新依赖关系”之后，“ plugins”文件夹已更新（请参见上面的屏幕截图），其中包含新的依赖关系条目以及一个不同类型的图标，表明我的依赖关系不同插件依赖性。

指挥中


为了使用户更加高效，命令提示符已得到增强。现在，它可以选择项目，并且可以固定到位（因此不会意外关闭），但是最重要的是，它现在具有历史记录功能。历史记录使您能够快速重新运行相同的命令或运行类似的命令。要访问提示，可以使用相同的“ claw”按键（在Windows机器上为Alt + Shift + Ctrl + G ），或者如果手没有弯曲，则可以使用grails透视图工具栏上的新grails图标启动它。grails图标旁边的下拉菜单提供了另一种访问命令历史记录的途径，该命令显示了命令及其运行所针对的项目：

在下拉历史记录中选择任何条目将打开带有该命令预填充的提示，可以在执行前对其进行编辑或简单地重新运行。如果提示是打开的，则访问历史记录的另一种方法就是按向上箭头：

为了加快应用程序的浏览速度，STS现在为grails项目提供了许多新的键盘快捷键。在控制器，服务，taglib或域类中时，可以使用Alt + G，然后使用D，C，S，T中的一个跳转到编辑器中的相关域类/控制器/服务/ taglib。单击工具栏上的新图标也可以跳转：

管理插件


为了使组织使用的插件更加容易，我们提供了一个新的grails插件管理器。无需记住您想要的插件的名称，您可以打开管理器（通过上下文菜单RightClick> Grails Tools> Grails Plugin Manager… ，或通过按Alt + G，M ）并浏览可用的插件：

插件管理器使您能够：


  • 安装新的插件

  • 更新现有插件

  • 卸载插件


它还提供了详细的描述和任何可用的文档链接。

可用插件列表已被缓存，但是您可以按Refresh以确保您使用的是最新列表。默认情况下，管理器会向您显示所有可用的插件，但是当您向下滚动时，将看到您已经安装的插件（已安装的插件带有绿色的勾号）。重要的是，如果已安装的任何插件均已发布更新，则UI会向您指示更新，并且您可以根据需要进行更新（蓝色向上箭头图标表示有可用的更新）。由于安装/卸载插件的操作不是即时的（grails实际需要花费一些时间），因此当您与UI交互并将插件标记为install / update / etc时，基本上就是在安排一系列grails命令运行。您所请求的更改将不会真正执行，除非您按OK（确定）按钮以关闭管理器。如果您在任何时候认为自己搞砸了，可以取消管理器（或重置管理器），这样您的项目将处于原始状态。
