视频：Spring Roo-不仅仅是另一个RAD工具！

新闻与活动
克洛伊·杰克逊（Chloe Jackson）
2012年8月16日

你们中的大多数人可能都知道Spring Roo是基于Java的快速应用程序开发工具，该工具使用AspectJ和命令行外壳程序来构建Spring应用程序。但是，在那些设置应用程序组件的巧妙命令之下，隐藏着一个更加强大且可扩展的平台，您可以使用该平台来构建所需的任何东西。

在此演示文稿中，SpringSource的Josh Long和Spring Roo in Action作者Ken Rimple和Srini Penchikala介绍了Spring Roo 1.2，然后进一步介绍了Roo强大的基于插件的弱点。他们介绍了Roo的OSGi捆绑软件支持，并介绍了如何使用附加组件来生成代码，安装模板，响应注释的添加/删除，以及公开源代码库和内部公司库供开发人员使用。最后，他们通过讨论最近开发的Tailor插件进行总结，并将展示如何使用它完全定制开发人员的Shell环境。

  3.  The ZeroTurnaround blog has a  really cool little blog on rapid development with Spring and Hibernate. 
     Of course, Zero Turnaround has a handy little software agent that lets you reload Java classes on the fly. So that's a huge gain in productivity right there. That, coupled with XML-free Spring 3.1 and Hibernate 4.1, and you have yourself a really awesome combination. To learn more, check out the blog!  The example he illustrates are also well represented in this sample project on GitHub. The example even  includes web.xml, even though it isn't required, just as was done in that blog post, specifically because it's more reliable on the buggy versions of JBoss 7 and Servlet 3 environments aren't ubiquitous, yet. 
        This is a seriously cool blog post, be sure to read the followups!

  5. 之前我们已经讨论过这一点，但是InfoQ的报道也很棒。

  13. 调试队长在使用Spring 3.1的配置文件以XML定义特定于环境的bean方面有一篇很棒的文章。

  14. Halycon代码博客上有一篇关于使用Spring的子上下文的文章，可让您在import语句中利用Spring的property-placeholder解析。通常情况下，这些语句都无法看到当前的应用程序上下文的属性占位符，但嵌套上下文可以看到父上下文的属性占位符。解？只需实例化子上下文中的所有内容，以便import语句可以使用父上下文的属性 ！这个把戏有点偷偷摸摸！我喜欢。

    记住，子上下文提供了作用域-一个孩子中定义的bean看不到另一个孩子中定义的bean。这对于诸如Spring Batch Admin之类的事情非常有用，它使您可以即时上传和启动新的Spring Batch作业（它们本身基于Spring配置）。


  15. Java Assist博客上有一篇不错的文章，介绍了在3.0或更高版本中使用Spring MVC bean验证 。凉！

Spring Data REST 1.0.0。RC2发布

发布
乔恩·布里斯宾
2012年7月31日

我很高兴地宣布发布Spring Data REST 1.0.0。RC2！除了许多错误修复之外，此版本还增加了对JSONPE（具有错误处理功能的JSONP）的支持，具有使用@RestResourceannotation关闭CRUD方法的功能，并且现已针对Servlet 3.0 API进行了构建和测试（尽管尚未完成） 3.0专用，因此在Servlet 2.5容器中仍然可以正常工作）。

新功能包括：

  • JSONPE-只需添加一个URL参数，即可将结果包装在对您指定的Javascript函数的调用中。还通过将错误转换为HTTP 200并将原始状态代码作为错误处理程序的第一个参数传递来处理服务器错误。
  • 关闭CRUD方法-导出器现在在CRUD方法上使用@RestResourceannotation。只需重写要关闭的CrudRepository的方法，然后使用@RestResource(exported = false)对其进行注释。
  • 与现有Spring MVC应用程序的更好集成-简化了内部Spring MVC配置，使其与现有Spring MVC应用程序的集成更加容易。只要将RepositoryRestMvcConfiguration bean包含到您自己的配置中就可以了。
